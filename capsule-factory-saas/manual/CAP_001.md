# capsule-factory-saas - CAP_001

## Overview

This is a production-grade, multi-tenant SaaS platform engineered for capsule manufacturers who need real-time traceability, batch-level quality control, and scalable production orchestration. Built for DevOps-minded factory operators and pharma-grade compliance teams, the system eliminates spreadsheet hell and legacy ERP friction.

## What is Capsule Factory SaaS?

Capsule Factory SaaS is a cloud-native operations hub that digitizes the entire capsule production lifecycle—from raw material blending to encapsulation, drying, sorting, and packaging. It enforces ISO 9001 / GMP workflows, integrates with PLCs and SCADA systems via MQTT or OPC UA, and exposes a fully versioned REST + GraphQL API for custom factory dashboards.

## Hardcore Features

### 1. Real-Time Batch Genealogy with Immutable Audit Logs
Every capsule batch is tracked through a directed acyclic graph (DAG) of production events—material lot, machine ID, operator, temperature/humidity telemetry, and timestamp. All mutations are hashed and stored in an append-only ledger (backed by PostgreSQL + event sourcing). Non-repudiation is baked in: any tamper attempt breaks the hash chain, triggering an instant alert to compliance.

### 2. Dynamic Encapsulation Recipe Engine with Multi-Variable Optimization
Define recipes as parameterized YAML/JSON manifests. The engine uses a constraint solver (via OptaPlanner or OR-Tools) to adjust fill weight, shell thickness, and drying time in real time based on sensor feedback. Supports A/B testing of recipe variants per production line, with automated rollback if yield drops below a configurable threshold.

### 3. Zero-Downtime Multi-Tenant Deployment with Tenant-Aware CI/CD
Each tenant (factory) gets an isolated Kubernetes namespace with dedicated PostgreSQL schema and object storage bucket. Deployments are orchestrated via ArgoCD with tenant-specific canary analysis. Rollouts are gated by synthetic health checks that validate capsule quality metrics before traffic is shifted. No shared state, no noisy neighbors.

## How to Use

### 1. Onboard a New Factory Tenant via API
```bash
curl -X POST https://api.capsule-factory.io/v1/tenants \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "tenant_id": "factory_berlin_01",
    "plan": "enterprise",
    "region": "eu-central-1",
    "compliance_profile": "GMP_2024"
  }'
```
The system auto-provisions a dedicated namespace, applies network policies, seeds baseline recipe templates, and returns a tenant-specific API key. Monitor provisioning status via `GET /v1/tenants/factory_berlin_01/status`.

### 2. Define and Deploy a Capsule Recipe
Create a recipe file `recipe_hard_gelatin_v3.yaml`:
```yaml
apiVersion: capsule-factory.io/v1
kind: Recipe
metadata:
  name: hard-gelatin-v3
spec:
  shell:
    material: gelatin_250_bloom
    plasticizer: glycerin_12%
    thickness_mm: 0.8
  fill:
    active_ingredient: paracetamol_500mg
    excipient: microcrystalline_cellulose
    target_weight_mg: 600
  drying:
    temp_celsius: 35
    humidity_rh: 20
    duration_min: 45
  constraints:
    max_yield_loss_percent: 2.0
    min_hardness_n: 12
```
Push it to the platform:
```bash
capsule-cli apply -f recipe_hard_gelatin_v3.yaml --tenant factory_berlin_01
```
The recipe is validated against your tenant’s material inventory and machine capabilities. On success, it becomes available for production scheduling.

### 3. Trigger a Production Run with Real-Time Monitoring
```bash
capsule-cli run --recipe hard-gelatin-v3 --batch-size 100000 --line L3 --tenant factory_berlin_01
```
The platform spawns a streaming pipeline that ingests PLC telemetry every 500ms. Use the WebSocket endpoint `wss://api.capsule-factory.io/v1/stream?tenant=factory_berlin_01&batch_id=20241015_001` to subscribe to live metrics: fill weight deviation, shell integrity index, and predicted yield. If any metric exceeds thresholds, an automated pause command is sent to the line controller.

## Best Practices

### 1. Implement Tenant-Level Rate Limiting and Cost Attribution
Use a sidecar proxy (Envoy or Linkerd) with per-tenant rate limits to prevent noisy tenants from starving others. Tag all cloud resource usage (CPU, storage, API calls) with tenant labels. Export these to a billing system (e.g., Stripe or custom invoicing) so each factory pays proportionally. This avoids cross-subsidization and keeps unit economics transparent.

### 2. Automate Compliance Reporting with Pre-Built Templates
Regulatory audits (FDA, EMA) demand batch records within 48 hours. Configure the platform to auto-generate PDF/CSV batch reports at the end of each run, signed with a tenant-specific GPG key. Store reports in a separate immutable S3 bucket with versioning enabled. Use AWS Lambda or a cron job to email reports to your QA team and regulatory archive. This turns a manual 4-hour task into a 2-minute verification.
