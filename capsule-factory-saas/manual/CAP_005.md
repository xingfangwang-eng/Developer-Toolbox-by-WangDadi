# capsule-factory-saas - CAP_005

## Overview

This is a production-grade, multi-tenant SaaS platform engineered for managing capsule manufacturing workflows — from gelatin blending and encapsulation to polishing, sorting, and lot traceability. Built for pharma-grade compliance, the system integrates real-time SCADA feeds, batch genealogy, and yield analytics out of the box.

## What is capsule-factory-saas?

capsule-factory-saas is a domain-specific operations platform that replaces fragmented spreadsheets, legacy MRP modules, and paper-based GMP logs. It provides a unified command layer for capsule producers — handling recipe versioning, equipment OEE tracking, defect root cause mapping, and regulatory audit trails (FDA 21 CFR Part 11, EU GMP Annex 11). The stack is designed for high-volume, high-variance environments where downtime and cross-contamination risk must be minimized programmatically.

## Core Features

### 1. Real-Time Batch Genealogy Engine
Every capsule batch is tracked from raw gelatin powder to finished polished capsules. The engine automatically builds a directed acyclic graph (DAG) of batch dependencies — linking raw material lots, encapsulation runs, drying cycles, and quality test results. Any deviation in moisture content or fill weight triggers an instant trace-back to the exact source lot and machine parameters.

### 2. Multi-Stage Yield & OEE Dashboard
Live OEE (Overall Equipment Effectiveness) is calculated per encapsulation line, factoring in planned downtime, speed loss, and quality rejects. The dashboard overlays yield curves against target PPM (parts per million) defect thresholds, with drill-downs into specific rejection codes — split capsules, dented caps, misaligned locks, or weight outliers. Historical trends feed into predictive maintenance scheduling.

### 3. GMP-Compliant Audit Trail with Digital Signatures
Every state change — recipe approval, machine parameter override, QC hold, or batch release — is logged with a tamper-evident hash chain and mandatory e-signature (FIPS 140-2 validated). The system auto-generates batch manufacturing records (BMRs) and deviation reports in PDF/A format, ready for regulatory submission without manual transcription.

## How to Use

### 1. Configure a New Capsule Recipe with Multi-Variant Parameters
Navigate to the Recipe Manager and define a new master formula. Specify gelatin bloom strength, plasticizer ratio (glycerin/sorbitol), colorant concentration, and drying profile (temperature ramp, RH setpoints). For each variant — e.g., size 00 vs. size 1 — set fill weight targets and pin gauge tolerances. The system validates against your existing BOM and flags any incompatible raw material lots (e.g., expired gelatin, inconsistent viscosity).

### 2. Trigger a Batch Run with Real-Time Machine Binding
From the Production Planner, select the recipe variant and target encapsulation machine. The platform pushes the recipe parameters directly to the PLC via OPC UA — no manual entry. As the run starts, the SCADA bridge streams live sensor data (pin temperature, pin withdrawal speed, drying tunnel airflow) into the batch record. You can set conditional auto-hold rules: e.g., if the moisture content drifts >0.5% for more than 10 seconds, the line pauses automatically and a QC alert fires.

### 3. Run a Root Cause Analysis from Defect Data
Open the Defect Analytics module and filter by date range, machine ID, or capsule size. The system groups defects by Pareto categories — split caps, pinholes, stuck pins, incomplete fills. Click any category to see the correlated process parameters (e.g., high withdrawal speed + low drying temp). The engine suggests probable root causes using a Bayesian inference model trained on historical batch data. Export the analysis as a CAPA (Corrective and Preventive Action) report directly into your QMS.

## Best Practices

### 1. Enforce Strict Recipe Versioning Before Any Production Run
Never reuse a recipe ID across formulation changes. Each revision (e.g., v1.2 → v1.3) should trigger a mandatory re-validation step in the platform. This prevents silent drift in capsule hardness or dissolution profiles, especially when switching gelatin suppliers or adjusting plasticizer ratios. Use the built-in diff viewer to compare parameter changes side-by-side before approval.

### 2. Automate Lot Segregation with Dynamic Material Reservation
Configure the system to reserve raw material lots based on their remaining shelf life and current QC status. For example, gelatin lots with <90 days until expiry should be flagged and reserved for low-volume or R&D runs only. This reduces the risk of using near-expiry material in high-volume commercial batches, which could lead to capsule brittleness or microbial growth during storage.
