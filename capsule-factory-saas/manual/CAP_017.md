# capsule-factory-saas - CAP_017

## Overview

This is a production-grade, multi-tenant SaaS platform purpose-built for capsule manufacturers and contract packagers. It streamlines batch tracking, gelatin inventory management, and compliance reporting across ISO 9001 and GMP environments. The system is optimized for high-throughput facilities operating 24/7 production lines.

## What is capsule-factory-saas?

`capsule-factory-saas` is a domain-driven, API-first platform that replaces spreadsheet-based workflows and legacy ERP modules. It provides real-time visibility into capsule shell production, from raw gelatin hydration to finished goods palletization. The platform supports both hard gelatin and vegetarian capsule lines, with native integration for HMI/SCADA systems.

## Core Features

### 1. Real-Time Batch Genealogy with Immutable Audit Trail
Every capsule batch is tagged with a unique `CAP-UID` that links gelatin lot numbers, machine parameters (temperature, humidity, pin speed), and operator IDs. The audit trail is write-once, stored as JSON blobs in a PostgreSQL column with `TIMESTAMP` and `USER_ID` — no accidental edits. This satisfies FDA 21 CFR Part 11 requirements out of the box.

### 2. Dynamic Yield Optimization Engine
The system ingests real-time sensor data from encapsulation machines and runs a lightweight linear regression model to predict yield deviations. When a parameter drifts (e.g., pin temperature exceeds ±1.5°C), the engine auto-adjusts the machine setpoints via OPC-UA or triggers an alert to the shift lead. This reduces scrap by up to 12% in field tests.

### 3. Multi-Tenant Inventory Locking with FIFO Enforcement
Each tenant (plant) gets an isolated inventory schema. Gelatin drums are tracked by receipt date and moisture content. The system enforces strict FIFO (First-In, First-Out) during production planning — no manual overrides allowed unless escalated via a 2FA approval flow. This prevents stale gelatin from being used in high-spec batches.

## How to Use

### Step 1: Configure Your Production Line Profile
Navigate to `Settings > Lines` and create a new line profile. Input the machine type (e.g., `Capsugel CFS 1200`), target speed (capsules/min), and the OPC-UA endpoint. The system will auto-discover tags (e.g., `ns=2;s=Temperature.Zone1`). Validate connectivity by clicking "Test Link" — a green checkmark confirms the handshake. If it fails, check firewall rules for port 4840.

### Step 2: Initiate a Batch with Raw Material Traceability
Go to `Production > New Batch`. Scan the QR code on the gelatin drum to auto-populate lot number, supplier, and moisture %. Set the batch size (e.g., 500,000 capsules) and target fill weight (e.g., 500 mg). The system will calculate the theoretical gelatin requirement and lock the necessary inventory. Click "Start Batch" — the line profile is now active, and data logging begins.

### Step 3: Monitor and Close the Batch with a Compliance Report
During production, the dashboard shows live metrics: yield %, reject rate, and temperature trends. When the batch completes, hit "Finalize Batch". The system generates a PDF report containing the CAP-UID, all sensor readings (timestamped), operator signatures, and QC test results (e.g., disintegration time). This report is stored in S3 and indexed for search. Archive it by clicking "Export to Vault".

## Best Practices

### 1. Implement a Pre-Shift Calibration Routine
Before the first batch of the day, run a manual calibration check on all temperature sensors. Use the platform's "Line Health" module to log the readings. If any sensor deviates by more than 0.5°C from the reference standard, flag it for immediate recalibration. This prevents drift-induced yield loss and ensures data integrity for audits.

### 2. Use the API for Automated Batch Reconciliation
Integrate `capsule-factory-saas` with your ERP (e.g., SAP, Oracle) via the REST API. After each batch is finalized, push the `CAP-UID`, yield %, and inventory consumption to your ERP's production order. This eliminates manual double-entry and reduces reconciliation errors by 90%. Set up a webhook on the `batch.finalized` event to trigger the push automatically.
