## What is capsule-factory-saas?

capsule-factory-saas is a cloud-native orchestration platform designed to bridge the gap between IoT sensor data, ERP systems, and capsule production workflows. It eliminates manual tracking, reduces waste, and enforces batch-level traceability for pharmaceutical, nutraceutical, and cosmetic capsule manufacturers. Think of it as a real-time control plane for your capsule factory floor.

## Core Features

1. **Real-Time Batch Traceability with Immutable Audit Logs**  
   Every capsule batch is tracked from raw material intake to final packaging. Using blockchain-backed hashing, the system generates an immutable audit trail that satisfies FDA 21 CFR Part 11 and EU GMP Annex 11 compliance. No more spreadsheets or fragmented logs.

2. **Predictive Yield Optimization via ML-Driven Anomaly Detection**  
   The platform ingests sensor data (temperature, humidity, pressure, fill weight) from your production lines and trains a lightweight ML model to predict capsule defects before they occur. Alerts are pushed to Slack, PagerDuty, or your custom webhook within milliseconds — reducing scrap rates by up to 18%.

3. **Multi-Tenant Isolation with Granular RBAC and API Rate Limiting**  
   Each factory tenant gets a fully isolated environment with custom role-based access control (RBAC) down to the machine level. The RESTful API enforces per-endpoint rate limiting and supports OAuth 2.0 + JWT for secure integrations with your existing MES or SCADA systems.

## How to Use

### 1. Onboard a Production Line via the CLI

```bash
capsule-factory-saas init --line-id "LINE-07" --sensor-endpoint "mqtt://192.168.1.50:1883" --output-format json
```

This command registers a new production line, binds it to a local MQTT broker, and validates the sensor schema. The CLI auto-generates a tenant-specific API key and stores it in `~/.capsule-factory/config.yaml`. You can also specify a custom `--webhook-url` for real-time event streaming.

### 2. Configure Batch Rules and Quality Gates

Navigate to the web dashboard or use the `batch` subcommand:

```bash
capsule-factory-saas batch create --batch-id "BATCH-20231004-001" --recipe "gelatin_softgel_v2" --quality-gate "weight_tolerance=±2%" --auto-hold-on-failure
```

This defines a batch with a specific recipe and a quality gate that automatically pauses production if fill weight deviates beyond ±2%. The system logs every sensor reading and triggers a hold event if the threshold is breached.

### 3. Monitor and Export Production Analytics

```bash
capsule-factory-saas monitor --line-id "LINE-07" --metric "defect_rate" --interval "1h" --export-csv "./reports/line07_defect_rate_weekly.csv"
```

This streams live defect rate data for the last hour and exports a CSV file suitable for Tableau or Power BI. You can also pipe the output to `jq` for custom JSON parsing. For real-time dashboards, use the built-in Grafana data source plugin.

## Best Practices

- **Implement a Two-Phase Rollout for Multi-Tenant Deployments**  
  Start with a single factory tenant to validate sensor calibration and batch rules. Once the model achieves <2% false positive rate for anomaly detection, roll out to additional tenants. This reduces operational risk and ensures your ML pipeline is production-grade before scaling.

- **Use Environment-Specific Configurations with GitOps**  
  Store all `capsule-factory-saas` configuration files (YAML/JSON) in a dedicated Git repository. Use branches for staging, production, and disaster recovery. This enables auditability, rollback, and automated CI/CD pipelines for infrastructure changes. Never hardcode API keys — rely on environment variables or a secrets manager like HashiCorp Vault.
