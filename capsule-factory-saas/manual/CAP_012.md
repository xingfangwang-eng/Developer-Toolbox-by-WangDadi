# capsule-factory-saas - CAP_012

## Overview

**capsule-factory-saas** is a cloud-native platform engineered for end-to-end capsule production lifecycle management. It bridges the gap between pharmaceutical-grade manufacturing requirements and real-time operational intelligence. Built for production engineers, QA managers, and supply chain operators who demand zero-downtime traceability and data-driven batch optimization.

## Core Features

### 1. Real-Time Batch Genealogy Engine
Every capsule batch is automatically linked to its raw material lot, encapsulation machine parameters, environmental sensor logs, and QC test results. The engine constructs a **full lineage graph** that can be queried via GraphQL or exported as a GDPR-compliant audit trail. No manual reconciliation, no data silos.

### 2. Predictive Encapsulation Calibration (PEC)
Leveraging on-device ML models trained on historical fill-weight variance, temperature drift, and humidity data, PEC dynamically adjusts machine settings before a batch run starts. It reduces scrap rate by up to 18% and eliminates the need for manual pre-run calibration cycles.

### 3. Multi-Tenant Inventory Locking with Real-Time ATP
Each tenant (factory) manages its own capsule SKU catalog with atomic inventory reservations. The system exposes a RESTful API for **Available-to-Promise (ATP)** queries that consider current WIP, scheduled maintenance, and batch hold status. No double-booking, no stockouts.

## How to Use

### 1. Onboard a Production Line via CLI
Use the `capsule-factory-saas-cli` to register a new encapsulation line:
```bash
capsule-factory-saas-cli line register \
  --name "Line-7" \
  --location "Building B, Floor 2" \
  --machine-type "KAPS-5000" \
  --api-key $YOUR_TENANT_API_KEY
```
The system validates the machine type against the supported model list, creates a unique line ID, and instantly activates telemetry ingestion.

### 2. Create a Batch with Dynamic Recipe Binding
Send a POST request to the `/api/v1/batches` endpoint:
```json
{
  "line_id": "line-7-uuid",
  "recipe_id": "recipe-gelatin-12",
  "target_quantity": 50000,
  "raw_material_lots": ["lot-A23", "lot-B89"],
  "auto_calibrate": true
}
```
The system performs a **pre-flight check** — verifying material availability, machine status, and calibration readiness. If `auto_calibrate` is true, PEC is triggered automatically before production starts.

### 3. Monitor Batch Health via Live Dashboard
Navigate to the **Batch Monitor** view in the web UI. Each batch is displayed with real-time KPIs:
- Fill weight variance (μ ± σ)
- Machine OEE (Overall Equipment Effectiveness)
- Predicted yield vs actual yield
- Anomaly alerts (e.g., sudden temperature spike)

Click any batch ID to drill into its genealogy graph, showing every sensor reading, operator action, and QC result timestamped to millisecond precision.

## Best Practices

### 1. Enforce Strict Material Lot Traceability from Day One
Before any batch creation, ensure that all raw material lots are registered in the system with their COA (Certificate of Analysis) metadata. This enables automatic **hold/release** logic: if a lot’s COA expires or fails a quality check, the system will block its use in any new batch. No manual gatekeeping needed.

### 2. Use the ATP API for Multi-Site Demand Planning
Integrate the ATP endpoint into your ERP or planning tool to query capsule availability across all factories in real time. This prevents over-promising to customers and allows dynamic re-routing of inventory between sites when a line goes down. Typical latency is under 50ms per query.
