### What is capsule-factory-saas?

**capsule-factory-saas** is not just another ERP add-on. It is a **real-time, event-driven production control platform** designed for high-mix, high-volume capsule filling lines. It decouples the physical PLC layer from your planning stack, exposing a clean REST/gRPC API for recipe management, batch tracking, and line OEE prediction.

---

### Core Features (Hardcore)

1. **Dynamic Recipe Binding with PLC Handshake**  
   Every capsule recipe (fill weight, shell composition, tamping force) is versioned and auto-deployed to line controllers via a secure MQTT bridge. The system validates the hardware state before unlocking the next batch—no manual checks, no drift.

2. **Real-Time Yield Anomaly Detection Engine**  
   Leveraging streaming telemetry from fill heads and checkweighers, the engine flags yield deviations at the individual capsule level. It integrates with your existing SCADA via OPC UA, reducing false positives by 40% compared to threshold-based alarms.

3. **Multi-Tenant Batch Lineage with Immutable Audit Trails**  
   Each capsule batch is traced from raw material lot to final packaging, stored as an immutable event log on a private blockchain ledger. This satisfies 21 CFR Part 11 and EU GMP Annex 11 without requiring a separate validation module.

### How to Use (Step-by-Step)

1. **Deploy the Ingestion Agent on Your Line Controller**  
   Install the `capsule-factory-agent` binary on your edge gateway (Linux x86_64 or ARM64). Configure the `config.yaml` with your MQTT broker endpoint and PLC register mappings. Run `./agent start` – the agent will automatically discover fill heads and publish telemetry to the cloud backend.

2. **Create a Production Order via the CLI**  
   Use the `capsule-cli` tool:  
   ```bash
   capsule-cli order create \
     --recipe "Gelatin_Size0_500mg" \
     --quantity 100000 \
     --line "Line-03" \
     --due "2025-07-15T08:00:00Z"
   ```  
   The system validates material availability, locks the recipe version, and returns a `batch_id` (e.g., `CAP-2025-07-12-0042`). This batch ID is used for all subsequent queries.

3. **Monitor Real-Time Dashboard via WebSocket**  
   Open the built-in dashboard at `https://<your-instance>.capsule-factory.io/dash/live`.  
   Subscribe to the batch channel:  
   ```javascript
   const ws = new WebSocket('wss://api.capsule-factory.io/ws/batch/CAP-2025-07-12-0042');
   ws.onmessage = (event) => {
     const data = JSON.parse(event.data);
     console.log(`Yield: ${data.yield}% | Reject Rate: ${data.rejectRate} ppm`);
   };
   ```  
   The dashboard updates every 200ms with fill head status, cumulative yield, and predicted completion time.

### Best Practices (Industry-Level)

1. **Always Pin Your Recipe Versions**  
   Never use `latest` as your recipe identifier in production. Always reference a specific version hash (e.g., `recipe:gelatin-size0-500mg:v3.2.1`). This ensures reproducibility and prevents silent failures when a recipe is updated mid-batch.

2. **Set Up Alerts on Fill Head Drift, Not Just Faults**  
   Most teams only alert on hardware faults. Instead, configure a drift alert when the average fill weight deviates by more than 1.5σ from the target for 10 consecutive capsules. This catches mechanical wear early, saving you from a full line stoppage.
