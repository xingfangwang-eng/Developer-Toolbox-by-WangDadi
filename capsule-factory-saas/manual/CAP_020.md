# capsule-factory-saas - CAP_020

## Overview

This is a production-grade SaaS toolkit engineered for modern capsule manufacturing environments—think real-time batch tracking, defect prevention via statistical process control (SPC), and multi-tenant isolation for pharma-grade compliance. Built for DevOps engineers, quality assurance leads, and plant operations managers who demand zero-downtime deployments and audit-ready data pipelines.

## What is capsule-factory-saas?

`capsule-factory-saas` is a cloud-native platform that transforms raw capsule production data into actionable intelligence. It eliminates silos between filling lines, encapsulation machines, and QA labs by providing a unified API-first architecture. Whether you're scaling from 10k to 10M capsules per batch, this tool ensures traceability from polymer powder to blister pack.

## Core Features

### 1. Real-Time SPC Dashboard with Outlier Detection
Embedded Shewhart control charts and CUSUM algorithms run directly on streaming sensor data (temperature, humidity, fill weight). The system auto-triggers alerts when process drifts beyond 3-sigma limits—no manual threshold tuning required. Perfect for reducing scrap rates in high-speed rotary encapsulation.

### 2. Immutable Batch Ledger with Blockchain-Ready Hashing
Every production event (raw material lot, machine calibration, QA test) is hashed and stored in an append-only log. This creates a tamper-evident chain for FDA 21 CFR Part 11 compliance. Export audit trails as JSON or CSV for regulatory inspections without breaking a sweat.

### 3. Multi-Tenant Resource Scheduler with Priority Queues
Dedicated Kubernetes namespaces per tenant, each with configurable resource quotas (CPU, memory, storage). The scheduler uses weighted fair queuing (WFQ) to ensure critical production orders aren't starved by background analytics jobs. Supports both on-premise and cloud clusters.

## How to Use

### 1. Deploy the Core Stack via Helm Chart
```bash
helm repo add capsule-factory https://charts.capsule-factory.io
helm install prod-cluster capsule-factory/capsule-stack \
  --namespace capsule-prod \
  --set global.tenantId=acme-pharma \
  --set persistence.storageClass=fast-ssd \
  --set monitoring.prometheus.enabled=true
```
This spins up the API gateway, worker nodes, and persistent volume claims. Confirm health with `kubectl get pods -n capsule-prod`.

### 2. Ingest Production Data via REST API
POST sensor readings to the `/v1/batches/{batchId}/events` endpoint. Use the provided Python SDK for bulk uploads:
```python
from capsule_sdk import Client
client = Client(api_key="sk-...", base_url="https://api.capsule-factory.io")
client.ingest_events(batch_id="BATCH-2023-001", events=[
    {"timestamp": "2023-10-05T14:30:00Z", "metric": "fill_weight_mg", "value": 502.3},
    {"timestamp": "2023-10-05T14:30:01Z", "metric": "temperature_c", "value": 24.1}
])
```
All events are validated against schema definitions stored in the tenant's metadata store.

### 3. Trigger Automated QA Workflows
Define SPC rules in YAML and apply them to specific production lines:
```yaml
apiVersion: capsule-factory.io/v1
kind: SPCControlPlan
metadata:
  name: fill-weight-control
spec:
  metric: fill_weight_mg
  chartType: xbar-r
  limits:
    upper: 510.0
    lower: 490.0
  actions:
    - condition: "out_of_control"
      alertSlack: "#qa-alerts"
      autoStopLine: true
```
Apply with `kubectl apply -f spc-plan.yaml`. The system will monitor the streaming data and execute actions (stop conveyor, notify QA, log to ledger) when violations occur.

## Best Practices

### 1. Implement Canary Deployments for Rule Changes
Before rolling out new SPC rules or ML models to all production lines, deploy them to a single "canary" line (e.g., Line 7). Monitor false-positive rates for 24 hours. Use the built-in A/B testing framework to compare baseline vs. new rule performance. This prevents catastrophic line shutdowns from misconfigured thresholds.

### 2. Use Partitioned Time-Series Storage for Cost Efficiency
Store raw sensor data in time-series databases (e.g., InfluxDB or TimescaleDB) with retention policies: 30 days hot (in-memory), 365 days warm (SSD), 5 years cold (S3/Glacier). The API automatically routes queries to the correct tier. For example, real-time dashboards hit the hot tier, while audit reports pull from cold storage. This reduces storage costs by up to 70% while maintaining sub-second query response for recent data.
