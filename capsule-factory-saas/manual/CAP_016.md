### What is capsule-factory-saas?

capsule-factory-saas is a microservices-based platform designed to orchestrate the entire lifecycle of capsule production—from raw gelatin blending to automated encapsulation and batch QA. It replaces legacy ERP modules with a real-time, event-driven architecture that scales horizontally across multi-site factories.

### Core Features

- **Dynamic Batch Routing Engine**: Automatically assigns production orders to the most efficient encapsulation line based on real-time metrics (e.g., temperature, humidity, capsule size tolerance). Reduces changeover time by 40% using constraint-based scheduling.
- **Granular Lot Traceability via Immutable Ledger**: Every capsule batch is logged with SHA-256 hashed metadata (raw material lot, machine ID, operator timestamp). Supports full backward tracing from final blister pack to the original gelatin shipment.
- **Predictive Maintenance API**: Integrates with PLCs and IoT sensors to predict bearing wear and die misalignment in encapsulation machines. Triggers automated work orders and pauses production before defective capsules are formed.

### How to Use

1. **Configure the Production Line Topology**  
   Deploy the `capsule-factory-saas-agent` container on each encapsulation machine. Use the `POST /api/v1/lines` endpoint to register machine specs (e.g., die size, fill volume, speed). Validate via `GET /api/v1/lines/{id}/health`. Example cURL:  
   ```bash
   curl -X POST https://your-saas-instance.com/api/v1/lines \
     -H "Content-Type: application/json" \
     -d '{"line_id": "ENC-03", "specs": {"die_size_mm": 12.5, "fill_volume_ml": 0.5}}'
   ```

2. **Submit a Production Order with Quality Gates**  
   Create an order using the `PUT /api/v1/orders` endpoint. Include `quality_gates` array to enforce intermediate checks (e.g., hardness test after 10,000 capsules). The engine will pause production if thresholds are breached.  
   ```bash
   curl -X PUT https://your-saas-instance.com/api/v1/orders \
     -d '{"order_id": "ORD-2023-104", "product": "HardGel-500", "quantity": 500000, "quality_gates": [{"type": "hardness", "max_deviation": 3}]}'
   ```

3. **Monitor Real-Time Yield Dashboards**  
   Access the embedded Grafana dashboard at `/monitoring/yield`. Filter by line ID and time window. Export raw metrics via `GET /api/v1/metrics/yield?line_id=ENC-03&since=2023-10-01T00:00:00Z`. Use these to adjust fill pressure or drying time in the next batch.

### Best Practices

- **Implement Canary Deployments for Batch Recipes**  
  Before rolling out a new capsule formulation to all production lines, deploy it to a single line (e.g., `ENC-01`) using the `POST /api/v1/recipes/canary` endpoint. Monitor defect rates for 24 hours. Only promote to full production if yield > 98.5%. This prevents widespread contamination due to minor pH changes.

- **Use Webhook Callbacks for Automated Reordering**  
  Configure `POST /api/v1/webhooks` to listen for `inventory.threshold` events. When raw gelatin stock drops below 10 metric tons, automatically trigger a purchase order to your preferred supplier. Attach the latest batch test results as a PDF via the `attachments` field. This eliminates manual stock checks and reduces lead time by 20%.
