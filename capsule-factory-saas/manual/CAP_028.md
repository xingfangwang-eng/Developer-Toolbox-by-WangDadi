# capsule-factory-saas - CAP_028

## Overview

This is a production-grade, multi-tenant SaaS platform purpose-built for capsule manufacturing operations. It streamlines batch tracking, gelatin sourcing, and fill-weight compliance across pharmaceutical, nutraceutical, and cosmetic supply chains. Optimized for English-speaking DevOps and manufacturing engineers.

## What is capsule-factory-saas?

A cloud-native ERP-lite system tailored for capsule factories. It replaces fragmented spreadsheets and legacy on-prem software with a unified API-first architecture. Supports real-time OEE dashboards, FDA 21 CFR Part 11 traceability, and dynamic BOM generation for hard/soft capsule lines.

## Core Features

### 1. Dynamic Batch Genealogy with Immutable Audit Logs
Every capsule batch is tracked from raw gelatin lot to final curing. The system generates a Merkle-tree-style chain of custody, exportable as a signed JSON or PDF for regulatory audits. Supports GS1 barcode integration for downstream serialization.

### 2. Real-Time Fill-Weight Drift Detection
Integrates with load cells and checkweighers via OPC UA or MQTT. The platform applies statistical process control (SPC) to detect drift in fill-weight variance. Alerts trigger auto-pause of the filling line and push a corrective action ticket to the operator’s mobile UI.

### 3. Multi-Tenant Inventory Pooling with Smart Allocation
Each tenant (factory line or site) gets isolated inventory pools, but the system can dynamically reallocate gelatin, colorants, and packaging based on demand signals from a centralized scheduler. Uses a constraint-satisfaction algorithm to minimize cross-contamination risk.

## How to Use

### 1. Connect Your Production Line via OPC UA
Navigate to `Settings > Device Integration` and select OPC UA as the protocol. Enter your server endpoint (e.g., `opc.tcp://192.168.1.100:4840`). The system will auto-discover available tags. Map `FillWeight`, `BatchID`, and `LineSpeed` to the corresponding data points. Click “Validate Connection” to confirm real-time data ingestion.

### 2. Configure a Batch Recipe with Dynamic BOM
Go to `Production > Batch Recipes` and create a new recipe. Define the target capsule size (e.g., Size 0), gelatin type, and colorant ratio. Enable “Dynamic BOM” to allow the system to adjust gelatin weight based on real-time humidity sensor input. Save and assign the recipe to a specific production line.

### 3. Trigger a Batch Run and Monitor SPC Charts
From the `Dashboard`, click “Start Batch”. The system will lock the recipe, verify inventory, and begin data collection. Open the `SPC Chart` widget to see real-time X-bar and R charts. If a data point falls outside ±3σ, an alert will appear. Click “View Root Cause” to see the exact sensor reading and timestamp.

## Best Practices

### 1. Use Separate OPC UA Sessions for Each Production Line
To avoid data collisions and latency spikes, configure a dedicated OPC UA session per physical line. This ensures that fill-weight data from Line A never interferes with Line B’s SPC calculations. The system supports up to 64 concurrent sessions per tenant.

### 2. Schedule Inventory Rebalancing During Off-Peak Hours
Enable “Smart Allocation” with a cron schedule (e.g., 2:00 AM UTC). This prevents allocation conflicts during active production. The algorithm respects quarantine periods for gelatin lots and prevents mixing of kosher and non-kosher materials. Logs are written to a separate audit bucket in your cloud storage.
