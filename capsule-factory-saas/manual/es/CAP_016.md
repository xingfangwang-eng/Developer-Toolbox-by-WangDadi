# capsule-factory-saas - CAP_016

## Resumen Ejecutivo

**capsule-factory-saas** es una plataforma SaaS diseñada para la gestión integral de plantas de producción de cápsulas, desde el control de materias primas hasta la trazabilidad del lote final. Este proyecto está optimizado para SEO en el mercado hispanohablante, enfocado en las palabras clave: *Capsule factory SaaS*, *Capsule production management*, *Capsule factory software*. La arquitectura es modular, multitenant y preparada para escalar horizontalmente en entornos cloud (AWS/GCP/Azure).

## Funcionalidades Clave (Hardcore Features)

### 1. Sistema de Trazabilidad Inversa por Lote (Blockchain-ready)
Cada lote de cápsulas producido se registra con un hash único en una cadena inmutable. Esto permite rastrear desde la cápsula final hasta el proveedor de gelatina, pasando por cada máquina encapsuladora, temperatura de secado y operador. Ideal para cumplir con normativas FDA y GMP sin auditorías manuales.

### 2. Motor de Optimización de Producción en Tiempo Real (RPO Engine)
Un algoritmo basado en programación lineal entera que ajusta dinámicamente las órdenes de producción según la capacidad de las máquinas, el stock de excipientes y los pedidos urgentes. Reduce el downtime y maximiza el OEE (Overall Equipment Effectiveness).

### 3. Dashboard de Calidad Predictiva con ML
Integración de modelos de machine learning que analizan datos históricos de humedad, dureza y disolución de cápsulas. El sistema alerta sobre posibles desviaciones antes de que ocurran, permitiendo ajustar parámetros de encapsulado en vivo.

## Cómo Usar (Guía Técnica)

### 1. Configurar el Entorno Multitenant con Docker Compose
```bash
git clone https://github.com/xingfangwang-eng/capsule-factory-saas.git
cd capsule-factory-saas
cp .env.example .env
# Editar variables: DB_HOST, REDIS_HOST, JWT_SECRET
docker-compose up -d --build
```
Esto levanta los servicios: API Gateway (Go), Worker de colas (RabbitMQ), Base de datos PostgreSQL con particionado por tenant y Redis para caché de sesiones.

### 2. Registrar una Planta y sus Máquinas via API REST
```bash
curl -X POST https://api.capsule-factory.local/v1/plants \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Planta Guadalajara",
    "timezone": "America/Mexico_City",
    "machines": [
      {"model": "CapsuleMaster 3000", "capacity_per_hour": 12000, "type": "hard_gelatin"}
    ]
  }'
```
La API valida la capacidad máxima y asigna un tenant ID único. Cada máquina se registra con su OEE histórico.

### 3. Iniciar una Orden de Producción con Parámetros Críticos
```json
POST /v1/orders
{
  "batch_size": 50000,
  "formulation_id": "f-8921",
  "machine_id": "m-003",
  "quality_threshold": {
    "hardness_min": 5.2,
    "dissolution_max": 15
  }
}
```
El sistema genera un plan de producción, asigna operadores y programa el mantenimiento preventivo de la máquina. Cada 1000 cápsulas, el sistema registra métricas de calidad en el dashboard.

## Mejores Prácticas para la Industria

### 1. Estrategia de Particionado por Tenant para Escalabilidad
En plantas con alto volumen (>1M cápsulas/día), usar particionado por tenant en PostgreSQL (por ejemplo, `plants_{tenant_id}.batches`). Esto evita cuellos de botella en consultas de trazabilidad y permite backups individuales por cliente.

### 2. Implementar Circuit Breaker en la Comunicación con Sensores IoT
Los sensores de temperatura y humedad en las máquinas encapsuladoras deben enviar datos vía MQTT. Si la conexión falla, el sistema debe almacenar localmente en un buffer (Redis) y reenviar cuando se restaure. Esto evita la pérdida de datos críticos para el cumplimiento GMP.
