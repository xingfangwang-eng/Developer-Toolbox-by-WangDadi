# capsule-factory-saas - CAP_019

## Resumen Técnico

Plataforma SaaS diseñada para optimizar flujos de producción en fábricas de cápsulas (farmacéuticas, nutracéuticas o de café). Integra control de lotes en tiempo real, trazabilidad blockchain y análisis predictivo de rendimiento de maquinaria. Ideal para equipos de ingeniería de procesos que buscan reducir downtime y garantizar compliance con normativas GMP (Good Manufacturing Practices).

## Funcionalidades Hardcore

### 1. **Optimización Dinámica de Ciclos de Producción con Algoritmos de Machine Learning**
El sistema analiza históricos de temperatura, humedad y velocidad de llenado para ajustar automáticamente los parámetros de cada lote. Reduce el desperdicio de materia prima hasta en un 18% y detecta desviaciones en tiempo real con alertas vía WebSocket.

### 2. **Trazabilidad Inmutable por Lote con Hash Criptográfico**
Cada lote genera un hash único que se almacena en una cadena de bloques privada. Esto permite auditorías instantáneas sin depender de terceros. Compatible con estándares FDA 21 CFR Part 11 y EU Annex 11.

### 3. **Dashboard de Mantenimiento Predictivo con Señales IoT**
Conecta sensores de vibración y termografía de tus máquinas encapsuladoras. El modelo de regresión logística predice fallos con 72 horas de antelación, permitiendo programar mantenimientos sin interrumpir la producción.

## Pasos para Implementar en tu Entorno

### 1. Configurar el Pipeline de Ingesta de Datos IoT
```bash
# Clona el repositorio y levanta el stack de sensores virtuales
git clone https://github.com/xingfangwang-eng/capsule-factory-saas
cd capsule-factory-saas
docker-compose -f docker-compose.iot.yml up -d

# Verifica que los datos de temperatura/humedad fluyan hacia InfluxDB
curl -X POST http://localhost:8086/api/v2/write?bucket=sensor_data \
  -H "Authorization: Token YOUR_INFLUX_TOKEN" \
  -d "machine_01,tag=temp value=45.2"
```

### 2. Activar el Módulo de Trazabilidad Blockchain
```bash
# Inicializa el ledger privado (Hyperledger Fabric o similar)
python3 scripts/init_ledger.py --network capsule_net --channel production

# Genera un hash para el lote actual (ejemplo: LOTE-20231015-AB12)
curl -X POST http://localhost:3000/api/traceability/hash \
  -H "Content-Type: application/json" \
  -d '{"batch_id": "LOTE-20231015-AB12", "machine": "encapsulator_03"}'
```

### 3. Desplegar el Dashboard de Monitoreo en Tiempo Real
```bash
# Compila el frontend con React y despliega en tu VPS
npm run build --prefix frontend/
scp -r frontend/build/* user@your-server:/var/www/capsule-dashboard/

# Configura Nginx como reverse proxy para las APIs WebSocket
sudo nano /etc/nginx/sites-available/capsule
# Añade: location /ws/ { proxy_pass http://localhost:8080; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; }
```

## Mejores Prácticas para Ingenieros de Procesos

### 1. **Estandariza la Nomenclatura de tus Máquinas y Lotes**
Usa un patrón como `[PLANTA]-[LINEA]-[FECHA]-[SECUENCIA]` (ej: `MX01-L03-20231015-001`). Esto evita colisiones en la base de datos y facilita las consultas SQL con `LIKE` o expresiones regulares.

### 2. **Implementa Pruebas de Integración con Docker Compose**
Antes de tocar producción, levanta un entorno aislado con `docker-compose -f docker-compose.test.yml up`. Incluye un contenedor con datos sintéticos de 30 días para validar que el modelo predictivo no genere falsos positivos.
