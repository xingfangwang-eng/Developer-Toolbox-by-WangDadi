## Performance Tuning for High Concurrency Scenarios

1. **Increase the Number of Worker Threads**

To improve system throughput under high concurrency, consider increasing the number of worker threads in your Java application. This can be achieved by setting the `corePoolSize` and `maximumPoolSize` properties in your `ThreadPoolExecutor`.

Example:
```properties
threadpool:
  core-pool-size: 10
  maximum-pool-size: 20
```

2. **Optimize Redis Cache Settings**

To improve Redis performance under high load, consider adjusting the following settings:

* `maxmemory` to control memory usage and prevent excessive caching
* `maxclients` to limit the number of concurrent connections

Example:
```properties
redis:
  maxmemory: 1000m
  maxclients: 500
```

By applying these performance tuning strategies, you can significantly improve the scalability and responsiveness of your capsule factory SaaS application.
