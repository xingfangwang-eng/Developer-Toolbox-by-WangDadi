# capsule-factory-saas - CAP_026

## 概要

capsule-factory-saas は、カプセル製造ラインのリアルタイム統合管理を実現するエンタープライズ向け SaaS プラットフォームです。生産計画から充填、包装、在庫管理までを一元制御し、GMP（医薬品適正製造規範）準拠のワークフローを自動化します。本プロジェクトは、カプセル工場のデジタルツイン構築を前提に設計されています。

## 詳細な内容

### capsule-factory-saas とは

capsule-factory-saas は、カプセル製造業界に特化したクラウドネイティブな生産管理システムです。従来のオンプレミス型ERPでは実現できなかった、**ライン停止予測**、**原料トレーサビリティの完全可視化**、**リアルタイムOEE（Overall Equipment Effectiveness）算出** を標準機能として提供します。特に、IoTセンサーデータと連携した異常検知エンジンにより、ダウンタイムを平均40%削減します。

### コア機能

1. **マルチテナント対応の動的キャパシティプランナー**
   各テナント（工場）ごとに異なる生産スケジュールを、機械学習ベースの需要予測モデルで最適化。充填機の稼働率、型替え時間、クリーンルームの使用状況を考慮した**動的スロット割り当て**により、ボトルネックを自動解消。API経由で外部ERP（SAP、Oracle）とも双方向連携可能。

2. **GMP準拠のデジタルバッチレコード（eBR）自動生成**
   各ロットの原料ロット番号、充填温度、湿度、機械パラメータをタイムスタンプ付きでブロックチェーンに記録。監査対応の**完全な改ざん防止ログ**を自動生成し、FDA 21 CFR Part 11 に準拠。異常値が検出された場合、即座にライン停止シグナルを発報します。

3. **リアルタイムOEEダッシュボード × 異常予知保全**
   PLC（プログラマブルロジックコントローラ）から直接収集したデータを基に、**パフォーマンス、可用性、品質指数**を秒単位で可視化。過去の故障パターンから次回のメンテナンス最適日時を提案する**予知保全エンジン**を内蔵。アラートはSlack/Teamsへ直接プッシュ可能。

### 使い方

1. **テナント初期設定とライン構成のインポート**
   管理コンソールから新規テナントを作成後、`/api/v1/tenants/{id}/lines` エンドポイントにCSVまたはJSON形式でライン構成（機械ID、センサー閾値、クリーンルームクラス）をPOST。設定完了後、即座にシミュレーションモードで動作検証が可能。例：
   ```bash
   curl -X POST https://api.capsule-factory-saas.com/v1/tenants/abc123/lines \
     -H "Authorization: Bearer {token}" \
     -d '{"lines": [{"machine_id": "FILL-01", "type": "hard_gelatin", "sensor_threshold": {"temperature_max": 25.0}}]}'
   ```

2. **生産ジョブの作成と動的スケジューリング**
   画面上で「新規ジョブ作成」を選択し、製品コード、数量、納期を入力。システムが自動で最適な充填機と時間枠を割り当て、**Ganttチャート**上に反映。スケジュール確定後、`/api/v1/jobs/{id}/start` でライン起動。異常が発生した場合、自動で代替ラインへの**フェイルオーバー**が実行されます。

3. **eBRの監査エクスポートとブロックチェーン検証**
   バッチ完了後、`/api/v1/batches/{batch_id}/ebr` からPDFまたはJSON形式でデジタルバッチレコードをダウンロード。各レコードにはSHA-256ハッシュが付与されており、ブロックチェーンエクスプローラで改ざん有無を検証可能。監査時は、**ワンクリックで全トレーサビリティレポート**を生成します。

### ベストプラクティス

- **ライン停止リスクの軽減には「カナリアデプロイ」を導入せよ**
   新たな生産パラメータや原料ロットを本番投入する前に、テスト用の小ロット（カナリア）で異常検知エンジンを検証する。capsule-factory-saas の `simulation mode` を活用し、実際のラインに影響を与えずにリスクを評価する。これにより、大規模なロット不良を未然に防ぐことができる。

- **OEEデータは週次ではなく、リアルタイムで監視せよ**
   多くの工場ではOEEを週単位で集計するが、これでは異常の兆候を見逃す。本ツールのダッシュボードで**15分単位のスループット**と**瞬時停止回数**を監視し、閾値超過時に自動でアラートを発報する設定を推奨する。特に、充填機の温度偏差が±0.5°Cを超えた場合、即座にライン停止と品質再検査をトリガーするルールを実装すべきである。
