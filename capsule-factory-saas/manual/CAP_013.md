## What is capsule-factory-saas?

**capsule-factory-saas** is a multi-tenant, API-first platform designed to orchestrate end-to-end capsule manufacturing workflows — from raw material traceability and batch formulation to real-time production line monitoring and compliance reporting. It eliminates spreadsheet chaos and siloed ERP systems by providing a unified control plane for capsule factories of any scale.

## Core Features

### 1. Real-Time Batch Traceability with Immutable Audit Logs  
Every capsule batch is tracked from raw material lot to finished good, with cryptographic hash anchoring on-chain for FDA/EMA audit readiness. The system auto-generates a **Digital Batch Record (DBR)** that is immutable, exportable, and searchable via GraphQL.

### 2. Dynamic Production Line Scheduler with Gantt-Aware Constraints  
The scheduler ingests real-time machine telemetry (OEE, downtime, changeover time) and automatically re-sequences production orders to minimize idle time and maximize throughput. Supports **multi-factory, multi-line, multi-shift** constraints with drag-and-drop manual overrides.

### 3. AI-Powered Yield Optimization Engine  
Using historical batch data and environmental variables (humidity, temperature, machine wear), the engine predicts optimal fill weight and shell hardness settings per capsule type. Reduces scrap rate by up to 18% in pilot deployments.

## How to Use

### 1. Onboard a New Capsule Factory Tenant  
Run the CLI command to bootstrap a tenant with isolated database schema and API keys:

```bash
capsule-factory-saas bootstrap --tenant "NutraCaps GmbH" --region eu-west-1 --plan enterprise
```

This creates a dedicated PostgreSQL schema, provisions a subdomain (`nutracaps.app.capsule-factory.io`), and generates a JWT-based admin token. All resources are managed via Terraform under the hood.

### 2. Integrate with Existing PLC/SCADA Systems  
Deploy the **edge connector** as a Docker container on your factory floor:

```bash
docker run -d \
  --name capsule-edge-connector \
  -e TENANT_ID="nutracaps" \
  -e MQTT_BROKER="tcp://192.168.1.100:1883" \
  -e API_ENDPOINT="https://api.capsule-factory.io/v1/telemetry" \
  capsule-factory/edge-connector:2.3.1
```

The connector maps OPC-UA tags (machine speed, temperature, pressure) to the platform’s standardized telemetry schema. Data is buffered locally and replayed on network failure.

### 3. Trigger a Batch Production Run via API  
Use the REST endpoint to initiate a batch with custom parameters:

```bash
curl -X POST https://api.capsule-factory.io/v1/batches \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": "vitamin-d3-1000iu",
    "quantity": 500000,
    "target_weight_mg": 500,
    "shell_type": "vegetarian",
    "line_id": "line-03",
    "priority": 1
  }'
```

The platform validates inventory, checks line availability, and returns a batch ID with an estimated completion time. Real-time status is streamed via WebSocket.

## Best Practices

### 1. Segregate Production Data by Tenant Using Schema-Based Multi-Tenancy  
Avoid shared-table architectures. Use dedicated PostgreSQL schemas per tenant to ensure data isolation, simplify backup/restore per tenant, and allow per-tenant schema migrations without downtime. This aligns with SOC 2 Type II requirements.

### 2. Implement Idempotency Keys for All Write Operations  
When triggering batch runs or updating machine parameters, always include an `Idempotency-Key` header. This prevents duplicate production orders in case of network retries or client timeouts — critical in high-throughput factory environments where a double order could cause line overcommit.
