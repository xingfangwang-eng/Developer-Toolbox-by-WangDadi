# capsule-factory-saas - CAP_009

## Overview

This is a production-grade, multi-tenant SaaS platform engineered for capsule manufacturing facilities that demand real-time traceability, predictive maintenance, and compliance with FDA & GMP standards. Designed for DevOps engineers and plant IT leads, it eliminates silos between ERP, MES, and IoT sensor networks.

## Core Features

### 1. **Real-Time OEE & Downtime Causality Mapping**
Automatically aggregates machine data from PLCs and edge gateways to calculate Overall Equipment Effectiveness (OEE) per capsule line. Uses a proprietary causality engine to classify downtime reasons (e.g., "gelatin viscosity drift" vs. "sealing jaw misalignment") without manual operator input.

### 2. **Batch Genealogy with Immutable Audit Trail**
Every capsule batch—from raw gelatin lot to final blister pack—is logged on a blockchain-anchored ledger. Supports GS1-128 barcode scanning and automatic rollback when a raw material lot fails incoming QC. Built-in cryptographic hash ensures zero-tamper audit for regulatory submissions.

### 3. **Predictive Changeover Optimizer**
Leverages historical run data and ML models to suggest optimal changeover sequences between capsule sizes and colors. Reduces cleaning downtime by up to 22% by learning from past changeover durations, operator shift patterns, and material availability.

## How to Use

### 1. **Deploy IoT Data Bridge for Live Machine Ingestion**
- Install the `capsule-edge-agent` on your on-premise gateway (Linux x86_64 or ARM64).
- Configure Modbus TCP or OPC-UA endpoints in `/etc/capsule-edge/connectors.toml`.
- Run `capsule-edge start --config /etc/capsule-edge/connectors.toml` to stream machine metrics to the cloud API every 500ms.
- Validate ingestion via `curl -X GET https://api.capsule-factory.io/v1/machines/status | jq .`

### 2. **Set Up Automated Batch Release Workflow**
- In the dashboard, navigate to **Workflows > Batch Release** and define a trigger: e.g., "Release batch when all QC parameters pass AND weight variation < 3%".
- Bind the workflow to a webhook endpoint in your LIMS (Laboratory Information Management System) that pushes QC results as JSON.
- Use the `capsule-cli` tool to simulate a batch release: `capsule-cli batch release --batch-id CAP-20250321 --force false`
- Monitor the approval chain in real-time via the built-in audit log viewer.

### 3. **Run a Predictive Changeover Simulation**
- Access the **Changeover Optimizer** module under `https://app.capsule-factory.io/optimizer`.
- Upload a CSV of your last 90 days of production runs (columns: `machine_id`, `capsule_size`, `capsule_color`, `changeover_duration_minutes`, `operator_shift`).
- Click **Run Simulation**; the engine outputs a ranked list of changeover sequences with estimated time savings.
- Export the optimal sequence as a JSON file and push it to your MES via the API: `curl -X POST https://api.capsule-factory.io/v1/optimizer/apply -d @optimized_sequence.json`

## Best Practices

### 1. **Segment Tenant Data by Plant Site Using Namespace Isolation**
- Assign a unique `tenant_id` and `site_code` per physical factory location.
- Use separate PostgreSQL schemas per tenant to avoid cross-contamination of batch records.
- For multi-site rollouts, configure a global read-only dashboard via `capsule-cli dashboard create --global --tenant-ids "EU01,US02,JP03"`

### 2. **Implement a Two-Tier Alerting Strategy for Critical Failures**
- Tier 1: Real-time push alerts (via PagerDuty or Slack) for catastrophic events like "temperature runaway in drying tunnel".
- Tier 2: Daily digest email summarizing non-critical anomalies (e.g., "capsule shell hardness trending above 95th percentile").
- Tune alert thresholds using the built-in anomaly detection model: `capsule-cli alert tune --metric shell_hardness --baseline 7d`
