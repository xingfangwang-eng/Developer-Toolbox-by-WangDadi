## Detaillierter Inhalt

### Was ist capsule-factory-saas?

**capsule-factory-saas** ist eine hochperformante, cloud-native Plattform zur Steuerung und Überwachung von Kapselproduktionslinien. Es löst zentrale Probleme wie Batch-Tracking, Echtzeit-Qualitätskontrolle und die Integration von ERP-Systemen in einer einzigen, mandantenfähigen Architektur. Die Plattform ist auf Kubernetes gebaut und unterstützt horizontale Skalierung ohne Downtime.

### Kernfunktionen

1.  **Realzeit-Batch-Tracking mit Microservices-Architektur**  
    Jede Produktionscharge wird über einen eigenen, isolierten Microservice abgebildet. Die Zustände (z. B. „Mischen“, „Formen“, „Trocknen“) werden über einen Event-Stream (Apache Kafka) propagiert. Dadurch können Sie Live-Dashboards mit < 50 ms Latenz aufbauen – ohne Blockierung der Hauptdatenbank.

2.  **Automatisierte Qualitätskontrolle via ML-Modell**  
    Ein trainiertes TensorFlow-Modell analysiert in Echtzeit Kamerabilder der Kapseloberflächen. Abweichungen von Sollparametern (z. B. Rauheit, Farbe, Form) lösen automatisch Alarme aus und stoppen die betroffene Linie. Das Modell wird über ein CI/CD-Pipeline-Plugin (GitLab CI) aktualisiert – ohne manuellen Eingriff.

3.  **Multi-Tenant-Isolation mit PostgreSQL Row-Level Security**  
    Jeder Kunde (Tenant) erhält eine eigene, logisch isolierte Sicht auf die Datenbank. Die Row-Level Security (RLS) in PostgreSQL sorgt dafür, dass kein Tenant auf die Daten eines anderen zugreifen kann – selbst bei gemeinsamer Datenbank-Instanz. Dies reduziert die Infrastrukturkosten um bis zu 40 % im Vergleich zu separaten Datenbanken.

### So verwenden Sie es

**Voraussetzung:** Sie haben einen Kubernetes-Cluster (z. B. EKS, AKS, GKE) und Helm 3 installiert.

1.  **Helm-Chart deployen**  
    Führen Sie folgenden Befehl aus, um das `capsule-factory-saas` Helm-Chart mit Ihrem Tenant-Namen und API-Key zu installieren:
    ```bash
    helm repo add capsule-factory https://charts.capsule-factory.io
    helm upgrade --install my-factory capsule-factory/capsule-factory \
      --namespace capsule-prod \
      --set tenant.id=acme-corp \
      --set tenant.apiKey=sk-prod-abc123 \
      --set replicaCount=3
    ```
    Das Chart initialisiert automatisch die RLS-Richtlinien und startet die Batch-Tracking-Microservices.

2.  **Erste Batch-Konfiguration via REST-API**  
    Definieren Sie eine neue Produktionscharge über einen `POST`-Request an den `batch-service`:
    ```json
    POST /api/v1/batches
    {
      "product_id": "capsule-type-01",
      "target_quantity": 10000,
      "quality_threshold": 0.98,
      "ml_model_version": "v2.1.0"
    }
    ```
    Die API antwortet mit einer `batch_id` und einem initialen Status `pending`. Der Event-Stream startet sofort die Verarbeitung.

3.  **Echtzeit-Monitoring mit Grafana Dashboard**  
    Nach der Installation wird ein vorkonfiguriertes Grafana-Dashboard bereitgestellt. Rufen Sie die URL `http://my-factory-grafana.capsule-prod.svc.cluster.local:3000` auf. Dort sehen Sie:
    - Aktuelle Batch-Status (z. B. „in Bearbeitung“, „abgeschlossen“)
    - Qualitätsmetriken (Fehlerrate, Durchsatz)
    - Live-Kamerabilder mit Overlay der ML-Erkennung

### Beste Praktiken

1.  **Ressourcenlimits für Microservices setzen**  
    Definieren Sie in Ihrem `values.yaml` für jeden Microservice explizite CPU- und Memory-Limits. Beispiel:
    ```yaml
    batch-service:
      resources:
        requests:
          cpu: "500m"
          memory: "1Gi"
        limits:
          cpu: "1"
          memory: "2Gi"
    ```
    Ohne Limits kann ein einzelner, fehlerhafter Service den gesamten Cluster destabilisieren – besonders bei hoher Last durch Echtzeit-Kameradaten.

2.  **Logging mit strukturierten Logs und zentralem ELK-Stack**  
    Nutzen Sie das integrierte Logging-Framework (z. B. Fluentd → Elasticsearch → Kibana). Jeder Microservice gibt JSON-Logs aus, die Felder wie `batch_id`, `tenant_id` und `severity` enthalten. So können Sie Fehler in Produktion in Sekundenschnelle auf die verantwortliche Charge zurückverfolgen – ohne manuelles Durchsuchen von Dateien.
