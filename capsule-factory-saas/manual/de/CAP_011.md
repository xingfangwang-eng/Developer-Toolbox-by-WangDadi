# capsule-factory-saas - CAP_011

## Überblick

capsule-factory-saas ist eine hochperformante, cloud-native Plattform zur vollständigen Automatisierung und Überwachung von Kapselproduktionslinien. Es vereint Echtzeit-Datenanalyse, IoT-Integration und KI-gestützte Qualitätskontrolle in einer einzigen, mandantenfähigen Architektur. Entwickelt für mittelständische und industrielle Pharma- und Nahrungsergänzungsmittelhersteller, die ihre OEE (Overall Equipment Effectiveness) maximieren und Ausschussraten minimieren möchten.

## Kernfunktionen (Hardcore-Features)

1.  **Multi-Tenant Produktions-Runtime mit Echtzeit-OEE-Dashboard**  
    Jede Produktionslinie wird als isolierter Tenant in einer Kubernetes-basierten Microservices-Umgebung abgebildet. Das Dashboard aggregiert Maschinendaten (Zyklen, Temperatur, Druck) und berechnet automatisch OEE, Verfügbarkeit, Leistung und Qualität – granular bis auf einzelne Kapselchargen.

2.  **KI-gestützte Fehlerklassifikation & Predictive Maintenance**  
    Ein trainiertes Modell (CNN + LSTM) analysiert Bilddaten aus Hochgeschwindigkeitskameras und klassifiziert Defekte (Risse, Füllmengenabweichungen, Versiegelungsfehler) in Echtzeit. Parallel prognostiziert ein zweites Modell den optimalen Wartungszeitpunkt für Stempel und Matrizen, basierend auf Verschleißmustern und Betriebsstunden.

3.  **Audit-Trail & Batch-Rückverfolgung über Blockchain-ähnlichen Log-Ledger**  
    Jede Charge erhält einen eindeutigen, kryptografisch signierten Hash, der alle Produktionsparameter (Rohstofflos, Maschineneinstellungen, Umgebungsbedingungen) unveränderbar speichert. Dies ermöglicht lückenlose Rückverfolgung für GMP- und FDA-Audits – direkt aus dem Dashboard exportierbar.

## So verwenden Sie es (Detaillierte, professionelle Schritte)

1.  **Konfiguration der Produktionslinie über die YAML-basierte Tenant-Definition**  
    Erstellen Sie im Admin-Panel eine neue Tenant-Konfiguration (`POST /api/tenants`). Definieren Sie die Maschinen-ID, Sensormapping (z. B. `temp_sensor_01 → `temperature_capsule_zone`), und die Qualitätsparameter (z. B. `max_fill_deviation: 0.02g`). Nutzen Sie die mitgelieferte CLI (`capsule-cli tenant init --config tenant.yaml`) zur Validierung der Struktur gegen das JSON-Schema.

2.  **Einrichtung des Echtzeit-Monitorings mit Prometheus & Grafana**  
    Aktivieren Sie in der Tenant-Konfiguration das Flag `metrics.enabled: true`. Das System startet automatisch einen Prometheus-Exporter auf Port `:9101`. Importieren Sie das vordefinierte Grafana-Dashboard `capsule-factory-oee.json` aus dem Repository. Konfigurieren Sie Alerts (z. B. `OEE < 75%` oder `predictive_maintenance_score > 0.8`) über den integrierten Alertmanager.

3.  **Trigger eines KI-Qualitätsaudits für eine Charge**  
    Nach Abschluss einer Charge (z. B. Charge-ID `CH-20231027-001`) rufen Sie den Audit-Endpoint auf:  
    `POST /api/quality/audit/CH-20231027-001`  
    Das System extrahiert die gespeicherten Bilder und Sensordaten, führt das KI-Modell aus und liefert einen detaillierten Report (JSON/PDF) mit Defektklassifikation, Konfidenzwerten und empfohlenen Korrekturmaßnahmen. Speichern Sie den Report im integrierten Dokumenten-Manager oder exportieren Sie ihn via Webhook in ein DMS.

## Beste Praktiken (Industry Best Practices)

1.  **Batch-Größen-Optimierung durch Simulation vor Produktionsstart**  
    Nutzen Sie den integrierten `simulation`-Befehl (`capsule-cli simulate --batch-size 100000 --machine-type KAP-5000`), um Engpässe, Temperaturverläufe und Füllmengenverteilung vorherzusagen. Dies reduziert Ausschuss um bis zu 15 % und spart teure Nachjustierungen in der laufenden Produktion.

2.  **Zero-Downtime-Deployment von Modell-Updates**  
    Verwenden Sie das Feature `model-canary-deployment` im Admin-Panel. Statt das KI-Modell sofort auf alle Produktionslinien zu pushen, deployen Sie es zuerst auf eine Test-Tenant. Überwachen Sie Metriken wie `false_positive_rate` und `inference_latency`. Erst nach erfolgreicher Validierung wird das Update per Rolling-Update auf alle Tenants ausgerollt – ohne Produktionsunterbrechung.
