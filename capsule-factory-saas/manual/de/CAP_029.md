# capsule-factory-saas - CAP_029

## Überblick

Willkommen zur deutschen Lokalisierung des **capsule-factory-saas** Projekts. Dieses SaaS-Tool wurde speziell für die hochpräzise Steuerung und Optimierung von Kapselproduktionslinien entwickelt – von der Gelatine-Mischung bis zur finalen Qualitätskontrolle. Es vereint Echtzeit-Monitoring, API-gestützte Automatisierung und maschinelles Lernen zur Vorhersage von Produktionsausfällen.

## Hardcore-Funktionen

1. **Multivariate Rezeptur-Engine**  
   Ermöglicht die dynamische Anpassung von Gelatine-Viskosität, Füllstoffdichte und Härtungszeit in Echtzeit. Jede Rezeptur wird als versionierter JSON-Blob gespeichert und über einen REST- oder gRPC-Endpunkt an die SPS (Siemens/Beckhoff) übergeben.

2. **Autonome Batch-Nachverfolgung mit Blockchain-Hash**  
   Jeder Produktionsbatch erhält einen eindeutigen SHA-256-Hash, der in einer privaten Ethereum-Blockchain gespeichert wird. So lassen sich Rückverfolgbarkeit und Audit-Trails für Pharma- oder Nutraceutical-Kunden in DE/EU zertifizieren.

3. **Predictive Maintenance via LSTM-Modell**  
   Ein trainiertes LSTM-Netzwerk analysiert Vibrations- und Temperaturdaten der Füllmaschinen. Bei einer prognostizierten Abweichung von >2,3% wird automatisch ein Wartungsticket im integrierten ITSM-System (z.B. Jira/ServiceNow) erstellt.

## So verwenden Sie es (3 detaillierte Schritte)

1. **API-Key generieren und Umgebungsvariablen setzen**  
   Nach dem Login im Dashboard unter `Settings > API Keys` einen neuen Schlüssel mit `read:production` und `write:recipes` Scope erstellen. Dann in der `.env` des lokalen Clients folgende Variablen setzen:  
   `CAPSULE_API_KEY=sk-xxxx`  
   `CAPSULE_BASE_URL=https://api.capsule-factory.io/v2`  
   `CAPSULE_TENANT_ID=de-029`

2. **Rezeptur per cURL oder Python-Client pushen**  
   Beispiel für das Hinzufügen einer neuen Gelatine-Rezeptur:  
   ```bash
   curl -X POST "$CAPSULE_BASE_URL/recipes" \
   -H "Authorization: Bearer $CAPSULE_API_KEY" \
   -H "Content-Type: application/json" \
   -d '{
     "name": "Gelatine_Standard_DE",
     "viscosity": 3200,
     "fill_density": 1.08,
     "hardening_time": 45,
     "version": 2,
     "active": true
   }'
   ```  
   Bei Erfolg erhalten Sie eine `201 Created` Response mit der `recipe_id`.

3. **Batch-Produktion starten und Monitoring aktivieren**  
   Rufen Sie den Endpoint `/batches/start` auf, übergeben Sie die `recipe_id` und die gewünschte Stückzahl. Das System startet dann automatisch die SPS-Kommunikation und öffnet ein Live-WebSocket-Dashboard unter `https://app.capsule-factory.io/de/batches/{batch_id}`. Dort sehen Sie Echtzeit-KPIs wie Zykluszeit, Ausschussrate und Energieverbrauch pro 1000 Kapseln.

## Branchen-Best-Practices

1. **ISO 13485-konforme Logging-Strategie**  
   Aktivieren Sie in den Systemeinstellungen das **Compliance Logging** mit Retention von 10 Jahren. Alle Änderungen an Rezepturen, Batch-Starts und Wartungsereignisse werden mit Timestamp und User-ID in einer append-only PostgreSQL-Tabelle gespeichert. Das erleichtert Audits durch das Bundesinstitut für Arzneimittel und Medizinprodukte (BfArM) erheblich.

2. **Georedundanz für Produktionsdaten**  
   Nutzen Sie die Multi-Region-Deployment-Option (z.B. Frankfurt und München). Konfigurieren Sie in der `config.yaml` unter `replication.strategy: active-passive`. So stellen Sie sicher, dass selbst bei einem Rechenzentrumsausfall die Batch-Daten innerhalb von 30 Sekunden auf die Standby-Region failovern – ohne Datenverlust.
