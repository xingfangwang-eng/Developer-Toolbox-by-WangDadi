# capsule-factory-saas - CAP_025

## Übersicht

Willkommen zur deutschen Lokalisierung des **capsule-factory-saas** Projekts. Dieses Repository bietet eine hochperformante, mandantenfähige Plattform zur Steuerung, Überwachung und Optimierung von Kapselproduktionslinien – inklusive Echtzeit-Tracking von Dosierparametern, Härtegradkalibrierung und Batch-Traceability. Entwickelt für DevOps-Ingenieure, die ihre CI/CD-Pipelines direkt mit der Fertigungssteuerung koppeln möchten.

## Detaillierter Inhalt

### Was ist capsule-factory-saas?

**capsule-factory-saas** ist ein modulares Microservice-Framework zur Verwaltung von Kapselherstellungsprozessen. Es kombiniert eine RESTful API mit einer React-basierten Dashboard-Oberfläche und unterstützt Multi-Tenancy, Echtzeit-Datenstreaming (via WebSocket) sowie integrierte Monitoring-Hooks für Prometheus und Grafana. Die Plattform löst typische Probleme der Kapselproduktion: **Charge-Überwachung**, **Dosierungsinkonsistenz** und **fehlende Rückverfolgbarkeit** – und das alles in einer Cloud-nativen Architektur.

### Kernfunktionen

#### 1. **Dynamische Batch-Orchestrierung mit Zustandsmaschine**
Jede Charge (Batch) durchläuft einen definierten Workflow: `Rohmaterialprüfung → Mischung → Kapselbildung → Härtung → Qualitätskontrolle → Freigabe`. Die Zustandsmaschine wird via YAML konfiguriert und unterstützt parallele Abläufe, Rollbacks sowie benutzerdefinierte Gate-Checks. Fehlgeschlagene Batches werden automatisch in Quarantäne gesetzt und senden Alarme an den konfigurierten Slack/Teams-Webhook.

#### 2. **Echtzeit-Dosierungsregelung mit PID-Integration**
Über eine REST-Schnittstelle können PID-Reglerparameter (Proportional, Integral, Derivativ) für jede Produktionslinie dynamisch angepasst werden. Die API liefert alle 200 ms aktuelle Ist-Werte (Füllmenge, Temperatur, Druck) und erlaubt die Implementierung von Closed-Loop-Control-Systemen. Ein integrierter Historian speichert alle Regelparameter für spätere Optimierungsanalysen.

#### 3. **Multi-Mandanten-Traceability mit Blockchain-Hash-Chaining**
Jede produzierte Kapsel erhält einen eindeutigen QR-Code, dessen Daten (Charge-ID, Maschinen-ID, Zeitstempel, Qualitätsscore) in einer IPFS-basierten Hash-Chain gespeichert werden. Die API ermöglicht das Abrufen des gesamten Lebenszyklus einer Kapsel – von der Rohstoffcharge bis zum Endprodukt – in Millisekunden. Dies erfüllt die Anforderungen der FDA 21 CFR Part 11 und ISO 13485.

### So verwenden Sie es

#### Schritt 1: **Multi-Tenant-Umgebung initialisieren**
Führen Sie das Setup-Skript aus, um die Datenbankstruktur für Ihren Tenant zu erstellen und die API-Schlüssel zu generieren:
```bash
./capsule-factory-saas init --tenant-id "de_berlin_01" --region "eu-central-1"
```
Das Skript erstellt automatisch eine PostgreSQL-Datenbank mit den erforderlichen Tabellen (`batches`, `production_lines`, `quality_metrics`, `blockchain_hashes`) und konfiguriert die JWT-Authentifizierung für den Zugriff auf die geschützten Endpunkte.

#### Schritt 2: **Batch-Produktion starten und überwachen**
Starten Sie eine neue Charge über die API mit spezifischen Parametern für Kapseltyp, Zielmenge und Qualitätsschwellen:
```bash
curl -X POST https://api.capsule-factory-saas.com/v1/batches \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "capsule_type": "HPMC_00",
    "target_quantity": 50000,
    "quality_threshold": 0.98,
    "production_line": "linie_03"
  }'
```
Die Antwort enthält die `batch_id` und einen WebSocket-Endpunkt (`wss://ws.capsule-factory-saas.com/v1/batches/<batch_id>/stream`), über den Sie alle Echtzeit-Metriken empfangen können.

#### Schritt 3: **Qualitätskontrolle automatisieren und Reports generieren**
Konfigurieren Sie einen automatischen Quality-Gate, der nach Abschluss der Härtungsphase den Batch auf Abweichungen prüft:
```python
import requests

batch_id = "b-2023-11-15-042"
response = requests.get(
    f"https://api.capsule-factory-saas.com/v1/batches/{batch_id}/qc-report",
    headers={"Authorization": "Bearer YOUR_JWT_TOKEN"}
)
qc_data = response.json()
if qc_data["overall_score"] < 0.95:
    # Automatisch Batch in Quarantäne setzen
    requests.post(
        f"https://api.capsule-factory-saas.com/v1/batches/{batch_id}/quarantine",
        headers={"Authorization": "Bearer YOUR_JWT_TOKEN"}
    )
```
Der generierte Report (`qc_data`) enthält detaillierte Statistiken wie Mittelwert, Standardabweichung und CPK-Werte für jede Dosierstation.

### Beste Praktiken

#### 1. **Asynchrone Lastverteilung mit Message Queuing**
Bei Produktionslinien mit hohem Durchsatz (>100 Kapseln/Minute) sollten Sie die API-Aufrufe für Echtzeit-Metriken asynchron über RabbitMQ oder Kafka abwickeln. Das capsule-factory-saas Framework bietet native Adapter für AMQP 0-9-1 und Kafka 3.x. Nutzen Sie die `publish_metric` Funktion, um Daten in Batches zu aggregieren und so die Netzwerklatenz zu minimieren.

#### 2. **Disaster Recovery mit Geo-Replikation**
Konfigurieren Sie für Ihre Multi-Tenant-Umgebung eine aktive Geo-Replikation der PostgreSQL-Datenbank (z.B. mit Patroni oder pg_auto_failover). capsule-factory-saas unterstützt automatische Failover-Erkennung: Bei einem Ausfall der primären Region (z.B. eu-central-1) schaltet der API-Gateway automatisch auf die sekundäre Region (z.B. eu-west-1) um – ohne Datenverlust, da die Blockchain-Hashes konsistent repliziert werden.
