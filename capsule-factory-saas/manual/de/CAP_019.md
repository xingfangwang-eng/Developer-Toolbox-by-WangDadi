# capsule-factory-saas - CAP_019

## Übersicht

Willkommen bei **capsule-factory-saas** – der hochperformanten Open-Source-Lösung für die Verwaltung und Optimierung von Kapselproduktionslinien. Diese SaaS-Plattform wurde speziell für Ingenieure, Produktionsleiter und DevOps-Teams entwickelt, die in der Hartkapsel- oder Weichkapselherstellung tätig sind. Sie kombiniert Echtzeit-Monitoring, maschinelles Lernen zur Qualitätskontrolle und mandantenfähige Microservices, um Ausfallzeiten zu minimieren und die Gesamtanlageneffektivität (OEE) zu maximieren.

## Detaillierter Inhalt

### Was ist capsule-factory-saas?

capsule-factory-saas ist eine modulare, cloud-native Architektur, die die gesamte Wertschöpfungskette einer Kapselfabrik abbildet – von der Rohstoffverwaltung über die Füll- und Verschließprozesse bis hin zur Endverpackung und Chargenrückverfolgung. Anders als monolithische ERP-Systeme setzt diese Plattform auf **eventgetriebene Stream-Processing-Pipelines** (Apache Kafka, Flink) und eine **Headless-CMS-Struktur**, die sich nahtlos in bestehende MES- und SCADA-Infrastrukturen integriert. Die Plattform ist vollständig auf Deutsch lokalisiert und unterstützt ISO 9001 sowie GMP-konforme Audit-Trails.

### Kernfunktionen

1. **Prädiktive Inline-Qualitätskontrolle via Computer Vision**  
   Ein auf YOLOv8 und TensorRT optimiertes Modell analysiert in Echtzeit Kapselgeometrie, Nahtdichtigkeit und Farbabweichungen direkt am Förderband. Fehlerhafte Kapseln werden automatisch aussortiert – die Fehlerrate sinkt um bis zu 97 % im Vergleich zu manuellen Stichproben.

2. **Multi-Tenant-Produktionsplanung mit dynamischem Lastausgleich**  
   Ein auf OR-Tools basierender Planungsalgorithmus verteilt Produktionsaufträge über mehrere Kapselpressen und Füllstationen, wobei Rüstzeiten, Materialverfügbarkeit und Wartungsfenster berücksichtigt werden. Die Planung aktualisiert sich bei jeder Änderung in unter 500 ms.

3. **Blockchain-basierte Chargenrückverfolgung & Echtzeit-Audit-Trail**  
   Jede Charge wird mit einem eindeutigen, unveränderlichen Hash auf einer privaten Hyperledger-Fabric-Blockchain versehen. Vom Rohstoff-Lieferanten bis zum Endkunden – alle Transaktionen sind manipulationssicher und für Behörden jederzeit einsehbar.

### So verwenden Sie es

**Schritt 1: Lokale Entwicklungsumgebung mit Docker Compose aufsetzen**  
Klone das Repository und starte die gesamte Infrastruktur (PostgreSQL, Kafka, Redis, MinIO, ML-Serving-API) mit einem Befehl:
```bash
git clone https://github.com/xingfangwang-eng/capsule-factory-saas.git
cd capsule-factory-saas
docker compose -f deploy/docker-compose.dev.yml up --build
```
Nach dem Start ist die Swagger-Dokumentation unter `http://localhost:8000/docs` verfügbar. Führe die initiale Datenbankmigration aus:
```bash
docker compose exec backend alembic upgrade head
```

**Schritt 2: Produktionsauftrag über die gRPC-API anlegen**  
Nutze den bereitgestellten Python-Client, um einen neuen Auftrag mit spezifischen Parametern zu starten:
```python
import grpc
from proto import production_pb2, production_pb2_grpc

channel = grpc.insecure_channel('localhost:50051')
stub = production_pb2_grpc.ProductionServiceStub(channel)
order = production_pb2.OrderRequest(
    product_id="CAP-1001",
    quantity=50000,
    capsule_material="Gelatine",
    fill_volume_ml=0.5,
    priority=2
)
response = stub.CreateOrder(order)
print(f"Auftrag erstellt: {response.order_id}")
```
Die Antwort enthält eine UUID, die als Referenz für alle weiteren Operationen dient.

**Schritt 3: Qualitätsdaten über das ML-Dashboard visualisieren**  
Öffne das Grafana-Dashboard unter `http://localhost:3000/d/quality`. Hier siehst du:
- **Defect Heatmap**: Welche Kapseltypen an welcher Station ausfallen.
- **Confidence Score**: Die Wahrscheinlichkeit, dass eine Kapsel fehlerfrei ist (basierend auf dem aktuellen Modell).
- **Real-Time Alerts**: Bei Überschreitung der Fehlertoleranz (Standard: 0,5 %) wird automatisch ein Webhook an Slack/Teams gesendet.

### Beste Praktiken

1. **Separate Kafka-Topics pro Produktionslinie**  
   Um Latenz und Datenkonsistenz zu gewährleisten, solltest du für jede physische Kapselpresse ein eigenes Kafka-Topic anlegen. Nutze die Partition-Key-Strategie `machine_id + timestamp`, damit die Event-Reihenfolge garantiert bleibt – essenziell für die korrekte Rückverfolgung.

2. **ML-Modell-Rollout mit Canary-Deployment**  
   Führe neue Versionen des Computer-Vision-Modells zunächst nur auf einer einzelnen Station aus (z. B. Station 3). Überwache die False-Positive-Rate für 24 Stunden. Erst wenn die Metrik unter 0,1 % liegt, rolle das Modell auf alle Stationen aus. Verwende dazu die integrierte Feature-Flag-Konfiguration in `config/ml_rollout.yaml`.
