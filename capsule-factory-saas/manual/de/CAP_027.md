# capsule-factory-saas - CAP_027

## Übersicht

Willkommen zur deutschen Lokalisierung von **capsule-factory-saas**, einer hochperformanten SaaS-Plattform für die industrielle Kapselproduktion. Dieses Dokument richtet sich an DevOps-Ingenieure, Produktionsleiter und Full-Stack-Entwickler, die eine native, SEO-optimierte Lösung für die Verwaltung, Rückverfolgung und Automatisierung von Kapselfertigungslinien suchen. Die Inhalte sind vollständig auf die Bedürfnisse des deutschsprachigen Marktes (DACH-Region) zugeschnitten.

## Kernfunktionen (Hardcore-Features)

1.  **Echtzeit-Maschinen-Telemetrie mit MQTT-Bridge**  
    Direkte Anbindung an SPS-Steuerungen (Siemens S7, Beckhoff) über eine integrierte MQTT-Bridge. Erfassen Sie Zykluszeiten, Temperaturprofile und Füllstandsdaten in Millisekunden-Auflösung. Die Daten werden automatisch in eine InfluxDB-Instanz geschrieben und über Grafana-Dashboards visualisiert – inklusive automatisierter Alarmierung bei Abweichungen vom Sollwert.

2.  **Batch-Rückverfolgung via Blockchain-Anchor**  
    Jeder produzierte Kapselbatch erhält einen eindeutigen, unveränderlichen Hash, der in einer privaten Hyperledger-Fabric-Chain verankert wird. Über eine REST-API können Sie jederzeit den vollständigen Lebenszyklus einer Charge abrufen: von Rohstofflieferung über Abfüllzeitpunkt bis zur Auslieferung an den Kunden. Ideal für Audits nach ISO 22000 und GMP-Richtlinien.

3.  **Dynamische Kapazitätsplanung mit KI-Optimierung**  
    Ein auf Reinforcement Learning basierender Algorithmus analysiert historische Produktionsdaten und prognostiziert Engpässe bis zu 72 Stunden im Voraus. Das System schlägt automatisch optimale Rüstreihenfolgen vor und gleicht Maschinenauslastungen aus – ohne manuelle Eingriffe. Integration in SAP S/4HANA und Odoo via REST-Webhooks möglich.

## Detaillierte Anleitung (3 professionelle Schritte)

### Schritt 1: Maschinen-Connector einrichten und Telemetrie aktivieren

1.  **MQTT-Broker konfigurieren:** Installieren Sie den `capsule-factory-agent` auf Ihrem Edge-Device (Raspberry Pi, Industrial PC). Öffnen Sie die Konfigurationsdatei `/etc/capsule-agent/config.yml` und setzen Sie die Werte für `mqtt.broker.url` und `mqtt.topic.prefix` (z.B. `fabrik1/linieA/`).  
2.  **SPS-Mapping erstellen:** Nutzen Sie die integrierte `opcua-discover` CLI (Befehl: `capsule-cli discover --interface eth0`), um automatisch alle OPC-UA-Knoten Ihrer SPS zu scannen. Weisen Sie die Knoten-IDs den Standard-Metriken zu (z.B. `ns=2;s=Temperature` -> `metric:kapsel_temperatur`).  
3.  **Datenstrom testen:** Starten Sie den Agenten mit `systemctl start capsule-agent && journalctl -u capsule-agent -f`. Überprüfen Sie im Live-Monitor (Dashboard unter `https://<deine-instanz>/monitor`), ob die eingehenden Daten korrekt dekodiert werden. Ein grüner Haken neben der Maschinen-ID bestätigt die erfolgreiche Verbindung.

### Schritt 2: Batch-Produktion starten und Blockchain-Anchor setzen

1.  **Batch initialisieren:** Rufen Sie die REST-API auf:  
    `POST /api/v1/batches` mit JSON-Body:  
    `{"product_id": "KAPSEL-A-123", "target_quantity": 50000, "machine_id": "linieA_01", "raw_material_lot": "RM-2024-09"}`  
    Die API antwortet mit einer `batch_id` (z.B. `BATCH-9F2A`).  
2.  **Produktionsdaten streamen:** Während der Fertigung sendet der Agent automatisch alle 5 Sekunden einen Datenpunkt an den Endpunkt `/api/v1/telemetry`. Die Daten werden lokal in einer SQLite-Datenbank gepuffert, falls die Verbindung unterbrochen wird.  
3.  **Anchor finalisieren:** Nach Produktionsende rufen Sie auf:  
    `POST /api/v1/batches/{batch_id}/anchor`  
    Das System berechnet den SHA-256-Hash des gesamten Batch-Journals und schreibt ihn in die Blockchain. Die Transaktions-ID erhalten Sie als Response. Überprüfen Sie die Unveränderlichkeit mit dem Befehl:  
    `capsule-cli verify --batch BATCH-9F2A --chain hyperledger`

### Schritt 3: KI-Kapazitätsplanung aktivieren und manuelle Eingriffe minimieren

1.  **Modell trainieren (optional):** Wenn Sie bereits historische Daten haben, führen Sie aus:  
    `capsule-cli train --data-dir /var/lib/capsule/history --model-type rl`  
    Das Modell wird als `rl_model_v2.onnx` im Arbeitsverzeichnis gespeichert.  
2.  **Planungs-Engine starten:** Aktivieren Sie die Planung über die Admin-Oberfläche:  
    `Settings > Production Planning > Enable AI Scheduling`  
    Setzen Sie den `Optimization Horizon` auf `72h` und wählen Sie `Objective: Minimize Setup Time`.  
3.  **Vorschläge übernehmen:** Das System zeigt im Dashboard eine priorisierte Liste von Rüstreihenfolgen an. Klicken Sie auf **Apply Schedule**, um die Vorschläge als CSV zu exportieren oder direkt an Ihr MES-System zu senden (via Webhook: `Settings > Integrations > MES Webhook URL`). Der Status jeder Maschine wird automatisch auf „Geplant“ gesetzt.

## Best Practices für die Industrie (2 Empfehlungen)

1.  **Datenredundanz für Ausfallsicherheit**  
    Konfigurieren Sie mindestens zwei unabhängige MQTT-Broker (z.B. einen lokalen auf dem Edge-Device und einen cloudbasierten). Nutzen Sie die `capsule-agent`-Option `--broker-failover` um automatisch umzuschalten. So stellen Sie sicher, dass auch bei einem Netzwerkausfall keine Produktionsdaten verloren gehen – kritisch für die Einhaltung von FDA- und EU-Richtlinien zur Chargenrückverfolgung.

2.  **Rollierende Datenbereinigung mit Retention-Policies**  
    Setzen Sie in Ihrer InfluxDB eine automatische Retention Policy (z.B. `RETENTION 90d` für Rohdaten, `RETENTION 365d` für aggregierte Stundendaten). Nutzen Sie die integrierte Funktion `capsule-cli compress --older-than 30d`, um alte Batch-Journale in ein komprimiertes Parquet-Format zu konvertieren. Dies reduziert Speicherkosten um bis zu 70%, ohne die Audit-Fähigkeit zu beeinträchtigen.
