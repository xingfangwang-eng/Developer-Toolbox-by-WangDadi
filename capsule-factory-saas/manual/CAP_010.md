# capsule-factory-saas - CAP_010

## Overview

This is a high-performance, cloud-native SaaS platform purpose-built for industrial capsule manufacturing operations. Targeting production managers, quality engineers, and plant IT teams, capsule-factory-saas delivers real-time batch traceability, predictive maintenance triggers, and multi-tenant isolation for pharmaceutical and nutraceutical factories. The platform is fully API-first and optimized for CI/CD pipelines in modern DevOps environments.

## What is capsule-factory-saas?

capsule-factory-saas is a zero-downtime, event-driven production management system that replaces legacy spreadsheets and on-premise ERP modules. It orchestrates capsule filling, encapsulation, polishing, and inspection workflows across distributed factory lines, with built-in compliance for FDA 21 CFR Part 11 and EU GMP Annex 11.

## Core Features

### 1. Real-Time Batch Genealogy with Immutable Ledger
Every capsule batch is tracked through a tamper-proof audit trail, linking raw material lots, machine parameters, operator actions, and environmental sensor data. The system automatically generates a digital batch record (DBR) that can be exported for regulatory submission without manual intervention.

### 2. Predictive Anomaly Detection via ML-Driven Sensor Fusion
Ingest live data from PLCs, temperature/humidity sensors, and fill-weight scales. The built-in anomaly detection engine flags deviations in capsule weight uniformity, dissolution rate, or seal integrity before they exceed tolerance thresholds, reducing scrap rates by up to 34% in controlled trials.

### 3. Multi-Tenant Workflow Orchestration with Role-Based Access
Each factory tenant gets isolated namespaces with custom SOP workflows, approval chains, and notification rules. Granular RBAC ensures that only authorized personnel can modify recipe parameters, release batches, or override quality holds.

## How to Use

### 1. Connect Your Factory Line via OPC UA or MQTT Bridge
Deploy the lightweight edge agent on your plant gateway (Windows/Linux or Docker). Configure the OPC UA endpoint or MQTT broker URL in the `capsule-factory-saas` admin panel. The agent auto-discovers PLC tags and maps them to predefined capsule production metrics (fill weight, tamping force, capsule body temperature). Validate the connection using the live dashboard—latency should be under 50ms for real-time monitoring.

### 2. Define a Batch Recipe and Trigger Production
Navigate to the `Recipes` module and create a new recipe: set target fill weight (e.g., 500mg ± 2%), encapsulation speed (e.g., 1200 caps/min), and material lot IDs. Assign the recipe to a specific machine line. Click `Start Batch`—the system will lock the recipe parameters, initiate sensor logging, and push the digital batch record to the immutable ledger. Operators see live feedback on HMI panels via WebSocket push.

### 3. Run a Post-Batch Compliance Check and Release
After the batch completes, the system automatically runs a compliance check against predefined quality gates (weight variation, dissolution, microbial limits). If all gates pass, the batch status changes to `Pending Release`. Authorized QA personnel can review the digital batch record, sign off with a digital signature (FIPS 140-2 compliant), and trigger an automated email notification to downstream logistics. The batch is now ready for shipment with full traceability.

## Best Practices

### 1. Implement a "Gold Batch" Baseline for Anomaly Detection
Before deploying predictive anomaly detection in production, run a "gold batch" under optimal conditions. Capture sensor data for a full run (e.g., 30 minutes at 1200 caps/min). Use this baseline to calibrate the ML model—this reduces false positives during initial deployment and accelerates operator trust in the system.

### 2. Enforce a Strict Separation of Duties via RBAC
To maintain audit readiness, assign distinct roles: `Recipe Author` can create/modify recipes but cannot start a batch; `Production Operator` can start/stop batches but cannot edit recipes; `QA Reviewer` can only view and sign off batches. This prevents unauthorized parameter changes and ensures every batch release has a clear chain of custody.
