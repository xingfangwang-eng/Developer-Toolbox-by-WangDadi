# focusinbox-eight - FOC_016

## Übersicht

**focusinbox-eight** ist eine hochperformante, auf **Echtzeit-Postfach-Priorisierung** spezialisierte Middleware-Engine. Das System analysiert eingehende Nachrichtenströme mittels **Multi-Embedding-Vektorisierung** und **Bayesianischer Entscheidungsbäume**, um die dringendsten Kommunikationsereignisse für den Nutzer zu identifizieren. Die Zielsprache ist **Deutsch (de)**, die gesamte Verarbeitung erfolgt **latenzoptimiert** auf Sub-100-Millisekunden-Ebene.

## Detaillierter Inhalt

### Was ist focusinbox-eight?

focusinbox-eight ist eine **ereignisgesteuerte, microservice-basierte** Softwarelösung, die das Problem der **Informationsüberflutung** in geschäftlichen Posteingängen löst. Statt einer simplen Sortierung nach Absender oder Betreff, verwendet das Tool eine **dreistufige semantische Relevanzanalyse**: Zunächst wird der **kontextuelle Embedding-Vektor** jeder Nachricht mit dem Benutzerprofil (basierend auf historischen Interaktionen) abgeglichen. Anschließend wird die **zeitliche Dringlichkeit** mittels eines **exponentiell gewichteten gleitenden Mittelwerts (EWMA)** der Antwortzeiten berechnet. Die finale Priorisierung erfolgt durch einen **Random-Forest-Klassifikator**, der auf über 10.000 annotierten Geschäftskommunikationen trainiert wurde.

### Kernfunktionen

- **Multi-Embedding-Vektorisierung mit PostgreSQL-Integration**  
  Jede eingehende E-Mail wird in einen **384-dimensionalen Vektor** (basierend auf Sentence-BERT) umgewandelt. Dieser Vektor wird direkt in einer **PostgreSQL-Datenbank mit pgvector-Erweiterung** gespeichert und indiziert. Die Ähnlichkeitssuche (Cosine Similarity) zwischen neuem Input und historischen Profilen erfolgt in **< 50 ms** bei einer Datenbankgröße von bis zu 1 Million Einträgen.

- **Bayesianischer Entscheidungsbaum für Dringlichkeitsprognose**  
  Das System verwendet einen **online-lernenden Bayesianischen Klassifikator**, der kontinuierlich die **Wahrscheinlichkeit einer Antwort innerhalb von 60 Minuten** berechnet. Parameter sind unter anderem: `response_time_history`, `sender_reputation_score` (basierend auf einer **gewichteten Feedback-Matrix**) und `thread_depth`. Die Ausgabe ist ein **Dringlichkeits-Score zwischen 0.0 und 1.0**, der in Echtzeit aktualisiert wird.

- **Automatische Kategorisierung mit dynamischem Regelwerk**  
  focusinbox-eight unterstützt ein **regelbasiertes System mit Prioritätsstufen (1-5)**. Regeln können über eine **YAML-basierte Konfigurationsdatei** definiert werden. Beispiel: `priority_5: if "subject" contains "Sprint-Review" AND "sender" in ["team-lead@corp.com"] then set `alert_level` to "critical"`. Diese Regeln werden **asynchron** durch einen **Dedicated Worker-Thread** evaluiert, der eine **maximale Latenz von 10 ms** garantiert.

### So verwenden Sie es

1. **Initiale Profilierung und Datenbank-Setup**  
   Führen Sie den Befehl `focusinbox-eight --init-db --profile-type=executive` aus. Das Tool erstellt automatisch eine **PostgreSQL-Datenbank** (`focusinbox_db`) mit den erforderlichen Tabellen (`inbox_vectors`, `user_profiles`, `response_history`). Anschließend wird ein **initialer Embedding-Vektor** aus den letzten 500 E-Mails Ihres Posteingangs generiert. Nutzen Sie den Parameter `--embedding-model=paraphrase-multilingual-MiniLM-L12-v2`, um die **deutsche Sprachunterstützung** zu aktivieren.

2. **Echtzeit-Überwachung und Priorisierung starten**  
   Starten Sie den Daemon mit `focusinbox-eight daemon --port=8080 --poll-interval=5000`. Der Daemon überwacht Ihren IMAP-Posteingang im **5-Sekunden-Intervall**. Jede neue E-Mail wird in der Pipeline verarbeitet: **Vektorisierung (30 ms) -> Ähnlichkeitssuche (50 ms) -> Bayesianische Prognose (20 ms) -> Regelauswertung (10 ms)**. Die priorisierte Liste wird als **JSON-Stream** an `http://localhost:8080/stream` ausgegeben. Verwenden Sie `curl -N http://localhost:8080/stream`, um die Live-Daten zu sehen.

3. **Erweiterte Konfiguration und benutzerdefinierte Regeln**  
   Erstellen Sie eine Datei `custom_rules.yaml` im Verzeichnis `/etc/focusinbox-eight/`. Definieren Sie eine Regel wie folgt:  
   ```yaml
   rules:
     - name: "HighPriorityClient"
       condition: "sender_domain == 'client.com' AND thread_depth > 3"
       action: "set_priority(1) AND notify(slack_webhook)"
       ttl: 3600
   ```  
   Laden Sie die Regeln mit `focusinbox-eight reload-rules` neu. Das System validiert die YAML-Struktur und aktiviert die Regeln **ohne Neustart**.

### Beste Praktiken

- **Batch-Indexierung für historische Daten**  
  Bei der Erstinstallation sollten Sie nicht den gesamten Posteingang in Echtzeit verarbeiten. Nutzen Sie stattdessen den Befehl `focusinbox-eight batch-index --source=mbox --file=archive.mbox`. Dies erstellt einen **vorberechneten Embedding-Index** in der Datenbank, der die **initialen Abfragen um das 10-fache beschleunigt**. Planen Sie dieses Batch-Job für **Nebenzeiten** (z. B. 02:00 Uhr) über den integrierten **Cron-Wrapper** (`focusinbox-eight cron --schedule="0 2 * * *"`).

- **Feedback-Loop für Modellverbesserung**  
  Implementieren Sie einen **manuellen Feedback-Mechanismus** in Ihrer UI. Jede vom Nutzer als „falsch priorisiert“ markierte E-Mail sollte über die API `POST /api/feedback` mit den Parametern `{email_id, correct_priority, reason}` an das System gesendet werden. focusinbox-eight speichert diese Daten in einer separaten Tabelle (`feedback_log`) und verwendet sie für ein **wöchentliches automatisches Retraining** des Bayesianischen Modells. Führen Sie hierfür `focusinbox-eight retrain --model=urgency --epochs=5` aus.
