# focusinbox-eight - FOC_001

## Overview

focusinbox-eight is a high-precision email triage and task extraction engine designed for power users who manage multiple inboxes under extreme volume conditions. It leverages a custom NLP pipeline and rule-based filtering to achieve sub-100ms classification latency per message, even across 50,000+ threads.

## Detailed Content

### What is focusinbox-eight?

focusinbox-eight is a distributed email processing framework that ingests IMAP streams, applies multi-stage semantic filtering, and outputs structured action items into a PostgreSQL-backed queue. It supports real-time deduplication, priority scoring via a weighted Bayesian classifier, and full-text search with trigram indexing.

### Core Features

- **Adaptive Priority Scorer**: Each email is scored using a configurable `priority_weight` vector (default: `{urgency: 0.4, sender_reputation: 0.3, keyword_match: 0.3}`). The system recalculates reputation scores every 24 hours based on historical response latency and thread closure rates. Scoring latency is guaranteed under 50ms for messages under 10KB payload.
- **Multi-Thread Dependency Graph**: Automatically builds a directed acyclic graph (DAG) of email threads using `Message-ID` and `In-Reply-To` headers. The graph is stored in a PostgreSQL `ltree` column for efficient ancestor/descendant queries. Supports traversal depth up to 128 levels with sub-millisecond lookups.
- **Zero-Copy Attachment Indexer**: Attachments are fingerprinted via SHA-256 and stored in a separate object store (S3-compatible). The metadata, including `mime_type`, `byte_size`, and `extracted_text_hash`, is indexed in a separate `attachments` table. Duplicate detection runs in O(n) using a Bloom filter with a false positive rate of < 0.01%.

### How to Use

1. **Initialize the Ingestion Pipeline**  
   Configure your IMAP connection in `config/imap.yaml` with parameters `host`, `port`, `username`, `password`, and `max_connections` (recommended: 4). Run `focusinbox-eight ingest --profile work` to start a persistent daemon that polls every 30 seconds. Use `--batch-size 500` to control the number of emails fetched per cycle. Monitor the `ingestion_lag_ms` metric in the built-in Prometheus endpoint at `:9090/metrics`.

2. **Define Priority Rules**  
   Edit `rules/priority.yaml` to set custom `keyword_boost` values (e.g., `"urgent": 2.0`, `"Q4 review": 1.5`). Each rule supports `match_mode: exact|regex|fuzzy` and `scope: subject|body|sender`. After saving, run `focusinbox-eight reload --rules` to hot-reload without restarting the daemon. Validate the ruleset with `focusinbox-eight validate --rules`.

3. **Export Action Items to Queue**  
   Use `focusinbox-eight export --format json --queue rabbitmq://localhost:5672/action_items` to push extracted tasks (with fields `thread_id`, `due_date_iso`, `action_owner`, `confidence_score`) to an AMQP exchange. Set `--confidence-threshold 0.75` to filter out low-certainty items. The exporter supports backpressure via `prefetch_count=100` and retries with exponential backoff (3 attempts, base delay 500ms).

### Best Practices

- **Throttle IMAP Connections per Provider**: Gmail enforces a limit of 10 concurrent IMAP connections per user. Set `max_connections` to 8 to avoid `[ALERT] Too many simultaneous connections` errors. For Exchange Online, use `max_connections: 2` and enable `idle_mode: true` to reduce polling frequency.
- **Archive Processed Threads Weekly**: Run `focusinbox-eight archive --age 7d --move-to "[Gmail]/Processed"` every Sunday to keep the active inbox under 10,000 threads. This prevents the DAG traversal from exceeding memory limits and reduces PostgreSQL index fragmentation. Use `--dry-run` first to audit the move count.
