# focusinbox-eight - FOC_008

## 概要

focusinbox-eight は、受信トレイのノイズをミリ秒単位でフィルタリングし、真に重要なタスクのみに集中できる次世代型集中管理エンジンです。特に、複数チャネル（メール、Slack、GitHub Issue）の通知をリアルタイムで統合・優先順位付けし、ユーザーの認知負荷を劇的に低減します。

## 詳細な内容

### focusinbox-eight とは

focusinbox-eight は、**分散型アテンション管理プラットフォーム**です。従来の「未読ゼロ」アプローチではなく、**重要度スコアリングアルゴリズム**（`priority_score`）に基づき、緊急度と関連性を動的に評価。PostgreSQL 上の時系列データと連携し、過去の対応パターンから学習することで、個人のワークフローに最適化された「集中すべき1件」を提示します。

### コア機能

1. **マルチチャネルリアルタイム集約エンジン**
   - パラメータ: `channel_type`（email/slack/github）、`latency_threshold`（デフォルト: 500ms）
   - 各チャネルのWebhookを独自の**イベントバス**で受信し、重複排除と正規化を実施。PostgreSQLの`NOTIFY/LISTEN`機構を用いて、**50ms以内**に`priority_queue`テーブルへ格納。

2. **動的優先順位スコアリング**
   - パラメータ: `sender_weight`（0.0〜1.0）、`keyword_boost`（設定可能なキーワードリスト）、`response_time_history`（過去30日間の平均応答時間）
   - スコア計算式: `priority_score = (urgency_factor * 0.4) + (sender_importance * 0.3) + (keyword_match * 0.2) - (age_decay * 0.1)`
   - 結果は**ミリ秒単位**で更新され、`focusinbox-eight`ダッシュボードに即時反映。

3. **集中モード自動トリガー**
   - パラメータ: `focus_duration`（デフォルト: 25分）、`break_interval`（デフォルト: 5分）、`notification_blocklist`（特定送信者/チャネルの一時停止）
   - 集中モード開始時、全チャネルの通知を**論理的にブロック**し、`focus_session`テーブルにセッションログを記録。終了時に未処理アイテムのサマリーを生成。

### 使い方

1. **初期セットアップとチャネル統合**
   - プロジェクトルートで `./focusinbox-eight init` を実行。
   - 指示に従い、メール（IMAPサーバー設定）、Slack（OAuthトークン）、GitHub（Personal Access Token）を登録。
   - 設定ファイル `~/.focusinbox/config.yml` に `channel_type: all` と `sync_interval: 30s` が自動生成されることを確認。

2. **優先順位ルールのカスタマイズ**
   - `focusinbox-eight rules edit` でルールエディタを起動。
   - 例: `sender_weight: 0.9` を `client@example.com` に割り当て、`keyword_boost: ["urgent", "deploy"]` を設定。
   - 変更後、`focusinbox-eight rules apply` でPostgreSQL上の`priority_rules`テーブルを更新。即座にスコアリングに反映される。

3. **集中セッションの開始と分析**
   - `focusinbox-eight focus start --duration 45` を実行。全通知がブロックされ、`focus_session`テーブルにセッションIDが発行される。
   - 終了時、`focusinbox-eight focus report --session <ID>` で、ブロックした通知数、対応したタスク数、平均応答時間の変化を**ミリ秒精度**で出力。

### ベストプラクティス

- **ルールの段階的適用**: 初回は`keyword_boost`のみを設定し、1週間のログ（`response_time_history`）を蓄積してから`sender_weight`を調整する。急激なスコア変動を防ぎ、アルゴリズムの収束を安定させる。
- **集中モードの時間帯最適化**: 自身の生産性ピーク時間帯（例: 午前9時〜11時）に`focus_duration`を45分に設定し、`break_interval`を10分に短縮。非ピーク時は25分/5分に戻すことで、リズムを崩さずに持続可能な集中を維持。
