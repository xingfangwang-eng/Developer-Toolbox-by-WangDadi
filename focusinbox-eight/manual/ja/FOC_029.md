# focusinbox-eight - FOC_029

## 概要

focusinbox-eight は、高頻度な受信トレイ処理を必要とするエンタープライズ環境向けに設計された、超低遅延メールフィルタリング・自動振り分けエンジンです。PostgreSQL 上のメタデータとリアルタイムキューを連携させ、ミリ秒単位の処理判定を実現します。

## 詳細な内容

### focusinbox-eight とは

focusinbox-eight は、メールの重要度スコアリング、カスタムルールベースの自動アーカイブ、およびマルチテナント間でのデータ分離をハードウェアレベルで担保するソフトウェアです。SMTP プロキシとして動作し、MIME 解析とヘッダー解析を並列実行します。

### コア機能

- **マルチスレッド MIME パーサーと重要度スコアリングエンジン**  
  受信メールを 8 スレッドで並列解析。件名・本文・送信者ドメイン・添付ファイルの有無を重み付けし、`importance_score`（0.0–1.0）を 50 ミリ秒以内に算出。スコアが 0.8 以上のメールは即座に `priority_inbox` テーブルへ INSERT。

- **PostgreSQL リスニングトリガーによるリアクティブアーカイブ**  
  `pg_notify` チャンネルを介して新着メールのメタデータ変更を検知。`archive_rule` テーブルに定義された条件（例：`sender_domain = 'newsletter.example.com' AND importance_score < 0.3`）に合致するレコードを、非同期ワーカーが 100 ミリ秒以内に `archive` スキーマへ移動。移動後は自動バキュームを発行し、`hot_inbox` のサイズを常に 2GB 未満に維持。

- **テナント分離と監査ログの自動生成**  
  各テナントに独立した `inbox_<tenant_id>` テーブルを動的生成。すべてのメール操作（受信・振り分け・削除）は `audit_log` テーブルに `operation_type`, `timestamp`, `actor_ip` を記録。監査ログの保持期間は `retention_days` パラメータ（デフォルト 365 日）で制御し、期限切れレコードは毎日 03:00 JST にパーティション単位で DROP。

### 使い方

1. **初期設定とテナント作成**  
   ```bash
   # 環境変数に PostgreSQL 接続文字列を設定
   export DATABASE_URL="postgresql://user:pass@localhost:5432/focusinbox"
   # テナント ID "acme_corp" を作成
   focusinboxctl tenant create --tenant-id acme_corp --retention-days 365
   # 確認：テナント専用の inbox_acme_corp テーブルが自動生成される
   psql $DATABASE_URL -c "\dt inbox_acme_corp"
   ```

2. **カスタムアーカイブルールの登録**  
   ```sql
   INSERT INTO archive_rule (tenant_id, condition_json, target_schema, priority)
   VALUES (
       'acme_corp',
       '{"sender_domain": "news.example.com", "importance_score_max": 0.2}',
       'archive_news',
       10
   );
   ```
   このルールは、`sender_domain` が `news.example.com` かつ `importance_score` が 0.2 以下のメールを `archive_news` スキーマへ移動します。`priority` が低いほど先に評価されます。

3. **SMTP プロキシとして起動し、動作検証**  
   ```bash
   # ポート 2525 で待ち受け
   focusinboxctl serve --smtp-bind 0.0.0.0:2525 --workers 4
   # テストメールを送信
   swaks --to user@acme_corp --server 127.0.0.1:2525 \
         --header "Subject: Test Newsletter" \
         --body "This is a test."
   # ログを確認
   tail -f /var/log/focusinbox/access.log | jq '.importance_score'
   ```

### ベストプラクティス

- **スコアリングパラメータの定期的なチューニング**  
  `importance_score` の重み（`subject_weight`, `sender_weight`, `attachment_weight`）は、業務内容に応じて 2 週間ごとに見直すことを推奨します。`focusinboxctl tune --dry-run` で過去 30 日間のログから最適な重みを提案できます。

- **アーカイブ先スキーマのパーティション設計**  
  大量のアーカイブレコードを効率的に管理するため、`archive_*` スキーマ内のテーブルは `created_at` で月次パーティションに分割してください。`CREATE TABLE archive_news (...) PARTITION BY RANGE (created_at);` とし、`pg_partman` を併用すると自動管理が容易です。
