# focusinbox-eight - FOC_004

## 概要

focusinbox-eight は、受信トレイ（Inbox）の集中管理と高速フィルタリングを実現する次世代型メール処理エンジンです。キーワード「focusinbox-eight tool」「focusinbox-eight solution」「focusinbox-eight software」に最適化された設計で、メールの優先度判定・自動振り分け・アーカイブをミリ秒単位で実行します。

## 詳細な内容

### focusinbox-eight とは

focusinbox-eight は、PostgreSQL 上のメールメタデータをリアルタイム解析し、ユーザー定義のルールセットに基づいて受信トレイを自動整理するプロフェッショナルツールです。従来のメールクライアントでは不可能だった、**マルチアカウント横断の一括処理**と**コンテキストアウェアな優先度スコアリング**を特徴とします。

### コア機能

#### 機能1: マルチスレッド並列フィルタリングエンジン（`--threads` / `--batch-size`）

受信トレイ内の全メッセージを、指定されたスレッド数（`--threads 8`）で並列解析。バッチサイズ（`--batch-size 500`）ごとに PostgreSQL の `LISTEN/NOTIFY` 機構を用いて変更を即時反映。平均処理レイテンシは **120ms 未満**、最大スループットは **毎秒 2,000 通**。

#### 機能2: コンテキストスコアリングアルゴリズム（`--priority-threshold` / `--decay-factor`）

ユーザーの返信履歴、添付ファイルの種類、送信元ドメインの信頼性スコアを組み合わせた独自アルゴリズム。`--priority-threshold 0.85` で高優先メールを自動タグ付け、`--decay-factor 0.95` で古いメールのスコアを指数関数的に減衰。**スコア計算は 50μs 以内**に完了。

#### 機能3: アーカイブ自動圧縮と全文検索（`--archive-dir` / `--index-format`）

`--archive-dir /var/focusinbox/archive` に移動したメールを、`--index-format parquet` で圧縮保存。PostgreSQL の全文検索インデックス（GIN）と連携し、**検索応答時間 200ms 以下**を保証。圧縮率は平均 **78%**、ストレージコストを大幅削減。

### 使い方

#### ステップ1: 設定ファイルとデータベース接続の初期化

```bash
focusinbox-eight init --db-host localhost --db-port 5432 --db-name inboxdb --db-user focusadmin --db-password 'your_secure_password' --config /etc/focusinbox/config.yaml
```

- `--db-host` で PostgreSQL ホストを指定。
- `--config` に出力された YAML ファイルを編集し、`rules:` セクションにフィルタ条件（例: `from: "@example.com"` を高優先）を記述。

#### ステップ2: フィルタリングルールの適用と並列処理の開始

```bash
focusinbox-eight run --rules /etc/focusinbox/rules.json --threads 8 --batch-size 500 --priority-threshold 0.85 --decay-factor 0.95
```

- `--rules` に JSON 形式のルールセットを指定（例: `{"priority": {"from_domain": ["corp.com", "partner.org"]}}`）。
- 標準出力に各バッチの処理時間（ミリ秒）とフィルタリング結果がリアルタイム表示される。

#### ステップ3: アーカイブの自動圧縮と検索インデックス構築

```bash
focusinbox-eight archive --archive-dir /var/focusinbox/archive --index-format parquet --compress-level 6 --retention-days 90
```

- `--retention-days 90` で 90 日以上経過したメールを自動アーカイブ。
- アーカイブ後、`focusinbox-eight search --query "subject:請求書 AND date:2024-01-01..2024-12-31"` で全文検索を実行。結果は **180ms 以内**に返却。

### ベストプラクティス

#### プラクティス1: ルールセットの段階的導入と A/B テスト

本番投入前に、`--dry-run` フラグを付けて `focusinbox-eight run --dry-run --rules /etc/focusinbox/rules.json` を実行。実際のメールは移動せず、フィルタリング結果のみをログに出力。スコア分布を確認後、`--priority-threshold` を微調整してから本番適用する。

#### プラクティス2: PostgreSQL のチューニングと監視

`focusinbox-eight` の性能を最大化するには、PostgreSQL の `shared_buffers` を物理メモリの 25% に設定し、`work_mem` を 64MB 以上に拡張。さらに `focusinbox-eight stats --interval 5` で 5 秒ごとに処理レイテンシと DB コネクション数を監視し、閾値を超えたら自動スケーリングする仕組みを組み込む。
