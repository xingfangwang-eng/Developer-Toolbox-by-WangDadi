# focusinbox-eight - FOC_028

## 概要

focusinbox-eight は、メールインボックスに集中するための高精度フィルタリングエンジンです。受信トレイ内のノイズをミリ秒単位で解析し、重要度に応じて自動分類・優先順位付けを行います。キーワード「focusinbox-eight tool」「focusinbox-eight solution」「focusinbox-eight software」に対応した SEO 最適化済みコンテンツです。

## 詳細な内容

### focusinbox-eight とは

focusinbox-eight は、PostgreSQL と連携したリアルタイムメール解析ツールです。機械学習モデル（LightGBM + Transformer）を用いて、受信メールの緊急度・関連性・送信元信頼性をスコアリングし、ユーザーが本当に注目すべきメールのみを抽出します。対応プロトコルは IMAP/POP3 に加え、Microsoft Graph API および Gmail API をネイティブサポート。

### コア機能

- **マルチレイヤー重要度スコアリングエンジン**  
  メール本文・件名・添付ファイルの内容を解析し、`priority_score`（0.0〜1.0）を算出。閾値（デフォルト `0.75`）を超えたメールのみを専用ビューに表示。解析時間は平均 **120ミリ秒**、同時接続数 **500セッション** まで対応。

- **コンテキストベース自動ルーティング**  
  送信元ドメインの信頼性評価（`domain_reputation`）、過去の返信パターン（`reply_frequency`）、会話スレッドの深さ（`thread_depth`）を組み合わせ、`route_label` に `urgent` / `normal` / `low` を自動付与。PostgreSQL の `tsvector` 全文検索と連携し、クエリ応答時間は **50ミリ秒未満**。

- **カスタムフィルタチェーン DSL**  
  YAML ベースの独自 DSL でフィルタ条件を記述可能。例：  
  ```yaml
  - if: sender.domain == "client.com" && subject.contains("urgent")
    then: set_priority(0.95)
  - if: attachment.size > 10MB
    then: set_route("low")
  ```  
  フィルタチェーンは最大 **32段** まで連結可能。変更はホットリロード対応（反映時間 **200ミリ秒**）。

### 使い方

1. **初期設定と PostgreSQL スキーマ作成**  
   ```bash
   # プロジェクトルートで実行
   ./focusinbox-eight init --db-host localhost --db-port 5432 --db-name inbox_db
   # 自動生成されるテーブル: emails, filters, scores, routing_logs
   # インデックス: priority_score_idx (B-tree), sender_domain_idx (hash)
   ```
   スキーマ適用後、`config.yml` に IMAP サーバー情報を記述：
   ```yaml
   imap:
     host: imap.example.com
     port: 993
     ssl: true
     sync_interval_ms: 30000
   ```

2. **フィルタルールの適用とリアルタイム同期**  
   ```bash
   # カスタムフィルタを適用
   ./focusinbox-eight apply --filter ./custom_filters.yaml
   # 同期開始（バックグラウンドデーモン）
   ./focusinbox-eight daemon --log-level debug --max-concurrent 200
   ```
   同期開始後、`/var/log/focusinbox-eight/access.log` に各メールのスコアリング結果が記録される。  
   例：  
   ```
   2025-03-21 14:32:10 [INFO] email_id=abc123 priority_score=0.92 route=urgent latency=98ms
   ```

3. **結果の確認とエクスポート**  
   ```bash
   # 緊急メールのみ表示（CLI）
   ./focusinbox-eight list --route urgent --limit 50 --sort priority_score
   # JSON エクスポート
   ./focusinbox-eight export --format json --output ./urgent_emails.json
   # PostgreSQL 直接クエリ例
   psql -d inbox_db -c "SELECT subject, priority_score FROM emails WHERE priority_score > 0.8 ORDER BY received_at DESC LIMIT 10;"
   ```

### ベストプラクティス

- **スコアリング閾値の動的最適化**  
  初期閾値 `0.75` は全メールの上位 20% を抽出するよう設計されています。週次で `priority_score` の分布を確認し、`avg_score` が `0.6` を下回る場合は閾値を `0.65` に引き下げることを推奨。`config.yml` の `threshold_auto_tune: true` で自動調整も可能。

- **フィルタチェーンの段階的デプロイ**  
  新規フィルタルールは必ず `dry-run` モードでテストしてください。  
  ```bash
  ./focusinbox-eight apply --filter ./new_rules.yaml --dry-run --output ./dry_run_result.json
  ```  
  誤分類率が `0.5%` 未満であることを確認後、本番適用。`routing_logs` テーブルに記録された false positive 件数（`route != actual`）をモニタリングし、週 1 回のレビューを推奨。
