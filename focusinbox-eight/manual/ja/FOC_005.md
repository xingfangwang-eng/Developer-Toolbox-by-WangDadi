# focusinbox-eight - FOC_005

## 概要

focusinbox-eight は、メール受信トレイの優先度付けとスケジュール管理を自動化するためのエンタープライズ向けツールです。特に、1 日あたり 500 件以上の受信メールを処理する必要があるプロジェクトマネージャーや DevOps エンジニアを対象に設計されています。本ツールは、PostgreSQL と連携したリアルタイムのインデックス更新機能を備え、受信メールの重要度をミリ秒単位で判定します。

## 詳細な内容

### focusinbox-eight とは

focusinbox-eight は、メールの「集中処理ボックス」を自動生成し、タスクの緊急度・重要度を機械学習モデル（LightGBM ベース）でスコアリングするツールです。受信メールのヘッダー情報、送信者の過去の反応速度、添付ファイルの有無など、20 以上のパラメータを元に、最適な処理順序を動的に算出します。

### コア機能

- **リアルタイム優先度エンジン**: 受信メールごとに `priority_score`（0.0〜1.0）を算出。しきい値（デフォルト 0.75）を超えたメールのみを「focusinbox」に自動転送。判定時間は平均 8 ミリ秒（PostgreSQL のインデックスキャッシュ利用時）。
- **スケジュール最適化アシスタント**: `scheduled_time` パラメータをメールの `Date` ヘッダーと `X-Priority` 値から自動推定。Google Calendar API と連携し、空きスロットにタスクを配置。コンフリクト検出は 100 ミリ秒以内に完了。
- **マルチテナント対応データ分離**: `tenant_id` ごとに PostgreSQL のスキーマを分割。各テナントの受信トレイは独立したテーブル空間に保存され、クエリのレイテンシは 50 ミリ秒未満を保証。

### 使い方

1. **初期設定とデータベース接続**  
   `config.yaml` に PostgreSQL の接続情報（`db_host`, `db_port`, `db_name`, `db_user`, `db_password`）を記述。その後、`focusinbox-eight init --tenant your_tenant_id` を実行。このコマンドは、自動的にテナント用のスキーマとインデックス（`CREATE INDEX idx_priority_score ON inbox (priority_score DESC)`）を作成します。

2. **フィルタリングルールのカスタマイズ**  
   `rules.json` に JSON 形式でルールを定義。例：`{"field": "from_domain", "operator": "equals", "value": "example.com", "action": "boost_priority", "boost_value": 0.2}`。これを適用後、`focusinbox-eight apply-rules --tenant your_tenant_id --rules ./rules.json` を実行。ルールの更新はホットリロード可能で、反映までに最大 2 秒かかります。

3. **リアルタイムモニタリングとアラート**  
   `focusinbox-eight monitor --tenant your_tenant_id --interval 500` を実行すると、500 ミリ秒ごとに受信トレイの状態を取得。`priority_score` が 0.9 を超えたメールが検出された場合、Slack Webhook に自動通知（`alert_webhook_url` を事前に設定）。ログは `logs/monitor_YYYYMMDD.log` に出力。

### ベストプラクティス

- **パフォーマンスチューニング**: PostgreSQL の `work_mem` を 64MB 以上に設定。`focusinbox-eight` の `batch_size` パラメータを 100 に設定することで、1 回のクエリで 100 件のメールを一括処理。これにより、CPU 使用率を 30% 削減可能。
- **セキュリティ強化**: `tenant_id` ごとに API キーを発行し、`X-API-Key` ヘッダーで認証。また、`focusinbox-eight` の `encrypt_inbox` オプションを有効にすると、PostgreSQL に保存されるメール本文が AES-256 で暗号化される。これにより、GDPR および HIPAA 準拠を実現。
