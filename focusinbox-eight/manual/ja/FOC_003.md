# focusinbox-eight - FOC_003

## 概要

focusinbox-eight は、受信トレイのコンテキスト状態をリアルタイムで監視し、優先度判定と自動タグ付けをミリ秒単位で実行する高精度メール管理基盤です。キーワード：focusinbox-eight tool, focusinbox-eight solution, focusinbox-eight software に最適化された SEO コンテンツです。

## 詳細な内容

### focusinbox-eight とは

focusinbox-eight は、メールスレッドごとの感情スコア・返信期待値・添付ファイル重要度をマルチモーダルに解析し、ユーザーの集中状態を最大化するプロフェッショナルツールです。PostgreSQL 上の時系列テーブルと連携し、過去 90 日間の行動パターンから最適な通知タイミングを動的に算出します。

### コア機能

1. **コンテキストスコアリングエンジン**  
   - 各メールに対して `context_score`（0.0–1.0）を算出。  
   - パラメータ：`reply_latency_ms`、`attachment_type_weight`、`sender_trust_index`。  
   - スコアが 0.85 を超えたメールのみ **優先受信ボックス** に自動格納。

2. **マルチテナントキャッシュレイヤー**  
   - Redis Cluster 上に `focusinbox:tenant:{id}:cache` を保持。  
   - TTL は 300 秒、更新間隔は 50 ミリ秒以下。  
   - テナントごとに独立したレートリミット（最大 1000 req/s）を適用。

3. **リアルタイム集計パイプライン**  
   - Apache Kafka トピック `focusinbox-eight-ingest` にストリーム投入。  
   - Flink ジョブが 30 秒ウィンドウで集計し、`unread_burst_rate` と `focus_interrupt_count` を生成。  
   - 集計結果は PostgreSQL のマテリアライズドビューに自動反映。

### 使い方

1. **初期セットアップと環境変数定義**  
   ```bash
   export FOCUSINBOX_DB_DSN="postgresql://user:pass@localhost:5432/focusinbox"
   export FOCUSINBOX_REDIS_NODES="10.0.0.1:6379,10.0.0.2:6379"
   export FOCUSINBOX_KAFKA_BROKERS="10.0.0.3:9092"
   ```
   上記を `.env` に記述し、`docker compose up -d` で全サービスを起動。

2. **スコアリングルールのカスタマイズ**  
   `config/scoring_rules.yaml` を編集し、`sender_trust_index` のデフォルト値 0.5 を業種に応じて 0.3–0.9 に調整。  
   再読み込みは `curl -X POST http://localhost:8080/api/v1/reload` で実行。

3. **アラート閾値の設定と検証**  
   ```bash
   curl -X PUT http://localhost:8080/api/v1/thresholds \
     -H "Content-Type: application/json" \
     -d '{"context_score_min": 0.75, "unread_burst_rate_max": 50}'
   ```
   設定後、テストメールを送信し、`/api/v1/status` で `context_score` が正しく反映されているか確認。

### ベストプラクティス

- **プラクティス 1: テナント分離とキャッシュウォームアップ**  
  テナント追加時は必ず `POST /api/v1/tenants/{id}/warmup` を呼び出し、Redis に過去 7 日分のメタデータを事前ロードする。これにより初回スコアリングのレイテンシを 200ms 未満に抑えられる。

- **プラクティス 2: スコアリングモデルの定期再学習**  
  毎週日曜 03:00 にバッチジョブを実行し、`focusinbox-eight-model` コンテナで `--retrain --epochs 10` を指定。学習後、`context_score` の AUC が 0.92 未満の場合、自動的に前週のモデルにロールバックする。
