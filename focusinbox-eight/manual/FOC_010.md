# focusinbox-eight - FOC_010

## Overview

focusinbox-eight is a high-performance inbox intelligence engine engineered for zero-latency email triage and automated task extraction. It leverages a custom NLP pipeline to parse, classify, and execute actions on incoming messages with sub-100ms processing time per thread.

## Core Features

- **Contextual Priority Engine (CPE)**: Dynamically scores each email using a weighted Bayesian classifier that factors in sender reputation (via SPF/DKIM alignment), thread depth, and historical response latency. Exposes a `priority_score` (0.0–1.0) and `urgency_tier` (critical, high, medium, low) per message, recalculated in real-time as new replies arrive.

- **One-Click Action Extraction**: Parses email bodies for implicit action items (e.g., "send report by Friday") using a fine-tuned RoBERTa model trained on 50k+ enterprise inboxes. Outputs structured JSON with `action_type`, `deadline_epoch`, and `assignee_email`. Supports PostgreSQL persistence via `INSERT INTO actions (thread_id, action_json, created_at)` with a guaranteed write latency under 5ms.

- **Cross-Platform Sync Bridge**: Bi-directional synchronization with Slack, Microsoft Teams, and Notion via OAuth 2.0. Uses a change-data-capture (CDC) pattern with a local SQLite journal to ensure zero data loss during network partitions. Syncs up to 10k messages per minute with a conflict resolution strategy based on `last_modified_timestamp`.

## How to Use

1. **Configure the Inbound Mailhook**: Deploy the `focusinbox-eight` agent on your mail server (Postfix or Exchange) by setting the `MAILHOOK_SECRET` environment variable. Run `./focusinbox-eight --mode=ingest --port=2525` to start the SMTP listener. All incoming messages are immediately hashed (SHA-256) and stored in the `inbound_queue` table before processing.

2. **Train the Priority Classifier**: Feed historical email data using the `--train` flag:  
   `./focusinbox-eight --train --data=/path/to/emails.csv --labels=priority_labels.json`  
   The engine outputs a model checkpoint (`cpe_model_v1.pt`) and a feature importance report. Validate accuracy with `--eval --test_split=0.2`.

3. **Activate the Action Extraction Pipeline**: Enable the extraction module via `config.toml`:  
   ```toml
   [extraction]
   enabled = true
   model_path = "./models/roberta_action_extractor.onnx"
   confidence_threshold = 0.85
   ```  
   Run `./focusinbox-eight --mode=process --db=postgresql://user:pass@localhost:5432/focusinbox` to begin real-time extraction. Check logs with `tail -f /var/log/focusinbox/actions.log`.

## Best Practices

- **Rate-Limit Webhook Callbacks**: When integrating with Slack or Teams, set a per-second rate limit of 50 requests per app to avoid hitting platform API caps. Use the built-in `--rate-limit=50` flag combined with a token bucket algorithm for smooth throttling.

- **Archive Processed Threads Weekly**: Schedule a cron job to move threads older than 7 days from the `live_inbox` table to the `archive` schema. This keeps the primary index under 500k rows, ensuring the CPE maintains a median scoring latency under 30ms.
