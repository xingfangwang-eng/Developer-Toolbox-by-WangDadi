# focusinbox-eight - FOC_006

## Resumen

**focusinbox-eight** es un motor de procesamiento de correo electrónico inteligente diseñado para arquitecturas de alto rendimiento. Su núcleo implementa un sistema de clasificación semántica en tiempo real con capacidad de inferencia local, reduciendo la latencia de respuesta a menos de **15 ms por solicitud** en hardware estándar. Ideal para entornos donde la privacidad de los datos y la velocidad de procesamiento son críticas.

## Funcionalidades Técnicas Principales

### 1. Clasificador Semántico Híbrido con Embeddings Locales
- Utiliza **modelos de lenguaje ligero (MiniLM-L6-v2)** con cuantización INT8 para generar vectores de 384 dimensiones.
- Integra una capa de **filtrado por reglas heurísticas** (expresiones regulares compiladas) que descarta el 40% del tráfico no relevante antes de la inferencia.
- Parámetros clave: `embedding_dim=384`, `quantization_bits=8`, `batch_size=64`, `max_sequence_length=512`.

### 2. Motor de Priorización Asíncrona con Colas de Mensajes
- Implementa **Redis Streams** como backend de cola, con TTL configurable (por defecto `ttl=3600s`).
- Soporta **sharding por hash del remitente** para distribuir la carga entre 8 workers concurrentes.
- Métricas en tiempo real: `throughput > 1200 msg/s`, `p99_latency < 50 ms`, `queue_depth` monitoreable vía Prometheus.

### 3. Integración Directa con PostgreSQL para Almacenamiento de Metadatos
- Esquema optimizado con **índices GIN** sobre campos JSONB (`metadata`), permitiendo consultas full-text en menos de **2 ms**.
- Política de **particionamiento por fecha** (`created_at`) con ventanas de retención configurables (por defecto 90 días).
- Parámetros de conexión: `pool_size=20`, `statement_timeout=30000ms`, `ssl_mode=require`.

## Cómo Usar (Pasos Detallados)

### Paso 1: Despliegue del Clasificador con Docker Compose
```yaml
services:
  classifier:
    image: focusinbox-eight/classifier:2.1.0
    environment:
      - MODEL_PATH=/models/minilm-l6-v2-int8
      - REDIS_URL=redis://redis:6379/0
      - DB_DSN=postgresql://user:pass@db:5432/focusinbox
    ports:
      - "8080:8080"
    volumes:
      - ./models:/models
```
Luego ejecutar:
```bash
docker compose up -d
```
Verificar el estado con `curl http://localhost:8080/health`.

### Paso 2: Configuración de la Cola de Prioridades
Editar el archivo `config/queue.yaml`:
```yaml
queue:
  type: redis_streams
  stream_key: "inbox:priority"
  consumer_group: "focusinbox-workers"
  workers: 8
  ttl_seconds: 3600
  retry_policy:
    max_retries: 3
    backoff: exponential
```
Reiniciar el servicio worker:
```bash
docker compose restart worker
```

### Paso 3: Ingesta de Correos vía API REST
```bash
curl -X POST https://api.focusinbox-eight.local/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "from": "cliente@ejemplo.com",
    "subject": "Urgente: problema de producción",
    "body": "El servidor principal ha caído...",
    "metadata": {"priority": "high", "source": "smtp"}
  }'
```
Respuesta esperada (tiempo real):
```json
{
  "status": "queued",
  "message_id": "foc-8a3f9c2e",
  "estimated_delay_ms": 12,
  "queue_position": 1
}
```

## Mejores Prácticas Recomendadas

### 1. Estrategia de Escalado Horizontal con Auto-Scaling
Configurar un **HPA (Horizontal Pod Autoscaler)** en Kubernetes basado en la métrica `queue_depth` de Redis. Recomendamos un umbral de **500 mensajes por worker** para activar un nuevo pod. Esto evita picos de latencia y garantiza un throughput constante incluso bajo cargas de 10,000 msg/s.

### 2. Auditoría de Rendimiento con Trazabilidad Distribuida
Implementar **OpenTelemetry** para rastrear cada mensaje desde la ingesta hasta la clasificación. Configurar un **exportador Jaeger** con muestreo del 10% (head-based sampling). Esto permite identificar cuellos de botella en la capa de embeddings o en la escritura a PostgreSQL, reduciendo el tiempo de depuración de horas a minutos.
