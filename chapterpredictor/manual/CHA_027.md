## What is chapterpredictor?

**chapterpredictor** is a deterministic, transformer-based chapter segmentation engine designed for long-form content (10K+ tokens). It leverages a custom-trained BERTopic variant to infer logical breakpoints in unstructured text—no human annotation required. Unlike naive NLP chunkers, it respects semantic coherence, narrative arcs, and topical drift, outputting a structured chapter map with confidence scores.

This is **not** a generic summarizer. It’s a precision tool for content architects, technical writers, and SEO engineers who need to programmatically generate chapter-level outlines for documentation, ebooks, or knowledge bases.

---

## Core Features (Hardcore, Not Fluff)

### 1. **Semantic Breakpoint Detection via Hierarchical Coherence Scoring**
Instead of fixed-length or sliding-window chunking, chapterpredictor uses a multi-scale coherence matrix. It computes local and global topic divergence across 3 sliding windows (128, 256, 512 tokens) and identifies inflection points where semantic drift exceeds a tunable threshold. This yields chapter boundaries that align with human-perceived topic shifts—not paragraph boundaries.

### 2. **Confidence-Weighted Chapter Map with Entropy Filtering**
Each predicted chapter boundary includes a **confidence score** (0.0–1.0) derived from a softmax over the coherence differential. Low-confidence boundaries (entropy > 0.7) are automatically flagged or merged. This prevents over-segmentation in noisy text (e.g., code snippets, tables) and ensures the output chapter count is both stable and reproducible across runs.

### 3. **SEO-Aware Chapter Title Generation with Keyword Injection**
Beyond segmentation, chapterpredictor generates a **descriptive chapter title** for each segment using a lightweight T5 model fine-tuned on long-form content. It automatically injects high-value LSI keywords into the title while preserving readability. This is critical for internal linking and content silo architecture in SEO-heavy projects.

## How to Use (Production-Ready Workflow)

### Step 1: Initialize the Engine with Custom Coherence Thresholds
```python
from chapterpredictor import ChapterPredictor

predictor = ChapterPredictor(
    model_name="bert-base-uncased",
    coherence_window=256,
    min_chapter_length=500,   # tokens
    entropy_threshold=0.7,
    keyword_bias=["API", "tutorial", "architecture"]
)
```
This sets the semantic sensitivity. For technical documentation, use a smaller window (128) to catch code block transitions. For narrative content, use 512.

### Step 2: Run Batch Inference on Raw Text with Context Preservation
```python
text = load_long_document("your_10k_word_doc.txt")
results = predictor.segment(
    text,
    return_confidences=True,
    merge_low_confidence=True,
    max_chapters=12
)
```
The engine returns a list of `ChapterSegment` objects, each containing:
- `start_index`, `end_index` (character offsets)
- `confidence` (float)
- `generated_title` (string)
- `keyword_density` (dict)

### Step 3: Export to Structured JSON for CMS Integration
```python
import json

export = []
for seg in results.segments:
    export.append({
        "title": seg.generated_title,
        "offset": [seg.start_index, seg.end_index],
        "confidence": seg.confidence,
        "content": text[seg.start_index:seg.end_index]
    })

with open("chapter_map.json", "w") as f:
    json.dump(export, f, indent=2)
```
This JSON can be directly ingested by headless CMS platforms (e.g., Contentful, Strapi) or used to generate an anchor-link table of contents.

## Best Practices (From a SaaS Architect’s Playbook)

### 1. **Pair with a Content Silo Strategy for SEO Gains**
Don’t just segment—use the chapter titles to build a **topic cluster**. Map each generated chapter to a specific subtopic in your content silo. Link them internally using the generated titles as anchor text. This signals topical authority to Google’s passage indexing algorithm.

### 2. **Run A/B Tests on Coherence Window Size**
For long-form tutorials or multi-author documents, the default 256-token window may over-segment. Run a batch test with 128 and 512 windows across 20 documents. Measure chapter count stability and confidence variance. Choose the window that minimizes entropy spikes—this will yield the most consistent user experience.
