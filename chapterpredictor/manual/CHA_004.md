## What is chapterpredictor?

## Core Features (Hardcore)

1. **Semantic Pacing & Entropy-Based Chapter Break Prediction**  
   Instead of static word counts, chapterpredictor evaluates *semantic entropy*—the information density shift between paragraphs. It identifies where reader cognitive load drops and auto-suggests chapter splits that maximize retention. You can tune the entropy threshold per genre (e.g., thriller vs. tutorial).

2. **Multi-Model Structural Forecasting**  
   The tool runs parallel inference across three models: a **Markov chain** for sequential probability, a **transformer-based sequence predictor** for long-range dependencies, and a **Bayesian hierarchical model** for chapter length distribution. It merges outputs into a single confidence score per proposed chapter boundary.

3. **Retention Curve Optimization & A/B Chaptering**  
   chapterpredictor integrates real-world reader behavior data (via optional analytics import) to simulate how different chapter splits affect drop-off rates. It can generate **3 alternative chapter structures** for the same content and rank them by predicted engagement lift, using a proprietary *Retention Delta Score*.

## How to Use (Professional Steps)

### Step 1: Ingest & Normalize Content  
Feed your raw text or Markdown file into the CLI or API endpoint. The tool automatically strips formatting, normalizes whitespace, and runs a **tokenization pipeline** that respects multilingual punctuation (CJK, Latin, Cyrillic). Use the `--lang-model` flag to select a language-specific tokenizer (e.g., `--lang-model en` for English).  
*Example:* `chapterpredictor analyze --input draft.md --lang-model en`

### Step 2: Configure Prediction Parameters  
Set your target chapter count, minimum/maximum chapter length in tokens, and the **entropy sensitivity** (0.0 = conservative, 1.0 = aggressive splitting). For serialized content, enable `--series-mode` to enforce structural consistency across volumes. You can also pass a custom **reader retention curve** CSV via `--retention-profile`.  
*Example:* `chapterpredictor predict --target-chapters 12 --entropy 0.7 --series-mode`

### Step 3: Review & Export Predicted Structure  
The tool outputs a ranked list of chapter boundaries with confidence scores, token counts, and predicted retention impact. Use the `--export-json` flag to get a machine-readable schema for CI/CD pipelines. For visual review, run `chapterpredictor visualize --output chart.html` to generate a **semantic entropy heatmap** over your content.  
*Example:* `chapterpredictor visualize --output chapter-map.html --highlight-boundaries`

## Industry Best Practices

### Best Practice 1: Pair with Real Analytics for Iterative Refinement  
Don’t rely solely on default retention curves. Integrate chapterpredictor with your analytics platform (e.g., Google Analytics, Mixpanel) by exporting actual user drop-off data. Use the `--retention-profile` flag to feed real metrics back into the model. This creates a **closed-loop optimization** cycle: predict → publish → measure → retrain. Over 3–4 iterations, your chapter breaks will become nearly optimal for your specific audience.

### Best Practice 2: Use Multi-Version A/B Testing for Critical Content  
For high-stakes content (e.g., onboarding docs, subscription-series premieres), generate 3 distinct chapter structures using `--alternative-count 3`. Deploy each version to a separate user segment via feature flags. Track engagement metrics (time on page, scroll depth, conversion) for 48 hours, then promote the winning structure. chapterpredictor’s **Retention Delta Score** correlates with actual user behavior at ~0.89 Pearson—enough to trust as a proxy metric when A/B testing is impractical.
