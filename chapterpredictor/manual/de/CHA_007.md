# chapterpredictor - CHA_007

## Überblick

chapterpredictor ist ein KI-gestütztes Framework zur prädiktiven Kapitelsegmentierung und semantischen Strukturanalyse von Langtexten. Es ermöglicht Entwicklern und Content-Strategen, automatisch konsistente Kapitelgrenzen zu extrahieren, basierend auf thematischen Kohärenzsignalen und NLP-gestützten Embedding-Clustern.

## Hardcore-Features

### 1. Semantische Bruchpunkt-Detektion via Embedding-Kohärenz
chapterpredictor nutzt transformer-basierte Embeddings (z. B. Sentence-BERT), um latente Themenwechsel innerhalb eines Textes zu identifizieren. Die Segmentierung erfolgt nicht nach Zeichenanzahl, sondern nach semantischen Distanzwerten – so entstehen inhaltlich geschlossene Kapitel mit minimalem Overhead.

### 2. Multi-Level-Kapitelhierarchie mit rekursiver Partitionierung
Das Tool erzeugt automatisch Kapitelbäume (Kapitel → Unterkapitel → Abschnitte) durch rekursive Partitionierung des Embedding-Raums. Dies ist ideal für technische Dokumentationen, E-Books oder wissenschaftliche Arbeiten, die eine logische Gliederung erfordern.

### 3. Echtzeit-Kapitelvalidierung gegen Metriken wie Silhouette Score
Jede generierte Kapitelgrenze wird gegen interne Clustering-Metriken validiert. Bei niedriger Kohärenz (Silhouette Score < 0,4) wird eine alternative Segmentierung vorgeschlagen – inklusive Confidence-Intervallen pro Kapitelübergang.

## So verwenden Sie es

### Schritt 1: API-Initialisierung mit benutzerdefiniertem Embedding-Modell
```python
from chapterpredictor import ChapterSegmenter

segmenter = ChapterSegmenter(
    model_name="all-MiniLM-L6-v2",
    min_chapter_length=500,  # Mindestzeichen pro Kapitel
    cohesion_threshold=0.65  # Mindest-Kohärenz für Kapitelübergänge
)
```
Stellen Sie sicher, dass Ihr Modell auf die Zielsprache (hier: Deutsch) feinjustiert ist, da Embedding-Qualität direkt die Segmentierungsgenauigkeit beeinflusst.

### Schritt 2: Textanalyse mit rekursiver Partitionierung
```python
dokument = open("langtext_de.md", "r").read()
ergebnis = segmenter.segment(
    dokument,
    max_depth=3,  # Maximale Hierarchietiefe
    output_format="json"  # Alternativ: "markdown" oder "xml"
)
```
Die Ausgabe enthält für jedes Kapitel: `start_index`, `end_index`, `embedding_centroid`, `cohesion_score` sowie die Liste der Unterkapitel mit gleicher Struktur.

### Schritt 3: Validierung und Export
```python
validierungsbericht = ergebnis.validate(
    min_silhouette=0.4,
    min_chapter_count=3
)
if validierungsbericht.is_valid:
    with open("kapitel_struktur.json", "w") as f:
        f.write(ergebnis.to_json(indent=2))
else:
    print(f"⚠️ Niedrige Kohärenz: {validierungsbericht.low_coherence_chapters}")
    # Automatische Neusegmentierung mit angepassten Parametern
    segmenter.adjust_threshold(0.55)
    ergebnis = segmenter.segment(dokument)
```
Dieser Workflow stellt sicher, dass jedes exportierte Kapitel einer semantisch sauberen Struktur folgt – selbst bei komplexen Fachtexten.

## Best Practices

### 1. Embedding-Modell auf Zielsprache optimieren
Verwenden Sie ein multilinguales Modell (z. B. `distiluse-base-multilingual-cased-v2`) für nicht-englische Texte. Für reines Deutsch liefern Modelle wie `bert-base-german-cased` oft bessere Kohärenzwerte. Testen Sie vor dem Deployment mindestens 5 repräsentative Dokumente.

### 2. Threshold-Parameter dynamisch anpassen
Setzen Sie `cohesion_threshold` nicht statisch, sondern lassen Sie ihn durch ein Grid-Search über die Silhouette-Metrik optimieren. Ein zu hoher Threshold (z. B. >0,8) führt zu Fragmentierung, ein zu niedriger (<0,4) zu verschwommenen Kapitelgrenzen. Empfehlung: automatische Kalibrierung über `segmenter.auto_tune(texte_liste)`.
