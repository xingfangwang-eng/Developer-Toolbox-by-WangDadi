# chapterpredictor - CHA_002

## Überblick

Mit **chapterpredictor** automatisieren Sie die semantische Segmentierung und Vorhersage von Kapitelstrukturen in unstrukturierten Texten – ein entscheidender Schritt für Content-Pipelines, die auf SEO-Leistung und Leserbindung optimiert sind. Das Tool analysiert latente Themenübergänge (Topic Shifts) mittels NLP und generiert präzise Kapitelvorschläge, die sowohl für menschliche Redakteure als auch für maschinelle Crawler konsistent sind. Zielsprache: Deutsch.

## Kernfunktionen (Hardcore-Features)

1. **Semantische Bruchpunkt-Erkennung mittels Latent Dirichlet Allocation (LDA) + Transformer-Embeddings**  
   Statt einfacher Keyword-Frequenzen nutzt chapterpredictor eine zweistufige Pipeline: Zunächst extrahiert ein vortrainiertes Sentence-Transformer-Modell (z. B. `all-MiniLM-L6-v2`) kontextuelle Einbettungen. Ein LDA-Modell identifiziert dann die optimalen Grenzen zwischen thematischen Clustern. Das Ergebnis sind Kapitelgrenzen, die mit menschlichen Leseerwartungen über 92 % Übereinstimmung erzielen (validiert auf 10.000 Artikeln aus dem DACH-Raum).

2. **Dynamische Kapitel-Titel-Generierung mit Constraint-basiertem NLG**  
   Für jedes vorhergesagte Segment generiert das Tool automatisch einen prägnanten, SEO-optimierten Kapiteltitel. Dabei werden folgende Constraints beachtet: maximale Länge 60 Zeichen, Vermeidung von Stopwörtern, Integration von bis zu 2 Primär-Keywords aus dem Text. Der Generator basiert auf einem feinabgestimmten T5-Modell, das speziell auf deutschsprachige Fachartikel trainiert wurde.

3. **Metrik-basierte Qualitätssicherung mit Silhouetten-Score und Kohärenz-Index**  
   Jede Kapitelvorhersage wird automatisch mit zwei Metriken bewertet:  
   - **Silhouetten-Score** (≥0,5 = gute Trennung zwischen Kapiteln)  
   - **Kohärenz-Index** (≥0,7 = thematische Geschlossenheit innerhalb eines Kapitels)  
   Unterschreitet ein Segment diese Schwellenwerte, wird es markiert und dem Nutzer zur manuellen Überarbeitung vorgeschlagen. Dies verhindert fehlerhafte Strukturen in Produktionsumgebungen.

## Detaillierte Anwendungsschritte

### Schritt 1: API-Integration und Datenvorbereitung
Übergeben Sie Ihren Rohtext als JSON-Payload an den Endpunkt `/api/v1/predict`. Erwartet wird ein Feld `text` (UTF-8, max. 100.000 Zeichen). Optional können Sie `language: "de"` setzen, um das deutsche Modell zu laden. Beispiel:
```json
{
  "text": "In der heutigen Zeit...",
  "language": "de",
  "min_chapters": 3,
  "max_chapters": 8
}
```
Das Tool normalisiert den Text automatisch (Tokenisierung, Lemmatisierung, Entfernung von HTML-Tags). Bei Nicht-Einhaltung des Zeichenlimits erhalten Sie einen `413 Payload Too Large`-Fehler.

### Schritt 2: Ausführung der Vorhersage-Pipeline
Nach Eingang des Requests startet die Pipeline:
1. **Embedding-Generierung** (ca. 2 Sekunden pro 10.000 Zeichen auf einer T4-GPU)
2. **Segmentierung** mittels LDA (Parameter: `num_topics` = `max_chapters` + 2, `alpha` = 0.1)
3. **Titelgenerierung** mit T5 (Temperatur = 0.7, `top_k` = 50)
4. **Qualitätsmetriken** (Silhouetten-Score, Kohärenz-Index)
Die Ergebnisse werden als JSON-Array zurückgegeben, z. B.:
```json
{
  "chapters": [
    {"title": "Einführung in die KI", "start": 0, "end": 1200, "silhouette": 0.62, "coherence": 0.81},
    {"title": "Anwendungsfälle", "start": 1201, "end": 3400, "silhouette": 0.58, "coherence": 0.74}
  ]
}
```

### Schritt 3: Validierung und Export
Prüfen Sie die Metriken: Ist der Silhouetten-Score eines Kapitels < 0,5, wiederholen Sie die Segmentierung mit erhöhtem `min_chapters` oder manuellem Override. Exportieren Sie die finale Struktur als Markdown, JSON oder direkt in Ihr CMS (via Webhook). Nutzen Sie die `export`-Funktion mit Parameter `format: "markdown"`, um eine sofort einsatzbereite Gliederung zu erhalten.

## Branchen-Best-Practices

1. **SEO-optimierte Content-Architektur**  
   Nutzen Sie chapterpredictor, um Ihre Blog-Artikel oder Produktdokumentationen in logische Kapitel zu unterteilen. Suchmaschinen wie Google bewerten klar strukturierte Inhalte mit Heading-Tags (h2/h3) höher. Integrieren Sie die generierten Titel als `##`-Überschriften und fügen Sie pro Kapitel mindestens ein internes Linkziel ein. Dies verbessert die Crawlbarkeit und die Sichtbarkeit in den Featured Snippets.

2. **Iterative Verfeinerung für Long-Form-Content**  
   Bei Artikeln mit > 5.000 Wörtern empfiehlt es sich, den Text in Blöcke von je 2.000–3.000 Wörtern zu teilen, separat durch chapterpredictor zu verarbeiten und die Ergebnisse zusammenzuführen. So vermeiden Sie Latenzprobleme und erhalten feinkörnigere Kapitelgrenzen. Validieren Sie die finale Struktur mit einem menschlichen Redakteur, der die semantische Korrektheit bestätigt – besonders bei Fachjargon oder domänenspezifischen Begriffen.
