# chapterpredictor - CHA_030

## Überblick

**chapterpredictor** ist ein KI-gestütztes Vorhersagetool für semantische Kapitelstrukturen in Langtexten. Es analysiert latente Themenübergänge, Satzembedding-Vektoren und narrative Kohärenz, um automatisch optimale Kapitelgrenzen zu generieren – ideal für technische Redakteure, Content-Strategen und SaaS-Dokumentationsteams.

## Kernfunktionen (Hardcore-Features)

### 1. Semantische Bruchpunkt-Detektion via Transformer-Embeddings
Statt auf Wortfrequenz oder Länge zu setzen, nutzt chapterpredictor ein feinabgestimmtes Sentence-BERT-Modell, um Themenwechsel auf Satzebene zu erkennen. Der Algorithmus berechnet cosine distances zwischen aufeinanderfolgenden Embeddings und identifiziert abrupte Änderungen im semantischen Fluss – selbst bei konsistenter Terminologie.

### 2. Adaptive Kapitel-Clustering mit dynamischer K-Bestimmung
Das Tool wendet ein modifiziertes HDBSCAN-Clustering auf die Satzvektoren an, wobei die minimale Clustergröße automatisch an die Textlänge und Varianz angepasst wird. So entstehen Kapitel, die nicht nur logisch, sondern auch visuell ausgewogen sind – ohne manuelle Schwellenwerte.

### 3. Export-Engine mit Markdown-Tree und Metadaten-Anreicherung
chapterpredictor generiert nicht nur Kapitelüberschriften, sondern einen vollständigen Markdown-Tree mit hierarchischen IDs, durchschnittlichen Embedding-Distanzen und Confidence-Scores pro Kapitelgrenze. Diese Metadaten sind direkt in CI/CD-Pipelines oder CMS-Systeme integrierbar.

## Detaillierte Anwendung (3 professionelle Schritte)

### Schritt 1: Textvorbereitung und Embedding-Generierung
Rohdaten (Markdown, HTML oder Plaintext) werden in Satzsegmente zerlegt. Verwende die Funktion `generate_embeddings()` mit dem Modell `all-MiniLM-L6-v2` – achte auf eine minimale Satzlänge von 5 Tokens. Bei technischen Dokumenten mit Code-Blöcken empfehle ich eine Preprocessing-Pipeline, die Code-Blöcke durch Platzhalter ersetzt, um Rauschen zu minimieren.

```python
from chapterpredictor import TextProcessor, EmbeddingGenerator
tp = TextProcessor(language='de', min_token_length=5)
sentences = tp.split_into_sentences(raw_text)
embeddings = EmbeddingGenerator(model='all-MiniLM-L6-v2').encode(sentences)
```

### Schritt 2: Dynamische Kapitelgrenzen-Berechnung
Rufe die `predict_chapters()`-Methode mit dem Parameter `method='hdbscan'` und `min_cluster_size=auto` auf. Das Tool gibt eine Liste von `ChapterBoundary`-Objekten zurück, die jeweils den Startindex, Endindex, Confidence-Wert (0–1) und die durchschnittliche Distanz zum Nachbarkapitel enthalten. Für eine optimale Segmentierung bei hybriden Texten (z. B. Tutorial + API-Referenz) setze `sensitivity=0.75`.

```python
from chapterpredictor import ChapterPredictor
predictor = ChapterPredictor(method='hdbscan', sensitivity=0.75)
boundaries = predictor.predict_chapters(embeddings)
for b in boundaries:
    print(f'Kapitel {b.id}: Index {b.start}-{b.end}, Confidence {b.confidence:.2f}')
```

### Schritt 3: Markdown-Export mit Metadaten-Integration
Exportiere die Struktur als validiertes Markdown mit `export_to_markdown()`. Nutze den Parameter `include_metadata=True`, um einen Metadaten-Block einzufügen, der die Confidence-Werte und Embedding-Distanzen als Kommentare enthält. Dies ermöglicht eine spätere manuelle Überprüfung oder automatische Qualitätssicherung.

```python
md_output = predictor.export_to_markdown(boundaries, sentences, 
                                          include_metadata=True, 
                                          header_level=2)
with open('kapitelstruktur.md', 'w', encoding='utf-8') as f:
    f.write(md_output)
```

## Branchen-Best Practices

### 1. Hybride Validierung: KI + menschliche Prüfung
Setze chapterpredictor als Preprocessing-Tool ein, aber implementiere einen manuellen Review-Schritt für die Confidence-Werte unter 0,6. In agilen Dokumentationsteams hat sich ein zweistufiger Workflow bewährt: Automatische Generierung → menschliche Anpassung der Grenzen → erneutes Embedding und Confidence-Rechnung. Dies reduziert Redundanz bei gleichbleibender Qualität.

### 2. Versionierung der Embedding-Modelle
Da sich die Qualität der Kapitelvorhersage direkt auf die Embedding-Modelle stützt, empfehle ich, die Modellversion (z. B. `all-MiniLM-L6-v2` vs. `gte-small`) im Metadaten-Header jedes Exportes zu speichern. Verwende eine `model_version`-Variable in deiner CI/CD-Pipeline, um bei Modell-Updates alle historischen Strukturen automatisch neu berechnen zu können. So bleibst du rückwärtskompatibel und vermeidest inkonsistente Ergebnisse über verschiedene Releases hinweg.
