# chapterpredictor - CHA_003

## Übersicht

**chapterpredictor** ist eine hochperformante, KI-gestützte Engine zur prädiktiven Kapitelstrukturierung von Langtexten. Optimiert für Content-Pipelines, die auf semantische Segmentierung und automatisierte Inhaltsnavigation angewiesen sind.  
Keywords: *Kapitelvorhersage, semantische Segmentierung, Content-Strukturierung, NLP-basierte Kapitelgenerierung*

## Detaillierter Inhalt

### Was ist chapterpredictor?

chapterpredictor analysiert Rohdokumente auf Makroebene und extrahiert mittels transformerbasierter Modelle latente Kapitelgrenzen. Es ist kein einfacher Text-Splitter – es versteht kontextuelle Brüche, Themenwechsel und narrative Bögen, um menschenlesbare, SEO-optimierte Kapitelstrukturen zu generieren. Ideal für technische Dokumentationen, E-Books, Blog-Serien und API-Dokumentationen.

### Kernfunktionen

1. **Semantische Grenzdetektion mit adaptiven Schwellwerten**  
   Nutzt ein hybrides Modell aus Sentence-BERT und Topic-Coherence-Scores, um Kapitelgrenzen nicht nur nach Wortanzahl, sondern nach inhaltlicher Kohärenz zu setzen. Der Schwellwert passt sich dynamisch an die Textlänge und -komplexität an.

2. **Metadaten-gestützte Kapitelgenerierung mit automatischer TOC-Erstellung**  
   Jedes vorhergesagte Kapitel erhält automatisch einen Titel, eine Zusammenfassung (max. 50 Wörter) und eine Prioritätsebene (1–5). Diese Metadaten werden direkt in JSON oder YAML exportiert – ideal für Headless-CMS und Static-Site-Generatoren.

3. **Multilinguale Embedding-Normalisierung für Cross-Lingual-Transfer**  
   Die Embeddings werden über einen mehrsprachigen Encoder (z. B. XLM-R) normalisiert, sodass Kapitelvorhersagen auch bei gemischten Sprachanteilen (z. B. Deutsch mit englischen Fachbegriffen) stabil bleiben. Dies minimiert False Positives bei Code-Blöcken oder Zitaten.

### So verwenden Sie es

1. **Dokument vorverarbeiten und in Chunks zerlegen**  
   Nutzen Sie den integrierten `TextPreprocessor`, um das Dokument zu normalisieren (HTML-Tags entfernen, Unicode normalisieren, Zeilenumbrüche glätten). Dann rufen Sie `chapterpredictor.predict(file_path, chunk_size=1024, overlap=128)` auf – dies erzeugt eine Liste von `ChapterCandidate`-Objekten mit Konfidenzwerten.

2. **Konfidenzschwellwert anpassen und Export konfigurieren**  
   Setzen Sie `min_confidence=0.65` im Konstruktor, um Rauschen zu filtern. Anschließend exportieren Sie die Struktur via `export_to_markdown(toc=True, include_summary=True)`. Das Ergebnis ist ein validiertes Markdown-Dokument mit `<!-- chapterpredictor:metadata -->`-Blöcken für jedes Kapitel.

3. **In CI/CD-Pipeline integrieren (z. B. GitHub Actions)**  
   Fügen Sie einen Step hinzu, der `chapterpredictor` auf jede Pull-Request-Datei anwendet und die Kapitelstruktur als Kommentar postet. Beispiel:  
   ```yaml
   - name: Generate chapter structure
     run: |
       pip install chapterpredictor
       chapterpredictor --input docs/ --output .chapterpredictor/ --format json
   ```  
   Das erzeugte JSON kann dann von Hugo, Docusaurus oder Next.js direkt gelesen werden.

### Beste Praktiken

1. **Dokumente vorab in logische Blöcke segmentieren**  
   chapterpredictor arbeitet optimal, wenn Sie das Dokument bereits in sinnvolle Abschnitte (z. B. `<section>`-Tags oder `---`-Trenner) vorstrukturieren. Dies reduziert die Latenz und erhöht die Genauigkeit der Kapitelvorhersage um bis zu 40 %.

2. **Metadaten als Grundlage für dynamische Navigation nutzen**  
   Verwenden Sie die exportierte Kapitelstruktur, um eine dynamische Seitenleiste oder ein Breadcrumb-Navigationssystem zu generieren. Kombinieren Sie die `priority`-Werte mit einem Filter (`priority > 3`), um nur Hauptkapitel anzuzeigen – ideal für mobile Ansichten.
