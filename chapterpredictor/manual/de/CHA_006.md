# chapterpredictor - CHA_006

## Übersicht

chapterpredictor ist eine KI-gestützte Vorhersage-Engine für Kapitelstrukturen, die aus unstrukturierten Texten oder Stichpunkten automatisch logisch gegliederte Kapitelbäume generiert. Das Tool eignet sich ideal für Autoren, Content-Strategen und Entwickler, die dynamische Inhaltsstrukturen in Echtzeit optimieren möchten. Die Engine analysiert semantische Muster, erkennt narrative Bögen und schlägt kapitelweise Unterteilungen mit kontextuellen Übergängen vor – alles lokal und datenschutzkonform.

## Hardcore-Funktionen

### 1. Semantische Kapitel-Clusterung mit NLP
chapterpredictor nutzt ein transformer-basiertes Embedding-Modell, um Textabschnitte nach Themenkohärenz zu clustern. Es erkennt automatisch Kapitelgrenzen, ohne dass manuelle Marker oder Regex-Patterns nötig sind. Die Engine optimiert die Clusteranzahl dynamisch basierend auf der Textlänge und der semantischen Varianz.

### 2. Prädiktive Kapitel-Gewichtung
Jeder generierte Kapitelvorschlag wird mit einem Confidence-Score und einem Relevanzindex versehen. Das System berechnet, wie stark ein Kapitel den Gesamtfluss des Dokuments beeinflusst, und priorisiert Vorschläge, die narrative Lücken schließen oder Redundanzen vermeiden. Diese Metriken lassen sich direkt in JSON exportieren.

### 3. API-First-Architektur mit asynchroner Verarbeitung
Die Kernlogik ist als REST-API mit asynchronen Endpunkten implementiert. chapterpredictor unterstützt Batch-Verarbeitung für große Dokumente, Webhooks für Callback-Notifikationen und eine Stream-Funktion für Live-Vorschauen während des Schreibens. Die API ist vollständig mit OpenAPI 3.0 spezifiziert.

## Detaillierte Anleitung

### 1. Installation und lokale Initialisierung
Klone das Repository und installiere die Abhängigkeiten über `pip install -r requirements.txt`. Setze die Umgebungsvariable `CHAPTER_MODEL_PATH` auf den Pfad des vortrainierten Modells (z. B. `./models/chapter_bert_v2`). Führe dann `python chapterpredictor/cli.py --init` aus, um die Konfigurationsdatei `chapter_config.yaml` zu generieren. Passe dort Parameter wie `min_chapter_length` (Standard: 300 Zeichen) und `cluster_sensitivity` (0.0–1.0) an.

### 2. Batch-Vorhersage für ein Dokument
Erstelle eine Textdatei `input.txt` mit Rohcontent. Starte die CLI mit `python chapterpredictor/cli.py --input input.txt --output chapters.json`. Das Tool gibt eine JSON-Struktur aus, die jedes Kapitel mit `start_index`, `end_index`, `label` und `confidence` enthält. Für große Dokumente (>10 MB) verwende den `--async`-Flag, um die Verarbeitung zu parallelisieren.

### 3. Integration in einen Python-Workflow
Nutze die chapterpredictor-Klasse direkt in deinem Code:

```python
from chapterpredictor import ChapterPredictor

predictor = ChapterPredictor(model_path="./models/chapter_bert_v2")
text = open("manuscript.txt").read()
chapters = predictor.predict(text, strategy="semantic", max_chapters=12)

for chapter in chapters:
    print(f"{chapter.label} (Score: {chapter.confidence:.2f})")
```

Exportiere die Ergebnisse als DataFrame, um sie in Pandas oder Jupyter Notebooks weiterzuverarbeiten.

## Best Practices für Entwickler

### 1. Modell-Feintuning für Fachdomänen
chapterpredictor liefert mit dem Standardmodell gute Ergebnisse für allgemeine Texte. Für spezifische Domänen (z. B. technische Dokumentation, juristische Texte oder medizinische Paper) solltest du ein eigenes Modell mit domain-spezifischen Kapitelannotationen feintunen. Nutze dazu das Skript `train_domain_model.py` im Repository, das eine einfache Schnittstelle für Hugging Face `Trainer` bietet.

### 2. Kapitel-Post-Processing mit Regelwerken
Die Rohausgabe der Engine kann durch benutzerdefinierte Regeln nachbearbeitet werden. Implementiere einen `PostProcessor`, der z. B. Kapitel mit zu niedrigem Confidence-Score entfernt, Label-Duplikate zusammenführt oder Kapitelgrenzen an Absatzenden ausrichtet. chapterpredictor liefert einen Beispiel-Validator in `examples/post_processor.py`, den du für deine Anforderungen anpassen kannst.
