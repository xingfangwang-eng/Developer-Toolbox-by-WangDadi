# chapterpredictor - CHA_014

## Überblick

chapterpredictor ist ein hochperformantes, KI-gestütztes Tool zur prädiktiven Kapitelstrukturierung von Content. Es analysiert semantische Muster, extrahiert latente Themenhierarchien und generiert automatisch logisch kohärente Kapitelabfolgen. Optimiert für SEO-kritische Textstrukturen, Long-Form-Content und modulare Dokumentation.

## Hardcore-Funktionen

### 1. Semantische Cluster-Detektion mit Transformer-Embeddings
chapterpredictor nutzt vortrainierte Sprachmodelle (z. B. Sentence-BERT), um Textsegmente in einen hochdimensionalen Embedding-Space zu projizieren. Über hierarchisches Clustering (HDBSCAN) werden thematische Grenzen präzise detektiert – unabhängig von Absatzlängen oder Formatierungsartefakten.

### 2. Dynamische Kapitel-Tiefensteuerung per Entropy-Threshold
Das Tool berechnet die semantische Entropie pro Textfenster und schlägt automatisch Kapitelgrenzen vor, sobald die Entropie einen konfigurierbaren Schwellwert überschreitet. Entwickler können über `--entropy-threshold 0.75` steuern, wie granular die Kapitelstruktur ausfallen soll – von flachen 3 Kapiteln bis zu tiefen 12+ Ebenen.

### 3. SEO-optimierte Kapitel-Headline-Generierung mit Keyword-Embedding
Statt generischer Überschriften generiert chapterpredictor pro Kapitel eine Headline, die semantisch zum Inhalt passt und gleichzeitig die Ziel-Keywords (z. B. „Kapitelvorhersage“, „Content-Strukturierung“) optimal platziert. Die Keywords werden über einen internen TF‑IDF‑Ranker gewichtet und in die Überschrift eingewoben – ohne Keyword-Stuffing.

## So verwenden Sie es – Schritt für Schritt

### Schritt 1: Installation & Konfiguration des Embedding-Modells
```bash
pip install chapterpredictor
# Modell herunterladen (empfohlen: multilingual-e5-large)
chapterpredictor download-model --model multilingual-e5-large
```
Danach in `config.yaml` den Pfad zum Modell sowie den gewünschten Entropy-Threshold setzen:
```yaml
model_path: ./models/multilingual-e5-large
entropy_threshold: 0.65
output_format: markdown
target_language: de
```

### Schritt 2: Rohdaten vorbereiten & semantische Analyse starten
Speichern Sie Ihren Text als `.txt` oder `.md` und führen Sie chapterpredictor aus:
```bash
chapterpredictor analyze --input ./dokument.md --output ./kapitel.md
```
Das Tool zerlegt den Text in semantische Chunks (Standardgröße: 256 Tokens), berechnet Embeddings und clustert diese. Im Terminal erscheint eine Live-Entropy-Kurve, die zeigt, wo Kapitelgrenzen gesetzt werden.

### Schritt 3: Kapitelstruktur validieren & Headlines optimieren
Nach der Analyse öffnen Sie die generierte `kapitel.md` und prüfen die vorgeschlagenen Überschriften. Mit dem Befehl:
```bash
chapterpredictor refine --input kapitel.md --keywords "Kapitelvorhersage, Content-Strukturierung, SEO"
```
werden die Headlines noch einmal auf Keyword-Relevanz und semantische Dichte getrimmt. Das Ergebnis ist eine markdown-ready Kapitelstruktur, die Sie direkt in Ihr CMS oder Ihre Dokumentation einbinden können.

## Best Practices für Profis

### 1. Entropy-Threshold an Content-Typ anpassen
Für technische Dokumentation (z. B. API-Referenzen) empfehle ich einen **niedrigen Threshold** (0,4–0,5), um viele kleine, präzise Kapitel zu erhalten. Für narrative Longform-Content (Blog, Whitepaper) ist ein **höherer Threshold** (0,7–0,8) ideal – er erzeugt weniger, aber thematisch dichtere Kapitel. Testen Sie beide Extreme mit `--entropy-threshold 0.4` vs. `0.8` und vergleichen Sie die Lesbarkeit.

### 2. Keyword-Embedding mit lokaler Glossar-Datei anreichern
Erstellen Sie eine `keywords.txt` mit branchenspezifischen Begriffen (z. B. „maschinelles Lernen“, „Text-Clustering“, „SEO-Architektur“). chapterpredictor importiert diese Liste automatisch und boostet die Gewichtung dieser Begriffe im Headline-Generator. So stellen Sie sicher, dass Ihre Kapitelüberschriften nicht nur semantisch passen, sondern auch für Ihre Zielgruppe optimiert sind.
