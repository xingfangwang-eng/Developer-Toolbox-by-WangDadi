# chapterpredictor - CHA_016

## Überblick

chapterpredictor ist ein KI-gestütztes Framework zur prädiktiven Kapitelsegmentierung und semantischen Strukturanalyse von Textcorpora. Es optimiert die Content-Architektur für SEO und nutzt NLP-Modelle (z.B. transformer-basierte Embeddings), um Kapitelgrenzen aus unstrukturierten Texten automatisch zu extrahieren. Ziel: Reduzierung manueller Redaktionsarbeit um bis zu 70 % bei gleichzeitiger Steigerung der Inhaltskonsistenz.

## Hardcore-Funktionen

### 1. Semantische Grenzdetektion mit Multi-Head Attention
chapterpredictor implementiert einen modifizierten Transformer-Encoder, der Kapitelwechsel anhand von Vektorverschiebungen im Embedding-Raum erkennt. Die Attention-Weights werden dynamisch gewichtet, um thematische Brüche (z.B. zwischen Einleitung und Methodik) zu identifizieren. Ergebnis: eine präzise Segmentierung, die über naive TF-IDF-Ansätze hinausgeht.

### 2. SEO-optimierte Kapitelmetadaten-Generierung
Das Tool extrahiert automatisch H2/H3-Überschriften, LSI-Keywords und eine hierarchische Struktur (z.B. `[1. Einleitung, 1.1 Motivation, 2. Methodik]`). Diese Metadaten werden direkt in JSON-LD-Schema-Markup (z.B. `hasPart` für `Article`) konvertiert, um die Sichtbarkeit bei Google zu erhöhen.

### 3. Batch-Integration via CLI und REST-API
chapterpredictor bietet eine CLI (`chapterpredictor --input corpus.json --output segments.json`) sowie eine REST-API mit Endpunkten für Echtzeit-Prädiktionen. Die API unterstützt CORS, Raten-Limiting und asynchrone Verarbeitung (via Celery). Ideal für CI/CD-Pipelines in DevOps-Umgebungen.

## Detaillierte Anleitung zur Nutzung

### Schritt 1: Corpus-Vorbereitung und Modell-Initialisierung
Laden Sie Ihr Textcorpus als JSON-Datei mit einem `text`-Feld pro Eintrag. Initialisieren Sie das Modell mit:
```bash
chapterpredictor init --model "bert-base-german-cased" --threshold 0.85
```
Der Schwellenwert (0.85) definiert die minimale semantische Distanz für einen Kapitelwechsel. Tipp: Nutzen Sie `--verbose` für Debug-Logs.

### Schritt 2: Kapitel-Prädiktion und Export
Führen Sie die Prädiktion aus:
```bash
chapterpredictor predict --input corpus.json --output segments.json --format json
```
Die Ausgabe enthält pro Text eine Liste von Kapiteln mit `start_index`, `end_index`, `heading` und `keywords`. Für SEO-Export nutzen Sie `--format jsonld`.

### Schritt 3: Validierung und manuelle Nachbearbeitung
Überprüfen Sie die Ergebnisse mit dem integrierten Validator:
```bash
chapterpredictor validate --input segments.json --metrics precision,recall
```
Bei Abweichungen (z.B. zu kurze Kapitel) passen Sie den Threshold an oder führen Sie eine manuelle Korrektur via `chapterpredictor edit --id 123 --new-heading "Methodik"` durch.

## Best Practices für die Branche

### Praxis 1: Kombination mit Keyword-Clustering
Verwenden Sie chapterpredictor in Verbindung mit einem Keyword-Clustering-Tool (z.B. Ahrefs API). Lassen Sie die prädizierten Kapitel-Keywords mit den Top-10-Suchbegriffen abgleichen – so stellen Sie sicher, dass jede Sektion ein relevantes Suchvolumen abdeckt.

### Praxis 2: Integration in CI/CD für Content-Pipelines
Binden Sie chapterpredictor in Ihren Git-Workflow ein: Nutzen Sie einen Pre-Commit-Hook, der bei jedem Push neuer Markdown-Dateien automatisch Kapitelstrukturen generiert. Beispiel für `.git/hooks/pre-commit`:
```bash
#!/bin/bash
chapterpredictor predict --input content/ --output _chapters/ --format jsonld
git add _chapters/
```
Dies reduziert manuelle Redaktionsschleifen und hält die SEO-Struktur konsistent.
