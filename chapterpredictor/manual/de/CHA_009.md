# chapterpredictor - CHA_009

## Übersicht

**chapterpredictor** ist ein KI-gestütztes Framework zur prädiktiven Kapitelsegmentierung und semantischen Inhaltsstrukturierung. Es analysiert Textkorpora auf latente Themencluster und generiert automatisch kohärente Kapitelgrenzen – optimiert für Content-Management-Systeme, E-Learning-Plattformen und technische Dokumentationen. Das Tool eignet sich besonders für mehrsprachige Projekte (10+ Sprachen) und reduziert manuelle Redaktionsarbeit um bis zu 80%.

## Detaillierter Inhalt

### Was ist chapterpredictor?

chapterpredictor ist ein **professionelles Tool** zur automatischen Kapitelvorhersage und Inhaltssegmentierung. Es verwendet **Transformer-basierte Sprachmodelle** (z. B. GPT-4, BERT) und **hierarchisches Clustering**, um Dokumente in logische Kapitel zu unterteilen. Im Gegensatz zu simplen Text-Tokenizern erkennt es semantische Brüche, Themenwechsel und narrative Strukturen – selbst in stark technischen Dokumenten oder kreativen Texten. Das Tool unterstützt **Batch-Verarbeitung**, **API-Integration** und **Export in PDF/EPUB/Markdown**.

### Kernfunktionen

1. **Prädiktive Kapitelgrenzen-Erkennung mit Embedding-basiertem Similarity-Scanning**  
   chapterpredictor berechnet für jedes Textsegment (z. B. 500 Token) ein Embedding über ein vortrainiertes Sentence-Transformer-Modell. Ein dynamischer Schwellenwert-Algorithmus identifiziert dann abrupte Änderungen im semantischen Vektorraum – z. B. wenn der Kosinus-Abstand zwischen aufeinanderfolgenden Segmenten > 0,7 beträgt. Das Ergebnis: präzise Kapitelgrenzen ohne Overfitting.

2. **Multilinguale Themen-Cluster-Analyse mit LDA und HDBSCAN**  
   Das Tool führt eine **Latent Dirichlet Allocation (LDA)** auf dem gesamten Dokument durch, gefolgt von **HDBSCAN-Clustering** (hierarchische Dichte-basierte Methode). So werden Themencluster automatisch erkannt und Kapitelüberschriften generiert – inklusive sprachspezifischer Stop-Word-Filter (z. B. für Deutsch, Spanisch, Japanisch). Das ermöglicht eine konsistente Kapitelstruktur über Sprachgrenzen hinweg.

3. **Dynamische Kapitel-Kompression und Abstract-Generierung**  
   Nach der Segmentation extrahiert chapterpredictor für jedes Kapitel einen **Abstract** (max. 3 Sätze) mithilfe eines **T5-Modells** (Text-To-Text Transfer Transformer). Der Abstract wird als Metadaten im JSON-Output gespeichert und kann direkt für SEO-Beschreibungen oder Inhaltsverzeichnisse verwendet werden. Die Kompression reduziert die Kapitellänge um 40–60 %, ohne semantische Verluste.

### So verwenden Sie es

1. **Initiale Dokumentvorbereitung und Konfiguration der Embedding-Pipeline**  
   Laden Sie Ihr Dokument (PDF, Markdown, TXT oder HTML) in das Tool. Konfigurieren Sie die `embedding_model`-Variable (z. B. `sentence-transformers/all-MiniLM-L6-v2`) und setzen Sie den `similarity_threshold` auf 0,65–0,75 für deutsche Texte (empirisch getestet). Rufen Sie dann die Methode `predict_chapters()` auf, die die Embedding-Matrix berechnet und die Kapitelgrenzen als Liste von Start- und Endindizes ausgibt. Beispiel:  
   ```python
   from chapterpredictor import ChapterPredictor
   predictor = ChapterPredictor(model_name="all-MiniLM-L6-v2", threshold=0.7)
   chapters = predictor.predict("dokument.txt")
   ```

2. **Feinjustierung der Cluster-Parameter für mehrsprachige Korpora**  
   Wenn Ihr Dokument gemischtsprachig ist (z. B. Deutsch und Englisch), setzen Sie `language_mix=True` und definieren Sie ein `stop_words_dict` mit sprachspezifischen Stop-Wörtern. Rufen Sie `cluster_topics()` auf, um die LDA-Modellparameter (z. B. `num_topics=10`) anzupassen. Das Tool gibt ein DataFrame mit Themen-Clustern und deren Relevanz-Scores aus. Beispiel:  
   ```python
   df_clusters = predictor.cluster_topics(num_topics=8, language_mix=True)
   print(df_clusters[['chapter_id', 'topic_label', 'relevance']])
   ```

3. **Export der segmentierten Kapitel mit Metadaten in JSON-Format**  
   Nach erfolgreicher Segmentation rufen Sie `export_chapters()` auf. Definieren Sie das Output-Format als `json` und aktivieren Sie `include_abstracts=True`. Das Tool generiert eine strukturierte JSON-Datei mit Feldern wie `chapter_id`, `start_index`, `end_index`, `abstract` und `topic_label`. Diese Datei kann direkt in ein CMS oder eine API integriert werden. Beispiel:  
   ```python
   predictor.export_chapters(output_format="json", include_abstracts=True, file_name="kapitel.json")
   ```

### Beste Praktiken

1. **Batch-Verarbeitung für große Dokumente mit Memory-Caching**  
   chapterpredictor kann Dokumente > 10.000 Token verarbeiten, aber für maximale Performance empfehlen wir die **Batch-Verarbeitung** mit einer Chunk-Größe von 2.048 Token. Aktivieren Sie das **Memory-Caching** über `cache_dir="./cache"`, um wiederholte Embedding-Berechnungen zu vermeiden. Das reduziert die Laufzeit um bis zu 50 % bei wiederholten Analysen.  
   *Warum?* Große Embedding-Matrizen (z. B. 500 Segmente x 768 Dimensionen) belasten den RAM. Caching vermeidet unnötige Neuberechnungen.

2. **Hybride Segmentierung: Embedding + regelbasierte Heuristiken für technische Dokumente**  
   Für technische Handbücher oder Code-Dokumentationen kombinieren Sie die Embedding-basierte Segmentierung mit **regelbasierten Heuristiken** (z. B. Erkennung von Überschriftenmustern wie `# Kapitel 1` oder `### 1.1`). Setzen Sie `heuristic_rules=True` und definieren Sie eine Regex-Liste in `heading_patterns`. Diese hybride Methode verbessert die Genauigkeit bei stark formatieren Dokumenten um 20–30 %.  
   *Warum?* Reine Embedding-Segmentierung kann bei konsistenten Überschriftenstrukturen (z. B. LaTeX oder Markdown) versagen, weil der semantische Abstand zwischen Kapiteln zu gering ist.
