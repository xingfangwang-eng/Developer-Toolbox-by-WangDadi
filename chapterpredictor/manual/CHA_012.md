## What is ChapterPredictor?

ChapterPredictor is a production-grade SaaS tool engineered to **automatically predict, segment, and generate chapter-level structures** from long-form content — whether it’s a novel, technical documentation, podcast transcript, or video script. It leverages a hybrid NLP model combining transformer-based sequence labeling with structural coherence scoring, delivering chapter boundaries with sub-sentence precision.

Unlike generic summarizers, ChapterPredictor is purpose-built for **content architects, editors, and AI pipeline developers** who need deterministic chapterization for downstream publishing, indexing, or LLM fine-tuning.

---

## Core Features

### 1. **Semantic Boundary Detection with Coherence Gap Analysis**  
ChapterPredictor doesn’t just look for keywords or paragraph breaks. It analyzes **latent topic shift vectors** using a fine-tuned BERT encoder, then applies a custom coherence gap metric to detect natural chapter breaks. This ensures chapters are semantically meaningful, not just syntactically separated.

### 2. **Adaptive Chapter Schema Generator**  
Define your own chapter schema (e.g., max chapter length, narrative arc type, or topic density constraints). The engine dynamically re-balances predicted chapters to fit your schema, using a **recursive splitting algorithm** that respects both content integrity and structural rules — ideal for publishing pipelines with strict formatting requirements.

### 3. **Multi-Format Export with Embedded Chapter Metadata**  
Export predicted chapters as structured JSON, Markdown TOC, EPUB chapter markers, or SRT-based chapter timestamps for video. Each export includes **confidence scores, topic labels, and sentence-level boundary offsets**, making it trivial to integrate with CMS, video editors, or LLM training datasets.

## How to Use

### 1. **Configure Detection Parameters via YAML or API Payload**  
Start by defining your detection profile. For example, in a YAML config:  

```yaml
chapter_predictor:
  model: "bert-base-chapter-v3"
  min_chapter_length: 500
  max_chapter_length: 3000
  coherence_threshold: 0.65
  schema: "narrative_arc"
  output_format: "json_with_metadata"
```

Or via API:

```json
{
  "text": "<your long-form content>",
  "params": {
    "min_chapter_length": 500,
    "coherence_threshold": 0.65,
    "schema": "narrative_arc"
  }
}
```

This allows you to tune the engine for different genres — from dense technical docs to episodic fiction.

### 2. **Run Prediction and Review the Confidence Heatmap**  
Execute the prediction. The engine returns a **chapter heatmap** — a visual timeline of coherence dips and peaks. Each proposed chapter boundary is annotated with a confidence score (0.0–1.0). You can accept, merge, or split boundaries directly in the dashboard before finalizing. This is critical for editors who need human-in-the-loop control.

### 3. **Export with Metadata for Downstream Integration**  
Once finalized, export the chapter structure. For example, a JSON export includes:

```json
{
  "chapters": [
    {
      "index": 1,
      "start_offset": 0,
      "end_offset": 2450,
      "confidence": 0.94,
      "topic_label": "Introduction to Neural Networks",
      "suggested_title": "The Rise of Deep Learning"
    }
  ]
}
```

You can then pipe this directly into a CMS, an EPUB generator, or a video timeline editor.

## Best Practices

### 1. **Pre-process Content with Consistent Formatting**  
ChapterPredictor performs best when input text is clean and structurally consistent. Strip extraneous HTML, normalize line breaks, and ensure no embedded scripts or markdown artifacts. For best results, run a **text normalization pipeline** (e.g., Unicode normalization, whitespace collapse) before feeding content to the API. This reduces false positives in boundary detection by up to 30%.

### 2. **Use Schema Constraints for Production Pipelines**  
If you're integrating ChapterPredictor into an automated publishing or video production pipeline, **always define a schema** with explicit `min_chapter_length` and `max_chapter_length`. This prevents the engine from generating overly granular chapters (e.g., 50-word snippets) or absurdly long ones that break your UX. For video scripts, pair schema constraints with `coherence_threshold` values between 0.6 and 0.7 to balance granularity with narrative flow.
