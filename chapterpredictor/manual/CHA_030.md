## What is chapterpredictor?

**chapterpredictor** is a deterministic‑yet‑adaptive NLP engine that ingests raw long‑form content (blogs, documentation, transcripts) and returns a semantically segmented chapter tree with confidence scores, timing estimates, and hierarchical headings. It is not a “summarizer” – it reconstructs the logical narrative arc, identifies latent topic transitions, and outputs a structured chapter map ready for CMS ingestion, SEO optimization, or audio/video chapter markers.

---

## Core Features

### 1. Latent Topic Boundary Detection with Confidence Calibration
Unlike naive sliding‑window approaches, chapterpredictor uses a custom transformer‑based **transition‑aware segmentation model** trained on 50k+ multi‑genre documents. It outputs not only chapter boundaries but a **boundary confidence score** (0.0–1.0) per segment. This allows downstream systems to auto‑merge low‑confidence splits or flag ambiguous transitions for human review.

### 2. Hierarchical Chapter Tree with Weighted Headings
The tool generates a **multi‑level chapter hierarchy** (H1 → H2 → H3) derived from intra‑document semantic similarity and lexical cohesion. Each node carries a **structural weight** – enabling automatic generation of nested tables of contents, breadcrumb trails, and JSON‑LD `hasPart` markup for search engines.

### 3. Real‑time Streaming Chapter Preview
chapterpredictor exposes a **WebSocket‑enabled streaming endpoint** that emits partial chapter predictions as the model processes content in chunks. This enables live previews in editors, progressive rendering in video chapter tools, and early‑termination logic for cost‑sensitive pipelines.

## How to Use

### Step 1: Install the CLI and Configure the API Key
```bash
npm install -g chapterpredictor
chapterpredictor config --api-key YOUR_API_KEY
```
The CLI automatically reads `CHAPTERPREDICTOR_API_KEY` from your environment. For CI/CD pipelines, use `--env-file` to load credentials from a vault‑managed `.env` file. The tool validates the key against the `/v1/auth/verify` endpoint before any operation.

### Step 2: Run Chapter Prediction with Custom Segmentation Sensitivity
```bash
chapterpredictor run input.txt \
  --output-format json \
  --min-chapter-length 300 \
  --boundary-threshold 0.65 \
  --hierarchy-depth 2
```
- `--min-chapter-length` (chars) prevents sub‑300‑character fragments from becoming standalone chapters.  
- `--boundary-threshold` (0–1) controls how aggressive the model is at splitting topics. Lower values yield more granular chapters.  
- `--hierarchy-depth` limits the tree to `H1` and `H2` levels, ideal for blog posts or documentation.

The JSON output includes `chapters[]` with `start_offset`, `end_offset`, `confidence`, `heading`, and `children[]`.

### Step 3: Integrate Streaming Chapter Preview into Your Editor
```javascript
import { ChapterPredictorStream } from 'chapterpredictor/client';

const stream = new ChapterPredictorStream({
  apiKey: process.env.API_KEY,
  onChapter: (chapter) => {
    // chapter.heading, chapter.confidence, chapter.start_offset
    appendToEditorPreview(chapter);
  },
  onComplete: (tree) => {
    buildTableOfContents(tree);
  },
});

await stream.feed(document.body.innerText);
```
The `ChapterPredictorStream` class handles WebSocket reconnection, backpressure, and automatic chunking. You can call `stream.abort()` to cancel mid‑prediction without wasting API quota.

## Best Practices

### 1. Pre‑normalize Content for Higher Boundary Accuracy
Strip extraneous HTML tags, inline styles, and non‑semantic line breaks before feeding content to chapterpredictor. Use a library like `@mozilla/readability` to extract the main article body. The model performs best when input is a clean, contiguous text stream – markdown is preferred over raw HTML. This alone can improve boundary F1‑score by 8–12%.

### 2. Use the Chapter Tree for Automated Breadcrumb + Schema Generation
After receiving the hierarchical chapter tree, map each heading level to a `BreadcrumbList` item and inject `hasPart` schema into your page’s `<head>`. This directly impacts Google’s “Key moments” rich results and improves CTR for long‑form content. Example mapping: `H1 → item.name`, `H2 → item.description`, `H3 → item.position`.
