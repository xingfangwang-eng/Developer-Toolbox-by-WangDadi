# chapterpredictor - CHA_018

## 概要

chapterpredictor は、コンテンツの章構成を自動予測・生成するためのプロフェッショナル向け SaaS ツールです。このツールは、ドキュメント、ブログ記事、技術書、e ラーニング教材など、あらゆるテキストベースのコンテンツに対して、最適な章立てとセクション分割を AI で提案します。SEO 対策としても、構造化されたコンテンツが検索エンジンに評価されやすいという特性を活かし、内部リンク構造の最適化にも貢献します。

## 詳細な内容

### chapterpredictor とは？

chapterpredictor は、機械学習モデルを用いてテキストのトピック遷移や重要度を解析し、人間の編集者では見落としがちな論理的な章構成を自動生成するツールです。単なる見出し生成ツールではなく、コンテンツの流れを考慮した「ストーリーテリングの設計図」を提供します。特に、大量のコンテンツを扱うメディア企業や、技術ドキュメントの標準化が必要な SaaS 企業において、その真価を発揮します。

### コア機能

1. **動的トピッククラスタリングとセクション自動分割**
   入力されたテキストに対して、自然言語処理（NLP）エンジンがトピックの変化点を検出し、自動的にセクションを分割します。さらに、各セクションに最適な見出し（H2/H3 レベル）と、そのセクションがカバーするキーワードの重要度スコアを出力します。これにより、SEO に強い階層構造を数秒で構築できます。

2. **コンテキストアウェアな章順序最適化エンジン**
   単なる分割ではなく、読者の理解度や学習曲線を考慮した章の並び替えを提案します。例えば、技術解説記事であれば「基礎→応用→実践」の流れを、物語性のあるコンテンツであれば「導入→対立→解決」の構造を自動で認識し、最適な順序を提示します。このエンジンは、過去の類似コンテンツのパフォーマンスデータ（滞在時間、離脱率）も参照します。

3. **マルチリンガル対応の見出し生成とメタデータ抽出**
   日本語、英語、中国語、ドイツ語、スペイン語など、主要 10 言語に対応。各言語の文法ルールや SEO ベストプラクティスに従った見出しを生成します。さらに、生成された各章に対して、メタディスクリプション候補や内部リンクに適したアンカーテキストも自動生成。これにより、国際展開するコンテンツの一貫性と品質を保証します。

### 使い方

1. **API 経由でのコンテンツ投入と解析パラメータ設定**
   まず、`POST /api/v1/chapter/analyze` エンドポイントに、解析したいテキスト（Markdown またはプレーンテキスト）と、ターゲット言語コード（例：`ja`）、および出力する章の最大数（例：`max_chapters: 8`）を JSON で送信します。この際、`tone` パラメータで「フォーマル」「カジュアル」「技術寄り」などのスタイル指定も可能です。レスポンスとして、解析ジョブの ID が返却されます。

   ```json
   {
     "text": "ここに解析したい本文を入力...",
     "language": "ja",
     "max_chapters": 8,
     "tone": "technical"
   }
   ```

2. **生成された章構成のレビューと手動調整**
   解析が完了すると、`GET /api/v1/chapter/result/{job_id}` で結果を取得できます。結果は、各章の見出し、開始位置、推定重要度スコア、そして推奨される内部リンク先のリストを含みます。Web ダッシュボード上では、ドラッグ＆ドロップで章の順序を変更したり、特定のセクションを結合・分割したりする編集機能を提供。編集後は、`PUT /api/v1/chapter/refine` で調整内容を反映させ、再度スコアリングを実行できます。

3. **生成結果のエクスポートと CMS 連携**
   最終的な章構成は、Markdown、JSON、または WordPress や Contentful などの主要 CMS 向けの専用フォーマットでエクスポート可能です。エクスポート時に、各見出しに自動生成されたメタデータ（SEO タイトル、メタディスクリプション）を埋め込むオプションも提供。さらに、`POST /api/v1/chapter/publish` を使用すれば、連携済みの CMS に直接下書きとして投稿することもできます。

### ベストプラクティス

1. **長文コンテンツの事前セグメンテーション**
   一度に 10,000 文字を超えるテキストを投入する場合は、事前に 3,000〜5,000 文字程度のセグメントに分割してから API に送信することを推奨します。これにより、トピッククラスタリングの精度が向上し、より粒度の細かい章構成が得られます。分割した各セグメントは、`session_id` パラメータで紐付けることで、全体として一貫性のある構成を維持できます。

2. **定期的なモデル再学習とフィードバックループの構築**
   chapterpredictor は、ユーザーが手動で調整した章構成のデータをフィードバックとして学習できます。週次または月次で、`POST /api/v1/model/feedback` に、元の自動生成結果とユーザーが最終的に採用した構成の差分を送信することで、プロジェクト固有の文体や業界用語に特化したカスタムモデルを構築できます。このフィードバックループを 3 ヶ月以上継続すると、初期状態と比較して 40% 以上の構成精度向上が確認されています。
