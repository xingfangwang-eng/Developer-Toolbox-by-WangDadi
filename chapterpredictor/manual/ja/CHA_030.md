# chapterpredictor - CHA_030

## 概要

chapterpredictor は、機械学習ベースのコンテンツ構造解析エンジンです。動画・ドキュメント・ポッドキャストなどのシーケンシャルデータから、論理的な区切り（チャプター）を自動検出・予測・生成します。単なるタイムスタンプ分割ではなく、意味論的セグメンテーションを実現します。

## 詳細な内容

### chapterpredictor とは？

chapterpredictor は、コンテンツの流れを解析し、**「いつ」「なぜ」「どのように」** チャプターが区切られるべきかを予測するツールです。自然言語処理と時系列パターン認識を組み合わせ、従来のルールベース分割では捉えられなかった文脈の変化点を特定します。動画編集者、コンテンツマーケター、Podcast プロデューサーに最適です。

### コア機能

1. **セマンティック・チャプタリング**
   - BERT ベースの埋め込みベクトルを時系列で比較し、意味的トピックの遷移点を検出。従来の「沈黙検出」や「キーワード出現」に依存せず、文脈の変化を捉えます。
   - 例: インタビュー動画で「質問の切り替わり」や「話題の転換」を、会話の流れから正確に予測。

2. **動的チャプター要約生成**
   - 各チャプターの内容を抽出型・抽象型のハイブリッドで要約。GPT モデルと組み合わせ、チャプタータイトルと 1 行サマリーを自動生成します。
   - 既存のチャプター情報がないコンテンツでも、後付けで構造化が可能。

3. **ストリーム対応リアルタイム予測**
   - ライブ配信や長時間録画に対応。チャンク単位で逐次処理し、遅延 500ms 以下でチャプター境界を提案。WebSocket 経由でリアルタイムフィードバックを提供します。

### 使い方

1. **データ前処理とモデルロード**
   - 入力データ（動画の音声テキスト、Markdown 文書、Podcast トランスクリプト）を chapterpredictor の `load_pipeline()` に渡します。内部で Whisper による音声認識（動画の場合）と、Sentence-BERT による文書ベクトル化が自動実行されます。
   - コード例:
     ```python
     from chapterpredictor import ChapterPipeline
     pipeline = ChapterPipeline(model_name="jp-bert-base")
     segments = pipeline.load("input_video.mp4", source_type="video")
     ```

2. **予測パラメータ調整**
   - `predict_chapters()` メソッドで、感度（`sensitivity=0.7`）と最小チャプター長（`min_chapter_sec=30`）を指定。`sensitivity` が高いほど細かい区切りが検出されます。
   - 出力は JSON 形式で、`start_time`, `end_time`, `topic_label`, `summary` を含むリスト。
     ```json
     [
       {"start": 0.0, "end": 45.2, "label": "導入", "summary": "ツールの概要説明"},
       {"start": 45.2, "end": 120.0, "label": "設定方法", "summary": "初期設定の手順を解説"}
     ]
     ```

3. **結果のエクスポートと編集**
   - 予測結果を `export_chapters()` で SRT、VTT、CSV、またはカスタムテンプレートに出力。GUI モードでは、予測境界をドラッグ＆ドロップで微調整可能。編集後は `save_adjustments()` でフィードバックをモデルに反映させ、次回の予測精度を向上させます。

### ベストプラクティス

- **ノイズ除去と前処理の徹底**
  - 動画コンテンツでは、BGMや効果音が予測精度を低下させます。chapterpredictor の `AudioCleaner` モジュールを事前に適用し、音声区間のみを抽出してからパイプラインに渡すことを推奨します。`pipeline.enable_denoiser(True)` で組み込み処理も利用可能。

- **ドメイン適応ファインチューニング**
  - 汎用モデルでも高精度ですが、特定のジャンル（例: テクニカルチュートリアル、ニュース番組）で使う場合は、少量の教師データ（20〜50 の正解チャプター）でファインチューニングすると、`sensitivity` を 0.5 以下に下げても精度が維持されます。`pipeline.fine_tune(labeled_data)` で実行可能。
