## What is chapterpredictor?

**chapterpredictor** is a lightweight, API-first SaaS tool that ingests raw text or outlines and predicts optimal chapter boundaries using statistical NLP and pattern recognition. It eliminates manual chapter drafting by analyzing semantic density, topic shift signals, and narrative pacing — delivering a ready-to-use chapter tree for documentation, ebooks, or course modules.

---

## Core Features

### 1. Semantic Boundary Detection Engine
Uses a hybrid of TF-IDF cosine similarity and sliding-window entropy to detect natural topic shifts. Unlike naive paragraph counters, it identifies where a new chapter *should* begin based on lexical divergence and concept drift — not just whitespace.

### 2. Adaptive Chapter Depth Calibration
Dynamically adjusts the number of predicted chapters based on document length, readability score, and target audience complexity (e.g., beginner vs. expert). Supports configurable depth thresholds via JSON config or CLI flags.

### 3. Multi-Format Export with SEO Metadata Injection
Exports predicted chapter structures as Markdown, JSON, or HTML. Each chapter includes auto-generated H2/H3 headings, meta descriptions, and keyword density suggestions — ready for direct integration into static site generators or CMS platforms.

## How to Use

### Step 1: Prepare Your Source Content
Feed the tool a plain text file, a Markdown document, or a URL pointing to raw content. Ensure no extraneous formatting (e.g., inline styles, embedded images) — the parser expects clean semantic text. For best results, strip boilerplate headers and footers.

```bash
chapterpredictor --input ./draft.md --lang en
```

### Step 2: Run the Prediction with Custom Sensitivity
Adjust the `--sensitivity` flag (range 0.1–1.0) to control how aggressively chapter boundaries are split. A lower value (0.2) yields fewer, broader chapters; a higher value (0.8) produces granular segments. For technical documentation, start with 0.5.

```bash
chapterpredictor --input ./draft.md --sensitivity 0.6 --output chapters.json
```

### Step 3: Validate and Export the Chapter Tree
Review the generated JSON structure for logical flow. Use `--export-format md` to output a ready-to-publish Markdown file with frontmatter and SEO tags. Optionally pipe into a linter to check heading hierarchy.

```bash
chapterpredictor --input ./draft.md --export-format md --output ./chapters.md
```

## Best Practices

### 1. Pre-process with a Content Cleaner
Run your source text through a deduplication and whitespace normalizer before feeding it to chapterpredictor. Inconsistent line breaks or repeated boilerplate (e.g., “back to top” links) can skew the boundary detection algorithm. Use `sed` or a lightweight Python script for this.

### 2. Pair with a Topic Modeling Pipeline for Domain-Specific Content
For highly technical or niche content (e.g., API docs, medical manuals), combine chapterpredictor with a topic modeling step (e.g., LDA or BERTopic). This refines chapter labels and improves the semantic coherence of each segment — especially when dealing with multi-domain documents.
