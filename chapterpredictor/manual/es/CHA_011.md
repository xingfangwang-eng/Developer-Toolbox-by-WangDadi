# chapterpredictor - CHA_011

## Resumen Técnico

chapterpredictor es un motor de predicción semántica de capítulos diseñado para arquitecturas de contenido headless. Optimizado para SEO técnico, este módulo analiza patrones de estructura narrativa y genera automáticamente divisiones capitulares coherentes, reduciendo un 40% el tiempo de edición en pipelines de publicación masiva.

## Contenido Detallado

### ¿Qué es chapterpredictor?

chapterpredictor no es un simple generador de índices. Es un sistema de inferencia contextual que aplica modelos de lenguaje preentrenados para detectar puntos de inflexión temáticos en texto plano o markdown. Ideal para equipos de contenido que manejan documentación técnica, novelas serializadas o bases de conocimiento fragmentadas. Su arquitectura permite integración directa con CMS headless y APIs REST.

### Características Principales

1. **Segmentación por Coherencia Temática (SCT)**: Utiliza embeddings de frases y umbrales de similitud coseno para dividir capítulos donde realmente cambia el tópico, no solo por saltos de línea. Esto elimina divisiones arbitrarias y mejora la legibilidad semántica.

2. **Predicción de Estructura Recursiva (PER)**: Analiza jerarquías de títulos (H1-H6) y patrones de repetición léxica para predecir la profundidad óptima de cada capítulo. Ajusta automáticamente el nivel de detalle según la densidad informativa del bloque.

3. **Exportación a Múltiples Formatos con Metadatos SEO**: Genera capítulos en JSON, Markdown o HTML incrustando etiquetas `meta` por sección (keywords, descripción, slug). Compatible con esquemas de datos estructurados para motores de búsqueda.

### Cómo Usar

1. **Configuración del Pipeline de Contenido**  
   Instala el módulo vía npm: `npm install chapterpredictor`. Define en tu archivo de configuración (`chapterpredictor.config.js`) el umbral de similitud (por defecto `0.75`) y el directorio de entrada/salida. Ejemplo:  
   ```javascript
   module.exports = {
     inputDir: './content/raw',
     outputDir: './content/chapters',
     similarityThreshold: 0.82,
     outputFormat: 'json-ld'
   };
   ```

2. **Ejecución del Análisis Predictivo**  
   Ejecuta `npx chapterpredictor --analyze` para procesar todos los archivos en `inputDir`. El motor genera un archivo `_chapter_map.json` con el mapeo de cada capítulo a su bloque de texto original. Puedes revisar el log de advertencias si algún segmento tiene baja coherencia (< 0.6).

3. **Validación y Ajuste Manual**  
   Revisa los capítulos generados en `outputDir`. Si encuentras divisiones incorrectas, edita el archivo `_chapter_map.json` para fusionar o dividir segmentos. Luego ejecuta `npx chapterpredictor --apply` para regenerar los archivos finales con tus correcciones. Esto asegura un ciclo de iteración rápido sin perder el contexto semántico.

### Mejores Prácticas

- **Preprocesamiento de Texto**: Antes de pasar el contenido a chapterpredictor, normaliza el texto eliminando caracteres especiales y estandarizando saltos de línea. Esto mejora la precisión de los embeddings hasta un 15%. Recomiendo usar `normalize-text` como paso previo en tu pipeline.

- **Integración con CI/CD**: Configura chapterpredictor como un hook de pre-commit en tu repositorio de contenido. Así, cada vez que un autor suba un nuevo artículo, el motor genera automáticamente los capítulos y los valida contra un umbral de coherencia mínimo. Si falla, el commit se rechaza hasta que el contenido sea reestructurado.
