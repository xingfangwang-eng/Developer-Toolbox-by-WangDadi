# chapterpredictor - CHA_006

## Resumen Técnico

Este proyecto implementa un motor de predicción semántica de capítulos basado en transformers, optimizado para contenido SEO y estructuras narrativas complejas. La herramienta analiza patrones de coherencia textual y genera fragmentos de capítulos con alta precisión contextual, ideal para automatización de contenido en blogs, documentación técnica y storytelling.

## Funcionalidades Clave

### 1. Predicción de Capítulos con Modelos de Lenguaje Fine-Tuned
chapterpredictor utiliza un modelo BERT-Span-based entrenado con corpus multilingüe (10 idiomas) para predecir el contenido del próximo capítulo a partir de un resumen o keywords. El sistema soporta **ajuste dinámico de ventana de contexto** (hasta 512 tokens) y **weighted attention** para priorizar términos clave SEO.

### 2. Generación de Esquemas Jerárquicos Automáticos
No solo predice capítulos, sino que **estructura jerárquicamente** el contenido: genera títulos H2, H3 y párrafos de ejemplo con densidad de palabras clave controlada. Incluye un validador de **coherencia temática** que evita saltos abruptos entre capítulos.

### 3. Optimización en Tiempo Real para SEO On-Page
Integra un módulo de **análisis de intención de búsqueda** que ajusta automáticamente la longitud, tono y frecuencia de palabras clave en cada capítulo generado. Soporta **modo batch** para procesar 100+ capítulos en paralelo con latencia <2s por unidad.

## Instrucciones de Uso (Flujo Profesional)

### 1. Configuración del Contexto de Predicción
Inicializa el objeto `ChapterPredictor` con parámetros de idioma (`lang='es'`), modelo base (`model_path='./models/bert-base-spanish'`) y umbral de confianza (`confidence_threshold=0.85`). Usa el método `set_context()` para cargar un archivo de texto o string con el contenido previo. Ejemplo:
```python
predictor = ChapterPredictor(lang='es', model='bert-base-spanish')
predictor.set_context('Resumen del capítulo anterior: análisis de tendencias SEO en 2024')
```

### 2. Generación de Predicciones con Parámetros Avanzados
Ejecuta `predict_chapter()` con `num_tokens=300` y `temperature=0.7` para controlar creatividad. Usa `keywords=['chapter prediction', 'content generation']` para forzar inclusión semántica. El método retorna un objeto `Chapter` con atributos `title`, `summary` y `probability_score`.
```python
chapter = predictor.predict_chapter(num_tokens=300, temperature=0.7, keywords=['predicción de capítulos'])
print(chapter.title, chapter.probability_score)
```

### 3. Validación y Refinamiento Automático
Después de la generación, llama a `validate_coherence()` que compara el capítulo con el contexto usando **cosine similarity** (umbral >0.6). Si falla, el sistema ejecuta automáticamente `refine_chapter()` con un **algoritmo de retropropagación semántica** que ajusta embeddings hasta cumplir el umbral.
```python
if predictor.validate_coherence(chapter, context_vector):
    chapter.save('output/capitulo_5.md')
else:
    refined = predictor.refine_chapter(chapter, iterations=3)
```

## Mejores Prácticas para Desarrolladores

### 1. Integración con Pipelines CI/CD
Para equipos de contenido automatizado, recomiendo envolver chapterpredictor en un **microservicio FastAPI** con endpoints REST. Usa el modo `async` para procesar lotes de capítulos en paralelo y cachea los modelos en memoria con `torch.jit.script` para reducir latencia en producción.

### 2. Estrategia de Fine-Tuning por Nicho
No uses el modelo base genérico para dominios especializados (ej. medicina, fintech). En lugar de eso, entrena un adaptador LoRA con 500 ejemplos de tu corpus. El repositorio incluye un script `train_lora.py` que ajusta el modelo en menos de 1 hora con una GPU T4, logrando un incremento del 23% en precisión de predicción.
