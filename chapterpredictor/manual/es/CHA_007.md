# chapterpredictor - CHA_007

## Resumen Técnico

chapterpredictor es un motor de predicción estructural de contenido basado en NLP, diseñado para arquitectos de contenido y equipos de SEO técnico. Permite anticipar la segmentación de capítulos en documentos extensos, optimizando la navegación semántica y la indexación diferencial en motores de búsqueda. Este proyecto resuelve el problema de la fragmentación de contenido no estructurado, generando automáticamente una jerarquía de capítulos predictiva que mejora el tiempo de carga cognitivo del usuario y la densidad de palabras clave por sección.

## Contenido Detallado

### ¿Qué es chapterpredictor?

chapterpredictor es una herramienta de análisis predictivo de capítulos que utiliza modelos de lenguaje entrenados en corpus técnicos y de documentación. A diferencia de los simples generadores de esquemas, este sistema evalúa la coherencia temática y la intención de búsqueda para proponer una estructura de capítulos que maximice la retención de usuarios y la relevancia SEO. Está diseñado para integraciones serverless y despliegues en entornos cloud-native, con una API RESTful que soporta peticiones asíncronas para procesamiento de grandes volúmenes de texto.

### Características Principales

- **Segmentación Semántica Predictiva con Transformer**: Utiliza una arquitectura BERT fine-tuned para identificar puntos de quiebre temáticos en tiempo real. El modelo no solo divide el texto, sino que asigna un peso de relevancia a cada capítulo basado en la frecuencia de términos LSI y la densidad de entidades nombradas, permitiendo priorizar secciones para indexación diferencial en Google.
- **Generación de Metadatos de Capítulo Automatizada**: Cada capítulo predicho incluye metadatos enriquecidos: título optimizado para SERP, extracto de 160 caracteres, y etiquetas h1/h2 sugeridas. Estos metadatos se generan mediante un pipeline de NLG que respeta la coherencia semántica del documento original, evitando keyword stuffing y cumpliendo con las directrices E-E-A-T.
- **Validación de Estructura contra Competidores**: Integra un módulo de competitive gap analysis que compara la estructura de capítulos generada con las de los top 10 resultados de Google para la misma keyword cluster. El sistema ajusta la segmentación para cubrir lagunas de contenido no abordadas por la competencia, mejorando la probabilidad de rankear en featured snippets.

### Cómo Usar

1. **Configuración del Pipeline de Predicción**: Clona el repositorio y ejecuta `npm run setup-pipeline`. Esto instalará las dependencias de Node.js (v18+), descargará el modelo pre-entrenado desde el registro de Hugging Face, y configurará la conexión a tu base de datos vectorial (Weaviate o Pinecone). Luego, define el archivo `config.yaml` con los parámetros de chunking (tamaño de ventana: 512 tokens, solapamiento: 128 tokens) y el umbral de similitud coseno (0.75) para agrupar segmentos.
2. **Ejecución del Análisis Predictivo**: Proporciona tu contenido en formato Markdown o JSON (estructura de array de objetos con `{id, text, url}`) mediante una petición POST a `/api/v1/predict`. El endpoint acepta un body con `{content: string, lang: "es", model: "chapterpredictor-v2"}`. La respuesta devuelve un array de objetos `{chapter_id, title, start_index, end_index, weight, metadata}`. Para documentos de más de 10,000 tokens, usa el endpoint asíncrono `/api/v1/predict/async` que devuelve un `job_id` para consultar el estado con GET `/api/v1/jobs/{job_id}`.
3. **Post-procesamiento y Exportación**: Una vez obtenida la predicción, puedes exportar la estructura en formato JSON-LD para schema.org (tipo `Article` con `hasPart`). Usa el comando `chapterpredictor export --format json-ld --input predictions.json --output article-schema.json`. Para integración directa en CMS (WordPress, Strapi), ejecuta `chapterpredictor deploy --cms wordpress --site-url https://tusitio.com --api-key WP_API_KEY`. El sistema actualizará automáticamente los campos de título y extracto de cada capítulo en el editor visual.

### Mejores Prácticas

- **Optimización de Chunking para Documentos Técnicos**: Para manuales de API o documentación de SDK, configura el tamaño de ventana en 256 tokens con un solapamiento de 64 tokens. Esto evita que las definiciones de funciones se dividan en capítulos separados, manteniendo la coherencia del contexto. Además, activa el flag `--preserve-code-blocks` en la configuración para que el modelo no intente predecir capítulos dentro de bloques de código, lo que degradaría la precisión de la segmentación.
- **Estrategia de Indexación Diferencial**: Usa los pesos de capítulo (`weight`) generados por el modelo para priorizar la indexación en Google. Los capítulos con peso superior a 0.8 deben ser marcados con `noindex` si son introductorios y de poco valor, o con `index` si contienen contenido único. Implementa esto mediante la etiqueta `<meta name="robots" content="max-snippet:-1, max-image-preview:large">` en los capítulos de alto peso. Para capítulos con peso entre 0.5 y 0.8, usa `max-snippet:50` para limitar el fragmento mostrado y evitar canibalización de keywords.
