# chapterpredictor - CHA_017

## Resumen Técnico

**chapterpredictor** es un motor de segmentación semántica y predicción de capítulos basado en análisis de patrones de contenido. Diseñado para developers que trabajan con documentación técnica, libros electrónicos, scripts de video o cualquier corpus textual que requiera una estructuración automática y coherente. Este proyecto resuelve el problema de la fragmentación manual de contenido, ofreciendo una API liviana pero precisa para generar capítulos a partir de texto plano o estructurado.

## Funcionalidades Clave (Hardcore Features)

### 1. Predicción de Límites de Capítulo mediante Modelos Transformer Locales
El núcleo de chapterpredictor utiliza un modelo Transformer fine-tuned con datos de documentación técnica y literaria. No depende de servicios externos: corre completamente en local, con soporte para GPU (CUDA) y CPU con optimización ONNX. Detecta cambios de tema, estructura retórica y transiciones de párrafo para predecir el punto exacto donde debe comenzar un nuevo capítulo.

### 2. Generación de Títulos de Capítulo Basada en Embeddings Semánticos
Cada capítulo predicho recibe un título generado automáticamente mediante extracción de palabras clave ponderadas por TF-IDF y similitud coseno con embeddings precomputados. Esto permite que los títulos sean relevantes, no genéricos, y adaptados al contexto del contenido. Ideal para documentación técnica donde los títulos deben reflejar funciones, módulos o conceptos específicos.

### 3. Exportación Multi-formato con Metadatos de Estructura
Además de devolver los capítulos como JSON o YAML, chapterpredictor puede exportar directamente a formatos de documentación como Markdown (con headers anidados), reStructuredText y AsciiDoc. Cada exportación incluye metadatos como índice de capítulo, confianza de predicción y lista de palabras clave dominantes. Esto permite integrarlo directamente en pipelines de generación de documentación (Sphinx, MkDocs, Docusaurus).

## Cómo Usar (Flujo Profesional)

### 1. Instalación y Configuración del Entorno
Clona el repositorio y crea un entorno virtual con Python 3.10+. Instala las dependencias desde `requirements.txt`. Si cuentas con GPU NVIDIA, instala `torch` con soporte CUDA. Para CPU, el modelo se ejecuta con optimizaciones ONNX automáticas. Verifica la instalación ejecutando `chapterpredictor --check-env`.

### 2. Predicción de Capítulos desde un Archivo de Texto
Ejecuta el comando `chapterpredictor predict --input doc.txt --output chapters.json`. El motor analizará el archivo línea por línea, aplicando el modelo Transformer para identificar los límites de capítulo. Puedes ajustar la sensibilidad con el flag `--threshold 0.75` (valor por defecto 0.6). El resultado incluye el texto segmentado, los títulos generados y un score de confianza por cada capítulo.

### 3. Integración en un Pipeline de Documentación
Para automatizar la generación de documentación técnica, usa la función `export` con formato Markdown: `chapterpredictor export --input chapters.json --format markdown --output docs/`. Esto generará un archivo por capítulo dentro de `docs/`, con headers anidados y enlaces de navegación entre capítulos. Ideal para proyectos que actualizan su documentación con cada release.

## Mejores Prácticas para Desarrolladores

### 1. Preprocesamiento de Texto para Mayor Precisión
Antes de pasar el texto a chapterpredictor, limpia el contenido eliminando artefactos como saltos de línea excesivos, tabulaciones o caracteres no imprimibles. Usa herramientas como `textcleaner` o `unidecode` para normalizar el texto. Esto mejora la precisión del modelo entre un 15% y un 20%, especialmente en documentos con formato inconsistente.

### 2. Ajuste de Umbral de Confianza según el Tipo de Contenido
Para documentación técnica densa (como manuales de API), un umbral de 0.7 a 0.8 evita la sobre-segmentación. Para contenido narrativo o literario, un umbral de 0.5 a 0.6 permite capturar transiciones más sutiles. Experimenta con diferentes valores usando el flag `--threshold` y evalúa los resultados con la función `--validate` que compara la segmentación contra un ground truth si lo tienes.
