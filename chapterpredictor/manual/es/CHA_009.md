# chapterpredictor - CHA_009

## Resumen

chapterpredictor es un motor de predicción estructural de contenidos basado en NLP, diseñado para arquitectos de contenido y desarrolladores SaaS que necesitan segmentar, predecir y generar capítulos coherentes a partir de datos no estructurados. Este proyecto permite transformar cualquier flujo de texto en una jerarquía de capítulos predecible, optimizada para SEO y legibilidad.

## Contenido Detallado

### ¿Qué es chapterpredictor?

chapterpredictor no es un simple generador de esquemas. Es un sistema de inferencia semántica que analiza patrones de coherencia temática, densidad de entidades y relaciones interoracionales para predecir la estructura de capítulos que un autor humano escribiría. Funciona como un orquestador de contenido, ideal para automatizar la creación de documentación técnica, guías de usuario y artículos de blog largos. Está construido sobre un pipeline de procesamiento de lenguaje natural que soporta 10 idiomas, incluyendo español, y se integra directamente en flujos CI/CD de generación de contenido.

### Características Principales

- **Predicción de corte temático con umbral de entropía ajustable**: Utiliza un algoritmo de segmentación basado en entropía de texto. Ajusta el umbral de coherencia (parámetro `--entropy-threshold`) para controlar la granularidad de los capítulos. Un valor bajo (0.1) genera capítulos muy detallados; un valor alto (0.6) agrupa temas amplios. Esto permite adaptar la salida a guías de API (granulares) o manuales de usuario (agregados).

- **Inferencia de título de capítulo mediante extracción de entidades clave (NER)**: No solo divide el texto, sino que extrae la entidad principal de cada segmento (persona, organización, concepto técnico) y la utiliza como título del capítulo. Esto garantiza que los títulos sean descriptivos y ricos en palabras clave, mejorando el SEO on-page de forma orgánica. Soporta modelos de NER preentrenados de spaCy y Hugging Face.

- **Modo de generación por contexto de API REST**: chapterpredictor expone un endpoint `POST /predict` que acepta un JSON con el texto original y metadatos (idioma, umbral, tipo de contenido). La respuesta incluye un array de capítulos con sus títulos, contenido y una puntuación de confianza (`confidence_score`) que indica la fiabilidad de la predicción. Esto permite integrarlo como middleware en cualquier stack de backend (Node.js, Python, Go) para servir contenido estructurado en tiempo real.

### Cómo Usar

1. **Configura el pipeline de segmentación para tu stack backend**  
   Instala el paquete vía npm o pip: `npm install chapterpredictor` o `pip install chapterpredictor`. Luego, inicializa el motor con el idioma objetivo. En tu archivo de configuración, define el modelo de lenguaje (por ejemplo, `es_core_news_md` para español) y el umbral de entropía. Ejemplo de inicialización en Node.js:  
   ```javascript
   const ChapterPredictor = require('chapterpredictor');
   const predictor = new ChapterPredictor({ lang: 'es', entropyThreshold: 0.3 });
   ```
   Esto prepara el motor para procesar texto en español con una granularidad media.

2. **Ejecuta la predicción desde tu flujo de datos**  
   Llama al método `predict(text)` con el contenido sin procesar. El motor devolverá un array de objetos `Chapter`. Cada objeto contiene `title` (string), `content` (string) y `confidence` (float entre 0 y 1). Para un artículo de 5000 palabras, el proceso tarda ~200ms en una instancia t2.micro de AWS. Ejemplo:  
   ```javascript
   const chapters = predictor.predict(largeText);
   console.log(chapters[0].title); // "Configuración del Entorno"
   ```

3. **Integra la salida en tu CMS o generador de sitios estáticos**  
   Mapea el array de capítulos a un formato de tu elección (Markdown, JSON, HTML). Por ejemplo, para generar un archivo Markdown con capítulos:  
   ```javascript
   const markdown = chapters.map(ch => `## ${ch.title}\n\n${ch.content}`).join('\n\n');
   fs.writeFileSync('output.md', markdown);
   ```
   Esto te permite automatizar la generación de documentación técnica o artículos SEO sin intervención manual.

### Mejores Prácticas

- **Ajusta el umbral de entropía según el tipo de contenido**: Para documentación técnica (manuales de API, guías de instalación), usa un umbral bajo (0.15-0.25) para obtener capítulos detallados y evitar mezclar conceptos. Para contenido editorial (blogs, guías de usuario), usa un umbral alto (0.4-0.6) para mantener una narrativa fluida. Ejecuta pruebas A/B con 10 documentos de muestra para encontrar el punto óptimo.

- **Preprocesa el texto con normalización de entidades**: Antes de pasar el texto a chapterpredictor, aplica un pipeline de limpieza que unifique términos técnicos (por ejemplo, "API REST" vs "API Restful") y elimine ruido (código incrustado, marcas HTML). Esto mejora la precisión de la inferencia de títulos y la coherencia de la segmentación. Usa herramientas como `unidecode` o `spaCy` para normalizar.
