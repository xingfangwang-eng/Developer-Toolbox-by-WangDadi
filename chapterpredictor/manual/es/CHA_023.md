# chapterpredictor - CHA_023

## Resumen Técnico

chapterpredictor es un motor de predicción de capítulos basado en NLP diseñado para analizar patrones semánticos y estructurales en contenido textual, generando automáticamente divisiones de capítulos optimizadas para SEO y legibilidad. Ideal para equipos de contenido que necesitan escalar la producción de artículos largos, documentación técnica o guías paso a paso.

## Funcionalidades Clave

### 1. Predicción de Capítulos Basada en Coherencia Semántica
Utiliza embeddings contextuales (modelo BERT fine-tuned) para detectar puntos de ruptura naturales en el texto. No depende de reglas fijas como número de palabras o títulos H2, sino que analiza la cohesión temática entre párrafos, generando capítulos con alto grado de relevancia interna.

### 2. Optimización Automática de Títulos de Capítulos para SEO
Cada capítulo predicho recibe un título generado dinámicamente mediante un algoritmo de extracción de keyphrases (TF-IDF + TextRank). Los títulos se ajustan automáticamente a la intención de búsqueda del usuario final, maximizando la densidad de palabras clave sin sacrificar naturalidad.

### 3. Integración con APIs de CMS y Workflows CI/CD
chapterpredictor expone una API RESTful que permite insertar capítulos predichos directamente en sistemas como WordPress, Contentful o Strapi. Además, soporta hooks de pre-commit para integrarse en pipelines de CI/CD, asegurando que cada nuevo artículo pase por el proceso de predicción antes de publicarse.

## Cómo Usar chapterpredictor en un Entorno de Producción

### Paso 1: Configurar el Modelo de Predicción
Ejecuta el script de inicialización para cargar el modelo preentrenado y ajustar los parámetros de sensibilidad (por ejemplo, `--threshold 0.75` para capítulos más largos y cohesivos). El comando típico en tu terminal:
```bash
python chapterpredictor/initialize.py --model bert-base-multilingual-cased --threshold 0.75
```
Esto descargará los pesos del modelo y generará un archivo `config.json` con los parámetros de predicción.

### Paso 2: Procesar un Documento de Entrada
Usa el cliente CLI para analizar un archivo Markdown o texto plano:
```bash
chapterpredictor predict --input article.md --output chapters.json --lang es
```
El sistema devolverá un JSON estructurado con cada capítulo, su título sugerido, y las oraciones que lo componen. Puedes inspeccionar la salida con `jq` para validar la calidad:
```bash
cat chapters.json | jq '.chapters[] | {title: .title, sentences: (.sentences | length)}'
```

### Paso 3: Integrar en tu CMS o Pipeline
Si usas WordPress con WP-CLI, puedes insertar los capítulos directamente como secciones de un post:
```bash
chapterpredictor publish --input chapters.json --cms wordpress --post-id 1234
```
Para un pipeline CI/CD (ej. GitHub Actions), añade un paso que ejecute la predicción antes del merge:
```yaml
- name: Predict chapters
  run: chapterpredictor predict --input ${{ github.event.pull_request.body }} --output pr_chapters.json
```

## Mejores Prácticas para Equipos de Contenido

### Práctica 1: Ajustar el Threshold Según el Tipo de Contenido
Para tutoriales técnicos o guías paso a paso, usa un threshold bajo (0.5–0.6) para generar capítulos cortos y muy específicos. Para artículos de opinión o análisis profundos, sube el threshold a 0.8–0.9, obteniendo capítulos más largos que cubren un tema completo. Guarda estas configuraciones en archivos por proyecto.

### Práctica 2: Validar la Coherencia con un Segundo Modelo de Resumen
Después de la predicción, ejecuta un resumen automático (por ejemplo, con BART) de cada capítulo. Compara el resumen con el título generado: si hay baja similitud coseno (<0.6), el capítulo probablemente necesita revisión manual. Esto reduce falsos positivos y asegura que cada sección tenga un hilo conductor claro.
