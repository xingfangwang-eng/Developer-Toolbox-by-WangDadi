## What is chapterpredictor?

chapterpredictor is a deterministic + probabilistic chapter inference engine that analyzes raw text, outlines, or fragmented content and generates structured, logically sequenced chapter proposals. Unlike generic outline generators, it uses a hybrid NLP pipeline (BERT embeddings + custom LSTM sequence models) to predict chapter boundaries, thematic transitions, and semantic coherence gaps. It’s designed for developers integrating into CMS platforms, documentation pipelines, or AI writing assistants.

---

## Core Features

### 1. **Semantic Boundary Detection via Transformer Embeddings**
chapterpredictor doesn’t rely on naive keyword frequency. It uses precomputed sentence-level embeddings from a fine-tuned RoBERTa model to detect latent topic shifts. This enables accurate chapter splits even in unstructured or noisy text, with configurable sensitivity thresholds for granularity control.

### 2. **Predictive Chapter Sequencing with Confidence Scoring**
Each predicted chapter is assigned a confidence score (0.0 to 1.0) based on cross-attention alignment with the source text. The engine can output multiple candidate chapter sequences ranked by coherence probability, allowing developers to implement fallback logic or A/B test chapter structures in real time.

### 3. **Customizable Schema Mapping for Multi-Format Output**
You can define output schemas (JSON, Markdown, HTML, or custom AST) and map chapter predictions to specific metadata fields (e.g., `chapter_title`, `summary`, `estimated_reading_time`, `keyword_density`). This makes chapterpredictor plug-and-play for static site generators, e-book compilers, or API-driven content workflows.

## How to Use

### 1. **Initialize the Predictor with a Source Document**
```python
from chapterpredictor import ChapterPredictor

predictor = ChapterPredictor(
    model_path="models/roberta-chapter-v2",
    confidence_threshold=0.65,
    output_schema="markdown"
)

doc = open("manuscript_raw.txt").read()
predictions = predictor.predict(doc)
```
The engine returns a list of `ChapterPrediction` objects, each containing `start_index`, `end_index`, `title_candidate`, and `confidence`.

### 2. **Fine-Tune Prediction Sensitivity via Context Windows**
Adjust the `context_window` parameter to control how far the model looks for semantic continuity. For dense technical documentation, use `context_window=5` (sentences). For narrative prose, use `context_window=12` to capture longer thematic arcs.
```python
predictions = predictor.predict(
    doc,
    context_window=8,
    merge_threshold=0.4  # merge adjacent low-confidence chapters
)
```

### 3. **Export Predictions to Structured Markdown with Metadata**
```python
for i, chap in enumerate(predictions, 1):
    print(f"## {chap.title_candidate}")
    print(f"*Confidence: {chap.confidence:.2f}*")
    print(f"*Estimated read time: {chap.estimated_read_time} min*")
    print(doc[chap.start_index:chap.end_index])
    print("---")
```
This produces a fully formatted chapter outline ready for review or direct publishing.

## Best Practices

### 1. **Preprocess Source Text for Higher Accuracy**
Strip boilerplate headers, footnotes, and navigation elements before feeding text into chapterpredictor. Use a lightweight HTML-to-text parser (e.g., `html2text` or `readability-lxml`) to normalize input. The model performs best on clean, continuous prose with minimal formatting noise.

### 2. **Use Confidence Thresholds to Build a Human-in-the-Loop Pipeline**
Set a lower confidence threshold (e.g., 0.4) for draft generation, then flag chapters below 0.7 for manual review. This hybrid approach reduces editorial overhead while maintaining quality. Integrate with your CI/CD pipeline to auto-generate chapter drafts for PRs or documentation updates.
