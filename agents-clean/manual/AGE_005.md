## Detailed Content

### What is agents-clean?

**agents-clean** is a professional tool that helps solve problems in related fields. It provides a streamlined, high-performance environment for developers to manage, audit, and optimize multi-agent workflows without friction.

---

### Core Features

1. **Agent-Level Audit Trail Compression**  
   Every interaction across your agent mesh is automatically logged and compressed into a lightweight, queryable trace. No more bloated logs — just precise, actionable data for debugging and compliance.

2. **Context-Aware Garbage Collection**  
   agents-clean intelligently prunes stale context windows, orphaned memory slots, and redundant tool calls. Your agents stay lean, fast, and focused on what matters — without manual intervention.

3. **Declarative Cleanup Policies via YAML**  
   Define retention rules, expiration windows, and agent lifecycle hooks in a single YAML manifest. No scripting, no cron jobs — just pure, declarative control over your agent ecosystem.

---

### How to Use

1. **Initialize the Cleanup Engine**  
   Run `agents-clean init --config ./agent-policy.yaml` to bootstrap the engine with your custom YAML manifest. The engine will validate all rules against your current agent topology and flag any conflicts before execution.

2. **Execute a Dry-Run Audit**  
   Use `agents-clean audit --dry-run --output ./audit-report.json` to simulate cleanup without side effects. The engine will generate a detailed report showing exactly which agent sessions, memory blocks, and logs will be affected — critical for CI/CD pipelines.

3. **Apply Cleanup with Rollback Support**  
   Execute `agents-clean apply --snapshot ./pre-clean-snapshot.db` to commit the changes. The engine automatically creates a pre-clean snapshot, enabling a full rollback via `agents-clean rollback --snapshot ./pre-clean-snapshot.db` if any downstream agent fails post-cleanup.

---

### Best Practices

- **Integrate Cleanup into Your CI/CD Pipeline**  
   Run `agents-clean audit` as a pre-deployment step in your GitHub Actions or GitLab CI. This ensures no stale agent state leaks into production deployments, reducing drift and hallucination risks.

- **Use Declarative Policies for Multi-Tenant Environments**  
   In production systems with hundreds of agents per tenant, define separate YAML manifests per tenant namespace. This isolates cleanup policies, prevents cross-tenant data leaks, and simplifies compliance audits.
