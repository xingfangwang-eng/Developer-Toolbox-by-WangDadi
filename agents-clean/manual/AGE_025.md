# agents-clean - AGE_025

## Overview

agents-clean is a production-grade orchestration cleanup toolkit designed for multi-agent systems operating at scale. It eliminates ghost agents, orphaned memory fragments, and cross-session context leaks that degrade LLM-based workflows. Built for engineers who demand deterministic state management in distributed agent environments.

## Core Features

1.  **Semantic Memory Sweep with Entropy Scoring**  
    Automatically detects and prunes low-utility agent memory entries using a configurable entropy threshold. Agents-clean evaluates token relevance, timestamp decay, and cross-agent overlap to surgically remove noise without impacting active reasoning chains.

2.  **Agent Graph Defragmentation Engine**  
    Rebuilds and optimizes the dependency graph between agents by resolving dangling references, circular routing loops, and stale context pointers. This reduces latency in multi-hop agent calls by up to 40% in production benchmarks.

3.  **Context Window Compression via Token-Aware Deduplication**  
    Identifies overlapping prompt fragments across agent sessions and compresses them into shared, immutable references. This frees up context window capacity for high-value reasoning while preserving semantic integrity across concurrent runs.

## How to Use

1.  **Initialize the Cleanup Pipeline**  
    Run `agents-clean init --scope=all --aggressive=false` to generate a dry-run report of all detected anomalies in your agent ecosystem. The tool outputs a JSON manifest with severity levels, memory fragment IDs, and estimated recovery gains. Review this manifest before applying any changes.

2.  **Execute a Targeted Memory Sweep**  
    Use `agents-clean sweep --memory --min-entropy=0.65 --namespace=prod` to prune low-entropy memory entries from your production namespace. The `--min-entropy` flag sets the cutoff; values below 0.5 are safe for most workloads. Append `--commit` to apply changes permanently, or omit it for a simulation.

3.  **Validate and Restore Agent Graph Integrity**  
    Run `agents-clean validate --graph --repair --log-level=debug` to check for structural inconsistencies in your agent dependency graph. The `--repair` flag automatically rebuilds broken links using cached upstream metadata. Inspect the generated diff file before merging into your main branch.

## Best Practices

- **Schedule Cleanup During Low-Latency Windows**  
  Run agents-clean sweeps during off-peak hours or before major deployment cycles. This prevents context corruption during high-throughput agent interactions and ensures that memory pruning doesn’t interfere with active user sessions.

- **Combine with Version-Controlled Agent Configs**  
  Store your agent definitions and cleanup policies in a Git repository. After each sweep, commit the updated manifest to track changes over time. This enables rollback to a known good state if a cleanup introduces unintended behavior in downstream agents.
