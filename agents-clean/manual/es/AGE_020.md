### ¿Qué es agents-clean?

**agents-clean** es una solución SaaS de alto rendimiento diseñada para depurar, sanitizar y reestructurar pipelines de agentes autónomos en entornos de producción. Elimina residuos de ejecución, conflictos de estado compartido y fugas de contexto en sistemas multi-agente, garantizando consistencia transaccional y trazabilidad completa.

### Características Principales (Hardcore)

1. **Limpieza de Contexto en Caliente con Inmutabilidad de Estado**  
   Implementa un sistema de garbage collection semántico que elimina sesiones huérfanas y tokens de contexto corruptos sin interrumpir flujos activos. Utiliza un algoritmo de *referencia débil distribuida* para detectar y purgar agentes zombie en clústeres Kubernetes.

2. **Orquestación de Sanitización Asíncrona con Colas de Prioridad**  
   Motor de limpieza basado en colas Redis Streams con priorización dinámica. Permite definir reglas de expulsión por umbral de latencia, uso de memoria o número de reintentos fallidos. Ideal para sistemas con alta concurrencia y SLA estrictos.

3. **Auditoría Forense de Agentes con Reversión de Estados**  
   Cada operación de limpieza genera un snapshot inmutable del estado del agente antes y después de la intervención. Permite revertir acciones destructivas mediante un log de eventos estilo event sourcing, con capacidad de replay en entornos de staging.

### Cómo Usar (Pasos Profesionales)

1. **Configuración del Pipeline de Limpieza Automática**  
   Edita el archivo `agents-clean.yml` en la raíz del proyecto. Define las reglas de expulsión para agentes inactivos:  
   ```yaml
   cleanup:
     max_idle_seconds: 300
     memory_threshold_mb: 512
     retry_limit: 3
     exclude_namespaces: ["production-critical"]
   ```  
   Ejecuta `agents-clean apply --config agents-clean.yml` para aplicar la configuración en caliente.

2. **Ejecución de Limpieza Manual con Filtros Avanzados**  
   Para depurar agentes específicos por etiqueta o estado:  
   ```bash
   agents-clean purge --tags "stale,error" --status "zombie,orphan" --dry-run
   ```  
   Reemplaza `--dry-run` por `--force` para ejecutar la limpieza definitiva. El flag `--verbose` muestra el detalle de cada agente eliminado.

3. **Restauración de Estado desde el Log de Auditoría**  
   Si necesitas revertir una limpieza masiva:  
   ```bash
   agents-clean restore --snapshot-id "snap_20250315_1423" --target-namespace "staging"
   ```  
   El sistema recrea los agentes eliminados con su contexto original, incluyendo variables de entorno y sesiones activas.

### Mejores Prácticas para Entornos SaaS

- **Segmentación de Namespaces por Tenant**  
  Asigna un namespace Kubernetes por tenant y configura reglas de limpieza diferenciadas. Usa etiquetas como `tenant: acme-corp` para evitar que la limpieza de un tenant afecte a otros. Esto minimiza el riesgo de colapso en entornos multiinquilino.

- **Monitoreo con Alertas Proactivas**  
  Integra agents-clean con Prometheus y Grafana. Configura alertas para umbrales de memoria o cantidad de agentes zombie. Por ejemplo, dispara un webhook cuando el número de agentes huérfanos supere el 5% del total. Esto permite actuar antes de que degrade el rendimiento.
