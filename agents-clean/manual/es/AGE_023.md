# agents-clean - AGE_023

## Resumen Técnico

Herramienta de limpieza y optimización para pipelines de agentes autónomos, diseñada para entornos multi-agente con alta concurrencia. Elimina residuos de ejecución, reconecta sesiones huérfanas y comprime logs de forma inteligente sin perder trazabilidad.

## Contenido Detallado

### ¿Qué es agents-clean?

agents-clean es un utility CLI y daemon diseñado para entornos de agentes autónomos (AutoGPT, CrewAI, LangGraph, etc.). Su núcleo resuelve la fragmentación de estado, la fuga de memoria contextual y la acumulación de artefactos temporales que degradan el rendimiento de los agentes tras ciclos prolongados de ejecución. Está construido en Rust con bindings para Python y JavaScript, ofreciendo una latencia de sub-milisegundo por operación de limpieza.

### Características Principales

1. **Purga Contextual con Preservación Semántica**  
   Analiza el contexto de cada agente usando un modelo ligero de embeddings (all-MiniLM-L6-v2) para identificar y eliminar tokens redundantes o irrelevantes sin perder la coherencia del diálogo. Reduce el tamaño del contexto hasta un 40% en sesiones de más de 50k tokens.

2. **Reconexión de Sesiones Huérfanas vía DAG de Dependencias**  
   Detecta agentes que quedaron en estado `idle` o `zombie` tras fallos de red o timeouts, y los reconecta automáticamente al flujo principal usando un grafo acíclico dirigido (DAG) de dependencias. Incluye rollback atómico para evitar corrupción de datos.

3. **Compresión Log-Eficiente con Indexación Temporal**  
   Convierte logs estructurados (JSON, NDJSON) en un formato binario propietario (`.acl`) que ocupa un 70% menos de espacio. Cada entrada se indexa con timestamps nanosegundo-precisos, permitiendo búsquedas por rango de tiempo sin descomprimir todo el archivo.

### Cómo Usar

#### 1. Instalación y Configuración Inicial del Daemon

```bash
# Instalar vía cargo (Rust) o pip (Python)
cargo install agents-clean --features daemon
# o
pip install agents-clean[daemon]

# Iniciar el daemon en modo background con configuración personalizada
agents-clean daemon --config /etc/agents-clean/config.toml --log-level debug

# Verificar estado del daemon
agents-clean status --json | jq '.connections_active, .bytes_freed'
```

#### 2. Limpieza Programática de un Pipeline Multi-Agente

```python
from agents_clean import Cleaner, ContextPurgePolicy

cleaner = Cleaner(
    policy=ContextPurgePolicy.SEMANTIC,
    min_similarity=0.85,
    max_tokens_per_agent=32000
)

# Ejecutar limpieza en un grupo de agentes específicos
result = cleaner.run(
    agent_ids=["agent-alpha", "agent-beta", "agent-gamma"],
    mode="aggressive",  # conservative | balanced | aggressive
    dry_run=True        # Simular sin aplicar cambios
)

print(f"Tokens eliminados: {result.tokens_purged}")
print(f"Sesiones reconectadas: {result.sessions_reconnected}")
```

#### 3. Monitoreo y Reporte de Eficiencia en Tiempo Real

```bash
# Ver estadísticas en vivo con formato de tabla
agents-clean stats --watch --interval 5s --format table

# Generar un reporte detallado de las últimas 24 horas
agents-clean report --since "2025-03-01T00:00:00Z" --until now --output report.html

# Exportar métricas a Prometheus para integración con Grafana
agents-clean metrics --push-gateway http://prometheus:9091 --job agents-clean-prod
```

### Mejores Prácticas

1. **Implementar una Estrategia de Limpieza Escalonada**  
   No ejecutes `agents-clean` en modo `aggressive` durante picos de carga. Programa limpiezas progresivas: `conservative` cada 10 minutos, `balanced` cada hora, y `aggressive` solo en ventanas de mantenimiento (ej. 3 AM UTC). Esto evita bloqueos en el DAG y mantiene la latencia por debajo de 5ms.

2. **Usar el Flag `--preserve-metadata` en Entornos Regulatorios**  
   Si trabajas con agentes que manejan datos sensibles (HIPAA, GDPR), activa `--preserve-metadata` para conservar cabeceras de auditoría y timestamps de origen. La compresión `.acl` soporta cifrado AES-256-GCM, pero la purga contextual debe configurarse con `min_similarity=0.95` para evitar la eliminación de términos críticos.
