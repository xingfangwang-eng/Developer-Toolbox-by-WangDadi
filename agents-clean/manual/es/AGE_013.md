# agents-clean - AGE_013

## Resumen

**agents-clean** es una herramienta SaaS de arquitectura limpia diseñada para orquestar, depurar y optimizar flujos de trabajo multi-agente en entornos de producción. Ideal para ingenieros que buscan eliminar la fricción en la gestión de agentes autónomos, reducir la latencia en pipelines complejos y garantizar la trazabilidad semántica en sistemas distribuidos. Con un enfoque en la eficiencia computacional y la integración nativa con APIs REST y GraphQL, agents-clean se posiciona como el estándar para equipos que escalan soluciones basadas en inteligencia artificial.

## Características Principales

### 1. **Orquestación Declarativa con DAG Dinámicos**
   - Define flujos de trabajo multi-agente mediante grafos acíclicos dirigidos (DAG) que se construyen en tiempo real. agents-clean permite mapear dependencias entre agentes, ejecutar tareas en paralelo y gestionar estados transaccionales con rollback automático. Ideal para pipelines de ETL, procesamiento de eventos en streaming o simulación de escenarios complejos.

### 2. **Limpieza y Normalización de Contextos en Tiempo Real**
   - Implementa un motor de limpieza que filtra, valida y normaliza datos de entrada/salida de cada agente. Soporta expresiones regulares, transformaciones JSON/XML y detección de anomalías mediante heurísticas configurables. Esto elimina la contaminación de datos en bucles recursivos y garantiza que cada agente reciba entradas predecibles.

### 3. **Monitorización y Trazabilidad Semántica con Logs Estructurados**
   - Cada acción de un agente se registra en un formato estructurado (JSON-LD) que incluye metadatos de contexto, tiempo de ejecución y hash de integridad. agents-clean integra dashboards en Grafana y alertas en Slack/Telegram para detectar cuellos de botella o fallos en tiempo real. Además, ofrece exportación directa a sistemas de observabilidad como OpenTelemetry.

## Cómo Usar

### **Paso 1: Instalación y Configuración del Entorno**
   - Clona el repositorio: `git clone https://github.com/xingfangwang-eng/agents-clean.git`
   - Instala dependencias con `pip install -r requirements.txt` (Python 3.9+) o usa Docker: `docker pull agents-clean:latest`.
   - Configura las variables de entorno en un archivo `.env`:
     ```
     AGENTS_CLEAN_API_KEY=tu_api_key
     AGENTS_CLEAN_LOG_LEVEL=INFO
     AGENTS_CLEAN_MAX_RETRIES=3
     ```
   - Verifica la instalación con `agents-clean --version` y ejecuta un test básico: `agents-clean run --config examples/basic_dag.yaml`.

### **Paso 2: Definir un DAG Multi-Agente**
   - Crea un archivo YAML (ej. `mi_dag.yaml`) que describa los agentes y sus dependencias:
     ```yaml
     dag:
       name: "Pipeline de Procesamiento de Datos"
       agents:
         - id: "extractor"
           type: "http_client"
           config:
             url: "https://api.ejemplo.com/data"
             headers: {"Authorization": "Bearer ${API_KEY}"}
         - id: "limpiador"
           depends_on: "extractor"
           type: "normalizer"
           config:
             rules: ["remove_nulls", "validate_email"]
         - id: "clasificador"
           depends_on: "limpiador"
           type: "ml_classifier"
           config:
             model: "distilbert-base-uncased"
     ```
   - Ejecuta el pipeline: `agents-clean run --config mi_dag.yaml`. agents-clean mostrará el progreso en consola con timestamps y métricas de rendimiento.

### **Paso 3: Depurar y Optimizar con Logs Estructurados**
   - Revisa los logs generados en `logs/` (por defecto en formato JSON-LD). Usa herramientas como `jq` para filtrar errores:
     ```bash
     cat logs/pipeline_20231027.json | jq 'select(.level == "ERROR") | {agent_id, message, timestamp}'
     ```
   - Si un agente falla, agents-clean ejecuta automáticamente reintentos configurables (por defecto 3). Para depurar manualmente, usa `agents-clean debug --config mi_dag.yaml --step limpiador` para aislar el agente y probar con datos mock.

## Mejores Prácticas

### **1. Modulariza tus Agentes con Contextos Aislados**
   - Evita que un agente modifique directamente el estado global. Usa el sistema de contexto de agents-clean para pasar datos entre agentes mediante referencias inmutables. Esto previene efectos secundarios no deseados en pipelines complejos y facilita la replicación de tests unitarios.

### **2. Implementa Pruebas de Carga con Datos Sintéticos**
   - Antes de desplegar en producción, simula escenarios de alta demanda usando el comando `agents-clean stress-test --config mi_dag.yaml --requests 1000`. agents-clean genera informes de latencia (p50, p95, p99) y consumo de memoria. Ajusta los límites de concurrencia (`max_workers`) y los timeouts (`agent_timeout`) según los resultados.
