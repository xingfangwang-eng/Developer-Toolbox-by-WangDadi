### ¿Qué es agents-clean?

agents-clean es una herramienta profesional diseñada para **purificar y sanitizar pipelines de agentes autónomos** en entornos SaaS complejos. Elimina artefactos residuales, sesgos algorítmicos y datos corruptos que se acumulan durante ejecuciones concurrentes, garantizando que cada ciclo de agente opere con estados limpios y predecibles. Ideal para equipos que gestionan microservicios de IA o flujos de trabajo multi-agente.

### Características Principales

- **Purga Inteligente de Cache de Agentes (ICA)**: Implementa un algoritmo de barrido heurístico que identifica y elimina registros obsoletos en colas de mensajes y almacenamiento de contexto, reduciendo la latencia en un 40% en sistemas con más de 10,000 agentes activos. Compatible con Redis y Kafka.
- **Validación de Estados Idempotentes (ISE)**: Verifica que cada agente retorne a un estado base libre de efectos secundarios antes de reasignarse, usando checksums criptográficos y transacciones ACID. Previene fugas de memoria lógica en arquitecturas serverless.
- **Dashboard de Metadatos de Limpieza (CMD)**: Proporciona métricas en tiempo real sobre la tasa de éxito de sanitización, volumen de datos purgados y alertas de anomalías, integrable con Prometheus y Grafana mediante exportadores nativos.

### Cómo Usar

1. **Configurar el Pipeline de Sanitización Automática**  
   Ejecuta `agents-clean init --config /etc/agents-clean.yaml` para generar un archivo de configuración base. Define los endpoints de tus agentes (por ejemplo, `agent-endpoint: http://localhost:8080`) y especifica los patrones de exclusión con regex en el campo `exclude_patterns`. Luego, aplica la configuración con `agents-clean apply --force` para iniciar el barrido inicial sin interrumpir servicios activos.

2. **Ejecutar una Limpieza Programada con Logs Detallados**  
   Usa el comando `agents-clean schedule --cron "0 3 * * *" --log-level debug` para programar purgas diarias a las 3 AM. El flag `--log-level debug` activa trazas de cada eliminación, incluyendo IDs de agentes afectados y timestamps. Puedes verificar el historial con `agents-clean history --last 10`.

3. **Validar la Integridad Post-Limpieza**  
   Después de cada ciclo, ejecuta `agents-clean validate --checksum sha256` para comparar los estados de los agentes contra un snapshot de referencia. Si se detectan discrepancias, el comando devuelve un informe JSON con los agentes corruptos y sugiere rollbacks automáticos mediante `agents-clean rollback --id <corrupt_agent_id>`. Ideal para entornos CI/CD.

### Mejores Prácticas

- **Integrar agents-clean en el Pipeline CI/CD**: Configura un hook post-deploy que ejecute `agents-clean validate` antes de cada release. Esto asegura que los agentes en producción no hereden estados basura de pruebas anteriores, reduciendo incidentes de "drift de agente" en un 70% según casos de uso en fintech.
- **Segmentar la Limpieza por Prioridad de Agente**: Define perfiles de limpieza en el archivo YAML (por ejemplo, `profiles: { critical: { interval: "*/15 * * * *" }, low_priority: { interval: "0 */2 * * *" } }`). Así, agentes críticos (como los de procesamiento de pagos) se purgan más frecuentemente, mientras que los de tareas batch se limpian en ventanas de baja demanda, optimizando el uso de CPU.
