# agents-clean - AGE_005

## Resumen Técnico

Herramienta de limpieza y optimización de agentes SaaS diseñada para entornos multi-idioma y arquitecturas serverless. Elimina residuos de agentes inactivos, reduce la latencia en pipelines de orquestación y asegura la consistencia del estado en despliegues distribuidos. Compatible con Node.js, Python y Go runtimes.

## Funcionalidades Clave

### 1. **Purga Selectiva de Estados Zombie**
Identifica y elimina automáticamente agentes en estado `stale` o `orphaned` mediante un algoritmo de poda basado en timestamps de última interacción y umbrales configurables. Integra hooks con sistemas de colas (RabbitMQ, Kafka) para evitar conflictos durante la limpieza.

### 2. **Reconciliación de Contextos Multi-Tenant**
Detecta y repara inconsistencias en los contextos de ejecución entre diferentes tenants sin interrumpir procesos activos. Utiliza un mecanismo de **snapshot diferencial** que compara el estado actual con la última versión limpia, aplicando parches atómicos.

### 3. **Compresión de Logs con Retención Semántica**
Reduce el volumen de logs generados por agentes en un 70% sin perder trazabilidad. Aplica compresión basada en patrones de eventos recurrentes y mantiene índices de búsqueda en memoria RAM para consultas rápidas. Soporta exportación a formatos Parquet y Avro.

## Cómo Usar

### Paso 1: Configurar el Pipeline de Limpieza Automática
Define un archivo `agents-clean.yml` en la raíz de tu proyecto:

```yaml
clean_policy:
  interval: 300  # segundos
  target_agents: ["worker-*", "cron-*"]
  zombie_threshold: 600  # segundos sin actividad
  context_repair: true
  log_compression:
    retention_days: 30
    format: "parquet"
```

Luego ejecuta:
```bash
agents-clean init --config agents-clean.yml
```

### Paso 2: Ejecutar una Limpieza Manual en Modo Dry-Run
Para validar cambios sin aplicar modificaciones:

```bash
agents-clean run --dry-run --verbose
```
Esto generará un reporte detallado de agentes a eliminar, contextos a reparar y logs a comprimir. Utiliza `--force` solo después de revisar el output.

### Paso 3: Integrar en CI/CD para Despliegues
Agrega un paso en tu pipeline (GitHub Actions, GitLab CI) que ejecute la limpieza post-deploy:

```yaml
- name: Clean agents after deploy
  run: |
    agents-clean run --target-agents "deploy-*" --context-repair --log-compression
  env:
    AGENTS_CLEAN_API_KEY: ${{ secrets.AGENTS_CLEAN_KEY }}
```

## Mejores Prácticas

### 1. **Segmentación por Entorno**
Configura políticas diferentes para `staging` y `production`. En staging, usa umbrales más agresivos (120 segundos) para probar la eliminación de zombies; en producción, mantén 600 segundos para evitar falsos positivos.

### 2. **Monitoreo Post-Limpieza**
Implementa un dashboard con métricas de `agents-clean` usando Prometheus:
- `agents_cleaned_total` (número de agentes eliminados)
- `context_repairs_total` (contextos reparados)
- `log_compression_ratio` (tasa de compresión)
Esto permite detectar picos anómalos que podrían indicar problemas en la orquestación de agentes.
