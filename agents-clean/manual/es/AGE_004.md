# agents-clean - AGE_004

## Resumen Técnico

**agents-clean** es una herramienta de limpieza y normalización de datos diseñada específicamente para entornos SaaS multiinquilino. Su núcleo se basa en pipelines de procesamiento asíncrono que eliminan ruido, duplicados y sesgos en conjuntos de datos generados por agentes automatizados. Ideal para equipos que manejan volúmenes masivos de logs, telemetría o registros de actividad.

## Funcionalidades Clave (Hardcore)

1. **Deduplicación Determinista con Hash Temporal**  
   Implementa un algoritmo de **bloom filter con ventana deslizante** que identifica y elimina registros duplicados en tiempo real, incluso si llegan con latencia variable. Soporta colisiones controladas mediante un hash compuesto por `(tenant_id + timestamp + payload_hash)`.

2. **Normalización Semántica de Esquemas Dinámicos**  
   Detecta automáticamente cambios en la estructura de datos entrantes (nuevos campos, tipos mutados, valores nulos) y aplica reglas de transformación basadas en **JSON Schema Draft 2020-12**. Ideal para microservicios que evolucionan sin control de versiones.

3. **Pipeline de Sanitización con Políticas Configurables**  
   Cada pipeline se define mediante un archivo YAML que permite encadenar filtros (regex, whitelist, blacklist), transformadores (normalización de fechas, encoding de texto) y validadores (umbrales de confianza, reglas de integridad referencial). Todo ejecutado en goroutines con backpressure controlado.

## Cómo Usar (Flujo Avanzado)

1. **Configurar el Pipeline de Limpieza**  
   Crea un archivo `clean-pipeline.yaml` en la raíz del proyecto. Define las fuentes de datos (S3, Kafka, API REST) y las reglas de sanitización. Ejemplo:
   ```yaml
   sources:
     - type: kafka
       topic: raw-events
       consumer_group: agents-clean-group
   steps:
     - filter: remove_nulls
     - transform: normalize_dates
     - validate: threshold_confidence(0.95)
   ```

2. **Ejecutar el Procesamiento Batch o Streaming**  
   Usa el comando `agents-clean run --config clean-pipeline.yaml` para iniciar el proceso. Si trabajas con datos históricos, añade `--mode batch` y especifica un rango de tiempo con `--start 2024-01-01 --end 2024-12-31`. Para streaming en vivo, usa `--mode stream`.

3. **Monitorear y Ajustar en Tiempo Real**  
   Accede al endpoint `/metrics` (puerto 9090 por defecto) para ver métricas en formato Prometheus. Las métricas clave son:
   - `agents_clean_records_processed_total`
   - `agents_clean_duplicates_removed_total`
   - `agents_clean_pipeline_errors_total`

   Si detectas un pico de errores, ajusta el parámetro `max_retries` en el pipeline YAML o incrementa el `buffer_size` en la configuración de Kafka.

## Mejores Prácticas (Industry Standards)

1. **Particionamiento por Tenant**  
   En entornos multiinquilino, fuerza el particionamiento de los datos por `tenant_id` desde la fuente. Esto evita cuellos de botella en la deduplicación y permite escalar horizontalmente. Implementa un `shard_key` basado en el hash del tenant para distribuir la carga en los workers.

2. **Versionado de Pipelines**  
   Cada cambio en el pipeline debe versionarse con un tag semántico (ej: `v1.2.0`). Usa Git para almacenar los archivos YAML y despliega con CI/CD. Esto garantiza trazabilidad y permite rollbacks rápidos si una regla de sanitización rompe la integridad de los datos.
