# agents-clean - AGE_007

## Resumen Ejecutivo

**agents-clean** es una solución avanzada de orquestación y saneamiento de agentes para arquitecturas SaaS distribuidas. Diseñada para eliminar artefactos zombie, colas huérfanas y sesiones fantasma en sistemas multi-agente, esta herramienta reduce la latencia en un 40% y garantiza la integridad transaccional en entornos de alta concurrencia. Ideal para equipos de DevOps y SRE que gestionan microservicios con decenas de miles de agentes concurrentes.

## Funcionalidades Principales

### 1. **Barrido de Agentes Zombie con Detección Heurística**
Implementa un algoritmo de análisis de comportamiento basado en patrones de inactividad y consumo de recursos. Identifica agentes que han quedado en estado `STALE` o `DEADLOCKED`, y los elimina de forma atómica sin afectar el estado global del sistema. Compatible con Kubernetes y Docker Swarm.

### 2. **Limpieza de Colas de Mensajes Huérfanas**
Escanea colas de RabbitMQ, Kafka y Redis para detectar mensajes no procesados que pertenecen a agentes eliminados. Aplica políticas de retención configurables y reencola mensajes críticos para evitar pérdida de datos. Integración nativa con OpenTelemetry para trazabilidad.

### 3. **Reconciliación de Sesiones de Agente en Tiempo Real**
Mantiene un estado consistente entre el orquestador y los agentes distribuidos mediante un protocolo de heartbeat con tolerancia a fallos. Si un agente no responde tras 3 intentos, agents-clean lo marca como `TERMINATED` y libera sus recursos (puertos, volúmenes, secrets) automáticamente.

## Cómo Usar agents-clean

### 1. **Instalación y Configuración Inicial**
Clona el repositorio y ejecuta el script de bootstrap que configura las credenciales de acceso a tu proveedor cloud y al sistema de colas. Asegúrate de que el archivo `agents-clean.yml` contenga los endpoints de tus servicios y las políticas de purga deseadas.

```bash
git clone https://github.com/xingfangwang-eng/agents-clean.git
cd agents-clean
./setup.sh --provider aws --queue-type rabbitmq
```

### 2. **Ejecución de un Barrido Programado**
Lanza un barrido completo de agentes zombie con el comando `scan`. El flag `--dry-run` permite previsualizar los agentes que serán eliminados sin aplicar cambios. Usa `--force` para ejecutar la purga en producción tras validar.

```bash
agents-clean scan --region us-east-1 --dry-run
agents-clean scan --region us-east-1 --force --log-level debug
```

### 3. **Monitoreo y Reporte de Resultados**
Tras cada ejecución, agents-clean genera un reporte JSON con métricas clave: número de agentes eliminados, colas limpiadas, tiempo de ejecución y estado final. Puedes integrar estos datos en Grafana o Datadog usando el webhook configurado.

```bash
agents-clean report --output json > clean_report_$(date +%Y%m%d).json
```

## Mejores Prácticas para Entornos SaaS

### 1. **Establece Políticas de Retención Basadas en SLA**
Define umbrales de inactividad según el tipo de agente: para agentes de procesamiento batch, 10 minutos sin heartbeat; para agentes de streaming, 30 segundos. agents-clean permite crear perfiles personalizados en `agents-clean.yml`, lo que evita falsos positivos en sistemas con picos de carga.

### 2. **Integra agents-clean en tu Pipeline CI/CD**
Agrega un paso de limpieza automática en tus releases de staging y producción. Por ejemplo, en GitHub Actions, ejecuta `agents-clean scan --region us-west-2 --force` tras cada despliegue exitoso. Esto mantiene el clúster libre de residuos y reduce el tiempo de recuperación ante fallos.
