# agents-clean - AGE_026

## Resumen Técnico

**agents-clean** es una solución SaaS de limpieza y orquestación de agentes autónomos, diseñada para entornos multi-agente en producción. Elimina residuos de estado, colisiones de contexto y fugas de memoria en sistemas basados en LLM. Ideal para equipos que gestionan pipelines de agentes en Node.js, Python o Rust.

## Características Principales (Hardcore)

1. **Limpieza de Contexto con Hash-Isolation**  
   Cada agente ejecuta su sesión en un contexto aislado mediante hashing criptográfico (SHA-256). agents-clean purga automáticamente datos huérfanos, evitando contaminación cruzada entre ejecuciones paralelas. Soporta integración nativa con Redis y PostgreSQL.

2. **Orquestación de Ciclo de Vida con Garbage Collection Predictivo**  
   Implementa un recolector de basura basado en TTL y umbrales de memoria. Detecta agentes zombies (inactivos > 5 minutos) y los termina con señales SIGTERM, liberando recursos sin afectar el throughput. Compatible con Docker y Kubernetes.

3. **Auditoría de Trazas con Replay Determinista**  
   Cada acción de limpieza genera un log estructurado (formato NDJSON) que permite reproducir el estado exacto del sistema en cualquier punto. Ideal para debugging post-mortem y compliance en entornos financieros o de salud.

## Cómo Usar (Pasos Detallados)

### 1. Configuración del Entorno Multi-Agente

```bash
# Instalar agents-clean CLI
npm install -g agents-clean

# Inicializar proyecto con soporte para 3 agentes simultáneos
agents-clean init --agents 3 --ttl 300 --storage redis
```

Esto genera un archivo `agents-clean.config.yml` con parámetros de aislamiento de contexto y umbrales de garbage collection.

### 2. Ejecución de Limpieza Programada

```javascript
// En tu script de Node.js
import { AgentCleaner } from 'agents-clean';

const cleaner = new AgentCleaner({
  strategy: 'lazy', // o 'eager' para limpieza inmediata
  hashIsolation: true,
  maxMemory: '512MB'
});

// Forzar limpieza manual antes de una tarea crítica
await cleaner.purgeOrphans({ force: true });
```

El método `purgeOrphans` elimina agentes sin referencias activas y libera memoria fragmentada.

### 3. Monitoreo y Replay de Trazas

```bash
# Ver logs en tiempo real
agents-clean watch --format json

# Reconstruir estado de un agente específico
agents-clean replay --agent-id "a1b2c3" --timestamp 2024-01-15T10:30:00Z
```

Esto permite inspeccionar el historial de limpieza y detectar fugas de contexto en entornos de staging.

## Mejores Prácticas (Industry Standards)

1. **Aplica Hash-Isolation en Entornos Multi-Tenant**  
   Si ejecutas agentes para distintos clientes en el mismo cluster, activa `hashIsolation: true` en la configuración. Esto evita que un agente malicioso o defectuoso acceda a datos de otro tenant. Combínalo con políticas de TTL agresivas (60 segundos) para minimizar riesgos de seguridad.

2. **Usa Replay Determinista para Depuración en Producción**  
   Antes de desplegar cambios críticos, ejecuta `agents-clean replay` con logs de producción filtrados por errores. Esto te permite identificar patrones de fuga de memoria sin necesidad de replicar el entorno exacto. Ideal para equipos que usan CI/CD con pruebas de integración.
