## Contenido Detallado

### ¿Qué es agents-clean?

agents-clean es una herramienta profesional de código abierto que simplifica la depuración, el mantenimiento y la reestructuración de pipelines de agentes en entornos SaaS complejos. A diferencia de soluciones monolíticas, agents-clean se integra directamente en tu flujo CI/CD, permitiendo la limpieza automática de estados obsoletos, la eliminación de dependencias circulares y la optimización de recursos en tiempo real. Está construido para escalar horizontalmente, soportando desde microservicios hasta arquitecturas multi-agente distribuidas.

### Funcionalidades Clave (Hardcore)

1.  **Purga Inteligente de Estados Zombie (Zombie State Purging)**: Detecta y elimina automáticamente estados de sesión huérfanos generados por agentes fallidos o timeout. Utiliza un algoritmo de *mark-and-sweep* en memoria distribuida (Redis/Etcd) para liberar recursos sin afectar agentes activos, reduciendo el consumo de RAM hasta un 40% en clústeres de alta concurrencia.

2.  **Reconciliación de DAGs Asíncronos (Async DAG Reconciliation)**: Implementa un motor de reconciliación que analiza grafos acíclicos dirigidos (DAGs) de dependencias entre agentes. agents-clean reordena y limpia automáticamente tareas huérfanas o ciclos no resueltos, garantizando que el flujo de ejecución sea determinista y libre de deadlocks. Ideal para pipelines de ETL o automatización de procesos.

3.  **Limpieza de Metadatos con Políticas de Retención (Metadata Cleanup with Retention Policies)**: Permite definir políticas de retención granulares (por tiempo, tamaño o tipo de dato) para logs, trazas y estados intermedios. agents-clean ejecuta una limpieza asíncrona en segundo plano, utilizando *garbage collection* basado en TTLs y compactación de índices en bases de datos NoSQL, sin bloquear el acceso a los agentes en producción.

### Cómo Usar (Pasos Detallados)

1.  **Configurar el Contexto de Limpieza**:
    - Ejecuta el comando `agents-clean init --context=production --retention=72h` para definir un contexto de limpieza con una política de retención de 72 horas.
    - Verifica la configuración con `agents-clean status --context=production`. Asegúrate de que el archivo `.agents-clean.yaml` incluya las rutas a tus colas de mensajes (Kafka/RabbitMQ) y la URI de tu base de datos de estados.

2.  **Ejecutar una Poda Selectiva de Estados**:
    - Lanza una poda manual de estados huérfanos con `agents-clean purge --state=zombie --force`.
    - Para una limpieza más quirúrgica, usa `agents-clean reconcile --dag=./workflows/dag_prod.json --dry-run` para simular la reconciliación de tu DAG antes de aplicarla con `--apply`.
    - Monitorea el progreso en tiempo real con `agents-clean watch --output=json | jq '.purged_count'`.

3.  **Automatizar la Limpieza en CI/CD**:
    - Integra agents-clean en tu pipeline de GitHub Actions o GitLab CI. Añade un paso de post-despliegue:
      ```yaml
      - name: Clean up stale agents
        run: agents-clean purge --state=orphan --retention=24h --context=staging
      ```
    - Para entornos Kubernetes, despliega el operador `agents-clean-operator` que ejecuta limpiezas periódicas basadas en cron jobs. Usa `kubectl apply -f https://raw.githubusercontent.com/xingfangwang-eng/agents-clean/main/deploy/operator.yaml`.

### Mejores Prácticas para la Industria

1.  **Estrategia de Limpieza Proactiva vs. Reactiva**: No esperes a que los agentes fallen. Implementa limpiezas programadas (cada 6 horas) usando `agents-clean schedule --interval=6h --policy=aggressive`. Esto reduce la latencia en picos de carga y evita la degradación del rendimiento en sistemas multi-tenant.

2.  **Auditoría y Rollback de Limpiezas**: Siempre activa el modo `--dry-run` antes de ejecutar una purga en producción. agents-clean genera un manifiesto JSON de los elementos a eliminar. Si algo sale mal, puedes restaurar el estado anterior usando `agents-clean restore --manifest=./backup_20231027.json`. Esto es crítico en entornos regulatorios (GDPR, SOC2).
