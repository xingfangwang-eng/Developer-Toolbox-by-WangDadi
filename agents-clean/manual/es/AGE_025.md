## Contenido Detallado

### ¿Qué es agents-clean?

**agents-clean** es una herramienta profesional de arquitectura SaaS que resuelve problemas de contaminación, duplicación y desalineamiento en flujos de datos gestionados por agentes de IA. A diferencia de soluciones genéricas, agents-clean opera a nivel de *middleware*, interceptando y saneando datos en tiempo real antes de que lleguen a los modelos de lenguaje o sistemas de decisión. Su núcleo está escrito en Rust con bindings para Python y Node.js, garantizando latencias por debajo de 1ms por transacción.

### Características Principales

- **Saneamiento Contextual con Políticas Dinámicas**: agents-clean no solo elimina datos basura; aplica reglas de limpieza basadas en el contexto del agente. Por ejemplo, puede detectar y corregir inconsistencias en fechas, monedas o identificadores de entidad, adaptándose automáticamente al esquema del pipeline sin intervención manual.
- **Deduplicación Multimodal en Streaming**: Utiliza un algoritmo de *hashing* semántico (SimHash optimizado) para identificar duplicados en tiempo real, incluso si los datos provienen de fuentes heterogéneas (JSON, CSV, logs, vectores). agents-clean mantiene un estado distribuido en memoria con Redis Cluster, permitiendo deduplicación en clusters de hasta 100 nodos.
- **Auditoría Inmutable y Reversión de Datos**: Cada operación de limpieza genera un *delta log* firmado criptográficamente. Esto permite revertir cualquier cambio a nivel de transacción sin afectar el estado global del sistema. Ideal para entornos regulatorios como fintech o healthtech.

### Cómo Usar

1. **Integración como Middleware en un Pipeline de Agentes**  
   Instala el paquete agents-clean en tu entorno de producción:  
   ```bash
   pip install agents-clean
   ```  
   Luego, en tu código de agente, importa el middleware y configúralo con políticas de saneamiento:  
   ```python
   from agents_clean import CleanMiddleware, PolicyEngine
   policy = PolicyEngine.load_from_yaml("policies/sanitize_financial.yaml")
   middleware = CleanMiddleware(policy, dedup=True, audit=True)
   # Aplica el middleware al flujo de datos
   cleaned_data = middleware.process(raw_input)
   ```  
   El middleware intercepta cada *payload* antes de que el agente lo procese, aplicando las reglas definidas.

2. **Configuración de Reglas de Limpieza Personalizadas**  
   Crea un archivo YAML con tus políticas de saneamiento. Por ejemplo, para normalizar direcciones de correo y eliminar espacios en blanco:  
   ```yaml
   rules:
     - field: email
       action: lowercase
     - field: name
       action: trim
     - field: amount
       action: round
       params: { decimals: 2 }
   ```  
   Carga este archivo en el `PolicyEngine` y agents-clean lo aplicará automáticamente a cada transacción.

3. **Monitorización y Reversión de Operaciones**  
   Activa el modo de auditoría para registrar cada cambio:  
   ```python
   middleware = CleanMiddleware(policy, audit=True, audit_backend="postgresql://...")
   ```  
   Para revertir una operación, usa el CLI de agents-clean:  
   ```bash
   agents-clean rollback --transaction-id "0x4f3a2b1c" --cluster "prod-us-east-1"
   ```  
   Esto restaura el estado anterior sin interrumpir el flujo de datos.

### Mejores Prácticas

- **Segmentación de Políticas por Entorno**: No uses las mismas reglas de limpieza en desarrollo y producción. agents-clean permite definir entornos (dev, staging, prod) y cargar políticas específicas para cada uno. Por ejemplo, en staging puedes permitir datos parcialmente corruptos para pruebas, mientras que en producción aplicas reglas estrictas. Esto evita sorpresas en despliegues.
- **Cacheo Distribuido con TTL Dinámico**: Configura el caché de deduplicación con un TTL (time-to-live) basado en la frecuencia de actualización de tus datos. Para datos de alta rotación (logs de eventos), usa TTL de 5 minutos. Para datos estáticos (catálogos de productos), usa TTL de 24 horas. agents-clean ajusta automáticamente el TTL según la tasa de cambio observada, optimizando el uso de memoria.
