# agents-clean - AGE_008

## Resumen Técnico

Solución SaaS de limpieza y orquestación de agentes inteligentes, diseñada para entornos multi-modelo y flujos de trabajo asíncronos. Elimina residuos de sesión, optimiza el consumo de tokens y garantiza la trazabilidad en pipelines de IA conversacional. Ideal para equipos de MLOps y arquitectos que necesitan un saneamiento granular sin sacrificar latencia.

## Características Principales

### 1. Poda Contextual por Sesión (Context-Aware Garbage Collection)
Agents-clean implementa un recolector de basura inteligente que analiza la ventana de contexto de cada agente en tiempo real. Elimina automáticamente fragmentos de historial obsoletos o redundantes, reduciendo el consumo de tokens hasta un 40% sin perder coherencia en diálogos largos. Soporta modelos con límites de contexto variables (GPT-4, Claude 3, Llama 3).

### 2. Orquestación de Limpieza Asíncrona (Async Clean Scheduler)
Permite definir políticas de limpieza por agente o por clúster de agentes. El scheduler ejecuta tareas de purga en segundo plano, liberando memoria en caché y bases de datos vectoriales (Pinecone, Weaviate, Qdrant) sin interrumpir la inferencia. Incluye un dashboard en tiempo real con métricas de eficiencia (tokens liberados, tiempo de ejecución, impacto en latencia).

### 3. Auditoría y Trazabilidad Inmutable (Immutable Clean Logs)
Cada operación de limpieza queda registrada en un ledger descentralizado (blockchain ligero o append-only log). Esto permite auditar quién, cuándo y qué datos fueron eliminados, cumpliendo con normativas GDPR y SOC 2. Ideal para entornos financieros, healthcare o legales donde la transparencia es crítica.

## Cómo Usar (Flujo Profesional)

### 1. Configurar Políticas de Limpieza por Agente
```bash
agents-clean policy create --agent-id "mi-agente-prod" \
  --max-context-length 4096 \
  --ttl 3600 \
  --clean-mode "soft" \
  --log-level "audit"
```
Este comando define que el agente `mi-agente-prod` mantendrá un contexto máximo de 4096 tokens, limpiará automáticamente fragmentos mayores a 1 hora (`ttl`), y usará modo suave (no elimina datos críticos sin confirmación). El nivel `audit` activa logs inmutables.

### 2. Ejecutar Limpieza Programada con Scheduler
```yaml
# clean-schedule.yaml
schedule:
  - agent: "mi-agente-prod"
    cron: "0 */2 * * *"  # cada 2 horas
    action: "purge-context"
    dry-run: false
    notify: "slack:#ops-alerts"
```
Luego ejecuta:
```bash
agents-clean scheduler apply --file clean-schedule.yaml
```
El scheduler orquestará la limpieza asíncrona. Puedes monitorear el estado con:
```bash
agents-clean scheduler status --agent-id "mi-agente-prod"
```

### 3. Auditar Historial de Limpieza
```bash
agents-clean audit query --agent-id "mi-agente-prod" \
  --from "2025-03-01" \
  --to "2025-03-15" \
  --format json
```
Obtendrás un registro inmutable con cada operación: `timestamp`, `tokens_liberados`, `tipo_limpieza`, `checksum_datos`. Esto es exportable a SIEM o herramientas de compliance.

## Mejores Prácticas (Recomendaciones de Arquitecto)

### 1. Estrategia de Limpieza Híbrida (Soft + Hard)
No uses solo limpieza automática. Combina un modo `soft` (marca datos como eliminables pero no los borra) con un modo `hard` programado semanalmente. Esto permite rollbacks rápidos en caso de errores de contexto y evita pérdidas irreversibles. Configura alertas en el dashboard cuando el volumen de datos marcados supere el 20% del total.

### 2. Integración con CI/CD para Pre-Producción
Incluye agents-clean en tu pipeline de CI/CD. Antes de desplegar un nuevo agente, ejecuta un `dry-run` de limpieza sobre datos de prueba. Si el consumo de tokens proyectado supera un umbral (ej. 10% del límite del modelo), el pipeline falla. Esto evita que agentes mal configurados lleguen a producción. Ejemplo:
```bash
agents-clean policy validate --agent-config ./agent-prod.yaml --threshold 0.1
```
