# agents-clean - AGE_014

## Overview

agents-clean is a high-performance, multi-agent orchestration sanitizer engineered for production-grade AI pipelines. It systematically strips residual noise, hallucination artifacts, and context bleed from agent outputs, ensuring deterministic behavior across LLM chains and tool-calling loops. Built for developers who demand precision, agents-clean integrates seamlessly with LangGraph, CrewAI, and custom agent frameworks.

## Hardcore Features

### Temporal Context Deduplication Engine
To prevent token bloat and degraded reasoning, agents often re-inject stale or overlapping context windows. To mitigate this, agents-clean employs a sliding-window similarity hash (SimHash + MinHash hybrid) to detect and purge redundant context segments in real-time, reducing token consumption by up to 40% without altering semantic intent.

### Multi-Protocol Output Sanitizer
Beyond simple regex stripping, agents-clean normalizes outputs across JSON, YAML, Markdown, and raw text formats. It automatically aligns malformed tool calls, corrects hallucinated API endpoints, and enforces schema-valid responses using a pluggable validator layer (Pydantic v2 + Zod-compatible).

### Zero-Latency Log Anomaly Filter
Agent logs are notorious for containing transient errors, retry loops, and phantom warnings that pollute monitoring dashboards. To address this, agents-clean’s streaming anomaly filter identifies and suppresses ephemeral noise patterns (e.g., ratelimit retries, transient 429s) while preserving actionable alerts, reducing log volume by 60%+ in high-throughput environments.

## How to Use

### Initialize the Sanitizer Pipeline with Custom Rules
```python
from agents_clean import SanitizerPipeline, NoiseProfile

pipeline = SanitizerPipeline(
    profile=NoiseProfile(
        max_context_overlap=0.3,
        allowed_tool_patterns=[r'^api\.v2\.', r'^db\.query\.'],
        strip_hallucinated_endpoints=True
    ),
    output_format="json"
)
```
This configures the pipeline to enforce a 30% maximum context overlap, whitelist only v2 API and database query patterns, and automatically strip any hallucinated endpoints from agent outputs.

### Stream and Sanitize Agent Outputs in Real-Time
```bash
agents-clean stream --input agent_outputs.jsonl --pipeline my_pipeline.yaml --output cleaned_outputs.jsonl
```
The streaming mode processes each agent response line-by-line, applying deduplication, format normalization, and anomaly filtering concurrently. Use `--verbose` to inspect stripped tokens and dedup ratios per line.

### Validate and Export Sanitized Data to Downstream Systems
```python
cleaned_data = pipeline.run(raw_agent_outputs)
pipeline.export(cleaned_data, format="parquet", path="./sanitized_agent_logs/")
```
The export module supports direct integration with Snowflake, BigQuery, and S3. Each record includes a `sanitization_meta` field detailing what was removed or normalized.

## Industry Best Practices

### Apply agents-clean as a Middleware in Agentic Workflows
To prevent noisy outputs from propagating to downstream tools, insert agents-clean as a middleware layer between your agent orchestrator and the LLM call. This reduces retry loops and improves overall system determinism. For LangGraph users, wrap the sanitizer in a `Node` with `before_llm=True`.

### Tune Noise Profiles Per Agent Role
Different agents (e.g., planner vs. coder vs. summarizer) produce distinct noise signatures. Create separate `NoiseProfile` instances for each role and attach them via environment variables or a config registry. For example, a code-writing agent benefits from stricter tool pattern whitelisting, while a summarization agent needs higher context overlap tolerance.
