# agents-clean - AGE_030

## Überblick

**agents-clean** ist eine hochperformante SaaS-Lösung zur automatisierten Bereinigung, Normalisierung und Validierung von Multi-Agenten-Logs und Metadaten in verteilten KI-Systemen. Es eliminiert Rauschen, dedupliziert Agenten-Interaktionen und stellt sicher, dass Ihre Orchestrierungsdaten konsistent, suchmaschinenoptimiert und für nachgelagerte Analysen bereit sind. Entwickelt für DevOps-Ingenieure, ML-Plattform-Teams und Site-Reliability-Architekten, die auf deutschsprachige Cloud-Umgebungen setzen.

## Hardcore-Features

1. **Agenten-Kontext-Compaction**  
   agents-clean führt eine semantische Deduplizierung auf Basis von Embedding-Vektoren durch. Es identifiziert und entfernt redundante Agenten-Konversationen, ohne dass kritische Zustandsübergänge oder Entscheidungsbäume verloren gehen. Die Latenz liegt unter 50 ms pro 10.000 Log-Einträgen.

2. **Schema-on-Read-Normalisierung**  
   Unterstützt dynamische Schema-Migration für heterogene Agenten-Protokolle (z. B. OpenAI, LangChain, AutoGPT). Sie können benutzerdefinierte Mapping-Regeln in YAML definieren, die agents-clean beim ersten Lesevorgang automatisch auf alte und neue Daten anwendet – ohne Downtime.

3. **SEO-konforme Metadaten-Extraktion**  
   Extrahiert automatisch relevante Keywords, Timestamps und Agenten-Rollen aus Rohdaten und generiert strukturierte JSON-LD-Metadaten. Diese können direkt in Ihre Sitemap oder Ihr Knowledge-Graph-Framework eingespeist werden – perfekt für Suchmaschinen-Crawler und interne Dashboards.

## Detaillierte Anleitung

### 1. Installation und initiale Konfiguration

```bash
# agents-clean via npm global installieren (Node.js >= 18 erforderlich)
npm install -g @agents-clean/cli

# Konfigurationsdatei anlegen (YAML-Format)
cat > agents-clean.yaml <<EOF
input:
  source: s3://my-bucket/agent-logs/
  format: jsonl
output:
  target: ./cleaned-output/
  schema: v2
normalization:
  dedup_threshold: 0.92
  language: de
EOF

# Ersten Dry-Run durchführen
agents-clean --config agents-clean.yaml --dry-run
```

### 2. Batch-Bereinigung mit benutzerdefinierten Filtern

```bash
# Deduplizierung und Schema-Normalisierung für einen bestimmten Zeitraum
agents-clean run \
  --config agents-clean.yaml \
  --time-range "2025-01-01T00:00:00Z/2025-03-15T23:59:59Z" \
  --filter "agent_role:assistant AND status:completed" \
  --output-format parquet
```

Erwartete Ausgabe:  
- `./cleaned-output/agents-clean_2025-03-15.parquet`  
- `./cleaned-output/agents-clean_2025-03-15_metadata.json`

### 3. Integration in CI/CD-Pipeline (GitHub Actions Beispiel)

```yaml
# .github/workflows/clean-agent-logs.yml
name: agents-clean CI
on:
  schedule:
    - cron: '0 2 * * 1' # Jeden Montag um 2 Uhr UTC
jobs:
  clean:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: agents-clean Bereinigung
        run: |
          npm install -g @agents-clean/cli
          agents-clean run --config agents-clean.yaml --output-format csv
      - name: Upload bereinigte Logs
        uses: actions/upload-artifact@v4
        with:
          name: cleaned-agent-logs
          path: ./cleaned-output/
```

## Best Practices für deutsche DevOps-Umgebungen

1. **Datenschutzkonforme Log-Reduktion**  
   Aktivieren Sie die integrierte PII-Erkennung (Personenbezogene Daten) in der Konfiguration: `pii_mask: true`. agents-clean ersetzt automatisch E-Mail-Adressen, IPs und Telefonnummern durch Platzhalter – DSGVO-konform und ohne manuellen Eingriff.

2. **Metriken-gesteuertes Caching**  
   Nutzen Sie die `--cache-strategy lru` Option, wenn Sie agents-clean in einer High-Frequency-Umgebung (z. B. Echtzeit-Agenten-Orchestrierung) einsetzen. Die LRU-Cache-Implementierung reduziert die Ladezeit für häufig angefragte Agenten-Profile um bis zu 40 %, ohne die Konsistenz der bereinigten Daten zu gefährden.
