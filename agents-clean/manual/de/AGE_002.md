# agents-clean - AGE_002

## Überblick

Mit agents-clean eliminierst du systematisch Datenmüll, inkonsistente Agenten-Outputs und veraltete Kontextfenster in Multi-Agent-Systemen. Dieses Tool wurde für Entwickler entwickelt, die ihre KI-Pipelines auf Produktionsniveau skalieren – ohne manuelle Bereinigung oder fragmentierte Logs. Keywords: agents-clean tool, agents-clean solution, agents-clean software.

## Detaillierter Inhalt

### Was ist agents-clean?

agents-clean ist ein spezialisiertes Framework zur automatisierten Bereinigung, Normalisierung und Validierung von Agenten-Kommunikationen in verteilten LLM-Architekturen. Es erkennt Anomalien in Prompt-Antwort-Ketten, entfernt redundante Kontextdaten und stellt sicher, dass jeder Agent nur die relevantesten Informationen erhält – für niedrigere Latenz, geringere Token-Kosten und reproduzierbare Ergebnisse.

### Kernfunktionen

1. **Kontext-Kompression mit semantischer Deduplizierung**  
   agents-clean analysiert den gesamten Dialogverlauf eines Agenten-Netzwerks und entfernt automatisch semantisch redundante oder veraltete Nachrichten. Dabei bleibt die logische Kohärenz erhalten – ideal für Langzeit-Sessions oder Chain-of-Thought-Pipelines.

2. **Agenten-spezifische Output-Validierung**  
   Jeder Agenten-Output wird gegen ein konfigurierbares Schema geprüft (JSON, YAML, benutzerdefinierte Formate). Fehlerhafte, unvollständige oder nicht-konforme Ergebnisse werden markiert, korrigiert oder zurückgewiesen – bevor sie in den nächsten Schritt fließen.

3. **Automatische Token-Budget-Verwaltung**  
   agents-clean berechnet dynamisch das optimale Token-Limit für jeden Agenten basierend auf der aktuellen Kontextgröße, der Aufgabenkomplexität und den verfügbaren Ressourcen. Überschüssige Token werden freigegeben, Engpässe werden gemeldet.

### So verwenden Sie es

1. **Initiale Konfiguration des Bereinigungs-Pipelines**  
   Definiere in einer `agents-clean.yaml`-Datei die Regeln für dein System: Welche Agenten-Typen (z. B. `planner`, `executor`, `validator`) sollen bereinigt werden? Welche Felder sind obligatorisch? Welche Metriken (z. B. `max_context_tokens`, `min_similarity_score`) sollen angewendet werden? Führe dann `agents-clean init --config ./agents-clean.yaml` aus, um die Pipeline zu initialisieren.

2. **Integration in deine bestehende Agenten-Schleife**  
   Füge in deinem Orchestrator (z. B. LangChain, AutoGen, eigene Implementierung) einen Middleware-Hook ein:  
   ```python
   from agents_clean import Cleaner
   cleaner = Cleaner(config_path="./agents-clean.yaml")
   cleaned_context = cleaner.process(agent_output, session_id="session_42")
   ```  
   Der Cleaner gibt ein bereinigtes `dict` zurück, das direkt an den nächsten Agenten übergeben werden kann.

3. **Monitoring und Audit-Logs aktivieren**  
   Starte das integrierte Dashboard mit `agents-clean monitor --port 8080`. Hier siehst du in Echtzeit, welche Agenten-Outputs bereinigt wurden, wie viele Token eingespart wurden und ob es Validierungsfehler gab. Exportiere die Logs als JSON oder CSV für Compliance-Zwecke.

### Beste Praktiken

1. **Setze harte Grenzen für Kontextfenster, aber weiche für Semantik**  
   Definiere ein maximales Token-Limit pro Agent (z. B. 4096), aber erlaube dem Cleaner, semantisch ähnliche Nachrichten zusammenzufassen, anstatt sie einfach zu löschen. Das reduziert Token-Kosten ohne Informationsverlust.

2. **Führe regelmäßige Regressionstests mit gespeicherten Sessions durch**  
   Nutze die `agents-clean replay`-Funktion, um historische Agenten-Sessions erneut zu bereinigen und die Ergebnisse mit früheren zu vergleichen. So stellst du sicher, dass neue Regeln keine unerwarteten Seiteneffekte haben.
