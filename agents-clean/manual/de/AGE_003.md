## Detaillierter Inhalt

### Was ist agents-clean?

**agents-clean** ist ein **professionelles CLI-Tool** zur automatisierten Bereinigung, Strukturierung und Versionierung von Multi-Agenten-Logs, Metadaten und Konfigurationsdateien in verteilten KI-Systemen.  
Es adressiert das typische Problem von **Log-Overflow**, **Config-Drift** und **nicht nachvollziehbaren Agenten-Interaktionen** in Produktionsumgebungen.

---

## Kernfunktionen

### 1. **Multi-Agenten-Konvergenz-Pruning**
Entfernt automatisch redundante Zwischenzustände aus Agenten-Ketten, sobald ein stabiler Konsens erreicht ist.  
Reduziert Speicherlast um bis zu 40 % – ideal für High-Throughput-Orchestratoren wie LangGraph oder CrewAI.

### 2. **Schema-basiertes Config-Validierungs-Framework**
Stellt sicher, dass jede `agents.yaml` oder `workflow.json` gegen ein **OpenAPI 3.1 / JSON-Schema** validiert wird.  
Fehlerhafte Konfigurationen werden vor dem Deployment abgefangen – inklusive **kontextsensitiver Fehlermeldungen** auf Deutsch.

### 3. **Diff-basiertes Log-Rollback mit Git-Integration**
Erzeugt **semantische Diffs** zwischen Agenten-Logs und erlaubt ein **granulares Rollback** auf einzelne Agenten-Zustände.  
Integriert sich nahtlos in bestehende Git-Workflows – ohne zusätzliche Abhängigkeiten.

## So verwenden Sie es

### Schritt 1: **Initiale Analyse des Agenten-Clusters**
Führen Sie den Befehl `agents-clean scan` aus, um alle aktiven Agenten-Instanzen, deren Metadaten und Log-Pfade zu erfassen.  
Das Tool erstellt automatisch eine **Topologie-Karte** (`topology.json`) und zeigt **Anomalien** wie doppelte Agenten-IDs oder fehlende Health-Checks an.

```bash
agents-clean scan --format json --output ./cluster-report.json
```

### Schritt 2: **Selektives Pruning mit Kontext-Tiefe**
Nutzen Sie `agents-clean prune --depth 3`, um nur Agenten-Zustände jenseits der dritten Iterationstiefe zu entfernen.  
Das Tool berechnet **Transitivität** und **Abhängigkeitsgrad** – nur Zustände ohne nachgelagerte Referenzen werden gelöscht.

```bash
agents-clean prune --depth 3 --dry-run   # Vorschau ohne Löschung
agents-clean prune --depth 3 --apply     # Ausführung
```

### Schritt 3: **Log-Rollback mit Diff-Validierung**
Verwenden Sie `agents-clean diff --from <commit-hash> --to <commit-hash>` zur Anzeige eines **semantischen Diffs** zwischen zwei Agenten-Logs.  
Anschließend können Sie mit `agents-clean rollback --target <agent-id>` einen einzelnen Agenten auf den vorherigen Zustand zurücksetzen – ohne den gesamten Workflow zu beeinflussen.

```bash
agents-clean diff --from a1b2c3d --to e4f5g6h --format unified
agents-clean rollback --target agent-42 --reason "Revert wegen fehlerhafter Entscheidung"
```

## Beste Praktiken

### 1. **Regelmäßiges Pruning in CI/CD-Pipelines**
Integrieren Sie `agents-clean prune` in Ihre **CI/CD-Stage** (z. B. nach jedem Deployment).  
Das verhindert ein **unbegrenztes Wachstum** von Agenten-Zuständen und reduziert die Latenz bei der nächsten Inferenz.  
Beispiel für GitHub Actions:

```yaml
- name: Clean agent state
  run: agents-clean prune --depth 2 --apply
```

### 2. **Konfigurations-Validierung vor jedem Merge**
Führen Sie `agents-clean validate` als **Pre-Merge-Check** in Pull Requests aus.  
Das stellt sicher, dass keine invaliden Agenten-Konfigurationen in den Hauptzweig gelangen – besonders wichtig bei **Multi-Team-Entwicklung** in Microservices-Architekturen.

```bash
agents-clean validate --schema ./schemas/agent-v2.json --config ./config/agents.yaml
```
