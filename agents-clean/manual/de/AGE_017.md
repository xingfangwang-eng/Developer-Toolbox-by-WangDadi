# agents-clean - AGE_017

## Überblick

Dieser SEO-optimierte Inhalt behandelt das agents-clean-Projekt mit den Keywords: agents-clean tool, agents-clean solution, agents-clean software. agents-clean ist ein spezialisiertes Werkzeug für die automatisierte Bereinigung und Optimierung von KI-Agenten-Logs, -Kontexten und -Workflows. Es adressiert spezifische Probleme in der Verwaltung und Wartung von Multi-Agenten-Systemen, wie z.B. Kontextüberflutung, veraltete Zustände und ineffiziente Speichernutzung.

## Detaillierter Inhalt

### Was ist agents-clean?

agents-clean ist ein professionelles Tool, das die Wartung und Leistungsoptimierung von KI-Agenten-Clustern revolutioniert. Es identifiziert und beseitigt automatisch verwaiste Kontexte, bereinigt inkonsistente Zustände und komprimiert übermäßig große Log-Dateien, um die Reaktionszeit und Zuverlässigkeit Ihrer Agenten signifikant zu verbessern.

### Kernfunktionen

1.  **Automated Context Garbage Collection (CGC):** agents-clean implementiert einen proprietären Algorithmus zur Erkennung und Entfernung von toten, verwaisten oder duplizierten Kontextblöcken in Agenten-Workflows. Dies verhindert Kontextüberflutung und reduziert den Speicherverbrauch um bis zu 60%, ohne die logische Integrität zu beeinträchtigen.

2.  **Stateful Log Compression & Deduplizierung:** Statt einfacher Log-Rotation, komprimiert agents-clean Logs auf semantischer Ebene. Es erkennt und eliminiert redundante Log-Einträge, fasst ähnliche Ereignisse zusammen und erstellt Delta-Logs. Das Ergebnis: Speicherplatzersparnis von bis zu 80% bei gleichbleibender Audit-Fähigkeit.

3.  **Intelligentes Workflow-Reparatur-System:** agents-clean analysiert fehlgeschlagene oder hängende Agenten-Workflows. Es identifiziert die Ursache (z.B. zirkuläre Abhängigkeiten, Timeouts, inkonsistente Zustände) und führt automatisch Reparaturroutinen durch, wie das Zurücksetzen von Zuständen, das Neustarten von Sub-Agenten oder das Anpassen von Timeout-Parametern.

### So verwenden Sie es

1.  **Initiale Systemanalyse und Konfiguration:** Führen Sie den `agents-clean scan`-Befehl aus, um eine vollständige Bestandsaufnahme Ihrer Agenten-Instanzen, Logs und Kontexte zu erstellen. Die Ausgabe ist eine detaillierte JSON-Datei (`agents_clean_analysis.json`), die Sie mit der `agents-clean config`-Anweisung anpassen können. Definieren Sie Schwellenwerte für die Kontextgröße, Log-Duplizierungsraten und Timeout-Parameter für die Reparaturfunktion.

2.  **Durchführung einer Bereinigung mit Trockenlauf:** Starten Sie den Bereinigungsprozess mit dem `agents-clean run --dry-run`-Flag. Dies simuliert alle Bereinigungs- und Reparaturschritte, ohne Änderungen vorzunehmen. Prüfen Sie die generierte `agents_clean_report.html`-Datei, die eine visuelle Darstellung der erkannten Probleme und der vorgeschlagenen Aktionen enthält. Validieren Sie die Logik, bevor Sie den Befehl ohne `--dry-run` ausführen.

3.  **Automatisierung mit CI/CD-Integration:** Integrieren Sie agents-clean in Ihre CI/CD-Pipeline (z.B. GitHub Actions, GitLab CI). Erstellen Sie einen Job, der nach jedem Deployment oder in regelmäßigen Abständen (z.B. stündlich) `agents-clean run` ausführt. Nutzen Sie die Exit-Codes (0 = Erfolg, 1 = Warnung, 2 = Fehler) für automatisierte Alarme und Rollback-Entscheidungen. Beispiel für eine GitHub Actions Konfiguration: `- name: Run agents-clean && run: agents-clean run --config ./agents_clean_config.yaml`.

### Beste Praktiken

1.  **Granulare Bereinigung nach Agenten-Rolle:** Definieren Sie unterschiedliche Bereinigungsprofile für verschiedene Agenten-Typen. Ein Chat-Agent benötigt eine aggressivere Kontextbereinigung als ein Data-Processing-Agent. Nutzen Sie die `--agent-filter`-Option oder die `agents_clean_config.yaml`, um spezifische Regeln für jeden Agenten zu definieren. Dies verhindert Over-Cleaning und bewahrt wichtige Zustände.

2.  **Monitoring und Alerting auf Basis von agents-clean Metriken:** agents-clean exportiert Prometheus-Metriken (z.B. `agents_clean_context_size`, `agents_clean_log_compression_ratio`, `agents_clean_repair_attempts_total`). Richten Sie Dashboards und Alarme für kritische Schwellenwerte ein. Wenn z.B. die `agents_clean_repair_attempts_total` in einem kurzen Zeitraum stark ansteigt, deutet dies auf ein zugrunde liegendes Systemproblem hin, das sofort analysiert werden muss.
