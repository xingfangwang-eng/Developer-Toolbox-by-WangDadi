## Detaillierter Inhalt

### Was ist agents-clean?

agents-clean ist kein weiteres Monitoring-Tool. Es ist ein **proaktives Korrektur- und Optimierungs-Framework** für agentenbasierte Pipelines. Es erkennt semantische Inkonsistenzen zwischen Agenten, entfernt redundante oder veraltete Zustände und führt **deterministische Cleanups** auf Basis von Kontext-Metadaten durch. Ideal für **Kubernetes-native Agenten-Cluster** und **event-getriebene Architekturen**.

### Kernfunktionen

1. **Semantische Konflikt-Resolution in Echtzeit**  
   agents-clean analysiert Agenten-Logs und -Zustände auf semantische Widersprüche (z. B. widersprüchliche Intent-Parameter, doppelte Task-Zuweisungen). Es bereinigt diese automatisch durch **kontextbewusste Merging-Strategien** und minimiert so Latenz und Fehlerraten in Multi-Agenten-Systemen.

2. **Adaptive Zustandsbereinigung mit Rollback-Versionierung**  
   Statt statischer Cleanup-Regeln nutzt agents-clean **ML-basierte Anomalieerkennung**, um veraltete oder korrupte Agenten-Zustände zu identifizieren. Alle Bereinigungen werden versioniert und sind jederzeit **rollback-fähig** – ohne Unterbrechung des laufenden Betriebs.

3. **Automatische Workflow-Orchestrierung nach Cleanup**  
   Nach einer Bereinigung erstellt agents-clean automatisch eine **optimierte Workflow-Sequenz** unter Berücksichtigung aller verbleibenden Agenten. Es passt Timeouts, Retry-Parameter und Dependencies dynamisch an – reduziert unnötige Wartezeiten um bis zu **40 %**.

### So verwenden Sie es

1. **Initialisieren des Cleanup-Contexts**  
   ```bash
   agents-clean init --context ./agent_config.yaml --mode strict
   ```  
   Der Befehl scannt alle aktiven Agenten-Zustände und erstellt einen **Context-Graphen**. `--mode strict` aktiviert die semantische Konfliktprüfung auf allen Ebenen. Optional: `--rollback-dir /var/agents-clean/rollbacks` für sofortige Versionierung.

2. **Ausführen einer gezielten Bereinigung**  
   ```bash
   agents-clean run --target agent-23 --cleanup-type state --force-merge
   ```  
   Dies führt eine **tiefgehende Zustandsbereinigung** für Agent `agent-23` durch. `--force-merge` erzwingt das Zusammenführen von doppelten Zuständen, selbst wenn Konflikte bestehen. Die Ausgabe zeigt detailliert, welche Zustände gelöscht, gemergt oder archiviert wurden.

3. **Überprüfen und Validieren des Ergebnisses**  
   ```bash
   agents-clean verify --context ./agent_config.yaml --metric consistency
   ```  
   Nach der Bereinigung wird die **semantische Konsistenz** aller Agenten geprüft. Der Befehl liefert einen `consistency_score` (0–1) sowie eine Liste aller noch offenen Konflikte. Bei `score < 0.95` erfolgt automatisch eine **Warnung mit Handlungsempfehlung**.

### Beste Praktiken

- **Führen Sie agents-clean in einer dedizierten Staging-Umgebung durch**  
  Nutzen Sie `--dry-run` vor dem ersten Einsatz in Produktion. So sehen Sie alle geplanten Änderungen, ohne dass Agenten-Zustände tatsächlich manipuliert werden. Testen Sie verschiedene Cleanup-Modi (`aggressive`, `conservative`) anhand Ihrer typischen Lastprofile.

- **Integrieren Sie agents-clean in Ihre CI/CD-Pipeline**  
  Rufen Sie `agents-clean verify` nach jedem Deployment auf. So stellen Sie sicher, dass neue Agenten-Versionen keine semantischen Inkonsistenzen einführen. Kombinieren Sie dies mit **automatischen Rollbacks** bei `consistency_score < 0.9` – das reduziert Ausfallzeiten drastisch.
