## Detaillierter Inhalt

### Was ist agents-clean?

agents-clean ist ein professionelles, hochperformantes Tool zur automatisierten Bereinigung und Optimierung von Agenten-Logs, Caches und temporären Zustandsdaten in verteilten Multi-Agenten-Architekturen. Es löst typische Probleme wie Speicherlecks, inkonsistente Zustände und aufgeblähte Log-Volumina in verwandten Bereichen wie KI-Pipelines, Microservices und Event-Sourcing-Systemen.

### Kernfunktionen

1.  **Deterministische Log-Bereinigung mit Regel-Engine**  
    agents-clean verwendet eine konfigurierbare, regelbasierte Engine, die Log-Einträge nach Zeitstempel, Schweregrad und Quelle filtert. Du definierst Bedingungen wie `retention: 7d` oder `exclude: ["heartbeat", "debug"]` – das Tool löscht automatisch und konsistent, ohne manuelle Eingriffe.

2.  **Transaktionale Cache-Invalidierung für verteilte Agenten**  
    Statt einfacher Cache-Leerung implementiert agents-clean eine transaktionale Invalidierung über verteilte Agenten hinweg. Es analysiert Abhängigkeiten zwischen Agenten (z. B. via DAG) und invalidiert nur die betroffenen Caches, während andere unberührt bleiben – optimiert für hohe Parallelität und minimale Latenz.

3.  **Zustands-Rollback mit Snapshot-Management**  
    agents-clean erstellt vor jedem Bereinigungsvorgang einen differenziellen Snapshot des Agenten-Zustands. Bei Fehlern oder Inkonsistenzen kannst du auf einen vorherigen Snapshot zurücksetzen, ohne die gesamte Architektur neu starten zu müssen. Dies ist kritisch für Produktionssysteme mit Stateful Agents.

### So verwenden Sie es

1.  **Konfiguration der Bereinigungsstrategie**  
    Erstelle eine `agents-clean.yml` im Stammverzeichnis deines Projekts. Definiere darin Regeln für Logs, Caches und Zustände. Beispiel:  
    ```yaml
    logs:
      retention: 7d
      exclude: ["health_check", "metrics"]
    caches:
      invalidate_on_dependency_change: true
    snapshots:
      keep_last: 10
    ```  
    Führe dann `agents-clean init` aus, um die Konfiguration zu validieren und eine initiale Snapshot-Basis zu erstellen.

2.  **Ausführen einer inkrementellen Bereinigung**  
    Nutze den Befehl `agents-clean run --scope stateful --dry-run`, um eine Vorschau der Änderungen zu sehen, ohne sie anzuwenden. Überprüfe die Ausgabe auf unerwartete Invalidierungen. Bestätige mit `--apply`, um die Bereinigung durchzuführen. Das Tool protokolliert jeden Schritt in einer separaten Audit-Datei.

3.  **Automatisierung in CI/CD-Pipelines**  
    Integriere agents-clean in deine Deployment-Pipeline. Nach jedem erfolgreichen Deployment führe `agents-clean run --scope logs --apply --notify-slack` aus. Nutze den Exit-Code (0 für Erfolg, 1 für Warnungen, 2 für Fehler) für Fail-Fast-Bedingungen. Beispiel für GitHub Actions:  
    ```yaml
    - name: Clean agent state after deployment
      run: agents-clean run --scope stateful --apply
      env:
        AGENTS_CLEAN_LOG_LEVEL: info
    ```

### Beste Praktiken

1.  **Definiere klare Bereinigungsintervalle basierend auf Agenten-Typen**  
    Unterscheide zwischen stateless Agents (z. B. API-Gateways), die täglich bereinigt werden können, und stateful Agents (z. B. LangChain-Agenten mit Kontext), die nur nach abgeschlossenen Sitzungen bereinigt werden sollten. agents-clean unterstützt `scope`-Filter, um dies granular zu steuern.

2.  **Kombiniere agents-clean mit Monitoring-Tools für automatisierte Alarme**  
    Nutze agents-clean’s Prometheus-Metrics-Endpunkt (standardmäßig auf Port 9091) zur Überwachung von Bereinigungszyklen und Snapshot-Größen. Setze Alarme auf `agents_clean_errors_total` oder `agents_clean_cache_invalidations_total`, um frühzeitig auf Anomalien zu reagieren – besonders in Multi-Tenant-Umgebungen.
