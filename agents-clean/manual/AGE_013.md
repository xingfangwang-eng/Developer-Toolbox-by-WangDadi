# agents-clean - AGE_013

## Overview

**agents-clean** is a high-performance, multi-agent orchestration sanitizer designed for production AI pipelines. It eliminates stale agent state, resolves context conflicts, and enforces deterministic cleanup across distributed agent runtimes. Built for DevOps engineers and AI architects who demand zero-latency teardown and memory-safe agent lifecycle management.

## Detailed Content

### What is agents-clean?

agents-clean is a lightweight, embeddable runtime utility that automates the teardown and sanitization of AI agent sessions in multi-agent environments. It solves the critical problem of agent state pollution—where lingering context, cached tool outputs, or orphaned sub-agents degrade downstream reasoning. By hooking directly into agent lifecycle hooks (pre-spawn, post-execute, on-error), agents-clean ensures every agent spawns into a pristine namespace, reducing hallucination rates and improving decision coherence by up to 40% in production benchmarks.

### Core Features

1. **Deterministic Namespace Eviction** – Each agent session is assigned an isolated context namespace. On termination, agents-clean performs a recursive memory sweep, clearing all ephemeral state, tool caches, and inter-agent message buffers. Supports both soft eviction (graceful) and hard eviction (force-kill) with configurable timeout backpressure.

2. **Cross-Agent Dependency Graph Cleanup** – Agents-clean maintains a directed acyclic graph (DAG) of inter-agent dependencies. When a parent agent is terminated, it automatically cascades cleanup to all child agents, preventing orphaned processes. Includes a built-in cycle detection engine to prevent deadlock during teardown.

3. **Zero-Latency Hook Injection** – Integrates via a single-line decorator or middleware layer (Python/TypeScript/Go). No external daemon or sidecar required. Hooks fire synchronously or asynchronously, with optional Prometheus metrics export for monitoring teardown latency and failure rates.

### How to Use

1. **Integrate the Cleanup Hook into Your Agent Runtime**  
   Install via `pip install agents-clean` or `npm install agents-clean`. In your agent factory, wrap the agent instantiation with the `@agents_clean` decorator. This automatically registers pre-spawn and post-execute hooks. Example:  
   ```python  
   from agents_clean import agents_clean  

   @agents_clean(namespace="user_session_123", eviction_policy="hard")  
   class MyAgent:  
       def execute(self, task):  
           # Your agent logic here  
           pass  
   ```

2. **Configure Eviction Policies and Metrics**  
   Create a `clean_config.yaml` in your project root. Define eviction timeout (default 5s), retry count (default 3), and whether to export metrics to your existing Prometheus endpoint. Example:  
   ```yaml  
   eviction:  
     timeout_ms: 3000  
     retry_on_failure: true  
     max_retries: 2  
   metrics:  
     enabled: true  
     endpoint: "http://localhost:9090/metrics"  
   ```  
   Then call `agents_clean.init(config_path="clean_config.yaml")` at application startup.

3. **Trigger Manual Cleanup for Debugging**  
   In development or staging, you can programmatically trigger a cleanup of all active agent namespaces via:  
   ```python  
   from agents_clean import force_evict_all  
   force_evict_all(namespace_filter="dev_*", dry_run=True)  
   ```  
   The `dry_run` flag logs which agents would be cleaned without actually terminating them. Use this to validate your dependency graph before production deployment.

### Best Practices

- **Always pair agents-clean with a session ID generator** – Use UUID v7 or KSUIDs for agent namespaces. This prevents namespace collisions in high-throughput environments and ensures clean eviction paths. Avoid sequential IDs; they create predictable cleanup bottlenecks.

- **Set up a health check that validates agent state after cleanup** – After each teardown cycle, run a lightweight probe that checks for residual context or orphaned threads. If the probe detects contamination, trigger an alert and escalate to a manual eviction. This is critical for compliance-heavy workflows (e.g., finance, healthcare).
