### What is agents-clean?

agents-clean is a high-performance, multi-agent orchestration framework designed to sanitize, deduplicate, and normalize messy agent-generated outputs at scale. It eliminates hallucinations, redundant data, and format inconsistencies across distributed AI pipelines, ensuring your downstream systems receive only clean, production-ready inputs. Built for developers who demand precision, it integrates seamlessly with CI/CD workflows and real-time streaming architectures.

### Core Features

- **Intelligent Deduplication Engine**: Leverages semantic fingerprinting and locality-sensitive hashing (LSH) to detect and merge near-identical agent outputs, reducing storage overhead by up to 60% without sacrificing context integrity.
- **Cross-Lingual Sanitization Pipeline**: Automatically normalizes multilingual agent responses (10+ languages) using Unicode-aware regex and context-aware tokenization, stripping out noise like broken HTML, malformed JSON, or mixed-script artifacts.
- **Real-Time Anomaly Injection Detection**: Employs a lightweight Bayesian classifier to flag and quarantine hallucinated or adversarial content injected by rogue agents, with configurable confidence thresholds for zero-trust environments.

### How to Use

1. **Initialize the Cleaner with a Custom Rule Set**  
   Instantiate the `AgentsCleaner` class by passing a YAML configuration that defines your sanitization rules, language profiles, and deduplication sensitivity. For example, set `dedup_threshold: 0.85` for strict matching or `0.75` for lenient merging. Then call `cleaner.load_pipeline()` to compile the rules into an optimized DAG.

2. **Batch Process Agent Outputs via Streaming**  
   Feed agent outputs as a generator or async stream using the `cleaner.ingest_stream()` method. Each item is processed through the pipeline: first, the anomaly detector scans for outliers; then, the deduplication engine checks against a Redis-backed bloom filter; finally, the sanitizer normalizes the output. Use `cleaner.flush_batch()` to commit clean data to your sink (e.g., S3, Kafka, or PostgreSQL).

3. **Validate and Export Cleaned Logs**  
   After processing, run `cleaner.validate_consistency()` to generate a JSON report of flagged anomalies, deduplication stats, and language distribution. Export the cleaned dataset using `cleaner.export_to_parquet()` for high-compression storage or `cleaner.export_to_ndjson()` for real-time dashboards. Integrate with your monitoring stack via the built-in Prometheus exporter.

### Best Practices

- **Implement Pre-Processing Hooks for High-Volume Pipelines**  
  Use the `pre_process` hook to strip obvious noise (e.g., timestamps, debug logs) before the main pipeline runs. This reduces false positives in anomaly detection and speeds up deduplication by 30%. Configure it in your YAML under `hooks.pre_process: "lambda x: x.replace('DEBUG:', '')"`.

- **Adopt Incremental Deduplication for Long-Running Agents**  
  For agents that generate continuous streams (e.g., real-time chatbots), enable incremental deduplication by setting `incremental_mode: true` and pointing to a persistent store like Redis or DynamoDB. This avoids reprocessing historical data and keeps memory usage constant, even after weeks of uptime.
