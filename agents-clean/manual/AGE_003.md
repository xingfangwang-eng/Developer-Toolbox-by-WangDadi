# agents-clean - AGE_003

## Overview

`agents-clean` is a precision-engineered SaaS toolkit designed for teams managing multi-agent orchestration pipelines. It systematically eliminates stale agent states, resolves dependency deadlocks, and enforces deterministic cleanup across distributed agent runtimes. Built for production-grade AI workflows, this tool ensures your agent lifecycle stays lean, auditable, and failure-resilient.

## Hardcore Features

### 1. **State Garbage Collector with Causality Tracking**
Agents often leave behind orphaned memory fragments or partial execution contexts. `agents-clean` implements a DAG-based causality tracker that recursively identifies and prunes zombie states across agent graphs — without breaking active lineage chains.

### 2. **Deadlock-Aware Agent Teardown Scheduler**
Instead of brute-force termination, the tool uses a topological teardown scheduler. It analyzes inter-agent dependencies in real-time, then executes a reversed dependency-aware shutdown sequence. This prevents cascading failures and resource leaks in complex multi-agent topologies.

### 3. **Deterministic Replay Log Cleaner**
When debugging agent failures, replay logs accumulate fast. `agents-clean` compresses and indexes replay logs using a time-windowed bloom filter, enabling lossless deduplication and context-preserving cleanup. You retain full audit trails while reclaiming up to 70% storage overhead.

## How to Use

### 1. **Initialize the Cleaner with a Target Agent Graph**
```bash
agents-clean init --graph-id agent-cluster-prod-01 --policy causal-prune
```
This command binds the cleaner to your agent cluster’s state graph. The `causal-prune` policy ensures only orphaned or unreachable agent states are flagged, leaving active execution paths untouched.

### 2. **Run a Dry-Clean Audit**
```bash
agents-clean audit --dry-run --output json > cleanup_report.json
```
Before executing any teardown, run a dry audit. The tool outputs a detailed JSON report listing each candidate state, its dependency depth, and the estimated memory reclaim. This step is critical for CI/CD pipelines where you want zero disruption.

### 3. **Execute a Scheduled Cleanup with Grace Period**
```bash
agents-clean schedule --interval 12h --grace-period 300s --force-orphan-age 24h
```
This schedules a recurring cleanup job. The grace period (300 seconds) allows any in-flight agent to finish its current task before being considered for cleanup. The `force-orphan-age` flag ensures states older than 24 hours with no active references are hard-reclaimed.

## Best Practices

### **1. Integrate Cleanup into Your Agent Deployment Pipeline**
Don’t treat cleanup as an afterthought. Embed `agents-clean` as a post-deployment hook in your CI/CD pipeline. After each agent version rollout, run an audit to detect and remove stale states from the previous version. This prevents version drift and ensures your agent graph stays consistent across deployments.

### **2. Use Tiered Cleanup Policies for Multi-Tenant Environments**
In shared agent clusters, apply different cleanup policies per tenant namespace. For example, use `causal-prune` for low-latency production tenants and `aggressive-reclaim` for staging tenants. This prevents one tenant’s cleanup job from affecting another’s performance, while still maintaining overall cluster hygiene.
