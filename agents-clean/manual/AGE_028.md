# agents-clean - AGE_028

## Overview

`agents-clean` is a high-performance, multi-agent orchestration sanitizer engineered for DevOps engineers and AI pipeline architects who demand deterministic cleanup in distributed agent ecosystems. It eliminates state drift, ghost processes, and orphaned artifacts across heterogeneous runtime environments—without manual intervention or fragile shell scripts.

## Core Features

### 1. **Deterministic Agent State Reconciliation Engine**
Agents-clean implements a lock-free, idempotent reconciliation loop that compares the declared agent lifecycle state against the actual runtime footprint. It leverages a Merkle-style diff tree to detect and purge zombie processes, stale socket files, and leaked environment variables—down to the PID namespace level. No polling, no race conditions.

### 2. **Multi-Tenant Artifact Garbage Collection with Causal Cut Semantics**
Built for high-density deployments, agents-clean performs distributed garbage collection across agent pods, sidecars, and ephemeral volumes. It uses a vector-clock-based causal cut algorithm to determine which artifacts are safe to remove without breaking in-flight transactions. Supports overlay filesystems, tmpfs, and bind mounts.

### 3. **Declarative Cleanup Policy Engine via HCL/JSON**
Define cleanup policies as code. Agents-clean parses HCL or JSON-based policy manifests to enforce resource quotas, TTL-based eviction, namespace-scoped whitelists, and pre/post-hook execution chains. Policies are validated against a schema and compiled into a DAG of cleanup actions—executed in topological order with rollback support.

## How to Use

### 1. **Initialize the Agent Cleanup Context**
```bash
agents-clean init --namespace=prod-us-east --policy=./policies/cleanup.hcl \
  --state-store=etcd://localhost:2379/agents-clean \
  --ttl=72h
```
This bootstraps the reconciliation context, connects to the distributed state store, and loads the policy DAG. The `--ttl` flag sets the global artifact expiry window.

### 2. **Execute a Dry-Run with Causal Trace**
```bash
agents-clean run --dry-run --trace-depth=3 \
  --scope=agent-*,sidecar-* \
  --exclude=critical-service
```
The dry-run mode emits a structured JSON trace of all cleanup actions, including causal dependencies and conflict detection. Use `--trace-depth` to control recursion into nested agent groups. The `--exclude` flag prevents cleanup of critical workloads.

### 3. **Apply Cleanup with Rollback Capability**
```bash
agents-clean apply --policy=./policies/cleanup.hcl \
  --rollback-snapshot=/tmp/cleanup_snapshot.json \
  --concurrency=8 --timeout=120s
```
This executes the cleanup DAG with configurable concurrency and timeout. A pre-snapshot of affected resources is written to `--rollback-snapshot`, enabling a full revert via `agents-clean rollback --snapshot=/tmp/cleanup_snapshot.json`.

## Best Practices

### 1. **Always Run a Dry-Run in Staging First**
Before touching production, execute `agents-clean run --dry-run --trace-depth=3` against a staging environment that mirrors your production topology. Validate the trace output for unintended removals—especially artifacts with cross-service dependencies. Integrate this step into your CI/CD pipeline as a gating check.

### 2. **Version-Control Your Cleanup Policies**
Treat your `.hcl` or `.json` policy files as infrastructure code. Store them in a Git repository with semantic versioning. Use `agents-clean validate --policy=./policy.hcl` to lint and schema-check before deployment. Combine policy changes with feature flags to roll out cleanup logic gradually across namespaces.
