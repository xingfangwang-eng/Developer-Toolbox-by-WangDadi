# agents-clean - AGE_016

## Overview

`agents-clean` is a lightweight, high‑performance agent‑side cleanup and orchestration toolkit designed for CI/CD pipelines, multi‑agent simulations, and ephemeral sandbox environments. It eliminates stale agent state, orphaned processes, and residual artifacts without sacrificing auditability. Built for developers who demand deterministic teardown and zero‑leak agent lifecycle management.

## Core Features

### 1. Declarative Agent Garbage Collection (AGC)
Define cleanup rules as YAML or JSON manifests. `agents-clean` automatically resolves dependency graphs, kills zombie sub‑agents, and purges temporary volumes—even across distributed runners. Supports regex‑based path targeting and recursive namespace eviction.

### 2. Idempotent State Reset with Rollback Snapshots
Every cleanup operation generates a lightweight Btrfs‑compatible snapshot before execution. On failure, `agents-clean` rolls back to the last known good state without manual intervention. Snapshot metadata is indexed by agent UUID and timestamp, enabling forensic replay for debugging.

### 3. Parallelized Cleanup with Resource Contention Detection
Leveraging Go’s goroutine pool, `agents-clean` executes cleanup tasks concurrently while monitoring for file locks, network socket collisions, and PID namespace conflicts. Automatically falls back to serial execution if contention exceeds a configurable threshold (default: 5 retries).

## How to Use

### 1. Create a Cleanup Manifest
```yaml
# .agents-clean.yaml
version: "2.0"
rules:
  - target: "/tmp/agent-*"
    recursive: true
    exclude: ["*.lock", "*.pid"]
  - target: "/var/run/agent-*"
    action: unlink
  - target: "docker://agent-*"
    action: prune --force
snapshot:
  enabled: true
  retention: 3
```
Run `agents-clean apply --manifest .agents-clean.yaml` to execute the defined rules.

### 2. Execute a Dry‑Run with Diff Output
```bash
agents-clean plan --manifest .agents-clean.yaml --format diff
```
This prints a unified diff of files to be deleted, processes to be killed, and containers to be removed. Use `--format json` for programmatic consumption in CI.

### 3. Trigger Cleanup via Webhook (GitHub Actions / GitLab CI)
```yaml
# .github/workflows/cleanup.yml
jobs:
  agent-cleanup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run agents-clean
        run: |
          curl -X POST https://cleanup.example.com/hooks/runner \
            -H "Authorization: Bearer ${{ secrets.AGENTS_CLEAN_TOKEN }}" \
            -d '{"repo": "${{ github.repository }}", "agent_id": "${{ github.run_id }}"}'
```
The webhook endpoint parses the payload and executes the cleanup manifest associated with the repository.

## Best Practices

1. **Always run `agents-clean plan` before `apply` in CI.**  
   This prevents accidental deletion of files that are still in use by concurrent agents. Integrate `plan` as a required step in your pipeline’s pre‑merge checks.

2. **Set snapshot retention to at least 3 for production runners.**  
   In case of a failed deploy, you can quickly roll back to the previous clean state without rebuilding the entire agent environment. Use `agents-clean restore --uuid <snapshot_uuid>` for granular recovery.
