# agents-clean - AGE_022

## Overview

`agents-clean` is a production-grade, multi-agent orchestration framework designed to streamline LLM-powered agent pipelines by pruning noise, redundant context, and hallucination-prone behaviors. It enforces deterministic agent handoffs, cleanses intermediate outputs, and ensures each agent operates on sanitized, high-signal data. Built for teams scaling agentic workflows in CI/CD, customer support automation, and autonomous code review systems.

## Core Features

### Contextual Sanitization Engine
Each agent receives a relevance-scored context window with dynamic removal of stale embeddings, conflicting instructions, and off-topic history using a sliding-window entropy filter. This reduces token waste by up to 40% in multi-turn agent conversations and prevents context poisoning.

### Deterministic Agent Handoff with Schema Locking
Agents no longer misinterpret output schemas. `agents-clean` enforces typed contracts between agents using Protocol Buffers, validating the payload against a predefined schema before an agent passes control. Any deviation triggers a retry or rollback, ensuring zero ambiguity in multi-agent state transitions.

### Hallucination Audit Trail
Every agent response is logged with a provenance graph that maps claims back to their source context. `agents-clean` integrates a lightweight verifier that flags ungrounded statements in real-time, enabling automated re-prompting or human-in-the-loop escalation. This is critical for regulated industries like fintech or healthcare.

## How to Use

### Define Agent Contracts with Protobuf
Create a `contracts/` directory and define your agent handoff schemas using Protocol Buffers. For example, an intent classification agent must output a `UserIntent` message with a `type` enum and a `confidence` float. `agents-clean` will enforce this contract at runtime.

```protobuf
syntax = "proto3";
message UserIntent {
  enum IntentType { UNKNOWN = 0; BOOKING = 1; CANCEL = 2; INQUIRY = 3; }
  IntentType type = 1;
  float confidence = 2;
}
```

### Initialize a Clean Pipeline
Instantiate a pipeline with built-in sanitization and handoff validation. Each agent is a callable class that implements `AgentInterface`. The pipeline automatically wraps each agent with context pruning and output validation.

```python
from agents_clean import Pipeline, AgentInterface

class IntentClassifier(AgentInterface):
    def run(self, context: dict) -> UserIntent:
        # Your LLM call here
        return UserIntent(type=UserIntent.BOOKING, confidence=0.92)

pipeline = Pipeline(
    agents=[IntentClassifier()],
    sanitizer="entropy",
    schema_lock=True
)
result = pipeline.run({"input": "Book a flight to Tokyo"})
```

### Enable Audit Trail and Retry Logic
Wrap your pipeline with the hallucination auditor. Configure automatic retries for low-confidence outputs or schema mismatches. The audit log outputs a JSON Lines file for later analysis.

```python
from agents_clean import Auditor

auditor = Auditor(pipeline, log_path="./audit.log")
auditor.run({"input": "Cancel my hotel reservation"})
# On schema violation or hallucination flag, auditor triggers a re-prompt
```

## Best Practices

### Use Schema Locking in Production
Never rely on LLM output parsing alone. Always define Protobuf schemas for agent handoffs. This eliminates silent failures and makes your agent graph debuggable.

### Implement Context Pruning Before Every Agent Call
Run context sanitization before each agent invocation to remove up to 30% of irrelevant tokens per hop, directly reducing latency and cost. For long-running agents (e.g., code review bots), schedule pruning every 5 turns to maintain signal integrity.
