# agents-clean - AGE_020

## Overview

`agents-clean` is a zero-trust sanitization runtime purpose-built for multi-agent systems handling untrusted I/O streams. It intercepts, validates, and scrubs agent outputs before they pollute downstream memory, vector stores, or execution pipelines. Designed for high-throughput AI orchestration layers, it enforces strict data hygiene across distributed agent topologies.

---

## Hardcore Features

1. **Schema-Aware Injection Guard**  
   Each agent output is validated against a declarative `output_schema.yaml` at runtime. Mismatched types, hallucinated fields, or adversarial payloads are stripped or rejected before propagation. Supports nested JSON, protobuf, and custom Pydantic models.

2. **Taint Tracking & Propagation Audit**  
   Every token emitted by an agent is tagged with a taint hash (SHA-256 of agent_id + timestamp + context_id). Downstream consumers can query the taint lineage via `agent_clean audit --trace <hash>` to isolate data contamination sources.

3. **Adaptive Rate-Limiting with Backpressure**  
   Dynamically throttles agent output velocity based on downstream sink latency (vector DB write QPS, LLM context window pressure). Integrates with Redis-based sliding window counters and emits structured logs for observability stacks (OpenTelemetry, Datadog).

## How to Use

### 1. Inject the Sanitization Middleware into Your Agent Pipeline

```python
from agents_clean import SanitizerMiddleware
from your_agent_framework import AgentPipeline

pipeline = AgentPipeline()
pipeline.use(
    SanitizerMiddleware(
        schema_path="./configs/output_schema.yaml",
        taint_enabled=True,
        strict_mode=True
    )
)
pipeline.run(input_data)
```

This inserts a blocking middleware layer that intercepts every agent output before it reaches the next handler.

### 2. Run a Forensic Audit on a Tainted Output

```bash
agent_clean audit --hash 3f7c1a2b... --depth all --format json
```

Returns a full propagation chain: which agent generated it, which downstream agents received it, and whether any sanitization rules were violated. Use `--depth 1` for immediate upstream only.

### 3. Configure Adaptive Rate Limits per Agent Class

```yaml
# configs/rate_limit.yaml
rate_limits:
  "summary_agent":
    max_tokens_per_sec: 500
    backpressure_sink: "vector_db"
    cooldown_ms: 250
  "code_gen_agent":
    max_tokens_per_sec: 1500
    backpressure_sink: "llm_context"
    cooldown_ms: 100
```

Apply via CLI:

```bash
agent_clean apply-rules --config configs/rate_limit.yaml --env production
```

## Best Practices

1. **Always pair `strict_mode=True` with a formal output schema in production.**  
   Without a schema, the sanitizer defaults to a lenient pass-through. In multi-tenant agent deployments, a single malformed output can cascade into corrupted embeddings or hallucinated downstream reasoning. Schema enforcement is your last line of defense before persistence.

2. **Enable taint tracking only on agents that interact with external APIs or user inputs.**  
   Taint hashing adds ~2ms per token. For internal orchestration agents (e.g., routing, aggregation), disable it via `taint_enabled=False`. Reserve taint tracking for boundary agents—those that face the open web, databases, or user-provided prompts.
