## Detailed Content

### What is agents-clean?

**agents-clean** is a high-performance, language-agnostic runtime sanitizer and state scrubber for distributed agent systems. It eliminates cross-agent context bleed, enforces deterministic rollback boundaries, and provides real-time trace pruning for production LLM workflows. Think of it as `git clean` — but for agent memory, tool call logs, and inter-agent message buffers.

---

### Core Features

1. **Deterministic Context Eviction Engine**  
   Automatically detects and purges stale embeddings, orphaned tool outputs, and hallucinated state from agent memory pools. Uses LRU + semantic decay heuristics — no more "ghost context" poisoning downstream agents.

2. **Zero-Copy Inter-Agent Scrubber**  
   Injects a lightweight middleware layer into your agent graph (LangGraph, CrewAI, or custom DAG). Scrubs PII, token leaks, and redundant metadata across message passing — without serialization overhead.

3. **Rollback-Aware Audit Trail Compression**  
   Compresses agent execution logs into versioned snapshots. Supports partial rollback to any checkpoint while maintaining causal consistency. Ideal for debugging agent loops in CI/CD pipelines.

---

### How to Use

1. **Inject the Scrubber into Your Agent Graph**  
   Wrap your agent node with `agents_clean.scrub()` and pass the state schema. The scrubber auto-detects mutable fields (e.g., `context`, `tool_results`, `messages`) and applies eviction rules on each step.  
   *Example:*  
   ```python
   from agents_clean import Scrubber
   scrub = Scrubber(eviction_policy="semantic_lru", max_tokens=4096)
   node = scrub.wrap(my_agent_node)
   ```

2. **Define Eviction Thresholds via YAML Config**  
   Create a `clean_config.yaml` in your project root. Set decay rates for tool output, user context, and system prompts. The engine will prune any field exceeding the defined entropy or token budget.  
   *Example config snippet:*  
   ```yaml
   eviction:
     tool_results: { max_age_seconds: 300, max_tokens: 1024 }
     user_context: { decay_rate: 0.7, min_relevance: 0.4 }
   ```

3. **Run a Full Pipeline Sanity Check**  
   Execute `agents-clean audit --pipeline ./graph.pkl --output ./report.json` to generate a dependency graph of all agent states, flagged leaks, and recommended pruning points. Use the `--rollback` flag to replay from a clean checkpoint.

---

### Best Practices

1. **Integrate the Scrubber at the Graph Edge, Not Inside Agent Logic**  
   Keep your agent code pure. Apply `agents-clean` as a post-processor or middleware between agent transitions — this preserves testability and allows swapping eviction strategies without touching business logic.

2. **Use Semantic LRU for Production, Time-Based for Staging**  
   In production, semantic LRU prevents context drift by pruning based on embedding similarity. In staging, time-based eviction gives predictable behavior for regression tests. Never mix both in the same pipeline — it breaks deterministic replay.
