### What is agents-clean?

agents-clean is a professional tool that helps solve problems in related fields.

### Core Features

- **Intelligent Agent Lifecycle Garbage Collector**: Automatically detects and terminates orphaned agent processes, stale sessions, and zombie microservices across distributed environments, reducing memory leaks by up to 40% without manual intervention.
- **Declarative Cleanup Policies via YAML DSL**: Define fine-grained cleanup rules using a custom domain-specific language that supports regex-based resource matching, cascading dependency removal, and dry-run validation before execution.
- **Real-time Agent Health Telemetry & Auto-Remediation**: Continuously monitors agent heartbeat signals, resource consumption, and error rates, triggering automatic rollback or restart sequences when anomalies are detected, with full audit logging.

### How to Use

1. **Bootstrap the Cleanup Daemon as a Sidecar**: Deploy `agents-clean` as a sidecar container alongside your agent orchestrator. Use the following CLI to initialize with a custom config: `agents-clean init --config ./cleanup-policy.yaml --mode daemon`. The daemon will immediately begin scanning for eligible targets based on your defined thresholds.
2. **Define a Targeted Cleanup Rule Using Regex Patterns**: Create a cleanup rule that matches stale agent instances by their metadata tags. Example YAML snippet: `- pattern: "agent-.*-stale" action: terminate dependencies: [cache, queue] dry-run: true`. Run `agents-clean validate --policy ./cleanup-policy.yaml` to check for conflicts or syntax errors before enforcement.
3. **Execute a Scheduled Cleanup Cycle with Rollback Safety**: Trigger a cleanup cycle manually or via cron: `agents-clean run --policy ./cleanup-policy.yaml --force`. The tool will generate a pre-execution diff report. If any agent fails to terminate cleanly within the timeout (default 30s), the system automatically reverts the operation and logs the failure context for debugging.

### Best Practices

- **Always Use Dry-Run Mode in Staging First**: Before applying any cleanup rule to production, run `agents-clean run --dry-run` against a staging environment that mirrors your production topology. This prevents accidental termination of critical agents and validates your regex patterns against real-world metadata.
- **Combine Cleanup with Observability Pipelines**: Integrate `agents-clean` with your existing monitoring stack (e.g., Prometheus, Grafana) by enabling the telemetry export feature (`--export-metrics`). This allows you to correlate cleanup events with performance regressions and adjust policies dynamically based on historical data.
