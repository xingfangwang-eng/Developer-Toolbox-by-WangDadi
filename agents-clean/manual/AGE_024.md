### What is agents-clean?

agents-clean is a professional tool that helps solve problems in related fields.

### Core Features

- **Multi-Agent Orchestration with Conflict Resolution** – agents-clean dynamically coordinates multiple AI agents in a single pipeline, automatically detecting and resolving conflicting outputs using a proprietary consensus engine. This ensures clean, non-redundant results even when agents disagree on data labels or extraction logic.
- **Contextual Memory Sweep with Token-Level Sanitization** – Unlike basic deduplication, agents-clean performs a deep context-aware sweep that removes hallucinated entities, orphaned references, and malformed JSON structures at the token level. It preserves semantic integrity while eliminating noise that standard regex-based cleaners miss.
- **Real-Time Agent Behavior Audit Trail** – Every cleaning action is logged with agent ID, timestamp, confidence score, and the original vs. cleaned payload. This gives you full traceability for debugging multi-agent workflows and ensures compliance with data governance standards.

### How to Use

1. **Initialize a Cleaning Pipeline with Custom Agent Profiles**  
   Define agent profiles in a YAML config file, specifying each agent’s domain (e.g., `ner`, `summarization`, `classification`) and cleaning rules. Run `agents-clean --config pipeline.yaml` to load the pipeline. The tool will automatically validate agent compatibility and inject sanitization hooks before execution.

2. **Execute a Targeted Sweep on a Batch of Agent Outputs**  
   Use the `--batch` flag with a JSONL file containing raw agent outputs. agents-clean will process each entry sequentially, applying conflict resolution and memory sweep. Monitor progress via the built-in CLI dashboard, which shows real-time deduplication ratios and error counts per agent.

3. **Export a Cleaned Dataset with Full Audit Metadata**  
   After cleaning, export the results using `--output cleaned_dataset.jsonl --audit audit_log.json`. The audit log includes every modification made, enabling you to rollback or replay specific cleaning steps. This is critical for CI/CD pipelines where agent outputs are fed into downstream systems.

### Best Practices

- **Always run a dry-run with `--simulate` before applying cleaning to production data.** This flags potential conflicts between agents (e.g., one agent extracting entities that another marks as noise) without modifying your dataset. Adjust agent rules based on the simulation report to avoid over-cleaning.
- **Integrate agents-clean as a post-processing step in your CI/CD pipeline.** After each agent inference run, pipe the output through agents-clean before storing in your data lake. This prevents corrupted or hallucinated data from propagating to downstream dashboards or training sets.
