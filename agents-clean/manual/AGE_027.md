### What is agents-clean?

agents-clean is a professional tool that helps solve problems in related fields.

### Core Features

- **Zero-Trace Agent Sweep**: Automatically detects and removes orphaned agent processes, stale session states, and residual event listeners across distributed microservice environments — no manual `kill -9` or container restarts required.
- **Declarative Cleanup Pipeline**: Define cleanup rules as YAML or JSON schemas; the engine executes a dependency-aware, parallelized purge across all registered agents, ensuring no service mesh or message broker is left with zombie references.
- **Real-Time Audit & Rollback Shield**: Every cleanup action is logged with a pre-execution snapshot. If an agent misbehaves post-clean, you can rollback to the last known good state in under 500ms — no full redeploy needed.

### How to Use

1. **Inject the Cleanup Hook into Your Agent Bootstrap**  
   Add a single line to your agent’s `init()` or `startup.sh`:  
   `./agents-clean --hook --config ./clean-policy.yaml`  
   This registers a lifecycle callback that will fire on shutdown or crash. The hook captures the agent’s PID, open file descriptors, and active goroutines before any cleanup begins.

2. **Define a Cleanup Policy with Dependency Graph**  
   Create a `clean-policy.yaml` file:  
   ```yaml
   agents:
     - name: data-sync-agent
       depends_on: [auth-agent, cache-agent]
       cleanup:
         - type: kill_process
           signal: SIGTERM
           timeout: 10s
         - type: flush_redis_keys
           pattern: "session:*"
   ```  
   Run `agents-clean validate clean-policy.yaml` to lint the schema and resolve dependency cycles before execution.

3. **Execute a Targeted Cleanup with Dry-Run Mode**  
   For a production environment, always start with:  
   `agents-clean run --dry-run --scope data-sync-agent`  
   This prints every action that would be taken (including which sockets would be closed and which cache entries removed) without actually applying changes. Once verified, drop `--dry-run` to execute.

### Best Practices

- **Always Pair Cleanup with Health Check Integration**  
   Don’t just clean — verify. After running `agents-clean`, immediately trigger a health check on the affected agents using your existing monitoring stack (e.g., Prometheus probes). This prevents false positives where an agent appears clean but is actually in a degraded state.

- **Version Your Cleanup Policies with CI/CD**  
   Treat `clean-policy.yaml` as first-class infrastructure code. Store it in your monorepo alongside agent manifests, and enforce schema validation in your CI pipeline. This ensures that a rollback of agent code also reverts the cleanup logic — no drift between what runs and what’s documented.
