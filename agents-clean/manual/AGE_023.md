# agents-clean - AGE_023

## Overview

agents-clean is a high-performance, modular agent sanitization framework designed for large-scale, distributed multi-agent orchestration environments. It eliminates context pollution, hallucination cascades, and state leakage across LLM-based agent pipelines, ensuring deterministic agent behavior in complex SaaS ecosystems.

## Core Features

### Temporal State Purging Engine
Automatically detects and strips residual agent context across conversational turns and parallel execution branches using a configurable decay window and entropy-based contamination scoring. This ensures each agent invocation starts with a pristine cognitive slate, free from stale memory leaks or cross-agent bleed.

### Schema-Locked Output Normalizer
Enforces strict output schemas per agent role using JSON Schema validation and runtime coercion. Any agent output that deviates from the defined type contract is either auto-corrected via a fallback transformer or quarantined for review, eliminating silent data corruption in multi-step agent chains.

### Latent Intent Scrubber
Analyzes agent-generated intermediate tokens for hidden biases, prompt injection artifacts, or unintended reasoning patterns using a lightweight BERT-based classifier. This flags and neutralizes semantic drift before it propagates to downstream agents or user-facing responses, preventing potential bias or emotional contamination.

### Define Agent Output Contracts
```python
from agents_clean import OutputContract, Field

contract = OutputContract(
    agent_id="summarizer-v3",
    fields=[
        Field(name="summary", type="string", max_length=500),
        Field(name="confidence", type="float", range=[0.0, 1.0]),
        Field(name="sources", type="list", item_type="string")
    ],
    strict_mode=True,
    fallback_action="transform"  # or "quarantine"
)
session.register_contract(contract)
```
Each agent must conform to its contract. If the summarizer returns a `confidence` of `1.2`, the normalizer either clamps it to `1.0` or quarantines the output depending on your `fallback_action`.

The scrubber runs inference on intermediate tokens before they reach the final output, preventing potential bias or emotional contamination from leaking into the response.

## Best Practices

### Use Per-Agent Decay Windows
Don't apply a global decay window across all agents. Set shorter windows (e.g., 120s) for high-frequency agents like chat responders, and longer windows (e.g., 600s) for slow, analytical agents like report generators. This prevents unnecessary state flushes while maintaining low latency.

### Quarantine Instead of Transform in Production
In staging, use `fallback_action="transform"` to test schema compliance. In production, switch to `fallback_action="quarantine"` and log all quarantined outputs to a dedicated S3 bucket. This gives you full auditability without silently corrupting downstream agent chains.
