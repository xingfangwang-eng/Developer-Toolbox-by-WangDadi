# agents-clean - AGE_029

## Overview

`agents-clean` is a high-performance runtime sanitizer and orchestration layer designed for multi-agent systems that accumulate state pollution, hallucination drift, and token waste over long execution cycles. Unlike generic cleanup scripts, `agents-clean` operates at the agent graph level, analyzing dependency trees and pruning stale context nodes without breaking downstream workflows.

## Core Features

### Contextual Dependency Pruning Engine
In real-time, `agents-clean` introspects the agent call graph to identify orphaned memory blocks, circular references, and expired tool outputs. By applying a topological sort combined with a time-to-live (TTL) heuristic, it safely evicts nodes that no longer contribute to the active inference path, reducing token consumption by up to 40% in long-running agent loops without sacrificing response coherence.

### Multi-Model State Reconciliation
When agents switch between LLM providers, `agents-clean` normalizes and deduplicates the accumulated state. It maps schema mismatches, aligns embedding vectors, and strips provider-specific metadata that would otherwise cause silent failures or hallucination cascades. This ensures seamless handoffs in polyglot agent architectures.

### Deterministic Rollback & Snapshot Injection
Every cleanup cycle produces a cryptographically signed state snapshot. `agents-clean` allows you to inject a previous snapshot into a running agent, enabling deterministic rollback to a known-good state without restarting the entire pipeline. This is critical for debugging, A/B testing, and compliance auditing in regulated environments.

## How to Use

### Instrument the Agent Graph
Wrap your agent execution loop with the `agents-clean` context manager. Pass the agent's internal state object (e.g., LangGraph `StateGraph`, AutoGen `ConversableAgent`, or custom dict) and define a cleanup policy using the `CleanPolicy` builder:

```python
from agents_clean import CleanPolicy, AgentCleaner

policy = CleanPolicy(
    max_context_tokens=4096,
    ttl_seconds=300,
    prune_orphans=True,
    reconcile_providers=["openai", "anthropic", "google"]
)

cleaner = AgentCleaner(policy)
with cleaner.watch(my_agent_state) as session:
    # Run your agent loop here
    session.run()
```

### Trigger a Targeted Cleanup
Instead of running a full sweep, use `session.clean(scope="tool_outputs")` to evict only tool-generated artifacts, or `session.clean(scope="memory")` to compress conversational history. You can also set auto-clean thresholds:

```python
session.auto_clean(
    trigger="token_usage",
    threshold=0.8,  # 80% of max_context_tokens
    strategy="lru"  # least recently used nodes first
)
```

### Inject a Snapshot for Rollback
To restore a previous state, call `session.inject(snapshot_id="abc123")`. `agents-clean` will replay the snapshot into the agent’s memory, re-hydrating only the nodes that existed at that point. This is especially useful when you detect a hallucination cascade or a logic loop:

```python
if agent_detects_loop():
    session.inject(snapshot_id=session.last_stable_snapshot)
    session.resume()
```

## Best Practices

### Use Agent-Specific TTL Profiles
Not all agent nodes are equal. Set a short TTL (e.g., 60s) for tool outputs like web search results or API responses, but a longer TTL (e.g., 600s) for user-provided context or system prompts. This prevents premature eviction of critical instructions while aggressively pruning transient data.

### Run Cleanup as a Sidecar Process
In high-throughput production environments, run `agents-clean` as a separate sidecar container or background thread. This isolates cleanup overhead from the main inference path and allows for parallel state inspection. Use the provided gRPC endpoint to trigger cleanups from your orchestrator without blocking the agent loop.
