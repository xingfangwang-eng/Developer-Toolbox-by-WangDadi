# agents-clean - AGE_010

## Overview

**agents-clean** is a battle-tested orchestration framework for sanitizing, normalizing, and deduplicating multi-agent system outputs. Designed for production LLM pipelines, it eliminates hallucination residue, format drift, and agent collision artifacts in real-time. Whether you're chaining autonomous agents or federating model outputs, agents-clean ensures your downstream data stays pristine.

## Core Features

### 1. **Agent Collision Detection & Resolution Engine**
Automatically detects overlapping or contradictory outputs from parallel agents using semantic fingerprinting and temporal conflict analysis. Resolves via configurable consensus strategies (majority, confidence-weighted, or LLM-mediated arbitration) — no manual triage needed.

### 2. **Schema-Native Output Normalization**
Enforces a strict, pluggable schema layer that transforms ragged, multi-format agent responses (JSON, YAML, markdown, raw text) into a unified, type-safe structure. Supports custom validators, coercion rules, and fallback templates — ideal for downstream API consumption or database ingestion.

### 3. **Incremental Deduplication with Vector Hashing**
Leverages approximate nearest neighbor (ANN) indexing to detect semantically identical or near-identical agent outputs across streaming batches. Uses locality-sensitive hashing (LSH) for O(1) lookup and configurable similarity thresholds — reduces storage bloat and downstream processing costs by up to 60%.

## How to Use

### 1. **Initialize the Cleaner with a Pipeline Config**
```python
from agents_clean import CleanPipeline

pipeline = CleanPipeline(
    schema="agent_output_v2",
    dedup_threshold=0.92,
    collision_strategy="confidence_weighted",
    log_level="DEBUG"
)
```
This configures collision detection, deduplication sensitivity, and output normalization in a single declarative object. No boilerplate.

### 2. **Stream Agent Outputs Through the Pipeline**
```python
raw_outputs = [
    {"agent": "planner", "content": "Deploy to us-east-1", "confidence": 0.95},
    {"agent": "validator", "content": "Deploy to us-west-2", "confidence": 0.87}
]

cleaned = pipeline.process(raw_outputs)
# Returns: {"content": "Deploy to us-east-1", "source": "planner", "dedup_key": "a3f2c1"}
```
The pipeline auto-resolves collisions, normalizes the schema, and assigns a dedup hash — all in a single call.

### 3. **Export to Target Format with Audit Trail**
```python
export = pipeline.export(
    format="json",
    include_audit=True,
    compress=False
)
# Output: {"cleaned": [...], "audit": {"collisions_resolved": 1, "duplicates_removed": 0, "normalization_errors": []}}
```
Every transformation is logged. You can replay, debug, or reprocess any batch from the audit trail — critical for compliance-heavy environments.

## Best Practices

- **Always run collision detection before deduplication.** Collisions often produce near-identical outputs that skew dedup thresholds. agents-clean’s pipeline order is intentional: resolve conflicts first, then deduplicate. This reduces false positives by ~40% in multi-agent setups.
- **Use schema versioning in production.** When agent schemas evolve (e.g., adding a `priority` field), pin your pipeline to a specific schema version (`"agent_output_v2"` instead of `"latest"`). This prevents silent failures when upstream agents change output format without notice.
