# agents-clean - AGE_030

## Overview

`agents-clean` is a production-grade, multi-agent orchestration sanitizer designed to eliminate context poisoning, token bleed, and residual hallucination artifacts in LLM-driven workflows. It provides deterministic cleansing pipelines for agentic systems operating in high-stakes environments—finance, compliance, and automated code generation.

## Core Features

### Recursive Agent Memory Scrubbing (RAMS)
`agents-clean` detects and purges cross-session hallucination residues from agent memory pools using entropy-based anomaly scoring. This ensures that no degraded context propagates into downstream agent calls. For example, when integrating with a high-concurrency chatbot, RAMS prevents the accumulation of unwanted contextual biases, ensuring accurate responses.

### Multi-Protocol Output Normalizer (MPON)
The `agents-clean` tool standardizes agent outputs across heterogeneous LLM backends (OpenAI, Anthropic, Llama) into a unified schema. MPON enforces strict type coercion, null-value stripping, and response-time variance correction—critical for CI/CD pipelines expecting deterministic artifacts.

### Latent Contamination Audit Trail (LCAT)
`agents-clean` generates immutable, cryptographically signed logs of every cleansing action. Each audit entry includes pre/post state diffs, cleansing rule triggers, and confidence scores. LCAT is SOC 2 and ISO 27001 compliant out of the box.

## Technical Challenges

### Challenge 1: Handling High Concurrency
When integrating `agents-clean` with a high-concurrency chatbot, we need to ensure that RAMS operates efficiently across multiple threads. To achieve this, we utilize asynchronous processing and thread-safe data structures, minimizing the impact on system performance.

### Challenge 2: Optimizing Indexes for Efficient Cleansing
To accelerate cleansing operations, we implement optimized indexing strategies for both agent memory pools and output normalizations. This enables `agents-clean` to quickly identify and purge hallucination residues, reducing overall processing times.

### Challenge 3: Detecting and Preventing Token Bleed
To prevent token bleed from propagating into downstream agent calls, `agents-clean` integrates with LLM backends using secure, encrypted communication channels. This ensures that sensitive information is protected throughout the cleansing process.

## How to Use

### Initializing the Cleansing Pipeline
```bash
agents-clean init --config ./cleansing-rules.json --backend openai
```
 Define rules in JSON: set `max_entropy_threshold: 0.45`, `token_bleed_window_ms: 1200`, and `enable_recursive_scrub: true`. The pipeline validates your config against the target LLM’s tokenizer before bootstrapping.

### Executing a Targeted Cleanse on a Multi-Turn Agent Log
```python
from agents_clean import Cleanser

cleanser = Cleanser(profile="production")
result = cleanser.run(
    agent_log="path/to/agent_session.json",
    mode="deep",
    ruleset=["ram", "mpon", "lcat"],
    output_format="parquet"
)
```
The `deep` mode triggers RAMS across all turns. MPON normalizes the output, and LCAT writes the audit trail to `./audit/`. Use `--parallel-workers=8` for logs exceeding 10k turns.

### Validating Cleanse Integrity
```bash
agents-clean audit --trail ./audit/2025-04-08T14:22:00Z.lcat --verify-signature
```
The CLI outputs a human-readable diff summary and a SHA-3 hash match status. If any cleansing step failed, the tool surfaces the exact rule ID and the offending token sequence for manual review.

## Best Practices

### Integrate Cleansing into Your CI/CD Pipeline as a Pre-Deploy Gate
Insert `agents-clean` as a step before model deployment in your GitHub Actions or GitLab CI. Use the `--fail-on-high-entropy` flag to block any agent artifact that exceeds your contamination threshold. This prevents hallucination cascades in production without adding latency to the development loop.

### Schedule Weekly Recursive Scrubs for Long-Running Agentic Systems
For agents that persist state across weeks (e.g., customer support bots, trading assistants), configure a cron job to run `agents-clean scrub --mode full --retention-policy 7d`. This prevents latent contamination from accumulating in long-term memory stores, especially when using vector databases with soft-delete semantics.
