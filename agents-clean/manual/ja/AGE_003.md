# agents-clean - AGE_003

## 概要

agents-clean は、AI エージェントのログ・キャッシュ・メモリを自動クリーンアップするための軽量 CLI ツールです。エージェント開発の「ビルド→実行→デバッグ」サイクルを高速化し、不要なデータ汚染を防ぎます。本プロジェクトは、LangChain、AutoGPT、独自エージェントフレームワークとの統合を前提に設計されています。

## 詳細な内容

### agents-clean とは？

agents-clean は、マルチエージェント環境における**メモリリーク**と**ステートの衝突**を解決するために開発された、プロフェッショナル向けクリーンアップツールです。従来の手動削除やシェルスクリプトでは対応しきれない、エージェント固有のセッションID、ベクトルストアのゴミデータ、一時ファイルの残骸を、設定ベースで一括除去します。

### コア機能

1. **セッションスコープのメモリパージ**  
   各エージェントの会話履歴・コンテキストキャッシュを、指定したセッションID単位で完全削除。LangChainの`ConversationBufferMemory`やPineconeのインデックスにも対応。

2. **ベクトルストアのゴミデータ除去**  
   ChromaDBやWeaviateに蓄積された、参照切れのエンベッディングを自動検出・削除。類似度検索の精度低下を防ぐ。

3. **マルチエージェントログの正規化**  
   異なるエージェントフレームワーク（AutoGPT、BabyAGI、CrewAI）のログフォーマットを統一し、重複・不完全なエントリをフィルタリング。デバッグ時間を50%削減。

### 使い方

1. **ターゲットエージェントの設定ファイルを生成**  
   ```bash
   agents-clean init --framework langchain --memory-type conversation_buffer --output ./config/agent_clean.yml
   ```
   これにより、対象エージェントのメモリタイプとストレージパスが自動検出され、クリーンアップルールが定義される。

2. **ドライランで影響範囲を確認**  
   ```bash
   agents-clean plan --config ./config/agent_clean.yml --session-id "session_20250410_abc123"
   ```
   削除対象のファイル数・ベクトル数・ログエントリ数をプレビュー。`--dry-run`フラグで実際の削除は行われない。

3. **本番クリーンアップを実行**  
   ```bash
   agents-clean apply --config ./config/agent_clean.yml --session-id "session_20250410_abc123" --force
   ```
   `--force`を指定することで、確認プロンプトをスキップしCI/CDパイプラインに組み込み可能。

### ベストプラクティス

- **エージェントの終了フックに組み込む**  
   各エージェントの`shutdown`または`stop`メソッド内で`agents-clean apply`を呼び出すことで、セッション終了時に自動クリーンアップを実行。メモリリークを未然に防止。

- **定期クリーンアップをcronジョブで設定**  
   開発サーバーでは1時間ごと、本番環境では1日1回のスケジュールで`agents-clean plan`を実行し、しきい値を超えたデータのみ削除。ログローテーションと併用することで、ストレージ使用量を一定に保つ。
