# agents-clean - AGE_023

## 概要

本プロジェクトは、エージェントベースのワークフローにおけるデータクレンジングと状態管理を最適化するための専用ツールです。特に、マルチエージェントシステムにおけるコンテキスト汚染やメモリリークを防止し、エージェント間の一貫性とパフォーマンスを維持します。

## 詳細な内容

### agents-clean とは？

agents-clean は、エージェントが生成するノイズや重複データを自動検出・除去し、システム全体の応答品質とスループットを向上させるプロフェッショナルツールです。大規模なエージェントネットワークにおいて、セッション管理、キャッシュクリア、コンテキスト圧縮をリアルタイムで実行します。

### コア機能

1. **コンテキスト圧縮エンジン**  
   エージェントが保持する会話履歴や状態データを、意味的損失を最小限に抑えながら圧縮。メモリ使用量を最大40%削減し、応答速度を向上させます。

2. **ノイズフィルタリング & デデュプリケーション**  
   重複したプロンプトや無関係なログエントリを自動検出し、エージェントの推論精度を維持。特に、長期稼働するエージェントにおける「会話の迷子」を防止します。

3. **動的セッションクリーンアップ**  
   一定時間アクティブでないエージェントセッションを自動的にクリーンアップ。リソースリークを防ぎ、サーバー負荷を最適化します。

### 使い方

1. **インストールと初期設定**  
   `pip install agents-clean` でインストール後、`agents_clean init` を実行。設定ファイル `agents_clean.yaml` に、対象エージェントのIDやクリーンアップ間隔（例: `cleanup_interval: 600`）を指定します。

2. **カスタムフィルタールールの定義**  
   `agents_clean filter --config custom_rules.json` で、特定のキーワードやパターンに基づくフィルタリングルールを追加。例えば、`{"pattern": "error|traceback", "action": "remove"}` のように、エラーログのみを対象に除去できます。

3. **リアルタイムモニタリングと強制クリーン**  
   `agents_clean monitor --session-id <session_id>` で特定セッションのメモリ使用量を監視。`agents_clean purge --force` で全セッションの強制クリーンを実行し、システムを初期状態にリセットします。

### ベストプラクティス

- **定期クリーンアップのスケジューリング**  
  本番環境では、エージェントのアイドル時間が長くなる前に、cronジョブやKubernetesのCronJobを使って定期的に `agents_clean purge` を実行することを推奨します。特に、夜間バッチ処理後に実行することで、翌日のパフォーマンスを最大化できます。

- **ログレベルの適切な設定**  
  デバッグ時以外は `log_level: WARNING` に設定し、過剰なログ出力によるメモリ消費を防ぎます。また、`agents_clean filter --log-level ERROR` でエラーのみを抽出し、障害解析を効率化します。
