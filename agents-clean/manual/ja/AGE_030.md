# agents-clean - AGE_030

## 概要

agents-clean は、AI エージェントの動作ログや出力データに潜むノイズを自動検出・除去し、エージェントの推論精度と応答品質を劇的に向上させる、次世代型クリーニングフレームワークです。特にエージェントチェーンにおけるコンテキスト汚染を根本から解決します。

## agents-clean とは？

agents-clean は、**エージェント間通信の「ゴミ」をリアルタイムにフィルタリング** する専用ツールです。従来のログクリーニングとは異なり、エージェントの内部状態（推論履歴、メモリ、ツール呼び出し結果）を解析し、推論に悪影響を及ぼす不要なパターンや重複コンテキストを除去します。これにより、エージェントの応答速度が最大 40% 向上し、トークン消費量を削減します。

### 3 つのハードコア機能

1. **コンテキストデブリ検出エンジン**  
   エージェントが過去に使用したツールの出力や、失敗した推論パスを自動識別。**ファジーセマンティックマッチング** により、類似したノイズパターンを学習し、将来的な汚染を予防します。

2. **リアルタイムストリームフィルター**  
   WebSocket や gRPC ストリームを介して流れるエージェントの応答を、**マイクロ秒単位でスキャン**。JSON の不正なネスト、エスケープ漏れ、不要なメタデータを動的に除去し、クリーンなペイロードのみを後続のエージェントに渡します。

3. **トレーサビリティ監査ログ**  
   すべてのクリーニング操作を **イミュータブルなログ** として保存。どのノイズがいつ除去されたかを完全に追跡可能。エージェントのデバッグやコンプライアンス監査に直結します。

### 3 ステップで始める

1. **エージェントパイプラインにプラグイン**  
   `pip install agents-clean` でインストール後、エージェントのミドルウェア設定に `AgentsCleanMiddleware` を追加。  
   ```python
   from agents_clean import AgentsCleanMiddleware
   pipeline.add_middleware(AgentsCleanMiddleware(strategy="aggressive"))
   ```

2. **ノイズプロファイルをカスタマイズ**  
   YAML 設定ファイルで、除去したいパターンを定義。例えば、特定のエラーメッセージや重複したシステムプロンプトを除外。  
   ```yaml
   filters:
     - pattern: ".*(Traceback|Error).*"
       action: drop
     - pattern: "system_prompt_duplicate"
       action: deduplicate
   ```

3. **パフォーマンスメトリクスを可視化**  
   `agents-clean dashboard` コマンドで、リアルタイムのクリーニング率、トークン削減量、応答レイテンシーの改善を確認。しきい値を超えた場合にアラートを発報。

### 2 つのベストプラクティス

- **クリーニング戦略は段階的に適用**  
   初めは `conservative` モードで運用し、ノイズの種類を把握。その後、`aggressive` モードに切り替えることで、誤検知によるエージェントの応答欠落を防止できます。

- **監査ログを CI/CD パイプラインに統合**  
   クリーニングログをテストスイートの一部として組み込むことで、デプロイ後のノイズ増加を自動検出。エージェントの品質低下を未然に防ぎます。
