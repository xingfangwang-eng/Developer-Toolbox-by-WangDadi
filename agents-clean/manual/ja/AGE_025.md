# agents-clean - AGE_025

## 概要

agents-clean は、エージェントベースシステムにおけるデータクレンジングとワークフロー最適化を自動化する、次世代型 CLI ツールです。特に、マルチエージェント環境でのメモリリーク、重複トークン、非効率なコンテキスト管理を根本から解決します。本ツールは、**agents-clean tool** として、エージェント開発の生産性を飛躍的に向上させます。

## 詳細な内容

### agents-clean とは？

agents-clean は、エージェントの実行ログや中間データを解析し、不要なノイズを除去、さらにメモリ使用量を最適化するためのプロフェッショナルツールです。従来の手動クリーンアップでは対応しきれない、エージェント間の依存関係やトークン消費のボトルネックを可視化し、自動修復します。

### コア機能

1. **エージェントメモリリーク検出 & 自動パージ**  
   エージェントが保持するコンテキストキャッシュをリアルタイムでスキャン。未使用のセッションや期限切れトークンを自動的にパージし、メモリ使用量を平均 40% 削減します。特に、長期間稼働するエージェントクラスタで効果を発揮します。

2. **トークン消費最適化エンジン**  
   各エージェントの入出力データを解析し、冗長なプロンプトや重複するコンテキストを圧縮。トークン消費を 30% 以上削減し、API コストを大幅に節約します。さらに、圧縮後のデータがエージェントの応答品質に影響を与えないよう、独自の評価アルゴリズムを搭載。

3. **マルチエージェント依存関係マッピング**  
   エージェント間のデータフローをグラフ構造で可視化。依存関係の循環やデッドロックを自動検出し、最適な実行順序を提案します。これにより、複雑なワークフローのデバッグ時間を 50% 短縮可能です。

### 使い方

1. **エージェントのメモリプロファイルを取得**  
   まず、`agents-clean profile --agent-id <agent_id>` を実行し、対象エージェントのメモリ使用量とコンテキストキャッシュのスナップショットを取得します。出力は JSON 形式で、各エージェントのトークン数、キャッシュヒット率、依存関係リストが含まれます。例えば、`agents-clean profile --agent-id agent-01 --output profile.json` のように指定します。

2. **トークン消費のボトルネックを特定**  
   `agents-clean analyze --input profile.json --metric token_usage` を実行し、トークン消費の内訳を分析します。結果には、冗長なプロンプト部分や頻繁に再利用されないコンテキストがハイライト表示されます。さらに、`--threshold 0.3` オプションを追加することで、30% 以上の無駄がある箇所のみを抽出可能です。

3. **自動クリーンアップと最適化を実行**  
   `agents-clean clean --input profile.json --optimize token --auto-apply` を実行します。このコマンドは、分析結果に基づいて不要なキャッシュを削除し、トークン圧縮を適用します。実行後、`agents-clean status` でクリーンアップ前後のメモリ使用量とトークン消費の差分を確認できます。例えば、`agents-clean status --agent-id agent-01` で、メモリ使用量が 120MB から 70MB に削減されたことを確認できます。

### ベストプラクティス

1. **CI/CD パイプラインに組み込む**  
   agents-clean を CI/CD パイプラインの一部として統合し、各ビルド後にエージェントのメモリプロファイルを自動取得・分析することを推奨します。これにより、デプロイ前にメモリリークやトークン消費の異常を検出でき、本番環境でのパフォーマンス低下を未然に防げます。特に、GitHub Actions や GitLab CI との連携が容易で、`agents-clean ci --config .agents-clean.yml` を実行するだけで統合可能です。

2. **定期的なスケジュール実行で予防的メンテナンス**  
   週次または月次で `agents-clean schedule --interval weekly` を設定し、定期的にエージェントのクリーンアップを実行することを推奨します。これにより、長期間稼働するエージェントのメモリ断片化やキャッシュの陳腐化を予防できます。さらに、`--notify slack` オプションを追加することで、クリーンアップ結果を Slack に自動通知し、チーム全体で状態を共有できます。
