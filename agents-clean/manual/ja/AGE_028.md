# agents-clean - AGE_028

## 概要

agents-clean は、AI エージェントのログデータやメモリキャッシュを自動的にクレンジング・最適化するためのプロフェッショナルツールです。本プロジェクトは、エージェントシステムのパフォーマンス劣化を防ぎ、長期運用におけるデータの一貫性を維持するために設計されています。キーワード：agents-clean ツール、エージェントクレンジング、AI メモリ最適化。

## 詳細な内容

### とは agents-clean？

agents-clean は、マルチエージェントシステムにおけるメモリリークやログ肥大化を検出・除去し、エージェントの推論精度と応答速度を維持するための軽量クレンジングフレームワークです。特に、長期稼働するエージェントのコンテキストウィンドウの断片化を解消し、トークン消費を最小限に抑えます。

### コア機能

- **メモリフラグメンテーション検出と自動デフラグ**：エージェントのセッション履歴を解析し、不要な重複データや古いコンテキストを自動的に圧縮・削除。メモリ使用率を最大 40% 削減。
- **トークン使用量のリアルタイム監視とクレンジングスケジューリング**：各エージェントのトークン消費パターンを追跡し、閾値を超えた場合に自動クレンジングジョブをトリガー。API コストを最適化。
- **エージェント間データ整合性チェックと同期クレンジング**：分散エージェント間で共有されるキャッシュやログの整合性を検証し、不整合データを自動修復。データ競合によるエラーを防止。

### 使い方

1. **クレンジングポリシーの定義と初期設定**  
   `agents-clean init --policy strict --max-token 4096` を実行し、クレンジングの厳格度と最大トークン閾値を設定。設定ファイルは `~/.agents-clean/config.yaml` に自動生成されるため、エージェント ID やクレンジング間隔を手動で調整可能。

2. **特定エージェントのセッションクレンジング実行**  
   `agents-clean run --agent-id agent-42 --mode deep` で、指定エージェントの全セッション履歴を深層解析。不要な会話ターンや重複エンティティを削除し、クレンジング後に `cleaned_stats.json` を出力。削除されたトークン数と残存メモリサイズを確認できる。

3. **定期的なバッチクレンジングジョブのスケジューリング**  
   `agents-clean schedule --cron "0 3 * * *" --batch-size 10` で、毎日午前 3 時に最大 10 エージェントのクレンジングを自動実行。ジョブ完了後は Slack または Webhook 経由で通知が送信され、失敗したエージェントはリトライキューに追加される。

### ベストプラクティス

- **クレンジング前のスナップショット保存を常に有効化**：`agents-clean run --snapshot` オプションを使用し、クレンジング前にメモリのスナップショットを保存。誤って重要なデータを削除した場合でも、`agents-clean restore --snapshot-id <id>` で即座に復元可能。
- **エージェントの重要度に応じたポリシー階層化**：本番エージェントには `policy: strict`、開発用エージェントには `policy: moderate` を適用。`config.yaml` でエージェントごとに異なるクレンジングルールを定義し、リソースを効率的に配分する。
