# agents-clean - AGE_005

## 概要

agents-clean は、AI エージェントの出力品質を自動検証・整形するための次世代型クリーンナップツールです。特にマルチエージェントシステムにおけるデータ汚染やトークン浪費を防ぎ、一貫性のあるアウトプットを保証します。

## 詳細な内容

### agents-clean とは？

agents-clean は、AI エージェントが生成するテキストやデータに対して、自動的にノイズ除去、フォーマット統一、重複排除を実行するプロフェッショナルツールです。LLM の出力が持つ「揺らぎ」を吸収し、後続のパイプラインにクリーンなデータを渡すことができます。

### コア機能

1. **エージェント出力の自動正規化エンジン**  
   エージェントが返す JSON、Markdown、コードブロックなどを検出し、スキーマに沿った形に自動変換。バリデーションエラーがあれば即座に警告し、修復提案も行います。

2. **トークンレベルでのノイズフィルタリング**  
   不要な改行、特殊文字、LLM 特有の「お詫び表現」や「補足文言」をパターンマッチングで除去。出力サイズを平均 18% 削減し、トークンコストを削減します。

3. **マルチエージェント間の一貫性チェック**  
   複数のエージェントが生成した結果を比較し、矛盾や重複を検出。優先度ルールに基づいて自動的にマージし、信頼性の高い最終出力を生成します。

### 使い方

1. **エージェント出力のパイプラインに組み込む**  
   `agents-clean` を npm または pip でインストール後、エージェントの出力を受け取る箇所に以下のように配置します。  
   ```bash
   pip install agents-clean
   ```
   その後、Python スクリプト内で `from agents_clean import Cleaner` し、`cleaner.process(agent_output)` を呼び出すだけで、自動的にノイズ除去とフォーマット整形が行われます。

2. **カスタムルールセットを定義する**  
   プロジェクトのルートに `agents_clean_rules.yaml` を作成し、除去したいパターンや許容するスキーマを記述します。  
   ```yaml
   rules:
     - pattern: "申し訳ありませんが、"
       action: remove
     - schema: 
         type: object
         properties:
           result: { type: string }
   ```
   これにより、エージェントが出力する「謝罪文」や「不要な前置き」を自動で排除できます。

3. **CI/CD パイプラインに組み込んで自動検証**  
   GitHub Actions や GitLab CI に `agents-clean` のバリデーションステップを追加します。プルリクエスト時にエージェントの出力がルールに違反していれば、自動でコメントを投稿し、修正を促します。  
   ```yaml
   - name: Validate agent outputs
     run: agents-clean validate --dir ./outputs
   ```

### ベストプラクティス

1. **エージェントごとに専用のルールセットを用意する**  
   各エージェントが生成するデータの特性は異なります。例えば、コード生成エージェントには構文チェックルールを、レポート生成エージェントには Markdown 整形ルールを設定することで、精度が飛躍的に向上します。

2. **定期的にルールセットをアップデートする**  
   LLM のアップデートやエージェントの変更に伴い、出力パターンも変化します。`agents-clean` の `--audit` モードで、現在のルールにマッチしない出力を可視化し、新しいパターンをルールに追加するサイクルを推奨します。
