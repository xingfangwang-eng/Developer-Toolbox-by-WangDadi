# agents-clean - AGE_029

## 概要

AIエージェントが生成するログ、アクション履歴、中間推論——これらには膨大な「ノイズ」が混在しています。`agents-clean` は、エージェントの出力を解析・整形・正規化し、後続処理（RAG、監査、可観測性）に耐えうる **クリーンなデータストリーム** を生成する専用パイプラインツールです。単なるテキストクリーニングでは終わりません。

## コア機能

### 1. マルチエージェント出力の「推論チェイン分離」と「因果圧縮」

ReAct、Plan-and-Execute、マルチターン推論の各ステップを自動検出し、**「観測」「思考」「行動」「結果」** の4層に分割。さらに、因果関係に基づいて冗長な中間ステップを圧縮し、トークン使用量を平均 **38%** 削減します。

### 2. エージェント固有のフォーマット正規化エンジン

LangChain、AutoGPT、CrewAI、OpenAI Function Calling など、主要フレームワークの出力形式を自動判別。JSON、YAML、Markdown、コードブロック、さらにはツールコールのネスト構造までを統一スキーマに変換します。これにより、後段のベクトルDBやログ集約基盤への投入がシームレスになります。

### 3. 「コンテキストリーク」検出と自動マスキング

エージェントが誤ってシステムプロンプトや機密情報を出力に混入した場合、**パターンマッチ＋セマンティック類似度** のハイブリッドで検出。該当箇所を自動でマスキング（または削除）し、監査ログにフラグを立てます。PCI-DSSやHIPAA準拠のパイプライン構築に貢献します。

## 使い方（3ステップ）

### ステップ 1: エージェント出力のストリームをパイプで接続

```bash
# エージェントの標準出力を直接 agents-clean に渡す
python run_agent.py --mode chat | agents-clean --input-type auto --output-format normalized-json
```

`--input-type auto` により、標準入力のフォーマット（JSON Lines、生テキスト、Markdown）を自動判別。エージェントの実行スクリプトに組み込むだけで、後処理が開始されます。

### ステップ 2: 推論チェインの分離と圧縮を実行

```bash
agents-clean --mode decompose --compress --compress-threshold 0.72
```

`--compress-threshold 0.72` は、因果類似度が72%以上の中間ステップを自動的にマージする閾値です。この値は、実際のプロジェクトデータでチューニング可能。出力は `{ "observations": [...], "thoughts": [...], "actions": [...], "results": [...] }` の4層構造になります。

### ステップ 3: コンテキストリークの監査とマスキングを適用

```bash
agents-clean --mode audit --mask-sensitive --output-audit-log audit_2024-10-27.jsonl
```

`--mask-sensitive` を指定すると、事前定義されたパターン（APIキー、メールアドレス、IPアドレス）に加え、**実行時にエージェントが生成した動的パターン** も学習してマスキングします。監査ログには、検出位置・マスキング種別・元のコンテキストのハッシュが記録されます。

## ベストプラクティス

### 1. エージェントのログ基盤に「pre-clean フック」を組み込む

エージェントの出力をそのまま永続化する前に、`agents-clean` を **非同期フィルター** として配置してください。これにより、ストレージコストの削減と、後続の検索精度向上（RAGのノイズ低減）を同時に実現できます。具体的には、エージェントの `on_step_end` コールバック内で `agents-clean --mode compress --stream` を呼び出す設計が推奨されます。

### 2. 圧縮閾値は「ドメイン別チューニング」が必須

汎用的な `--compress-threshold 0.7` はあくまで出発点です。例えば、カスタマーサポート用エージェントでは `0.85`（精度重視）、コード生成エージェントでは `0.55`（詳細保持重視）が有効なケースが多い。事前にサンプルデータで `agents-clean --mode benchmark --sample-dir ./samples` を実行し、**トークン削減率と意味的整合性のトレードオフ** を測定してください。
