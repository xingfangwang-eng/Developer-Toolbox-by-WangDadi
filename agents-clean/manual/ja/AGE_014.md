## 詳細な内容

### agents-clean とは

agents-clean は、マルチエージェントシステムにおける「ゴミ出力」「ハルシネーション」「コンテキスト汚染」を検出・除去するためのプロフェッショナルツールです。従来のルールベースフィルターでは捉えきれない、エージェント間の不整合やトークン廃棄を、独自のスコアリングエンジンでリアルタイムにクリーンアップします。

### コア機能

1. **エージェント間コンテキスト整合性チェッカー**  
   複数エージェントの会話ログを横断的に解析し、矛盾する発言や重複した指示を自動検出。スコアリングに基づいて不要なターンを削除し、コンテキストウィンドウの最適化を実現します。

2. **ハルシネーションフィルター with 外部知識ベース連携**  
   エージェント出力を外部API（例：Wikipedia、社内ナレッジベース）と突き合わせ、事実と異なる記述を特定。検出されたハルシネーションは自動でマスクまたはリライト候補として提示します。

3. **トークン廃棄可視化ダッシュボード**  
   クリーンアップ前後のトークン使用量を可視化。どのエージェントがどれだけの無駄な出力を生成しているかをパーセンテージで表示し、コスト最適化のためのインサイトを提供します。

### 使い方

1. **MCPサーバーへのプラグインインストール**  
   `npm install agents-clean-plugin` でパッケージを導入後、`mcp.config.json` に以下の設定を追加します。  
   ```json
   {
     "plugins": ["agents-clean"],
     "agents-clean": {
       "mode": "strict",
       "knowledgeBaseUrl": "https://your-kb.example.com/api"
     }
   }
   ```
   これにより、全エージェントの出力が自動的にバリデーション対象となります。

2. **カスタムルールの定義**  
   プロジェクトルートに `agents-clean.rules.yaml` を作成し、エージェントごとのクリーンアップルールを記述します。  
   ```yaml
   rules:
     - agent: "customer-support"
       maxTokens: 2048
       blacklist: ["個人情報", "クレジットカード"]
     - agent: "code-generator"
       enforceFormat: "markdown"
       forbiddenPatterns: ["TODO:", "FIXME"]
   ```
   ルールはYAMLスキーマに従い、動的にリロード可能です。

3. **クリーンアップの実行と結果確認**  
   CLIコマンド `agents-clean run --log ./logs/agent_output.json` を実行。出力結果は標準出力と同時に `cleaned_output.json` に保存されます。  
   実行後、ダッシュボードで「削除されたトークン数」「修正されたハルシネーション数」を確認。必要に応じて `agents-clean revert --id <session_id>` でロールバックも可能です。

### ベストプラクティス

- **CI/CDパイプラインへの組み込み**  
   agents-clean を GitHub Actions や GitLab CI のテストステージに追加し、エージェント出力の品質をデプロイ前に自動検証。特に、プロダクション環境に出す前にハルシネーション率が 5% を超える場合はパイプラインを停止する設定を推奨します。

- **エージェントごとのプロファイル最適化**  
   各エージェントの役割に応じて、`agents-clean.rules.yaml` で `maxTokens` や `forbiddenPatterns` を個別設定。例えば、カスタマーサポートエージェントには個人情報のマスキングを強制し、コード生成エージェントには未実装タグの除去を徹底することで、トークンコストを平均 30% 削減できます。
