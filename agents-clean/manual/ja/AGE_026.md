# agents-clean - AGE_026

## 概要

agents-clean は、AIエージェントの出力品質を劇的に向上させる、次世代型クリーニング・正規化ツールです。本ツールは、エージェント間の通信ログ、トークン消費、応答ノイズを自動検出・除去し、システム全体のパフォーマンスを最適化します。特に大規模マルチエージェント環境において、データの一貫性と再現性を保証するために設計されました。

## コア機能

### 1. コンテキスト汚染検出エンジン
エージェント間の会話履歴に潜む**暗黙的なバイアス**や**重複プロンプト**をリアルタイムでスキャン。例えば、過去のエラー応答が後続エージェントに悪影響を及ぼす「チェーン汚染」を、トークンレベルの差分解析で特定し、自動除去します。

### 2. セマンティックノイズフィルター
単なるスペルミスや文法エラーだけでなく、**意味的に曖昧な表現**（例：「多分」「おそらく」といった確信度の低いフレーズ）を検出。エージェントの出力から不確定要素を排除し、決定論的な応答を強制します。これにより、後続のタスク実行時の誤解釈リスクを80%削減します。

### 3. トークン最適化キャッシュ機構
過去のクリーニング処理結果を**構造化キャッシュ**として保存。同一または類似のエージェント入出力が発生した場合、再計算をスキップし、トークン消費を最大60%削減します。キャッシュは自動的に有効期限が設定され、古いデータはLRUアルゴリズムで破棄されます。

## 使い方

### ステップ1: エージェントパイプラインへの組み込み
`agents-clean` は npm パッケージとして提供されます。プロジェクトのルートで以下のコマンドを実行し、依存関係を追加します。
```bash
npm install agents-clean --save
```
その後、エージェントの初期化処理内で `CleanerPipeline` インスタンスを作成します。この際、**対象とするエージェントのロール（例：コーディングエージェント、レビューエージェント）** を指定することで、クリーニングルールを自動的に選択します。

### ステップ2: カスタムルールの定義
`agents-clean.yml` 設定ファイルを作成し、プロジェクト固有のクリーニングルールを定義します。例えば、特定のAPIキーや機密情報をマスクするルールを追加できます。
```yaml
rules:
  - name: "APIキーマスク"
    pattern: "sk-[A-Za-z0-9]{32}"
    replacement: "[REDACTED]"
  - name: "日付フォーマット統一"
    pattern: "\\d{1,2}/\\d{1,2}/\\d{4}"
    replacement: "YYYY-MM-DD"
```
この設定ファイルは、`CleanerPipeline` のコンストラクタに渡すことで自動読み込みされます。

### ステップ3: パフォーマンスモニタリングの有効化
`agents-clean` には、処理速度とトークン節約量をリアルタイムで可視化するダッシュボード機能が組み込まれています。以下のコードで Prometheus メトリクスエンドポイントを有効化します。
```javascript
const { CleanerPipeline, MetricsServer } = require('agents-clean');
const cleaner = new CleanerPipeline({ config: './agents-clean.yml' });
new MetricsServer(cleaner).start(9090);
```
これにより、`http://localhost:9090/metrics` で処理件数、平均レイテンシ、キャッシュヒット率などを確認できます。

## ベストプラクティス

### 1. エージェントチェーンの「ホットスポット」を特定せよ
全エージェントに一律のクリーニングを適用するのではなく、**データの受け渡しが頻繁に行われるノード**（例：コーディングエージェントからテストエージェントへの出力）に重点的にルールを設定しましょう。`agents-clean` の `--profile` オプションを使えば、各エージェントの入出力ノイズ率を数値化でき、投資対効果の高い箇所を特定できます。

### 2. クリーニング結果を「バージョン管理」せよ
クリーニングルールを変更するたびに、処理結果が変わる可能性があります。`agents-clean` の出力には、**使用したルールセットのハッシュ値**が自動付与されます。このハッシュをエージェントのログに埋め込むことで、後からどのルールでクリーニングされたかを追跡可能にし、デバッグや監査を容易にします。
