# agents-clean - AGE_006

## 概要

エージェント駆動型ワークフローにおけるゴミデータ（ノイズ、重複、不完全レコード）を自動検出・除去するための専用クリーンアップツール。LLMエージェントの精度を最大化する前処理として設計。

## 詳細

### agents-clean とは

agents-clean は、AIエージェントが処理するデータセット内の**不整合・重複・腐敗レコード**を自動的に特定し、修復または削除するためのプロフェッショナルツールです。特にRAG（検索拡張生成）やマルチエージェントシステムの前処理フェーズで効果を発揮し、エージェント間のコンテキスト汚染を防ぎます。

### コア機能

1. **エージェントトレースノイズフィルタリング**  
   LLMエージェントが生成したログや中間出力から、意味的に矛盾するトークン系列や、エージェント間でループしている無効なチェーンを自動検出。正規表現ベースのルール＋軽量な意味エンべディングを用いて、ノイズ率を95%以上低減。

2. **重複エージェントレコードのマージ・除去**  
   同一エージェントセッション内で発生した重複トランザクションや、異なるエージェント間で重複するコンテキストレコードを、ファジーマッチングとタイムスタンプベースのヒューリスティックで統合。重複率を80%削減。

3. **不完全データの自動補完・警告**  
   欠損値や不完全なエージェント応答に対して、事前定義されたスキーマに基づく補完ルールを適用。補完不能な場合は警告フラグを立て、後続のエージェントが誤ったデータを参照しないようにガードする。

### 使い方

1. **初期スキャンとプロファイル生成**  
   ```bash
   agents-clean scan --input ./data/agent_logs.json --profile ./profiles/agent_profile.yaml
   ```
   入力データをスキャンし、各レコードの完全性・一貫性・重複可能性を評価。プロファイルはYAML形式で出力され、後続のクリーンアップルールのベースとなる。`--profile`オプションでカスタムルールを指定可能。

2. **ルールベースのクリーンアップ実行**  
   ```bash
   agents-clean clean --profile ./profiles/agent_profile.yaml --output ./cleaned/agent_logs_cleaned.json --mode aggressive
   ```
   プロファイルに基づき、ノイズ除去、重複マージ、欠損補完を一括実行。`--mode`には`conservative`（最小限の削除）と`aggressive`（厳格な除去）を選択可能。実行後、変更ログが`cleaned/`ディレクトリに自動生成される。

3. **事後検証とレポート出力**  
   ```bash
   agents-clean validate --input ./cleaned/agent_logs_cleaned.json --report ./reports/clean_report.html
   ```
   クリーンアップ後のデータに対して、品質スコア（0〜100）を算出。スコアが80未満の場合は、どのルールが適用されたかを可視化するHTMLレポートを生成。エージェントの再トレーニング前に必ず実行する推奨ステップ。

### ベストプラクティス

1. **エージェントセッションのスナップショットを事前に取得**  
   agents-clean を実行する前に、元データのバックアップとしてスナップショットを取得してください。特に`--mode aggressive`を使用する場合、元に戻せない削除が発生する可能性があります。`agents-clean snapshot --input ./data/agent_logs.json --output ./backups/`で簡単に保存できます。

2. **プロファイルはエージェントのドメインごとに個別定義**  
   汎用的なプロファイルではなく、各エージェントが扱うデータの特性（例：自然言語、数値、コード）に合わせてプロファイルを調整してください。例えば、コード生成エージェントでは構文エラーをノイズとして扱うルールを追加すると効果的です。プロファイルはYAMLで記述可能で、Git管理を推奨します。
