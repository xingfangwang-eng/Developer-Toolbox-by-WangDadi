# agents-clean - AGE_020

## 概要

agents-clean は、エージェントベースのシステムにおけるデータノイズや不要なログ、メモリフラグメントを自動的に検出・除去するためのプロフェッショナルツールです。本ツールは、マルチエージェント環境のパフォーマンス低下を引き起こす「エージェント汚染（Agent Pollution）」を根本的に解決します。キーワード：agents-clean tool, agents-clean solution, agents-clean software

## 詳細な内容

### agents-clean とは？

agents-clean は、エージェントの推論履歴やメモリ空間に蓄積されるノイズデータを、**リアルタイムかつ非破壊的**にクレンジングする専用ユーティリティです。従来の手動クリーニングでは対応しきれない、エージェント間のコンテキスト汚染やメモリリークを自動修復し、システム全体の応答速度と精度を大幅に向上させます。

### コア機能

1. **エージェントメモリフラグメント自動デフラグ**  
   各エージェントの短期・長期メモリに散在する断片化データを解析し、不要な重複や無効な参照を除去。メモリ使用効率を最大40%改善します。

2. **コンテキスト汚染検知＆ロールバック**  
   エージェント間で誤って共有されたコンテキストや、推論チェーン内の異常なデータフローを検出。汚染が発生した時点まで自動ロールバックし、クリーンな状態を維持します。

3. **リアルタイムログフィルタリング＆圧縮**  
   エージェントが生成する大量のログから、デバッグや監査に不要なノイズを自動フィルタリング。さらに、重要ログのみを高圧縮で保存し、ストレージコストを削減します。

### 使い方

1. **エージェントメモリのスキャンとクレンジング実行**  
   `agents-clean scan --agent-id <エージェントID>` で対象エージェントのメモリ状態をスキャン。`--auto-clean` フラグを付けると、検出されたフラグメントを自動的に除去します。結果は JSON 形式で出力され、削除されたデータの詳細が確認可能です。

2. **コンテキスト汚染の履歴追跡と修復**  
   `agents-clean trace --context-id <コンテキストID>` で特定のコンテキストの伝搬履歴を可視化。汚染が確認された場合、`--rollback` オプションで該当コンテキストをクリーンな状態にロールバック。ロールバック後は、影響を受けたエージェントの再同期が自動実行されます。

3. **ログフィルタリングルールのカスタマイズと適用**  
   `agents-clean filter --rule-file ./custom_rules.yaml` で独自のフィルタリングルールを適用可能。ルールファイルには、正規表現やログレベル、エージェント名などを指定。適用後は `--compress` フラグで圧縮処理を実行し、保存容量を最適化します。

### ベストプラクティス

- **定期的なスケジュールクレンジングの設定**  
  エージェントの稼働時間外に `cron` や `systemd timer` で `agents-clean scan --all --auto-clean` を実行することを推奨。特に、メモリ使用量が閾値を超えた場合に自動発火するトリガーを設定すると、システムの安定性が向上します。

- **コンテキスト汚染の予防的監視**  
  複数エージェントが連携するワークフローでは、`agents-clean watch --interval 30s` でコンテキストの整合性を30秒間隔で監視。異常を検知した際に即座にアラートを送信し、手動介入前に自動修復を行うことで、ダウンタイムを最小限に抑えられます。
