# agents-clean - AGE_007

## 概要

agents-clean は、マルチエージェントシステムにおける非効率なリソース消費とノイズの多いログデータを排除するために設計された、次世代型クリーニングフレームワークです。特に大規模言語モデル（LLM）ベースのエージェント群が生成するメモリリークや重複コンテキストを自動検出・圧縮し、システム全体のパフォーマンスを最適化します。本ツールは、エージェントの動作履歴を解析し、不要なプロセスを刈り込むことで、レイテンシを最大40%削減します。

## コア機能

### 1. 動的コンテキストプルーニングエンジン
エージェントが保持する会話履歴やタスクコンテキストから、意味的に冗長なフラグメントをリアルタイムで特定し、削除します。トークン使用量を平均30%削減し、APIコストの最適化に直結します。内部では、TF-IDFと埋め込みベクトルの類似度スコアを組み合わせたハイブリッドアルゴリズムを採用。

### 2. マルチエージェントメモリデフラグ
分散エージェント間で共有されるメモリプールの断片化を検出し、自動的に再編成します。これにより、エージェント間の通信オーバーヘッドが低減し、タスク完了時間が平均25%短縮されます。特に、非同期メッセージングパターンを持つシステムで効果を発揮。

### 3. プロアクティブノイズフィルタリング
エージェントが生成するログやデバッグ出力から、エラーハンドリングに無関係なノイズ（例：タイムスタンプの重複、ステータスコードの誤報）を事前にフィルタリング。アラートの誤検出率を90%以上低減し、運用チームの認知負荷を劇的に軽減します。

## 使い方

### ステップ 1: クリーニングポリシーの定義
プロジェクトルートに `agents-clean.yml` を作成し、プルーニングの閾値（例：`similarity_threshold: 0.85`）とターゲットメモリプール（例：`memory_pool: shared_context`）を指定します。次のコマンドでポリシーを検証：
```bash
agents-clean validate --config ./agents-clean.yml
```

### ステップ 2: エージェントプロファイルのスキャン
全エージェントのメモリフットプリントを収集するため、以下のコマンドを実行：
```bash
agents-clean scan --agent-pid all --output profile.json
```
このJSONファイルには、各エージェントのトークン消費量、コンテキストヒストリー長、メモリ断片化率が含まれます。分析には `jq` との連携が推奨されます。

### ステップ 3: 自動最適化の適用
スキャン結果をもとに、クリーニングを実行：
```bash
agents-clean optimize --profile profile.json --strategy aggressive
```
`--strategy` フラグでは `conservative`（安全優先）または `aggressive`（パフォーマンス優先）を選択可能。実行後、`clean_report.json` に削除されたトークン数とパフォーマンス改善率が出力されます。

## ベストプラクティス

### CI/CDパイプラインへの統合
agents-clean をGitHub ActionsやGitLab CIに組み込み、プルリクエストごとにエージェントのメモリ健全性を自動チェックしましょう。`.github/workflows/agents-clean.yml` にスキャンステップを追加し、しきい値を超えた場合はビルドを失敗させることで、プロダクション環境でのメモリリークを未然に防げます。

### カスタムプルーニングルールの作成
デフォルトのルールセットでは対応できないドメイン特化型のノイズ（例：金融取引エージェントでの重複オーダーID）がある場合、`agents-clean` のプラグインAPIを使って独自のフィルターを実装しましょう。`rules/` ディレクトリにPythonスクリプトを配置し、`agents-clean run --custom-rules ./rules` で適用できます。
