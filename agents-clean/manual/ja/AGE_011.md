# agents-clean - AGE_011

## 概要

agents-clean は、エージェントベースのワークフローにおける不要なノイズや冗長なプロセスを自動検出・除去するための高パフォーマンスツールです。特にマルチエージェントシステムの運用において、メモリリークやコンテキスト汚染を防止し、エージェントの応答精度を最大化します。本ツールは、AI エージェントの動作ログを解析し、不要なプロンプトチェーンや重複するタスク実行を特定して排除します。

## 詳細な内容

### とは agents-clean?

agents-clean は、エージェントシステムの「クリーンアップ」に特化したプロフェッショナルツールです。エージェント間の通信ログやメモリ状態をリアルタイムで監視し、冗長なコンテキストや不要なエージェント呼び出しを自動的に除去します。これにより、エージェントの応答時間を最大 40% 短縮し、トークン消費を削減します。LangChain や AutoGen といった主要フレームワークとの統合も可能です。

### コア機能

1. **コンテキスト圧縮エンジン**  
   エージェントの会話履歴やメモリ内の冗長な情報をリアルタイムで検出し、重要度スコアに基づいて圧縮します。これにより、コンテキストウィンドウの効率的な利用とエージェントの応答品質向上を実現します。具体的には、TF-IDF と意味的類似度を組み合わせた独自アルゴリズムで、不要なプロンプトセグメントを特定します。

2. **エージェントループ検出器**  
   マルチエージェントシステムで発生する無限ループや循環参照を自動検出します。エージェント間のメッセージフローをグラフ構造で解析し、特定のパターン（例：エージェント A が B を呼び出し、B が再度 A を呼び出す）を検出した場合、即座にループを中断し、代替パスを提案します。

3. **プロンプトキャッシュ最適化**  
   エージェントが頻繁に使用するプロンプトテンプレートやシステムメッセージをキャッシュし、重複する API 呼び出しを削減します。キャッシュの有効期限や更新ポリシーを動的に調整し、トークン消費を最小化しながら応答速度を向上させます。

### 使い方

1. **エージェントログの解析とクリーンアップ**  
   `agents-clean analyze --log-path ./agent_logs/` を実行し、エージェントの動作ログを解析します。ツールは自動的に冗長なプロンプトチェーンや不要なエージェント呼び出しを特定し、クリーンアップ候補をリストアップします。`--auto-clean` フラグを追加すると、検出された問題を自動的に修正します。

2. **カスタムルールの設定**  
   `agents-clean config --rules ./custom_rules.yaml` で、特定のエージェントパターンやコンテキストの除外ルールを定義します。例えば、`ignore_patterns: ["debug_*", "temp_agent"]` のように設定することで、特定のエージェントやプロンプトをクリーンアップ対象から除外できます。

3. **リアルタイムモニタリングとアラート**  
   `agents-clean monitor --interval 10s` を実行し、エージェントシステムの状態をリアルタイムで監視します。メモリ使用量やコンテキストサイズが閾値を超えた場合、Slack やメールにアラートを送信します。`--threshold 80` で閾値をパーセンテージで指定可能です。

### ベストプラクティス

- **エージェントのライフサイクル管理**  
  エージェントのタスク完了後、即座にコンテキストをクリーンアップするよう設定します。`agents-clean` の `--auto-clean` オプションを有効にし、各エージェントセッション終了時に不要なメモリやログを自動削除することで、システム全体のパフォーマンスを維持します。

- **プロダクション環境での段階的導入**  
  まずはステージング環境で `agents-clean` をテストし、クリーンアップ後のエージェント応答品質を検証します。特に、コンテキスト圧縮がエージェントの判断に悪影響を与えないことを確認するため、`--dry-run` モードで影響範囲を事前に評価することを推奨します。
