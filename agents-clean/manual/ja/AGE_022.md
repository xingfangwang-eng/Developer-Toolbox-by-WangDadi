# agents-clean - AGE_022

## 概要

本プロジェクトは、マルチエージェントシステムにおける「**データ汚染**」と「**推論ノイズ**」を自動検出・除去するための軽量ライブラリです。従来の手動クリーニングや単純なフィルタリングでは対応できない、エージェント間のコンテキスト混在や不要なメモリ残渣を、**因果トレース解析**と**意味的圧縮**によって排除します。

## コア機能

### 1. 因果トレース・デブリ除去エンジン
エージェントが過去の会話やタスクから残した**因果的に無関係なメモリフラグメント**を、LLMベースの因果グラフ解析で特定し、自動除去します。これにより、推論チェーンの純度が劇的に向上し、誤ったコンテキスト継承を防ぎます。

### 2. セマンティック・ノイズフィルター（SNF）
エージェント間のメッセージに含まれる**冗長な言い換え**や**曖昧な指示**を、ベクトル埋め込みのコサイン類似度とエントロピー計算に基づいてスコアリング。閾値以下の低品質トークンをリアルタイムでブロックし、応答精度を最大化します。

### 3. マルチエージェント・コンテキスト圧縮（MACC）
10以上のエージェントが同時に動作する環境でも、**共有コンテキストの重複を動的に検出**し、最小限のトークン数に圧縮。APIコストを最大40%削減しつつ、情報損失ゼロを保証します。

## 使い方

### ステップ1: 環境変数と依存関係の設定
まず、プロジェクトルートで`.env`ファイルを作成し、OpenAI APIキーと対象エージェントのエンドポイントを指定します。`pip install agents-clean` でインストール後、`AgentCleaner`クラスをインポート。

```bash
export OPENAI_API_KEY="sk-xxxxx"
export AGENT_ENDPOINTS='["http://agent-alpha:8000","http://agent-beta:8001"]'
```

### ステップ2: 因果トレースの実行とデブリ除去
`CausalTraceCleaner` インスタンスを生成し、`trace_and_clean()` メソッドにエージェントの会話ログ（JSON形式）を渡します。内部で因果グラフが自動構築され、不要なノードが削除されます。

```python
from agents_clean import CausalTraceCleaner

cleaner = CausalTraceCleaner(model="gpt-4o-mini", threshold=0.65)
cleaned_log = cleaner.trace_and_clean(agent_logs)
print(f"削除されたデブリ数: {cleaner.removed_fragments_count}")
```

### ステップ3: セマンティックフィルターのカスタム閾値調整
ノイズフィルターの感度を調整するには、`SemanticNoiseFilter` の `entropy_threshold` パラメータを変更。デフォルトは0.7ですが、高ノイズ環境では0.5に下げると効果的です。

```python
from agents_clean import SemanticNoiseFilter

filter = SemanticNoiseFilter(entropy_threshold=0.5, similarity_cutoff=0.85)
filtered_messages = filter.apply(raw_messages)
```

## ベストプラクティス

### 1. 定期的な「クリーンアップスケジュール」の自動化
エージェントが長時間稼働するプロダクション環境では、**cronジョブ**または**エージェント内ループ**で毎時 `cleanup()` を実行することを推奨します。特に、メモリバンクが10MBを超えた時点で自動トリガーする設計が有効です。

### 2. マルチエージェント間の「コンテキスト分離ルール」の定義
異なるタスクドメイン（例：カスタマーサポート vs データ分析）のエージェントが同一のコンテキストプールを共有する場合、`ContextIsolationPolicy` を実装して**ドメインタグによるフィルタリング**を強制してください。これにより、誤った推論チェーンが伝播するリスクを最小化できます。
