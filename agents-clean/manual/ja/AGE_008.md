# agents-clean - AGE_008

## 概要

エージェント駆動型システムにおけるデータ汚染を根本から解決する、次世代クレンジングフレームワーク。agents-clean は、マルチエージェント環境で発生するノイズ、重複、コンテキストの断片化を自動検出・修復するプロフェッショナルツールです。SaaS アーキテクチャに特化し、スケーラブルなデータ整合性を実現します。

## 詳細な内容

### agents-clean とは

agents-clean は、エージェント間の通信ログ、メモリ状態、タスク履歴をリアルタイムで分析し、不要なエントリを除去・正規化するクレンジングエンジンです。従来のバッチ処理とは異なり、ストリーム処理とメタデータベースのハイブリッドアプローチを採用。特に、LLM ベースのエージェントが生成する幻覚データや、非決定的な出力による不整合を、ポリシーベースで自動補正します。

### コア機能

1. **コンテキストフラグメント修復エンジン**  
   マルチエージェント間で発生するコンテキストの部分欠落やタイムスタンプのずれを、分散トレーシングと因果グラフを用いて自動補完。修復後は整合性ハッシュを付与し、再発を防止。

2. **動的ノイズフィルタリング**  
   エージェントの動作パターンを学習し、異常値や外れ値をリアルタイムで特定。閾値ベースではなく、確率モデル（ベイズ推定）を用いてノイズとシグナルを分離。誤検知率を従来比で 60% 削減。

3. **ポリシー駆動型重複排除**  
   同一セッション内で発生する重複エントリを、セマンティック類似度とメタデータのタイムラインから判定。ユーザー定義のポリシー（例：最新のみ保持、論理和マージ）に従って自動マージ。履歴は監査ログとして保存。

### 使い方

1. **初期セットアップとポリシー定義**  
   まずは `agents-clean init` を実行し、対象となるエージェントのデータソース（Kafka トピック、PostgreSQL テーブル、Redis ストリーム）を登録。その後、`config/policy.yaml` にクレンジングルールを記述。例えば、`dedup: { strategy: "semantic_merge", threshold: 0.85 }` と設定すれば、85% 以上の類似度を持つエントリを自動マージする。設定後は `agents-clean validate` でポリシーの整合性チェックを実施。

2. **ストリーム処理の実行と監視**  
   `agents-clean run --stream --source kafka://events --sink postgresql://cleaned` でリアルタイム処理を開始。処理中は `agents-clean stats` でスループット（1 秒あたりの処理エントリ数）とエラー率を監視。異常が検出された場合、`agents-clean pause --reason "high_error_rate"` で一時停止し、ログを確認。再開は `agents-clean resume`。

3. **バックフィルと整合性検証**  
   過去データのクレンジングが必要な場合、`agents-clean backfill --from 2024-01-01 --to 2024-12-31 --batch-size 10000` を実行。処理後は `agents-clean verify --check integrity_hash` で全エントリの整合性を検証。不一致が検出された場合は `agents-clean repair --auto` で自動修復。修復履歴は `agents-clean audit --since 2024-01-01` でエクスポート可能。

### ベストプラクティス

- **段階的ロールアウトとカナリアデプロイ**  
  本番環境に適用する前に、まず `agents-clean dry-run` で影響範囲をシミュレーション。特に、重複排除の閾値は最初は緩め（0.7 程度）に設定し、1 週間の運用データを元に段階的に厳しくする。カナリアリリースでは、全エージェントの 10% にのみ適用し、エラー率が 0.1% 未満であることを確認してから全面展開する。

- **監査ログの定期エクスポートとバックアップ**  
  クレンジング操作はすべて `agents-clean audit --format parquet --output s3://bucket/audit/` で日次エクスポート。これにより、万が一のデータ損失時でも、元の状態にロールバック可能。また、ポリシー変更時は必ず `agents-clean snapshot` で現在の状態をスナップショットとして保存し、変更後に `agents-clean diff` で差分を確認する。
