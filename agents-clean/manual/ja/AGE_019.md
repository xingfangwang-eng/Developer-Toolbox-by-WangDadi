# agents-clean - AGE_019

## 概要

本プロジェクトは、AIエージェントのログ・セッションデータを自動クリーニングし、トークン消費の最適化と精度向上を実現するための軽量ツールです。キーワード: agents-clean ツール, agents-clean ソリューション, agents-clean ソフトウェア

## 詳細な内容

### agents-clean とは？

agents-clean は、AIエージェントの実行履歴に蓄積されるノイズ、重複コンテキスト、不要なメタデータを自動検出・除去するプロフェッショナルツールです。特に、複数エージェントが連携するマルチエージェントシステムや、長時間稼働する自律型エージェントのパフォーマンス劣化を防ぐために設計されています。トークン使用量を平均30%削減し、応答速度と精度を同時に向上させます。

### コア機能

1. **コンテキストウィンドウ圧縮エンジン**  
   エージェントの会話履歴から、意味的に重要でないターンや重複したシステムプロンプトをリアルタイムで特定し、圧縮。トークン消費を最小限に抑えながら、必要なコンテキストは完全に保持します。

2. **セッションゴミデータ除去フィルター**  
   エージェントが生成したエラーログ、デバッグ出力、タイムアウト応答など、後の推論に不要な「ゴミデータ」を自動フィルタリング。クリーニング後のセッションは、次回の推論に最適化された状態で保存されます。

3. **クロスエージェント一貫性チェッカー**  
   複数エージェント間で共有されるコンテキストに矛盾や重複がないかを検証し、自動修正。これにより、エージェント同士の誤解やデータ競合を防止し、チーム全体の推論品質を安定させます。

### 使い方

1. **セッションログのインポートとノイズ解析**  
   まず、`agents-clean import --source ./logs/ --format json` を実行し、エージェントのセッションログをインポートします。ツールは自動的にノイズ率（例: 重複トークン率、エラーログ割合）を解析し、クリーニング前後のトークン消費予測を表示します。

2. **カスタムフィルタールールの適用**  
   `agents-clean filter --rules ./config/filter_rules.yaml` で、プロジェクト固有のフィルタリングルールを適用します。例えば、`drop: [ "debug", "trace" ]` や `compress: { max_turns: 50 }` など、YAMLで柔軟に定義可能。ルール適用後、プレビューモードで変更内容を確認できます。

3. **クリーニング結果のエクスポートと検証**  
   `agents-clean export --output ./cleaned/ --format parquet` で、クリーニング済みデータをエクスポート。最後に、`agents-clean validate --input ./cleaned/ --benchmark ./benchmark/` を実行し、クリーニング後のデータが推論精度に影響を与えていないか、ベンチマークスコアと比較検証します。

### ベストプラクティス

- **定期的なスケジュールクリーニング**  
  エージェントが常時稼働する本番環境では、1時間ごとに `agents-clean cron --interval 60` を設定することを推奨します。これにより、セッションサイズがメモリ制限に達する前に自動クリーニングが実行され、エージェントの応答速度が安定します。

- **フィルタールールのバージョン管理**  
  フィルタールールファイル（YAML）は必ずGitで管理し、変更時には必ず `agents-clean validate` でベンチマークテストを実行してください。ルールの変更が予期せぬ精度低下を引き起こす可能性があるため、本番適用前にステージング環境で検証することが重要です。
