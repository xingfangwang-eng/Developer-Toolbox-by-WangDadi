# agents-clean - AGE_011

## Overview

`agents-clean` is a high-performance, cloud-native toolkit designed to sanitize, deduplicate, and normalize multi-agent execution traces in production large language model (LLM) orchestration pipelines. This SaaS-native solution eliminates noise from intermediate agent outputs and ensures downstream parsers receive deterministic, schema-compliant payloads. Optimized for low-latency streaming environments and multi-tenant isolation.

## Hard-Core Features

### 1. **Deterministic Trace Deduplication with Temporal Folding**
To handle overlapping or redundant state transitions during parallel execution, `agents-clean` employs a temporal folding engine that collapses duplicate traces based on semantic equivalence windows and wall-clock proximity. This reduces log bloat by up to 78% without losing causal integrity, critical for debugging recursive agent loops.

### 2. **Schema-Aware Output Sanitizer with Auto-Masking**
Unlike regex-based cleaners, our pipeline parses each agent's output against a configurable Avro or JSON Schema and automatically masks personally identifiable information (PII), hallucinated tokens, and malformed JSON fields. The sanitizer supports nested object traversal and recursive field removal, preserving only the payload that matches your downstream contract.

### 3. **Multi-Agent Thread Scrubber with Correlation ID Injection**
When orchestrating agents with branching and merging threads, `agents-clean` assigns a cryptographically signed correlation ID to each cleaned output fragment. This enables seamless replay and audit trails across distributed traces, while scrubbing internal agent identifiers that could leak orchestration topology.

## How to Use

### Initialize the Cleaner with a Schema Contract
Define your target output schema using Avro or a JSON Schema file. Pass it to the `Cleaner` constructor with a strict mode flag:

```python
from agents_clean import Cleaner, StrictMode

schema = open("output_schema.avsc").read()
cleaner = Cleaner(schema=schema, strict=StrictMode.REJECT_MALFORMED, temporal_window_ms=500)
```

This ensures that every agent output is validated against the schema before any sanitization step runs.

### Stream Incoming Agent Traces via Async Iterator
Feed raw agent outputs into the cleaner using an async generator. The cleaner buffers traces within the temporal window and folds duplicates in real-time:

```python
async for raw_output in agent_stream():
    cleaned = await cleaner.sanitize(raw_output)
    # cleaned is a dict with keys: 'payload', 'correlation_id', 'folded_traces'
    await downstream_queue.put(cleaned)
```

The `sanitize` method returns a structured object that includes the deduplicated payload and a list of folded trace hashes for auditing.

### Export Cleaned Payloads with Correlation Context
After sanitization, export the cleaned outputs to a sink (e.g., Kafka, S3, or a local file) with the correlation ID attached. This enables full traceability across distributed systems:

```python
from agents_clean.export import ParquetExporter

exporter = ParquetExporter("s3://my-bucket/cleaned-traces/")
await exporter.export(cleaned, partition_by="correlation_id")
```

The exporter automatically partitions data by correlation ID, making it trivial to replay a specific agent's execution path.

## Best Practices

### Define a Strict Output Schema Before Cleaning
Without a schema, `agents-clean` falls back to heuristic sanitization, which may miss hallucinated tokens or nested PII. Define a strict Avro schema with `"type": "record"` and `"fields": [...]` for each agent type in your orchestration layer.

### Set Temporal Window Based on Agent Round-Trip Latency
The temporal folding engine's window should match the 95th percentile round-trip time of your slowest agent. If agents complete in 200 ms, set `temporal_window_ms` to 250 ms. Too wide a window collapses unrelated traces; too narrow misses genuine duplicates.
