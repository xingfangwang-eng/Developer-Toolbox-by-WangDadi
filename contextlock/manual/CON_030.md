# contextlock - CON_030

## Overview

ContextLock is a **persistent AI memory engine** designed for developers who need to maintain coherent, long-running conversations with LLMs without losing critical context across sessions, branches, or toolchains. It solves the fragmentation problem in AI-assisted development workflows by locking contextual state into reusable, version-controlled snapshots.

## Core Features

### 1. **Context Graph Snapshots (CGS)**
ContextLock captures not just raw chat history, but the semantic dependency graph of your conversation — including resolved variables, inferred intent, and decision trees. Each snapshot is a self-contained DAG that can be replayed, branched, or merged like code. This eliminates the "what did I tell the AI last week?" problem.

### 2. **Multi-Provider State Synchronization**
Seamlessly bridge context between OpenAI, Anthropic, local models (via Ollama), and custom endpoints. ContextLock normalizes token-aware state into a unified JSON schema, ensuring that a context exported from Claude can be resumed in GPT-4 without drift. Built-in conflict resolution handles schema mismatches and token budget warnings.

### 3. **Context Hooks & CI/CD Integration**
Inject context into your pipeline. Use ContextLock as a pre-commit hook to auto-attach relevant conversation history to issue tracking, PR descriptions, or changelogs. The CLI tool supports `contextlock push` and `contextlock pull` commands, enabling teams to share AI context alongside code in version control.

## How to Use

### 1. **Initialize a Context Lock File**
```bash
contextlock init --project ./my-ai-app --provider openai --model gpt-4-turbo
```
This generates a `.contextlock/config.yaml` with your provider credentials, token budget (default: 4096), and a `context.graph` file. The graph is initially empty — it will populate as you interact.

### 2. **Capture and Tag a Context Snapshot**
During a developer session, run:
```bash
contextlock snapshot --tag "auth-refactor-v2" --branch "feature/oauth"
```
This serializes the current conversation history, including all system prompts, user messages, and assistant responses, into a timestamped snapshot. The `--branch` flag enables parallel context trees — useful for A/B testing prompt strategies.

### 3. **Restore and Diff Context Across Sessions**
To resume work from a previous snapshot:
```bash
contextlock restore --tag "auth-refactor-v2" --diff
```
The `--diff` flag compares the restored context with the current state of your project (e.g., changes in `src/` or `docs/`). ContextLock highlights semantic mismatches — for example, if a function signature changed since the snapshot was taken — and suggests a context merge or rollback.

## Best Practices

### **1. Version Control Your Context Like Code**
Treat `.contextlock/` as a first-class citizen in your repository. Include it in `.gitignore` only if you're dealing with sensitive tokens. For team projects, use `contextlock push` to sync context to a shared remote (e.g., S3 or Git LFS). This ensures that every developer can pick up where another left off, reducing onboarding friction by up to 40%.

### **2. Use Context Locking for Regression Testing**
Before deploying a prompt change or model update, run a context replay test: `contextlock replay --snapshot "prod-v1" --target-model "gpt-4o-mini"`. This replays the exact conversation graph against the new model, flagging any deviation in output. Combine with `contextlock diff` to generate a report of semantic drift — critical for maintaining deterministic AI behavior in production.
