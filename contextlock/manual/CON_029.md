# contextlock - CON_029

## Overview

ContextLock is a developer-first AI context persistence layer designed for teams building multi-turn LLM workflows, RAG pipelines, and autonomous agents. It eliminates context drift, token fragmentation, and memory loss across sessions—without forcing you into a proprietary runtime. Built for TypeScript and Python ecosystems, it hooks directly into your existing toolchain.

## Hardcore Features

### 1. **Deterministic Context Snapshots with Immutable Checkpoints**
Each interaction is hashed and stored as an immutable checkpoint. You can roll back, fork, or replay any context state—like `git bisect` for your LLM session. No more "I lost the thread" when debugging agent behavior.

### 2. **Zero-Copy Context Injection via Shared Memory Mapping**
ContextLock uses `mmap`-backed storage for ultra-low-latency context reads across processes. Your agent or pipeline can access persistent memory without serialization overhead. Ideal for real-time streaming or parallel inference.

### 3. **Declarative Context Schemas with Auto-Eviction Policies**
Define context structure using Zod or Pydantic schemas. ContextLock enforces schema validation at write time and supports LRU, TTL, and sliding window evictions. No more bloated or stale context poisoning your prompts.

## How to Use

### 1. **Initialize a Context Lock Instance**
```bash
npm install contextlock
# or
pip install contextlock
```

```typescript
import { ContextLock } from 'contextlock';

const lock = new ContextLock({
  namespace: 'agent-session-42',
  schema: z.object({
    userIntent: z.string(),
    previousSteps: z.array(z.string()).max(5),
    lastError: z.string().optional()
  }),
  eviction: 'lru',
  maxTokens: 4096
});
```

### 2. **Inject Context into Your LLM Call**
```typescript
const context = await lock.getContext();
const response = await openai.chat.completions.create({
  model: 'gpt-4',
  messages: [
    ...context.messages,
    { role: 'user', content: 'Continue debugging the last step.' }
  ]
});
await lock.append({ role: 'assistant', content: response.choices[0].message.content });
```

### 3. **Snapshot and Revert for Debugging**
```typescript
const snapshotId = await lock.snapshot(); // returns a SHA-256 hash
// ... later ...
await lock.revert(snapshotId); // rollback to exact state
```

## Best Practices

- **Use ContextLock as a Sidecar in Microservices**: Instead of embedding memory logic into each agent, run ContextLock as a lightweight sidecar process. This decouples context management from business logic and allows you to scale memory independently.
- **Implement Session-Aware Eviction Policies**: Don’t use a single global TTL. Instead, set per-session `maxTokens` and `slidingWindow` based on expected conversation depth. For long-running agents, combine LRU with a priority flag for critical context entries (e.g., user authentication tokens).
