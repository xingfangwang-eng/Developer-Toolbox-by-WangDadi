# contextlock - CON_006

## Overview

This is a no-fluff, developer-first breakdown of ContextLock — a persistent AI context management tool engineered for teams shipping at scale. If your AI agents keep forgetting who they're talking to, or you're drowning in prompt boilerplate, this is your fix. No wrappers, no fluff.

## What is ContextLock?

ContextLock is a lightweight, embeddable context engine that gives your AI agents long-term memory without the overhead of vector databases or brittle prompt engineering. It's designed for CI/CD pipelines, dev tooling, and multi-agent architectures where context drift kills productivity.

## Core Features

1. **Session-Aware Context Freezing**  
   ContextLock snapshots the entire conversational state (system prompt, user intents, tool outputs) into a deterministic hash. On resume, it restores the exact semantic vector — no hallucination, no drift. This is not a chat history buffer; it's a stateful context graph.

2. **Zero-Dependency Token Budget Manager**  
   Automatically trims and prioritizes context tokens based on recency, relevance, and user-defined weights. No more hitting token limits mid-query. You define the budget; ContextLock handles eviction with LRU + semantic similarity scoring.

3. **Multi-Agent Context Sync**  
   When you have multiple AI agents (code review, docs, testing) working the same task, ContextLock syncs their context via a shared lockfile. No race conditions, no duplicate work. Each agent sees the same "ground truth" without polling or Redis.

## How to Use

1. **Initialize a ContextLock session in your CLI tool**  
   ```bash
   contextlock init --session-id "pr-237-review" --budget 4096
   ```  
   This creates a `.contextlock` file in your project root. The session ID ties all subsequent interactions to a single context graph. The budget flag sets the max token count before pruning kicks in.

2. **Inject context into your AI agent's system prompt**  
   In your Python script or Makefile, pipe the context string directly:  
   ```python
   from contextlock import ContextLock
   cl = ContextLock.load("pr-237-review")
   system_prompt = cl.render(style="openai")  # returns a formatted system message
   ```  
   The `render()` method outputs a compressed, token-efficient string that includes only the most relevant context chunks. You can also export as JSON for custom parsers.

3. **Commit and sync context across team workflows**  
   After your agent finishes, run:  
   ```bash
   contextlock commit --message "completed code review for PR-237" --tag "reviewer:alice"
   ```  
   This updates the lockfile with the new state. Other agents (e.g., a docs generator) can then `contextlock pull --session-id "pr-237-review"` to get the latest context without re-inventing the wheel.

## Best Practices

- **Use context tags for multi-stage pipelines**  
  Tag each commit with the agent role (e.g., `reviewer`, `tester`, `writer`). This lets you filter context by stage when debugging. In production, we've seen teams reduce context reconstruction time by 70% just by tagging.

- **Set a strict token budget per agent type**  
  Don't use the same budget for a code generator (needs more context) and a linter (needs less). Define budgets per agent in your `contextlock.yaml` config. This prevents expensive agents from starving smaller ones.
