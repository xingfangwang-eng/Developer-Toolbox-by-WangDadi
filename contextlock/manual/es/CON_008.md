# contextlock - CON_008

## Resumen

ContextLock es una herramienta CLI diseñada para desarrolladores que trabajan con modelos de lenguaje grandes (LLMs) y necesitan preservar el estado del contexto entre sesiones de trabajo. Elimina la pérdida de memoria contextual al permitirte congelar, restaurar y compartir configuraciones de contexto complejas entre equipos distribuidos.

## Contenido Detallado

### ¿Qué es ContextLock?

ContextLock resuelve el problema crítico de la volatilidad del contexto en flujos de trabajo con IA. Cuando trabajas con asistentes de código, chatbots o pipelines de generación, el contexto se pierde al cerrar una sesión o cambiar de rama. ContextLock permite serializar el estado completo del contexto —incluyendo historial de conversaciones, variables de entorno, configuraciones de modelo y metadatos de proyecto— en un archivo cifrado y versionable.

### Características Principales

- **Snapshot Contextual Versionado**: Cada vez que ejecutas `contextlock freeze`, se genera un hash SHA-256 del contexto actual. Puedes etiquetar snapshots con metadatos semánticos (ej: `feature/auth-flow-v3`) y revertir a cualquier punto anterior sin pérdida de datos. El sistema soporta diff automático entre snapshots, mostrando exactamente qué cambió en el contexto.

- **Contexto Compartido con Control de Acceso Granular**: Los snapshots se almacenan en un repositorio local cifrado con AES-256-GCM. Puedes compartir contextos específicos con miembros del equipo mediante claves de acceso temporales, sin exponer el historial completo. Ideal para equipos que colaboran en prompts complejos o pipelines de generación de código.

- **Hooks de Pre/Post Contexto**: Integra ContextLock en tu CI/CD. Define hooks que se ejecutan automáticamente al congelar o restaurar contexto. Por ejemplo, un hook post-freeze puede enviar el snapshot a un bucket S3, mientras que un hook pre-restore puede validar que las dependencias del proyecto coincidan con el contexto guardado.

### Cómo Usar

1. **Congelar el contexto actual con metadatos**: Ejecuta `contextlock freeze --tag "refactor-db-layer" --meta "branch:feature/db-optimization,commit:abc123"`. Esto genera un snapshot cifrado en `~/.contextlock/snapshots/`. El flag `--meta` permite añadir pares clave-valor que se indexan automáticamente para búsqueda posterior.

2. **Restaurar contexto desde un snapshot específico**: Usa `contextlock restore --snapshot "refactor-db-layer" --env production`. Esto restaura el historial de conversación, variables de entorno y configuraciones de modelo. Si el entorno no coincide con el del snapshot, ContextLock muestra un warning y permite forzar la restauración con `--force`.

3. **Buscar snapshots por contenido**: `contextlock search "autenticación OAuth" --limit 5`. ContextLock indexa el contenido textual de cada snapshot (conversaciones, prompts, respuestas) usando TF-IDF. Puedes buscar por términos específicos y obtener snapshots relevantes, incluso meses después de haberlos creado.

### Mejores Prácticas

- **Estandariza las etiquetas de snapshots**: Define un naming convention para tus tags, como `{tipo}/{feature}/{version}` (ej: `prompt/login-flow/v2`). Esto facilita la búsqueda y evita colisiones. En equipos grandes, recomiendo usar un script de pre-commit que valide que los tags sigan el patrón definido.

- **Automatiza el congelado de contexto en puntos críticos**: Integra `contextlock freeze` en tus hooks de Git (pre-commit, post-merge) y en tu pipeline de CI/CD. Por ejemplo, después de un merge exitoso, congela automáticamente el contexto de la rama principal. Esto crea un historial de contexto que puedes usar para debugging retrospectivo o para entrenar modelos internos.
