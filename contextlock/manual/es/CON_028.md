# contextlock - CON_028

## Resumen Técnico

ContextLock es un gestor de contexto persistente para desarrolladores que trabajan con APIs de lenguaje natural. Elimina la pérdida de memoria contextual entre sesiones de IA, permitiendo mantener hilos de trabajo coherentes sin reinicios ni repeticiones. Ideal para equipos que integran modelos de lenguaje en pipelines CI/CD, asistentes de código o herramientas de análisis automatizado.

## Funcionalidades Clave

### 1. Bloqueo de Contexto con Hash de Sesión
Cada interacción con la API se vincula a un hash único generado a partir del estado actual del proyecto (ramas Git, variables de entorno, archivos activos). Esto asegura que el contexto no se mezcle entre diferentes tareas o entornos, eliminando colisiones en equipos distribuidos.

### 2. Persistencia Asíncrona con Reinteligencia
El contexto se almacena en un backend ligero (SQLite o Redis) con un sistema de reinteligencia: si detecta que el modelo ha perdido el hilo (por ejemplo, tras un timeout o reinicio del servidor), ContextLock inyecta automáticamente los últimos 3 bloques de contexto relevante sin intervención manual.

### 3. Compresión de Contexto con Priorización Semántica
No todo el historial es igual de importante. ContextLock analiza la relevancia semántica de cada bloque de texto usando embeddings locales (sin dependencia externa) y descarta automáticamente fragmentos redundantes o de baja prioridad, manteniendo el contexto dentro del límite de tokens sin perder información crítica.

## Cómo Usar ContextLock en tu Proyecto

### 1. Inicializa el Contexto en tu Pipeline
```bash
contextlock init --project mi-proyecto --backend sqlite
```
Esto genera un archivo `.contextlockrc` en la raíz del proyecto y un almacén local. Asegúrate de que tu CI/CD lo ignore (añádelo a `.gitignore`). El hash de sesión se calcula automáticamente al primer uso.

### 2. Vincula el Contexto a tus Llamadas API
En tu script de integración con la IA, envuelve cada llamada con el decorador `@contextlock`:
```python
from contextlock import contextlock

@contextlock(session="mi-sesion", max_tokens=4096)
def consultar_ia(prompt):
    # Tu lógica de llamada a la API
    return respuesta
```
ContextLock inyecta automáticamente el contexto previo en el prompt y gestiona la compresión si excedes el límite de tokens.

### 3. Reanuda Sesiones Interrumpidas
Si el proceso falla o se reinicia, simplemente ejecuta:
```bash
contextlock resume --session mi-sesion --force
```
Esto restaura el último estado coherente, omitiendo bloques corruptos o incompletos. Ideal para procesos largos como análisis de logs o generación de documentación.

## Mejores Prácticas para Desarrolladores

### 1. Segmenta por Módulos, No por Tiempo
No uses una sola sesión para todo el proyecto. Crea sesiones independientes para cada módulo o funcionalidad (por ejemplo, `sesion-backend`, `sesion-frontend`, `sesion-tests`). Esto evita que el contexto de una parte del sistema contamine a otra, especialmente en proyectos monolíticos.

### 2. Configura un Límite de Tokens Realista
No uses el máximo permitido por la API (por ejemplo, 32k tokens). Establece un límite del 70-80% para dejar espacio a la respuesta del modelo. ContextLock te permite definir `max_tokens` por sesión; ajústalo según la complejidad de las consultas. Para tareas rutinarias, 2048 tokens suele ser suficiente.
