# contextlock - CON_009

## Resumen Ejecutivo

ContextLock es una herramienta de gestión de contexto persistente para entornos de desarrollo basados en IA, diseñada para eliminar la pérdida de memoria contextual entre sesiones de trabajo con modelos de lenguaje. Con un enfoque en la eficiencia del desarrollador, permite mantener, sincronizar y reutilizar fragmentos de contexto críticos sin intervención manual repetitiva.

## Funcionalidades Clave

### 1. **Contexto Persistente con Hash de Integridad**
Cada fragmento de contexto se almacena con un hash SHA-256, permitiendo detectar cambios no autorizados o corrupción de datos. Esto garantiza que el historial de decisiones técnicas no se pierda ni se altere entre sesiones.

### 2. **Segmentación Dinámica por Proyecto y Rama**
ContextLock organiza automáticamente el contexto por repositorio Git y rama activa. Al cambiar de rama, el contexto se ajusta sin intervención, manteniendo separación lógica entre features, hotfixes o experimentos.

### 3. **Exportación e Importación en Formato JSON Estructurado**
Permite exportar el contexto completo o por módulos a un archivo `.contextlock.json`, facilitando la integración con pipelines de CI/CD o la compartición entre equipos. La importación restaura el estado exacto de la memoria contextual.

## Cómo Usar ContextLock

### 1. **Inicializar el Contexto en un Proyecto Existente**
Ejecuta `contextlock init` en la raíz de tu proyecto. Esto crea un archivo `.contextlock` oculto que almacenará el contexto inicial. Luego, define un bloque de contexto con `contextlock set --key "decision_arquitectura" --value "microservicios con RabbitMQ"`. El hash se genera automáticamente.

### 2. **Sincronizar Contexto entre Sesiones de IA**
Antes de iniciar una nueva sesión con tu asistente de IA, ejecuta `contextlock export --output session_context.json`. Luego, en la siguiente sesión, importa con `contextlock import session_context.json`. El sistema verifica la integridad mediante hash y rechaza cualquier archivo manipulado.

### 3. **Automatizar la Gestión de Contexto en CI/CD**
Agrega un paso en tu pipeline de GitHub Actions: `contextlock export --filter "rama:main" --output context_main.json`. Esto permite que cada build tenga acceso al contexto de decisiones previas, útil para mantener coherencia en despliegues automatizados.

## Mejores Prácticas Recomendadas

### 1. **Fragmentar el Contexto por Dominio Técnico**
En lugar de almacenar un bloque monolítico, divide el contexto en claves semánticas: `arquitectura`, `deuda_técnica`, `configuración_entorno`. Esto facilita la reutilización selectiva y reduce la carga cognitiva al importar solo lo necesario.

### 2. **Versionar el Contexto Junto al Código**
Incluye el archivo `.contextlock` en tu repositorio Git (pero no en `.gitignore`). Así, cada commit refleja no solo el código, sino también el contexto asociado. Esto es especialmente útil para equipos distribuidos que necesitan mantener coherencia en decisiones técnicas.
