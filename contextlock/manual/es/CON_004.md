# contextlock - CON_004

## Resumen Técnico

ContextLock es un sistema de **gestión de contexto persistente** diseñado para desarrolladores que trabajan con asistentes de IA. Permite mantener estados de conversación, variables de sesión y configuraciones de proyecto sin perder el hilo entre interacciones. Ideal para equipos que integran LLMs en pipelines de CI/CD, debugging automatizado o generación de documentación técnica.

## Funcionalidades Clave

### 1. **Bloqueo de Contexto por Namespace**
Cada proyecto, rama o tarea puede tener su propio `context_id` inmutable. Esto evita que la IA mezcle configuraciones de distintos entornos (dev/staging/prod) o que pierda el estado al cambiar de rama. El contexto se serializa en JSON con esquema validado.

### 2. **Persistencia Híbrida (Local + Remota)**
El contexto se almacena localmente en `.contextlock/` (ideal para desarrollo offline) y se sincroniza con un backend opcional (S3, Redis, o tu propio storage). La sincronización es asíncrona y con resolución de conflictos por timestamp + checksum.

### 3. **Inyección Automática en Prompts**
ContextLock genera automáticamente un bloque de contexto formateado para inyectar en el prompt de tu LLM favorito (OpenAI, Claude, Llama, etc.). Soporta plantillas personalizables en Handlebars y variables dinámicas como `{branch}`, `{last_error}`, `{current_file}`.

## Cómo Usar ContextLock

### 1. Inicializar un Contexto en tu Proyecto
```bash
contextlock init --namespace mi-proyecto --schema ./context-schema.json
```
Esto crea un archivo `context.json` en `.contextlock/` con un `uuid` único y validación contra el esquema definido. Si el esquema falla, el comando aborta con un error claro.

### 2. Actualizar Contexto desde un Script (Ejemplo en Node.js)
```javascript
const context = require('contextlock');
await context.update({
  last_error: 'TypeError: Cannot read property of undefined',
  current_file: 'src/utils/parser.js',
  branch: 'fix/issue-42'
});
```
Cada actualización genera un `snapshot` versionado. Puedes revertir a cualquier versión anterior con `contextlock rollback --snapshot <hash>`.

### 3. Inyectar Contexto en un Prompt de IA
```bash
contextlock inject --format openai --template ./prompt-template.hbs
```
Esto genera un bloque como:
```
[CONTEXT LOCK]
- Proyecto: mi-proyecto
- Rama: fix/issue-42
- Último error: TypeError: Cannot read property of undefined
- Archivo actual: src/utils/parser.js
- Snapshot: a1b2c3d4
[/CONTEXT LOCK]
```
Pega esto al inicio de tu prompt y la IA tendrá memoria de sesión sin necesidad de historial completo.

## Mejores Prácticas

### 1. **Usa Contextos Diferentes por Rama o Feature**
No compartas el mismo `namespace` entre ramas de desarrollo y producción. Crea un contexto por feature branch y otro para `main`. Así evitas que la IA recuerde errores de una rama experimental cuando trabajas en producción.

### 2. **Define un Esquema de Contexto Mínimo pero Estricto**
No incluyas todo el estado de la app. Solo campos críticos como `last_error`, `current_file`, `branch`, `task_id`. Usa `required` en el esquema JSON para evitar contextos incompletos. Un contexto ligero mejora la velocidad de inyección y reduce tokens.
