# contextlock - CON_006

## Resumen Técnico

ContextLock es una herramienta de gestión de contexto persistente para desarrolladores que trabajan con asistentes de IA y modelos de lenguaje. Elimina la fragmentación de memoria en sesiones de trabajo, permitiendo que tu agente de IA recuerde decisiones arquitectónicas, preferencias de estilo y dependencias entre módulos sin necesidad de reinyectar manualmente el contexto en cada interacción. Ideal para equipos que integran LLMs en flujos CI/CD o entornos de desarrollo.

## Funcionalidades Clave

### 1. Bloqueo de Contexto por Namespace Jerárquico
Cada proyecto o submódulo puede definir su propio `contextlock` con herencia de ámbito. Por ejemplo, un namespace `backend/api/v2` hereda automáticamente las reglas de `backend/api`, pero permite sobrescribir variables de contexto sin afectar al resto del árbol. Esto evita colisiones en equipos grandes que trabajan con múltiples microservicios.

### 2. Memoria Persistente con Hash de Dependencias
ContextLock genera un hash SHA-256 del árbol de dependencias de cada sesión. Si el hash cambia (por ejemplo, al actualizar una librería crítica), el sistema invalida automáticamente el contexto cacheado y fuerza una reinyección completa. Esto garantiza que la IA nunca opere con información obsoleta.

### 3. Exportación de Contexto como Payload JSON-LD
Cada bloque de contexto puede serializarse en formato JSON-LD, permitiendo su consumo directo por motores de búsqueda semántica o pipelines de RAG. Incluye metadatos como timestamp, versión del proyecto y nivel de confianza del contexto (por ejemplo, `verified`, `tentative`). Ideal para auditoría y trazabilidad.

## Guía de Uso Avanzada

### 1. Inicializar un Bloque de Contexto en un Proyecto Existente
```bash
contextlock init --namespace "frontend/componentes" --scope "local"
```
Esto crea un archivo `.contextlock.json` en el directorio raíz. El flag `--scope local` limita el contexto al directorio actual, mientras que `--scope global` lo propaga a todos los submódulos. Revisa el archivo generado: incluye un array `memory` vacío y un campo `hash` que se actualizará automáticamente.

### 2. Inyectar Contexto Dinámico desde un Pipeline de CI
```yaml
- name: Inyectar contexto de IA
  run: |
    contextlock inject --from-env --output .contextlock.json
    echo "Contexto sincronizado con variables de entorno: $CONTEXT_LOCK_VARS"
```
El flag `--from-env` lee variables como `API_KEY`, `BRANCH` o `BUILD_NUMBER` y las inyecta como claves-valor en el bloque de contexto. Útil para que la IA sepa en qué rama o entorno se ejecuta el pipeline.

### 3. Validar Consistencia del Contexto Antes de un Commit
```bash
contextlock validate --strict --ignore "node_modules"
```
Este comando recorre todos los archivos del proyecto (excepto los ignorados) y verifica que las referencias a módulos, variables de entorno y dependencias coincidan con el contexto registrado. Si encuentra discrepancias, lanza un error con el diff exacto. Ideal como hook de pre-commit.

## Mejores Prácticas para el Uso en Producción

### 1. Segmenta el Contexto por Ciclo de Vida del Proyecto
No mantengas un único bloque de contexto para todo el repositorio. Crea namespaces separados para `desarrollo`, `staging` y `producción`. Usa el flag `--env` al inyectar contexto para que la IA distinga entre configuraciones de pruebas y parámetros de producción. Esto evita que un cambio en desarrollo afecte la memoria de la IA en producción.

### 2. Automatiza la Limpieza de Contexto Obsoleto
Configura un cron job que ejecute `contextlock prune --older-than 30d` semanalmente. Esto elimina bloques de contexto que no se han actualizado en más de 30 días, evitando que la IA arrastre información de sprints antiguos o dependencias deprecadas. Combínalo con `contextlock archive` para mantener un historial sin saturar el espacio de trabajo.
