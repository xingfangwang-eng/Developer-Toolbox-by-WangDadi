# contextlock - CON_017

## Resumen

ContextLock es la solución definitiva para desarrolladores que necesitan mantener contexto persistente entre sesiones de IA sin perder el hilo ni reiniciar prompts. Olvídate de tener que reexplicar tu arquitectura cada vez que abres una nueva conversación. Con ContextLock, tu asistente IA recuerda exactamente dónde lo dejaste, incluso después de cerrar el terminal o cambiar de rama en Git.

## Contenido Detallado

### ¿Qué es ContextLock?

ContextLock es una herramienta CLI y SDK diseñada para equipos de desarrollo que trabajan con asistentes de IA (ChatGPT, Claude, Copilot, etc.) y necesitan mantener un estado de conversación coherente a lo largo del tiempo. En lugar de depender de ventanas de chat volátiles o de copiar manualmente resúmenes, ContextLock inyecta y recupera automáticamente el contexto relevante desde un almacenamiento local cifrado. Es ideal para sesiones de debugging largas, revisión de PRs complejas o mantenimiento de proyectos con múltiples microservicios.

### Características Principales

- **Persistencia de Contexto con Hashing Diferencial**: ContextLock no guarda todo el historial, sino que calcula un hash diferencial del contexto activo. Solo almacena los cambios significativos entre sesiones, reduciendo el overhead de almacenamiento y permitiendo revertir a estados anteriores con un simple comando `ctxlock rollback --snapshot <hash>`. Cada snapshot está firmado criptográficamente para evitar corrupción o manipulación.

- **Inyección Contextual Automática en Prompts**: Cuando inicias una nueva sesión con tu IA favorita, ContextLock inyecta automáticamente el contexto guardado en el prompt inicial. Soporta formatos nativos de cada plataforma (Markdown para ChatGPT, XML para Claude, JSON para Copilot). Además, puedes definir reglas de filtrado para que solo se inyecten fragmentos relevantes según el tipo de tarea (ej: solo contexto de debugging, no de planificación).

- **Sincronización Multi-entorno con Git Hooks**: ContextLock se integra directamente con tu flujo de trabajo Git. Cada vez que haces commit o cambias de rama, un hook de Git actualiza automáticamente el contexto asociado a ese branch. Así, si estás trabajando en una feature branch, el contexto de la IA se ajusta automáticamente al código y decisiones de esa rama, sin interferir con el contexto de `main` o `develop`.

### Cómo Usar

1. **Inicializar ContextLock en tu proyecto**: Ejecuta `ctxlock init` en la raíz de tu repositorio. Esto crea un archivo `.ctxlock/config.yaml` donde defines el alcance del contexto (archivos, directorios, variables de entorno). Por ejemplo, puedes especificar que solo se capture el contexto de los archivos en `src/` y `tests/`, ignorando `node_modules/` o `vendor/`. Luego, ejecuta `ctxlock capture` para tomar la primera instantánea del contexto actual.

2. **Guardar y restaurar contexto durante una sesión de IA**: Mientras trabajas con tu asistente, si necesitas pausar y retomar más tarde, usa `ctxlock push` para guardar el estado actual. Cuando regreses, ejecuta `ctxlock pull` para recuperar el contexto exacto. Si estás usando la API, puedes integrar el comando directamente en tu script: `ctxlock pull --format json | jq '.context' > prompt_context.txt` y luego inyectarlo en tu llamada a la API de OpenAI o Anthropic.

3. **Automatizar con Git hooks**: Configura un hook post-checkout en `.git/hooks/post-checkout` que ejecute `ctxlock switch --branch $1`. De esta manera, cada vez que cambies de rama, ContextLock carga automáticamente el contexto asociado a esa rama. Si estás en `feature/login`, el asistente sabrá que estás trabajando en autenticación, no en pagos. Para hooks pre-commit, puedes agregar `ctxlock capture --auto` para que antes de cada commit se actualice el contexto con los últimos cambios.

### Mejores Prácticas

- **Segmenta el contexto por dominio y no por sesión**: En lugar de tener un solo contexto gigante, crea múltiples archivos de contexto en `.ctxlock/contexts/` (ej: `backend.md`, `frontend.md`, `devops.md`). Luego, en tu prompt inicial, referencia solo el contexto relevante usando `ctxlock inject --context backend`. Esto evita que la IA se distraiga con información de otro módulo y mejora la precisión de las respuestas.

- **Usa ContextLock como parte de tu pipeline de CI/CD**: En lugar de depender de que cada desarrollador recuerde ejecutar `ctxlock push`, integra el comando en tu pipeline de integración continua. Por ejemplo, después de cada merge a `main`, ejecuta `ctxlock capture --label "release-v2.3"`. Así, si necesitas debuggear un bug en producción, puedes restaurar el contexto exacto de esa release con `ctxlock restore --label "release-v2.3"`, obteniendo el estado de la IA tal como estaba cuando se lanzó esa versión.
