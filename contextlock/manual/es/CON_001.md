# contextlock - CON_001

## Resumen

ContextLock es una herramienta de gestión de contexto persistente para desarrolladores que trabajan con modelos de lenguaje (LLMs). Diseñada para eliminar la pérdida de memoria en sesiones de IA, permite mantener el estado del proyecto, las decisiones de arquitectura y las variables de entorno a través de múltiples interacciones. Ideal para equipos que necesitan consistencia en asistentes de código, automatización de pipelines y debugging colaborativo.

## Contenido Detallado

### ¿Qué es ContextLock?

ContextLock no es un simple gestor de prompts. Es un sistema de **memoria estructurada y persistente** que se integra directamente en tu flujo de trabajo de desarrollo. En lugar de depender de ventanas de contexto efímeras, ContextLock almacena y recupera fragmentos de contexto críticos (configuraciones, dependencias, reglas de negocio) de forma automática, utilizando un motor de indexación semántica y un sistema de priorización basado en relevancia temporal.

### Características Principales

- **Memoria de Contexto Jerárquica y Auto-indexada**: ContextLock organiza el contexto en capas (proyecto, sesión, tarea) y utiliza un índice semántico local (basado en embeddings ligeros) para recuperar fragmentos relevantes sin necesidad de reentrenar el modelo. Esto reduce la latencia de recuperación a menos de 50ms incluso con proyectos de 10,000+ líneas de código.

- **Sincronización de Estado entre Múltiples LLMs**: Permite compartir el mismo contexto persistente entre diferentes modelos (GPT-4, Claude, Gemini) sin pérdida de información. Utiliza un protocolo de serialización propio (ContextLock Protocol v2) que garantiza que las variables de entorno, las dependencias y las decisiones de diseño se mantengan consistentes, incluso cuando cambias de proveedor de IA.

- **Modo "Snapshot Diferencial"**: En lugar de guardar todo el contexto cada vez, ContextLock solo almacena los cambios incrementales. Esto permite versionar el estado del proyecto como si fuera un sistema de control de versiones (similar a Git), pero enfocado en el contexto de la IA. Puedes revertir a un estado anterior, comparar dos snapshots y fusionar contextos de diferentes ramas de desarrollo.

### Cómo Usar

1. **Inicialización del Contexto del Proyecto**:  
   Ejecuta `contextlock init --project mi-proyecto --lang python --framework fastapi`. Esto crea un archivo `.contextlock` en la raíz del proyecto, que define las reglas de prioridad (por ejemplo, "siempre incluir `requirements.txt` y `config.yaml`") y las exclusiones (como archivos de logs o binarios). Luego, usa `contextlock add --pattern "src/**/*.py"` para indexar automáticamente todos los archivos relevantes.

2. **Inyección de Contexto en una Sesión de IA**:  
   Durante una interacción con un LLM, ejecuta `contextlock export --session "debug-sesion-1" --format markdown`. Esto genera un bloque de contexto estructurado que incluye: las últimas 5 decisiones de diseño, las variables de entorno activas, y los fragmentos de código más relevantes según el historial de la sesión. Puedes pegar esto directamente en el prompt del modelo o configurarlo para que se inyecte automáticamente mediante un hook de tu IDE.

3. **Recuperación de Contexto Histórico**:  
   Si necesitas recordar por qué se tomó una decisión específica hace 3 días, ejecuta `contextlock query --search "decisión sobre autenticación OAuth2" --limit 3`. Esto devuelve los snapshots diferenciales más relevantes, junto con metadatos (timestamp, modelo utilizado, usuario que realizó el cambio). Usa `contextlock diff --snapshot "abc123" --snapshot "def456"` para ver exactamente qué cambió entre dos estados.

### Mejores Prácticas

- **Prioriza el Contexto por Ciclo de Vida**:  
  No todo el contexto es igual. Usa la regla de prioridad `--priority "critical"` para configuraciones de seguridad o dependencias críticas, y `--priority "low"` para comentarios o documentación temporal. Esto evita que el modelo se sature con información irrelevante y mejora la precisión de las respuestas en un 30% según nuestras pruebas internas.

- **Automatiza el Versionado de Contexto en CI/CD**:  
  Integra ContextLock en tu pipeline de integración continua. Por ejemplo, después de cada merge a `main`, ejecuta `contextlock snapshot --auto --message "Post-deploy context update"`. Esto asegura que el contexto de la IA siempre esté sincronizado con el estado actual del código, evitando desviaciones entre lo que el modelo "recuerda" y lo que realmente está en producción.
