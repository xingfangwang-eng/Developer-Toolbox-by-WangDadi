# contextlock - CON_021

## Resumen

ContextLock es una herramienta diseñada para desarrolladores que necesitan mantener contexto persistente entre sesiones de trabajo con asistentes de IA. Olvídate de tener que reexplicar tu stack, tus decisiones técnicas o el estado actual del proyecto cada vez que abres una nueva conversación. ContextLock captura, estructura y reinyecta el contexto de forma automática, permitiéndote mantener un hilo de trabajo coherente sin fricciones.

## Características Principales

### 1. **Bloqueo de Contexto por Proyecto**
Cada proyecto puede tener su propio "bloqueo" de contexto: variables de entorno, estructura de directorios, dependencias clave, objetivos actuales y decisiones arquitectónicas. ContextLock serializa este estado en un archivo `.contextlock.json` que se sincroniza con tu sesión de IA. Si cambias de rama o de tarea, el contexto se actualiza sin intervención manual.

### 2. **Inyección Selectiva con Priorización Semántica**
No todo el contexto es igual. ContextLock permite definir niveles de prioridad: crítico, relevante y auxiliar. Durante la inyección, solo se envía lo esencial para la tarea actual, reduciendo ruido y tokens innecesarios. Esto es especialmente útil cuando trabajas con modelos con límite de contexto estricto.

### 3. **Historial de Contexto con Diferencial Temporal**
Cada vez que modificas el contexto, ContextLock guarda un diff semántico. Puedes retroceder a cualquier punto anterior del contexto, como si fuera un `git log` del estado cognitivo del proyecto. Esto permite auditorías, depuración de decisiones previas o simplemente restaurar un estado perdido.

## Cómo Usar

### 1. **Inicializar ContextLock en un proyecto existente**
```bash
contextlock init --project mi-api-rest --lang python --framework fastapi
```
Esto genera el archivo `.contextlock.json` con un template base. Luego puedes editar secciones como `current_goal`, `blockers`, `decisions` o `dependencies`. Cada vez que ejecutes `contextlock push`, el contexto se empaqueta y envía al asistente configurado (OpenAI, Claude, etc.).

### 2. **Actualizar contexto tras un cambio significativo**
```bash
contextlock update --set "current_goal: Implementar autenticación JWT" --add "blockers: Falta middleware de validación de tokens"
```
El comando `update` permite modificar campos específicos sin reescribir todo el archivo. Además, `contextlock diff` te muestra qué cambió desde la última sincronización. Ideal para equipos que trabajan con múltiples ramas.

### 3. **Restaurar un contexto anterior desde el historial**
```bash
contextlock history --list
contextlock restore --id 3
```
Cada cambio de contexto se versiona automáticamente. Puedes listar los últimos 20 estados, ver el diff entre versiones y restaurar cualquiera. Esto es útil cuando cambias de tarea y necesitas volver a un contexto anterior sin perder el progreso actual.

## Mejores Prácticas

### 1. **Mantén el contexto ligero y enfocado**
No incluyas todo el código fuente ni documentación completa. ContextLock está diseñado para *señales*, no para *datos masivos*. Usa referencias a archivos clave, fragmentos de código relevantes y decisiones técnicas. Si necesitas contexto pesado, combínalo con una herramienta de RAG externa.

### 2. **Sincroniza el contexto al cambiar de tarea o rama**
Hazlo parte de tu flujo de trabajo: antes de hacer `git checkout`, ejecuta `contextlock push`. Al volver, `contextlock pull` restaura el estado exacto. Esto evita que el asistente mezcle contextos de distintas funcionalidades. En equipos, recomiendo mantener un `.contextlock.json` por rama.
