# contextlock - CON_018

## Resumen

ContextLock es una herramienta de gestión de contexto persistente para desarrolladores que trabajan con asistentes de IA. Permite mantener el estado, los historiales de conversación y las configuraciones específicas del proyecto sin perder información entre sesiones, eliminando la fricción de tener que reexplicar el contexto cada vez que interactúas con un modelo de lenguaje.

## Contenido Detallado

### ¿Qué es ContextLock?

ContextLock es una solución profesional de memoria persistente para IA que soluciona el problema fundamental de la pérdida de contexto en flujos de trabajo de desarrollo. En lugar de depender de ventanas de conversación efímeras o de tener que cargar manualmente archivos de contexto, ContextLock implementa un sistema de almacenamiento estructurado que retiene el estado del proyecto, las preferencias del desarrollador y las interacciones previas con el asistente de IA. Esto es especialmente crítico en entornos de desarrollo donde las sesiones de trabajo se extienden por horas o días, y donde cada reinicio de conversación implica una pérdida de productividad significativa.

### Características Principales

#### 1. **Memoria de Contexto Jerárquico con Priorización Automática**
ContextLock implementa un sistema de almacenamiento de contexto basado en una estructura de árbol con pesos dinámicos. Cada nodo de contexto (archivo, dependencia, configuración, historial de interacción) se evalúa según su relevancia temporal y frecuencia de uso. El motor de priorización automática descarta automáticamente los nodos de baja relevancia cuando se alcanza el límite de tokens, manteniendo siempre los elementos críticos del proyecto. Esto elimina la necesidad de gestionar manualmente qué información conservar.

#### 2. **Sincronización Bidireccional con Sistemas de Archivos y APIs**
No solo guarda el contexto en memoria, sino que sincroniza automáticamente los cambios con el sistema de archivos local y con servicios externos como GitHub, GitLab o Bitbucket. Cada vez que realizas un commit, actualizas una dependencia o modificas un archivo de configuración, ContextLock actualiza su base de contexto en tiempo real. Además, permite exportar el contexto completo como un archivo JSON estructurado para integrarlo con pipelines de CI/CD o herramientas de documentación automática.

#### 3. **Plugin de Contexto para Editores y Terminales**
ContextLock se integra directamente como plugin en VS Code, JetBrains, Neovim y terminales como iTerm2 o tmux. Esto significa que el contexto se mantiene vivo incluso cuando cambias de herramienta o de ventana. Por ejemplo, puedes iniciar una sesión de depuración en la terminal, y el asistente de IA recordará exactamente qué archivos estabas editando, qué errores encontraste y qué soluciones ya probaste, sin necesidad de reexplicar nada.

### Cómo Usar

#### 1. **Inicialización del Contexto del Proyecto**
Ejecuta el siguiente comando en la raíz de tu proyecto para crear un archivo de configuración de contexto:

```bash
contextlock init --project "nombre-del-proyecto" --lang "python" --framework "fastapi"
```

Esto genera un archivo `.contextlock.json` que define las reglas de persistencia, los directorios a monitorear y los patrones de exclusión. Puedes personalizar la configuración editando este archivo directamente o usando el comando `contextlock config set`.

#### 2. **Carga y Restauración de Sesiones de Trabajo**
Para cargar el contexto de una sesión anterior, usa:

```bash
contextlock load --session "2025-03-15-feature-auth"
```

ContextLock restaurará automáticamente:
- El historial de conversaciones con el asistente de IA.
- Los archivos abiertos y sus posiciones de cursor.
- Las variables de entorno y configuraciones específicas del proyecto.
- Las dependencias y versiones activas en el momento de la sesión.

Para guardar la sesión actual antes de cerrar:

```bash
contextlock save --tag "antes-de-refactor"
```

#### 3. **Integración con Asistentes de IA (OpenAI, Claude, Gemini)**
ContextLock expone un endpoint local que los asistentes de IA pueden consultar para obtener el contexto actual. Configura tu cliente de IA para que llame a `http://localhost:8765/context` antes de cada interacción. Ejemplo de configuración para un script de Python:

```python
import requests
from openai import OpenAI

client = OpenAI()
context = requests.get("http://localhost:8765/context").json()

response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": f"Contexto del proyecto: {context}"},
        {"role": "user", "content": "Continúa con la implementación del endpoint de login"}
    ]
)
```

Esto asegura que el asistente siempre tenga la información más actualizada del proyecto sin necesidad de cargarla manualmente.

### Mejores Prácticas

#### 1. **Segmentación de Contexto por Módulos y Features**
No almacenes todo el proyecto en un solo contexto. Divide el contexto por módulos funcionales (autenticación, base de datos, frontend, etc.) y usa etiquetas para activar o desactivar contextos específicos según la tarea que estés realizando. Esto reduce el ruido y mejora la precisión de las respuestas del asistente de IA. Por ejemplo, si estás trabajando en la API de usuarios, carga solo el contexto del módulo de autenticación y base de datos.

#### 2. **Automatización de la Limpieza de Contexto Obsoleto**
Configura reglas de expiración automática para contextos que ya no son relevantes. Por ejemplo, después de 7 días sin modificaciones, ContextLock puede archivar automáticamente sesiones antiguas y liberar espacio. Usa el comando `contextlock cleanup --days 7` para ejecutar esta limpieza de forma programada mediante cron o task scheduler. Esto evita la acumulación de contextos huérfanos que podrían ralentizar el sistema y generar confusión en las respuestas de la IA.
