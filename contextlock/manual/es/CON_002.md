# contextlock - CON_002

## Resumen Ejecutivo

ContextLock es la primera herramienta de gestión de contexto persistente para desarrolladores que trabajan con asistentes de IA. Diseñada para eliminar la pérdida de memoria en sesiones largas y garantir que cada interacción con la IA retenga el contexto crítico de tu proyecto. Keywords: **AI Context Management, Persistent AI Memory, Developer Productivity Tool, contexto persistente, gestión de memoria de IA**.

---

## ¿Qué es ContextLock?

ContextLock es una solución profesional para desarrolladores que lidian con la fragmentación de contexto en herramientas de IA. En lugar de repetir instrucciones o perder el hilo en sesiones complejas, ContextLock inyecta y bloquea el contexto relevante de manera transparente. Es ideal para equipos que trabajan con asistentes de IA en arquitecturas modulares, monorepos o proyectos con múltiples microservicios.

---

## Características Principales

### 1. Inyección de Contexto por Capas (Layered Context Injection)
ContextLock permite definir capas de contexto (proyecto, módulo, tarea) que se inyectan automáticamente según la sesión actual. No más copiar y pegar archivos de configuración o `README.md` enteros. Cada capa se fusiona de forma inteligente, priorizando la información más reciente o relevante.

### 2. Persistencia de Memoria con Hash de Estado
Cada cambio en el contexto se registra con un hash criptográfico (SHA-256). Esto permite detectar automáticamente cuándo el contexto ha cambiado (por ejemplo, después de un commit o un cambio de dependencias) y actualizar la memoria de la IA sin intervención manual. Ideal para equipos que usan CI/CD.

### 3. Bloqueo de Contexto por Sesión (Session Lock)
ContextLock permite "bloquear" el contexto de una sesión específica, evitando que la IA lo modifique o lo pierda. Esto es crucial para depuración en vivo, revisiones de código o sesiones de arquitectura donde cada detalle cuenta. El bloqueo se puede desbloquear solo con un token de autorización.

## Cómo Usar ContextLock

### 1. Configuración Inicial del Contexto Global
Ejecuta el siguiente comando en la raíz de tu proyecto para inicializar el archivo de contexto global:
```bash
contextlock init --scope global --file .contextlock/global.json
```
Este archivo define variables de entorno, dependencias clave y reglas de negocio. Se recomienda incluirlo en el control de versiones para que todo el equipo tenga la misma base.

### 2. Inyección Dinámica en Sesiones de IA
Para inyectar el contexto en una sesión activa de tu asistente de IA (por ejemplo, ChatGPT o Claude), usa:
```bash
contextlock inject --session current --layers global,module
```
Esto fusiona el contexto global con el contexto del módulo actual (detectado automáticamente por el directorio de trabajo). La salida es un bloque de texto formateado que puedes copiar y pegar directamente en el chat de la IA.

### 3. Bloqueo de Contexto para Depuración
Si estás depurando un bug crítico y necesitas que la IA no pierda el hilo, bloquea la sesión:
```bash
contextlock lock --session current --token YOUR_TOKEN
```
Esto genera un snapshot del contexto actual y lo protege contra modificaciones. Para liberarlo, usa `contextlock unlock --session current --token YOUR_TOKEN`. El token se genera automáticamente durante la inicialización.

## Mejores Prácticas

### 1. Contexto Modular por Microservicio
En proyectos con múltiples microservicios, define un archivo de contexto por servicio (por ejemplo, `contextlock --scope service-auth`). Esto evita que la IA confunda dependencias o variables de entorno entre servicios. Además, reduce el ruido en cada sesión.

### 2. Automatización del Contexto en CI/CD
Integra ContextLock en tu pipeline de CI para que, antes de cada deploy, se actualice el contexto global con los últimos cambios. Por ejemplo, en un script de GitHub Actions:
```yaml
- name: Update AI Context
  run: contextlock update --scope global --commit
```
Esto garantiza que cualquier sesión de IA posterior tenga la información más reciente, evitando errores por desactualización.
