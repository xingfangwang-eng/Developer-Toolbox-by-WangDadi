# contextlock - CON_012

## Resumen

ContextLock es una herramienta de gestión de contexto persistente para desarrolladores que trabajan con modelos de lenguaje de IA. Diseñada para eliminar la pérdida de estado en sesiones de desarrollo, permite mantener memoria contextual entre interacciones con APIs de LLMs, optimizando flujos de trabajo complejos y reduciendo la redundancia en prompts. Ideal para equipos que integran IA en pipelines de CI/CD, automatización de código o asistentes personalizados.

## Contenido Detallado

### ¿Qué es ContextLock?

ContextLock es una solución liviana pero robusta que actúa como una capa de persistencia de contexto para cualquier interacción con modelos de lenguaje. A diferencia de soluciones genéricas, este proyecto se enfoca en desarrolladores que necesitan mantener un estado coherente entre múltiples llamadas a APIs de IA, ya sea para debugging, generación de documentación o asistentes de código. Está construido sobre principios de inmutabilidad y versionado de contexto, permitiendo revertir, clonar o compartir estados de sesión sin fricción.

### Características Principales

1. **Context Snapshots Versionados**  
   Cada interacción con la IA se almacena como un snapshot inmutable con metadatos de timestamp, hash de contenido y etiquetas personalizadas. Puedes restaurar cualquier punto anterior del diálogo sin perder el historial completo, ideal para debugging de prompts complejos.

2. **Context Injection Automatizada via Hooks**  
   Integra ContextLock directamente en tu pipeline de desarrollo mediante hooks de pre/post ejecución. Por ejemplo, al lanzar un test suite, el contexto de la sesión anterior se inyecta automáticamente en el nuevo prompt, evitando tener que reescribir instrucciones repetitivas.

3. **Exportación/Importación de Contexto en JSON con Validación de Esquema**  
   Exporta cualquier sesión de contexto a un archivo JSON estructurado, con validación de esquema mediante JSON Schema. Esto permite compartir configuraciones de contexto entre miembros del equipo, o versionarlas en tu repositorio de Git.

### Cómo Usar

1. **Inicialización y Configuración del Contexto Base**  
   Ejecuta `contextlock init --name "mi-sesion" --model "gpt-4"` para crear un nuevo archivo de contexto. Esto genera un directorio `.contextlock/` con un archivo `context.json` que contiene el contexto vacío. Luego, define variables de entorno como `CONTEXT_LOCK_PATH` y `CONTEXT_LOCK_TTL` para controlar la persistencia y expiración.

2. **Inyección de Contexto en Llamadas a la API**  
   En tu script de Python, importa `contextlock` y usa `ContextLock.load("mi-sesion")` para obtener el contexto actual. Luego, al hacer una llamada a la API de OpenAI, pasa el contexto como parte del array de mensajes:  
   ```python
   from contextlock import ContextLock
   cl = ContextLock("mi-sesion")
   messages = cl.get_messages() + [{"role": "user", "content": "Nuevo prompt"}]
   response = openai.ChatCompletion.create(model="gpt-4", messages=messages)
   cl.update(response)
   ```  
   Esto asegura que cada respuesta se agregue al contexto de forma automática.

3. **Rollback a un Snapshot Anterior**  
   Si una respuesta de la IA corrompe el flujo, ejecuta `contextlock rollback --session "mi-sesion" --snapshot "abc123"`. Esto restaura el contexto al snapshot especificado, manteniendo los snapshots posteriores como historial. Luego, puedes reanudar desde ese punto sin pérdida de datos.

### Mejores Prácticas

1. **Versiona tus Contextos en Git**  
   Incluye el directorio `.contextlock/` en tu repositorio (excepto archivos sensibles). Esto permite que cada miembro del equipo tenga acceso al mismo estado de contexto, facilitando la colaboración en prompts complejos. Usa tags de Git para marcar snapshots importantes (ej. `v1.0-context-estable`).

2. **Define TTLs por Tipo de Sesión**  
   Para sesiones de desarrollo rápido, usa un TTL bajo (ej. 1 hora) para evitar acumulación de contexto obsoleto. Para sesiones de producción o debugging, usa TTLs altos (ej. 7 días) y combínalos con exportaciones periódicas a un bucket de S3. Esto optimiza el rendimiento y evita la saturación de memoria.
