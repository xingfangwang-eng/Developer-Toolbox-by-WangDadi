# contextlock - CON_029

## Resumen Técnico

ContextLock es una herramienta de gestión de contexto persistente para desarrolladores que trabajan con asistentes de IA. Elimina la pérdida de memoria entre sesiones, permitiendo que los modelos mantengan un estado coherente y contextualizado sin necesidad de reinicializar prompts cada vez. Ideal para equipos que integran LLMs en pipelines de desarrollo, automatización de tareas o sistemas multi-agente.

## Características Principales

### 1. **Persistencia de Contexto con Hash de Estado Diferencial**
Cada interacción se almacena con un hash SHA-256 del estado anterior, permitiendo detectar cambios mínimos en el contexto. Esto evita la redundancia en el almacenamiento y asegura que solo se guarde lo nuevo, optimizando el uso de memoria y reduciendo costos de API.

### 2. **Inyección de Contexto Dinámico con Priorización por Pesos**
Los fragmentos de contexto se asignan con un peso de relevancia (0–100). ContextLock prioriza automáticamente los fragmentos más relevantes al reconstruir el contexto para el LLM, evitando saturar el límite de tokens con información obsoleta o de baja prioridad.

### 3. **Sincronización Multi-sesión con Lock de Contexto Distribuido**
Permite que múltiples instancias de un agente compartan el mismo contexto en tiempo real mediante un sistema de locks distribuidos basado en Redis. Ideal para equipos que trabajan con múltiples agentes colaborativos o pipelines de CI/CD que requieren coherencia de estado.

## Cómo Usar

### 1. **Inicializar un Contexto Persistente**
Ejecuta el siguiente comando para crear un nuevo contexto con un identificador único:
```bash
contextlock init --id "mi-proyecto-v1" --max-tokens 4096
```
Esto genera un archivo `.contextlock` en el directorio actual. Puedes especificar el límite de tokens para evitar exceder el contexto del modelo.

### 2. **Guardar y Priorizar Fragmentos de Contexto**
Usa el flag `--weight` para asignar relevancia a cada fragmento:
```bash
contextlock push --id "mi-proyecto-v1" --data "Configuración de base de datos: PostgreSQL 15" --weight 90
contextlock push --id "mi-proyecto-v1" --data "Log de errores: conexión fallida a Redis" --weight 30
```
Los fragmentos con mayor peso se inyectan primero en el prompt del LLM.

### 3. **Reconstruir Contexto para una Nueva Sesión**
Al iniciar una nueva interacción con el LLM, reconstruye el contexto optimizado:
```bash
contextlock build --id "mi-proyecto-v1" --target-tokens 2048
```
Esto genera un JSON con los fragmentos más relevantes hasta alcanzar el límite de tokens especificado. Puedes pasar este JSON directamente al sistema de prompts de tu LLM.

## Mejores Prácticas

### 1. **Estrategia de Pesos Dinámicos**
No asignes pesos estáticos a todos los fragmentos. Implementa un sistema de decaimiento temporal: reduce el peso de fragmentos antiguos cada 24 horas. Esto evita que información obsoleta domine el contexto y mantiene la relevancia actualizada.

### 2. **Uso de Contextos Anidados para Microservicios**
Si trabajas con múltiples microservicios, crea contextos independientes para cada uno y un contexto global que los agrupe. ContextLock permite referencias cruzadas entre contextos usando el flag `--link`. Esto facilita la depuración y el mantenimiento sin saturar el contexto de un solo agente.
