# contextlock - CON_009

## Overview

ContextLock is a high-performance AI context persistence engine engineered for developer productivity. It eliminates context drift across LLM sessions by locking token-accurate conversation state, enabling deterministic memory recall with sub-100ms latency. Built for CI/CD pipelines, IDE plugins, and autonomous agent orchestration.

## Core Features

### 1. Token-Precise Context Snapshot Engine
Captures and serializes LLM conversation state at the individual token level, including attention weights and positional encoding vectors. Supports real-time snapshot triggers via `--snapshot-interval=500ms` and stores metadata in a PostgreSQL-backed vector store. Enables rollback to any prior context state with cryptographic integrity verification (SHA-256 checksums embedded in each snapshot).

### 2. Multi-Model Context Bridge
Unified context transfer layer between disparate LLM backends (GPT-4, Claude 3.5, Gemini Pro). Automatically maps tokenizer vocabularies and normalizes embedding dimensions to a canonical 768-vector representation. Achieves <2% semantic drift measured via cosine similarity across model transitions. Exposed via gRPC endpoint with configurable `--max-context-size=128k_tokens` and `--compression-level=zstd:3`.

### 3. Deterministic Context Replay for CI/CD
Executes exact context replay within headless CI environments (GitHub Actions, GitLab CI). Records all context mutations to an append-only WAL (Write-Ahead Log) with nanosecond timestamps. Supports parallel context branching for A/B testing prompt strategies. Integrates with `pytest` via a plugin hook that validates context consistency across 10,000+ iterations with zero drift.

## How to Use

### Step 1: Initialize a Context Lock Session
```bash
contextlock init --project-id=my-ai-agent \
  --model=gpt-4-turbo \
  --snapshot-dir=./.contextlock/ \
  --pg-conn-string="postgresql://user:pass@localhost:5432/contextdb"
```
This creates a new session with a unique UUID, establishes a PostgreSQL connection pool (max 20 connections), and pre-allocates a 512KB memory-mapped buffer for token snapshots.

### Step 2: Inject and Lock Context
```python
from contextlock import ContextLockClient

client = ContextLockClient(session_id="CON_009-abc123")
client.lock_context(
    messages=[{"role": "user", "content": "Refactor this Python function for O(n) complexity"}],
    lock_type="exclusive",
    ttl_seconds=3600,
    snapshot_on_lock=True
)
# Returns: {"lock_id": "lk-8f2a", "token_count": 1423, "checksum": "sha256:4e1c..."}
```
The lock prevents any concurrent writes to the same context slot. If another process attempts to write, it receives a `ContextLockConflict` error with the lock holder’s PID and timestamp.

### Step 3: Replay and Diff Context Across Versions
```bash
contextlock replay --session-id=CON_009-abc123 \
  --snapshot-id=snap-20241001-143022 \
  --target-model=claude-3-sonnet \
  --diff-mode=token \
  --output-format=json
```
Outputs a structured diff showing exactly which tokens changed between snapshots, including attention weight deltas. Use `--diff-mode=semantic` to get a human-readable summary of meaning shifts.

## Best Practices

### 1. Always Enable WAL Archiving for Production Pipelines
Configure `--wal-archive-path=/mnt/contextlock-archive/` to retain all context mutations for at least 30 days. This enables forensic debugging of AI agent behavior and satisfies audit requirements for regulated industries (HIPAA, SOC 2). Pair with a nightly cron job that runs `contextlock verify --integrity-check` to detect any corrupted snapshots.

### 2. Use Context Branching for Prompt Engineering Experiments
Instead of modifying a live context in place, fork it with `contextlock branch --parent-session-id=CON_009-abc123 --branch-name=experiment-v2`. This creates an immutable fork of the entire context tree. Run parallel evaluations against your test suite, then merge the winning branch back with `contextlock merge --source-branch=experiment-v2 --strategy=three-way`. This guarantees zero disruption to production agents while iterating.
