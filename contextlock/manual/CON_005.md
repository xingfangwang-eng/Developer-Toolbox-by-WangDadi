### What is ContextLock?

ContextLock is a **persistent context orchestration engine** for AI-assisted development. Unlike ephemeral chat windows or stateless API calls, ContextLock binds your AI’s working memory to your project’s actual file tree, git history, and runtime environment. It eliminates the "cold start" problem—your AI assistant remembers exactly where you left off, even after a reboot, branch switch, or team handoff. Think of it as a **session-persistent, version-controlled context layer** for your IDE.

### Core Features

1. **Immutable Context Snapshots with Git-Aware Diffing**  
   ContextLock automatically snapshots your AI’s working context (open files, conversation history, linting state) on every `git commit`. When you switch branches, it restores the exact context from that branch’s last snapshot—no more re-explaining your architecture to the AI after a `git checkout`.

2. **Multi-Model Context Bridging**  
   Seamlessly share a single persistent context across OpenAI, Claude, Gemini, or local LLMs. ContextLock normalizes token windows, prompt templates, and embedding vectors, so switching models mid-session preserves your conversation’s semantic memory without re-prompting.

3. **Zero-Latency Context Injection via File System Watcher**  
   The agent monitors your workspace for file saves, terminal output, and debugger breakpoints. It injects relevant context (e.g., the last 50 lines of a log file, the current function signature) into the AI’s prompt automatically—no manual copy-paste or `/context` commands.

### How to Use

1. **Initialize ContextLock in Your Project**  
   Run `npx contextlock init` inside your project root. This creates a `.contextlock/` directory and a `contextlock.config.yml` file. Configure the `models` array (e.g., `["gpt-4", "claude-3-opus"]`) and set `snapshot_on_commit: true` to enable automatic git-aware snapshots.

2. **Attach Context to a Specific File or Symbol**  
   Open a file in your editor, then run `contextlock attach --scope function --name "handlePayment"`. This binds the AI’s memory to the `handlePayment` function. Any subsequent AI interaction (chat, code generation, refactoring) will automatically include the function’s signature, docstring, and call graph as context.

3. **Restore a Previous Session from a Git Branch**  
   After switching branches, run `contextlock restore --branch feature/payment-v2`. ContextLock will load the most recent snapshot for that branch, re-open all previously tracked files in your editor, and inject the last 20 messages from the AI conversation into the current chat window. Use `--dry-run` to preview the restore without modifying your workspace.

### Best Practices

- **Use Scoped Contexts for Monorepo Work**  
   In a monorepo, run `contextlock scope --path ./packages/payment-service` to isolate context to a single package. This prevents cross-package context pollution and keeps token usage low. Combine with `--max-tokens 8192` to enforce a hard ceiling per scope.

- **Integrate with CI/CD for Regression Testing**  
   Add `contextlock verify --snapshot baseline` to your CI pipeline. This compares the current AI context (e.g., prompt templates, embedding vectors) against a baseline snapshot. If the context drifts beyond a configurable threshold (e.g., cosine similarity < 0.95), the pipeline fails—catching silent context degradation before it affects production.
