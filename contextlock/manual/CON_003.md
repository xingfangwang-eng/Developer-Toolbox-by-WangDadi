### What is ContextLock?

ContextLock is a professional tool that helps solve problems in related fields.

### Core Features

- **Deterministic Context Snapshots**: Freeze and restore AI conversation state at any point, ensuring reproducible outputs across sessions. No more "forgetting" mid-prompt.
- **Semantic Memory Compression**: Automatically compress verbose context into token-efficient embeddings, reducing API costs by up to 40% while preserving semantic fidelity.
- **Cross-Platform Context Bridge**: Seamlessly sync context across VS Code, CLI, and browser-based AI tools via a unified JSON schema. No vendor lock-in.

### How to Use

1. **Initialize a Context Lock**: Run `contextlock init --project my-ai-app` to create a `.contextlock` directory. This generates a base context schema with default compression thresholds and memory retention policies. Edit the `context.config.yaml` to set your preferred embedding model (e.g., `text-embedding-3-small`) and max token budget.
2. **Inject Context into a Session**: Use `contextlock inject --session-id dev-123 --from-file ./contexts/latest.json` to push a pre-compiled context into your AI tool's active session. The tool automatically applies semantic deduplication and re-ranks tokens by relevance score.
3. **Rollback to a Previous State**: Run `contextlock rollback --snapshot-id snap_20250315_1442` to restore a prior context state. This is critical for debugging regressions in AI-driven code generation or multi-turn agent workflows. The rollback is atomic and preserves all metadata tags.

### Best Practices

- **Version Control Your Contexts**: Treat `.contextlock` snapshots like code commits. Store them in a Git LFS repository to enable collaborative debugging and audit trails for AI-assisted development.
- **Tune Compression Per Use Case**: For code generation, set `compression_mode: "logical"` to preserve variable names and function signatures. For conversation analysis, use `compression_mode: "semantic"` to retain intent and sentiment. Profile with `contextlock analyze --profile` before deploying.
