# contextlock - CON_007

## Overview

ContextLock is a developer-first AI context management tool designed to eliminate session fragmentation and preserve persistent memory across LLM interactions. Built for engineers who demand deterministic behavior from AI assistants, it ensures your conversation state, project context, and toolchain configurations remain intact across sessions, APIs, and even model switches.

## Core Features

### 1. **Deterministic Context Snapshots with Version Control**
ContextLock captures a fully serialized snapshot of your AI session — including system prompts, tool definitions, conversation history, and embedded files — and stores it as a versioned, diff-compatible artifact. You can roll back to any prior context state, fork from a snapshot, or replay a session exactly as it occurred. This eliminates the "black box" problem in AI development workflows.

### 2. **Multi-Model Context Bridging**
Switch between GPT-4, Claude, Gemini, or local models without losing context. ContextLock normalizes and maps session state across model-specific token limits, system prompt formats, and tool-calling schemas. It automatically truncates or compresses history based on the target model's context window, ensuring seamless continuity without manual re-prompting.

### 3. **Context-Aware CI/CD Integration**
Inject locked context into your CI/CD pipelines. ContextLock exposes a CLI and REST API that lets you attach a context lock to automated code reviews, test generation, or documentation workflows. Your AI agents in CI always operate with the same project context, eliminating hallucination caused by missing or stale background information.

## How to Use

### 1. **Initialize a Context Lock in Your Project**
Run `contextlock init` in your project root. This creates a `.contextlock/` directory containing a `lock.json` manifest. The manifest automatically scans your `.gitignore`, `package.json`, `requirements.txt`, or `go.mod` to infer your tech stack and toolchain. You can then manually append files or directories with `contextlock attach ./src/models` to embed specific codebases.

### 2. **Snapshot and Tag a Session**
While interacting with any AI model, invoke `contextlock snapshot "feature/auth-flow-v2"` at any point. This captures the full session state — including all messages, tool calls, and outputs — and tags it with a semantic label. Use `contextlock list` to view all snapshots, and `contextlock restore "feature/auth-flow-v2"` to replay or continue from that exact point, even on a different machine.

### 3. **Integrate into Your CI Pipeline**
Add the following to your `.github/workflows/ci.yml`:
```yaml
- name: Lock project context
  run: contextlock attach --ci --model gpt-4-turbo
- name: Generate unit tests with locked context
  run: contextlock exec "Generate unit tests for src/auth/ with 90% coverage"
```
This ensures your AI test generator always has the full project structure, dependency graph, and coding conventions — not just a truncated prompt.

## Best Practices

### 1. **Use Context Lock as a Peer Review Artifact**
When submitting a PR, attach the context lock snapshot used during development. Reviewers can replay your AI-assisted coding sessions, inspect the exact prompts and outputs, and verify that no hallucinated code was introduced. This turns your AI interactions into auditable, reproducible artifacts — a must for compliance-heavy environments.

### 2. **Version Your Context Locks Alongside Code**
Treat `.contextlock/` as a first-class citizen in your repository. Commit context lock snapshots that correspond to major feature branches or releases. This allows any developer to instantly reconstruct the exact AI context used for a specific feature, enabling consistent collaboration across teams and time zones. Always run `contextlock prune` before committing to remove redundant or oversized snapshots.
