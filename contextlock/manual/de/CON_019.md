# contextlock - CON_019

ContextLock ist kein weiteres Kontext-Tool. Es ist eine **deterministische Memory-Engine** für KI-Assistenten, die Kontextverluste eliminiert und Token-Budgets optimiert. Entwickler in Deutschland nutzen es, um komplexe Multi-Session-Workflows stabil zu halten.

## 🔥 3 Hardcore-Features (die andere Tools nicht haben)

### 1. Deterministic Context Pin (DCP)
Erzwingt, dass bestimmte Kontextblöcke (z. B. System-Prompts, API-Schemas, Compliance-Regeln) **unabhängig von der Token-Limitierung** immer aktiv bleiben. Kein "Vergessen" nach 10 Iterationen.

### 2. Token-Budget-Controlled Memory Layer
Automatische Kompression von historischen Dialogen auf **< 5 % des Original-Token-Verbrauchs** ohne semantischen Verlust. Ideal für Langzeit-Sessions mit GPT-4 oder Claude.

### 3. Session-Persistenz-Hook für CI/CD
ContextLock kann als **Middleware** in CI/CD-Pipelines integriert werden. Kontext wird über Build-Jobs hinweg beibehalten – z. B. für automatische Code-Reviews oder Doku-Generierung.

## 🛠️ 3 Profi-Schritte zur Einrichtung

### Schritt 1: Kontext-Pin definieren
```bash
contextlock init --pin "SYSTEM_PROMPT:./prompts/system.md" --pin "SCHEMA:./api/v2/schema.json"
```
Dadurch werden zwei Kontextblöcke als **unveränderlich** markiert. Jeder Loop in der KI ignoriert sie nicht mehr.

### Schritt 2: Memory-Kompression aktivieren
```yaml
# contextlock.yml
memory:
  compression: true
  strategy: "semantic_chunk"
  max_tokens_per_session: 12000
```
Nach 12.000 Tokens wird der älteste Teil automatisch komprimiert, aber nie gelöscht. Der Entwickler kann über `contextlock restore --session-id` jederzeit den Originalverlauf einsehen.

### Schritt 3: Session-Persistenz in CI/CD einbinden
```bash
# .gitlab-ci.yml
before_script:
  - contextlock restore --session-id $CI_JOB_ID
script:
  - make build
after_script:
  - contextlock save --session-id $CI_JOB_ID --output ./context_snapshots/
```
Damit bleibt der Kontext über mehrere Pipeline-Stages hinweg erhalten – kein Neulernen bei jedem Job.

## 🧠 2 Best Practices für deutsche Entwickler

### 1. Compliance-getriebene Kontext-Sicherung
Setze DCP für **DSGVO-relevante Kontextblöcke** (z. B. Kundendaten, Audit-Logs). ContextLock stellt sicher, dass diese Blöcke **niemals** aus dem Kontext fallen – auch nicht bei Token-Limit-Überschreitung.

### 2. Token-Budget-Monitoring in der IDE
Integriere `contextlock stats --watch` in deine VS Code- oder JetBrains-Umgebung. So siehst du live, wie viele Tokens dein aktueller Kontext verbraucht, und kannst vor dem nächsten Prompt optimieren. Spart bares Geld bei API-Kosten.
