# contextlock - CON_029

## Überblick

ContextLock ist ein spezialisiertes Tool für **AI Context Management** und **Persistent AI Memory**, das speziell für Developer entwickelt wurde, die mit großen Sprachmodellen (LLMs) arbeiten. Es eliminiert Kontextverluste und ermöglicht eine nahtlose Wiederverwendung von Sitzungsdaten. Optimiert für die deutsche Entwickler-Community, die Wert auf Effizienz, Datensouveränität und Workflow-Konsistenz legt.

## Detaillierter Inhalt

### Was ist ContextLock?

ContextLock ist kein einfacher Prompt-Manager – es ist eine **persistente Kontext-Engine**, die AI-Sitzungen über mehrere Projekte und Tools hinweg synchronisiert. Statt jedes Mal von Null zu beginnen, speichert ContextLock den vollständigen Kontext (inklusive System-Prompts, User-Inputs und Modell-Responses) in einer versionierten, durchsuchbaren Datenbank. Das Tool ist ideal für Teams, die mit ChatGPT, Claude, Gemini oder lokalen Modellen wie Llama 3 arbeiten.

### Kernfunktionen

1. **Kontext-Versionierung und Rollback**  
   Jede AI-Sitzung wird mit einem eindeutigen Hash und Zeitstempel gespeichert. Du kannst nahtlos zu einem früheren Kontext zurückkehren – ohne Datenverlust. Perfekt für Debugging oder A/B-Testing von Prompts.

2. **Cross-Platform Kontext-Sync**  
   ContextLock synchronisiert Kontexte zwischen CLI, IDE (VS Code, JetBrains) und Web-UI. Wechselst du vom Terminal zur IDE, bleibt der gesamte Dialogverlauf erhalten. Kein manuelles Exportieren oder Kopieren mehr.

3. **Semantische Kontext-Verknüpfung**  
   Das Tool erstellt automatisch semantische Beziehungen zwischen verschiedenen Sitzungen. Suchst du nach einem bestimmten Fehler oder einer Lösung, findest du nicht nur den exakten Kontext, sondern auch ähnliche Fälle aus der Vergangenheit – in Sekunden.

### So verwenden Sie es

1. **Initiale Konfiguration und Projekt-Bindung**  
   Installiere ContextLock via npm oder pip:  
   ```bash
   npm install -g contextlock
   ```
   Initialisiere es in deinem Projektverzeichnis:  
   ```bash
   contextlock init --project "mein-llm-projekt" --storage local
   ```
   Dabei wird eine `.contextlock`-Datei erstellt, die alle Konfigurationen (z. B. API-Keys, Modell-Präferenzen) speichert. Optional kannst du einen Remote-Storage (S3, GCS) angeben.

2. **Kontext-Session starten und speichern**  
   Starte eine neue Session mit einem spezifischen System-Prompt:  
   ```bash
   contextlock start --system "Du bist ein DevOps-Experte, der Terraform-Code optimiert."
   ```
   Jeder Input und jede Response wird automatisch geloggt. Um den Kontext explizit zu speichern:  
   ```bash
   contextlock save --tag "terraform-optimierung-v2"
   ```
   Das Tag dient als semantischer Anker für spätere Suchen.

3. **Kontext-Wiederherstellung und semantische Suche**  
   Rufe einen gespeicherten Kontext ab:  
   ```bash
   contextlock restore --tag "terraform-optimierung-v2"
   ```
   Oder führe eine semantische Suche durch:  
   ```bash
   contextlock search "Fehler bei Terraform-Plan"
   ```
   ContextLock gibt dir die relevantesten Sitzungen zurück – inklusive des vollständigen Dialogs. Du kannst direkt in der gefundenen Session weiterarbeiten.

### Beste Praktiken

1. **Kontext-Branches für parallele Experimente**  
   Nutze die Versionierungsfunktion wie Git-Branches. Starte einen neuen Branch für jede Experiment-Runde (z. B. `feature/prompt-optimierung`). So behältst du den Überblick über mehrere Prompt-Strategien und kannst später die effektivste auswählen.

2. **Automatische Kontext-Bereinigung und Retention**  
   Konfiguriere eine automatische Retention-Policy, um alte Kontexte zu archivieren oder zu löschen. Beispiel: Lösche alle Kontexte älter als 30 Tage, die keine semantischen Tags haben. Das hält die Datenbank schlank und die Suche schnell. Verwende dazu den Befehl:  
   ```bash
   contextlock cleanup --retention 30d --untagged-only
   ```
