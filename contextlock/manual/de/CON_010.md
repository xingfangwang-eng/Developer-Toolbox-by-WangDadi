# contextlock - CON_010

## Überblick

ContextLock ist ein KI-gestütztes Kontextmanagement-Tool für Developer, die persistente AI-Memory-Sessions über mehrere Tools und Terminals hinweg benötigen. Es eliminiert Kontextverluste und beschleunigt Workflows durch strukturierte, wiederherstellbare Kontext-Snapshots.

## Detaillierter Inhalt

### Was ist ContextLock?

ContextLock ist ein CLI-first-Tool zur Verwaltung von AI-Kontexten in Entwicklungsumgebungen. Es speichert, synchronisiert und lädt Kontextdaten zwischen verschiedenen AI-Assistenten, Terminals und CI/CD-Pipelines – ohne Datenverlust oder manuelles Kopieren.

### Kernfunktionen

- **Persistente Context-Snapshots mit automatischer Versionierung**: Jeder Kontext wird als versioniertes JSON-Schema gespeichert. Änderungen werden automatisch getrackt, inklusive Diff-Logs, die bei Bedarf auf ältere Zustände zurückgesetzt werden können.
- **Multi-Protocol Context-Streaming**: ContextLock streamt Kontextdaten über gRPC, WebSocket und HTTP/2 – ideal für verteilte CI/CD-Pipelines oder Remote-Entwicklungsserver.
- **Integrierte Context-Expiry & Garbage Collection**: Kontexte können mit TTL (Time-to-Live) markiert werden. Ein dedizierter GC-Prozess löscht veraltete Snapshots automatisch, was Speicherkosten senkt und Compliance-Anforderungen erfüllt.

### So verwenden Sie es

1. **Initialen Kontextsnapshot erstellen**:  
   Führen Sie `contextlock init --name "dev-session-01" --format yaml` aus. ContextLock erstellt ein `.contextlock/`-Verzeichnis mit einer `context.yaml`, die Metadaten (UUID, Timestamp, Tags) enthält. Fügen Sie dann `contextlock add --key "current_branch" --value "feature/ai-memory"` hinzu, um Schlüssel-Wert-Paare zu definieren.

2. **Kontext zwischen zwei Terminals synchronisieren**:  
   Starten Sie im ersten Terminal `contextlock watch --port 9090 --sync-interval 5s`. Im zweiten Terminal verbinden Sie sich mit `contextlock connect --remote localhost:9090`. Jede Änderung im ersten Terminal (z. B. `contextlock update --key "last_query" --value "SELECT * FROM users"`) wird automatisch auf das zweite Terminal repliziert.

3. **Kontext in einer CI/CD-Pipeline wiederherstellen**:  
   In Ihrer GitHub Actions Workflow-Datei fügen Sie nach dem Checkout-Schritt ein:  
   ```yaml
   - name: Restore ContextLock Snapshot
     run: |
       contextlock restore --snapshot-id "snap_20250315_abc123" --output-dir ./context_backup
   ```  
   ContextLock lädt den Snapshot und stellt alle Schlüssel-Wert-Paare sowie geloggte Diffs in das angegebene Verzeichnis. Anschließend kann der Pipeline-Kontext auf Basis dieser Daten fortgesetzt werden.

### Beste Praktiken

- **Context-Tagging für Multi-Environment-Workflows**: Verwenden Sie Tags wie `env:prod`, `env:staging` oder `team:backend`, um Kontexte nach Umgebungen oder Teams zu organisieren. ContextLock unterstützt Filterung über `contextlock list --tag "env:staging"`, was das Auffinden relevanter Snapshots in großen Projekten beschleunigt.
- **Regelmäßige Context-Archivierung für Audits**: Setzen Sie einen Cron-Job auf, der täglich `contextlock archive --older-than 30d --output-dir /backups/contexts` ausführt. So bleiben Kontexte für Compliance-Prüfungen erhalten, ohne die aktive Datenbank zu belasten.
