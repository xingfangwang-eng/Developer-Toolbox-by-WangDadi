# contextlock - CON_017

## Überblick

ContextLock ist ein KI-gestütztes Tool zur persistenten Kontextverwaltung für Entwickler, die KI-Assistenten in ihren Workflows nutzen. Es eliminiert das lästige Wiederholen von Kontext in jeder neuen Session und sorgt für eine nahtlose, memory-basierte Interaktion mit LLMs. Optimiert für Developer Productivity und AI Context Management.

## Detaillierter Inhalt

### Was ist ContextLock?

ContextLock ist kein weiteres Chat-Plugin. Es ist eine **Context-Engine**, die KI-Kontext über Sitzungen, Projekte und sogar Tools hinweg synchronisiert. Statt jedes Mal denselben Projektkontext, Coding-Richtlinien oder Architektur-Entscheidungen neu zu erklären, speichert ContextLock diese als **strukturierte Context-Snapshots** und stellt sie bei Bedarf automatisch wieder her. Es ist das Gedächtnis, das deinem KI-Assistenten bisher gefehlt hat.

### Kernfunktionen

1. **Context-Snapshot-Versionierung**  
   Jeder Snapshot wird mit einem eindeutigen Hash und Metadaten (Projekt, Branch, Timestamp) versehen. Du kannst Snapshots vergleichen, alte Versionen wiederherstellen oder gezielt bestimmte Kontextblöcke (z. B. nur API-Spezifikationen) in eine neue Session laden.

2. **Automatische Context-Injektion via Git-Hooks**  
   ContextLock integriert sich direkt in deinen Git-Workflow. Ein `pre-commit`-Hook speichert automatisch den aktuellen Projektkontext (Code-Struktur, offene Issues, Architektur-Diagramme) als Snapshot. Beim Checkout eines Branches wird der dazugehörige Kontext automatisch geladen – ohne manuelles Zutun.

3. **Multi-LLM-Provider-Synchronisation**  
   ContextLock ist nicht an einen bestimmten KI-Anbieter gebunden. Es synchronisiert denselben Kontext nahtlos zwischen OpenAI, Claude, Gemini und lokalen Modellen (via Ollama). Du wechselst den Provider, ohne dass der KI-Assistent den Faden verliert.

### So verwenden Sie es

1. **Context-Snapshot erstellen und versionieren**  
   ```bash
   contextlock snapshot create --project "my-app" --branch "feature/auth" --include ".env.example, docs/architecture.md"
   ```  
   Dieser Befehl erstellt einen Snapshot aus den angegebenen Dateien und speichert ihn lokal sowie optional in einem Remote-Repository (z. B. S3 oder GitHub Releases). Der Snapshot wird automatisch mit einem Git-Tag verknüpft.

2. **Context in einer neuen KI-Session laden**  
   ```bash
   contextlock session start --snapshot "my-app:feature/auth:latest" --provider "claude"
   ```  
   Startet eine Claude-Session mit dem gespeicherten Kontext. Der gesamte Projektkontext wird als System-Prompt injiziert – inklusive Code-Struktur, Coding-Guidelines und offenen TODOs. Du kannst direkt weiterarbeiten, ohne etwas neu erklären zu müssen.

3. **Context zwischen Projekten teilen**  
   ```bash
   contextlock snapshot export --snapshot "my-app:feature/auth:v2" --format "json" | contextlock snapshot import --project "my-app-v2" --branch "feature/auth-refactor"
   ```  
   Exportiert einen Snapshot als JSON und importiert ihn in ein anderes Projekt. Ideal für Monorepos oder wenn du Architektur-Entscheidungen von einem Projekt in ein anderes übertragen möchtest.

### Beste Praktiken

1. **Snapshots mit Git-Tags verknüpfen**  
   Verwende `contextlock snapshot tag --snapshot "my-app:main:v3" --git-tag "v1.2.3"`, um Snapshots mit Releases zu verknüpfen. So kannst du später genau nachvollziehen, welcher Kontext zu welcher Version gehörte – essenziell für Debugging und Audits.

2. **Context-Granularität bewusst wählen**  
   Nicht jede Datei gehört in einen Snapshot. Beschränke dich auf **Architektur-Dokumente, Coding-Richtlinien und API-Spezifikationen**. Generierte Dateien (`node_modules`, `dist`) und temporäre Logs haben im Kontext nichts verloren. Nutze `.contextlockignore` analog zu `.gitignore`.
