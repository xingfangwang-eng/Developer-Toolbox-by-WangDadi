## What is ContextLock?

ContextLock is a developer-first tool engineered to solve the fragmentation problem in AI-assisted workflows. It persists and synchronizes AI context across sessions, eliminating the need to re-explain project state, codebase structure, or user intent. Designed for teams using LLMs in CI/CD, IDE plugins, or CLI pipelines, it ensures every AI interaction is grounded in the latest, relevant context.

## Core Features

### 1. **Session-Aware Context Snapshots**
Automatically captures and stores the full state of an AI conversation—including code snippets, file references, and user commands—into a versioned snapshot. Each snapshot is tagged with metadata (timestamp, branch, commit hash), enabling rollback or replay of any context without data loss.

### 2. **Multi-Model Context Bridge**
Enables seamless context transfer between different AI models (e.g., GPT-4, Claude, Gemini) by normalizing their input/output schemas. Developers can switch models mid-task without losing conversation history or project-specific memory.

### 3. **Granular Access Control with RBAC**
Integrates Role-Based Access Control (RBAC) for shared context repositories. Teams can define who can read, write, or delete context snapshots, making it suitable for enterprise environments where compliance and audit trails are critical.

## How to Use

### Step 1: Initialize a Context Lock File
Run `contextlock init --project ./my-app` in your terminal. This creates a `.contextlock` directory in your project root, containing a `schema.yaml` file for defining context rules (e.g., `max_snapshots: 50`, `auto_sync: true`). The tool also generates a `context.lock` file that tracks the current active snapshot.

### Step 2: Inject Context into AI Prompts
Use the `contextlock inject` command to prepend the current context to any AI prompt. For example:  
`contextlock inject --prompt "Refactor this function" --model gpt-4`  
This appends metadata like `[Context: v2.3 | Branch: feature/auth | Last commit: a1b2c3d]` to the prompt, ensuring the AI understands the exact project state.

### Step 3: Rollback or Merge Contexts
When debugging, run `contextlock diff --snapshots v2.2 v2.3` to compare two context versions. If a mistake is detected, execute `contextlock restore --snapshot v2.2` to revert the active context. For team collaboration, use `contextlock merge --source v2.3 --target v2.4` to resolve conflicts automatically using a three-way merge algorithm.

## Best Practices

### 1. **Version Control Integration for Context Snapshots**
Treat context snapshots as first-class artifacts in your CI/CD pipeline. Hook `contextlock snapshot` into your pre-commit or pre-push git hooks. This ensures every code commit is paired with a corresponding context snapshot, making it easy to reproduce AI-assisted decisions later.

### 2. **Context Segmentation by Domain**
Avoid monolithic context files. Instead, segment context by domain (e.g., `frontend.context`, `backend.context`, `infra.context`). Use `contextlock scope --domain frontend` to load only relevant context, reducing token overhead and improving AI response accuracy. This is especially critical for large monorepos.
