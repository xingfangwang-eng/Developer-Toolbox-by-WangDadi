## What is ContextLock?

ContextLock is a **persistent AI context management tool** designed for developers who rely on LLMs (like GPT-4, Claude, or Gemini) for complex, multi-session workflows. Instead of losing context when you close a chat or restart a session, ContextLock **snapshots, serializes, and restores** your entire AI conversation state — including system prompts, tool outputs, and token-level memory — across any environment.

It’s not just a chat logger; it’s a **stateful context engine** that ensures your AI assistant remembers exactly where you left off, even after a system reboot or a model switch.

## How to Use

### Step 1: Initialize a Context Session in Your Terminal
```bash
contextlock init --session "feature-auth-v2" --model gpt-4-turbo --system-prompt ./prompts/architect.md
```
This creates a new context session with a unique session ID, binds it to a specific system prompt, and sets the initial token budget. The session is stored locally (or in your cloud bucket) as a `.ctx` file.

### Step 2: Push Context After Each AI Interaction
After every LLM call (via API or CLI), run:
```bash
contextlock push --session "feature-auth-v2" --last-response "response.json" --include-tool-calls
```
This appends the latest response and any tool call outputs (e.g., file writes, API responses) to the session’s context graph. ContextLock automatically deduplicates and compresses redundant tokens, ensuring your context stays within the model’s window without losing critical information.

### Step 3: Restore Context in a New Environment
If you switch machines, restart your terminal, or change models, restore the full state with:
```bash
contextlock restore --session "feature-auth-v2" --output-format openai
```
This outputs a ready-to-inject `messages` array (or Anthropic-compatible format) that you can directly paste into your API call or chat UI. ContextLock also validates that the restored context fits within the new model’s token limit, truncating intelligently if needed.

## Best Practices

### 1. **Use ContextLock for Long-Running Refactoring Sessions**
When refactoring a large codebase, your AI assistant needs to remember the entire dependency graph, architectural decisions, and previous refactoring steps. Instead of re-pasting the same context every time, use ContextLock to **persist the session across multiple days**. This reduces token waste by up to 40% and ensures the AI never loses sight of the big picture.

### 2. **Integrate ContextLock into Your CI/CD Pipeline for Automated Code Reviews**
Set up a ContextLock session per pull request. When a developer opens a PR, inject the entire PR diff, relevant codebase context, and previous review comments into the AI. The AI can then generate a consistent, context-aware code review summary. If the PR is updated, just push the new diff into the same session — the AI will remember the previous review and only comment on new changes.
