# contextlock - CON_029

## 概要

ContextLock は、AI セッション間でコンテキストを完全に保持・再利用するための開発者向け永続化レイヤーです。ChatGPT、Claude、Cursor など複数の LLM 環境にまたがる会話の断絶を解消し、プロジェクト全体の文脈を一貫して維持します。

## 3つのコア機能

### 1. マルチセッションコンテキストマージ
異なる AI ツール間で発生した会話履歴を、セマンティック重複除去 + タイムスタンプ順マージで統合。単なるログ結合ではなく、同じ話題の断片を自動検出し、一つの連続したコンテキストツリーに再構築します。

### 2. コンテキストスナップショット & ロールバック
任意の時点でのコンテキスト状態をスナップショットとして保存。後続の会話で誤った方向に進んだ場合、過去の正確な文脈状態にワンコマンドでロールバック可能。Git ライクな差分管理でストレージ効率も最適化。

### 3. クロスツールコンテキストブリッジ
Cursor のコードエディタ内で行ったリファクタリングの意図を、そのまま ChatGPT の会話に持ち越し。JSON スキーマベースのポータブルコンテキストフォーマットにより、ツール間のコンテキスト転送がシームレスに実現。

## セットアップ & 基本操作

### ステップ 1: コンテキストストアの初期化
プロジェクトルートで以下のコマンドを実行し、`.contextlock/` ディレクトリを作成。`contextlock init --project-type node` のようにプロジェクトタイプを指定することで、自動的に `.gitignore` に追記し、Node.js プロジェクト向けのデフォルトフィルタリングルールが適用されます。

```bash
contextlock init
```

### ステップ 2: アクティブセッションへのコンテキストアタッチ
開発中の AI セッションに現在のコンテキストをバインド。`contextlock attach --session-id <UUID>` で特定セッションを指定するか、`contextlock attach --auto` で最も最近更新されたセッションに自動アタッチ。アタッチ後は、すべての会話が自動的にコンテキストストアに同期されます。

```bash
contextlock attach --auto
```

### ステップ 3: コンテキストのエクスポート & インポート
別の AI ツールにコンテキストを移行する場合、`contextlock export --format json --output ./context-snapshot.json` でポータブル形式にエクスポート。移行先で `contextlock import --file ./context-snapshot.json` を実行するだけで、会話の履歴、決定事項、未解決タスクが完全復元されます。

```bash
contextlock export --format json --output context-snapshot.json
contextlock import --file context-snapshot.json
```

## ベストプラクティス

### プラクティス 1: コンテキストのライフサイクルを明確に区切る
大規模リファクタリングやアーキテクチャ変更の前には必ず `contextlock snapshot --tag "refactor-before-v2.3"` でスナップショットを作成。機能ブランチごとにコンテキストを分離し、マージ時に `contextlock merge --source branch-A --target main` でコンテキストツリーを統合するのが効果的です。

### プラクティス 2: コンテキストサイズの定期的なクリーンアップ
長期間運用するとコンテキストが肥大化し、LLM のトークン制限に抵触する可能性があります。週次で `contextlock prune --max-tokens 8000` を実行し、重要度の低い会話（挨拶、試行錯誤の軌跡など）を自動トリミング。重要度は発言の編集距離と後続参照頻度でスコアリングされます。
