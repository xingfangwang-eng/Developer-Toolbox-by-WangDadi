# contextlock - CON_021

## 概要

ContextLock は、AI コンテキスト管理の永続化とセッション間一貫性を実現するエンタープライズグレードのツールです。AI エージェントのメモリ保持問題を解決し、開発者の生産性を飛躍的に向上させます。

## 詳細な内容

### ContextLock とは

ContextLock は、AI セッション間でコンテキストをロック・保持・復元するためのプロフェッショナルツールです。PostgreSQL との連携により、コンテキストデータをミリ秒単位で永続化し、セッション断絶後も完全な状態を復元します。

### コア機能

- **マルチセッションコンテキストロック機構**: 各セッションのコンテキストを UUID ベースのロックキーで管理。同時実行時の競合を 5ms 以内で解決し、デッドロックを完全防止。PostgreSQL のアドバイザリロックを利用した分散環境対応。
- **コンテキスト差分圧縮エンジン**: 前回セッションとの差分のみを Zstandard 圧縮で保存。保存容量を平均 87% 削減。復元速度は 200μs 以下を達成。圧縮レベルは `--compression-level 3` で調整可能。
- **コンテキストアンカーポイント自動生成**: セッション内の重要トピックを自動検出し、タイムスタンプ付きアンカーを作成。`--anchor-threshold 0.85` で類似度閾値を設定。アンカー間のジャンプは O(log n) で実行。

### 使い方

1. **コンテキストセッションの初期化とロック取得**:
   ```bash
   contextlock init --session-id "dev-2024-01" --db-url "postgresql://localhost:5432/contextdb" --lock-timeout 30000
   ```
   `--lock-timeout` はミリ秒単位で指定。指定時間内にロックが取得できない場合はエラーコード `CON_021_ERR_LOCK_TIMEOUT` を返却。

2. **コンテキストデータの差分保存と圧縮**:
   ```bash
   contextlock save --context-json "./context.json" --diff-only --compression-level 3 --output "./context.clk"
   ```
   `--diff-only` フラグで前回保存時との差分のみを抽出。`--compression-level` は 1〜9 の整数で指定。出力ファイルは `.clk` 形式で保存され、SHA-256 ハッシュが自動付与。

3. **セッション復元とアンカー指定ジャンプ**:
   ```bash
   contextlock restore --session-id "dev-2024-01" --anchor "2024-01-15T14:30:00Z" --verify-hash --output "./restored_context.json"
   ```
   `--anchor` で特定の時点のコンテキストを指定。`--verify-hash` で保存時のハッシュと復元時の整合性を検証。不一致の場合は `CON_021_ERR_HASH_MISMATCH` を出力。

### ベストプラクティス

- **コンテキスト分割による粒度最適化**: 単一の巨大なコンテキストではなく、機能単位でコンテキストを分割保存する。推奨分割サイズは 512KB 以下。これにより復元時間を 40% 短縮し、競合確率を 0.1% 未満に抑制。
- **定期的なアンカー再生成とガベージコレクション**: 24 時間ごとに `contextlock gc --session-id "dev-2024-01" --retention-days 30` を実行。保持期間超過のアンカーを自動削除し、`--vacuum` フラグで PostgreSQL のテーブルを最適化。インデックス断片化率が 20% を超えた場合に実行推奨。
