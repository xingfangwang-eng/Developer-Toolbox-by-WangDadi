# contextlock - CON_023

## 概要

ContextLock は、AI 開発者向けの**永続的コンテキスト管理エンジン**です。LLM セッション間でコンテキストを保持し、開発ワークフローの断絶を解消します。本ツールは「AI に記憶を持たせる」という課題に対し、ファイルベースの軽量アプローチで応えます。

## 詳細な内容

### ContextLock とは

ContextLock は、AI アシスタントとの対話における**コンテキストの再初期化問題**を解決するためのプロフェッショナルツールです。従来のチャット UI では、セッションを跨ぐたびに前提知識やコードベースの状態を再説明する必要がありました。ContextLock は、プロジェクト固有のコンテキストファイル（`.ctxlock`）を自動生成・管理し、AI が常に最新のプロジェクト状態を把握できるようにします。

### コア機能

1. **コンテキストスナップショット自動生成**
   コードベースの変更を検知し、`.ctxlock` ファイルに差分のみを追跡。AI に渡すコンテキストを最小限かつ高精度に保ちます。Git フックと連携し、コミット時に自動更新が可能。

2. **セッション間コンテキスト継承**
   異なる AI ツール（Cursor、Copilot、ChatGPT など）間でも、同一プロジェクトのコンテキストを共有。`contextlock pull` で最新のコンテキストを取得し、`contextlock push` で更新を反映。マルチツール環境での一貫性を担保します。

3. **コンテキスト重み付けフィルタリング**
   プロジェクトの重要度に応じて、コンテキスト項目に優先順位を付与。`contextlock prioritize --weight 0.8` で特定のモジュールや変数を強調表示。AI が無関係なノイズに埋もれず、核心的な情報に集中できます。

### 使い方

1. **プロジェクト初期化とコンテキスト設定**
   ```bash
   # プロジェクトルートで初期化
   contextlock init --project my-app --lang python

   # コンテキスト設定ファイルを編集
   vi .contextlock/config.yml
   # 例: 監視対象ディレクトリ、除外パターン、重み付けルールを定義
   ```

2. **コンテキストのプッシュとプル**
   ```bash
   # AI セッション開始前に最新コンテキストを取得
   contextlock pull --remote origin --branch main

   # 開発中にコンテキストを更新（コード変更を反映）
   contextlock push --message "feat: add authentication module"
   # 自動的に .ctxlock ファイルが更新され、Git にステージされる
   ```

3. **コンテキストの検証とデバッグ**
   ```bash
   # 現在のコンテキスト内容をプレビュー
   contextlock inspect --format json | jq '.context'

   # コンテキストの整合性チェック（重複・欠損を検出）
   contextlock validate --strict
   # 出力例: "Error: Missing import 'os' in module 'auth.py'"
   ```

### ベストプラクティス

- **コンテキストの粒度は「モジュール単位」に設定する**
  ファイル単位では細かすぎ、プロジェクト全体では粗すぎます。モジュール（例: `src/auth/`）単位でコンテキストを定義し、AI に渡す情報量を最適化しましょう。`contextlock init --granularity module` で自動設定できます。

- **CI/CD パイプラインにコンテキストチェックを組み込む**
  プルリクエスト作成時に `contextlock validate` を実行し、コンテキストの整合性を自動検証。AI が古いコンテキストを参照して誤ったコードを生成するリスクを軽減します。GitHub Actions 用のアクションも提供しています。
