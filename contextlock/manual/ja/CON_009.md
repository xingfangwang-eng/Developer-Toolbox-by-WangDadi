# contextlock - CON_009

## 概要

ContextLock は、AI エージェントのコンテキスト管理をハードウェアレベルで最適化する永続メモリフレームワークです。開発者の生産性を向上させるために、セッションを跨いだコンテキストの保持と復元をミリ秒単位で実行します。

## 詳細な内容

### ContextLock とは

ContextLock は、AI ワークフローの文脈情報を永続化し、再現性のある開発環境を提供するプロフェッショナルツールです。PostgreSQL とインメモリキャッシュを連携させ、コンテキストの整合性と高速アクセスを両立します。

### コア機能

- **セッションレスコンテキストブリッジ**  
  AI モデル間でコンテキストを共有可能にする機能。`context_hash` による一意のセッションID を生成し、Redis と PostgreSQL の二重保存で障害耐性を確保。復元レイテンシは平均 12ms 以下。

- **コンテキスト差分圧縮エンジン**  
  前回のコンテキストとの差分のみを記録する `delta_storage` モジュール。Zstandard 圧縮アルゴリズムを採用し、ストレージ使用量を最大 87% 削減。圧縮率は `compression_level` パラメータ（1～22）で調整可能。

- **マルチレイヤーメモリロック機構**  
  コンテキストの変更をトランザクション単位でロックし、同時実行性を担保。`lock_timeout_ms` を 500ms に設定することで、デッドロックを防止しながら書き込み競合を解決。`isolation_level` は `SERIALIZABLE` をデフォルトに設定。

### 使い方

1. **コンテキストの初期化と永続化**  
   ```bash
   contextlock init --session-id "dev-2025-03-28" --storage postgresql://user:pass@localhost:5432/contextdb --cache redis://localhost:6379
   ```
   上記コマンドでセッションを作成。`--ttl 3600` オプションでキャッシュの有効期限を秒単位で指定可能。

2. **コンテキストの差分更新**  
   ```python
   from contextlock import ContextDelta
   delta = ContextDelta(session_id="dev-2025-03-28")
   delta.push(key="current_task", value="refactoring auth module", compression_level=3)
   delta.commit()
   ```
   `commit()` 実行時に差分が自動圧縮され、PostgreSQL に保存。`batch_size` を 100 に設定するとバルクインサートが有効化。

3. **コンテキストの完全復元**  
   ```bash
   contextlock restore --session-id "dev-2025-03-28" --output json --include-metadata
   ```
   `--include-metadata` フラグで最終更新時刻や圧縮率の統計情報も出力。復元結果は標準出力または `--output-file` でファイルに保存可能。

### ベストプラクティス

- **コンテキストのライフサイクル管理**  
  長期運用時は `contextlock gc --older-than 7d` で 7 日以上経過した差分を自動削除。PostgreSQL の `VACUUM` と組み合わせることで、テーブル膨張を抑制しクエリ性能を維持。

- **マルチエージェント環境での分離戦略**  
  エージェントごとに異なる `namespace` を割り当て、`contextlock isolate --namespace agent-a` で名前空間を分離。PostgreSQL のスキーマ分割と Redis のキープレフィックスを自動設定し、データ混在を防止。
