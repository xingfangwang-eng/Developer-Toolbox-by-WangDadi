# contextlock - CON_010

## 概要

ContextLock は、AI セッション間でコンテキストを自動的に継承・最適化する、次世代の開発者向けコンテキスト管理ツールです。大規模なプロジェクトや複数エージェント連携において、コンテキストの断絶や再初期化による生産性低下を根本から解決します。

## 詳細な内容

### ContextLock とは

ContextLock は、AI アシスタントとの対話におけるコンテキストを「ロック」し、セッションを跨いで一貫性を保つツールです。特に、コードベースの大規模リファクタリングやマルチモジュール開発において、AI が過去の決定や設計意図を忘れることなく、継続的に最適な提案を行うことを可能にします。

### コア機能

#### 1. コンテキストスナップショット & 自動復元
AI との対話終了時に、現在のコンテキスト（変数状態、依存関係、未解決タスク）を JSON スキーマでエクスポート。次回セッション開始時に、スナップショットを自動ロードし、中断箇所からシームレスに再開します。

#### 2. コンテキスト依存関係グラフ
プロジェクト内のファイル間の依存関係を解析し、コンテキストの「スコープ」を視覚化。特定のモジュールにフォーカスしたコンテキストロックが可能で、不要なコンテキストの混入を防ぎます。

#### 3. コンテキストバージョン管理 & ロールバック
コンテキストの変更履歴を Git ライクに管理。誤ったコンテキスト更新や、AI の誤認識によるコンテキスト破損が発生した場合、任意の過去の状態にロールバックできます。

### 使い方

#### ステップ 1: コンテキストスナップショットの作成
```bash
contextlock snapshot --scope ./src --format json
```
プロジェクトの `src` ディレクトリ配下の全ファイルの依存関係と、現在の AI セッションのコンテキストをスナップショットとして保存します。出力は `./contextlock/snapshots/` にタイムスタンプ付きで保存されます。

#### ステップ 2: コンテキストの自動復元と差分適用
```bash
contextlock restore --snapshot latest --diff-only
```
最新のスナップショットからコンテキストを復元し、前回セッションからの変更差分のみを AI に提示します。これにより、毎回フルコンテキストを送信する必要がなくなり、トークン消費を大幅に削減します。

#### ステップ 3: コンテキストバージョンのロールバック
```bash
contextlock rollback --to 2025-03-15 --force
```
指定した日付のコンテキスト状態に強制ロールバックします。`--dry-run` フラグを付けると、実際のロールバック前に影響範囲をプレビューできます。

### ベストプラクティス

#### 1. コンテキストスコープの明示的な指定
大規模プロジェクトでは、`--scope` オプションでコンテキストの対象範囲を限定してください。全ファイルを含めると、AI がノイズに埋もれ、精度が低下します。例えば、`--scope ./src/modules/payment` のように、現在のタスクに関連するモジュールのみに絞り込みます。

#### 2. スナップショットの定期クリーンアップ
コンテキストスナップショットは、デフォルトで無制限に保存されます。`contextlock cleanup --older-than 7d` を CI/CD パイプラインに組み込み、1週間以上前のスナップショットを自動削除することで、ストレージの肥大化を防ぎます。
