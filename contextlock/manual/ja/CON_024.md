# contextlock - CON_024

## プロジェクト概要

ContextLock は、AI 開発ワークフローにおいて「コンテキストの永続化」と「メモリ管理」を自動化する、次世代型のプロダクティビティツールです。  
大規模言語モデル（LLM）との対話において、セッションを跨いでも一貫性のあるコンテキストを保持し、開発者の思考の断絶を防ぎます。

**ターゲットキーワード**: AI Context Management, Persistent AI Memory, Developer Productivity Tool

---

## 3つのハードコア機能

### 1. セマンティックコンテキストロック（Semantic Context Lock）
従来の単純なテキスト保存ではなく、ベクトル埋め込みを利用してコンテキストの意味的構造を保持。  
セッション終了後も、類似したクエリに対して自動的に最適な過去コンテキストを復元します。  
これにより、AI が「前回の会話を覚えていない」問題を根本的に解決。

### 2. メモリヒエラルキーエンジン（Memory Hierarchy Engine）
短期・中期・長期の3層メモリモデルを実装。  
各層で保持するデータの重要度と鮮度を自動評価し、不要な情報は圧縮・破棄、重要な情報は永続化。  
開発者が手動で「何を残すか」を判断する手間をゼロに。

### 3. クロスセッションコンテキストブリッジ（Cross-Session Context Bridge）
異なるプロジェクトやブランチ間でも、関連するコンテキストを自動リンク。  
例えば、API設計の議論をしていたセッションから、実装コードのセッションに移行しても、必要な制約や前提条件を自動継承。

## 詳細な操作手順

### 手順1: セマンティックコンテキストの初期化とロック
```bash
contextlock init --project my-ai-project --mode semantic
```
- このコマンドで、プロジェクトルートに `.contextlock/` ディレクトリが生成され、ベクトルDB（デフォルト: ChromaDB）が初期化されます。
- `--mode semantic` を指定することで、単なるテキストログではなく、埋め込みベクトルによる意味的コンテキスト管理が有効に。
- 初期化後、`.contextlock/config.yaml` でメモリ階層のパラメータ（保持期間、圧縮閾値）を調整可能。

### 手順2: コンテキストの手動ロックと自動復元
```bash
# 現在のセッションコンテキストを明示的にロック
contextlock lock --tag "API設計フェーズ1"

# 後日、別セッションで復元
contextlock restore --tag "API設計フェーズ1"
```
- `lock` サブコマンドで、現在のチャット履歴・コードスニペット・エラーログをスナップショットとして保存。
- `restore` 時に、保存されたコンテキストが現在のLLMプロンプトに自動注入されるため、完全な文脈を再現。
- `--tag` オプションで複数コンテキストを管理可能。

### 手順3: メモリ階層の手動最適化とガベージコレクション
```bash
contextlock optimize --strategy aggressive --dry-run
```
- メモリヒエラルキーエンジンが自動で管理するが、明示的に最適化を実行したい場合に使用。
- `--strategy aggressive` で、長期メモリに昇格すべきデータを強制的に評価し、短期メモリの不要データを削除。
- `--dry-run` で実際の削除前に影響範囲をプレビュー可能。本実行は `--apply` を追加。

## 業界ベストプラクティス

### 1. プロジェクト単位でのコンテキスト分離とタグ運用
大規模プロジェクトでは、機能単位やフェーズ単位でタグを付与し、コンテキストを分割管理することを推奨。  
例えば、`frontend-v2`, `backend-auth`, `bugfix-sprint-12` のようにタグを付けることで、復元時のノイズを最小化。  
ContextLock のセマンティック検索機能を使えば、タグを忘れても類似コンテキストを自動提案してくれる。

### 2. CI/CDパイプラインへのコンテキストスナップショット組み込み
GitHub Actions や GitLab CI のジョブ終了時に `contextlock lock --auto` を実行し、ビルドログやテスト結果を自動保存。  
これにより、後日「なぜこの変更をしたのか」という文脈を失わずにコードレビューやデバッグが可能。  
特に、ホットフィックスやリグレッション対応時に、過去のコンテキストが即座に復元できるため、開発速度が大幅に向上する。
