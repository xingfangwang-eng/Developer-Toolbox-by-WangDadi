# contextlock - CON_005

## 概要

ContextLock は、AI コンテキスト管理と永続メモリ機能を統合した次世代型開発者向けツールです。大規模言語モデル（LLM）との対話において、セッションを跨いだコンテキストの一貫性を担保し、開発生産性を劇的に向上させます。

## 詳細な内容

### ContextLock とは？

ContextLock は、AI との対話における「コンテキスト断絶問題」を解決するために設計されたプロフェッショナルツールです。従来のチャットベース AI では、セッションごとにコンテキストがリセットされ、過去の決定事項やプロジェクト固有の知識が失われる問題がありました。ContextLock は、これを **PostgreSQL をバックエンドとした永続化レイヤー** で解決し、ミリ秒単位のコンテキスト復元を実現します。

### コア機能

1. **コンテキストスナップショット & ロールバックエンジン**
   - AI との対話セッションを任意の時点でスナップショットとして保存。`snapshot_id` を指定することで、過去のコンテキスト状態に **3ms 以内** でロールバック可能。
   - パラメータ: `--snapshot-interval 300`（秒単位の自動スナップショット間隔）、`--max-snapshots 100`（保持する最大スナップショット数）

2. **セマンティックコンテキストマージ**
   - 複数のプロジェクトやブランチで異なるコンテキストを自動マージ。ベクトル埋め込み（`text-embedding-3-small` 互換）を用いて意味的重複を検出し、重複排除と優先順位付けを実行。
   - パラメータ: `--merge-strategy latest|priority|weighted`、`--similarity-threshold 0.85`（コサイン類似度閾値）

3. **コンテキスト依存関係グラフの自動構築**
   - 各コンテキストブロック間の依存関係を DAG（有向非循環グラフ）として自動生成。変更影響範囲を可視化し、部分的なコンテキスト更新を **200ms 未満** で実行。
   - パラメータ: `--graph-export-format dot|json|mermaid`、`--dependency-depth 5`（最大依存深度）

### 使い方

1. **初期コンテキストのロックと永続化**
   ```bash
   contextlock init --project my-ai-app --storage postgresql://user:pass@localhost:5432/contextdb
   ```
   上記コマンドで、プロジェクト固有のコンテキストストレージを初期化。`contextlock.config.yaml` に自動生成される `storage_connection_string` と `project_id` を確認。

2. **セッション跨ぎのコンテキスト復元**
   ```bash
   contextlock restore --session-id SESSION_20250315_001 --target-context "前回のコードレビューの指摘事項"
   ```
   過去セッションの特定コンテキストのみを抽出。`--fuzzy-match` フラグを付与すると、部分一致による復元が可能（Levenshtein距離 < 3）。

3. **コンテキストの差分検出と自動同期**
   ```bash
   contextlock diff --source session:latest --target file:./CONTEXT.md --output unified
   ```
   現在のセッションコンテキストとローカルファイルの差分を `unified diff` 形式で出力。`--auto-sync` を追加すると、差分を自動適用し、コンテキストの一貫性を担保。

### ベストプラクティス

1. **コンテキストスコープの明示的分離**
   - `contextlock scope create --name "フロントエンド設計" --tags ui,react,state-management` のように、機能単位でスコープを分割。スコープごとに独立したコンテキスト履歴を管理し、大規模プロジェクトでのコンテキスト汚染を防止。スコープ数は最大 50 まで推奨。

2. **定期的なコンテキストガベージコレクション**
   - `contextlock gc --age 7d --dry-run` で 7 日以上更新のないコンテキストを確認後、`--execute` で削除。`--compression-level 9` を指定すると、削除前に zstd 圧縮でアーカイブ。ストレージ使用量を月次で 40% 削減可能。
