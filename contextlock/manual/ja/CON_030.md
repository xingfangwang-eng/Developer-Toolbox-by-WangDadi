# contextlock - CON_030

## 概要

ContextLockは、AI開発セッションにおけるコンテキスト断絶問題を根本的に解決する、**コンテキスト永続化＆復元特化型ミドルウェア**です。大規模言語モデル（LLM）との対話において、セッションを跨いでも開発コンテキストを完全に保持し、再初期化コストをゼロにします。

## なぜContextLockなのか？

AIアシスタントとの作業で最もフラストレーションが溜まるのは、「さっきまであれだけ詳細に説明したのに、新しいチャットを開いたら全て忘れている」という状況です。ContextLockは、この問題をアーキテクチャレベルで解決します。

## コア機能（3つのハードコア機能）

### 1. セマンティックコンテキストスナップショット

単なる会話ログの保存ではありません。**ベクトル埋め込みと依存関係グラフ**を用いて、会話の意味構造を保持したままスナップショットを作成。復元時には、直前の会話だけでなく、関連する過去の決定事項や技術的制約も自動的に再構築します。

```
// 例: コンテキストスナップショットの内部構造
{
  "session_id": "ctx_20240315_abc123",
  "semantic_graph": {
    "nodes": ["API設計", "エラーハンドリング", "認証フロー"],
    "edges": [{"from": "API設計", "to": "認証フロー", "type": "依存"}]
  },
  "vector_index": "pgvector://...",
  "timestamp": 1710489600
}
```

### 2. マルチモーダルコンテキストブリッジ

テキストだけでなく、**コードブロック、ターミナル出力、ファイル構造、さらには画像の説明**まで含めたマルチモーダルなコンテキストを保持。特にReactコンポーネントのJSXやPythonの型ヒントなど、言語固有のシンタックスを認識して構造化保存します。

### 3. コンテキストデルタ同期

変更があった部分のみを差分同期。毎回フルコンテキストを送信するのではなく、**前回からの変更点（デルタ）だけを効率的に転送**。これにより、大規模プロジェクトでもAPIコールのトークン消費を最大70%削減します。

## セットアップ＆使い方（3ステップ）

### ステップ1: コンテキストロックの初期化

```bash
# npm / yarn / pnpm でインストール
npm install @contextlock/core

# プロジェクトルートで初期化
npx contextlock init --project my-ai-app --storage local

# 出力例:
# ✓ contextlock.config.yml を作成しました
# ✓ .contextlock/ ディレクトリを初期化しました
# ✓ デフォルトストレージアダプター: file-system
```

### ステップ2: セッションの開始とコンテキスト注入

```typescript
import { ContextLock } from '@contextlock/core';

// AIアシスタントセッションを開始
const lock = new ContextLock({
  project: 'my-ai-app',
  mode: 'persistent', // 永続モード
  autoSnapshot: true, // 自動スナップショット
});

// セッション開始時に、過去のコンテキストを自動復元
await lock.restore(); // 前回のセッションコンテキストを完全復元

// 新しいコンテキストを追加
lock.addContext({
  type: 'code',
  content: 'export const handler = async (req, res) => {...}',
  metadata: { file: 'api/handler.ts', line: 12 },
});
```

### ステップ3: コンテキストのエクスポートと共有

```bash
# 現在のセッションコンテキストをエクスポート
npx contextlock export --format markdown --output ./context_backup.md

# 出力例:
# 📦 コンテキストエクスポート完了
# - 含まれる要素: コードブロック(23), 会話履歴(156行), 決定事項(8)
# - 推定復元コスト: 1,234 トークン
# - 保存先: ./context_backup.md

# チームメイトに共有して、同じコンテキストで再現可能
npx contextlock import ./context_backup.md
```

## ベストプラクティス（2つの業界推奨）

### 1. コンテキスト分割戦略（マイクロコンテキストパターン）

AIセッションが肥大化すると、コンテキストウィンドウの制限に達しやすくなります。**機能単位でコンテキストを分割**し、必要なときだけマージする設計が効果的です。

**推奨パターン:**
- `@contextlock/context-auth` — 認証関連のコンテキスト
- `@contextlock/context-db` — データベース設計のコンテキスト
- `@contextlock/context-ui` — UIコンポーネントのコンテキスト

これにより、各セッションは必要なコンテキストだけをロードし、トークン消費を最適化できます。

### 2. CI/CDパイプラインへの統合

コンテキストスナップショットをGitリポジトリにコミットし、**プルリクエストレビュー時にAIが過去の設計判断を自動参照**できるようにします。

```yaml
# .github/workflows/context-sync.yml
name: Context Sync
on:
  push:
    branches: [main]
jobs:
  snapshot:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npx contextlock snapshot --commit --message "Auto-snapshot: ${{ github.sha }}"
```

これにより、チーム全体で一貫したAIコンテキストを共有し、コードレビューの効率が飛躍的に向上します。
