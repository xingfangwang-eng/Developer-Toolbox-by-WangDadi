# giveawaytracker - GIV_028

## 概要

giveawaytracker は、大規模プロモーションイベントにおけるギブアウェイの参加者管理、抽選処理、配布実績追跡をミリ秒単位で実行する専用プラットフォームです。PostgreSQL との連携により、100 万件規模のエントリーデータをリアルタイムで処理し、重複排除や異常検知を自動実行します。

## 詳細な内容

### giveawaytracker とは

giveawaytracker は、SNS キャンペーンやライブ配信連動型ギブアウェイにおいて、参加者のエントリー情報を統合管理し、抽選ロジックのカスタマイズ、当選者通知、配布ステータスのトラッキングまでを一貫して行うプロフェッショナルツールです。特に、複数プラットフォーム（Twitter、Discord、Web フォーム）からのエントリーを自動統合し、重複エントリーの排除と不正検知をリアルタイムで実行します。

### コア機能

- **マルチプラットフォームエントリー統合エンジン**: Twitter API v2、Discord Bot、カスタム Webhook から非同期でエントリーデータを受信し、PostgreSQL の `ON CONFLICT` 句を用いた upsert 処理で重複排除を実行。エントリー受信から統合完了までのレイテンシは平均 120ms 以下。
- **重み付き抽選アルゴリズムエンジン**: 参加者の属性（フォロワー数、過去参加回数、アカウント作成日）に基づく重み係数を動的に計算し、`CASE WHEN` 文による条件分岐ロジックで抽選確率を制御。抽選実行は 10 万エントリーあたり 2.3 秒で完了。
- **配布ステータスリアルタイムトラッキング**: 当選者への連絡（メール、Discord DM）からコード発行、利用状況までを `status` カラム（pending/sent/redeemed/expired）で管理。PostgreSQL の `LISTEN/NOTIFY` 機能を用いて、ステータス変更を 50ms 以内にフロントエンドへプッシュ通知。

### 使い方

1. **エントリー受信チャネルの設定**  
   `config/entry_channels.yaml` に、Twitter の Bearer Token、Discord Bot Token、Webhook の署名キーを記述。`channel_type: "twitter"` の場合は `tweet_keywords` パラメータで監視するキーワードを指定。`rate_limit: 100/15min` で API 呼び出し制限を設定。

2. **抽選条件の定義**  
   `config/draw_rules.json` に `weight_formula: "CASE WHEN followers_count > 1000 THEN 1.5 ELSE 1.0 END"` のような SQL 式を直接記述。`draw_count: 50` で当選者数を指定し、`exclude_previous_winners: true` で過去当選者を除外。

3. **配布コードの一括発行と通知**  
   `scripts/generate_codes.py` を実行し、`--alphanumeric 12` で 12 桁の英数字コードを生成。生成後、`notification/templates/discord_dm.md` に基づいて Discord DM を送信。送信失敗時は `retry_count: 3`、`retry_interval: 300s` で自動リトライ。

### ベストプラクティス

- **エントリー受信時のバリデーション強化**  
  `entry_validator.py` に `is_bot_account` 判定ロジックを実装し、アカウント作成日が 30 日未満のユーザーを自動除外。さらに、同一 IP からの短時間連続エントリーを `rate_limit: 5/min` で制限することで、bot による不正参加を 99.2% 削減可能。

- **抽選結果の監査ログ出力**  
  抽選実行時に `draw_logs` テーブルへ全エントリーの `entry_id`、`weight`、`random_seed`、`result` を記録。`seed: "2024-01-15T12:00:00Z+user_id"` のように日時とユーザー ID を組み合わせて決定論的乱数を生成し、後日監査可能な状態を担保。
