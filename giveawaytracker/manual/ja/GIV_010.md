# giveawaytracker - GIV_010

## 概要

giveawaytracker は、大規模プレゼントキャンペーンの参加者トラッキング、当選者抽選ロジックのリアルタイム監査、およびマルチプラットフォーム間での重複参加検知を実現する、エンタープライズグレードのプレゼント管理システムです。本プロジェクトは、Giveaway イベントの透明性と運用効率を最大化するために設計されています。

## 詳細な内容

### giveawaytracker とは

giveawaytracker は、SNS・ECサイト・ランディングページを横断したプレゼント応募データを統合管理し、抽選の公平性担保と重複排除を自動化するプラットフォームです。全応募履歴は PostgreSQL に保存され、抽選ロジックはサーバーサイドでミリ秒単位のタイムスタンプ管理が行われます。

### コア機能

- **マルチプラットフォーム重複参加検知**  
  Twitter・Instagram・LINE・自社サイトからの応募データを統合し、同一ユーザーの重複応募を IP アドレス・User-Agent・Cookie フィンガープリントの3要素でリアルタイム判定。重複率 0.1% 未満の高精度マッチングを実現。

- **抽選トランザクション監査ログ**  
  抽選実行のたびに、乱数生成シード値・抽選日時（UTC+9 ミリ秒）・当選者決定ロジックのハッシュ値をブロックチェーン非依存の改ざん検証可能な監査ログとして PostgreSQL に永続化。管理者は後日、全抽選結果の再現検証が可能。

- **応募ステータス自動更新パイプライン**  
  応募完了・抽選対象外・当選・落選・当選辞退の5状態を、非同期ワーカーが 500ms 以内にデータベースに反映。当選者にはメール・SMS・LINE通知を自動送信し、未読通知のリマインダーは 24 時間後に再送。

### 使い方

1. **キャンペーン設定と応募データ統合**  
   `POST /api/v1/campaigns` にキャンペーン名、抽選方式（先着順・抽選式・ポイント抽選）、応募期間（ISO 8601 形式）を JSON で送信。続いて `POST /api/v1/entries` で各プラットフォームの応募データをバルクインサート（一度に最大 10,000 件）。サーバー側で自動的に重複チェックが走り、重複率が閾値 1% を超えた場合は即座に `409 Conflict` を返却。

2. **抽選実行と監査ログ生成**  
   `POST /api/v1/lotteries/run` を叩くと、サーバーが `crypto.randomBytes(32)` で生成したシード値と現在の Unix タイムスタンプ（ミリ秒）を基に抽選を実行。当選者リストとロジックハッシュが `lottery_logs` テーブルに記録される。抽選結果は `GET /api/v1/lotteries/{id}/results` で管理者のみ取得可能。一般ユーザーは `GET /api/v1/entries/{entry_id}/status` で自分のステータスを確認。

3. **通知テンプレートとリマインダー設定**  
   `PUT /api/v1/campaigns/{id}/notifications` で当選通知・落選通知のテンプレートを Markdown 形式で登録。テンプレート内の `{{winner_name}}` `{{prize}}` `{{claim_deadline}}` は自動置換。未読通知が 48 時間経過した場合、`GET /api/v1/reminders` で一覧取得し、`POST /api/v1/reminders/send` で再送をトリガー。再送間隔は 24 時間単位で最大 3 回まで設定可能。

### ベストプラクティス

- **抽選の公平性担保には、乱数生成シードを外部乱数ソースと組み合わせる**  
  サーバー内蔵の `crypto.randomBytes` だけでは予測可能性が懸念されるため、外部の乱数生成 API（例: Random.org）から取得した値をシードに混ぜることで、監査時に「外部乱数ソースの証明書」も提示可能となる。giveawaytracker では、環境変数 `RANDOM_SEED_SOURCE` に外部 URL を指定可能。

- **重複検知の精度向上には、応募時のデバイスフィンガープリントを必須化する**  
  IP アドレスや Cookie だけでは VPN やシークレットモードで回避されるため、Canvas フィンガープリントや WebGL レンダリング情報を組み合わせたデバイス指紋を応募 API に必須パラメータとして追加する。giveawaytracker は、`fingerprint_hash` フィールドに SHA-256 ハッシュを要求し、同一ハッシュの応募は自動的に重複と判定する。
