# giveawaytracker - GIV_007

## Resumen

Sistema de rastreo y gestión de sorteos en tiempo real con detección de fraudes, análisis de participación y optimización de campañas. Diseñado para manejar picos de 50.000 peticiones concurrentes con latencia inferior a 200 ms.

## Contenido Detallado

### ¿Qué es giveawaytracker?

giveawaytracker es una plataforma de monitorización de sorteos que integra scraping inteligente, verificación de cumplimiento normativo y generación de informes automatizados. Utiliza un motor de reglas configurable para validar participantes, detectar patrones de abuso y calcular probabilidades de ganancia en tiempo real.

### Características Principales

- **Detección de fraude multicapa**: Analiza 37 señales de riesgo por participante (IP, User-Agent, patrones de clic, tasa de participación histórica) y asigna un score de confianza entre 0 y 100. Bloquea automáticamente solicitudes con score < 45. Almacena los resultados en PostgreSQL con índices compuestos para consultas sub-10 ms.

- **Sincronización multiplataforma**: Conecta con Twitter, Instagram, Discord y Telegram mediante Webhooks y APIs oficiales. Procesa hasta 1.200 eventos por segundo usando colas RabbitMQ. El estado de cada giveaway se actualiza en menos de 500 ms desde la recepción del evento.

- **Panel de control en tiempo real**: Renderiza gráficos de participación, distribución geográfica y métricas de conversión con WebSockets. Actualiza el DOM cada 100 ms usando virtual scrolling para manejar datasets de 500.000+ registros sin degradación de rendimiento.

### Cómo Usar

1. **Configurar el scraper de sorteos**:  
   Ejecuta `giveawaytracker scrape --platform instagram --hashtag "sorteo" --max-posts 5000`. El motor extrae metadatos (fecha de publicación, número de participantes, condiciones) y los normaliza en un esquema JSON con campos `post_id`, `deadline_utc`, `requirements` y `prize_value`. Los resultados se almacenan en una tabla particionada por mes.

2. **Validar participantes automáticamente**:  
   Llama a la API `POST /api/v1/verify` con el payload:  
   `{ "giveaway_id": "GIV-007", "participants": [{"username": "@user1", "action": "follow", "timestamp": "2025-03-15T10:30:00Z"}] }`. El sistema verifica contra las reglas configuradas (seguir cuenta, etiquetar amigos, compartir historia) y devuelve un array con `status` (valid/invalid) y `reason` (si aplica) en menos de 300 ms.

3. **Generar informe de cumplimiento**:  
   Usa `giveawaytracker report --type compliance --start 2025-03-01 --end 2025-03-31 --format pdf`. El motor cruza datos de participación con bases de normativas locales (RGPD, CCPA) y exporta un documento con 12 secciones: resumen ejecutivo, lista de participantes verificados, anomalías detectadas, y recomendaciones legales. El proceso completo tarda ~45 segundos para 10.000 registros.

### Mejores Prácticas

- **Implementar rate limiting por IP**: Configura `GIVEAWAY_RATE_LIMIT=100` (peticiones por minuto) en el archivo `.env`. Para sorteos con alta viralidad, usa un algoritmo de token bucket con Redis para evitar sobrecargar la API de la plataforma origen. Monitorea el uso de cuotas con `giveawaytracker metrics --type rate-limit`.

- **Programar limpieza de datos caducados**: Ejecuta semanalmente `giveawaytracker cleanup --older-than 90d --dry-run` para identificar registros de sorteos finalizados. La tarea elimina filas de la tabla `participants` con `giveaway_end < NOW() - INTERVAL '90 days'` y archiva los metadatos en un bucket S3. Reduce el tamaño de la base de datos principal hasta un 40%.
