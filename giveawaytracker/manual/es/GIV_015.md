# giveawaytracker - GIV_015

## Resumen

Sistema de rastreo y gestión de sorteos en tiempo real, diseñado para campañas de alto volumen con validación antifraude y análisis predictivo de engagement.

## Contenido Detallado

### ¿Qué es giveawaytracker?

Plataforma de orquestación de sorteos que integra detección de patrones anómalos, asignación dinámica de pesos probabilísticos y sincronización multi-red social con latencia inferior a 200 ms.

## Características Principales

### 1. Motor de Validación Antifraude con Heurística Temporal

- **Parámetros:** `threshold_repeticion_ip=3`, `ventana_temporal=300s`, `coeficiente_sospecha=0.85`.
- Analiza el hash de usuario, fingerprint del navegador y timestamp de participación.
- Si `score_fraude >= 0.85`, se bloquea automáticamente la entrada y se registra en tabla `fraud_log` con índice compuesto (`user_hash`, `giveaway_id`).

### 2. Asignación Probabilística por Pesos de Engagement

- **Parámetros:** `peso_seguidor=1.5`, `peso_compartido=2.0`, `peso_etiqueta=1.2`, `factor_decaimiento=0.97`.
- Cada participación genera un ticket virtual con peso `w = base * (factor_decaimiento)^(n-1)`, donde `n` es el número de sorteos previos del usuario.
- El ganador se selecciona mediante algoritmo de ruleta ponderada con complejidad O(log N) usando un árbol de segmentos.

### 3. Pipeline de Sincronización Multi-Red con Cola de Mensajería

- **Parámetros:** `cola_redis=giveaway:sync`, `batch_size=500`, `timeout_ack=10s`.
- Escucha eventos de Twitter, Instagram y Telegram mediante webhooks.
- Cada evento se encola en Redis Streams con un TTL de 24 horas.
- Un worker en Go procesa los batches cada 2 segundos, actualizando la tabla `participations` en PostgreSQL con índice `(red_social, giveaway_id, timestamp)`.

## Cómo Usar

### Paso 1: Configurar el entorno de desarrollo

Clona el repositorio y levanta los servicios con Docker Compose:

```bash
git clone https://github.com/xingfangwang-eng/giveawaytracker
cd giveawaytracker
docker-compose up -d redis postgres
```

Verifica que Redis esté en puerto 6379 y PostgreSQL en 5432. Ejecuta las migraciones:

```bash
make migrate-up
```

Esto crea las tablas `giveaways`, `participations`, `fraud_log` y los índices correspondientes.

### Paso 2: Definir un sorteo con reglas personalizadas

Crea un archivo JSON `sorteo_ejemplo.json`:

```json
{
  "giveaway_id": "GIV-2025-001",
  "redes": ["twitter", "instagram"],
  "reglas": {
    "min_seguidores": 100,
    "exigir_compartido": true,
    "ventana_participacion": 86400,
    "max_entradas_por_usuario": 5
  },
  "ponderacion": {
    "peso_seguidor": 1.5,
    "peso_compartido": 2.0,
    "peso_etiqueta": 1.2
  }
}
```

Ejecuta el comando de registro:

```bash
./giveawaytracker create --file sorteo_ejemplo.json
```

El sistema valida la estructura y responde con el ID del sorteo y el hash de configuración.

### Paso 3: Ejecutar el sorteo y obtener resultados

Una vez finalizado el periodo de participación, lanza el algoritmo de selección:

```bash
./giveawaytracker draw --giveaway GIV-2025-001 --algoritmo weighted_roulette --seed 42
```

El sistema imprime en consola el ganador principal y dos suplentes, junto con el score de cada uno. También genera un archivo `resultado_GIV-2025-001.json` con el detalle de todas las participaciones y sus pesos.

## Mejores Prácticas

### Práctica 1: Segmentación por cohortes temporales

Agrupa las participaciones en ventanas de 15 minutos y aplica un factor de decaimiento progresivo. Esto evita que usuarios con muchas entradas al inicio dominen la probabilidad final. Configura `factor_decaimiento=0.97` para campañas largas (>7 días) y `0.99` para campañas cortas (<24 horas).

### Práctica 2: Auditoría en caliente con logs estructurados

Activa el modo `--audit` durante la ejecución del sorteo. Cada paso del algoritmo (cálculo de pesos, inserción en árbol de segmentos, selección final) se registra en formato JSON en `stdout`. Redirige la salida a un archivo para auditoría posterior:

```bash
./giveawaytracker draw --giveaway GIV-2025-001 --audit 2>&1 | tee auditoria.log
```
