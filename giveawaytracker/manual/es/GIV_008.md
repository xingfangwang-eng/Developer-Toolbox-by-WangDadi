# giveawaytracker - GIV_008

## Resumen Técnico

Sistema de rastreo y análisis de sorteos en tiempo real, optimizado para campañas multicanal con precisión de milisegundos. Integra detección de fraudes mediante machine learning y sincronización nativa con PostgreSQL para auditoría transaccional.

## Funcionalidades Principales

### 1. Motor de Rastreo Multiplataforma con Captura de Eventos en Tiempo Real
- **Parámetros clave**: `platform_id`, `event_timestamp (μs)`, `user_session_hash`
- **Tecnología**: WebSocket + colas Redis para procesamiento de 10.000 eventos/segundo con latencia < 50ms
- **Validación**: Checksum SHA-256 en cada payload para evitar manipulaciones

### 2. Sistema de Puntuación Dinámica con Umbrales Adaptativos
- **Algoritmo**: Ponderación basada en `engagement_score` (0.0-1.0) y `viral_coefficient` (>1.2)
- **Almacenamiento**: Tablas particionadas en PostgreSQL con índices GIN para búsquedas de texto completo en logs de participación
- **Precisión**: Cálculo de `giveaway_rank` con 8 decimales mediante funciones PL/pgSQL

### 3. Dashboard de Auditoría con Reconstrucción de Sesiones
- **Parámetros**: `session_id (UUIDv7)`, `ip_geolocation (ISO 3166-2)`, `device_fingerprint`
- **Rendimiento**: Carga de 50.000 registros en <2s mediante lazy loading y virtualización de tablas
- **Seguridad**: Encriptación AES-256 en reposo para datos de participantes

## Cómo Usar

### 1. Inicializar el Rastreador de Campañas
```bash
giveawaytracker init --campaign "black_friday_2024" \
  --platforms "twitter,instagram,telegram" \
  --rules-file "./rules/engagement_v2.json" \
  --db-uri "postgresql://user:pass@localhost:5432/giveaways"
```
El sistema generará automáticamente tablas `campaign_events` con particionamiento por día.

### 2. Configurar Reglas de Puntuación Personalizadas
```json
{
  "weight_rules": [
    {"action": "share", "base_score": 2.5, "decay_factor": 0.95},
    {"action": "referral", "base_score": 5.0, "bonus_condition": "referral_count > 3"}
  ],
  "fraud_detection": {
    "ip_threshold": 5,
    "device_time_window_ms": 100
  }
}
```
Ejecutar `giveawaytracker apply-rules --file custom_rules.json` para actualizar en caliente.

### 3. Generar Reporte de Auditoría Forense
```bash
giveawaytracker audit --campaign "black_friday_2024" \
  --time-range "2024-11-29T00:00:00Z/2024-12-02T23:59:59Z" \
  --output-format "parquet" \
  --include-fingerprints
```
Obtendrás un archivo con estructura columnar comprimido (ZSTD) listo para análisis en Apache Spark.

## Mejores Prácticas

### 1. Estrategia de Particionamiento Temporal para Bases de Datos
- Crear tablas `events_YYYY_MM` con índices BRIN en `event_timestamp` para consultas de rango
- Programar `VACUUM` automático cada 6 horas en ventanas de baja actividad (02:00-04:00 UTC)
- Usar `pg_partman` para gestión automática de particiones con retención de 90 días

### 2. Mitigación de Abusos mediante Rate Limiting Adaptativo
- Implementar bucket de tokens por `user_fingerprint` con capacidad de 10 solicitudes/segundo
- Activar bloqueo temporal (30 min) tras 3 intentos de manipulación de `referral_code`
- Registrar todos los eventos de rate limiting en tabla `security_audit_log` con índices en `blocked_at`
