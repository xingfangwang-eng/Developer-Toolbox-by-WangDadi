# giveawaytracker - GIV_003

## Resumen

Sistema de rastreo y gestión de sorteos en tiempo real, diseñado para escalar hasta 1 millón de eventos concurrentes con latencia inferior a 200 ms. Integra detección de fraudes, análisis de participación y notificaciones push automatizadas.

## Contenido Detallado

### ¿Qué es giveawaytracker?

Plataforma modular que centraliza el ciclo de vida completo de sorteos: desde la creación de campañas, seguimiento de participantes, hasta la validación de ganadores. Utiliza un motor de reglas configurable con soporte para 15+ tipos de métricas personalizadas (ej: `engagement_score`, `retention_rate`). Cada sorteo se identifica mediante un UUID v4 único y se almacena en una base de datos PostgreSQL con particionamiento por fecha.

### Características Principales

- **Detección de fraude en tiempo real**  
  Analiza patrones de participación usando un modelo de machine learning (Random Forest con 200 árboles) que evalúa variables como `ip_repetition`, `device_fingerprint` y `time_between_entries`. Si el score de riesgo supera 0.85, el participante es bloqueado automáticamente y se registra en una tabla `fraud_logs` con timestamp en milisegundos.

- **Dashboard dinámico con WebSockets**  
  Transmite datos de participación y métricas clave (`total_entries`, `unique_users`, `conversion_rate`) cada 500 ms mediante WebSockets. El backend usa Redis Streams para encolar eventos y un worker en Node.js (cluster de 4 núcleos) que actualiza el frontend en React con gráficos D3.js.

- **API RESTful con rate limiting y caching**  
  Endpoints protegidos con tokens JWT y límite de 1000 requests/minuto por IP. Las respuestas de consultas frecuentes (ej: `GET /giveaways/{id}/stats`) se cachean en Redis con TTL de 60 segundos. Soporta filtros por `start_date`, `end_date` y `status` (active, closed, cancelled).

### Cómo Usar

1. **Crear una campaña de sorteo**  
   Ejecuta `POST /api/v1/giveaways` con el siguiente payload JSON:  
   ```json
   {
     "title": "Sorteo Navidad 2024",
     "max_entries": 5000,
     "start_date": "2024-12-01T00:00:00Z",
     "end_date": "2024-12-25T23:59:59Z",
     "rules": {
       "min_account_age_days": 30,
       "max_entries_per_user": 3
     }
   }
   ```  
   La API devuelve un `giveaway_id` y un `secret_key` para firmar las entradas.

2. **Registrar la participación de un usuario**  
   Envía una solicitud firmada con HMAC-SHA256:  
   ```bash
   curl -X POST https://api.giveawaytracker.com/v1/entries \
     -H "Authorization: Bearer <JWT_TOKEN>" \
     -H "X-Signature: <HMAC_SIGNATURE>" \
     -d '{"giveaway_id": "abc123", "user_id": "user_456", "timestamp": 1704067200000}'
   ```  
   El servidor valida la firma, verifica el rate limit (máx. 10 entradas/minuto por usuario) y si pasa, almacena la entrada en la tabla `entries` con índice compuesto por `giveaway_id` y `user_id`.

3. **Seleccionar ganador y enviar notificación**  
   Ejecuta `POST /api/v1/giveaways/{id}/draw` con parámetros opcionales como `number_of_winners=1` y `method=weighted_random`. El sistema calcula el ganador usando un algoritmo de selección ponderada (basado en `entry_weight`) y emite un evento a través de RabbitMQ. El worker de notificaciones envía un email con plantilla HTML y un SMS vía Twilio.

### Mejores Prácticas

- **Configurar umbrales de fraude dinámicos**  
  Ajusta el score de riesgo según la estacionalidad. Por ejemplo, durante Black Friday, incrementa el umbral a 0.90 para evitar falsos positivos. Usa la variable de entorno `FRAUD_THRESHOLD` y monitorea la tasa de bloqueos en el log `fraud_metrics.log`.

- **Implementar backup de Redis con persistencia RDB**  
  Configura Redis para hacer snapshots cada 5 minutos en un volumen montado en EBS. Si el servidor se cae, al reiniciar carga el último snapshot y replica los eventos perdidos desde PostgreSQL usando un script de reconciliación que compara `entry_id` entre ambos sistemas.
