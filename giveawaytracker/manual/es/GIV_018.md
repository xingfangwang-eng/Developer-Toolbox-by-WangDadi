# giveawaytracker - GIV_018

## Resumen

Sistema de rastreo y gestión de sorteos en tiempo real, diseñado para plataformas de e-commerce y redes sociales. Permite monitorizar la participación, validar reglas de elegibilidad y calcular la probabilidad de ganar con precisión de microsegundos. Integra PostgreSQL para persistencia de datos y Redis para caché de alta concurrencia.

## Contenido Detallado

### ¿Qué es giveawaytracker?

giveawaytracker es una plataforma de orquestación de sorteos que resuelve la problemática de fraude por múltiples cuentas, desincronización de fechas límite y distribución injusta de premios. Utiliza un motor de reglas dinámico con soporte para restricciones geográficas (GeoIP), límites de participación por IP/huella digital del dispositivo y cálculo de cuotas de ganancia basado en el histórico de actividad del usuario.

### Características Principales

- **Validación Antifraude en Tiempo Real**: Analiza más de 20 señales de comportamiento (velocidad de clics, tiempo entre acciones, patrones de navegación) para detectar bots o usuarios duplicados. Parámetros clave: `max_attempts_per_ip=5`, `session_timeout_ms=30000`, `device_fingerprint_ttl=86400s`. Rechaza automáticamente participaciones sospechosas con una latencia inferior a 50 ms.

- **Motor de Probabilidad Dinámica**: Asigna pesos de victoria basados en métricas como `engagement_score` (puntuación de interacción), `account_age_days` y `referral_bonus`. Utiliza un algoritmo de selección ponderada con complejidad O(log n) mediante un árbol de Fenwick. Permite configurar umbrales mínimos de participación (`min_participants=100`) para activar el sorteo.

- **Sincronización Multiplataforma**: Conecta con APIs de Twitter, Instagram y Discord mediante webhooks con reintentos exponenciales (backoff máximo de 5 segundos). Almacena el estado de cada sorteo en PostgreSQL con índices compuestos (`(giveaway_id, user_id, status)`) y utiliza triggers para actualizar la tabla de estadísticas en milisegundos.

### Cómo Usar

1. **Configuración del Sorteo**: Define los parámetros en un archivo YAML:
   ```yaml
   giveaway:
     id: "GIV-2025-03"
     prize_pool: ["item_001", "item_002"]
     eligibility_rules:
       min_account_age: 30d
       required_followers: 10
       geo_restrictions: ["MX", "ES", "AR"]
     schedule:
       start: "2025-04-01T00:00:00Z"
       end: "2025-04-07T23:59:59Z"
   ```
   Ejecuta `giveawaytracker init --config config.yaml` para crear la estructura en PostgreSQL y Redis.

2. **Inicio del Monitoreo**: Activa el rastreador con `giveawaytracker run --mode realtime`. El sistema comenzará a escuchar eventos de las plataformas configuradas. Verifica el estado con `giveawaytracker status --giveaway-id GIV-2025-03` para obtener métricas en vivo: `participants_count=1234`, `unique_ips=987`, `fraud_detections=12`.

3. **Finalización y Distribución**: Al alcanzar la fecha límite, ejecuta `giveawaytracker draw --giveaway-id GIV-2025-03 --algorithm weighted`. El sistema genera un hash criptográfico del resultado (SHA-256) y lo publica en un endpoint público. Los ganadores se notifican automáticamente mediante webhooks configurados previamente.

### Mejores Prácticas

- **Segmentación de Premios**: Para sorteos con múltiples premios, utiliza el parámetro `tiered_distribution` para asignar probabilidades decrecientes (ej: primer premio 5%, segundo 10%, tercero 15%). Esto evita que un solo usuario acumule todos los premios y mejora la percepción de equidad.

- **Auditoría Post-Sorteo**: Habilita el modo `--audit-log` para registrar cada acción (participación, validación, selección) en una tabla separada de PostgreSQL. Esto permite realizar análisis forenses en caso de disputas y cumple con requisitos legales de transparencia en sorteos comerciales.
