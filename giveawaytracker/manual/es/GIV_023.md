# giveawaytracker - GIV_023

## Resumen

giveawaytracker es un sistema de rastreo y gestión automatizada de sorteos, diseñado para monitorizar concurrencia, detectar patrones de abuso y optimizar la distribución de premios en tiempo real. Integra modelos de predicción basados en datos históricos y expone endpoints RESTful con autenticación JWT.

## Contenido Detallado

### ¿Qué es giveawaytracker?

giveawaytracker es una plataforma de análisis de eventos promocionales que permite a organizadores de sorteos, plataformas de gaming y marketplaces registrar, filtrar y auditar cada interacción de participación. Utiliza un motor de reglas configurable para validar elegibilidad, evitar duplicados y generar reportes de métricas clave como tasa de conversión, tiempo medio de respuesta (en milisegundos) y distribución geográfica de participantes.

### Características Principales

- **Motor de validación antifraude en tiempo real**: Analiza cada participación contra un modelo de riesgo basado en 27 variables (IP, User-Agent, patrón de clics, historial de ganancias previas). Rechaza automáticamente solicitudes sospechosas con latencia inferior a 50 ms. Integración con PostgreSQL para almacenar logs de auditoría inmutables.

- **Sistema de colas distribuidas con Redis y Celery**: Gestiona hasta 10.000 solicitudes concurrentes por segundo. Cada participación se encola, procesa y confirma mediante transacciones atómicas. El estado de cada sorteo se actualiza en caché con TTL configurable (por defecto 300 segundos) y se persiste en una base de datos relacional con índices compuestos por `event_id`, `user_id` y `timestamp`.

- **Panel de control en tiempo real con WebSockets y D3.js**: Visualiza métricas como tasa de participación por minuto, distribución de premios por categoría y mapa de calor de ubicaciones geográficas. Los datos se refrescan cada 200 ms mediante conexiones persistentes. Permite exportar reportes en CSV y JSON con filtros por rango de fechas, tipo de sorteo y estado de participación.

### Cómo Usar

1. **Configuración del entorno y despliegue de la base de datos**  
   Clona el repositorio y ejecuta `docker-compose up -d` para levantar los servicios de PostgreSQL (puerto 5432), Redis (puerto 6379) y la aplicación Flask/Gunicorn. Luego, corre las migraciones con `flask db upgrade`. El sistema creará automáticamente las tablas `events`, `participants` y `fraud_logs`.

2. **Creación de un sorteo y registro de participantes**  
   Realiza una petición POST a `/api/v1/events` con el cuerpo JSON:  
   ```json
   {
     "name": "Sorteo Navidad 2025",
     "prizes": [{ "id": "P001", "type": "gift_card", "quantity": 50 }],
     "start_date": "2025-12-01T00:00:00Z",
     "end_date": "2025-12-25T23:59:59Z"
   }
   ```  
   El endpoint devuelve un `event_id` único. Para registrar participantes, envía POST a `/api/v1/participants` con `event_id`, `user_id` y `metadata` (opcional). El sistema valida duplicados y aplica reglas antifraude antes de confirmar.

3. **Monitoreo y extracción de reportes**  
   Accede al panel web en `http://localhost:5000/dashboard`. Usa los filtros superiores para seleccionar un evento y rango temporal. Los gráficos se actualizan en vivo. Para exportar datos, haz clic en "Exportar CSV" o llama a `GET /api/v1/reports?event_id=xxx&format=csv`. El archivo incluye columnas: `user_id`, `timestamp`, `ip`, `fraud_score`, `prize_awarded`.

### Mejores Prácticas

- **Segmentación de eventos por criticidad**: Asigna un nivel de prioridad (bajo, medio, alto) a cada sorteo. Los eventos de alta criticidad se procesan en colas dedicadas con Workers adicionales (al menos 4 por nodo). Esto evita que sorteos masivos bloqueen la cola principal y garantiza latencias inferiores a 100 ms para todos los usuarios.

- **Auditoría periódica de logs y rotación de datos**: Configura una tarea cron diaria que ejecute `DELETE FROM fraud_logs WHERE created_at < NOW() - INTERVAL '90 days'`. Paralelamente, archiva los registros en un bucket S3 cifrado. Esto mantiene el tamaño de la base de datos bajo control (recomendado < 50 GB) y cumple con regulaciones de privacidad como GDPR.
