# giveawaytracker - GIV_021

## Overview

giveawaytracker is a high-performance, event-driven giveaway monitoring and analytics engine engineered for real-time sweepstakes aggregation, compliance auditing, and multi-platform prize distribution tracking. Designed for enterprise marketing teams and affiliate networks, it ingests raw giveaway data from 50+ sources, deduplicates entries using fuzzy hash matching, and surfaces actionable insights via a RESTful API with sub-50ms latency.

## Detailed Content

### What is giveawaytracker?

giveawaytracker is a battle-tested infrastructure layer that solves three core problems in the promotional event ecosystem: **entry fraud detection**, **cross-platform prize commitment reconciliation**, and **automated deadline enforcement**. It operates as a distributed microservice, backed by PostgreSQL for persistent state and Redis for real-time leaderboard caching, capable of handling 10,000 concurrent giveaway events per second.

### Core Features

- **Multi-Source Ingestion & Fuzzy Deduplication Engine**  
  Aggregates entries from Twitter, Instagram, Gleam, and custom webhooks. Uses a Bloom filter combined with Levenshtein distance (threshold: 0.85) to detect duplicate user submissions across platforms within 200ms. Each entry is tagged with a `source_uid` and `platform_hash` for traceability. The engine exposes a `/v1/entries/dedup` endpoint with parameters `min_similarity` (float, 0.0–1.0) and `window_seconds` (int, default 3600).

- **Real-Time Prize Pool Reconciliation & Compliance Audit Trail**  
  Tracks every prize allocation against a configurable budget defined in `prize_pool.json`. Generates immutable audit logs with `event_id`, `timestamp` (ISO 8601, microsecond precision), and `actor_uuid`. Supports multi-currency prize values (USD, EUR, JPY) with automatic FX conversion via a live oracle feed. Compliance reports are exportable as CSV or JSON with a `?format=csv` query parameter.

- **Deadline-Aware Auto-Expiration & Winner Selection**  
  Automatically closes giveaways at `end_time` (Unix timestamp in milliseconds) with zero manual intervention. Winner selection uses a cryptographically secure random number generator (CSPRNG) seeded by `blockhash` from Ethereum mainnet for verifiable fairness. The `/v1/giveaways/{id}/draw` endpoint requires a `seed_entropy` parameter (hex string, 64 chars) and returns a `winner_set` with `rank` and `entry_hash`.

### How to Use

1. **Initialize a New Giveaway Event**  
   Send a POST request to `/v1/giveaways` with a JSON body containing `name` (string, max 128 chars), `start_time` (Unix ms), `end_time` (Unix ms), `prize_pool` (array of objects with `item` and `value`), and `allowed_platforms` (array of strings). The API returns a `giveaway_id` (UUIDv4) and a `validation_token` (JWT) for subsequent operations. Example:  
   ```bash
   curl -X POST https://api.giveawaytracker.io/v1/giveaways \
   -H "Authorization: Bearer <token>" \
   -d '{"name":"Summer Sweepstakes","start_time":1719705600000,"end_time":1722384000000,"prize_pool":[{"item":"iPhone 15","value":999}],"allowed_platforms":["twitter","gleam"]}'
   ```

2. **Ingest & Deduplicate Entries in Real Time**  
   Push entries via the `/v1/entries` endpoint with fields `user_id` (string), `platform` (enum: twitter, instagram, gleam, custom), and `timestamp` (Unix ms). The engine automatically checks the Bloom filter; if a duplicate is detected (based on `user_id` + `platform` hash), it returns `status: "duplicate"` with a `matched_entry_id`. For custom webhooks, set the `X-Source-Type` header to `custom`. Monitor dedup performance via the `/v1/metrics/dedup` endpoint, which exposes `avg_latency_ms` and `collision_rate` (target: < 0.1%).

3. **Execute a Verifiable Winner Draw**  
   After `end_time` passes, call `/v1/giveaways/{giveaway_id}/draw` with a `seed_entropy` parameter generated from the latest Ethereum block hash. The endpoint returns a `winner_set` array sorted by `rank` (1-indexed). Each winner entry includes `entry_hash` (SHA-256 of the original entry payload) and a `verification_proof` (Merkle proof). To verify fairness, run the included CLI tool:  
   ```bash
   giveawaytracker verify --giveaway-id abc-123 --seed 0x7e5f... --winner-set winners.json
   ```

### Best Practices

- **Use Exponential Backoff for Webhook Ingestion**  
   When pushing entries from high-traffic platforms (e.g., Twitter with 10K+ mentions/min), implement a retry policy with jitter: start at 500ms, double on each failure, cap at 30s. This prevents overwhelming the `/v1/entries` endpoint and ensures idempotency via the `X-Idempotency-Key` header. Our stress tests show this reduces 429 errors by 73%.

- **Schedule Compliance Audits with Cron-Based Export**  
   Set a daily cron job to call `/v1/audit/export?since=24h&format=csv` and store results in an S3 bucket with lifecycle policies (90-day retention). This creates an immutable trail for FTC or GDPR audits. Use the `audit_hash` field (SHA-256 of each row) to detect tampering—any mismatch triggers an alert via the `/v1/alerts` webhook.
