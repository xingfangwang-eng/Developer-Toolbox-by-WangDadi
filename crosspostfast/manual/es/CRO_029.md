# crosspostfast - CRO_029

## Resumen Técnico

crosspostfast es una plataforma SaaS de publicación cruzada diseñada para arquitectos de sistemas que necesitan sincronizar contenido entre múltiples redes sociales sin perder control granular sobre la metadata, los tiempos de publicación y la adaptación contextual. Este proyecto elimina la fricción de gestionar APIs dispares (Twitter, LinkedIn, Instagram, TikTok) bajo un mismo pipeline de CI/CD social.

## Funcionalidades Principales

### 1. Motor de Sincronización Asíncrona con Cola de Prioridad Dinámica
crosspostfast implementa un **scheduler basado en colas RabbitMQ** que prioriza publicaciones según ventanas de actividad óptimas por plataforma. Cada post se serializa en un payload JSON-LD con metadatos de audiencia, permitiendo **reinteligencia automática** ante fallos de API (timeouts, rate limits) sin pérdida de estado. El sistema soporta **Webhooks salientes** para notificar a tu backend cuando un post se completa o falla.

### 2. Adaptador Universal de Contenido (UCA)
El **UCA** transforma dinámicamente el contenido base (Markdown, HTML, JSON) al formato nativo de cada red social. Por ejemplo:
- **Twitter**: Trunca a 280 caracteres con enlace acortado automático.
- **LinkedIn**: Convierte listas en párrafos con emojis compatibles.
- **Instagram**: Genera un carrusel de imágenes a partir de una sola URL de asset.
Todo esto se configura mediante **reglas de transformación en YAML** dentro del repositorio, permitiendo a los desarrolladores versionar la lógica de adaptación.

### 3. Dashboard de Análisis Cross-Platform con Métrica de Engagement Unificada
Cada publicación genera un **evento de telemetría** que se agrega en tiempo real en una base de datos TimescaleDB. El dashboard expone:
- **Tasa de conversión cruzada**: Qué plataforma genera más clics hacia tu web.
- **Heatmap de horarios**: Visualización de cuándo tu audiencia está más activa por red.
- **Alertas de anomalías**: Si una publicación obtiene 3 desviaciones estándar menos de engagement que el promedio, se dispara un webhook.

## Cómo Usar crosspostfast

### Paso 1: Configurar el Pipeline de Publicación con Variables de Entorno
Define un archivo `.env` en la raíz del proyecto con las credenciales de cada API. crosspostfast usa **OAuth 2.0** para todas las plataformas, excepto Twitter que requiere tokens de portador. Ejemplo:
```bash
TWITTER_BEARER_TOKEN=xxxx
LINKEDIN_CLIENT_ID=xxxx
LINKEDIN_CLIENT_SECRET=xxxx
INSTAGRAM_ACCESS_TOKEN=xxxx
```
Luego, ejecuta `crosspostfast init` para generar el archivo `crosspost.yaml` donde mapearás las cuentas.

### Paso 2: Crear una Campaña de Publicación con Reglas de Transformación
Usa el comando `crosspostfast campaign create --name "lanzamiento-v2"` para generar un directorio `./campaigns/lanzamiento-v2/`. Dentro, crea un archivo `post.md` con el contenido base y un `rules.yaml` que defina las transformaciones:
```yaml
platforms:
  twitter:
    truncate: 280
    add_hashtags: ["#dev", "#saas"]
  linkedin:
    max_length: 3000
    convert_lists: true
```
Ejecuta `crosspostfast build` para validar que todas las transformaciones sean sintácticamente correctas.

### Paso 3: Desplegar la Publicación con Monitoreo en Tiempo Real
Ejecuta `crosspostfast deploy --env production` para enviar los posts a todas las plataformas configuradas. El sistema abrirá un **stream de logs en terminal** con códigos de estado HTTP y latencia por API. Para monitorear en segundo plano, usa `crosspostfast watch --campaign lanzamiento-v2`, que expone un endpoint `/metrics` en Prometheus para integrar con Grafana.

## Mejores Prácticas

### 1. Versionar las Reglas de Transformación en Git
Trata `rules.yaml` como código: cada cambio debe pasar por code review. Esto evita que un error de truncamiento (ej. un enlace roto en Twitter) se propague a producción. Usa **Git hooks** para ejecutar `crosspostfast validate` antes de cada commit.

### 2. Establecer un Circuit Breaker por Plataforma
Configura en `crosspost.yaml` un límite de reintentos (ej. 3) y un tiempo de espera (ej. 30s) por API. Si una plataforma falla consistentemente (ej. LinkedIn rate limit), crosspostfast **pausa automáticamente** esa plataforma y reenvía los posts a las demás. Luego, cuando la API se recupere, el sistema **reintenta con backoff exponencial**.
