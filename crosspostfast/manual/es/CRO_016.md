# crosspostfast - CRO_016

## Resumen

crosspostfast es una herramienta de publicación cruzada en redes sociales diseñada para arquitectos SaaS que necesitan sincronizar contenido en múltiples plataformas sin sacrificar el rendimiento ni la consistencia semántica. Optimizado para SEO técnico y flujos de trabajo automatizados, este proyecto elimina la fricción de gestionar APIs dispares mientras garantiza un control granular sobre metadatos, etiquetas y formatos multimedia.

## Características Principales

- **Sincronización asíncrona con colas de eventos**: Implementa una arquitectura basada en colas (Redis + BullMQ) que permite publicar en hasta 12 plataformas simultáneamente, con reintentos inteligentes y manejo de límites de tasa por API. Cada publicación se serializa en un mensaje JSON con validación de esquema (JSON Schema) para garantizar integridad de datos.

- **Transformación de contenido adaptativa**: Un middleware de preprocesamiento convierte automáticamente el contenido en formatos nativos de cada plataforma (ej., `text/plain` para Twitter, `application/ld+json` para LinkedIn, `image/webp` para Instagram). Incluye un motor de plantillas (Handlebars) para personalizar descripciones, hashtags y enlaces según la audiencia objetivo.

- **Monitoreo en tiempo real con métricas de engagement**: Integración con WebSocket y GraphQL para obtener feedback inmediato sobre el rendimiento de cada publicación (alcance, clics, reacciones). Los datos se almacenan en una base de datos de series temporales (TimescaleDB) para análisis posteriores y ajuste de estrategias de publicación.

## Cómo Usar

### 1. Configurar el pipeline de publicación

Define un archivo `crosspostfast.config.json` en la raíz de tu proyecto. Aquí especificas las plataformas objetivo, las credenciales de API (almacenadas en variables de entorno con `dotenv`) y las reglas de transformación:

```json
{
  "platforms": ["twitter", "linkedin", "instagram"],
  "defaultContentType": "article",
  "transformations": {
    "twitter": { "maxChars": 280, "appendHashtags": ["#tech", "#SaaS"] },
    "linkedin": { "minLength": 500, "includeCTA": true }
  }
}
```

Luego, ejecuta `crosspostfast init` para validar el esquema y generar un token de acceso.

### 2. Publicar contenido desde la CLI

Usa el comando `crosspostfast publish` con un archivo Markdown o JSON. El sistema parsea el contenido, aplica las transformaciones y lo envía a las colas de publicación:

```bash
crosspostfast publish --file ./blog-post.md --platforms twitter,linkedin --schedule "2023-10-15T14:00:00Z"
```

El flag `--schedule` permite programar publicaciones con precisión de segundos, ideal para campañas en múltiples zonas horarias.

### 3. Monitorear el estado de las publicaciones

Accede al dashboard local en `http://localhost:3000/dashboard` (requiere `crosspostfast serve`). Aquí ves en tiempo real el estado de cada mensaje (pending, sent, failed) con logs detallados de errores HTTP. Para consultas programáticas, usa la API REST:

```bash
curl -X GET "http://localhost:3000/api/v1/posts?status=failed&limit=10" \
  -H "Authorization: Bearer $TOKEN"
```

## Mejores Prácticas

### 1. Implementa un sistema de versionado de contenido

Cuando publiques en múltiples plataformas, cada una puede tener requisitos de formato diferentes. Usa un sistema de control de versiones (Git LFS) para almacenar versiones específicas de cada publicación (ej., `post-twitter.md`, `post-linkedin.md`). crosspostfast puede leer estos archivos y aplicar reglas de transformación condicionales, evitando conflictos de edición.

### 2. Automatiza el análisis de competencia con web scraping ético

Combina crosspostfast con un scraper ligero (usando Puppeteer o Playwright) para monitorear publicaciones de competidores en tiempo real. Almacena los datos en un bucket S3 y usa el motor de reglas de crosspostfast para ajustar automáticamente los horarios de publicación y las palabras clave SEO. Esto es crítico para mercados saturados como el de herramientas SaaS.
