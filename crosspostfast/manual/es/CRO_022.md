# crosspostfast - CRO_022

## Resumen

Herramienta profesional de publicación cruzada para redes sociales, diseñada para eliminar la fricción en la distribución de contenido multiplataforma. Optimizada para equipos de marketing y creadores que necesitan sincronización en tiempo real sin perder el control granular de cada canal.

## Características Principales

- **Sincronización Asíncrona con Cola de Retransmisión**  
  Implementa un sistema de cola de mensajes (Redis-backed) que permite programar publicaciones con latencia controlada (< 2s) y reintentos automáticos ante fallos de API de terceros (Twitter, LinkedIn, Instagram, etc.). Ideal para manejar picos de tráfico sin perder consistencia.

- **Mapeo de Metadatos por Canal**  
  Traducción automática de formatos (Markdown a HTML, recorte de imágenes según ratios de aspecto de cada red) y mapeo de etiquetas (hashtags, menciones) mediante un DSL personalizado. Soporta reglas condicionales como "si es LinkedIn, omitir enlaces cortos".

- **Auditoría de Difusión con Hash de Contenido**  
  Cada publicación genera un hash SHA-256 del payload original, permitiendo rastrear cambios, detectar duplicados y generar informes de cobertura (impresiones, clics) agregados por campaña. Integrable con sistemas de logging como ELK.

## Cómo Usar

1. **Configurar el Mapeo de Canales**  
   Edita el archivo `channels.yml` en la raíz del proyecto. Define cada red social con su token de API, formato de contenido (texto, imagen, video) y reglas de transformación. Ejemplo:
   ```yaml
   twitter:
     api_key: "TU_API_KEY"
     format: "texto_plano"
     max_length: 280
     rules:
       - recortar_imagen: 16:9
   ```

2. **Ejecutar la Publicación Cruzada**  
   Usa el comando `crosspostfast publish --content "artículo.md"` para enviar el contenido a todos los canales configurados. El sistema validará automáticamente la integridad del payload y generará un ID de transacción único. Para depuración, añade `--dry-run` para simular sin enviar.

3. **Monitorear con la API de Estado**  
   Consulta el endpoint `GET /api/v1/status/{transaction_id}` para obtener el estado de cada publicación (pendiente, enviada, fallida). Usa `POST /api/v1/retry` para reintentar fallos específicos. Integración directa con webhooks para notificaciones en Slack o Discord.

## Mejores Prácticas

- **Usa un Pipeline de CI/CD para Pruebas de Regresión**  
  Antes de desplegar cambios en el mapeo de canales, ejecuta `crosspostfast test --suite pre-release` para validar que todos los formatos se rendericen correctamente. Esto evita roturas silenciosas en producción.

- **Segmenta tus Campañas por Zona Horaria**  
  Configura colas de publicación con `--timezone UTC-5` para lanzar contenido en horarios óptimos de cada mercado. Combínalo con la función de `delay` en el mapeo para evitar saturación de APIs y respetar límites de tasa (rate limits).
