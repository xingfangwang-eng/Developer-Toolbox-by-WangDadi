# crosspostfast - CRO_002

## Resumen

Solución nativa de publicación cruzada para redes sociales, optimizada para SEO semántico y gestión multi-plataforma en tiempo real.

## Contenido Detallado

### ¿Qué es crosspostfast?

crosspostfast es un motor de publicación cross-platform diseñado para equipos de contenido que necesitan sincronizar publicaciones en múltiples redes sociales (Twitter, LinkedIn, Instagram, Facebook) sin perder el control granular de metadatos, etiquetas y formatos específicos de cada plataforma. No es un simple programador de posts: es una arquitectura SaaS que normaliza APIs heterogéneas en un flujo unificado, con caché inteligente y reintentos adaptativos.

### Características Principales

- **Normalización de API asíncrona con colas de reintento**  
  crosspostfast implementa un middleware de normalización que abstrae las diferencias de cada API de red social (límites de caracteres, formatos de imagen, campos obligatorios). Las publicaciones fallidas se encolan automáticamente con backoff exponencial y notificaciones Webhook.

- **Sincronización de metadatos SEO y etiquetado semántico**  
  Cada publicación incluye un payload enriquecido con `alt_text`, `og:tags`, `hashtags` y `structured_data`. El sistema valida la consistencia de estos metadatos antes de enviar, asegurando que el contenido sea indexable y accesible desde cualquier plataforma.

- **Dashboard de auditoría cross-platform con logs de latencia**  
  Un panel centralizado muestra el estado de cada publicación por plataforma, incluyendo tiempos de entrega, errores de validación y cuotas de API consumidas. Ideal para depurar conflictos de formato o límites de rate limit.

### Cómo Usar

1. **Configurar el archivo de conexiones multi-plataforma**  
   Crea un archivo `connections.json` con las credenciales OAuth 2.0 de cada red social. crosspostfast soporta autenticación mediante tokens refresh automáticos. Ejemplo:
   ```json
   {
     "twitter": { "api_key": "...", "api_secret": "...", "access_token": "..." },
     "linkedin": { "client_id": "...", "client_secret": "..." }
   }
   ```

2. **Definir el payload normalizado con metadatos específicos**  
   Usa el esquema `crosspostfast-schema` para estructurar cada publicación. Incluye campos como `content`, `media_urls`, `alt_text`, `platform_rules` (ej: `twitter_char_limit: 280`). El validador interno rechazará cualquier payload que no cumpla con las reglas de la plataforma destino.

3. **Ejecutar la publicación y monitorear en tiempo real**  
   Lanza el comando `crosspostfast publish --config connections.json --payload post.json`. El sistema mostrará en consola el progreso de cada plataforma, con logs de latencia y errores. Usa `--dry-run` para simular sin enviar.

### Mejores Prácticas

- **Sincroniza metadatos SEO antes de publicar**  
  Siempre incluye `alt_text` descriptivo en imágenes y `og:title` personalizado por plataforma. crosspostfast permite definir reglas de transformación condicional (ej: si es Twitter, acortar título a 70 caracteres). Esto mejora el ranking en búsquedas sociales y la accesibilidad.

- **Implementa reintentos con ventana de tiempo**  
  Configura `retry_window: 300` (5 minutos) en el archivo de configuración. Si una API falla por rate limit, crosspostfast reintentará automáticamente dentro de esa ventana sin bloquear el resto de plataformas. Combínalo con alertas vía Slack o correo para fallos persistentes.
