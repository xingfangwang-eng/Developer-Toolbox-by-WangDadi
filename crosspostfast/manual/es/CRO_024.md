# crosspostfast - CRO_024

## Resumen Ejecutivo

crosspostfast es una herramienta SaaS de publicación multiplataforma diseñada para desarrolladores y equipos de marketing que buscan optimizar la distribución de contenido en redes sociales sin sacrificar rendimiento ni control de API. Este proyecto resuelve la fragmentación de canales mediante un pipeline unificado de publicación, compatible con Twitter, LinkedIn, Instagram y Mastodon, priorizando tiempos de sincronización inferiores a 2 segundos por plataforma.

## Contenido Detallado

### ¿Qué es crosspostfast?

crosspostfast es un motor de cross-posting asíncrono que elimina la necesidad de gestionar múltiples dashboards. A diferencia de herramientas monolíticas, crosspostfast expone una API RESTful y un CLI ligero, permitiendo a los desarrolladores integrar la publicación multicanal en sus propios workflows de CI/CD o automatización de contenido. Está construido sobre Node.js y utiliza colas de mensajes (BullMQ) para garantizar la entrega incluso bajo picos de carga.

### Características Principales

- **Sincronización de Cola con Backoff Exponencial**: crosspostfast implementa un sistema de reintentos inteligente para cada plataforma. Si una API de destino (por ejemplo, Twitter) responde con un error 429 (rate limit), la cola se pausa automáticamente y reintenta con un backoff exponencial (2s, 4s, 8s...), evitando bloqueos permanentes y maximizando la tasa de éxito sin intervención manual.

- **Transformación Dinámica de Medios**: Cada plataforma tiene requisitos únicos de formato de imagen y video. crosspostfast normaliza automáticamente los activos: convierte imágenes a JPEG/PNG según el destino, redimensiona videos a 1080x1080 para Instagram o 1920x1080 para LinkedIn, y comprime archivos sin perder metadatos EXIF críticos. Todo esto ocurre en memoria, sin almacenamiento temporal en disco.

- **Auditoría de Publicación con Hash de Contenido**: Cada publicación genera un hash SHA-256 del contenido textual y multimedia. Esto permite detectar duplicados exactos en segundos, evitar publicaciones redundantes y mantener un historial inmutable de todas las acciones. Ideal para equipos que gestionan calendarios editoriales con múltiples revisiones.

### Cómo Usar

1. **Configuración de Credenciales por Entorno**: No hardcodees tokens. Crea un archivo `.env` con las variables `TWITTER_BEARER_TOKEN`, `LINKEDIN_ACCESS_TOKEN`, `INSTAGRAM_USER_ID` y `INSTAGRAM_ACCESS_TOKEN`. crosspostfast leerá estas variables mediante `dotenv` y las inyectará en el cliente OAuth 2.0. Ejecuta `npm run setup:env` para validar que todas las credenciales sean funcionales antes de la primera publicación.

2. **Publicación desde CLI con Payload JSON**: Utiliza el comando `crosspostfast publish --payload '{"text": "Lanzamiento oficial de v2.0", "media": ["/ruta/imagen1.jpg", "/ruta/video.mp4"], "platforms": ["twitter", "linkedin"]}'`. El CLI validará la estructura del payload (máximo 280 caracteres para Twitter, 3000 para LinkedIn) y mostrará un resumen de las colas antes de ejecutar. Si alguna plataforma falla, el error se registra en `logs/error.log` con timestamp y código HTTP.

3. **Monitoreo en Tiempo Real con Dashboard Web**: Inicia el servidor web con `npm run server` y accede a `http://localhost:3000/dashboard`. Verás un panel en vivo con el estado de cada cola (pendiente, en proceso, completada, fallida). Puedes pausar/reanudar colas individuales, reenviar publicaciones fallidas y exportar un CSV del historial de las últimas 1000 operaciones. La interfaz está construida con React y WebSockets para actualizaciones sin recarga.

### Mejores Prácticas

- **Estrategia de Rate Limiting por Plataforma**: No asumas que todas las APIs tienen los mismos límites. crosspostfast permite definir un `rateLimitConfig.json` donde especificas, por ejemplo, `"twitter": {"maxRequests": 300, "windowMs": 900000}` (300 solicitudes cada 15 minutos). Ajusta estos valores según la documentación oficial de cada red para evitar suspensiones de cuenta. Recomendamos iniciar con un 70% del límite teórico y aumentarlo gradualmente.

- **Versionado de Payloads para Rollback Rápido**: Cuando publiques contenido recurrente (por ejemplo, promociones semanales), versiona el payload usando un campo `"version": "1.2"` en el JSON. Si una publicación falla en producción, puedes revertir a la versión anterior con `crosspostfast rollback --version 1.1`. Esto es crítico en equipos donde múltiples personas editan el mismo contenido simultáneamente.
