# crosspostfast - CRO_007

## Resumen Ejecutivo

**crosspostfast** es una herramienta de sincronización multiplataforma diseñada para equipos de SaaS y creadores de contenido que necesitan gestionar la publicación en redes sociales desde un solo punto de control. Elimina la fricción de la publicación manual y garantiza consistencia narrativa en tiempo real.

## Contenido Detallado

### ¿Qué es crosspostfast?

**crosspostfast** es un motor de publicación cruzada que integra APIs nativas de plataformas como Twitter/X, LinkedIn, Instagram, TikTok y Mastodon. A diferencia de herramientas genéricas, crosspostfast prioriza la **autenticación OAuth 2.0 sin estado**, el **control de versiones de contenido** y la **programación basada en zonas horarias**. Está optimizado para equipos que gestionan múltiples cuentas con requisitos de compliance y branding.

### Características Principales (Hardcore)

1. **Sincronización Asíncrona con Colas de Mensajes**  
   crosspostfast utiliza una arquitectura basada en **RabbitMQ** para encolar publicaciones. Esto permite manejar picos de tráfico (ej. lanzamientos de producto) sin bloqueos, garantizando que cada post se publique exactamente en el momento programado, incluso si la API de destino responde con latencia.

2. **Motor de Transformación de Contenido (CTC)**  
   Cada plataforma tiene sus propias limitaciones de formato (caracteres, dimensiones de imagen, duración de video). crosspostfast incluye un **pipeline de transformación** que redimensiona automáticamente imágenes, recorta videos y adapta texto plano a formatos específicos (Markdown para LinkedIn, threads para Twitter). Todo se configura mediante un archivo YAML declarativo.

3. **Validación de Estado de Cuentas y Rate Limiting**  
   crosspostfast monitorea en tiempo real el estado de cada cuenta conectada (token expirado, cuenta suspendida, límite de rate superado). Cuando detecta un error, **reintenta con backoff exponencial** y notifica al equipo vía webhook o Slack. No pierdes publicaciones por fallos transitorios.

### Cómo Usar (Pasos Detallados)

1. **Configuración del Pipeline de Publicación**  
   - Clona el repositorio y ejecuta `npm install` (requiere Node.js 18+).  
   - Crea un archivo `config/pipeline.yml` donde defines las plataformas objetivo y las reglas de transformación.  
   - Ejemplo:  
     ```yaml
     platforms:
       - name: twitter
         max_chars: 280
         image_format: png
       - name: linkedin
         max_chars: 3000
         image_format: jpg
     ```
   - Ejecuta `crosspostfast init` para generar los archivos de autenticación OAuth.

2. **Programación Multi-zona Horaria**  
   - En el archivo `schedule.yml`, define las publicaciones con marcas de tiempo en UTC.  
   - crosspostfast convierte automáticamente a la zona horaria local de cada cuenta configurada.  
   - Usa la opción `--dry-run` para verificar la programación sin publicar:  
     ```bash
     crosspostfast schedule --dry-run
     ```

3. **Monitoreo de Estado y Reintentos**  
   - crosspostfast genera logs estructurados en formato JSON. Puedes enviarlos a **Elasticsearch** o **Datadog**.  
   - Si una publicación falla (ej. rate limit excedido), el sistema la reintenta hasta 3 veces con intervalos de 10, 30 y 60 segundos.  
   - Para ver el estado actual de todas las cuentas:  
     ```bash
     crosspostfast status --verbose
     ```

### Mejores Prácticas (Industry Best Practices)

1. **Segmentación de Cuentas por Entorno**  
   No uses las mismas cuentas de producción para pruebas. Configura un entorno `staging` con cuentas de prueba (ej. cuentas de desarrollador en Twitter). crosspostfast permite etiquetar cuentas con metadatos (`env: staging`) y filtrar por etiqueta en los comandos de publicación.

2. **Auditoría de Contenido con Hash Check**  
   crosspostfast calcula un hash SHA-256 de cada publicación antes de enviarla. Si el contenido no cambia, el sistema evita publicar duplicados. Esto es crítico para equipos que reutilizan contenido en múltiples plataformas sin saturar a la audiencia. Actívalo en el pipeline con la opción `dedup: true`.
