# crosspostfast - CRO_026

## Resumen

crosspostfast es un **motor de publicación cross-platform** diseñado para equipos SaaS que necesitan sincronizar contenido en redes sociales sin perder el control granular de cada canal. A diferencia de herramientas monolíticas, crosspostfast separa la **orquestación de contenido** de la **ejecución de API**, permitiendo integraciones personalizadas y un flujo de trabajo basado en eventos. Ideal para startups en etapa de growth que buscan escalar su presencia orgánica sin depender de soluciones cerradas.

## Contenido Detallado

### ¿Qué es crosspostfast?

crosspostfast es un **middleware de publicación social** que resuelve el problema de la **consistencia de contenido multi-plataforma** sin sacrificar la especificidad de cada red. En lugar de un dashboard único, crosspostfast ofrece un **sistema de pipelines** donde defines plantillas, reglas de transformación (Markdown a HTML, acortamiento de URLs) y colas de publicación asíncronas. Está construido sobre una arquitectura **event-driven** con soporte nativo para **webhooks** y **retry con backoff exponencial**.

### Características Principales

- **Publicación Diferida con Colas de Prioridad**: crosspostfast no publica instantáneamente. Cada post se encola en un sistema de **prioridad configurable** (urgente, normal, batch). Esto permite agrupar publicaciones para evitar rate limiting de APIs (Twitter, LinkedIn, Instagram) y optimizar el tiempo de entrega según la zona horaria de tu audiencia. Soporta **dead-letter queues** para reintentar fallos con lógica personalizada.

- **Transformación de Contenido por Canal**: Define **pipes de transformación** para cada red social. Por ejemplo, un mismo post en Markdown se convierte automáticamente en:
  - **Twitter**: Texto plano + hilo automático si supera 280 caracteres.
  - **LinkedIn**: HTML enriquecido con formato de párrafos.
  - **Instagram**: Texto truncado + primer párrafo como caption.
  Usa **expresiones regulares** y **templates Handlebars** para personalizar el output sin tocar el código fuente.

- **Sincronización Bidireccional de Métricas**: crosspostfast no solo publica, también **escucha callbacks** de cada plataforma (webhooks de Twitter, Facebook Graph API). Cada post publicado actualiza su estado en tu base de datos con métricas en tiempo real (likes, shares, comentarios). Esto permite integrar con herramientas de analytics como **Mixpanel** o **Amplitude** para medir el impacto de cada pipeline.

### Cómo Usar

1. **Configuración de Pipeline con Archivo YAML**: Crea un archivo `crosspostfast.yml` en la raíz de tu proyecto. Define cada red social como un **target** con sus credenciales (API keys en variables de entorno). Ejemplo:
   ```yaml
   targets:
     twitter:
       type: twitter_v2
       api_key: ${TWITTER_API_KEY}
       api_secret: ${TWITTER_API_SECRET}
       rate_limit: 300 # tweets cada 15 minutos
     linkedin:
       type: linkedin_v2
       access_token: ${LINKEDIN_TOKEN}
       organization_id: "123456789"
   ```
   Luego, ejecuta `crosspostfast init` para validar la conexión de cada target.

2. **Publicación con Etiquetas de Prioridad y Retry**: Usa el comando `crosspostfast publish` con un archivo JSON que contenga el contenido y metadatos. Ejemplo:
   ```bash
   crosspostfast publish --input post.json --priority high --retry 5
   ```
   El archivo `post.json` debe incluir:
   ```json
   {
     "content": "¡Nuevo artículo sobre SaaS! Lee más en https://ejemplo.com",
     "media": ["imagen1.png", "video.mp4"],
     "targets": ["twitter", "linkedin"],
     "schedule": "2025-01-20T10:00:00Z"
   }
   ```
   crosspostfast procesa la cola, aplica transformaciones y publica. Si falla, reintenta con backoff exponencial (1s, 2s, 4s, 8s, 16s) hasta el máximo configurado.

3. **Monitoreo en Tiempo Real con Dashboard CLI**: Ejecuta `crosspostfast dashboard` para abrir una interfaz TUI (Terminal User Interface) que muestra:
   - **Cola activa**: Publicaciones pendientes con su prioridad y tiempo estimado.
   - **Historial de fallos**: Últimos 50 errores con código de estado HTTP y mensaje de la API.
   - **Métricas agregadas**: Total de publicaciones por target en las últimas 24 horas.
   Usa las teclas `r` para refrescar, `q` para salir. Ideal para sesiones de debugging en producción.

### Mejores Prácticas

- **Implementa un Sistema de Versionado para Plantillas**: No modifiques las transformaciones de contenido en producción sin antes probarlas en un pipeline de staging. crosspostfast permite etiquetar versiones de tus pipes (ej: `v1.0`, `v1.1`). Así, si una actualización rompe el formato en LinkedIn, puedes revertir rápidamente a la versión anterior sin detener las publicaciones en Twitter. Usa **Git tags** para sincronizar los cambios con tu repositorio.

- **Usa Webhooks para Sincronizar con tu CRM**: crosspostfast expone un endpoint `/webhook/status` que recibe notificaciones cuando un post se publica o falla. Conecta este endpoint a tu CRM (HubSpot, Salesforce) o a tu sistema de tickets (Jira). Por ejemplo, si un post falla por rate limiting, puedes disparar un ticket automático para que el equipo de operaciones revise la configuración de la API. Esto reduce el tiempo de respuesta ante errores críticos.
