# crosspostfast - CRO_015

## Resumen Técnico

**crosspostfast** es un orquestador de publicación multiplataforma diseñado para equipos SaaS que necesitan sincronizar contenido en redes sociales sin perder el control granular de cada canal. Optimizado para SEO técnico y velocidad de ejecución, este proyecto elimina la fricción entre la creación de contenido y su distribución masiva.

## Funcionalidades Duras

### 1. **Pipeline de Publicación Asíncrona con Cola de Prioridad**
Cada post se encola en un sistema de prioridad configurable (por red, por hora, por tipo de contenido). El motor interno gestiona reintentos automáticos con backoff exponencial y valida códigos de respuesta HTTP en tiempo real. Ideal para campañas que requieren lanzamientos sincronizados en múltiples zonas horarias.

### 2. **Normalización Automática de Media por Plataforma**
crosspostfast detecta automáticamente el formato de imágenes y videos (JPEG, PNG, MP4, WebM) y los reescala, recorta o convierte según las especificaciones de cada red social. No más errores por dimensiones incorrectas en Instagram o límites de duración en Twitter.

### 3. **Hash de Contenido y Detección de Duplicados**
Cada publicación genera un hash SHA-256 del texto + media. Si intentas publicar contenido idéntico en la misma red dentro de un período configurable, el sistema lo rechaza o lo agenda como actualización. Esto evita penalizaciones por spam y mantiene la coherencia del feed.

## Cómo Usar (Flujo de Integración)

### **Paso 1: Configurar el Archivo de Mapeo de Redes**
Crea un archivo YAML (`networks.yaml`) donde definas las credenciales OAuth 2.0 para cada plataforma (Twitter, LinkedIn, Instagram, etc.). crosspostfast no almacena tokens en texto plano; usa variables de entorno o un vault externo. Ejemplo mínimo:
```yaml
twitter:
  api_key: ${TWITTER_API_KEY}
  api_secret: ${TWITTER_API_SECRET}
  access_token: ${TWITTER_ACCESS_TOKEN}
linkedin:
  client_id: ${LINKEDIN_CLIENT_ID}
  client_secret: ${LINKEDIN_CLIENT_SECRET}
```

### **Paso 2: Construir el Payload de Publicación**
Usa el endpoint `POST /publish` con un JSON que contenga:
- `text`: string del contenido (máximo 280 caracteres para Twitter, 3000 para LinkedIn).
- `media`: array de URLs públicas o rutas locales (el sistema las descarga y normaliza).
- `schedule`: timestamp ISO 8601 opcional para publicación diferida.
- `networks`: array de strings con los nombres de las redes destino.

Ejemplo:
```json
{
  "text": "Lanzamos crosspostfast v2.0 - ahora con soporte para Instagram Reels.",
  "media": ["https://cdn.example.com/demo.mp4"],
  "schedule": "2025-04-15T10:00:00Z",
  "networks": ["twitter", "linkedin", "instagram"]
}
```

### **Paso 3: Monitorear el Estado con Webhooks**
Configura un webhook en `POST /webhook` para recibir notificaciones en tiempo real sobre el estado de cada publicación. crosspostfast envía eventos como `published`, `failed`, `rate_limited`, o `media_converted`. Esto te permite integrar alertas en Slack, Discord o tu propio dashboard.

## Mejores Prácticas para el Ecosistema SaaS

### **1. Segmenta por Zona Horaria con Cronogramas Dinámicos**
No publiques todo a la vez. crosspostfast permite definir ventanas de publicación por red (ej: LinkedIn solo en horario laboral CET, Instagram a las 8 PM EST). Usa la función `time_window` en el payload para evitar tasas de rebote altas y maximizar el engagement.

### **2. Estrategia de Reinteligencia con Backoff Exponencial**
Cuando una red devuelve `429 Too Many Requests`, crosspostfast no reintenta inmediatamente. Implementa un backoff que duplica el tiempo de espera hasta un máximo de 5 minutos. Si falla tras 3 intentos, el post se marca como `failed` y se notifica al webhook. Esto respeta los límites de API y mantiene la reputación de tu cuenta.
