# crosspostfast - CRO_030

## Resumen

Herramienta SaaS de publicación cruzada optimizada para SEO y rendimiento, diseñada para equipos de marketing y desarrolladores que gestionan múltiples cuentas en redes sociales (LinkedIn, Twitter, Instagram, TikTok) desde un solo pipeline CI/CD.

## Contenido Detallado

### ¿Qué es crosspostfast?

crosspostfast es un orquestador de contenido multi-plataforma que elimina la fricción de la publicación manual. Integra APIs nativas de cada red social, soporta colas de publicación asíncronas, y ofrece un dashboard en tiempo real con métricas de engagement y tasa de error por plataforma. Ideal para agencias digitales y SaaS que necesitan escalar su presencia social sin sacrificar consistencia de marca.

### Características Principales

- **Pipeline de publicación con cola de reintentos inteligente**: Cada post se encola con prioridad configurable. Si una API falla (por rate limiting o timeout), crosspostfast reintenta automáticamente con backoff exponencial y notifica por webhook. Soporta hasta 50.000 solicitudes concurrentes por hora.
- **Normalización automática de contenido**: Convierte automáticamente formatos de imagen/video según las especificaciones de cada plataforma (relación de aspecto, tamaño máximo, duración). Ejemplo: un carrusel de Instagram se transforma en un hilo de Twitter sin perder metadata.
- **Dashboard de rendimiento con alertas en tiempo real**: Muestra métricas como tiempo de publicación medio, tasa de éxito por plataforma, y coste estimado de API calls. Las alertas se envían por Slack o correo si la tasa de error supera el 5% en los últimos 10 minutos.

### Cómo Usar

1. **Configuración inicial del pipeline de publicación**  
   Ejecuta `crosspostfast init` para generar el archivo `crosspostfast.yml`. Define las plataformas objetivo, las credenciales OAuth 2.0 (almacenadas en variables de entorno), y las reglas de normalización. Ejemplo:  
   ```yaml
   platforms:
     - name: linkedin
       api_version: v2
       max_retries: 3
       content_rules:
         image_max_size: 5MB
         video_max_duration: 10min
     - name: twitter
       api_version: v2
       max_retries: 5
   ```

2. **Publicación programada con cola de prioridad**  
   Usa `crosspostfast publish --queue high` para enviar contenido urgente (ej. anuncios de producto) antes que el contenido orgánico. La cola se procesa con un scheduler interno que respeta los rate limits de cada API. Para ver el estado de la cola: `crosspostfast queue status --verbose`.

3. **Monitoreo y depuración en tiempo real**  
   Abre el dashboard web con `crosspostfast dashboard --port 8080`. Filtra por plataforma, fecha o estado (éxito/error). Para errores, haz clic en el ID del post para ver el log completo de la API, incluyendo el código HTTP y el payload enviado. Ejemplo de comando CLI: `crosspostfast logs --post-id 1234 --format json`.

### Mejores Prácticas

- **Implementa un sistema de rate limiting por plataforma**: Cada red social tiene límites diferentes (Twitter: 300 tweets/3h, LinkedIn: 100 posts/día). Configura `max_requests_per_minute` en `crosspostfast.yml` para evitar bloqueos. Usa la función de cola con backoff para repartir la carga.
- **Automatiza la revisión de contenido con un pre-commit hook**: Antes de publicar, ejecuta `crosspostfast validate` para verificar que las imágenes no contengan texto oculto (OCR) y que los enlaces no estén rotos. Integra esto en tu CI/CD para evitar publicar contenido defectuoso.
