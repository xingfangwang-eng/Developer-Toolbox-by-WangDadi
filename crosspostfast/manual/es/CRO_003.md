# crosspostfast - CRO_003

## Resumen Técnico

**crosspostfast** es un motor de publicación cross-platform optimizado para SEO, diseñado para eliminar la fricción en la sindicación de contenido a través de redes como Twitter/X, LinkedIn, Mastodon, y Bluesky. Este proyecto se centra en la automatización de metadatos, la gestión de colas inteligentes y la normalización de formatos de publicación para maximizar el alcance orgánico sin depender de APIs de terceros con límites restrictivos.

## Funcionalidades Clave de Ingeniería

### 1. **Motor de Normalización de Contenido Multiformato**
El sistema analiza y transforma automáticamente el contenido de entrada (Markdown, HTML, texto plano) a formatos nativos de cada plataforma. Por ejemplo, convierte encabezados `##` en hashtags para Twitter, y listas anidadas en bullet points de LinkedIn, todo mientras preserva la semántica SEO mediante la inyección de `alt text` en imágenes y `structured data` en los posts.

### 2. **Cola de Publicación con Priorización Temporal y Algoritmo de Rate-Limiting**
Implementa un scheduler basado en `cron` que ajusta dinámicamente los intervalos de publicación según los patrones de actividad de la audiencia objetivo (detectados por análisis de engagement histórico). Incluye un módulo de `backoff exponencial` para evitar bloqueos por parte de las APIs, y un sistema de reintentos con `circuit breaker` para fallos transitorios.

### 3. **Generación Automática de Metadatos SEO Cross-Platform**
Cada publicación se enriquece con `Open Graph` tags, `Twitter Cards` y `JSON-LD` específicos para la plataforma destino. Además, el sistema extrae palabras clave del contenido y las integra en los `meta descriptions` y `titles` de los snippets generados, mejorando la visibilidad en motores de búsqueda y en feeds sociales.

## Guía de Uso Avanzada

### 1. **Configuración del Pipeline de Publicación Automatizada**
- **Preparación del entorno**: Clona el repositorio y ejecuta `npm install` para instalar dependencias. Luego, configura las variables de entorno en un archivo `.env` con las claves API de cada plataforma (ej. `TWITTER_BEARER_TOKEN`, `LINKEDIN_ACCESS_TOKEN`).
- **Definición de la cola**: En el archivo `config/schedule.json`, define los horarios de publicación usando sintaxis cron. Por ejemplo, `"0 8,12,18 * * 1-5"` publica a las 8 AM, 12 PM y 6 PM de lunes a viernes.
- **Ejecución del worker**: Inicia el worker con `node src/worker.js`. Este escuchará la cola de Redis y procesará cada tarea aplicando las reglas de normalización y rate-limiting configuradas.

### 2. **Personalización de Reglas de Normalización por Plataforma**
- **Editar el módulo de transformación**: Ve a `src/transformers/`. Cada plataforma tiene su propio archivo (ej. `twitter.js`, `linkedin.js`). Modifica las funciones `transformContent()` para ajustar cómo se manejan los hashtags, menciones o enlaces.
- **Ejemplo práctico**: Para que los enlaces se acorten automáticamente en Twitter, agrega una regla en `twitter.js` que llame a un servicio de acortamiento (ej. `bitly`) antes de la publicación.

### 3. **Monitoreo y Depuración del Flujo de Publicación**
- **Logs estructurados**: Todos los eventos se registran en `logs/` con formato JSON. Usa `tail -f logs/app.log | jq .` para ver en tiempo real el estado de cada publicación.
- **Dashboard de métricas**: Ejecuta `node src/dashboard.js` para iniciar un servidor HTTP en `localhost:3000`. Aquí verás estadísticas de éxito/fallo, latencia de APIs y cuota de rate-limit restante.

## Mejores Prácticas para Desarrolladores

### 1. **Estrategia de Rate-Limiting Adaptativo**
No uses intervalos fijos. Implementa un sistema que lea los encabezados `X-RateLimit-Remaining` de cada respuesta de API y ajuste dinámicamente el tiempo de espera entre peticiones. Por ejemplo, si el límite restante es bajo, aumenta el `delay` en un 50%. Esto evita bloqueos y maximiza el throughput.

### 2. **Versionado de Contenido para SEO**
Cada vez que publiques un post, guarda una copia en `storage/versions/` con un hash del contenido. Si necesitas actualizar un post (ej. corregir un enlace), el sistema detecta el cambio y genera un nuevo post con un `canonical URL` apuntando al original, evitando penalizaciones por contenido duplicado.
