## What is crosspostfast?

**crosspostfast** is a headless content syndication layer engineered for dev-first teams and growth hackers.  
It eliminates the friction of multi-platform scheduling by providing a unified API, deterministic deduplication, and platform-aware formatting.  
Think of it as a **Webhook + CI/CD pipeline for your social media content**.

---

## Core Features

### 1. Deterministic Content Hash & Dedup Engine
Every post is fingerprinted using SHA-256 of its normalized content + target platform ID.  
This prevents accidental re-posting, enforces idempotency across retries, and allows you to trace exactly which version of a post landed on which platform.

### 2. Platform-Aware Media Transcoding Pipeline
Automatically converts and crops media assets (images, video thumbnails, GIFs) to match each platform’s exact specs (e.g., Twitter 16:9, LinkedIn 1.91:1, Instagram 4:5).  
Uses FFmpeg + Sharp under the hood, with zero-downtime fallback to original media if conversion fails.

### 3. Atomic Batch Publishing with Rollback
Publish to 10+ platforms in a single atomic transaction. If any platform fails (rate limit, auth error, malformed payload), the entire batch is rolled back automatically.  
No orphan posts, no partial visibility, no manual cleanup.

## How to Use

### Step 1: Initialize the Crosspost Client
```bash
npm install crosspostfast-client
```
```js
import { CrosspostClient } from 'crosspostfast-client';

const client = new CrosspostClient({
  apiKey: process.env.CROSSPOST_FAST_API_KEY,
  defaultPlatforms: ['twitter', 'linkedin', 'mastodon'],
  dedupMode: 'strict', // 'strict' | 'soft' | 'off'
});
```

### Step 2: Define a Multi-Platform Post with Metadata
```js
const post = client.createPost({
  body: 'Our new release ships with zero-copy streaming. Check the changelog →',
  media: ['./assets/demo.mp4', './assets/thumbnail.png'],
  platforms: {
    twitter: { tags: ['#dev', '#releases'] },
    linkedin: { visibility: 'public', articleMode: false },
    mastodon: { visibility: 'unlisted', language: 'en' },
  },
  scheduledAt: '2025-05-20T14:00:00Z',
});
```

### Step 3: Execute Atomic Publish & Handle Rollback
```js
const result = await client.publishAtomic(post);

if (result.status === 'committed') {
  console.log(`Published to ${result.platforms.length} platforms`);
} else if (result.status === 'rolled_back') {
  console.error(`Failed platforms: ${result.failedPlatforms.join(', ')}`);
  // No cleanup needed — rollback was automatic
}
```

## Best Practices

### 1. Use Staging Platforms for Integration Testing
Before pushing to production platforms, create a private test environment (e.g., a secondary Mastodon instance or a dummy Twitter account).  
Run `publishAtomic()` with `dryRun: true` to validate payloads, media sizes, and formatting without hitting real APIs.

### 2. Implement a Content Versioning Strategy
Maintain a `version` field in your post metadata. When you update a post (e.g., fix a typo or swap a thumbnail), increment the version.  
Crosspostfast’s dedup engine will treat it as a new post, ensuring updates propagate across all platforms without duplicate content penalties.
