# crosspostfast - CRO_017

## Overview

**crosspostfast** is a high-performance, multi-platform content syndication engine built for developers who need to blast content across LinkedIn, Twitter, Mastodon, and custom REST endpoints — without ever touching a browser. It’s not a dashboard; it’s a CLI-first, API-driven automation framework designed to eliminate repetitive publishing workflows and reduce time-to-post by up to 90%.

**Keywords:** Social media cross-platform posting, Multi-platform content publishing, Social media management tool

---

## Core Features

### 1. **Declarative Platform Profiles with Smart Field Mapping**
Define platform-specific templates using YAML or JSON. Each profile maps your content fields (title, body, image, tags) to the target platform’s API schema. crosspostfast automatically handles character limits, media resizing, and link formatting — no manual tweaks per platform.

### 2. **Atomic Batch Publishing with Rollback**
Queue up to 50 posts per batch and publish them synchronously across multiple platforms. If any single platform fails (e.g., rate limit or auth error), the entire batch is rolled back using a two-phase commit pattern. Your content state stays consistent across all destinations.

### 3. **Webhook-Driven Continuous Syndication**
Hook crosspostfast into your CI/CD pipeline or CMS webhook. Every time a new blog post, PR, or release note is published, crosspostfast automatically detects the event, transforms the content, and pushes it to all configured platforms — zero manual intervention.

## How to Use

### 1. **Initialize a Multi-Platform Configuration**
Run `crosspostfast init` to generate a `crosspost.yml` file. Define your platforms with API keys and field mappings:

```yaml
platforms:
  linkedin:
    api_key: env://LINKEDIN_TOKEN
    fields:
      title: "{{ .Title }}"
      text: "{{ .Body | truncate 3000 }}"
  twitter:
    api_key: env://TWITTER_TOKEN
    fields:
      text: "{{ .Title }}: {{ .Body | truncate 240 }}"
```

### 2. **Draft and Validate a Post Locally**
Create a Markdown post with frontmatter, then run:

```bash
crosspostfast validate post.md --platform linkedin,twitter
```

This dry-runs the transformation, checks character limits, and previews the output for each platform. No API calls are made — perfect for debugging field mappings or truncation rules.

### 3. **Publish with Atomic Rollback**
Once validated, publish:

```bash
crosspostfast publish post.md --platform linkedin,twitter,mastodon --atomic
```

If any platform fails (e.g., Twitter returns 429), crosspostfast triggers a rollback: it deletes the post from successful platforms using their respective delete endpoints. The entire operation is treated as a single transaction.

## Best Practices

### 1. **Use Environment Variables for Secrets**
Never hardcode API keys in your config files. crosspostfast supports `env://` syntax to pull tokens from environment variables. Keep a `.env` file locally, and store secrets in your CI/CD vault (e.g., GitHub Actions secrets, HashiCorp Vault). This ensures your config is portable and secure across environments.

### 2. **Leverage Pre-commit Hooks for Content Validation**
Integrate `crosspostfast validate` into your Git pre-commit hook. Before any content commit, validate that all posts meet platform-specific constraints. This catches truncation issues, broken links, or oversized media early — before they ever reach production.
