# crosspostfast - CRO_012

## Overview

**crosspostfast** is a high-performance, multi-platform content syndication engine designed for developers, growth hackers, and SaaS teams who need to publish, schedule, and synchronize content across social media channels with zero friction. Built for speed, SEO compliance, and API-first extensibility.

## What is crosspostfast?

crosspostfast eliminates the bottleneck of manual cross-posting by providing a unified API and CLI interface that connects to major social platforms—Twitter, LinkedIn, Mastodon, Reddit, Telegram, and more. It's not just a scheduler; it's a real-time content distribution layer that respects platform-specific formatting, rate limits, and SEO metadata injection.

## Core Features (Hardcore)

### 1. **Atomic Multi-Platform Dispatch with Platform-Aware Formatting**
Each post is transformed on-the-fly to match the target platform's constraints—character limits, media types, link previews, and hashtag conventions. crosspostfast automatically truncates, reformats, or enriches content without breaking intent. Supports Markdown-to-HTML, auto-shortlink generation, and platform-specific thumbnail injection.

### 2. **SEO-Optimized Post Injection and Canonical Link Management**
Every syndicated post carries a hidden canonical reference back to your original source. crosspostfast injects `rel="canonical"` metadata, Open Graph tags, and Twitter Cards into platform-compatible formats. This prevents duplicate content penalties across search engines while maximizing cross-platform visibility.

### 3. **Idempotent Scheduling with Retry Logic and Rate-Limit Awareness**
Built-in exponential backoff and queue-based retry ensure zero post loss. crosspostfast maintains a local event log and syncs with platform webhooks to confirm delivery. If a platform rejects a post due to rate limits, the engine re-queues it with adaptive timing—no manual intervention required.

## How to Use (Professional Workflow)

### 1. **Initialize a Multi-Platform Post Configuration**
Create a `crosspost.yml` file in your project root:
```yaml
platforms:
  - twitter: { api_key, api_secret, access_token }
  - linkedin: { client_id, client_secret, redirect_uri }
  - mastodon: { instance_url, access_token }
defaults:
  canonical_url: "https://yourdomain.com/post/{slug}"
  retry_policy: { max_retries: 3, backoff_factor: 2.0 }
```
Run `crosspostfast init` to validate credentials and test connectivity. The CLI will output a health check summary for each platform.

### 2. **Define and Dispatch a Post with Platform-Specific Metadata**
Use the `crosspost` command with a JSON payload:
```bash
crosspostfast post --file post.json --schedule "2025-04-01T10:00:00Z"
```
Example `post.json`:
```json
{
  "title": "Scaling SaaS with crosspostfast",
  "body": "Learn how atomic cross-posting reduces SEO risk.",
  "media": ["thumbnail.png"],
  "platform_specific": {
    "twitter": { "thread": true, "poll": null },
    "linkedin": { "article_format": true, "tags": ["SaaS", "SEO"] }
  }
}
```
The engine will validate each platform’s constraints before queuing.

### 3. **Monitor Delivery and Audit Logs**
Track every post’s lifecycle:
```bash
crosspostfast logs --platform twitter --status delivered
```
Output includes platform post ID, timestamp, canonical URL, and retry history. For failed posts, run `crosspostfast retry --id <post_id>` to manually re-dispatch with updated rate-limit windows.

## Best Practices (Industry-Proven)

### 1. **Use Canonical Backlinks as a SEO Shield**
Always set a `canonical_url` in your post config. This tells search engines that your original source is the authoritative version, even if the same content appears on Twitter, LinkedIn, or Reddit. crosspostfast automatically appends a `?utm_source=platform` parameter for analytics tracking.

### 2. **Implement Platform-Specific Content Variants**
Don’t treat all platforms equally. For Twitter, keep threads under 280 characters per tweet. For LinkedIn, use rich article formatting with subheadings. crosspostfast supports conditional content blocks—define a `variant` field per platform to tailor tone, length, and media without duplicating your entire post.
