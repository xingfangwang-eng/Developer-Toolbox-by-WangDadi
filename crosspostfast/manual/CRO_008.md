## Detailed Content

### What is crosspostfast?

crosspostfast is a lean, API-first SaaS tool built for developers and content ops teams who need to blast content across Twitter, LinkedIn, Mastodon, and Bluesky in seconds—without bloated dashboards or vendor lock-in. It handles rate limits, retries, and idempotency out of the box.

### Core Features

- **Idempotent Queue Engine with Per-Platform Throttling**: Each post is pushed through a distributed queue that respects per-platform API rate limits (e.g., Twitter’s 300 tweets/3h, LinkedIn’s 100 shares/day). Failed posts auto-retry with exponential backoff, and dedup prevents double-posting even if the client crashes mid-flight.
- **Webhook-Driven Real-Time Sync for CI/CD Pipelines**: Trigger crossposting from GitHub Actions, Zapier, or any HTTP endpoint. Use our `POST /v1/publish` endpoint with a JSON payload—crosspostfast returns a `202 Accepted` immediately, then fires a callback to your webhook URL with the final status (published, rate-limited, or failed) within 30 seconds.
- **Dynamic Content Templating with per-Platform Rendering**: Define a single Markdown source, and crosspostfast auto-truncates for Twitter (280 chars), renders rich cards for LinkedIn, and strips unsupported HTML for Mastodon. Use `{{platform}}` and `{{character_count}}` variables in your templates for fine-grained control.

### How to Use

1. **Configure API Keys and Webhook Endpoint**  
   Head to your crosspostfast dashboard, generate a read-write API token under **Settings > API Tokens**. Then, in your CI/CD pipeline (e.g., GitHub Actions), set environment variables:  
   ```yaml
   - name: crosspost
     run: |
       curl -X POST https://api.crosspostfast.io/v1/publish \
         -H "Authorization: Bearer ${{ secrets.CROSSPOST_FAST_TOKEN }}" \
         -H "Content-Type: application/json" \
         -d '{
           "content": "Your post content here #crosspostfast",
           "platforms": ["twitter", "linkedin"],
           "webhook_url": "https://your-app.com/callback"
         }'
   ```  
   The callback will send a `POST` with `{ "post_id": "abc123", "status": "published", "platform": "twitter" }`.

2. **Create a Multi-Platform Template with Conditional Logic**  
   In the **Templates** tab, define a base template:  
   ```
   {{content}}  
   {{#if platform == "twitter"}}  
   (max 280 chars)  
   {{/if}}  
   {{#if platform == "linkedin"}}  
   Read the full article: {{url}}  
   {{/if}}  
   ```  
   Then, when publishing, pass `"template_id": "tpl_456"` in your API call. crosspostfast will compile the template per platform before sending.

3. **Monitor Queue Performance via Prometheus Metrics**  
   Enable the `/metrics` endpoint in your dashboard. Expose it to your monitoring stack (e.g., Grafana) to track:  
   - `crosspost_posts_total{status="published"}`  
   - `crosspost_queue_depth`  
   - `crosspost_rate_limit_remaining{platform="twitter"}`  
   Set up alerts for `crosspost_queue_depth > 100` to catch bottlenecks.

### Best Practices

- **Use Idempotency Keys for Retry Safety**: When calling the publish API, always include an `idempotency_key` (e.g., a SHA-256 of your content + timestamp). This prevents duplicate posts if your client retries due to a network timeout. crosspostfast deduplicates within a 24-hour window.
- **Prefer Batch Publishing Over Sequential Calls**: Instead of firing one POST per platform, batch all target platforms in a single request. crosspostfast parallelizes the dispatch internally, reducing total latency by ~40% compared to sequential calls. For high-volume ops (100+ posts/hour), use the `/v1/publish-batch` endpoint.
