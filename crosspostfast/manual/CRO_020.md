# crosspostfast - CRO_020

## Overview

crosspostfast is a headless social media syndication engine built for developers who need to blast content across multiple platforms without sacrificing per-platform optimization. It’s not a scheduler—it’s a programmatic cross-posting middleware that handles content transformation, rate-limit-aware dispatch, and platform-specific media adaptation.

**Primary Keywords:** Social media cross-platform posting, Multi-platform content publishing, Social media management tool  
**Secondary Keywords:** API-first social posting, multi-channel content distribution, cross-platform media syndication

---

## Hardcore Features

### 1. Platform-Aware Content Morphing Engine

crosspostfast automatically transforms your original post into platform-optimized variants. It strips unsupported markdown, truncates to character limits, reformats hashtags per platform syntax (e.g., `#Topic` on Twitter vs `[Topic]` on LinkedIn), and converts rich media (e.g., video → thumbnail + link for platforms that don’t support native video). You can define custom morphing rules via a YAML config file or hook into the transformation pipeline with middleware.

### 2. Rate-Limit-Aware Dispatch with Backpressure

Stop getting 429s. crosspostfast maintains a per-platform token bucket that respects API rate limits and dynamically throttles outgoing requests. It supports exponential backoff with jitter and queues failed posts into a dead-letter buffer for retry or manual inspection. The dispatch engine also respects platform-specific cooldowns (e.g., no more than 5 identical links per hour on Twitter).

### 3. Programmatic Cross-Posting API with Idempotency Keys

Each crosspost request accepts an `idempotency_key` header or body field. If a request fails mid-flight (e.g., network timeout after Facebook accepted but before Instagram did), you can safely retry the same key—crosspostfast will skip already-delivered platforms and resume the remaining ones. This enables reliable multi-platform syndication from CI/CD pipelines, webhooks, or cron jobs.

## How to Use

### Step 1: Configure Platform Profiles in `config/platforms.yaml`

Define each destination platform with its API credentials, content limits, and media rules:

```yaml
platforms:
  twitter:
    api_key: ${TWITTER_API_KEY}
    api_secret: ${TWITTER_API_SECRET}
    max_chars: 280
    media_max_count: 4
    media_formats: [jpg, png, gif, mp4]
    morph_rules:
      strip_markdown: true
      convert_links: short
  linkedin:
    api_key: ${LINKEDIN_API_KEY}
    max_chars: 3000
    media_max_count: 9
    media_formats: [jpg, png, pdf]
    morph_rules:
      wrap_hashtags_with_brackets: true
```

### Step 2: Write a Crosspost Script Using the SDK

```python
from crosspostfast import Client

client = Client(config_path="config/platforms.yaml")

post = {
    "text": "Check out our new release! 🚀 #crosspostfast #devtools",
    "media": ["screenshot.png", "demo.mp4"],
    "idempotency_key": "deploy-20250321-v2"
}

response = client.crosspost(
    platforms=["twitter", "linkedin", "mastodon"],
    content=post,
    dry_run=True  # validates without sending
)

print(response.delivery_status)
# {
#   "twitter": "queued",
#   "linkedin": "blocked: link limit exceeded",
#   "mastodon": "queued"
# }
```

### Step 3: Monitor and Retry Failed Deliveries

```bash
crosspostfast retry --batch-id 47a2b1c9 --platform linkedin --force-media-conversion
```

This re-dispatches only the failed platform for the given batch, re-applies media conversion (e.g., PDF to thumbnail + link), and respects the original idempotency key so no duplicates land on already successful platforms.

## Best Practices

### 1. Use Idempotency Keys for CI/CD Pipelines

When integrating crosspostfast into a deployment pipeline, always generate deterministic idempotency keys (e.g., `sha256(commit_sha + timestamp)`). This ensures that if your pipeline restarts mid-way or a webhook fires twice, your release notes don’t end up duplicated across platforms. It also lets you safely retry failed runs without manual deduplication.

### 2. Run `dry_run=True` Before Production Syndication

Always validate your content morphing rules before blasting to live accounts. crosspostfast’s dry-run mode returns a full report of how each platform would render the post, including truncated text, rejected media formats, and violated rate limits. This is especially critical when you’re syndicating programmatic content (e.g., changelogs, automated reports) that might contain unsupported characters or malformed links.
