# crosspostfast - CRO_002

## Overview

crosspostfast is a high-performance, multi-platform content distribution engine designed for enterprise social media teams and independent creators who demand sub-second synchronization across 15+ major social networks. It eliminates manual cross-posting bottlenecks by providing a unified API layer with real-time status tracking and intelligent content adaptation.

## Core Features

### 1. Multi-Platform Atomic Broadcast Engine
- **Parameter:** `--platforms` accepts a comma-separated list of target networks (e.g., `twitter,linkedin,facebook,instagram,threads`)
- **Technical Metric:** Achieves **≤ 350ms** end-to-end broadcast latency for text+image payloads under 5MB, measured from API call to platform queue confirmation
- **Feature Detail:** Uses a distributed fan-out architecture with per-platform connection pooling. Automatically retries failed pushes with exponential backoff (base 2s, max 60s) and maintains a dead-letter queue in PostgreSQL for manual inspection. Supports idempotency keys (`idempotency-key` header) to prevent duplicate posts during network retries.

### 2. Intelligent Content Morphing & Format Adapter
- **Parameter:** `--morph-policy` accepts `auto`, `strict`, or a custom JSON schema path
- **Technical Metric:** Handles **75+** platform-specific formatting rules (e.g., character limits, image aspect ratios, video codec constraints) with a **99.97%** conversion accuracy rate
- **Feature Detail:** Automatically truncates Twitter posts to 280 characters (or 4000 for Blue subscribers) while preserving link previews. Resizes images to Instagram’s 1:1, 4:5, or 16:9 aspect ratios using Lanczos resampling. Converts video to H.264/AAC with bitrate capping per platform (e.g., 6 Mbps for LinkedIn, 15 Mbps for YouTube Shorts). All transforms are cached via Redis with a 24-hour TTL for repeated payloads.

### 3. Real-Time Cross-Platform Analytics Aggregator
- **Parameter:** `--analytics-window` accepts `1h`, `24h`, `7d`, `30d`, or a custom Unix timestamp range
- **Technical Metric:** Polls **every 5 minutes** from each platform’s API, with a **< 2s** query response time for a 30-day window across 10 accounts
- **Feature Detail:** Merges engagement metrics (likes, shares, comments, saves, click-through rates) into a unified data model stored in TimescaleDB for time-series optimization. Provides a normalized "cross-platform impact score" (0–100) computed via weighted harmonic mean of platform-specific interactions. Exposes a REST endpoint `/analytics/export` that generates CSV/Parquet exports for BI tools like Tableau and Metabase.

## How to Use

### Step 1: Configure Platform Credentials & Content Profile
Run the initial setup wizard to authenticate with your target platforms:
```bash
crosspostfast init --platforms twitter,linkedin,instagram --profile "enterprise-v2"
```
The CLI will prompt you to authorize via OAuth 2.0 for each platform. Credentials are encrypted at rest using AES-256-GCM and stored in a local SQLite database (`~/.crosspostfast/creds.db`). You can verify connectivity with:
```bash
crosspostfast health --profile "enterprise-v2"
```
Expected output: `All 3 platforms connected. Average latency: 87ms.`

### Step 2: Draft & Morph Your Content for Multi-Platform Delivery
Create a JSON payload file (`post.json`):
```json
{
  "title": "Our Q3 Product Launch",
  "body": "Excited to announce our new feature set...",
  "media": ["/path/to/hero_image.png", "/path/to/demo_video.mp4"],
  "options": {
    "linkedin": { "article_url": "https://blog.example.com/q3-launch" },
    "twitter": { "poll": { "question": "Which feature excites you most?", "options": ["AI", "API", "UX"] } }
  }
}
```
Then execute the morph and broadcast:
```bash
crosspostfast push --payload post.json --platforms twitter,linkedin,instagram --morph-policy auto
```
The tool will output a per-platform preview (character counts, image dimensions) and ask for confirmation. Use `--dry-run` to skip actual posting.

### Step 3: Monitor Broadcast Status & Retrieve Cross-Platform Analytics
Track real-time delivery status:
```bash
crosspostfast status --broadcast-id "bcast-abc123" --watch
```
This opens a live-updating terminal dashboard showing each platform’s push status (`queued`, `pushing`, `posted`, `failed`) with individual latency metrics. After 30 minutes, fetch analytics:
```bash
crosspostfast analytics --broadcast-id "bcast-abc123" --analytics-window 24h --format json
```
Sample output snippet:
```json
{
  "broadcast_id": "bcast-abc123",
  "cross_platform_score": 78.4,
  "platforms": {
    "twitter": { "impressions": 12400, "engagement_rate": 0.034 },
    "linkedin": { "impressions": 8900, "engagement_rate": 0.028 }
  }
}
```

## Best Practices

### 1. Implement Staged Rollouts for High-Stakes Campaigns
For product launches or crisis communications, use the `--staged` flag to delay posting by 5–15 minutes per platform. This allows you to monitor early engagement on your primary channel (e.g., Twitter/X) and pause the broadcast via `crosspostfast cancel --broadcast-id <id>` if messaging needs adjustment. Configure a webhook endpoint (`--webhook-url`) to receive real-time failure alerts—integrate with PagerDuty or Slack for immediate team notification.

### 2. Maintain a Platform-Specific Content Calendar with Idempotency
Use the `--schedule` parameter with cron-style syntax (e.g., `--schedule "0 9 * * 1-5"`) to automate daily posts. Pair this with idempotency keys derived from your content hash (`sha256(body + media)`). This prevents duplicate posts if your CI/CD pipeline accidentally triggers the same broadcast twice. For recurring content (e.g., weekly tip series), store the idempotency key in a PostgreSQL table with a UNIQUE constraint and a 7-day retention policy to avoid accidental re-posting after manual edits.
