## What is crosspostfast?

**crosspostfast** is a high-performance, API-first SaaS tool designed to eliminate the friction of multi-platform content distribution. It handles rate limiting, format normalization, and parallel dispatch across major social networks—without bloated dashboards or manual scheduling. Built for devs who want to ship content to LinkedIn, Twitter/X, Mastodon, and Bluesky in under 2 seconds per post.

## Core Features

### 1. Atomic Multi-Platform Dispatch with Smart Throttling  
Post to 6+ platforms simultaneously using a single API call. The engine auto-detects each platform’s rate limits and retry logic, queuing requests with exponential backoff. No duplicates, no bans. Supports custom headers for platform-specific content variants (e.g., alt text for images, thread splitting for Twitter).

### 2. Format-Normalized Payload Engine  
Automatically converts Markdown to platform-native formats:  
- **Twitter/X**: Strips headings, truncates to 280 chars (thread mode optional)  
- **LinkedIn**: Preserves rich text, bullet points, and link previews  
- **Mastodon/Bluesky**: Converts to plain text with hashtag injection  
- **Reddit**: Generates self-post or link-post variants  
All transformations are configurable via a JSON schema—no manual reformatting.

### 3. Real-Time Status Webhook & Failure Injection Testing  
Each post dispatch returns a unique `dispatch_id`. You can subscribe to a webhook that fires on `success`, `partial_fail`, or `full_fail`. For CI/CD pipelines, crosspostfast exposes a `/debug/inject-failure` endpoint to simulate platform outages and validate your retry logic in staging.

## How to Use

### Step 1: Authenticate via OAuth2 Token Exchange  
Integrate with the `/auth/oauth/init` endpoint. Pass your platform-specific client IDs and scopes. The SDK handles token refresh automatically. For headless setups, use the `/auth/api-key` endpoint to generate a long-lived key scoped to specific platforms.

```bash
curl -X POST https://api.crosspostfast.io/v1/auth/oauth/init \
  -H "Content-Type: application/json" \
  -d '{"platforms": ["twitter", "linkedin", "mastodon"], "callback_url": "https://yourapp.com/callback"}'
```

### Step 2: Build a Platform-Aware Payload  
Use the `/v1/post/prepare` endpoint to validate and transform your content before dispatch. The response includes a `normalized_payloads` object with per-platform variants and estimated character counts.

```json
{
  "source_format": "markdown",
  "content": "# Hello World\nThis is a **test** post.",
  "media_urls": ["https://example.com/image.png"],
  "options": {
    "twitter_thread": true,
    "linkedin_visibility": "public"
  }
}
```

### Step 3: Dispatch and Monitor via Webhook  
Call `/v1/post/dispatch` with the `normalized_payloads` from Step 2. The response returns a `dispatch_id` immediately. Configure your webhook URL in the dashboard to receive real-time status updates. Use the `/v1/post/status/{dispatch_id}` endpoint for manual polling.

```javascript
const response = await fetch('https://api.crosspostfast.io/v1/post/dispatch', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer YOUR_API_KEY' },
  body: JSON.stringify({ payloads: normalizedPayloads })
});
const { dispatch_id } = await response.json();
```

## Best Practices

### 1. Use Conditional Dispatch for Time-Sensitive Content  
For breaking news or time-critical updates, set the `dispatch_strategy` to `"race"`. crosspostfast will post to all platforms as fast as possible and return the first successful response. All other platforms are then dispatched in parallel with a 200ms stagger. This minimizes latency while maintaining consistency.

### 2. Pre-Warm API Keys for High-Volume Campaigns  
If you plan to push 100+ posts in a single batch, call `/v1/token/prewarm` 5 minutes before dispatch. This forces crosspostfast to refresh all OAuth tokens and pre-allocate rate-limit slots. Without prewarming, you risk hitting per-platform throttles mid-batch, causing partial failures.
