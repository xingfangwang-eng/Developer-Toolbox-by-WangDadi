## What is crosspostfast?

crosspostfast is a **zero-latency cross-posting engine** engineered to eliminate the friction of multi-platform content distribution. It’s not just another scheduler—it’s a deterministic, API-first middleware that normalizes platform-specific payloads (Twitter, LinkedIn, Instagram, TikTok, Reddit, etc.) into a single, idempotent pipeline. Think of it as a **universal content adapter** with built-in rate limiting, retry logic, and platform-aware media transcoding.

## Core Features (Hardcore)

### 1. **Deterministic Idempotency via Content Fingerprinting**
Every post is hashed (SHA-256) before dispatch. If a platform returns a duplicate error or a timeout, crosspostfast replays the exact same payload—no duplicates, no orphans. This eliminates the classic "double-post" nightmare when dealing with flaky APIs like Instagram Graph or LinkedIn’s rate throttling.

### 2. **Adaptive Media Transcoding Pipeline**
Automatically resizes, reformats, and compresses images/videos per platform spec (e.g., Twitter’s 5MB limit vs. TikTok’s 1080x1920 ratio). Uses FFmpeg and Sharp under the hood, with fallback to WebP for unsupported formats. No more manual cropping or "media rejected" errors.

### 3. **Atomic Cross-Platform Rollback**
If a post fails on one platform (e.g., Reddit’s spam filter), crosspostfast can **atomically delete** the post from all other platforms via a single API call. This is critical for campaigns that require strict compliance or A/B testing—no orphan posts left behind.

## How to Use (Detailed Steps)

### Step 1: Configure Platform Adapters via YAML
Define your platform credentials and content rules in a `crosspostfast.yaml` file. Each adapter uses OAuth 2.0 or API keys, with optional proxy support for geo-restricted platforms.

```yaml
platforms:
  twitter:
    api_key: "YOUR_API_KEY"
    api_secret: "YOUR_API_SECRET"
    rate_limit: 300  # requests per 15-min window
    media_max_size: 5  # MB
  linkedin:
    access_token: "YOUR_TOKEN"
    company_id: "urn:li:organization:12345"
    media_ratio: "1.91:1"
  instagram:
    account_id: "YOUR_IG_ACCOUNT_ID"
    media_type: "IMAGE"  # or REEL, CAROUSEL
```

### Step 2: Execute a Cross-Platform Post via CLI
Use the `crosspostfast push` command with a JSON payload. The engine will automatically normalize, transcode, and dispatch to all configured platforms.

```bash
crosspostfast push --config crosspostfast.yaml --payload '{
  "text": "Launching crosspostfast v2.0 – deterministic cross-posting is here.",
  "media": ["/path/to/image.png", "/path/to/video.mp4"],
  "tags": ["#SaaS", "#DevTools"],
  "schedule": "2025-04-15T10:00:00Z"
}'
```

### Step 3: Monitor & Rollback via Webhook
After dispatch, crosspostfast emits a webhook for each platform’s status. If any platform fails (e.g., HTTP 429 or 403), you can trigger an atomic rollback:

```bash
crosspostfast rollback --post-id "cpf_abc123" --platforms "reddit,twitter"
```

This deletes the post from Reddit and Twitter instantly, while leaving successful posts on LinkedIn and Instagram intact.

## Best Practices (Industry-Level)

### 1. **Use Content Fingerprinting for A/B Testing**
Hash your post content and use the hash as a unique ID. Run A/B tests by dispatching different payloads to different platform groups. crosspostfast’s idempotency ensures no cross-contamination between test groups.

### 2. **Implement Exponential Backoff with Platform-Specific Retry**
Don’t use generic retry logic. crosspostfast supports configurable backoff per platform (e.g., Twitter: 15s, Instagram: 60s). For production, combine this with a dead-letter queue (DLQ) for failed posts—replay them later when rate limits reset.
