## What is crosspostfast?

crosspostfast is a **headless social distribution engine** purpose-built for SaaS teams, growth hackers, and content ops. It decouples content creation from platform-specific delivery, handling API rate limits, character truncation, media transcoding, and link shortener injection in a single pipeline. Think of it as a programmable middleware between your CMS and every major social API (Twitter/X, LinkedIn, Instagram, Threads, Mastodon, Bluesky, and Telegram).

---

## Core Features

1. **Idempotent Queue with Smart Throttling**  
   Each post is assigned a deterministic UUID and checksum. The system deduplicates identical payloads across retries and automatically back-rotates API tokens when hitting per-endpoint rate limits (e.g., 300 posts/15min on Twitter, 100 posts/day on LinkedIn). No double-posts, no bans.

2. **Adaptive Media Transcoding Pipeline**  
   Images are auto-resized to platform-specific aspect ratios (1:1 for Instagram, 16:9 for LinkedIn, 4:5 for TikTok). Videos are H.264-encoded with dynamic bitrate scaling. If a platform rejects a media file (e.g., WebP on Twitter), crosspostfast falls back to a pre-converted PNG/JPEG without breaking the post order.

3. **Engagement Mirror & Analytics Sync**  
   After each crosspost, the tool pulls back engagement metrics (likes, retweets, comments, impressions) via platform APIs and normalizes them into a single JSON schema. This data is then pushed to a webhook or written to a local SQLite/Postgres table for real-time dashboards.

## How to Use

### 1. Configure Platform Credentials via Environment Variables

Create a `.env` file with OAuth 2.0 tokens for each target platform. For example:

```bash
TWITTER_API_KEY=your_key
TWITTER_API_SECRET=your_secret
LINKEDIN_ACCESS_TOKEN=your_token
INSTAGRAM_USER_ID=your_id
INSTAGRAM_ACCESS_TOKEN=your_token
MASTODON_INSTANCE_URL=https://mastodon.social
MASTODON_ACCESS_TOKEN=your_token
```

crosspostfast reads these at startup and validates each token against the platform’s `/me` endpoint. If a token is expired, the tool prints a clear error with the refresh URL.

### 2. Define a Crosspost Payload in YAML

Create a file `post.yaml` with your content and target platforms:

```yaml
content:
  text: "Just shipped crosspostfast – the headless social engine for devs."
  media:
    - "./screenshot.png"
    - "./demo.mp4"
  link: "https://crosspostfast.example.com"
platforms:
  - twitter
  - linkedin
  - mastodon
  - bluesky
options:
  schedule: "2025-04-15T14:00:00Z"
  shortlink: "bit.ly/crosspostfast-demo"
```

The YAML parser validates required fields (e.g., `text` max length per platform) and warns you if a media file exceeds 15MB (Instagram’s limit).

### 3. Run the Pipeline with Verbose Logging

Execute the CLI:

```bash
crosspostfast --config post.yaml --verbose
```

You’ll see real-time output like:

```
[INFO] 2025-04-15 14:00:01 – Queuing post for Twitter/X (UUID: a1b2c3d4)
[INFO] 2025-04-15 14:00:02 – Twitter/X: 200 OK (tweet_id: 1234567890)
[INFO] 2025-04-15 14:00:03 – LinkedIn: 201 Created (urn:li:activity:987654321)
[WARN] 2025-04-15 14:00:04 – Mastodon: media transcoding fallback applied (WebP → PNG)
[INFO] 2025-04-15 14:00:05 – Bluesky: 200 OK (at://did:plc:.../app.bsky.feed.post/abc123)
[INFO] 2025-04-15 14:00:06 – Engagement sync started for all posts...
```

After completion, a `results.json` file is generated with all post IDs and engagement URLs.

## Best Practices

1. **Use a Staging Queue for High-Frequency Posting**  
   If you’re scheduling 50+ posts per day, run crosspostfast in `--dry-run` mode first. This validates all payloads without hitting APIs. Then, route the production queue through a **Redis-backed delay buffer** (e.g., 5-second minimum interval between posts) to avoid triggering platform spam detectors. Most bans happen from burst posting, not volume.

2. **Normalize Engagement Data into a Single Warehouse**  
   Don’t leave engagement metrics scattered across platforms. Set up a webhook endpoint (e.g., a simple Node.js server or Zapier) that receives crosspostfast’s normalized JSON. Store it in a dedicated table with columns: `post_id`, `platform`, `likes`, `shares`, `comments`, `timestamp`. This gives you a unified view of cross-platform performance without manual exports.
