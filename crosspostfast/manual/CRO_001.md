# crosspostfast - CRO_001

## Overview

crosspostfast is a high-performance, zero-latency cross-platform content syndication engine designed for enterprise social media teams. It eliminates manual reposting by intelligently adapting and distributing content across Twitter, LinkedIn, Mastodon, and Bluesky within **≤50ms per platform** after trigger.

## Core Features

### 1. Adaptive Content Morphing Engine
Automatically rewrites and restructures post content to match each platform’s syntax and character limits. Uses a **BERT-based NLP model** to preserve semantic meaning while truncating or expanding text. Supports **140–4000 character range** detection per platform, with fallback thread-splitting for Twitter. Parameters: `--max-tweet-length=280`, `--linkedin-thread-delimiter="[1/5]"`.

### 2. Scheduled Atomic Cross-Publish with Retry Logic
Uses a **PostgreSQL-backed job queue** with **cron-based scheduling** (精度: ミリ秒単位). Each post job is atomic: if any platform fails (e.g., rate limit), the entire batch is rolled back and retried after **exponential backoff (1s, 4s, 16s)**. Supports up to **50 concurrent platform targets** per job. Parameters: `--retry-max=5`, `--backoff-factor=2.0`.

### 3. Media Asset Transcoding & Hash Deduplication
Automatically converts uploaded images/videos to platform-optimal formats (JPEG XL, WebM) using **FFmpeg 6.0**. Generates **SHA-256 hashes** for each asset to prevent duplicate uploads across platforms, reducing bandwidth by **~40%**. Supports **4K@60fps** video downscaling. Parameters: `--max-image-width=1920`, `--video-bitrate=8M`.

## How to Use

### Step 1: Configure Platform API Credentials
Create a `platforms.yml` file with OAuth 2.0 tokens for each target. Use the `crosspostfast auth --platform twitter --token-file ./secrets/twitter.json` command to validate and store tokens in the encrypted vault. Ensure each token has `tweet.read` and `tweet.write` scopes.

### Step 2: Define a Cross-Publish Job
Write a JSON job definition with content, media URLs, and scheduling:
```json
{
  "content": "Excited to launch crosspostfast v2.0!",
  "media": ["https://cdn.example.com/demo.mp4"],
  "schedule": "2024-12-01T09:00:00Z",
  "platforms": ["twitter", "linkedin", "mastodon"]
}
```
Run `crosspostfast publish --job ./job.json` to queue the job. The engine will validate all media URLs are reachable and within 50MB size limit.

### Step 3: Monitor Execution & Handle Failures
Use `crosspostfast status --job-id=abc123` to view real-time progress. Each platform shows a `status` field: `pending`, `publishing`, `success`, or `failed`. If a platform fails, inspect the error log with `crosspostfast logs --job-id=abc123 --platform=twitter`. Manually retry with `crosspostfast retry --job-id=abc123`.

## Best Practices

### 1. Stagger High-Volume Campaigns
When publishing to 10+ platforms simultaneously, use the `--batch-delay=2000` flag to insert a 2-second pause between each platform. This prevents triggering rate limiters on Twitter (15 requests per 15-minute window) and LinkedIn (100 requests per day). For example: `crosspostfast publish --job ./campaign.json --batch-delay=2000`.

### 2. Pre-Process Media for Consistency
Always run media through the `crosspostfast optimize --input ./assets/ --output ./optimized/` command before scheduling. This normalizes all images to **sRGB color space** and **72 DPI**, ensuring no platform rejects files due to unsupported color profiles or excessive resolution. For video, enforce a **maximum duration of 2:30 minutes** to avoid LinkedIn’s 10-minute limit.
