# crosspostfast - CRO_022

## Überblick

**crosspostfast** ist kein weiteres Social-Media-Tool – es ist eine **asynchrone Publishing-Pipeline**, die Content über API-Gateways parallel an 10+ Plattformen ausliefert. Optimiert für Developer, die ihre Reichweite maximieren wollen, ohne manuelle Workarounds oder Browser-Automation. Keywords: Multi-Plattform-Publishing, Cross-Posting API, Social-Media-Automation.

## Hardcore-Funktionen

### 1. **Parallel-Execution Engine mit Retry-Logik**
Jeder Post wird als unabhängiger Goroutine-Job behandelt. Fehlgeschlagene Zustellungen (z. B. 429 Rate-Limits) werden automatisch mit exponentiellem Backoff wiederholt – konfigurierbar bis zu 5 Retries. Kein Single-Point-of-Failure, selbst wenn Twitter API down ist.

### 2. **Content-Normalizer für plattformspezifische Syntax**
Automatische Konvertierung von Markdown in plattformnative Formate:  
- Twitter: Kürzung auf 280 Zeichen + Link-Shortening  
- LinkedIn: Erhalt von Headlines und Bullet Points  
- Instagram: Extraktion von Hashtags aus dem Text in den Caption-Bereich  
- Reddit: Flair- und Subreddit-Tagging basierend auf Keywords  

### 3. **Scheduling mit Cron-Expression-Unterstützung**
Nicht nur „Post jetzt“ oder „Post später“ – du definierst präzise Zeitfenster via Cron (z. B. `0 9 * * 1-5` für werktags 9:00 UTC). Das Tool verteilt Posts automatisch auf die besten Zeitfenster pro Plattform, basierend auf historischen Engagement-Daten.

## Detaillierte Anleitung: Post auf 5 Plattformen in 3 Minuten

### 1. **API-Key-Konfiguration via YAML**
Erstelle eine `config.yaml` im Projekt-Root. Definiere für jede Plattform die Credentials und Rate-Limits:

```yaml
platforms:
  twitter:
    api_key: "your_twitter_api_key"
    api_secret: "your_twitter_api_secret"
    rate_limit: 300  # requests per 15-min window
  linkedin:
    access_token: "your_linkedin_token"
    rate_limit: 100
```

### 2. **Content-Erstellung mit Frontmatter**
Erstelle einen Markdown-Post mit Metadaten:

```markdown
---
title: "Neues Release v2.3.0"
platforms: [twitter, linkedin, reddit, devto, mastodon]
schedule: "0 10 * * *"  # täglich 10:00 UTC
tags: [release, update, dev]
---
Heute veröffentlichen wir **v2.3.0** mit verbesserter API-Performance.  
- 30% schnellere Response-Zeiten  
- Neue Endpoints für Batch-Processing  
- Bugfixes im Auth-Flow
```

### 3. **Asynchrones Publishing starten**
Führe den Befehl aus:

```bash
crosspostfast publish --file ./posts/release-v2-3-0.md --dry-run
```

Der `--dry-run`-Flag zeigt dir, wie der Post auf jeder Plattform aussehen wird (inkl. Längenprüfung und Formatierung). Ohne `--dry-run` wird der Post sofort in die Pipeline gegeben. Überwache den Status mit `crosspostfast status --job-id <id>`.

## Best Practices für Developer

### 1. **Rate-Limits vorausberechnen**
Nutze die `--simulate`-Option, bevor du einen Massen-Post planst. Das Tool berechnet, ob deine API-Keys die Last in einem 15-Minuten-Fenster stemmen. Bei Überschreitung schlägt es automatisch Splitting vor – z. B. 2 Posts pro Minute statt 10 auf einmal.

### 2. **Content-Versionierung via Git**
Speichere alle Posts in einem dedizierten `posts/`-Verzeichnis mit Datums- und Plattform-Tags. So hast du ein vollständiges Audit-Log, kannst ältere Posts neu schedulen (z. B. für Evergreen-Content) und via CI/CD automatisch deployen. Ein `git commit -m "Scheduled Q4 announcements"` ist dein Deployment-Trigger.
