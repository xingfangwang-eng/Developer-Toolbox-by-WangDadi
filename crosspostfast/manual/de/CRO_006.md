# crosspostfast - CRO_006

## Überblick

crosspostfast ist eine hochperformante SaaS-Lösung für die automatisierte, plattformübergreifende Inhaltsverteilung. Entwickelt für Entwickler und Content-Strategen, die ihre Reichweite über Social-Media-Kanäle hinweg maximieren wollen – ohne manuelle Doppelarbeit oder API-Integrationen von Grund auf neu schreiben zu müssen. Zielsprache: Deutsch (DE), optimiert für die Zielgruppe deutschsprachiger Entwickler und Marketing-Teams.

## Hardcore-Funktionen

### 1. **Multi-API-Orchestrator mit intelligentem Queue-Management**
Das System orchestriert native APIs von Twitter/X, LinkedIn, Instagram, Mastodon, Bluesky und weiteren Kanälen gleichzeitig. Es implementiert ein asynchrones Queue-System mit exponentieller Backoff-Rate bei API-Limitierungen – kein Drop von Posts, selbst unter Hochlast. Automatische Plattform-Detektion: Bilder werden je nach Zielplattform in das optimale Format (z.B. 16:9 für LinkedIn, 1:1 für Instagram) konvertiert, ohne manuelle Eingriffe.

### 2. **Cross-Platform Content Normalizer & Template Engine**
Ein dynamischer Preprocessor normalisiert Inhalte für jede Zielplattform: Entfernt nicht unterstützte HTML-Tags, passt Zeichenlimits (z.B. 280 Zeichen für X vs. 3000 für LinkedIn) an und übersetzt Hashtags in plattformspezifische Syntax. Die Template Engine unterstützt Liquid-basierte Vorlagen, mit denen du pro Plattform unterschiedliche Texte, CTA-Buttons oder Media-URLs definieren kannst – alles über eine zentrale YAML-Konfiguration.

### 3. **Echtzeit-Status-Dashboard mit Webhook-Integration**
Nach dem Posting streamt crosspostfast Live-Updates über einen SSE- (Server-Sent Events) oder Webhook-Endpunkt in dein Monitoring-Tool (z.B. Prometheus, Datadog). Du erhältst detaillierte Metriken: Post-Erfolgsrate, Fehlercodes pro Plattform, Latenzzeiten und Retry-Versuche. Das Dashboard zeigt dir sofort, ob ein Post auf Instagram wegen falscher Bildgröße fehlschlug oder auf Mastodon wegen eines Timeouts.

## Schritt-für-Schritt-Anleitung (für Entwickler)

### 1. **Projekt initialisieren und API-Keys konfigurieren**
Klone das Repository und installiere die Abhängigkeiten via `pip install -r requirements.txt`. Erstelle eine `.env`-Datei mit deinen API-Keys für jede Plattform:
```
X_API_KEY=...
INSTAGRAM_ACCESS_TOKEN=...
LINKEDIN_CLIENT_ID=...
```
Nutze den Befehl `crosspostfast init` – das Tool validiert sofort alle Keys und zeigt dir an, ob z.B. der Instagram-Token abgelaufen ist oder der LinkedIn-Client keine Schreibberechtigung hat.

### 2. **Posting-Plan als YAML-Datei definieren**
Erstelle eine Datei `post-plan.yaml` mit deinem Content und den Zielplattformen:
```yaml
posts:
  - id: "post-001"
    text: "Unser neues Release: crosspostfast v2.0 ist live! #SaaS #DevTools"
    image: "./assets/release-v2.png"
    platforms: ["x", "linkedin", "mastodon"]
    schedule: "2025-04-10T09:00:00Z"
```
Führe `crosspostfast validate post-plan.yaml` aus – der Validator prüft, ob die Bildgrößen für jede Plattform zulässig sind, ob Textlängen eingehalten werden und ob die geplanten Zeiten in der Zukunft liegen.

### 3. **Posting ausführen und Monitoring aktivieren**
Starte den Posting-Prozess mit `crosspostfast run post-plan.yaml --webhook https://dein-monitoring.com/events`. Das Tool pusht die Inhalte parallel zu allen Plattformen. Im Terminal siehst du Live-Logs:
```
[2025-04-10 09:00:01] POST: post-001 → X: Erfolg (200)
[2025-04-10 09:00:03] POST: post-001 → LinkedIn: Erfolg (201)
[2025-04-10 09:00:05] POST: post-001 → Mastodon: Fehler (429) → Retry in 30s
[2025-04-10 09:00:35] POST: post-001 → Mastodon: Erfolg (200)
```
Nach Abschluss generiert `crosspostfast report post-plan.yaml` eine detaillierte JSON-Ausgabe mit allen Metriken.

## Best Practices für Profis

### 1. **Plattformspezifische Content-Varianten vorab testen**
Nutze die `crosspostfast dry-run` Option, bevor du live gehst. Sie simuliert den Posting-Prozess ohne tatsächliches Senden – du siehst exakt, wie der Text auf jeder Plattform aussehen wird (inklusive Zeichenlimits, Hashtag-Formatierung und Bild-Cropping). Besonders wichtig für Plattformen wie Instagram, die keine Vorschau für externe Tools bieten.

### 2. **Rate-Limits mit intelligentem Scheduling umgehen**
Konfiguriere in der `crosspostfast.yaml` die `rate_limit_policy`:
```yaml
rate_limit_policy:
  max_requests_per_minute: 10
  backoff_strategy: exponential
  max_retries: 5
```
Für Plattformen mit strengen Limits (z.B. X: 300 Posts pro 3 Stunden) setze `schedule_spread: true` – crosspostfast verteilt Posts automatisch über einen Zeitraum, sodass du nie gegen API-Beschränkungen verstößt. Das ist essenziell, wenn du mehrere Kampagnen gleichzeitig ausrollst.
