## Detaillierter Inhalt

### Was ist crosspostfast?

crosspostfast ist ein **professionelles SaaS-Tool**, das die Herausforderungen des **Multi-Channel-Content-Managements** löst: kein manuelles Posten, keine Formatierungsinkonsistenzen, keine verlorenen Metadaten. Es synchronisiert Beiträge in Echtzeit über Plattformen wie LinkedIn, Twitter/X, Mastodon und Bluesky – mit **einem API-Call**.

### Kernfunktionen (Hardcore-Features)

1.  **Deklaratives Cross-Post-Schema (YAML/JSON)**: Definiere dein Posting-Modell zentral. Ein `post.yaml` definiert Variablen wie `{title}`, `{platform_specific_hashtags}` und `{media_attachment}`. crosspostfast rendert automatisch plattformspezifische Formate (z.B. Threads für X, Karussells für LinkedIn).
2.  **Atomic Broadcast Queue mit Rollback**: Jeder Post wird als atomare Transaktion behandelt. Schlägt ein API-Call auf einer Plattform fehl (z.B. Rate-Limit bei Twitter/X), rollt crosspostfast **alle bereits gesendeten Posts** auf anderen Plattformen zurück – inklusive Löschung der Media-Assets. Keine fragmentierten Kampagnen.
3.  **Zero-Downtime Multi-Account Rotation**: Für Power-User: Definiere mehrere API-Keys pro Plattform. crosspostfast rotiert automatisch zwischen Accounts, um Rate-Limits zu umgehen. Bei 5 Twitter/X-Accounts und 3 LinkedIn-Seiten kannst du **300 Posts/Minute** ohne Drosselung pushen.

### So verwenden Sie es (Detaillierte Schritte)

#### Schritt 1: Konfiguration des `crosspost.yaml`-Manifests

Erstelle im Stammverzeichnis deines Projekts eine `crosspost.yaml`-Datei. Definiere deine Plattformen und das Posting-Schema.

```yaml
platforms:
  - name: linkedin
    api_key: ${LINKEDIN_API_KEY}
    account_id: "urn:li:organization:123456"
  - name: twitter
    api_key: ${TWITTER_API_KEY}
    account_id: "1234567890"
    
post:
  title: "Mein neuer Blogbeitrag über SaaS"
  body: "Hier ist der Inhalt. **Fett** und *kursiv* werden automatisch konvertiert."
  media:
    - type: image
      url: "https://example.com/image.png"
      alt: "SaaS-Diagramm"
```

**Wichtig**: Nutze Umgebungsvariablen (`${...}`), um Secrets aus deinem `.env`-File zu laden. crosspostfast validiert das Schema beim Start und gibt Fehler aus, wenn ein API-Key fehlt.

#### Schritt 2: Ausführen eines synchronen Cross-Posts

Führe den Befehl im Terminal aus:

```bash
crosspostfast publish --config crosspost.yaml
```

Das Tool:
1.  Validiert die API-Keys gegen die Plattformen (Ping-Test).
2.  Rendert den `body` für jede Plattform (z.B. entfernt Markdown für Twitter/X, behält es für LinkedIn).
3.  Sendet den Post **atomar**: Zuerst an LinkedIn, dann an Twitter/X.
4.  Gibt bei Erfolg eine JSON-Ausgabe mit den Post-IDs und URLs aus.

**Beispiel-Output**:
```json
{
  "status": "success",
  "posts": [
    {"platform": "linkedin", "post_id": "abc123", "url": "https://linkedin.com/feed/update/..."},
    {"platform": "twitter", "post_id": "def456", "url": "https://x.com/status/..."}
  ]
}
```

#### Schritt 3: Automatisierung mit Cron-Job (CI/CD-Integration)

Erstelle eine `cron`-Regel auf deinem Server, die crosspostfast jede Stunde ausführt. Beispiel für `crontab`:

```bash
0 * * * * /usr/local/bin/crosspostfast publish --config /path/to/crosspost.yaml >> /var/log/crosspostfast.log 2>&1
```

**Erweiterte Option**: Nutze den `--dry-run`-Flag für Tests:

```bash
crosspostfast publish --config crosspost.yaml --dry-run
```

Das simuliert den gesamten Prozess ohne tatsächliches Posten. Perfekt für CI/CD-Pipelines (z.B. GitHub Actions).

### Best Practices für die Industrie

1.  **Implementiere eine Posting-Strategie mit Time-Affinity**: crosspostfast erlaubt das Setzen von `post_at`-Timestamps. Analysiere die Aktivitätsdaten deiner Zielgruppe (z.B. LinkedIn: 8-10 Uhr MEZ, Twitter/X: 12-14 Uhr MEZ) und definiere eine **Posting-Rotation**. Nutze `crosspostfast schedule --config crosspost.yaml --timezone "Europe/Berlin"` für zeitgesteuerte Veröffentlichungen. Das erhöht die Engagement-Rate um bis zu 40 % (laut Buffer-Studie 2024).
2.  **Nutze das Atomic-Rollback-Feature für A/B-Tests**: Erstelle zwei `crosspost.yaml`-Konfigurationen (z.B. `variant_a.yaml` und `variant_b.yaml`). Poste beide gleichzeitig. Wenn eine Variante auf einer Plattform fehlschlägt (z.B. wegen eines Formatierungsfehlers), rollt crosspostfast die gesamte Kampagne zurück. So vermeidest du **halbe A/B-Tests** und erhältst saubere Daten. Führe den Test dann mit einer korrigierten Konfiguration erneut aus.
