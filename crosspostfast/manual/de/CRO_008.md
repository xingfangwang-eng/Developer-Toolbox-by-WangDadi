## Detaillierter Inhalt

### Was ist crosspostfast?

crosspostfast ist ein professionelles, quelloffenes Tool für die plattformübergreifende Content-Synchronisation. Es löst das Kernproblem fragmentierter Social-Media-Workflows: Statt manuelles Posten auf 5+ Plattformen (LinkedIn, X, Mastodon, Threads, Bluesky) erlaubt es eine deklarative, YAML-basierte Konfiguration mit automatischer Formatierung, Bildoptimierung und zeitgesteuerter Auslieferung.

### Kernfunktionen

1. **Multi-Platform Normalizer Engine**  
   Automatische Umwandlung von Markdown-Content in plattformspezifische Formate: z.B. Threads für X (280 Zeichen), Rich-Text für LinkedIn, Alt-Text für Mastodon. Inklusive intelligenter Hashtag-Extraktion und Emoji-Ersetzung auf Basis der Ziel-API.

2. **Inhaltsdifferenzierung per Kontext-Mapping**  
   Definiere für jede Plattform eigene Content-Varianten (z.B. `de_linkedin`, `en_mastodon`) innerhalb einer YAML-Datei. crosspostfast merged automatisch globale und plattformspezifische Felder – ohne Duplikate im Repository.

3. **CI/CD-Integration mit GitHub Actions**  
   Build- und Deploy-Pipeline, die bei jedem `git push` auf `main` die Konfiguration validiert, API-Rate-Limits prüft und Content asynchron ausliefert. Inklusive Dry-Run-Modus für Previews.

### So verwenden Sie es

1. **Konfiguration via YAML-Schema**  
   Erstelle eine `crosspostfast.yaml` im Repository-Stamm. Definiere Plattformen, API-Keys (via Secrets) und Content-Templates. Beispiel:
   ```yaml
   platforms:
     linkedin: { variant: de, api_key: ${{ secrets.LINKEDIN_KEY }} }
     mastodon: { variant: en, api_key: ${{ secrets.MASTODON_KEY }} }
   content:
     - file: posts/2025-04-release.md
       schedule: '2025-04-15T10:00:00Z'
   ```

2. **Lokales Preflight-Testing**  
   Führe `npx crosspostfast validate` aus. Das Tool prüft auf Syntax-Fehler, API-Erreichbarkeit und Rate Limits. Erhalte eine detaillierte JSON-Ausgabe mit geplanten Posts und potenziellen Konflikten.

3. **Automatisierte Deployment-Pipeline**  
   Nach erfolgreichem Test: `git push` triggert die GitHub Action. Das Tool führt `crosspostfast deploy` im Dry-Run-Modus aus, zeigt eine Diff-Liste an, und nach manueller Bestätigung (via Issue-Comment) erfolgt die Live-Auslieferung.

### Beste Praktiken

- **Content-Versionierung mit Branching**  
  Verwende Feature-Branches für Kampagnen. Jeder Branch enthält eigene `crosspostfast.yaml`-Varianten. Merge nur nach erfolgreichem CI-Lint und manueller Review. So vermeidest du versehentliches Posten auf Produktions-Kanälen.

- **Rate-Limit-Management via Token-Rotation**  
  Nutze die integrierte `rate_limit`-Direktive in der Konfiguration (z.B. `max_posts_per_hour: 5`). Für Hochlast-Kampagnen: Implementiere eine Queue mit asynchroner Verarbeitung. crosspostfast unterstützt nativ Redis-basierte Cooldown-Speicherung.
