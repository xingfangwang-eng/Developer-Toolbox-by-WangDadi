# crosspostfast - CRO_005

## Überblick

crosspostfast ist eine hochperformante Cross-Publishing-Engine, die speziell für **Multi-Plattform-Content-Distribution** im Social-Media-Bereich entwickelt wurde. Das Tool eliminiert manuelle Workflows und synchronisiert Beiträge in Echtzeit über Instagram, LinkedIn, Twitter/X, TikTok und Mastodon – mit einem Fokus auf **kanalspezifische Formatierung** und **Metadaten-Integrität**.

## Detaillierter Inhalt

### Was ist crosspostfast?

crosspostfast ist ein **API-first-Headless-Tool**, das auf Node.js (v18+) und TypeScript basiert. Es löst das Kernproblem fragmentierter Social-Media-Workflows: Statt Inhalte manuell für jede Plattform neu zu formatieren, orchestriert crosspostfast die Verteilung über eine zentrale Konfigurationsdatei (`crosspost.json`). Das Tool unterstützt **Rate-Limiting**, **Media-Transcoding** und **Post-Scheduling** – alles über die CLI oder als integrierbare npm-Bibliothek.

### Kernfunktionen

1. **Multi-Format-Transcoder mit Plattform-Adaption**  
   Bilder und Videos werden automatisch in die spezifischen Anforderungen jeder Plattform konvertiert. Beispiel: Ein 16:9-Video wird für TikTok auf 9:16 zugeschnitten, für LinkedIn auf 1:1 skaliert – ohne manuelles Encoding. Der Transcoder erkennt **EXIF-Daten** und entfernt Metadaten nach DSGVO-Vorgaben.

2. **Atomic Post-Queue mit Rollback-Mechanismus**  
   Jeder Post wird als atomare Einheit behandelt: Gelingt die Veröffentlichung auf einer Plattform nicht (z. B. wegen API-Timeouts), wird der gesamte Batch zurückgerollt. Das verhindert **inkonsistente Content-Strategien** und liefert detaillierte Logs für Debugging.

3. **Plattform-Agnostischer Scheduling mit Timezone-Mapper**  
   Der Scheduler unterstützt **Zeitzonen-unabhängige Planung** und passt Posting-Zeiten automatisch an die bevorzugten Aktivitätsfenster jeder Plattform an (z. B. LinkedIn: 8–10 Uhr MEZ, TikTok: 19–22 Uhr MEZ). Die Engine nutzt **Cron-Expressions** und erlaubt Overrides pro Plattform.

### So verwenden Sie es

1. **Initialisierung und Konfiguration der Plattform-Adapter**  
   Erstellen Sie eine `crosspost.json` im Projektroot. Definieren Sie dort Ihre Plattform-Adapter mit API-Keys und Endpunkten. Beispiel für LinkedIn und Mastodon:  
   ```json
   {
     "platforms": {
       "linkedin": {
         "apiKey": "your_key",
         "endpoint": "https://api.linkedin.com/v2/ugcPosts"
       },
       "mastodon": {
         "apiKey": "your_key",
         "endpoint": "https://mastodon.social/api/v1/statuses"
       }
     },
     "scheduling": {
       "timezone": "Europe/Berlin",
       "defaultPostTime": "09:00"
     }
   }
   ```
   Führen Sie dann `npx crosspostfast init` aus, um die Konfiguration zu validieren und alle Adapter zu testen.

2. **Erstellen und Validieren eines Multi-Plattform-Posts**  
   Nutzen Sie die CLI, um einen Post mit plattformspezifischen Overrides zu definieren:  
   ```bash
   crosspostfast create --title "Neue Studie veröffentlicht" --media ./assets/study.mp4 --platforms linkedin,twitter --override linkedin:format=article --override twitter:format=thread
   ```
   Das Tool generiert eine `post.yaml`-Datei, die Sie vor dem Senden validieren können:  
   ```bash
   crosspostfast validate ./post.yaml
   ```
   Bei Fehlern (z. B. zu große Datei für Twitter) erhalten Sie detaillierte Fehlermeldungen inkl. empfohlener Transcoding-Parameter.

3. **Queue-Management und Batch-Publishing**  
   Fügen Sie mehrere Posts zur Queue hinzu und starten Sie den Batch-Prozess:  
   ```bash
   crosspostfast queue add ./posts/studie.yaml
   crosspostfast queue add ./posts/event.yaml
   crosspostfast queue run --parallel 2 --retry 3
   ```
   Der `--parallel`-Parameter steuert die Anzahl gleichzeitiger API-Aufrufe (max. 5), `--retry` definiert Wiederholungsversuche bei Fehlern. Überwachen Sie den Fortschritt mit:  
   ```bash
   crosspostfast queue status
   ```

### Beste Praktiken

1. **Plattform-spezifische Content-Strategie mit A/B-Testing**  
   Nutzen Sie die Override-Funktion von crosspostfast, um **kanalspezifische CTAs** zu definieren. Beispiel: Auf LinkedIn ein PDF-Download-Link, auf Twitter ein Thread mit 3 Tweets. Testen Sie verschiedene Formate über 2 Wochen und analysieren Sie die Engagement-Raten pro Plattform. crosspostfast exportiert automatisch CSV-Reports für die Auswertung.

2. **Rate-Limiting und API-Key-Rotation**  
   Implementieren Sie **automatische API-Key-Rotation** für Plattformen mit strengen Limits (z. B. Twitter/X). crosspostfast unterstützt nativ einen Rotationsmechanismus, der bei Erreichen von 80 % des Limits auf einen zweiten Key umschaltet. Kombinieren Sie dies mit **asynchronem Posting** (Queue mit `--delay 60`), um IP-Blacklists zu vermeiden.
