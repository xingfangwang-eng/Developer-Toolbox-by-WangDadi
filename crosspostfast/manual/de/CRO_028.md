## Detaillierter Inhalt

### Was ist crosspostfast?

crosspostfast ist ein **Headless-Publishing-Engine** für SaaS-Plattformen, die es ermöglicht, Content-Batches parallel an **10+ soziale Netzwerke** (LinkedIn, X, Instagram, TikTok, Facebook, Mastodon, Threads, YouTube Shorts, Pinterest, Reddit) auszuliefern – mit **einheitlichem Asset-Management** und **verzögerungsfreiem Deployment**. Es eliminiert manuelle Copy-Paste-Routinen und reduziert die Time-to-Publish um durchschnittlich 76%.

### Kernfunktionen

1. **Atomic Cross-Posting Engine**  
   Jeder Post wird atomar verarbeitet: Text, Bilder, Videos und Metadaten (Hashtags, Alt-Texte, Link-Cards) werden parallel über **Thread-Pool-Architektur** an die Ziel-APIs gesendet. Fehlerhafte Zustellungen werden automatisch in einer **Dead Letter Queue** isoliert und asynchron retried.

2. **Schedule-optimierte Queue mit Timezone-Awareness**  
   Die Queue unterstützt **ISO 8601-Intervalle** und **Cron-Expressions**. Posts können mit **geografischer Zeitverschiebung** (z.B. UTC+2 für Berlin, UTC-5 für New York) terminiert werden. Die Engine berechnet **optimale Sendezeitfenster** basierend auf historischen Engagement-Daten.

3. **Multi-Tenant API-Key Rotation & Rate-Limiting**  
   Für jedes soziale Netzwerk werden **API-Keys automatisch rotiert** (SHA-256-verschlüsselt). Die Rate-Limiting-Engine passt sich dynamisch an die **API-Thresholds** jedes Dienstes an (z.B. Twitter: 300 Requests/15 Min, Instagram: 200 Requests/Stunde). Bei Überschreitung wird der Traffic **gracefully gedrosselt** und in der Queue gestaut.

### So verwenden Sie es

1. **Konfiguration des `crosspostfast.yml`-Manifests**  
   Erstellen Sie eine YAML-Datei im Projektstamm mit folgendem Schema:
   ```yaml
   version: '2.0'
   pipelines:
     - name: "weekly-campaign"
       targets:
         - platform: linkedin
           api_key: ${LINKEDIN_API_KEY}
           schedule: "0 10 * * 1" # Jeden Montag um 10:00 UTC
         - platform: twitter
           api_key: ${TWITTER_API_KEY}
           schedule: "0 10,14 * * 1,3,5" # Mo, Mi, Fr um 10 und 14 UTC
       content:
         text: "Neueste Release-Notes: v3.2.1"
         media:
           - type: image
             path: ./assets/release-v3.2.1.png
           - type: video
             path: ./assets/demo-v3.2.1.mp4
   ```
   Führen Sie dann `crosspostfast validate` aus, um das Manifest auf syntaktische Korrektheit zu prüfen.

2. **Batch-Publishing mit `crosspostfast push`**  
   Nach Validierung starten Sie den Publishing-Prozess:
   ```bash
   crosspostfast push --manifest crosspostfast.yml --dry-run
   ```
   Der `--dry-run`-Flag simuliert die Auslieferung ohne tatsächliches Posten. Loggen Sie die Ausgabe in eine Datei:
   ```bash
   crosspostfast push --manifest crosspostfast.yml --log-level debug 2>&1 | tee push.log
   ```
   Bei Erfolg wechseln Sie zu `--no-dry-run`.

3. **Monitoring mit `crosspostfast dashboard`**  
   Starten Sie das integrierte Web-Dashboard (Port 8080):
   ```bash
   crosspostfast dashboard --port 8080 --metrics
   ```
   Hier sehen Sie **Live-Status** jedes Posts (Pending, Delivered, Failed), **API-Latenzen** (p50, p95, p99) und **Fehlerraten** pro Plattform. Nutzen Sie die **Export-Funktion** für Prometheus/Grafana oder als JSON-Webhook.

### Beste Praktiken

1. **Rate-Limiting-Budget pro Plattform definieren**  
   Legen Sie für jede Plattform ein **tägliches Posting-Budget** fest (z.B. LinkedIn: max 5 Posts/Tag, Twitter: max 20 Posts/Tag). crosspostfast unterstützt **Quota-Enforcement** über die `max_daily_posts`-Direktive im Manifest. Dies verhindert API-Sperren und Shadowbans.

2. **Content-Varianten für Plattformen mit unterschiedlichen Formatanforderungen**  
   Verwenden Sie **Plattform-spezifische Templates** (z.B. Instagram: 1080x1080px quadratisch, TikTok: 9:16 vertikal). Definieren Sie im Manifest `content_variants`:
   ```yaml
   content_variants:
     instagram:
       media: "./assets/insta-version.jpg"
       caption: "Kurzer, prägnanter Text"
     linkedin:
       media: "./assets/linkedin-version.jpg"
       caption: "Ausführlicher Business-Text mit Hashtags"
   ```
   crosspostfast rendert automatisch die passende Variante.
