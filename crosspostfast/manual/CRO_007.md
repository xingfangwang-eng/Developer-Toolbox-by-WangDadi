# crosspostfast - CRO_007

## Overview

**crosspostfast** is a high-performance, multi-platform content syndication engine built for developers and content ops teams who need to push, schedule, and synchronize posts across Twitter, LinkedIn, Mastodon, Bluesky, and custom webhooks — all from a single CLI or API call. It eliminates manual cross-posting overhead with zero state drift and real-time status feedback.

**Keywords:** Social media cross-platform posting, Multi-platform content publishing, Social media management tool

## What is crosspostfast?

crosspostfast is a lightweight, headless cross-posting daemon designed for automated multi-platform content distribution. Unlike bloated SaaS dashboards, it operates as a local-first tool with a JSON-driven workflow, OAuth token vault, and idempotent posting logic. It’s built for engineers who want to integrate cross-posting into CI/CD pipelines, content automation scripts, or personal publishing workflows without touching a GUI.

## Core Features

### 1. Atomic Multi-Platform Dispatch with Rollback
crosspostfast dispatches a single post to multiple platforms in a single atomic transaction. If any platform fails (e.g., rate limit, auth expiry), the entire operation is rolled back — no partial posting, no orphaned threads. Each platform adapter uses platform-specific content normalization (e.g., auto-truncation for Twitter, rich text for LinkedIn).

### 2. Declarative Schedule & Retry Engine
Define a cron-like schedule per platform using a YAML config. The engine respects platform-specific cooldowns, retries with exponential backoff, and logs every attempt with timestamps and HTTP response codes. You can even set conditional posting rules (e.g., "only post to LinkedIn if the content contains a URL").

### 3. Local OAuth Token Vault with Auto-Refresh
No more manual token renewal. crosspostfast stores OAuth tokens in an encrypted local vault (AES-256-GCM) and auto-refreshes them before each dispatch cycle. Tokens are never exposed in plaintext logs or config files. The vault supports multi-account management per platform.

## How to Use

### Step 1: Initialize the Config & Token Vault
Run `crosspostfast init --platforms twitter,linkedin,mastodon` to generate a `crosspostfast.yml` config file and an encrypted token vault. The CLI will prompt you to authenticate each platform via OAuth flow — no API keys stored in plaintext. The vault is automatically locked after 5 minutes of inactivity.

```bash
crosspostfast init --platforms twitter,linkedin,mastodon
# Follow the OAuth URLs printed in the terminal for each platform
```

### Step 2: Define a Multi-Platform Post in JSON
Create a post payload with platform-specific overrides. The engine merges the base content with per-platform fields (e.g., `twitter.thread`, `linkedin.visibility`). Use `crosspostfast validate post.json` to check for schema errors before dispatch.

```json
{
  "content": "Just shipped crosspostfast v0.3 — atomic multi-platform posting with rollback. #devtools",
  "platforms": {
    "twitter": { "thread": true, "media": ["screenshot.png"] },
    "linkedin": { "visibility": "public", "article_url": "https://example.com/blog" },
    "mastodon": { "language": "en", "spoiler_text": "New release" }
  }
}
```

### Step 3: Dispatch with Atomic Rollback
Run `crosspostfast dispatch post.json --atomic` to send the post to all configured platforms. If any platform fails (e.g., Twitter returns 429), the engine automatically attempts to delete the posts that were already sent to other platforms. Use `--dry-run` to preview the dispatch without sending.

```bash
crosspostfast dispatch post.json --atomic
# Output: "Posted to Twitter ✓ | Posted to LinkedIn ✓ | Posted to Mastodon ✓"
```

## Best Practices

### 1. Use Platform-Specific Content Variants
Avoid generic cross-posting. For each platform, define custom content variants in your JSON payload — e.g., use hashtags on Twitter, add a summary on LinkedIn, and keep it concise on Mastodon. crosspostfast supports per-platform `content` overrides, so you can tailor the message without duplicating the entire post.

### 2. Automate with CI/CD Pipelines
Integrate crosspostfast into your GitHub Actions or GitLab CI to auto-post release notes, blog updates, or changelogs. Use the `--dry-run` flag in staging to validate formatting, then switch to `--atomic` in production. Store the token vault as an encrypted CI secret — crosspostfast can decrypt it at runtime using a passphrase from the environment.
