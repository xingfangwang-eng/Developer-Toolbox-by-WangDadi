# crosspostfast - CRO_005

## Overview

crosspostfast is a high-performance, API-first social media cross-posting engine designed for SaaS teams, content agencies, and indie hackers who need to schedule, publish, and synchronize content across multiple platforms—without hitting rate limits or breaking format consistency. It’s built for speed, reliability, and SEO-native output.

## What is crosspostfast?

crosspostfast is a lightweight, developer-friendly tool that automates multi-platform content publishing by abstracting platform-specific API quirks into a unified, idempotent pipeline. Think of it as a cross-posting middleware that handles auth refresh, media transcoding, retry logic, and post deduplication—so you don’t have to.

## Core Features

### 1. **Idempotent Multi-Platform Dispatch with Auto-Retry**
Each post is assigned a deterministic UUID. The engine ensures that even if a network failure or API timeout occurs mid-flight, the post is never duplicated. Failed deliveries are queued for exponential backoff retry with per-platform failure isolation. No more accidental double-posts across LinkedIn and Twitter.

### 2. **Media Transcoding & Platform-Native Format Mapping**
Automatically converts images, GIFs, and short videos to the optimal format for each target platform (e.g., 16:9 for YouTube, 1:1 for Instagram, 4:5 for Facebook). Supports WebP to JPEG fallback, video resolution downscaling, and alt-text injection for accessibility and SEO compliance.

### 3. **SEO-Optimized Post Templates with Dynamic Metadata Injection**
Define reusable post templates with placeholders for `{title}`, `{url}`, `{hashtags}`, and `{locale}`. The engine injects platform-specific metadata (Open Graph tags, Twitter Cards, schema markup) directly into the post payload. Perfect for cross-posting blog content while preserving link previews and search visibility.

## How to Use

### Step 1: Configure Your Platform Credentials via Environment Variables
Create a `.env` file with your API tokens and secrets for each platform. crosspostfast uses a unified credential manager that supports OAuth 2.0 refresh token rotation. Example:

```env
TWITTER_API_KEY=your_key
TWITTER_API_SECRET=your_secret
LINKEDIN_ACCESS_TOKEN=your_token
INSTAGRAM_BUSINESS_ID=your_id
```

Run `crosspostfast init` to validate all credentials and store them securely in an encrypted local vault.

### Step 2: Define Your Post Payload as a JSON Object
Use the CLI or SDK to craft a structured post:

```json
{
  "title": "How We Scaled Our SaaS to 10K Users",
  "content": "In this post, we break down the architecture decisions that mattered.",
  "media": ["https://cdn.example.com/thumbnail.png"],
  "platforms": ["twitter", "linkedin", "facebook"],
  "schedule": "2025-06-15T14:00:00Z",
  "metadata": {
    "og:image": "https://cdn.example.com/og-image.png",
    "twitter:card": "summary_large_image"
  }
}
```

Call `crosspostfast publish --payload post.json` to queue the post.

### Step 3: Monitor Delivery Status via the Event Log
Use the built-in event stream to track each post’s lifecycle:

```bash
crosspostfast logs --tail --filter=error
```

You’ll see real-time delivery status per platform: `queued → encoding → posting → posted` or `failed (retry in 30s)`. The log output is JSON-formatted for easy ingestion into your monitoring stack (e.g., Datadog, Grafana).

## Best Practices

### 1. **Use Platform-Specific Post Templates for Higher Engagement**
Don’t blast the same copy everywhere. Create a master template with conditional blocks for each platform. For example, Twitter needs a shorter lead + a link, while LinkedIn benefits from a longer narrative and a call-to-action. crosspostfast supports `if_platform` directives in your template engine.

### 2. **Schedule Posts During Platform Peak Hours Using Timezone-Aware Queues**
Each platform has optimal posting windows. Use crosspostfast’s built-in timezone mapping to schedule posts in the target audience’s local time. For example, schedule a LinkedIn post for 9:00 AM EST while the same content goes to Twitter at 8:00 AM PST. The engine handles the offset automatically.
