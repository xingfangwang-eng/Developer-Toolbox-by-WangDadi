## Detailed Content

### What is crosspostfast?

**crosspostfast** is a high-performance, multi-tenant SaaS engine designed to eliminate the friction of cross-platform content distribution. It transforms a single content draft into platform-optimized posts for Twitter, LinkedIn, Instagram, Mastodon, Bluesky, and more — all from one unified API or dashboard. Built for agencies, growth hackers, and indie developers who need speed, reliability, and zero manual overhead.

### Core Features

- **Atomic Draft Sync with Platform-Aware Rendering**  
  Define your content once as a JSON payload. The engine automatically resizes images, truncates text, and applies platform-specific formatting (e.g., thread splitting for X/Twitter, carousel generation for Instagram). No more copy-paste errors.

- **Distributed Queue with Idempotency & Retry Logic**  
  Each post is processed through an idempotent, fault-tolerant queue. If a platform API fails (rate limit, timeout, auth expiry), the job is retried with exponential backoff. You get a deterministic outcome per post ID — no duplicates, no ghost failures.

- **Real-Time Cross-Platform Analytics Webhook**  
  After publishing, crosspostfast fires a webhook payload containing per-platform engagement metrics (impressions, clicks, shares) aggregated in near real-time. Integrate directly into your own BI pipeline or dashboard.

### How to Use

1. **Define a Post Payload with Conditional Fields**  
   Create a JSON object with a `content` block and a `platforms` array. Each platform entry can override fields like `image_url`, `max_length`, or `thread_delimiter`. Example:  
   ```json
   {
     "content": { "text": "Ship fast. Break nothing.", "media": ["https://example.com/img.png"] },
     "platforms": [
       { "id": "twitter", "max_length": 280, "thread": true },
       { "id": "linkedin", "max_length": 3000, "media": [] }
     ]
   }
   ```  
   The engine validates the payload schema and returns a `post_id` immediately.

2. **Submit via REST API with Async Polling**  
   POST to `/v1/publish` with an `X-Idempotency-Key` header. The endpoint returns a `202 Accepted` with a `job_id`. Poll `GET /v1/jobs/{job_id}` to get the status per platform. Use the `status` field (`queued`, `publishing`, `published`, `failed`) to drive your UI logic.

3. **Handle Webhook Events for Post-Mortem Analysis**  
   Register a webhook URL in the dashboard. After all platforms complete, crosspostfast sends a `POST` with the full result object:  
   ```json
   {
     "post_id": "abc123",
     "results": {
       "twitter": { "status": "published", "url": "https://x.com/...", "impressions": 1200 },
       "linkedin": { "status": "failed", "error": "INVALID_IMAGE_FORMAT" }
     }
   }
   ```  
   Use this to trigger alerts, update your DB, or re-queue failed posts with corrected payloads.

### Best Practices

- **Use Idempotency Keys for All Mutations**  
  Always generate a unique `X-Idempotency-Key` (UUIDv4) for each publish request. This prevents double-posting even if your client retries due to network timeouts. Store the key alongside your draft ID for traceability.

- **Pre-Validate Assets Before Submission**  
  Run media files through a local check (size, format, aspect ratio) before sending to crosspostfast. For example, Instagram Reels require MP4 with max 60s duration. Pre-validation reduces API errors and speeds up the queue. Use the `/v1/validate` endpoint to test payloads without publishing.
