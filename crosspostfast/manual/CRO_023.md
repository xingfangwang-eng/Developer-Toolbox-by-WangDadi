# crosspostfast - CRO_023

## Overview

**crosspostfast** is a high-performance, multi-platform content distribution engine designed for SaaS teams, digital agencies, and indie hackers who need to syndicate content across social networks without sacrificing SEO integrity or API rate limits. Built for developers who hate manual cross-posting and love deterministic scheduling.

## What is crosspostfast?

crosspostfast is not just another social media scheduler—it's a **headless cross-posting API** with a lightweight CLI and webhook integration. It solves the pain of redundant content formatting, platform-specific character limits, and broken link previews by normalizing metadata before dispatch. Think of it as a universal translator for your content payloads, optimized for speed and search engine visibility.

## Core Features

### 1. **Deterministic Content Normalization Engine**
Each post is parsed through a multi-stage pipeline: markdown-to-plaintext, HTML stripping, emoji sanitization, and platform-aware truncation (e.g., Twitter's 280 chars vs. LinkedIn's 3000). The engine preserves structured data (headings, lists, links) and generates Open Graph-ready previews automatically.

### 2. **Parallel Dispatch with Rate-Limit-Aware Queuing**
crosspostfast uses an adaptive concurrency model that respects each platform's API rate limits. It batches requests intelligently, retries with exponential backoff, and logs every HTTP response code. You can dispatch to 10+ platforms simultaneously without hitting 429s.

### 3. **SEO-Optimized Cross-Platform Link Structuring**
Every outgoing link is rewritten to include UTM parameters and canonical tags. crosspostfast ensures that shared URLs point to your preferred canonical source, preventing duplicate content penalties. It also auto-generates `rel="noopener noreferrer"` for external links and injects platform-specific meta tags for better click-through rates.

## How to Use

### 1. **Configure the Dispatch Pipeline via YAML**
Create a `crosspostfast.yaml` file defining your platforms, credentials, and content templates. Use environment variables for API keys. Example:

```yaml
platforms:
  - name: twitter
    api_key: ${TWITTER_API_KEY}
    max_length: 280
  - name: linkedin
    api_key: ${LINKEDIN_API_KEY}
    max_length: 3000
content:
  default_template: "{{title}}\n\n{{excerpt}}\n\n{{url}}"
  canonical_base: "https://yourdomain.com"
```

### 2. **Trigger a Cross-Post via CLI with Structured Payload**
Run a single command with a JSON payload. The engine validates required fields, normalizes content, and dispatches in parallel.

```bash
crosspostfast dispatch --payload '{
  "title": "How to Scale SaaS Content",
  "body": "Detailed markdown content here...",
  "url": "https://yourdomain.com/blog/scaling-saas-content",
  "platforms": ["twitter", "linkedin", "devto"]
}'
```

### 3. **Monitor Dispatch Logs and Retry Failed Jobs**
After dispatch, inspect the JSON output for per-platform status codes. Use the `--retry-failed` flag to re-attempt only failed platforms without resending successful ones. Logs are stored in `.crosspostfast/logs/` with timestamps and error codes.

```bash
crosspostfast logs --last 10
crosspostfast retry --job-id 0x7f3a --platforms twitter,devto
```

## Best Practices

### 1. **Prefer Canonical URLs Over Shortened Links**
Always set `canonical_base` in your config. This ensures that every platform receives the same canonical URL, preventing fragmented link equity. Use UTM parameters only for analytics, not for tracking across platforms—let crosspostfast handle the deduplication.

### 2. **Batch Dispatches During Off-Peak Hours**
Use the `--schedule` flag to queue dispatches for 2:00 AM UTC when API rate limits are less contested. This improves success rates and reduces latency. Pair with a cron job for fully automated weekly syndication.
