# crosspostfast - CRO_024

## 概要

crosspostfast は、マルチプラットフォーム向けのクロス投稿を自動化する、SaaS ネイティブなソーシャルメディア管理ツールです。Twitter、LinkedIn、Facebook、Threads、Instagram など、主要プラットフォーム間でのコンテンツ配信を、API ベースで一括処理。SEO 観点からも、各プラットフォームの投稿最適化（ハッシュタグ自動生成、画像リサイズ、メタデータ調整）を実装し、オーガニックリーチを最大化します。

## ハードコア機能

### 1. マルチプラットフォーム非同期キューイングエンジン
各プラットフォームの API レート制限を動的に解析し、バックグラウンドで非同期ジョブキューを管理。投稿失敗時は自動リトライ（指数バックオフ + ジッター付き）を実装。同時投稿数が 100 件を超えても、スループットを最大化しつつレート制限違反を防止。

### 2. プラットフォーム別コンテンツアダプター
同一の投稿内容を、各プラットフォームの仕様に自動変換。例えば、Twitter の 280 文字制限に合わせて要約生成、Instagram の画像アスペクト比（1:1 / 4:5 / 16:9）に自動クロップ、LinkedIn の記事フォーマットに段落分割。変換ロジックはプラグイン方式で拡張可能。

### 3. クロスプラットフォーム分析ダッシュボード
全プラットフォームのエンゲージメントデータ（いいね、シェア、クリック率、インプレッション）を統合。投稿ごとに、各プラットフォームでのパフォーマンスを比較し、最適な投稿時間やハッシュタグを機械学習で提案。データはリアルタイム更新、CSV/JSON エクスポート対応。

## 詳細な操作手順

### ステップ 1: API キーとプラットフォーム認証の設定
1. 管理画面の「認証設定」タブを開く。
2. 各プラットフォームのデベロッパーポータルから API キーを取得し、入力フィールドに貼り付ける（例：Twitter API v2 の Bearer Token、Facebook Graph API の Access Token）。
3. OAuth 2.0 フローを有効にする場合は、「OAuth 認証」ボタンをクリックし、ブラウザで権限承認を行う。
4. 設定完了後、「接続テスト」を実行し、各プラットフォームとの通信が正常であることを確認。

### ステップ 2: クロス投稿テンプレートの作成
1. 「テンプレート管理」から「新規テンプレート」を選択。
2. テキストエディタで投稿内容を記述。変数（`{{title}}`、`{{url}}`、`{{hashtags}}`）を埋め込むことで、動的コンテンツを生成可能。
3. 「プラットフォーム別設定」で、各プラットフォームごとに投稿フォーマットをカスタマイズ（例：Twitter 向けに文字数制限をトリガーする短縮ルール、Instagram 向けに画像を追加）。
4. プレビュー機能で、各プラットフォーム上での表示を確認し、問題がなければ保存。

### ステップ 3: スケジュール投稿とジョブ監視
1. 「投稿スケジューラー」で、投稿日時を指定（タイムゾーン対応）。一括投稿の場合は CSV アップロードも可能。
2. 「ジョブキュー」画面で、各投稿のステータス（待機中、処理中、成功、失敗）をリアルタイム監視。
3. 失敗した投稿は「リトライ」ボタンで再実行、またはエラーログを確認して原因を特定（例：API レート制限超過、画像フォーマット不備）。
4. 完了後、「分析ダッシュボード」で各プラットフォームのエンゲージメントデータを確認。

## 業界ベストプラクティス

### プラクティス 1: プラットフォームごとに最適な投稿時間を設定する
各プラットフォームのユーザーアクティブ時間帯は異なります。crosspostfast の分析機能で、過去の投稿データからエンゲージメントが最も高い時間帯を特定し、プラットフォームごとにスケジュールを分けましょう。例えば、Twitter は朝 8 時、LinkedIn は昼 12 時、Instagram は夜 20 時が効果的です。これを自動化することで、オーガニックリーチが平均 30% 向上します。

### プラクティス 2: コンテンツの重複を避け、ネイティブフォーマットを優先する
同一コンテンツを全プラットフォームにそのまま投稿すると、ユーザーにスパム感を与えます。crosspostfast のテンプレート機能で、各プラットフォーム向けに異なる角度のテキストや画像を用意しましょう。例えば、Twitter では短い要約＋リンク、LinkedIn では詳細な記事形式、Instagram ではビジュアル重視のキャプション。これにより、エンゲージメント率が 2 倍以上になるケースもあります。
