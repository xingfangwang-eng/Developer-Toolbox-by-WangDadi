# crosspostfast - CRO_001

## 概要

crosspostfast は、ソーシャルメディアのクロスプラットフォーム投稿を最適化するプロフェッショナルツールです。マルチプラットフォームのコンテンツパブリッシングを効率化し、SEO に強いメタデータ管理と CMS 連携を実現します。キーワード「Social media cross-platform posting」「Multi-platform content publishing」「Social media management tool」に対応した設計です。

## 詳細な内容

### crosspostfast とは

crosspostfast は、複数のソーシャルメディアプラットフォーム（Twitter、LinkedIn、Facebook、Instagram など）に対して、一括でコンテンツを投稿・スケジュール管理できる SaaS ツールです。特に、API レート制限やプラットフォームごとのフォーマット差異を自動解決し、SEO メタデータ（OGP、JSON-LD）を動的に生成する点が特徴です。開発者向けには RESTful API と Webhook を提供し、CI/CD パイプラインとの統合も容易です。

### コア機能

1. **動的メタデータ最適化エンジン**  
   各プラットフォームの仕様に合わせて、タイトル、説明文、画像のアスペクト比、ハッシュタグを自動変換。さらに、JSON-LD 構造化データを動的に生成し、Google 検索のリッチリザルト対応を強化します。例えば、Twitter カードと Facebook OGP を同時に最適化する「ハイブリッドメタモーフィング」を実装。

2. **クロスプラットフォームスケジューリング & レート制限回避**  
   Twitter の 1 時間あたりの投稿制限や Instagram の API クオータを考慮し、自動で最適な投稿間隔を計算。また、プラットフォームごとのタイムゾーン差分を吸収し、ユーザーの指定したローカル時間に正確に投稿します。障害発生時には自動リトライとフォールバック投稿を実行。

3. **コンテンツテンプレート & バージョン管理**  
   Markdown ベースのテンプレートエンジンにより、変数（`{{title}}`, `{{date}}`, `{{hashtag}}`）を埋め込んだテンプレートを管理。Git リポジトリと連携し、テンプレートの差分管理とロールバックが可能。さらに、A/B テスト用に複数バージョンの投稿を同時にスケジュールし、エンゲージメント率を比較できます。

### 使い方

1. **API キーと Webhook のセットアップ**  
   まず、crosspostfast のダッシュボードで各プラットフォームの API キーを登録します。次に、CI/CD パイプライン（GitHub Actions や GitLab CI）に Webhook URL を設定し、コードのプッシュ時に自動で投稿がトリガーされるようにします。Webhook のペイロードには、`{ "platform": ["twitter", "linkedin"], "content_id": "123" }` のような JSON を指定。

2. **テンプレートの作成とプレビュー**  
   ダッシュボードのテンプレートエディタで、Markdown ベースのテンプレートを作成。例として、`# {{title}}` と `{{description}}` を含むテンプレートを保存。プレビュー機能で各プラットフォームのレンダリング結果を確認し、必要に応じて変数を調整します。プレビューは実際の API を呼び出さずにシミュレートされるため、コストを抑えられます。

3. **スケジュール投稿とモニタリング**  
   スケジュール画面で、投稿日時とプラットフォームを選択。タイムゾーンは UTC またはユーザーのローカルタイムを指定可能。投稿後は、ダッシュボードの分析タブでエンゲージメント（いいね、リツイート、クリック数）をリアルタイムでモニタリング。異常なレート制限エラーや投稿失敗があれば、Slack やメールに通知が飛びます。

### ベストプラクティス

1. **プラットフォームごとに異なるハッシュタグ戦略を自動化**  
   Twitter ではトレンドワードを、LinkedIn では業界固有のキーワードを自動で選定する「ハッシュタグジェネレーター」を活用。crosspostfast の動的メタデータ最適化エンジンと組み合わせることで、各プラットフォームのアルゴリズムに最適化された投稿が可能です。特に、Instagram ではハッシュタグの最大数（30 個）を自動で埋め込む設定が推奨されます。

2. **CI/CD パイプラインに組み込んでリリースノートを自動投稿**  
   コードのリリース時に、Git タグをトリガーとして crosspostfast の Webhook を呼び出し、リリースノートを Twitter と LinkedIn に自動投稿。これにより、手作業のミスを防ぎ、開発チームの負荷を軽減します。テンプレートには `{{version}}` や `{{changelog}}` を埋め込み、GitHub のリリースノートから動的に取得するのがポイントです。
