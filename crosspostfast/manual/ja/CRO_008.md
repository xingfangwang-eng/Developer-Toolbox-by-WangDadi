# crosspostfast - CRO_008

## 概要

crosspostfast は、ソーシャルメディアのクロスプラットフォーム投稿を自動化するためのプロフェッショナルツールです。マルチプラットフォームコンテンツパブリッシングを効率化し、ソーシャルメディア管理の手間を大幅に削減します。本プロジェクトは、SEO最適化されたコンテンツ戦略と高速なクロスポスト機能を融合させ、マーケターやコンテンツクリエイターに最適なソリューションを提供します。

## 詳細な内容

### crosspostfast とは？

crosspostfast は、複数のソーシャルメディアプラットフォーム（Twitter、Instagram、LinkedIn、Facebook、TikTokなど）に同時に投稿できるオープンソースのSaaSツールです。APIファーストの設計により、プラグイン不要で各プラットフォームのネイティブ機能を最大限活用。特に、**クロスプラットフォーム間のフォーマット自動変換**と**スケジューリング機能**が特徴で、コンテンツの一貫性を保ちながら、最適なタイミングで配信できます。

### コア機能

1. **フォーマット自動最適化エンジン**  
   各プラットフォームの仕様（文字数制限、画像比率、ハッシュタグ形式）を自動検出し、投稿内容をリアルタイムで変換。例えば、Twitter向けに280文字以内に要約し、Instagram向けにはカルーセル形式に自動分割。手動調整はゼロで、クロスプラットフォーム間のフォーマット差異を吸収します。

2. **マルチアカウント一括管理ダッシュボード**  
   最大50アカウントまで同時管理可能。APIキーを設定するだけで、各アカウントの投稿履歴、エンゲージメント率、投稿失敗ログを一元表示。失敗時には自動リトライ機能（最大3回、指数バックオフ付き）が動作し、ネットワークエラーやレート制限にも耐性があります。

3. **コンテンツカレンダー連携スケジューラー**  
   GoogleカレンダーやNotionとの双方向同期に対応。投稿予定日時をブロック単位で設定し、タイムゾーン自動調整機能で世界中のオーディエンスに最適なタイミングで配信。さらに、AIが過去のエンゲージメントデータを分析し、推奨投稿時間を提案するインテリジェントモードも搭載。

### 使い方

1. **環境セットアップとAPI認証**  
   `git clone https://github.com/xingfangwang-eng/crosspostfast` でリポジトリをクローン後、`config.yaml` に各プラットフォームのAPIキー（Twitter Bearer Token、Instagram Graph API Token、LinkedIn Access Tokenなど）を記述。`make setup` を実行すると、依存関係（Node.js v18+、Python 3.10+、Docker）が自動インストールされ、ローカル環境で即座に起動可能。

2. **クロスポストジョブの定義と実行**  
   コマンドラインから `crosspostfast publish --platforms twitter,linkedin --content "新しいブログ記事を公開しました！" --media ./image.png --schedule "2025-04-01 10:00:00 JST"` を入力。`--schedule` オプションを省略すると即時投稿。内部では、各プラットフォームのAPIに非同期リクエストを送信し、レスポンスタイムアウト（デフォルト30秒）を超えた場合のみエラーログを出力。

3. **パフォーマンス監視とログ分析**  
   `crosspostfast monitor --interval 60` を実行すると、60秒ごとに各アカウントの投稿ステータスをチェック。`logs/` ディレクトリにJSON形式のログが蓄積され、`crosspostfast analyze` でエンゲージメント率（いいね数、シェア数、コメント数）をCSVエクスポート。失敗した投稿は `--retry` フラグで再実行可能。

### ベストプラクティス

- **プラットフォームごとにコンテンツを微調整する**  
  クロスポストの自動フォーマット変換を過信せず、各プラットフォームの文化に合わせたハッシュタグやトーンを手動で追加するのが効果的。例えば、LinkedInでは専門性を強調し、Twitterでは簡潔さを重視。crosspostfast の `--custom-message` オプションでプラットフォーム別のメッセージを指定できます。

- **レート制限を事前に計算してスケジュールを調整する**  
  Twitterのレート制限（15分間で300投稿）やInstagramのAPI制限（1時間あたり200リクエスト）を考慮し、`config.yaml` で `rate_limit_threshold` を設定。crosspostfast は自動的にリクエストをキューイングし、制限を超えないよう調整します。大規模キャンペーン時は、スケジュールを24時間以上に分散させるのが安全です。
