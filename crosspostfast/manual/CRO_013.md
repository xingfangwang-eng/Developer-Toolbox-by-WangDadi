# crosspostfast - CRO_013

## Overview

This is a high-performance cross-posting engine designed for developers and content teams who need to blast content across multiple social platforms without sacrificing SEO integrity or post structure. Built for speed, schema consistency, and API-first workflows.

## What is crosspostfast?

crosspostfast is a headless social media distribution layer that decouples content creation from platform-specific formatting. It handles rate limiting, retry logic, and metadata normalization across Twitter, LinkedIn, Mastodon, Bluesky, and custom REST endpoints—all from a single YAML config.

## Core Features

- **Schema-Aware Platform Adapters**  
  Each platform adapter (Twitter, LinkedIn, etc.) auto-transforms your content into platform-native formats: thread splitting for Twitter, rich card embeds for LinkedIn, and markdown-to-HTML sanitization for Mastodon. No manual re-formatting.

- **Atomic Post Bundles with Failover**  
  Define a single post bundle with fallback text, image variants, and link previews. If one platform rejects a media type, crosspostfast automatically downgrades to the next viable variant without breaking the entire pipeline.

- **Idempotent Queue with Webhook Confirmations**  
  Every post is assigned a deterministic UUID. The engine deduplicates retries, logs each delivery attempt, and fires a webhook once all platforms confirm success or permanent failure. Built for CI/CD pipelines.

## How to Use

1. **Initialize a project config**  
   Run `crosspostfast init --platforms twitter,linkedin,mastodon` to generate a `crosspost.yaml` with API keys, rate limits, and platform-specific field mappings. Use environment variables or a `.env` file to keep secrets out of version control.

2. **Define a post bundle**  
   Create a `post.yaml` with `title`, `body`, `media` (URLs or local paths), and `fallback_text`. Use the `variants` key to specify platform-specific overrides—e.g., a shorter summary for Twitter, a full article link for LinkedIn.

3. **Execute and monitor**  
   Run `crosspostfast publish post.yaml --watch` to push the bundle. The CLI streams real-time logs per platform, including HTTP status codes, rate limit headers, and retry backoff intervals. Use `--dry-run` first to validate the payloads without sending.

## Best Practices

- **Always pin platform adapter versions**  
  Social APIs change fast. Lock your `crosspost.yaml` to specific adapter versions (e.g., `twitter: v2.1`) to avoid silent failures from deprecated endpoints. Run `crosspostfast update adapters` periodically in a staging environment.

- **Use a staging platform group for testing**  
  Create a separate platform group (e.g., `staging`) pointing to test accounts or sandbox APIs. Run your CI pipeline against this group before promoting to production. This catches rate limit mismatches and media format issues early.
