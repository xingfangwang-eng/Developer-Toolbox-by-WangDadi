## Detailed Content

### What is crosspostfast?

`crosspostfast` is not just another social media scheduler; it's a **headless, API-first microservice** designed to eliminate the friction of multi-platform content syndication. It leverages a unified content model to normalize posts across disparate APIs (Twitter v2, Instagram Graph, LinkedIn, TikTok), handling rate limits, media transcoding, and thread splitting automatically. For developers, it’s the missing middleware for your CMS or CI/CD pipeline.

### Core Features

- **Atomic Cross-Platform Normalization & Thread Splitting:** Automatically adapts a single `Post` object to platform-specific constraints. It intelligently splits long-form content (e.g., a 5000-character blog summary) into a threaded Twitter/X thread, a single LinkedIn article, or a multi-slide Instagram carousel, preserving the semantic structure and media attachments.
- **Declarative Scheduling with Idempotency Tokens:** Define a cron-like schedule or a specific UTC timestamp. The system uses a deterministic `idempotency_key` (derived from the post content hash + platform ID) to guarantee exactly-once delivery, even after network failures or server restarts. No more duplicate posts.
- **Real-time Webhook-Driven Status Dashboard:** Instead of polling, `crosspostfast` provides a robust webhook system. Each post attempt triggers a `post.scheduled`, `post.published`, or `post.failed` event with a detailed error payload (e.g., `429: rate_limit_exceeded; retry_after: 300`). This allows you to build your own monitoring or integrate with Sentry/Datadog.

### How to Use

1.  **Initialize the Client & Configure Platform Credentials:** First, instantiate the `CrossPostClient` with your API key. Then, register each social platform using OAuth 2.0 or API tokens. The client abstracts away the underlying authentication flow.
    ```python
    from crosspostfast import CrossPostClient, Platform
    
    client = CrossPostClient(api_key="YOUR_API_KEY")
    client.add_platform(Platform.TWITTER, credentials={"bearer_token": "..."})
    client.add_platform(Platform.LINKEDIN, credentials={"access_token": "..."})
    ```

2.  **Build a Unified Post Object with Media Attachments:** Construct a `Post` object. The library automatically handles media compression (e.g., converting a PNG to JPEG for Twitter) and uploading to the platform's CDN. Use `PostBuilder` for complex workflows.
    ```python
    from crosspostfast.models import Post, Media, PostBuilder
    
    post = PostBuilder() \
        .set_title("New Release: crosspostfast v2.0") \
        .set_body("Check out our new feature for atomic thread splitting...") \
        .add_media(Media.from_url("https://example.com/hero.png", alt_text="Dashboard UI")) \
        .set_schedule("2024-03-15T14:00:00Z") \
        .build()
    ```

3.  **Execute the Cross-Post with Error Handling & Retry Logic:** Call the `publish` method. The SDK automatically retries on transient errors (e.g., 503 Service Unavailable) with exponential backoff. For permanent failures, it logs the error and triggers the webhook.
    ```python
    try:
        result = client.publish(post, platforms=[Platform.TWITTER, Platform.LINKEDIN])
        print(f"Published: {result.status}")  # Returns a dict with per-platform status
    except crosspostfast.exceptions.PlatformAuthorizationError as e:
        print(f"Auth failed for {e.platform}: {e}")
        # Trigger your own alerting system here
    ```

### Best Practices

- **Leverage the `idempotency_key` for CI/CD Pipelines:** When integrating `crosspostfast` into a GitHub Action or GitLab CI for automated release announcements, always use the same `idempotency_key` (e.g., derived from the Git commit hash). This prevents duplicate posts if the pipeline is re-run or if a deployment fails mid-way.
- **Implement a Two-Phase Media Upload for Large Assets:** For videos (e.g., TikTok or Instagram Reels), use the `Media.upload_in_chunks()` method instead of `Media.from_url()`. This provides a `media_upload_id` that can be passed to the `Post` object. It significantly reduces timeouts and allows you to track upload progress via the webhook.
