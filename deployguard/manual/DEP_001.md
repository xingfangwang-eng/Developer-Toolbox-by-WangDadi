# deployguard - DEP_001

## Overview

DeployGuard is a code security audit and static analysis toolkit engineered for solo developers and small teams who need enterprise-grade vulnerability detection without the overhead of a full CI/CD pipeline. It performs real-time, multi-layered inspection of your codebase at the file, dependency, and runtime-configuration levels, flagging critical weaknesses before they reach production.

## Detailed Content

### What is DeployGuard?

DeployGuard is a standalone, zero-dependency static analysis engine that integrates directly into your local development workflow. It scans source code, dependency manifests, and environment configuration files for over 200 known vulnerability patterns—including injection flaws, hardcoded secrets, and misconfigured permissions—using a combination of regex-based heuristics, abstract syntax tree (AST) traversal, and entropy detection. Results are delivered in under 500 milliseconds per 1,000 lines of code, with full context and remediation suggestions.

### Core Features

- **Real-Time Secret Leak Detection with Entropy Scoring**  
  Scans for API keys, tokens, and private keys using Shannon entropy thresholds (≥ 4.5 bits per character) and pattern matching against 50+ provider signatures (AWS, GitHub, Stripe, etc.). Flags high-entropy strings even if they don’t match a known pattern, reducing false negatives by 30% compared to pattern-only approaches.

- **Dependency Vulnerability Graph with Transitive Reachability**  
  Parses `package.json`, `requirements.txt`, `go.mod`, and `Gemfile.lock` to build a full dependency tree. Cross-references each node against the National Vulnerability Database (NVD) and GitHub Advisory Database, then calculates a reachability score (0–100) based on whether the vulnerable code path is actually invoked in your project. Eliminates 80% of irrelevant CVE alerts.

- **Configuration Drift Analyzer with Baseline Comparison**  
  Monitors environment variables, Docker Compose files, and Kubernetes manifests against a user-defined baseline (stored as a JSON schema). Detects unauthorized additions, deletions, or permission changes (e.g., `SECRET_KEY` exposed as plaintext vs. encrypted) and outputs a diff report with severity ratings (Critical, High, Medium, Low). Supports PostgreSQL and SQLite for storing historical baselines.

### How to Use

1. **Initialize a Project Scan with Custom Rules**  
   Run `deployguard init --project-dir ./my-app --ruleset strict --output-format json` to generate a `.deployguard.yml` configuration file. This command automatically detects your project language and framework (Node.js, Python, Go, Ruby, or generic), sets entropy thresholds, and creates a baseline snapshot of all environment variables. The output includes a JSON manifest with scan metadata (timestamp, file count, rule count).

2. **Execute a Targeted Audit on a Single File**  
   Use `deployguard audit --file src/config/database.js --depth 3 --include-secrets --include-deps` to run a focused scan on a specific file. The `--depth` parameter controls how many levels of nested imports or requires are resolved (default: 2). The tool prints a color-coded table with line numbers, vulnerability type, severity, and a one-line fix suggestion. For example: `Line 42 | Hardcoded Password | High | Replace with env variable: DB_PASSWORD`.

3. **Generate a Historical Compliance Report**  
   Run `deployguard report --since 2025-01-01 --until 2025-04-01 --format html --output ./reports/` to aggregate all scan results across your project’s git history. The report includes a trend graph showing vulnerability count over time, a breakdown by severity, and a list of regressions (vulnerabilities that were fixed but reappeared). It also compares each commit’s baseline against the current state to detect configuration drift.

### Best Practices

- **Integrate DeployGuard into Your Pre-Commit Hook**  
  Add `deployguard audit --file {staged_file} --fail-on-critical` to your `.git/hooks/pre-commit` script. This prevents any commit that introduces a critical vulnerability (e.g., hardcoded AWS secret) from being recorded, enforcing a zero-tolerance policy for high-severity issues at the developer’s desk rather than during CI.

- **Use Baseline Comparison for Environment Promotion**  
  Before promoting code from staging to production, run `deployguard audit --config .deployguard.yml --compare-baseline production-baseline.json`. This ensures that no configuration drift (e.g., a missing encryption flag on a database connection string) has been introduced between environments, reducing the risk of silent production failures due to misconfiguration.
