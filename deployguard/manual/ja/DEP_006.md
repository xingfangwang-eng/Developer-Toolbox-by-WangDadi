# deployguard - DEP_006

## 概要

DeployGuardは、ソロ開発者向けに設計された高精度なコードセキュリティ監査ツールです。静的解析エンジンとリアルタイム脆弱性検出を融合し、デプロイ前に重大なセキュリティリスクを自動排除します。キーワード：Code Security Audit, Solo Developer Tools, Static Analysis Tool

## 詳細な内容

### DeployGuardとは

DeployGuardは、コードベースの静的特性をミリ秒単位でスキャンし、OWASP Top 10に基づく脆弱性パターンを検出するプロフェッショナルツールです。独自のルールエンジンにより、偽陽性率を業界平均の15%未満に抑制し、ソロ開発者でもエンタープライズレベルのセキュリティ監査を実現します。

### コア機能

- **リアルタイム依存関係脆弱性スキャン**  
  パッケージマニフェスト（`package.json`、`requirements.txt`）を監視し、既知のCVEデータベースと照合。`--cve-db-update-interval 3600` パラメータで更新間隔を秒単位で制御可能。検出結果は `deployguard_report.json` に出力され、重大度（Critical/High/Medium/Low）に分類。

- **カスタムルールベースの静的解析**  
  `deployguard.rules` ファイルにYAML形式でルールを記述。正規表現パターンやASTノードタイプを指定し、機密情報（APIキー、DB接続文字列）のハードコーディングを検出。解析速度は1ファイルあたり平均12ミリ秒（10万行のコードベースで約2秒）。

- **デプロイ前自動ゲートチェック**  
  CI/CDパイプラインに統合可能な `deployguard check --exit-code` オプション。重大度「High」以上の脆弱性を検出した場合、終了コード `1` を返し、デプロイをブロック。PostgreSQLとの連携により、過去の監査結果を `deployguard_audit_log` テーブルに保存し、トレンド分析を実行。

### 使い方

1. **初期セットアップとルール設定**  
   `deployguard init --project-dir ./myapp --rules-template strict` を実行。カレントディレクトリに `deployguard.rules` と `.deployguardignore` が生成される。`--rules-template` で `strict`（厳格）、`moderate`（標準）、`custom`（カスタム）を選択可能。

2. **単一ファイルのセキュリティ監査**  
   `deployguard scan --file ./src/auth.js --format json --output ./audit_result.json` を実行。`--format` で `json`、`csv`、`html` を指定可能。結果には脆弱性の行番号、該当コードスニペット、推奨修正方法が含まれる。

3. **CI/CDパイプラインへの統合**  
   GitHub Actions用に `deployguard-action` を利用。`.github/workflows/deployguard.yml` に以下を記述：
   ```yaml
   - name: Run DeployGuard
     uses: xingfangwang-eng/deployguard-action@v1
     with:
       scan-path: './src'
       exit-on-high: true
       notify-slack: ${{ secrets.SLACK_WEBHOOK_URL }}
   ```
   これにより、プルリクエスト作成時に自動スキャンが実行され、Slackに通知が送信される。

### ベストプラクティス

- **ルールセットの段階的適用**  
  初回導入時は `moderate` テンプレートを使用し、偽陽性を確認しながら `strict` に移行する。週次で `deployguard.rules` をレビューし、新たな脆弱性パターン（例：Log4j系のJNDIインジェクション）を追加する。

- **監査結果のバージョン管理**  
  `deployguard_audit_log` テーブルに保存された履歴を活用し、各リリースのセキュリティスコア（`deployguard score --since v2.0.0`）を追跡する。スコアが5%以上低下した場合、即座にホットフィックスブランチを作成する。
