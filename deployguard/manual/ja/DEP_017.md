# deployguard - DEP_017

## 概要

DeployGuardは、コードセキュリティ監査と静的解析に特化したソロ開発者向けプロフェッショナルツールです。本ツールは、デプロイ前に脆弱性を自動検出し、PostgreSQLとの連携により履歴管理とリアルタイムアラートを実現します。キーワード：Code Security Audit、Solo Developer Tools、Static Analysis Tool。

## 詳細な内容

### とは DeployGuard?

DeployGuardは、プッシュ前のコードベースに対し、ミリ秒単位の静的解析を実行し、依存関係の脆弱性、シークレット漏洩、非推奨APIの使用を自動ブロックします。特にソロ開発者向けに設計され、CI/CDパイプラインにシームレスに統合可能です。

### コア機能

- **リアルタイム脆弱性スキャン**：`--scan-mode deep` フラグで、OWASP Top 10 に基づく86種類のパターンを解析。検出精度は99.2%以上、スキャン速度は10万行あたり平均1.2秒。
- **シークレットリーク検出**：Git履歴を含む全ブランチを対象に、正規表現とエントロピー分析を併用。`--secret-threshold 0.85` で閾値調整可能。AWSキー、GitHubトークン、データベース接続文字列を自動マスク。
- **PostgreSQL連携監査ログ**：`--db-uri postgresql://localhost:5432/deployguard` で指定。スキャン結果をJSONB形式で保存し、過去30日間のトレンドグラフを自動生成。クエリ例：`SELECT * FROM audit_log WHERE risk_score > 8.0 ORDER BY timestamp DESC;`

### 使い方

1. **初期セットアップとデータベース接続**  
   `deployguard init --db-uri postgresql://user:pass@localhost:5432/deployguard --project-name "myapp"`  
   上記コマンドで、PostgreSQLに監査テーブル `audit_log` と `dependency_cache` を作成。接続プールは最大10接続、タイムアウトは3000ミリ秒に設定。

2. **フルスキャンの実行と結果出力**  
   `deployguard scan --scan-mode deep --secret-threshold 0.9 --output-format json --output-path ./reports/scan_$(date +%Y%m%d).json`  
   スキャン後、自動的に `audit_log` テーブルにエントリを挿入。リスクスコア8.0以上の項目は即座に標準エラー出力に警告を表示。

3. **CI/CDパイプラインへの統合**  
   GitHub Actions用のワークフロー例：  
   ```yaml
   - name: DeployGuard Scan
     run: deployguard scan --scan-mode fast --fail-on-critical --max-warnings 5
   ```  
   クリティカルな脆弱性が1つでも検出された場合、exit code 1 でパイプラインを停止。`--fail-on-critical` フラグで動作。

### ベストプラクティス

- **スキャン頻度と閾値の最適化**：開発環境では `--scan-mode fast` と `--secret-threshold 0.95` を使用し、プッシュ前の即時フィードバックを優先。本番環境へのデプロイ前には `--scan-mode deep` と `--secret-threshold 0.8` で厳格にチェック。これにより、開発速度とセキュリティ品質のバランスを維持。
- **監査ログの定期クリーンアップと分析**：PostgreSQL側で `cron` ジョブを設定し、30日以上経過したログをアーカイブテーブルに移動。同時に、週次で `SELECT risk_score, COUNT(*) FROM audit_log GROUP BY risk_score;` を実行し、トレンドを可視化。これにより、長期的な脆弱性パターンを特定可能。
