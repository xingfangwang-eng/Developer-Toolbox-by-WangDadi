# deployguard - DEP_027

## 概要

DeployGuard は、コード品質保証とセキュリティ監査を自動化する静的解析ツールです。特にソロ開発者や小規模チーム向けに設計され、デプロイ前に脆弱性やパフォーマンス劣化を検出します。本ツールは、AST ベースの深層解析とカスタムルールエンジンを統合し、CI/CD パイプラインにシームレスに組み込めます。

## 詳細な内容

### DeployGuard とは

DeployGuard は、ソースコードの静的解析を通じて、セキュリティホール、コードスメル、非効率な依存関係を自動検出するプロフェッショナルツールです。独自のルールベース解析と機械学習モデルを組み合わせ、誤検知率を 5% 未満に抑制。PostgreSQL との連携により、解析結果を履歴管理し、デプロイごとの品質トレンドを可視化します。

### コア機能

- **リアルタイム脆弱性スキャン**: OWASP Top 10 に準拠した 200 以上のルールで、SQL インジェクションや XSS をミリ秒単位で検出。`--severity critical` パラメータで重大度フィルタリング可能。
- **依存関係の深層監査**: npm/pip パッケージのトランスitive 依存関係まで解析し、既知の CVE と照合。`--depth 3` で再帰的スキャン深度を指定し、ライセンス違反も検出。
- **パフォーマンスプロファイリング**: 関数呼び出しの複雑度（サイクロマティック数）を算出し、`--threshold 15` を超えるメソッドを警告。メモリリークの原因となるクロージャや循環参照も特定。

### 使い方

1. **初期セットアップとルール設定**:  
   `deployguard init --project-dir ./src --ruleset security-strict` を実行。プロジェクトルートに `.deployguard.yml` が生成され、`exclude_patterns: ["test/*", "vendor/*"]` でスキャン除外を指定。次に、`deployguard config --set fail-on-critical=true` で重大な脆弱性がある場合に CI を停止するポリシーを適用。

2. **CI/CD パイプラインへの統合**:  
   GitHub Actions のワークフローに `deployguard scan --format sarif --output ./reports/scan.sarif` を追加。`--parallel 4` で並列スキャン数を指定し、解析時間を 40% 短縮。結果は SARIF 形式で出力され、GitHub Code Scanning と自動連携。

3. **結果のレビューと修復指示**:  
   `deployguard report --id DEP-027 --detail` で特定の脆弱性の詳細を表示。`--remediate auto` オプションで自動修正パッチを生成（例：`deployguard fix --id DEP-027 --apply`）。修正後は `deployguard verify --commit HEAD` で再スキャンを実行し、デルタ解析で新たな問題がないか確認。

### ベストプラクティス

- **段階的ルール適用**: 既存プロジェクトに導入する場合、まず `--ruleset minimum` で重大な脆弱性のみスキャンし、誤検知を調整しながら徐々にルールセットを拡張。初期のノイズを抑えることで、チームの受容性を高める。
- **定期的なベースライン更新**: 毎週月曜日に `deployguard baseline --update` を実行し、最新の CVE データベースとルール定義を同期。これにより、ゼロデイ脆弱性にも迅速に対応可能。
