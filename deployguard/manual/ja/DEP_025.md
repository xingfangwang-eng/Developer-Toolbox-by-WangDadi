# deployguard - DEP_025

## 概要

DeployGuard は、コードセキュリティ監査、静的解析、デプロイ前チェックを統合した、個人開発者向けの次世代型静的解析ツールです。特に、CI/CD パイプラインに組み込むことで、脆弱性の早期発見とデプロイ時のリスク低減を実現します。

## 詳細な内容

### DeployGuard とは？

DeployGuard は、Go 言語で実装された高性能な静的解析エンジンを搭載。依存関係の透過的スキャン、シークレット情報の漏洩検知、カスタムルールに基づくコードレビューを、ミリ秒単位で実行します。PostgreSQL との連携により、過去の監査結果を蓄積・比較し、セキュリティスコアの推移を可視化します。

### コア機能

- **動的依存関係スキャン (DDS)**  
  プロジェクトの `go.mod`、`package.json`、`requirements.txt` などをリアルタイムで解析。既知の脆弱性（CVE）データベースと照合し、危険度（Critical/High/Medium/Low）を 100ms 以内に判定。`--cve-db-update-interval=24h` でデータベース更新間隔を設定可能。

- **シークレットリーク検知 (SLD)**  
  `.env` ファイルやコード内に埋め込まれた API キー、トークン、パスワードを、正規表現とエントロピー解析のハイブリッド手法で検出。`--secret-patterns=custom_patterns.json` で独自パターンを追加可能。検出感度は `--sensitivity=high` で調整。

- **デプロイ前チェックサム検証 (PCV)**  
  デプロイ対象のバイナリやコンテナイメージの SHA-256 ハッシュを自動生成。前回のデプロイ時と比較し、改ざんやビルドミスを検出。`--checksum-db=postgres://user:pass@localhost:5432/deployguard` で PostgreSQL に結果を保存。検証失敗時は `--rollback-on-failure=true` で自動ロールバック。

### 使い方

1. **初期設定とデータベース接続**  
   ターミナルで `deployguard init --db=postgres://user:pass@localhost:5432/deployguard` を実行。PostgreSQL にテーブルが自動作成され、過去の監査データを保存可能に。`--cve-db-path=./cve_data` でローカル CVE データベースのパスを指定。

2. **CI/CD パイプラインへの統合**  
   GitHub Actions の YAML に以下を追加：  
   ```yaml
   - name: DeployGuard Scan
     run: deployguard scan --path=./src --output=json --fail-on=high
   ```  
   `--fail-on=high` により、High 以上の脆弱性が検出された場合にパイプラインを停止。結果は `deployguard_report.json` に出力。

3. **結果の可視化とレポート生成**  
   `deployguard report --from=2024-01-01 --to=2024-12-31 --format=html` を実行。PostgreSQL に保存されたデータから、脆弱性の推移グラフ、修正推奨リスト、スコアカードを含む HTML レポートを生成。`--output-dir=./reports` で出力先を指定。

### ベストプラクティス

- **デプロイ前の全チェックを自動化**  
  デプロイのたびに手動でチェックするのではなく、CI/CD パイプラインに DeployGuard を組み込み、コミットごとに自動スキャンを実行。`--fail-on=critical` に設定し、Critical な脆弱性がある場合はデプロイをブロック。これにより、本番環境へのリスクを最小化。

- **カスタムルールで組織固有のポリシーに対応**  
  業界標準のルールセットに加え、`--custom-rules=./org_rules.yaml` で自社のセキュリティポリシーを定義。例えば、「AWS キーは必ず環境変数から取得する」などのルールを追加。ルールは YAML 形式で記述し、チーム内で共有可能。
