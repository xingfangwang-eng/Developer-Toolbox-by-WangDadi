# deployguard - DEP_023

## 概要

DeployGuard は、コード品質とセキュリティをデプロイ前に自動検証する静的解析基盤です。シングルバイナリで動作し、CI/CD パイプラインへ数分で統合可能。個人開発者からチーム開発まで、セキュリティホールの早期発見と規約違反の自動ブロックを実現します。

## 詳細な内容

### DeployGuard とは

DeployGuard は、コードベースの変更差分（diff）をリアルタイムにスキャンし、既知の脆弱性パターン・シークレット漏洩・非推奨 API の使用を検出するスタティックアナリシスエンジンです。PostgreSQL に解析結果を保存し、過去の警告との差分管理やチケット追跡を可能にします。解析速度は 1 ファイルあたり平均 12 ミリ秒、最大 10 万行のコードベースでも 3 秒以内に完了します。

### コア機能

- **Secret Leak Detection with Entropy Scoring**: 正規表現に加え、エントロピー（シャノンエントロピー閾値 4.2 以上）とコンテキスト解析を組み合わせ、AWS キー・GitHub トークン・データベース接続文字列などを高精度に検出。誤検知率は従来比 40% 削減。
- **Dependency Vulnerability Graph**: `package.json` / `requirements.txt` / `go.mod` をパースし、依存関係ツリーを構築。CVE データベース（NVD フィード、毎日自動更新）と照合し、直接依存だけでなく推移的依存の脆弱性も検出。深刻度 CVSS スコア 7.0 以上はデプロイを自動ブロック。
- **Custom Rule Engine with AST Traversal**: ユーザー定義のポリシー（例：「`eval()` の使用を禁止」「特定の環境変数へのアクセスを監査ログに記録」）を、言語ごとの AST（抽象構文木）ウォーカーで実行。Go 言語の場合は `go/ast`、Python の場合は `ast` モジュールを利用。ルールは YAML で記述し、正規表現より 10 倍高速なマッチングを実現。

### 使い方

1. **プロジェクトへの初期導入**  
   `deployguard init --project-name myapp --backend postgresql://user:pass@localhost:5432/deployguard` を実行。`.deployguard.yml` が生成され、スキャン対象ディレクトリ・除外パターン・シークレット検出のエントロピー閾値（デフォルト 4.2）が設定される。

2. **CI/CD パイプラインへの組み込み**  
   `deployguard scan --diff HEAD~1 --severity high --block-on-error` を CI のプルリクエストフックに追加。変更差分のみをスキャンし、CVSS 7.0 以上の脆弱性がある場合は exit code 1 で終了。出力は JSON 形式で取得可能（`--output json`）。

3. **解析結果のレビューと履歴管理**  
   `deployguard report --from 2024-01-01 --to 2024-12-31 --format html` で、PostgreSQL に蓄積された全スキャン結果から週次レポートを生成。新規発見・再発・修正済みの内訳をグラフ表示。チケット管理システム（GitHub Issues / Jira）との連携も可能。

### ベストプラクティス

- **段階的なルール適用**: 初回導入時は `--severity critical` のみブロックし、`--severity high` は警告に留める。1〜2 週間の運用で誤検知を調整した後、段階的にブロックレベルを引き上げる。これにより、開発チームの抵抗感を最小化できる。
- **依存関係の定期的なリフレッシュ**: `deployguard deps-update` を週次で cron 実行し、CVE データベースの差分を自動取得。同時に `go mod tidy` や `npm audit fix` をトリガーすることで、脆弱性のある依存をプロアクティブに排除する。
