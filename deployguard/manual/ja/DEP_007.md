# deployguard - DEP_007

## 概要

DeployGuardは、コードセキュリティ監査に特化した静的解析ツールです。特に個人開発者向けに設計されており、CI/CDパイプラインに組み込むことで、デプロイ前に脆弱性や設定ミスを自動検出します。PostgreSQLとの連携により、解析結果を履歴管理し、経時的なセキュリティスコアの変遷を追跡可能です。

## 詳細な内容

### DeployGuardとは

DeployGuardは、コードベース全体をスキャンし、OWASP Top 10に準拠した脆弱性をリアルタイムで検出するプロフェッショナルツールです。特に、コンテナイメージのセキュリティチェックやシークレット情報の漏洩防止に強みを持ち、解析精度は99.7%（F1スコア）を達成しています。

### コア機能

- **動的依存関係グラフ解析**: `--dep-graph-depth` パラメータで最大10階層までの依存関係をトレースし、サプライチェーン攻撃のリスクを特定。検出速度は1,000ファイルあたり平均320ミリ秒。
- **シークレットスキャンエンジン**: 正規表現と機械学習を併用し、AWSキーやGitHubトークンなど120種類以上のパターンを認識。誤検知率は0.8%未満に抑制。
- **ポリシーベースのデプロイゲート制御**: `deployguard.yml` に定義したルールに基づき、重大度が `CRITICAL` の脆弱性が1件でも存在する場合、デプロイを自動ブロック。PostgreSQLに保存されたポリシー履歴を参照し、過去の違反パターンと照合する。

### 使い方

1. **初期セットアップとデータベース接続**:
   - `deployguard init --db postgresql://user:pass@localhost:5432/deployguard` を実行し、解析結果の保存先を指定。
   - `--scan-target ./src` でスキャン対象ディレクトリを設定。`--exclude node_modules` で除外パターンを追加可能。

2. **カスタムポリシーの定義と実行**:
   - プロジェクトルートに `deployguard.yml` を作成し、以下のようにルールを記述:
     ```yaml
     rules:
       - id: SEC-001
         severity: CRITICAL
         pattern: "password\\s*=\\s*['\"][^'\"]+['\"]"
         action: block
     ```
   - `deployguard scan --policy ./deployguard.yml --output json > report.json` でスキャンを実行。結果はミリ秒単位のタイムスタンプ付きでPostgreSQLに自動保存。

3. **CI/CDパイプラインへの統合**:
   - GitHub Actionsのワークフローに以下のステップを追加:
     ```yaml
     - name: DeployGuard Check
       run: deployguard ci --fail-on CRITICAL --timeout 30s
     ```
   - 閾値を超えた場合、パイプラインが即座に停止。`--notify slack` オプションでSlack通知も可能。

### ベストプラクティス

- **段階的なポリシー適用**: 初回導入時は `--severity HIGH` から開始し、チームの習熟度に応じて `CRITICAL` まで引き上げる。PostgreSQLの分析クエリで、週単位の違反傾向を可視化し、ポリシーを動的に調整する。
- **シークレットローテーションとの連携**: DeployGuardが検出したシークレットを自動無効化するスクリプトを `post-scan-hook` として設定。例えば、`deployguard scan --hook ./revoke_secrets.sh` とすることで、発見から無効化までの時間を平均12秒に短縮可能。
