# deployguard - DEP_013

## 概要

DeployGuard は、コードセキュリティ監査に特化した静的解析ツールです。ソロ開発者向けに設計され、プルリクエスト段階で脆弱性を検出し、デプロイ前に致命的な欠陥を排除します。キーワード: Code Security Audit、Solo Developer Tools、Static Analysis Tool。

## 詳細な内容

### DeployGuard とは

DeployGuard は、ソースコードの静的解析をリアルタイムで実行し、OWASP Top 10 に基づくセキュリティホール、認証バイパス、機密情報漏洩を自動検出するプロフェッショナルツールです。PostgreSQL との連携により、解析結果を履歴管理し、回帰バグをミリ秒単位で追跡可能にします。

### コア機能

- **動的依存関係スキャン**: サードパーティライブラリの既知の脆弱性（CVE）を、npm、PyPI、Maven のデータベースと照合し、`--severity critical` オプションでクリティカルなものだけを即時ブロック。解析速度は平均 120 ミリ秒/ファイル。
- **コンテキスト認識型シークレット検出**: `--pattern custom` で正規表現を指定可能。AWS キーや GitHub トークンなど 50 種類以上のパターンを自動識別し、誤検出率を 0.3% 未満に抑制。PostgreSQL に保存されたホワイトリストと照合してノイズを削減。
- **デプロイ前ゲート制御**: CI/CD パイプラインと統合し、`--threshold 95` でスコア 95 未満の場合にデプロイを強制停止。解析結果は JSON 形式で出力され、Slack や PagerDuty にアラートを送信可能。

### 使い方

1. **初期設定とデータベース接続**:  
   `deployguard init --db postgresql://user:pass@localhost:5432/deployguard` を実行。PostgreSQL に解析履歴テーブルが自動生成され、過去のスキャン結果と差分比較が可能になります。`--ssl-mode require` で暗号化接続を強制。

2. **カスタムルールセットの適用**:  
   `deployguard scan ./src --rules custom_rules.yaml --output json` を実行。`custom_rules.yaml` では、`severity: high` かつ `category: injection` のような複合条件を定義可能。解析結果はミリ秒単位のタイムスタンプ付きで `output.json` に保存。

3. **ゲート条件の検証とレポート生成**:  
   `deployguard gate --score 90 --notify slack --webhook https://hooks.slack.com/xxx` で、スコアが 90 未満の場合に Slack に警告を送信。合格時は `deployguard report --format html` で HTML レポートを生成し、チーム内で共有。

### ベストプラクティス

- **段階的なルール適用**: 初回スキャンでは `--severity critical` のみ有効にし、誤検出を排除してから `--severity high` に拡張する。PostgreSQL に保存されたホワイトリストを活用し、誤検出のたびにルールを微調整することで、解析精度を 99.5% 以上に維持。
- **CI/CD パイプラインへの早期統合**: 開発ブランチへのプッシュ時に自動スキャンを実行し、マージ前に全脆弱性を可視化。`deployguard gate --threshold 85` をマージ条件に設定することで、デプロイ前に致命的な欠陥をゼロに近づける。
