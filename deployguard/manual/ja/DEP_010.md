# deployguard - DEP_010

## 概要

DeployGuardは、個人開発者向けに設計されたコードセキュリティ監査・静的解析ツールです。デプロイ前のコードベースに対して、脆弱性スキャン、依存関係の整合性検証、環境変数の漏洩チェックをミリ秒単位で実行。セキュリティホールを未然に防ぎ、安全なデプロイを保証します。

## 詳細な内容

### DeployGuardが解決する課題

- デプロイ直後に発覚するシークレットキーのハードコード漏洩
- サードパーティライブラリの既知の脆弱性（CVE）を見逃すリスク
- 手動レビューによる見落としと人的ミスの削減

### コア機能

- **動的依存関係スキャン（DDS-5000）**  
  `package.json`、`requirements.txt`、`go.mod` など12種類のパッケージマニフェストを解析。各依存パッケージのバージョンをNVD（National Vulnerability Database）とリアルタイム照合し、CVSSスコアが7.0以上の脆弱性を検出した場合、即座にデプロイをブロック。検出精度は99.7%、遅延は平均80ミリ秒。

- **環境変数リーク検出エンジン（EVL-200）**  
  コード内の`.env`、`config.yml`、環境変数参照パターン（`process.env.*`、`os.Getenv`）を正規表現とAST（抽象構文木）で解析。Git管理下に誤ってコミットされたシークレットキー（AWSアクセスキー、GitHubトークン、データベースパスワード）を検出し、該当行番号とコミットハッシュを出力。スキャン範囲は最大10万行のコードを3秒未満で完了。

- **静的コードプロファイリング（SCP-100）**  
  Python、JavaScript、Go、Rustの4言語に対応。未使用のインポート、到達不能コード、型の不一致を検出し、修正候補を自動生成。特にデプロイ前のコード品質スコアを0〜100で算出し、閾値（デフォルト80）を下回る場合は警告を発する。スコアリングには独自のヒューリスティック重み付けアルゴリズムを採用。

### 使い方

1. **初期設定とプロジェクトスキャン**  
   `deployguard init --project-dir ./myapp --language python` を実行。カレントディレクトリに`.deployguard.yml`が生成される。次に `deployguard scan --severity high --output json` で全コードベースのスキャンを開始。結果は`deployguard_report.json`に保存される。

2. **脆弱性レポートのレビューとフィルタリング**  
   `deployguard report --format html --filter CVE-2024-*` でHTML形式のレポートを生成。特定のCVE IDを指定してフィルタリング可能。レポートには影響を受けるファイルのパス、行番号、推奨アップグレードバージョンが記載される。

3. **CI/CDパイプラインへの統合**  
   GitHub Actionsのワークフローに以下を追加：  
   ```yaml
   - name: DeployGuard Check
     run: deployguard ci --block-on-error --threshold 90
   ```  
   `--block-on-error` フラグにより、脆弱性が検出された場合にパイプラインが停止。`--threshold` で品質スコアの下限を設定可能。

### ベストプラクティス

- **デプロイ前に必ずフルスキャンを実行する**  
  開発環境でのコミット前フック（`pre-commit`）に `deployguard scan --quick` を組み込み、差分のみのスキャンで高速チェック。ただし、本番デプロイ前には必ず `--full` フラグで全ファイルスキャンを実施し、見逃しをゼロにする。

- **脆弱性データベースの定期更新を自動化する**  
  `cron` ジョブで毎日 `deployguard update-db` を実行し、NVDの最新データを取得。これにより、ゼロデイ脆弱性にも迅速に対応可能。更新後の初回スキャンは必ず手動で確認し、誤検出がないかを検証する。
