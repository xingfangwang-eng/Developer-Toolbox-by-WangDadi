# deployguard - DEP_024

## 概要

DeployGuardは、コードセキュリティ監査と静的解析に特化した、ソロ開発者向けプロフェッショナルツールです。プッシュ前の脆弱性検出、依存関係の整合性チェック、デプロイ構成の自動検証を1コマンドで実行。CI/CDパイプラインへの組み込みを前提とした、軽量かつ高精度な静的解析エンジンを搭載しています。

## 詳細な内容

### コア機能

- **リアルタイム依存関係トレーサー**：`--dep-depth=5` オプションで最大5階層の推移的依存関係を解析。PostgreSQLとの連携により、過去30日間のバージョン変動をミリ秒単位で追跡し、既知の脆弱性（CVE）が混入した時点を特定します。

- **マルチターゲット・デプロイ構成バリデーター**：`--target-env=staging,production` で複数環境の設定ファイルを同時検証。環境変数の過不足、シークレットキーのハードコーディング、TLS証明書の有効期限（残日数表示）を自動検出。検出精度は99.7%以上。

- **ゼロダウンタイム・ロールバック予測エンジン**：`--rollback-sim=true` を指定すると、デプロイ前にロールバック手順のシミュレーションを実行。データベースマイグレーションの逆順適用が可能か、スキーマ変更によるデータ損失リスクを評価し、リスクスコア（0〜100）を算出します。

### 使い方

1. **プロジェクト初期化と解析設定**
   ```bash
   deployguard init --project-name=myapp --scan-mode=full --exclude=node_modules,dist
   ```
   実行後、`.deployguard/config.yml` が生成されます。ここで `threshold: 85`（脆弱性スコアの警告閾値）を設定。次に `deployguard analyze --ci-mode=true --output-format=sarif` で静的解析を実行。結果はJSONとSARIF形式で出力され、CI/CDツールとの統合が可能です。

2. **デプロイ前のセキュリティゲート通過**
   ```bash
   deployguard check --branch=main --commit-hash=$(git rev-parse HEAD) --block-on-critical=true
   ```
   クリティカルな脆弱性が1件でも検出された場合、終了コード1を返しデプロイをブロック。`--block-on-high=true` を追加すれば、Highレベルの脆弱性もブロック対象にできます。出力例：`CRITICAL: 2, HIGH: 5, MEDIUM: 12, PASS: 89%`

3. **ロールバック計画の自動生成と検証**
   ```bash
   deployguard rollback-plan --release-tag=v2.3.1 --target-env=production --dry-run=true
   ```
   生成されたロールバック手順を `rollback-v2.3.1.md` に保存。`--dry-run=false` で実際のロールバック実行前に、データベースのスナップショットとの差分確認を強制。差分がある場合は実行を中断し、詳細レポートを出力します。

### ベストプラクティス

- **プルリクエスト単位でのゲート設置**：GitHub ActionsやGitLab CIで、PR作成時に `deployguard check --ci-mode=true --block-on-critical=true` を自動実行。マージ前に脆弱性をゼロにすることで、デプロイ後のインシデントを80%削減できます。また、`--report-url=https://your-team.slack.com/archives/` を指定し、検出結果をSlack通知することも推奨します。

- **依存関係の定期監査と自動アップデート**：週次で `deployguard dep-audit --depth=5 --cve-db-update=auto --notify-on-new-cve=true` をcronジョブで実行。新しいCVEが公開された際、影響を受けるパッケージとその修正バージョンをメールまたはWebhookで即時通知。これにより、ゼロデイ脆弱性への対応時間を平均2時間以内に短縮できます。
