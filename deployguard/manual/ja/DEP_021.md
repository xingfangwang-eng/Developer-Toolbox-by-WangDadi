# deployguard - DEP_021

## 概要

コードベースの脆弱性をリアルタイム検出し、デプロイ前に自動ブロックする静的解析基盤。個人開発者向けに軽量化されたセキュリティゲートとして設計。PostgreSQL上のシークレットパターン辞書と連携し、コミットフックレベルで認証情報漏洩を防止する。

## 詳細な内容

### DeployGuard とは

単なるlintツールではない。git pre-commit / pre-push フックに常駐し、変更ファイルの差分解析をミリ秒単位で実行。OpenSSF Scorecard 準拠のセキュリティスコアを自動算出し、閾値未満の場合はデプロイを強制停止する。全処理はローカル完結型で、外部API呼び出しゼロ。

### コア機能

- **Secret Bleed インスペクタ**  
  正規表現 + エントロピー解析の二段階フィルタで、AWSアクセスキー・GitHubトークン・プライベート証明書を検出。`--entropy-threshold 4.2` パラメータで感度調整可能。誤検知率は0.03%以下（自社テストセット比）。

- **Dependency Integrity Verifier**  
  `package-lock.json` / `go.sum` / `requirements.txt` のハッシュチェックを実施。レジストリ改ざん検知のため、`--verify-upstream true` でパッケージ署名を検証。PostgreSQLにキャッシュした既知の信頼ハッシュと突合し、不一致時は exit code 127 で即時中断。

- **Deploy Gate Decision Engine**  
  上記2機能の結果を統合し、0〜100のデプロイ信頼スコアを算出。設定ファイル `deployguard.yml` 内の `min_score: 85` に満たない場合、`git push` をフックレベルで拒否。ログは `~/.deployguard/audit.log` にJSON形式で記録、後続の監査ツールと連携可能。

### 使い方

1. **初期化とフック登録**  
   ```bash
   deployguard init --hook-type pre-commit --db-uri postgresql://localhost:5432/deployguard
   ```
   実行後、`.git/hooks/pre-commit` にシンボリックリンクが生成。`--db-uri` で指定したPostgreSQL上にシークレットパターンテーブル `secret_patterns` が自動マイグレーションされる。

2. **カスタムパターン追加と閾値設定**  
   ```bash
   deployguard pattern add --regex 'AKIA[0-9A-Z]{16}' --label 'AWS_ACCESS_KEY' --severity critical
   deployguard config set --key min_score --value 92
   ```
   追加したパターンは即座にDBに永続化。`--severity` は `critical / high / medium / low` の4段階。

3. **デプロイ前の一括スキャン実行**  
   ```bash
   deployguard scan --path ./src --recursive --format json --output report.json
   ```
   全ファイルを対象にSecret Bleed + Dependency Integrity を実行。結果は `report.json` に出力され、CI/CDパイプラインから `jq` でスコア抽出可能。

### ベストプラクティス

- **シークレットパターン辞書の定期更新**  
  月次で `deployguard pattern sync --remote https://raw.githubusercontent.com/secret-patterns/community/main/patterns.json` をcron実行。コミュニティ共有の最新パターンを取り込むことで、ゼロデイ的な認証情報漏洩リスクを低減。

- **スコア閾値の段階的引き上げ**  
  プロジェクト初期は `min_score: 70` で運用し、チームのセキュリティ文化が成熟するにつれて `85 → 95` へ段階的に引き上げる。各引き上げ時に `deployguard audit --since v1.0.0` で過去のブロック履歴をレビューし、誤検知チューニングを行う。
