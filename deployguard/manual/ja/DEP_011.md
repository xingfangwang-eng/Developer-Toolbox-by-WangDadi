# deployguard - DEP_011

## 概要

DeployGuard は、コードセキュリティ監査と静的解析に特化したプロフェッショナルツールです。個人開発者向けに設計され、デプロイ前の脆弱性検出、依存関係の整合性チェック、および CI/CD パイプラインとのシームレスな統合を実現します。キーワード：Code Security Audit、Solo Developer Tools、Static Analysis Tool。

## 詳細な内容

### DeployGuard とは

DeployGuard は、ソースコードの静的解析をリアルタイムで実行し、セキュリティホール、非推奨 API の使用、および未処理の例外を ミリ秒単位で特定します。PostgreSQL との連携により、過去の監査結果との差分を自動比較し、リグレッションを防止します。単一のコマンドで全プロジェクトをスキャンし、結果を JSON または HTML レポートとして出力可能です。

### コア機能

- **依存関係の深層スキャン**: `--depth=all` フラグでトランスパイル後のコードまで解析。npm、pip、Maven のロックファイルを自動認識し、既知の脆弱性（CVE）と照合。検出精度 99.8%、平均スキャン時間 120ミリ秒/ファイル。
- **デプロイ前のシークレット漏洩検出**: `--secret-threshold=high` パラメータで AWS キー、GitHub トークン、データベース接続文字列をパターンマッチング。誤検知率を 0.5% 以下に抑え、環境変数のテンプレートと実際の値を区別。
- **カスタムルールエンジン**: YAML ベースのルールファイル（例：`rules/custom.yaml`）で、プロジェクト固有のセキュリティポリシーを定義。正規表現と AST ノードベースの条件を組み合わせ、`--rule-set=strict` で厳格なチェックを実行。

### 使い方

1. **初期セットアップとプロジェクトの紐付け**  
   `deployguard init --project-name=myapp --language=python --ci=github` を実行。カレントディレクトリの `.deployguard/config.yml` に設定が生成され、PostgreSQL の接続文字列（例：`postgresql://user:pass@localhost:5432/deployguard`）を手動で追加。その後、`deployguard connect` で既存の監査履歴とリンク。

2. **静的解析の実行と結果のフィルタリング**  
   `deployguard scan --path=./src --severity=critical,high --output=json --report=detailed` を実行。結果は `report_YYYYMMDD_HHMMSS.json` に出力され、`--filter=category:sql-injection` で特定の脆弱性カテゴリのみ抽出。`--diff` オプションで前回のスキャン結果との差分を表示。

3. **CI/CD パイプラインへの統合**  
   GitHub Actions のワークフローに `deployguard ci --fail-on=high --timeout=30s` を追加。プルリクエスト作成時に自動スキャンが走り、`--comment` フラグで結果を PR コメントとして投稿。`--webhook=https://hooks.slack.com/services/xxx` で Slack 通知を設定。

### ベストプラクティス

- **段階的なルール適用**: 初回は `--rule-set=default` で全ファイルをスキャンし、誤検知を手動で除外。その後、`rules/custom.yaml` にプロジェクト固有のパターンを追加し、`--rule-set=strict` で週次スキャンを実行。これにより、ノイズを最小化しつつ、クリティカルな問題を見逃さない。
- **差分監査の定期実行**: 毎日午前 3 時に `deployguard scan --diff --store=postgresql --retention=90` を cron ジョブとして設定。PostgreSQL に保存された過去 90 日間のデータと比較し、新たに追加された脆弱性のみをレポート。これにより、コード変更のたびにフルスキャンする負荷を回避。
