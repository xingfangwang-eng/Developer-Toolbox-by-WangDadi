# deployguard - DEP_018

## 概要

DeployGuard は、コード品質保証とセキュリティ監査を自動化するための静的解析ツールです。特に個人開発者向けに設計され、CI/CD パイプラインに組み込むことで、デプロイ前に脆弱性やパフォーマンス低下を引き起こすコードを検出・ブロックします。

## 詳細な内容

### DeployGuard とは

DeployGuard は、ソースコードの静的解析と依存関係のセキュリティスキャンを統合したプロフェッショナルツールです。Node.js、Python、Go など主要言語のプロジェクトに対応し、デプロイ前にコードベース全体のリスク評価を実行します。特に、個人開発者が見落としがちなサードパーティライブラリの脆弱性や、環境変数の誤設定を自動検出します。

### コア機能

- **ゼロデイ脆弱性スキャン**: 依存パッケージのバージョンと CVE データベース（NVD、OSV）をリアルタイム照合。検出後は `--block-on-critical` フラグでビルドを強制停止。スキャン時間は 100,000 依存関係あたり平均 1.2 秒（PostgreSQL キャッシュ利用時）。
- **環境変数リーク検出**: `.env` や CI 設定ファイル内のシークレットトークン（AWS_ACCESS_KEY_ID 等）を正規表現とエントロピー分析で検出。`--leak-threshold=0.85` で感度調整可能。誤検知率は 0.3% 以下。
- **デプロイ前差分監査**: 前回のデプロイ時とのコード差分を自動比較し、新たに追加された脆弱性パターン（SQL インジェクション、XSS など）をブロック。`audit --diff-base=main` でブランチ指定可能。差分解析はミリ秒単位で完了。

### 使い方

1. **プロジェクト初期化とルール設定**  
   ターミナルで `deployguard init --project-name=myapp --language=node` を実行。カレントディレクトリに `.deployguard.yml` が生成される。このファイル内で `severity: high` や `exclude-dirs: ["tests"]` など、スキャンルールをカスタマイズする。

2. **CI パイプラインへの統合**  
   GitHub Actions のワークフローに以下のステップを追加:  
   ```yaml
   - name: DeployGuard スキャン
     run: deployguard scan --ci --fail-on=high --output=json > report.json
   ```  
   `--fail-on=high` で重大度「高」以上の脆弱性がある場合にビルドを失敗させる。出力された JSON レポートは後続の通知ステップで利用可能。

3. **結果の確認と脆弱性修正**  
   スキャン完了後、`deployguard report --format=html --open` を実行すると、ブラウザで詳細レポートが開く。各脆弱性には修正推奨バージョンや該当コード行が表示される。修正後は `deployguard recheck --commit=abc123` で特定コミットのみ再スキャン可能。

### ベストプラクティス

- **スキャンはデプロイ前の必須ゲートとする**: CI パイプラインの最終ステージに DeployGuard を配置し、`--block-on-critical` フラグを有効にする。これにより、重大な脆弱性を含むコードが本番環境に到達するのを防ぐ。特に、個人開発ではセキュリティレビューの代替として機能する。
- **依存関係の自動更新と組み合わせる**: Dependabot や Renovate と併用し、DeployGuard が検出した脆弱性のあるパッケージを自動でアップグレードするワークフローを構築する。例えば、DeployGuard の JSON 出力をパースして、該当パッケージのマイグレーション PR を自動作成するスクリプトを用意する。
