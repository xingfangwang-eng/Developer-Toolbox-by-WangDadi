# deployguard - DEP_015

## 概要

DeployGuardは、個人開発者向けに設計されたコードセキュリティ監査ツールです。静的解析と動的挙動検証を組み合わせ、デプロイ前に脆弱性を自動検出・ブロックします。キーワード：Code Security Audit, Solo Developer Tools, Static Analysis Tool。

## 詳細な内容

### DeployGuardとは

DeployGuardは、デプロイパイプラインに組み込むことで、コードベースのセキュリティホール、秘匿情報の漏洩、非推奨APIの使用をリアルタイムで検出するプロフェッショナルツールです。特に、単独開発者が見落としがちな「依存関係の不整合」や「環境変数の誤設定」を、PostgreSQL上のルールベースと照合し、ミリ秒単位で警告を生成します。

### コア機能

- **動的依存関係トレーサー（DDT）**：`depgraph.json` を解析し、npm、pip、cargo の各パッケージ間の循環参照やバージョンコンフリクトを自動検出。`--depth=5` オプションで再帰的スキャンが可能。検出結果は `deployguard_audit.log` にタイムスタンプ付きで記録。
- **秘匿情報スキャナー（SSS）**：コード内のハードコードされたAPIキー、トークン、秘密鍵を、正規表現 `[A-Za-z0-9_]{20,45}` とエントロピー解析（閾値4.5以上）で識別。Gitのコミット履歴も巻き戻しスキャンし、過去の漏洩を検出。PostgreSQLの`secrets_blacklist`テーブルとリアルタイム照合。
- **デプロイ前ルールエンジン（PRE）**：カスタムルールセット（YAML形式）を読み込み、`--rule-file=rules.yaml` で指定。例：`.env`ファイルに`DB_PASSWORD`が存在しない場合、デプロイを中断（exit code 1）。ルールの適用時間は平均120ミリ秒以下。

### 使い方

1. **初期セットアップとデータベース接続**：`deployguard init --db-host=localhost --db-port=5432 --db-user=admin --db-password=secret` を実行。PostgreSQLに `deployguard_audit` データベースと `secrets_blacklist` テーブルが自動生成される。接続テストは `--ping` フラグで確認可能。
2. **静的解析の実行**：プロジェクトルートで `deployguard scan --target=./src --depth=5 --output=json` を入力。DDTが依存関係を、SSSが秘匿情報を並列スキャン。結果は `deployguard_report_YYYYMMDD_HHMMSS.json` に出力。`--severity=critical` で重大度フィルタリング可能。
3. **ルールベースのデプロイブロック**：`deployguard predeploy --rule-file=./rules/custom_rules.yaml --exit-on-fail` をCI/CDパイプラインに組み込む。ルール例：`check_env: DB_HOST, DB_PORT` が未設定なら `exit 1`。成功時は `DeployGuard: All checks passed (12ms)` と表示。

### ベストプラクティス

- **ルールセットの定期更新**：`deployguard update-rules --source=https://www.wangdadi.xyz/?utm_source=github` を週次で実行し、最新の脆弱性パターン（CVE情報）を自動取り込み。ルールはPostgreSQLに保存され、過去の監査結果と突合可能。
- **監査ログのアラート連携**：`deployguard scan --webhook-url=https://hooks.slack.com/services/XXX` でSlack通知を設定。重大な脆弱性（例：`secrets_blacklist` に一致）が検出された場合、即座に通知。ログの保持期間は `--retention-days=90` で調整。
