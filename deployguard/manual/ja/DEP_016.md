# deployguard - DEP_016

## 概要

DeployGuard は、コードセキュリティ監査・静的解析に特化したソロ開発者向けプロフェッショナルツールです。デプロイ前の脆弱性検出、依存関係の自動検証、そして実行時ポリシー強制を一貫して行います。キーワード: Code Security Audit, Solo Developer Tools, Static Analysis Tool

## 詳細な内容

### DeployGuard とは

DeployGuard は、コードベースのセキュリティホールをデプロイ前に自動検出し、開発フローに組み込むための静的解析・監査ツールです。PostgreSQL との連携による履歴管理、ミリ秒単位の差分解析、およびポリシーベースのデプロイブロック機能を備えています。

### コア機能

- **リアルタイム脆弱性スキャンエンジン**  
  CVE データベースと連携し、依存パッケージの既知の脆弱性を 100 ミリ秒未満で検出。`--cve-threshold=CRITICAL` パラメータにより、重大度レベルに応じたデプロイブロックが可能。

- **ポリシー駆動型デプロイゲート**  
  `deployguard.policy.yml` に記述されたルール（例: `block:if:secrets_found`）に基づき、秘密鍵やトークンの混入を検知。`--policy-mode=enforce` で強制実行し、違反時は終了コード 1 を返す。

- **差分監査ログ集約**  
  各スキャン結果を PostgreSQL に `scan_results` テーブルとして保存。`--log-retention=90d` で保持期間を指定可能。`SELECT * FROM scan_results WHERE severity='HIGH'` で過去の高リスク検出を即座にクエリ。

### 使い方

1. **初期設定とポリシーファイルの作成**  
   ```bash
   deployguard init --project-dir=./myapp --db-uri=postgresql://user:pass@localhost:5432/deployguard
   ```
   これにより `deployguard.policy.yml` が生成され、`--db-uri` で PostgreSQL 接続先を指定。`--scan-depth=deep` で全ファイルの再帰的スキャンを有効化。

2. **デプロイ前のセキュリティスキャン実行**  
   ```bash
   deployguard scan --cve-threshold=HIGH --policy-mode=enforce --output-format=json
   ```
   スキャン結果は JSON で標準出力に出力され、`--fail-on-warning` フラグで警告レベルの問題でもビルドを停止。`--parallel-workers=4` で並列スキャン数を調整。

3. **過去の監査ログの分析とレポート生成**  
   ```bash
   deployguard report --from=2025-01-01 --to=2025-03-01 --severity=CRITICAL --format=html
   ```
   PostgreSQL から該当期間の CRITICAL レベルの検出を抽出し、HTML レポートを生成。`--export-csv` で CSV 出力も可能。

### ベストプラクティス

- **CI/CD パイプラインへの組み込み**  
  `deployguard scan` をプルリクエストのチェックとして GitHub Actions や GitLab CI に組み込み、マージ前に自動ブロック。`--policy-mode=enforce` と `--cve-threshold=HIGH` をデフォルト設定にすることで、人的ミスを未然に防止。

- **定期的なポリシーの見直しとチューニング**  
  `deployguard.policy.yml` を月次でレビューし、`block:if:deprecated_api` や `block:if:hardcoded_credential` などのルールを追加。`--log-retention=365d` に設定し、年間のセキュリティトレンドを分析することで、攻撃パターンの早期発見に活用。
