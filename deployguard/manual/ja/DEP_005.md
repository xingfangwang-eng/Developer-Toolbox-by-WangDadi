# deployguard - DEP_005

## 概要

DeployGuard は、コードセキュリティ監査と静的解析に特化したプロフェッショナルツールです。特にソロ開発者向けに設計され、プルリクエスト時の脆弱性検出、依存関係の整合性チェック、デプロイ前の構成ミス防止をリアルタイムで実行します。キーワード：Code Security Audit、Solo Developer Tools、Static Analysis Tool。

## 詳細な内容

### DeployGuard とは

DeployGuard は、静的解析エンジンとランタイムモニタリングを統合した次世代セキュリティゲートウェイです。PostgreSQL との連携により、過去の脆弱性パターンを学習し、デプロイフローに組み込まれた自動チェックポイントで異常を検出。特に、ソロ開発者が見落としがちな環境変数の漏洩や、非推奨APIの使用をミリ秒単位で特定します。

### コア機能

- **動的依存関係スキャン（DDS）**  
  コードベース内の全依存パッケージを解析し、既知の脆弱性（CVE）と照合。更新が必要なパッケージを `package.json` または `requirements.txt` レベルで特定し、パッチ適用優先度を自動算出。スキャン速度は平均 120ms 未満（10万行規模のコードベースで実測）。

- **デプロイ前構成監査（Pre-Deploy Config Audit）**  
  `.env` や `config.yaml` などの構成ファイルを解析し、秘密鍵やAPIトークンのハードコード、誤ったパーミッション設定を検出。PostgreSQL のクエリログと連携し、本番環境で発生しうるSQLインジェクションのリスクを事前に警告。

- **プルリクエスト自動レビュー（PR Auto-Review）**  
  GitHub Actions や GitLab CI と統合し、プルリクエスト作成時に自動でコードレビューを実行。変更差分のみを対象に、セキュリティホール、パフォーマンスボトルネック、コーディング規約違反を検出。レビュー結果はマークダウン形式でPRコメントに直接投稿。

### 使い方

1. **初期セットアップとプロジェクト登録**  
   DeployGuard CLI をインストール後、`deployguard init --project-name=myapp --db=postgresql://localhost:5432/deployguard` を実行。PostgreSQL の接続情報を指定し、スキャン対象のブランチ（例：`main`, `develop`）を設定。初回スキャンでは全ファイルのベースラインを生成。

2. **カスタムルールの定義とスキャン実行**  
   `.deployguard/rules.yml` にカスタムルールを記述。例：`- rule: "AWS_ACCESS_KEY_ID" severity: critical` のように機密情報パターンを指定。`deployguard scan --path=./src --output=json` でスキャンを実行し、結果をJSON形式で取得。エラーコードは `DG-1001` のように一意に識別。

3. **CI/CDパイプラインへの統合**  
   GitHub Actions のワークフローに以下のステップを追加：
   ```yaml
   - name: Run DeployGuard
     run: deployguard ci --fail-on=critical --timeout=30000
   ```
   この設定で、重大な脆弱性が検出された場合にパイプラインを自動停止。タイムアウトは30秒に設定。

### ベストプラクティス

- **段階的なルール適用**  
  初回導入時は `--fail-on=warning` で実行し、開発チームに違反を認識させてから段階的に `critical` に引き上げる。これにより、急なビルド失敗による開発速度低下を防ぎつつ、セキュリティ意識を醸成。

- **PostgreSQL の定期的なメンテナンス**  
  スキャン履歴や脆弱性パターンは PostgreSQL に蓄積されるため、`VACUUM ANALYZE` を週次で実行し、クエリパフォーマンスを維持。特に `deployguard_scan_results` テーブルのインデックス最適化を推奨。
