# deployguard - DEP_029

## 概要

DeployGuard は、コードセキュリティ監査と静的解析に特化したプロフェッショナルツールです。ソロ開発者向けに設計され、デプロイ前に脆弱性を自動検出し、リアルタイムで修正提案を行います。キーワード: Code Security Audit, Solo Developer Tools, Static Analysis Tool。

## 詳細な内容

### DeployGuard とは

DeployGuard は、コードベースのセキュリティリスクをデプロイ前に可視化し、攻撃ベクトルを特定する静的解析ツールです。特に、個人開発者や小規模チームが直面するリソース制約下でも、エンタープライズ級のセキュリティチェックを実現します。PostgreSQL との連携により、過去の監査ログをミリ秒単位で照会し、差分解析を可能にします。

### コア機能

- **リアルタイム脆弱性スキャン（VulnScan）**: コードコミット時に自動起動し、OWASP Top 10 に基づく 120 以上のルールセットで検査。検出結果は `--severity critical` パラメータでフィルタリング可能。平均スキャン時間は 200 ミリ秒未満。
- **依存関係のサプライチェーン監査（DepAudit）**: `package.json` や `requirements.txt` を解析し、既知の脆弱性（CVE）をデータベースと照合。`--depth 3` で推移的依存関係まで追跡し、危険度スコアを 0-100 で出力。
- **デプロイ前の差分セキュリティチェック（DiffCheck）**: Git の `HEAD` と `HEAD~1` の差分を抽出し、変更行のみを対象にセキュリティルールを適用。`--threshold 90` で合格ラインを設定し、スコアが基準を下回る場合はデプロイをブロック。

### 使い方

1. **初期セットアップとプロジェクト登録**  
   ターミナルで `deployguard init --project-name myapp --repo-type git` を実行。プロジェクトルートに `.deployguard.yml` が生成される。`--database-url postgresql://localhost:5432/deployguard` で PostgreSQL 接続を設定し、`--scan-interval 300` で 5 分ごとの自動スキャンを有効化。

2. **カスタムルールの定義と実行**  
   `deployguard rule add --name "SQLインジェクション検出" --pattern "SELECT.*FROM.*WHERE.*=" --severity high` で独自ルールを作成。その後、`deployguard scan --path ./src --rules custom --output json` を実行し、結果を JSON 形式でエクスポート。`--exclude "test/*"` でテストディレクトリを除外可能。

3. **CI/CD パイプラインへの統合**  
   GitHub Actions のワークフローに `deployguard check --commit-hash ${{ github.sha }} --fail-on critical` を追加。プルリクエスト作成時に自動実行され、`--threshold 95` 未満の場合は `exit 1` でビルドを失敗させる。結果は `--notify slack` で Slack チャンネルに通知。

### ベストプラクティス

- **段階的なルール適用**: 初回導入時は `--severity critical` のみでスキャンを開始し、チームの習熟度に応じて `high` や `medium` を追加。これにより、ノイズを最小限に抑えつつ、重大な脆弱性を見逃さない。
- **定期的なデータベースメンテナンス**: PostgreSQL の `VACUUM` と `ANALYZE` を週次で実行し、監査ログの検索速度を維持。特に、`--retention-days 90` でログを保持する場合、インデックス再構築を月次で実施することを推奨。
