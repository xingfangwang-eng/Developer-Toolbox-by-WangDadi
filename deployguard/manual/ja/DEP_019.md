# deployguard - DEP_019

## 概要

DeployGuardは、コード品質とセキュリティをデプロイ前に徹底検証する静的解析エンジンです。個人開発者向けに設計され、CI/CDパイプラインに組み込むことで、脆弱性の早期発見とコードベースの健全性維持を実現します。

## コア機能

### 1. リアルタイム依存関係脆弱性スキャン（VulnTrace）
- `--vuln-db-update-interval` パラメータで更新間隔をミリ秒単位で指定可能（デフォルト: 3600000ms）
- OWASP Top 10およびCVEデータベースと連携し、npm/pip/goモジュールの既知の脆弱性を自動検出
- 検出結果は `deployguard_vuln_report.json` に出力、重大度（CRITICAL/HIGH/MEDIUM/LOW）でフィルタリング可能

### 2. マルチランタイム構文検証（PolyCheck）
- `--runtime` フラグでNode.js 18/20、Python 3.9/3.11、Go 1.21/1.22を同時指定可能
- 各ランタイムの非互換構文（例: Node.jsのESM/CJS混在、Pythonのasync/await非互換）を静的解析
- 解析速度は1ファイルあたり平均12ms、1000ファイルのプロジェクトを12秒以内で完了

### 3. デプロイ前設定ファイル整合性監査（ConfigGuard）
- `--config-pattern` で監視対象ファイルを正規表現指定（例: `.*\.(yaml|json|env)$`）
- 環境変数の未定義参照、YAMLのインデントミス、JSONのキー重複を自動修正提案
- PostgreSQLやRedisの接続文字列フォーマットを検証し、不正なポート番号を検出（例: `--port-range 1024-65535`）

## 使い方

### ステップ 1: 初期セットアップと設定ファイル生成
```bash
deployguard init --project-name my-app --ci-provider github-actions --output .deployguard.yml
```
このコマンドで `.deployguard.yml` が生成されます。`scan-depth: deep` を追加すると、全依存関係の推移的スキャンが有効になります。

### ステップ 2: カスタムルールと閾値の定義
```yaml
# .deployguard.yml
rules:
  vuln-scan:
    severity-threshold: HIGH
    ignore-patterns: ["test/**", "examples/**"]
  config-audit:
    required-files: ["docker-compose.yml", ".env.production"]
    port-validation: true
```
`--validate-only` フラグで設定ファイルの文法チェックのみを実行可能。

### ステップ 3: CI/CDパイプラインへの統合とレポート生成
```bash
deployguard scan --config .deployguard.yml --output-format html --report-dir ./reports
```
GitHub Actionsでは以下のスニペットで自動実行:
```yaml
- name: Run DeployGuard
  run: deployguard scan --exit-code --fail-on CRITICAL
```
`--exit-code` で脆弱性検出時に非ゼロ終了コードを返し、パイプラインを停止。

## ベストプラクティス

### プラクティス 1: 段階的な閾値適用による誤検知の最小化
初期導入時は `--severity-threshold CRITICAL` で重大な脆弱性のみ検出し、チームの習熟度に応じて `HIGH` → `MEDIUM` と段階的に厳しくする。`--baseline` フラグで既存の既知問題を無視リストに追加可能。

### プラクティス 2: マルチブランチスキャンによる回帰バグの防止
`--branch-compare` オプションで現在のブランチと `main` ブランチの差分のみをスキャン。新たに導入された脆弱性のみに集中でき、スキャン時間を平均70%削減。週次で全ブランチのフルスキャンを実行し、レポートをSlack/webhookで通知。
