# deployguard - DEP_020

## 概要

DeployGuard は、コードセキュリティ監査・静的解析・依存関係検証を統合した、個人開発者向けの高精度プリデプロイチェッカーです。マージ前の最終ゲートとして、脆弱性の混入、シークレットリーク、非推奨APIの使用を自動検出し、デプロイ失敗率を最大 **87%** 削減します。

## 詳細な内容

### DeployGuard とは？

DeployGuard は、CI/CD パイプラインの最終段階に挿入するエージェントレス型セキュリティゲートウェイです。Go 製の単一バイナリで動作し、PostgreSQL に保存された既知の脆弱性データベース（CVE/NVD ミラー）とリアルタイム照合を行いながら、ソースコード・コンテナイメージ・IaC テンプレートを横断的にスキャンします。スキャン結果は **ミリ秒単位** でレポートされ、ブロッキングルールに基づいてパイプラインを自動停止します。

### コア機能

- **SecretLeak 検出エンジン**  
  正規表現 + エントロピー分析のハイブリッド方式により、ハードコードされた API キー、JWT トークン、AWS クレデンシャルを検出。誤検知率は **0.3% 未満**、検出閾値は `--entropy-threshold 4.2` で調整可能。スキャン速度は **1 ファイルあたり平均 12ms**。

- **依存関係ポリシー強制モジュール**  
  `deployguard.policy.yml` に定義された許可リスト/拒否リストに基づき、`go.mod`、`package.json`、`requirements.txt` を解析。ライセンス違反（GPL の商用利用など）や、バージョンが `>=2.0.0` 未満の古いパッケージをブロック。PostgreSQL 上のポリシーテーブルと自動同期し、チーム全体に即時反映。

- **IaC セキュリティスキャナー（Terraform / CloudFormation）**  
  `resource` ブロックの構成ミスを静的解析。例：S3 バケットの `acl = "public-read"` や、SecurityGroup の `cidr_blocks = ["0.0.0.0/0"]` を検出し、`--severity critical` オプションで重大度フィルタリング。スキャン結果は **JSON または SARIF** 形式で出力可能。

### 使い方

1. **初期セットアップとポリシー定義**  
   ```bash
   deployguard init --db postgres://user:pass@localhost:5432/deployguard
   deployguard policy create --name "production-strict" \
     --block-severity critical \
     --allow-licenses MIT,Apache-2.0 \
     --block-packages "lodash@<4.17.21"
   ```
   PostgreSQL にマイグレーションが自動実行され、`policies` テーブルにレコードが作成されます。

2. **CI/CD パイプラインへの統合**  
   ```yaml
   # .github/workflows/deployguard.yml
   - name: Run DeployGuard
     run: |
       deployguard scan \
         --path ./ \
         --policy "production-strict" \
         --output sarif \
         --fail-on critical \
         --cache-ttl 300s
   ```
   スキャン結果が `deployguard-results.sarif` に出力され、critical 以上の違反があると exit code 1 でパイプラインが停止します。

3. **結果の確認とバイパス管理**  
   ```bash
   deployguard report --input deployguard-results.sarif --format table
   deployguard bypass create --fingerprint "sha256:abc123" \
     --reason "既知のテスト用トークン" \
     --expires-at "2025-12-31T23:59:59Z"
   ```
   バイパスは `bypasses` テーブルに記録され、次回以降のスキャンで同一 fingerprint はスキップされます。

### ベストプラクティス

- **スキャン結果を Slack / PagerDuty に通知**  
  `--webhook-url https://hooks.slack.com/services/...` オプションを使用し、critical 違反検出時に即座に通知。デプロイブロックの平均応答時間を **2.3 分** に短縮できます。

- **週次ベースライン更新の自動化**  
  `deployguard db update --cve-mirror https://your-mirror.local` を crontab に登録し、PostgreSQL 上の脆弱性データベースを毎週土曜 3:00 に更新。これにより、ゼロデイ脆弱性への対応が **最大 48 時間** 早まります。
