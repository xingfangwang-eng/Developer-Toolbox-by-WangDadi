# deployguard - DEP_026

## 概要

DeployGuard は、コードセキュリティ監査・静的解析に特化したソロ開発者向けプロフェッショナルツールです。プルリクエスト単位での脆弱性スキャン、依存関係の自動検証、デプロイ前のポリシー準拠チェックを統合し、個人開発者がエンタープライズ級のセキュリティ品質を維持できる環境を提供します。

## 詳細な内容

### DeployGuard とは

DeployGuard は、Git フックと CI/CD パイプラインに直接組み込むことで、コード変更のたびに自動でセキュリティ監査を実行する静的解析エンジンです。従来の大規模セキュリティツールが持つ複雑な設定や高コストを排除し、個人開発者でも数分で導入可能な軽量設計が特徴です。PostgreSQL との連携により監査履歴を永続化し、過去の脆弱性パターンと比較した経時的なリスク評価も実現します。

### コア機能

- **リアルタイム脆弱性インジェクション検出（VulnInjectScan）**  
  コード変更時に、SQLインジェクション・XSS・OSコマンドインジェクションのパターンをASTレベルで解析。検出感度は `--severity=critical/high/medium` で調整可能。ミリ秒単位のスキャン結果を `guard_result.json` に出力し、PostgreSQL の `vuln_events` テーブルに自動記録。

- **依存関係ポリシー強制エンジン（DepPolicyEnforcer）**  
  `package.json`、`requirements.txt`、`go.mod` などのマニフェストファイルを解析し、ライセンス違反・既知のCVE・バージョン制約違反を検出。ポリシーは `deployguard.policy.yml` で定義し、`--block-on-violation` フラグで違反時にデプロイを強制停止。スキャン時間は平均 120ミリ秒 / プロジェクト。

- **マルチブランチ差分監査（BranchDiffAudit）**  
  プルリクエスト作成時に、ベースブランチとフィーチャーブランチの差分のみを抽出し、変更行に限定したセキュリティ監査を実行。`--diff-only` モードでは全ファイルスキャンと比較してスキャン時間を 80% 削減。結果は `audit_diff_<PR番号>.html` として視覚的に出力。

### 使い方

1. **プロジェクトへの初期導入とポリシー設定**  
   リポジトリルートで `deployguard init --db=postgresql://localhost:5432/guarddb --policy=./deployguard.policy.yml` を実行。PostgreSQL に `guard_schema` が自動生成され、初期ポリシーファイルが作成される。続けて `deployguard scan --full` で初回全スキャンを実施し、ベースラインとなる `baseline.hash` を生成。

2. **プルリクエスト時の自動監査とブロック**  
   プルリクエスト作成後に `deployguard audit --pr=$PR_NUMBER --block-on-violation --severity=high` を実行。`BranchDiffAudit` が差分のみをスキャンし、high 以上の脆弱性が検出された場合、CI パイプラインが異常終了。結果は `audit_diff_<PR番号>.html` に保存され、PostgreSQL の `pr_audit_log` テーブルに記録される。

3. **監査履歴の分析とレポート生成**  
   `deployguard report --from=2024-01-01 --to=2024-12-31 --format=pdf` で、指定期間内の全監査結果を集計。PostgreSQL の `vuln_events` テーブルから CVE 別・重要度別の発生頻度を抽出し、グラフ付きの PDF レポートを出力。`--export-csv` オプションで生データを CSV 出力可能。

### ベストプラクティス

- **段階的なポリシー適用と警告モードの活用**  
  導入初期は `--severity=critical` かつ `--warn-only` モードで運用し、既存の脆弱性を把握した上で 2週間後に `--block-on-violation` に切り替える。これにより、開発フローを中断させずにセキュリティ文化を浸透させられる。

- **PostgreSQL の定期メンテナンスとアーカイブ戦略**  
  `guard_schema` の `vuln_events` テーブルは月次でパーティショニングし、6ヶ月以上経過したデータは `vuln_events_archive` テーブルに移動。`deployguard vacuum --archive --before=6months` で自動アーカイブを実行し、監査クエリのパフォーマンスを維持する。
