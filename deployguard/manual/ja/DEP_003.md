# deployguard - DEP_003

## 概要

DeployGuard は、コード品質とデプロイ安全性を両立する静的解析エンジンです。特にソロ開発者向けに設計され、プッシュ前のコード監査・脆弱性検出・構成ミス防止を自動化します。キーワード：Code Security Audit, Solo Developer Tools, Static Analysis Tool

## 詳細な内容

### DeployGuard とは

DeployGuard は、プルリクエスト作成前の段階でコードベース全体をスキャンし、セキュリティホール、非推奨APIの使用、環境変数の漏洩、依存関係の既知の脆弱性（CVE）を検出する CLI ツールです。独自のルールエンジンとプラグインアーキテクチャにより、プロジェクト固有のポリシーに柔軟に対応します。

### コア機能

1. **マルチレイヤー静的セキュリティスキャン**
   - 引数 `--depth` でスキャン深度を指定可能（デフォルト: 3）。`--severity critical` を指定すると、CVSS 9.0 以上の脆弱性のみを即時ブロック。
   - 内部で Tree-sitter を用いた AST 解析を実行し、変数スコープ外へのシークレット（AWSキー、DBパスワード）の流出を検出。検出精度は 99.2%（社内ベンチマーク）。
   - 出力は JSON/HTML 形式に対応し、`--output report.html` で可視化レポートを生成。

2. **デプロイ前差分監査（Preflight Diff Audit）**
   - `deployguard diff --base main` で、main ブランチとの差分のみを対象に監査。変更行数が 500 行を超える場合は自動でバッチ分割処理（`--batch-size 200`）。
   - 各差分ブロックに対して、PostgreSQL のスキーマ変更検出（カラム削除、型変更）を実施し、互換性がない変更（例：`ALTER TABLE DROP COLUMN`）を検出した場合は exit code 1 で CI を停止。
   - 解析時間は 1000 行の差分あたり平均 320 ミリ秒（Intel Core i7-12700 実測値）。

3. **依存関係 CVE 自動マッピング＆パッチ提案**
   - `--cve-db ./cve.db` でローカル CVE データベースを指定。デフォルトでは NVD フィードを自動同期（更新間隔: 6 時間）。
   - 検出された脆弱性パッケージに対し、利用可能なパッチバージョンとマイグレーション手順を `--suggest-fix` フラグで自動提案。提案は `package.json` や `requirements.txt` の書き換え候補として表示。
   - 対応言語: Python, Node.js, Go, Rust。各言語のロックファイル（`poetry.lock`, `yarn.lock`, `Cargo.lock`）を直接解析。

### 使い方

1. **初期セットアップとクイックスキャン**
   - プロジェクトルートで `deployguard init` を実行。`.deployguard.yml` が生成される。
   - スキャン対象ディレクトリ、除外パターン（`exclude: ["vendor/*", "*.min.js"]`）、通知先 Slack Webhook URL を設定。
   - `deployguard scan --severity high --output json | jq '.vulnerabilities[] | select(.cvss >= 7.0)'` で重要度高の脆弱性のみ抽出。

2. **CI/CD パイプラインへの組み込み**
   - GitHub Actions のワークフローに以下のステップを追加：
     ```yaml
     - name: Run DeployGuard
       run: deployguard diff --base ${{ github.event.pull_request.base.ref }} --fail-on critical
     ```
   - プルリクエスト作成時に自動実行。`--fail-on critical` により、Critical レベルの問題が1つでもあると CI が失敗。
   - 結果は `deployguard report --format markdown` で PR コメントとして自動投稿。

3. **カスタムルールの作成と適用**
   - `deployguard rules new --name no_hardcoded_secrets` でルールテンプレートを生成。
   - ルールファイル（YAML）内で、正規表現パターン（`pattern: "(?i)password\\s*=\\s*['\"][^'\"]+['\"]"`）とメッセージ、重大度を定義。
   - `deployguard scan --rules ./custom_rules/` でカスタムルールを適用。標準ルールと併用可能。

### ベストプラクティス

1. **段階的なルール適用とノイズ低減**
   初回導入時は `--severity critical` および `--ignore-warnings` フラグを併用し、警告レベルのノイズを排除。チーム内でルールの合意が取れた段階で `--severity high` に引き上げる。また、`.deployguard-ignore` ファイルを用いて既知の false positive を管理する。

2. **デプロイ前のフルスキャンと差分スキャンの使い分け**
   週次で `deployguard scan --full --output weekly-report.html` を実行し、全コードベースのセキュリティ姿勢を監視。日常のプルリクエストでは `deployguard diff` で変更部分のみを高速チェック。これにより、解析時間を 80% 削減しつつ、回帰バグを防止する。
