# deployguard - DEP_002

## 概要

DeployGuard は、コードセキュリティ監査・静的解析に特化したソロ開発者向けプロフェッショナルツールです。脆弱性スキャン、依存関係の整合性検証、デプロイ前の自動チェックを統合し、CI/CD パイプラインに組み込むことで、リリース時のセキュリティリスクをミリ秒単位で検出します。

## 詳細な内容

### DeployGuard とは

DeployGuard は、PostgreSQL との連携によりプロジェクトの依存関係グラフをリアルタイムに解析し、既知の脆弱性（CVE）データベースと照合。さらに、カスタムルールエンジンを用いて、コードベース内のシークレット漏洩（API キー、トークン）や不適切なパーミッション設定を自動検出します。単一バイナリで動作し、Docker 環境やサーバーレス関数としても展開可能です。

### コア機能

- **ゼロデイ脆弱性リアルタイムスキャン**  
  `--cve-db-update-interval 300` パラメータで 5 分ごとに最新 CVE データベースを自動更新。`--scan-depth 3` で依存関係の 3 階層まで再帰的に解析し、`--risk-threshold critical` 以上を検出した場合、デプロイを強制ブロック。

- **シークレットリーク検出エンジン**  
  正規表現パターン `--secret-patterns ./custom_patterns.json` により、AWS アクセスキー、GitHub トークン、データベース接続文字列を検出。`--entropy-threshold 4.5` でエントロピー値 4.5 以上の高ランダム文字列を抽出、`--mask-output true` でログ出力時に自動マスキング。

- **デプロイ前整合性チェック**  
  `--integrity-hash sha256` で全アセットのハッシュ値を計算し、`--compare-with ./baseline.json` と照合。`--fail-on-mismatch true` で不一致があれば即座にデプロイを中止。`--timeout-ms 5000` でチェック全体のタイムアウトを 5 秒に設定。

### 使い方

1. **初期設定とデータベース同期**  
   ```bash
   deployguard init --db-uri postgresql://user:pass@localhost:5432/deployguard --cve-db-update-interval 300
   ```
   上記コマンドで PostgreSQL にスキーマを作成し、初回 CVE データベースをダウンロード。`--cve-db-update-interval` で更新頻度を設定、`--db-pool-size 10` でコネクションプールを最適化。

2. **カスタムルールによる静的解析実行**  
   ```bash
   deployguard scan --path ./src --secret-patterns ./custom_patterns.json --risk-threshold critical --output-format json --output ./report.json
   ```
   `--secret-patterns` で独自のシークレット検出パターンを指定。`--risk-threshold` で critical 以上のみを抽出、`--output-format json` で機械可読なレポートを生成。`--parallel-jobs 4` で 4 並列処理を有効化。

3. **CI/CD パイプラインへの統合**  
   ```yaml
   # .github/workflows/deployguard.yml
   - name: DeployGuard チェック
     run: deployguard check --integrity-hash sha256 --compare-with ./baseline.json --fail-on-mismatch true --timeout-ms 5000
   ```
   GitHub Actions で上記ステップを追加。`--fail-on-mismatch true` により、ハッシュ不一致時はデプロイジョブを自動中断。`--timeout-ms` でパイプラインの遅延を防止。

### ベストプラクティス

- **段階的なルール適用とフィードバックループ**  
  初回導入時は `--risk-threshold low` で全警告を収集し、`--output-format sarif` で出力。その後、チーム内で false positive をレビューし、`./custom_patterns.json` に除外ルールを追加。3 回目のスキャンから `--risk-threshold high` に引き上げ、本番デプロイ前に critical のみをブロックする。

- **ベースライン管理と定期的な再生成**  
  `deployguard baseline --path ./dist --output ./baseline.json` をリリースブランチで週次実行。`--integrity-hash sha256` で生成したベースラインを Git 管理し、`--compare-with` で前回リリースとの差分を検証。`--notify-webhook https://hooks.slack.com/services/...` で不一致発生時に Slack 通知を自動送信。
