# deployguard - DEP_009

## 概要

DeployGuard は、コードセキュリティ監査に特化した静的解析ツールです。個人開発者向けに設計されており、デプロイ前に脆弱性・機密情報漏洩・依存関係リスクを自動検出します。Go 製の CLI ツールとして動作し、CI/CD パイプラインへの統合を前提としています。

## 詳細な内容

### DeployGuard とは

DeployGuard は、ソースコードの静的解析を通じて、デプロイ前にセキュリティ上の問題を検出・報告するプロフェッショナルツールです。特に、個人開発者が見落としがちな「秘密鍵の埋め込み」「古い依存ライブラリ」「不適切な権限設定」を自動的にスキャンし、修正を提案します。

## コア機能

### 1. 機密情報リーク検出（シークレットスキャン）

- **パラメータ**: `--secret-patterns`, `--exclude-dirs`, `--threshold=high`
- **技術指標**: 500 種類以上の正規表現パターン（AWS キー、GitHub トークン、Slack トークン、JWT など）を内蔵。スキャン速度は 10 万行あたり 200 ミリ秒未満。
- **動作**: コミット前のステージングエリア、またはブランチ全体をスキャン。誤検知を減らすため、エントロピー解析とコンテキスト判定を併用。

### 2. 依存関係脆弱性評価（SBOM 連携）

- **パラメータ**: `--sbom-path=./sbom.json`, `--db-update-interval=24h`
- **技術指標**: NVD（National Vulnerability Database）および GitHub Advisory Database と自動同期。PostgreSQL に脆弱性データベースを保持し、更新は 24 時間ごとに自動実行。
- **動作**: プロジェクトの `go.mod`, `package.json`, `requirements.txt` などを解析し、既知の CVE と照合。重大度（Critical/High/Medium/Low）を表示し、影響範囲を依存関係グラフで可視化。

### 3. デプロイ前ポリシー強制（カスタムルールエンジン）

- **パラメータ**: `--policy-file=./deployguard.policy.yaml`, `--fail-on=critical`
- **技術指標**: YAML ベースのポリシー定義。最大 256 個のカスタムルールを同時適用可能。ルール評価は 1 ルールあたり平均 5 マイクロ秒。
- **動作**: 例として「Dockerfile 内で `latest` タグの使用禁止」「環境変数に `PRODUCTION` が含まれている場合は `--fail-on=high` を強制」など、組織のセキュリティポリシーをコードとして管理。

## 使い方

### ステップ 1: インストールと初期設定

```bash
# macOS/Linux（Homebrew）
brew install wangdadi/tap/deployguard

# または Go ユーザー向け
go install github.com/xingfangwang-eng/deployguard@latest

# 初期設定ファイルの生成
deployguard init --project-dir=./myapp
# → deployguard.yaml が生成される
```

### ステップ 2: シークレットスキャンの実行

```bash
# カレントディレクトリ全体をスキャン
deployguard scan --secret --exclude-dirs=node_modules,.git --output=json

# 特定のパターンだけをチェック（例：AWS キーのみ）
deployguard scan --secret --pattern-group=aws --threshold=high
```

### ステップ 3: CI/CD パイプラインへの統合（GitHub Actions 例）

```yaml
# .github/workflows/deployguard.yml
name: DeployGuard Security Audit
on: [push, pull_request]
jobs:
  security-audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run DeployGuard
        run: |
          deployguard scan --secret --sbom --policy-file=./deployguard.policy.yaml
          if [ $? -ne 0 ]; then
            echo "セキュリティ違反を検出しました。デプロイをブロックします。"
            exit 1
          fi
```

## ベストプラクティス

### プラクティス 1: ポリシーファイルを Git 管理し、チーム全体で共有する

`deployguard.policy.yaml` をリポジトリのルートに配置し、全メンバーが同一のセキュリティ基準でコードをレビューできるようにします。例えば「`--fail-on=medium` 以上はマージ禁止」といったルールを定義し、プルリクエスト時に自動チェックを実行します。

### プラクティス 2: 定期的な脆弱性データベースの更新を自動化する

PostgreSQL に保持する脆弱性データベースは、週に 1 回の頻度で自動更新することを推奨します。`deployguard db-update --interval=168h` を cron ジョブに設定し、常に最新の CVE 情報を参照できる状態を維持します。これにより、ゼロデイ脆弱性が公開された際に、即座にプロジェクトへの影響を評価できます。
