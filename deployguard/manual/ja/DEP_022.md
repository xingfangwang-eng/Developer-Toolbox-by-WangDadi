# deployguard - DEP_022

## 概要

DeployGuardは、コードセキュリティ監査と静的解析に特化したプロフェッショナル向けツールです。個人開発者や小規模チームがデプロイ前に脆弱性を自動検出し、攻撃ベクターを事前に排除することを目的としています。キーワード: Code Security Audit, Solo Developer Tools, Static Analysis Tool

## 詳細な内容

### DeployGuardとは？

DeployGuardは、静的解析エンジンとリアルタイム依存関係スキャナーを統合した次世代型セキュリティゲートウェイです。デプロイパイプラインに組み込むことで、コードベース全体の機密情報漏洩、既知のCVE（Common Vulnerabilities and Exposures）、および不適切なアクセス制御パターンをミリ秒単位で検出します。PostgreSQLとの連携により、過去の監査ログと比較したトレンド分析も可能です。

### コア機能

- **機密情報リーク検出エンジン**: 正規表現と機械学習モデルを組み合わせ、AWSキー、GitHubトークン、データベース接続文字列など、50種類以上のシークレットパターンを自動特定。--secret-threshold=high オプションで誤検知率0.1%未満を実現。
- **CVE事前スキャンとバージョンロック**: npm、PyPI、Maven Centralの依存関係をリアルタイムで照合し、--cve-db-update-interval=3600（秒）で1時間ごとに更新。脆弱性が見つかった場合、--block-on-critical=true でデプロイを即時停止。
- **アクセス制御パターン監査**: コード内のRBAC（ロールベースアクセス制御）実装を自動解析し、--audit-depth=3（深さ3レベル）までトレース。過剰権限や未認証エンドポイントをJSONレポートとして出力。

### 使い方

1. **初期セットアップとパイプライン統合**  
   `deployguard init --project-type=node --output-format=json` を実行し、プロジェクトルートに設定ファイル（deployguard.config.yml）を生成。次にCI/CDパイプライン（例：GitHub Actions）に `deployguard scan --fail-on-severity=high` を追加し、プルリクエスト時に自動実行。

2. **カスタムルールとシークレットパターンの追加**  
   `deployguard rules add --name=custom-aws-key --pattern="AKIA[0-9A-Z]{16}" --severity=critical` で独自のシークレットパターンを登録。さらに `deployguard rules test --file=./test-secrets.txt` で正規表現の精度を検証し、誤検知を排除。

3. **結果のレビューと修復提案の適用**  
   `deployguard report --since=2024-01-01 --severity=critical` で過去の全監査結果をPostgreSQLから取得。`deployguard remediate --auto-fix=true --backup=true` を実行すると、検出された脆弱性に対して自動修正パッチを生成し、バックアップ付きで適用。

### ベストプラクティス

- **段階的な閾値設定とチーム内レビュー**  
  初回導入時は `--fail-on-severity=critical` のみで運用し、誤検知率を確認しながら `--fail-on-severity=high` へ段階的に引き上げる。週次で監査ログをチーム共有し、false positiveのパターンをdeployguard.config.ymlのignoreリストに追加することで、ノイズを最小化する。

- **依存関係の定期的なリフレッシュとキャッシュ戦略**  
  CI/CDパイプライン内で `deployguard scan --cve-cache-clean=true` を毎日実行し、CVEデータベースのキャッシュをリフレッシュ。同時に `--dependency-cache-ttl=7200`（秒）で2時間ごとに依存関係のハッシュを再計算し、古いバージョンによる誤検出を防ぐ。
