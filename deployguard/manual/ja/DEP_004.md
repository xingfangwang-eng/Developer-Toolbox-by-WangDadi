# deployguard - DEP_004

## 概要

DeployGuard は、コードセキュリティ監査・静的解析に特化したソロ開発者向けプロフェッショナルツールです。秘密鍵やトークン漏洩、依存関係の脆弱性、デプロイ前の設定ミスを自動検出し、CI/CD パイプラインに組み込むことで、セキュアなリリースを担保します。

## コア機能

### 1. リアルタイムシークレットスキャンエンジン（RSSE）
- コミット前フック（pre-commit）および CI ステージで、正規表現 + エントロピー解析（Shannonエントロピー 4.5 以上）により、AWS キー、GitHub Token、DB パスワードなどを **ミリ秒単位** で検出。
- 検出時は即座にコミットをブロックし、エラーコードと行番号を出力。
- パラメータ：`--min-entropy 4.5`、`--exclude-pattern "*.md"`

### 2. マルチバージョン依存関係脆弱性比較（MVDVC）
- `package.json`、`requirements.txt`、`Gemfile.lock` を解析し、**PostgreSQL 上の CVE データベース**（毎日自動更新）と照合。
- 過去 3 バージョン分の差分をスナップショット保存し、**バージョン間の脆弱性増減をヒートマップで可視化**。
- パラメータ：`--db-url postgresql://deployguard:pass@localhost:5432/cve_db`、`--history-depth 3`

### 3. デプロイ前設定整合性チェッカー（DPC）
- Kubernetes マニフェスト、Docker Compose、Terraform の変数と実際の環境変数（`.env`）を比較し、**未定義変数・型不一致・ポート競合** を検出。
- チェック結果は **JSON 形式で出力** し、Slack / Discord への Webhook 通知に対応。
- パラメータ：`--manifest-dir ./k8s/`、`--env-file .env.production`、`--webhook-url https://hooks.slack.com/...`

## 使い方

### ステップ 1: インストールと初期設定
```bash
# macOS / Linux
curl -sSL https://deployguard.io/install.sh | bash

# 初期化（PostgreSQL 接続情報を設定）
deployguard init --db-url postgresql://user:pass@localhost:5432/deployguard --min-entropy 4.5
```
- `~/.deployguard/config.yml` に設定が保存されます。

### ステップ 2: リポジトリ全体のスキャン実行
```bash
# 全ファイルのシークレットスキャン + CVE チェック
deployguard scan --path ./my-repo/ --ci-mode --output json > report.json
```
- `--ci-mode` を指定すると、エラー時に終了コード 1 を返すため、CI パイプラインで利用可能。
- 出力例（抜粋）：
```json
{
  "secrets_found": 2,
  "cve_high": 1,
  "cve_critical": 0,
  "config_errors": 3
}
```

### ステップ 3: マルチバージョン比較とレポート生成
```bash
# 過去 3 バージョンの脆弱性差分をヒートマップで出力
deployguard diff --history-depth 3 --format html --output ./reports/vuln-history.html
```
- 生成された HTML はブラウザで開き、**バージョンごとの脆弱性数・種別・対応推奨日** を一覧表示。

## ベストプラクティス

### 1. CI/CD パイプラインへのフル統合
- プルリクエスト作成時（GitHub Actions の `pull_request` イベント）に `deployguard scan --ci-mode` を実行し、**マージブロッカー** として設定。
- 検出されたシークレットは GitHub Secrets に自動マスクされ、ログに平文が残らないよう `--redact` オプションを併用。

### 2. 週次ベースライン更新
- 毎週月曜 9:00 に cron ジョブで `deployguard update-db` を実行し、PostgreSQL の CVE データベースを最新化。
- 更新後、全リポジトリの `deployguard diff` を自動実行し、**新たに発見された脆弱性をメールまたは Slack で通知**。
