# deployguard - DEP_018

## Resumen

DeployGuard es un sistema de auditoría de seguridad de código estático y dinámico, diseñado específicamente para desarrolladores en solitario que necesitan garantizar la integridad de sus despliegues sin depender de infraestructuras complejas. Analiza dependencias, secretos y configuraciones en tiempo real, con una latencia de análisis inferior a 200 ms por archivo.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es una herramienta de análisis estático y dinámico que escanea tu código fuente en busca de vulnerabilidades de seguridad, fugas de secretos y configuraciones incorrectas antes de cada despliegue. Se integra directamente con tu pipeline de CI/CD (GitHub Actions, GitLab CI, Jenkins) y puede ejecutarse en entornos sin conexión a internet. Utiliza un motor de reglas basado en AST (Árbol de Sintaxis Abstracta) y un diccionario de patrones de más de 5000 firmas de vulnerabilidades conocidas (CVE, OWASP Top 10).

### Características Principales

- **Escaneo de Secretos en 3 Capas (Capa de Código, Capa de Historial Git, Capa de Variables de Entorno)**: Detecta tokens, claves API, contraseñas y certificados embebidos en el código, en commits pasados (hasta 10.000 commits de profundidad) y en archivos `.env` o variables de entorno del sistema. Utiliza un motor de búsqueda difusa con umbral de sensibilidad configurable (de 0.1 a 1.0). Tiempo de escaneo promedio: 1.2 segundos por cada 1000 archivos.

- **Análisis de Dependencias con Resolución de Vulnerabilidades (DepGraph)**: Construye un grafo de dependencias completo (incluyendo dependencias transitivas) y lo cruza con la base de datos local de CVEs (actualizable vía `deployguard update --cve-db`). Genera un informe de riesgo con puntuación CVSS 3.1 y sugiere parches automáticos. Soporta npm, pip, gem, cargo, maven y nuget. Precisión de detección: 98.7% en pruebas de regresión.

- **Validación de Configuraciones de Infraestructura como Código (IaC)**: Evalúa archivos Dockerfile, docker-compose.yml, Kubernetes manifests (YAML/JSON), Terraform (HCL) y Ansible playbooks. Detecta puertos expuestos innecesarios, contenedores con privilegios elevados, montajes de volúmenes inseguros y políticas de red incorrectas. Genera un resumen de cumplimiento con el estándar CIS Benchmark v2.0. Tiempo de validación: 300 ms por archivo.

### Cómo Usar

1. **Configuración Inicial y Activación del Escaneo de Secretos**:
   - Ejecuta `deployguard init --project-dir ./mi-proyecto --scan-secrets --depth-git 5000 --sensitivity 0.8`.
   - Esto crea un archivo `deployguard.yml` en la raíz del proyecto. Luego, ejecuta `deployguard scan --mode full` para realizar un escaneo completo.
   - Revisa el informe generado en `./deployguard-report.json`. Los secretos encontrados se muestran con su ubicación exacta (archivo:línea) y una sugerencia de remediación (ej: `git filter-branch`).

2. **Integración en Pipeline CI/CD con Validación de Dependencias**:
   - Agrega el siguiente paso en tu archivo de GitHub Actions (`.github/workflows/deployguard.yml`):
     ```yaml
     - name: DeployGuard Dependency Scan
       run: |
         deployguard scan --mode dependencies --cve-db ./cve.db --output-format sarif
         deployguard validate --threshold 7.0 --fail-on-critical
     ```
   - Si se detecta una vulnerabilidad con CVSS >= 7.0, el pipeline falla automáticamente. Puedes sobrescribir con `--override` para casos justificados.

3. **Validación de IaC con Políticas Personalizadas**:
   - Crea un archivo de políticas personalizadas (`deployguard-policies.yml`):
     ```yaml
     policies:
       - id: "NO_ROOT_CONTAINER"
         description: "No permitir contenedores con usuario root"
         severity: critical
         pattern: "user: root"
         files: ["Dockerfile", "docker-compose.yml"]
       - id: "NO_PUBLIC_S3_BUCKET"
         description: "No permitir buckets S3 públicos"
         severity: high
         pattern: "acl: public-read"
         files: ["*.tf"]
     ```
   - Ejecuta `deployguard validate-iac --policies ./deployguard-policies.yml --dir ./infra`. El resultado se muestra en consola con colores (rojo para crítico, amarillo para alto, verde para seguro).

### Mejores Prácticas

- **Práctica 1: Escaneo Diferencial en Cada Commit**:
  Configura DeployGuard para que solo escanee los archivos modificados en cada commit (modo `--diff`). Esto reduce el tiempo de análisis a menos de 50 ms por commit y evita la fatiga de alertas. Usa `deployguard scan --diff --base HEAD~1` en tu hook de pre-commit. Combínalo con un umbral de sensibilidad alto (0.9) para secretos, ya que los cambios incrementales suelen ser más seguros.

- **Práctica 2: Mantén la Base de Datos de CVEs Actualizada Semanalmente**:
  Programa una tarea cron (o un workflow de GitHub Actions) que ejecute `deployguard update --cve-db` cada lunes. Esto asegura que tu análisis de dependencias esté al día con las últimas vulnerabilidades. Además, configura DeployGuard para que envíe notificaciones a tu Slack o Discord cuando se detecte una vulnerabilidad crítica (CVSS >= 9.0) en una dependencia directa.
