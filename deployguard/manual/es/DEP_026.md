# deployguard - DEP_026

## Resumen

Sistema de auditoría de seguridad de código con análisis estático en tiempo real para desarrolladores en solitario. Detecta vulnerabilidades críticas en menos de 200 ms por archivo, con soporte para 14 lenguajes de programación y reglas personalizables vía YAML.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es un analizador estático de código fuente diseñado para desarrolladores individuales y equipos pequeños. Escanea el árbol sintáctico abstracto (AST) de cada archivo en busca de patrones de vulnerabilidad, fugas de credenciales y configuraciones inseguras. Se integra directamente con el hook `pre-commit` de Git, bloqueando commits que contengan fallos de seguridad antes de que lleguen al repositorio remoto. Utiliza un motor de reglas basado en Tree-sitter para lograr una precisión del 99.2% en la detección de inyección SQL, XSS y deserialización insegura.

### Características Principales

- **Escáner de Credenciales Embebidas**: Detecta claves API, tokens OAuth y contraseñas en texto plano mediante 47 expresiones regulares predefinidas y un modelo de aprendizaje automático ligero (Random Forest de 50 árboles) con una tasa de falsos positivos inferior al 1.5%. Parámetro configurable: `--credential-depth [1-5]` para controlar la profundidad de búsqueda en archivos binarios y de configuración.

- **Análisis de Dependencias con Graph de Vulnerabilidades**: Construye un grafo dirigido de dependencias transitivas (hasta 3 niveles de profundidad) y lo cruza contra la base de datos local de CVEs, actualizada cada 6 horas desde NVD. Tiempo de análisis para un proyecto con 200 dependencias: 1.2 segundos en un Intel i7-12700. Parámetro: `--cve-db-update-interval 360` (en minutos).

- **Bloqueo de Patrones de Código Inseguro**: Identifica 23 familias de antipatrones de seguridad (como `eval()` en JavaScript, `exec()` en Python, o `mysql_query()` en PHP sin escape) con resaltado de línea exacta y sugerencia de corrección automática. Soporta inyección de reglas personalizadas mediante archivos `.dg-rules.yaml` con sintaxis de patrones glob y operadores lógicos AND/OR.

### Cómo Usar

1. **Configuración del Hook Pre-Commit**: Ejecuta `deployguard init --hook pre-commit --threshold high` para generar el archivo `.git/hooks/pre-commit`. Este hook ejecuta `deployguard scan --exit-code 1 --severity critical,high` antes de cada commit. Si se detecta alguna vulnerabilidad de severidad "critical" o "high", el commit se cancela y se muestra el reporte en la terminal con colores ANSI y enlaces directos a las líneas ofensivas.

2. **Escaneo Programático con Pipeline CI/CD**: Integra DeployGuard en GitHub Actions con el siguiente comando: `deployguard scan --path ./src --format sarif --output report.sarif --fail-on any`. El archivo SARIF generado puede subirse a GitHub Security Code Scanning. Para proyectos con más de 5000 archivos, usa el flag `--parallel 4` para habilitar escaneo multihilo con 4 workers, reduciendo el tiempo total en un 62% en promedio.

3. **Creación de Reglas Personalizadas**: Crea un archivo `custom-rules.yaml` en la raíz del proyecto. Ejemplo de regla que bloquea el uso de `pickle.load()` en Python sin validación de origen:
   ```yaml
   - id: PY-DES-001
     pattern: "pickle.load({data})"
     condition: "{data} != 'trusted_source'"
     severity: critical
     message: "Deserialización de pickle sin verificar origen. Usa json.loads() en su lugar."
   ```
   Luego ejecuta `deployguard scan --rules custom-rules.yaml` para aplicar solo esas reglas.

### Mejores Prácticas

- **Segmentación de Severidades por Entorno**: Configura thresholds diferentes para desarrollo y producción. En desarrollo, permite advertencias de severidad "low" para no ralentizar el flujo de trabajo, pero en producción (mediante variable de entorno `DEPGUARD_ENV=production`) eleva todas las advertencias "medium" a "high". Esto evita que vulnerabilidades menores bloqueen commits en etapas tempranas, pero garantiza seguridad absoluta en despliegues.

- **Rotación y Actualización Automática de Reglas**: Suscríbete al feed de reglas comunitarias vía `deployguard update-rules --source community --auto-apply`. Esto descarga nuevas reglas cada 24 horas desde el repositorio oficial y las aplica automáticamente si pasan una validación de sintaxis. Mantén siempre una copia de seguridad de tus reglas personalizadas en un repositorio Git separado para evitar pérdidas por actualizaciones conflictivas.
