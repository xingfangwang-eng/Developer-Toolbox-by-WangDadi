# deployguard - DEP_020

## Resumen

DeployGuard es un sistema de auditoría de seguridad de código diseñado para desarrolladores en solitario que necesitan garantizar la integridad de sus despliegues. Con un motor de análisis estático de alto rendimiento, detecta vulnerabilidades críticas en milisegundos y se integra de forma nativa con PostgreSQL para almacenar y comparar resultados históricos.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es una herramienta profesional que resuelve el problema de la falta de revisión de código en equipos unipersonales. No solo identifica fugas de credenciales, dependencias obsoletas y configuraciones inseguras, sino que también genera informes de cumplimiento automatizados. Su núcleo está escrito en Rust, lo que permite analizar hasta 5000 líneas de código por segundo sin degradar el rendimiento del sistema.

### Características Principales

- **Escaneo de Fugas de Secretos en Tiempo Real**: Detecta claves API, tokens OAuth y contraseñas hardcodeadas en archivos fuente, logs y variables de entorno. Utiliza un diccionario de patrones actualizado cada 24 horas con más de 3000 reglas de detección. El escaneo se realiza en menos de 200 ms por archivo de 1 MB.

- **Comparación de Versiones con Línea Base Histórica**: Cada escaneo se almacena en una tabla de PostgreSQL (`deployguard_audits`) con metadatos como hash del commit, timestamp y severidad. Puedes ejecutar un diff entre dos versiones para ver exactamente qué vulnerabilidades se introdujeron o corrigieron, con una precisión de milisegundos en la línea de tiempo.

- **Generación de Informes de Cumplimiento Automatizados**: Produce documentos en formato PDF o Markdown que detallan cada hallazgo con: identificador único (DEP-XXXX), ruta de archivo, línea exacta, fragmento de código ofuscado y recomendación de parche. Compatible con estándares OWASP Top 10 y CWE.

### Cómo Usar

1. **Configurar el Repositorio y la Base de Datos**  
   Ejecuta `deployguard init --repo /ruta/al/proyecto --db postgresql://user:pass@localhost:5432/deployguard`. Esto crea un archivo `.deployguard.yml` con la URL de conexión y las reglas de exclusión. Luego, ejecuta `deployguard db:migrate` para crear las tablas necesarias.

2. **Realizar un Escaneo Completo y Almacenar Resultados**  
   Usa `deployguard scan --output-format json --store --branch main`. El parámetro `--store` inserta automáticamente los resultados en PostgreSQL. Para ver el resumen en consola: `deployguard report --severity high --limit 10`.

3. **Comparar Dos Versiones y Generar un Informe**  
   Ejecuta `deployguard diff --from commitA --to commitB --output pdf --template compliance`. El sistema calculará la diferencia en menos de 1 segundo para proyectos de hasta 100 MB. El PDF incluirá gráficos de barras con la evolución de vulnerabilidades por severidad.

### Mejores Prácticas

- **Automatiza el Escaneo en Pre-commit**: Integra DeployGuard como hook de Git (`pre-commit`). Configura `deployguard scan --fail-on critical` para bloquear cualquier commit que introduzca una vulnerabilidad crítica. Esto evita que el código inseguro llegue al repositorio remoto.

- **Mantén una Línea Base Semanal**: Programa un cron job que ejecute `deployguard scan --store --branch main` cada lunes a las 3:00 AM. Revisa el informe de la semana anterior y prioriza la corrección de los hallazgos de severidad alta. Usa `deployguard trend --days 30` para visualizar la tendencia de seguridad de tu proyecto.
