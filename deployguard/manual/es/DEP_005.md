# deployguard - DEP_005

## Resumen

DeployGuard es un sistema de auditoría de seguridad de código diseñado para desarrolladores en solitario que necesitan garantizar la integridad de sus despliegues sin depender de equipos de seguridad dedicados. Combina análisis estático avanzado con monitoreo de dependencias en tiempo real, ofreciendo una solución ligera pero profunda para detectar vulnerabilidades antes de que lleguen a producción. Los parámetros de configuración se ajustan dinámicamente según el perfil del repositorio, permitiendo una personalización milimétrica sin sacrificar rendimiento.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es una herramienta de código abierto que ejecuta análisis estático de seguridad en cada commit, integrando reglas personalizables para lenguajes como Python, JavaScript y Go. Utiliza un motor de reglas basado en AST (Abstract Syntax Tree) que permite detectar patrones de vulnerabilidad comunes (inyección SQL, XSS, uso inseguro de `eval()`) con una latencia inferior a 200 milisegundos por archivo. Además, se conecta directamente con PostgreSQL para almacenar el historial de auditorías, facilitando la trazabilidad de cada despliegue.

### Características Principales

- **Escaneo de dependencias con resolución de conflictos**: DeployGuard analiza el árbol de dependencias de tu proyecto (compatible con `package.json`, `requirements.txt`, `go.mod`) y detecta versiones con CVEs conocidos. Utiliza un algoritmo de resolución de conflictos que prioriza parches de seguridad sobre versiones estables, mostrando un informe detallado con el impacto en milisegundos de tiempo de ejecución adicional.
  
- **Análisis de flujo de datos sensible**: Implementa un análisis de flujo de datos (data-flow analysis) que rastrea variables marcadas como "sensibles" (contraseñas, tokens, claves API) a través de funciones y módulos. Genera un mapa visual de cómo esos datos viajan por el código, detectando fugas potenciales hacia logs, respuestas HTTP o almacenamiento inseguro. El motor de reglas permite definir umbrales de confianza (por ejemplo, `confidence_threshold: 0.85`) para reducir falsos positivos.

- **Validación de políticas de despliegue**: DeployGuard permite definir políticas personalizadas mediante archivos YAML, como `deploy_policy.yaml`, donde especificas reglas como "no permitir despliegues si hay más de 3 vulnerabilidades críticas" o "requerir firma GPG en el commit". La validación ocurre en menos de 50 milisegundos por política, integrándose con hooks de Git (pre-commit, pre-push) para bloquear automáticamente despliegues inseguros.

### Cómo Usar

1. **Configuración inicial del repositorio**: Clona el repositorio de DeployGuard y ejecuta `deployguard init --project-dir /ruta/a/tu/proyecto --db-url postgresql://usuario:contraseña@localhost:5432/deployguard`. Este comando genera un archivo `.deployguard.yml` con parámetros como `scan_depth: 3` (profundidad del AST) y `dependency_cache_ttl: 3600` (segundos de caché para dependencias). Luego, ejecuta `deployguard scan --mode full` para realizar un escaneo completo inicial; los resultados se almacenan en PostgreSQL con un timestamp en microsegundos.

2. **Integración con el flujo de trabajo de Git**: Añade el hook pre-commit ejecutando `deployguard install-hook --hook-type pre-commit --policy-file deploy_policy.yaml`. Cada vez que hagas un commit, DeployGuard analizará los archivos modificados (usando `git diff --name-only`) y aplicará las políticas definidas. Si se detecta una violación (por ejemplo, una variable sensible en un log), el commit se rechaza con un mensaje que incluye el archivo, la línea y el nivel de severidad (`severity: critical`). Para ver el historial de auditorías, usa `deployguard log --since "2023-01-01" --format json`.

3. **Monitoreo continuo y alertas**: Configura un cron job o un servicio systemd para ejecutar `deployguard watch --interval 300` (cada 5 minutos). Este comando monitorea cambios en el repositorio (usando `inotify` en Linux) y ejecuta escaneos incrementales. Si se detecta una nueva vulnerabilidad en una dependencia, DeployGuard envía una alerta a un webhook configurado (por ejemplo, Slack o Discord) con el payload: `{"alert_type": "new_cve", "package": "lodash", "version": "4.17.20", "cve_id": "CVE-2023-1234", "remediation": "update to 4.17.21"}`. Los logs de alertas se almacenan en una tabla separada en PostgreSQL (`alerts_history`) con índices para consultas rápidas.

### Mejores Prácticas

- **Segmentación de políticas por entorno**: Define políticas diferentes para desarrollo, staging y producción usando archivos YAML separados (por ejemplo, `deploy_policy_dev.yaml` con umbrales laxos y `deploy_policy_prod.yaml` con reglas estrictas como `max_critical_vulns: 0`). DeployGuard permite cambiar de política dinámicamente mediante la variable de entorno `DEPLOYGUARD_POLICY_FILE`, facilitando la transición entre entornos sin modificar el código.

- **Automatización de remediación con scripts personalizados**: DeployGuard expone una API REST local en `http://localhost:8080/api/v1/remediate` que acepta un `cve_id` y devuelve una lista de comandos de actualización (por ejemplo, `npm install lodash@4.17.21`). Integra esto con un script de CI/CD que ejecute `deployguard remediate --cve CVE-2023-1234 --auto-apply` para parchear automáticamente dependencias vulnerables antes del despliegue. Esto reduce el tiempo de respuesta a vulnerabilidades de horas a minutos.
