# deployguard - DEP_009

## Resumen

DeployGuard es un sistema de análisis estático y auditoría de seguridad diseñado específicamente para desarrolladores en solitario (solo developers) que necesitan proteger sus despliegues sin un equipo de seguridad dedicado. Escanea dependencias, secretos hardcodeados y configuraciones inseguras en pipelines de CI/CD, todo en menos de 200 ms por archivo.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es un motor de reglas personalizables que intercepta fugas de credenciales, tokens de API y claves SSH antes de que lleguen a producción. Se integra directamente con tu repositorio Git y ejecuta un análisis diferencial en cada commit, bloqueando el push si se detecta un secreto válido. No depende de servicios externos para el análisis; todo se ejecuta en local mediante un binario de 12 MB.

### Características Principales

- **Escaneo de Secretos con Validación de Contexto**: Detecta claves de AWS (`AKIA[0-9A-Z]{16}`), tokens de GitHub (`ghp_[0-9a-zA-Z]{36}`) y cadenas de conexión a PostgreSQL (`postgresql://user:password@host:5432/db`). Además, valida si el secreto es funcional realizando una petición HTTP sin almacenar la respuesta, usando un timeout de 500 ms por verificación. Si el secreto está rotado o es un falso positivo, lo ignora automáticamente.

- **Análisis de Dependencias con Score de Vulnerabilidad Ponderado**: Escanea `package.json`, `requirements.txt` y `Cargo.toml` comparando cada dependencia contra la base de datos local de CVEs (actualizada cada 6 horas). Asigna un score de 0 a 100 basado en la gravedad del CVE, la versión mínima afectada y la popularidad del paquete. Si el score supera 75, el pipeline falla con un código de error `DEP-ERR-045`.

- **Auditoría de Configuración de CI/CD**: Analiza archivos YAML de GitHub Actions, GitLab CI y Jenkins. Detecta permisos excesivos (ej: `write-all` en un job de prueba), uso de `pull_request_target` sin restricciones, y variables de entorno expuestas en logs. Genera un reporte en JSON con la línea exacta y una sugerencia de corrección usando expresiones regulares predefinidas.

### Cómo Usar

1. **Instalación y Configuración del Hook de Pre-commit**:
   - Descarga el binario para tu sistema operativo desde el release más reciente.
   - Ejecuta `deployguard init` en la raíz de tu proyecto. Esto crea un archivo `.deployguard.yml` con reglas por defecto.
   - Modifica el archivo para incluir tus patrones personalizados, por ejemplo:
     ```yaml
     secrets:
       patterns:
         - regex: "sk-[a-zA-Z0-9]{32}"
           name: "Stripe Secret Key"
           validate: true
     ```
   - Agrega `deployguard scan --pre-commit` a tu hook de pre-commit. Si se detecta un secreto válido, el commit se rechaza con el mensaje: `[DEP-ERR-023] Secreto encontrado: Stripe Secret Key en src/config.py:15`.

2. **Ejecución de un Escaneo Completo del Repositorio**:
   - Corre `deployguard scan --path ./ --output report.json --verbose`.
   - El escaneo recorre todos los archivos ignorando los listados en `.gitignore`. Para cada archivo, aplica las reglas de secretos, dependencias y CI/CD.
   - El reporte `report.json` incluye:
     ```json
     {
       "timestamp": "2025-03-15T10:30:00Z",
       "total_files": 142,
       "secrets_found": 0,
       "vulnerabilities": [
         {
           "package": "lodash",
           "version": "4.17.20",
           "cve": "CVE-2021-23337",
           "score": 82,
           "suggested_fix": "npm install lodash@4.17.21"
         }
       ],
       "ci_issues": [
         {
           "file": ".github/workflows/deploy.yml",
           "line": 12,
           "issue": "Permiso 'write-all' en job de pruebas",
           "severity": "high"
         }
       ]
     }
     ```
   - Si el score de alguna vulnerabilidad supera 75, el proceso termina con código de salida 1.

3. **Integración Continua con GitHub Actions**:
   - Agrega el siguiente paso a tu workflow:
     ```yaml
     - name: DeployGuard Security Scan
       uses: deployguard/action@v2
       with:
         api_key: ${{ secrets.DEPGUARD_API_KEY }}
         fail_on_score: 75
         validate_secrets: true
     ```
   - La acción descarga automáticamente la última versión del binario, ejecuta el escaneo y publica los resultados como un comentario en el PR si se detectan issues. Si el escaneo falla, el workflow se marca como fallido y se envía una notificación a tu canal de Slack configurado.

### Mejores Prácticas

- **Rotación Automática de Secretos Detectados**: Configura DeployGuard para que, al detectar un secreto válido, genere automáticamente un nuevo secreto usando la API del proveedor (ej: AWS Secrets Manager, GitHub Secrets) y lo reemplace en el archivo. Esto reduce el tiempo de exposición a menos de 2 segundos desde la detección hasta la rotación. Para habilitarlo, añade `auto_rotate: true` en la regla del secreto.

- **Uso de Reglas por Entorno con Prioridades**: Define diferentes perfiles de escaneo para desarrollo, staging y producción. Por ejemplo, en desarrollo permite secretos falsos (ej: `password=12345`) pero en producción los bloquea con severidad crítica. Usa `deployguard scan --profile production` para aplicar el perfil más restrictivo. Esto evita falsos positivos durante el desarrollo mientras mantienes la seguridad en producción.
