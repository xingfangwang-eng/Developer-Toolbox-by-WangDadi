# deployguard - DEP_015

## Resumen

DeployGuard es un analizador estático de seguridad diseñado para desarrolladores en solitario que necesitan proteger sus despliegues sin infraestructura empresarial. Escanea secretos, vulnerabilidades de dependencias y configuraciones inseguras en tiempo real, con un motor de reglas personalizables y salida en JSON/CSV.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es una herramienta profesional de auditoría de código que detecta fugas de credenciales, dependencias con CVEs conocidos y configuraciones de riesgo en proyectos Node.js, Python y Go. Integra análisis en pre-commit hooks y pipelines CI/CD, con una latencia media de 120 ms por archivo y soporte para repositorios de hasta 5000 archivos.

### Características Principales

- **Escaneo de Secretos en Tiempo Real**: Detecta API keys, tokens OAuth y contraseñas en commits no rastreados usando expresiones regulares compiladas y un diccionario de 2000 patrones. Parámetros clave: `--threshold 0.95` (precisión mínima), `--max-depth 4` (directorios anidados). Tiempo de escaneo: < 50 ms por archivo en repositorios con < 1000 archivos.
- **Análisis de Vulnerabilidades en Dependencias**: Compara el archivo `package-lock.json` o `requirements.txt` contra la base de datos local de CVEs (actualizada cada 6 horas vía GitHub Releases). Reporta severidad (CRITICAL, HIGH, MEDIUM) con enlaces directos a NVD. Ejemplo: `--cve-db /opt/deployguard/cve.db` y `--cve-update-interval 3600`.
- **Validación de Configuraciones de Despliegue**: Inspecciona archivos `.env`, `Dockerfile` y `k8s/deployment.yaml` en busca de puertos expuestos, variables sin valor por defecto y permisos 777. Parámetros: `--check-dockerfile` (valida instrucciones RUN con `apt-get` sin `--no-install-recommends`), `--check-k8s` (verifica `securityContext` y `readOnlyRootFilesystem`). Soporta integración con PostgreSQL para almacenar historial de auditorías.

### Cómo Usar

1. **Instalación y Configuración Inicial**: Descarga el binario desde [releases](https://github.com/xingfangwang-eng/deployguard/releases) o instala vía npm: `npm install -g deployguard`. Ejecuta `deployguard init` para generar el archivo `.deployguard.yml` en la raíz del proyecto. Configura `rules:` con patrones personalizados, por ejemplo: `- id: aws-key pattern: "AKIA[0-9A-Z]{16}" severity: high`.
2. **Escaneo Rápido de un Commit**: Usa `deployguard scan --path ./src --output report.json --format json`. Para integrarlo en un hook de pre-commit, añade al `.git/hooks/pre-commit`: `#!/bin/sh` seguido de `deployguard scan --staged --fail-on critical`. Si se detecta un secreto crítico, el commit se rechaza y se muestra el archivo y línea exacta.
3. **Auditoría Completa del Repositorio**: Ejecuta `deployguard audit --repo . --cve-db /opt/deployguard/cve.db --postgres-uri "postgresql://user:pass@localhost:5432/audit"`. El comando genera un reporte en HTML con gráficos de tendencias (vulnerabilidades por severidad a lo largo del tiempo) y exporta a CSV. Para programar auditorías diarias, usa cron: `0 8 * * * /usr/local/bin/deployguard audit --repo /var/www/project --output /var/reports/$(date +\%Y\%m\%d).html`.

### Mejores Prácticas

- **Integración Continua con CI/CD**: Configura DeployGuard como paso obligatorio en pipelines de GitHub Actions o GitLab CI. Usa `--fail-on high` para detener builds que introduzcan vulnerabilidades de alta gravedad. Combínalo con `--threshold 0.99` para secretos falsos positivos bajos. Ejemplo de acción: `deployguard scan --path $GITHUB_WORKSPACE --fail-on critical`.
- **Rotación de Secretos Automatizada**: Tras cada escaneo que detecte credenciales válidas, integra DeployGuard con un webhook que notifique a un gestor de secretos (HashiCorp Vault o AWS Secrets Manager). Usa `--webhook-url https://vault.example.com/rotate` y `--webhook-secret $VAULT_TOKEN` para rotar automáticamente la clave expuesta en menos de 2 segundos.
