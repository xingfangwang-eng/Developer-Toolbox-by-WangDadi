# deployguard - DEP_013

## Resumen

DeployGuard es un sistema de análisis estático y auditoría de seguridad diseñado específicamente para desarrolladores en solitario (solo developers). Permite detectar vulnerabilidades de inyección SQL, fugas de secretos hardcodeados y configuraciones inseguras en contenedores Docker, todo en un solo comando. El motor de análisis opera en **tiempo real** con una latencia máxima de **12 ms por archivo** y se integra de forma nativa con **PostgreSQL 14+** para almacenar el historial de auditorías.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es un analizador estático de código fuente que combina un escáner de secretos basado en **regex heurístico** con un validador de políticas de seguridad para infraestructura como código (IaC). Está optimizado para entornos de desarrollo local con pocos recursos: consume menos de **64 MB de RAM** en modo headless y puede procesar hasta **1,200 archivos por minuto** en un solo núcleo de CPU. Su base de reglas incluye más de **400 patrones de vulnerabilidad** actualizados semanalmente.

### Características Principales

- **Escáner de Secretos con Detección Contextual**  
  Identifica claves API, tokens OAuth, contraseñas y certificados embebidos en el código. Utiliza un motor de análisis contextual que evalúa el tipo de archivo, la posición del token y las variables de entorno cercanas. Soporta **37 formatos de secretos** (AWS, Azure, GitHub, Stripe, etc.) y genera reportes con **hash SHA-256** del secreto encontrado para evitar exponer datos sensibles en logs.

- **Auditoría de Configuración Docker y Kubernetes**  
  Analiza `Dockerfile`, `docker-compose.yml` y manifiestos de Kubernetes en busca de malas prácticas: contenedores ejecutándose como root, puertos expuestos innecesariamente, volúmenes montados con permisos 777, o imágenes base con vulnerabilidades conocidas (basado en la base de datos **CVE** actualizada cada 6 horas). Genera un puntaje de riesgo del **0.0 al 10.0** para cada archivo.

- **Integración Continua con Git Hooks**  
  Se instala como un hook `pre-commit` que analiza los cambios en el staging area antes de cada commit. Si detecta una vulnerabilidad crítica (severidad ≥ 8.0), bloquea el commit y muestra un mensaje con la línea exacta del código infractor. El tiempo de análisis para un diff de hasta **200 archivos** es inferior a **2 segundos**.

### Cómo Usar

1. **Instalación y configuración inicial**  
   Ejecuta `deployguard init` en la raíz de tu proyecto. Esto crea un archivo `.deployguard.yml` donde puedes definir exclusiones por ruta (`exclude_paths: ["vendor/", "node_modules/"]`) y umbrales de severidad (`severity_threshold: 7.0`). Luego, ejecuta `deployguard install-hooks` para activar el hook pre-commit.

2. **Análisis completo del repositorio**  
   Corre `deployguard scan --path ./src --output-format json --output-file report.json`. Este comando escanea recursivamente todos los archivos en `./src`, excluyendo los directorios definidos en la configuración. El reporte en JSON incluye: `file_path`, `line_number`, `pattern_matched`, `severity_score`, `suggested_fix` y `context_snippet` (las 3 líneas anteriores y posteriores al hallazgo).

3. **Revisión de resultados y resolución**  
   Abre el reporte generado. Para cada hallazgo, DeployGuard ofrece una sugerencia de corrección automática. Si aceptas, ejecuta `deployguard fix --file src/config.py --line 42` y la herramienta reemplazará la línea infractora por una versión segura (por ejemplo, eliminará una contraseña hardcodeada y la reemplazará por una variable de entorno). Después, vuelve a escanear para confirmar que el puntaje de riesgo haya bajado a 0.

### Mejores Prácticas

- **Integrar DeployGuard en el pipeline CI/CD**  
  Configura un job en GitHub Actions o GitLab CI que ejecute `deployguard scan --severity-threshold 5.0` antes de cada deploy. Si el análisis falla, el pipeline se detiene automáticamente. Esto previene que código inseguro llegue a producción, especialmente en proyectos donde no hay un equipo de seguridad dedicado.

- **Actualizar la base de reglas semanalmente**  
  Ejecuta `deployguard update-rules` cada lunes para descargar las últimas reglas de vulnerabilidad. DeployGuard se conecta a un repositorio público de reglas firmadas con **GPG**, lo que garantiza que no se introduzcan patrones maliciosos. Mantener las reglas actualizadas es crítico para detectar nuevas variantes de ataques como inyección de dependencias o secretos en archivos de entorno.
