# deployguard - DEP_024

## Resumen

Sistema de auditoría de seguridad de código estático para desarrolladores en solitario. Analiza vulnerabilidades en tiempo real, detecta fugas de secretos y evalúa riesgos de dependencias en proyectos Python, JavaScript y Go. Integración directa con CI/CD mediante hooks nativos.

## Contenido Detallado

### ¿Qué es DeployGuard?

Herramienta de análisis estático que escanea tu código fuente en busca de vulnerabilidades OWASP Top 10, secretos hardcodeados y dependencias obsoletas. Opera en modo offline con una base de reglas actualizable vía `deployguard update --ruleset 2024.11`. Compatible con repositorios Git, extrae el diff automáticamente tras cada commit.

### Características Principales

- **Análisis de secretos en tiempo real**  
  Detecta tokens de AWS, claves SSH, API keys de Stripe y más de 120 patrones predefinidos. Usa el parámetro `--secret-threshold 0.95` para ajustar la sensibilidad. Escanea archivos binarios y comprimidos mediante `--scan-archives true`. Precisión del 99.2% en pruebas con 10.000 muestras.

- **Auditoría de dependencias con base de datos local**  
  Conecta con PostgreSQL para almacenar el historial de vulnerabilidades conocidas (CVE). Ejecuta `deployguard deps --db postgresql://localhost:5432/deployguard --update-interval 3600` para sincronizar cada hora. Reporta riesgo de severidad (crítico, alto, medio) con tiempo de respuesta inferior a 200 ms por paquete.

- **Análisis de flujo de datos sensibles**  
  Rastrea variables que contienen datos personales (PII) o financieros a través del árbol de sintaxis abstracta (AST). Usa `--data-flow-depth 5` para profundidad de 5 niveles. Genera gráficos DOT exportables con `--export-graph ./reporte.dot`. Detecta fugas en cadenas de concatenación y literales de plantilla.

### Cómo Usar

1. **Inicializar el proyecto**  
   Ejecuta `deployguard init --project mi-app --lang python --ruleset owasp-2024` para generar el archivo de configuración `.deployguard.yml`. Define exclusiones con `exclude: ["tests/", "build/"]`. Verifica con `deployguard check --config .deployguard.yml`.

2. **Ejecutar escaneo completo**  
   Corre `deployguard scan --path ./src --severity critical --output json > reporte.json`. Para análisis incremental, usa `deployguard scan --diff HEAD~1 --format html --open-report`. El reporte HTML incluye fragmentos de código resaltados con la línea exacta de la vulnerabilidad.

3. **Integrar con CI/CD**  
   Agrega al pipeline de GitHub Actions:  
   ```yaml
   - name: DeployGuard Scan
     run: deployguard scan --path . --fail-on critical --slack-webhook ${{ secrets.SLACK_WEBHOOK }}
   ```  
   Si se detectan vulnerabilidades críticas, el pipeline falla con código de salida 1. Para ignorar falsos positivos, usa `deployguard ignore --rule-id SECRET-123 --reason "token de prueba"`.

### Mejores Prácticas

- **Programar escaneos nocturnos**  
  Configura un cron job con `deployguard scan --path /var/www --schedule daily --email-alerts admin@dominio.com` para recibir informes comprimidos en PDF cada madrugada. Almacena los resultados en PostgreSQL para análisis de tendencias semanales.

- **Combinar con análisis dinámico**  
  Usa DeployGuard como pre-filtro antes de ejecutar pruebas de penetración. Los resultados estáticos reducen el tiempo de escaneo dinámico en un 40% al eliminar falsos positivos comunes (como comentarios de código). Integra con OWASP ZAP mediante `deployguard export --format zap --output ./zap_rules.json`.
