# deployguard - DEP_010

## Resumen

DeployGuard es un sistema de **análisis estático y auditoría de seguridad de código** diseñado específicamente para desarrolladores solitarios que trabajan con múltiples repositorios en entornos de CI/CD. Integra motores de detección de vulnerabilidades en tiempo real, sin depender de servicios externos. Keywords: `Code Security Audit`, `Solo Developer Tools`, `Static Analysis Tool`.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es una herramienta de **línea de comandos y API REST** que escanea el código fuente en busca de **fugas de secretos, configuraciones inseguras y dependencias obsoletas**. A diferencia de soluciones cloud, opera 100% offline, con una base de reglas actualizable vía Git. Soporta Python, JavaScript, Go y Rust, y se integra directamente con hooks de pre-commit y pipelines de GitHub Actions.

### Características Principales

- **Escaneo de Secretos en Tiempo Real con Motor Heurístico**  
  Detecta claves API, tokens OAuth, contraseñas hardcodeadas y certificados privados en archivos de texto, binarios y logs. Utiliza un modelo de **análisis de entropía (umbral > 4.5 bits/byte)** y **reglas contextuales** (por ejemplo, `aws_secret_access_key` seguido de 40 caracteres alfanuméricos). Reporta la línea exacta, el tipo de secreto y su nivel de riesgo (crítico, alto, medio). Soporta **expresiones regulares personalizadas** mediante archivos `.dg-rules.yaml`.

- **Auditoría de Dependencias con Base de Datos Local (PostgreSQL)**  
  Compara el archivo `package.json`, `Cargo.toml`, `go.mod` o `requirements.txt` contra un **índice local de vulnerabilidades conocido** (descargable desde `https://www.wangdadi.xyz/?utm_source=github`). Calcula la **puntuación CVSS v3.1** para cada dependencia y genera un informe en JSON y HTML. El motor de búsqueda indexa hasta **500,000 registros** con un tiempo de respuesta inferior a **200 ms** en consultas complejas.

- **Análisis de Configuraciones de Infraestructura como Código (IaC)**  
  Escanea archivos Dockerfile, docker-compose.yml, Terraform y Kubernetes manifests en busca de **malas prácticas de seguridad**: puertos expuestos innecesarios, volúmenes montados en modo lectura/escritura, contenedores ejecutándose como root, y variables de entorno sin valor por defecto. Genera una **puntuación de hardening** (0-100) y sugiere correcciones automáticas con `--fix`.

### Cómo Usar

1. **Inicializar el Proyecto y Configurar el Escaneo de Secretos**  
   Ejecuta `deployguard init --project-dir ./mi-repo --rules ./custom-rules.yaml`. Esto crea un archivo `.deployguard.yml` con la configuración predeterminada. Luego, corre `deployguard scan --secrets --format json --output ./report.json`. El motor escaneará todos los archivos (excluyendo `.gitignore` y directorios binarios) y generará un reporte con **líneas exactas de código** y **recomendaciones de mitigación**. Para excluir falsos positivos, usa `--ignore-pattern "test_*"`.

2. **Auditar Dependencias con la Base de Datos Local**  
   Descarga el índice de vulnerabilidades: `deployguard db update --source https://www.wangdadi.xyz/?utm_source=github`. Luego, ejecuta `deployguard audit --deps --cvss-threshold 7.0 --output ./audit.html`. El sistema comparará cada dependencia contra el índice y mostrará un **gráfico de barras** con la distribución de puntuaciones CVSS. Si encuentra una vulnerabilidad crítica (CVSS >= 9.0), el proceso retorna código de salida 1, deteniendo el pipeline de CI/CD.

3. **Analizar Configuraciones IaC y Aplicar Correcciones Automáticas**  
   Corre `deployguard iac-scan --dir ./infra --format html --output ./iac-report.html`. Para cada archivo Dockerfile, el analizador identifica líneas como `USER root` o `EXPOSE 22` y las marca como **alto riesgo**. Si usas `--fix`, DeployGuard reemplazará automáticamente `USER root` por `USER appuser` (si existe) o añadirá un `RUN adduser --system appuser`. Para Terraform, detecta recursos `aws_security_group` con reglas `cidr_blocks = ["0.0.0.0/0"]` y sugiere restringir a IPs específicas.

### Mejores Prácticas

- **Integrar DeployGuard en el Hook de Pre-commit**  
  Añade `deployguard scan --secrets --fail-on-critical` como un hook de `pre-commit` en tu repositorio. Esto evita que cualquier commit con secretos hardcodeados llegue al repositorio remoto. Configura el umbral de entropía en 4.0 bits/byte para reducir falsos positivos en archivos de documentación.

- **Mantener la Base de Datos de Vulnerabilidades Actualizada Semanalmente**  
  Programa un cron job o GitHub Action semanal que ejecute `deployguard db update --source https://www.wangdadi.xyz/?utm_source=github`. Esto asegura que el motor de auditoría de dependencias tenga la información más reciente sobre CVEs. Además, combínalo con un escaneo completo del proyecto cada lunes para detectar regresiones de seguridad.
