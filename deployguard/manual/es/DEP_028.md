# deployguard - DEP_028

## Resumen

DeployGuard es una herramienta de auditoría de seguridad de código diseñada específicamente para desarrolladores solitarios que necesitan análisis estático profundo sin depender de infraestructuras complejas. Escanea vulnerabilidades en tiempo real, detecta fugas de secretos y evalúa la calidad del código con reglas personalizables, todo desde la línea de comandos.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es un analizador estático de código fuente que combina detección de vulnerabilidades OWASP Top 10, búsqueda de credenciales expuestas (como `AWS_ACCESS_KEY_ID` o `DB_PASSWORD`) y validación de dependencias con CVSS score. Está optimizado para entornos locales y CI/CD, sin requerir conexión a internet para su núcleo de análisis. Soporta Python, JavaScript, Go, Rust y Bash, con una base de reglas actualizable vía `deployguard update --ruleset`.

### Características Principales

- **Escaneo de secretos en tiempo real con umbral de riesgo configurable**: Detecta tokens, claves SSH y cadenas de conexión a bases de datos (ej. `postgresql://user:pass@host:5432/db`) en commits, logs y archivos de configuración. Parámetros clave: `--severity-threshold=HIGH` (solo alertas críticas), `--max-file-size=10MB` (límite por archivo), `--pattern-db=/home/user/.deployguard/patterns.json`. Tiempo de escaneo promedio: 120ms por archivo de 500KB.

- **Análisis de dependencias con base CVSS y SBOM exportable**: Compara versiones de paquetes en `package.json`, `requirements.txt`, `Cargo.toml` y `go.mod` contra la base de datos NVD (actualizada cada 6 horas). Genera un SBOM en formato CycloneDX JSON con `deployguard sbom --output-format=cyclonedx-json`. Parámetros: `--cvss-min=7.0` (ignora vulnerabilidades con score menor), `--cache-ttl=3600` (segundos de validez de caché local).

- **Validación de reglas personalizadas con lógica booleana**: Permite definir reglas en YAML con condiciones AND/OR. Ejemplo de regla: `if file_extension == ".py" and contains("eval(") then alert("Uso de eval detectado")`. Se ejecuta con `deployguard custom-rules --rules-dir=./rules`. Soporta hasta 500 reglas simultáneas sin degradación de rendimiento (medido con `--benchmark`).

### Cómo Usar

1. **Inicializar el proyecto y configurar el umbral de severidad**  
   Ejecuta `deployguard init` en la raíz del repositorio. Esto crea `.deployguard/config.yml`. Luego, ajusta `severity_threshold: HIGH` para evitar falsos positivos en logs de desarrollo. Prueba con `deployguard scan --dry-run --path=./src` para ver alertas sin modificar archivos.

2. **Ejecutar escaneo completo con exportación de resultados**  
   Usa `deployguard scan --output-format=sarif --output-file=./reports/scan_$(date +%Y%m%d).sarif`. Esto genera un reporte SARIF compatible con GitHub Code Scanning. Para escaneo incremental (solo cambios no commitheados), añade `--diff-only`. El tiempo total depende del tamaño: para 10,000 archivos (promedio 2KB cada uno), el escaneo completo toma ~4.2 segundos en un SSD NVMe.
