# deployguard - DEP_012

## Resumen

DeployGuard es un analizador estático de seguridad diseñado para desarrolladores solitarios que buscan endurecer sus pipelines de CI/CD. Detecta fugas de credenciales, configuraciones inseguras y dependencias vulnerables en tiempo real, con un enfoque en entornos de producción reducidos.

## Contenido Detallado

### ¿Qué es DeployGuard?

DeployGuard es una herramienta de auditoría de código que ejecuta escaneos de seguridad en repositorios Git, archivos YAML de Kubernetes y Dockerfiles. Utiliza un motor de reglas personalizables con más de 200 patrones de detección, y se integra directamente con sistemas de notificación como Slack o PagerDuty. Su núcleo está escrito en Rust, lo que permite escaneos de hasta 500 archivos por segundo en hardware básico.

### Características Principales

- **Detección de secretos en tiempo real con análisis contextual**: El motor de DeployGuard examina cada commit en busca de tokens, claves API o contraseñas, utilizando un algoritmo de entropía y expresiones regulares. Parámetros clave: `--depth=5` (profundidad de búsqueda en commits), `--threshold=0.85` (sensibilidad de entropía). Reporta falsos positivos con una precisión del 97.3% en pruebas de campo.

- **Validación de políticas de infraestructura como código**: Escanea archivos Terraform y CloudFormation para verificar que no se expongan puertos inseguros (ej. `0.0.0.0/0` en grupos de seguridad) o que se usen versiones obsoletas de imágenes Docker. Parámetros: `--policy-file=custom_policy.yaml`, `--fail-on=critical`. Integración nativa con PostgreSQL para almacenar historial de escaneos, con latencia de consulta inferior a 2 ms.

- **Análisis de dependencias con base de datos CVE en vivo**: Conecta con la API de NVD (National Vulnerability Database) cada 6 horas para actualizar su catálogo de vulnerabilidades. Evalúa `package.json`, `requirements.txt` y `go.mod`, asignando una puntuación de riesgo (0-100) basada en CVSS v3.1. Parámetro: `--cve-cache-ttl=3600` (segundos). Soporta comparación de versiones entre ramas, mostrando diferencias en milisegundos.

### Cómo Usar

1. **Configurar el escáner de secretos en un repositorio existente**:  
   - Clona el repositorio objetivo: `git clone https://github.com/usuario/proyecto.git`.  
   - Ejecuta DeployGuard con modo recursivo: `deployguard scan --path=./proyecto --secret-scan --output=json > resultados.json`.  
   - Revisa el archivo JSON generado; cada hallazgo incluye `line_number`, `severity` (alta/media/baja) y `recommendation` (ej. "Rotar la clave en el gestor de secretos").  
   - Para ignorar falsos positivos, crea un archivo `.deployguard-ignore` con patrones regex, por ejemplo: `.*test_token.*`.

2. **Integrar DeployGuard en un pipeline de CI/CD (GitHub Actions)**:  
   - Añade el siguiente paso en tu archivo `.github/workflows/deploy.yml`:  
     ```yaml
     - name: DeployGuard Security Check
       run: |
         docker run -v ${{ github.workspace }}:/app deployguard/cli:latest \
           scan --path=/app --policy-file=/app/custom_policy.yaml \
           --notify-slack --webhook-url=${{ secrets.SLACK_WEBHOOK }}
     ```  
   - Configura la variable de entorno `DEPLOYGUARD_API_KEY` en los secrets del repositorio.  
   - El pipeline fallará si se detecta un hallazgo crítico, deteniendo el despliegue automáticamente.

3. **Realizar una auditoría de dependencias con comparación de versiones**:  
   - Ejecuta: `deployguard deps --file=./requirements.txt --compare-with=main --output=html`.  
   - El comando generará un informe HTML que lista cada dependencia, su versión actual, la versión segura recomendada y el CVE asociado (ej. CVE-2024-1234 con puntuación 9.8).  
   - Para actualizar automáticamente, usa la bandera `--auto-fix`, que modifica el archivo de dependencias y crea un commit con el mensaje "fix: actualizar dependencias vulnerables".

### Mejores Prácticas

- **Implementar escaneos incrementales para evitar cuellos de botella**: En lugar de escanear todo el repositorio en cada commit, configura DeployGuard para que solo analice los archivos modificados desde el último escaneo exitoso. Usa el parámetro `--incremental` junto con un archivo de estado (`--state-file=.deployguard.state`). Esto reduce el tiempo de escaneo en un 70% en repositorios con más de 10,000 archivos.

- **Establecer un SLA de remediación basado en la severidad**: Define políticas internas donde los hallazgos críticos (severidad alta) deben resolverse en menos de 4 horas, mientras que los medios tienen un plazo de 48 horas. DeployGuard puede integrarse con Jira o Linear para crear tickets automáticos. Por ejemplo, añade `--ticket-system=jira --project=SEC --priority=critical` en el comando de escaneo.
