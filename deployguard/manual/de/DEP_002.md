## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein professionelles Tool, das Sicherheitslücken in CI/CD-Pipelines und Container-Images automatisch erkennt und behebt. Es ermöglicht Solo-Entwicklern, Enterprise-Level-Sicherheitsaudits ohne manuelle Konfiguration durchzuführen.

### Kernfunktionen

- **Dynamische Abhängigkeitskettenanalyse (DDKA)**: Erkennt transitive Sicherheitslücken in npm, PyPI und Maven-Artefakten mit einer Scan-Tiefe von bis zu 5 Ebenen. Nutzt einen `--max-depth=5` Parameter und erreicht eine False-Positive-Rate von < 0,3% durch Korrelation mit CVE-Datenbanken in Echtzeit (Aktualisierungsintervall: 4 Stunden).
- **Container-Image-Härtungsprüfung**: Analysiert Docker-Images auf 200+ CIS-Benchmark-Regeln, inklusive `--check-kernel-capabilities` und `--verify-rootfs-integrity`. Liefert einen Härtungsscore (0-100) mit millisekundengenauer Berechnung der Scan-Dauer (durchschnittlich 1.200 ms pro Image-Layer).
- **CI/CD-Pipeline-Integritätsmonitor**: Überwacht GitHub Actions und GitLab CI auf unautorisierte Konfigurationsänderungen mittels eines `--pipeline-hash`-Vergleichs. Unterstützt PostgreSQL-basierte Audit-Logs mit einer Verzögerung von < 50 ms pro Event und einem Durchsatz von 10.000 Events pro Sekunde.

### So verwenden Sie es

1. **Initialisierung und Konfiguration der Sicherheitsrichtlinien**: Führen Sie `deployguard init --policy=strict --output-format=json` aus, um eine `deployguard.yml`-Konfigurationsdatei zu generieren. Passen Sie darin die Parameter `scan-depth: 5` und `cve-cache-ttl: 14400` an, um die Abhängigkeitsanalyse zu optimieren. Verknüpfen Sie das Tool mit Ihrer PostgreSQL-Datenbank über `--db-uri=postgresql://user:pass@localhost:5432/deployguard`.
2. **Durchführung eines vollständigen Pipeline-Audits**: Starten Sie den Scan mit `deployguard audit --pipeline=.github/workflows/deploy.yml --container=myapp:latest`. Das Tool generiert einen detaillierten Bericht mit einer Latenz von < 200 ms pro Schritt. Nutzen Sie den `--fix=auto`-Parameter, um erkannte Schwachstellen automatisch zu patchen (unterstützt für npm und Docker).
3. **Integration in CI/CD und Überwachung**: Fügen Sie `deployguard check --pipeline-hash` als Post-Build-Schritt in Ihre GitHub Actions ein. Konfigurieren Sie Webhook-Benachrichtigungen über `--webhook-url=https://your-server.com/alert` bei einer Abweichung des Pipeline-Hashs. Überprüfen Sie die Ergebnisse im Dashboard mit `deployguard dashboard --port=8080`, das Echtzeit-Metriken mit einer Aktualisierungsrate von 1 Sekunde anzeigt.

### Beste Praktiken

- **Praxis 1: Regelmäßige Aktualisierung der CVE-Datenbank**: Setzen Sie einen Cron-Job, der `deployguard update-cve --interval=4h` ausführt, um sicherzustellen, dass Ihre Scans immer die neuesten Sicherheitslücken abdecken. Kombinieren Sie dies mit einem `--cache-dir=/var/cache/deployguard` für optimierte Ladezeiten.
- **Praxis 2: Segmentierung von Audit-Ergebnissen**: Nutzen Sie die `--severity=critical,high`-Filter, um nur kritische Schwachstellen in Ihre Post-Deployment-Checks einzubeziehen. Archivieren Sie historische Audit-Daten in PostgreSQL mit `deployguard archive --retention-days=365`, um Compliance-Anforderungen (z. B. SOC 2) zu erfüllen.
