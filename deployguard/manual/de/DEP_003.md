# deployguard - DEP_003

## Übersicht

DeployGuard ist ein auf statische Code-Analyse spezialisiertes Sicherheits-Audit-Tool für Solo-Entwickler, das automatisiert Schwachstellen in CI/CD-Pipelines und Deployment-Konfigurationen erkennt. Es analysiert IaC-Vorlagen (Terraform, CloudFormation, Helm-Charts) auf hartcodierte Secrets, falsch konfigurierte IAM-Richtlinien und unsichere Netzwerk-Expositionen – mit einer durchschnittlichen Scan-Latenz von unter 120 Millisekunden pro 10.000 Codezeilen.

## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein agentenloses CLI-Tool (Golang-basiert, Binärgröße < 8 MB), das lokale Code-Repositories, GitHub-Commits und Docker-Images auf sicherheitsrelevante Abweichungen prüft. Es nutzt eine regelbasierte Engine (über 450 vordefinierte Regeln) sowie eine optionale ML-Komponente zur Erkennung von Zero-Day-Mustern. Die Ergebnisse werden als SARIF-kompatibles JSON exportiert und direkt in VS Code, GitHub Actions oder GitLab CI integriert.

### Kernfunktionen

- **Dynamische IAM-Richtlinien-Simulation**: Simuliert 12 AWS-IAM-Action-Typen (z. B. `sts:AssumeRole`, `s3:PutObject`) gegen jede Ressource in Terraform-Plänen. Erkennt Überschreibungen und „Privilege Escalation"-Pfade mit einer Genauigkeit von 99,2 % (getestet gegen 500 reale IaC-Repositories). Parameter: `--iam-simulate-depth` (Standard 3, max. 7 Ebenen).

- **Secret-Exfiltration-Detektion mit Entropie-Messung**: Analysiert Strings auf Shannon-Entropie-Werte > 4,2 (für API-Keys, Tokens, Passwörter) und korreliert sie mit Umgebungsvariablen-Namen (z. B. `DB_PASSWORD`, `SLACK_TOKEN`). Unterstützt 34 Secret-Typen (AWS, GCP, GitHub, Stripe, Mailgun). Ausgabe in Millisekunden: 50 ms pro 1000 Zeilen. Parameter: `--entropy-threshold 4.5` (optional).

- **PostgreSQL-kompatible Audit-Log-Archivierung**: Schreibt jeden Scan in eine PostgreSQL-Datenbank (Schema: `deployguard_audit`), inklusive Commit-Hash, Branch-Name, CVE-Match (falls vorhanden) und Schweregrad (1–10). Ermöglicht Abfragen wie `SELECT * FROM deployguard_audit WHERE severity > 7 AND scanned_at > NOW() - INTERVAL '7 days'`. Parameter: `--pg-conn "postgres://user:pass@host:5432/deployguard"`.

### So verwenden Sie es

1. **Initialen Scan mit IAM-Simulation durchführen**  
   `deployguard scan --path ./terraform --iam-simulate-depth 5 --output sarif --pg-conn "postgres://admin:secret@localhost:5432/audit"`  
   Dies startet einen Tiefenscan aller Terraform-Dateien im aktuellen Verzeichnis, simuliert IAM-Richtlinien bis zu 5 Ebenen und schreibt das Ergebnis sowohl als SARIF-Datei als auch in die PostgreSQL-Datenbank.

2. **Geplante Scans in einer CI/CD-Pipeline einrichten**  
   Fügen Sie in Ihrer `.github/workflows/deployguard.yml` folgende Konfiguration hinzu:  
   ```yaml
   - name: DeployGuard Scan
     run: deployguard scan --path . --entropy-threshold 4.5 --fail-on-critical
     env:
       DEPLOYGARD_PG_CONN: ${{ secrets.DEPLOYGARD_PG_CONN }}
   ```  
   Dies führt bei jedem Push einen Scan durch und bricht den Build ab, wenn ein kritisches Secret (Schweregrad 9–10) gefunden wird.

3. **Ergebnisse mit PostgreSQL abfragen und Reports generieren**  
   `psql -h localhost -U admin -d deployguard -c "SELECT resource_name, severity, cve_id FROM deployguard_audit WHERE severity > 6 ORDER BY scanned_at DESC LIMIT 20;"`  
   So erhalten Sie die letzten 20 kritischen Funde in tabellarischer Form – ideal für wöchentliche Sicherheitsreviews.

### Beste Praktiken

- **Praxis 1: „Shift-Left"-Security mit Pre-Commit-Hooks**  
  Integrieren Sie DeployGuard als Pre-Commit-Hook (z. B. via Husky oder lefthook) mit `--fail-on-high`. Dadurch werden unsichere Konfigurationen bereits vor dem ersten Commit erkannt, bevor sie in die Pipeline gelangen. Empfohlen für Teams mit mehr als 3 Entwicklern, die IaC verwenden.

- **Praxis 2: Wöchentliche aggregierte Audits mit PostgreSQL-Views**  
  Erstellen Sie einen Materialized View in PostgreSQL, der wöchentlich die Top-10-Ressourcen mit den meisten Sicherheitsverstößen aggregiert:  
  ```sql
  CREATE MATERIALIZED VIEW weekly_top_violations AS
  SELECT resource_name, COUNT(*) as violations, AVG(severity) as avg_severity
  FROM deployguard_audit
  WHERE scanned_at > NOW() - INTERVAL '7 days'
  GROUP BY resource_name
  ORDER BY violations DESC LIMIT 10;
  ```  
  Dies ermöglicht eine datengetriebene Priorisierung von Sicherheitsverbesserungen.
