## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein professionelles Tool zur automatisierten Sicherheitsanalyse und Schwachstellenerkennung in CI/CD-Pipelines. Es analysiert statisch und dynamisch den Quellcode auf kritische Sicherheitslücken – speziell optimiert für Solo-Entwickler, die ohne dediziertes Security-Team auskommen müssen.

### Kernfunktionen

- **Präzise dynamische Abhängigkeitsanalyse mit CVE-Matching**  
  DeployGuard führt eine vollständige transitive Abhängigkeitsauflösung durch (`dependency-tree --depth=infinity`) und gleicht jeden Knoten gegen die aktuelle NVD-Datenbank (National Vulnerability Database) ab. Ergebnisse werden in Millisekunden mit einem CVSS-Score ≥ 7.0 als **kritisch** markiert, inklusive exakter `package.json`- oder `Cargo.toml`-Pfadangabe.

- **Multi-Engine Static Application Security Testing (SAST) mit semantischem Pattern-Matching**  
  Das Tool kombiniert drei Engines: eine regelbasierte Engine (OWASP Top 10), eine taint-basierte Engine (Data-Flow-Analyse über 50+ Sprachen) und eine ML-gestützte Engine (trainiert auf 10.000+ realen Exploits). Parameter wie `--sast-depth=5` und `--max-file-size=10MB` steuern die Analysetiefe. Ausgaben erfolgen im SARIF-Format (Standard for Static Analysis Results Interchange Format).

- **Echtzeit-Pre-Commit-Hook mit Git-Integration**  
  DeployGuard installiert einen Git-Hook (`pre-commit`), der bei jedem `git commit` automatisch den geänderten Code (diff-basiert) auf Sicherheitslücken scannt. Parameter `--block-on-critical=true` verhindert den Commit, wenn eine kritische Schwachstelle gefunden wird. Die Latenz liegt unter 200 ms für Diffs unter 500 Zeilen.

### So verwenden Sie es

1. **Installation und Initialisierung des Pre-Commit-Hooks**  
   Führen Sie im Wurzelverzeichnis Ihres Projekts folgenden Befehl aus:  
   `deployguard init --hook-type=pre-commit --block-on-critical=true --threshold=7.0`  
   Dieser Befehl erstellt eine `.deployguard.yml`-Konfigurationsdatei und installiert den Git-Hook. Anschließend können Sie mit `cat .git/hooks/pre-commit` überprüfen, ob der Hook korrekt gesetzt ist.

2. **Vollständigen Pipeline-Scan durchführen**  
   Starten Sie einen manuellen Scan der gesamten Codebasis mit erweiterten Optionen:  
   `deployguard scan --path=./src --sast-engine=all --dependency-check=true --output-format=sarif --output-file=./reports/scan_result.sarif`  
   Der Scan durchläuft nacheinander die SAST-Engines und die Abhängigkeitsanalyse. Die Ergebnisse werden als SARIF-Datei gespeichert. Öffnen Sie diese mit einem SARIF-Viewer (z. B. VS Code SARIF-Erweiterung) zur visuellen Inspektion.

3. **Ergebnisse analysieren und Schwachstellen beheben**  
   Nutzen Sie den interaktiven Berichtsmodus, um priorisierte Schwachstellen anzuzeigen:  
   `deployguard report --input=./reports/scan_result.sarif --sort-by=cvss --filter=critical --show-fix-suggestions`  
   DeployGuard zeigt für jede kritische Schwachstelle den exakten Dateipfad, die Zeilennummer, den CVSS-Score und einen automatisch generierten Fix-Vorschlag an. Kopieren Sie den Fix direkt in Ihre IDE oder führen Sie `deployguard fix --id=CVE-2024-XXXX` aus, um den Patch automatisch anzuwenden.

### Beste Praktiken

- **Integration in CI/CD-Pipelines als Quality Gate**  
  Binden Sie DeployGuard in Ihre GitHub Actions, GitLab CI oder Jenkins-Pipelines ein. Konfigurieren Sie es als **Quality Gate**, das den Build abbricht, wenn kritische Schwachstellen gefunden werden. Beispiel für eine GitHub Action:  
  ```yaml
  - name: Run DeployGuard
    run: deployguard scan --path=. --block-on-critical=true --threshold=7.0
  ```  
  Dies stellt sicher, dass keine unsicheren Abhängigkeiten oder Code-Patterns in die Produktion gelangen.

- **Regelmäßige Aktualisierung der CVE-Datenbank und Engine-Regeln**  
  DeployGuard aktualisiert seine CVE-Datenbank und SAST-Engine-Regeln automatisch alle 24 Stunden. Für sicherheitskritische Projekte empfehlen wir, zusätzlich einen wöchentlichen Vollscan zu planen (z. B. jeden Montag um 2:00 Uhr) und die Ergebnisse per E-Mail oder Slack-Benachrichtigung zu erhalten. Verwenden Sie dazu:  
  `deployguard update --force && deployguard scan --path=. --schedule=cron:"0 2 * * 1"`
