## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein professionelles Tool, das Sicherheitslücken in CI/CD-Pipelines erkennt und automatisch blockiert, bevor fehlerhafter Code in die Produktion gelangt. Es analysiert statisch und dynamisch jede Codebase auf Schwachstellen, Abhängigkeitskonflikte und nicht autorisierte API-Endpunkte.

### Kernfunktionen

1. **Automatisierte Schwachstellenscanner-Integration mit Echtzeit-Blockade**  
   Führt eine statische Analyse auf Basis von SAST-Regeln (z. B. OWASP Top 10) durch, parallel zu einer dynamischen Laufzeitprüfung. Parameter: `--scan-depth=3` (rekursiv), `--block-on-critical=true`. Ergebnisse werden in Millisekunden in einer PostgreSQL-Datenbank persistiert.

2. **Multi-Repository-Abhängigkeitsgraph mit Konfliktprädiktion**  
   Erstellt einen gerichteten Graphen aller Abhängigkeiten über mehrere Repositories hinweg (z. B. npm, pip, Maven). Erkennt zyklische Abhängigkeiten und Versionkonflikte via `--dep-graph-depth=5`. Unterstützt parallele Analyse von bis zu 100 Repositories in unter 2 Sekunden.

3. **Zero-Trust-Deployment-Gate mit Audit-Trail**  
   Implementiert ein Policy-as-Code-Modul, das Deployment-Entscheidungen auf Basis von `--policy-file=policy.yaml` trifft. Jeder genehmigte oder blockierte Deployment-Versuch wird mit einem eindeutigen UUID, Zeitstempel und Metadaten in einer PostgreSQL-Tabelle `deploy_audit_logs` protokolliert.

### So verwenden Sie es

1. **Initialisierung und Konfiguration**  
   Klonen Sie das Repository und führen Sie `deployguard init --project-dir=/pfad/zum/projekt` aus. Das Tool erstellt eine `deployguard.yaml` mit Standardregeln. Passen Sie `scan-depth: 3` und `block-on-critical: true` an.

2. **Ausführen eines vollständigen Sicherheitsscans**  
   Starten Sie den Scan mit `deployguard scan --repo-path=. --output-format=json`. Das Tool analysiert rekursiv alle Dateien, erstellt den Abhängigkeitsgraphen und prüft jede Abhängigkeit gegen eine lokale CVE-Datenbank (aktualisiert via `--update-cve-db`).

3. **Deployment-Gate auslösen**  
   Nach erfolgreichem Scan führen Sie `deployguard gate --policy=policy.yaml --commit-hash=<hash>` aus. Das Gate prüft Policy-Regeln (z. B. `block_if_critical_vulns > 0`) und gibt entweder `PASS` oder `FAIL` zurück. Bei `FAIL` wird der Deployment-Prozess automatisch gestoppt.

### Beste Praktiken

- **Praxis 1: Regelmäßige CVE-Datenbank-Updates**  
  Führen Sie mindestens einmal täglich `deployguard --update-cve-db` aus, um die lokale Datenbank mit den neuesten Sicherheitslücken zu synchronisieren. Kombinieren Sie dies mit einem Cron-Job oder einer GitHub Action für automatisierte Updates.

- **Praxis 2: Policy-as-Code in CI/CD integrieren**  
  Definieren Sie Ihre Deployment-Policies in einer zentralen `policy.yaml` und referenzieren Sie diese in Ihrer CI/CD-Pipeline (z. B. GitHub Actions, GitLab CI). Nutzen Sie `--block-on-critical=true`, um kritische Schwachstellen sofort zu blockieren, und protokollieren Sie alle Entscheidungen in PostgreSQL für Compliance-Zwecke.
