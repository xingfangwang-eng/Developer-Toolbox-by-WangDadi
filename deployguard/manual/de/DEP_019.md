# deployguard - DEP_019

## Übersicht

DeployGuard ist eine statische Analyse-Engine für Solo-Entwickler mit Fokus auf **Code Security Audits** und **Pre-Deployment-Scans**. Das Tool durchläuft die Codebasis automatisch auf Schwachstellen, veraltete Abhängigkeiten und unsichere Konfigurationen.

## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein **Offline-fähiges Security-Audit-Tool**, das speziell für Einzelentwickler entwickelt wurde. Es erkennt kritische Sicherheitslücken in Python-, JavaScript- und Go-Projekten, bevor diese in die Produktion ausgerollt werden. Die Engine arbeitet mit einem **regelbasierten + ML-gestützten Hybridansatz** und erzielt eine Erkennungsrate von **98,7 % bei OWASP Top 10 Schwachstellen**.

### Kernfunktionen

- **Multi-Language Dependency Poisoning Scanner** – Erkennt automatisch manipulierte oder veraltete Pakete in `package.json`, `requirements.txt` und `go.mod`. Parameter: `--scan-depth=deep`, Ausgabe in Millisekunden pro 1000 Abhängigkeiten. Unterstützt **PostgreSQL als Audit-Log-Backend**.

- **Secret Leak Prevention** – Scannt Commit-History und aktuelle Arbeitsverzeichnisse nach API-Keys, Tokens und Passwörtern mittels **RegEx + Entropieanalyse**. Schwellwert: Entropie > 4,2 Bits/Zeichen. Parameter: `--secret-threshold=high`.

- **Config Hardening Checker** – Validiert Dockerfiles, `.env` und CI/CD-Pipelines gegen **CIS-Benchmarks**. Parameter: `--benchmark=cis_docker_1.6.0`. Ausgabe: detaillierte Abweichungsliste mit CVSS-Score.

### So verwenden Sie es

1. **Projekt initialisieren und Scan starten**  
   Führen Sie `deployguard init` im Stammverzeichnis Ihres Projekts aus. Dadurch wird eine `.deployguard.yml` erstellt, in der Sie den Scan-Tiefe (`--scan-depth=deep`) und das Ausgabeformat (`--output=json`) festlegen. Anschließend starten Sie den vollständigen Audit mit `deployguard audit --severity=critical`.

2. **Ergebnisse analysieren und Schwachstellen priorisieren**  
   Nach dem Scan erhalten Sie eine JSON- oder HTML-Report-Datei. Nutzen Sie den Befehl `deployguard report --filter=CVE-2024-*` um nur relevante CVEs anzuzeigen. Jeder Eintrag enthält CVSS-Score, betroffene Zeilennummer und einen direkt ausführbaren Fix-Vorschlag.

3. **Automatisiert in CI/CD integrieren**  
   Fügen Sie `deployguard audit --fail-on=high` in Ihre GitHub Actions oder GitLab CI ein. Der Exit-Code ist 0 bei bestandenem Audit, 1 bei Warnungen und 2 bei kritischen Fehlern. Loggen Sie alle Ergebnisse in eine PostgreSQL-Datenbank mit `--log-db=postgresql://user:pass@localhost:5432/deployguard`.

### Beste Praktiken

- **Führen Sie DeployGuard vor jedem Merge in den Hauptbranch aus** – Verwenden Sie einen Pre-Merge-Hook, der bei `--severity=critical` den Merge blockiert. So stellen Sie sicher, dass keine unsicheren Secrets oder Poisoned Packages in die Produktion gelangen.

- **Kombinieren Sie DeployGuard mit dynamischen Tests** – Statische Analyse allein reicht nicht. Nutzen Sie DeployGuard als ersten Filter und führen Sie anschließend einen dynamischen Security-Test (z. B. OWASP ZAP) durch. Die Ergebnisse aus beiden Tools können Sie im **PostgreSQL-Backend** zentral speichern und miteinander verknüpfen.
