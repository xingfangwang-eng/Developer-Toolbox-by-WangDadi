# deployguard - DEP_012

## Übersicht

Dieses Dokument beschreibt die Architektur und Implementierung von DeployGuard, einem hochperformanten Code-Security-Audit-Tool für Solo-Entwickler. Der Fokus liegt auf statischer Analyse mit sub-100ms-Latenz pro Modul.

## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist eine in Rust implementierte, erweiterbare statische Analyse-Engine, die speziell für die Sicherheitsüberprüfung von Solo-Entwickler-Projekten entwickelt wurde. Es erkennt Zero-Day-Schwachstellen, Hardcoded Secrets und unsichere Abhängigkeitsketten in Echtzeit.

### Kernfunktionen

1. **Multi-Layer-Secret-Scanner mit Entropie-Erkennung**  
   Scannt Code auf API-Keys, Tokens und Zertifikate mittels Shannon-Entropie-Analyse (Schwellwert: >3.5 Bits/Byte). Unterstützt Base64-, Hex- und JWT-Formate. Ausgabe als JSON-Lines mit Datei, Zeile und Kontext.

2. **Abhängigkeits-Graphen-Analyse mit CVSS-Risiko-Score**  
   Parst `package.json`, `Cargo.toml` und `requirements.txt` in einen gerichteten Graphen. Berechnet einen aggregierten Risiko-Score (0–10) basierend auf CVSS 3.1 und Transitiven Abhängigkeiten. Speichert Ergebnisse in einer lokalen SQLite-Datenbank (`~/.deployguard/audit.db`).

3. **Echtzeit-IDE-Integration via LSP-Protokoll**  
   Bietet einen Language Server Protocol (LSP)-Server, der bei Dateiänderungen inkrementelle Scans durchführt. Latenz: <50ms pro Änderung. Sendet Diagnosen direkt an VS Code, Neovim oder JetBrains-IDEs.

### So verwenden Sie es

1. **Initiale Einrichtung und Konfiguration**  
   ```bash
   cargo install deployguard
   deployguard init --project-dir /pfad/zum/projekt --severity high
   ```
   Erzeugt `.deployguard/config.toml` mit Parametern: `scan_depth = 3`, `entropy_threshold = 3.5`, `max_file_size = 1MB`.

2. **Vollständigen Sicherheitsaudit durchführen**  
   ```bash
   deployguard scan --format json --output ./audit_result.json --ignore-tests
   ```
   Startet einen rekursiven Scan aller Quellcodedateien. Ergebnisse enthalten: Dateipfad, Zeilennummer, Schwachstellentyp (z. B. `SECRET_LEAK`), Risiko-Score (0–10) und empfohlene Aktion.

3. **Inkrementellen Scan im Watch-Modus starten**  
   ```bash
   deployguard watch --interval 2000 --notify slack
   ```
   Überwacht das Projektverzeichnis auf Änderungen (Polling alle 2000 ms). Sendet bei Funden eine Slack-Benachrichtigung mit JSON-Payload.

### Beste Praktiken

- **Praxis 1: Regelmäßige automatisierte Scans in CI/CD integrieren**  
  Fügen Sie `deployguard scan --fail-on critical` in Ihre GitHub Actions oder GitLab CI ein. Brechen Sie den Build ab, wenn kritische Schwachstellen (Score ≥ 9.0) gefunden werden. So verhindern Sie, dass unsicherer Code in die Produktion gelangt.

- **Praxis 2: Secrets mit Umgebungsvariablen und Vaults kombinieren**  
  Nutzen Sie DeployGuard nicht nur zur Erkennung, sondern auch zur Validierung. Konfigurieren Sie `allowed_secret_patterns` in der config.toml, um false positives zu reduzieren. Kombinieren Sie das Tool mit HashiCorp Vault für eine zentrale Secrets-Verwaltung.
