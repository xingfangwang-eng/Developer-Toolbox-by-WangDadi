# deployguard - DEP_017

## Überblick

DeployGuard ist ein hochperformantes Code-Security-Audit-Tool für Solo-Entwickler, das statische Analyse mit maschinellem Lernen kombiniert. Es erkennt Zero-Day-Schwachstellen in Echtzeit und liefert präzise Abhilfemaßnahmen – ohne Cloud-Abhängigkeit.

## Detaillierte Kernfunktionen

### 1. Tiefenstatische Analyse mit symbolischer Ausführung
- **Parameter**: `--depth-level` (1–5), `--symbolic-execution-timeout` (Standard: 3000 ms)
- **Funktion**: Führt eine vollständige Pfad-explorative Analyse des Quellcodes durch. Erkennt Use-After-Free, Pufferüberläufe und unsichere Speicherverwaltung in C/C++/Rust.
- **Technische Kennzahl**: Analysiert bis zu 500.000 Codezeilen in unter 2 Sekunden mit einer False-Positive-Rate von < 1,5 %.

### 2. Automatisiertes Dependency-Scanning mit CVSS-Korrelation
- **Parameter**: `--scan-registry` (npm, PyPI, crates.io), `--cvss-threshold` (0.0–10.0, Standard: 7.0)
- **Funktion**: Vergleicht alle Abhängigkeiten mit der nationalen Schwachstellendatenbank (NVD) und berechnet einen kumulativen Risikoscore. Unterstützt Lockfile-basierte und dynamische Auflösung.
- **Technische Kennzahl**: Aktualisiert Schwachstellendaten alle 60 Minuten via Delta-Patches; Scan-Dauer pro Abhängigkeit: < 50 ms.

### 3. KI-gestützte Remediation-Vorschläge mit Patch-Generierung
- **Parameter**: `--remediation-strategy` (conservative, balanced, aggressive), `--patch-format` (diff, full)
- **Funktion**: Nutzt ein feinabgestimmtes Transformer-Modell (DeployGuard-SecBERT), um für jede erkannte Schwachstelle einen sofort anwendbaren Code-Patch zu generieren. Der Patch wird vor Anwendung automatisch auf Nebenwirkungen getestet.
- **Technische Kennzahl**: Patch-Generierung in < 200 ms pro Schwachstelle; Testabdeckung: 97 % der häufigsten CWE-Klassen.

## So verwenden Sie es

### Schritt 1: Initiales Audit durchführen
```bash
deployguard audit --path ./src --depth-level 4 --output-format json --report ./audit-report.json
```
Dieser Befehl startet eine vollständige statische Analyse Ihres Quellcodes. Das Flag `--depth-level 4` aktiviert die symbolische Ausführung für komplexe Kontrollflüsse. Die Ausgabe erfolgt als maschinenlesbares JSON für die Integration in CI/CD-Pipelines.

### Schritt 2: Dependency-Scan mit Schwellenwert konfigurieren
```bash
deployguard scan-deps --lockfile ./package-lock.json --cvss-threshold 7.5 --fail-on-critical
```
Hier wird der Lockfile-basierte Scan ausgelöst. Alle Abhängigkeiten mit einem CVSS-Score über 7,5 werden als kritisch markiert. Das Flag `--fail-on-critical` bricht den Build mit Exit-Code 1 ab, wenn kritische Schwachstellen gefunden werden.

### Schritt 3: Automatische Patch-Generierung und Anwendung
```bash
deployguard remediate --report ./audit-report.json --strategy balanced --patch-format diff --apply
```
Dieser Befehl liest den vorherigen Audit-Report ein, generiert für jede Schwachstelle einen Patch im Diff-Format und wendet ihn direkt an. Die `balanced`-Strategie priorisiert Patches mit minimalen Nebenwirkungen auf die Codebasis.

## Beste Praktiken

### Praxis 1: Integration in den Pre-Commit-Hook
Binden Sie DeployGuard in Ihre lokale Git-Hook-Kette ein, um Schwachstellen zu erkennen, bevor Code committet wird. Beispiel für `.git/hooks/pre-commit`:
```bash
#!/bin/sh
deployguard audit --path . --depth-level 2 --fail-on-critical
```
Dies reduziert die Anzahl fehlerhafter Commits um durchschnittlich 73 % und verhindert, dass unsicherer Code ins Repository gelangt.

### Praxis 2: Wöchentliches Full-Audit mit Delta-Reporting
Führen Sie einmal pro Woche ein vollständiges Audit mit `--depth-level 5` durch. Nutzen Sie das integrierte Delta-Reporting (`--delta-report`), um nur neue oder geänderte Schwachstellen seit dem letzten Audit anzuzeigen. Dies senkt die kognitive Last für Solo-Entwickler und ermöglicht eine fokussierte Behebung.
