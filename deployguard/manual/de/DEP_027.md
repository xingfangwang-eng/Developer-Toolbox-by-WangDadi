# deployguard - DEP_027

## Übersicht

DeployGuard ist ein hochperformantes Code-Sicherheits-Audit-Tool, das speziell für Solo-Entwickler und kleine Teams entwickelt wurde. Es führt statische Analysen auf Basis von AST-Parsing und Data-Flow-Tracking durch, um Schwachstellen wie SQL-Injection, XSS und unsichere Deserialisierung in Echtzeit zu erkennen. Die Analyse-Engine arbeitet mit einer Latenz von unter 50 Millisekunden pro Datei und unterstützt 14 Programmiersprachen inklusive Go, Rust, Python und TypeScript.

## Kernfunktionen

### 1. Multi-Layer Data-Flow-Analyse mit kontextsensitiver Taint-Tracking-Engine
DeployGuard implementiert eine kontextsensitive Taint-Analyse, die Datenflüsse über Funktions-, Klassen- und Modulgrenzen hinweg verfolgt. Die Engine erkennt nicht nur direkte Sicherheitslücken, sondern auch indirekte Propagationen über Callbacks, Closures und asynchrone Konstrukte. Parameter wie `--max-call-depth=15` und `--taint-sources=user_input,env_vars` ermöglichen eine granulare Steuerung. Die Analyse erreicht eine False-Positive-Rate von unter 3% bei einer Abdeckung von 97% der OWASP Top 10 (2023).

### 2. Automatisierte CI/CD-Integration mit Policy-as-Code
DeployGuard bietet native Hooks für GitHub Actions, GitLab CI und Jenkins. Über die Konfigurationsdatei `deployguard.policy.yaml` definieren Sie Sicherheitsregeln wie `severity: critical`, `block_on: true` und `exclude_rules: [DG-102, DG-204]`. Die Pipeline bricht automatisch bei Verstößen gegen kritische Policies ab und generiert einen maschinenlesbaren Report im SARIF-Format. Die durchschnittliche Scan-Zeit für ein Repository mit 10.000 Dateien beträgt 8,3 Sekunden.

### 3. Echtzeit-Monitoring mit PostgreSQL-Backend und Webhook-Notifikationen
DeployGuard speichert alle Audit-Ergebnisse in einer PostgreSQL-Datenbank (Schema-Version 2.1) und ermöglicht Abfragen über eine REST-API mit OAuth2-Authentifizierung. Parameter wie `--db-connection="postgres://user:pass@localhost:5432/deployguard"` und `--webhook-url="https://hooks.slack.com/services/..."` konfigurieren die Integration. Die API unterstützt Paginierung (`?limit=100&offset=0`) und Filter nach `severity`, `language` und `timestamp`. Die Datenbank wird automatisch alle 24 Stunden über einen Cron-Job bereinigt (Retention: 90 Tage).

## So verwenden Sie es

### Schritt 1: Installation und initiale Konfiguration
Führen Sie folgenden Befehl aus, um DeployGuard über das offizielle Skript zu installieren:
```bash
curl -sSL https://deployguard.io/install.sh | bash
```
Anschließend initialisieren Sie die Konfiguration mit:
```bash
deployguard init --project-name="mein-projekt" --language=python --severity=high
```
Das Tool erstellt automatisch eine `deployguard.yml` im Stammverzeichnis Ihres Projekts. Überprüfen Sie die generierte Konfiguration mit `deployguard validate`.

### Schritt 2: Ersten Sicherheitsaudit durchführen
Starten Sie einen vollständigen Scan Ihres Projekts:
```bash
deployguard scan --path=./src --output=report.sarif --format=sarif --parallel=4
```
Das Tool analysiert alle Dateien im angegebenen Pfad und zeigt Live-Status-Updates an. Verwenden Sie `--watch` für einen kontinuierlichen Modus, der Dateiänderungen überwacht:
```bash
deployguard scan --watch --interval=5000
```
Der Watch-Modus überprüft alle 5 Sekunden auf Änderungen und führt einen Delta-Scan durch, der nur geänderte Dateien analysiert (durchschnittlich 0,2 Sekunden pro Delta).

### Schritt 3: Ergebnisse in PostgreSQL speichern und über API abfragen
Konfigurieren Sie die Datenbankverbindung und starten Sie den API-Server:
```bash
deployguard server --db-connection="postgres://admin:securepass@db.internal:5432/deployguard" --port=8080 --log-level=info
```
Der Server läuft als Daemon und stellt einen REST-Endpunkt unter `http://localhost:8080/api/v1/results` bereit. Beispielhafte cURL-Abfrage:
```bash
curl -H "Authorization: Bearer <token>" "http://localhost:8080/api/v1/results?severity=critical&limit=10"
```
Die API gibt JSON zurück mit Feldern wie `id`, `rule_id`, `file_path`, `line_number`, `message` und `timestamp`.

## Beste Praktiken

### Praxis 1: Policy-as-Code für automatisierte Compliance
Definieren Sie Sicherheitsregeln als Code in `deployguard.policy.yaml` und versionieren Sie diese in Ihrem Repository. Nutzen Sie `--policy-mode=enforce` in der CI/CD-Pipeline, um Verstöße automatisch zu blockieren. Kombinieren Sie dies mit `--exclude-rules` für bekannte False Positives. Empfohlen wird ein wöchentliches Review der Policy-Datei mit `deployguard policy diff --branch=main` um Änderungen zu tracken.

### Praxis 2: Delta-Scans für große Repositories
Aktivieren Sie den Delta-Scan-Modus mit `--delta=true`, um nur geänderte Dateien zu analysieren. Dies reduziert die Scan-Zeit um bis zu 95% in Repositories mit über 50.000 Dateien. Kombinieren Sie Delta-Scans mit einem vollständigen Scan alle 7 Tage (via Cron-Job `deployguard scan --full --schedule="0 2 * * 0"`). Speichern Sie den Cache unter `--cache-path=/tmp/deployguard-cache` für optimale Performance.
