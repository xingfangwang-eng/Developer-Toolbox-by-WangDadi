## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein **offline-fähiger, statischer Security-Linter** für monolithische und Microservice-Repositories. Es analysiert Abhängigkeitsgraphen, Konfigurationsdateien und Quellcode auf CVEs, Hardcoded Secrets und unsichere API-Aufrufe – ohne Daten an Dritte zu senden. Die Engine arbeitet mit einer **Regelbasis von über 1.200 Pattern** und erzeugt Ergebnisse in **Millisekunden pro 1.000 LOC**.

### Kernfunktionen

- **Multi-Layer Dependency Graph Analyzer**  
  Extrahiert den gesamten Abhängigkeitsbaum (npm, pip, go.mod, Maven) und gleicht ihn gegen eine **lokal gecachte CVE-Datenbank** ab. Parameter: `--depth=5` (Rekursionstiefe), `--threshold=critical` (nur CVSS >= 9.0). Unterstützt **Delta-Scans**: Nur geänderte Pakete seit letztem Scan.

- **Secret Pattern Matcher mit Kontext-Engine**  
  Erkennt API-Keys, Tokens, Private Keys und Connection Strings in Code, YAML, Dockerfile, `.env`. Nutzt **Regex + Entropie-Analyse** (Shannon-Entropie > 4.2). Parameter: `--entropy-threshold=4.5`, `--exclude-patterns=test/*,*.md`. Ausgabe im **SARIF-Format** für direkte Integration in VS Code, GitHub CodeQL oder GitLab SAST.

- **CI/CD Native Blocking Policy Engine**  
  Definiere Schwellwerte pro Branch: `--fail-on=high` bricht den Build ab, wenn mehr als 2 High-Severity-Funde. Parameter: `--policy-file=.deployguard.yml`. Unterstützt **Pre-Commit-Hooks** (via `pre-commit` oder `lefthook`) und **GitHub Action** mit automatischem Kommentar im PR. Latenz: < 200ms pro 100 Dateien.

### So verwenden Sie es

1. **Initialscan und Baseline erstellen**  
   ```bash
   deployguard scan --path=./src --output=baseline.json --format=sarif
   ```
   Dies erzeugt eine `baseline.json` mit allen aktuellen Schwachstellen. Nutze `--ignore-existing` um alte Funde auszublenden. Der Scan durchläuft alle Dateien mit **max. 4 parallelen Threads** (einstellbar via `--workers`).

2. **Policy-basierte PR-Prüfung einrichten**  
   ```yaml
   # .deployguard.yml
   policy:
     branches:
       main:
         fail-on: high
         max-secrets: 0
       develop:
         fail-on: medium
         max-secrets: 2
   ```
   Führe im CI aus:  
   ```bash
   deployguard check --diff=origin/main...HEAD --policy=.deployguard.yml
   ```
   Der Befehl vergleicht nur geänderte Dateien und stoppt den Build, wenn die Policy verletzt wird. Ergebnis wird als **Exit Code 0/1** zurückgegeben.

3. **Secrets-Prüfung mit automatischer Maskierung**  
   ```bash
   deployguard secrets --path=./config --entropy-threshold=4.5 --mask-output
   ```
   Alle gefundenen Secrets werden im Terminal mit `***` maskiert. Exportiere als **CSV** (`--format=csv`) für Compliance-Reports. Nutze `--exclude-patterns=*.log` um Rauschen zu vermeiden.

### Beste Praktiken

- **Scheduled Full Scan + Daily Delta Scan**  
  Führe einmal pro Woche einen vollständigen Scan (`deployguard scan --path=.`) durch, um neue CVEs im gesamten Repository zu erkennen. Für tägliche Läufe nutze `--diff` nur auf geänderte Dateien – das reduziert die Laufzeit um **80 %** und verhindert Alarmmüdigkeit.

- **Secrets-Prüfung in der Pre-Commit-Phase**  
  Integriere `deployguard secrets --path=. --entropy-threshold=4.5` als Pre-Commit-Hook. So gelangen keine Tokens in den Git-History. Kombiniere mit `git-secrets` oder `trufflehog` für eine zweite Ebene. Die Hook-Latenz sollte unter **500ms** bleiben – deployguard erreicht dies durch Caching der Entropie-Matrix.
