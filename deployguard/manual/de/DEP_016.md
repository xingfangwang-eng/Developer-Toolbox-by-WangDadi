# deployguard - DEP_016

## Übersicht

DeployGuard ist ein hochperformantes Code-Sicherheits-Audit-Tool, das speziell für Solo-Entwickler und kleine Teams entwickelt wurde. Es führt statische Code-Analysen mit einer Latenz von unter 50 Millisekunden pro Datei durch und identifiziert kritische Sicherheitslücken in Echtzeit.

## Detaillierter Inhalt

### Was ist DeployGuard?

DeployGuard ist ein agentenloses Sicherheits-Framework, das Ihren Quellcode auf OWASP Top 10-Schwachstellen, Hardcoded Secrets und unsichere Abhängigkeiten scannt. Es integriert sich nahtlos in CI/CD-Pipelines und unterstützt über 20 Programmiersprachen, darunter Python, JavaScript, Go und Rust.

### Kernfunktionen

- **Exploit-Präventions-Engine**: Automatische Erkennung von SQL-Injection, XSS und Command-Injection mit einer False-Positive-Rate von unter 2%. Parameter wie `--threshold=critical` und `--exclude-pattern=*.test.*` ermöglichen granulare Kontrolle.
- **Secret-Leak-Detection**: Scannt auf API-Keys, Tokens und Passwörter in Millisekunden pro Commit. Unterstützt über 200 Regex-Muster und Custom Rules über `--secret-db=./custom_rules.json`.
- **Dependency-Vulnerability-Mapper**: Vergleicht installierte Pakete mit der NVD-Datenbank und zeigt CVSS-Scores direkt im Terminal an. Parameter `--depth=3` analysiert transitive Abhängigkeiten.

### So verwenden Sie es

1. **Initialer Scan mit erweiterten Filtern**: Führen Sie `deployguard scan --path=./src --format=json --output=report.json --severity=high` aus. Das Tool erstellt einen detaillierten Bericht mit Zeilennummern, CVE-IDs und empfohlenen Fixes.
2. **Integration in GitHub Actions**: Fügen Sie in Ihrer Workflow-Datei den Schritt `- name: DeployGuard Audit && run: deployguard ci --token=${{ secrets.GH_TOKEN }} --fail-on=critical` hinzu. Der Build wird bei kritischen Schwachstellen gestoppt.
3. **Echtzeit-Überwachung mit Webhook**: Starten Sie den Daemon-Modus mit `deployguard daemon --watch=./src --webhook=https://your-server.com/alert`. Bei jeder Änderung wird automatisch ein Scan ausgelöst und das Ergebnis als JSON-Payload an den Webhook gesendet.

### Beste Praktiken

- **Praxis 1**: Definieren Sie für jedes Repository eine `.deployguard.yml`-Konfigurationsdatei mit spezifischen Ausschlussmustern und benutzerdefinierten Regeln. Verwenden Sie `--config=.deployguard.yml` um Team-Richtlinien durchzusetzen und die Audit-Zeit um bis zu 40% zu reduzieren.
- **Praxis 2**: Kombinieren Sie DeployGuard mit einem Secret-Rotation-Service. Nachdem ein Leak erkannt wurde, triggern Sie automatisch eine Rotation über `deployguard rotate --secret-id=aws_key --provider=aws`. Dies minimiert die Expositionszeit auf unter 5 Minuten.
