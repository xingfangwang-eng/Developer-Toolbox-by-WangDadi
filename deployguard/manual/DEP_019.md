# deployguard - DEP_019

## Overview

DeployGuard is a zero-configuration static analysis engine purpose-built for solo developers and small teams who need production-grade code security audits without the overhead of enterprise CI/CD pipelines. It scans your repository for hardcoded secrets, misconfigured deployment manifests, and vulnerable dependency chains — all within a single CLI command.

## Detailed Content

### What is DeployGuard?

DeployGuard acts as a local security auditor that runs in under 300 milliseconds per 10,000 lines of code. It parses your source tree, cross-references known CVE databases, and flags insecure patterns before they ever reach a staging environment. Unlike generic linters, DeployGuard is trained specifically on deployment-stage vulnerabilities: exposed API keys in environment files, overly permissive IAM roles in YAML, and outdated packages with active exploit payloads.

### Core Features

- **Secret Leak Detection with Regex-Entropy Hybrid Engine**  
  Scans for 47 predefined secret patterns (AWS keys, GitHub tokens, JWT secrets) using a combination of regular expression matching and Shannon entropy scoring. Detects base64-encoded credentials and obfuscated tokens with a false positive rate below 2.3%. All matches are logged with exact line numbers and surrounding context snippets.

- **Deployment Manifest Validation Against OWASP Top 10**  
  Parses Docker Compose, Kubernetes manifests, and Terraform HCL files to enforce least-privilege principles. Flags containers running as root, exposed ports without TLS, and storage volumes with world-readable permissions. Integrates with a local rule set stored in `~/.deployguard/rules/` that can be extended via custom YAML policies.

- **Dependency Graph Vulnerability Mapping**  
  Resolves your `package-lock.json`, `requirements.txt`, or `go.sum` against a local snapshot of the National Vulnerability Database (NVD), updated daily via `deployguard update --db`. Outputs a dependency tree with CVSS scores, affected versions, and remediation commands. Supports transitive dependency analysis for npm, PyPI, and Go modules.

### How to Use

1. **Initialize a Scan Profile**  
   Run `deployguard init --project-name my-app --scan-depth all` in your project root. This generates a `.deployguard.yaml` configuration file where you can set ignore patterns (`ignore-paths: ["node_modules", "vendor"]`) and custom entropy thresholds (`entropy-threshold: 4.5`). The tool will also create a baseline snapshot of your current dependency tree.

2. **Execute a Full Audit**  
   Execute `deployguard audit --format json --output report.json` to scan the entire repository. The engine will process all files matching `*.{py,js,yaml,yml,json,env,go}` and output a structured JSON report containing `leaks`, `manifest_violations`, and `vulnerabilities` arrays. Each entry includes a `severity` field (critical, high, medium, low) and a `fix` suggestion.

3. **Review and Suppress False Positives**  
   Use `deployguard review --report report.json --interactive` to step through flagged items one by one. For each finding, you can mark it as `--suppress` with a reason string (e.g., `--reason "test credential for local dev only"`). Suppressed items are written to `.deployguard/suppressions.json` and excluded from future scans unless you run `deployguard audit --reset-suppressions`.

### Best Practices

- **Run DeployGuard as a Pre-Commit Hook**  
  Integrate `deployguard audit --fail-on critical --min-ratio 0.8` into your `.git/hooks/pre-commit` script. This ensures no commit containing a critical-level secret leak or manifest violation enters your repository. The `--min-ratio` flag enforces that at least 80% of files are scanned before the hook passes, preventing accidental partial scans.

- **Schedule Weekly Dependency Updates with CVE Sync**  
  Automate `deployguard update --db && deployguard audit --focus vulnerabilities` via a cron job or GitHub Actions scheduled workflow. This keeps your local NVD snapshot fresh and re-scans only the dependency tree, which typically completes in under 2 seconds for projects with fewer than 500 dependencies. Pair this with `--output html --open` to generate a browsable report.
