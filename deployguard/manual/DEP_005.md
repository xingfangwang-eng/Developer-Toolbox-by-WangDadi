# deployguard - DEP_005

## Overview

DeployGuard is a static analysis and code security audit engine designed specifically for solo developers and small teams who need enterprise-grade vulnerability detection without the overhead of a full CI/CD pipeline. It integrates directly into your local development workflow, scanning for OWASP Top 10 vectors, dependency chain exploits, and misconfigured deployment manifests before they reach production.

## Detailed Content

### What is DeployGuard?

DeployGuard is a zero-configuration, file-level static analysis tool that audits your codebase for security flaws, credential leaks, and unsafe API usage patterns. It operates on a per-commit basis, producing a severity-ranked report with actionable fix suggestions. Unlike cloud-based scanners, DeployGuard runs entirely offline, ensuring your source code never leaves your machine.

### Core Features

- **Real-time Secret Leak Detection with Contextual Redaction**: Scans for hardcoded API keys, tokens, and private keys using entropy-based pattern matching (`--scan-depth=deep`). Supports 40+ provider signatures (AWS, GitHub, Stripe). On detection, it automatically replaces the exposed string with `[REDACTED]` in the output log and logs the file path, line number, and surrounding code context into a local SQLite database (`~/.deployguard/secrets.db`). Detection latency is under 50ms per file.

- **Dependency Chain Exploit Mapping**: Parses `package-lock.json`, `requirements.txt`, and `go.sum` to build a transitive dependency tree. Uses a locally cached CVE database (updated via `deployguard update --cve-db`) to flag any package version with a known vulnerability. Outputs a directed acyclic graph (DAG) showing the exact import path from your source to the vulnerable module. Supports `--fail-on-critical` to block commits if any critical CVE is found.

- **Deployment Manifest Validation**: Validates Kubernetes YAML, Docker Compose files, and Terraform HCL against a set of 120+ security rules (e.g., `container.privileged: false`, `network.policy.egress: restricted`). Runs a dry-run simulation (`--dry-run --format=json`) that outputs a compliance score (0–100) and a list of violations with remediation commands. Integrates with `pre-commit` hooks to prevent insecure manifests from being committed.

### How to Use

1. **Initialize a new audit session**  
   Run `deployguard init --project-name=myapp --scan-root=./src --output-format=html`. This creates a `.deployguard/config.yaml` file with default scan parameters. The tool will then perform a full recursive scan of all `.js`, `.py`, `.ts`, and `.yaml` files, generating a baseline report in `./reports/initial-audit.html`.

2. **Run a targeted secret scan on a single file**  
   Execute `deployguard scan --file=./src/config.py --secret-only --context-lines=3 --severity=high`. This command skips dependency and manifest checks, focusing only on credential leaks. It outputs a JSON object to stdout with the structure `{"secrets_found": 2, "entries": [{"line": 42, "type": "AWS_ACCESS_KEY", "confidence": 0.97}]}`. The tool will also write a log entry to `~/.deployguard/audit.log` with a timestamp and file hash.

3. **Generate a compliance report for CI integration**  
   Use `deployguard ci --branch=main --fail-on=medium --report-path=./ci-report.json`. This runs all three scan modes (secrets, dependencies, manifests) and exits with code 1 if any medium or higher severity issue is found. The JSON report includes a `summary` object with `total_issues`, `critical_count`, and `fix_commands` array. This is designed to be piped into GitHub Actions or GitLab CI.

### Best Practices

- **Integrate DeployGuard into your pre-commit hook**  
  Add `deployguard scan --pre-commit --fail-on=high` to your `.git/hooks/pre-commit` script. This ensures no commit with a high-severity secret leak or CVE-flagged dependency ever reaches your repository. Pair this with a `.deployguard/ignore.yaml` file to whitelist false positives (e.g., test fixtures or mock data).

- **Schedule weekly full-audit runs with CVE database updates**  
  Set up a cron job or CI scheduled trigger to run `deployguard update --cve-db && deployguard scan --full --output-format=pdf --email-report=you@domain.com`. This keeps your local CVE cache fresh and generates a comprehensive PDF report that can be archived for compliance audits (e.g., SOC 2, ISO 27001). The report includes a timestamp, scan duration, and a breakdown by severity.
