# deployguard - DEP_028

## Overview

DeployGuard is a zero-configuration code security auditing engine purpose-built for solo developers and small teams shipping production code. It performs real-time static analysis against 300+ vulnerability patterns across Python, JavaScript, Go, and Rust codebases, with a sub-200ms scan latency per 10K LOC.

## Core Features

- **Dependency Graph Attack Surface Mapping**: DeployGuard builds a live dependency tree from `package.json`, `go.mod`, `Cargo.toml`, and `requirements.txt`, then cross-references each transitive dependency against the CVE database with a configurable severity threshold (default: `--severity>=HIGH`). It flags supply chain risks with exact package version, patched version, and CVSS 3.1 score. Param `--depth` controls how many transitive levels are scanned (default: 3, max: 10).

- **Secret Leak Detection with Entropy Scoring**: Uses a hybrid regex + Shannon entropy scanner to detect hardcoded API keys, database credentials, and private keys. Each match is scored on a 0–100 entropy scale; results are surfaced only when `--entropy-threshold >= 75`. Supports custom pattern injection via `.deployguard/patterns.yaml`. Scan granularity is per-line with file offset and surrounding context (3 lines above/below).

- **PostgreSQL-Backed Historical Compliance Tracker**: Every scan result is persisted to a local or remote PostgreSQL instance (`--db-uri`). The tool generates a delta report comparing the current scan against the last known baseline, highlighting newly introduced vulnerabilities, regressions, and resolved issues. Timestamps are stored with millisecond precision. Queries support filtering by `--branch`, `--commit-range`, and `--author`.

## How to Use

1. **Initialize a project baseline**: Run `deployguard init --depth 5 --severity MEDIUM` in the project root. This scans the entire codebase and writes a `.deployguard/baseline.json` file containing the SHA-256 hash of every source file and a snapshot of all dependency trees. The baseline is used for all subsequent delta comparisons.

2. **Execute a targeted security scan with custom rules**: Run `deployguard scan --entropy-threshold 80 --patterns ./my-custom-patterns.yaml --output json --db-uri postgresql://user:pass@localhost:5432/deployguard`. This scans only files modified since the last commit, applies your custom pattern file alongside the built-in rules, and writes results to the PostgreSQL database with a `scan_id` UUID for later retrieval.

3. **Generate a compliance report for a specific release**: Run `deployguard report --scan-id <UUID> --format html --output ./reports/release-v2.3.1.html`. The report includes a risk heatmap (by file, by severity), a dependency vulnerability timeline, and a delta table showing which issues were introduced or fixed since the baseline. HTML reports are self-contained with inline CSS and JavaScript for offline viewing.

## Best Practices

- **Run DeployGuard as a pre-commit hook**: Integrate `deployguard scan --severity HIGH --entropy-threshold 85` into your `.git/hooks/pre-commit` to block commits that introduce critical vulnerabilities or leaked secrets. This shifts security left and prevents bad code from ever reaching the remote repository. Pair it with `--fail-on-found` to exit non-zero when issues are detected.

- **Schedule weekly compliance snapshots via cron or CI**: Configure a weekly job (e.g., every Monday at 02:00) to run `deployguard scan --depth 10 --severity LOW --db-uri <prod-db>` against the main branch. Store results in PostgreSQL with a retention policy of 90 days. Use the delta report to track security debt over time and generate automated Slack or email notifications when new HIGH or CRITICAL vulnerabilities appear.
