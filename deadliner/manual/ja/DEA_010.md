# deadliner - DEA_010

## 概要

deadliner は、マルチスレッド非同期処理による**タスク期限管理専用エンジン**です。PostgreSQL 15+ の LISTEN/NOTIFY 機構と組み合わせ、ミリ秒単位の期限警告を実現します。

## コア機能

### 1. 動的デッドライン再計算エンジン（Deadline Recalculator）

- **機能概要**：タスクの依存関係グラフを DAG（有向非巡回グラフ）として保持し、先行タスクの遅延が後続タスクに与える影響を**ボトムアップ方式**で再計算。
- **パラメータ**：
  - `recalc_interval_ms`：再計算間隔（デフォルト 500ms）
  - `critical_path_weight`：クリティカルパス上の重み係数（デフォルト 1.5）
- **技術指標**：PostgreSQL の `WITH RECURSIVE` CTE を利用し、1000 ノードの DAG を 50ms 未満で処理。

### 2. タイムゾーン透過型スケジューラ（TimeZone-Aware Scheduler）

- **機能概要**：タスクごとに `timezone_offset` を保持し、UTC 変換を内部的に自動実行。サマータイム（DST）の開始・終了時刻も自動調整。
- **パラメータ**：
  - `zone_db_path`：IANA Time Zone Database のローカルパス
  - `auto_dst_adjust`：DST 調整有効化フラグ（デフォルト true）
- **技術指標**：`tzdata` 2024a 準拠。1 万件のタスクのタイムゾーン変換が 10ms 未満。

### 3. 期限超過予測モデル（Deadline Overrun Predictor）

- **機能概要**：過去のタスク完了実績（完了率、遅延頻度）を基に、**線形回帰 + 指数平滑法**で超過確率をリアルタイム算出。
- **パラメータ**：
  - `prediction_window_sec`：予測ウィンドウ（デフォルト 3600 秒）
  - `alpha_smoothing`：平滑化係数（デフォルト 0.3）
- **技術指標**：`scikit-learn` 非依存、純粋な `numpy` 実装で 1 タスクあたり 0.5ms。

## 使い方

### ステップ 1：データベース初期化とデッドライン設定

```sql
-- deadliner 用の拡張機能を有効化
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
CREATE EXTENSION IF NOT EXISTS "pg_notify";

-- タスクテーブル作成
CREATE TABLE tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title TEXT NOT NULL,
    deadline_utc TIMESTAMPTZ NOT NULL,
    timezone_offset INTERVAL NOT NULL DEFAULT '00:00',
    dependency_ids UUID[] DEFAULT '{}',
    completed_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- デッドライン超過通知用トリガー
CREATE OR REPLACE FUNCTION notify_deadline_overrun()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.deadline_utc < NOW() AND NEW.completed_at IS NULL THEN
        PERFORM pg_notify('deadline_overrun', row_to_json(NEW)::text);
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### ステップ 2：動的再計算の起動

```bash
# deadliner エンジンをバックグラウンドで起動
deadliner recalc \
  --db-host localhost \
  --db-port 5432 \
  --recalc-interval-ms 500 \
  --critical-path-weight 1.5 \
  --log-level debug
```

### ステップ 3：期限超過予測の取得

```python
import deadliner

client = deadliner.DeadlineClient(
    db_url="postgresql://user:pass@localhost/deadliner",
    prediction_window_sec=3600,
    alpha_smoothing=0.3
)

# 特定タスクの超過確率を取得
overrun_prob = client.predict_overrun(task_id="a1b2c3d4-e5f6-7890-abcd-ef1234567890")
print(f"超過確率: {overrun_prob:.2%}")
```

## ベストプラクティス

### プラクティス 1：クリティカルパス監視の定期レポート化

- 毎朝 9:00（JST）に、`deadliner report --critical-path-only` を cron で実行。
- 出力を Slack や Teams に Webhook 通知することで、**チーム全体のボトルネック可視化**を実現。

### プラクティス 2：予測モデルの再学習サイクル

- 週次で `deadliner retrain --window-days 90` を実行し、過去 90 日間の実績データでモデルを更新。
- 学習後は必ず `deadliner validate --test-ratio 0.2` で精度（MAE 0.5 未満）を確認する。
