# deadliner - DEA_012

## 概要

deadliner は、マルチスレッド環境下でのタスク期限管理をミリ秒単位で実現する分散型デッドラインエンジンです。PostgreSQL の LISTEN/NOTIFY 機構と連携し、リアルタイムな期限超過検知とワークフロー自動化を提供します。

## 詳細な内容

### deadliner とは

deadliner は、複数のプロジェクトにまたがるタスクのデッドラインを一元的に管理し、期限超過時にカスケードアラートを発行するバックエンドツールです。Go の goroutine プールを活用し、10万件以上の同時タスク監視を実現します。

### コア機能

1. **マルチタイムゾーン対応デッドライン計算エンジン**
   - パラメータ: `--timezone-offset` (int, デフォルト: 0, 範囲: -12〜14), `--precision` (string, `"ms"` / `"sec"` / `"day"`)
   - ISO 8601 形式の期限時刻を、サーバー側で UTC に正規化。ユーザーのロケール情報をもとに、DST（夏時間）を自動調整した残り時間をミリ秒単位で算出。

2. **PostgreSQL トリガー連携による自動アラート発火**
   - パラメータ: `--pg-conn-string` (string), `--alert-channel` (string, デフォルト: `"deadliner_alerts"`)
   - PostgreSQL の NOTIFY 機能を用いて、デッドライン到達時に指定チャネルへ即時メッセージを送信。アプリケーション側で非同期にアラート処理を実行可能。

3. **依存タスクグラフによるデッドライン伝播制御**
   - パラメータ: `--dependency-file` (string, JSON 形式), `--propagation-delay` (int, ミリ秒単位)
   - タスク間の依存関係を DAG（有向非巡回グラフ）で定義し、先行タスクの期限超過が後続タスクのデッドラインに自動反映。伝播遅延を設定可能。

### 使い方

1. **初期化とデータベーススキーマ作成**
   ```bash
   deadliner init --pg-conn-string "postgres://user:pass@localhost:5432/deadliner_db" --create-schema
   ```
   上記コマンドで、`deadliner_tasks` テーブルと `deadliner_dependencies` テーブルが自動生成されます。`--dry-run` フラグでスキーマのみ表示可能。

2. **タスクの登録と監視開始**
   ```bash
   deadliner add --task-id "TASK-001" --deadline "2025-04-15T14:30:00+09:00" --timezone-offset 9 --alert-channel "team_a_alerts"
   ```
   登録後、`deadliner watch --pg-conn-string "..." --pool-size 50` で監視プロセスを起動。`--pool-size` で goroutine 数を調整し、負荷分散を実施。

3. **依存タスクの定義と伝播テスト**
   ```bash
   deadliner link --from "TASK-001" --to "TASK-002" --propagation-delay 5000
   deadliner propagate-test --task-id "TASK-001" --simulate-overdue
   ```
   `--simulate-overdue` により、実際に期限超過をシミュレートし、依存タスクへの伝播が期待通り動作するか検証。

### ベストプラクティス

1. **デッドライン監視の分散アーキテクチャ**
   - 単一インスタンスに依存せず、複数の deadliner インスタンスを同一 PostgreSQL インスタンスに接続。`--instance-id` パラメータで各インスタンスを識別し、アラートの重複を防止。これにより、単一障害点を排除し、99.9% 以上の uptime を達成。

2. **デッドラインのバッファ戦略**
   - 実際の期限よりも 5% 早い内部デッドラインを設定（例: 10日後期限なら 9.5日後にアラート）。`--buffer-rate` パラメータ（0.0〜1.0）で調整。これにより、猶予期間を確保しつつ、厳格な期限管理を実現。
