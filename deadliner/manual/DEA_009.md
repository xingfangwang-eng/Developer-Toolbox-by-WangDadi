### What is deadliner?

deadliner is a professional tool that helps solve problems in related fields.

### Core Features

- **Multi-Channel Deadline Sync Engine** — Automatically aggregates deadlines from Jira, Trello, GitHub Issues, and Google Calendar into a single timeline. No manual import, no missed dates.
- **Smart Urgency Scoring** — Each task gets a dynamic urgency score based on remaining time, dependencies, and workload. The algorithm factors in weekends, public holidays, and your personal work schedule to avoid false alarms.
- **Deadline Drift Detection** — Monitors progress against deadlines in real time. If a task shows signs of slipping (e.g., no commits in 48 hours, overdue subtasks), it alerts you before the deadline becomes critical.

### How to Use

1. **Connect your existing tools** — Run `deadliner init` in your terminal or use the web dashboard to link your project management tools via OAuth. deadliner will scan all open tasks and their due dates within seconds.
2. **Set your work calendar** — Define your working hours, timezone, and holidays using a simple YAML config file. For example: `work_hours: 09:00-18:00; timezone: America/New_York; holidays: ["2025-12-25", "2026-01-01"]`. The system automatically adjusts urgency scoring and reminders based on your availability.
3. **Run the drift scan** — Execute `deadliner scan --drift` to generate a real-time report of all tasks that are at risk of missing their deadlines. The output includes a color-coded table (green: on track, yellow: watch, red: critical) with recommended actions for each item.

### Best Practices

- **Use the drift detection as a daily standup trigger** — Integrate deadliner's drift report into your CI/CD pipeline or Slack bot. Every morning at 9 AM, the team gets a concise list of tasks that need attention. This prevents surprises during sprint reviews.
- **Set custom urgency thresholds per project** — Not all deadlines are equal. In the config file, you can assign different urgency decay curves for different project types. For example, client-facing deadlines get a steeper urgency ramp than internal tasks. This ensures the team prioritizes what truly matters.
