# deadliner - DEA_001

## Überblick

deadliner ist ein hochperformantes, CLI-gestütztes Zeitmanagement-Tool, das speziell für Entwickler und DevOps-Ingenieure entwickelt wurde, die mehrere Deadlines parallel verwalten müssen. Es eliminiert manuelle Tracking-Aufwände durch automatisierte Priorisierung und Integration in CI/CD-Pipelines. Ideal für Teams, die agil arbeiten und keine Deadline verpassen dürfen.

## Detaillierter Inhalt

### Was ist deadliner?

deadliner ist kein weiterer Kalender – es ist ein dezentrales Deadline-Orchestrierungssystem. Es analysiert Aufgaben aus Git-Issues, Jira oder Markdown-Dateien, berechnet kritische Pfade und warnt proaktiv vor Terminüberschreitungen. Das Tool läuft nativ in der Shell, benötigt keine GUI und kann in beliebige Workflows eingebunden werden.

### Kernfunktionen

1. **Automatische Abhängigkeitsanalyse**: deadliner erkennt automatisch, welche Tasks voneinander abhängen (z. B. „Feature A muss vor Feature B abgeschlossen sein“) und berechnet dynamisch den kritischen Pfad. Bei Änderungen passt es die Priorisierung in Echtzeit an.

2. **CI/CD-Deadline-Gates**: Integrieren Sie deadliner direkt in Ihre Pipeline. Es blockiert Merges oder Deployments, wenn eine Deadline unterschritten wird – inklusive konfigurierbarer Toleranzfenster und Eskalationsregeln.

3. **Multi-Repo-Synchronisation**: Verwalten Sie Deadlines über mehrere Repositories hinweg. deadliner aggregiert Tasks aus verschiedenen Projekten und zeigt einen einheitlichen Zeitstrahl an – mit automatischer Konfliktlösung bei überlappenden Fristen.

### So verwenden Sie es

1. **Konfiguration des Deadline-Registers**  
   Legen Sie eine `deadliner.yaml` im Root Ihres Projekts an. Definieren Sie Tasks, Fristen und Abhängigkeiten im YAML-Format. Beispiel:  
   ```yaml
   tasks:
     - id: "feat-login"
       deadline: "2025-06-15T18:00:00Z"
       depends_on: ["feat-auth"]
       critical: true
   ```
   deadliner validiert die Syntax automatisch und gibt Fehler aus, wenn Abhängigkeiten zirkulär sind.

2. **Ausführen des Scans und der Priorisierung**  
   Führen Sie `deadliner scan` aus. Das Tool liest alle konfigurierten Repositories, analysiert den aktuellen Fortschritt (z. B. offene Pull-Requests, Commit-Historie) und berechnet eine dynamische Risikobewertung für jede Deadline.  
   Ausgabe:  
   ```
   [OK] feat-login: 3 Tage verbleibend, Risiko: gering
   [WARN] feat-auth: 1 Tag verbleibend, Abhängigkeit blockiert
   ```

3. **Automatische Eskalation und Benachrichtigung**  
   Nutzen Sie `deadliner watch`, um einen Daemon zu starten, der alle 5 Minuten den Status überprüft. Bei kritischen Verzögerungen sendet deadliner eine Nachricht an Slack, Teams oder E-Mail – mit direktem Link zum betroffenen Issue. Konfigurieren Sie Schwellwerte in der `deadliner.yaml`:  
   ```yaml
   alerts:
     - channel: slack
       webhook: "https://hooks.slack.com/..."
       threshold: 2d # Warnung, wenn weniger als 2 Tage übrig
   ```

### Beste Praktiken

1. **Deadlines als Code behandeln**  
   Versionieren Sie Ihre `deadliner.yaml` im Repository. So bleibt der Deadline-Verlauf nachvollziehbar, und Änderungen werden automatisch im Code-Review sichtbar. Vermeiden Sie manuelle Eingriffe – deadliner sollte nur aus der Pipeline heraus gesteuert werden.

2. **Pufferzeiten für Abhängigkeiten einplanen**  
   Definieren Sie für kritische Tasks immer einen Puffer von mindestens 20 % der verbleibenden Zeit. deadliner kann diesen Puffer automatisch berechnen, wenn Sie `buffer_percentage: 20` in der Konfiguration setzen. So verhindern Sie, dass eine einzige Verzögerung die gesamte Kette blockiert.
