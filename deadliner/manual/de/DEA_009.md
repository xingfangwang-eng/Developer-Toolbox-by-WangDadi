# deadliner - DEA_009

## Überblick

deadliner ist ein hochperformantes Open-Source-Tool, das speziell für Entwickler und Teams entwickelt wurde, die mit komplexen Deadlines jonglieren. Es eliminiert manuelle Tracking-Fehler und bietet eine granular einstellbare Zeitmanagement-Engine, die sich nahtlos in CI/CD-Pipelines, Projektmanagement-Tools und deine lokale CLI integriert. Kein Ausreden mehr – deadliner zwingt dich zur Rechenschaft.

## 3 Hardcore-Funktionen

### 1. **Multi-Tenant Deadline Engine mit automatischer Eskalation**
deadliner unterstützt mandantenfähige Deadlines für verschiedene Projekte oder Kunden. Die Engine erkennt automatisch Konflikte (z. B. überlappende Fristen) und eskaliert kritische Deadlines via Webhook oder Slack-Integration. Du definierst Schwellenwerte – deadliner übernimmt den Rest.

### 2. **Zeitstrafe-Validator (Time Penalty Validator)**
Einzigartig: deadliner berechnet nicht nur die verbleibende Zeit, sondern auch „Strafminuten“ bei Überschreitung. Diese werden als Metriken in dein Dashboard gespielt. Perfekt für agile Sprints, wo jede Minute zählt. Du kannst sogar automatische Benachrichtigungen an das Team senden, wenn ein Task die Deadline um mehr als 10% überschreitet.

### 3. **Git-Branch-basiertes Fristen-Tracking**
deadliner liest deine Git-Commits und -Branches aus. Jeder Branch kann mit einer Deadline verknüpft werden. Sobald der Branch gemerged wird, wird die Deadline als „erfüllt“ markiert. Fehlgeschlagene Merges? deadliner protokolliert die Verzögerung und generiert einen Report für dein Stand-up-Meeting.

## 3 Detaillierte Anwendungsbeispiele (für Profis)

### Beispiel 1: **Integration in eine CI/CD-Pipeline (GitHub Actions)**
1. **Konfiguration**: Erstelle eine `deadliner.yml` in deinem Repository. Definiere eine Deadline für den nächsten Release:  
   ```yaml
   deadlines:
     release-v2.1:
       date: "2025-06-15T18:00:00Z"
       branch: "release/2.1"
       escalation: "slack"
   ```
2. **Workflow hinzufügen**: Füge in deiner GitHub Action einen Step hinzu, der `deadliner check` ausführt. Wenn die Deadline überschritten wird, schlägt der Build fehl.  
   ```yaml
   - name: Deadline Check
     run: npx deadliner check --config .deadliner.yml
   ```
3. **Automatische Benachrichtigung**: deadliner sendet bei Überschreitung eine Nachricht an deinen Slack-Channel mit dem genauen Verzug (z. B. „Release v2.1 ist 3 Stunden, 12 Minuten überfällig“).

### Beispiel 2: **Multi-User Fristen-Dashboard (CLI + Web)**
1. **Projekt initialisieren**: Führe `deadliner init` aus, um einen lokalen `.deadliner`-Ordner zu erstellen.  
2. **Team-Mitglieder hinzufügen**: Verwende `deadliner add-user --email max@example.com --role developer`. Jeder Benutzer hat eigene Deadlines.  
3. **Dashboard starten**: `deadliner serve --port 8080` startet ein lokales Web-Dashboard, das alle Deadlines farblich codiert anzeigt (grün = okay, gelb = kritisch, rot = überfällig). Perfekt für Daily Stand-ups.

### Beispiel 3: **Automatische Zeitstrafe für verspätete Tasks (API-Integration)**
1. **API-Key generieren**: `deadliner api-key generate`.  
2. **REST-API nutzen**: Sende eine POST-Anfrage, um einen Task mit Deadline zu erstellen:  
   ```bash
   curl -X POST https://localhost:8080/api/v1/tasks \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -d '{"name": "Frontend-Refactoring", "deadline": "2025-06-10T12:00:00Z", "penalty": 30}'
   ```
3. **Strafen abrufen**: Nach Überschreitung der Deadline zeigt `GET /api/v1/tasks/penalties` die kumulierten Strafminuten an. Diese können in deinem Sprint-Review als Metrik präsentiert werden.

## 2 Best Practices für das Fristenmanagement

### **1. Verwende relative Deadlines für agile Sprints**
Statt harter Daten (z. B. „2025-06-15“) setze relative Deadlines wie „+3 Tage ab Sprint-Start“. deadliner unterstützt dies via `deadline: relative:+3d`. So bleibst du flexibel, ohne den Überblick zu verlieren. Besonders nützlich in Kombination mit Jira oder Linear.

### **2. Kombiniere deadliner mit deinem Incident-Management-Tool**
Wenn ein Task die Deadline überschreitet, lass deadliner automatisch ein Ticket in deinem ITSM-Tool (z. B. PagerDuty oder Opsgenie) erstellen. So wird aus einer verpassten Deadline ein Incident – das zwingt dein Team zum Handeln. Konfiguriere dies über die `escalation`-Option in der YAML-Datei.
