# deadliner - DEA_030

## Überblick

deadliner ist ein hochperformantes CLI-Tool zur automatisierten Deadlinetracking-Verwaltung, das sich nahtlos in CI/CD-Pipelines und DevOps-Workflows integrieren lässt. Es eliminiert manuelle Tracking-Aufwände durch intelligente Zeitstrahl-Analyse und bietet Echtzeit-Warnungen bei kritischen Terminüberschreitungen. Entwickelt für Teams, die präzise Fristen über mehrere Projekte hinweg orchestrieren müssen – ohne Overhead.

## Kernfunktionen

1. **Deadline-Heatmap mit KI-gestützter Risikoprognose**  
   deadliner analysiert historische Task-Daten und erstellt eine farbcodierte Heatmap, die nicht nur anstehende Fristen anzeigt, sondern auch das Risiko einer Terminüberschreitung auf Basis von Zeitpuffern und Teamgeschwindigkeit vorhersagt. Jeder Task erhält einen `risk_score` (0–100), der dynamisch aktualisiert wird.

2. **Multi-Repository-Synchronisation mit Webhook-Trigger**  
   Verknüpfen Sie Deadlines aus GitHub, GitLab und Jira in einem zentralen Dashboard. deadliner registriert Webhooks bei Repository-Änderungen und aktualisiert automatisch alle Fristen. Bei Merge-Conflicts oder Branch-Freezes werden Warnungen direkt an den Projekt-Chat (Slack/Teams) gesendet.

3. **Time-Boxed Code-Freeze-Integration**  
   Setzen Sie automatische Code-Freeze-Zeiträume für Releases. deadliner blockiert Merge-Requests in definierten Zeitfenstern und erstellt einen `deadline_lock`-Status, der nur durch manuelles Override mit 2FA aufgehoben werden kann. Ideal für Sprint-Endphasen.

## So verwenden Sie es

### 1. Installation und initiale Konfiguration
```bash
# Global installieren
npm install -g @deadliner/cli

# API-Key generieren und Umgebungsvariablen setzen
export DEADLINER_API_KEY="your-api-key"
export DEADLINER_DEFAULT_TZ="Europe/Berlin"

# Erstes Projekt initialisieren
deadliner init --project "SaaS_MVP" --deadline "2025-06-30T18:00:00+02:00" --risk-threshold 75
```
Der Befehl erstellt eine `deadliner.config.yml` im Projektstamm und registriert automatisch einen Cron-Job (Linux/macOS) oder eine geplante Aufgabe (Windows), der alle 30 Minuten den Risikoscore prüft.

### 2. Tasks mit automatischer Abhängigkeitserkennung hinzufügen
```bash
# Task mit Sub-Tasks und Abhängigkeiten definieren
deadliner add --task "API-Integration" \
  --deadline "2025-06-15" \
  --depends-on "Auth-Modul" \
  --priority critical \
  --assignee @dev-team

# Abhängigkeiten automatisch aus Commit-Messages extrahieren
deadliner scan --repo ./my-repo --branch develop --parse-commits
```
deadliner liest Commit-Messages wie `fix: resolves #42 (depends on #39)` und erstellt automatisch eine Dependency-Graph. Bei Zyklus-Erkennung wird eine Warnung ausgegeben.

### 3. Dashboard starten und Echtzeit-Monitoring
```bash
# Lokales Dashboard auf Port 3000 starten
deadliner dashboard --port 3000 --theme dark

# Oder: Headless-Modus für CI/CD
deadliner check --ci-mode --export-json ./deadline_report.json
```
Das Dashboard zeigt eine Gantt-ähnliche Ansicht mit farbcodierten Risikobalken. Im CI-Modus wird eine JSON-Datei ausgegeben, die in GitLab CI oder GitHub Actions als Artefakt weiterverarbeitet werden kann. Bei Überschreitung von `risk-threshold` wird der Build mit `exit code 1` beendet.

## Beste Praktiken

1. **Deadline-Puffer in der Konfiguration festlegen**  
   Nutzen Sie den Parameter `buffer_days` in der `deadliner.config.yml`, um automatisch 20% Pufferzeit zu jedem Task zu addieren. Beispiel: `buffer_days: 2` für Tasks mit einer Deadline unter 10 Tagen. Das verhindert unnötige Risikowarnungen bei realistischen Verzögerungen.

2. **Integration in den Daily Standup-Prozess**  
   Lassen Sie deadliner jeden Morgen um 9:00 Uhr eine Slack-Nachricht mit den Top-5-Risikotasks senden. Verwenden Sie dazu den integrierten Webhook-Client:  
   ```bash
   deadliner schedule --cron "0 9 * * 1-5" --webhook-url "https://hooks.slack.com/services/XXX" --template "daily_risk"
   ```  
   Das reduziert manuelle Status-Updates und fokussiert das Team auf kritische Pfade.
