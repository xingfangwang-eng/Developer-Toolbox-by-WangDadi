# deadliner - DEA_018

## Überblick

deadliner ist ein hochperformantes, quelloffenes Kommandozeilen-Tool zur präzisen Deadline-Verwaltung und Aufgabenverfolgung für DevOps-Ingenieure und agile Teams. Anders als herkömmliche Projektmanagement-Suiten arbeitet deadliner komplett lokal, JSON-basiert und lässt sich nahtlos in CI/CD-Pipelines sowie Git-Hooks integrieren. Es eliminiert manuelle Aufwände bei der Fristverfolgung und liefert Push-Benachrichtigungen direkt in die Konsole oder über Webhook-Endpunkte.

## Kernfunktionen

- **Distributed Deadline Synchronization (DDS)**  
  deadliner erlaubt das Verknüpfen von Aufgaben mit externen Systemen via REST-API oder WebSocket. Änderungen an Deadlines in Jira, Linear oder GitHub Issues werden automatisch abgeglichen – ohne manuelles Mapping.

- **Time-Aware Dependency Graph (TADG)**  
  Das Tool modelliert Abhängigkeiten zwischen Tasks als gerichteten Graphen mit zeitlichen Constraints. Verzögert sich ein Vorgänger, berechnet deadliner automatisch den kritischen Pfad neu und warnt vor Kaskadeneffekten.

- **Deadline Drift Detection & Alerting**  
  Mittels Machine-Learning-gestützter Trendanalyse erkennt deadliner frühzeitig, ob ein Task wahrscheinlich seine Deadline verfehlt. Alarme werden über systemd, Slack oder E-Mail ausgelöst – mit konfigurierbarem Schwellwert.

## So verwenden Sie es

1. **Initiale Konfiguration und Task-Erstellung**  
   Installieren Sie deadliner via `npm install -g deadliner` (Node.js 18+ erforderlich). Erstellen Sie eine `deadliner.json` im Projektstamm mit folgender Struktur:
   ```json
   {
     "tasks": [
       { "id": "feat-42", "title": "Authentifizierung implementieren", "deadline": "2025-04-15T18:00:00Z", "depends_on": [], "priority": 1 }
     ],
     "alerts": { "slack_webhook": "https://hooks.slack.com/..." }
   }
   ```
   Führen Sie `deadliner init` aus, um den Watchdog zu starten.

2. **Abhängigkeiten definieren und Graph visualisieren**  
   Fügen Sie in `depends_on` die IDs von Vorgänger-Tasks hinzu. Nutzen Sie `deadliner graph` zur Ausgabe des DAG im DOT-Format – ideal zur Integration in Grafana oder Mermaid-Diagramme. Beispiel:
   ```bash
   deadliner graph --format mermaid > dependencies.mmd
   ```

3. **Automatisiertes Deadline-Update und Drift-Erkennung**  
   Binden Sie deadliner in Ihren CI-Trigger ein:  
   ```yaml
   # .gitlab-ci.yml
   after_script:
     - deadliner update --task-id feat-42 --progress 85
   ```
   deadliner berechnet automatisch die Restzeit und löst bei Überschreitung des Drift-Schwellwerts (Standard: 20%) einen Alert aus. Überprüfen Sie den Status mit `deadliner status --json`.

## Beste Praktiken

- **Deadline als Code (DaC)**  
  Lagern Sie Ihre Deadline-Konfiguration in einer eigenen YAML-Datei im Repository aus und versionieren Sie diese. So bleiben Fristen nachvollziehbar und können im Code-Review geprüft werden. Kombinieren Sie deadliner mit Git Hooks, um bei jedem Commit die aktuelle Deadline-Verletzung zu checken.

- **Granulare Alert-Eskalation**  
  Definieren Sie für kritische Tasks mehrere Eskalationsstufen:  
  `info` (20% Drift) → `warning` (40%) → `critical` (60%). Nutzen Sie unterschiedliche Kanäle (z. B. Slack für Info, PagerDuty für Critical). So vermeiden Sie Alarmmüdigkeit und stellen sicher, dass nur relevante Verzögerungen eskalieren.
