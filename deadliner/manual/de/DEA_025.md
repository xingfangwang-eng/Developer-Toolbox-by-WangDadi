# deadliner - DEA_025

## Überblick

deadliner ist eine hochperformante, plattformunabhängige Fristen-Engine für Entwickler, die präzises Task-Deadline-Tracking in CI/CD-Pipelines, Microservices-Architekturen und agilen Workflows ermöglicht. Statt generischer To-Do-Listen setzt deadliner auf ISO-konforme Zeitlogik, automatische Eskalationspfade und maschinenlesbare Metriken – ideal für Teams, die ihre Deadlines nicht nur verwalten, sondern **garantieren** müssen.

## Harte Kernfunktionen

### 1. Automatische Fristen-Eskalation mit Multi-Chain-Propagation

deadliner erkennt Abhängigkeiten zwischen Tasks und propagiert Verzögerungen automatisch entlang der gesamten Abhängigkeitskette. Wenn ein vorgelagerter Task seine Deadline überschreitet, werden alle abhängigen Deadlines dynamisch neu berechnet – inklusive Pufferzeitkorrektur und Benachrichtigung über Webhooks, Slack oder E-Mail. Kein manuelles Nachjustieren mehr.

### 2. ISO 8601-konforme Zeitlogik mit Zeitzonen- und Kalender-Engine

Integrierte Unterstützung für wiederkehrende Deadlines, Feiertagskalender (z. B. DE, AT, CH) und Arbeitszeitmodelle. deadliner interpretiert `R/2025-12-31T23:59:00+01:00/P1M` korrekt und berechnet exakte Restlaufzeiten unter Berücksichtigung von Wochenenden und gesetzlichen Feiertagen. Perfekt für internationale Teams mit verteilten Zeitzonen.

### 3. Deadliner-API mit GraphQL-Schema und Event-Streaming

Bietet eine vollständige GraphQL-Schnittstelle für CRUD-Operationen auf Deadlines, Tasks und Eskalationsregeln. Zusätzlich ein WebSocket-basierter Event-Stream für Echtzeit-Updates – ideal für Dashboards, CI/CD-Integrationen oder eigene Monitoring-Tools. Alle Mutationen sind idempotent und unterstützen optimistisches Locking.

## Detaillierte Anleitung

### So integrieren Sie deadliner in Ihre CI/CD-Pipeline

1. **Initialisieren Sie das Repository mit dem deadliner CLI-Tool**  
   Führen Sie `deadliner init --project-id=DEA_025 --timezone=Europe/Berlin` aus. Dadurch wird eine `.deadliner.yml`-Konfigurationsdatei erstellt, in der Sie globale Fristen, Eskalationsregeln und Webhook-Endpunkte definieren. Beispiel:  
   ```yaml
   deadlines:
     build: "2025-06-30T18:00:00+02:00"
     deploy: "2025-07-01T09:00:00+02:00"
   escalation:
     - type: slack
       channel: "#build-alerts"
       threshold: 120m
   ```

2. **Verknüpfen Sie Tasks mit Ihrem CI/CD-System**  
   Nutzen Sie den `deadliner schedule`-Befehl direkt in Ihrer Pipeline (z. B. GitHub Actions, GitLab CI). Beispiel:  
   ```bash
   deadliner schedule --task=build --deadline="2025-06-30T18:00:00+02:00" --depends-on=test
   ```  
   Der Task wird automatisch in der Abhängigkeitskette registriert. Bei Überschreitung löst deadliner den konfigurierten Eskalations-Webhook aus.

3. **Überwachen Sie Echtzeit-Deadlines mit dem Dashboard**  
   Starten Sie den integrierten WebSocket-Server mit `deadliner serve --port=8080`. Öffnen Sie `http://localhost:8080/graphql` und führen Sie eine Abfrage aus:  
   ```graphql
   query {
     tasks(projectId: "DEA_025") {
       name
       deadline
       remaining
       status
     }
   }
   ```  
   Sie erhalten eine Live-Liste aller Tasks mit Restlaufzeit in Millisekunden und Eskalationsstatus.

## Best Practices

### 1. Nutzen Sie relative Deadlines für wiederkehrende Aufgaben

Statt absoluter Daten (`2025-12-31`) verwenden Sie relative Angaben wie `P30D` oder `R/2025-07-01/P1W`. deadliner berechnet diese automatisch neu, wenn sich der Startzeitpunkt verschiebt. So bleiben Ihre Deadlines robust gegenüber Sprint-Verschiebungen oder unerwarteten Verzögerungen.

### 2. Implementieren Sie eine zweistufige Eskalation

Konfigurieren Sie für kritische Tasks zwei Eskalationsschwellen: eine moderate (z. B. 50 % der Pufferzeit überschritten) für eine automatische Slack-Benachrichtigung und eine kritische (90 %) für einen PagerDuty- oder Opsgenie-Alarm. deadliner unterstützt beliebig viele Eskalationsstufen mit unterschiedlichen Kanälen – ideal für SLA-gebundene Projekte.
