# deadliner - DEA_005

## Übersicht

Deadliner ist ein **agiles Deadline-Management-Tool** für Developer, die mit sprunghaften Anforderungen und parallelen Sprints jonglieren. Es eliminiert die manuelle Tracking-Last und fokussiert auf **dynamische Zeitlinien**, nicht auf starre Fristen. Das Tool ist ideal für Remote-Teams, die eine **echtzeitfähige, dezentrale Deadline-Synchronisation** benötigen.

Optimiert für: **Deadline management, Task deadline tracking, Time management tool**

## Detaillierter Inhalt

### Was ist deadliner?

Deadliner ist kein weiterer Kalender. Es ist ein **ereignisgesteuerter Task-Scheduler**, der Deadlines nicht als fixe Termine, sondern als **variable Zeitfenster** behandelt. Es integriert sich tief in CI/CD-Pipelines und Git-Workflows, um automatisch Fristen zu generieren, die auf tatsächlicher Arbeitsgeschwindigkeit basieren. Es löst das Problem der **kognitiven Überlastung** durch manuelles Deadline-Tracking und bietet stattdessen eine **probabilistische Prognose** für Aufgabenabschlüsse.

### Kernfunktionen

1.  **Dynamisches Deadline-Floating (DDF):** Statt festen Daten berechnet Deadliner automatisch **Pufferzeiten** basierend auf historischer Sprint-Geschwindigkeit und aktueller Task-Komplexität. Deadlines verschieben sich intelligent, wenn Abhängigkeiten blockieren oder Prioritäten wechseln – ohne manuelles Nachjustieren.
2.  **Git-basierte Task-Verankerung:** Jeder Task wird direkt an einen **Git-Branch oder Commit** gebunden. Deadliner liest aus `git log` und `git blame` automatisch den Fortschritt. Ein Merge Request löst eine Deadline-Neuberechnung aus. Kein manuelles Status-Update mehr.
3.  **Probabilistische Zeitprognose (PTP):** Deadliner gibt keine fixen Daten aus, sondern eine **Wahrscheinlichkeitsverteilung** (z.B. „80% Wahrscheinlichkeit für Abschluss bis Freitag 14:00 Uhr“). Dies basiert auf Monte-Carlo-Simulationen über vergangene Sprint-Daten. Das reduziert die Illusion von Sicherheit und fördert realistische Planung.

### So verwenden Sie es

1.  **Task-Verkettung und Abhängigkeitsdefinition:**
    Definieren Sie Tasks in einer YAML-Datei (`deadliner.yml`) im Root Ihres Repos. Nutzen Sie die `depends_on`-Direktive, um Abhängigkeiten zu anderen Tasks oder externen API-Endpunkten zu setzen. Beispiel: `task: "API-Integration" depends_on: ["Backend-Commit", "Swagger-Final"]`. Deadliner parst diese Datei beim `git push` und erstellt automatisch einen **Deadline-Graphen**.

2.  **Integration in den CI/CD-Pipeline (GitHub Actions/GitLab CI):**
    Fügen Sie einen `deadliner sync` Schritt in Ihre Pipeline ein. Dieser Schritt ruft die Deadliner-API auf, um den Task-Status zu aktualisieren. Nutzen Sie den Output (`deadliner_output.json`) für automatisierte Benachrichtigungen (z.B. Slack). Beispiel: `- run: deadliner sync --token ${{ secrets.DEADLINER_TOKEN }} --branch ${{ github.ref_name }}`. Deadliner kalibriert die Prognose automatisch mit jedem Pipeline-Lauf.

3.  **Nutzung des probabilistischen Dashboards:**
    Öffnen Sie das integrierte Dashboard (Port 8080 nach `deadliner serve`). Filtern Sie nach **Team**, **Sprint** oder **Task-Komplexität**. Nutzen Sie den „What-If“-Slider, um zu sehen, wie sich eine Ressourcenverschiebung auf die Deadline-Wahrscheinlichkeiten auswirkt. Exportieren Sie die Prognose als JSON für Ihr Reporting-Tool.

### Beste Praktiken

1.  **Agiles Puffer-Management mit DDF:** Nutzen Sie das dynamische Deadline-Floating nicht als Ausrede für Aufschieberitis. Definieren Sie einen **minimalen Puffer** (z.B. 10% der geschätzten Zeit) und lassen Sie Deadliner diesen nur bei nachgewiesenen Blockaden (z.B. CI-Fail) automatisch erhöhen. Bei manuellem Überschreiben wird eine Warnung im Log ausgegeben.
2.  **Probabilistische Kommunikation im Team:** Ersetzen Sie feste Deadlines in Ihrem Daily Standup durch die **Wahrscheinlichkeitswerte** aus dem Dashboard. Sagen Sie nicht „Wir sind bis Freitag fertig“, sondern „Wir haben eine 70% Wahrscheinlichkeit für Freitag 16:00 Uhr“. Dies reduziert Druck und fördert eine datengetriebene Diskussion über Risiken und Abhängigkeiten.
