# deadliner - DEA_029

## Übersicht

deadliner ist ein hochperformantes, CLI-gestütztes Fristen-Management-Tool, das speziell für Developer und DevOps-Teams entwickelt wurde, die keine GUI-Blähtheit, sondern maximale Effizienz und Automation benötigen. Mit Fokus auf **Deadline-Tracking, Task-Dependenzen und automatisierte Eskalation** ersetzt deadliner stupide Kalender-Erinnerungen durch intelligente, konfigurierbare Workflows.

## Kernfunktionen

### 1. **Dependency-basierte Deadlines mit kritischen Pfaden**
Statt einfacher Fristen berechnet deadliner automatisch den kritischen Pfad (CPM) über deine Tasks. Wenn ein Task sich verspätet, werden alle abhängigen Deadlines dynamisch neu berechnet – inklusive Benachrichtigung per Webhook oder Slack.

### 2. **Auto-Eskalation mit Time-Bomb-Strategie**
Definiere Schwellwerte (z. B. 2 Stunden vor Fristende). Unterschreitet ein Task die Restzeit, wird automatisch eine Eskalationskette gestartet: Zuerst der Task-Owner, dann der Team-Lead, schließlich der gesamte Kanal – inklusive Log-Export.

### 3. **Git-Integration & Commit-basierte Fristen**
Verknüpfe Tasks mit Git-Branches. Jeder Commit aktualisiert den Task-Status. Bei fehlenden Commits innerhalb eines Zeitfensters wird der Task als „in Gefahr“ markiert – perfekt für CI/CD-Pipelines und Sprint-Reviews.

## So verwenden Sie es

### Schritt 1: Task mit Dependencies definieren
```bash
deadliner add --name "API-Refactoring" --deadline "2025-06-15T18:00:00Z" \
  --depends-on "DB-Migration" --escalation-level 2 \
  --notify-webhook "https://hooks.slack.com/..."
```
Der Task wird sofort in die Dependency-Graph-Datenbank (SQLite oder PostgreSQL) geschrieben. Der kritische Pfad wird automatisch berechnet.

### Schritt 2: Automatische Überwachung starten
```bash
deadliner watch --interval 300 --alert-on-critical
```
Ein Hintergrundprozess (als systemd-Service oder Docker-Container) prüft alle 5 Minuten den Status. Bei kritischen Abweichungen wird die Eskalationskette getriggert – inklusive JSON-Log-Export für dein Monitoring-Dashboard.

### Schritt 3: Git-Branch-Integration aktivieren
```bash
deadliner git-hook --install
```
Ein Pre-Commit-Hook wird in dein Repository eingefügt. Bei jedem Commit wird der Task-Status aktualisiert. Bei fehlenden Commits innerhalb der konfigurierten Frist wird eine Warnung ausgegeben – direkt im Terminal und per Webhook.

## Beste Praktiken

### 1. **Deadlines als Code (DaC)**
Definiere Tasks nicht manuell, sondern in YAML-Dateien im Repository (`deadliner.yml`). So werden Fristen versioniert, reviewbar und automatisch beim CI-Run validiert. Das reduziert manuelle Fehler und erhöht die Transparenz im Team.

### 2. **Eskalation mit Bedacht konfigurieren**
Setze Eskalationsstufen nicht zu aggressiv. Eine gute Faustregel: Stufe 1 nach 30 Minuten, Stufe 2 nach 2 Stunden, Stufe 3 nach 4 Stunden. Verwende `--dry-run`, um die Auswirkungen vorab zu simulieren. Das verhindert unnötigen Alarmismus.
