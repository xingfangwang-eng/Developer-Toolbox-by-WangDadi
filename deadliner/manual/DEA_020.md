## Overview

Deadliner is not just another todo app. It's a **deadline-first task orchestration engine** built for developers, project leads, and indie hackers who need to ship on time—without micromanagement. This SEO-optimized guide covers the tool's core value proposition: deadline management, task deadline tracking, and time management for engineering teams.

## What is deadliner?

Deadliner is a lightweight, CLI-first time management tool that treats deadlines as first-class citizens. Unlike bloated project management suites, deadliner integrates directly into your dev workflow—tracking task deadlines, surfacing bottlenecks, and enforcing time-boxed execution without leaving your terminal.

## Core Features

### 1. Deadline Dependency Graph (DDG)
Deadliner automatically resolves task dependencies and visualizes critical paths. If Task A is blocked by Task B, the tool recalculates all downstream deadlines and alerts you before the domino effect hits. No more manual Gantt chart tweaking.

### 2. Time-Boxed Sprints with Auto-Pacing
Define sprint durations and per-task time budgets. Deadliner tracks your actual work velocity against estimates, then dynamically adjusts future deadlines. If you're consistently over-running, it flags the pattern and suggests re-scoping—not just crunching.

### 3. Git-Integrated Deadline Hooks
Deadliner hooks into your Git workflow. Commit messages can include deadline tags (e.g., `git commit -m "fix: auth bug #deadliner:2025-03-15"`). The tool then cross-references commit timestamps with task deadlines, generating a real-time "deadline drift" report. Miss a deadline? You'll see it in your next standup.

## How to Use

### Step 1: Initialize a Project with Deadline Constraints
Run `deadliner init --deadline 2025-04-01 --scope "backend API v2"` in your project root. This creates a `.deadliner.yml` config file. Define milestones as YAML arrays, each with a strict deadline and optional dependency list. Example:
```yaml
milestones:
  - name: "auth module"
    deadline: 2025-03-20
    depends_on: []
  - name: "payment integration"
    deadline: 2025-04-01
    depends_on: ["auth module"]
```
Deadliner will parse this and generate a critical path map.

### Step 2: Track Tasks via CLI or VS Code Extension
Use `deadliner add --task "Implement OAuth2" --estimate 4h --deadline 2025-03-18` to log a task. The tool automatically groups tasks by milestone and calculates slack time. For real-time tracking, install the VS Code extension: it shows a status bar with remaining time per task and highlights overdue items in red.

### Step 3: Generate a Deadline Drift Report
At the end of each sprint, run `deadliner report --drift`. This compares your planned vs. actual completion times. The output is a JSON array with per-task drift values (in hours) and a "risk score" for each milestone. Use this data to refine your estimation model. Example output:
```json
[
  {"task": "OAuth2", "planned": 4, "actual": 6, "drift": 2, "risk": "high"},
  {"task": "DB schema", "planned": 3, "actual": 3, "drift": 0, "risk": "low"}
]
```

## Best Practices

### 1. Use Deadliner as a Pre-Commit Hook
Integrate deadliner's drift check into your CI/CD pipeline. Before merging a PR, run `deadliner check --branch` to ensure no task in the current sprint is overdue. This prevents deadline creep from reaching production. For teams using GitHub Actions, a simple workflow can block merges if drift exceeds 10%.

### 2. Combine Deadliner with Timeboxing for High-Risk Tasks
For tasks with high uncertainty (e.g., "research new auth protocol"), set a hard timebox of 2 hours using `deadliner timebox --task "auth research" --max 2h`. Deadliner will auto-stop the timer and log a "timebox hit" event. This prevents perfectionism from blowing up your sprint. After the timebox, reassess and either close the task or escalate.
