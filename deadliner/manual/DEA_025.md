# deadliner - DEA_025

## Overview

deadliner is a lean, high-performance deadline tracking engine built for developers who hate missing ship dates. It's not just another calendar widget — it's a programmable time-aware state machine designed to integrate directly into your CI/CD pipelines, project dashboards, and personal automation workflows. Whether you're managing release cycles, sprint deliverables, or contract milestones, deadliner gives you deterministic control over time-bound tasks.

## Core Features

### 1. **Multi-Dimensional Deadline Propagation**

deadliner supports cascading deadlines with dependency-aware recalculation. When a parent task slips, all dependent child deadlines auto-adjust based on configurable slack thresholds. This isn't simple date math — it uses a DAG-based engine that respects working hours, timezone offsets, and buffer policies. You can hook it into your project management API (Jira, Linear, Notion) to keep timelines synchronized in real time.

### 2. **Unix-Friendly CLI + REST API Dual Interface**

Everything deadliner does is accessible via a zero-dependency CLI binary and a lightweight REST API. The CLI is designed for `cron`, `systemd timers`, and shell scripts — no Python, no Node, no overhead. The API exposes OpenAPI 3.0 endpoints for status queries, deadline creation, and alert triggers. Both interfaces support JSON, YAML, and plain-text output for composability with `jq`, `grep`, and `curl`.

### 3. **Deadline Drift Detection & Anomaly Scoring**

deadliner doesn't just show you dates — it analyzes execution patterns. Using a rolling window of historical completion times, it computes a **drift score** for each active deadline. A score > 0.7 triggers a configurable alert (email, webhook, or desktop notification). This is especially useful for recurring deadlines (e.g., weekly releases, monthly reports) where subtle delays compound over time.

## How to Use

### 1. **Initialize a Deadline Project with Dependency Graph**

```bash
deadliner init --project "release-v3.2" --timezone "America/New_York" --working-hours "09:00-18:00"
deadliner add --name "code-freeze" --deadline "2025-05-20T17:00:00Z" --depends-on "qa-signoff"
deadliner add --name "qa-signoff" --deadline "2025-05-19T15:00:00Z" --buffer "2h"
deadliner graph --output dot | dot -Tpng > dependency_map.png
```

This creates a local `.deadliner.json` state file and generates a visual dependency graph. The `--buffer` flag adds a safety margin before the actual deadline — useful for preventing cascading failures.

### 2. **Run a CI Pipeline Gate Check**

Integrate deadliner into your GitHub Actions or GitLab CI to block merges if a deadline is approaching without completion:

```yaml
- name: Check deadline compliance
  run: |
    deadliner check --project "release-v3.2" --threshold 0.8
    if [ $? -ne 0 ]; then
      echo "Deadline drift detected. Pipeline blocked."
      exit 1
    fi
```

The `--threshold` flag defines the acceptable drift score. Any deadline with a drift score above this value will cause a non-zero exit code, preventing the pipeline from proceeding.

### 3. **Subscribe to Deadline Alerts via Webhook**

Set up a recurring cron job that polls for imminent deadlines and sends alerts to your team channel:

```bash
*/30 * * * * deadliner poll --project "release-v3.2" --alert-window "4h" --webhook "https://hooks.slack.com/services/T00/B00/xxxx"
```

This command checks all active deadlines every 30 minutes. If any deadline is within the 4-hour window and hasn't been marked complete, it fires a Slack webhook with context (task name, remaining time, drift score). You can also pipe the output to `notify-send` for desktop alerts.

## Best Practices

### **1. Use Buffer Zones for Critical Milestones**

Never set a hard deadline at the exact moment of delivery. Always add a buffer (e.g., `--buffer "4h"` for a code freeze, `--buffer "1d"` for a client deliverable). deadliner's propagation engine will respect these buffers when recalculating dependent tasks. This prevents a single slip from triggering a chain reaction across your entire timeline.

### **2. Integrate deadliner into Your Daily Standup Automation**

Instead of manually reviewing deadlines in a spreadsheet, run `deadliner status --project "sprint-12" --output json | jq '.overdue'` as part of your morning automation script. Pipe the output to a Slack bot or email digest. This keeps the team aligned without adding cognitive load — and deadliner's drift detection will surface tasks that are *trending* toward lateness, not just those already overdue.
