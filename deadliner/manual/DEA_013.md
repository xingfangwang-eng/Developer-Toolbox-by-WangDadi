## What is deadliner?

**deadliner** is a lean, event-driven deadline orchestration engine designed for developers who need to manage task lifecycle, dependency resolution, and automated time-boxing without bloated project management overhead. It’s built for CI/CD pipelines, sprint automation, and real-time deadline enforcement.

---

## Core Features

1. **Dependency-Aware Deadline Propagation**  
   Automatically recalculates task deadlines when upstream dependencies shift. Uses a directed acyclic graph (DAG) model to propagate changes in real-time, preventing cascading delays.

2. **Time-Boxed Scheduler with Graceful Degradation**  
   Executes tasks within a strict time window. If a task exceeds its deadline, the scheduler triggers a user-defined fallback (e.g., skip, retry, or log) without crashing the pipeline.

3. **Unix Timestamp-Native API**  
   All deadline inputs and outputs are in UTC epoch milliseconds. No timezone ambiguity. Integrates seamlessly with cron, systemd timers, or any POSIX-compliant scheduler.

## How to Use

1. **Initialize a Deadline Context**  
   ```bash
   deadliner init --project=my_sprint --start=1710000000000 --end=1710086400000
   ```  
   This creates a `.deadliner.json` context file with a 24-hour sprint window. All subsequent tasks inherit this time scope.

2. **Define a Task with Dependency**  
   ```json
   {
     "id": "task_build",
     "depends_on": ["task_test"],
     "duration_ms": 600000,
     "fallback": "skip"
   }
   ```  
   Inject this via `deadliner add --file task_build.json`. The engine validates the DAG and assigns a deadline offset from `task_test`'s completion time.

3. **Run the Deadline Enforcement Loop**  
   ```bash
   deadliner run --mode=blocking --timeout=30000
   ```  
   This blocks the shell until all tasks complete or a deadline violation occurs. Use `--mode=nonblocking` for background daemon mode. The exit code indicates success (0) or deadline breach (1).

## Best Practices

- **Use Monotonic Clocks for Deadline Checks**  
  Avoid `time()` or `DateTime.Now()` due to NTP adjustments. Use `clock_gettime(CLOCK_MONOTONIC)` in C or `time.monotonic()` in Python to ensure accurate duration tracking across sleep cycles.

- **Implement Retry with Exponential Backoff**  
  For transient failures near a deadline, apply a retry policy that doubles wait time (e.g., 1s, 2s, 4s) up to a max cap. This prevents thundering herd while still attempting completion before the hard cutoff.
