# deadliner - DEA_010

## Overview

**deadliner** is a precision deadline orchestration engine built for engineers who refuse to let tasks slip through the cracks. Unlike bloated project management suites, deadliner focuses on one thing: **brutally accurate task deadline tracking** with zero overhead. It’s designed for developers, freelancers, and small teams who need a lightweight, CLI-first time management tool that integrates seamlessly into existing workflows.

## What is deadliner?

deadliner is a deterministic deadline management system that eliminates ambiguity from project timelines. It parses natural language deadlines, enforces hard cutoffs, and surfaces schedule conflicts before they become blockers. Think of it as a **time-aware linter** for your task list.

## Core Features

### 1. Natural Language Deadline Parser & Auto-Scheduler
Write deadlines like `"submit PR by next Friday 3pm PST"` or `"deploy v2.1 before EOD +2"`. deadliner auto-resolves relative times, timezones, and recurring patterns, then schedules tasks into your existing calendar or todo system via webhook.

### 2. Conflict Detection & Cascade Alerts
When two deadlines overlap or a dependency slips, deadliner fires **cascade alerts** — not just to you, but to downstream task owners. It highlights critical path items and suggests rebalancing with a single command.

### 3. Immutable Deadline Log with Git-Like History
Every deadline change is versioned. You can `git diff` your deadline timeline, roll back to a previous schedule, or export a forensic audit trail. Perfect for compliance-heavy environments or post-mortem analysis.

## How to Use

### 1. Initialize a Project & Set Your First Deadline
```bash
deadliner init --project "api-v2" --timezone "America/New_York"
deadliner add "Finish auth middleware" --due "2025-06-20 17:00" --priority critical
```
deadliner validates the date, checks for existing conflicts, and appends the task to your local `.deadliner` file. It also emits a JSON event to your configured webhook (e.g., Slack, Discord, or custom endpoint).

### 2. Visualize Your Deadline Graph
```bash
deadliner graph --format mermaid
```
This generates a Mermaid timeline diagram showing all tasks, dependencies, and slack time. You can pipe it directly into your documentation or CI pipeline. Use `--blockers` to highlight tasks that are currently blocked.

### 3. Trigger a Deadline Audit Before a Milestone
```bash
deadliner audit --milestone "v2.1-release" --dry-run
```
deadliner scans all tasks tagged with the milestone, checks for overdue items, recalculates risk scores, and outputs a JSON report. With `--dry-run`, nothing is changed — you get a preview of what will be flagged. Combine with `--auto-reschedule` to let deadliner propose a new timeline.

## Best Practices

### 1. Integrate deadliner into Your CI/CD Pipeline
Add a `deadliner audit` step before every deployment. If any high-priority task is overdue or a deadline has slipped beyond the buffer, fail the build. This prevents shipping code when the schedule is already broken.

### 2. Use Immutable Logs for Client-Facing Deadlines
For agencies or consultancies, enable the immutable log and share the SHA of the deadline state with clients. This creates an auditable, tamper-proof timeline that builds trust and eliminates "I thought we said Tuesday" disputes.
