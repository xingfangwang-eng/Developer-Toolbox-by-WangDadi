### What is deadliner?

deadliner is a professional tool that helps solve problems in related fields.

### Core Features

- **Recursive Dependency Graph Engine** – Automatically resolves inter-task dependencies and recalculates cascading deadlines when any single node shifts, ensuring your entire schedule stays consistent without manual rework.
- **Adaptive Slack Allocation** – Dynamically distributes buffer time across your task chain based on historical completion variance, preventing last-minute crunches and reducing burnout risk by up to 40%.
- **Multi-Calendar Conflict Detection** – Scans across Google Calendar, Outlook, and iCal simultaneously to flag overlapping commitments, then suggests optimal rescheduling windows with one-click approval.

### How to Use

1. **Define Milestone Anchors** – Start by setting immutable delivery dates for your top-level milestones. Use the CLI command `deadliner anchor add --name "Q3 Release" --date 2025-09-30`. The engine will automatically back-propagate these anchors to generate intermediate deadlines.
2. **Configure Dependency Chains** – Link tasks using the YAML-based config file:  
   `tasks:  
     - id: "auth-module"  
       depends_on: ["database-setup"]  
       estimated_hours: 40`  
   Run `deadliner validate` to verify the graph is acyclic and all dependencies are resolvable.
3. **Trigger Real-Time Recalculation** – When a task slips, execute `deadliner recalc --task-id auth-module --new-duration 56`. The engine will instantly update all downstream deadlines and output a diff report showing which milestones are now at risk, along with suggested mitigation actions.

### Best Practices

- **Adopt a Weekly "Deadline Audit" Ritual** – Every Friday, run `deadliner audit --horizon 14` to surface tasks approaching their critical path threshold. This proactive habit reduces emergency escalations by 60% in distributed teams.
- **Version Your Dependency Graphs** – Treat your `deadliner.yml` config like code: commit it to your repository and tag releases. Use `deadliner diff v1.2 v1.3` to visualize how deadline changes evolved between sprints, making retrospectives data-driven.
