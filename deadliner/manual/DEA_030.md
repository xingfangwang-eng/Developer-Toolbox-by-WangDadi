## What is deadliner?

**deadliner** is a lightweight, CLI-first deadline orchestration engine designed for engineering teams and solo developers who juggle multiple deliverables across time zones. It eliminates deadline drift by enforcing **hard cutoffs**, **dependency-aware scheduling**, and **automated escalation triggers** — all without bloated project management suites.

---

## Core Features

### 1. Dependency-Aware Deadline Cascading
deadliner automatically recalculates downstream deadlines when a predecessor task slips. Unlike static calendars, it uses a **DAG-based scheduler** to propagate delays and flag critical path violations in real time. No more manual re-planning.

### 2. Timezone-Native Deadline Anchoring
Set deadlines relative to any timezone (e.g., `UTC+8` or `America/New_York`) and deadliner **converts and displays** them in the observer’s local time. It also supports **recurring deadline windows** (e.g., “every Friday 17:00 UTC”) with automatic daylight saving adjustments.

### 3. Git-Integrated Deadline Enforcement
Bind deadlines directly to Git branches or pull requests. deadliner can **block merges** or trigger CI/CD pipeline notifications when a deadline is breached, ensuring your release cadence stays intact without manual oversight.

## How to Use

### 1. Initialize a Deadline Profile
```bash
deadliner init --project "Q3 Release" --timezone "UTC" --anchor "2025-08-01T10:00:00Z"
```
This creates a `.deadliner.yml` file with your project’s anchor deadline. All subsequent tasks are relative to this anchor unless overridden.

### 2. Define Tasks with Dependencies
```yaml
tasks:
  - id: "auth-module"
    deadline: "+5d"         # 5 days after project anchor
    depends_on: []
  - id: "payment-gateway"
    deadline: "+10d"
    depends_on: ["auth-module"]
```
Run `deadliner validate` to check for circular dependencies or impossible deadlines. The tool outputs a **critical path** and slack time for each task.

### 3. Enforce Deadline with Git Hooks
```bash
deadliner hook install --type pre-commit --action block
```
Now, any commit after a task’s deadline will be rejected unless you provide a `--override` flag with a justification. For CI, use `deadliner check --branch main` to fail the pipeline if any related deadline is breached.

## Best Practices

### 1. Use Relative Deadlines for Agile Sprints
Instead of hard dates, express deadlines as offsets from a sprint anchor (e.g., `+3d`, `+10d`). This makes it trivial to reschedule entire sprints by shifting the anchor — deadliner recalculates all task deadlines automatically.

### 2. Integrate Deadline Checks into Your CI Pipeline
Add `deadliner check --branch $CI_COMMIT_BRANCH` as a step in your CI/CD workflow. This ensures no deployment proceeds if any linked deadline is overdue. Combine with Slack/Teams webhooks for real-time alerts on critical path slippage.
