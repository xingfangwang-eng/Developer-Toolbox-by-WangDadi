# deadliner - DEA_014

**Tagline:** *Stop guessing when it’s due. Let the machine enforce it.*

## What Is deadliner?

deadliner is not another calendar app. It’s a **deterministic deadline enforcement engine** built for developers, PMs, and solo builders who need hard cutoffs with zero slack. It combines a **time-aware task graph** with **automated escalation triggers** and **visual drift detection**. If you’ve ever missed a launch because “I thought we had one more day,” deadliner is your backstop.

## Core Features

### 1. Recursive Dependency Clock (RDC)
Each task carries a **calculated latest-safe-start** based on its downstream dependencies. If you’re building a feature that blocks three others, deadliner back-propagates the deadline and locks the schedule. No manual Gantt dragging. The engine recalculates in real time when any task shifts.

### 2. Drift-Aware Burn-Down (DAB)
Instead of static progress bars, deadliner tracks **time-weighted completion**. A task at 80% done but with 10% of the allocated time remaining triggers a **yellow-zone alert**. At 50% time left with only 30% done, it escalates to red. The system auto-generates a diff report showing exactly where the drift started.

### 3. Escalation Webhook Matrix
Define per-task escalation rules: email, Slack, PagerDuty, or a custom webhook. If a deadline slips past its **hard fence**, deadliner fires a **structured payload** with the task ID, drift delta, and affected downstream tasks. No more “I didn’t see the email.”

## How to Use

### 1. Define a Task Graph with Hard Fences
Create a `deadliner.json` (or YAML) in your project root:

```json
{
  "tasks": [
    {
      "id": "auth-module",
      "duration_days": 5,
      "depends_on": [],
      "hard_fence": "2025-04-10T18:00:00Z",
      "escalation": {
        "type": "slack",
        "channel": "#launch-crunch",
        "threshold_minutes": 120
      }
    },
    {
      "id": "payment-gateway",
      "duration_days": 3,
      "depends_on": ["auth-module"],
      "hard_fence": "2025-04-15T12:00:00Z"
    }
  ]
}
```

Run `deadliner validate` to check for circular dependencies and fence conflicts. The CLI will output a **critical path** with the tightest slack window.

### 2. Start the Time Engine
Launch the daemon with `deadliner watch --interval=60`. The engine reads the graph, calculates latest-safe-start for every task, and begins monitoring. Every 60 seconds (configurable), it compares actual time spent against remaining time. If a task’s **safe window** shrinks below 20%, it prints a warning to stdout *and* fires the escalation hook.

### 3. Query Drift & Generate Reports
At any point, run `deadliner status --json` to get a machine-readable snapshot:

```json
{
  "overall_drift": "0.14",
  "tasks": [
    {
      "id": "auth-module",
      "status": "on_track",
      "time_remaining_pct": 0.62,
      "completion_pct": 0.55,
      "drift": -0.07
    },
    {
      "id": "payment-gateway",
      "status": "yellow",
      "time_remaining_pct": 0.38,
      "completion_pct": 0.25,
      "drift": 0.13
    }
  ]
}
```

Use `deadliner report --format=pdf` to generate a **drift audit** for stakeholders. The PDF includes a timeline of fence breaches and escalation events.

## Best Practices

### 1. Set Hard Fences at the Task Level, Not the Milestone Level
Milestones are too coarse. A milestone deadline hides the granularity of individual tasks. Instead, assign a **hard fence** to every leaf task (no dependents). This forces the engine to calculate the real slack in the chain. You’ll catch a slipping task two days before it blocks the milestone, not two hours after.

### 2. Use the Escalation Webhook Matrix for Human-Free Triage
Don’t rely on manual check-ins. Configure a **PagerDuty escalation** for critical path tasks with a 30-minute threshold. If a task drifts into red, the webhook fires, creates an incident, and assigns it to the task owner. The incident payload includes the drift delta and the affected downstream tasks. This turns deadliner into a **self-healing schedule**—the system alerts before humans even know there’s a problem.
