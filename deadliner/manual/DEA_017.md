# deadliner - DEA_017

## Overview

**deadliner** is a precision-engineered time and task deadline tracking tool built for developers, freelancers, and SaaS teams who refuse to miss a cutoff. It eliminates the overhead of manual scheduling, surfaces real-time deadline pressure, and integrates directly into your CLI, IDE, or CI/CD pipeline.

Optimized for: **Deadline management, Task deadline tracking, Time management tool**

## What is deadliner?

deadliner is not just another calendar widget. It's a lightweight, CLI-first engine that ingests task metadata from your project files (JSON, YAML, or Markdown frontmatter), evaluates dependencies, and generates a live countdown + risk score for every deliverable. It's designed to be embedded into your dev workflow — no bloated UI, no cloud dependency.

---

## Core Features

### 1. Dependency-Aware Deadline Cascading
deadliner automatically detects task dependencies (e.g., `task_b` blocks `task_c`) and recalculates downstream deadlines when a parent task shifts. If `task_a` slips by 3 hours, deadliner instantly propagates the delay and flags the new critical path. No more manual chain updates.

### 2. Real-Time Risk Scoring via Contextual Heuristics
Each task is assigned a **Deadline Risk Score (0–100)** based on historical completion velocity, remaining buffer time, and task complexity parsed from your metadata. Scores update every 30 seconds. A red score (>75) triggers a desktop notification or Slack webhook — you act before the deadline breaks.

### 3. Git-Aware Snapshotting & Blame Tracking
deadliner hooks into your Git commit history to map task IDs to actual code changes. It can identify which commits contributed to a deadline miss, and even suggest rollback candidates when a feature branch is at risk. This turns deadline management into a forensic, data-driven process.

## How to Use

### Step 1: Define Tasks with Metadata in Your Project Root

Create a `.deadliner/tasks.yaml` file in your project root. Each task must include an `id`, `deadline` (ISO 8601), `dependencies` (array of IDs), and an optional `effort` field in hours.

```yaml
tasks:
  - id: "auth_rewrite"
    deadline: "2025-03-15T18:00:00Z"
    dependencies: []
    effort: 16
  - id: "api_rate_limit"
    deadline: "2025-03-18T12:00:00Z"
    dependencies: ["auth_rewrite"]
    effort: 8
```

Run `deadliner init` to validate the schema and register the project.

### Step 2: Launch the Live Monitoring Session

Execute `deadliner watch --interval 30 --alert slack` to start the real-time engine. It will:
- Parse all tasks and compute the risk score.
- Open a persistent terminal dashboard (or a tui) showing each task, its countdown, and the current risk level.
- Send alerts via Slack webhook when any task crosses the 75 risk threshold.

You can also run `deadliner status` for a one-shot snapshot without the live loop.

### Step 3: Integrate with Git Hooks for Automated Risk Lock

Add a pre-push hook that runs `deadliner check --git` before any push. This will:
- Compare the current branch's commit history against active tasks.
- If any task's deadline is within 24 hours and the branch contains unmerged changes, the push is blocked with a warning: `"Push blocked: task 'api_rate_limit' due in 4h — uncommitted changes detected."`
- Override with `git push --no-verify` only if you explicitly acknowledge the risk.

## Best Practices

### 1. Use Deadlines as Immutable Anchors, Not Negotiable Targets
Treat every deadline in deadliner as a hard cutoff. If a task slips, never adjust the deadline — instead, create a new task with a later date and mark the original as `missed`. This builds a transparent historical record of estimation accuracy and helps refine your velocity metrics over time.

### 2. Pair Risk Scores with Automated Escalation Escalation Chains
Don't rely on manual review. Configure deadliner's webhook to trigger a PagerDuty or Opsgenie alert when a risk score exceeds 85. This ensures that critical path blockers are surfaced to the right person within minutes, not hours. For lower-risk tasks, set up a daily digest via email or Slack.
