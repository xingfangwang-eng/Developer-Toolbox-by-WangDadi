# deadliner - DEA_001

## Overview

**deadliner** is a lightweight, high-precision task deadline tracking engine designed for developers and project leads who need to manage time-sensitive deliverables across distributed teams. It eliminates manual follow-ups by automating deadline monitoring, escalation, and calendar integration—keeping your pipeline on schedule without the overhead of heavyweight project management suites.

## What is deadliner?

deadliner is a CLI + API-based tool that ingests task metadata (due dates, assignees, dependencies) from your existing workflow (GitHub Issues, Jira, linear, or plain Markdown) and surfaces actionable alerts before deadlines slip. It's built for teams that want to **own their time**, not be owned by it.

## Hardcore Features

1. **Deadline Drift Detection Engine**  
   Automatically calculates schedule slippage by comparing actual task progress (via commit frequency, PR activity, or custom webhook payloads) against the planned timeline. Alerts are triggered when drift exceeds a configurable threshold—no more surprise overdue tasks.

2. **Multi-Channel Escalation Matrix**  
   Route deadline notifications through Slack, Discord, email, or a custom webhook chain. Define escalation tiers: first reminder at 48h out, critical alert at 6h, and a final "auto-assign to backup" trigger if no response within the window. Each channel supports conditional logic based on task priority.

3. **Timezone-Aware Deadline Sync**  
   Automatically converts all deadlines to each assignee's local timezone using IANA timezone database. Supports recurring deadlines (daily standups, sprint reviews, deployment windows) and adjusts for daylight saving transitions. Outputs a unified timeline view across global teams.

## How to Use

### 1. Initialize a Deadline Tracking Session

```bash
deadliner init --source github --repo your-org/your-repo --token $GH_TOKEN
```
This command scans all open issues with a `deadline:` label (customizable) and creates a local tracking database. It automatically parses ISO 8601 dates from the issue body or a dedicated `due` field in your project board. The session persists across reboots and syncs every 15 minutes.

### 2. Define Escalation Rules

Create a `.deadliner.yml` in your repo root:

```yaml
escalation:
  - threshold: 48h
    action: notify
    channel: slack
    message: "⏰ Task {{ task_id }} is due in 48 hours. Current progress: {{ progress }}%"
  - threshold: 6h
    action: notify
    channel: email
    cc: ["lead@team.com"]
    message: "🚨 Critical: {{ task_id }} overdue in 6h. Please confirm ETA."
  - threshold: 0h
    action: reassign
    fallback: "backup-engineer"
```

Run `deadliner apply` to load the rules. The tool validates the YAML schema and tests each channel connection before activating.

### 3. Monitor & Audit

```bash
deadliner dashboard --view timeline --group assignee
```
Opens a terminal-based interactive dashboard (powered by `bubbletea`) showing all active deadlines, drift percentages, and escalation status. Use `--export json` for CI/CD pipelines or `--export ical` to sync with Google Calendar/Outlook. The audit log (`deadliner.log`) records every notification sent and escalation triggered for compliance.

## Best Practices

1. **Tag Early, Tag Often**  
   Attach a `deadline:` label to every issue **at creation**, even if it's a placeholder. This allows deadliner to start tracking dependencies and resource conflicts from day one. Retroactively adding deadlines loses historical drift data.

2. **Combine with Automated CI Gates**  
   Integrate deadliner's `--check` flag into your CI pipeline (e.g., GitHub Actions). If a PR's linked issues have deadlines within 24h and the PR isn't merged, fail the build with a message. This forces teams to address time-critical tasks before shipping new code.
