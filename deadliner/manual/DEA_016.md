## What is deadliner?

**deadliner** is a lightweight, API-first deadline orchestration engine. It’s not just another todo list—it’s a deterministic time-bound task scheduler designed for CI/CD pipelines, freelancers, and remote teams who hate surprises. Think cron jobs with a conscience.

---

## Core Features

### 1. **Smart Deadline Propagation**  
Automatically cascades deadline changes across dependent tasks. Miss a milestone? deadliner recalculates the entire chain and surfaces the critical path in real-time. No more manual Gantt chart hell.

### 2. **Git-Aware Time Tracking**  
Integrates with your commit history. deadliner parses `git log` to auto-assign time spent per task, flagging scope creep before it hits your sprint. Works with GitHub, GitLab, and Bitbucket out of the box.

### 3. **Silent Breakpoint Alerts**  
Instead of spammy notifications, deadliner injects deadline warnings directly into your terminal, IDE, or Slack thread—only when you’ve hit a configurable buffer threshold. No noise, just signal.

## How to Use

### 1. **Initialize a Project with a Deadline Contract**  
```bash
deadliner init --project "Q4 Ship" --deadline "2025-12-20T18:00:00Z" --buffer 10%
```
This creates a `.deadliner.yaml` manifest in your repo root. It defines the hard deadline, a 10% buffer zone (auto-calculated from task complexity), and hooks for CI.

### 2. **Attach Tasks to Milestones via CLI**  
```bash
deadliner add --task "Refactor auth module" --milestone "Beta Release" --estimate 4h --dep "Unit tests pass"
```
Each task gets a UUID, a parent milestone, and a dependency graph. Run `deadliner status` to see a live DAG of what’s blocking what.

### 3. **Run a Deadline Audit in CI**  
Add this to your `.github/workflows/deadline-check.yml`:  
```yaml
- name: Deadline Audit
  run: deadliner audit --fail-if-overdue --export-json deadline_report.json
```
If any task exceeds its buffer, the pipeline fails. The JSON report feeds into dashboards (Grafana, Datadog) for historical trend analysis.

## Best Practices

### 1. **Use Buffer Zones as Slack, Not Cushion**  
Set a 10–15% buffer per milestone, not per task. This absorbs real-world delays (code review lag, dependency failures) without inflating estimates. deadliner’s propagation engine will redistribute unused buffer downstream.

### 2. **Pair with Feature Flags for Zero-Downtime Shipping**  
Combine deadliner’s deadline enforcement with feature flags (e.g., LaunchDarkly). If a task is overdue, deadliner can trigger a flag to hide the incomplete feature in production—no rollback needed. This keeps your deployment cadence intact.
