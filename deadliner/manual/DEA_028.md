# deadliner - DEA_028

## Overview

deadliner is a lightweight, high-precision deadline orchestration engine built for developers who need to manage task timelines, track delivery windows, and enforce time-based workflows without bloated project management overhead. It’s not another calendar app—it’s a programmable time-bound context manager designed for CI/CD pipelines, sprint tracking, and personal productivity automation.

## What is deadliner?

deadliner is a CLI + API tool that ingests task definitions with absolute or relative deadlines, evaluates them against system time, and outputs actionable states (pending, due, overdue, or grace). It supports recurring deadlines, dependency-based scheduling, and timezone-aware comparisons. Think `cron` meets `deadline` with JSON I/O.

## Core Features

1. **Deadline Dependency Graph**  
   Tasks can declare upstream blockers. deadliner resolves the DAG and only marks a task as “active” once all dependencies are cleared. No more manually checking “is X done before Y starts.”

2. **Grace Window & Escalation Engine**  
   Each deadline can have a configurable grace period (e.g., +2 hours) and an escalation action: webhook, stdout alert, or a custom script. This turns deadliner into a real-time responsibility trigger.

3. **Timezone-Aware Relative Offsets**  
   Define deadlines as `+3d` or `next Friday 17:00 UTC` and deadliner auto-resolves based on the system’s TZ or an explicit `--tz` flag. No more off-by-one errors across timezones.

## How to Use

### 1. Define a Task with Dependencies

Create a JSON file `tasks.json`:

```json
[
  {
    "id": "api-docs",
    "deadline": "+2d",
    "grace": "1h",
    "on_overdue": "webhook:https://hooks.slack.com/..."
  },
  {
    "id": "code-review",
    "depends_on": ["api-docs"],
    "deadline": "+4d"
  }
]
```

Run:

```bash
deadliner evaluate --file tasks.json --tz America/New_York
```

deadliner outputs a table with status, remaining time, and escalation triggers.

### 2. Use as a CI Gate

In your GitHub Actions workflow:

```yaml
- name: Check deadline compliance
  run: |
    deadliner evaluate --file .deadliner/tasks.json --fail-on-overdue
```

If any task is overdue, the pipeline fails. This enforces timeboxing without manual tracking.

### 3. Monitor in Real Time

Run deadliner in watch mode:

```bash
deadliner watch --file tasks.json --interval 30s
```

It refreshes every 30 seconds, printing updated statuses. Combine with `--notify` to push alerts to your terminal or a webhook when a deadline is within 10% of its window.

## Best Practices

- **Use relative offsets for recurring tasks**  
  Instead of hardcoding dates, use `+1w` or `+3d 09:00`. This keeps your task definitions portable across environments and timezones, and deadliner recalculates accurately on each run.

- **Integrate deadliner as a pre-commit hook**  
  Run `deadliner evaluate --file .deadliner/tasks.json` before every commit. This ensures no overdue tasks slip into your repo, especially useful in teams where deadline accountability is shared across timezones.
