### What is deadliner?

deadliner is a precision-engineered, developer-first deadline tracking and time management tool designed to eliminate missed deadlines and reduce cognitive overhead in complex project environments. It integrates seamlessly into existing workflows, providing real-time visibility into task dependencies, buffer times, and critical path analysis without adding friction. Unlike generic calendar apps, deadliner treats every deadline as a dynamic constraint that can be monitored, adjusted, and optimized through code-driven configuration.

### Core Features

- **Dependency-Aware Deadline Propagation**: Automatically recalculates all dependent task deadlines when a single milestone shifts. Uses a directed acyclic graph (DAG) model to propagate changes upstream and downstream, flagging cascading risks before they become blockers. Supports both hard and soft dependency types with configurable slack windows.

- **Buffer-Intelligent Time Boxing**: Implements a probabilistic buffer allocation engine based on historical task completion variance. Instead of static padding, deadliner dynamically assigns and reclaims buffer time across tasks using Monte Carlo simulation, ensuring that critical path items have statistically adequate protection while non-critical tasks stay lean.

- **CLI-First with Git Integration Hook**: Operates as a lightweight CLI tool that hooks directly into your Git workflow. Automatically reads commit timestamps, branch metadata, and merge events to infer progress and adjust deadline estimates. No manual entry required—deadliner updates its internal timeline based on your actual development cadence.

### How to Use

1. **Initialize a Project Context with Dependency Mapping**  
   Run `deadliner init --project-dir ./my-app --deadline 2025-06-30` to create a `.deadliner.yml` configuration file. Use the `--deps` flag to define task dependencies in YAML format, e.g., `task_a: [task_b, task_c]`. deadliner will validate the DAG for cycles and output a topological sort with estimated completion windows. For existing repos, use `deadliner scan --git-log` to auto-generate task nodes from commit history.

2. **Set Dynamic Buffers via Historical Variance Analysis**  
   Execute `deadliner calibrate --history 90d` to analyze the last 90 days of task completion data. deadliner computes mean and standard deviation per task type, then applies a configurable confidence interval (default 95%) to set buffer sizes. Override manually with `deadliner set-buffer --task task_a --p50 2d --p95 5d`. The system will then display a risk heatmap showing which tasks are most likely to slip.

3. **Monitor Real-Time Deadline Drift with Alerts**  
   Run `deadliner watch --interval 30m` to start a background daemon that checks progress against the planned timeline. deadliner compares actual Git activity (commits, PR merges) against expected velocity. When deviation exceeds a threshold (default 15%), it triggers a console notification and writes a JSON report to `./.deadliner/alerts.json`. Use `deadliner status --json` for a machine-readable snapshot suitable for CI/CD pipelines.

### Best Practices

- **Adopt a Weekly Recalibration Cadence**: Run `deadliner calibrate` every Monday to adjust buffers based on the previous week's actuals. This prevents buffer bloat from stale historical data and keeps your deadline estimates honest. In fast-moving sprints, consider a daily `deadliner status --diff` to catch drift early.

- **Integrate deadliner into Your CI/CD Pipeline**: Add a `deadliner check --fail-on-risk` step to your pre-merge checks. This forces teams to address deadline overruns before merging, turning deadline management into a first-class code review criterion. Pair it with a Slack webhook for real-time team visibility.
