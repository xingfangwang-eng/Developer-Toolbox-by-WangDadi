## Contenido Detallado

### ¿Qué es deadliner?

deadliner no es solo otro calendario digital. Es un **motor de priorización contextual** que analiza tu carga de trabajo, identifica cuellos de botella y te alerta proactivamente antes de que un deadline se convierta en una crisis. Diseñada para desarrolladores que odian los recordatorios genéricos y necesitan granularidad a nivel de commit.

### Características Principales (Hardcore Features)

1.  **Algoritmo de Holgura Dinámica (Slack Forecasting)**
    deadliner calcula automáticamente el "slack" (tiempo de holgura) de cada tarea basándose en tu historial de productividad y dependencias entre issues. Si una tarea tiene solo un 15% de holgura respecto a su fecha límite, el sistema eleva su prioridad visualmente y te sugiere reasignar recursos antes de que sea tarde. No es un simple contador regresivo; es un análisis predictivo de riesgos.

2.  **Integración con Git Hooks y CI/CD**
    Conecta deadliner directamente a tu pipeline de desarrollo. Configura un webhook para que, al hacer `git push`, la herramienta actualice automáticamente el estado de la tarea vinculada. Si un PR no se mergea antes del deadline, deadliner puede bloquear el siguiente deploy o enviar una notificación al Tech Lead. Esto elimina el "olvido" humano y fuerza la rendición de cuentas en tiempo real.

3.  **Vista de "Time-to-Completion" (TTC)**
    A diferencia de un simple Gantt, deadliner te muestra una proyección estadística de cuándo *realmente* terminarás cada tarea, basada en tu velocidad actual (throughput) y la complejidad estimada en story points. La vista TTC te permite responder preguntas como: "Si empiezo esta tarea ahora, ¿llegaré al deadline del sprint?" con un 95% de confianza estadística.

### Cómo Usar deadliner (Guía Técnica)

1.  **Configurar el Enganche de Repositorio (Git Hook)**
    Instala el hook de deadliner en tu repositorio local o remoto. Ejecuta:
    ```bash
    deadliner init --hook pre-push
    ```
    Esto configurará un script que, cada vez que hagas `git push`, extraerá el ID de la tarea desde el mensaje del commit (ej: `git commit -m "fix: corregir bug #DEA-006"`) y actualizará su estado en deadliner automáticamente. Luego, vincula tu repositorio:
    ```bash
    deadliner repo link --url https://github.com/tu-org/tu-repo
    ```

2.  **Definir Plazos con Dependencias Complejas**
    Crea una tarea con dependencias cruzadas usando la CLI:
    ```bash
    deadliner task create --title "Implementar API de autenticación" \
      --deadline "2024-12-15T18:00:00Z" \
      --depends-on "DEA-003, DEA-004" \
      --priority critical \
      --estimated-hours 16
    ```
    deadliner calculará automáticamente la ruta crítica. Si `DEA-003` se retrasa, el deadline de tu tarea se recalculará y recibirás una alerta de "riesgo de bloqueo en cascada".

3.  **Ejecutar un Análisis de Rendimiento (Sprint Retrospective)**
    Al final de un sprint, genera un reporte de desviación de tiempo:
    ```bash
    deadliner report sprint --period "2024-11-01..2024-11-15" --format json
    ```
    Este comando devuelve un JSON con métricas como: `avg_slack_consumption`, `tasks_on_time`, `tasks_late`, y `top_blockers`. Úsalo para ajustar tus estimaciones futuras y mejorar la precisión de tu planificación.

### Mejores Prácticas (Industry Standards)

1.  **Establece "Zonas de Alarma" (Trigger Zones)**
    No esperes al día del deadline. Configura deadliner para que active notificaciones cuando una tarea entre en la "zona roja" (ej: menos del 20% de tiempo restante vs. esfuerzo estimado). Esto te permite hacer *rescue sprints* o renegociar plazos con el cliente antes de que sea un problema. Recomendación: usa la regla 80/20; el 80% del trabajo debe estar completo cuando quede el 20% del tiempo.

2.  **Integra con tu Herramienta de Ticketing (Jira/Linear)**
    deadliner es más potente cuando no está aislado. Usa el webhook bidireccional para sincronizar estados. Por ejemplo, cuando un issue pasa a "In Progress" en Jira, deadliner inicia el cronómetro de la tarea automáticamente. Cuando pasa a "Done", deadliner registra el tiempo real vs. estimado y actualiza tu modelo predictivo. Esto elimina la fricción de tener que actualizar dos sistemas.
