# deadliner - DEA_011

## Resumen

deadliner es una herramienta SaaS minimalista pero letal para el seguimiento de plazos críticos, diseñada específicamente para equipos de desarrollo que operan bajo presión. Con un enfoque en la gestión de deadlines, el rastreo de tareas con fecha límite y la optimización del tiempo, deadliner elimina el ruido y te deja solo con lo que importa: cumplir.

## Funcionalidades Clave

### 1. **Auto-Notificaciones con Umbral de Riesgo Dinámico**
deadliner no te avisa solo cuando el deadline está cerca. Calcula en tiempo real el riesgo de incumplimiento basado en tu historial de productividad, la complejidad estimada de la tarea y el tiempo restante. Si el riesgo supera el 70%, te envía una alerta prioritaria por Slack, Telegram o correo.

### 2. **Vista de Línea de Tiempo con Dependencias Reales**
No es un simple calendario. deadliner mapea automáticamente las dependencias entre tareas (bloqueantes, bloqueadas, paralelas) y te muestra el impacto visual de retrasar una sola tarea. Ideal para equipos que trabajan con sprints o entregas concatenadas.

### 3. **Modo “Deadline Cero” para Tareas Urgentes**
Activa este modo para tareas que deben entregarse “ahora”. deadliner bloquea cualquier distracción: elimina notificaciones no críticas, oculta tareas no prioritarias y fuerza un temporizador Pomodoro integrado. Perfecto para cuando el cliente está esperando.

## Cómo Usar deadliner (Pasos Profesionales)

### 1. **Configurar un Proyecto con Dependencias**
Desde el dashboard, crea un nuevo proyecto y agrega tareas. Usa la sintaxis `@dependencia` para vincular una tarea a otra. Ejemplo: `Desplegar servidor @build-finalizado`. deadliner calculará automáticamente la ventana de tiempo realista.

### 2. **Activar el Umbral de Riesgo Personalizado**
Ve a `Configuración > Alertas > Umbral de Riesgo`. Define un porcentaje (recomendado: 70%). Luego, conecta tu canal de notificaciones (Slack, Telegram, correo). deadliner empezará a monitorear y te alertará solo cuando el riesgo supere ese umbral.

### 3. **Ejecutar una Tarea en Modo “Deadline Cero”**
Selecciona una tarea urgente y haz clic en `Modo Deadline Cero`. deadliner ocultará todas las demás tareas, activará un temporizador Pomodoro de 25 minutos y bloqueará notificaciones externas. Al finalizar, recibirás un resumen de productividad.

## Mejores Prácticas para Equipos de Desarrollo

### 1. **Integra deadliner con tu CI/CD pipeline**
Conecta deadliner con GitHub Actions o GitLab CI. Cuando un build falle o un PR esté atrasado, deadliner puede crear automáticamente una tarea urgente y asignarla al responsable. Esto evita que los deadlines se escapen por errores técnicos.

### 2. **Usa el historial de deadlines para ajustar estimaciones**
deadliner guarda el tiempo real que tomó cada tarea. Revisa semanalmente el reporte de desviación (estimado vs real) para mejorar la planificación de sprints. Si una tarea tipo “bug crítico” siempre toma 2 días más de lo estimado, ajusta tus plazos futuros.
