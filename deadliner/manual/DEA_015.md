# deadliner - DEA_015

## Overview

deadliner is a lean, CLI-first time management engine built for developers who hate context switching. It doesn't just track deadlines—it enforces a deterministic workflow that surfaces bottlenecks before they blow up. Designed for CI/CD pipelines, sprint planning, and personal productivity, deadliner integrates directly into your existing toolchain with zero overhead.

## Core Features

### 1. **Deterministic Deadline Propagation (DDP)**
deadliner calculates cascading deadline dependencies using a topological sort over your task graph. When a parent task slips, every downstream deadline is automatically recalculated and flagged in your terminal or CI output. No manual recalibration.

### 2. **Unix-Pipeable `deadline` Command**
Chain deadliner with `grep`, `jq`, or `curl` for maximum composability. Output structured JSON or plaintext—feed it into Slack webhooks, calendar APIs, or your own dashboards. The tool is stateless by default, but supports optional SQLite-backed persistence for long-running projects.

### 3. **Timebox Enforcement with Hard Halt**
Set a hard stop per task. When the timer expires, deadliner can execute a user-defined hook (e.g., `kill -9` a process, push a Git tag, or send a webhook to a CI/CD pipeline). This is not a suggestion—it's a contract.

## How to Use

### 1. **Initialize a Deadline Profile**
```bash
deadliner init --project "sprint-42" --timezone "America/New_York"
```
This creates a `.deadliner.yml` file in your project root. Define tasks, dependencies, and hard deadlines using YAML. Example:
```yaml
tasks:
  - id: "auth-module"
    deadline: "2025-05-10T18:00:00Z"
    depends_on: []
    halt_action: "notify-slack"
  - id: "payment-gateway"
    deadline: "2025-05-12T12:00:00Z"
    depends_on: ["auth-module"]
    halt_action: "git-tag"
```

### 2. **Run a Deadline Audit**
```bash
deadliner audit --json | jq '.overdue[] | {id, overdue_by}'
```
This parses your profile, checks current system time, and outputs any tasks past their deadline. Combine with `cron` or GitHub Actions for automated daily reports. The JSON output is designed for machine consumption—feed it into your monitoring stack.

### 3. **Execute a Timeboxed Session**
```bash
deadliner box --task "auth-module" --duration 90m --halt "notify-slack"
```
Starts a 90-minute timer for the `auth-module` task. When time expires, deadliner executes the `halt_action` defined in your profile (or overridden here). The session is logged to `~/.deadliner/sessions.db` for post-mortem analysis.

## Best Practices

### **1. Integrate deadliner into CI/CD as a Gating Mechanism**
Add `deadliner audit --exit-code` to your pre-merge pipeline. If any task is overdue, the build fails. This shifts deadline awareness left, forcing teams to resolve scheduling conflicts before code hits production. Pair it with a Slack webhook for visibility.

### **2. Use Timeboxing for Spike Tasks**
When exploring unknowns (e.g., a new library or API), set a hard 45-minute timebox with `halt_action: "git-commit"`. This forces a decision point: either commit the spike as a working prototype or kill it. Prevents "just one more hour" rabbit holes.
