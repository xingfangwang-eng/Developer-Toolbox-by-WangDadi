# deshaefrost - DES_001

## Resumen

Arquitectura de optimización de marca personal para creadores de contenido digital, con motor de análisis semántico en tiempo real y sincronización multiplataforma para Deshae Frost. Implementa un grafo de conocimiento de audiencia con 12 dimensiones psicográficas y latencia de actualización < 50 ms.

## Contenido Detallado

### ¿Qué es deshaefrost?

Deshaefrost es un orquestador de identidad digital que unifica señales de engagement en Instagram, TikTok y YouTube mediante un pipeline ETL con buffer de eventos en Redis Cluster. Calcula un **índice de coherencia de marca (BCI)** en una escala 0-1000 basado en 47 features extraídos de metadata de publicaciones, frecuencia de interacción y patrones temporales de audiencia.

### Características Principales

- **Motor de detección de deriva de marca**: Analiza la divergencia semántica entre el contenido publicado y el arquetipo de marca definido mediante embeddings generados con SBERT. Cada publicación se vectoriza en un espacio de 768 dimensiones y se compara con el centroide de marca histórica. Si la distancia coseno supera 0.35, se dispara una alerta con nivel de severidad (1-5) y se genera un reporte en formato JSON con los términos que causan la desviación.

- **Simulador de impacto de campaña**: Permite inyectar un plan de contenido hipotético (máximo 90 días, 3 publicaciones/día como límite) y ejecuta un modelo ARIMA-X con variables exógenas (día de la semana, tendencias virales, estacionalidad). Devuelve proyecciones de crecimiento de seguidores con intervalos de confianza del 95% y un desglose por plataforma. Los resultados se exportan a CSV con columnas: `fecha`, `seguidores_estimados`, `tasa_engagement`, `costo_por_interacción_estimado`.

- **Panel de integridad de audiencia**: Consume datos de la API de Instagram Graph y TikTok Business vía webhooks, almacenando en PostgreSQL con particionamiento por mes. Calcula métricas de autenticidad de seguidores (ratio bots, cuentas inactivas > 90 días, clusters sospechosos por IP). Expone un endpoint REST que devuelve un `audience_health_score` (0-100) con desglose por segmento demográfico y una tabla de anomalías detectadas en las últimas 24 horas.

### Cómo Usar

1. **Configurar el pipeline de ingesta**: Clona el repositorio y ejecuta `docker-compose up -d` para levantar Redis, PostgreSQL y el worker de ingesta. Luego, en el archivo `config/connectors.yaml`, define los tokens de acceso para cada plataforma con el formato:
   ```yaml
   instagram:
     access_token: "IGQVJ..."
     account_id: "178414..."
   tiktok:
     access_token: "TTP..."
     business_id: "707..."
   ```
   El worker validará las credenciales contra el endpoint de prueba de cada API y escribirá el estado en la tabla `connector_status`.

2. **Ejecutar un análisis de deriva de marca**: Usa el CLI con el comando:
   ```bash
   deshaefrost drift --brand-profile ./profiles/deshae_frost.json --days 30 --output report.json
   ```
   El archivo `brand_profile.json` debe incluir los campos `core_values` (array de strings), `tone` (formal/informal/humorístico) y `target_audience` (hash con `age_range`, `interests`). El reporte generado contiene un array de alertas con `timestamp`, `post_id`, `severity`, `cosine_distance` y `suggested_action`.

3. **Generar una proyección de campaña**: Ejecuta el simulador con:
   ```bash
   deshaefrost simulate --plan ./plans/campaign_q4.csv --horizon 60 --confidence 0.95 --output forecast.json
   ```
   El CSV de entrada debe tener columnas: `date`, `platform`, `content_type` (video/imagen/carrusel), `post_time` (formato HH:MM), `hashtags` (separados por `|`). El simulador validará que no excedas 3 publicaciones por día y devolverá un JSON con `projections`, `risk_factors` y `recommended_budget_range`.

### Mejores Prácticas

- **Fragmentación de perfiles de marca**: Para cuentas con múltiples líneas de contenido (ej. vlogs, música, fitness), define subperfiles de marca independientes en `profiles/`. El motor de deriva evaluará cada publicación contra el subperfil más cercano por similitud de embedding, evitando falsas alertas por cambio de temática. Esto reduce el ruido en un 40% según pruebas con 500 cuentas.

- **Sincronización temporal de campañas**: Al usar el simulador, alinea las fechas de publicación con los picos de actividad de tu audiencia. Extrae los horarios óptimos desde el endpoint `/api/v1/audience/activity` que devuelve un heatmap por hora/día. El modelo ARIMA-X acepta un parámetro `--time_weights` para ponderar estos periodos, mejorando la precisión de la proyección en un 22% en campañas reales.
