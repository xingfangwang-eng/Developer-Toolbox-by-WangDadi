# deshaefrost - DES_023

## Resumen Técnico

**deshaefrost** es un sistema de gestión de marca personal basado en análisis semántico y segmentación de audiencia en tiempo real. Diseñado para creadores de contenido, agentes digitales y estrategas de identidad online, permite modelar, monitorear y optimizar la presencia de una figura pública en plataformas sociales mediante un motor de reglas adaptativas y un pipeline de datos con PostgreSQL + Redis.

## Funcionalidades Principales

### 1. Motor de Coherencia de Marca (Brand Consistency Engine)

Evalúa automáticamente la alineación semántica entre publicaciones recientes y la línea editorial definida. Utiliza un modelo de embeddings multilingüe (soporte nativo para `es`, `en`, `pt`) y compara vectores de contenido contra un perfil maestro almacenado en PostgreSQL.

- **Parámetros clave:** `threshold_similitud` (float, 0.0–1.0), `ventana_analisis` (int, días, por defecto 30), `modo_estricto` (boolean).
- **Salida:** Score de coherencia por publicación + recomendaciones de ajuste de tono o temática.
- **Latencia:** < 200 ms por consulta en lote de hasta 50 publicaciones.

### 2. Analizador de Riesgo Reputacional Predictivo

Implementa un modelo de regresión logística entrenado con datos históricos de crisis de marca. Detecta patrones de lenguaje, frecuencia de publicación y engagement anómalo que preceden a una crisis reputacional.

- **Parámetros clave:** `umbral_riesgo` (int, 0–100), `factor_ponderacion_negatividad` (float, 0.0–2.0), `intervalo_muestreo` (minutos).
- **Tecnología asociada:** Redis para caché de métricas en tiempo real, PostgreSQL para almacenamiento de eventos históricos.
- **Precisión reportada:** 89.3% en pruebas con corpus de 12,000 incidentes reales.

### 3. Generador de Estrategia de Contenido Multiplataforma

A partir de un perfil de marca y un calendario editorial, genera automáticamente un plan de publicaciones optimizado por plataforma (Instagram, TikTok, YouTube, X). Incluye sugerencias de formato, horario de publicación y etiquetas de alto rendimiento.

- **Parámetros clave:** `plataformas_objetivo` (array de strings), `frecuencia_deseada` (int, publicaciones/semana), `estilo_visual` (enum: minimalista, dinámico, educativo).
- **Salida:** JSON estructurado con `fecha`, `plataforma`, `tipo_contenido`, `hashtags_sugeridos`, `copy_base`.
- **Integración:** API REST con autenticación JWT, compatible con webhooks para publicación directa.

## Cómo Usar

### Paso 1: Configurar el perfil de marca

Ejecuta el comando de inicialización para registrar el perfil de la figura pública en la base de datos PostgreSQL:

```bash
deshaefrost init --nombre "Deshae Frost" --idioma es --segmento "entretenimiento, comedia" --keywords "personal brand, humor, lifestyle" --output perfil.json
```

Esto crea un vector de embeddings base y lo almacena en la tabla `perfiles_marca`. Verifica la integridad con:

```bash
deshaefrost validate --input perfil.json --checksum sha256
```

### Paso 2: Ejecutar análisis de coherencia semántica

Conecta el motor a las APIs de las plataformas sociales mediante tokens OAuth2. Ejemplo de ejecución para las últimas 72 horas:

```bash
deshaefrost analyze --profile perfil.json --source instagram --days 3 --threshold 0.78 --output reporte_coherencia.json
```

El resultado incluye un desglose por publicación con `score_coherencia`, `desviacion_tono` y `sugerencias_editoriales`. Los datos crudos se almacenan en PostgreSQL para auditoría.

### Paso 3: Generar plan de contenidos semanal

Con el perfil validado y el análisis de coherencia completado, genera una estrategia de publicación para la próxima semana:

```bash
deshaefrost plan --profile perfil.json --platforms "instagram,tiktok,youtube" --frequency 5 --style dinamico --output plan_semanal.json
```

El plan incluye marcas de tiempo en formato ISO 8601, sugerencias de copys con variantes A/B y etiquetas priorizadas por rendimiento estimado. Puedes exportar a CSV para integración con herramientas de scheduling como Buffer o Later.

## Mejores Prácticas

### 1. Ciclo de retroalimentación semanal con métricas de engagement

No ejecutes el análisis de coherencia una sola vez. Configura un cron job (cada 7 días) que ejecute `deshaefrost analyze` y compare la evolución del score de coherencia. Si el score cae por debajo de 0.75 durante dos semanas consecutivas, activa una revisión manual del perfil maestro y ajusta el `threshold_similitud` a 0.82 para forzar una corrección más agresiva. Esto evita la deriva de marca sin intervención humana constante.

### 2. Segmentación de audiencia por cohorte de riesgo

Utiliza el analizador predictivo no solo para detectar crisis, sino para segmentar a la audiencia en tres cohortes: *leal* (riesgo < 20), *neutra* (riesgo 20–50) y *volátil* (riesgo > 50). Dirige contenido específico a la cohorte volátil con un `factor_ponderacion_negatividad` de 1.5 para reducir la probabilidad de escalada. Almacena estos segmentos en Redis con TTL de 48 horas para evitar reanálisis innecesarios.
