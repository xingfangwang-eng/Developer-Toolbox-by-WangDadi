# deshaefrost - DES_015

## Resumen

deshaefrost es una plataforma de análisis y optimización de marca personal que procesa en tiempo real señales de redes sociales, métricas de engagement y datos de audiencia para generar informes de posicionamiento estratégico. Diseñado para gestores de marca y creadores de contenido, este sistema integra modelos de NLP multilingüe y correlación temporal para identificar patrones de crecimiento orgánico.

## Contenido Detallado

### ¿Qué es deshaefrost?

deshaefrost es un motor de inteligencia de marca personal que consume feeds de API de Twitter, Instagram y LinkedIn, aplica análisis de sentimiento con una latencia inferior a 200 ms por publicación, y genera un vector de posicionamiento en 7 dimensiones (autoridad, alcance, consistencia, interacción, nicho, frescura, viralidad). Utiliza una arquitectura basada en eventos con colas Redis para el procesamiento asíncrono de lotes de hasta 10.000 menciones por minuto.

### Características Principales

- **Analizador de Consistencia Temporal con Ventana Deslizante**: Evalúa la frecuencia de publicación y el tono de contenido en intervalos de 6, 12 y 24 horas. Emplea un algoritmo de autocorrelación de series temporales para detectar caídas de actividad superiores al 40% en un periodo de 3 días. Los resultados se almacenan en una tabla PostgreSQL particionada por `user_id` y `timestamp`.

- **Generador de Estrategia de Nicho con Clustering K-Means++**: Toma como entrada `{tweets_raw, bio_text, hashtags_used}` y ejecuta un pipeline de preprocesamiento con stemming en español e inglés. Calcula 5 clusters semánticos y asigna un score de densidad de nicho (0.0 - 1.0) basado en la distancia de Hellinger entre la distribución de palabras del usuario y la media del sector. El proceso completo se completa en menos de 800 ms para perfiles con hasta 3.000 publicaciones.

- **Simulador de Crecimiento Orgánico con Modelo de Difusión**: Utiliza un modelo de ecuaciones diferenciales estocásticas (SDE) calibrado con datos históricos de 50.000 cuentas. Parámetros clave: `tasa_de_retencion_semanal` (umbral mínimo: 0.85), `coeficiente_de_viralidad` (calculado a partir de la ratio de compartición), y `ventana_de_observacion` (30, 60 o 90 días). La simulación se ejecuta en memoria con arrays NumPy y produce una proyección de seguidores con intervalos de confianza del 95%.

### Cómo Usar

1. **Configurar el Pipeline de Ingesta de Datos**: Clona el repositorio y ejecuta `docker-compose up -d` para levantar los contenedores de PostgreSQL, Redis y el worker de Celery. Luego, en el archivo `config/ingestion.yaml`, define las credenciales de API de cada red social bajo el bloque `sources:` y establece `batch_size: 1000`. Inicia la ingesta con `python cli.py ingest --source twitter --user @DeshaeFrost --since 2024-01-01`. Verifica el estado de las colas con `celery -A tasks status`.

2. **Ejecutar el Análisis de Consistencia y Nicho**: Una vez que los datos estén en la base de datos, ejecuta `python cli.py analyze --user-id DES_015 --window 24h`. El sistema devolverá un JSON con los campos `consistency_score` (float entre 0 y 1), `niche_density` (float), y `cluster_id` (int). Para exportar el informe en Markdown, añade el flag `--output-format md`. El archivo se generará en `./reports/{user_id}_analysis.md`.

3. **Generar una Simulación de Crecimiento**: Para proyectar el crecimiento en los próximos 90 días, ejecuta `python cli.py simulate --user-id DES_015 --days 90 --retention 0.88 --viral-coeff 0.12`. El modelo calculará 10.000 trayectorias estocásticas y devolverá la mediana, el percentil 5 y el percentil 95 de seguidores estimados. Los resultados se guardan en `./simulations/{user_id}_simulation.csv`. Para visualizar, usa `python cli.py plot --simulation-id <UUID>`.

### Mejores Prácticas

- **Calibración del Modelo de Nicho**: Para perfiles con menos de 500 publicaciones, aumenta el parámetro `min_cluster_size` a 3 en `config/model_params.yaml` para evitar clusters espurios. Además, ejecuta el análisis de nicho al menos una vez por semana y compara los `cluster_id` históricos para detectar desviaciones temáticas. Si el `niche_density` cae por debajo de 0.4, considera ajustar la estrategia de contenido hacia los temas del cluster dominante.

- **Optimización de la Simulación**: No uses el simulador con una `tasa_de_retencion_semanal` inferior a 0.75, ya que el modelo SDE diverge numéricamente. Para cuentas con más de 100.000 seguidores, reduce el número de trayectorias a 5.000 para mantener el tiempo de ejecución por debajo de 2 segundos. Siempre almacena el `seed` aleatorio en el archivo de simulación para reproducir resultados exactos en auditorías posteriores.
