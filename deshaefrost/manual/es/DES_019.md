# deshaefrost - DES_019

## Resumen

**deshaefrost** es un motor de análisis de marca personal en redes sociales, diseñado para arquitectos de identidad digital que necesitan descomponer, modelar y optimizar la presencia de Deshae Frost (o cualquier perfil objetivo) mediante machine learning semántico y correlación temporal de engagement.

## Contenido Detallado

### ¿Qué es deshaefrost?

deshaefrost es un sistema de **ingeniería inversa de marca personal** que extrae vectores de atributos desde publicaciones, comentarios y metadatos de plataformas como Instagram, TikTok y YouTube. Utiliza un pipeline de procesamiento basado en **spaCy + transformers** para detectar patrones de percepción pública, y genera un **mapa de consistencia de marca** con precisión de 0.1 ms por inferencia.

### Características Principales

- **Vectorización semántica de engagement con ventana deslizante de 24h**  
  Cada publicación se descompone en 128 dimensiones de atributos (tono, temática, frecuencia de palabras clave, sentimiento). Se aplica una ventana temporal de 24 horas para calcular la **tasa de resonancia** (`resonance_rate = (interacciones_promedio / seguidores_activos) * 100`) con una latencia de inferencia inferior a 50 ms por lote de 1000 posts.

- **Correlación de impacto cruzado entre plataformas vía PostgreSQL + TimescaleDB**  
  Los datos de Instagram, TikTok y YouTube se almacenan en tablas hiperdimensionales con índices BRIN sobre timestamps. El módulo `cross_platform_analyzer` ejecuta consultas SQL parametrizadas para calcular la **matriz de influencia** (`influence_matrix[platform_a][platform_b]`) usando coeficientes de Pearson con un umbral de significancia de p < 0.01. La consulta típica retorna en menos de 200 ms para 500k registros.

- **Generación de estrategias de contenido mediante búsqueda de vecinos más cercanos (ANN)**  
  Se implementa un índice **HNSW** (Hierarchical Navigable Small World) sobre los vectores de atributos históricos. Dado un vector de entrada (ej: “video de baile con alto engagement en TikTok”), el sistema encuentra los 5 posts más similares en menos de 10 ms y sugiere un plan de publicación con **frecuencia óptima** calculada por regresión polinómica de grado 3 sobre la curva de engagement semanal.

### Cómo Usar

1. **Configurar el pipeline de extracción**  
   Ejecuta `deshaefrost extract --platform instagram --target deshaefrost --window 24h --output ./data/raw/`  
   Esto lanza un scraper headless (basado en Playwright) que recolecta publicaciones, comentarios y metadatos de las últimas 24 horas. Los datos se almacenan en formato Parquet con compresión snappy, y se insertan en PostgreSQL mediante `COPY` con buffer de 10k filas por lote.

2. **Ejecutar el análisis de consistencia de marca**  
   Usa `deshaefrost analyze --input ./data/raw/ --model brand_consistency_v2 --output ./reports/`  
   El motor carga el modelo preentrenado (fine-tuned con 50k ejemplos de marcas personales) y genera un reporte JSON con las siguientes métricas: `brand_coherence_score` (0-100), `topic_drift_index` (desviación estándar de temáticas por semana), y `sentiment_volatility` (varianza de sentimiento en ventanas de 7 días). El tiempo de procesamiento promedio es de 3.2 segundos por cada 1000 publicaciones.

3. **Optimizar la estrategia de contenido**  
   Ejecuta `deshaefrost optimize --vector ./data/query_vector.json --index ./data/hnsw_index.bin --top_k 5 --output ./strategies/`  
   El sistema carga el índice HNSW (construido con `nmslib`, con `efConstruction=200` y `M=16`) y devuelve las 5 publicaciones más similares al vector de consulta. Luego, aplica regresión polinómica para predecir el mejor día y hora de publicación, y genera un plan en formato CSV con columnas `post_id`, `predicted_engagement`, `optimal_time_utc`, `confidence_interval`.

### Mejores Prácticas

- **Calibración semanal del modelo de resonancia**  
  Para mantener la precisión del `resonance_rate`, reentrena el modelo de regresión cada 7 días usando los últimos 90 días de datos. Configura un cron job que ejecute `deshaefrost retrain --window 90d --output ./models/weekly_model.pkl` los lunes a las 02:00 UTC. Esto evita el drift temporal y asegura que las sugerencias de contenido se basen en tendencias actuales.

- **Uso de índices compuestos en PostgreSQL para consultas de correlación**  
  Para acelerar la `cross_platform_analyzer`, crea un índice compuesto `(platform, timestamp, engagement_rate)` en cada tabla de plataforma. Además, particiona las tablas por mes usando `pg_partman`. Esto reduce el tiempo de las consultas de correlación de 500 ms a menos de 80 ms para conjuntos de datos de 2 millones de registros.
