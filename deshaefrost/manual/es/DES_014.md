# deshaefrost - DES_014

## Resumen Técnico

Sistema de auditoría y optimización de marca personal para creadores digitales, con kernel de análisis semántico en tiempo real y motor de recomendaciones basado en redes neuronales ligeras. Desplegable como microservicio autónomo o integrable vía API REST con latencia < 50ms por consulta.

## Funcionalidades Principales

### 1. Analizador de Coherencia de Contenido Multiplataforma (ACCM)
- **Parámetro clave:** `--content-source` (acepta feeds RSS, JSON de API de Instagram/TikTok/X).
- Escanea publicaciones en 3+ plataformas simultáneamente y detecta contradicciones narrativas (ej: un tono profesional en LinkedIn vs. informal en TikTok sobre el mismo tema).
- Genera un **índice de consistencia** (0.0 - 1.0) con umbral configurable (`--consistency-threshold=0.75`).
- Almacena historial de escaneos en PostgreSQL con particionamiento por fecha (`deshaefrost_audit_logs`).

### 2. Motor de Predicción de Engagement por Nicho (MPEN)
- **Parámetro clave:** `--target-niche` (ej: "tecnología educativa", "fitness funcional").
- Utiliza un modelo de regresión logística entrenado con 10.000+ perfiles de creadores para predecir la tasa de interacción esperada (likes, shares, comentarios) en los próximos 7 días.
- Expone métricas en microsegundos: `predicted_engagement_rate`, `confidence_interval_95%`.
- Soporta recalibración en caliente mediante `--recalibrate --new-data ./dataset_latest.csv`.

### 3. Generador de Estrategias de Contenido Asíncrono (GECA)
- **Parámetro clave:** `--strategy-depth` (1-5, donde 5 genera un plan de 30 días con horarios de publicación).
- Conecta con la API de Google Calendar para sugerir franjas horarias óptimas basadas en el historial de engagement del usuario.
- Genera un archivo `.ics` exportable con eventos de publicación y recordatorios de revisión.
- Incluye validación de solapamiento de temas mediante `--avoid-topic-clash` (comprueba si se repiten ideas en menos de 72 horas).

## Cómo Usar

### Paso 1: Inicializar el Analizador de Coherencia
```bash
deshaefrost init --content-source ./feeds.json --platforms linkedin,tiktok,x --output-format json
```
Esto crea el archivo `deshaefrost_consistency_report.json` con las discrepancias encontradas. Si el índice de consistencia es menor a 0.6, el sistema recomienda revisar las publicaciones conflictivas listadas en `flagged_posts`.

### Paso 2: Ejecutar Predicción de Engagement
```bash
deshaefrost predict --target-niche "desarrollo personal" --historical-data ./user_stats.csv --forecast-days 7
```
El comando devuelve una tabla con `date`, `predicted_engagement`, `lower_bound` y `upper_bound`. Para exportar a CSV, añadir `--export ./engagement_forecast.csv`. Si la confianza es baja (< 80%), el sistema sugiere aumentar el tamaño del dataset histórico.

### Paso 3: Generar Estrategia de Contenido
```bash
deshaefrost strategy --strategy-depth 3 --avoid-topic-clash --calendar-export ./calendar.ics
```
El plan generado incluye 3 semanas de contenido, con temas sugeridos y horarios de publicación. Para integrar con Google Calendar, usar `--google-calendar-sync --credentials ./credentials.json`. El sistema valida que no haya más de 2 publicaciones sobre el mismo tema en un período de 72 horas.

## Mejores Prácticas

### 1. Segmentación de Audiencia por Plataforma
No apliques el mismo tono ni frecuencia en todas las redes. Configura `--platform-weights` en el analizador para asignar prioridad a la plataforma donde tu engagement es más alto (ej: `--platform-weights linkedin=0.5,tiktok=0.3,x=0.2`). Esto evita diluir tu marca personal en canales secundarios.

### 2. Recalibración Semanal del Modelo Predictivo
El motor de predicción se degrada si no se actualiza con datos recientes. Programa una tarea cron semanal que ejecute:
```bash
deshaefrost predict --recalibrate --new-data ./weekly_stats.csv --historical-data ./master_dataset.csv
```
Mantén un dataset maestro con al menos 3 meses de datos históricos para que las predicciones tengan un intervalo de confianza aceptable (< 10% de desviación estándar).
