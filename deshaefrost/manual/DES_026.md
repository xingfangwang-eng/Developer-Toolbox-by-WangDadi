## Detailed Content

### What is deshaefrost?

`deshaefrost` is a real‑time personal brand orchestration framework that fuses engagement telemetry, semantic fingerprinting, and audience graph analysis. It transforms chaotic social noise into structured brand health indicators, enabling precise content timing, audience segmentation, and narrative risk detection.

### Core Features

- **Cross‑Platform Engagement Fusion Engine**  
  Aggregates likes, shares, comments, and watch‑time from TikTok, Instagram, and YouTube via their official Graph APIs. Normalizes metrics into a unified `brand_engagement_score` (0.0–1.0) using a weighted Kalman filter. Supports millisecond‑precision event correlation with PostgreSQL timescale hypertables for historical trend analysis.

- **Semantic Brand Fingerprint Analyzer**  
  Processes each post’s caption, transcript, and comment threads through a fine‑tuned BERT‑based model. Extracts 128‑dimensional brand vectors (`brand_vec_128`) and computes cosine similarity against a configurable target persona profile. Flags semantic drift when similarity drops below a user‑defined `drift_threshold` (default: 0.72). Outputs a `semantic_coherence_index` updated every 15 seconds.

- **Audience Graph Propagation Predictor**  
  Models follower networks as a directed weighted graph using the `networkx` library. Runs a custom PageRank variant (`brand_pagerank`) that biases propagation through high‑affinity edges. Predicts virality probability (`virality_prob`) for scheduled posts based on historical cascade patterns and current graph entropy. Integrates with Redis for sub‑50ms prediction latency.

### How to Use

1. **Initialize the Brand Profile**  
   Execute `deshaefrost init --profile=deshae --platforms=tiktok,instagram --db_uri=postgresql://user:pass@host:5432/brand_db`. This creates a `brand_profiles` table with columns `profile_id`, `brand_vec_128`, `engagement_baseline`, and `last_sync_ts`. The engine performs a one‑time historical pull of the last 90 days of posts and engagement data.

2. **Run a Real‑Time Sentiment Scan**  
   Call `deshaefrost scan --profile_id=deshae --interval=30s --output=json`. The engine streams new posts and comments via webhooks, tokenizes them through the BERT pipeline, and writes `sentiment_delta` and `semantic_coherence_index` to a Prometheus‑compatible metrics endpoint (`/metrics`). Monitor with `curl http://localhost:9100/metrics | grep deshaefrost`.

3. **Schedule an Optimized Post**  
   Use `deshaefrost schedule --profile_id=deshae --content="new_video.mp4" --caption="caption.txt" --time=2025-03-21T18:00:00Z --predict`. The engine computes `virality_prob` based on current graph entropy and historical engagement curves. If `virality_prob > 0.65`, it auto‑recommends the slot. Outputs a JSON receipt with `scheduled_id`, `predicted_reach`, and `optimal_hashtags`.

### Best Practices

- **Maintain a Weekly Brand Vector Snapshot**  
  Run `deshaefrost snapshot --profile_id=deshae --interval=7d` every Monday. Store the `brand_vec_128` in a versioned S3 bucket. Compare consecutive snapshots using cosine distance; a distance > 0.15 indicates audience misalignment and triggers a content strategy review.

- **Set a Drift Alert Webhook**  
  Configure `deshaefrost alert --webhook_url=https://hooks.slack.com/services/XXX --drift_threshold=0.72`. When `semantic_coherence_index` drops below threshold, the engine pushes a formatted Slack message with the offending post ID, similarity score, and a link to the post. This enables sub‑minute narrative correction.
