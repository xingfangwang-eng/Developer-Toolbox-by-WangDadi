# deshaefrost - DES_019

## Overview

`deshaefrost` is a high-performance personal brand intelligence engine designed for creators, influencers, and digital strategists who need to audit, optimize, and scale their social media presence with surgical precision. It ingests cross-platform signals—engagement velocity, sentiment drift, and content decay curves—to generate actionable brand equity scores in real time.

## Core Features

### 1. Cross-Platform Brand Signal Aggregation Engine
Aggregates raw interaction data from Instagram, TikTok, YouTube, and Twitter via OAuth 2.0 APIs. Calculates a unified **Brand Resonance Index (BRI)** using a weighted formula: `BRI = (engagement_rate * 0.45) + (sentiment_score * 0.35) + (share_of_voice * 0.20)`. Supports configurable time windows (24h, 7d, 30d) with millisecond-level refresh latency. Data is normalized against a baseline of 10,000 peer accounts per niche.

### 2. Content Decay & Republishing Scheduler
Analyzes historical post performance to model decay curves using an exponential moving average (EMA) with a half-life parameter (default: 72 hours). Identifies optimal republishing windows based on audience overlap and platform fatigue thresholds. Outputs a time-stamped queue of posts to recycle, with auto-generated caption variants and thumbnail swaps. Integrates directly with Buffer and Later via webhook.

### 3. Competitor Sentiment Drift Detector
Tracks up to 50 competitor handles per project. Uses a fine-tuned RoBERTa model (trained on 1.2M social media comments) to classify sentiment into 7 categories: admiration, envy, criticism, indifference, humor, confusion, and spam. Flags drift events when the rolling 48-hour sentiment variance exceeds 2.3 standard deviations from the mean. Outputs a structured JSON alert with the offending post URL, sentiment delta, and recommended counter-strategy.

## How to Use

### Step 1: Initialize a Brand Profile
Create a new project via CLI or API:
```bash
deshaefrost init --handle @DeshaeFrost --platforms instagram,tiktok --timezone UTC --decay_half_life 48
```
This generates a local `brand_profile.json` containing your baseline metrics, connected API tokens, and a 14-day historical snapshot. The engine immediately begins polling platform endpoints at 5-minute intervals.

### Step 2: Run a Full Audit with Parameter Tuning
Execute a deep scan with custom thresholds:
```bash
deshaefrost audit --bri_weight_engagement 0.50 --bri_weight_sentiment 0.30 --sentiment_model roberta --competitor_watchlist "handle1,handle2" --output_format pdf
```
The audit produces a 40+ page report including: platform-by-platform BRI breakdown, content decay heatmaps, top 10 underperforming posts with republish recommendations, and a competitor sentiment drift timeline. Processing time scales linearly with data volume—expect ~2.3 seconds per 1,000 posts on a standard M2 Mac.

### Step 3: Deploy the Republishing Pipeline
Schedule automatic republishing with custom rules:
```bash
deshaefrost schedule --min_bri 65 --max_republishes 3 --cooldown_hours 96 --platform instagram --dry_run false
```
This activates a cron-based daemon that checks your content library every 6 hours. Posts with a current BRI below 65 are flagged for republishing, but only if they haven't been recycled more than 3 times and the last republish was at least 96 hours ago. The daemon writes to a PostgreSQL-backed queue; failed publishes are retried with exponential backoff (max 3 retries).

## Best Practices

### Practice 1: Segment Your Audience by Engagement Velocity
Instead of treating your entire follower base as a monolith, use `deshaefrost` to cluster followers by their engagement velocity (likes + comments per minute post-publish). High-velocity segments (top 5%) should receive exclusive, time-sensitive content; low-velocity segments (bottom 20%) should be targeted with repurposed, evergreen material. This strategy alone can lift overall BRI by 12–18% over 30 days.

### Practice 2: Set Sentiment Drift Alerts at 2.0 Sigma, Not 2.3
While the default drift threshold is 2.3 standard deviations, industry data shows that catching negative sentiment shifts earlier (at 2.0 sigma) reduces brand recovery time by 40%. Configure your project with `--drift_threshold 2.0` and pair it with a Slack webhook. The extra false positives are negligible compared to the cost of a viral backlash that goes undetected for 6+ hours.
