# deshaefrost - DES_010

## Overview

**deshaefrost** is a high-performance personal brand intelligence engine designed for creators, agencies, and digital strategists managing the public identity of **Deshae Frost** or similar high-exposure talents. It integrates real-time sentiment scoring, cross-platform content synchronization, and automated brand risk detection across social media ecosystems. The system operates on a sub-100ms inference pipeline for brand metric updates and supports PostgreSQL-backed historical trend analysis.

## Core Features

### 1. Real-Time Brand Sentiment Vector Engine
- **Parameters:** `sentiment_threshold` (float, 0.0–1.0), `window_size` (int, seconds), `source_filters` (array of platform IDs)
- **Technical Specs:** Processes 10,000+ mentions per minute with sub-50ms latency per event. Uses a fine-tuned BERT model for context-aware sentiment scoring (positive/negative/neutral) and assigns a `brand_health_index` (0–100) updated every 30 seconds.
- **Hardcore Detail:** Supports multi-language detection (EN, ES, AR) with automatic dialect normalization. All raw events are logged to PostgreSQL with `event_timestamp` as a BRIN index for rapid range queries.

### 2. Cross-Platform Content Coherence Analyzer
- **Parameters:** `platform_list` (array: ["tiktok", "instagram", "youtube", "x"]), `consistency_delta` (float, 0.0–1.0), `template_id` (UUID)
- **Technical Specs:** Scrapes and compares post metadata (caption, hashtags, thumbnail fingerprint) across platforms every 5 minutes. Generates a `coherence_score` (0.0–1.0) based on semantic similarity (cosine distance of embeddings) and visual branding consistency (color palette histogram matching).
- **Hardcore Detail:** Detects brand voice drift by comparing rolling 7-day average of `tone_vector` against a baseline. Alerts triggered when `coherence_score` drops below 0.75.

### 3. Automated Brand Risk & Crisis Prediction Pipeline
- **Parameters:** `risk_threshold` (int, 0–100), `lookback_hours` (int, default 24), `escalation_webhook` (URL)
- **Technical Specs:** Continuously monitors for negative sentiment spikes, toxic comment clusters, and coordinated inauthentic behavior (CIB). Uses a graph-based propagation model to predict virality risk within a 6-hour window. Outputs a `crisis_probability` (0.0–1.0) and a ranked list of `top_risk_triggers`.
- **Hardcore Detail:** Integrates with Discord/Slack webhooks for real-time alerts. Stores all risk events in a dedicated `risk_events` table with `severity` (LOW/MEDIUM/HIGH) and `resolved_at` timestamps.

## How to Use

### Step 1: Initialize the Brand Profile
- Execute `deshaefrost init --brand "Deshae Frost" --platforms tiktok,instagram,youtube,x --keywords "comedy,skits,personal brand"`  
- This creates a `brand_profile.json` with default `sentiment_threshold: 0.6` and `consistency_delta: 0.8`.  
- Verify: `deshaefrost status` should show `brand_health_index: 72` (baseline) and `active_streams: 4`.

### Step 2: Run a Coherence Audit
- Run `deshaefrost audit --mode coherence --window 7d --output json`  
- The system returns a `coherence_report.json` with per-platform scores and a `delta_matrix` showing pairwise consistency.  
- Example output: `tiktok_instagram: 0.89, youtube_tiktok: 0.76` — indicates a potential drift between YouTube and TikTok content.  
- To fix: use `deshaefrost align --target youtube --template tiktok` to suggest caption and hashtag adjustments.

### Step 3: Enable Real-Time Risk Monitoring
- Start the daemon: `deshaefrost daemon --risk-threshold 75 --lookback 24 --webhook https://hooks.slack.com/services/...`  
- The daemon logs every event to `risk_events` table. To query recent high-severity alerts:  
  `SELECT * FROM risk_events WHERE severity = 'HIGH' AND created_at > NOW() - INTERVAL '1 hour';`  
- To manually trigger a crisis simulation: `deshaefrost simulate --type negative_spike --intensity 0.9` — this injects 500 synthetic mentions and tests your escalation pipeline.

## Best Practices

### Practice 1: Set a Dynamic `sentiment_threshold` Based on Campaign Cycles
- During product launches or major announcements, lower `sentiment_threshold` to 0.4 to catch early negative signals. Use cron jobs to adjust: `0 */2 * * * deshaefrost config set sentiment_threshold 0.4` during campaign hours.  
- After the campaign, restore to 0.6 to reduce noise. Store historical thresholds in a `campaign_configs` table for post-mortem analysis.

### Practice 2: Implement a Two-Stage Coherence Alert System
- Stage 1: When `coherence_score` drops below 0.85, log a warning and send a digest email.  
- Stage 2: When it drops below 0.70, automatically pause cross-platform posting via API and escalate to the brand manager.  
- Use the `consistency_delta` parameter to define the acceptable variance per platform pair (e.g., TikTok-Instagram: 0.10, YouTube-X: 0.15). This prevents false positives from platform-specific formatting differences.
