# deshaefrost - DES_024

## Overview

**deshaefrost** is a high-performance personal brand intelligence engine engineered for real-time social media signal analysis and automated reputation orchestration. It ingests multi-platform engagement streams, applies NLP-driven sentiment decomposition, and executes deterministic content scheduling with sub‑second latency. Designed for creators, executives, and agencies who demand surgical precision over organic reach.

## Core Features

### 1. Cross-Platform Sentiment Vector Engine
- **Parameters**: `--platform-instance` (Twitter, Instagram, TikTok), `--sentiment-threshold` (0.0–1.0), `--time-window` (ms)
- **Technical Specs**: Processes up to 50,000 posts/min per instance; performs real‑time token‑level sentiment decomposition using a fine‑tuned BERT model; outputs a 768‑dimensional sentiment vector alongside a confidence score. All results are stored in a PostgreSQL time‑series partition (`sentiment_raw_YYYY_MM`) for sub‑second historical queries.

### 2. Automated Brand Narrative Generator
- **Parameters**: `--tone-profile` (JSON schema), `--content-cadence` (cron expression), `--variation-count` (int)
- **Technical Specs**: Accepts a tone profile defining lexical constraints (e.g., formality index, emoji density). Generates 3–5 unique content variants per scheduled slot using a GPT‑4o‑mini fine‑tune. Each variant is scored against a brand‑specific engagement predictor (RMSE < 0.12) before the highest‑scoring variant is auto‑published. All generation and scoring metrics are logged to an internal Prometheus endpoint.

### 3. Reputation Anomaly Detection & Alerting
- **Parameters**: `--anomaly-sensitivity` (0–100), `--alert-channel` (Slack webhook, Discord webhook, email), `--cooldown-period` (seconds)
- **Technical Specs**: Continuously monitors sentiment vector drift using an Isolation Forest model trained on the last 30 days of baseline data. When a deviation exceeds the sensitivity threshold (default: 85), an alert is dispatched within 200 ms. The alert payload includes the top‑3 contributing posts, a delta‑vector visualization, and a recommended response template.

## How to Use

1. **Initialize the Platform Connector**  
   Run `deshaefrost init --platform-instance twitter --api-key <YOUR_KEY> --api-secret <YOUR_SECRET>`.  
   This creates a persistent WebSocket connection to the Twitter v2 filtered stream, configures a local buffer (Redis Streams, max 10,000 entries), and registers a Prometheus counter for ingested tweets.

2. **Configure the Sentiment Baseline**  
   Execute `deshaefrost calibrate --time-window 604800000 --output /var/deshaefrost/baseline.json`.  
   This processes the last 7 days of ingested data, computes per‑hour sentiment vectors, and writes a JSON baseline file. The calibration also populates the PostgreSQL `sentiment_baseline` table with mean and standard deviation for each hour bucket.

3. **Deploy the Automated Narrative Pipeline**  
   Use `deshaefrost schedule --tone-profile ./profiles/executive.json --content-cadence "0 */2 * * *" --variation-count 3`.  
   This registers a cron job inside deshaefrost’s internal scheduler (backed by a PostgreSQL job queue). Every two hours, it generates 3 content variants, scores them against the engagement predictor, and publishes the top variant. All lifecycle events are emitted to a configurable Kafka topic (`deshaefrost.content.lifecycle`).

## Best Practices

- **Maintain a Rolling Baseline Window**  
  Re‑calibrate the sentiment baseline every 14 days to capture seasonal shifts in audience behavior. Use `deshaefrost calibrate --time-window 1209600000 --update-baseline` to merge new data without losing historical context. This prevents anomaly detectors from flagging organic trend changes as false positives.

- **Implement a Staged Alerting Policy**  
  Set `--anomaly-sensitivity` to 90 for high‑profile accounts and 75 for experimental ones. Use a cooldown period of 300 seconds to avoid alert fatigue. Route critical alerts (sensitivity > 95) to a PagerDuty integration, while lower‑severity alerts go to a Slack channel. Always include the `--alert-channel` parameter with a dedicated webhook per severity tier.
