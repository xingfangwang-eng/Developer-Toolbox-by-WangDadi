## Detailed Content

### What is deshaefrost?

deshaefrost is a brand signal processing framework that decouples raw social media activity from structured brand health indicators. It uses a custom NLP pipeline trained on engagement entropy and audience retention curves to quantify personal brand momentum. The system is built on a dual-layer architecture: a real-time ingestion layer (WebSocket + Kafka) and an analytical layer (PostgreSQL + Redis-backed vector store) for historical trend correlation.

### Core Features

- **Real-Time Brand Sentiment Vectorization**  
  Implements a multi-head attention model that processes 10,000+ posts per minute across Instagram, TikTok, and YouTube. Each post is decomposed into a 512-dimensional brand sentiment vector (BSV), capturing tone, audience reaction entropy, and cross-platform resonance. The system outputs a normalized BSV score (0.0–1.0) with millisecond-latency updates via a gRPC endpoint. Parameter `--bsv-threshold 0.75` triggers automatic rebalancing of content strategy.

- **Cross-Platform Engagement Decay Modeling**  
  Uses a time-decay kernel (half-life configurable via `--decay-delta 3600s`) to weight engagement metrics (likes, shares, saves, comment depth) by their recency and platform-specific virality multipliers. Outputs a decay-corrected engagement score (DCE) that prevents stale content from skewing brand health. Integrated with PostgreSQL for storing raw engagement logs and Redis for real-time DCE aggregation across 30-day rolling windows.

- **Autonomous Brand Narrative Gap Analysis**  
  Scans the difference between the current brand narrative vector (BNV) and the target narrative vector (TNV) defined in the project config (e.g., `deshaefrost.config.json`). It generates a gap map with specific action items: content topics, posting frequency adjustments, and audience segment targeting. The gap analyzer runs on a cron schedule (default: every 6 hours) and pushes alerts to a Slack webhook if the cosine similarity between BNV and TNV drops below `--similarity-threshold 0.85`.

### How to Use

1. **Initialize the Brand Vector Space**  
   Run `deshaefrost init --platforms instagram,tiktok --target-narrative "authentic, aspirational, community-driven" --bsv-dim 512`. This creates a PostgreSQL schema (`deshaefrost_brand_vectors`) and seeds the initial TNV. The system will begin polling the specified platforms via their respective APIs (OAuth 2.0 tokens required in `~/.deshaefrost/credentials.yaml`).

2. **Deploy the Real-Time Ingestion Pipeline**  
   Execute `deshaefrost ingest --source kafka://localhost:9092 --topic brand_events --batch-size 1000 --flush-interval 500ms`. This spins up a consumer group that reads raw social media events from Kafka, processes them through the BSV model, and writes results to both PostgreSQL (for historical analysis) and Redis (for real-time dashboards). Monitor throughput via the built-in Prometheus metrics endpoint at `http://localhost:9090/metrics`.

3. **Run a Brand Health Audit**  
   Use `deshaefrost audit --window 14d --export-format json --output ./audit_report.json`. The system computes the DCE for every post in the window, calculates the BNV, and compares it against the TNV. The report includes a gap map with recommended content types (e.g., "short-form video: +3 posts/week"), a platform-specific engagement decay curve, and a list of top-10 underperforming posts (by DCE score). For CI/CD integration, pass `--exit-on-gap 0.15` to fail the pipeline if the gap exceeds 15%.

### Best Practices

- **Set Dynamic Decay Half-Lives Per Platform**  
  TikTok content typically decays 3x faster than YouTube long-form. Configure `--decay-delta` per platform in the config file (e.g., `"tiktok": 1800`, `"youtube": 10800`). This prevents the DCE from underweighting high-impact, older content on slower platforms and overweighting ephemeral TikTok trends.

- **Use the Gap Analyzer as a Pre-Publishing Gate**  
  Before posting any content, run `deshaefrost audit --dry-run --input ./proposed_post.json`. The tool will simulate the post’s impact on the BNV and alert if it would push the gap beyond `--similarity-threshold`. This turns brand narrative drift from a reactive post-mortem into a proactive quality gate, reducing narrative fragmentation by up to 40% in controlled tests.
