# deshaefrost - DES_016

## Overview

`deshaefrost` is a high-performance personal brand intelligence engine designed for creators, influencers, and digital strategists managing the public identity of figures like Deshae Frost. It ingests cross-platform social signals, applies NLP-driven sentiment clustering, and outputs actionable brand positioning insights with sub‑50ms latency per query.

## Core Features

### 1. Cross‑Platform Identity Fingerprint Engine
Ingests raw JSON feeds from Instagram, TikTok, YouTube, and X (Twitter) via OAuth‑2.0 webhooks. Each post is tokenized and vectorized using a fine‑tuned RoBERTa model, then mapped to a **Brand Attribute Vector (BAV)** of 128 dimensions. The engine detects persona drift in real time by comparing rolling 7‑day BAV cosine similarity against a baseline snapshot. Drift exceeding **0.15** triggers an alert. All vectors are persisted in a PostgreSQL + pgvector hybrid store for sub‑10ms nearest‑neighbor lookups.

### 2. Sentiment‑Weighted Narrative Graph
Every mention, comment, and reply is parsed into a directed graph where nodes represent entities (e.g., “Deshae Frost”, “collab brand”) and edges carry a **sentiment weight** between –1.0 and +1.0. The graph is rebuilt every 15 minutes using a streaming Apache Kafka pipeline. The tool exposes a REST endpoint `/narrative/trends?window=7d` that returns the top 5 fastest‑growing positive and negative edges, enabling pre‑emptive reputation management.

### 3. Predictive Content Resonance Score
Each drafted post is scored before publishing using a multi‑head transformer that considers historical engagement patterns, current trending hashtags, and audience time‑zone segmentation. The model outputs a **Resonance Score (0–100)** along with three concrete recommendations: optimal posting hour (UTC), ideal caption length (characters), and top‑3 hashtags ranked by expected CTR. The inference pipeline runs on a single NVIDIA T4 GPU with a median inference time of **34 ms** per post.

## How to Use

1. **Initialize the Brand Profile**  
   Run `deshaefrost init --profile deshae --platforms instagram,tiktok,youtube --webhook-secret <your_secret>` to bootstrap the identity fingerprint engine. This creates a PostgreSQL schema `df_profiles` and starts the Kafka consumer group. Wait for the initial sync log:  
   `[INFO] Ingested 1,247 posts across 3 platforms in 4.3s. Baseline BAV computed.`

2. **Query Real‑Time Narrative Trends**  
   Use the CLI to inspect sentiment shifts:  
   `deshaefrost narrative trends --profile deshae --window 24h --format json`  
   The response includes a `drift_edges` array with `{source, target, delta_sentiment, velocity}`. Any edge with `velocity > 3.0` should be reviewed immediately via the `/narrative/alert` endpoint.

3. **Score a Draft Post**  
   Save your draft as a JSON file:  
   `{"body": "New collab dropping soon 🔥", "media_type": "carousel", "scheduled_time": "2025-06-15T18:00:00Z"}`  
   Then run:  
   `deshaefrost score --profile deshae --draft draft.json --output optimized`  
   The tool returns a resonance score of **87** and suggests: `post at 20:00 UTC, trim caption to 110 characters, use #exclusive #behindthescenes #sneakpeek`.

## Best Practices

- **Set a weekly BAV baseline refresh** – Schedule a cron job every Monday at 03:00 UTC to recompute the 7‑day rolling baseline. This prevents seasonal content shifts (e.g., holiday promos) from triggering false drift alerts. Use `deshaefrost baseline update --profile deshae --force`.

- **Monitor edge velocity on public holidays** – High‑velocity negative edges often spike during events like New Year’s Eve. Create a dedicated Grafana dashboard that queries the `/narrative/trends` endpoint with a 1‑hour window and triggers a PagerDuty alert if `velocity > 5.0`. This gives you a 15‑minute head start before the narrative goes viral.
