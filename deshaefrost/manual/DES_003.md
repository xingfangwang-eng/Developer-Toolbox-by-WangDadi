# deshaefrost - DES_003

## Overview

deshaefrost is a high-performance personal brand intelligence engine engineered for digital identity architects, social media strategists, and reputation analysts. It ingests cross-platform behavioral signals from Instagram, TikTok, and YouTube to compute a real-time **Brand Equity Score (BES)** with sub‑second latency, enabling data‑driven decisions for Deshae Frost‑tier talent management.

## Core Features

### 1. Multi‑Platform Sentiment & Engagement Decay Analysis

- **Parameter**: `--decay-window=720h` (30‑day sliding window, configurable in hours).
- **Metric**: `Engagement Decay Rate (EDR)` — quantifies the percentage drop in average interaction rate per post over the window, normalized against platform‑specific baselines.
- **Technical detail**: Uses a **Holt‑Winters triple exponential smoothing model** (α=0.3, β=0.1, γ=0.05) to forecast expected engagement and flag anomalies when observed values fall below the 95% prediction interval. Output is a JSON payload with `{edr, forecast_band_lower, forecast_band_upper, anomaly_timestamps}`.

### 2. Automated Content Gap & Opportunity Mapping

- **Parameter**: `--topic-cluster-depth=5` (number of hierarchical LDA topic nodes per platform).
- **Metric**: `Content Gap Score (CGS)` — a 0–100 scalar indicating how much of the top‑performing keywords in the niche are absent from the subject’s posting history.
- **Technical detail**: Connects to **PostgreSQL** via a connection pool (min 10, max 50) and runs a nightly batch job that executes `pg_trgm` similarity joins against a curated corpus of 50,000+ viral post embeddings. Results are written to a materialized view `content_gaps_mv` with a refresh interval of 6 hours. Output includes a ranked list of up to 20 recommended content themes with predicted reach (impressions) and estimated production effort (hours).

### 3. Real‑Time Brand Risk & Crisis Detection Pipeline

- **Parameter**: `--risk-threshold=0.75` (float, 0–1, triggers alert when sentiment volatility index exceeds threshold).
- **Metric**: `Sentiment Volatility Index (SVI)` — computed as the standard deviation of the 15‑minute moving average of sentiment scores (range -1 to +1) over the last 4 hours.
- **Technical detail**: Leverages a **Kafka** stream (topic `brand_sentiment_raw`, 3 partitions, replication factor 2) consuming webhook data from social APIs. A **Flink** job (parallelism 4, checkpointing interval 30 s) runs a sliding window of 256 events, applies a pre‑trained BERT‑based sentiment classifier (ONNX runtime, inference < 15 ms per event), and emits an alert to a dedicated Slack webhook if SVI > threshold for 3 consecutive windows. All alerts are logged to `alert_history` table in PostgreSQL with a TTL of 90 days.

## How to Use

### Step 1: Initialize the Brand Profile with Historical Data

```bash
deshaefrost init --profile=deshae_frost --platforms=instagram,tiktok,youtube \
  --backfill-days=90 --pg-conn-string="postgresql://user:pass@host:5432/deshaefrost"
```

This command:
- Creates a new profile `deshae_frost` in the `profiles` table.
- Launches three parallel backfill workers (one per platform) that fetch all posts, comments, and metadata from the last 90 days.
- Stores raw data in `raw_posts` and `raw_interactions` tables, partitioned by platform and ingestion timestamp.
- Outputs a summary of ingested records (`posts: 1,247`, `interactions: 89,312`) and the initial BES score.

### Step 2: Run a Full‑Cycle Brand Health Scan

```bash
deshaefrost scan --profile=deshae_frost --output-format=json \
  --decay-window=720h --topic-cluster-depth=5 --risk-threshold=0.75 \
  --export-path=./reports/scan_2025_04_10.json
```

The scan executes three parallel pipelines (sentiment decay, content gaps, risk detection). The JSON report includes:
- `bes`: current Brand Equity Score (float, 0–100).
- `edr`: Engagement Decay Rate with forecast bands.
- `content_gaps`: array of 20 recommended themes, each with `{theme, predicted_reach, effort_hours, gap_score}`.
- `risk_alerts`: array of any triggered alerts with `{timestamp, svi_value, sentiment_mean, suggested_action}`.

### Step 3: Schedule Automated Daily Scans with Alerting

```bash
crontab -e
# Add the following line to run at 06:00 UTC daily:
0 6 * * * /usr/local/bin/deshaefrost scan --profile=deshae_frost \
  --output-format=json --decay-window=720h --topic-cluster-depth=5 \
  --risk-threshold=0.75 --export-path=/var/deshaefrost/daily_reports/$(date +\%Y_\%m_\%d).json \
  --slack-webhook=https://hooks.slack.com/services/T00/B00/xxx
```

This ensures:
- A fresh scan runs every morning before peak social hours.
- If SVI > 0.75 for three consecutive windows, the pipeline sends an immediate Slack alert with a pre‑formatted message including the top 5 anomalous posts and a recommended crisis response template.

## Best Practices

### Practice 1: Align Decay Window with Platform Algorithm Cycles

Set `--decay-window` to **720 hours (30 days)** for Instagram and TikTok, where the algorithm heavily weights recent engagement. For YouTube, use **2160 hours (90 days)** because long‑tail discovery is common. Mismatched windows produce misleading EDR values — for example, a 30‑day window on YouTube may show an artificial decay spike when a high‑performing video from 45 days ago drops out of the calculation. Always validate the window against the platform’s official content freshness guidelines (Instagram: 30‑day lookback, YouTube: 90‑day lookback per Creator Studio analytics).

### Practice 2: Maintain a Versioned Content Gap Corpus

The `content_gaps_mv` materialized view should be rebuilt weekly using an updated corpus of viral embeddings. Store each corpus version in a separate schema (e.g., `corpus_v1`, `corpus_v2`) and configure `--topic-cluster-depth` to match the granularity of the latest corpus. If the corpus grows stale (older than 14 days), the CGS will recommend themes that are already saturated or irrelevant. Automate the corpus refresh via a cron job that runs `deshaefrost corpus-update --source=./data/viral_corpus_2025_04.csv` every Sunday at 02:00 UTC.
