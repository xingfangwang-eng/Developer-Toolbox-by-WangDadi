# deshaefrost - DES_008

## 概要

deshaefrostは、Deshae Frostのパーソナルブランド管理に特化したプロフェッショナルツールです。ソーシャルメディア上のブランドアセットを統合管理し、エンゲージメント最適化とクロスプラットフォーム一貫性を自動検証します。キーワード「Deshae Frost」「パーソナルブランド管理」「ソーシャルメディアパーソナルブランド」に対応したSEO最適化コンテンツ生成エンジンを内蔵しています。

## 詳細な内容

### コア機能

1. **クロスプラットフォームブランドアセット整合性検証エンジン**
   - パラメーター: `--platform-check-delay=300ms`（プラットフォーム間のポーリング間隔）、`--consistency-threshold=0.95`（95%以上の一致率で警告抑制）
   - Twitter、Instagram、YouTube、TikTokのプロフィール画像、ヘッダー、バイオ、リンク先URLを同時取得し、PostgreSQL上のブランドテンプレート（`brand_assets`テーブル）と比較。差分が0.5%を超えた場合、Slack Webhook経由で即時アラート送信。

2. **エンゲージメントスコアリング＆トレンド予測モジュール**
   - パラメーター: `--engagement-window=7d`（直近7日間のデータを分析）、`--forecast-method=prophet`（Facebook Prophetアルゴリズム使用）
   - 各プラットフォームのいいね、シェア、コメント、保存数を10分間隔で収集し、時系列DB（TimescaleDB）に格納。過去30日間のデータからエンゲージメントスコア（0-100）を算出し、次週の予測スコアを±3%の誤差範囲で出力。予測が閾値（デフォルト: スコア20以下）を下回る場合、コンテンツ改善提案を自動生成。

3. **パーソナルブランド一貫性スキャン＆レポート自動生成**
   - パラメーター: `--scan-depth=full`（全アカウント全投稿スキャン）、`--report-format=pdf`（出力形式）
   - 全プラットフォームの全投稿（上限10,000件）をテキストマイニングし、ブランドキーワード（例: “Deshae Frost”, “comedy”, “motivation”）の出現頻度、トーン（ポジティブ/ネガティブ/ニュートラル）、ハッシュタグの一貫性を分析。結果をA4 PDFレポートに集約し、改善点を優先度（高/中/低）でタグ付け。

### 使い方

1. **初期セットアップとプラットフォーム連携**
   - `deshaefrost init --platforms=twitter,instagram,youtube,tiktok` を実行。OAuth2.0認証用のリダイレクトURIを自動生成し、各プラットフォームのデベロッパーポータルで登録。認証後、`~/.deshaefrost/credentials.json`にトークンが暗号化保存される。
   - 環境変数 `DESHAEFROST_DB_URL=postgresql://user:pass@localhost:5432/deshaefrost` を設定し、`deshaefrost db migrate` でスキーマ（`brand_assets`, `engagement_logs`, `scan_reports`）を自動適用。

2. **ブランドテンプレートの設定と整合性検証の実行**
   - `deshaefrost template set --avatar=./profile.png --header=./header.png --bio="Deshae Frost | Comedian & Motivational Speaker" --website=https://deshaefrost.com` でテンプレートを登録。画像は自動でリサイズ（アバター: 400x400px、ヘッダー: 1500x500px）され、`brand_assets`テーブルにBase64で保存。
   - `deshaefrost check consistency --platform=all --notify-slack=https://hooks.slack.com/services/XXX` をcronで1時間ごとに実行。結果はJSON形式で標準出力に出力され、差分がある場合はSlackに`⚠️ プラットフォームXXでブランド不一致を検出`とメンション付きメッセージが送信される。

3. **エンゲージメント分析とレポート出力**
   - `deshaefrost analyze engagement --window=7d --forecast --export=./report_2024Q1.pdf` を実行。分析完了後、`engagement_logs`テーブルから直近7日間のデータを集計し、エンゲージメントスコアを棒グラフ、トレンド予測を折れ線グラフで可視化したPDFを生成。
   - レポート内には「改善提案」セクションが自動挿入され、例えば「TikTokでのエンゲージメントスコアが12と低いため、#DeshaeFrostのハッシュタグ使用頻度を現在の5%から15%に増やすことを推奨」といった具体的なアクションが記載される。

### ベストプラクティス

1. **ブランドテンプレートのバージョン管理と定期的な更新**
   - ブランドテンプレートはGitで管理し、変更履歴を追跡する。`deshaefrost template history` で過去10バージョンまで参照可能。新しいプロフィール画像やバイオを反映する際は、`deshaefrost template update --version=2` でバージョンを明示的に指定することで、誤った古いテンプレートによる整合性チェックを防止する。また、四半期ごとにテンプレートをレビューし、ブランドガイドラインの変更を反映することを推奨。

2. **エンゲージメントスコアの閾値設定とアラートの多段階化**
   - エンゲージメントスコアの閾値は、プラットフォームごとに個別設定する（例: YouTubeはスコア30以上、TikTokはスコア15以上）。`deshaefrost config set engagement.threshold.youtube=30` のように設定可能。閾値を下回った場合のアラートは、Slackだけでなく、メール（SMTP設定必須）やPagerDuty（APIキー必須）にも送信できる。さらに、スコアが10を下回った場合は、自動でコンテンツカレンダーに「緊急リライトタスク」を追加するワークフローを構築すると効果的。
