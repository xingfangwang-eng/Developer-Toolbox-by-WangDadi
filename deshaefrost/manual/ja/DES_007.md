# deshaefrost - DES_007

## 概要

deshaefrost は、Deshae Frost のパーソナルブランド管理を自動化・最適化するための統合プラットフォームです。SNS 上のブランド露出、エンゲージメント率、コンテンツ一貫性をリアルタイムで監視・分析し、戦略的なブランド構築を支援します。

## コア機能

### 1. ブランドエンゲージメント・スコアリングエンジン（BESE）

SNS プラットフォーム（Twitter、Instagram、TikTok）から投稿データを収集し、`engagement_rate`、`sentiment_score`、`brand_consistency_index` を ミリ秒単位で算出。`post_frequency` と `audience_overlap` をパラメータとして、ブランドの影響力を数値化します。

### 2. クロスプラットフォーム・コンテンツ同期パイプライン（CSP）

PostgreSQL をバックエンドに、各プラットフォームの投稿スケジュールを一元管理。`content_hash` による重複排除と `publish_timestamp` の自動調整機能を備え、同一コンテンツの重複投稿を防止しつつ、最適なタイミングで配信します。

### 3. パーソナルブランド・リスク検出モジュール（PBRD）

`keyword_blacklist` と `image_similarity_threshold` を設定可能なルールベースエンジン。投稿前にブランド毀損リスクをスキャンし、`risk_score` が 0.85 を超えた場合、自動で公開をブロック。監査ログは `audit_trail` テーブルに保存され、後日分析が可能です。

## 使い方

1. **初期設定とデータソースの登録**  
   `config.yml` に `platforms: [twitter, instagram, tiktok]` を定義し、各プラットフォームの API トークンを `credentials.enc` に暗号化して保存。`bootstrap.sh` を実行し、PostgreSQL のスキーマを自動生成します。

2. **ブランドプロファイルの作成**  
   `profile create --name "Deshae Frost" --keywords "comedy, acting, personal brand"` を実行。内部で `keyword_vector` が生成され、BESE が初期スコアを算出します。`--sentiment_baseline 0.7` で感情スコアの基準値を設定可能。

3. **継続的モニタリングとアラート設定**  
   `monitor start --interval 300` で 5 分間隔のスキャンを開始。`alert --risk_threshold 0.9` で高リスク投稿を即時通知。結果は `dashboard` コマンドでリアルタイム可視化されます。

## ベストプラクティス

- **コンテンツカレンダーの定期レビュー**  
  毎週日曜日に `csp sync --dry-run` を実行し、次週の投稿スケジュールの重複やタイミングのズレを事前にチェック。本番同期は `csp sync --apply` で実施します。

- **リスク検出の閾値調整**  
  ブランドの成長段階に応じて `risk_threshold` を動的に変更。初期は 0.7 に設定し、ブランド認知が安定したら 0.85 に引き上げることで、過剰なブロックを防ぎつつリスク管理を継続します。
