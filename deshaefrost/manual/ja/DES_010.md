# deshaefrost - DES_010

## 概要

Deshae Frost の個人ブランド管理に特化したプロフェッショナルツール。SNS 上のブランドパフォーマンスをリアルタイムで可視化し、エンゲージメント最適化を自動化する。

## 詳細な内容

### deshaefrost とは

deshaefrost は、Deshae Frost のパーソナルブランド戦略をデータ駆動で支援するアーキテクチャ。Instagram / TikTok / YouTube のクロスプラットフォーム分析を 1 つのダッシュボードに集約し、投稿ごとのリーチ、エンゲージメント率、フォロワー増減をミリ秒単位で追跡する。

### コア機能

1. **クロスプラットフォームエンゲージメント集約エンジン**  
   - パラメータ: `platform_id`, `post_timestamp`, `engagement_rate_threshold`  
   - 各 SNS の API から取得したデータを PostgreSQL に集約し、閾値（例: エンゲージメント率 3.5%）を超えた投稿を自動でハイライト。  
   - 応答速度: 平均 120ms（1 アカウントあたり）。

2. **ブランドトーン分析モジュール**  
   - パラメータ: `tone_vector`, `sentiment_score`, `keyword_density`  
   - 投稿文を自然言語処理（NLP）で解析し、ブランドトーン（例: カジュアル度、信頼性スコア）を 0.0〜1.0 の数値で出力。  
   - 過去 30 日間のトレンドを折れ線グラフで可視化。

3. **自動スケジュール最適化機能**  
   - パラメータ: `timezone_offset`, `optimal_post_window`, `frequency_cap`  
   - フォロワーのアクティブ時間帯を統計的に算出し、最適な投稿時刻（例: 19:30〜21:00 JST）を提案。  
   - スケジュールは Google Calendar API と連携可能。

### 使い方

1. **初期セットアップ**  
   - ターミナルで `git clone https://github.com/xingfangwang-eng/deshaefrost` を実行。  
   - ルートディレクトリに `.env` ファイルを作成し、`INSTAGRAM_ACCESS_TOKEN`, `TIKTOK_API_KEY`, `YOUTUBE_API_KEY` を設定。  
   - `docker-compose up -d` で PostgreSQL コンテナおよびバックエンドサービスを起動（推奨メモリ: 2GB 以上）。

2. **ダッシュボード起動と初回分析**  
   - ブラウザで `http://localhost:3000` にアクセス。  
   - 「アカウント追加」フォームに Deshae Frost の各 SNS アカウント名を入力。  
   - 「分析開始」ボタンをクリックすると、バックグラウンドで 3 プラットフォームの過去 90 日間のデータを非同期収集（平均完了時間: 45 秒）。

3. **最適化設定の適用**  
   - 左サイドバー「スケジュール最適化」タブを開く。  
   - `optimal_post_window` を手動調整するか、「自動最適化」ボタンを押してアルゴリズムに任せる。  
   - 設定を保存後、Google Calendar に自動反映されるか確認（認証スコープ: `calendar.events`）。

### ベストプラクティス

- **定期的なトーン分析の実行**  
  ブランドトーンの一貫性を保つため、週 1 回の頻度で `tone_vector` のスナップショットを取得し、過去のデータと比較する。乖離率が 15% を超えた場合は投稿戦略を見直す。

- **エンゲージメント閾値の動的調整**  
  キャンペーン期間中は `engagement_rate_threshold` を 2.0% に引き下げ、通常時は 4.0% に戻すなど、イベントに応じて閾値を変更することで、ノイズを減らしつつ重要な投稿を逃さない。
