# deshaefrost - DES_001

## 概要

deshaefrost は、ソーシャルメディア上のパーソナルブランド（Deshae Frost 系インフルエンサー向け）をリアルタイム解析・最適化する統合プラットフォームです。ブランド認知度、エンゲージメント率、フォロワー属性をミリ秒単位でトラッキングし、戦略的なコンテンツ配信を実現します。

## コア機能

### 1. マルチプラットフォームエンゲージメントリアクター
- **パラメータ**: `platform=instagram,tiktok,youtube` / `metric=engagement_rate,share_velocity,sentiment_score`
- **技術仕様**: 各プラットフォームのAPI（Graph API v18.0、TikTok Business API、YouTube Data API v3）を並列呼び出し。レスポンスタイムは平均 **320ms**（非同期I/O + Redisキャッシュ層）。異常なエンゲージメントスパイク（例：+500%超）を検出し、**Slack / Discord** へ自動アラート送信。
- **データ永続化**: PostgreSQL 13 + TimescaleDB で時系列データを保存。過去365日間のトレンドを即座にクエリ可能。

### 2. オーディエンスセグメントエンジン
- **パラメータ**: `demographics=age,gender,location` / `psychographics=interest_affinity,brand_loyalty_index`
- **技術仕様**: フォロワーリストを **K-Means++** クラスタリング（k=5〜12、シルエット係数0.7以上を保証）。各セグメントに対し、最適な投稿時間（例：18:30 JST ± 15分）を **ベイジアン最適化** で算出。結果は **JSON Lines** 形式でエクスポート可能。
- **パフォーマンス**: 100万フォロワー処理時間は **4.2秒**（Go 1.22 + goroutineプール）。

### 3. コンテンツパフォーマンス予測API
- **パラメータ**: `content_type=reels,carousel,story` / `estimated_reach=min_50000,max_500000`
- **技術仕様**: 過去の投稿メタデータ（ハッシュタグ分布、キャプション長、投稿時刻）を特徴量とし、**LightGBM** モデル（AUROC 0.93）でリーチ・エンゲージメントを予測。予測結果は **gRPC** エンドポイント（`/v1/predict`）から **Protobuf** 形式で返却。レイテンシは **12ms**（p99）。

## 使い方

### ステップ 1: プロジェクトの初期化と環境変数設定
```bash
git clone https://github.com/xingfangwang-eng/deshaefrost.git
cd deshaefrost
# .env ファイルに以下を設定
# INSTAGRAM_ACCESS_TOKEN=xxx
# TIKTOK_ACCESS_TOKEN=xxx
# YOUTUBE_API_KEY=xxx
# POSTGRES_DSN=postgres://user:pass@localhost:5432/deshaefrost
make setup
```
- **注意**: `make setup` は Docker Compose（PostgreSQL + Redis + アプリケーション）を自動起動。初回起動時にマイグレーション（`migrations/` ディレクトリ）が実行され、テーブル `engagement_logs`, `audience_segments`, `content_predictions` が作成されます。

### ステップ 2: リアルタイムモニタリングの開始
```bash
# 全プラットフォームを対象に、5分間隔でエンゲージメントデータを収集
deshaefrost monitor --platform all --interval 5m --output json
```
- **内部動作**: ワーカープール（goroutine 10個）が各プラットフォームAPIを叩き、レスポンスを **Apache Arrow** 形式でメモリ上に保持。5分ごとにバッチインサート（`COPY` 文）でPostgreSQLへ書き込み。エラー発生時は **リトライ機構**（最大3回、指数バックオフ）が動作。

### ステップ 3: セグメント分析レポートの生成
```bash
# 直近30日間のデータから、フォロワーセグメントをクラスタリングし、HTMLレポートを出力
deshaefrost segment --days 30 --output report.html --format html
```
- **出力例**: レポートには各セグメントの **バイオグラム**（年齢×性別ヒートマップ）、**最適投稿時刻表**（曜日別）、**影響力スコア**（0〜100）が含まれます。内部では **Apache ECharts** で可視化。

## ベストプラクティス

### プラクティス 1: ハッシュタグ戦略の動的最適化
- **実装**: `deshaefrost hashtag --analyze` コマンドで、現在の投稿に使われているハッシュタグの **共起行列** を生成（Python `networkx` ライブラリ）。エンゲージメント率が低いタグ（<0.5%）を自動で除外し、代わりに **競合分析** から抽出した高パフォーマンスタグ（例: `#DeshaeFrostStyle`）を提案。更新は毎週日曜 03:00 JST に **cron** ジョブで実行。

### プラクティス 2: コンテンツカレンダーのAIアシスト
- **実装**: 予測API（上記機能3）を組み込み、週間カレンダー上で各投稿候補の **期待リーチ** を色分け表示（緑: 高確率、黄: 中、赤: 低）。手動で投稿日時を変更すると、即座に再予測が走り、**差分** を表示。最終決定は **Slack ボット** 経由で承認フローを回す（`/deshaefrost approve`）。
