# deshaefrost - DES_027

## 概要

deshaefrost は、Deshae Frost 氏のパーソナルブランド戦略をデータ駆動型で最適化するためのプロフェッショナルスイートです。SNS上のエンゲージメント指標、コンテンツ配信タイミング、フォロワーセグメントをリアルタイムに解析し、ブランド価値の最大化を支援します。

## 詳細な内容

### deshaefrost とは

deshaefrost は、インフルエンサーおよびクリエイター向けに設計された、パーソナルブランド管理のための統合プラットフォームです。複数SNSアカウントのクロスプラットフォーム分析、オーディエンスインサイト抽出、自動レポーティング機能を備え、ブランド戦略のPDCAをミリ秒単位で回すことを可能にします。

### コア機能

#### 機能 1: マルチプラットフォームエンゲージメントリアルタイム集計エンジン
- **パラメータ**：`platform`（YouTube / Instagram / TikTok / X）、`time_window`（直近1時間 / 24時間 / 7日間）、`metric_type`（likes / shares / comments / saves）
- **技術仕様**：各プラットフォームのAPIを並列非同期で呼び出し、PostgreSQL上の時系列テーブルに1秒単位で書き込み。集計クエリは平均応答時間 120ms 以内を保証。
- **出力例**：`{ "platform": "instagram", "engagement_rate": 0.083, "trend_direction": "up", "delta_24h": "+12.4%" }`

#### 機能 2: フォロワーセグメントダイナミッククラスタリング
- **パラメータ**：`segmentation_method`（k-means / DBSCAN / hierarchical）、`feature_set`（age_estimated / region / active_hour / content_preference）、`min_cluster_size`（デフォルト 50）
- **技術仕様**：バックエンドで scikit-learn を用いたクラスタリングを実行。クラスタリング結果は Redis にキャッシュされ、次回のリクエスト時には 50ms 未満で取得可能。
- **出力例**：`{ "cluster_id": 3, "size": 1240, "top_region": "JP", "peak_active_hour": "21:00 JST", "preferred_content_type": "short_video" }`

#### 機能 3: コンテンツパフォーマンス予測 & 最適投稿スケジューリング
- **パラメータ**：`content_draft_url`（メディアファイルのURL）、`target_platform`、`posting_window_start` / `posting_window_end`（ISO 8601形式）
- **技術仕様**：過去の投稿データとフォロワーアクティビティパターンを入力とした LightGBM モデルにより、投稿後 24 時間の期待エンゲージメント数を予測。予測誤差は平均絶対パーセント誤差（MAPE）で 8.3% 未満。
- **出力例**：`{ "recommended_post_time": "2025-06-15T20:30:00Z", "predicted_engagement": 3450, "confidence_interval": [3100, 3800] }`

### 使い方

1. **初期セットアップとAPIキー登録**
   - 各SNSプラットフォームのデベロッパポータルからアクセストークンを取得。
   - deshaefrost の管理画面で `POST /api/v1/credentials` エンドポイントに `platform` と `access_token` をJSONで送信。
   - 応答として `credential_id` と `status: "verified"` が返却されることを確認。

2. **ターゲットオーディエンスのセグメント分析実行**
   - `POST /api/v1/segmentation/run` に `segmentation_method` と `feature_set` を指定。
   - バックグラウンドジョブが完了するまでポーリング（`GET /api/v1/segmentation/{job_id}/status`）。
   - ジョブ完了後、`GET /api/v1/segmentation/{job_id}/clusters` で各クラスタの詳細を取得。

3. **最適投稿スケジュールの生成と適用**
   - 投稿予定のコンテンツを `POST /api/v1/content/analyze` にアップロード（マルチパートフォーム）。
   - 応答の `recommended_post_time` をカレンダーアプリやSNS管理ツールに連携。
   - 実際の投稿後、`POST /api/v1/content/{content_id}/feedback` で実績エンゲージメントを送信し、モデルを更新。

### ベストプラクティス

- **プラクティス 1: クロスプラットフォームの一貫性維持**
  各プラットフォームでブランドトーンとビジュアルアイデンティティを統一するため、deshaefrost の `brand_guidelines` エンドポイントにカラーパレット、フォント、口調ルールをJSONで事前登録し、コンテンツ分析時に自動チェックさせる。これにより、プラットフォーム間のブランド乖離を防止できる。

- **プラクティス 2: 予測モデルの定期再学習の自動化**
  週次で `POST /api/v1/model/retrain` をcronジョブから実行し、最新のエンゲージメントデータでLightGBMモデルを再学習させる。再学習後は必ず `GET /api/v1/model/validation` でMAPEが 10% 未満であることを確認。閾値を超えた場合は過去データとの差分を調査する。
