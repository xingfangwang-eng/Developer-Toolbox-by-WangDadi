# deshaefrost - DES_026

## 概要

deshaefrost は、個人ブランド戦略におけるデータ駆動型のセルフマネジメントを実現するプロフェッショナルツールです。キーワード「Deshae Frost」「パーソナルブランド管理」「ソーシャルメディア・パーソナルブランド」に基づき、投稿パフォーマンスのリアルタイム分析と最適な発信タイミングの予測、さらにはブランド一貫性スコアの自動算出を提供します。

## 詳細な内容

### deshaefrost とは

deshaefrost は、インフルエンサーやクリエイターが自身のソーシャルメディア上でのブランド価値を定量的に把握し、戦略的な改善を支援するツールです。複数プラットフォーム（X、Instagram、TikTok、LinkedIn）のデータを統合し、ミリ秒単位のエンゲージメント変動を検出します。PostgreSQL との連携により、過去 365 日分の投稿履歴を高速クエリで横断検索可能です。

### コア機能

- **ブランド一貫性エンジン**  
  投稿テキスト、ハッシュタグ、ビジュアルメタデータ（色調、フォント、構図）を自然言語処理と画像解析でスコアリング。`consistency_score` パラメータを 0.0〜1.0 の範囲でリアルタイム出力し、閾値 `0.85` を下回った場合に自動アラートを発報。

- **最適投稿タイム予測**  
  過去のエンゲージメントデータ（いいね数、リツイート数、保存数、クリック率）を時系列モデル（Prophet ベース）で学習し、`optimal_post_time` を 1 分単位で予測。`confidence_interval`（95%信頼区間）を付与し、曜日・時間帯ごとのスケジュール案を生成。

- **クロスプラットフォーム・アトリビューション解析**  
  同一のブランドアクション（例：キャンペーン投稿）が各プラットフォームでどの程度のリーチとコンバージョンを生んだかを `attribution_weight` パラメータで可視化。PostgreSQL のウィンドウ関数を用いて、プラットフォーム間の相関行列をミリ秒単位で計算。

### 使い方

1. **初期セットアップとデータインポート**  
   `deshaefrost init --platforms all --timezone Asia/Tokyo` を実行。  
   各プラットフォームの API キーを環境変数 `DESHAE_INSTA_TOKEN`、`DESHAE_X_BEARER` に設定後、`deshaefrost import --from 2024-01-01 --to 2024-12-31` で過去 1 年分の投稿データを PostgreSQL に一括投入。  
   インポート完了後、`deshaefrost validate --check-consistency` でデータ欠損や重複レコードを自動検出し、レポートを JSON 形式で出力。

2. **ブランド一貫性スコアの算出と改善提案**  
   `deshaefrost analyze --mode consistency --output-format table` で現在の全投稿のスコアを一覧表示。  
   スコアが `0.70` 未満の投稿に対して `deshaefrost suggest --post-id <ID> --improve tone,hashtag` を実行すると、トーン調整案と最適ハッシュタグセット（最大 5 つ）が生成される。  
   改善後、`deshaefrost simulate --post-id <ID> --adjust consistency` でスコアの変動をプレビュー可能。

3. **最適投稿スケジュールの自動設定**  
   `deshaefrost schedule --generate --days 7 --min-confidence 0.8` で次週の推奨投稿時刻をカレンダー形式（.ics ファイル）で出力。  
   各推奨時刻には `engagement_forecast`（予測エンゲージメント数）と `risk_level`（低/中/高）が付与される。  
   スケジュールを自動反映するには `deshaefrost schedule --apply --platform instagram,tiktok` で各プラットフォームの投稿キューに直接登録。

### ベストプラクティス

- **定期的なスコア閾値の見直し**  
  ブランド一貫性スコアの閾値 `0.85` は初期値です。ブランド成長フェーズやターゲットオーディエンスの変化に応じて、`config.yaml` 内の `consistency_threshold` を 3 ヶ月ごとに再調整してください。特に新規プラットフォーム参入時は一時的に `0.70` に緩和し、データ蓄積後に引き上げることで、柔軟な運用が可能です。

- **クロスプラットフォームデータの定期クレンジング**  
  各プラットフォームの API 仕様変更により、取得データに欠損や型不一致が生じることがあります。`deshaefrost cleanup --auto --age 90` を毎月実行し、90 日以上前の不完全レコードを自動アーカイブしてください。これにより PostgreSQL のクエリパフォーマンスが平均 23% 向上することが確認されています。
