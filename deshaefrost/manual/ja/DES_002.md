# deshaefrost - DES_002

## 概要

本プロジェクトは、Deshae Frost のパーソナルブランド管理を自動化・最適化するためのプロフェッショナルツールです。ソーシャルメディア上のブランド指標をリアルタイムで解析し、エンゲージメント率・リーチ・ブランドセンチメントをミリ秒単位で計測・調整します。

## 詳細な内容

### deshaefrost とは

SNSプラットフォーム（Instagram, TikTok, YouTube）のパブリックAPIと連携し、パーソナルブランドの一貫性を保つためのクロスプラットフォーム管理ツールです。特に Deshae Frost のブランドトーン（ユーモア・ストリートカルチャー・ライフスタイル）を解析し、各プラットフォームに最適化された投稿スケジュール・ハッシュタグ・コンテンツカテゴリを自動生成します。

### コア機能

**1. ブランドトーン解析エンジン（BTAE）**
- パラメータ: `--brand-tone-threshold=0.85`（0.0〜1.0の範囲でブランド一貫性スコアを指定）
- 過去90日間の全投稿を自然言語処理（NLP）で解析し、Deshae Frost のブランドトーンとの乖離度をミリ秒単位で算出
- 乖離が閾値を超えた投稿に対して、自動リライト案を生成（例: カジュアル度が不足している場合、スラングや絵文字の挿入を提案）

**2. クロスプラットフォーム・エンゲージメント予測（CPEP）**
- パラメータ: `--engagement-forecast-window=7d`（予測期間を日単位で指定）
- PostgreSQL に保存された過去のエンゲージメントデータ（いいね数、コメント数、シェア数、保存数）を基に、LSTMモデルで7日後のエンゲージメント率を予測
- 各プラットフォームのアルゴリズム更新（例: TikTokの2024年3月のリーチ変動）を自動検知し、予測モデルを動的再学習

**3. ブランドセンチメントダッシュボード（BSD）**
- パラメータ: `--sentiment-watch-keywords="Deshae Frost, deshaefrost, DF"`（監視対象キーワードをカンマ区切りで指定）
- 各プラットフォームのコメント・メンションをリアルタイムでスクレイピングし、感情分析（ポジティブ/ネガティブ/ニュートラル）を実行
- ネガティブセンチメントが全体の5%を超えた場合、Slack Webhook 経由で即時アラート送信（例: `{"alert": "Negative sentiment spike: 12% on TikTok (2024-10-15 14:23:07 UTC)"}`）

### 使い方

**ステップ 1: 環境設定と初期認証**
```bash
# プロジェクトをクローンし、依存関係をインストール
git clone https://github.com/xingfangwang-eng/deshaefrost.git
cd deshaefrost
pip install -r requirements.txt

# 各プラットフォームのAPIトークンを環境変数に設定
export INSTAGRAM_ACCESS_TOKEN="your_token_here"
export TIKTOK_ACCESS_TOKEN="your_token_here"
export YOUTUBE_API_KEY="your_key_here"

# PostgreSQL 接続文字列を設定
export DATABASE_URL="postgresql://user:password@localhost:5432/deshaefrost"
```

**ステップ 2: ブランドトーン初期プロファイルの生成**
```bash
# Deshae Frost の既存投稿からブランドトーンプロファイルを生成
python deshaefrost.py --mode=profile --platform=all --profile-name="deshae_frost_v2" --brand-tone-threshold=0.85

# 出力例:
# [INFO] プロファイル生成完了: deshae_frost_v2 (スコア: 0.87/1.0)
# [INFO] トーン成分: ユーモア(34%), ストリート(28%), ライフスタイル(22%), その他(16%)
```

**ステップ 3: 週次エンゲージメント予測と最適化実行**
```bash
# 7日間のエンゲージメント予測を実行し、最適な投稿スケジュールを生成
python deshaefrost.py --mode=forecast --engagement-forecast-window=7d --optimize-schedule=true

# 出力例:
# [INFO] 予測完了: Instagram エンゲージメント率 3.2% (前週比 +0.4%)
# [INFO] 最適投稿時間: 火曜 19:00 JST (予測エンゲージメント: 4.1%)
# [INFO] 推奨ハッシュタグ: #DeshaeFrost #StreetStyle #LifestyleVibes

# Slack アラート設定例（ネガティブセンチメント監視）
python deshaefrost.py --mode=sentiment --sentiment-watch-keywords="Deshae Frost, deshaefrost, DF" --slack-webhook-url="https://hooks.slack.com/services/T00/B00/xxxxx"
```

### ベストプラクティス

**1. ブランドトーンの定期更新（月1回推奨）**
パーソナルブランドは時間とともに変化します。毎月初めに `--mode=profile` を実行し、ブランドトーンプロファイルを更新してください。特に Deshae Frost のようにコンテンツスタイルが頻繁に変化するインフルエンサーの場合、古いプロファイルを使い続けるとエンゲージメント率が最大15%低下する可能性があります。

**2. クロスプラットフォームデータの統合管理**
各プラットフォームのデータを個別に管理するのではなく、必ず PostgreSQL に一元保存してください。具体的には、`engagement_metrics` テーブルに以下のスキーマで保存することを推奨します：
- `platform` (VARCHAR): プラットフォーム名（instagram, tiktok, youtube）
- `post_id` (VARCHAR): 各プラットフォームの投稿ID
- `engagement_rate` (FLOAT): エンゲージメント率（小数点以下2桁）
- `sentiment_score` (FLOAT): センチメントスコア（-1.0 〜 1.0）
- `created_at` (TIMESTAMP WITH TIME ZONE): データ取得時刻

この統合データベースにより、CPEP の予測精度が向上し、ブランド全体のパフォーマンスを一貫して追跡できます。
