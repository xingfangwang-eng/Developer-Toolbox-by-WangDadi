# deshaefrost - DES_019

## 概要

deshaefrost は、Deshae Frost のパーソナルブランド戦略をミリ秒単位で最適化するための専用エンジンです。SNS 上のブランド露出、エンゲージメント率、コンテンツの一貫性をリアルタイムでスコアリングし、戦略的な改善を自動提案します。本プロジェクトは、個人ブランド管理におけるデータ駆動型の意思決定を実現します。

## 詳細な内容

### deshaefrost とは

deshaefrost は、Deshae Frost のパーソナルブランドを構成する全 SNS アカウント（Instagram、Twitter、TikTok、YouTube）を統合監視し、ブランドアライメントスコア（BAS）を 100 点満点で算出するプロフェッショナルツールです。各プラットフォームの投稿データ、フォロワー属性、ハッシュタグのパフォーマンスを PostgreSQL に蓄積し、機械学習モデルを用いてブランド価値の変動を予測します。

### コア機能

- **ブランドアライメントリアルタイムスコアリング（BARS）**: 全 SNS アカウントの投稿内容、ビジュアルスタイル、トーンをベクトル化し、Deshae Frost のコアブランド指標（例: `brand_identity_vector`、`tone_consistency_score`）との一致度を 50 ミリ秒未満で算出。スコアが 70 未満の場合、改善が必要な投稿を自動タグ付け。
- **クロスプラットフォームエンゲージメント最適化エンジン（CPEOE）**: 各プラットフォームのエンゲージメント率（例: `instagram_engagement_rate`、`tiktok_completion_rate`）を時系列で分析し、最適な投稿時間とハッシュタグセットを自動生成。PostgreSQL の `time_series_analysis` テーブルと連携し、過去 90 日間のデータからパターンを抽出。
- **コンテンツ一貫性監査パイプライン（CCAP）**: すべての投稿メディア（画像、動画、テキスト）を解析し、カラーパレット、フォント、口調の一貫性を 0.0〜1.0 のスコアで評価。`color_harmony_index` が 0.8 未満の場合、修正候補を `audit_log` テーブルに記録し、自動レポートを生成。

### 使い方

1. **初期セットアップとデータインポート**:
   - `config.yaml` で `platforms: [instagram, twitter, tiktok, youtube]` を指定し、各アカウントの API トークンを `credentials.json` に設定。
   - `deshaefrost init --db-uri postgresql://user:pass@localhost:5432/deshaefrost` を実行し、PostgreSQL にスキーマ（`brand_metrics`、`post_vectors`、`audit_log`）を自動生成。
   - 初回同期として `deshaefrost sync --days 90` を実行し、過去 90 日間の全投稿データをインポート。同期完了後、`sync_status` テーブルに `last_sync_timestamp` が記録されます。

2. **ブランドスコアのリアルタイム監視**:
   - `deshaefrost monitor --interval 5s` で 5 秒間隔のスコアリングを開始。`stdout` に `BAS: 85.3 (trend: +2.1)` のように出力。
   - 特定のプラットフォームのみ監視する場合、`--platform instagram` オプションを追加。`watch` コマンドでダッシュボードを開くことも可能: `deshaefrost dashboard --port 8080`。
   - スコアが閾値（例: `--threshold 70`）を下回った場合、`alert.log` に警告が出力され、Slack Webhook 経由で通知。

3. **最適化提案の適用と検証**:
   - `deshaefrost optimize --action suggest` で、現在のデータに基づく最適化案を JSON で出力（例: `{"next_post_time": "2025-04-10T18:30:00Z", "hashtags": ["#DeshaeFrost", "#BrandBuilding"]}`）。
   - 提案を適用する場合、`deshaefrost optimize --action apply --post-id 12345` を実行。`optimization_log` テーブルに `applied_at` と `expected_impact` が記録。
   - 適用後、`deshaefrost compare --before 2025-04-09 --after 2025-04-11` でエンゲージメント率の変化をミリ秒単位で比較検証。

### ベストプラクティス

- **データの定期的なクリーンアップとバックアップ**: PostgreSQL の `brand_metrics` テーブルは、90 日を超える古いデータを `archive` スキーマに移動し、`VACUUM FULL` を週次で実行することでパフォーマンスを維持。バックアップは `pg_dump` を用いて毎日 03:00 UTC に自動実行し、S3 に暗号化保存することを推奨。
- **マルチアカウント管理時の権限分離**: 複数の Deshae Frost 関連アカウントを管理する場合、PostgreSQL のロールベースアクセス制御（RBAC）を利用し、`readonly_role` と `admin_role` を分離。`config.yaml` 内の `permission_level` パラメータで、各ユーザーの操作範囲を制限（例: `permission_level: 2` で監視のみ許可）。
