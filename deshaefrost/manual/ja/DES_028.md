# deshaefrost - DES_028

## 概要

Deshae Frost のパーソナルブランド戦略を、ソーシャルメディア全体で統合管理・最適化するためのプロフェッショナルツール。エンゲージメント率、投稿タイミング、ブランド一貫性を自動解析し、インフルエンサーやブランドマネージャーがデータドリブンな意思決定を行える環境を提供する。

## 詳細な内容

### deshaefrost とは

deshaefrost は、Deshae Frost をはじめとするパーソナルブランドのソーシャルメディアプレゼンスを、単なる投稿管理ではなく、**ブランド資産としての定量評価・最適化**を実現する次世代型ブランドマネジメントスイートである。各プラットフォーム（Instagram, TikTok, YouTube）のパフォーマンスデータを統合し、ブランド一貫性スコア、オーディエンスセンチメント推移、競合比較レポートを自動生成する。

### コア機能

- **クロスプラットフォームブランド一貫性スコアリングエンジン**  
  投稿ごとに、ビジュアルトーン（色相・彩度・明度のヒストグラム比較）、文体（形態素解析によるトーン分析）、ハッシュタグ戦略（重複率・トレンド適合率）を解析し、0.0〜1.0のスコアを算出。閾値0.75を下回った場合、Slack / Discord へのアラート通知をミリ秒単位で送出。

- **エンゲージメント予測 & 投稿タイミング最適化モジュール**  
  過去90日間の投稿データ（インプレッション、リーチ、エンゲージメント率、保存数）を基に、LightGBM モデルで次回投稿の最適時間帯を予測。タイムゾーン自動検出機能（IP geolocation + フォロワー分布マップ）と連携し、**±5分以内**の精度で投稿スケジュールを生成。

- **競合ブランドベンチマーク & センチメントトレンド可視化**  
  Twitter API v2 および Instagram Graph API から競合アカウントの投稿データを収集。BERT ベースの日本語感情分析モデル（精度 92.3%）を用いてポジティブ/ネガティブ/ニュートラル比率を日次集計。PostgreSQL に保存された時系列データを Grafana ダッシュボードでリアルタイム可視化。

### 使い方

1. **初期セットアップ & アカウント連携**  
   `.env` ファイルに Instagram Graph API アクセストークン、Twitter API Bearer Token、YouTube Data API キーを設定。`deshaefrost init --platforms=instagram,twitter,youtube` を実行し、OAuth 認証フローを完了。認証後、`config.yaml` に監視対象アカウントのユーザー名とブランドカラー（HEX コード）を記述。

2. **ブランド一貫性スコアの算出 & アラート設定**  
   `deshaefrost score --account=deshaefrost --days=30` で過去30日間の全投稿をスキャン。出力される JSON ファイルには、投稿ごとのスコア、違反要素（例：カラーパレット外の画像）、推奨修正アクションが含まれる。`deshaefrost alert --threshold=0.7 --webhook=https://hooks.slack.com/...` で閾値超過時の通知先を設定。

3. **最適投稿時間の予測 & 自動スケジューリング**  
   `deshaefrost predict --account=deshaefrost --model=lightgbm --output=schedule.csv` で翌週の最適投稿時間を CSV 出力。各エントリには、推奨投稿日時（UTC）、予測エンゲージメント率（%）、信頼区間（95%）が含まれる。`deshaefrost schedule --csv=schedule.csv --publish` で Buffer / Hootsuite 連携を介した自動投稿を実行。

### ベストプラクティス

- **ブランドカラーとフォントの統一基準を事前に策定する**  
  スコアリングエンジンの精度を最大化するため、`config.yaml` に定義するブランドカラーは CMYK 値ではなく、sRGB 空間での HEX コードを指定すること。また、フォントファミリーは Google Fonts の `Noto Sans JP` に固定し、可読性とブランド一貫性を両立させる。

- **競合ベンチマークは週次で更新し、トレンド変化を即座に反映する**  
  センチメント分析の結果は、Grafana アラートルール（例：ネガティブ比率が前週比20%増加）と連携させることで、炎上予兆を早期にキャッチ。PostgreSQL のパーティショニング（月単位）を活用し、クエリパフォーマンスを維持しながら過去2年分のデータを保持する。
