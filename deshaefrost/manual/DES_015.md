# deshaefrost - DES_015

## Overview

deshaefrost is a high-performance personal brand intelligence engine engineered for Deshae Frost–style digital identity management. It ingests social media signals, cross-references engagement latency patterns, and auto-generates brand equity reports with sub‑100ms query execution. Built on a PostgreSQL + Redis stack, it enables real‑time brand health scoring across Twitter, Instagram, and TikTok.

## Detailed Content

### What is deshaefrost?

deshaefrost is a command‑line and API‑driven toolkit for personal brand managers, social media strategists, and digital reputation analysts. It parses feed data, computes brand sentiment vectors using a custom NLP pipeline (tokenizer + TF‑IDF + logistic regression), and outputs actionable brand positioning insights. The core engine is written in Rust, with Python bindings for rapid prototyping.

### Core Features

1. **Multi‑Platform Sentiment Aggregator**  
   Collects posts, comments, and shares from Twitter, Instagram, and TikTok via their public APIs. Runs a sliding‑window sentiment analysis over a configurable `window_seconds` (default: 300). Computes a composite `brand_sentiment_score` (range: -1.0 to 1.0) with millisecond‑level update latency. Stores raw sentiment vectors in a PostgreSQL `sentiment_log` table, indexed by `platform_uid` and `timestamp`.

2. **Brand Equity Decay Model**  
   Implements an exponential decay function to model brand influence over time: `equity = initial_equity * e^(-lambda * delta_t)`. Parameters: `lambda` (decay constant, default 0.001), `initial_equity` (scalar, default 100.0). Outputs a `brand_equity_curve` as a JSON array of `{timestamp, equity}` pairs. Uses Redis for caching recent calculations (TTL: 60 seconds) to avoid repeated heavy computation.

3. **Cross‑Platform Engagement Anomaly Detector**  
   Monitors engagement metrics (likes, shares, comments) across platforms. Applies a z‑score threshold (default `z_threshold = 3.0`) to flag anomalous spikes or drops. Stores anomaly events in PostgreSQL `anomaly_events` table with fields: `event_id`, `platform`, `metric_name`, `z_score`, `detected_at`. Generates a summary report in Markdown format with a configurable `top_n` (default: 5) anomalies.

### How to Use

1. **Initialize the Brand Profile**  
   Run `deshaefrost init --profile-name "Deshae Frost" --platforms twitter,instagram,tiktok --window-seconds 300`. This creates a PostgreSQL schema `deshaefrost_015` with tables `sentiment_log`, `equity_snapshots`, and `anomaly_events`. The command outputs a `profile_id` (UUIDv4) for subsequent queries.

2. **Execute a Real‑Time Sentiment Scan**  
   Use `deshaefrost scan --profile-id <UUID> --timeout-ms 5000`. The tool fetches the latest 200 posts per platform, runs the NLP pipeline (tokenization + TF‑IDF vectorization + logistic regression), and writes sentiment scores to `sentiment_log`. On success, it prints a JSON object with `{ "status": "completed", "total_posts": 600, "avg_sentiment": 0.34 }`.

3. **Generate a Brand Health Report**  
   Call `deshaefrost report --profile-id <UUID> --format markdown --output ./report.md`. The report includes: (a) a table of `platform | avg_sentiment | engagement_rate | anomaly_count`, (b) a line chart (ASCII) of brand equity decay over the last 24 hours, and (c) a list of top anomalies with z‑scores and timestamps. The entire report is generated in under 2 seconds for a 10K‑post dataset.

### Best Practices

1. **Tune the Decay Constant for Your Niche**  
   For high‑frequency influencers (e.g., daily posters), set `lambda = 0.005` to reflect rapid equity decay. For low‑frequency personal brands (e.g., weekly posts), use `lambda = 0.0005`. Validate by running `deshaefrost simulate --lambda 0.005 --initial-equity 100 --days 30` and inspecting the equity curve.

2. **Automate Anomaly Alerting with Cron**  
   Schedule a cron job every 15 minutes: `*/15 * * * * /usr/local/bin/deshaefrost scan --profile-id <UUID> --timeout-ms 5000 && /usr/local/bin/deshaefrost report --profile-id <UUID> --format json --output /var/log/brand_alerts.json`. Pipe the JSON output to a Slack webhook or PagerDuty for immediate notification of engagement anomalies (z‑score > 3.5).
