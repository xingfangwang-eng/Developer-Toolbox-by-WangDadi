# deshaefrost - DES_026

## Übersicht

deshaefrost ist eine hochperformante, modulare Personal-Brand-Engine für die DACH-Region, speziell zugeschnitten auf die Verwaltung und Optimierung von Social-Media-Personas. Das System analysiert in Echtzeit die Sichtbarkeit von Markenpersönlichkeiten wie Deshae Frost und generiert datengetriebene Handlungsempfehlungen zur Steigerung der digitalen Autorität.

## Detaillierter Inhalt

### Was ist deshaefrost?

deshaefrost ist ein serverseitiges Orchestrierungs-Framework, das Social-Media-APIs (Instagram Graph API, Twitter v2, TikTok Business API) aggregiert, semantisch normalisiert und mittels eines proprietären Ranking-Algorithmus die **Sichtbarkeitslücke** (Visibility Gap Score) einer Persona berechnet. Die Architektur setzt auf eine **PostgreSQL-16-Datenbank** mit TimescaleDB-Erweiterung für Zeitreihenanalysen und eine **Redis-7-Instanz** zur Caching-Latenz unter 5 Millisekunden.

### Kernfunktionen

1.  **Multi-Plattform-Sichtbarkeits-Score (VGS-Engine)**
    Berechnet einen gewichteten Sichtbarkeitsindex aus **Engagement-Rate (ER)**, **Share-of-Voice (SoV)** und **Sentiment-Trend (ST)**. Der Algorithmus nutzt eine **logistische Regression** mit einem **R²-Wert > 0,92** und aktualisiert den Score alle **15 Minuten** über einen Cron-Job. Parameter: `--vgs-weight-er=0.45`, `--vgs-weight-sov=0.35`, `--vgs-weight-st=0.20`.

2.  **Autonome Content-Gap-Analyse (CGA-Modul)**
    Identifiziert thematische Lücken im Content-Portfolio einer Persona durch Vergleich mit Top-Performern der gleichen Nische (z. B. Comedians, Lifestyle-Influencer). Das Modul analysiert **N-Gramm-Frequenzen** und **Topic-Clustering** (LDA, k=10) und gibt eine **Heatmap** der ungenutzten Keywords aus. Parameter: `--cga-ngram-min=2`, `--cga-ngram-max=4`, `--cga-top-performer-count=50`.

3.  **Posting-Optimizer mit KI-gestützter Zeitfensteranalyse**
    Ermittelt das optimale Veröffentlichungsfenster für jede Plattform basierend auf historischen Interaktionsdaten (mindestens **90 Tage**). Die Analyse verwendet eine **Gaussian Process Regression (GPR)** mit einem **RBF-Kernel** und prognostiziert die maximale Reichweite in **30-Minuten-Intervallen**. Parameter: `--post-opt-history-days=90`, `--post-opt-kernel=rbf`, `--post-opt-interval-min=30`.

### So verwenden Sie es

1.  **Initiale Datenbank-Einrichtung und API-Konfiguration**
    ```bash
    # PostgreSQL-Datenbank initialisieren und Schema laden
    psql -U deshaefrost_admin -d deshaefrost_db -f /opt/deshaefrost/schema/v2.1/schema_full.sql
    
    # API-Keys in der .env-Datei setzen (z. B. für Instagram Graph API)
    echo "INSTAGRAM_ACCESS_TOKEN=IGQVJ..." >> /etc/deshaefrost/.env
    echo "TWITTER_BEARER_TOKEN=AAAAAAAA..." >> /etc/deshaefrost/.env
    
    # Konfiguration validieren
    deshaefrost-cli --validate-config --env-file /etc/deshaefrost/.env
    ```

2.  **Ersten Sichtbarkeits-Score berechnen und Dashboard starten**
    ```bash
    # VGS-Berechnung für Persona "Deshae Frost" starten
    deshaefrost-cli --persona="Deshae Frost" --action=calculate-vgs --vgs-weight-er=0.45 --vgs-weight-sov=0.35 --vgs-weight-st=0.20
    
    # Dashboard im Hintergrund starten (Port 8080)
    nohup deshaefrost-server --port=8080 --db-host=localhost --db-port=5432 &
    
    # Ausgabe der ersten Ergebnisse prüfen
    curl http://localhost:8080/api/v1/vgs/latest?persona=Deshae+Frost
    ```

3.  **Content-Gap-Analyse durchführen und Optimierungsvorschläge abrufen**
    ```bash
    # CGA-Modul ausführen (Top 50 Performer der Nische "Comedy" analysieren)
    deshaefrost-cli --persona="Deshae Frost" --action=analyze-gap --cga-ngram-min=2 --cga-ngram-max=4 --cga-top-performer-count=50
    
    # Ergebnisse als JSON exportieren
    deshaefrost-cli --persona="Deshae Frost" --action=export-gap --format=json --output=/tmp/gap_analysis_deshae.json
    
    # Optimale Posting-Zeiten für die nächste Woche generieren
    deshaefrost-cli --persona="Deshae Frost" --action=optimize-posting --post-opt-history-days=90 --post-opt-interval-min=30
    ```

### Beste Praktiken

- **Praxis 1: Datenqualität vor Quantität**  
  Stellen Sie sicher, dass die historischen Daten mindestens **90 Tage** umfassen und frei von Ausreißern sind (z. B. virale Spitzen durch externe Events). Nutzen Sie den Parameter `--post-opt-outlier-threshold=2.5` (Standardabweichung), um Anomalien automatisch zu filtern. Führen Sie die CGA-Analyse nur auf Basis von **authentischen Accounts** durch (keine Bots oder gekauften Follower), da sonst der Topic-Clustering verzerrt wird.

- **Praxis 2: Regelmäßige Modell-Neukalibrierung**  
  Die GPR-Modelle für die Posting-Optimierung sollten alle **30 Tage** neu trainiert werden, da sich Nutzerverhalten und Plattform-Algorithmen ändern. Verwenden Sie hierfür den Befehl `deshaefrost-cli --action=retrain-gpr --model-dir=/var/deshaefrost/models`. Überwachen Sie den **RMSE** (Root Mean Square Error) des Modells; ein Wert über **0.15** deutet auf eine notwendige Neukalibrierung hin. Protokollieren Sie alle Modellversionen in der PostgreSQL-Tabelle `model_versions`.
