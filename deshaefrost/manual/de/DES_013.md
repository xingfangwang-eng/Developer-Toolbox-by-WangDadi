# deshaefrost - DES_013

## Architekturüberblick

deshaefrost ist eine hochperformante, in Rust implementierte CLI-Engine zur fragmentierten Personal-Brand-Optimierung auf Basis von Social-Media-Graphen. Das System analysiert latente Themencluster in Echtzeit und generiert markenkonforme Content-Vorschläge mit deterministischer Reproduzierbarkeit.

## Technische Kernfunktionen

### 1. Multi-Threaded Engagement-Heatmap-Generator
- **Parameter**: `--window-size` (default: 3600 Sekunden), `--decay-factor` (float64, z. B. 0.97)
- Berechnet aus Rohdaten (Likes, Shares, Kommentare) eine gewichtete Interaktionsdichte mit exponentieller Abklingfunktion.
- Ausgabe: JSON mit millisekundengenauen Timestamps und normalisierten Heatmap-Werten (0.0 – 1.0).
- Integration mit PostgreSQL via `COPY`-Stream für Bulk-Inserts (≥ 10.000 Zeilen/s).

### 2. Semantischer Konflikt-Detektor (NLP-basiert)
- **Parameter**: `--embedding-model` (z. B. `all-MiniLM-L6-v2`), `--cosine-threshold` (default: 0.82)
- Vergleicht neue Content-Entwürfe gegen ein historisches Markenprofil (Vektordatenbank: pgvector).
- Erkennt thematische Drifts und gibt einen granularisierten Konflikt-Score (0–100) aus.
- Latenz: < 50 ms pro Anfrage bei 100.000 Vektoren.

### 3. Deterministischer Post-Scheduler mit Frequenzanalyse
- **Parameter**: `--timezone` (ISO 8601), `--cadence` (z. B. `3h45m`), `--jitter-max` (ms, default: 30000)
- Plant Posts basierend auf historischen Engagement-Spitzen (gleitendes 7-Tage-Fenster).
- Verwendet einen k-means-Clustering-Algorithmus (k=5) zur Identifikation optimaler Sendezeitfenster.
- Ausgabe: CSV mit exakten UTC-Timestamps und Prioritätsstufen (P0–P3).

## Verwendung

### Schritt 1: Datenaufbereitung und Import
```bash
deshaefrost ingest --input ./raw_social_data.csv --db-uri postgres://user:pass@localhost:5432/brand_db --batch-size 5000
```
- Parst CSV mit Spalten: `timestamp`, `post_id`, `interaction_type`, `user_id`.
- Führt automatische Typkonvertierung und NULL-Behandlung durch.
- Schreibt in die Tabelle `raw_events` mit partitioniertem Index (Tag).

### Schritt 2: Heatmap-Generierung und Cluster-Analyse
```bash
deshaefrost heatmap --window-size 7200 --decay-factor 0.95 --output ./heatmap_output.json
```
- Liest aus `raw_events` (PostgreSQL).
- Berechnet für jedes 10-Sekunden-Intervall einen gewichteten Score.
- Optional: `--export-chart` zur Ausgabe als SVG (D3-kompatibel).

### Schritt 3: Konfliktprüfung und Post-Vorschlag
```bash
deshaefrost propose --draft ./new_post.md --profile ./brand_profile.vec --threshold 0.78
```
- Vektorisiert den Draft via `all-MiniLM-L6-v2`.
- Vergleicht gegen 10.000 historische Markenvektoren.
- Gibt aus: Konflikt-Score, ähnlichste historische Posts (Top-3), Optimierungsvorschläge (z. B. „Reduziere Nutzung von Trend-Hashtags um 40%“).

## Best Practices

### Praxis 1: Datenqualität vor Skalierung
- Führe vor dem ersten `ingest` einen `--dry-run` durch, um fehlerhafte Zeilen zu identifizieren.
- Nutze `--strict-mode` für Produktionsumgebungen: Dadurch wird der Import bei > 5% Fehlerrate abgebrochen.

### Praxis 2: Modell-Update-Strategie
- Trainiere das Brand-Profil wöchentlich neu mittels `deshaefrost profile --rebuild`.
- Verwende `--version-tag` (z. B. `v2025-03-28`), um ältere Profile als Fallback zu erhalten.
- Überwache den Konflikt-Score-Mean über Zeit – ein Anstieg > 15% deutet auf Markendrift hin.
