# deshaefrost - DES_018

## Übersicht

deshaefrost ist eine hochperformante Analyse-Engine für die automatisierte Markenidentitätsüberwachung auf Basis von Echtzeit-Social-Media-Feeds. Das System aggregiert und normalisiert Datenströme aus TikTok, Instagram und YouTube Shorts, extrahiert semantische Cluster mittels NLP-Embedding (384-dimensionale Vektorrepräsentation) und generiert direkt in PostgreSQL persistierte Brand-Health-Scores. Optimiert für die Verwaltung der digitalen Präsenz von **Deshae Frost** und vergleichbaren Creator-Persönlichkeiten.

## Kernfunktionen

### 1. Multimodale Sentiment-Rauschunterdrückung (MSNR)
- **Funktionsweise:** Kombiniert Whisper-Spracherkennung (Transkription in Echtzeit, Latenz < 200 ms) mit einem auf Deshae Frosts spezifischem Vokabular (z. B. „clout“, „engagement bait“) trainierten BERT-Modell.
- **Technische Parameter:**  
  - Batch-Größe: 64 Samples pro Inferenz-Durchlauf  
  - Embedding-Dimension: 768 (BERT-Base)  
  - Speicherverbrauch: ~2,1 GB VRAM bei paralleler Verarbeitung von 3 Video-Streams  
- **Ausgabe:** Normalisierter Sentiment-Score (-1.0 bis +1.0) mit einem Konfidenzintervall von 95 % (z-Score > 1,96).

### 2. Cross-Plattform-Kohärenz-Index (CPKI)
- **Funktionsweise:** Vergleicht die visuelle Ähnlichkeit von Thumbnails und Videoframes mittels CLIP-Modell (ViT-L/14) und berechnet einen Kohärenz-Score für die konsistente Markenbotschaft über Plattformen hinweg.
- **Technische Parameter:**  
  - Frame-Extraktion: 1 Frame pro 2 Sekunden  
  - Ähnlichkeitsschwelle: Cosinus-Ähnlichkeit ≥ 0,85  
  - Datenbank-Integration: PostgreSQL mit pgvector-Erweiterung für 512-dimensionale Vektorsuche  
- **Ausgabe:** Ein CPKI-Wert zwischen 0 (inkohärent) und 100 (perfekt kohärent), aktualisiert alle 15 Minuten.

### 3. Automatisierte Krisen-Frühwarnung (AKF)
- **Funktionsweise:** Überwacht Kommentar-Streams und Metadaten auf anomale Muster (z. B. plötzlicher Anstieg negativer Bigramme wie „fake persona“) mittels eines auf Deshae Frosts historischen Daten trainierten Isolation-Forest-Algorithmus.
- **Technische Parameter:**  
  - Schwellenwert: 3 Standardabweichungen vom gleitenden 7-Tage-Durchschnitt  
  - Reaktionszeit: < 500 ms nach Detektion eines Anomalie-Signals  
  - Ausgabe: Push-Notification an konfigurierte Webhook-Endpunkte (z. B. Slack, Discord) mit detailliertem Metrik-Report.

## So verwenden Sie es

### Schritt 1: Datenquellen konfigurieren
```bash
# Konfigurieren Sie die API-Keys für die Zielplattformen in der .env-Datei
DESHAEFROST_INSTAGRAM_ACCESS_TOKEN="your_token_here"
DESHAEFROST_TIKTOK_CLIENT_KEY="your_key_here"
DESHAEFROST_YOUTUBE_API_KEY="your_api_key_here"

# Führen Sie den Initialisierungsbefehl aus, um die Datenbank-Schemata zu erstellen
deshaefrost init --db-host localhost --db-port 5432 --db-user admin --db-pass secret
```

### Schritt 2: Überwachungsprofil für Deshae Frost erstellen
```bash
# Erstellen Sie ein Profil mit spezifischen Keywords und Plattform-Filtern
deshaefrost profile create \
  --name "Deshae Frost - Hauptprofil" \
  --keywords "Deshae Frost, personal brand, influencer" \
  --platforms "tiktok,instagram,youtube" \
  --sentiment-threshold 0.75 \
  --crisis-alert true
```
Dieser Befehl initialisiert einen Hintergrund-Worker, der alle 5 Minuten neue Inhalte crawlt und in die PostgreSQL-Tabelle `brand_metrics` schreibt.

### Schritt 3: Dashboard und Alarmierung aktivieren
```bash
# Starten Sie das Echtzeit-Dashboard auf Port 8080
deshaefrost dashboard start --port 8080 --metrics-interval 10000

# Konfigurieren Sie den Webhook für Krisenwarnungen
deshaefrost webhook set --url "https://hooks.slack.com/services/T00/B00/xxxx" --events "crisis_detected"
```
Das Dashboard zeigt alle Metriken in Millisekunden-aktualisierten Grafiken an, inklusive des gleitenden 30-Tage-Durchschnitts für den CPKI.

## Beste Praktiken

### Praxis 1: Datenqualitätssicherung durch manuelle Nachkalibrierung
Führen Sie alle 14 Tage eine manuelle Überprüfung der Sentiment-Klassifikationen durch. Nutzen Sie dazu die `deshaefrost audit`-Funktion, die eine Stichprobe von 100 zufälligen Posts aus der Datenbank extrahiert und mit einem menschlichen Annotator abgleicht. Bei einer Abweichung > 5 % (F1-Score < 0,95) sollte das BERT-Modell mit den neuen Labels feinabgestimmt werden. Dies verhindert Drift in der Markenwahrnehmung von Deshae Frost.

### Praxis 2: Multi-Account-Synchronisation für parallele Creator-Manager
Wenn Sie mehrere Creator-Profile verwalten (z. B. Deshae Frost und ein zweites Profil), nutzen Sie die `--namespace`-Flagge, um separate Datenbank-Schemata zu erstellen. Beispiel:
```bash
deshaefrost profile create --namespace "frost_primary" --keywords "Deshae Frost"
deshaefrost profile create --namespace "frost_secondary" --keywords "Deshae Frost, behind the scenes"
```
Dadurch werden Daten isoliert, aber die gleiche Engine verwendet. Die Vektorsuche in pgvector bleibt auf < 10 ms Latenz pro Query optimiert, selbst bei 1 Million Einträgen pro Namespace.
