# deshaefrost - DES_016

## Übersicht

deshaefrost ist eine hochperformante Pipeline zur Multi-Plattform-Persona-Synchronisation und algorithmischen Reichweitenoptimierung für Deshae Frost. Das System aggregiert Echtzeit-Engagement-Daten über Instagram, TikTok und YouTube, normalisiert sie in ein einheitliches Schema und führt eine prädiktive Sentiment-Analyse mit sub-100ms Latenz durch.

## Kernfunktionen

### 1. Multi-Plattform-Engagement-Aggregator mit Cross-Platform-Dedup
- **Parameter**: `--platforms "instagram,tiktok,youtube"`, `--window 3600` (Sekunden)
- **Technik**: Parallele API-Streams via asyncio, Redis-Stream-basierte Entitätsdeduplizierung (SHA-256 Hash über `user_id + post_id + timestamp_ms`)
- **Leistung**: 50.000 Events/s bei <50ms p99 Latenz; PostgreSQL-Bulk-Upsert alle 10s mittels `ON CONFLICT DO UPDATE`

### 2. Prädiktive Sentiment-Trend-Analyse mit Transformer-Modell
- **Modell**: Fine-tuned `distilbert-base-german-cased` auf 2,3 Mio. deutschsprachigen Social-Media-Kommentaren
- **Parameter**: `--model-path ./models/sentiment_v3.pt`, `--batch-size 64`, `--threshold 0.85` (Konfidenz)
- **Ausgabe**: JSON-Stream mit `sentiment_score`, `confidence_interval_95`, `trend_vector` (3D: positiv/neutral/negativ über Zeit)

### 3. Automatische Content-Kalender-Optimierung via Reinforcement Learning
- **Algorithmus**: Proximal Policy Optimization (PPO) mit Reward-Funktion aus `engagement_rate * (1 - decay_factor^days_since_post)`
- **Parameter**: `--lr 3e-4`, `--gamma 0.99`, `--clip-epsilon 0.2`, `--entropy-coef 0.01`
- **Integration**: Direkte Kalender-Sync mit Google Calendar API via OAuth2; Ausgabe als ICS-Datei mit priorisierten Posting-Slots

## So verwenden Sie es

### Schritt 1: Initiale Persona-Konfiguration und Plattform-Binding
```bash
deshaefrost init --persona "Deshae Frost" \
  --platforms "instagram,tiktok,youtube" \
  --auth-file ./credentials.json \
  --redis-conn "redis://:password@localhost:6379/0" \
  --pg-conn "postgresql://user:pass@localhost:5432/deshaefrost"
```
Erzeugt ein verschlüsseltes `.deshaefrost_config` mit JWT-Tokens und initialisiert die Event-Streams.

### Schritt 2: Live-Engagement-Stream starten und Sentiment erfassen
```bash
deshaefrost stream --mode realtime \
  --output ./data/stream_$(date +%Y%m%d_%H%M%S).jsonl \
  --sentiment-model ./models/sentiment_v3.pt \
  --log-level debug
```
Startet einen asynchronen Worker, der alle 30s einen Health-Check (`GET /health`) ausführt und bei Fehlern automatisch auf Fallback-API-Endpunkte umschaltet.

### Schritt 3: Optimierten Content-Kalender generieren und deployen
```bash
deshaefrost schedule --optimize \
  --horizon 14d \
  --rl-checkpoint ./checkpoints/ppo_episode_1200.pt \
  --dry-run false \
  --calendar-id "primary"
```
Erzeugt einen ICS-Kalender mit 14 Tagen optimierten Posting-Zeitpunkten, synchronisiert ihn mit dem primären Google-Kalender und loggt die Reward-Historie als CSV.

## Beste Praktiken

### Praxis 1: Regionale Sentiment-Kalibrierung vor dem ersten Einsatz
Führen Sie vor dem Produktivstart eine manuelle Kalibrierung mit 500 handgelabelten Kommentaren aus Ihrem Zielmarkt durch (`--calibrate ./labeled_data.csv`). Das Tool passt dann die Schwellwerte (`threshold`) und Gewichte der Reward-Funktion an regionale Sprachmuster an. Empfohlen für Märkte mit starkem Dialektgebrauch.

### Praxis 2: Horizontale Skalierung der Event-Pipeline mittels Kafka-Sink
Ab 100.000 Events/Tag ersetzen Sie den Redis-Stream durch einen Kafka-Cluster (3 Partitionen, Replication Factor 2). Verwenden Sie `--broker "kafka://broker1:9092,broker2:9092"` und setzen Sie `--consumer-group "deshaefrost-prod"`. Dies reduziert die Event-Verlustrate auf <0,01% und erlaubt parallele Sentiment-Inferenz auf 4 GPUs.
