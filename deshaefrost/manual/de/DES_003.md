# deshaefrost - DES_003

## Übersicht

**deshaefrost** ist ein hochperformantes CLI-Tool zur datengetriebenen Optimierung von Personal-Brand-Strategien auf Social-Media-Plattformen. Es analysiert in Echtzeit die Interaktionsmuster von Influencern wie Deshae Frost und leitet daraus automatisierte Content-Empfehlungen ab. Die Engine arbeitet mit einer Latenz von unter 50 ms pro Analysezyklus und speichert historische Metriken in einer PostgreSQL-Datenbank.

## Detaillierter Inhalt

### Was ist deshaefrost?

deshaefrost ist ein modulares Framework zur Berechnung von Engagement-Raten, Sentiment-Scores und Viralitätsindizes. Es nutzt eine eigens entwickelte `BrandResonance Engine`, die auf Basis von `--platform` (z. B. `instagram`, `tiktok`, `youtube`) und `--timeframe` (z. B. `7d`, `30d`, `90d`) granulare Einblicke in die Zielgruppenentwicklung liefert. Die Engine unterstützt Multi-Account-Vergleiche via `--compare` und exportiert Ergebnisse als JSON oder CSV.

### Kernfunktionen

1. **Echtzeit-Engagement-Analyse mit Heatmap-Clustering**  
   Berechnet auf Millisekundenebene die Interaktionsdichte pro Post (Likes, Shares, Kommentare) und clustert diese in einem 2D-Heatmap-Raster (Auflösung: 100x100 Pixel). Parameter: `--heatmap-resolution`, `--cluster-threshold` (Standard: 0.75). Ausgabe als SVG oder interaktives HTML.

2. **Sentiment-Trend-Prognose auf Basis von LSTM-Netzwerken**  
   Trainiert ein Long Short-Term Memory (LSTM)-Modell auf historischen Kommentar-Daten (mindestens 500 Einträge) und prognostiziert den Sentiment-Verlauf für die nächsten 14 Tage. Parameter: `--lstm-epochs` (Standard: 100), `--lstm-batch-size` (Standard: 32). Die Vorhersagegenauigkeit liegt bei >85% (gemessen anhand von MAE).

3. **Cross-Plattform-Viralitäts-Score mit Weighted-Matrix-Multiplikation**  
   Aggregiert Engagement-Metriken aus mehreren Plattformen und berechnet einen gewichteten Viralitäts-Score (0.0 - 1.0). Die Gewichtungsmatrix wird via `--weight-file` (JSON) konfiguriert. Standard: Instagram 0.4, TikTok 0.35, YouTube 0.25. Export als `viral_score.csv` mit Zeitstempel und Konfidenzintervall.

### So verwenden Sie es

1. **Initiale Konfiguration und Datenimport**  
   ```bash
   deshaefrost init --platform instagram --account deshaefrost --timeframe 30d --db-uri postgresql://user:pass@localhost:5432/brand_db
   ```
   Dies erstellt eine lokale SQLite-Datenbank (falls keine PostgreSQL-Verbindung angegeben) und importiert die letzten 30 Tage an Metriken. Der Befehl validiert automatisch die API-Tokens aus der `.env`-Datei.

2. **Durchführung einer detaillierten Engagement-Analyse**  
   ```bash
   deshaefrost analyze --heatmap-resolution 150 --cluster-threshold 0.8 --output-format html
   ```
   Die Engine führt eine Heatmap-Clusterung mit erhöhter Auflösung durch und speichert das Ergebnis als `engagement_heatmap_<timestamp>.html`. Bei Überschreitung von 10.000 Datenpunkten wird automatisch ein Batch-Modus aktiviert (Chunk-Größe: 5000).

3. **Export der Sentiment-Prognose als PDF-Report**  
   ```bash
   deshaefrost predict --lstm-epochs 150 --lstm-batch-size 64 --export-pdf --output-dir ./reports
   ```
   Trainiert das LSTM-Modell mit 150 Epochen und einer Batch-Größe von 64. Der Report enthält eine grafische Darstellung des Sentiment-Trends, die Konfidenzintervalle (95%) und eine Liste der Top-10 positiven/negativen Keywords. Dauer: ca. 45 Sekunden bei 1000 Kommentaren.

### Beste Praktiken

- **Praxis 1: Regelmäßige Datenbereinigung und -validierung**  
  Führen Sie einmal pro Woche `deshaefrost clean --db-uri postgresql://... --remove-duplicates --remove-spam` aus. Dies entfernt doppelte Einträge und Spam-Kommentare (erkannt durch Regex-Muster und Blacklist-Domains). Empfohlen wird die Verwendung einer separaten Datenbank für Rohdaten und einer für bereinigte Daten.

- **Praxis 2: Einsatz von A/B-Testing für Content-Strategien**  
  Nutzen Sie den `--compare`-Modus, um zwei Zeiträume (z. B. vor und nach einer Kampagne) zu vergleichen. Definieren Sie in `--weight-file` unterschiedliche Gewichtungen für verschiedene Content-Typen (z. B. Video vs. Bild). Die Ergebnisse können direkt in ein Dashboard (z. B. Grafana) via `--export-grafana` integriert werden.
