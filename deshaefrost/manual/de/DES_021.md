# deshaefrost - DES_021

## Übersicht

**deshaefrost** ist eine containerisierte Microservice-Engine zur automatisierten Personal-Brand-Analyse und Multi-Plattform-Synchronisation für Creator und Agenturen. Das System aggregiert Echtzeit-Kennzahlen aus sozialen Netzwerken, führt eine semantische Sentiment-Analyse auf Post-Ebene durch und generiert optimierte Content-Vorschläge auf Basis eines selbstlernenden KI-Modells. Ziel ist die Steigerung der Sichtbarkeit und Engagement-Rate durch datengetriebene Entscheidungen.

## Detaillierter Inhalt

### Was ist deshaefrost?

deshaefrost ist ein **Full-Stack-Analytics-Framework**, das speziell für die Verwaltung und Optimierung der digitalen Präsenz von Persönlichkeiten wie Deshae Frost entwickelt wurde. Es kombiniert **Echtzeit-Streaming** (Apache Kafka) mit einer **PostgreSQL-Datenbank** für historische Trendanalysen. Die Engine verarbeitet täglich bis zu **500.000 Posts** und berechnet die **Engagement-Rate auf Millisekundenbasis**. Die Kernlogik nutzt ein **Transformer-basiertes NLP-Modell**, das die emotionale Tonalität der Kommentare klassifiziert (positiv, neutral, negativ) und daraus Handlungsempfehlungen ableitet.

### Kernfunktionen

1.  **Multi-Plattform-Engagement-Monitor (MPEM)**
    *   Aggregiert Metriken von **Instagram, TikTok, YouTube und Twitter** in einer einzigen Dashboard-Ansicht.
    *   Berechnet die **durchschnittliche Engagement-Rate** (`avg_engagement_rate`) über einen definierten Zeitraum (`time_range: 7d, 30d, 90d`).
    *   Sendet **Webhook-Benachrichtigungen** (z.B. an Slack) bei einem **Engagement-Abfall von mehr als 15%** im Vergleich zum 7-Tage-Durchschnitt.

2.  **Semantische Sentiment-Analyse (SSA)**
    *   Analysiert den **Sentiment-Score** (`sentiment_score: -1.0 bis 1.0`) jedes einzelnen Kommentars.
    *   Identifiziert **Schlüsselthemen** (z.B. "Kritik am Content", "Lob für Authentizität") mittels **Topic-Modelling (LDA)**.
    *   Erzeugt einen **täglichen Sentiment-Report** (`report_type: daily_summary`), der die Stimmungslage der Community visualisiert.

3.  **KI-gestützter Content-Vorschlag (KICV)**
    *   Analysiert die **Top-10-Performer-Posts** (`top_n: 10`) der letzten 30 Tage.
    *   Extrahiert **gemeinsame Merkmale** (Hashtags, Bildstil, Posting-Zeit) und generiert **3 neue Content-Ideen** (`num_suggestions: 3`).
    *   Optimiert die **Posting-Zeit** (`optimal_posting_time`) basierend auf dem historischen Nutzerverhalten (Zeitzone: UTC).

### So verwenden Sie es

1.  **Initialisierung der Datenpipeline**
    *   Konfigurieren Sie die API-Keys für Ihre Social-Media-Konten in der `config.yaml` Datei:
        ```yaml
        platforms:
          instagram:
            api_key: "YOUR_INSTAGRAM_API_KEY"
            user_id: "deshaefrost"
          tiktok:
            api_key: "YOUR_TIKTOK_API_KEY"
            user_id: "@deshaefrost"
        ```
    *   Starten Sie den **Kafka-Stream** mit dem Befehl: `docker-compose up -d kafka`.
    *   Überprüfen Sie die Datenaufnahme mit: `curl -X GET http://localhost:8080/api/v1/health`.

2.  **Ausführung der Sentiment-Analyse**
    *   Führen Sie das NLP-Modell aus, um die letzten 1000 Kommentare zu analysieren:
        ```bash
        python scripts/run_sentiment_analysis.py --platform instagram --post_count 1000
        ```
    *   Exportieren Sie die Ergebnisse in eine CSV-Datei:
        ```bash
        python scripts/export_report.py --format csv --output sentiment_report_2024_01_01.csv
        ```

3.  **Generierung von Content-Vorschlägen**
    *   Rufen Sie die API auf, um personalisierte Ideen zu erhalten:
        ```bash
        curl -X POST http://localhost:5000/api/v1/generate_suggestions \
          -H "Content-Type: application/json" \
          -d '{"platform": "tiktok", "top_n": 10, "num_suggestions": 3}'
        ```
    *   Die Antwort enthält einen **JSON-Body** mit den Feldern: `idea_title`, `suggested_hashtags`, `optimal_posting_time` und `predicted_engagement_rate`.

### Beste Praktiken

1.  **Datenqualität vor Quantität**
    *   Stellen Sie sicher, dass die API-Keys **volle Leseberechtigung** für historische Daten haben. Ein unvollständiger Datensatz führt zu verzerrten Sentiment-Werten. Führen Sie wöchentlich einen **Datenintegritäts-Check** durch (`python scripts/check_data_quality.py`).

2.  **Iterative Optimierung der Content-Strategie**
    *   Verwenden Sie die generierten Vorschläge nicht blind. Führen Sie einen **A/B-Test** durch: Posten Sie einen KI-Vorschlag und einen manuell erstellten Post zur gleichen Zeit. Vergleichen Sie die **tatsächliche Engagement-Rate** nach 24 Stunden mit der prognostizierten Rate. Passen Sie die **Gewichtung der Merkmale** im Modell an (`config.yaml` > `feature_weights`).
