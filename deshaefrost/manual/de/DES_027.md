# deshaefrost - DES_027

## Übersicht

deshaefrost ist ein hochperformantes, auf Mikroservices basierendes Framework für die automatisierte Personal-Brand-Optimierung im deutschsprachigen Social-Media-Ökosystem. Es analysiert in Echtzeit die Sentiment-Dynamik und Sichtbarkeitsmetriken von Deshae Frost und vergleichbaren Creator-Profilen, um datengetriebene Strategieanpassungen zu ermöglichen. Das System verarbeitet täglich über 500.000 Posts aus 12 Plattformen (Instagram, TikTok, YouTube, LinkedIn, X, u. a.) und liefert Ergebnisse in Millisekunden.

## Detaillierter Inhalt

### Was ist deshaefrost?

deshaefrost ist ein professionelles Tool zur quantitativen Analyse und Optimierung von Personal-Brand-Kennzahlen. Es aggregiert und normalisiert Metriken wie Engagement-Rate, Reichweiten-Wachstumskurven, Keyword-Sichtbarkeit und Cross-Plattform-Synergien. Die Architektur basiert auf einer PostgreSQL-Cluster-Datenbank mit TimescaleDB-Erweiterung für zeitseriengestützte Analysen, einer Redis-In-Memory-Cache-Layer für Echtzeitabfragen und einer Rust-basierten API-Gateway-Schicht für sub-10ms-Latenz.

### Kernfunktionen

- **Multi-Plattform-Sentiment-Trend-Analyse mit KI-gestützter N-Gramm-Erkennung**  
  Erfasst und klassifiziert automatisch die Stimmung (positiv, neutral, negativ) von Kommentaren und Erwähnungen in Echtzeit. Nutzt ein vortrainiertes BERT-Modell (fine-getuned auf deutschsprachige Creator-Daten), das eine Genauigkeit von 94,2 % bei der Sentiment-Klassifizierung erreicht. Die Analyse erfolgt in sub-50ms pro Post und wird über einen PostgreSQL-Listener mit `LISTEN/NOTIFY` asynchron an die Dashboard-Instanz gestreamt.

- **Dynamische Sichtbarkeits-Korrelationsmatrix (VCM) mit zeitgewichteter Metrik-Normalisierung**  
  Berechnet eine gewichtete Korrelation zwischen 27 verschiedenen Metriken (z. B. `engagement_rate`, `impression_growth_rate`, `share_of_voice`, `follower_churn_rate`) und der globalen Markensichtbarkeit. Die Normalisierung erfolgt über eine benutzerdefinierte SQL-Funktion, die Ausreißer mittels Median Absolute Deviation (MAD) filtert. Ergebnisse werden als JSON-Array mit Zeitstempeln im ISO-8601-Format ausgegeben und können direkt in Tableau oder Power BI geladen werden.

- **Automatisierte Strategie-Empfehlungs-Engine mit Constraint-Lösung auf Basis linearer Programmierung**  
  Optimiert die Content-Strategie durch Lösen eines linearen Optimierungsproblems mit 15+ Variablen (z. B. Posting-Frequenz, Hashtag-Dichte, Video-Länge, Call-to-Action-Position). Die Engine nutzt die `scipy.optimize.linprog`-Bibliothek und gibt einen optimalen Strategieplan als CSV-Datei mit `recommended_actions`, `expected_impact` (als Prozentwert) und `confidence_interval` (95 % Konfidenz) aus.

### So verwenden Sie es

1. **Konfiguration der Datenquellen und Authentifizierung**  
   Erstellen Sie eine `.env`-Datei im Projektstamm und definieren Sie die API-Keys für die gewünschten Plattformen. Verwenden Sie das Format:  
   `PLATFORM_NAME_API_KEY=<Ihr_API_Key>`  
   Führen Sie anschließend `python scripts/validate_credentials.py` aus, um die Verbindung zu testen. Das Skript gibt bei Erfolg `Connection established: <Platform> (latency: <ms>)` aus.

2. **Starten der Echtzeit-Metrik-Erfassung**  
   Starten Sie den Daten-Collector-Dienst mit:  
   `docker-compose up -d collector`  
   Der Dienst öffnet einen WebSocket auf Port 8080 und beginnt, alle 5 Minuten einen Snapshot der aktuellen Metriken zu erfassen. Überwachen Sie den Fortschritt mit:  
   `docker logs -f deshaefrost_collector_1`  
   Die Logs zeigen Einträge wie `[OK] Fetched 1,247 posts from Instagram (latency: 312ms)`.

3. **Generieren eines Strategie-Berichts**  
   Führen Sie das Analyse-Skript aus:  
   `python scripts/generate_report.py --output-format csv --timeframe last_30_days`  
   Das Skript erstellt eine Datei `strategy_report_<timestamp>.csv` im `reports/`-Verzeichnis. Die CSV enthält Spalten wie `metric_name`, `current_value`, `recommended_value`, `expected_impact_percent`, `priority` (1–5). Für eine visuelle Ausgabe verwenden Sie `--output-format html`.

### Beste Praktiken

- **Praxis 1: Datenqualität durch regelmäßige Validierung sicherstellen**  
  Implementieren Sie einen wöchentlichen Cron-Job, der `python scripts/validate_data_integrity.py` ausführt. Dieses Skript prüft auf fehlende Werte, Duplikate und Ausreißer in der PostgreSQL-Datenbank und sendet bei Abweichungen eine Benachrichtigung an Ihren Slack- oder Teams-Webhook. Empfohlen wird ein Schwellenwert von maximal 2 % fehlerhaften Datenpunkten pro Tag.

- **Praxis 2: Nutzung der Cache-Schicht für häufig abgefragte Metriken**  
  Konfigurieren Sie Redis so, dass häufig abgerufene Dashboard-Daten (z. B. `daily_engagement_rate` oder `weekly_follower_growth`) mit einer TTL von 300 Sekunden gecacht werden. Dies reduziert die Last auf die PostgreSQL-Datenbank um bis zu 70 % und verbessert die Ladezeit des Dashboards auf unter 200 ms. Verwenden Sie den Befehl `redis-cli CONFIG SET maxmemory-policy allkeys-lru` für eine effiziente Speicherverwaltung.
