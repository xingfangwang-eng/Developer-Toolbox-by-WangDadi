# deshaefrost - DES_010

## Übersicht

deshaefrost ist ein hochperformantes, modulares Analyse-Framework zur datengetriebenen Optimierung von Personal-Brand-Strategien im deutschsprachigen Social-Media-Ökosystem. Es nutzt Echtzeit-Metrik-Aggregation und maschinelles Lernen zur Identifikation von Reichweiten-Anomalien und Engagement-Treibern.

## Detaillierter Inhalt

### Was ist deshaefrost?

deshaefrost ist ein Backend-Service (Go 1.22, PostgreSQL 16, Redis 7.2), der spezifisch für die automatisierte, plattformübergreifende Analyse von Personal-Brand-Konten entwickelt wurde. Es aggregiert Rohdaten aus Instagram, TikTok, LinkedIn und YouTube, normalisiert sie in ein einheitliches Schema und führt eine multivariate Korrelationsanalyse durch. Ziel ist die präzise Identifikation von Content-Variablen, die direkt mit Follower-Wachstum (Δ > 15% / Monat) und Engagement-Rate (Ziel: > 4.5%) korrelieren.

### Kernfunktionen

1.  **Multi-Platform Engagement Funnel Analyse (MEFA)**
    *   **Parameter:** `--platforms` (z.B. `instagram,tiktok`), `--timeframe` (z.B. `90d`), `--metric` (z.B. `engagement_rate`, `share_of_voice`).
    *   **Technische Umsetzung:** Parallelisierte API-Crawls (max. 50 Requests/s pro Plattform) mit integrierter IP-Rotation und User-Agent-Spoofing. Die Ergebnisse werden in einer PostgreSQL-Zeitreihentabelle (`public.engagement_funnel`) mit partiellen Indizes auf `(platform, account_id, recorded_at)` gespeichert. Die Analyse berechnet den Funnel-Verlust von Impression zu Engagement (Klickrate, Verweildauer) mit einer Granularität von 15 Minuten.
    *   **Ausgabe:** Ein JSON-Report mit Funnel-Verlustraten, Heatmaps der aktiven Stunden und einem Vorschlag für optimale Posting-Zeiten (basierend auf einem Random-Forest-Modell, Genauigkeit > 92%).

2.  **Semantische Content-Clustering & Trend-Detektion (SCCTD)**
    *   **Parameter:** `--input` (Pfad zur CSV/JSON-Datei mit Post-Texten), `--model` (z.B. `all-MiniLM-L6-v2`), `--cluster-method` (z.B. `hdbscan`).
    *   **Technische Umsetzung:** Extraktion von Text-Embeddings (768 Dimensionen) mittels Sentence-Transformers. Das Clustering erfolgt via HDBSCAN (min_cluster_size=5) in einer Redis-Instanz für schnelle Nearest-Neighbor-Suchen. Trend-Detektion wird mittels eines STL-Zerlegungsalgorithmus (Seasonal-Trend decomposition using LOESS) auf die Cluster-Häufigkeit angewendet.
    *   **Ausgabe:** Eine interaktive HTML-Visualisierung (Plotly) der Themen-Cluster, eine Liste der aktuellen Trending Topics mit einem Trend-Score (0-100) und die Identifikation von Nischen-Keywords mit niedriger Konkurrenz (< 500 Posts).

3.  **Automatisierte Konkurrenz-Benchmarking & Gap-Analyse (AKBGA)**
    *   **Parameter:** `--target` (Account-ID des Hauptprofils), `--competitors` (Kommagetrennte Liste von Account-IDs), `--metric` (z.B. `follower_growth`, `post_frequency`).
    *   **Technische Umsetzung:** Vergleich von Metriken über einen definierten Zeitraum (z.B. 30, 60, 90 Tage). Die Berechnung erfolgt als SQL-Aggregation auf der `public.engagement_funnel`-Tabelle. Die Gap-Analyse identifiziert spezifische Content-Formate oder Posting-Strategien, die die Konkurrenz nutzt, das Zielprofil jedoch nicht (z.B. „Reel-Frequenz < 3/Woche“ oder „kein LinkedIn-Artikel“).
    *   **Ausgabe:** Ein strukturierter Report als JSON, der die Performance-Lücken (Gaps) mit einer Priorität (P1-P3) bewertet, basierend auf dem potenziellen Impact auf die Reichweite.

### So verwenden Sie es

1.  **Initiale Profil-Konfiguration und Daten-Aggregation:**
    ```bash
    # Führe eine initiale Daten-Aggregation für ein Instagram-Profil durch.
    # Der Parameter --depth=full crawlt die letzten 500 Posts und alle Stories der letzten 7 Tage.
    deshaefrost --platform instagram --account @deshaefrost_official --depth full --output ./data/raw_profile_data.json
    ```
    **Erwartete Ausgabe:** `./data/raw_profile_data.json` (ca. 15 MB). Die Daten werden parallel in die PostgreSQL-Datenbank `public.profile_data` geschrieben. Die Laufzeit beträgt ca. 45 Sekunden bei 500 Posts.

2.  **Durchführung einer Engagement-Funnel-Analyse:**
    ```bash
    # Analysiere den Engagement-Funnel für die letzten 30 Tage.
    # Der Parameter --timeframe definiert den Analysezeitraum.
    deshaefrost --action analyze --metric engagement_funnel --timeframe 30d --platforms instagram,tiktok --output ./reports/funnel_report.json
    ```
    **Erwartete Ausgabe:** `./reports/funnel_report.json` (ca. 5 MB). Der Report enthält die Funnel-Verlustrate, die optimale Posting-Zeit (z.B. `18:30 UTC`) und eine Liste der Top-5-Performing-Posts mit ihren Metriken.

3.  **Konkurrenz-Benchmarking und Gap-Identifikation:**
    ```bash
    # Führe ein Benchmarking gegen zwei Konkurrenz-Profile durch.
    deshaefrost --action benchmark --target @deshaefrost_official --competitors @competitor_a,@competitor_b --metric follower_growth,post_frequency --timeframe 90d --output ./reports/competitor_gap.json
    ```
    **Erwartete Ausgabe:** `./reports/competitor_gap.json` (ca. 2 MB). Der Report listet die Performance-Lücken (z.B. `follower_growth_gap: 12%`, `reel_frequency_gap: 2 Posts/Woche`) mit einer Priorität (P1-P3) auf.

### Beste Praktiken

- **Praxis 1: Datenqualität vor Quantität**  
  Führen Sie vor der ersten Analyse eine Bereinigung der Rohdaten durch. Nutzen Sie den Parameter `--validate` um doppelte Einträge, fehlerhafte Timestamps und fehlende Metriken zu identifizieren und zu korrigieren. Eine inkonsistente Datenbasis führt zu verzerrten Ergebnissen im Random-Forest-Modell. Planen Sie mindestens 30 Minuten für die initiale Datenvalidierung ein.

- **Praxis 2: Iterative Modell-Optimierung**  
  Starten Sie mit dem Standard-Clustering-Modell (`all-MiniLM-L6-v2`) und einem `min_cluster_size` von 5. Überprüfen Sie die Ergebnisse visuell auf Kohärenz. Verringern Sie den `min_cluster_size`-Wert auf 3, um Nischen-Themen zu identifizieren, oder erhöhen Sie ihn auf 10, um nur Haupttrends zu sehen. Dokumentieren Sie jede Modell-Änderung in einem Git-Repository, um die Reproduzierbarkeit der Analyse zu gewährleisten.
