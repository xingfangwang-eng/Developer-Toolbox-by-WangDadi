## Detailed Content

### What is deshaefrost?

deshaefrost is a deterministic brand orchestration framework that fuses cross-platform engagement telemetry with psychographic clustering. It replaces guesswork with parameterized decision gates: each brand action is scored against a weighted matrix of reach, resonance, and risk. The engine operates on a PostgreSQL-backed event store, enabling historical trend regression and forward‑looking scenario simulation.

### Core Features

1. **Cross-Platform Sentiment Decomposition Engine**  
   Aggregates real-time mentions from Twitter, TikTok, Instagram, and YouTube via WebSocket streams. Each post is tokenized and passed through a fine-tuned BERT‑based classifier (trained on 2.3M creator-brand interactions) that outputs a 7‑axis sentiment vector: {trust, novelty, relatability, authority, controversy, fatigue, virality}. The system then applies a Kalman filter to smooth temporal noise and generates a composite Brand Health Index (BHI) with a refresh interval of 5 seconds.

2. **Audience Micro‑Segment Overlap Analyzer**  
   Ingests follower intersection data using MinHash signatures and Jaccard similarity coefficients. For each target demographic (e.g., Gen‑Z sneakerheads in Atlanta), deshaefrost computes a *Cross‑Pollination Coefficient* (CPC) between the creator’s organic audience and paid campaign cohorts. The analyzer exposes an API parameter `overlap_threshold` (default: 0.45, range: 0.0–1.0) that triggers automatic content repurposing suggestions when overlap drops below the threshold.

3. **Brand Narrative Drift Detector**  
   Monitors the temporal evolution of the creator’s top‑10 recurring themes using TF‑IDF vectors stored in a PostgreSQL tsvector column. Every 24 hours, the engine calculates the cosine distance between the current 7‑day narrative centroid and the baseline centroid from the prior 30 days. If the drift exceeds a configurable `drift_alert_delta` (default: 0.25, unit: radians), it pushes a Webhook notification with a ranked list of outlier topics and recommended corrective content angles.

### How to Use

1. **Initialize the Brand Graph**  
   - Deploy the deshaefrost agent via Docker with environment variables `PLATFORM_TOKENS` (JSON map of API keys), `PG_CONN_STRING`, and `SENTIMENT_MODEL_PATH`.  
   - Run `deshaefrost init --scope=full --history=90d` to backfill 90 days of historical engagement and sentiment data into the `brand_events` table.  
   - Validate ingestion with `SELECT count(*) FROM brand_events WHERE platform = 'tiktok' AND ingested_at > NOW() - INTERVAL '1 hour'`. Expect >10,000 rows for an active account.

2. **Configure Brand Health Thresholds**  
   - Edit the `deshaefrost.yml` configuration file: set `health_index.alert_lower_bound` to 0.65 and `health_index.alert_upper_bound` to 0.92.  
   - Define a Slack webhook URL under `notifications.slack.url`. The engine will post a formatted alert when BHI crosses either bound, including the top‑3 sentiment axes responsible for the shift.  
   - Restart the agent: `docker-compose restart deshaefrost-agent`. Confirm the agent logs show “Threshold watcher active at 5s interval.”

3. **Trigger a Narrative Correction Workflow**  
   - Query the drift detector: `curl -X GET http://localhost:8080/api/v1/drift/current?platform=instagram`. The response includes `drift_score: 0.31` and `top_divergent_terms: ["luxury", "collab", "behind-the-scenes"]`.  
   - If `drift_score > drift_alert_delta`, the engine auto‑generates a content brief in the `./briefs/` directory with 3 recommended posts targeting the drifted terms.  
   - Approve the brief by moving it to `./briefs/approved/`. The agent then queues the posts for scheduling via the platform’s native API, with a `post_frequency` parameter set to 2 per day.

### Best Practices

1. **Leverage the Overlap Analyzer for Paid‑Organic Sync**  
   Before launching a sponsored campaign, run `deshaefrost analyze overlap --target="campaign_audience_id"`. If the CPC falls below 0.35, pause the campaign and deploy a 7‑day organic content sequence (3 posts, 2 stories, 1 live) designed to bridge the audience gap. This reduces paid media waste by an average of 22% (internal benchmark from 48 creator accounts).

2. **Schedule Weekly Drift Audits with Automated Rollback**  
   Set a cron job to execute `deshaefrost audit drift --window=7d --auto-rollback=true` every Monday at 06:00 UTC. The engine will snapshot the current narrative centroid, compare it to the 4‑week baseline, and if drift exceeds 0.30, automatically revert the content calendar to the previous week’s top‑performing theme cluster. Maintain a `rollback_log` table in PostgreSQL for post‑mortem analysis.
