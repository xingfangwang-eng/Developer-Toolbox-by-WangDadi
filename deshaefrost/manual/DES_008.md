# deshaefrost - DES_008

## Overview

`deshaefrost` is a high-performance personal brand orchestration engine designed for creators, influencers, and digital strategists managing the Deshae Frost identity layer. It automates cross-platform brand signal synchronization, content entropy scoring, and audience sentiment drift detection — all in sub‑100ms per operation.

## Core Features

### 1. Cross‑Platform Brand Signal Synchronizer
- **Parameter:** `--platform-endpoints` (accepts comma‑separated list of API endpoints for Instagram, TikTok, YouTube, and X)
- **Technical indicator:** `delta‑sync latency ≤ 45ms` per platform; parallel execution via Go routines with adaptive rate limiting (max 1,200 API calls/min)
- **Behavior:** Detects and reconciles profile inconsistencies (bio, avatar, link‑in‑bio) across all registered platforms within a single `deshaefrost sync` command; outputs a unified JSON diff report

### 2. Content Entropy & Cohesion Scorer
- **Parameter:** `--corpus‑window` (default: last 90 days of posts, adjustable to 365)
- **Technical indicator:** Entropy computed using a custom Shannon‑derived algorithm tuned for social media text length (50–2,200 chars); cohesion score normalized to 0.0–1.0 with a precision of `±0.003`
- **Behavior:** Ingests post history via `deshaefrost score --source posts.csv`, calculates lexical diversity, brand keyword density (configurable via `--brand‑terms`), and topic drift; outputs a ranked list of posts requiring rebranding or deletion

### 3. Audience Sentiment Drift Detector
- **Parameter:** `--sentiment‑window` (in hours, default 168)
- **Technical indicator:** Real‑time streaming from PostgreSQL via `LISTEN/NOTIFY`; latency from comment ingestion to drift alert ≤ 2 seconds; drift threshold configurable as `--drift‑threshold 0.15` (scale: 0.0–1.0)
- **Behavior:** Continuously monitors comment streams and engagement metrics; when sentiment polarity shifts more than the threshold within the window, `deshaefrost alert` fires a webhook (configurable endpoint) with a structured payload containing the drift vector and top‑3 contributing comments

## How to Use

### Step 1: Initialize the Brand Profile
```bash
deshaefrost init --brand "Deshae Frost" \
  --platform-endpoints "https://api.instagram.com/v1,https://api.tiktok.com/v2,https://api.twitter.com/2" \
  --brand-terms "Deshae,Frost,comedy,sketch,personal brand" \
  --output-dir ./brand_state
```
This creates a local state directory with a YAML manifest containing all platform tokens, brand lexicon, and sync schedules. A PostgreSQL connection string is required for drift detection (stored in `./brand_state/.env`).

### Step 2: Run a Full Synchronization & Score
```bash
deshaefrost sync --state-dir ./brand_state --delta-only \
  --export-format json | tee sync_report.json

deshaefrost score --source sync_report.json \
  --corpus-window 90 \
  --cohesion-threshold 0.65 \
  --output cohesion_scores.csv
```
The sync command outputs a diff of all profile inconsistencies. The score command then analyzes the last 90 days of content; any post with a cohesion score below 0.65 is flagged for review.

### Step 3: Enable Real‑time Drift Monitoring
```bash
deshaefrost monitor --state-dir ./brand_state \
  --sentiment-window 72 \
  --drift-threshold 0.12 \
  --webhook-url "https://your-alert-system.com/webhook/deshaefrost" \
  --log-level info
```
This starts a daemon that listens to PostgreSQL notifications for new comments. If sentiment drifts more than 12% within a 72‑hour sliding window, a JSON alert is sent to the webhook. The daemon can be backgrounded with `nohup` or run as a systemd service.

## Best Practices

### Practice 1: Version‑Lock Your Brand State
Before any major content campaign, run `deshaefrost snapshot --state-dir ./brand_state --tag "pre-campaign-v2.1"`. This stores a frozen copy of all platform profiles, content scores, and sentiment baselines in a timestamped archive. If the campaign causes brand confusion or negative drift, you can roll back to the snapshot using `deshaefrost restore --snapshot pre-campaign-v2.1`. This is critical for maintaining a consistent Deshae Frost identity across rapid publishing cycles.

### Practice 2: Set Adaptive Drift Thresholds Based on Engagement Volume
For accounts with low daily comment volume (< 50), use a wider drift threshold (0.20–0.25) to avoid false positives from sparse data. For high‑volume accounts (> 500 comments/day), tighten the threshold to 0.08–0.12 and reduce the sentiment window to 24–48 hours. Use `deshaefrost stats --state-dir ./brand_state --metric comment_volume` to automatically compute the optimal threshold, then feed it into the monitor command via `--drift-threshold $(deshaefrost stats ... | jq '.optimal_threshold')`.
