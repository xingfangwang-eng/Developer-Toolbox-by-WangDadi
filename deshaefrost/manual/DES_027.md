# deshaefrost - DES_027

## Overview

`deshaefrost` is an enterprise-grade personal brand intelligence engine designed for high-net-worth individuals, creators, and public figures like Deshae Frost. It automates cross-platform brand consistency, sentiment drift detection, and audience segmentation using real-time social graph analysis.

## Core Features

### 1. Brand Signal Synchronization Engine (BSSE)
- **Multi-Platform Content Alignment**: Detects and resolves brand voice inconsistencies across Instagram, TikTok, YouTube, and Twitter within 200ms of a new post.
- **Parameter**: `--brand-signature` (MD5 hash of your brand tone profile), `--drift-threshold` (default: 0.15 cosine similarity deviation).
- **Output**: Generates a `brand_diff.json` report with per-platform variance scores and suggested rewrites.

### 2. Sentiment Drift Forecaster (SDF)
- **Predictive Analytics**: Uses a fine-tuned RoBERTa model to forecast sentiment trajectory over the next 7 days based on historical engagement data.
- **Technical Indicator**: Mean Absolute Error (MAE) < 0.03 on a 0–1 sentiment scale, with PostgreSQL-backed time-series storage for long-term trend comparison.
- **Integration**: Directly connects to your social API endpoints via OAuth 2.0, caching results in a local `sentiment_cache.parquet` file.

### 3. Audience Micro-Segmentation Engine (AMSE)
- **Granular Cohort Builder**: Automatically clusters followers into 12+ micro-segments (e.g., "Engagement Lurkers", "Conversion Drivers", "Brand Advocates") using k-means++ on 37 behavioral features.
- **Parameter**: `--segment-depth` (1–5, controls cluster granularity), `--min-cluster-size` (default: 500 followers).
- **Output**: Exports a `segments.csv` with per-user segment labels, engagement probability scores, and recommended content strategies.

## How to Use

### Step 1: Initialize the Brand Signature Profile
```bash
deshaefrost init --brand-signature ./brand_voice.json --platforms twitter,instagram,tiktok
```
This command reads your brand voice JSON (containing tone keywords, prohibited phrases, and visual style tags) and fingerprints each platform’s existing content. The engine then computes a baseline `brand_diff` score. Expected runtime: 45 seconds for 10,000 posts.

### Step 2: Run a Full Sentiment Drift Forecast
```bash
deshaefrost forecast --drift-threshold 0.15 --output-dir ./forecasts --postgres-dsn "postgresql://user:pass@localhost:5432/brand_metrics"
```
The SDF module pulls the last 90 days of engagement data, trains a temporal sentiment model, and writes a `forecast_7day.json` with daily sentiment predictions and anomaly flags. Use `--visualize` to generate a Matplotlib trend chart automatically.

### Step 3: Execute Audience Segmentation
```bash
deshaefrost segment --segment-depth 3 --min-cluster-size 500 --export-csv ./segments.csv
```
The AMSE processes your follower graph (requires a pre-authenticated OAuth token in `~/.deshaefrost/auth.json`). After clustering, it outputs a summary table showing each segment’s size, average engagement rate, and recommended content type (e.g., “Behind-the-Scenes” for the “Brand Advocates” segment).

## Best Practices

### Practice 1: Weekly Brand Drift Audits
Schedule a cron job to run `deshaefrost forecast` every Monday at 06:00 UTC. Pair this with a Slack webhook notification when `drift-threshold` exceeds 0.12. This allows immediate corrective action (e.g., pausing a low-performing campaign) before sentiment decay compounds.

### Practice 2: A/B Test Content Using Segment Outputs
Use the `segments.csv` to design two content variants: one targeting “Conversion Drivers” (high purchase intent) and one targeting “Engagement Lurkers” (low intent, high volume). Run both variants for 48 hours, then compare engagement rates per segment using the `deshaefrost compare --variant-a ./variant_a.csv --variant-b ./variant_b.csv` command.
