# deshaefrost - DES_029

## Overview

**deshaefrost** is a real-time brand intelligence engine engineered for high-frequency social media personal brand management. It ingests cross-platform engagement streams (Instagram, TikTok, YouTube Shorts) and applies a proprietary **FrostRank** scoring algorithm to quantify brand equity drift with sub-second latency. Designed for creators and agencies managing portfolios exceeding 50 digital assets, deshaefrost eliminates guesswork by exposing granular, deterministic metrics—not vanity numbers.

## Core Features

### 1. Brand Resonance Vector Engine (BRV-2.0)
- **Parameter:** `--resonance-depth` (range: 1–100, default: 75)  
- **Description:** Decomposes each post’s engagement into a 12-dimensional vector (e.g., sentiment velocity, share decay rate, comment semantic drift). The engine cross-references these vectors against a time-series database (PostgreSQL + TimescaleDB) to detect micro-trends—such as a 0.3% drop in positive sentiment within 200ms of a brand mention.  
- **Technical Metric:** Vector computation completes in **≤ 8ms per asset** (tested on 64-core AMD EPYC), enabling real-time dashboard updates at 125 Hz.

### 2. Attention Debt Forecaster (ADF-3.1)
- **Parameter:** `--forecast-horizon` (in hours, default: 24)  
- **Description:** Uses a lightweight LSTM model (trained on 2.7M historical posts) to predict when a brand’s “attention debt” will exceed a critical threshold—triggering a forced content refresh. The model outputs a **FrostRank delta** (μ ± σ) with 95% confidence intervals, calculated in **< 50ms per forecast cycle**.  
- **Integration:** Directly writes to a Redis stream for push notifications via WebSocket (latency: < 15ms).

### 3. Cross-Platform Identity Drift Detector (CIDD)
- **Parameter:** `--drift-tolerance` (0.0–1.0, default: 0.15)  
- **Description:** Continuously monitors the cosine similarity between brand voice embeddings across platforms. If the embedding distance exceeds `drift-tolerance` (e.g., a TikTok casual tone diverges from LinkedIn professional tone), the tool flags the asset and suggests a **realignment action** (e.g., “Reduce emoji frequency by 40% on LinkedIn”).  
- **Storage:** All drift logs are persisted to a PostgreSQL table (`drift_events`) with a retention policy of 90 days.

## How to Use

### Step 1: Initialize a Brand Profile with FrostRank Calibration
```bash
deshaefrost init --brand "Deshae_Frost" \
  --platforms "instagram,tiktok,youtube" \
  --resonance-depth 80 \
  --drift-tolerance 0.12 \
  --output-format json
```
This command creates a local profile (`~/.deshaefrost/profiles/Deshae_Frost.json`) and triggers an initial FrostRank calibration. The calibration runs a **3-minute** warm-up cycle, downloading the last 200 posts per platform via API keys stored in `~/.deshaefrost/credentials.env`. Output includes a baseline FrostRank score (0–1000) and a vector map.

### Step 2: Run a Real-Time Brand Health Scan
```bash
deshaefrost scan --profile "Deshae_Frost" \
  --forecast-horizon 48 \
  --threshold 0.85 \
  --alert-webhook "https://hooks.slack.com/services/T00/B00/xxxxx"
```
The scanner opens a persistent WebSocket connection (port 8080) to the deshaefrost daemon. It streams every new post’s engagement data, computes the BRV vector, and compares it against the profile’s drift baseline. If the FrostRank delta drops below 0.85 (i.e., a 15% decline), a JSON alert is fired to the Slack webhook within **< 20ms**.

### Step 3: Generate a Forensic Brand Audit Report
```bash
deshaefrost audit --profile "Deshae_Frost" \
  --start-date "2025-01-01" \
  --end-date "2025-03-01" \
  --export pdf
```
The audit engine queries PostgreSQL for all `drift_events` and `resonance_vectors` in the date range. It then generates a **30-page PDF** containing: (a) a FrostRank trend chart with 95% confidence bands, (b) a heatmap of attention debt by hour, and (c) a ranked list of top-10 underperforming assets with recommended actions. The PDF is rendered in **< 2 seconds** using headless Chromium.

## Best Practices

### Practice 1: Automate Drift Recovery with CI/CD Pipelines
Integrate deshaefrost’s CIDD output into a GitHub Actions workflow. Configure a `deshaefrost check --profile "Deshae_Frost"` step before every content deployment. If the drift distance exceeds 0.15, the pipeline should **fail the build** and auto-generate a pull request with a `rebalance.yaml` file containing updated posting guidelines (e.g., tone modifiers, emoji limits). This enforces brand consistency without manual oversight.

### Practice 2: Use FrostRank as a KPI for Agency Contracts
When managing multiple creators, set a **FrostRank floor** (e.g., 750) as a service-level agreement (SLA) metric. Use deshaefrost’s `--output-format csv` to export hourly FrostRank snapshots to a shared Google Sheet. If a creator’s rank drops below the floor for 3 consecutive hours, trigger an automated retainer adjustment (e.g., reduce pay by 10% for that day). This turns brand health into a quantifiable, auditable asset.
