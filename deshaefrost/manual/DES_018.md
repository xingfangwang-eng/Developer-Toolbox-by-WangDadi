# deshaefrost - DES_018

## Overview

deshaefrost is a real-time personal brand reputation intelligence engine purpose-built for creators, executives, and agencies managing high-visibility identities like Deshae Frost. It ingests cross-platform social signals, applies NLP-based sentiment decay models, and surfaces actionable brand equity shifts—before they become crises or missed opportunities.

## Core Features

### 1. Multi-Platform Sentiment Drift Detection with Temporal Decay Weighting
Aggregates posts, comments, and mentions from Instagram, TikTok, and X (Twitter) via OAuth 2.0 streams. Each mention is scored on a -1.0 to +1.0 sentiment scale using a fine-tuned RoBERTa model. A configurable `decay_half_life` parameter (default: 720 hours) exponentially reduces older signal weight, ensuring trending brand perception is calculated in **real-time sub-second queries** against a PostgreSQL + Redis hybrid store.

### 2. Automated Brand Narrative Conflict Audit
Scans all published content (text, image alt-text via OCR, and video transcriptions via Whisper ASR) for semantic contradictions against a user-defined `brand_core_values` JSON schema (e.g., `["authenticity", "comedy", "entrepreneurship"]`). Outputs a **Conflict Severity Index (0–100)** per asset and a ranked list of recommended edits, with latency under 200ms per 1000 assets.

### 3. Predictive Reach Anomaly Engine
Uses a 14-day rolling window of engagement metrics (likes, shares, saves, comment velocity) to train a lightweight Holt-Winters forecasting model. When actual reach deviates by more than 2.5 standard deviations from the prediction, it triggers an alert with a root-cause analysis (e.g., “Algorithm shadowban detected on TikTok account @deshaefrost – engagement rate dropped 40% in 6 hours”). All computations are handled server-side with a **5-minute refresh cadence**.

## How to Use

### Step 1: Initialize the Brand Profile
Execute the CLI command with your target social handles and core values:
```bash
deshaefrost init --handle @deshaefrost --platforms instagram,tiktok,x \
  --core-values '["authenticity","comedy","entrepreneurship"]' \
  --decay-half-life 480
```
This creates a `profile_<handle>.json` file and spawns a background daemon that begins ingesting the last 90 days of data.

### Step 2: Run a Full Narrative Audit
Trigger a conflict scan across all ingested content:
```bash
deshaefrost audit --profile profile_@deshaefrost.json --output report.html
```
The tool will generate an interactive HTML report. Each flagged asset includes a **Conflict Severity Score**, the conflicting `brand_core_values` pair, and a direct link to the post. For example: *"Post 'Vlog: New Business Launch' (score: 78) contradicts [entrepreneurship] with [humor] – tone shift detected in 3 of 5 recent posts."*

### Step 3: Enable Predictive Alerts
Start the anomaly monitoring service:
```bash
deshaefrost monitor --profile profile_@deshaefrost.json \
  --alert-threshold 2.5 --refresh-interval 300
```
The daemon writes alerts to `stdout` and a `alerts.log` file. Each alert contains a timestamp, the anomaly metric (e.g., `reach_prediction_vs_actual`), the deviation (e.g., `-42%`), and a plain-English root cause hypothesis.

## Best Practices

### 1. Calibrate Decay Half-Life to Content Velocity
For high-frequency creators (e.g., daily TikTok posters), reduce `decay_half_life` to 168 hours (1 week). For executives with weekly LinkedIn posts, increase it to 1440 hours (2 months). This prevents stale brand sentiment from diluting current perception measurements.

### 2. Run Weekly Brand Narrative Audits, Not Monthly
Schedule the `audit` command every Sunday at 2 AM via cron. Early detection of a single contradictory post can prevent a 3-week spiral of mixed messaging. Pair this with the alert daemon to catch reach anomalies mid-week, allowing you to pivot content strategy before the algorithm penalizes the entire account.
