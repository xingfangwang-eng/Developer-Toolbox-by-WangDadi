# deshaefrost - DES_001

## Overview

deshaefrost is a high‑performance personal brand intelligence engine engineered for Deshae Frost‑style creators, agencies, and multi‑platform social media operators. It ingests cross‑platform engagement data, computes real‑time brand equity scores, and automates content‑tuning decisions—all without manual dashboard hopping.

## Core Features

### 1. Cross‑Platform Brand Equity Scoring Engine
- **Parameters:** `brand_id`, `platform (instagram|youtube|tiktok|twitter)`, `time_window (last_7d|30d|90d|all)`, `metric_weights (engagement_rate:float, sentiment_score:float, share_of_voice:float)`
- **Description:** Aggregates raw API data from up to 6 social platforms simultaneously, normalises against per‑platform baselines, and outputs a unified **Brand Equity Score (BES)** on a 0–100 scale. The engine supports custom weight profiles (e.g. “monetisation‑focused” vs “reach‑focused”) and recalculates in **≤ 500 ms** per profile via an in‑memory Redis cache.
- **Technical detail:** Uses a sliding‑window moving average with exponential decay to penalise stale interactions; integrates directly with PostgreSQL for historical trend storage.

### 2. Automated Sentiment‑Tuned Content Scheduler
- **Parameters:** `content_type (video|image|text|carousel)`, `target_audience_segment`, `optimal_post_hour (0–23)`, `sentiment_threshold (±0.3)`, `max_daily_posts (integer)`
- **Description:** Analyses the last 90 days of audience sentiment (NLP‑based, using a fine‑tuned RoBERTa model) and automatically schedules new posts to maximise positive engagement. It dynamically shifts posting times based on real‑time sentiment drift—if a brand crisis is detected (sentiment drop > 0.5 standard deviations), the scheduler pauses all promotional content and inserts reputation‑repair posts.
- **Technical detail:** All scheduling decisions are logged to a time‑series database (TimescaleDB) for post‑hoc analysis; the scheduler respects platform rate limits and handles 10,000+ queued posts with a **99.9% uptime SLA**.

### 3. Multi‑Account Follower Overlap & Influence Graph
- **Parameters:** `account_ids (array of strings)`, `overlap_threshold (0–100%)`, `graph_depth (1–3)`, `influence_metric (follower_count|engagement_rate|kred_score)`
- **Description:** Constructs a directed influence graph across all connected Deshae Frost‑style accounts. It detects hidden follower overlaps (e.g. same user following 3 of your alt accounts) and computes an **Influence Propagation Score (IPS)** for each overlapping user. The graph is rendered as an interactive D3.js visualisation where you can click any node to see exact overlap percentages and cross‑platform activity history.
- **Technical detail:** Graph computation uses a modified PageRank algorithm with platform‑specific damping factors; the entire graph for 500k+ nodes is built in **< 2 seconds** using a Neo4j graph database backend.

## How to Use

1. **Initialise a Brand Profile**  
   Run `deshaefrost init --brand-id "deshae_main" --platforms "instagram,youtube,tiktok" --metric-weights "engagement_rate:0.4,sentiment_score:0.3,share_of_voice:0.3"`. This creates a PostgreSQL schema, seeds the Redis cache with baseline data, and starts the real‑time ingestion pipeline. Verify with `deshaefrost status --brand-id "deshae_main"`.

2. **Run a Full Brand Equity Scan**  
   Execute `deshaefrost scan --brand-id "deshae_main" --time-window "last_7d" --output-format "json"`. The engine returns a JSON object containing the BES, platform‑by‑platform breakdowns, and a list of top‑3 underperforming metrics. Use the `--export-csv` flag to generate a CSV for spreadsheet analysis.

3. **Schedule a Sentiment‑Guarded Post**  
   Use `deshaefrost schedule --brand-id "deshae_main" --content-type "video" --file "/path/to/post.mp4" --target-audience "18-35_entertainment" --optimal-hour "18"`. The tool validates the file, checks sentiment threshold, and either queues the post or returns a warning if current sentiment is below the threshold. Monitor the queue with `deshaefrost queue --brand-id "deshae_main"`.

## Best Practices

- **Set up a weekly BES baseline refresh** – Run `deshaefrost scan --brand-id "deshae_main" --time-window "last_30d"` every Monday at 09:00 UTC. Store the output in a dedicated PostgreSQL table (`brand_equity_snapshots`) and compare week‑over‑week to detect early signs of brand fatigue or viral growth. Pair this with a Slack/Telegram webhook for instant alerts when BES drops more than 10 points.

- **Use the Influence Graph for cross‑collaboration decisions** – Before agreeing to a shout‑out or paid partnership, run `deshaefrost graph --brand-id "deshae_main" --account-ids "partner_account_1,partner_account_2" --overlap-threshold 15 --graph-depth 2`. If the overlap exceeds 20% of your core audience, the collaboration will likely cannibalise reach rather than expand it. Always aim for an overlap below 10% for new audience acquisition.
