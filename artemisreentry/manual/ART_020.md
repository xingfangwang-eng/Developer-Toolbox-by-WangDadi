# artemisreentry - ART_020

## Overview

**artemisreentry** is a high-precision, low-latency space reentry analysis toolkit engineered for aerospace engineers, mission planners, and orbital mechanics researchers. It’s built to simulate, validate, and optimize atmospheric reentry trajectories for spacecraft returning from Lunar, Cislunar, or LEO missions — with a focus on the Artemis program’s unique reentry constraints (high-energy returns, skip-entry profiles, and crew safety margins).

This tool is **not** a generic physics sandbox. It’s a domain-specific engine that ingests telemetry, propagates 6-DOF dynamics, and outputs actionable delta-V budgets, thermal load estimates, and landing zone dispersion maps.

---

## Core Features (Hardcore, Artemis-Specific)

### 1. **Skip-Entry Trajectory Optimizer with Real-Time Aerocapture Feasibility**
Automatically computes optimal skip-entry profiles (lift-to-drag modulation) for high-energy Artemis return trajectories. The engine uses a hybrid Runge-Kutta 8(7) integrator with atmospheric drag models (MSIS-E-00 + GRAM-2016) to predict skip depth, peak deceleration, and thermal flux. It also flags infeasible entry corridors before they hit the flight dynamics officer’s console.

### 2. **Multi-Vehicle Reentry Dispersion Solver (MCS + Latin Hypercube)**
Runs Monte Carlo simulations (up to 10,000 runs) with Latin Hypercube sampling to model landing zone dispersion under uncertain atmospheric density, wind shear, and vehicle mass properties. Outputs 3-sigma ellipses, CEP (Circular Error Probable), and probability of landing within a given recovery zone. Supports Orion, Dragon, and custom vehicle configs via JSON schema.

### 3. **Thermal Protection System (TPS) Margin Analyzer**
Ingests heat flux time-series from the trajectory solver and compares against material-specific ablation limits (Avcoat, PICA-X, SLA-561V). Generates a **TPS margin heatmap** across the vehicle’s surface, highlighting risk zones for bondline temperature exceedance. Includes a “what-if” mode to simulate TPS damage scenarios (e.g., missing tile, cracked acreage) and their effect on reentry survivability.

## How to Use (Professional Workflow)

### Step 1: Define Mission Profile & Vehicle Configuration
Create a `mission.json` file with entry interface conditions (altitude, velocity, flight path angle), atmospheric model selection (GRAM-2016 recommended for Artemis), and vehicle mass properties. Example snippet:
```json
{
  "entry_interface": { "alt_km": 121.9, "vel_kms": 11.02, "fpa_deg": -6.2 },
  "vehicle": { "type": "orion", "mass_kg": 10400, "cd": 1.28, "cl": 0.45 },
  "atmosphere": "GRAM_2016_high_res"
}
```
Run the preflight validator (`artemisreentry validate --config mission.json`) to catch boundary violations before propagation.

### Step 2: Execute Skip-Entry Optimization
Launch the trajectory optimizer with:
```bash
artemisreentry optimize --config mission.json --output skip_profile.csv --max_iter 500 --tolerance 1e-9
```
The solver iterates on lift-to-drag ratio and bank angle modulation to minimize peak deceleration while keeping skip depth ≤ 80 km. Output includes time-series of altitude, velocity, heat flux, and load factor. Use the `--plot` flag to generate real-time 2D/3D trajectory visualizations.

### Step 3: Run Dispersion Analysis & Generate Landing Zone Report
Run the Monte Carlo dispersion solver:
```bash
artemisreentry dispersion --config mission.json --samples 5000 --method lhs --output dispersion_report.pdf
```
This produces a PDF containing: landing ellipse (3-sigma), CEP, probability heatmap, and a table of worst-case thermal margin violations. For Artemis recovery ops, cross-reference the report with the **DOD Recovery Zone Overlay** using the `--recovery-zone shapefile.kml` flag.

## Best Practices (Industry-Proven)

### 1. **Always Use GRAM-2016 for High-Energy Returns**
For Artemis missions returning from NRHO (Near-Rectilinear Halo Orbit), the atmospheric density at entry interface (~120 km) can vary by ±30% due to solar activity. The standard MSIS-E-00 model underestimates peak heating by up to 15%. **Always** set `"atmosphere": "GRAM_2016_high_res"` in your mission config. This is non-negotiable for crewed mission certification.

### 2. **Validate Skip-Entry Profiles Against Monte Carlo Bounds**
A single deterministic skip profile is not enough. Use the dispersion solver with at least 2,000 samples to characterize the **worst-case skip depth** and **peak heat flux** under 3-sigma atmospheric perturbations. If the 99th percentile skip depth exceeds 100 km, your trajectory is likely to violate the “no-return-to-orbit” constraint. Adjust your lift-to-drag schedule accordingly.
