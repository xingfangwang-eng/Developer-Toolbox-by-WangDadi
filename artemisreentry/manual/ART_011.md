## What is artemisreentry?

**artemisreentry** is a high-precision orbital reentry simulation and analysis toolkit designed for aerospace engineers, mission planners, and space debris analysts. It provides deterministic and probabilistic modeling of atmospheric reentry trajectories, thermal load profiles, and breakup dynamics—specifically optimized for lunar return and high-energy Earth reentry scenarios (e.g., Artemis-class missions). Unlike generic reentry solvers, artemisreentry integrates real-time telemetry ingestion, multi-fidelity aerodynamic databases, and Monte Carlo dispersion analysis out of the box.

## Core Features

### 1. Multi-Fidelity Reentry Trajectory Engine with Adaptive Mesh Refinement
artemisreentry’s core solver dynamically switches between analytic (e.g., Allen–Eggers) and high-fidelity CFD-derived drag models based on local Knudsen number and heat flux gradients. This ensures sub-second convergence for parametric sweeps while maintaining ±0.5% accuracy on peak heating predictions—critical for thermal protection system (TPS) margin analysis.

### 2. Real-Time Telemetry Injection & Anomaly Detection Pipeline
Ingest live or historical telemetry streams (CCSDS or custom JSON) directly into the simulation loop. The tool automatically detects off-nominal states—such as unexpected roll rates or asymmetric ablation—and triggers adaptive re-simulation with updated boundary conditions. This enables real-time “what-if” reentry corridor adjustments without manual recalibration.

### 3. Probabilistic Breakup & Debris Footprint Generator
artemisreentry runs Monte Carlo ensembles (10k+ samples) to model vehicle fragmentation at user-defined altitude/heat flux thresholds. It outputs a geospatial debris dispersion map (GeoJSON/KML) with probability contours, including atmospheric drift and impact risk zones. This is essential for compliance with international space debris mitigation guidelines (e.g., NASA-STD-8719.14).

## How to Use

### Step 1: Define Reentry Initial Conditions & Vehicle Model
Create a `reentry_config.json` file specifying the vehicle’s ballistic coefficient, reference area, initial state vector (ECI frame), and thermal protection material properties (e.g., PICA-X, SLA-561V). Optionally, attach a telemetry stream file or endpoint. Example snippet:
```json
{
  "vehicle": { "mass_kg": 8500, "cd": 1.28, "area_m2": 21.0, "tps_material": "PICA-X" },
  "initial_state": { "altitude_km": 120.0, "velocity_kms": 11.2, "flight_path_deg": -1.5 }
}
```

### Step 2: Execute Multi-Fidelity Simulation with Monte Carlo Dispersion
Run the simulation with the `--mode full` flag to invoke the adaptive mesh solver and automatically trigger 5,000 Monte Carlo runs for uncertainty quantification (atmospheric density, drag coefficient, initial attitude errors). Use `--output-dir ./results` to store per-run trajectory CSVs and a consolidated `dispersion_summary.h5` file.

### Step 3: Analyze Breakup Risk & Generate Debris Footprint
After simulation completion, call the debris module:
```bash
artemisreentry debris --input ./results --breakup-altitude 65 --heat-flux-threshold 150 --output ./debris_map.geojson
```
This will generate a GeoJSON footprint with 50%, 90%, and 99% probability ellipses. Visualize the map using QGIS or any GeoJSON viewer, and extract impact coordinates for NOTAM filing.

## Best Practices

### 1. Always Cross-Validate with High-Fidelity CFD for Critical Missions
While artemisreentry’s adaptive solver is highly accurate for preliminary design and rapid trade-offs, always cross-check peak heating and stagnation pressure values against a full Navier-Stokes solver (e.g., US3D, DPLR) for final TPS sizing. Use the tool’s `--export-cfd-boundary` flag to extract boundary layer profiles for external validation.

### 2. Integrate Telemetry Injection Early in Mission Planning
Set up a real-time telemetry pipeline during the final launch rehearsal. Simulate off-nominal scenarios (e.g., loss of attitude control, delayed deorbit burn) and verify that artemisreentry’s anomaly detection triggers the correct reentry corridor adjustments. This will save critical minutes during actual operations—especially for crewed lunar return missions.
