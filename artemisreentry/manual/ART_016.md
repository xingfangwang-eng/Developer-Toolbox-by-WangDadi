## What is artemisreentry?

artemisreentry is a precision engineering toolkit designed for aerospace engineers and mission planners who need to simulate, validate, and optimize spacecraft reentry trajectories. It bridges the gap between raw orbital mechanics and actionable mission data, providing real-time atmospheric drag modeling, thermal load estimation, and landing zone prediction.

---

## Core Features

### 1. Multi-Phase Reentry Trajectory Engine
Unlike basic propagators, artemisreentry uses a hybrid numerical integrator that handles exoatmospheric coast, atmospheric entry, and terminal descent phases with adaptive step sizing. It accounts for variable ballistic coefficients, atmospheric density gradients (via NRLMSISE-00), and Earth rotation effects. This allows you to simulate skip reentry, guided lifting entry, or ballistic descent with sub-kilometer accuracy.

### 2. Real-Time Thermal & Structural Coupling
The tool integrates a lumped-capacitance thermal model with a finite-element mesh for critical heat shield nodes. You can define material ablation rates, TPS thickness, and bondline temperature limits. During simulation, artemisreentry outputs heat flux, surface temperature, and recession depth at each timestep, enabling you to validate against flight data or adjust margins before hardware testing.

### 3. Landing Zone Probability Contour Generator
Instead of a single impact point, artemisreentry runs Monte Carlo ensembles (1000+ iterations) with randomized initial state errors, wind profiles, and parachute deployment uncertainties. It outputs 2D probability density maps and 99.7% confidence ellipses, directly usable in mission planning and range safety documentation.

## How to Use

### 1. Define Your Reentry Profile in JSON
Create a `reentry_config.json` file with your spacecraft parameters:

```json
{
  "vehicle": {
    "mass": 8500,
    "reference_area": 12.6,
    "ballistic_coefficient": 220,
    "heat_shield_material": "PICA",
    "thickness": 0.045
  },
  "initial_state": {
    "altitude_km": 120.0,
    "velocity_kms": 7.8,
    "flight_path_deg": -1.5,
    "latitude_deg": 35.0,
    "longitude_deg": -120.0
  },
  "atmosphere": {
    "model": "NRLMSISE-00",
    "f107": 150,
    "ap": 15
  }
}
```

Then run:
```bash
artemisreentry --config reentry_config.json --output results/
```

### 2. Execute a Monte Carlo Landing Zone Analysis
Generate 2000 perturbed trajectories with realistic error sources:

```bash
artemisreentry --mc-samples 2000 \
  --uncertainty wind:5% \
  --uncertainty bc:3% \
  --uncertainty deploy_alt:2% \
  --output landing_zone_ensemble.nc
```

The tool will output a NetCDF file containing all trajectory data, plus a `landing_zone_contours.geojson` file for GIS integration.

### 3. Validate Against Flight Telemetry
If you have recorded flight data (CSV with time, altitude, velocity, heat flux), run a comparison:

```bash
artemisreentry --compare flight_data.csv \
  --config reentry_config.json \
  --metrics altitude velocity heat_flux \
  --plot residuals_comparison.png
```

This generates residual plots and a summary table with RMSE, max error, and correlation coefficient for each metric. Use this to calibrate your model or document validation for a design review.

## Best Practices

### 1. Always Run Sensitivity Analysis Before Full Monte Carlo
Start with a deterministic run, then vary one parameter at a time (e.g., ballistic coefficient ±5%, entry angle ±0.1°). This reveals which inputs dominate landing zone spread. You can then focus Monte Carlo sampling on the most sensitive parameters, reducing computational cost while preserving accuracy.

### 2. Use Atmospheric Model Selection Based on Mission Phase
For exoatmospheric coast, the NRLMSISE-00 model is sufficient. But for terminal descent below 30 km, switch to a High-Resolution Ensemble Forecast (HREF) or rawinsonde data if available. artemisreentry supports loading custom atmospheric profiles via `--atmo-profile custom_profile.csv`. This dramatically improves landing zone prediction in mountainous or coastal regions where local wind shear is significant.
