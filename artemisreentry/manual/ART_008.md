# artemisreentry - ART_008

## Overview

This is a precision-engineered flight dynamics toolkit for analyzing Artemis-class reentry trajectories and space return scenarios. The project provides deterministic modeling of atmospheric reentry, thermal protection system (TPS) margins, and abort-to-orbit (ATO) decision logic.

## What is artemisreentry?

`artemisreentry` is a command-line and API-driven framework that simulates ballistic and lifting reentry profiles for crewed and cargo return vehicles. It allows engineers to compute heat flux, deceleration loads, and landing dispersion ellipses under real-time mission constraints.

## Hardcore Features

### 1. Multi-Phase Reentry State Machine with Abort Node Injection
The tool models reentry as a discrete-event state machine (entry interface, blackout, drogue deploy, main chute, splashdown). Engineers can inject abort nodes at any phase—triggering a simulated ATO burn or skip-reentry logic—and observe the resulting trajectory perturbations in 6-DOF.

### 2. Real-Time TPS Margin vs. Ballistic Coefficient Solver
Unlike generic reentry calculators, this solver dynamically computes the required TPS thickness (e.g., PICA-X, SLA-561V) based on real-time stagnation point heat flux and shear stress. It outputs margin ratios against user-defined safety factors, making it ideal for trade studies on vehicle mass vs. thermal protection.

### 3. Landing Dispersion Ellipse Generator with Monte Carlo Overlay
Using a configurable covariance matrix for IMU noise, atmospheric density perturbations, and wind shear, the tool runs 10,000+ Monte Carlo runs per scenario. It outputs a 99.7% landing dispersion ellipse (3-sigma) overlaid on a WGS84 map, including probability density heatmaps for recovery zone planning.

## How to Use

### Step 1: Configure a Reentry Scenario via YAML
Create a `reentry_config.yaml` file defining the vehicle’s initial state vector (position, velocity, mass, drag area), atmospheric model (US Standard 1976 or NRLMSISE-00), and TPS material parameters. Use the CLI to validate the config schema:

```bash
artemisreentry validate --config reentry_config.yaml
```

The tool will print any schema violations (e.g., missing `heat_capacity` field) and suggest default values based on the selected TPS material.

### Step 2: Run a Deterministic Trajectory Simulation
Execute a single-pass simulation to generate key metrics:

```bash
artemisreentry simulate --config reentry_config.yaml --output trajectory.json
```

This outputs a JSON file with time-stamped entries for altitude, velocity, heat flux (W/cm²), dynamic pressure (Pa), and deceleration (g). Use the `--plot` flag to generate a Matplotlib figure showing heat flux vs. altitude with annotated blackout and drogue deployment zones.

### Step 3: Generate Monte Carlo Dispersion Analysis
To account for uncertainties, run the Monte Carlo engine:

```bash
artemisreentry monte-carlo --config reentry_config.yaml --runs 10000 --seed 42 --output dispersion.kml
```

The tool exports a KML file containing the 3-sigma ellipse and probability heatmap. Open it in Google Earth or QGIS to overlay on recovery zone maps. Use the `--summary` flag to print the centroid coordinates and ellipse semi-major/minor axes in meters.

## Best Practices

### Practice 1: Validate TPS Margins at Two Extreme Entry Angles
Always run the simulation with both a shallow entry angle (e.g., -5.5°) and a steep entry angle (e.g., -8.2°) to bound the heat flux envelope. The tool’s TPS margin solver will flag if the margin drops below 1.5x for any phase—this is a common failure point in preliminary design reviews.

### Practice 2: Use Abort Node Injection for Contingency Planning
Before finalizing a mission profile, inject an abort node at the worst-case blackout altitude (typically 60–70 km). Set the abort delta-V to 250 m/s (standard Orion ATO burn). The tool will compute the new landing ellipse and compare it to the nominal splashdown zone. If the ellipse overlaps with land mass or shipping lanes, adjust the abort trigger conditions.
