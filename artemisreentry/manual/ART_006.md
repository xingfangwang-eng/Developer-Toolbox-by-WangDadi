# artemisreentry - ART_006

## Overview

**ArtemisReentry** is a precision-engineered toolkit designed for modeling, validating, and optimizing atmospheric reentry trajectories for spacecraft returning from lunar or deep-space missions. Built for aerospace engineers, mission planners, and simulation architects, this tool eliminates guesswork by delivering deterministic reentry corridor analysis with sub-kilometer accuracy.

## Core Features

### 1. Multi-Phase Reentry Corridor Solver
Unlike generic trajectory calculators, ArtemisReentry implements a **coupled 6-DOF (degrees of freedom) + thermal load solver** that resolves aerodynamic heating, plasma blackout windows, and G‑load constraints in real time. It supports both ballistic and lifting-body reentry profiles, with automatic constraint violation detection for critical events like drogue deployment and main chute initiation.

### 2. Hypersonic Aero-Thermal Database (HAT-DB)
Embedded with a **validated aero-thermodynamic lookup table** sourced from NASA’s DPLR and LAURA datasets, this feature provides instantaneous reference for stagnation point heat flux, skin friction coefficients, and radiative heating at Mach 25–35. No external CFD coupling required—just plug in your vehicle geometry parameters and atmospheric model (GRAM‑99, NRLMSISE‑00, or custom).

### 3. Automated Mission Rule Engine (AMRE)
Define complex reentry constraints using a declarative YAML schema. AMRE validates **time-to-go vs. energy management**, cross-range margins, and blackout-to-acquisition handover latencies. It outputs a compliance scorecard with flagged violations and suggested corrective actions—ideal for pre-mission dry runs or post-flight anomaly reconstruction.

## How to Use

### Step 1: Configure Reentry Initial Conditions
Prepare a JSON configuration file with your spacecraft’s state vector at the Entry Interface (EI) point. Include altitude (typically 120 km), velocity (km/s), flight path angle (degrees), and vehicle mass properties. Use the built-in validator to ensure all fields conform to the ArtemisReentry schema:

```bash
artemisreentry init --config reentry_profile.json --check
```

The tool will auto-resolve missing parameters (e.g., bank angle schedule) against the default Artemis‑class vehicle template.

### Step 2: Execute Multi-Objective Trajectory Optimization
Run the solver in **optimization mode** to balance peak heat rate, deceleration limits, and landing footprint dispersion. Specify your cost function as a weighted sum (e.g., `--weights heat:0.4, g_load:0.3, cross_range:0.3`). The engine uses a direct transcription method with adaptive mesh refinement:

```bash
artemisreentry run --mode optimize --weights heat:0.4, g_load:0.3, cross_range:0.3 --output optimized_traj.csv
```

Monitor convergence via the built-in real‑time telemetry dashboard (port 8080 by default).

### Step 3: Generate Compliance & Visualization Report
After optimization, produce a **PDF mission brief** with annotated trajectory plots, critical event timelines, and constraint violation heatmaps:

```bash
artemisreentry report --input optimized_traj.csv --format pdf --output mission_brief.pdf
```

The report includes a **Blackout Window Analysis** table (start/end times, frequency attenuation) and a **Landing Ellipse Uncertainty** overlay derived from Monte Carlo dispersion runs.

## Best Practices

### Practice 1: Validate Against Known Flight Data
Before trusting any new vehicle model, run a **validation pass** using Apollo or Stardust reentry telemetry (included in the `examples/` directory). Compare your solver’s output against published peak heat flux and deceleration values. If discrepancies exceed 5%, adjust your atmospheric drag multiplier (`CD_scale`) in the config.

### Practice 2: Use Staged Constraint Hardening
When designing a new reentry profile, start with **soft constraints** (warning-only) in AMRE. Run the optimizer, inspect the compliance scorecard, and then harden constraints iteratively. This prevents the solver from failing silently on edge cases like off‑nominal atmospheric density or late bank reversals.
