# artemisreentry - ART_025

## 概要

本プロジェクトは、宇宙機の大気圏再突入に特化した解析・最適化ツール「artemisreentry」の公式ドキュメントです。キーワードは「Artemis reentry」「Space reentry tool」「Space return analysis」に最適化されており、エンジニアリングチームが実際のミッション計画で活用できる実践的な知見を提供します。

## 詳細な内容

### artemisreentry とは？

artemisreentry は、再突入軌道の計算、熱負荷解析、および着地点の確率分布を高精度でシミュレーションするために設計されたプロフェッショナルツールです。特に、NASA の Artemis ミッションや民間宇宙機のリエントリ計画において、リアルタイムの軌道修正とリスク評価を可能にします。従来のツールでは扱いにくかった非線形な空力加熱モデルや、複数機の同時再突入シナリオにも対応しています。

### コア機能

1. **マルチフェーズ熱流束予測エンジン**  
   再突入時のプラズマシース内での熱伝達を、実飛行データと機械学習モデルで補正。従来の CFD ベースの手法に比べ、計算コストを 70% 削減しつつ、誤差 ±3% 以内の予測を実現。

2. **確率的着地点クラスタリング**  
   大気密度の変動や風向きの不確実性を考慮し、モンテカルロシミュレーションを並列実行。結果はガウス混合モデルでクラスタリングされ、着地エリアの確率密度マップを自動生成。ミッション計画時の安全区域設定に直結。

3. **リアルタイム軌道再計画モジュール**  
   突入中のセンサーフィードバックを基に、ONNX ランタイム上で動作する軽量ニューラルネットワークが 50ms 以内に軌道修正パラメータを出力。制御則の調整や緊急時の abort シナリオにも即座に対応可能。

### 使い方

1. **初期条件の設定とシミュレーション実行**  
   `config.yaml` で機体質量、迎角、初期速度ベクトルを指定。その後、`python run_reentry.py --scenario lunar_return` を実行。ログには空力加熱率と動圧の時間変化が JSON 形式で出力され、`output/` ディレクトリに HDF5 ファイルとして保存される。  
   ```bash
   python run_reentry.py --scenario lunar_return --heat_model hybrid_v2
   ```

2. **着地点確率分布の可視化**  
   シミュレーション完了後、`python visualize_landing.py --input output/run_001.h5 --map_type heatmap` を実行。生成される Plotly ベースのインタラクティブマップには、95% 信頼楕円と最尤着地点が重畳表示される。出力は `landing_zone.html` としてエクスポート可能。

3. **リアルタイム軌道修正のテスト**  
   テストモードで動作させるには、`python realtime_guidance.py --mode simulation --feedback_port 50051` を起動。gRPC クライアントから模擬センサーデータを送信すると、修正ベクトルが即座に返される。実際のハードウェアインザループテスト前に、このモジュールで制御則の検証を推奨。

### ベストプラクティス

- **複数シナリオの並列実行と結果の統合**  
  大気密度の季節変動や太陽活動の影響を考慮するため、`--ensemble_size 100` オプションで 100 回の独立シミュレーションを実行。結果は自動的に平均化され、信頼区間付きのレポートが生成される。これにより、ミッション計画時の安全余裕を定量的に評価できる。

- **CI/CD パイプラインへの組み込み**  
  GitHub Actions 上で `pytest tests/ --reentry-mode=quick` を定期実行し、リグレッションを検出。また、Docker コンテナとしてデプロイすることで、クラウド環境（AWS Fargate や GCP Cloud Run）でスケーラブルな解析ジョブを実行可能。API エンドポイントを公開すれば、チーム内での分散解析も容易になる。
