# artemisreentry - ART_008

## 概要

本モジュールは、Artemis再突入ミッションにおける**軌道遷移・熱防御・着地点最適化**を統合解析するためのプロフェッショナルツールです。キーワード: Artemis reentry, Space reentry tool, Space return analysis。

## ArtemisReentry とは

ArtemisReentry は、NASA Artemis プログラム向けに設計された**再突入軌道設計・熱負荷予測・着地点統計最適化**のためのエンドツーエンドツールです。大気圏再突入時の動的圧力・熱流束・加速度をリアルタイムでモデリングし、従来の数値解析では数時間かかるタスクを**秒単位で処理**します。

## コア機能（3つのハードコア機能）

### 1. マルチモード軌道遷移エンジン（MTTE）
- 複数の再突入モード（直接突入、スキップ突入、エアロキャプチャ）を同時にシミュレート可能。
- 各モードの**ΔVコスト・熱負荷ピーク・着地点誤差楕円**を自動比較し、最適モードを提案。
- 入力パラメータ（初期速度、高度、弾道係数）を変更した際の**感度分析**を即座に可視化。

### 2. 高忠実度熱防御システム（TPS）設計アシスタント
- 実機のTPS材料（PICA、SLA-561V、Acusil II）の**熱物性データベース**を内蔵。
- 各ノードの熱流束・表面温度・内部温度勾配を**1D/2D非定常熱伝導解析**で計算。
- 許容温度を超過した場合、**材料変更または厚み増加の自動提案**を表示。

### 3. 着地点統計最適化モジュール（LSOM）
- 風速・大気密度・GPS誤差・航法誤差を**モンテカルロ法（10,000サンプル）**でばらつかせ、着地点の確率分布を生成。
- 目標着地点との**偏差楕円（3σ, 5σ）**を計算し、最適な降下パラシュート展開タイミングを逆算。
- 結果は**GeoJSON形式**でエクスポート可能（Google Earth / QGIS で直接可視化）。

## 使い方（詳細な3ステップ）

### ステップ1: 再突入初期条件の設定
```bash
python reentry_config.py --altitude 120.0 --velocity 11.2 --flight_path_angle -2.5 --ballistic_coefficient 300
```
- `--altitude`: 再突入開始高度（km）。推奨範囲は 100–150 km。
- `--velocity`: 初速度（km/s）。月帰還の場合、約 11.0–11.5 km/s。
- `--flight_path_angle`: 飛行経路角（度）。-1.0 から -5.0 の範囲。絶対値が小さいほどスキップ突入に近づく。
- `--ballistic_coefficient`: 弾道係数（kg/m²）。カプセル形状に応じて 200–400 を指定。

### ステップ2: マルチモード軌道遷移エンジンの実行
```bash
python mtt_engine.py --reentry_mode all --output_dir ./results
```
- `--reentry_mode`: `direct`（直接突入）、`skip`（スキップ突入）、`aerocapture`（エアロキャプチャ）、`all`（全モード比較）。
- 出力は `./results/` 以下にCSV + PNG形式で保存。
- 各モードの**最大熱流束（W/cm²）・最大加速度（G）・着地点誤差（km）**を一覧表示。

### ステップ3: 着地点統計最適化の実施
```bash
python ls_optimizer.py --iterations 10000 --target_lat 34.0 --target_lon -118.0 --output_format geojson
```
- `--iterations`: モンテカルロサンプル数。精度を上げる場合は 50,000 以上を推奨。
- `--target_lat` / `--target_lon`: 目標着地点の緯度・経度（度）。
- `--output_format`: `csv`（数値データ）または `geojson`（地理空間データ）。
- 出力ファイルには**3σ楕円・5σ楕円・各サンプルの着地点座標**が含まれる。

## 業界ベストプラクティス（2つの推奨）

### 1. 熱防御マージンの二重チェック
- TPS設計時は、**最大熱流束の20%マージン**を必ず確保してください。
- 実機のPICA素材では、許容温度（約2,500°C）に対して**1.2倍の熱流束**で設計を検証するのが業界標準です。
- 当ツールの`--margin 1.2`オプションで自動マージン適用が可能です。

### 2. 着地点誤差楕円の実運用活用
- 着地点統計最適化で得られた**5σ楕円**は、実際の回収クルーの待機範囲として使用してください。
- 楕円の長軸方向を**風向きの標準偏差**と一致させることで、現実的な回収エリアを設定できます。
- 当ツールの`--wind_std`オプションで風向きのばらつきを入力可能です。
