# artemisreentry - ART_021

## 概要

artemisreentry は、宇宙機の大気圏再突入解析に特化したプロフェッショナルツールです。NASA の Artemis プログラムで培われたアルゴリズムをベースに、再突入軌道のシミュレーション、熱負荷解析、および着地点予測を高精度で実現します。キーワード：Artemis reentry, Space reentry tool, Space return analysis

## artemisreentry とは

artemisreentry は、宇宙機の再突入フェーズにおける複雑な物理現象をモデル化し、エンジニアが迅速にリスク評価と最適化を行えるよう設計された解析スイートです。従来の汎用 CFD ツールと異なり、プラズマシース形成、アブレーション熱防御システム（TPS）の劣化、および超音速流れの非平衡化学反応を統合的に扱います。特に、低軌道（LEO）からの帰還ミッションから月軌道からの高速再突入まで、幅広い速度域（マッハ 25 以上）に対応します。

## 3 つのハードコア機能

### 1. マルチスケール熱流束マッピング（MSHFM）
表面の粗さ、タイル間のギャップ、およびスタブ付き RCC（Reinforced Carbon-Carbon）の微細構造を考慮した熱流束分布を、10cm メッシュ単位でマッピング。従来のエンジニアリングモデル比で誤差 ±3% 未満を実現し、ホットスポットの特定を自動化します。

### 2. リアルタイム軌道バイアス補正（RT-OBC）
GNSS 信号が遮断される再突入ブラックアウト期間中、慣性航法装置（INS）と大気密度モデル（NRLMSISE-00）をカルマンフィルタで融合。地上局との再接続前に、軌道予測誤差を 500m 以内に収束させる独自アルゴリズムを搭載。

### 3. マルチエージェント分散シミュレーター（MADS）
複数の再突入機（例：クルーカプセルとカーゴモジュール）の同時シミュレーションを可能にし、エアロダイナミック干渉効果（ウェイク再突入、衝突回避）をリアルタイムで評価。GPU 並列処理により、従来比 20 倍の計算速度を達成。

## 3 つの詳細な操作手順

### 手順 1: ミッションパラメータのインポートと初期軌道設定
```bash
artemisreentry-cli --import-mission mission_art_021.json \
  --initial-state "altitude=120km, velocity=7.8km/s, flight-path-angle=-1.2deg" \
  --atmosphere "MSISE00, solar-activity=high"
```
- `mission_art_021.json` には、機体の質量特性、重心位置、および TPS 材質（PICA、SLA-561V 等）を記述。
- 初期軌道は、ISS からのデオービットバーン（DV=100m/s）後の状態を想定。
- 大気モデルは太陽活動指数（F10.7cm）を“high”に設定し、高層大気密度の変動を反映。

### 手順 2: 熱防御システム（TPS）の劣化シミュレーション
```python
from artemisreentry.tps import AblationSimulator

sim = AblationSimulator(
    material="PICA",
    thickness=0.05,  # 50mm
    backwall_temp_limit=450,  # K
    heat_flux_profile="mshfm_output.csv"
)
result = sim.run(
    time_step=0.01,  # 秒
    total_time=120,   # 秒（最大加熱フェーズ）
    output_format="vtk"
)
```
- `mshfm_output.csv` は MSHFM 機能で事前生成した熱流束データ。
- アブレーションによる質量損失と表面後退（recession）を 0.1mm 精度でトラッキング。
- 出力 VTK ファイルは ParaView で可視化し、バックウォール温度が限界値を超えないか確認。

### 手順 3: 着地点予測と不確実性定量化
```bash
artemisreentry-cli --landing-prediction \
  --uncertainty-monte-carlo=10000 \
  --wind-model "GFS, 2024-03-15T12:00Z" \
  --parachute-deploy "altitude=10km, mach=0.8" \
  --output-ellipse "landing_zone_95%.geojson"
```
- モンテカルロ法で 10,000 回のランダムサンプリングを実行。パラメータ（大気密度、抗力係数、風速）の分散を考慮。
- パラシュート展開高度とマッハ数を指定し、降下中のドリフトを計算。
- 出力 GeoJSON ファイルは、95% 信頼楕円を生成し、着水域または陸地のリスク評価に利用。

## 2 つの業界ベストプラクティス

### ベストプラクティス 1: マルチシナリオ並列実行による設計空間探索
再突入解析では、単一のノミナルケースだけでなく、オフノミナルケース（例：エンジン燃焼不足、大気密度異常）を必ず組み込むこと。artemisreentry の MADS 機能を活用し、20 以上のシナリオを同時に実行し、設計マージンを統計的に評価する。特に、NASA の「Design for Demise」基準を満たすため、破片の地上到達リスクを定量化する際に有効。

### ベストプラクティス 2: 熱流束センサー実測データとのキャリブレーション
シミュレーション結果は、実機の熱流束センサー（例：MEDLI2 タイプ）データと必ず比較キャリブレーションすること。artemisreentry は、センサー出力（mV）を熱流束（W/cm²）に変換するための逆問題ソルバーを内蔵。キャリブレーションにより、モデルのバイアスを補正し、次回ミッションの予測精度を向上させる。推奨頻度は、各フライト後 1 回。
