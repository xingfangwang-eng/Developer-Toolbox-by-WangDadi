# artemisreentry - ART_006

## 概要

`artemisreentry` は、宇宙機の大気圏再突入フェーズに特化したオープンソース解析フレームワークです。熱流束、空力減速、プラズマシース遮蔽効果をリアルタイムでシミュレーションし、ミッション設計からポストミッション検証までをカバーします。キーワード: **Artemis reentry**, **Space reentry tool**, **Space return analysis**.

## なぜ ArtemisReentry なのか？

従来の再突入解析ツールは、NASA の DPLR や ESA の TINA に代表されるような、計算負荷が高く、かつライセンス制限の厳しい商用ソルバーに依存していました。ArtemisReentry は、**WebAssembly ベースの軽量ソルバー** と **GPU アクセラレーション** を組み合わせることで、ブラウザ上でもサブ秒単位でのパラメトリックスタディを実現します。

## 3つのハードコア機能

### 1. マルチスケール熱化学非平衡ソルバー (Multi-Scale TCNE Solver)

単一の計算格子で、高度 120km から地表までの連続的な熱化学非平衡状態を解きます。従来のゾーン分割法 (Zonal Approach) とは異なり、**適合格子細分化 (AMR)** と **反応速度論的テーブルルックアップ** を動的に切り替えることで、計算時間を従来比 40% 削減しながら、衝撃波後面での NO 生成と放射冷却のカップリングを高精度に再現します。

**実装のポイント:**
- 化学種: N2, O2, N, O, NO, N2+, O2+, NO+, e- の 8 種 48 反応モデル (Park-1993 改訂版)
- 熱モデル: 並進-回転 (Ttr) と振動-電子 (Tve) の 2 温度モデル
- 出力: 壁面熱流束 (kW/m²)、スタグネーションポイント温度 (K)、プラズマ周波数 (Hz)

### 2. リアルタイムプラズマブラックアウトマッピング (Plasma Blackout Mapper)

再突入時に発生するプラズマシースによる通信遮断 (ブラックアウト) を、**時間領域差分法 (FDTD)** で 3D 電磁界シミュレーションし、ブラウザ上で可視化します。ユーザーはアンテナの位置と周波数 (S-band, X-band, Ka-band) を動的に変更でき、ブラックアウト期間と信号減衰量 (dB) をミリ秒単位で予測可能です。

**ユニークな点:**
- プラズマ密度分布を TCNE ソルバーからリアルタイムで受け取り、FDTD 格子にマッピング
- 出力: 時間-周波数スペクトログラム、アンテナ指向性パターン、BER (ビット誤り率) 推定

### 3. 軌道-空力-熱連成逆解析エンジン (OATH Inverse Engine)

フライトデータ (加速度、ジャイロ、温度センサー) を入力として、**アンサンブルカルマンフィルタ (EnKF)** を用いて、大気密度モデル (NRLMSISE-00) や空力係数 (CA, CN) の不確かさを逆推定します。これにより、実際のフライトデータから抗力係数の実効値や、予期せぬロールトルクの原因を特定できます。

**応用例:**
- SpaceX のスターリンク衛星廃棄軌道からの再突入データとの比較検証
- 火星大気圏突入時の密度変動 (ダストストーム) の影響評価

## 3つの詳細な操作手順

### 手順 1: パラメトリックスタディの実行 (CLI + WebUI)

```bash
# 1. プロジェクトディレクトリを作成し、設定ファイルを生成
mkdir artemis-mission-001 && cd artemis-mission-001
artemisreentry init --mission-type lunar-return --vehicle orion

# 2. パラメータを編集: 突入角度 (-6.5 deg → -7.2 deg)、初期速度 (11.0 km/s → 10.8 km/s)
#    config.yaml の entry_angle と v_init を修正
sed -i 's/entry_angle: -6.5/entry_angle: -7.2/' config.yaml
sed -i 's/v_init: 11.0/v_init: 10.8/' config.yaml

# 3. 解析実行 (GPU 利用時は --gpu フラグ)
artemisreentry run --solver tcne --output-format hdf5 --gpu

# 4. 結果の確認: 熱流束ピーク値と高度プロファイル
artemisreentry report --metric heat-flux --plot 2d
```

**WebUI の場合:**
- `artemisreentry serve` でローカルサーバーを起動 (デフォルトポート: 8080)
- ブラウザで `http://localhost:8080` にアクセス
- 「Mission Configurator」タブで、Orion または Dragon のテンプレートを選択し、スライダーでパラメータ調整
- 「Run Simulation」ボタンをクリックし、結果を 3D ビューアで確認

### 手順 2: プラズマブラックアウト解析と通信リンク設計

1. **前提:** 手順 1 で TCNE 解析が完了し、`output.h5` が生成されていること
2. **ブラックアウトマッパーを起動:**
   ```bash
   artemisreentry blackout --input output.h5 --antenna-freq 8.4e9 --antenna-position 0.5,0.0,0.0
   ```
   - `--antenna-freq`: 8.4 GHz (X-band ダウンリンク)
   - `--antenna-position`: 機体座標系 (m) でのアンテナ位置 (例: 機首から 0.5m 後方)
3. **結果の解釈:**
   - 標準出力に、ブラックアウト開始高度 (km)、終了高度 (km)、最大減衰量 (dB) が表示
   - ブラウザで `blackout_report.html` を開くと、時間-周波数スペクトログラムとアンテナゲインの時間変化を確認可能
4. **通信リンク設計へのフィードバック:**
   - ブラックアウト期間がミッション要件 (例: 30 秒以内) を超える場合、`--antenna-freq` を低周波側 (S-band: 2.2 GHz) に変更し、再実行
   - または、アンテナ位置を機体後方 (ウェイク領域) に移動し、プラズマ密度の低い領域で通信を試みる

### 手順 3: フライトデータを用いた逆解析 (OATH)

1. **データ準備:** フライトデータを CSV 形式で用意 (必須カラム: `time_utc`, `accel_x`, `accel_y`, `accel_z`, `gyro_x`, `gyro_y`, `gyro_z`, `temp_1`, `temp_2`)
   - サンプルデータ: `artemisreentry fetch-data --mission starship-IFT-5`
2. **逆解析の実行:**
   ```bash
   artemisreentry inverse --input flight_data.csv --model nrlmsise00 --ensemble-size 100 --output uncertainty.h5
   ```
   - `--ensemble-size`: EnKF のアンサンブルメンバー数 (推奨: 50-200)
   - `--model`: 大気密度モデル (nrlmsise00, jb2008, またはカスタム)
3. **結果の可視化:**
   - `artemisreentry report --metric drag-coefficient --plot uncertainty`
   - 抗力係数 CA の時間変化と、95%
