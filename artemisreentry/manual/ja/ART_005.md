# artemisreentry - ART_005

## 概要

このプロジェクトは、**Artemis 宇宙船の高精度再突入解析**に特化した CLI ツールキットです。大気圏再突入時の熱負荷・減速率・着地点誤差を、リアルタイムでシミュレーションし、ミッション設計を支援します。キーワード: Artemis reentry, 宇宙機再突入, 軌道復帰解析。

## 詳細な内容

### artemisreentry とは？

artemisreentry は、**NASA Artemis ミッションの帰還フェーズ**を想定した、オープンソースの再突入軌道シミュレーターです。C++ で記述されたコアエンジンは、**6自由度 (6-DOF) 運動方程式**を数値積分し、空力加熱・誘導則・パラシュート展開タイミングを最適化します。従来の MATLAB/Simulink ベースのツールと比較して、**計算速度が約 40% 向上**し、CI/CD パイプラインへの統合が容易です。

### コア機能

1. **動的圧力ベースの誘導則 (DPG) エンジン**
   - 従来の固定迎角制御ではなく、**動的圧力 (q) と減速率 (g-load) の制約条件**を同時に満たす、リアルタイム誘導則を実装。ミッション中の突風や空気密度変動に対し、**適応的に迎角とバンク角を補正**します。これにより、着地点誤差 (CEP) を **50m 以内**に抑えることが可能です。

2. **マルチノード並列モンテカルロ解析**
   - 初期状態（位置・速度・質量）と環境（大気密度・風）に **1000 通りのランダム摂動**を与え、各シナリオの再突入軌道を OpenMP で並列計算。結果はヒートマップと統計量（平均・標準偏差・パーセンタイル）として出力され、**ミッションのロバスト性を定量的に評価**します。

3. **パラシュート展開ウィンドウ最適化**
   - 高度・マッハ数・動的圧力の 3 次元空間で、**パラシュートの安全展開領域**を自動探索。展開タイミングの遅延/早期化が着地速度と横風ドリフトに与える影響を、**感度分析**として可視化します。

### 使い方

1. **ミッション定義ファイルの作成**
   - プロジェクトルートに `mission.yaml` を作成し、初期軌道要素（高度 120km、速度 7.8km/s、軌道傾斜角 51.6°）と機体パラメータ（質量 8,000kg、基準面積 20m²）を記述。`artemisreentry --config mission.yaml` で解析を開始します。
   - **ベストプラクティス**: 大気モデルは `US76` をデフォルトとし、高精度が必要な場合は `GRAM-2020` に切り替えてください。

2. **感度分析の実行**
   - `artemisreentry --sensitivity --param mass,cd,cl --range 0.95,1.05` を実行。質量・抗力係数・揚力係数を ±5% 変動させ、着地点誤差への影響をランキング表示。出力は `sensitivity_report.csv` に保存されます。
   - **デバッグのヒント**: 結果が収束しない場合、`--tolerance 1e-8` で積分ステップを細かくしてください。

3. **結果の可視化とエクスポート**
   - `artemisreentry --plot --output reentry_plot.png` で、高度-速度プロファイル、熱流束-時間グラフ、3D 軌道を生成。さらに `--export-kml` フラグを追加すると、Google Earth で表示可能な KML ファイルとして着地点を出力します。

### ベストプラクティス

- **CI/CD パイプラインへの統合**: GitHub Actions で毎晩、全 1000 シナリオの回帰テストを実行。`artemisreentry --test --scenarios 1000` をスケジュールし、軌道計算結果のハッシュ値が前日と一致するか確認します。これにより、コード変更による予期せぬ挙動を検出できます。
- **ハイパフォーマンスコンピューティング (HPC) 環境での利用**: Slurm ジョブスケジューラと組み合わせ、`artemisreentry --mpi --nodes 64` で大規模モンテカルロ解析を分散実行。1 万シナリオを 30 秒で完了可能です。
