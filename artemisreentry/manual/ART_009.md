## What is artemisreentry?

artemisreentry is a precision toolchain for modeling, simulating, and validating spacecraft reentry trajectories. It’s built for engineers who need deterministic results, not approximations. Whether you’re handling lunar return profiles or atmospheric skip maneuvers, artemisreentry gives you sub-10ms latency on thermal load calculations and real-time debris dispersion modeling.

## Core Features

### 1. Real-Time Aerothermal Flux Mapping
artemisreentry computes stagnation-point heat flux and convective heating rates using a coupled Navier-Stokes solver. It outputs a 3D thermal gradient map across the TPS (Thermal Protection System) surface, updated every 50ms during simulation. No more waiting for post-processing—you get live feedback on ablation margins.

### 2. Multi-Phase Trajectory Spline Engine
The tool uses cubic Hermite splines to interpolate between atmospheric interface points, coast phases, and terminal descent. It automatically adjusts for gravitational perturbations (J2, J3) and atmospheric drag coefficients derived from NRLMSISE-00. The engine generates a continuous, differentiable trajectory path with error bounds under 0.1% for altitudes above 80 km.

### 3. Automated Contingency Branching (ACB)
artemisreentry simulates failure modes in parallel: parachute deployment failure, heat shield jettison delay, or navigation drift. Each branch is computed in a separate thread, and the tool outputs a decision tree with probability-weighted landing ellipses. You can set custom thresholds for abort triggers (e.g., dynamic pressure > 15 kPa or Mach number < 0.8).

## How to Use

### Step 1: Configure the Reentry Profile
Start by defining the initial state vector in the `config/reentry_profile.yaml` file. Set the altitude (km), velocity (m/s), flight path angle (degrees), and ballistic coefficient (kg/m²). For example:
```yaml
initial_conditions:
  altitude: 120.0
  velocity: 7800.0
  flight_path_angle: -1.5
  ballistic_coefficient: 250.0
```
Then, specify the atmospheric model (e.g., `model: NRLMSISE-00`) and the TPS material (e.g., `PICA-X`). Run `artemisreentry --validate-config` to check for parameter bounds.

### Step 2: Execute the Simulation
Launch the simulation with `artemisreentry --run --output-format csv`. The tool will generate a real-time log in the terminal showing key metrics: altitude, Mach number, heat flux (W/cm²), and deceleration (g). Use the `--visualize` flag to open an interactive 3D plot of the trajectory. For batch runs, use `--batch` with a list of state vectors in `batch_input.csv`.

### Step 3: Analyze Contingency Outputs
After the simulation, navigate to the `output/contingency/` directory. Each branch is stored as a separate CSV file (e.g., `branch_parachute_failure.csv`). Use the built-in `artemisreentry --analyze-contingency` command to generate a PDF report with landing ellipse boundaries and probability densities. The report includes a heatmap overlay of debris dispersion zones.

## Best Practices

### 1. Use High-Fidelity Atmospheric Models for Low-Altitude Simulations
For altitudes below 50 km, switch from `NRLMSISE-00` to `GRAM-2016` for better density gradient accuracy. This reduces landing ellipse error by up to 12% in crosswind scenarios. Update the config with `atmospheric_model: GRAM-2016` and re-run.

### 2. Parallelize Contingency Branches for Large-Scale Studies
When running more than 10 failure modes, set `--threads 8` to distribute the workload across CPU cores. artemisreentry automatically manages memory allocation for each branch, but ensure your system has at least 16 GB RAM for 20+ simultaneous threads. Monitor CPU usage with `htop` to avoid throttling.
