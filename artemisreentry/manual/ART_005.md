# artemisreentry - ART_005

## Overview

artemisreentry is a precision-engineered toolkit for modeling, simulating, and optimizing spacecraft reentry trajectories. Built for aerospace engineers and mission planners, it bridges the gap between atmospheric entry dynamics and operational decision-making. This is not a toy—it's a production-grade analysis engine for return-to-Earth scenarios.

## What is artemisreentry?

artemisreentry provides a modular, CLI-first environment for calculating heat flux, deceleration profiles, and landing dispersion for reentry vehicles. It ingests trajectory parameters, atmospheric models, and vehicle specs to output actionable telemetry and risk assessments. Designed for the Artemis program ecosystem, it scales from suborbital tests to lunar return missions.

## Core Features

### 1. Multi-Phase Trajectory Solver with Real-Time Aero-Thermal Feedback
The solver couples 6-DOF dynamics with a Chapman-sheared atmospheric model, updating heat flux and drag coefficients at each integration step. It supports variable ballistic coefficients and lift-to-drag ratios, outputting time-series data for G-load, stagnation temperature, and dynamic pressure. This allows engineers to validate entry corridors against thermal protection system (TPS) limits without spinning up a full CFD suite.

### 2. Landing Dispersion Ellipse Generator via Monte Carlo
Inject Gaussian noise into initial state vectors (position, velocity, attitude) and atmospheric density perturbations. The engine runs 10,000+ iterations to produce a 3-sigma landing ellipse. Output includes latitude/longitude centroids, semi-major axis orientation, and probability contours. Critical for range safety officers and recovery team staging.

### 3. Autonomous Skip-Entry Guidance Simulation
Implement a closed-loop guidance algorithm that modulates bank angle to control downrange and crossrange. The simulator models an Apollo-style predictor-corrector loop, calculating required lift vector adjustments every 10 seconds. Outputs include control surface deflection history, energy management plots, and final splashdown accuracy relative to the target.

## How to Use

### Step 1: Define Vehicle and Mission Parameters in a YAML Config
Create a `reentry_config.yaml` file with vehicle mass, reference area, drag coefficient table, and initial state vector (ECI coordinates). Specify atmospheric model (e.g., NRLMSISE-00 or US Standard 1976) and integration step size. Example snippet:

```yaml
vehicle:
  mass_kg: 8500
  ref_area_m2: 18.5
  drag_coeff: [0.8, 1.2, 0.6]  # Mach-dependent
initial_state:
  position_km: [6542.3, -1023.7, 2845.1]
  velocity_kms: [7.45, -0.89, 1.23]
atmosphere: "nrlmsise00"
```

Run `artemisreentry --config reentry_config.yaml` to validate the file.

### Step 2: Execute a Batch Monte Carlo Simulation
Use the `--mc-runs` flag to set the number of perturbed trajectories. The engine will sample initial state errors (3-sigma: 100 m position, 1 m/s velocity) and atmospheric density variations (10% uncertainty). Output is written to `dispersion_results.nc` (NetCDF format) with full time-history for each run.

```bash
artemisreentry --config reentry_config.yaml --mc-runs 5000 --output dispersion.nc
```

### Step 3: Analyze Heat Flux and Deceleration Profiles
Post-process the Monte Carlo output using the built-in `artemis-reentry-plot` utility. Generate overlay plots of heat flux vs. time for the 95th percentile cases, and identify peak G-load exceedances. Use the `--threshold` flag to flag trajectories that exceed TPS limits (e.g., > 150 W/cm²).

```bash
artemis-reentry-plot --input dispersion.nc --metric heat_flux --threshold 150
```

The tool outputs a JSON summary with violation counts and recommended entry angle adjustments.

## Best Practices

### 1. Validate Against Heritage Missions First
Before running your own parameters, test the solver against known Apollo or Orion reentry data. Use the `--heritage apollo` flag to load pre-validated vehicle models and compare output G-profiles against published telemetry. This ensures your atmospheric model and integration tolerances are calibrated correctly.

### 2. Run Sensitivity Analysis on Ballistic Coefficient
The most impactful single parameter is the ballistic coefficient (mass / (Cd * area)). Run a sweep from 200 to 600 kg/m² in 50 kg/m² increments, and plot peak heat flux vs. entry angle. This reveals the optimal corridor for your TPS margin. Use `--sweep ballistic_coefficient` to automate.
