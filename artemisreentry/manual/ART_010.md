# artemisreentry - ART_010

## Overview

ArtemisReentry is a high-precision, simulation-driven toolkit engineered for analyzing and optimizing spacecraft reentry trajectories, thermal protection system (TPS) margins, and return corridor constraints. Built for aerospace engineers and mission planners, this tool bridges the gap between orbital mechanics and atmospheric flight dynamics with a focus on Artemis-class return profiles.

## Core Features

### 1. Multi-Phase Trajectory Solver with Real-Time Atmospheric Coupling
Unlike traditional reentry tools that rely on static atmospheric models, ArtemisReentry integrates a time-varying, high-fidelity atmospheric interface (NRLMSISE-00 + local weather assimilation) to compute drag, heating, and lift-to-drag ratios dynamically across the entire entry, descent, and landing (EDL) sequence. This enables sub-1% error margins on peak heat flux predictions for lifting-body capsules.

### 2. TPS Margin Analyzer with Ablation Modeling
The tool includes a built-in finite-difference thermal solver that models char-layer ablation, pyrolysis gas injection, and surface recession for common TPS materials (PICA, SLA-561V, Avcoat). It automatically flags mission segments where the safety factor drops below 1.25, allowing engineers to iterate on trajectory shaping before committing to hardware constraints.

### 3. Return Corridor Optimization Engine
ArtemisReentry computes the feasible reentry corridor (in terms of flight-path angle vs. velocity) and applies a constrained nonlinear optimizer (SCP-based) to maximize cross-range capability while respecting peak g-load and heat rate limits. The output includes a Pareto front of viable trajectories, each annotated with delta-V requirements for mid-course corrections.

## How to Use

### Step 1: Define the Entry State and Vehicle Parameters
Create a JSON configuration file specifying the entry interface state (altitude, velocity, flight-path angle, heading) and vehicle aerodynamic coefficients (CL, CD as functions of Mach and angle of attack). Use the `artemisreentry validate-config` command to check for missing fields or out-of-range values. Example:
```bash
cat <<EOF > entry_config.json
{
  "entry_altitude_km": 121.9,
  "entry_velocity_kms": 11.0,
  "flight_path_angle_deg": -1.5,
  "vehicle": {
    "mass_kg": 8900,
    "reference_area_m2": 19.6,
    "cl_table": "cl_vs_mach.csv",
    "cd_table": "cd_vs_mach.csv"
  }
}
EOF
artemisreentry validate-config entry_config.json
```

### Step 2: Run the Multi-Phase Simulation
Execute the solver with the `--mode full` flag to propagate from entry interface to parachute deploy. The solver automatically switches between continuum, transitional, and free-molecular flow regimes based on Knudsen number. Monitor real-time diagnostics via the built-in WebSocket telemetry stream:
```bash
artemisreentry simulate entry_config.json --mode full --output trajectory.nc --stream :8080
```
Open `http://localhost:8080` in a browser to watch altitude, velocity, and heat rate plots update at 10 Hz.

### Step 3: Analyze TPS Margins and Optimize Corridor
Post-process the trajectory NetCDF output to generate a TPS margin report:
```bash
artemisreentry tps-margin trajectory.nc --material pica --thickness_mm 25.4 --report tps_report.pdf
```
Then run the corridor optimizer to find a trajectory that reduces peak heat rate by 15% while keeping cross-range above 800 km:
```bash
artemisreentry optimize-corridor entry_config.json --max-heat-rate 85 --min-cross-range 800 --output optimized_traj.nc
```
Compare the original and optimized trajectories using the built-in overlay plotter.

## Best Practices

### 1. Always Validate with Monte Carlo Dispersion Runs
Before finalizing any trajectory for a mission design review, run a 500-case Monte Carlo analysis that perturbs entry state, atmospheric density, and aerodynamic coefficients. Use the `artemisreentry monte-carlo` subcommand with a covariance matrix derived from navigation error budgets. This reveals whether your TPS margin holds under worst-case dispersions—critical for human-rated return vehicles.

### 2. Use the Corridor Optimizer Early in the Design Loop
Don’t wait until the vehicle geometry is locked. Run the optimizer with placeholder aerodynamic data to determine the required L/D and bank-angle modulation capability. This drives requirements for control surface sizing and reaction control system (RCS) propellant allocation. A 10% improvement in cross-range from optimization can eliminate the need for a costly skip-entry maneuver.
