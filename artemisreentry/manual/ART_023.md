# artemisreentry - ART_023

## Overview

**artemisreentry** is a domain-specific toolkit engineered for high-fidelity reentry trajectory simulation and post-mission atmospheric return analysis. Built for aerospace engineers, mission planners, and orbital debris researchers, this tool bridges the gap between raw telemetry data and actionable reentry insights. It is optimized for the **Artemis reentry** use case, supporting both crewed and uncrewed return profiles with sub-kilometer precision.

This release focuses on **real-time ablation modeling**, **multi-body separation dynamics**, and **heat flux anomaly detection** — three areas where existing tools fall short.

---

## Core Features

### 1. Ablation Layer Mapping with Real-Time Mass Loss Propagation
Unlike static thermal protection system (TPS) calculators, artemisreentry dynamically models **char layer recession** and **pyrolysis gas injection** across the vehicle’s surface. It couples a 3D finite-difference heat conduction solver with a material-specific ablation database (including PICA, SLA-561V, and Avcoat variants). The engine outputs **time-resolved mass loss curves** and **surface recession rates** at each mesh node, enabling engineers to validate TPS margins against actual flight data.

### 2. Multi-Body Separation Cascade Simulation (MBSCS)
For return vehicles deploying drogue chutes, main parachutes, or jettisoning service modules, artemisreentry simulates the **sequential separation cascade** using a multibody dynamics engine with contact-impact constraints. It accounts for aerodynamic interference, wake recirculation, and tether dynamics. The simulation yields **separation distance thresholds**, **relative velocity envelopes**, and **collision probability contours** — critical for abort scenarios or staged descent profiles.

### 3. Heat Flux Anomaly Detection via Spectral Gradient Analysis
The tool ingests **infrared thermography** or **embedded thermocouple data** and applies a wavelet-based spectral decomposition to identify anomalous heat flux spikes. By comparing real-time spectral signatures against a library of known reentry anomalies (e.g., boundary layer transition, shock-shock interaction, or tile gap heating), artemisreentry flags **pre-critical events** with a confidence score. This feature is designed for post-flight analysis and real-time mission support.

## How to Use

### Step 1: Configure the Reentry Profile and Material Stack
Define the trajectory using a `.csv` or `.json` file containing **time-stamped state vectors** (position, velocity, attitude) at a minimum of 100 Hz resolution. Then, specify the thermal protection system (TPS) layup in a `tps_config.yaml` file — include material type, thickness, and initial temperature distribution. Example snippet:

```yaml
layers:
  - material: "PICA-15"
    thickness_mm: 25.4
    initial_temp_K: 300
  - material: "Aluminum 2024"
    thickness_mm: 3.0
    initial_temp_K: 300
```

Run the preprocessor to validate mesh conformity and material compatibility:

```bash
artemisreentry preprocess --trajectory reentry_art_023.csv --tps tps_config.yaml --mesh vehicle_mesh.stl
```

### Step 2: Execute the Coupled Ablation and Separation Simulation
Launch the main solver with the `--coupled` flag to enable simultaneous ablation and separation dynamics. This is recommended for return vehicles with jettisonable components (e.g., Orion’s Launch Abort System). Use the following command to enable **multi-body separation** and **real-time ablation logging**:

```bash
artemisreentry run --coupled --ablation_log interval=0.1 --separation_output sepsim_art023.json
```

The solver will output a `reentry_summary.h5` file containing time-series data for: surface temperature, recession depth, mass loss, and separation distances between all defined bodies.

### Step 3: Run Spectral Anomaly Detection on the Heat Flux Data
After simulation completion (or during a live feed via UDP stream), invoke the anomaly engine:

```bash
artemisreentry detect --input reentry_summary.h5 --spectral wavelet_db4 --threshold 2.5
```

This will generate an `anomaly_report.md` file listing all flagged events with:
- **Timestamp** (seconds from entry interface)
- **Anomaly type** (e.g., “Boundary Layer Transition”)
- **Confidence score** (0.0–1.0)
- **Spectral energy shift** (dB)

For real-time telemetry, pipe data directly:

```bash
nc -u localhost 9000 | artemisreentry detect --stream --threshold 2.0
```

## Best Practices

### 1. Validate TPS Material Properties Against Flight Heritage
Do not rely solely on manufacturer datasheets. Prior to running ablation simulations, cross-reference material properties (e.g., thermal conductivity, specific heat, and pyrolysis gas enthalpy) with **ArcJet test data** or **post-flight inspection reports** from similar reentry profiles (e.g., Stardust, Hayabusa, or Artemis I). artemisreentry’s material database is extensible — add your own calibrated coefficients via the `materials/custom.json` interface.

### 2. Use Statistical Sensitivity Analysis for Separation Timing
When simulating multi-body separation, always perform a **Monte Carlo sweep** on the separation trigger conditions (e.g., altitude, dynamic pressure, or Mach number). artemisreentry includes a `--sensitivity` flag that runs 100+ variants and outputs a **Pareto front** of collision probability vs. separation delta-V. This is especially critical for crewed missions where abort windows are narrow. Run:

```bash
artemisreentry sensitivity --params separation_altitude,separation_mach --n_samples 500
```
