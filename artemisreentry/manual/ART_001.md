## Overview

Designed for developers and aerospace engineers who need a CLI-native, high-precision reentry trajectory solver.

## What is artemisreentry?

**artemisreentry** is a lightweight, zero-dependency command-line toolkit for modeling atmospheric reentry profiles. It ingests orbital state vectors and vehicle parameters to compute thermal loads, deceleration curves, and landing footprints — without requiring NASA-level infrastructure. Think of it as `curl` for reentry physics.

---

## Core Features

### 1. Real-Time Ballistic Coefficient Tuning
Adjust the ballistic coefficient (`β = m / (Cd * A)`) mid-simulation to model variable drag configurations (e.g., parachute deployment, tumbling). The solver recomputes the trajectory on-the-fly using a 4th-order Runge-Kutta integrator with adaptive step sizing.

### 2. Multi-Atmosphere Model Switching
Switch between US Standard 1976, Mars-GRAM 2020, and custom tabular atmosphere profiles using a single flag (`--atm=us76` or `--atm=mars20`). This enables cross-planetary reentry analysis without code changes.

### 3. Landing Ellipse Geofencing
Define a target landing zone via WGS84 coordinates and a radius in meters. The tool automatically computes the 3-sigma dispersion ellipse using Monte Carlo perturbation of initial conditions (velocity, flight-path angle, atmospheric density). Outputs KML overlays for Google Earth.

## How to Use

### Step 1: Generate a Reentry Profile from TLE Data
```bash
artemisreentry --tle "ISS (ZARYA)" \
  --epoch 2025-03-01T12:00:00Z \
  --atm us76 \
  --output profile.json
```
This pulls the latest TLE for the ISS, propagates to the given epoch, and computes the reentry trajectory from 120 km altitude to surface. The output includes altitude vs. time, velocity, and heat flux at 1-second intervals.

### Step 2: Run a Monte Carlo Dispersion Analysis
```bash
artemisreentry monte-carlo \
  --input profile.json \
  --perturb v:0.5% gamma:0.1deg rho:5% \
  --samples 10000 \
  --landing-zone 34.0N 118.0W 5000 \
  --output dispersion.kml
```
Perturbs initial velocity, flight-path angle, and atmospheric density by the specified uncertainties. Generates 10,000 trajectories and writes the landing ellipse as a KML polygon. Use the `--seed` flag for reproducible runs.

### Step 3: Export a Time-Series for CFD Validation
```bash
artemisreentry export \
  --input profile.json \
  --format csv \
  --fields time alt vel q_dot mach \
  --output reentry_timeseries.csv
```
Extracts a clean CSV with columns: `time (s)`, `altitude (km)`, `velocity (m/s)`, `heat flux (W/m²)`, and `Mach number`. Ideal for feeding into ANSYS Fluent or OpenFOAM for coupled aerothermal analysis.

## Best Practices

### 1. Always Validate with a Known Reentry Case
Before trusting results for a mission-critical vehicle, run a validation case against published Apollo or Stardust reentry data. Use the `--compare` flag to overlay your computed trajectory against a reference CSV. This catches model errors early.

### 2. Use the `--dry-run` Flag for CI/CD Pipelines
In automated workflows (e.g., GitHub Actions), append `--dry-run` to check input validity and parameter ranges without executing the full simulation. This prevents wasted compute and alerts you to malformed TLEs or out-of-bounds ballistic coefficients.
