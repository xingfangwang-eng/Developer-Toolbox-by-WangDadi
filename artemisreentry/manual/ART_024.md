# artemisreentry - ART_024

## Overview

ArtemisReentry is a precision reentry simulation engine purpose-built for trajectory analysts and space domain engineers working on return-to-Earth missions. It bridges the gap between high-fidelity atmospheric modeling and actionable mission planning, delivering deterministic outputs for real-time decision support.

## Hardcore Feature Highlights

### 1. **Multi-Phase Aero-Thermal Coupling Engine**
ArtemisReentry integrates CFD-grade heat flux modeling with 6-DOF trajectory propagation. Instead of relying on precomputed lookup tables, it dynamically couples aerodynamic drag, radiative heating, and ablation layer response in real time. This allows you to simulate hypersonic plasma sheath formation and its impact on comms blackout windows with sub-second granularity.

### 2. **Stochastic Reentry Corridor Mapping**
The tool runs Monte Carlo ensembles over atmospheric density perturbations, vehicle attitude jitter, and GPS dropout scenarios. It outputs probabilistic landing footprint ellipses with 95% and 99.7% confidence intervals, enabling robust abort site selection and risk-informed go/no-go decisions.

### 3. **Time-Varying Constraint Solver for TPS Margins**
You can define custom safety margins for thermal protection system (TPS) degradation over the entire reentry timeline. ArtemisReentry solves for maximum allowable entry angle and velocity under a given TPS thickness profile, providing a real-time “green-yellow-red” margin indicator that updates every 50 ms.

## Detailed Usage Steps

### Step 1: Configure Mission Parameters and Initial State Vector
Launch the simulation by loading a JSON configuration file that specifies the vehicle’s initial state (position, velocity, mass, reference area) and atmospheric model (NRLMSISE-00 or custom CIRA-86). Set the desired reentry interface altitude (typically 120 km) and define your TPS material properties: density, specific heat, and ablation enthalpy. ArtemisReentry validates all inputs against physical bounds—any out-of-range values trigger a structured error with suggested corrections.

### Step 2: Run the Coupled Aero-Thermal Propagation
Execute the solver with the `--coupled` flag to enable aero-thermal feedback. The engine discretizes the trajectory into 0.01-second timesteps and solves the coupled ODE system using an adaptive Runge-Kutta 5(4) scheme. Real-time telemetry streams to stdout: altitude, velocity, stagnation heat flux, and TPS recession depth. Use the `--verbose` flag to output per-timestep diagnostics for debugging or post-processing.

### Step 3: Analyze Probabilistic Footprint and Margin Reports
After propagation, generate the stochastic footprint map using the `--mc-ensemble 10000` flag. ArtemisReentry outputs a GeoJSON file with the 95% and 99.7% landing ellipses, plus a CSV of all Monte Carlo runs. For TPS margin analysis, run `--margin-report` to produce a timeline chart showing the safety margin against your defined thresholds. Use the built-in CLI viewer (`artemisreentry view --report margin_report.json`) to inspect the critical reentry phase where margins narrow.

## Industry Best Practices

- **Always run a sensitivity sweep on atmospheric density before critical missions.** ArtemisReentry’s `--density-perturb` flag lets you vary density by ±15% in 1% increments. This reveals how sensitive your landing footprint is to seasonal or latitudinal atmospheric anomalies—essential for polar or tropical reentry corridors.

- **Integrate ArtemisReentry into your CI/CD pipeline for automated margin regression.** Use the `--exit-on-violation` flag to fail the build if TPS margins drop below 10% at any point. This ensures every code change to your vehicle design or trajectory profile is automatically stress-tested against reentry safety constraints.
