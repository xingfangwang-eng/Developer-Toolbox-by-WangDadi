## Detaillierter Inhalt

### Was ist artemisreentry?

**artemisreentry** ist ein hochspezialisiertes, in Rust implementiertes CLI-Tool zur präzisen Simulation und Analyse von atmosphärischen Wiedereintrittsprofilen für Raumkapseln und Nutzlasten.  
Es berechnet reale thermodynamische Belastungen, Bahnkorrekturen und Aufprallpunkte – ohne auf Cloud-APIs angewiesen zu sein.  
Das Tool eignet sich sowohl für Rapid Prototyping in der Missionsplanung als auch für Post-Flight-Analysen.

---

### Kernfunktionen

#### 1. **Dynamische Heat-Flux-Vorhersage mit konvektiver Kopplung**
Das Modell berechnet den konvektiven Wärmestrom entlang des gesamten Wiedereintrittspfades unter Berücksichtigung von Atmosphärenschichtung (US Standard 1976, MSIS-E-00).  
Es integriert real-time die **Reynolds-Zahl**, **Prandtl-Zahl** und **Stanton-Zahl** für variable Anstellwinkel – und gibt eine zeitaufgelöste Heatmap aus.

#### 2. **Multi-Constraint Trajectory Optimizer (MCTO)**
Statt einfacher Euler-Integration nutzt artemisreentry einen **Runge-Kutta-Fehlberg-Adaptivschritt (RKF45)** mit bis zu 6 Freiheitsgraden.  
Der Optimierer erlaubt die gleichzeitige Berücksichtigung von **G-Limit**, **Max-Temp-Schwelle** und **Landezonen-Zielkoordinaten** – ideal für Trade-Off-Analysen zwischen Abbremsung und thermischer Belastung.

#### 3. **Native Rust-FFI für Echtzeit-Telemetrie-Streaming**
Über eine **FFI-Schnittstelle** (Foreign Function Interface) kann artemisreentry direkt aus Python oder C++ heraus gesteuert werden.  
Das ermöglicht Live-Streaming von Telemetriedaten (z. B. über ZeroMQ oder gRPC) in existierende Missionskontroll-Dashboards – ohne Serialisierungs-Overhead.

### So verwenden Sie es

#### Schritt 1: Installation und Konfiguration der Atmosphären-Datenbank
```bash
git clone https://github.com/xingfangwang-eng/artemisreentry.git
cd artemisreentry
cargo build --release --features msis-db
```
Laden Sie die MSIS-E-00-Atmosphärendaten herunter und platzieren Sie sie unter `~/.artemisreentry/atmo/`.  
Konfigurieren Sie dann die Startparameter in `config.toml`:

```toml
[entry_profile]
altitude_km = 120.0
velocity_km_s = 7.8
flight_path_deg = -1.2
mass_kg = 2500.0
drag_coefficient = 1.2
heat_capacity_J_per_kgK = 840.0
```

#### Schritt 2: Ausführen einer vollständigen Wiedereintrittssimulation
```bash
artemisreentry run --profile config.toml --output heatmap.csv --log-level debug
```
Das Tool gibt während der Laufzeit jede Sekunde den aktuellen **Heat-Flux (kW/m²)**, die **Oberflächentemperatur (K)** sowie den **aktuellen G-Load (g)** aus.  
Nach Abschluss wird eine CSV-Datei mit allen physikalischen Zuständen über die Zeit generiert.

#### Schritt 3: Analyse und Visualisierung der Ergebnisse
Nutzen Sie die integrierte Export-Funktion für **Matplotlib-kompatible JSON**:
```bash
artemisreentry export --input heatmap.csv --format matplotlib --output trajectory.json
```
Laden Sie die JSON in ein Python-Skript:
```python
import json, matplotlib.pyplot as plt
data = json.load(open('trajectory.json'))
plt.plot(data['time_s'], data['heat_flux_kW_m2'])
plt.xlabel('Zeit [s]')
plt.ylabel('Wärmestrom [kW/m²]')
plt.title('Wiedereintrittsprofil - Artemis Reentry')
plt.grid(True)
plt.show()
```

### Beste Praktiken

#### 1. **Verwenden Sie den MCTO-Modus für Trade-Off-Analysen**
Starten Sie den Optimierer mit mehreren Zielfunktionen (`--objectives max_landing_accuracy,min_peak_g,min_peak_temp`).  
Das Tool gibt eine Pareto-Front aus – ideal für Missionsplaner, die zwischen Landegenauigkeit und Crew-Komfort abwägen müssen.

#### 2. **Integrieren Sie Echtzeit-Telemetrie mit gRPC**
Binden Sie artemisreentry über die FFI in Ihre existierende Missionskontrolle ein (z. B. NASA OpenMCT oder eigene Web-Dashboards).  
Nutzen Sie den mitgelieferten `grpc_server`-Befehl:
```bash
artemisreentry grpc-server --port 50051
```
Damit können Sie live Heat-Flux-Daten streamen und direkt in Echtzeit-Alarme für kritische Schwellenwerte auslösen.
