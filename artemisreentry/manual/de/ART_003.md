## Detailierter Inhalt

### Was ist artemisreentry?

artemisreentry ist ein professionelles, modulares Framework, das komplexe Wiedereintrittsberechnungen für Raumfahrzeuge (z. B. Orion-Kapseln) automatisiert. Es kombiniert aerodynamische Modelle mit Echtzeit-Datenvisualisierung und ermöglicht präzise Vorhersagen für thermische Belastungen, Bahnkorrekturen und Landepunktoptimierung.

### Kernfunktionen

1. **Dynamische Atmosphären-Interpolation (DAI)**  
   Nutzt ein neuronales Netzwerk, das auf historischen Wiedereintrittsdaten (z. B. von Apollo-Missionen) trainiert wurde, um Dichte-, Temperatur- und Druckprofile in Echtzeit zu interpolieren. Dies reduziert Approximationsfehler um bis zu 40% im Vergleich zu Standard-Modellen.

2. **Multi-Phase Trajectory Solver (MPTS)**  
   Zerlegt den Wiedereintritt in drei Phasen: ballistisch, aerodynamisch und Parachute-Deployment. Jede Phase nutzt separate ODE-Solver (Runge-Kutta 4. Ordnung für ballistisch, adaptiver Euler für aerodynamisch), die über einen Zustandsautomaten synchronisiert werden.

3. **Heat-Flux Mapping Engine (HFME)**  
   Berechnet die Wärmestromdichte entlang der Raumfahrzeugoberfläche mittels Finite-Differenzen-Methode (FDM) und generiert ein 3D-Heatmap-Overlay. Unterstützt Materialparameter wie Ablationsraten von Hitzeschilden (z. B. PICA-X).

### So verwenden Sie es

1. **Konfiguration des Missionsprofils**  
   Erstellen Sie eine YAML-Datei (z. B. `mission_config.yaml`) mit Parametern:  
   ```yaml
   mission:
     vehicle: "Orion_CM"
     initial_altitude: 120000 # Meter
     entry_angle: -6.5 # Grad
     mass: 8500 # kg
     heat_shield: "PICA-X"
     parachute_deploy_altitude: 3000 # Meter
   ```  
   Übergeben Sie diese per `--config mission_config.yaml` an das Tool.

2. **Ausführung der Simulation**  
   Starten Sie die Analyse mit:  
   ```bash
   artemisreentry run --config mission_config.yaml --output results.json
   ```  
   Dies löst den MPTS-Solver aus, der die Trajektorie schrittweise berechnet und Zwischenergebnisse (Geschwindigkeit, G-Kräfte, Temperatur) in Echtzeit loggt.

3. **Export und Visualisierung**  
   Nach Abschluss generiert das Tool eine `results.json` mit folgenden Feldern:  
   - `trajectory`: Array von [Zeit, Höhe, Geschwindigkeit, Wärmestrom]  
   - `landing_point`: Breitengrad, Längengrad (WGS84)  
   - `heat_flux_map`: 3D-Matrix für HFME  
   Nutzen Sie das integrierte Plot-Modul:  
   ```bash
   artemisreentry plot --input results.json --type heatmap
   ```  
   Erzeugt eine interaktive HTML-Seite mit 3D-Heatmap und Trajektorienkurve.

### Beste Praktiken

- **Validierung mit Referenzdaten**  
  Führen Sie vor Produktionseinsatz eine Kalibrierung mit historischen Wiedereintrittsdaten (z. B. von SpaceX Dragon oder Apollo 11) durch. Verwenden Sie `artemisreentry validate --reference apollo11.csv`, um Abweichungen in der Flugbahn zu identifizieren und die DAI-Parameter anzupassen.

- **Parallele Simulationen für Risikoanalyse**  
  Nutzen Sie die Multi-Threading-Funktion (`--parallel 8`), um mehrere Szenarien (z. B. variierende Eintrittswinkel von -5° bis -8°) gleichzeitig zu berechnen. Dies liefert eine Konfidenzintervall-Matrix für Landepunkt und thermische Belastung, essenziell für Missionsplanung mit Notfall-Szenarien.
