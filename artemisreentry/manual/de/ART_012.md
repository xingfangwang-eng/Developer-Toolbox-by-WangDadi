# artemisreentry - ART_012

## Übersicht

**artemisreentry** ist ein spezialisiertes Open-Source-Tool zur präzisen Simulation und Optimierung von Wiedereintrittsprofilen für Raumfahrzeuge der Artemis-Generation. Es adressiert die komplexen thermischen, aerodynamischen und navigatorischen Herausforderungen beim Rücktransport von Nutzlasten aus dem cislunaren Raum. Entwickelt für Ingenieure, die hochpräzise, deterministische Analysen benötigen, ohne auf schwergewichtige kommerzielle Suiten angewiesen zu sein.

## Kernfunktionen

1. **Dynamische Hitzefluss-Prognose mit Multi-Modell-Fusion**  
   Integriert semi-empirische Korrelationen (z. B. nach Sutton-Graves) mit einem reduzierten Navier-Stokes-Löser für Echtzeit-Vorhersagen des konvektiven und radiativen Wärmestroms entlang der Flugbahn. Ermöglicht die Fehlerabschätzung durch Ensemble-Methoden direkt im CLI.

2. **Monte-Carlo-gestützte Korridoroptimierung**  
   Simuliert tausende Eintrittspfade unter Berücksichtigung von Unsicherheiten in Atmosphärendichte, Aerodynamik-Koeffizienten und initialen Zustandsvektoren. Liefert probabilistische Grenzen für den zulässigen Eintrittskorridor – essenziell für Missionsplanung unter realen Umweltbedingungen.

3. **Autonome Trajektorienregelung mittels modellprädiktiver Steuerung (MPC)**  
   Implementiert einen Echtzeit-MPC-Controller, der Sollwerte für Bank- und Anstellwinkel generiert, um vorgegebene thermische und strukturelle Lasten einzuhalten. Das Modell nutzt eine linearisierte Zustandsraumdarstellung mit adaptiven Zeitschritten für numerische Stabilität.

## Detaillierte Anwendungsschritte

### 1. Initialisierung des Szenarios und Validierung der Randbedingungen

Legen Sie eine Konfigurationsdatei im YAML-Format an, die alle missionsspezifischen Parameter definiert:

```yaml
mission:
  vehicle: orion_mpcv
  entry_mass_kg: 8800
  ballistic_coefficient: 320
  initial_state:
    altitude_km: 121.9
    velocity_kms: 11.2
    flight_path_deg: -1.5
  atmosphere_model: nrlmsise00
```

Führen Sie dann den integrierten Prüfmechanismus aus:
```bash
artemisreentry validate --config mission_art_012.yaml
```
Dieser Schritt überprüft Konsistenz zwischen den eingegebenen Parametern und physikalischen Grenzen (z. B. maximale dynamische Last < 5 g). Bei Abweichungen werden detaillierte Warnungen mit Korrekturvorschlägen ausgegeben.

### 2. Durchführung einer sensitiven Korridoranalyse

Starten Sie eine Monte-Carlo-Simulation mit 10.000 Läufen zur Bestimmung der Eintrittsunsicherheiten:
```bash
artemisreentry corridor --config mission_art_012.yaml --samples 10000 --output corridor_analysis.csv
```

Das Tool generiert eine CSV-Datei mit folgenden Schlüsselspalten: `max_heat_flux`, `peak_deceleration`, `landing_ellipse_sigma`. Anschließend visualisieren Sie die Ergebnisse:
```bash
artemisreentry plot corridor --input corridor_analysis.csv --heatmap
```
Die Heatmap zeigt die Wahrscheinlichkeitsdichte des Hitzeflussmaximums über dem Eintrittswinkel – kritisch für die Auswahl eines robusten Referenzpfads.

### 3. Implementierung der MPC-gestützten Echtzeitsteuerung

Aktivieren Sie den MPC-Controller für eine deterministische Trajektorie:
```bash
artemisreentry fly --config mission_art_012.yaml --controller mpc --horizon 30 --dt 0.1 --output trajectory_log.bin
```

Während der Simulation gibt das Tool sekündlich aktuelle Werte aus: `current_heat_flux`, `bank_angle_cmd`, `load_factor`. Nach Abschluss exportieren Sie die Logdaten zur Analyse:
```bash
artemisreentry export trajectory_log.bin --format parquet
```
Die Parquet-Datei kann direkt in Pandas oder MATLAB für detaillierte Nachbearbeitung geladen werden – inklusive aller Zwischenzustände des MPC-Optimierungsproblems.

## Best Practices für den industriellen Einsatz

1. **Kopplung mit Echtzeit-Telemetrie für adaptive Missionsführung**  
   Nutzen Sie die bereitgestellte Python-API (`artemisreentry.bindings`), um das Tool in eine bestehende Missionskontrollumgebung zu integrieren. Durch die direkte Einspielung von Telemetriedaten (z. B. GPS/INS-Fusion) kann der MPC-Controller während des Flugs seine Prädiktionen aktualisieren und so auf unerwartete Atmosphärenschwankungen reagieren – eine Fähigkeit, die in rein prädiktiven Tools oft fehlt.

2. **Automatisierte Regressionstests bei Änderungen der Fahrzeuggeometrie**  
   Definieren Sie eine Reihe von Referenzszenarien (z. B. nominal, worst-case thermal, off-nominal entry angle) in einem Testskript. Nach jeder Anpassung der aerodynamischen Datenbank oder der Massenverteilung führen Sie `artemisreentry test --suite regression_suite.yaml` aus. Das Tool vergleicht automatisch die neuen Ergebnisse mit den hinterlegten Baseline-Werten und meldet statistisch signifikante Abweichungen – unverzichtbar für die Qualitätssicherung in iterativen Entwicklungsprozessen.
