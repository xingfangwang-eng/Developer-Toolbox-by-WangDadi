# artemisreentry - ART_004

## Overview

**artemisreentry** is a precision-engineered toolkit for modeling, simulating, and validating atmospheric reentry trajectories and thermal protection system (TPS) margins for lunar return missions. Built for aerospace engineers and mission planners, it bridges the gap between high-fidelity physics and rapid iteration workflows.

## Core Features

### 1. Multi-Phase Trajectory Solver with Real-Time Aerothermal Coupling
Unlike single-shot propagators, artemisreentry integrates a **6-DOF rigid-body dynamics solver** with a **conjugate heat transfer (CHT) model** that updates material ablation rates at every time step. This allows you to visualize how TPS recession affects dynamic stability and heat flux in real time, without needing to stitch separate CFD and trajectory tools.

### 2. Automated Dispersion Analysis via Monte Carlo Sensitivity Funnels
Run **10,000+ perturbed reentry cases** in parallel using our built-in variance reduction engine. The tool automatically identifies which parameters (entry angle, atmospheric density, vehicle mass properties) dominate the dispersion of your landing ellipse or peak heat load. Outputs include tornado charts and probability density functions for risk review boards.

### 3. Native Flight Computer Interface for HITL/VV&A
artemisreentry exposes a **gRPC-based telemetry bridge** that speaks the same binary protocol as many flight-qualified avionics stacks. You can plug in a hardware-in-the-loop (HITL) unit and replay mission-critical “blackout” and “skip” scenarios, validating your guidance, navigation, and control (GNC) logic before committing to metal.

## How to Use

### Step 1: Define Your Reentry Corridor with Custom Atmosphere Profiles
Create a `corridor.yaml` file that specifies the entry interface (EI) altitude, inertial velocity, and flight path angle. For lunar return, set `atmosphere_model: "GRAM-2020"` and override seasonal density perturbations using the `perturbation_scale` parameter. Example snippet:

```yaml
corridor:
  ei_altitude_km: 122.0
  ei_velocity_kms: 11.0
  fpa_deg: -5.8
  atmosphere_model: "GRAM-2020"
  perturbation_scale: 0.95  # 5% below nominal density
```

### Step 2: Configure TPS Material Stack and Run the Solver
Specify the thermal protection system as a layered stack in `tps_stack.json`. Each layer requires a material ID, thickness (mm), and initial temperature. Then invoke the solver:

```bash
artemisreentry solve --corridor corridor.yaml --tps tps_stack.json --output results.h5
```

The solver will produce an HDF5 file containing time-series of heat flux, wall temperature, recession, and aerodynamic coefficients. Use the built-in `plot` command to overlay your results against reference trajectories:

```bash
artemisreentry plot --input results.h5 --overlay reference_traj.csv
```

### Step 3: Execute a Full Dispersion Sweep and Export a Risk Report
Generate a sweep configuration file that defines the statistical distributions for your uncertain parameters. For example, vary EI flight path angle with a Gaussian distribution (mean = -5.8°, σ = 0.2°). Run the sweep with:

```bash
artemisreentry sweep --config sweep_config.yaml --n 10000 --output sweep_results.h5
```

After completion, generate a PDF risk report that includes a landing ellipse probability map, peak heat load histogram, and TPS margin tables:

```bash
artemisreentry report --input sweep_results.h5 --format pdf --out risk_assessment.pdf
```

## Best Practices

### 1. Always Validate Against a Known Flight Dataset
Before trusting your dispersion results, run a single nominal case against publicly available reentry data (e.g., Apollo 4 or Orion EFT-1). Use the `--compare` flag with `artemisreentry solve` to compute the RMS error between your simulation and the reference telemetry. This catches subtle errors in atmosphere interpolation or vehicle aerodynamics early.

### 2. Treat TPS Margin as a Stochastic, Not a Safety Factor
Instead of applying a blanket 1.5× safety factor on TPS thickness, use the dispersion sweep to set a **probabilistic margin target**. For example, configure the sweep to find the 99.7th percentile (3σ) of peak wall temperature, then size your TPS to that value. This reduces unnecessary mass while maintaining the same risk posture. Export the cumulative distribution function (CDF) from the report to support your design review.
