## Core Features

### 1. **Real-Time Atmospheric Drag & Thermal Flux Modeling**
Leverage a hybrid U.S. Standard Atmosphere 1976 / NRLMSISE-00 model to compute real-time drag coefficients, stagnation-point heat flux, and convective heating rates. Supports both blunt-body (Apollo-style) and lifting-body (Shuttle-style) geometries with configurable material ablation profiles.

### 2. **Multi-Phase Trajectory Solver with Adaptive Mesh Refinement (AMR)**
Deploy a Runge-Kutta 4(5) adaptive integrator that automatically refines time steps around peak dynamic pressure (Max Q) and plasma blackout windows. Outputs are compatible with GMAT, STK, and custom Monte Carlo dispersion analysis pipelines.

### 3. **Automated Landing Zone & Recovery Corridor Optimization**
Given a target landing ellipse (e.g., Woomera, Edwards AFB, or Pacific splashdown), the tool computes optimal bank-angle modulation profiles and skip-entry corridors using a direct-shooting method. Includes real-time wind shear and GPS-denied navigation drift compensation.

## How to Use

### Step 1: Define the Entry State Vector & Vehicle Parameters
Initialize the simulation by specifying the Entry Interface (EI) state: altitude (typically 120 km), velocity (km/s), flight-path angle (gamma), and heading. Then load a JSON-based vehicle profile that includes mass, reference area, drag coefficient (Cd), lift-to-drag ratio (L/D), and nose radius for thermal calculations.

**Example snippet:**
```json
{
  "vehicle": "Orion_MPCV",
  "mass_kg": 10400,
  "ref_area_m2": 19.6,
  "Cd": 1.45,
  "L_over_D": 0.35,
  "nose_radius_m": 0.6
}
```

### Step 2: Run the Trajectory Solver with Constraint Tolerances
Execute the solver with user-defined tolerances for dynamic pressure (max 30 kPa for crewed missions), heat rate (max 150 W/cm²), and deceleration (max 5 G). The AMR engine logs every discontinuity. Output is a time-stamped CSV with columns for altitude, velocity, acceleration, heat flux, and plasma frequency.

### Step 3: Export & Visualize the Reentry Corridor
Generate a 2D/3D visualization of the trajectory overlaying the Earth’s curvature, atmospheric layers, and the target landing ellipse. Export the corridor as a KML file for Google Earth or as a MATLAB/NumPy array for post-processing. Use the built-in Monte Carlo module to run 1,000+ perturbed cases and produce a 99.7% dispersion ellipse.

## Best Practices

### 1. **Calibrate the Atmospheric Model Against Real Flight Data**
Before running mission-critical simulations, validate the atmospheric model using archived telemetry from Apollo, Shuttle, or SpaceX Dragon reentries. Adjust the solar flux index (F10.7) and geomagnetic index (Ap) to match the intended mission epoch. This reduces landing zone error by up to 15%.

### 2. **Use a Staged Optimization Workflow for Skip-Entry Profiles**
For high-energy lunar or Mars return trajectories, never solve the entire skip-entry profile in one pass. Instead, break it into three phases: (1) initial descent to 80 km, (2) skip-out to 120 km, (3) final descent. Optimize each phase independently, then stitch them with a cubic spline. This avoids singularities in the Jacobian and yields a 30% faster convergence.
