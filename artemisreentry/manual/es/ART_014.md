# artemisreentry - ART_014

## Resumen Técnico

Herramienta de simulación y análisis de reentrada atmosférica para misiones Artemis. Optimizada para ingenieros aeroespaciales que necesitan cálculos precisos de trayectorias balísticas, cargas térmicas y ventanas de recuperación en tiempo real.

## Funcionalidades Principales

### 1. Simulador de Trayectoria Balística con Corrección en Vuelo
Motor de simulación que integra modelos atmosféricos reales (NRLMSISE-00) y coeficientes aerodinámicos variables. Permite ajustar parámetros de reentrada como ángulo de inclinación, velocidad inicial y coeficiente de arrastre en tiempo real, visualizando la dispersión de impacto con precisión sub-kilométrica.

### 2. Análisis de Cargas Térmicas con Materiales Compuestos
Módulo de cálculo de flujo de calor convectivo y radiante sobre escudos térmicos ablativos (PICA, SLA-561V). Genera mapas de temperatura superficial y profundidad de ablación, permitiendo validar diseños de protección térmica para cápsulas Orion y módulos de servicio.

### 3. Optimización de Ventanas de Recuperación con Algoritmos Genéticos
Algoritmo que evalúa múltiples escenarios de reentrada (inclinación, altitud de inicio, velocidad de frenado) para maximizar la probabilidad de aterrizaje en zonas de recuperación predefinidas. Ideal para misiones con restricciones de tiempo y ubicación.

## Cómo Usar

### 1. Configurar Parámetros de Reentrada
Ejecuta el script principal con argumentos de línea de comandos o un archivo JSON de configuración:
```bash
python reentry_sim.py --config configs/artemis_014.json
```
El archivo debe incluir: `initial_altitude_km`, `entry_angle_deg`, `velocity_km_s`, `mass_kg`, `drag_coefficient` y `heat_shield_material`.

### 2. Ejecutar Simulación y Visualizar Resultados
Tras la simulación, el sistema genera un reporte HTML interactivo con gráficos de altitud vs tiempo, velocidad vs tiempo, y mapa de dispersión de impacto. Para exportar datos a CSV:
```python
from artemisreentry import SimulationEngine
sim = SimulationEngine.from_config('config.json')
results = sim.run()
results.export_csv('reentry_data.csv')
```

### 3. Analizar Cargas Térmicas y Optimizar Ventanas
Usa el módulo `thermal_analysis` para validar el escudo térmico:
```python
from artemisreentry import ThermalAnalyzer
ta = ThermalAnalyzer(results.trajectory)
ta.plot_heat_flux_profile(save_path='heat_flux.png')
ta.optimize_recovery_window(algorithm='genetic', population=100, generations=50)
```

## Mejores Prácticas

### 1. Validar con Datos de Misiones Reales
Compara tus simulaciones con telemetría de misiones Artemis previas (Artemis I, II). Ajusta los coeficientes de arrastre y modelos atmosféricos usando los logs de vuelo disponibles en el repositorio de datos abiertos de la NASA. Esto reduce el error de predicción en más del 15%.

### 2. Usar Versionado de Configuraciones
Mantén un repositorio Git con todos los archivos de configuración (`configs/`) y scripts de simulación. Cada ejecución debe ser reproducible. Recomendamos etiquetar cada simulación con el hash del commit y la fecha UTC para auditorías posteriores.
