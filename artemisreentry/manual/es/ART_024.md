# artemisreentry - ART_024

## Resumen

Herramienta de análisis de trayectorias de reentrada orbital diseñada para misiones Artemis. Calcula perfiles de retorno atmosférico con precisión de submilisegundo, integrando modelos de resistencia aerodinámica y puntos de aterrizaje optimizados para vehículos de exploración lunar.

## Contenido Detallado

### ¿Qué es artemisreentry?

artemisreentry es un simulador de reentrada espacial que resuelve problemas de predicción de impacto térmico y estructural en cápsulas de retorno. A diferencia de herramientas genéricas, implementa algoritmos de propagación de estado basados en ecuaciones diferenciales de alta fidelidad, adaptados a los perfiles de misión de la NASA y SpaceX.

### Características Principales

- **Modelo de resistencia aerodinámica con variación de densidad atmosférica**: Utiliza datos de la atmósfera estándar de EE.UU. 1976 para calcular la desaceleración en tiempo real, considerando la latitud y altitud del punto de reentrada. Esto permite predecir picos de calentamiento con error menor al 2%.

- **Optimización de punto de aterrizaje mediante algoritmos genéticos**: Implementa un optimizador multiobjetivo que ajusta el ángulo de ataque y la velocidad de descenso para minimizar la desviación del punto de impacto (máximo 500 metros de error). Ideal para misiones que requieren aterrizajes precisos en zonas remotas.

- **Visualización 3D de trayectorias con integración de datos de telemetría**: Renderiza en tiempo real la trayectoria de reentrada usando Three.js, permitiendo superponer datos de sensores reales (acelerómetros, giroscopios) para validar modelos. Incluye un módulo de exportación a CSV para análisis posterior.

### Cómo Usar

1. **Configuración del perfil de misión**: Define los parámetros de reentrada en el archivo `config.json`, incluyendo la masa del vehículo (kg), el coeficiente de arrastre (CD) y la altitud inicial (km). Ejemplo para la cápsula Orion: `{"mass": 8500, "cd": 1.2, "altitude_initial": 120}`. Luego ejecuta `python artemisreentry.py --config config.json` para iniciar la simulación.

2. **Ejecución de la simulación con ajuste de precisión**: Usa la bandera `--precision high` para activar el solver Runge-Kutta de cuarto orden (RK4) que reduce el error de integración a 0.1 segundos. Monitorea la salida en terminal con `--verbose`, que muestra cada 10 segundos la velocidad actual (m/s) y la temperatura de la capa límite (K). Para detener la simulación antes de tiempo, presiona Ctrl+C y el sistema guarda automáticamente el estado en `checkpoint.pkl`.

3. **Análisis de resultados con visualización interactiva**: Después de la simulación, genera un gráfico 3D de la trayectoria ejecutando `python plot_trajectory.py --input output.csv`. Puedes rotar el modelo con el ratón y hacer clic en puntos específicos para ver detalles (altitud, velocidad, tiempo). Exporta los datos a PDF con `--export pdf` para informes técnicos.

### Mejores Prácticas

- **Validación cruzada con datos de misiones reales**: Antes de usar los resultados para una misión crítica, compara las predicciones de artemisreentry con datos de telemetría de misiones similares (ej. Apollo 13, Crew Dragon Demo-2). Ajusta los coeficientes de arrastre en `config.json` para minimizar el error cuadrático medio (RMSE) entre simulación y realidad.

- **Uso de versionado semántico para configuraciones**: Guarda cada configuración de simulación en un repositorio Git con un tag semántico (ej. `v1.2.3`). Esto permite rastrear cambios en parámetros como la masa del vehículo o el modelo atmosférico, facilitando la reproducibilidad de resultados en equipos de ingeniería distribuidos.
