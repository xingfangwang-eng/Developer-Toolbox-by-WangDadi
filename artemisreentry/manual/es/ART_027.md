# artemisreentry - ART_027

## Resumen Ejecutivo

Artemisreentry es una herramienta de análisis de reentrada atmosférica diseñada para ingenieros aeroespaciales y desarrolladores que trabajan con misiones de retorno de carga útil. Este proyecto optimiza la simulación de trayectorias balísticas, la gestión de perfiles térmicos y la validación de datos de telemetría en tiempo real. Con un enfoque modular y escalable, Artemisreentry se integra directamente en pipelines de CI/CD para entornos de simulación espacial.

## Contenido Detallado

### ¿Qué es artemisreentry?

Artemisreentry es un framework de código abierto para modelar, simular y validar reentradas atmosféricas de vehículos espaciales. A diferencia de herramientas monolíticas, Artemisreentry utiliza un motor de eventos asíncrono (basado en Rust) para procesar datos de telemetría en tiempo real, combinado con un backend en Python para análisis estadísticos. Está pensado para equipos que necesitan reducir el tiempo de iteración en pruebas de reentrada, desde simulaciones Monte Carlo hasta ajuste de parámetros de guiado.

### Características Principales

1. **Motor de Eventos Asíncrono para Reentrada en Tiempo Real**  
   Implementa un pipeline de procesamiento de datos basado en Rust con canales MPMC (Multiple Producer, Multiple Consumer) que maneja hasta 10,000 eventos por segundo. Esto permite simular condiciones de reentrada con fluctuaciones de presión y temperatura sin pérdida de paquetes de telemetría. Ideal para pruebas de estrés en sistemas de guiado autónomo.

2. **Simulación Térmica Multifase con Modelos de Ablación**  
   Integra un solver de elementos finitos (FEM) optimizado para materiales ablativos, como PICA (Phenolic Impregnated Carbon Ablator) y SLA (Silicone Impregnated Reusable Ceramic Ablator). El módulo calcula la tasa de ablación en tiempo real basándose en flujos de calor convectivo y radiante, y exporta gráficos de evolución de espesor de material en formato SVG interactivo.

3. **Validación Automática de Trayectorias con Regresión de Monte Carlo**  
   Genera automáticamente 10,000 trayectorias de reentrada con perturbaciones aleatorias en parámetros clave (velocidad inicial, ángulo de entrada, densidad atmosférica). Luego, aplica un algoritmo de regresión logística para identificar las trayectorias que exceden los límites de seguridad térmica o estructural. Los resultados se muestran en un dashboard de Plotly con curvas de probabilidad de fallo.

### Cómo Usar

1. **Configuración del Pipeline de Simulación**  
   Clona el repositorio y ejecuta `cargo build --release` para compilar el motor de eventos. Luego, edita el archivo `config/simulation.toml` para definir los parámetros de reentrada: masa del vehículo, coeficiente de arrastre, y perfil de temperatura de la atmósfera. Ejemplo de configuración básica:
   ```toml
   [vehicle]
   mass_kg = 2500
   drag_coefficient = 1.2
   initial_velocity_kms = 7.8
   entry_angle_deg = 15.3
   ```
   Ejecuta `python run_simulation.py --config config/simulation.toml` para iniciar la simulación. El sistema mostrará en consola el progreso de cada fase de reentrada (exoatmosférica, ionización, ablación).

2. **Análisis de Datos con el Módulo de Regresión**  
   Después de la simulación, usa el script `scripts/regression_analysis.py` para procesar los archivos CSV generados en `/output/trajectories`. El script aplica una regresión logística con regularización L2 para identificar las trayectorias de alto riesgo. Ejecuta:
   ```bash
   python scripts/regression_analysis.py --input output/trajectories/*.csv --threshold 0.85
   ```
   El resultado es un archivo `risk_report.html` con gráficos de distribución de probabilidad de fallo y una tabla de las 10 trayectorias más críticas.

3. **Integración en Pipelines de CI/CD**  
   Para equipos que usan GitHub Actions, añade el siguiente workflow en `.github/workflows/reentry_test.yml`:
   ```yaml
   name: Reentry Simulation Test
   on: [push]
   jobs:
     simulate:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - name: Build and Simulate
           run: |
             cargo build --release
             python run_simulation.py --config config/simulation.toml
         - name: Validate Results
           run: |
             python scripts/regression_analysis.py --input output/trajectories/*.csv --threshold 0.85
   ```
   Esto ejecuta automáticamente una simulación de reentrada completa en cada push, asegurando que los cambios no introduzcan riesgos de fallo térmico.

### Mejores Prácticas

1. **Optimización de Parámetros para Simulaciones de Alta Fidelidad**  
   Para obtener resultados precisos, ajusta el coeficiente de arrastre (`drag_coefficient`) basándote en datos de viento reales de la misión (por ejemplo, del modelo MSIS-E-90). Además, usa el flag `--mesh-resolution high` en la simulación para activar el solver FEM con mallas de 0.5 mm, lo que reduce el error de ablación a menos del 2%. Esto es crítico cuando se validan materiales para misiones de reentrada tripuladas.

2. **Estrategia de Pruebas con Datos de Telemetría Reales**  
  Antes de desplegar en producción, importa archivos de telemetría reales (formato CSV o Parquet) usando el comando `python scripts/import_telemetry.py --file telemetry.csv`. Luego, ejecuta la simulación con el flag `--real-telemetry` para comparar los resultados simulados vs. reales. Esto permite identificar desviaciones en el modelo de arrastre o en la respuesta térmica del vehículo. Para misiones críticas, se recomienda realizar al menos 100 iteraciones con diferentes conjuntos de datos históricos.
