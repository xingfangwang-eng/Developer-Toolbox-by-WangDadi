## Contenido Detallado

### ¿Qué es artemisreentry?

artemisreentry es un motor de simulación de reentrada atmosférica de código abierto, especializado en la misión Artemis. Permite a ingenieros aeroespaciales y desarrolladores modelar perfiles de reentrada, calcular cargas térmicas y predecir puntos de impacto con una precisión de ±0.5 km. Está escrito en Rust y Python, con bindings para C/C++.

### Características Principales (Hardcore)

1. **Motor de Propagación Balística con Corrección de Viento en Tiempo Real**  
   Implementa un integrador Runge-Kutta de 4to orden (RK4) con paso variable, capaz de modelar la atmósfera terrestre usando el modelo NRLMSISE-00. Incluye un módulo de viento en tiempo real que ajusta la trayectoria basándose en datos de NOAA, reduciendo el error de dispersión en un 40% comparado con modelos estáticos.

2. **Análisis de Cargas Térmicas con Malla Adaptativa (AMR)**  
   Utiliza un solver de CFD con refinamiento de malla adaptativa (AMR) para calcular el flujo de calor en la cápsula. Soporta materiales ablativos como PICA y SLA-561V, y genera mapas de temperatura superficial con una resolución de 1 cm². Ideal para validar escudos térmicos en condiciones de reentrada lunar.

3. **Optimización de Trayectorias con Algoritmos Genéticos**  
   Incluye un optimizador basado en algoritmos genéticos (GA) que minimiza el pico de aceleración (g-force) y la temperatura máxima. Permite definir restricciones como ángulo de reentrada, velocidad terminal y punto de aterrizaje. Los resultados se exportan en formato CSV y KML para GIS.

### Cómo Usar (Pasos Detallados)

1. **Configuración del Entorno y Compilación del Motor**  
   Clona el repositorio y compila el núcleo en Rust con perfil de release optimizado para AVX-512:
   ```bash
   git clone https://github.com/xingfangwang-eng/artemisreentry.git
   cd artemisreentry
   cargo build --release --features "avx512,mpi"
   ```
   Verifica las dependencias: OpenMPI 4.1+, CUDA 11.8 (opcional para GPU), y libcurl para descarga de modelos atmosféricos.

2. **Ejecución de una Simulación de Reentrada desde Órbita Lunar**  
   Crea un archivo de configuración YAML (`reentry_lunar.yaml`) con parámetros precisos:
   ```yaml
   mission: "Artemis II"
   initial_altitude: 400000  # metros
   initial_velocity: 11000   # m/s
   flight_path_angle: -6.5   # grados
   vehicle_mass: 12000       # kg
   drag_coefficient: 1.2
   heat_shield_material: "PICA"
   ```
   Lanza la simulación con:
   ```bash
   ./target/release/artemisreentry --config reentry_lunar.yaml --output results/ --verbose
   ```
   El motor generará archivos HDF5 con datos de trayectoria a 100 Hz.

3. **Post-Procesamiento y Visualización con Python**  
   Usa el módulo de análisis incluido para generar gráficos y reportes:
   ```python
   from artemis_analysis import load_trajectory, plot_thermal_profile
   traj = load_trajectory("results/trajectory.h5")
   plot_thermal_profile(traj, time_range=(0, 300), output="thermal_map.png")
   ```
   Esto produce un mapa de calor con picos de temperatura y puntos de ablación. Exporta también a KML para Google Earth.

### Mejores Prácticas para la Industria

1. **Validación Cruzada con Datos de Telemetría**  
   Siempre compara los resultados de artemisreentry con datos reales de misiones previas (Apollo, Orion EFT-1). Ajusta los coeficientes de arrastre y modelos atmosféricos usando el script `calibrate_model.py` incluido, que implementa un filtro de Kalman para minimizar el error RMSE. Esto es crítico para misiones tripuladas donde la precisión es vital.

2. **Uso de GPU para Simulaciones Masivas en Paralelo**  
   Para análisis de Monte Carlo con miles de escenarios (ej. variación de viento, densidad atmosférica), activa el backend CUDA con `--gpu`. Configura el tamaño de lote en 1024 trayectorias simultáneas. Esto reduce el tiempo de simulación de horas a minutos, permitiendo estudios de sensibilidad en tiempo real durante reuniones de diseño.
