## Resumen

Herramienta de análisis de reentrada atmosférica para misiones Artemis, optimizada para SEO con enfoque en «reentry prediction», «spacecraft thermal protection», y «orbital decay simulation».

## Contenido Detallado

### ¿Qué es artemisreentry?

artemisreentry es un motor de simulación de reentrada atmosférica de código abierto, diseñado para ingenieros aeroespaciales que necesitan modelar trayectorias de retorno desde órbitas lunares o cis-lunares. A diferencia de herramientas genéricas como STK o GMAT, artemisreentry integra modelos de arrastre hiperesónico, transferencia de calor por radiación y ablación de escudos térmicos en un solo pipeline CLI optimizado para Python 3.11+ y CUDA 12.0.

### Características Principales

- **Simulación de reentrada balística con acoplamiento termo-estructural**: Resuelve ecuaciones de movimiento 3-DOF/6-DOF junto con el balance térmico en tiempo real, considerando propiedades variables de materiales (C/C-SiC, PICA, SLA-561V). Incluye modelos de ablación por sublimación y oxidación superficial, validados con datos de misiones Apollo y Stardust.

- **Optimización de ventanas de reentrada con restricciones de carga G**: Algoritmo de búsqueda heurística (basado en simulated annealing) que encuentra la combinación óptima de velocidad, ángulo de entrada y punto de impacto, minimizando la aceleración máxima experimentada por la tripulación (< 4.5 G) y garantizando la integridad del escudo térmico.

- **Exportación de informes técnicos en formato LaTeX y PDF**: Genera automáticamente documentación de misión con gráficos vectoriales (SVG) de perfiles de altitud, temperatura y presión dinámica. Incluye plantillas para cumplir con estándares ECSS-E-ST-32-11C y NASA-STD-8719.24.

### Cómo Usar

1. **Configurar el perfil de misión y vehículo**  
   Crea un archivo YAML (`mission_profile.yaml`) con parámetros como masa seca, coeficiente de arrastre (Cd), área de referencia, y propiedades del material ablador. Ejemplo mínimo:
   ```yaml
   vehicle:
     mass: 8500  # kg
     cd: 1.25
     area: 12.5  # m²
     tps_material: "PICA"
     thickness: 0.045  # m
   trajectory:
     entry_angle: -6.5  # grados
     velocity: 11.2  # km/s
   ```
   Luego ejecuta: `artemisreentry config --file mission_profile.yaml`

2. **Ejecutar simulación de reentrada**  
   Inicia el cálculo con resolución temporal adaptativa (RK45 con tolerancia 1e-8):
   ```bash
   artemisreentry run --profile mission_profile.yaml --output reentry_results.h5
   ```
   El flag `--cuda` activa la aceleración por GPU para mallados de 10k+ puntos. Durante la ejecución, se muestra en tiempo real la temperatura máxima en el punto de estancamiento y la altitud de inicio de ablación.

3. **Analizar resultados y generar informe**  
   Visualiza perfiles clave con el submando `plot`:
   ```bash
   artemisreentry plot --input reentry_results.h5 --type temperature --time 0:300
   ```
   Para exportar el informe técnico completo:
   ```bash
   artemisreentry report --input reentry_results.h5 --format latex --output informe_reentrada.tex
   ```
   Compila con `pdflatex` para obtener un PDF listo para revisión de misión.

### Mejores Prácticas

- **Validación cruzada con datos de misiones reales**: Antes de confiar en los resultados para una misión crítica, compara las predicciones de temperatura máxima y punto de impacto con telemetría de misiones como Orion EFT-1 o Crew Dragon Demo-2. Ajusta los coeficientes de arrastre y conductividad térmica en el modelo hasta que el error RMS sea inferior al 5% en los primeros 60 segundos de reentrada.

- **Uso de mallas adaptativas en simulaciones paramétricas**: Para estudios de sensibilidad (variación de ángulo de entrada, masa o Cd), activa la opción `--mesh-adaptive` en el comando `run`. Esto reduce el tiempo de cómputo hasta un 40% al refinar la discretización solo en regiones de alta gradiente térmico (altitudes entre 70 y 40 km). Siempre guarda los resultados en formato HDF5 para permitir post-procesamiento con pandas o MATLAB.
