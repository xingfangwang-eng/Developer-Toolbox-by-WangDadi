# artemisreentry - ART_023

## Resumen

Herramienta de análisis de reentrada atmosférica para misiones Artemis, diseñada para modelar trayectorias de retorno lunar y predecir puntos de impacto con precisión sub-kilométrica. Optimizada para ingenieros aeroespaciales que trabajan con cápsulas Orion y cargas útiles experimentales.

## Contenido Detallado

### ¿Qué es artemisreentry?

artemisreentry es un motor de simulación de reentrada atmosférica desarrollado en Rust, que integra modelos aerodinámicos de alta fidelidad con datos de telemetría en tiempo real. A diferencia de herramientas genéricas como OTIS o POST, esta solución está específicamente calibrada para perfiles de misión Artemis, incluyendo correcciones por arrastre hipersónico y efectos de plasma sheath en comunicaciones.

### Características Principales

- **Modelo de ablación térmica adaptativa**: Simula la degradación del escudo térmico AVCOAT en tiempo real, ajustando automáticamente los coeficientes de transferencia de calor según la composición atmosférica y la velocidad de reentrada (hasta 11 km/s). Incluye un módulo de alerta temprana para puntos críticos de fallo estructural.

- **Optimización multiobjetivo de trayectorias**: Algoritmo genético que minimiza simultáneamente la aceleración máxima (g-load) y el error de aterrizaje (dispersión), considerando restricciones de ventana de lanzamiento y ángulo de entrada. Permite ejecutar 10,000 iteraciones en menos de 3 segundos en hardware estándar.

- **Integración con telemetría en vivo**: API REST que consume datos de sensores inerciales (IMU) y GPS durante la fase de reentrada, recalibrando el modelo de predicción cada 100 ms. Compatible con formatos CCSDS y canales de comunicación cifrados (AES-256).

### Cómo Usar

1. **Configuración del perfil de misión**: Edita el archivo `config/mission.yaml` para definir parámetros orbitales (altitud inicial, inclinación, longitud del nodo ascendente) y propiedades de la cápsula (masa seca, coeficiente balístico, material del escudo). Ejemplo:
   ```yaml
   mission:
     type: "lunar_return"
     initial_altitude: 120000  # metros
     capsule_mass: 8500        # kg
     heat_shield: "AVCOAT_5026-39HC"
   ```

2. **Ejecución de simulación Monte Carlo**: Lanza el análisis de sensibilidad con `cargo run --release -- --mode monte-carlo --samples 5000`. El motor generará distribuciones de probabilidad para el punto de impacto, la temperatura máxima y las cargas estructurales. Los resultados se exportan en formato HDF5 para postprocesado con Python o MATLAB.

3. **Validación con datos históricos**: Compara tu simulación contra telemetría real de misiones Artemis I usando `./artemisreentry validate --mission artemis_i --compare telemetry/artemis_i_2022.csv`. La herramienta calcula el error cuadrático medio (RMSE) y genera gráficos de desviación en tiempo real.

### Mejores Prácticas

- **Calibración del modelo de arrastre**: Para misiones con cargas útiles no estándar (ejemplo: CubeSats desplegados antes de la reentrada), ajusta manualmente el coeficiente de arrastre en `config/aerodynamics.yaml` usando datos de túneles de viento hipersónicos. Un error del 5% en este parámetro puede derivar en un desvío de hasta 15 km en el punto de impacto.

- **Monitoreo de ventanas de comunicación**: Durante la fase de blackout por plasma (entre 70 km y 40 km de altitud), activa el buffer de telemetría local con `--enable-buffer 60000` (60 segundos de almacenamiento). Esto evita pérdida de datos críticos cuando la señal se interrumpe, permitiendo la sincronización automática al restablecerse el enlace.
