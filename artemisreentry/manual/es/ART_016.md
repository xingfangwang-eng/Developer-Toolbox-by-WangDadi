# artemisreentry - ART_016

## Resumen Ejecutivo

Herramienta de análisis de reentrada atmosférica diseñada específicamente para misiones Artemis. Ofrece simulación térmica en tiempo real, predicción de puntos de impacto con precisión sub-kilométrica y validación de trayectorias balísticas para vehículos de retorno lunar. Ideal para ingenieros aeroespaciales que necesitan reducir iteraciones de diseño en un 40%.

## Contenido Detallado

### ¿Qué es artemisreentry?

artemisreentry es un *framework* de simulación numérica que modela el comportamiento de cápsulas durante la reentrada en la atmósfera terrestre desde órbitas translunares. A diferencia de herramientas genéricas, este sistema integra datos de telemetría real de misiones Apollo y Orion, permitiendo calibrar coeficientes de arrastre y transferencia de calor con errores inferiores al 2%. Soporta perfiles de misión con velocidades de hasta 11 km/s y ángulos de reentrada variables entre -5° y -8°.

### Características Principales (Hardcore)

- **Motor de ablación térmica adaptativa**  
  Simula la degradación del escudo térmico PICA-X en tiempo real, considerando flujo de calor por radiación y convección. Algoritmo de Runge-Kutta de 4to orden con paso variable para precisión en zonas de plasma. Incluye base de datos de materiales con 12 variantes de carbono-fenólico.

- **Optimizador de trayectorias con restricciones G**  
  Resuelve el problema de dos puntos límite (Luna-Tierra) usando el método de disparo múltiple con gradiente reducido. Calcula automáticamente el punto de inicio de frenado atmosférico para mantener cargas G por debajo de 3.5g, incluso con vientos de alta altitud (modelo HWM-14 integrado).

- **Corrector de deriva por viento estratosférico**  
  Algoritmo de Monte Carlo con 10,000 iteraciones que modela perturbaciones de viento entre 30 y 50 km de altitud. Genera elipsoides de error del 95% para el punto de impacto, con resolución de 50 metros. Se conecta directamente a APIs de NOAA para datos meteorológicos en tiempo real.

### Cómo Usar (Pasos Avanzados)

1. **Configurar perfil de misión desde archivo YAML**  
   Crea un archivo `mission_profile.yaml` con los siguientes parámetros obligatorios:  
   ```yaml
   entry_velocity: 10800  # m/s
   entry_angle: -6.2      # grados
   mass: 8500             # kg
   heat_shield: "PICA-X_v3"
   ```
   Ejecuta `artemisreentry --load mission_profile.yaml` para validar la sintaxis. El sistema verificará automáticamente que la velocidad esté dentro del rango de reentrada lunar (10.5-11.2 km/s).

2. **Ejecutar simulación con monitoreo en vivo**  
   Lanza la simulación con `artemisreentry --simulate --live-plot`. Se abrirá una ventana con 4 gráficos en tiempo real: altitud vs velocidad, flujo de calor vs tiempo, carga G vs tiempo y desviación del punto de impacto. Presiona `Ctrl+C` para pausar y exportar datos a CSV. Para un análisis paramétrico, usa `--sweep entry_angle: -5.5,-6.5,0.1` que ejecutará 11 simulaciones automáticas.

3. **Analizar resultados con post-procesador**  
   Tras la simulación, ejecuta `artemisreentry --analyze output_20231027_1430.csv --heatmap`. Esto generará un mapa de calor 2D de la distribución de temperatura en el escudo térmico. Para obtener el punto de impacto más probable, usa `--landing-zone` que calcula la elipse de confianza del 95% y la exporta como archivo KML para Google Earth. Incluye la opción `--compare apollo_11.csv` para superponer la trayectoria simulada con datos históricos.

### Mejores Prácticas para el Sector

- **Integración continua con CI/CD aeroespacial**  
  Configura un pipeline en GitLab que ejecute `artemisreentry --test --regression` cada vez que modifiques el perfil de misión. Esto asegura que cambios en parámetros como el coeficiente de arrastre no degraden la precisión del punto de impacto. Usa la flag `--fail-on-drift 50` para que el pipeline falle si la desviación supera los 50 metros respecto a la simulación de referencia.

- **Validación cruzada con datos de vuelo reales**  
  Para misiones críticas, no confíes solo en simulaciones. Usa artemisreentry junto con datos de telemetría de misiones anteriores. Descarga archivos CSV de la base de datos pública de NASA (por ejemplo, Orion EFT-1) y ejecuta `artemisreentry --calibrate eft1_data.csv`. La herramienta ajustará automáticamente los coeficientes aerodinámicos para minimizar el error RMSE entre la simulación y los datos reales. Guarda estos coeficientes en `calibration/eft1_calib.json` para reutilizarlos.
