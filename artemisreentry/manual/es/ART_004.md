# artemisreentry - ART_004

## Resumen Ejecutivo

Artemis Reentry es una herramienta de análisis de reentrada atmosférica diseñada específicamente para misiones lunares y de espacio profundo. Optimizada para ingenieros aeroespaciales, permite modelar trayectorias de retorno con precisión suborbital, calcular cargas térmicas y predecir puntos de impacto con datos en tiempo real. Este proyecto integra algoritmos de propagación orbital avanzados y simulaciones de ablación térmica, todo en un entorno de línea de comandos eficiente.

## Características Principales

### 1. Simulación de Reentrada con Perfil de Ablación Adaptativa
Implementa un modelo de ablación térmica que ajusta dinámicamente la tasa de pérdida de masa del escudo térmico según la densidad atmosférica y la velocidad de reentrada. Utiliza datos de la atmósfera estándar internacional (ISA) y permite configurar materiales compuestos (PICA, SLA-561V) para simular escenarios de misión reales. Los resultados incluyen curvas de temperatura superficial y flujo de calor en tiempo real.

### 2. Optimización de Trayectoria con Algoritmo de Busqueda de Gradiente Conjugado
Incorpora un optimizador de trayectoria que minimiza el pico de aceleración (g-load) y la energía cinética residual en el punto de impacto. El algoritmo ajusta el ángulo de entrada y el coeficiente de sustentación (L/D) para cumplir con restricciones de aterrizaje en zonas específicas (océano, desierto). Soporta múltiples puntos de referencia orbitales (TLE) y calcula ventanas de reentrada con precisión de segundos.

### 3. Diagnóstico de Fallos con Análisis de Sensibilidad Monte Carlo
Ejecuta simulaciones de Monte Carlo para evaluar la robustez del perfil de reentrada frente a variaciones en parámetros críticos (densidad atmosférica, coeficiente de arrastre, masa inicial). Genera mapas de calor de probabilidad de impacto y alertas tempranas para desviaciones que excedan márgenes de seguridad (e.g., 3σ). Ideal para revisiones de diseño de misión (PDR/CDR).

## Cómo Usar

### 1. Configurar el Perfil de Misión y Materiales
Ejecuta el comando `artemisreentry --mission-profile lunar_return --heat-shield PICA`. Esto carga un perfil predeterminado para retorno lunar (velocidad de entrada ~11 km/s, ángulo de -6.5°). Para personalizar, edita el archivo YAML `config/reentry_config.yaml` y define parámetros como `initial_mass: 8500 kg`, `heat_shield_thickness: 0.045 m`, y `atmosphere_model: "MSIS-00"`. Luego, valida la configuración con `artemisreentry --validate-config`.

### 2. Ejecutar Simulación y Visualizar Resultados
Inicia la simulación con `artemisreentry run --output-dir ./results --verbose`. El motor genera archivos CSV con datos de altitud, velocidad, flujo de calor y ablación cada 0.1 segundos. Para visualizar, usa `artemisreentry plot --type thermal_profile --show`. Esto produce un gráfico interactivo con curvas de temperatura vs. tiempo y puntos críticos de ablación. Si prefieres datos crudos, exporta a JSON con `--format json`.

### 3. Realizar Análisis de Sensibilidad y Optimización
Para un estudio de robustez, ejecuta `artemisreentry monte-carlo --samples 10000 --parameters density, drag_coefficient --output sensitivity_report.pdf`. El comando genera un histograma de desviaciones de impacto y un informe PDF con percentiles 5%, 50% y 95%. Luego, optimiza la trayectoria con `artemisreentry optimize --target-lat 35.0 --target-lon -120.5 --max-g-load 4.5`. El optimizador itera hasta converger en un perfil que cumpla las restricciones (típicamente < 100 iteraciones).

## Mejores Prácticas

### 1. Validar Perfiles con Datos de Misiones Históricas
Antes de usar en un diseño real, compara tus simulaciones con datos de misiones como Apollo 11 o Orion EFT-1. Descarga archivos TLE de reentradas pasadas desde el catálogo de la NASA y ejecuta `artemisreentry --compare-mission apollo_11`. Si la desviación en el punto de impacto supera 5 km, ajusta los coeficientes de arrastre en `config/drag_coefficients.yaml`. Esto asegura que el modelo refleje la física real.

### 2. Integrar con Sistemas de Monitoreo en Tiempo Real
Para operaciones de misión, conecta Artemis Reentry a APIs de seguimiento orbital (e.g., Space-Track.org) mediante el script `scripts/live_tracking.py`. Configura la actualización automática de TLE cada 30 minutos y ejecuta `artemisreentry real-time --stream`. Esto permite recalcular la trayectoria de reentrada en vivo y emitir alertas si el punto de impacto previsto se desvía de la zona segura. Ideal para centros de control de misión.
