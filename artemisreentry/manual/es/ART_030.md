# artemisreentry - ART_030

## Resumen Ejecutivo

Herramienta de análisis de reentrada atmosférica para misiones Artemis, diseñada para modelar trayectorias balísticas y evaluar riesgos térmicos en vehículos espaciales de retorno. Optimizada para ingenieros aeroespaciales que trabajan con datos de telemetría en tiempo real y simulaciones Monte Carlo.

## Funcionalidades Clave

### 1. Simulación de Perfiles Térmicos con Acoplamiento CFD-Monte Carlo
Ejecuta simulaciones híbridas que integran dinámica de fluidos computacional (CFD) con métodos de Monte Carlo para predecir la distribución de temperatura en el escudo térmico durante el reingreso. Permite variar parámetros como ángulo de ataque, densidad atmosférica y coeficiente de arrastre, generando mapas de calor con resolución de 0.1°C en mallas no estructuradas.

### 2. Optimización de Trayectorias con Algoritmos Genéticos Adaptativos
Implementa un optimizador basado en algoritmos genéticos que ajusta automáticamente los puntos de ignición de los retropropulsores y el perfil de velocidad para minimizar la carga térmica máxima (Q_max). Soporta restricciones de ventana de aterrizaje y límites de aceleración (G-force), convergiendo en menos de 50 iteraciones para misiones de referencia.

### 3. Visualización 3D en Tiempo Real con WebGL y Datos de Telemetría
Renderiza trayectorias de reentrada en un visor 3D interactivo usando WebGL, superponiendo datos de telemetría como velocidad, altitud y flujo de calor. Incluye un modo de depuración que muestra vectores de fuerzas aerodinámicas y la evolución del ángulo de balance, ideal para revisiones post-misión.

## Guía de Uso Avanzada

### 1. Configuración de Escenario de Reentrada Lunar
Define los parámetros orbitales iniciales desde el punto de retorno lunar (TLI). Ejecuta `artemisreentry --mission LUNAR_RETURN --altitude 120km --velocity 11.2km/s` para cargar la atmósfera estándar USSA1976. Ajusta la masa del vehículo (dry mass + propellant) y el coeficiente balístico (β = m/(Cd·A)) en el archivo YAML de configuración. Verifica que el ángulo de trayectoria de entrada (γ) esté entre -5.5° y -6.5° para evitar skip o sobrecalentamiento.

### 2. Ejecución de Análisis de Sensibilidad Paramétrica
Usa el comando `--sensitivity` para barrer parámetros clave: `artemisreentry --sensitivity --param Cd --range 0.8:1.2 --step 0.05`. La herramienta generará un heatmap de la temperatura máxima en función de Cd y el ángulo de ataque. Exporta los resultados en formato CSV o HDF5 para post-procesamiento con Python/NumPy. Monitorea el log de convergencia para detectar inestabilidades numéricas en la integración Runge-Kutta de 4° orden.

### 3. Validación con Datos de Vuelo Reales
Importa telemetría de misiones previas (Apollo, SpaceX Crew Dragon) usando `--import-telemetry telemetry.csv --timestamp-format unix`. La herramienta alinea automáticamente los datos simulados con los reales mediante correlación cruzada. Genera un informe de diferencias (error cuadrático medio) para cada variable: altitud, velocidad, flujo de calor. Si el RMSE supera el 5%, revisa los modelos de atmósfera y la discretización temporal (paso Δt < 0.1s).

## Mejores Prácticas para Ingenieros Aeroespaciales

### 1. Estandariza la Referencia de Atmósfera
Siempre especifica el modelo atmosférico (USSA1976, NRLMSISE-00, o Mars-GRAM) al iniciar la simulación. Para misiones terrestres, usa USSA1976 con corrección de viento en altura. Para reentradas marcianas, activa el modelo de polvo estacional. Esto evita desviaciones de hasta 15% en la densidad atmosférica calculada.

### 2. Implementa un Pipeline de CI/CD para Validación de Trayectorias
Integra `artemisreentry` en tu flujo de integración continua. Cada commit en el repositorio de diseño del vehículo (masa, geometría) debe disparar una simulación de reentrada con parámetros nominales y de contingencia. Usa el flag `--ci-mode` para generar salidas en formato JSON, que luego se comparan con una base de datos de trayectorias validadas. Esto reduce errores humanos en la configuración de escenarios y garantiza trazabilidad.
