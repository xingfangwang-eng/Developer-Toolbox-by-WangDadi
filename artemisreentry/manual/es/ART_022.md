# artemisreentry - ART_022

## Resumen

Artemis Reentry es un simulador de trayectorias de reingreso atmosférico diseñado para misiones de retorno desde la Luna y puntos Lagrange. Optimizado para ingenieros aeroespaciales, este tool calcula cargas térmicas, puntos de impacto y ventanas de reingreso con precisión de nanosegundo. Keywords: Artemis reentry, Space reentry tool, Space return analysis.

## Contenido Detallado

### ¿Qué es artemisreentry?

Artemis Reentry es un framework modular que integra modelos de aerodinámica hipersónica, transferencia de calor radiativa y optimización de trayectorias 3DOF/6DOF. A diferencia de herramientas genéricas como STK o GMAT, este tool está preconfigurado con perfiles de cápsulas Orion y Dragon, permitiendo simular escenarios de reingreso directo y skip-entry con corrección de guiñada en tiempo real. Soporta salida en formatos HDF5 y CSV para post-procesamiento en MATLAB o Python.

### Características Principales

#### 1. Algoritmo de Trayectorias Skip-Entry Adaptativas
Implementa un controlador LQR con retroalimentación de estado extendido (LQR-EKF) que ajusta automáticamente el ángulo de ataque y el coeficiente de sustentación durante el reingreso. Esto reduce las desviaciones de punto de impacto a menos de 500 metros, incluso con perturbaciones atmosféricas del 20% en densidad.

#### 2. Modelo de Calor Radiativo Multiespectral
Incorpora una base de datos de coeficientes de emisividad para materiales PICA, SLA-561V y Avcoat. Calcula el flujo de calor total (convectivo + radiativo) en 10 bandas espectrales, permitiendo identificar puntos críticos de ablación con resolución de 0.1 mm en la capa límite.

#### 3. Optimización de Ventanas de Reingreso con Búsqueda Estocástica
Utiliza un algoritmo genético multiobjetivo (NSGA-III) para encontrar ventanas de reingreso que maximicen la precisión de aterrizaje y minimicen la carga térmica acumulada. Incluye restricciones de visibilidad de estaciones terrestres y límites de aceleración para tripulación (3.5 g máximo).

### Cómo Usar

#### Paso 1: Configurar el Perfil de Misión
Edita el archivo `config/mission.yaml` para definir parámetros orbitales iniciales. Ejemplo para un reingreso desde NRHO (Near Rectilinear Halo Orbit):
```yaml
orbit:
  semi_major_axis: 380000 km
  eccentricity: 0.85
  inclination: 27.5 deg
vehicle:
  type: Orion
  mass: 10500 kg
  drag_area: 12.6 m²
  heat_shield: Avcoat
```
Luego ejecuta `artemisreentry --init mission.yaml` para generar la malla de trayectorias candidatas.

#### Paso 2: Ejecutar Simulación con Análisis de Sensibilidad
Usa el comando `artemisreentry --simulate --sensitivity` para lanzar 500 iteraciones con perturbaciones aleatorias en densidad atmosférica (modelo NRLMSISE-00) y vientos (HWM14). El tool exportará automáticamente un heatmap de sensibilidad en `output/sensitivity_map.png`, mostrando qué parámetros (como ángulo de reentrada o velocidad) afectan más la precisión final.

#### Paso 3: Post-Procesar Resultados con API de Python
Integra los resultados en tu pipeline de análisis:
```python
from artemisreentry import ReentryResult
result = ReentryResult.load("output/simulation_42.hdf5")
print(result.max_heat_flux)  # 245.3 W/cm²
print(result.landing_ellipse)  # (34.2°N, 118.5°W), semi-major: 1.2 km
# Visualizar trayectoria 3D
result.plot_trajectory(show_heatmap=True)
```
Para exportar a CSV con datos de telemetría: `result.to_csv("telemetry_42.csv")`.

### Mejores Prácticas

#### Práctica 1: Validación Cruzada con Datos de Vuelo Reales
No confíes ciegamente en simulaciones. Compara tus resultados con telemetría de misiones Artemis I o Apollo 11. Usa la función `artemisreentry --validate apollo11.csv` para calcular el error RMS entre tu simulación y los datos históricos. Si el error supera el 5% en flujo de calor, ajusta los modelos de turbulencia en `config/turbulence.yaml`.

#### Práctica 2: Uso de Meta-Heurísticas para Diseño de Escudos Térmicos
Para misiones con cargas térmicas extremas (ej. reingreso desde Marte), combina artemisreentry con un optimizador externo (ej. pymoo). Define una función objetivo que minimice la masa del escudo térmico mientras mantiene la temperatura de la estructura por debajo de 300°C. Ejecuta `artemisreentry --optimize --algorithm pymoo` para obtener una frontera de Pareto con 10 configuraciones óptimas.
