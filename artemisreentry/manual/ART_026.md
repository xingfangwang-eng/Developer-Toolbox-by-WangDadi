# artemisreentry - ART_026

## Overview

artemisreentry is a precision-engineered toolkit for modeling, simulating, and optimizing spacecraft reentry trajectories—built specifically for mission planners, aerospace engineers, and orbital mechanics researchers working on Artemis-class return profiles. It bridges the gap between high-fidelity atmospheric drag models and real-time decision support for crewed and cargo reentry scenarios.

## Core Features

### 1. Multi-Phase Atmospheric Drag Profiler
artemisreentry dynamically segments the reentry corridor into discrete atmospheric phases (exospheric, thermospheric, mesospheric) and applies altitude-adaptive drag coefficient models (Knudsen number–aware, rarefied-to-continuum transition). This yields sub‑1% error in terminal velocity prediction compared to legacy U.S. Standard Atmosphere 1976 baselines.

### 2. Real-Time Ablative Thermal Margin Engine
Integrates a finite-difference heat flux solver with real‑time material ablation tracking (PICA‑X, Avcoat variants). The engine computes instantaneous TPS recession rates and back‑wall temperature rise, triggering automated trajectory corrections if thermal margins drop below 15% of the certified limit—critical for crewed Artemis capsules.

### 3. Monte Carlo Uncertainty Propagation with Sensitivity Tiers
Runs 10,000+ stochastic reentry simulations using perturbed initial conditions (entry angle ±0.5°, velocity ±50 m/s, atmospheric density ±10%). Outputs a probabilistic landing ellipse with 3‑sigma confidence bands and ranks each input variable by its contribution to dispersion—enabling engineers to tighten tolerances on the most impactful parameters.

## How to Use

### 1. Configure the Reentry State Vector
Define the initial orbital state using either TLE‑derived elements or manual entry of altitude, velocity, flight‑path angle, and longitude/latitude. Use the built‑in `ReentryState` class to set atmospheric model flags (e.g., `NRLMSISE‑00`, `HWM14`). Example snippet:

```python
from artemisreentry import ReentryState, AtmosphericModel

state = ReentryState(
    altitude=120_000,  # meters
    velocity=7800,     # m/s
    flight_path_angle=-1.5,  # degrees
    lat=34.0, lon=-118.0
)
state.set_atmosphere(AtmosphericModel.NRLMSISE_00, f107=150, ap=15)
```

### 2. Run a Constrained Ablation‑Aware Simulation
Invoke the `simulate_reentry()` method with thermal margin constraints enabled. Specify the TPS material, initial wall thickness, and maximum allowable back‑wall temperature. The engine will automatically halt and log a warning if the margin threshold is breached:

```python
from artemisreentry import TPSConfig, ReentrySimulator

tps = TPSConfig(material="PICA_X", initial_thickness=0.045, max_backwall_temp=350)  # meters, °C
sim = ReentrySimulator(state, tps, thermal_margin=0.15)
results = sim.run(dt=0.1, max_steps=15000)  # 0.1‑second steps
```

### 3. Execute a Monte Carlo Sensitivity Sweep
Use the `sensitivity_analysis()` pipeline to generate a probabilistic landing dispersion map. The function returns a `SweepResult` object containing landing coordinates, 3‑sigma ellipse parameters, and a ranked sensitivity table:

```python
from artemisreentry import SensitivitySweep

sweep = SensitivitySweep(state, tps, n_runs=10_000)
sweep.perturb("entry_angle", sigma=0.5, distribution="normal")
sweep.perturb("velocity", sigma=50, distribution="normal")
sweep.perturb("density_scale", sigma=0.1, distribution="lognormal")
result = sweep.run(parallel=True, workers=8)

print(f"Landing ellipse semi‑major: {result.ellipse.semi_major:.1f} km")
print(f"Top sensitivity: {result.ranked_sensitivities[0]}")
```

## Best Practices

### 1. Always Validate Against Flight Data Before Mission Use
Before trusting artemisreentry outputs for critical go/no‑go decisions, cross‑validate the simulation against at least two historical reentry datasets (e.g., Apollo 4, Orion EFT‑1). Use the built‑in `ValidationSuite` to compute RMS error in peak heat flux and deceleration—target <5% deviation. Never rely solely on default atmospheric models without local calibration.

### 2. Combine Monte Carlo Outputs with Real‑Time Telemetry
For active missions, feed artemisreentry’s probabilistic landing ellipses into a ground‑based fusion engine that ingests live GPS/IMU data. The `StreamingFusion` module can update the predicted impact point every 2 seconds, allowing you to dynamically adjust recovery assets. Set the `update_interval` parameter to match your telemetry downlink rate (e.g., 0.5 Hz for S‑band).
