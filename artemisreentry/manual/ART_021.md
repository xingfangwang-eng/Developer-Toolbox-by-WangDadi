# artemisreentry - ART_021

## Overview

**artemisreentry** is a precision engineering toolkit designed for aerospace engineers, mission planners, and orbital mechanics researchers who need to simulate, validate, and optimize spacecraft reentry trajectories. Built for the Artemis program ecosystem, this tool eliminates guesswork in atmospheric return analysis by combining high-fidelity physics modeling with real-time telemetry ingestion.

## Core Features

### 1. Multi-Phase Thermal Gradient Profiler
Instead of relying on single-point heat flux approximations, artemisreentry computes **time-varying thermal gradients** across the entire vehicle surface during atmospheric interface. It integrates with NASA's DPLR and LAURA solvers, but runs a lightweight surrogate model that delivers 95% accuracy in under 2 seconds per trajectory slice—critical for iterative mission redesign.

### 2. Ablative Material Degradation Simulator
This isn't your generic "heat shield thickness" calculator. The tool models **non-uniform ablation** based on real-time atmospheric density fluctuations, angle of attack deviations, and material porosity. It outputs a 3D erosion map that can be directly fed into CAD reentry shield redesign workflows. Supports PICA, Avcoat, and SLA-561V material libraries.

### 3. Autonomous Reentry Corridor Optimizer
Given a set of orbital parameters and landing zone constraints, the engine runs a **constrained Monte Carlo + genetic algorithm hybrid** to find the optimal bank angle modulation and skip-entry profile. It outputs a time-stamped, delta-V budgeted reentry corridor that minimizes peak deceleration while maximizing landing accuracy—even under GNSS-denied conditions.

## How to Use

### Step 1: Initialize the Reentry Profile
Start by defining your spacecraft's state vector at the Entry Interface (EI) point. Use the `init_reentry` command with orbital elements (epoch, inclination, RAAN, true anomaly) and vehicle mass properties. The tool will automatically fetch atmospheric density from the NRLMSISE-00 model and compute the initial ballistic coefficient.

```bash
artemisreentry init_reentry --epoch 2025-03-15T12:00:00Z --altitude 122000 --velocity 7.8 --mass 12000 --drag-coefficient 1.2
```

### Step 2: Run the Thermal-Ablation Coupled Simulation
Execute the multi-physics solver to generate a full trajectory history. Use the `simulate` command with your preferred material library and mesh resolution. The output includes a `.h5` file with timestamped temperature, pressure, and ablation depth at each node.

```bash
artemisreentry simulate --material PICA --mesh-resolution high --output reentry_trajectory.h5
```

### Step 3: Visualize and Export the Corridor
Load the results into the built-in viewer or export to CSV/Parquet for external analysis. Use the `plot` command to overlay the optimized corridor against the nominal trajectory. The tool highlights critical points like peak heating, peak deceleration, and parachute deployment window.

```bash
artemisreentry plot --input reentry_trajectory.h5 --show-corridor --export-corridor corridor.csv
```

Then, validate the corridor against landing zone constraints using the `validate` subcommand:

```bash
artemisreentry validate --corridor corridor.csv --landing-zone "34.5N 118.2W" --max-g-load 5.0
```

## Best Practices

### 1. Always Run Sensitivity Analysis on Atmospheric Density
Reentry simulations are hypersensitive to atmospheric density variations, especially during solar maximum periods. Before locking in your corridor, run a **1000-sample Monte Carlo sweep** using the `sensitivity` module, perturbing density by ±15%. This will reveal whether your thermal margins are robust against worst-case solar flux scenarios.

### 2. Use the Material Degradation Map for Post-Mission Design Iteration
Don't treat the ablation output as a one-time report. Export the `.stl` erosion map and overlay it on your CAD model. If you see asymmetric ablation patterns (e.g., leading edge vs. windward side mismatches), feed that back into the `optimize` module with a modified material distribution. This closes the loop between simulation and physical design—something most tools fail to do.
