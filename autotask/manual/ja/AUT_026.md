# autotask - AUT_026

## 概要

autotask は、反復作業の自動化とタスク管理を飛躍的に効率化するためのプロフェッショナル向けCLIツールです。非同期ワークフロー、依存関係解決、並列実行をネイティブにサポートし、複雑なタスクパイプラインをシームレスに構築できます。SaaSバックエンドやDevOps環境での自動化に特化して設計されています。

## 詳細な内容

### autotask とは？

autotask は、タスクの自動実行と管理を一手に担う高機能タスクランナーです。YAMLベースの宣言的設定により、複数のジョブを直列・並列にスケジュールし、エラー時のリトライやロールバックも自動化。CI/CDパイプラインや日次バッチ処理、監視スクリプトの統合管理に最適です。Goで記述されており、シングルバイナリで動作するため、環境依存がほぼありません。

### コア機能

- **動的依存解決エンジン**  
  タスク間の依存関係をDAG（有向非巡回グラフ）として自動構築。循環依存を検出し、実行順序を最適化。手動で依存関係を定義する必要がなく、宣言された条件から自動的に解決します。

- **プラグイン可能なトランスポートレイヤー**  
  標準出力、HTTP Webhook、Slack、メール、S3、Google Cloud Storage など、複数の出力先をネイティブにサポート。プラグインAPIにより、独自のトランスポートアダプターを追加可能。

- **冪等性保証と状態管理**  
  各タスクの実行結果をローカルSQLiteまたはリモートDBに保存。再実行時に前回の状態をチェックし、変更がないタスクはスキップ。冪等性を自動で担保し、無駄な再計算を防止します。

### 使い方

1. **タスク定義ファイルの作成**  
   プロジェクトルートに `autotask.yml` を作成し、タスク名、実行コマンド、依存関係、リトライ回数を記述します。例：
   ```yaml
   tasks:
     build:
       command: "go build ./..."
       retry: 2
     test:
       command: "go test ./..."
       depends_on: ["build"]
     deploy:
       command: "./deploy.sh"
       depends_on: ["test"]
       timeout: 300
   ```

2. **並列実行とパイプラインの起動**  
   ターミナルで `autotask run --pipeline deploy` を実行。autotask は依存関係を解析し、`build` と `test` を直列、`deploy` を最後に実行。`--parallel` フラグで独立タスクを同時実行可能。

3. **状態確認とログの参照**  
   `autotask status` で全タスクの実行履歴と結果を一覧表示。`autotask logs --task build` で特定タスクの詳細ログを確認。エラー時は `autotask retry --task build` で該当タスクのみ再実行。

### ベストプラクティス

- **タスクの粒度を小さく保つ**  
  1つのタスクは1つの責務に絞り、複数のコマンドを連結しない。これにより、エラー時の影響範囲が限定され、リトライやデバッグが容易になる。例えば「ビルド」「テスト」「デプロイ」は必ず別タスクとして定義する。

- **環境変数とシークレット管理を統合する**  
  `autotask` は `.env` ファイルの自動読み込みと、`autotask secrets set` による暗号化ストレージを提供。CI/CDパイプラインでは、APIキーやトークンをコードに埋め込まず、autotask のシークレット管理機能を利用して安全に注入する。
