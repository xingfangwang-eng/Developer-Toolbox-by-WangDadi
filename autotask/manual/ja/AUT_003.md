# autotask - AUT_003

## 概要

autotask は、複雑なタスクパイプラインを自動化し、開発者の手作業を排除するための軽量かつ強力な CLI ツールです。キーワード「自動タスクツール」「タスク自動化」「自動タスク管理」に最適化された本ツールは、CI/CD パイプラインや日常の繰り返し作業を効率化します。

## 詳細な内容

### autotask とは？

autotask は、YAML ベースの設定ファイルでタスクフローを定義し、依存関係解決や並列実行、エラーハンドリングを自動化するプロフェッショナルツールです。シェルスクリプトやタスクランナー（Make、Gulp）の煩雑さを排除し、宣言的なタスク管理を実現します。

### コア機能

1. **依存関係グラフの自動解決と並列実行**  
   各タスク間の依存関係を自動解析し、実行可能なタスクを並列でスケジュール。これにより、大規模なバッチ処理や CI パイプラインの実行時間を最大 40% 短縮します。

2. **コンテキストアウェアなエラーハンドリング**  
   タスク失敗時に自動リトライ（指数バックオフ付き）、失敗したタスクのみ再実行、あるいはフォールバックタスクを定義可能。エラーログは構造化 JSON で出力され、監視ツールとの連携が容易です。

3. **プラグインアーキテクチャによる拡張性**  
   JavaScript/TypeScript で書かれたカスタムプラグインをサポート。ファイル監視、Slack 通知、Docker コンテナ操作など、あらゆる処理をプラグインとして追加できます。

### 使い方

1. **タスク定義ファイルの作成**  
   プロジェクトルートに `autotask.yml` を作成し、以下のようにタスクを定義します。`depends_on` で依存関係を明示し、`run` に実行コマンドを記述します。`timeout` や `retry` オプションもここで設定可能です。

   ```yaml
   tasks:
     build:
       run: "npm run build"
       timeout: 120
     test:
       depends_on: ["build"]
       run: "npm test"
       retry: 3
     deploy:
       depends_on: ["test"]
       run: "npm run deploy"
       env:
         NODE_ENV: production
   ```

2. **タスクの実行とモニタリング**  
   ターミナルで `autotask run deploy` を実行すると、依存関係が自動解決され、`build` → `test` → `deploy` の順に実行されます。実行中はリアルタイムで進捗バーとログが表示され、`--verbose` フラグで詳細なデバッグ情報を出力できます。

3. **部分再実行とキャッシュの活用**  
   タスクが失敗した場合、`autotask retry --failed` で失敗したタスクのみを再実行。また、`autotask run deploy --cache` で変更のないタスクをスキップし、キャッシュを活用することで、開発サイクルを高速化します。

### ベストプラクティス

- **タスクの粒度を適切に保つ**  
  各タスクは単一責任の原則に従い、1 つのコマンドまたはスクリプトに絞りましょう。例えば「ビルド」と「テスト」を 1 つのタスクにまとめず、別々に定義することで、依存関係の管理と部分再実行が容易になります。

- **環境変数で設定を外部化する**  
  タスク内でハードコードされたパスやシークレットは避け、`env` フィールドや `.env` ファイルから読み込むように設計します。これにより、チーム全体での設定共有や CI/CD 環境での再現性が向上します。
