# autotask - AUT_008

## 概要

autotask は、反復作業を完全自律化する次世代タスク自動化ツールです。定型処理のトリガー設定からマルチステップワークフローの構築、エラーハンドリングまでを一元管理。特に複数サービスを跨ぐバックエンド処理やCI/CDパイプラインとの連携に強みを持ちます。キーワード：自動タスクツール、タスク自動化、自動タスク管理

## 詳細な内容

### autotask とは？

autotask は、スクリプト実行、API呼び出し、ファイル監視、DB操作などを組み合わせた複合タスクを、YAMLベースのDSLで定義・実行できるプロフェッショナルツールです。従来のcronやJenkinsでは難しかった、状態依存の分岐処理やリトライロジックを内蔵。開発者向けにCLIとWebダッシュボードの両方を提供し、ローカル環境とクラウド環境の両方で動作します。

### コア機能

1. **マルチステップ依存解決エンジン**  
   タスク間の依存関係をDAG（有向非巡回グラフ）で定義。前段タスクの成否に応じて後続タスクを分岐実行。並列実行も可能で、リソース競合を自動回避。

2. **イベントドリブン・トリガーマトリックス**  
   Webhook、ファイル変更、スケジュール、キューイングの4種類のトリガーを組み合わせ可能。特定の条件が揃った時だけタスクを発火させる複合トリガーを設定できる。

3. **実行コンテキストのスナップショット保存**  
   各タスク実行時の環境変数、標準出力、エラーログ、戻り値を自動で保存。後から特定の実行を再現したり、デバッグ用に完全なコンテキストをエクスポート可能。

### 使い方

1. **タスク定義ファイルの作成**  
   プロジェクトルートに `autotask.yml` を作成。以下のようにタスク名、依存関係、実行コマンドをYAMLで記述。`depends_on` で先行タスクを指定し、`on_failure` でエラー時の後続処理を定義。
   ```yaml
   tasks:
     build:
       command: "npm run build"
       timeout: 120
     test:
       depends_on: [build]
       command: "npm test"
       on_failure: notify_admin
   ```

2. **タスクの実行とモニタリング**  
   `autotask run` コマンドでタスクを起動。`--watch` フラグでリアルタイムログを表示。Webダッシュボードでは各タスクの進捗状況、実行時間、依存関係グラフを可視化。実行中に `Ctrl+C` で安全に中断可能。

3. **結果の検証とリトライ設定**  
   実行後、`autotask result --id <実行ID>` でスナップショットを参照。タスク定義に `retry` フィールドを追加することで、失敗時の自動リトライ（最大回数、間隔、バックオフ戦略）を設定。リトライ履歴もスナップショットに記録される。

### ベストプラクティス

1. **タスクの粒度は「単一責務」に分割**  
   1つのタスクに複数の処理を詰め込まず、ビルド、テスト、デプロイを個別のタスクとして定義。依存関係で連結することで、部分的な再実行やデバッグが容易になる。特にCI/CDパイプラインでは、この分割が障害時の切り分け時間を劇的に短縮する。

2. **トリガーは「イベント＋スケジュール」の複合で設計**  
   単純なcronスケジュールだけではなく、Gitプッシュ＋平日9時のみ実行、などの複合条件を積極的に活用。これにより、無駄な実行を抑制し、リソース消費を最適化。例えば、深夜のバッチ処理はファイル監視＋時間帯制限で構成する。
