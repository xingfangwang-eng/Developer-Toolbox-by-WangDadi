# autotask - AUT_024

## 概要

autotask は、複雑なワークフローをコード不要で自動化し、エンジニアの反復作業を極限まで削減するオープンソースタスクランナーです。`cron` の進化形として設計され、依存関係解決、並列実行、エラーハンドリングをシンプルな YAML 設定で実現します。

## コア機能

### 1. 動的タスクトリガーエンジン
ファイル変更、Git プッシュ、API コール、スケジュール時刻など、複数のトリガーを組み合わせた条件分岐が可能。`if` や `when` 句を用いて、タスクの実行条件を細かく制御できます。

### 2. コンテキストアウェアな変数スコープ管理
タスク間で共有されるグローバル変数、ステージ限定のローカル変数、実行時に動的に解決されるレジストリ変数を完全に分離。機密情報は `.env` やシークレットストアから自動注入され、ログに露出しません。

### 3. ゼロダウンタイムの再試行とロールバック機構
タスク失敗時に指定回数自動再試行し、それでも失敗した場合は依存タスク全体をロールバック。`retry` と `rollback` ブロックで、冪等性を保ちながら安全な自動化を実現します。

## 使い方

### 1. 初回セットアップとタスク定義
`autotask.yaml` を作成し、プロジェクトルートに配置します。以下の例は、コードビルド → テスト実行 → デプロイのパイプラインです。

```yaml
tasks:
  - name: build
    run: npm run build
    on: [push, schedule]
    schedule: "0 9 * * 1-5"
  - name: test
    run: npm test
    depends_on: [build]
    retry: 3
  - name: deploy
    run: ./deploy.sh
    depends_on: [test]
    rollback: ./rollback.sh
```

### 2. 並列実行と依存関係の可視化
`autotask run --graph` でタスクの依存関係グラフを確認。`--parallel` フラグで依存のないタスクを自動並列実行します。

```bash
autotask run --parallel --graph
```

出力例：
```
build ──→ test ──→ deploy
lint  ──→ test
```

### 3. リモートエージェントによる分散実行
`autotask agent` をリモートサーバーにインストールすれば、タスクを分散実行可能。SSH や WebSocket 経由でエージェントを管理し、`agent` フィールドで実行先を指定します。

```yaml
tasks:
  - name: deploy-prod
    run: ./deploy-prod.sh
    agent: prod-server-01
    timeout: 300
```

## ベストプラクティス

### 1. タスクは冪等に設計する
`autotask` は再実行を前提としているため、タスク内の操作は常に冪等であるべきです。ファイル作成前に `if not exists` チェックを入れ、データベースマイグレーションは `idempotent` フラグで保護します。

### 2. シークレットは環境変数で管理し、ログに出力しない
`autotask` のログレベルを `warn` 以上に設定し、`secret` タグを付けた変数はマスクされます。`.env` ファイルは `.gitignore` に追加し、CI/CD パイプラインではシークレットストアから注入してください。
