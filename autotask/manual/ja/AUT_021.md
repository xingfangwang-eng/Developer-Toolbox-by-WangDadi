# autotask - AUT_021

## 概要

autotask は、反復作業を完全自動化する次世代タスク管理エンジンです。キーワード「タスク自動化」「自動タスク管理」に最適化された本ツールは、CI/CD パイプラインからデイリーオペレーションまでを統合制御します。

## 詳細な内容

### autotask とは？

autotask は、YAML ベースのワークフロー定義とイベント駆動型トリガーを組み合わせた、プロフェッショナル向け自動化プラットフォームです。従来の cron や Jenkins の制約を超え、コンテキスト依存の条件分岐や外部 API とのリアルタイム連携をネイティブサポート。開発チームの「手作業地獄」を根本から解消します。

### コア機能

- **動的依存解決エンジン**: タスク間の依存関係を DAG（有向非巡回グラフ）で自動最適化。実行順序の衝突やデッドロックを防止し、並列実行効率を最大化。
- **コンテキストアウェアトリガー**: ファイル変更、Git プッシュ、Slack メッセージ、HTTP フックなど、20 以上のトリガーソースを統合。条件式（例：`branch == "main" && file_pattern == "*.js"`）で細粒度制御可能。
- **自己修復型実行ログ**: タスク失敗時に自動リトライ（指数バックオフ）、部分成功時のロールバック、異常終了時のスタックトレース自動収集。エラー原因特定のためのコンテキスト情報を構造化 JSON で出力。

### 使い方

1. **ワークフロー定義ファイルの作成**  
   プロジェクトルートに `autotask.yml` を作成し、以下のようにタスクチェーンを記述します。`on` セクションでトリガー条件、`steps` で実行ステップ、`env` で環境変数を定義。YAML スキーマバリデーションにより記述ミスを即座に検出。

   ```yaml
   version: '2.0'
   workflow: 'deploy-pipeline'
   on:
     push:
       branches: ['main', 'release/*']
       paths: ['src/**', 'deploy/**']
   env:
     NODE_ENV: production
   steps:
     - name: build
       run: npm run build
       retry: 3
     - name: test
       run: npm test
       needs: [build]
   ```

2. **CLI による手動実行とモニタリング**  
   `autotask run --workflow deploy-pipeline --env BRANCH=feature/new-auth` で特定ブランチを指定して即時実行。実行中は `autotask watch` でリアルタイムログをストリーミング表示。各ステップの所要時間とメモリ使用量がカラーマップで可視化されます。

3. **条件分岐とパラレル実行の設定**  
   ステップに `if` 条件を追加して分岐制御。`parallel` キーワードで複数ステップを同時実行。例えば、`if: env.BRANCH == 'main'` の条件下でのみデプロイジョブを起動し、テストと lint 検査を並列処理。

   ```yaml
   steps:
     - name: lint
       run: npm run lint
       parallel: true
     - name: unit-test
       run: npm run test:unit
       parallel: true
     - name: deploy
       run: ./deploy.sh
       if: env.BRANCH == 'main'
       needs: [lint, unit-test]
   ```

### ベストプラクティス

- **タスク粒度の適正化**: 1 ステップに複数のコマンドを詰め込まず、単一責務のステップに分割。これにより、失敗時の影響範囲を最小化し、リトライ効率を向上。例えば「ビルド→単体テスト→結合テスト→デプロイ」を 4 ステップに分離。
- **環境変数によるポータビリティ**: ワークフロー内のパスや認証情報はハードコードせず、`env` セクションまたは外部 `.env` ファイルで管理。`autotask run --env-file .env.production` で環境切り替えを実現し、開発・ステージング・本番間の設定差異を吸収。
