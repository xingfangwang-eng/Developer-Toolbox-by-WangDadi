# autotask - AUT_023

## 概要

autotask は、反復作業を完全自動化するためのタスクオーケストレーションツールです。複雑なワークフローをコード化し、手動介入を排除することで、開発チームの生産性を飛躍的に向上させます。キーワード: タスク自動化、ワークフローエンジン、自動タスク管理。

## 詳細な内容

### autotask とは何か？

autotask は、**宣言型の YAML 設定** と **プラグイン可能なアクションエンジン** を核とした、タスク自動化フレームワークです。単なる cron の代替ではなく、依存関係グラフに基づいたタスクの並列実行、失敗時のロールバック戦略、そして外部サービス（Slack、GitHub、AWS Lambda）とのシームレスな連携を標準装備。特に CI/CD パイプラインやデータパイプラインの自動化において、その真価を発揮します。

### コア機能

1. **宣言型ワークフローエンジン**: YAML で記述したタスクの依存関係を自動解決し、並列実行を最適化。`depends_on` と `retry_strategy` を組み合わせることで、複雑なパイプラインも簡潔に定義可能。
2. **プラグインアーキテクチャ**: カスタムアクションを TypeScript または Python で記述し、動的にロード。`@action` デコレータを使えば、わずか数行で独自のタスクアクションを追加できる。
3. **リアルタイムダッシュボード**: WebSocket ベースのダッシュボードで、各タスクの実行状態、ログ、メトリクスをリアルタイムで可視化。エラー発生時には、該当タスクのスタックトレースを即座に表示。

### 使い方

1. **ワークフロー定義の作成**: プロジェクトルートに `workflow.yml` を作成し、タスクを宣言します。例として、コードのビルド → テスト → デプロイ のパイプラインを定義します。各タスクには `action` と `input` を指定し、`depends_on` で依存関係を明示します。
2. **アクションの実装**: `actions/` ディレクトリにカスタムアクションを配置。TypeScript の例: `export default @action('build') async (ctx) => { await ctx.exec('npm run build'); }`。`ctx` オブジェクトを通じて、環境変数や前タスクの出力にアクセス可能です。
3. **タスクの実行と監視**: `autotask run workflow.yml` を実行。ターミナルにリアルタイムの進捗が表示されるほか、`autotask dashboard` でブラウザベースのダッシュボードを起動。特定タスクが失敗した場合、`autotask retry --task <task_id>` でリトライ可能。

### ベストプラクティス

- **冪等性の徹底**: 各アクションは必ず冪等に設計してください。特に外部 API を呼び出す場合は、リトライによる副作用を考慮し、`ctx.memoize` を利用して同一入力に対する結果をキャッシュすると安全です。
- **ワークフロー分割とテスト**: 巨大な単一ワークフローは避け、責務ごとにサブワークフローに分割。各ワークフローは `autotask test --workflow <path>` でユニットテスト可能なため、事前に依存関係の整合性を検証しておきましょう。
