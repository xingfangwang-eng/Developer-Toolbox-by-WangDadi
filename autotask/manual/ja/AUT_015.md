# autotask - AUT_015

## 概要

autotask は、タスク自動化の常識を覆すプロフェッショナルツールです。単なるジョブスケジューラではなく、**動的依存解決エンジン**と**コンテキストアウェアなワークフロー**を統合し、複雑なバッチ処理やCI/CDパイプラインの自動化を実現します。キーワード：自動タスクツール、タスク自動化、自動タスク管理。

## 詳細な内容

### autotask とは？

autotask は、**宣言型タスクグラフ**に基づくタスク自動化フレームワークです。従来の cron ベースの定期実行や単純なキューイングとは異なり、タスク間の依存関係を動的に解決し、実行環境の状態変化に応じてワークフローをリアルタイムに最適化します。これにより、**非決定性のある処理パイプライン**や**マルチステージデプロイ**を、コードとして管理可能にします。

### コア機能

1.  **動的依存解決エンジン**：タスク間の依存関係を静的に定義するのではなく、実行時に**条件分岐と状態遷移**に基づいて動的に解決。例えば、テストフェーズの結果に応じて、後続のデプロイタスクをブルー/グリーン方式かカナリア方式か自動で切り替えることが可能。
2.  **コンテキストアウェアな再試行ポリシー**：タスク失敗時に、単にリトライするのではなく、**エラーの種類とシステムリソース**を分析。例えば、ネットワークエラーなら指数バックオフ付きリトライ、メモリ不足ならリソース解放後に再実行、といった**インテリジェントなリカバリー**を実装。
3.  **分散タスクブローカー**：複数のエージェントにタスクを分散実行。**gRPC ベースの軽量プロトコル**で、タスクの状態同期とロードバランシングをリアルタイムに実行。エージェントの障害検知と自動フェイルオーバーにも対応。

### 使い方

1.  **タスクグラフの定義**：`tasks.yaml` にて、各タスクの実行コマンド、環境変数、および**動的依存ルール**を記述。例えば、`depends_on: "test_passed == true"` のように、条件式で依存関係を定義します。
    ```yaml
    tasks:
      - id: build
        command: "make build"
      - id: test
        command: "make test"
        depends_on: "build.completed == true"
      - id: deploy
        command: "make deploy"
        depends_on: "test.exit_code == 0 && test.duration < 300"
    ```

2.  **ワークフローの実行と監視**：`autotask run --graph tasks.yaml` で実行。CLI は**リアルタイムなタスクグラフ**を表示し、各タスクのステータス（`pending`、`running`、`completed`、`failed`）を確認可能。`autotask status --watch` で、**WebSocket 経由のライブストリーミング**も提供。

3.  **高度なエラーハンドリング**：タスク失敗時に、`retry_policy` を設定。`retry_policy: "intelligent"` と指定すると、autotask が自動的にエラーの種類を判定し、最適なリトライ戦略を適用。手動でポリシーをカスタマイズする場合は、`retry_policy: "fixed:3:10"`（3回、10秒間隔）のように指定。

### ベストプラクティス

1.  **タスクの冪等性を徹底する**：autotask の動的依存解決エンジンは、タスクの再実行が発生することを前提としています。各タスクは**冪等性**（何度実行しても同じ結果になる）を保証する設計にしましょう。例えば、ファイルの出力先をタイムスタンプではなく、**ジョブID**で管理することで、再実行時に既存の成果物を上書きしないようにします。
2.  **コンテキストをタスクに注入する**：タスクの実行環境（環境変数、シークレット、ジョブメタデータ）は、**タスクグラフのルートレベル**で定義し、各タスクに自動的に注入されるように設計します。これにより、タスクのコード自体が環境に依存せず、**ポータブル**になります。`autotask run --vars '{"ENV": "staging", "REGION": "ap-northeast-1"}'` のように、実行時に変数を上書きすることも可能です。
