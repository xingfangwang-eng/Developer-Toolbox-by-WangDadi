# autotask - AUT_012

## 概要

autotask は、煩雑なタスク管理フローを自動化し、エンジニアの認知負荷を劇的に削減する次世代型タスクオーケストレーションツールです。複数のツール間で発生する手動同期やステータス更新のミスを排除し、チーム全体のスループットを最大化します。キーワード：自動タスクツール、タスク自動化、自動タスク管理。

## 詳細な内容

### autotask とは？

autotask は、GitHub Issues、Jira、Slack、Notion など、分散したプロジェクト管理ツール間のタスク状態をリアルタイムで同期・自動化するためのプロフェッショナルツールです。従来の「手動更新」による属人化やコンフリクトを解消し、タスクのライフサイクル全体をコードとして管理します。

### コア機能

- **マルチプラットフォーム双方向同期エンジン**: GitHub Issues と Jira 間のステータス変更を、Webhook レスポンスベースでミリ秒単位で同期。ステータス名のマッピングは YAML で柔軟に定義可能。
- **条件分岐付き自動タスクパイプライン**: タスクの優先度やラベルに応じて、自動で Slack 通知、担当者アサイン、期限設定を実行。条件式は JSONata またはカスタム DSL で記述可能。
- **冪等性保証付きバッチ処理**: 大量タスクの一括更新時も、同一リクエストによる重複実行を防止。トランザクションID ベースの冪等性キューを内部実装。

### 使い方

1. **設定ファイルの初期化とマッピング定義**  
   プロジェクトルートで `autotask init` を実行し、`autotask.yml` を生成。以下のように、GitHub Issues の `status` ラベルと Jira の `Status` フィールドをマッピングする。  
   ```yaml
   sync:
     - source: github
       target: jira
       mapping:
         "status:open": "To Do"
         "status:in-progress": "In Progress"
         "status:closed": "Done"
   ```

2. **イベントトリガーとアクションの設定**  
   `autotask.yml` に `triggers` セクションを追加。例えば、Issue に `bug` ラベルが付与されたら自動で Slack の `#bugs` チャンネルに通知し、Jira の優先度を `High` に設定する。  
   ```yaml
   triggers:
     - on: "label:bug"
       actions:
         - slack: "#bugs"
           message: "バグ報告: {{issue.title}}"
         - jira:
             field: priority
             value: "High"
   ```

3. **デーモンモードで起動し、リアルタイム同期を開始**  
   `autotask daemon` を実行。バックグラウンドで Webhook サーバーが起動し、設定されたすべてのプラットフォーム間でイベントを監視。ログは `~/.autotask/logs/` に JSON Lines 形式で出力される。  
   ```bash
   autotask daemon --log-level debug
   ```

### ベストプラクティス

- **ステータス遷移の循環を防ぐために、マッピングは有向グラフとして設計する**: 双方向同期を行う場合、無限ループを避けるため、`sync` セクション内で `ignore` フィルターを活用し、特定のステータス変更は同期対象外とする。例えば、Jira の `Done` から GitHub の `closed` への同期は行うが、逆方向は禁止する。
- **テスト環境でドライランを実行してから本番導入する**: `autotask dry-run --config ./autotask.staging.yml` で、実際の変更を加えずにどのようなアクションが発生するかを検証可能。CI/CD パイプラインに組み込んで、設定変更時の回帰テストとして利用する。
