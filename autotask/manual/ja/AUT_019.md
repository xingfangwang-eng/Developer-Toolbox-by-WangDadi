# autotask - AUT_019

## 概要

autotask は、反復作業の完全自動化を実現する次世代型タスクオーケストレーションツールです。本プロジェクトは、「設定して忘れる」運用を徹底し、手動介入の削減とスループット最大化に特化しています。キーワード：自動タスクツール、タスク自動化、自動タスク管理。

## 詳細な内容

### autotask とは？

autotask は、複雑な依存関係を持つワークフローを YAML ベースの DSL で記述し、並列実行・エラーハンドリング・再試行戦略を自動制御するプロフェッショナルツールです。特に CI/CD パイプラインや定期バッチ処理において、単なる cron 代替を超えた高度なオーケストレーションを提供します。

### コア機能

1. **動的依存解決グラフエンジン**  
   タスク間の暗黙的依存関係を静的解析で自動検出。DAG（有向非巡回グラフ）をランタイムで最適化し、リソース競合を回避しながら最大並列度で実行。

2. **インテリジェントリトライ & ロールバック**  
   タスク失敗時に指数バックオフ付きリトライを自動適用。さらに、部分成功状態から安全にロールバックする「セーブポイント機構」を内蔵。DB トランザクションのようにワークフロー全体の整合性を担保。

3. **プラグイン可能なトリガーアダプター**  
   cron、Webhook、Git イベント、Kubernetes イベント、AWS SQS など複数ソースからのトリガーを統一的に処理。各トリガーは独自のペイロード変換ロジックをプラグインとして追加可能。

### 使い方

1. **ワークフロー定義ファイルの作成**  
   プロジェクトルートに `autotask.yaml` を作成し、タスク名、実行コマンド、依存関係、タイムアウト値を記述。例：
   ```yaml
   version: '2.0'
   tasks:
     build:
       command: 'make build'
       timeout: 300s
     test:
       depends_on: [build]
       command: 'make test'
       retry:
         max_attempts: 3
         backoff: exponential
   ```

2. **ローカルモードでの検証実行**  
   `autotask run --dry-run` で DAG の解析結果と実行順序をプレビュー。その後 `autotask run` で実際の実行を開始。`--watch` フラグでリアルタイムログを tail 可能。

3. **本番環境へのデプロイと常時監視**  
   `autotask deploy --target production` でワークフローをサーバーレス環境にデプロイ。`autotask status` で全ワークフローの実行状態を一覧表示し、`autotask logs --task build` で個別タスクの詳細ログを確認。

### ベストプラクティス

- **タスク粒度の設計**  
  1 タスク 1 責務の原則を徹底。単一のシェルコマンドやスクリプト呼び出しに分割し、タイムアウトを 5 分以内に設定。これにより失敗時の影響範囲を最小化し、再試行効率を最大化。

- **環境変数による動的設定**  
  ワークフロー内のパラメータはハードコードせず、`${ENV_VAR}` 構文で外部注入。シークレットは `.env` ファイルではなく、Vault や AWS Secrets Manager と連携するプラグインを使用して安全に管理。
