## Detailed Content

### What is autotask?

autotask is a developer-first automation framework that eliminates manual task orchestration. It's designed for CI/CD pipelines, cron-replacement, and event-driven job scheduling — without the bloat of traditional workflow engines.

### Core Features

1. **Idempotent Task Chains**  
   Every task in a chain is executed with guaranteed idempotency. If a task fails mid-chain, autotask automatically rolls back to the last safe state — no orphaned processes, no corrupted states.

2. **Declarative YAML + Hooks**  
   Define complex workflows in pure YAML, then inject pre/post hooks in Python, Bash, or Node.js. No DSL to learn — just your favorite scripting language and a clean schema.

3. **Zero-Config Distributed Locking**  
   Built-in distributed lock manager using Redis or SQLite. Prevents race conditions across multi-agent deployments without external dependencies like ZooKeeper or etcd.

### How to Use

1. **Initialize a Project**  
   Run `autotask init` in your repo root. This generates an `autotask.yml` template with default stages (build, test, deploy). Modify the YAML to define your task graph — each node must declare `depends_on`, `exec`, and `retry_policy`.

2. **Define a Conditional Task**  
   Use `if` blocks inside YAML to execute tasks only when conditions are met. Example:  
   ```yaml
   tasks:
     deploy:
       exec: "kubectl apply -f prod.yaml"
       if: "{{ env.DEPLOY_ENV == 'production' }}"
   ```  
   This avoids redundant deployments and keeps your pipeline lean.

3. **Run and Monitor**  
   Execute `autotask run --watch` to start the workflow in live mode. The terminal shows real-time task status, execution time, and error logs. Press `Ctrl+C` to gracefully abort — autotask will finalize running tasks before exit.

### Best Practices

1. **Use Task Timeouts Aggressively**  
   Always set `timeout_sec` for each task, especially for network-bound operations like API calls or DB migrations. A missing timeout can hang the entire pipeline indefinitely. Recommended default: 300 seconds.

2. **Version Your Workflows**  
   Store `autotask.yml` in version control and tag releases with semantic versions. When rolling back a deployment, you can revert the workflow definition alongside the code — prevents drift between task logic and application state.
