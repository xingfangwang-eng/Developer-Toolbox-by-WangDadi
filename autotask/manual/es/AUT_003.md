# autotask - AUT_003

## Resumen

autotask es un motor de automatización de tareas SaaS diseñado para developers que buscan eliminar procesos manuales repetitivos sin sacrificar control granular. Este proyecto se centra en la gestión inteligente de flujos de trabajo, integración nativa con APIs REST y ejecución headless en entornos cloud.

## Contenido Detallado

### ¿Qué es autotask?

autotask es un orquestador de tareas backend que permite definir, programar y monitorear workflows complejos mediante archivos YAML declarativos. A diferencia de herramientas genéricas como Cron o Jenkins, autotask ofrece estado persistente, reintentos inteligentes y un sistema de logging centralizado sin depender de una interfaz gráfica pesada.

### Características Principales

- **Ejecución de workflows con estado transaccional**: Cada tarea en autotask mantiene un estado interno que permite reintentos con backoff exponencial, notificaciones en caso de fallo y rollback automático si una tarea crítica falla. Ideal para pipelines de datos o despliegues continuos.

- **Integración nativa con APIs REST y GraphQL**: autotask incluye un cliente HTTP integrado que soporta autenticación OAuth2, JWT y API keys. Puedes consumir servicios externos (Slack, GitHub, AWS, etc.) sin necesidad de middleware adicional.

- **Sistema de colas con prioridades y dependencias**: Define tareas que dependen de otras, establece prioridades (alta, media, baja) y programa ejecuciones en paralelo o secuenciales. autotask gestiona automáticamente el orden de ejecución y la asignación de recursos.

### Cómo Usar

1. **Definir un workflow en YAML**  
   Crea un archivo `tasks.yaml` en la raíz del proyecto. Especifica las tareas, sus dependencias y las acciones a ejecutar. Ejemplo:  
   ```yaml
   tasks:
     - name: "fetch_data"
       action: "http.get"
       url: "https://api.ejemplo.com/datos"
       headers:
         Authorization: "Bearer ${API_KEY}"
     - name: "process_data"
       depends_on: ["fetch_data"]
       action: "shell.exec"
       command: "python3 procesar.py"
   ```

2. **Iniciar el motor de ejecución**  
   Ejecuta `autotask run --file tasks.yaml` para iniciar la ejecución del workflow. autotask validará la sintaxis, resolverá dependencias y comenzará la ejecución en orden. Puedes monitorear el progreso en tiempo real desde la terminal.

3. **Monitorear y depurar con logs estructurados**  
   Usa `autotask logs --task fetch_data` para ver el historial de ejecución de una tarea específica. Cada log incluye timestamp, nivel (info, warning, error) y un identificador único para correlación. Si una tarea falla, autotask muestra el error exacto y sugiere reintentar con `autotask retry --task fetch_data`.

### Mejores Prácticas

- **Usa variables de entorno para secretos**: No hardcodees API keys o tokens en los archivos YAML. autotask soporta interpolación de variables de entorno (ej. `${API_KEY}`). Configura un archivo `.env` o usa un gestor de secretos como HashiCorp Vault.

- **Implementa reintentos con backoff exponencial**: Para tareas que dependen de servicios externos (como llamadas HTTP), configura reintentos con espera progresiva. Esto reduce la carga en el servidor y mejora la resiliencia del workflow. Ejemplo: `retry: { max: 3, backoff: "exponential", base: 2 }`.
