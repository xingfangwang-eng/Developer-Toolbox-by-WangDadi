# autotask - AUT_018

## Resumen

autotask es un orquestador de tareas automatizadas diseñado para desarrolladores que buscan eliminar la fricción en flujos de trabajo repetitivos. Optimizado para SEO con palabras clave como «herramienta de tareas automáticas», «automatización de tareas» y «gestión automática de tareas», este proyecto se enfoca en la eficiencia sin comprometer el control granular.

## Contenido Detallado

### ¿Qué es autotask?

autotask no es solo un gestor de tareas; es un **motor de automatización declarativo** que permite definir pipelines complejos mediante archivos YAML o JSON. A diferencia de herramientas como Cron o systemd, autotask ofrece dependencias entre tareas, reinteligencia inteligente y logging estructurado. Ideal para entornos donde la fiabilidad y la trazabilidad son críticas, como despliegues CI/CD, sincronización de datos o mantenimiento de servidores.

### Características Principales

- **Ejecución con DAG (Directed Acyclic Graph)**: Define tareas como nodos en un grafo acíclico dirigido. autotask resuelve automáticamente el orden de ejecución, permitiendo paralelismo seguro y fallos aislados. Ejemplo: una tarea de backup puede depender de una tarea de compresión, pero ejecutarse en paralelo con una tarea de notificación.
- **Reinteligencia Adaptativa con Backoff Exponencial**: Si una tarea falla, autotask no solo reintenta; aplica un algoritmo de backoff exponencial con jitter para evitar tormentas de reintentos. Configurable por tarea, con límites de tiempo y número de intentos.
- **Logging Estructurado en Tiempo Real**: Cada ejecución genera logs en formato JSON, listos para ser ingeridos por sistemas como ELK o Grafana. Incluye metadatos como ID de ejecución, duración, estado y trazas de errores. Ideal para auditoría y depuración post-mortem.

### Cómo Usar

1. **Definir un Pipeline con Dependencias Explícitas**  
   Crea un archivo `autotask.yaml` en la raíz de tu proyecto. Usa la sintaxis `depends_on` para establecer relaciones entre tareas. Ejemplo:  
   ```yaml
   tasks:
     - name: fetch_data
       command: "curl -o data.json https://api.example.com/data"
     - name: process_data
       command: "python process.py"
       depends_on: [fetch_data]
     - name: notify
       command: "curl -X POST https://hooks.slack.com/notify"
       depends_on: [process_data]
   ```  
   Ejecuta con `autotask run --file autotask.yaml`. autotask validará el grafo y ejecutará en orden óptimo.

2. **Configurar Reinteligencia con Backoff**  
   Para tareas críticas, añade la sección `retry`:  
   ```yaml
   tasks:
     - name: deploy_app
       command: "ansible-playbook deploy.yml"
       retry:
         max_attempts: 5
         backoff: exponential
         jitter: true
         timeout: 300
   ```  
   Esto asegura que si el despliegue falla, autotask reintentará con intervalos crecientes (ej. 1s, 2s, 4s, 8s, 16s) más un jitter aleatorio para evitar picos.

3. **Monitorear con Logging Estructurado**  
   Activa el logging JSON con la flag `--log-format json`:  
   ```bash
   autotask run --file autotask.yaml --log-format json > logs.json
   ```  
   Los logs incluyen campos como `task_name`, `status`, `duration_ms`, `error_message` y `timestamp`. Puedes filtrar con `jq`:  
   ```bash
   cat logs.json | jq 'select(.status == "failed") | {task: .task_name, error: .error_message}'
   ```

### Mejores Prácticas

- **Usa Tareas Idempotentes**: Cada tarea debe poder ejecutarse múltiples veces sin efectos secundarios. Por ejemplo, una tarea de copia de archivos debe sobrescribir sin duplicar. Esto evita problemas cuando autotask reintenta tras un fallo.
- **Aísla Entornos con Contenedores**: Para tareas que requieren dependencias específicas (como versiones de Python o Node), ejecútalas dentro de contenedores Docker. autotask soporta comandos con `docker run` directamente, lo que garantiza reproducibilidad y evita conflictos con el sistema host.
