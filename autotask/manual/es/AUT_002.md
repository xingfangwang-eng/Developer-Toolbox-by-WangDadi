# autotask - AUT_002

## Resumen

autotask es un orquestador de tareas automatizadas diseñado para entornos SaaS que requieren ejecución programática sin intervención manual. Optimizado para flujos de trabajo asíncronos y pipelines de datos recurrentes, este proyecto elimina la fricción operativa en la gestión de tareas batch, sincronización entre microservicios y procesamiento de colas.

## Contenido Detallado

### ¿Qué es autotask?

autotask no es un simple gestor de tareas: es un motor de ejecución tolerante a fallos que prioriza la atomicidad y la trazabilidad en cada operación. Ideal para equipos de backend que necesitan orquestar procesos complejos sin depender de cron jobs frágiles o soluciones monolíticas. Con autotask, defines la lógica de negocio en archivos YAML o JSON, y el sistema se encarga de la ejecución distribuida, reintentos exponenciales y notificaciones en tiempo real.

### Características Principales

- **Ejecución atómica con rollback automático**: Cada tarea se encapsula como una transacción. Si un paso falla, autotask ejecuta automáticamente un rollback parcial o total según la configuración de dependencias, evitando estados inconsistentes en tu sistema.

- **Pipeline de tareas con DAG dinámico**: Define grafos acíclicos dirigidos (DAG) que se resuelven en tiempo de ejecución. Soporta bifurcaciones condicionales, paralelismo controlado y sincronización mediante barreras (barriers) sin necesidad de código adicional.

- **Observabilidad nativa con métricas y logs estructurados**: Cada tarea genera trazas OpenTelemetry compatibles con Prometheus y Grafana. Incluye un dashboard embebido para monitorear latencias, tasas de fallo y cuellos de botella en tiempo real.

### Cómo Usar

1. **Configuración del pipeline con dependencias condicionales**  
   Crea un archivo `autotask.yaml` con la definición de tareas y sus dependencias. Usa `depends_on` con condiciones lógicas (ej: `if: "{{ task_A.status == 'success' }}"`) para controlar el flujo. Ejecuta `autotask run --pipeline pipeline_name` para lanzar el DAG.

2. **Gestión de reintentos con backoff exponencial**  
   Para tareas propensas a fallos transitorios, define en el YAML `retry_policy` con parámetros como `max_retries: 5`, `backoff_factor: 2.0` y `jitter: true`. autotask aplica automáticamente un tiempo de espera creciente entre reintentos, minimizando la carga en servicios externos.

3. **Monitorización en vivo con alerts personalizados**  
   Integra autotask con tu sistema de logging centralizado usando `--log-format json`. Para recibir notificaciones en Slack o Discord, configura `notifiers` en el archivo de configuración global. Usa `autotask status --watch` para ver el progreso en tiempo real desde terminal.

### Mejores Prácticas

- **Divide tareas en micro-pasos idempotentes**: Cada tarea debe poder ejecutarse múltiples veces sin efectos secundarios. Usa identificadores únicos (UUID) para cada ejecución y almacena estados intermedios en un almacén persistente (Redis o PostgreSQL). Esto facilita la recuperación ante fallos y la depuración.

- **Establece timeouts y límites de recursos por tarea**: Define `timeout_seconds` y `max_memory_mb` en cada tarea para evitar que un proceso malicioso o con fugas de memoria degrade el rendimiento global. autotask aborta automáticamente tareas que excedan los límites, liberando recursos para otras operaciones.
