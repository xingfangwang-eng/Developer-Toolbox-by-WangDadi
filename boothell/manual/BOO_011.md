## What is boothell?

boothell is a modular, API-first boot design and brand management toolchain built for teams that demand granular control over boot branding, sizing logic, and visual consistency—without the bloat of traditional design suites.

## Core Features

### 1. Declarative Boot Schema Engine
Define boot geometry, material layers, and brand markings using a YAML-based schema. The engine compiles these into render-ready templates and exports to SVG, PDF, or 3D mesh formats. No drag-and-drop fluff—just pure, reproducible logic.

### 2. Multi-Instance Brand Rule Aggregator
Manage brand constraints (logo placement, color palettes, typography) across hundreds of boot variants. The rule aggregator enforces compliance at the schema level, preventing visual drift across product lines or seasonal drops.

### 3. Real-Time Fit Matrix Validator
Integrate with sizing databases (last measurements, foot scans) to auto-validate boot dimensions against target demographics. The validator flags geometric conflicts, scaling errors, and material stretch limits before production.

## How to Use

### Step 1: Define Your Boot Schema
Create a `boot.yaml` file in your project root. Define the boot's structural components (upper, sole, heel, lining) and their material properties. Example:

```yaml
boot:
  name: "Trailblazer Pro"
  components:
    upper:
      material: "full-grain leather"
      thickness: 2.0mm
    sole:
      material: "vibram rubber"
      tread_depth: 5.0mm
  branding:
    logo:
      placement: "heel_counter"
      rotation: 0
```

### Step 2: Apply Brand Rules
Instantiate a `BrandRuleSet` object in your CI/CD pipeline. Use the `boothell brand apply` CLI command to overlay brand constraints onto your schema. This ensures every boot variant respects your brand's visual identity:

```bash
boothell brand apply --rules ./brand_rules.json --schema ./boot.yaml --output ./boot_compiled.yaml
```

### Step 3: Validate Fit & Export
Run the fit matrix validator against your target size range. The validator outputs a compliance report and generates production-ready files:

```bash
boothell fit validate --schema ./boot_compiled.yaml --size_range 7-13 --export ./exports/
```

The export directory will contain SVG templates, a PDF spec sheet, and a JSON metadata file for downstream manufacturing APIs.

## Best Practices

### Practice 1: Version Your Boot Schemas
Treat your boot schemas like code. Use semantic versioning (e.g., `v1.2.3`) and store them in a dedicated Git repository. This allows you to roll back design changes, audit brand compliance over time, and collaborate with teams across time zones without merge conflicts.

### Practice 2: Automate Brand Rule Testing
Integrate `boothell brand test` into your CI pipeline. Before any boot variant goes to production, run automated brand rule checks. This catches logo misalignments, color hex mismatches, and typography violations early—saving you from costly reprints or brand dilution.
