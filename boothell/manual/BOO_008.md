## What is boothell?

boothell is a cloud-native, API-first boot design and customization engine. It enables teams to prototype, iterate, and deploy boot configurations at scale—without touching legacy CAD tools. Think of it as the headless CMS for boot manufacturing, with a developer-friendly CLI and a drag-and-drop UI for non-technical stakeholders.

## Core Features

- **Real-Time 3D Boot Configurator with WebGL Renderer**  
  Instantly visualize boot geometry changes (heel height, shaft length, material texture) in-browser. Supports glTF/GLB export for downstream CAD pipelines. No plugin required.

- **Multi-Tenant Brand Asset Manager**  
  Centralize all boot design assets (lasts, patterns, colorways) with version control and role-based access. Each brand tenant gets isolated namespaces, audit logs, and automated backups to S3-compatible storage.

- **AI-Powered Fit Prediction Engine**  
  Feed in foot scan data (3D mesh or CSV) and boothell predicts fit score across 12 anatomical zones. Uses a lightweight ONNX model that runs on edge or cloud. Outputs heatmaps and sizing recommendations in milliseconds.

## How to Use

### 1. Initialize a Boot Design Project via CLI

```bash
boothell init --project "trail-hiker-2025" --template "tactical-boot-v3"
```

This creates a Git-tracked project directory with a `boothell.yaml` manifest. The manifest defines default materials, last IDs, and export profiles. Run `boothell validate` to check all dependencies (e.g., Node 18+, Docker for local rendering).

### 2. Push a Custom Colorway to the Brand Asset Manager

```bash
boothell asset push --path ./colorways/olive-drab.json --tenant "alpha-outdoor"
```

The JSON schema follows boothell's `ColorwaySpec` v2.1: includes `{ name, hex, finish (matte/gloss), supplier_code, environmental_rating }`. The asset manager auto-generates a preview thumbnail and fires a webhook to your CI/CD pipeline.

### 3. Deploy a Boothell Configurator Widget to Your E-Commerce Site

```bash
boothell deploy --widget "3d-configurator" --target "https://shop.alphaboots.com/product/123"
```

This generates an embeddable `<iframe>` snippet with lazy loading, analytics tracking, and a fallback 2D view for low-bandwidth users. The widget communicates with boothell's API via signed JWTs. Monitor real-time usage with `boothell logs --widget <id>`.

## Best Practices

- **Always version your boothell.yaml manifest**  
  Treat it like a `Dockerfile`—lock material IDs, last versions, and export format. Use Git tags to correlate boothell releases with production boot runs. This prevents "it worked on my machine" syndrome in manufacturing.

- **Leverage the AI Fit Prediction before finalizing production batches**  
  Run fit tests on a representative sample of foot scans (min 50) before committing to injection molding. boothell's prediction engine can flag fit outliers at the 95th percentile, saving thousands in returns and warranty claims.
