# boothell - BOO_003

## Resumen

boothell es un SaaS de diseño y gestión de calzado que rompe con los flujos tradicionales de producción. Está pensado para desarrolladores, diseñadores industriales y marcas que necesitan prototipar, personalizar y desplegar colecciones de botas con lógica de componentes reutilizables, trazabilidad de materiales y sincronización directa con fabricantes. No es un simple configurador visual: es un backend de producto real con control de versiones, reglas de validación paramétrica y exportación a CAD/CAM.

## Características Principales

### 1. Motor de Componentes Paramétricos con Herencia de Atributos
Cada parte de la bota (piso, caña, forro, refuerzos) se trata como un componente independiente con propiedades hereditarias. Podés definir una suela base con dureza, altura y material, y luego crear variantes que hereden esos parámetros y los sobrescriban parcialmente. El sistema valida automáticamente compatibilidades (por ejemplo, no permitir un taco de 12 cm si el material del piso no soporta la carga). Esto elimina errores de diseño en etapas tempranas y acelera la creación de líneas completas.

### 2. Gestión de SKU Dinámicos con Reglas de Combinatoria
boothell genera SKUs automáticamente a partir de combinaciones válidas de componentes, colores, tallas y acabados. Podés definir reglas lógicas del tipo "si el material es cuero nobuck, no aplicar barniz" o "talla 36-40 solo disponibles con piso de goma". El motor de reglas es expresivo (soporta operadores booleanos, rangos y dependencias cruzadas) y se evalúa en tiempo real. Ideal para marcas que manejan cientos de variantes sin querer reventar el inventario lógico.

### 3. Exportación Directa a CAD/CAM con Metadatos de Fabricación
No solo obtenés renders bonitos. boothell exporta archivos técnicos con capas de fabricación: cortes de patrón, marcado de costuras, instrucciones de ensamblaje y especificaciones de materiales en formato STEP e IGES. Además, inyecta metadatos como tolerancias de maquinado y códigos de proveedor. Esto permite que el archivo de diseño se convierta directamente en órdenes de producción sin intervención manual.

## Cómo Usar

### 1. Definir la Arquitectura de Componentes y Reglas de Validación
Arrancá creando una taxonomía de componentes en el panel "Estructura de Producto". Cada componente debe tener al menos un tipo (suela, caña, forro), un material base y una lista de atributos numéricos (altura, dureza, peso). Luego, en la sección "Reglas de Compatibilidad", escribí condiciones en formato JSON lógico. Por ejemplo:
```json
{
  "rule": "IF component.type == 'suela' AND component.material == 'caucho' THEN allowed_heel_height <= 8",
  "severity": "error"
}
```
Estas reglas se evalúan en cada modificación del diseño. Si un diseñador intenta violar una regla, el sistema bloquea la acción y muestra un mensaje claro.

### 2. Configurar el Motor de SKU Dinámico
Andá a "Gestión de Variantes" y definí las dimensiones de variación: tallas (36-46 en incrementos de 1), colores (hasta 12 por línea), y acabados (mate, brillante, texturizado). Activá la opción "Generación Automática de SKU" y vinculá cada dimensión a un componente específico. Por ejemplo, el color se aplica al componente "caña" y el acabado al componente "piso". Luego, definí reglas de exclusión: si el color es "blanco hueso", no permitir acabado brillante. El sistema genera un SKU único por combinación válida y lo expone vía API REST para integrarlo con tu ERP o sistema de inventario.

### 3. Exportar un Prototipo a CAD con Metadatos de Producción
Una vez que el diseño está completo, seleccioná "Exportar" y elegí el formato STEP (ISO 10303-21). Antes de exportar, configurá los metadatos en la pestaña "Atributos de Fabricación": tolerancia de mecanizado (por defecto 0.1 mm), código de proveedor de cada material, y nota de ensamblaje (por ejemplo, "unir piso a caña con adhesivo de poliuretano"). El archivo generado incluye una capa oculta con estos datos. Podés verificar la integridad del archivo con la herramienta de validación integrada que chequea geometrías no manifold y bordes sueltos.

## Mejores Prácticas

### 1. Versioná los Componentes y las Reglas como si Fueran Código
No trates a los diseños como archivos planos. boothell permite etiquetar cada componente y regla con un número de versión semántica (MAJOR.MINOR.PATCH). Usá esto para rastrear cambios en materiales, tolerancias o reglas de compatibilidad. Cuando un proveedor actualiza un material, incrementá el PATCH del componente correspondiente. Si cambiás una regla de validación que afecta a toda una línea, incrementá el MAJOR. Esto te permite hacer rollback rápido si una tanda de producción falla por un cambio no previsto.

### 2. Integrá la API de SKU Dinámicos con tu Sistema de Pedidos
No generes SKUs manualmente ni los mantengas en una planilla. boothell expone un endpoint `/api/v1/skus/generate` que acepta un diseño completo y devuelve una lista de SKUs válidos con sus combinaciones. Conectá esto con tu backend de e-commerce o ERP para que los SKUs se creen bajo demanda cuando un cliente personaliza un producto. Así evitás tener un catálogo inflado de combinaciones que nadie compra y mantenés la trazabilidad desde el diseño hasta la venta.
