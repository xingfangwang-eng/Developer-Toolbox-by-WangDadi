# boothell - BOO_027

## Resumen

Boothell es un toolkit de branding y diseño de calzado conceptual que permite a desarrolladores y diseñadores industriales gestionar, prototipar y optimizar catálogos de botas mediante una arquitectura modular basada en JSON. Ideal para equipos que buscan estandarizar la personalización de calzado técnico (trekking, motocross, militar) y automatizar la generación de assets visuales para e-commerce.

## Contenido Detallado

### ¿Qué es boothell?

Boothell es un sistema de gestión de diseño de calzado que combina un schema semántico de componentes (suela, caña, cordones, refuerzos) con un motor de renderizado headless. Permite a equipos multidisciplinarios definir reglas de personalización, validar compatibilidad de materiales y exportar especificaciones técnicas listas para manufactura o integración con plataformas de venta como Shopify o Magento.

### Características Principales

- **Schema semántico de componentes (bootSchema v2)**  
  Define cada parte de la bota como un objeto JSON tipado con propiedades de material, color, tolerancia y compatibilidad. Ideal para equipos que necesitan validar combinaciones de componentes antes de prototipar.

- **Motor de reglas de compatibilidad (Rule Engine)**  
  Implementa lógica condicional para evitar combinaciones imposibles (ej. suela de tacón con caña de motocross). Las reglas se escriben en YAML y se evalúan en tiempo real durante la edición del modelo.

- **Exportación multi-formato con metadatos SEO**  
  Genera automáticamente archivos STL, OBJ y JSON con metadatos de producto (SKU, variantes, palabras clave) listos para alimentar catálogos digitales o sistemas de fabricación aditiva.

### Cómo Usar

1. **Clonar el repositorio e inicializar el entorno de desarrollo**  
   ```bash
   git clone https://github.com/xingfangwang-eng/boothell.git
   cd boothell
   npm install
   cp .env.example .env
   ```
   Configura las variables de entorno para conectar con tu sistema de gestión de inventarios (por defecto, usa SQLite para prototipado).

2. **Definir un modelo base mediante bootSchema**  
   Crea un archivo `models/mi-bota.json` con la siguiente estructura mínima:  
   ```json
   {
     "modelo": "TrailX",
     "componentes": {
       "suela": { "material": "goma_vulcanizada", "color": "#2C3E50" },
       "caña": { "material": "cuero_impermeable", "altura": 25 },
       "cordones": { "tipo": "redondos", "longitud": 120 }
     }
   }
   ```
   Ejecuta `node cli.js validate models/mi-bota.json` para verificar compatibilidad.

3. **Renderizar y exportar assets optimizados**  
   Usa el comando de exportación para generar los archivos técnicos y metadatos:  
   ```bash
   node cli.js export models/mi-bota.json --format stl,obj,json --seo-keywords "bota_trekking, calzado_impermeable, diseño_modular"
   ```
   Los archivos se guardarán en `dist/` con nombres normalizados y listos para subir a tu CMS.

### Mejores Prácticas

- **Versiona tus schemas con Git LFS**  
  Los archivos JSON de componentes pueden crecer rápido si incluyes texturas o mallas 3D. Usa Git LFS para evitar saturar el repositorio y mantener la trazabilidad de cambios en cada iteración de diseño.

- **Integra boothell con un pipeline CI/CD para validación automática**  
  Configura un hook de pre-commit que ejecute `boothell validate` sobre cualquier cambio en la carpeta `models/`. Así evitas que combinaciones inválidas lleguen a producción o a tu sistema de fabricación.
