# boothell - BOO_018

## Resumen

**boothell** es una herramienta SaaS de arquitectura modular diseñada para el diseño, personalización y gestión de marca de botas de alta gama. Optimizada para SEO multilingüe y desarrollada con un stack moderno (Node.js + React + GraphQL), permite a equipos de producto y marketing lanzar configuradores visuales de calzado en minutos, sin tocar una línea de código.

## Contenido Detallado

### ¿Qué es boothell?

boothell es un **motor de personalización visual** que resuelve el cuello de botella entre diseño industrial y experiencia de compra online. No es solo un configurador: es una plataforma de branding dinámico que sincroniza catálogos, precios y reglas de negocio en tiempo real. Ideal para marcas que quieren ofrecer personalización masiva (mass customization) sin sacrificar rendimiento ni escalabilidad.

### Características Principales (Hardcore)

1. **Motor de Reglas de Negocio Dinámicas (DRB Engine)**  
   Permite definir lógica condicional compleja: «Si el usuario selecciona cuero vegano + suela Vibram, aplicar descuento del 12% y bloquear opción de tacón metálico». Todo en YAML, con validación en tiempo real y sin tocar el backend.

2. **Generación de Variantes en Tiempo Real (VRT)**  
   Cada combinación de materiales, colores y acabados genera una URL única con metadatos SEO. El sistema renderiza previsualizaciones 3D en WebGL y produce automáticamente sitemaps, microdatos y etiquetas Open Graph para cada variante.

3. **API de Sincronización Multicanal (SyncMesh)**  
   Conecta boothell con Shopify, Magento, WooCommerce y sistemas ERP. Cada cambio en el configurador se refleja al instante en stock, precios y descripciones. Incluye webhooks y cola de reintentos con backoff exponencial.

### Cómo Usar (Pasos Detallados)

1. **Definir la taxonomía de producto en el panel de administración**  
   Accede a `/admin/taxonomy` y crea un árbol de atributos: tipo de piel, altura de caña, tipo de suela, color de cordones, etc. Cada atributo acepta valores con peso, precio adicional y reglas de compatibilidad. Usa el validador integrado para evitar conflictos lógicos antes de publicar.

2. **Configurar reglas de negocio en el archivo `rules.yaml`**  
   Edita el archivo en la raíz del proyecto o súbelo desde la UI. Ejemplo de regla:  
   ```yaml
   - if: { material: "cuero_full_grain", suela: "vibram_arctic" }
     then:
       discount: 0.10
       block: ["tacon_metalico", "plantilla_ortopedica"]
   ```  
   El sistema parsea y valida automáticamente cada regla; los errores se muestran en el panel de depuración.

3. **Desplegar el configurador en tu dominio**  
   Ejecuta `boothell deploy --domain mibotas.com` desde la CLI. El asistente genera un embed code (iframe o Web Component) que puedes insertar en cualquier página. boothell se encarga del SEO: genera automáticamente un `robots.txt`, un `sitemap.xml` y metaetiquetas para cada variante. Verifica con `boothell seo-check` que todo esté indexable.

### Mejores Prácticas (Industry Best Practices)

1. **Usa reglas de negocio como capa de testing A/B**  
   En lugar de hardcodear descuentos o bloqueos, define reglas condicionales que puedas activar/desactivar sin desplegar. Esto permite probar estrategias de precios dinámicos o limitar opciones según el canal de tráfico (orgánico, paid, email).

2. **Optimiza el tiempo de carga del configurador con lazy loading de assets 3D**  
   boothell permite cargar texturas y modelos solo cuando el usuario interactúa con una opción específica. Activa `lazyTextures: true` en el archivo `boothell.config.json` para reducir el peso inicial de la página y mejorar el Core Web Vitals (LCP, FID).
