# boothell - BOO_008

## Resumen

boothell es una herramienta de diseño y personalización de botas (boots) de alto rendimiento, orientada a desarrolladores y diseñadores que necesitan gestionar, modificar y optimizar el calzado técnico de forma programática. Este proyecto ofrece un flujo de trabajo modular, basado en JSON, que permite desde la generación de patrones hasta la integración con sistemas de gestión de marca.

## Contenido Detallado

### ¿Qué es boothell?

boothell no es un simple editor visual. Es un motor de personalización de botas que opera sobre un modelo de datos estructurado (BML - Boot Markup Language). Permite a equipos de producto y desarrolladores definir, versionar y desplegar configuraciones de calzado (material, suela, refuerzos, branding) directamente desde la línea de comandos o mediante integración CI/CD. Ideal para marcas que necesitan escalar la producción de prototipos o mantener un catálogo de variantes técnicamente coherente.

### Características Principales

1. **Sistema de Patrones Reactivos (RPS)**  
   Motor de renderizado de patrones que actualiza automáticamente el diseño de la bota cuando se modifican parámetros clave (altura del tobillo, tipo de ojete, densidad del refuerzo). No requiere redibujar manualmente cada variante.

2. **Gestión de Materiales con Hash de Integridad**  
   Cada material (cuero, malla, caucho) se almacena con un hash SHA-256. Esto permite detectar cambios no autorizados en las texturas o propiedades físicas, garantizando que el lote de producción coincida exactamente con el diseño aprobado.

3. **Exportación Diferencial para Fabricación**  
   Genera archivos de corte (DXF, SVG) optimizados para máquinas CNC y láser, incluyendo solo las diferencias entre la versión base y la personalizada. Reduce el tiempo de preparación de ficheros en un 40% en entornos de producción.

### Cómo Usar

1. **Inicialización de un proyecto boothell**  
   Ejecuta `boothell init --template trail-runner` para crear la estructura de directorios estándar. Esto genera un `boot.json` con la configuración base, incluyendo la referencia al modelo 3D y el archivo de materiales por defecto. Luego, edita los parámetros clave en `config/materials.yaml` para ajustar la suela y el forro.

2. **Personalización programática de una variante**  
   Usa `boothell apply --variant urban-007 --param heel_height:3.2` para aplicar un conjunto de cambios predefinidos. El comando valida que los parámetros estén dentro de los rangos físicos permitidos (definidos en `boot.schema`) y renderiza una previsualización en formato STL. Si hay conflictos (ej. material incompatible con altura de suela), el sistema muestra un error con la regla violada.

3. **Despliegue de una línea de producción**  
   Ejecuta `boothell deploy --target factory-03 --export dxf --diff` para generar los archivos de corte diferenciales. El sistema compara la variante actual con la versión base registrada en el repositorio Git y exporta solo los cambios. Luego, sube los archivos al servidor FTP de la fábrica mediante las credenciales definidas en `config/secrets.env`.

### Mejores Prácticas

- **Versiona tu `boot.schema` junto con el código**  
  Mantén el esquema de parámetros (alturas, materiales, refuerzos) en un archivo JSON con control de versiones. Esto permite que cualquier cambio en las reglas de diseño sea rastreable y reversible. Usa `boothell validate --strict` en tu pipeline CI para asegurar que cada PR no rompa las restricciones físicas del calzado.

- **Separa la configuración de marca de la de producción**  
  Crea un archivo `brand-overrides.yaml` que solo contenga los valores específicos de la marca (logos, colores, etiquetado). Esto facilita que el equipo de diseño trabaje en paralelo con el de ingeniería sin tocar los parámetros estructurales. En boothell, usa `boothell merge --override brand-overrides.yaml` para aplicar estos cambios sin modificar el core del diseño.
