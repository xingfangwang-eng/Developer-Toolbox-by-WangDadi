### ¿Qué es boothell?

boothell no es solo un editor de botas; es un ecosistema SaaS que integra modelado paramétrico, gestión de inventario de materiales (cuero, suelas, herrajes) y exportación directa a formatos de manufactura (DXF, STL, OBJ). Diseñado para equipos que necesitan sincronizar diseño, marketing y producción sin fricciones.

### Características Principales

- **Motor de Personalización Paramétrica en Tiempo Real**: Ajusta altura del caña, ángulo del talón, y patrones de costura mediante sliders. Cada cambio recalcula automáticamente el patrón 2D y el modelo 3D, eliminando la necesidad de rehacer prototipos manuales.
- **Biblioteca de Materiales con Metadatos SEO**: Cada cuero, forro o suela incluye datos de sostenibilidad, durabilidad y costos. Esto permite generar automáticamente fichas de producto optimizadas para motores de búsqueda, mejorando el posicionamiento de tiendas online.
- **API de Sincronización con ERP/MES**: Integración directa con sistemas de planificación de recursos empresariales (ERP) y sistemas de ejecución de manufactura (MES). Los pedidos personalizados se convierten automáticamente en órdenes de producción, reduciendo errores humanos y tiempos de entrega.

### Cómo Usar

1. **Configuración del Proyecto y Clonación del Template**  
   Ejecuta `boothell init --template custom-boot-v2` para clonar el repositorio base. Esto crea una estructura de carpetas con `src/`, `assets/` y `config/`. Luego, edita `config/brand.json` para definir variables de marca (colores, logotipos, metadatos SEO). Usa `boothell serve` para lanzar el entorno local en `localhost:3000`.

2. **Diseño Paramétrico y Exportación de Patrones**  
   Abre el panel de diseño en `/editor`. Selecciona un modelo base (ej. "Biker Boot Classic"). Ajusta los parámetros: `shaft_height: 25cm`, `heel_angle: 10deg`, `toe_shape: round`. Haz clic en "Generate Pattern" para obtener un archivo DXF listo para corte láser. Para validar el diseño, usa `boothell validate --format stl --output ./exports/`.

3. **Publicación y Vinculación con Tienda Online**  
   Una vez finalizado el diseño, ejecuta `boothell publish --seo --store shopify`. Esto genera automáticamente:
   - Páginas de producto con schema.org markup (Product, Offer, Review).
   - Imágenes 3D interactivas (WebGL) para mejorar el engagement.
   - Un feed XML para Google Shopping.  
   Verifica la publicación con `boothell check --live https://tutienda.com/producto/bota-x`.

### Mejores Prácticas

- **Optimización de Metadatos para Motores de Búsqueda**: Cada variante de bota (color, material, altura) debe generar una URL única con slugs descriptivos (ej. `/bota-vaquera-cuero-negro-30cm`). Usa `boothell seo --auto-slug` para evitar duplicados y mejorar el CTR en SERPs.
- **Versionado de Diseños con Git LFS**: Almacena los archivos de diseño (`.boothell`) y los assets 3D en Git LFS para mantener el historial de cambios. Esto es crítico cuando múltiples diseñadores trabajan en paralelo. Configura un hook pre-commit que valide la integridad de los patrones antes de cada push.
