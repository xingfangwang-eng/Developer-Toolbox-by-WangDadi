## What is boothell?

**boothell** is a high-performance, modular boot design and brand management toolkit built for modern SaaS teams and boot manufacturers. It eliminates the friction between design, prototyping, and production by providing a unified API-first platform for boot customization, version control, and brand compliance — all in one CLI-friendly environment.

Unlike traditional design tools that treat boots as static assets, boothell treats them as living, versioned components. It’s built for teams who need to ship branded boot configurations across multiple SKUs, regions, and supply chains without reinventing the wheel.

---

## Core Features (Hardcore, Not Fluff)

### 1. **Declarative Boot Schema Engine (DBSE)**
Define your boot model as a YAML/JSON schema. Boothell automatically generates a fully typed API surface, validation rules, and a visual editor from your schema. No more manual UI wiring. Supports nested components, conditional fields, and multi-tenant isolation out of the box.

### 2. **Real-Time Brand Compliance & Version Diffing**
Every boot configuration is tracked as a Git-like commit. Boothell’s diff engine highlights breaking changes in material, color, or sizing across versions. Built-in brand rules (e.g., “never use neon green on safety boots”) are enforced at the API level, not just in the UI. Integrates with GitHub Actions and CI pipelines for automated compliance checks.

### 3. **Multi-Locale Boot Configurator with Auto-Translation**
Boothell ships with a built-in i18n engine that auto-translates boot labels, sizing guides, and marketing copy into 10+ languages (including RTL support). It also handles locale-specific sizing conversions (US, EU, UK, JP) and currency formatting. No third-party translation service required.

## How to Use (3 Pro-Level Steps)

### Step 1: Initialize a Boot Schema
```bash
boothell init --type boot --brand "HeavyDuty" --output schema.yaml
```
This creates a boot schema with default fields: `model`, `size_range`, `material`, `color_palette`, `sole_type`, `toe_type`, and `brand_logo_url`. The schema is auto-validated against boothell’s built-in boot ontology. You can extend it with custom fields using the `--extend` flag.

### Step 2: Configure a Boot Variant via API
```python
import boothell

client = boothell.Client(api_key="sk-...")
variant = client.create_variant(
    schema="schema.yaml",
    overrides={
        "model": "HDB-9000",
        "material": "full-grain leather",
        "sole_type": "oil-resistant rubber",
        "color_palette": ["black", "brown"],
        "size_range": ["7", "8", "9", "10", "11", "12"],
        "locale": "de_DE"
    }
)
print(variant.id)  # Output: boot_variant_abc123
```
This triggers automatic brand compliance checks, locale-aware size conversion (US→EU), and generates a preview URL. The variant is immediately versioned and available for downstream integration.

### Step 3: Deploy a Brand Compliance Rule
```yaml
# .boothell/rules.yaml
rules:
  - name: "no_neon_on_safety"
    condition: "material == 'safety' && color in ['neon_green', 'neon_pink']"
    action: "reject"
    message: "Safety boots must use ANSI-approved reflective colors only."
```
Push this to your repo, and boothell will enforce it in CI. Any variant creation that violates the rule will return a `422` with the exact message. You can also set `action: warn` for soft enforcement.

## Best Practices (Industry-Level)

### 1. **Version Everything, Including Brand Rules**
Treat your boothell schemas, rules, and variants as code. Store them in a monorepo with semantic versioning. Use boothell’s `boothell diff` command to compare two versions of a boot config before deploying to production. This prevents silent regressions in sizing or material compliance.

### 2. **Use Locale-Aware Testing Before Shipment**
Before pushing a new boot variant to your e-commerce store or catalog, run `boothell test --locale all` to validate that all translations, size conversions, and currency formats are correct. Boothell will flag any missing or inconsistent locale data. This is especially critical for brands shipping to Japan, Germany, or the Middle East.
