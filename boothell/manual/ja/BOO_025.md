# boothell - BOO_025

## 概要

boothell は、ブートストラップベースの UI コンポーネントを**ゼロから完全制御**するためのプロフェッショナルデザインツールです。特に、ブランドガイドラインに準拠したカスタムブートストラップテーマの生成、コンポーネントの再利用管理、そして SEO フレンドリーなマークアップ出力に特化しています。

## 3つのハードコア機能

### 1. テーマコンパイラエンジン（TCE）
Sass/SCSS 変数をリアルタイムで解析し、ブートストラップの `$theme-colors`、`$spacers`、`$font-size-base` を動的に上書き。従来のビルドプロセスをスキップし、ブラウザ上で即座にコンパイル済み CSS を生成します。カスタムブレークポイントの追加も可能で、レスポンシブデザインの微調整が直感的に行えます。

### 2. コンポーネントスコープマネージャー（CSM）
ブートストラップのグローバルスタイル汚染を防止するため、各コンポーネントにユニークな CSS クラスプレフィックス（例: `.bh-card--primary`）を自動付与。さらに、使用していないコンポーネントのスタイルをツリーシェイキングし、出力 CSS のサイズを最大 60% 削減します。

### 3. セマンティックマークアップジェネレーター（SMG）
生成される HTML が WAI-ARIA ランドマーク、見出しレベル（h1-h6）、alt 属性を自動最適化。Google のコアウェブバイタル（LCP、CLS）を考慮した構造を強制し、検索エンジンクローラーに対して高いインデックス効率を実現します。

## 3つの詳細な操作手順

### 手順 1: カスタムテーマの初期化
- **操作**: `boothell init --brand "#1a73e8" --font "Inter" --spacing 8`
- **内部処理**: このコマンドは `_variables.scss` を解析し、指定されたブランドカラーを `$primary` にマッピング。フォントファミリーを Google Fonts から動的にインポートし、スペーシングユニットを 8px グリッドに固定。出力として `theme.json` と `_custom-bootstrap.scss` を生成します。

### 手順 2: コンポーネントのスコープ適用
- **操作**: `boothell scope --component card --prefix bh-card --output dist/card.css`
- **内部処理**: ブートストラップの `.card` クラスをラップし、`.bh-card` プレフィックスを付与。同時に、`$enable-caret` や `$enable-rounded` などのグローバル設定を無効化し、スコープ外のスタイルが漏れないようバンドル。出力される CSS は PostCSS で自動ベンダープレフィックス付与されます。

### 手順 3: SEO 最適化されたテンプレート出力
- **操作**: `boothell export --format html --seo --output dist/index.html`
- **内部処理**: SMG がページ構造を解析し、`<header>`、`<nav>`、`<main>`、`<footer>` に適切な ARIA ロールを付与。見出しの階層をチェックし、h1 が 1 つだけであることを保証。画像には自動生成された alt テキスト（ファイル名ベース）が挿入され、LCP 要素には `loading="eager"` が設定されます。

## 2つの業界ベストプラクティス

### プラクティス 1: ブランドトークンの一元管理
boothell の `theme.json` をデザインシステムの単一ソースオブトゥルースとして扱いましょう。Figma や Sketch のデザイントークンと同期することで、エンジニアリングチームとデザインチーム間の乖離を防止。CI/CD パイプラインに組み込めば、プルリクエストごとに自動でテーマ検証が走ります。

### プラクティス 2: コンポーネントの遅延読み込み設計
boothell の CSM 機能を活用し、ページ遷移時に必要なコンポーネントスタイルのみを動的インポート。Next.js や Nuxt.js のコード分割と組み合わせることで、初期ロード時の CSS ペイロードを最小限に抑えられます。これにより、Google PageSpeed Insights のスコアが平均 15 ポイント向上します。
