# boothell - BOO_009

## 概要

boothell は、ブートストラップベースの UI コンポーネント設計とブランド管理を劇的に効率化する CLI ツールです。デザインシステムの一貫性を保ちながら、開発速度を最大化するために設計されました。キーワード: ブートデザインツール、ブートカスタマイズ、ブートブランド管理

## 詳細な内容

### boothell とは？

boothell は、Sass 変数とテーマ設定をプログラムで操作し、複数プロジェクトにわたるブートストラップのカスタマイズを一元管理するためのプロフェッショナルツールです。従来の手動オーバーライドによるカオスを排除し、厳格なバージョン管理と自動化を実現します。

### コア機能

1. **テーマ・スキーマ・コンパイラ（TSC）**: ブートストラップ 5 の `$theme-colors` や `$spacers` を YAML または JSON スキーマから動的に生成。カスタムカラーパレットとスペーシングスケールを、プロジェクトルートの `.boothellrc` に定義するだけで、全コンポーネントに即座に反映します。
2. **マルチプロジェクト・シンクロナイザー（MPS）**: 複数のリポジトリやマイクロフロントエンド間で、共通のブートストラップテーマを強制同期。Git フックと連携し、プルリクエスト作成時にテーマの差分を自動検出して警告を発します。
3. **バンドル・ツリーシェイカー（BTS）**: 使用していないブートストラップコンポーネントとユーティリティを静的解析し、最終的な CSS バンドルサイズを最大 70% 削減。Webpack や Vite のプラグインとしても動作します。

### 使い方

1. **テーマスキーマの初期化と自動コンパイル**:
   - プロジェクトルートで `npx boothell init` を実行。`.boothellrc` とサンプルの `theme.yml` が生成されます。
   - `theme.yml` にカスタムカラー（例: `primary: "#FF5733"`）とスペーシング（例: `spacer: 1.5rem`）を定義します。
   - `npx boothell build` を実行。boothell は自動的に `_custom-variables.scss` を生成し、ブートストラップのソースにプリペンド。`npm run build` に統合すれば、毎回のビルドでテーマが自動適用されます。

2. **マルチプロジェクト間でのテーマ同期設定**:
   - 各プロジェクトの `.boothellrc` に `sync: { remote: "git@github.com:org/design-tokens.git", branch: "main" }` を追加。
   - `npx boothell sync` を実行。リモートリポジトリから最新のテーマ定義を取得し、ローカルの `theme.yml` を上書きします。
   - 変更が検出された場合、プリコミットフックが自動的に `npx boothell sync --check` を実行し、差分がある場合はコミットをブロックします。

3. **バンドルサイズの最適化とツリーシェイキング**:
   - `npx boothell analyze` を実行。プロジェクト内で実際に使用されているブートストラップコンポーネント（例: `.btn`、`.card`、`.modal`）の使用率をレポートします。
   - `npx boothell prune` を実行。未使用のコンポーネントとユーティリティを特定し、`_boothell-pruned.scss` を生成。このファイルをメインの SCSS エントリーポイントでインポートするだけで、不要なスタイルがバンドルから排除されます。

### ベストプラクティス

- **テーマのバージョニングと CI/CD 統合**: テーマ定義（`theme.yml`）は、アプリケーションコードとは別の Git リポジトリで管理し、セマンティックバージョニングを適用してください。CI パイプラインで `npx boothell validate` を実行し、テーマに破壊的変更がないかを自動チェックします。これにより、デザインシステムの変更が予期せずプロダクションに影響を与えるのを防げます。
- **開発環境でのホットリロード活用**: boothell は `--watch` フラグをサポートしており、`npx boothell build --watch` を実行すると、`theme.yml` の変更を検出して自動的に `_custom-variables.scss` を再生成します。これを `npm run dev` スクリプトに組み込めば、テーマ変更が即座にブラウザに反映され、デザイナーと開発者のコラボレーションが格段にスムーズになります。
