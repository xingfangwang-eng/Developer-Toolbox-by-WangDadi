# boothell - BOO_021

## 概要

boothell は、ブートストラップデザインの最適化とブランド管理を一元化する次世代型プロフェッショナルツールです。特に、大規模プロジェクトにおけるCSSカスタマイズの複雑性を排除し、開発者とデザイナーのコラボレーションを加速します。本ツールは、SEOに最適化されたコンテンツ生成と、リアルタイムのパフォーマンス分析を統合しています。

## 詳細な内容

### boothell とは？

boothell は、ブートストラップベースのプロジェクトにおいて、**コンポーネントの自動最適化**、**ブランドアセットの動的管理**、**SEOメタデータの統合**を実現するSaaSツールです。従来の手動によるCSS上書きや、複数ファイルの管理から解放され、開発者はロジックに集中できます。

### コア機能

1. **動的テーマエンジン (Dynamic Theme Engine)**  
   グローバルな変数（`$primary`, `$font-size-base` など）をリアルタイムで上書き可能。ブランドカラーやタイポグラフィを、プロジェクト全体に即座に反映。さらに、ダークモードやアクセシビリティ対応（WCAG 2.1 AA準拠）を自動生成。

2. **SEOメタデータインジェクター**  
   各ページの`<head>`タグに、ターゲット言語（ja）に最適化されたメタディスクリプション、OGP、構造化データ（JSON-LD）を自動挿入。Googleのクローラーが認識しやすい形で、ブートストラップコンポーネントのSEOスコアを向上。

3. **ブランドアセットマネージャー (BAM)**  
   ロゴ、フォント、アイコンセット（Bootstrap Icons / Font Awesome）を一元管理。CDN経由で配信し、バージョン管理とキャッシュ戦略を自動化。変更履歴はGitログに統合可能。

### 使い方

1. **テーマのカスタマイズと即時反映**  
   ```bash
   boothell init --theme dark --primary #1a73e8 --font-family "Noto Sans JP"
   ```
   上記コマンドで、プロジェクトに`_variables.scss`を生成。`boothell watch`を実行すると、SCSSの変更を検知し、ブラウザにホットリロード。さらに、`boothell seo --lang ja`で、日本語向けのメタタグを自動生成。

2. **ブランドアセットのインポートとCDN最適化**  
   ```bash
   boothell asset add --logo ./logo.svg --icon-set bootstrap@5.3
   ```
   アセットを`/assets`ディレクトリに自動コピーし、CDN（jsDelivr）のURLを生成。`boothell build --minify`で、アセットを圧縮し、`<link>`タグと`<script>`タグを自動挿入。

3. **SEOメタデータのバッチ更新**  
   ```json
   // boothell.config.json
   {
     "seo": {
       "title": "boothell - ブートストラップ最適化ツール",
       "description": "プロフェッショナル向けのブートストラップデザイン管理とSEO最適化を一元化。",
       "og": {
         "image": "https://example.com/og-image.jpg",
         "locale": "ja_JP"
       }
     }
   }
   ```
   設定ファイルを編集後、`boothell seo --apply`で全ページのメタデータを更新。`boothell seo --validate`で、Googleの構造化データテストツールと同等のバリデーションを実行。

### ベストプラクティス

- **コンポーネントの分離とスコープ管理**  
  グローバルなCSSクラス（`.btn`, `.card`）を直接上書きせず、`@extend`や`@mixin`を活用。boothellのテーマエンジンを使って、コンポーネント単位でスコープを設定（例：`.my-component { @include button-variant($custom-color); }`）。これにより、サードパーティプラグインとの競合を防止。

- **パフォーマンスとSEOのトレードオフ最適化**  
  ブートストラップの全コンポーネントを読み込まず、`boothell tree-shake`で未使用のCSSを削除。さらに、`boothell critical --viewport 375x812`で、モバイルファーストのクリティカルCSSを生成。Lighthouseのパフォーマンススコアを90以上に維持しつつ、SEOメタデータの自動更新を週次でスケジュール。
