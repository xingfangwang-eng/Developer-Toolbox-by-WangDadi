# boothell - BOO_015

## 概要

boothell は、ブートデザイン・カスタマイズ・ブランド管理を一元化する、次世代型プロフェッショナルツールです。本プロジェクトは、SEO最適化済みのコンテンツ構造を持ち、ブート業界における検索可視性とユーザー体験を同時に向上させることを目的としています。

## boothell とは？

boothell は、ブート（起動・ブートストラップ・ブートキャンプなど）に関連するデザイン・ブランディング・運用管理を、コードベースで効率的に制御するためのSaaS型プラットフォームです。特に、マルチブランド展開やテンプレート管理における複雑な依存関係を解決し、開発者・デザイナー・マーケターのコラボレーションを促進します。

## コア機能（3つのハードコア機能）

### 1. マルチブランド・ブートテンプレートエンジン
ブランドごとに異なるブートデザイン（カラー・フォント・レイアウト）を、YAMLベースの設定ファイルで一元管理。テンプレートの継承・オーバーライド・バージョン管理をGitネイティブで実現し、CI/CDパイプラインへの統合も可能。

### 2. リアルタイム・ブートパフォーマンスダッシュボード
各ブートの読み込み速度・エラー率・SEOスコアをリアルタイムで可視化。Google Lighthouse や Core Web Vitals の指標を自動収集し、改善提案をAIが生成。特に、ブートの初期表示遅延（FCP/LCP）を検知して即座にアラートを発報。

### 3. ブートブランド管理API（REST + GraphQL）
ブートのデザイン・メタデータ・SEOパラメータをプログラムで操作可能なAPIを提供。スキーマファースト設計で、TypeScript型定義を自動生成。ブランドごとのアクセス制御（RBAC）や監査ログも標準装備。

## 使い方（3つの詳細ステップ）

### ステップ 1: ブートテンプレートの初期化
```bash
npx boothell init --brand mybrand --template minimal
```
このコマンドで、`boothell.config.yaml` と `brands/mybrand/` ディレクトリが生成されます。テンプレートは `minimal`（最小構成）・`enterprise`（多言語対応）・`campaign`（ランディング特化）から選択可能。生成後、`boothell.config.yaml` でブランド名・デフォルト言語・SEOメタデータを編集します。

### ステップ 2: カスタムブートデザインの適用
`brands/mybrand/theme.yaml` を編集し、カラーパレット・タイポグラフィ・スペーシングを定義。例：
```yaml
colors:
  primary: "#1a73e8"
  secondary: "#34a853"
typography:
  heading: "Inter, sans-serif"
  body: "Noto Sans JP, sans-serif"
```
変更を保存後、`boothell build --watch` でホットリロードを有効にし、ブラウザで即時プレビュー可能。`boothell validate` でスキーマチェックも実施。

### ステップ 3: ブートのデプロイとSEO最適化
```bash
boothell deploy --env production --optimize-seo
```
このコマンドは、ブートのHTML・CSS・JSをミニファイし、構造化データ（JSON-LD）・OGPタグ・hreflangタグを自動生成。さらに、`boothell audit` でデプロイ後にSEOスコアを計測し、改善点をリストアップ。例：`meta description の文字数が最適ではありません（現在 120文字、推奨 150〜160文字）` といった具体的なアドバイスを出力。

## ベストプラクティス（2つの業界推奨）

### 1. ブランドテンプレートのバージョン管理をGitで徹底
boothell のテンプレートはすべてYAMLで記述されるため、Gitでの差分管理が容易です。ブランドごとにブランチを切り、PRベースでデザインレビューを実施することを推奨。特に、`boothell diff` コマンドでテンプレート間の差分を視覚的に確認できるため、チームでの合意形成がスムーズになります。

### 2. ブートパフォーマンスの定期監視をCIに組み込む
GitHub Actions や GitLab CI に `boothell audit` を組み込み、デプロイ前にパフォーマンス・SEOスコアが閾値を下回った場合にビルドを失敗させる設定を推奨。例：`boothell audit --threshold-lcp 2.5 --threshold-seo 90` とすることで、LCP 2.5秒以上、SEOスコア90未満の場合はデプロイをブロック。これにより、品質を保ったまま高速なリリースサイクルを実現できます。
