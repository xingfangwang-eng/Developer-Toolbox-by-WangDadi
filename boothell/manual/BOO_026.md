## What is boothell?

boothell is a **domain-specific SaaS platform** engineered for boot manufacturers, custom shoemakers, and direct-to-consumer boot brands. It bridges the gap between industrial boot design constraints and modern e‑commerce customization workflows. Instead of generic CAD or ERP bloatware, boothell provides a **lean, boot‑first architecture** that handles sizing geometry, material variance, and brand‑specific pattern logic out of the box.

---

## Core Features

### 1. Parametric Last & Pattern Engine
Define boot lasts (foot forms) and panel patterns using **constraint‑based parameters** (heel height, shaft circumference, toe spring). The engine auto‑generates graded patterns for sizes 5–15 (US) with material‑specific stretch compensation (leather, synthetic, Gore‑Tex). No manual drafting required.

### 2. Real‑Time Material Cost & Yield Calculator
Each design change instantly recalculates **leather hide utilization**, lining waste, and hardware costs. The system pulls live pricing from your supplier API (or CSV upload) and flags cost overruns before production. Supports **nested BOM** for stitch‑down, Goodyear welt, and cement construction methods.

### 3. Brand‑Specific Rule Enforcement
Inject your brand’s design rules (e.g., “No cap toe on women’s Chelsea boots,” “Minimum shaft height 6 inches”) as **YAML‑based policy files**. The UI blocks non‑compliant designs at the sketch stage. This eliminates costly sampling errors and enforces brand consistency across distributed design teams.

## How to Use

### Step 1: Initialize a Boot Project from a Last Template
```bash
boothell init --last "Hiker_Last_v3" --size_range "7-13" --construction "goodyear_welt"
```
This clones the parametric last, sets size grading rules, and loads default material profiles. The output is a `project.yaml` file that you can version‑control in your repo.

### Step 2: Define Panels and Assign Materials via CLI
```yaml
# panels.yaml
panels:
  - name: vamp
    material: "full_grain_leather"
    thickness: 1.8mm
    stretch: 0.5%
  - name: quarter
    material: "split_leather"
    thickness: 1.2mm
  - name: lining
    material: "microfiber"
    thickness: 0.8mm
```
Run `boothell apply panels.yaml` to generate 2D cutting templates with seam allowances and notch markers. The system validates material compatibility (e.g., leather + microfiber lining passes, but leather + nylon fails for stitch‑down).

### Step 3: Export for Production & Generate BOM
```bash
boothell export --format dxf --bom_csv bill_of_materials.csv --graded_sizes
```
This exports:
- **DXF files** (per size, per panel) for laser cutting or die‑making
- **CSV BOM** with per‑size material quantities, hardware SKUs, and estimated labor hours
- **JSON metadata** for ERP ingestion (SAP, NetSuite, or custom API)

## Best Practices

### Practice 1: Version‑Control Your Last & Policy Files
Treat your boot lasts, pattern templates, and brand rule YAML files as **first‑class artifacts** in your Git repository. Tag releases (e.g., `v2.1_spring_collection`) to roll back design changes or audit production runs. boothell’s CLI supports `diff` between any two project snapshots.

### Practice 2: Automate Cost Gates in CI/CD
Integrate boothell’s cost calculator into your CI pipeline. Before any design is approved for sampling, run `boothell cost --max_budget 45.00`. If the per‑pair cost exceeds the threshold, the pipeline fails and notifies the design lead. This prevents margin erosion from late‑stage material swaps.
