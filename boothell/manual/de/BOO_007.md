# boothell - BOO_007

## Überblick

boothell ist ein spezialisiertes CLI-gestütztes Tool für die **Markenverwaltung und Individualisierung von Stiefeln im B2B-Kontext**. Es richtet sich an Produktmanager und Designer, die eine konsistente Markenidentität über verschiedene Stiefel-Modelle hinweg sicherstellen müssen – von der Materialauswahl bis zur finalen Preisgestaltung. Die Lösung ist auf Performance und Skalierbarkeit ausgelegt, mit nativer Unterstützung für **Multi-Tenant-Konfigurationen** und **ISO-konforme Metadaten**.

## 3 Hardcore-Funktionen (die es von anderen Tools abheben)

1.  **Dynamische Material-Kaskadenlogik**  
    boothell erlaubt die Definition von **regelbasierten Materialketten** (z. B. Leder → Futter → Sohle → Beschläge), die automatisch auf Kompatibilität geprüft werden. Bei Änderungen an einem Bauteil werden alle abhängigen Preis- und Verfügbarkeitsdaten in Echtzeit neu berechnet.

2.  **Versioniertes Marken-Asset-Management**  
    Jede Stiefel-Konfiguration wird als **Git-kompatibler Snapshot** gespeichert. So können Markenmanager historische Designs wiederherstellen, A/B-Tests von Logovarianten durchführen oder Änderungen mit einem `diff`-Befehl nachvollziehen.

3.  **SEO-optimierter Export-Generator**  
    Das Tool generiert automatisch **strukturierte Daten (JSON-LD)** und **lokalisierte Metadaten** für jedes Stiefel-Modell. Das bedeutet: Deine Produktseiten sind von Haus aus für Suchmaschinen optimiert – ohne zusätzliches Plugin.

## 3 detaillierte Anwendungsschritte (für Profis)

### 1. Initiales Setup und Tenant-Konfiguration

```bash
boothell init --tenant "BrandA" --locale de-DE --currency EUR
```

Dieser Befehl legt einen neuen Mandanten an. Du wirst aufgefordert, ein **Marken-Handbuch (JSON)** zu importieren, das Farben, Schriftarten und Logos enthält. boothell validiert die Struktur sofort gegen ein vorgegebenes Schema.

### 2. Erstellen einer variantenreichen Stiefel-Konfiguration

```bash
boothell create --model "Lederstiefel-2025" \
  --material "Rindsleder_Schwarz" \
  --sole "Gummi_Profi" \
  --lining "Baumwolle_Atmungsaktiv" \
  --brand-logo "BrandA_Logo_2024.svg" \
  --price 249.99
```

Das Tool prüft, ob alle Materialien aus derselben **Kaskadengruppe** stammen. Falls nicht, wird eine Fehlermeldung mit Vorschlägen zur Korrektur ausgegeben. Nach erfolgreicher Prüfung wird die Konfiguration als `.boothell/model/`-Datei gespeichert.

### 3. SEO-Export und Deployment vorbereiten

```bash
boothell export --model "Lederstiefel-2025" --format html --seo
```

Dieser Befehl generiert eine **statische HTML-Datei** inklusive:
- Open-Graph-Tags
- Produktschema (Product, Offer, Brand)
- Mehrsprachige Alternativlinks (für `de`, `en`, `fr`)

Die Datei kann direkt in ein CMS oder auf einen S3-Bucket hochgeladen werden.

## 2 Best Practices (aus der Praxis deutscher Markenmanager)

1.  **Führe regelmäßige Marken-Audits mit `boothell audit` durch**  
    Das Audit prüft, ob alle aktiven Modelle noch mit dem aktuellen Markenhandbuch übereinstimmen. Besonders nützlich nach einem Rebranding. Der Befehl gibt eine **konkrete Liste von Abweichungen** aus – inklusive Zeilennummer der betroffenen Konfigurationsdatei.

2.  **Nutze die `--dry-run`-Flag bei Preisänderungen**  
    Bevor du Preise global anpasst (z. B. wegen Rohstoffpreissteigerung), simuliere die Auswirkungen mit `boothell update --price +5% --dry-run`. Das Tool zeigt dir, welche Modelle betroffen sind und wie sich die Marge verändert – ohne tatsächliche Änderungen vorzunehmen.
