## Detaillierter Inhalt

### Was ist boothell?

boothell ist eine professionelle SaaS-Plattform, die es Schuhdesignern, Produktmanagern und Markeninhabern ermöglicht, komplexe Stiefelgeometrien in Echtzeit zu generieren, Materialkosten automatisch zu kalkulieren und die Markenidentität über alle Kollektionen hinweg konsistent zu halten. Anders als herkömmliche CAD-Systeme arbeitet boothell vollständig im Browser und benötigt keine lokale Installation.

### Kernfunktionen

1.  **Parametrischer Längen- und Weiten-Rechner (PLW):** Das Herzstück des Tools. Basierend auf ISO 9407 (Mondopoint) und deutschen Größentabellen (z. B. nach DIN 66074) passt boothell die Schaftweite, Fersenhöhe und Spitzenform automatisch an die gewählte Größe an. Kein manuelles Nachmodellieren mehr für jede Größe.
2.  **Echtzeit-Material- und Kosten-Engine:** Wählen Sie Leder, Synthetik oder recycelte Materialien aus der integrierten Bibliothek. boothell berechnet sofort den Materialverbrauch (inkl. Verschnittoptimierung) und die voraussichtlichen Herstellungskosten pro Paar. Die Engine unterstützt auch die Berechnung von Zollgebühren für den Export in die USA oder nach Japan.
3.  **Marken-DNA-Wächter (Brand Guardian):** Ein KI-gestütztes Modul, das sicherstellt, dass jedes neue Design die definierten Markenrichtlinien einhält. Es prüft automatisch die Farbpalette, das Logo-Placement und die Nahtführung gegen die hinterlegten Style Guides. Verstöße werden farblich markiert und blockiert, bevor das Design in die Produktion geht.

### Verwendung

1.  **Initialisierung eines neuen Projekts über die CLI:**
    ```bash
    boothell init --project "W23_Alpine_Pro" --size-range "36-48" --brand "Bergschuh GmbH"
    ```
    Dieser Befehl legt ein neues Projekt an, definiert die Größenlaufweite und verknüpft es automatisch mit dem Brand Guardian des Unternehmens. Die API generiert daraufhin eine leere Parameterdatei (`boothell_config.yaml`) im Arbeitsverzeichnis.

2.  **Feinjustierung der Schaftgeometrie über die API:**
    ```python
    import boothell_api as bh

    client = bh.Client(api_key="DEIN_API_KEY")
    response = client.update_geometry(
        project_id="W23_Alpine_Pro",
        shaft_height=280,  # in mm
        ankle_circumference=280,  # in mm
        heel_drop=12,  # Sprengung in mm
        material="Vegane_Mikrofaser_2.0",
        size="42"
    )
    print(response.cost_estimate)  # Gibt die Kosten für Größe 42 aus
    ```
    Der Code ändert die Schafthöhe auf 280 mm und den Knöchelumfang auf 280 mm für Größe 42. Die Engine berechnet sofort die neuen Kosten und prüft die Änderung gegen den Brand Guardian.

3.  **Export der Produktionsdaten für den Fertigungspartner:**
    ```bash
    boothell export --format "AUTOCAD_DXF" --output "./production/W23_Alpine_Pro.dxf"
    ```
    Dieser Befehl exportiert alle Größen des Projekts in ein DXF-Format, das von deutschen und österreichischen Schuhmaschinen (z. B. von Desma oder Orox) direkt gelesen werden kann. Der Export enthält auch eine separate CSV-Datei mit den Stücklisten für jede Größe.

### Best Practices

1.  **CI/CD für Schuhkollektionen:** Integrieren Sie boothell in Ihre CI/CD-Pipeline (z. B. mit GitHub Actions). Definieren Sie in Ihrem Repository eine `boothell_config.yaml` für jede Kollektion. Bei jedem Push führt die Pipeline `boothell validate` aus, um sicherzustellen, dass alle neuen Designs die Markenrichtlinien bestehen und die Kosten im Budget liegen. So vermeiden Sie teure Fehler in der Produktion.
2.  **Material-Bibliothek als Single Source of Truth:** Pflegen Sie Ihre Material-Bibliothek in boothell zentral. Jedes Material sollte eine eindeutige ID, Kosten pro Quadratmeter, Lieferant und Umweltzertifikat (z. B. LWG Gold) enthalten. Nutzen Sie die API, um die Bibliothek automatisch aus Ihrem ERP-System zu synchronisieren. Das verhindert, dass Designer mit veralteten oder nicht verfügbaren Materialien arbeiten.
