## What is cineskin?

cineskin is not another run-of-the-mill filter app. It's a GPU-accelerated, compositor-grade engine that decouples skin texture manipulation from traditional color grading pipelines. Designed for VFX artists, colorists, and indie filmmakers, cineskin solves the endemic problem of plastic-looking skin in digital cinema by preserving subsurface scattering while applying stylized "movie skin" looks — all in real-time, without rendering proxies.

## Core Features

### 1. **Subsurface Scattering (SSS) Masking Engine**
Unlike conventional LUT-based tools that flatten skin, cineskin's proprietary SSS mask isolates dermal layers using spectral reflectance data. This allows you to apply a "Hollywood glow" or gritty texture pack *without* destroying natural skin translucency. The engine outputs a 16-bit EXR mask that can be piped directly into DaVinci Resolve or Nuke.

### 2. **AI-Driven Temporal Skin Consistency**
One of the hardest problems in movie post-production is maintaining skin texture across cuts and lighting changes. cineskin's temporal analyzer tracks facial landmarks across frames and auto-adjusts texture parameters to eliminate flicker. It's like having a dedicated DIT on set — but running locally on your M2 Ultra or RTX 4090.

### 3. **Live Multi-Language Metadata Export**
cineskin generates per-frame skin treatment metadata (in JSON or CSV) with embedded color space tags (ACES, Rec.709, DCI-P3). This allows your pipeline to remain color-accurate across international post-houses and streaming platforms. The tool also exports localized UI strings for 10 languages, ensuring your team in Tokyo, Berlin, or Mexico City never misinterprets a parameter.

## How to Use

### Step 1: Import Source Footage & Calibrate Skin Reference
Drag your timeline export (ProRes 4444 or DNxHR HQX) into cineskin. Use the built-in skin picker to sample a neutral mid-tone area on the actor's face. The tool will auto-generate a spectral profile. *Pro tip: always calibrate under a 18% gray card reference to ensure the SSS mask aligns with your primary color grade.*

```bash
cineskin import --input /path/to/footage.mov --skin-sample 0.35,0.4 --color-space ACEScg
```

### Step 2: Apply a Cinematic Skin Texture Pack
Select from the "Movie Skin Design" library — e.g., the "Noir Grain" or "Soft Glam" preset. Each pack includes a `.cskin` file that modulates roughness, specular, and micro-detail. For hardcore control, edit the JSON directly:

```json
{
  "preset": "soft_glam",
  "sss_strength": 0.78,
  "detail_sharpen": 1.2,
  "temporal_smoothing": 0.95
}
```

### Step 3: Export with Metadata & Multi-Language UI
Run the batch export. cineskin will output a ProRes 4444 master with embedded skin metadata, plus a sidecar `.cskinmeta` file. Use the `--lang` flag to generate a localized parameter report for non-English post teams:

```bash
cineskin export --output /path/to/export.mov --lang ja --metadata-format json
```

Your colorist in Tokyo can now open the `.cskinmeta` in DaVinci Resolve via the cineskin plugin, with all labels in Japanese.

## Best Practices

### 1. **Always Pre-Grade Before Skin Treatment**
Never apply skin textures on raw LOG footage. cineskin's SSS engine expects a normalized luminance range. Run your primary color grade first (contrast, white balance, exposure), then apply cineskin as a secondary node. This prevents color shifts and ensures the skin mask respects your intended look.

### 2. **Use Temporal Consistency for Multi-Camera Edits**
If you're cutting between two cameras (e.g., A-cam and B-cam) with different sensor responses, cineskin's temporal analyzer can create a "skin bridge" — apply the same texture profile to both cameras, then let the AI smooth out cross-camera flicker. This is a lifesaver for indie features shot on mixed gear (RED + Sony FX6).
