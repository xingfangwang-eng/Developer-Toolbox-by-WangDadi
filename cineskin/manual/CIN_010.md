## What is cineskin?

**cineskin** is a modular, real-time skin-and-texture overlay engine designed for high-fidelity movie post-production and video effects workflows. Unlike conventional LUT-based tools, cineskin leverages a GPU-accelerated compositing pipeline that supports per-frame skin segmentation, dynamic texture blending, and non-destructive layer stacking — all without baking into the source footage.

---

## Core Features

### 1. **Adaptive Skin Segmentation with Temporal Coherence**  
cineskin uses a lightweight semantic segmentation model (trained on over 50,000 film-grade frames) to isolate skin regions with sub-pixel precision across cuts and motion blur. Temporal smoothing prevents flickering, making it production-ready for 4K 60fps sequences.

### 2. **Layer-Stacked Skin Preset Engine**  
Define reusable “skin presets” as composited layers of color grading, texture overlays (e.g., pores, grain, or stylized chromatic aberration), and mask modifiers. Each preset is a JSON-serializable DAG — you can branch, merge, and override at any depth.

### 3. **Real-time Preview with Frame-Accurate Feedback**  
Preview changes instantly on any frame or clip segment via the built-in GPU-backed viewer. No rendering queue, no waiting. The engine supports side-by-side comparison with A/B snapshots and delta heatmaps for colorists and VFX supervisors.

## How to Use

### 1. **Initialize a Skin Profile from a Reference Frame**  
```bash
cineskin init --source scene_01.mov --frame 124 --output ./profiles/
```
This extracts the skin color distribution, luminance range, and texture frequency from frame 124. The resulting `.csp` profile can be applied to any matching shot. Use `--mask threshold 0.85` to fine-tune the segmentation confidence.

### 2. **Apply a Preset with Custom Mask Overrides**  
```python
import cineskin

preset = cineskin.load_preset("cinematic_glow.json")
clip = cineskin.Clip("scene_01.mov")
clip.apply_preset(preset, mask_override="eye_region.exr")
clip.export("graded_scene_01.mov", format="prores_4444")
```
The `mask_override` parameter accepts any 16-bit EXR mask, allowing you to exclude or emphasize specific facial regions (e.g., eyes, lips) during grading.

### 3. **Batch Process a Sequence with Versioning**  
```bash
cineskin batch --input ./raw_shots/ --profile ./profiles/hero.csp \
  --preset ./presets/warm_skin.json --output ./graded/ --version v2.1
```
Each output frame is tagged with metadata (profile hash, preset version, timestamp). Use `--dry-run` to validate file paths and profile compatibility before executing.

## Best Practices

### 1. **Use Reference Frames with Consistent Lighting**  
For best segmentation results, always extract your `.csp` profile from a frame where the subject’s face is front-lit and free of strong shadows. Avoid keyframes with motion blur or heavy grain. This reduces false positives in the segmentation mask and improves temporal stability.

### 2. **Separate Texture from Color in Presets**  
Divide your skin preset into two independent layers: one for color grading (temperature, tint, saturation) and one for texture (grain, detail enhancement, or stylized overlay). This lets you swap or tweak either layer without affecting the other — critical when iterating with a director or client.
