## What is cineskin?

**cineskin** is a high-performance, plugin-agnostic skinning engine designed specifically for cinematic video effects pipelines.  
It abstracts away the complexity of frame-level texture mapping and temporal coherence, allowing developers and VFX artists to apply, blend, and animate **custom film-grade visual skins** directly on video streams — without touching a single line of shader boilerplate.

Unlike generic overlay tools, cineskin treats each video frame as a layered skin canvas, enabling real-time compositing of color grades, lens distortions, grain overlays, and procedural material effects. It’s built for **modern post-production workflows** where speed, reproducibility, and non-destructive editing are non-negotiable.

## Core Features

### 1. Real-time Temporal Skin Warping Engine  
cineskin’s core is a **frame-aware skin warper** that automatically tracks motion vectors across frames. This means every applied skin — whether it’s a film grain texture, a vintage lens bloom, or a stylized color LUT — follows object movement with sub-pixel precision. No more manual keyframing or motion tracking in external compositors.

### 2. Multi-layer Skin Compositor with Non-destructive Blend Modes  
Stack up to 16 independent skin layers per clip, each with its own blend mode, opacity curve, and mask. The compositor supports **industry-standard blending** (screen, multiply, overlay, soft light, etc.) plus custom cinematic modes like *bleach bypass* and *teal-orange push*. All layers remain editable post-export.

### 3. GPU-accelerated Skin Preview with Real-time LUT Baking  
Preview your skin stack in full resolution at 60fps, even on 4K footage. The engine bakes LUTs and skin parameters into a single GPU-friendly pass, reducing export time by up to 40% compared to traditional node-based compositors. Perfect for iterative color grading and look development.

## How to Use

### Step 1: Initialize a Skin Pipeline in Your Project  
```bash
pip install cineskin
cineskin init --project my_film --resolution 3840x2160 --fps 24
```
This creates a `skin_pipeline.yaml` file and a `skins/` directory. The YAML defines your frame buffer settings, skin layer stack, and output codec. No manual config — just a clean, version-controllable pipeline scaffold.

### Step 2: Apply a Custom Skin to a Video Stream  
```python
from cineskin import SkinPipeline, FilmGrain, ColorGrading

pipeline = SkinPipeline.from_config("skin_pipeline.yaml")
pipeline.add_layer(FilmGrain(intensity=0.15, grain_size=2.0, seed=42))
pipeline.add_layer(ColorGrading(lut_path="./looks/teal_orange.cube", strength=0.8))

pipeline.render("input.mp4", "output_skinned.mp4", preview=True)
```
You can also chain skins with `.blend(mode="overlay")` and apply temporal masks using `cineskin.MotionMask`. The preview flag opens a real-time OpenGL window to inspect your skin before final export.

### Step 3: Export with Optimized Skin Baking  
```bash
cineskin bake --input output_skinned.mp4 --output final.mp4 --codec h264_nvenc --preset p4
```
The `bake` command flattens all skin layers into a single GPU-optimized pass, preserving metadata for later re-edits. Use `--dry-run` to estimate export time and memory usage. For batch processing, pipe multiple clips via `cineskin batch --glob "*.mp4"`.

## Best Practices

### 1. Use Motion Masks for Dynamic Skin Intensity  
Instead of applying a uniform grain or color grade across an entire clip, define **motion-based masks** using `cineskin.MotionMask(threshold=0.3)`. This automatically reduces skin intensity in high-motion areas (e.g., fast pans or action scenes) to avoid artifacts. It’s a pro-level trick for maintaining visual consistency without manual rotoscoping.

### 2. Version-Control Your Skin Pipeline Configs  
Treat `skin_pipeline.yaml` as part of your project’s source code. Commit it alongside your footage and LUTs. This ensures full reproducibility — a must for collaborative post-production or when revisiting a project months later. Use `cineskin diff` to compare pipeline versions between takes.
