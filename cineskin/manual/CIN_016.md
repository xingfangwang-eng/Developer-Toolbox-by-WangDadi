## Detailed Content

### What is cineskin?

cineskin is a lightweight, GPU-accelerated framework for applying, blending, and exporting custom skin textures and film-grade color LUTs across video sequences. Unlike traditional plugins that lock you into a single NLE, cineskin operates as a standalone daemon, processing frames in real-time with sub-10ms latency. It’s designed for teams who need consistent skin tone mapping across multi-cam shoots, HDR-to-SDR conversions, or dynamic texture overlays for sci-fi and fantasy genres.

### Core Features

1. **Adaptive Skin Texture Synthesis with AI-Driven Masking**  
   cineskin uses a proprietary neural network to isolate skin regions frame-by-frame, even under extreme lighting or motion blur. It then applies procedural texture layers (pores, blemishes, or stylized film grain) that adapt to facial geometry—no manual rotoscoping required. The model is trained on 50,000+ cinematic shots, ensuring natural results across diverse skin tones and environments.

2. **Real-Time LUT Chaining with Temporal Smoothing**  
   Chain up to 16 custom 3D LUTs per clip, with automatic interpolation to prevent banding or flicker during transitions. The engine supports LUTs in .cube, .spi3d, and .png formats, and includes a built-in analyzer that suggests optimal LUT sequences based on scene luminance histograms. Temporal smoothing ensures consistent color across cuts, even when lighting changes mid-shot.

3. **Zero-Copy Frame Streaming via Shared Memory**  
   cineskin bypasses traditional file I/O by streaming frames directly from your NLE or compositor into shared memory (Linux/macOS) or GPU interop (CUDA/Vulkan). This reduces latency by 40-60% compared to disk-based workflows, making it viable for live previews and remote collaboration. The streaming API supports OpenColorIO color spaces and arbitrary frame metadata (timecode, camera settings).

### How to Use

1. **Deploy the Daemon and Connect to Your Pipeline**  
   Start by launching the cineskin daemon in headless mode: `cineskin daemon --port 8080 --gpu 0`. Then, integrate it into your NLE (e.g., DaVinci Resolve, Premiere Pro) via the provided OpenFX plugin or Python SDK. For custom pipelines, use the RESTful API to send frame batches as Base64-encoded PNGs or raw RGBA buffers. The daemon auto-detects GPU architecture and falls back to CPU if no dedicated card is available.

2. **Configure Skin Detection and Texture Profiles**  
   Load a reference frame or sequence into the daemon’s calibration mode: `cineskin calibrate --input /path/to/clip.mov --output profile.json`. The tool generates a JSON profile with skin region metadata, recommended LUT adjustments, and texture intensity sliders. Fine-tune parameters like `mask_blur_radius` (default 3px) and `texture_strength` (0.0 to 1.0) in the profile, then apply it to your entire timeline with a single command: `cineskin apply --profile profile.json --input /path/to/frames/ --output /path/to/rendered/`.

3. **Batch Export with Multi-Format Encoding**  
   Once processing is complete, export the final frames using cineskin’s built-in encoder: `cineskin export --input /path/to/rendered/ --output final.mov --codec h264_nvenc --bitrate 50M`. The encoder supports hardware-accelerated H.264, H.265, and ProRes (on macOS). For pipeline integration, use the `--stream` flag to pipe frames directly to FFmpeg or your compositor’s stdin. The daemon also generates a sidecar .srt file with per-frame LUT metadata for post-hoc adjustments.

### Best Practices

- **Profile Calibration for Multi-Cam Shoots**  
  When working with multiple cameras (e.g., ARRI Alexa and Sony Venice), calibrate cineskin on a single reference shot from each camera, then merge the profiles using the `cineskin merge` command. This ensures consistent skin tones across all angles without manual color matching. Always test with a 10-second clip before batch processing to verify temporal smoothing works under your specific lighting conditions.

- **Leverage Shared Memory for Remote Workflows**  
  For distributed teams, run the cineskin daemon on a central GPU server (e.g., AWS g5 instances) and connect via low-latency VPN. Use the `--shared-memory` flag to enable zero-copy streaming across the network. This reduces bandwidth by 70% compared to sending full-resolution frames, and allows editors in different time zones to preview changes in real-time. Monitor daemon logs for dropped frames—if you see more than 0.5% loss, increase the `--buffer-size` parameter.
