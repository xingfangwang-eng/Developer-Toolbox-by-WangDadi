## Was ist cineskin?

cineskin ist ein **professionelles Tool zur automatisierten Erstellung und Optimierung von Movie-Skins** – visuelle Overlays, die in der Post-Production dynamisch auf Videomaterial angewendet werden. Es löst das Problem der manuellen, zeitaufwändigen Anpassung von Film-Looks, Farbkorrekturen und Effekt-Layern, indem es einen **komponentenbasierten, deklarativen Skin-Ansatz** bietet, der direkt in bestehende Workflows integrierbar ist.

---

## 3 Hardcore-Features (Tool-spezifisch)

1. **Deklarative Skin-DSL**  
   Definiere Movie-Skins mittels einer **eigenen domänenspezifischen Sprache (DSL)**, die auf YAML/JSON basiert. Jeder Skin besteht aus **parametrisierten Effekt-Knoten** (z. B. Color Grading, LUT-Overlay, Rauschfilter). Die DSL erlaubt **Bedingungslogik** (z. B. „wenn Szene dunkel, dann wende High-Contrast-Skin an“) und **Versionierung** via Git.

2. **Echtzeit-Preview-Engine mit GPU-Beschleunigung**  
   Die integrierte **WebGL-basierte Vorschau-Engine** rendert Skins in Echtzeit auf dem Quellvideo. Unterstützt **HDR-Farbraum (Rec. 2020, PQ)** und **Multi-Layer-Blending** (Additiv, Multiplikativ, Screen). Kein manuelles Exportieren – Änderungen am Skin sind sofort sichtbar.

3. **Automatische Skin-Generierung via KI-Analyse**  
   cineskin analysiert **Szenenwechsel, Belichtung, Farbhistogramm und Bewegungsvektoren** aus dem Quellmaterial. Basierend auf diesen Metadaten generiert es **adaptive Skins**, die sich an den visuellen Kontext anpassen – z. B. ein „Noir-Skin“ für dunkle Szenen, ein „Pastell-Skin“ für helle.

## 3 Detaillierte Anwendungsschritte (Professionell)

### 1. Skin-Definition in der DSL erstellen

Erstelle eine Datei `my_skin.yaml` im cineskin-Projektverzeichnis. Definiere einen **zweilagigen Skin** mit einer LUT und einem Farbkorrektur-Knoten:

```yaml
skin:
  name: "Cinematic_Retro"
  layers:
    - type: "lut"
      src: "luts/retro_1970.cube"
      blend_mode: "multiply"
      opacity: 0.8
    - type: "color_grade"
      saturation: 0.6
      contrast: 1.2
      temperature: 4500
```

Die DSL unterstützt **Variablen** (`$scene_type`) und **Bedingungen**:

```yaml
conditions:
  - if: "$scene_type == 'night'"
    then:
      - type: "noise_reduction"
        strength: 0.7
```

### 2. Skin auf Quellvideo anwenden und in Echtzeit optimieren

Führe den Befehl aus, um das Video mit dem Skin zu rendern:

```bash
cineskin apply --input input.mp4 --skin my_skin.yaml --output preview.mp4
```

Öffne die **Web-UI** (Port 8080) für die Echtzeit-Vorschau. Nutze die **Tastaturkürzel**:
- `Strg + Shift + P`: Skin-Parameter als Overlay einblenden
- `Strg + Shift + L`: LUT-Intensität in 10%-Schritten anpassen
- `Strg + Shift + R`: Skin zurücksetzen

### 3. Skin exportieren und in Post-Production-Workflow integrieren

Exportiere den optimierten Skin als **standalone JSON-Datei** für die Integration in DaVinci Resolve, Premiere Pro oder After Effects:

```bash
cineskin export --skin my_skin.yaml --format resolve
```

cineskin generiert eine **DaVinci Resolve PowerGrade-Datei** (`.drx`) oder eine **After Effects Preset-Datei** (`.ffx`). Die Export-API erlaubt **Batch-Export** für mehrere Skins:

```bash
cineskin export --glob "skins/*.yaml" --format all --output ./exports/
```

## 2 Best Practices für die Branche

### 1. **Skin-Versionierung in Git mit integrierten Tests**

Verwende cineskin’s **`cineskin test`** Befehl, um Skins automatisch auf **Kompatibilität, Syntaxfehler und Performance** zu prüfen. Führe vor jedem Commit aus:

```bash
cineskin test --glob "skins/*.yaml" --max-layers 5 --max-resolution 4K
```

So stellst du sicher, dass jeder Skin **fehlerfrei** und **performant** ist, bevor er in die Produktion geht. Kombiniere dies mit **Git Hooks** (pre-commit), um Broken Skins zu verhindern.

### 2. **Adaptive Skins mit Szenenanalyse automatisieren**

Nutze die **KI-Analyse-Funktion** von cineskin, um aus einem Rohvideo automatisch mehrere Skins zu generieren:

```bash
cineskin analyze --input raw_footage.mp4 --output ./adaptive_skins/ --style "film_noir, vintage, modern"
```

Die generierten Skins werden **automatisch mit Metadaten (Zeitstempel, Szenenindex)** verknüpft. Importiere sie direkt in deinen NLE-Workflow (z. B. Premiere Pro) über die **cineskin Bridge** – ein Plugin, das Skins als **dynamische Ebenen** einbindet.
