# cineskin - CIN_014

## Überblick

cineskin ist kein weiterer generischer Videoeffekt-Baukasten, sondern eine hochperformante Pipeline für kinematografische Skin-Texturierung und Echtzeit-Postproduktion. Optimiert für professionelle Compositing-Workflows unter Linux und macOS. Keine Bloatware, keine unnötigen Abstraktionen – nur präzise Kontrolle über Materialeigenschaften, Beleuchtungsmodelle und GPU-beschleunigte Berechnungen.

## Detaillierter Inhalt

### Was ist cineskin?

cineskin ist eine modulare, CLI- und API-first Engine zur algorithmischen Generierung und Anpassung von Film-Oberflächenstrukturen („Skins“) – von organischen Hauttexturen über synthetische Metalloxide bis hin zu prozeduralen Verschleißmustern. Das Tool ist darauf ausgelegt, klassische Compositing-Schritte wie Color Grading, Texture Baking und Displacement Mapping in einem einzigen, deterministischen Workflow zu vereinen. Es richtet sich an VFX-Künstler, Pipeline-TDs und Render-Engine-Entwickler, die keine Abhängigkeit von proprietären GUI-Tools eingehen möchten.

### Kernfunktionen (Hardcore)

1.  **Prozeduraler Skin-Compiler mit Multi-Material-Stacking** – cineskin kompiliert Skin-Definitionen aus einer deklarativen DSL (YAML/JSON) und erzeugt daraus GPU-optimierte Shader-Kernel. Jeder Layer (Diffuse, Specular, Normal, Displacement, Roughness, Subsurface) kann mit mathematischen Operatoren (Blend, Subtract, Overlay, Soft Light) kombiniert werden, ohne dass ein einziger Frame in Echtzeit gerendert werden muss. Das Ergebnis ist ein deterministischer Material-Output, der sich für Batch-Rendering und A/B-Testing eignet.

2.  **Dynamisches Luminanz-Mapping mit KI-gestützter Kantenerkennung** – Die Engine analysiert das Quellmaterial frame-basiert auf Luminanz, Chrominanz und Gradienten. Basierend auf diesen Daten erzeugt cineskin eine prozedurale Maske, die HDR-Bereiche, Schattenverläufe und Mikrokonturen automatisch segmentiert. Diese Maske dient als Grundlage für adaptive Skin-Anpassungen, z. B. gezieltes Aufrauen von Glanzlichtern oder lokales Verstärken von Subsurface-Scattering in Übergangszonen.

3.  **Deterministisches Caching mit inkrementellem Rebuild** – Jeder Skin-Processing-Durchlauf erzeugt einen eindeutigen Hash aus Quellmaterial, Skin-Definition und Parameter-Set. Ändert sich nur ein Parameter, wird ausschließlich der betroffene Layer neu berechnet – nicht der gesamte Stack. Das spart bei iterativen Workflows (z. B. Look-Development für 4K+ Sequenzen) massiv Rechenzeit und ermöglicht reproduzierbare Builds ohne Dirty-State-Probleme.

### So verwenden Sie es (Detaillierte Schritte)

**Schritt 1: Skin-Definition als YAML erstellen und validieren**

Erstellen Sie eine Datei `skin_definition.yaml` mit folgendem Schema:

```yaml
skin:
  name: "cinematic_skin_01"
  base_material: "organic_skin"
  layers:
    - type: "diffuse"
      source: "input_footage.exr"
      blend: "multiply"
      intensity: 0.85
    - type: "specular"
      source: "procedural_roughness"
      blend: "overlay"
      intensity: 0.45
    - type: "displacement"
      source: "height_map.exr"
      amplitude: 0.02
  output:
    format: "exr"
    compression: "piz"
    bit_depth: 16
```

Führen Sie dann die Validierung durch:

```bash
cineskin validate --definition skin_definition.yaml --output validation_report.json
```

Das Tool prüft auf fehlende Layer, inkonsistente Blend-Modi oder nicht unterstützte Bit-Tiefen. Bei Fehlern wird eine detaillierte JSON-Ausgabe mit Zeilenangabe erzeugt.

**Schritt 2: Batch-Processing mit inkrementellem Caching starten**

Starten Sie den eigentlichen Render-Durchlauf:

```bash
cineskin render --definition skin_definition.yaml --input ./frames/seq_%04d.exr --output ./rendered/seq_%04d.exr --cache ./cache/skin_cache --threads 8
```

cineskin erzeugt für jede Frame-Nummer einen Cache-Eintrag. Wird ein Frame bereits verarbeitet (gleicher Hash), wird der vorhandene Cache verwendet. Sie können den Cache-Status live überwachen:

```bash
cineskin cache --status --cache ./cache/skin_cache
```

**Schritt 3: Skin-Output in Compositing-Pipeline integrieren**

Verwenden Sie die exportierte Skin-Definition als Input für Ihr Compositing-Tool (z. B. Nuke, Fusion oder Blender Compositor). cineskin gibt zusätzlich eine JSON-Metadaten-Datei aus, die Layer-Reihenfolge, Blend-Modi und Intensitäten enthält. Beispiel für die Integration in Nuke über ein Python-Script:

```python
import nuke
import json

with open("skin_metadata.json") as f:
    meta = json.load(f)

for layer in meta["layers"]:
    read_node = nuke.createNode("Read")
    read_node["file"].setValue(layer["output_path"])
    # Weitere Node-Erstellung basierend auf blend mode
```

### Beste Praktiken (Industry Standards)

1. **Immer mit einem deterministischen Seed arbeiten** – cineskin unterstützt einen `--seed` Parameter für jede prozedurale Generierung (z. B. Rauschmuster, Mikrokonturen). Setzen Sie diesen Wert auf eine feste Zahl (z. B. die Frame-Nummer oder eine Projekt-ID), um reproduzierbare Ergebnisse zu garantieren. Ohne Seed generiert die Engine bei jedem Durchlauf ein anderes Muster, was bei iterativem Look-Development zu Inkonsistenzen führt.

2. **Layer-Priorität durch Weight-Maps steuern** – Statt globale Intensitäten zu verwenden, nutzen Sie Weight-Maps (Graustufen-EXR) für jeden Layer. Eine Weight-Map definiert pixelgenau, wie stark ein Layer an einer bestimmten Stelle wirkt. Beispiel: Eine Weight-Map für den Specular-Layer, die nur in hellen Bereichen aktiv ist, verhindert überschüssige Glanzlichter in Schattenzonen. cineskin akzeptiert Weight-Maps als optionales Feld in der Layer-Definition (`weight_map: "weight_specular.exr"`).
