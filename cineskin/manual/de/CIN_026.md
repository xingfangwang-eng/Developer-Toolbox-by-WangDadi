## Was ist cineskin?

cineskin ist ein **kommandozeilenbasiertes Framework** zur automatisierten Generierung und Anwendung von Film-Skins auf Rohmaterial. Es ersetzt manuelle Color-Grading-Workflows in der Postproduktion durch eine **konfigurationsgesteuerte Pipeline**, die direkt in CI/CD-Umgebungen oder lokal ausgeführt wird.

## Hardcore-Features

### 1. **Multi-Layer Skin Compositing Engine**
Erstelle und kombiniere **bis zu 12 unabhängige Skin-Layer** (LUT, Mask, Overlay, Grain) pro Frame. Jeder Layer unterstützt eigene Blending-Modi (Multiply, Screen, Overlay) und Alpha-Kanäle. Ideal für komplexe Looks ohne manuelles Compositing in DaVinci Resolve.

### 2. **Real-Time Previsualization via WebSocket**
Starte einen lokalen WebSocket-Server (`cineskin preview --port 8080`), der **Live-Vorschau** auf jedem Frame-Update streamt. Änderungen an Skin-Konfigurationen werden sofort im Browser gerendert – ohne Neuladen der Quelle.

### 3. **Batch-Export mit GPU-Beschleunigung (CUDA/Metal)**
Exportiere 4K-Footage in **<2s pro Frame** auf NVIDIA RTX oder Apple M-Serie. Nutze `--gpu=cuda` oder `--gpu=metal` für native Hardware-Beschleunigung. Unterstützt H.265, ProRes 4444 und OpenEXR.

## So verwenden Sie es – Detaillierte Schritte

### Schritt 1: Pipeline initialisieren und Konfiguration laden
```bash
cineskin init --project ./my_film --template cinematic_vintage
```
Erzeugt eine `cineskin.yaml` mit allen Skin-Parametern (LUT-Pfad, Maskenreferenz, Grain-Intensität). Passe die Datei direkt im Editor an – alle Werte sind **hot-reloadable**.

### Schritt 2: Batch-Processing mit benutzerdefiniertem Output-Ordner
```bash
cineskin process --input ./raw_footage --output ./graded_footage --config cineskin.yaml --gpu=metal --threads 8
```
Verarbeitet alle `.mov`/`.mp4`-Dateien im Input-Ordner. Nutzt **8 parallele Threads** und GPU-Beschleunigung. Fortschritt wird in Echtzeit an die Konsole ausgegeben (z.B. `Frame 142/500 – 28.3% – ETA 3m12s`).

### Schritt 3: Skin als eigenständiges Paket exportieren
```bash
cineskin export --name "noir_dark" --format .cube --output ./skins/
```
Extrahiert die aktuelle Skin-Konfiguration als **LUT-Datei (.cube)**. Diese kann direkt in Premiere Pro, Final Cut oder Resolve importiert werden. Optional: `--include-masks` für Alpha-basierte Overlays.

## Best Practices

### 1. **CI/CD-Integration für automatisierte Grading-Pipelines**
Nutze cineskin in GitHub Actions oder GitLab CI, um bei jedem Push auf den `main`-Branch automatisch alle neuen Raw-Files zu graden. Beispiel-Config:
```yaml
- name: Apply cineskin
  run: cineskin process --input ./uploads --output ./graded --config .cineskin/production.yaml --gpu=cuda
```
So stellst du sicher, dass **jeder Build denselben Look** erhält – ideal für Serienproduktionen oder Trailer.

### 2. **Versionierung von Skin-Konfigurationen mit Git LFS**
Speichere alle `.cube`- und `.yaml`-Dateien in einem separaten Repository mit **Git LFS** (Large File Storage). So bleiben Änderungen nachvollziehbar und du kannst bei Bedarf auf ältere Looks zurückgreifen. Beispiel:
```bash
git lfs track "*.cube"
git add cineskin.yaml noir_dark.cube
git commit -m "Add noir_dark skin v2.1 – adjusted contrast curve"
```
