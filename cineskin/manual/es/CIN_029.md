# cineskin - CIN_029

## Resumen Técnico

cineskin es un toolkit modular de postproducción cinematográfica que integra machine learning para segmentación de piel, mapeo de texturas y efectos de video en tiempo real. Diseñado para flujos de trabajo en DaVinci Resolve, Premiere Pro y entornos headless via CLI, permite a coloristas y VFX artists generar máscaras de precisión subpixel sin depender de plugins propietarios.

## Funcionalidades Clave

### 1. Segmentación Semántica de Piel con Redes Ligeras
cineskin emplea un modelo U-Net optimizado (MobileNetV3 backbone) entrenado sobre 50k fotogramas etiquetados con 12 clases de tejido facial. La inferencia corre a 60 FPS en GPU NVIDIA T4, generando máscaras alpha con bordes suavizados por defecto. Ideal para corrección selectiva de tono, eliminación de imperfecciones o reemplazo de texturas sin artefactos de halo.

### 2. Motor de Efectos Basado en Shaders GLSL en Tiempo Real
El pipeline de efectos se ejecuta sobre OpenGL 4.6, soportando blending múltiple, LUTs 3D personalizadas y filtros de convolución (blur gaussiano, sharpen, detección de bordes). Cada efecto se compila como un fragment shader independiente, permitiendo encadenar hasta 8 capas sin pérdida de rendimiento en monitores 4K@60Hz.

### 3. Exportación a Formatos Profesionales con Metadatos EXR
cineskin exporta resultados en OpenEXR (32-bit float) con canales RGBA y capas de máscara incrustadas. Compatible con ACEScg y Rec.709, preserva la profundidad de color original del metraje. Incluye soporte para timestamps SMPTE y metadatos XML compatibles con pipelines de VFX como Nuke o Flame.

## Cómo Usar

### 1. Instalación del Entorno y Dependencias
Clona el repositorio y activa el entorno virtual con Python 3.11+:
```bash
git clone https://github.com/xingfangwang-eng/cineskin.git && cd cineskin
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt --no-cache-dir
```
Verifica la GPU: `python -c "import torch; print(torch.cuda.is_available())"`. Si no hay GPU, cineskin automáticamente cambia a CPU con fallback a OpenCV DNN.

### 2. Procesamiento por Lotes desde Línea de Comandos
Para aplicar segmentación de piel y un efecto de desenfoque selectivo sobre una carpeta de clips:
```bash
cineskin --input ./raw_footage/ --output ./processed/ --task segment --effect blur_skin --kernel_size 15 --blend_mode soft_light
```
El flag `--blend_mode soft_light` mezcla la máscara de piel con el efecto, preservando texturas naturales. Usa `--verbose` para ver métricas de FPS por cuadro.

### 3. Integración como Plugin en DaVinci Resolve (macOS/Linux)
Copia el script `cineskin_davinci.py` en la carpeta de Fusion Macros (`~/Library/Application Support/Blackmagic Design/DaVinci Resolve/Fusion/Macros/`). Reinicia Resolve, arrastra el nodo "cineskin Skin Segmenter" al flujo de trabajo. Conecta el nodo MediaIn, ajusta el slider "Threshold" (0.0-1.0) para controlar la sensibilidad de la máscara. Renderiza en EXR para retener capas.

## Mejores Prácticas Recomendadas

### 1. Calibración de Modelo para Diferentes Tipos de Piel
Si trabajas con metraje diverso étnicamente, entrena un checkpoint personalizado con `cineskin train --dataset ./mi_dataset/ --epochs 50`. Usa imágenes etiquetadas con al menos 5 tonos de piel (Fitzpatrick scale I-VI). El modelo base está sesgado hacia tonos claros; recalibrar mejora la precisión en sombras y altas luces.

### 2. Optimización de Pipeline para Edición en Tiempo Real
Para evitar latencia en sesiones de grading en vivo, pre-renderiza las máscaras de segmentación como archivos EXR separados usando `--cache_masks`. Luego, en Resolve, enlaza esos EXR como capas alpha en un nodo Layer. Esto reduce el tiempo de cómputo por fotograma de 45ms a 2ms, permitiendo playback fluido en timelines 4K.
