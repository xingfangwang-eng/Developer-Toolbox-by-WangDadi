# cineskin - CIN_009

## Resumen

**cineskin** es un motor de postproducción cinematográfica basado en capas de inteligencia artificial, diseñado para aplicar y transferir paletas de color, texturas de granulado y estilos de iluminación de manera no destructiva. Este toolkit está optimizado para flujos de trabajo en DaVinci Resolve, Premiere Pro y composición en tiempo real con Python/OpenCV.

## Contenido Detallado

### ¿Qué es cineskin?

cineskin es un **framework de skinning visual** que permite a los desarrolladores y editores de vídeo incrustar perfiles de color y efectos de postproducción directamente en el pipeline de renderizado. A diferencia de los LUTs estáticos, cineskin genera **máscaras de profundidad dinámicas** que se adaptan al contenido de la escena, resolviendo problemas comunes de clipping de color y desincronización de granulado en tomas con alto rango dinámico.

### Características Principales

1. **Motor de Transferencia de Estilo Multimodal (MSTM)**  
   No solo transfiere el look general de una película de referencia, sino que segmenta por planos (fondo, sujeto, iluminación) y aplica matrices de corrección independientes. Soporta exportación directa a formato `cube` y `dctl` para integración en NLEs.

2. **Sistema de Granulado Fílmico Generativo (GFG)**  
   Genera granulado fotoquímico basado en la sensibilidad ISO simulada y el tipo de emulsión (Kodak 2383, Fuji Eterna, etc.). El algoritmo varía la textura en función del brillo y el contraste local, evitando el efecto "plano" de los overlays tradicionales.

3. **API de Skinning por Capas (LSA)**  
   Expone una API REST y una librería Python (`cineskin.core`) que permite a los desarrolladores inyectar efectos en tiempo real durante la reproducción. Ideal para herramientas de edición colaborativa o motores de juego que requieran un look cinematográfico consistente.

### Cómo Usar

#### 1. Configurar un Pipeline de Color Automatizado con Python

```python
from cineskin.core import SkinPipeline, FilmStock

# Inicializar pipeline con perfil de Kodak 2383
pipeline = SkinPipeline(stock=FilmStock.KODAK_2383, iso=500, grain_strength=0.8)

# Cargar clip de referencia (frame array numpy)
reference_frame = load_frame("path/to/clip.dpx")

# Aplicar skin con máscara de profundidad dinámica
output = pipeline.apply(
    frame=reference_frame,
    mask_type="depth",  # 'depth' | 'luma' | 'chroma'
    lut_path="custom_look.cube",
    preserve_highlights=True
)

# Exportar como DCTL para DaVinci Resolve
pipeline.export_dctl("look_kodak_2383.dctl")
```

#### 2. Integrar el Granulado Generativo en Premiere Pro

1. **Crear un ajuste preestablecido de efectos**  
   Desde el panel de `Efectos` de Premiere, arrastra `cineskin Grain Generator` sobre un clip. En los controles de efectos, selecciona `Emulsión: Fuji Eterna 500T` y ajusta `Grain Size: 0.5 µm`.

2. **Sincronizar con el metraje original**  
   Activa `Auto-Adaptive Grain` para que el granulado se atenúe en las sombras y se intensifique en las altas luces, imitando el comportamiento de la película real.

3. **Exportar como preset**  
   Guarda la configuración como un `.prfpset` para reutilizarlo en proyectos futuros. cineskin genera automáticamente un archivo de calibración que mantiene la consistencia entre tomas.

#### 3. Desplegar Skinning en Tiempo Real con la API REST

```bash
# Iniciar servidor de skinning en localhost:8080
cineskin serve --port 8080 --cache-size 512MB

# Enviar frame codificado en base64 para procesamiento
curl -X POST http://localhost:8080/skin \
  -H "Content-Type: application/json" \
  -d '{
    "frame": "<base64_encoded_frame>",
    "profile": "noir_style",
    "grain": true,
    "output_format": "png"
  }'
```

La respuesta incluye el frame procesado y metadatos como `applied_lut_id` y `grain_seed` para depuración.

### Mejores Prácticas

1. **Usar máscaras de profundidad en lugar de LUTs planos**  
   En tomas con iluminación mixta (luz natural + artificial), activa el modo `mask_type="depth"` para que cineskin corrija el color de manera independiente en el fondo y el primer plano. Esto evita el efecto "plastificado" que ocurre cuando se aplica un solo LUT a toda la imagen.

2. **Calibrar el granulado por resolución de salida**  
   Si tu proyecto final es 4K, configura `grain_resolution="4K"` en el pipeline. El granulado generativo se redimensiona y se difumina para coincidir con la densidad de píxeles real, evitando el aliasing en escalados descendentes (por ejemplo, de 6K a 4K).
