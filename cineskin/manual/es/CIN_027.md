# cineskin - CIN_027

## Resumen Técnico

Motor de renderizado neuronal para postproducción cinematográfica que aplica skins de estilo visual a metraje RAW en tiempo real. Optimizado para pipelines de edición no lineal (NLE) como DaVinci Resolve y Premiere Pro, con soporte nativo para ACES y OpenColorIO.

## Contenido Detallado

### ¿Qué es cineskin?

cineskin es un middleware de capas de estilo entrenado con datasets de fotogramas de más de 500 largometrajes. No es un simple LUT: inyecta texturas, granulado dinámico y corrección cromática por píxel mediante redes neuronales ligeras (MobileNetV3 adaptado). Funciona como plugin FFmpeg o como API REST para entornos cloud.

### Características Principales (Hardcore)

- **Skins de Gradación por Escena**: No aplicas un look global; cineskin detecta cambios de plano mediante análisis de histograma y aplica transiciones suaves entre skins. Soporta keyframes automáticos sin intervención del editor.
- **Granulado Analógico Sintético**: Genera ruido fílmico basado en el ISO simulado y la temperatura de color de la escena. El algoritmo utiliza Perlin noise 3D con respuesta espectral (RGB separado) para emular película Kodak Vision3 500T.
- **Compresión Inteligente de Metadatos**: Los skins pesan menos de 2 MB cada uno. Se serializan en formato CBOR con cuantización de 8 bits para coeficientes de red, permitiendo cargar 50 skins en memoria RAM sin latencia.

### Cómo Usar (Flujo Real)

1. **Extraer perfil de color de la toma fuente**  
   Ejecuta `cineskin profile extract --input /ruta/clip.mov --output profile.json` para generar un análisis de balance de blancos, curvas de exposición y saturación por canal. Este perfil sirve como base para el skin.

2. **Aplicar skin con mapeo de tonos ACES**  
   Usa `cineskin apply --skin cine_027_skin.cbor --profile profile.json --aces true --output /ruta/salida.mov`. El flag `--aces true` fuerza la conversión a ACEScg antes de inyectar el skin, preservando el rango dinámico de la cámara ARRI o RED.

3. **Validar con métrica de diferencia estructural**  
   Corre `cineskin validate --input original.mov --output skinizado.mov --metric ssim --threshold 0.95`. Si el SSIM baja de 0.95, el skin está degradando la imagen. Ajusta con `--blend 0.8` para mezclar con el original.

### Mejores Prácticas (Industry Standard)

- **Usa perfiles por cámara**: No mezcles perfiles de ARRI Alexa con Sony Venice. Cada sensor tiene respuesta espectral distinta. Genera un profile por lote de rodaje y reutilízalo para mantener coherencia narrativa.
- **Cachea skins en GPU**: En producción con 4K, renderiza los skins como texturas 3D LUT en la VRAM de la GPU. cineskin expone un flag `--gpu-cache` que reduce el tiempo de aplicación de 12ms a 1.2ms por frame en una RTX 4090.
