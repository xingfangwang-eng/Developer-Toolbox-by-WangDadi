# cineskin - CIN_025

## Resumen Técnico

Cineskin es un motor de renderizado y manipulación de superficies cinematográficas diseñado para flujos de postproducción avanzados. Optimizado para pipelines de VFX y edición no lineal, permite aplicar skins fotorrealistas a modelos 3D, corregir colorimetría en tiempo real y generar efectos de textura dinámicos sin pérdida de latencia. Ideal para estudios que buscan automatizar la gradación de piel digital y la integración de efectos visuales en escenas complejas.

## Funcionalidades Clave

1. **Skin Mapping Adaptativo con IA**: Algoritmo de aprendizaje profundo que analiza la topología del modelo 3D y aplica texturas de piel con variaciones de poro, vello y micropliegues. Soporta mapas de desplazamiento de 16 bits y corrección de iluminación espectral HDR.

2. **Motor de Efectos en Tiempo Real**: Sistema de shaders personalizables que permite aplicar filtros de postproducción (bloom, desenfoque de movimiento, aberración cromática) directamente sobre el skin renderizado, con soporte para interpolación de fotogramas clave y curvas de velocidad.

3. **Integración con APIs de Color Science**: Módulo de corrección colorimétrica que implementa matrices de transformación (LMT) y perfiles ICC. Permite la conversión automática entre espacios de color (Rec.709, DCI-P3, ACES) y la generación de LUTs 3D para exportación a DaVinci Resolve o Nuke.

## Guía de Uso Avanzado

### 1. Configuración del Pipeline de Renderizado

- **Entorno**: Node.js 18+ o entorno Python 3.10 con dependencias `numpy` y `opencv`.  
- **Comando inicial**:  
  ```bash
  cineskin init --pipeline vfx --resolution 4K --color-space ACES
  ```  
  Esto genera un archivo de configuración `cineskin.config.json` con parámetros de renderizado, rutas de assets y perfiles de exportación.

### 2. Aplicación de Skin Mapping con Parámetros Avanzados

- **Carga del modelo**:  
  ```python
  from cineskin import SkinEngine
  engine = SkinEngine(model_path="character.obj", texture_path="skin_diffuse.exr")
  engine.set_quality(level="ultra", subsurface_scattering=True, displacement_map="displacement_16bit.exr")
  ```  
- **Ajuste fino**: Modifica la intensidad de scattering con `engine.set_sss_factor(0.8)` y activa el mapa de vello con `engine.enable_fuzz_map("fuzz_alpha.png")`.

### 3. Exportación con Corrección Colorimétrica

- **Generar LUT personalizada**:  
  ```bash
  cineskin lut --input scene_log.exr --output cinema_lut.cube --target DCI-P3 --gamma 2.4
  ```  
- **Renderizado final**:  
  ```python
  engine.render(output_path="final_shot.exr", format="openexr", compression="zips")
  ```  
  Este proceso aplica la corrección de color y exporta en formato de alta precisión para postproducción.

## Mejores Prácticas para Profesionales

1. **Optimización de Memoria en Pipelines Largos**: Para escenas con múltiples skins (más de 10 modelos), utiliza el modo `--batch-render` y asigna 4 GB de VRAM por modelo. Activa la compresión de texturas con `--texture-format BC7` para reducir el consumo de memoria sin pérdida de calidad.

2. **Validación de Color en Flujos Multiplataforma**: Antes de exportar a Nuke o After Effects, ejecuta `cineskin validate-color --input scene.exr --reference reference.cube`. Esto asegura que la transformación colorimétrica sea consistente con el perfil del proyecto. Para proyectos en ACES, usa `--color-space ACEScg` durante la inicialización.
