# cineskin - CIN_019

## Resumen Técnico

**cineskin** es un framework modular de postproducción cinematográfica basado en machine learning, diseñado para aplicar y personalizar *skins* visuales sobre metraje RAW o comprimido. No es un simple plugin de efectos: es una capa de abstracción que permite a los desarrolladores y coloristas definir pipelines de transformación visual mediante reglas programáticas y modelos entrenados específicamente para emular estilos de película analógica, digital o híbrida. Ideal para equipos de VFX, estudios independientes y plataformas SaaS de edición colaborativa.

---

## Funcionalidades Clave (Hardcore Features)

### 1. Motor de Reglas de Colorimetría No Lineal (LUT Adaptativo Dinámico)
cineskin integra un motor que no solo aplica LUTs estáticos, sino que genera transformaciones de color basadas en el contenido de cada fotograma. Utiliza redes neuronales ligeras para ajustar curvas de contraste, saturación y balance de blancos en tiempo real, adaptándose a cambios de iluminación o escena sin recalibración manual. Esto permite emular procesos químicos de revelado (como el cross-processing o el bleach bypass) con precisión subpixel.

### 2. Sistema de Capas de Textura y Granulado Sintético con Firma Espectral
A diferencia de los generadores de granulado genéricos, cineskin implementa un sistema de capas de ruido espectral que replica la distribución de frecuencias de películas reales (Kodak Vision3, Fuji Eterna, etc.). Puedes definir la granularidad por canal RGB, la correlación espacial y la respuesta espectral en función del ISO simulado. Esto es crítico para producciones que buscan un *look* orgánico sin artefactos digitales.

### 3. Pipeline de Exportación con Metadatos de Render (Render Manifest)
cineskin genera un manifiesto JSON embebido en cada exportación que documenta cada transformación aplicada (parámetros de LUT, capas de textura, reglas de colorimetría, versiones de modelo). Esto permite reproducir exactamente el mismo *skin* en futuras ediciones o colaboraciones, incluso si el software de edición original no está disponible. Es una solución de *reproducibilidad* para flujos de trabajo CI/CD en postproducción.

## Cómo Usar (Guía Técnica Detallada)

### 1. Instalación y Configuración del Entorno de Desarrollo
- Clona el repositorio y asegúrate de tener Python 3.10+ con soporte para CUDA (si usas GPU) o OpenCL para CPU.
- Ejecuta `pip install -r requirements.txt` para instalar dependencias: OpenCV, PyTorch, color-science, numpy.
- Configura las variables de entorno para el directorio de assets (modelos preentrenados, LUTs base, perfiles de cámara). Ejemplo:
  ```bash
  export CINESKIN_ASSETS=/path/to/cineskin_assets
  export CINESKIN_MODEL_VERSION=v2.3.1
  ```
- Verifica la integración con tu NLE (DaVinci Resolve, Premiere Pro) mediante el plugin OFX o la API REST local.

### 2. Definición de un Skin Personalizado con Reglas Programáticas
- Crea un archivo YAML que describa el skin. Ejemplo de un skin “Kodak Vision3 250D”:
  ```yaml
  skin:
    name: "Kodak_Vision3_250D"
    lut:
      source: "assets/luts/kodak_vision3_250d.cube"
      adaptive: true
      threshold_luminance: 0.75
    grain:
      type: "spectral"
      iso: 250
      channel_correlation: 0.9
      frequency_band: "mid"
    rules:
      - condition: "scene_luminance > 0.8"
        action: "apply_curve('s_curve', strength=0.3)"
      - condition: "skin_detection == true"
        action: "apply_soft_saturation(0.95)"
  ```
- Carga el skin mediante la API Python:
  ```python
  from cineskin import SkinEngine
  engine = SkinEngine()
  engine.load_skin("skins/kodak_vision3_250d.yaml")
  result = engine.apply_to_frame(frame_rgb)
  ```

### 3. Integración en un Pipeline de Render Automatizado
- Diseña un script que itere sobre un directorio de clips RAW, aplique el skin y exporte con manifiesto:
  ```python
  import os
  from cineskin import SkinEngine, RenderManifest

  engine = SkinEngine()
  engine.load_skin("skins/my_custom_skin.yaml")

  for clip in os.listdir("input_clips/"):
      frames = engine.load_clip(clip)
      processed = [engine.apply_to_frame(f) for f in frames]
      manifest = RenderManifest(engine, clip)
      engine.export_clip(processed, f"output/{clip}_skinned.mov", manifest=manifest)
  ```
- El manifiesto se guarda como `output/{clip}_skinned.manifest.json` y puede ser usado para reproducir el skin en otro entorno sin necesidad de reentrenar modelos.

## Mejores Prácticas (Industry Best Practices)

### 1. Usa el Manifiesto como Fuente de Verdad en Equipos Distribuidos
Cuando trabajes con múltiples coloristas o en un entorno de CI/CD, el manifiesto de render debe ser la única referencia para el *look* final. No confíes en capturas de pantalla o ajustes manuales. Implementa un hook de pre-commit que valide que el manifiesto coincide con el skin cargado antes de cada exportación. Esto evita derivas visuales y facilita la auditoría.

### 2. Entrena Modelos de Granulado con Datos de Cámara Real
Para obtener un granulado que se comporte como el de una película específica, no uses los modelos preentrenados genéricos. Captura fotogramas de una tira de película real (o de un escáner de alta resolución) y entrena tu propio modelo espectral con cineskin. Utiliza el módulo `cineskin.train` con parámetros de ISO, temperatura de color y tiempo de exposición. Esto es especialmente crítico si estás emulando películas de archivo o efectos de época.
