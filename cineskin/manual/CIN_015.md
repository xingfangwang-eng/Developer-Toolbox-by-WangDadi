# cineskin - CIN_015

## Overview

`cineskin` is a high-performance, API-first toolkit engineered for filmmakers, VFX artists, and post-production engineers who demand precision-grade control over cinematic skin texture, lighting simulation, and color grading pipelines. It bridges the gap between real-time rendering and offline compositing workflows, targeting professionals working with Unreal Engine, DaVinci Resolve, and Nuke.

## Core Features

### 1. **Subsurface Scattering Emulation Engine (SSSE)**
Unlike traditional SSS shaders that rely on brute-force sampling, `cineskin` implements a spectral multi-lobe scattering model that accurately simulates light transport through dermal layers. It supports wavelength-dependent absorption coefficients, enabling photorealistic skin tones under mixed lighting conditions without precomputed textures.

### 2. **Real-Time Chroma-Aware Denoising (RCAD)**
Leveraging a lightweight convolutional neural network trained on 4K+ film scans, RCAD removes noise from skin regions while preserving micro-details like pores, wrinkles, and subsurface blur. It operates at 60fps on consumer GPUs and integrates directly with OpenColorIO (OCIO) color spaces.

### 3. **Procedural Skin Aging & Condition Simulator**
Generate hyper-realistic blemishes, wrinkles, sun damage, or dermatological conditions (e.g., rosacea, eczema) using a node-based graph system. Each node exposes physiological parameters like collagen density, melanin distribution, and blood flow modulation, allowing artists to script aging sequences or medical visualizations.

## How to Use

### Step 1: Initialize a Skin Profile with Custom Spectral Data
```bash
cineskin init --profile "caucasian_adult" --spectral "custom_skin_reflectance.csv" --output "./profiles/"
```
This command parses a CSV of wavelength-to-reflectance values (380–780nm) and builds a compressed `.cskin` profile. For optimal accuracy, use data from a spectrophotometer or the built-in `--default` presets (e.g., `fair`, `olive`, `dark`).

### Step 2: Apply RCAD Denoising to a Skin Mask in DaVinci Resolve
```python
import cineskin

node = cineskin.connect_resolve_node("Serial1")
mask = cineskin.extract_skin_mask(node, method="HSV_threshold")
denoised = cineskin.rcad_apply(mask, strength=0.85, preserve_edges=True)
node.output = denoised
```
This snippet connects to DaVinci Resolve’s Fusion page, extracts a skin mask using HSV ranges, and denoises only the skin region. The `preserve_edges` flag ensures sharp boundaries around eyes, lips, and hair.

### Step 3: Script a Procedural Aging Timeline in Nuke
```python
import cineskin.nuke as cn

age_node = cn.AgingNode()
age_node.set_parameter("collagen_degradation", 0.3)
age_node.set_parameter("melanin_spread", 0.6)
age_node.set_parameter("capillary_dilation", 0.2)
age_node.set_parameter("time_scale", "years")

cn.animate(age_node, frame_start=1000, frame_end=2000, loop=False)
```
This creates a node that procedurally ages a character over 1000 frames. Parameters are keyframeable and can be driven by audio or motion data for dynamic storytelling.

## Best Practices

### 1. **Always Calibrate Your Monitor with OCIO v2**
`cineskin` assumes ACEScg as its working color space. Before any skin grading or denoising, ensure your Nuke or Resolve project is set to ACES 1.3 with the latest OCIO config. Mismatched color spaces will cause artifacts in SSS and denoising outputs.

### 2. **Use Multi-Resolution Baking for Real-Time Applications**
When exporting skin profiles for Unreal Engine or Unity, bake the `.cskin` file into multiple LODs (e.g., 512x512 for close-ups, 256x256 for mid-shots). This reduces memory overhead without sacrificing fidelity. Use the `--lod 4` flag during export to auto-generate mipmap-like variants.
