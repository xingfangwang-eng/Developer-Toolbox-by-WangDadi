# cineskin - CIN_015

## 概要

cineskinは、映像ポストプロダクションにおける**映画スキンデザイン**と**ビデオエフェクト編集**を、コードベースで精密に制御するためのプロフェッショナルツールです。従来のGUIベースのツールでは実現困難な、ピクセル単位のスキンマッピングとリアルタイムプレビューを、CLIとAPIの両方で提供します。

## 詳細な内容

### cineskinとは？

cineskinは、映像制作パイプラインに直接組み込める**スキンエフェクトエンジン**です。特に、肌の質感補正、映画調のカラーグレーディング、テクスチャオーバーレイを自動化し、ポストプロダクションの工数を劇的に削減します。C++で記述されたコアエンジンは、GPUアクセラレーションに対応し、4K/8K映像でもリアルタイム処理が可能です。

### コア機能

1. **ダイナミックスキンマスク生成**  
   ディープラーニングベースの肌領域セグメンテーションにより、髪の毛やアクセサリーを自動除外。肌色の変動や照明変化にロバストで、サブピクセル精度のマスクを出力します。

2. **カスタムLUT連動型スキンシェーダー**  
   DaVinci ResolveやFinal Cut ProのLUTとシームレスに連携。スキン領域のみに適用するカーブ調整や、肌のテクスチャ（毛穴・シワ・艶）をパラメトリックに制御できます。

3. **バッチ処理 & パイプライン統合**  
   JSONまたはYAMLで定義したワークフローを、複数クリップに一括適用。FFmpegやNukeとのパイプ接続により、既存のポストプロダクションパイプラインに数行のコードで統合可能です。

### 使い方

1. **スキンマスクの生成と調整**  
   以下のコマンドで、入力映像からスキンマスクを生成します。`--threshold` で肌検出の感度を調整可能です。
   ```bash
   cineskin mask --input input.mov --output mask.exr --threshold 0.85
   ```
   生成されたEXRマスクは、アルファチャンネルとしてNukeやAfter Effectsに直接読み込めます。

2. **スキンシェーダーの適用**  
   カスタムLUTとスキンシェーダーを組み合わせて適用します。`--lut` でLUTファイルを指定し、`--shader` でスキン専用のシェーダーパラメータを設定します。
   ```bash
   cineskin shade --input input.mov --mask mask.exr --lut cinema_lut.cube --shader skin_shader.json --output shaded.mov
   ```
   シェーダーJSONでは、肌の明度、彩度、テクスチャ強度を個別に制御できます。

3. **バッチ処理による自動化**  
   複数のクリップを一括処理するには、ワークフローYAMLを作成し、`--batch` フラグで実行します。
   ```yaml
   workflow:
     - input: clip1.mov
       mask: { threshold: 0.8 }
       shade: { lut: "film_lut.cube", shader: "skin_shader.json" }
     - input: clip2.mov
       mask: { threshold: 0.9 }
       shade: { lut: "warm_lut.cube", shader: "skin_shader.json" }
   ```
   ```bash
   cineskin batch --workflow workflow.yaml --output-dir ./processed/
   ```

### ベストプラクティス

1. **LUTとスキンシェーダーの分離管理**  
   カラーグレーディング用のLUTと、スキン専用のシェーダーは、別々のファイルで管理しましょう。これにより、プロジェクト間での再利用性が高まり、チーム内でのバージョン管理が容易になります。特に、肌の質感だけを微調整したい場合、LUTを変更せずにシェーダーパラメータのみを調整することで、作業効率が向上します。

2. **マスク精度の確認と手動補正**  
   自動生成されたスキンマスクは、必ずフレーム単位で確認してください。特に、髪の毛や背景に肌色が含まれるシーンでは、`--threshold` の調整や、手動でのマスク補正が必要です。cineskinは、`--refine` オプションで、ユーザー指定のペイントマスクとのブレンドもサポートしています。
