# cineskin - CIN_026

## 概要

cineskin は、映画ポストプロダクションにおけるスキンデザインと映像エフェクトを効率化する、プロフェッショナル向けツールです。本プロジェクトは、映像制作ワークフローにおけるカスタムルック作成とエフェクト適用を、コードベースで管理・自動化することを目的としています。キーワード: ムービースキンデザイン、映像エフェクトツール、映画ポストプロダクション、カラーグレーディング自動化、VFXパイプライン。

## 詳細な内容

### cineskin とは？

cineskin は、映像作品の「スキン」（ルック＆フィール）を、コードと設定ファイルで定義・バージョン管理できるツールです。従来のGUIベースのグレーディングやエフェクト適用を、宣言型の設定に置き換え、チーム間の再現性と効率を飛躍的に向上させます。特に、複数のプロジェクトで一貫したルックを適用したい場合や、CI/CDパイプラインに組み込んで自動グレーディングを行いたい場合に威力を発揮します。

### コア機能

1.  **宣言型スキン定義エンジン**: YAML/JSONベースの設定ファイルで、LUT、カラーコレクション、プラグインパラメータを一元的に管理。`cineskin apply` コマンド一つで、指定したクリップにスキンを適用。バージョン管理と差分比較が可能で、チーム全体でルックを共有・再利用できます。

2.  **AIアシストマッチバック**: 参照映像（リファレンス）から自動でカラーパレットとトーンカーブを抽出し、対象クリップに適用。従来の手動マッチバック作業を80%削減。抽出したパラメータは、スキン設定としてエクスポート可能。

3.  **プラグインフックシステム**: DaVinci Resolve、Nuke、After Effectsなど、主要なポストプロダクションツールのプラグインを、cineskinのワークフローに統合。プラグインのパラメータをスキン設定内で直接指定でき、複雑なエフェクトチェーンもコードで再現可能。

### 使い方

1.  **スキンファイルの作成**: プロジェクトルートに `cineskin.yaml` を作成。以下のように、カラーコレクションとプラグイン設定を記述します。
    ```yaml
    # cineskin.yaml
    version: 1.0
    name: "cinematic_look_v2"
    color:
      lut: "path/to/filmic.cube"
      primaries:
        saturation: 0.8
        contrast: 1.1
    plugins:
      - name: "com.example.grain"
        params:
          amount: 0.05
          size: 1.0
    ```

2.  **スキンの適用**: ターミナルで以下のコマンドを実行。`--recursive` フラグでサブディレクトリ内の全クリップに適用可能。
    ```bash
    cineskin apply --input ./footage/ --output ./output/ --skin cinematic_look_v2
    ```

3.  **バージョン管理とロールバック**: `cineskin history` で適用履歴を確認。`cineskin rollback <commit_id>` で以前の状態に瞬時に戻せます。これにより、クライアントからの修正依頼にも迅速に対応可能。

### ベストプラクティス

-   **スキンのモジュール化**: シーンごと、カメラごとにスキンを分割し、`cineskin.yaml` で `import` 文を使って合成する。これにより、特定のシーンだけルックを変更する際に、全体の設定を書き換える必要がなくなります。
-   **CI/CD パイプラインへの組み込み**: バージョン管理システム（Gitなど）にスキン設定をコミットし、プルリクエスト時に自動でグレーディング結果をプレビュー生成する。これにより、ルックのレビューと承認プロセスを効率化できます。
