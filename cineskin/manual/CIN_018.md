## What is cineskin?

cineskin is a lightweight yet deeply extensible framework for crafting, testing, and deploying real-time skin-tone-aware visual effects for motion picture and broadcast workflows. It bridges the gap between color grading pipelines and procedural skin rendering, giving VFX artists and pipeline TDs a deterministic layer for per-shot skin response modeling.

## Core Features

1. **Per-Pixel Skin Response Mapping**  
   Leverages a custom-trained luminance-chroma model to isolate skin regions with sub-pixel accuracy, then applies parametric color transforms (LUTs, CDL, or OpenColorIO) *only* to those regions — without masking artifacts or edge bleeding. No more roto for skin.

2. **Real-Time Wrapper for OpenFX & Nuke**  
   Ships as a native OpenFX plugin and a Nuke gizmo, with GPU-accelerated inference via CUDA or Metal. Supports live feedback at 4K 60fps, with per-frame serialization for shot-based consistency checks in review pipelines.

3. **Look-Up Table (LUT) Generation from Reference Stills**  
   Feed cineskin a reference frame, and it reverse-engineers a skin-specific LUT that matches the reference’s chroma distribution. Exports as .cube, .spi1d, or .spi3d — ready for on-set DIT or color assist workflows.

## How to Use

### Step 1: Initialize a Skin Profile from a Reference Frame

```bash
cineskin profile init --reference /path/to/reference.exr --output profile.json
```

This extracts the skin chroma cluster from the reference using a CIELAB-based Gaussian mixture model. The resulting profile stores per-channel deviation, luminance range, and a fallback mask for non-skin areas.

### Step 2: Apply the Profile to a Shot Sequence

```bash
cineskin apply --input /shots/seq_01/ --profile profile.json --output /renders/seq_01_skin/
```

The engine processes each frame through its inference pipeline: skin detection → mask generation → per-pixel color transform. Use `--preview` to dump a side-by-side comparison without full render overhead.

### Step 3: Export a Skin-Specific LUT for Grading

```bash
cineskin lut --profile profile.json --format .cube --size 33 --output skin_lut_33.cube
```

Generates a 3D LUT that can be loaded directly into DaVinci Resolve, Baselight, or Nuke. The LUT only affects skin tones, leaving everything else untouched — ideal for shot-matching across camera sources.

## Best Practices

1. **Always Profile Per Lighting Setup**  
   Skin response varies drastically under tungsten, HMI, or LED arrays. Generate a new profile for each lighting configuration — don’t reuse a profile from a different shoot. Store profiles in version control alongside your project files.

2. **Validate Mask Outputs Before Rendering**  
   Run `cineskin validate --profile profile.json --mask` on a representative frame to inspect the skin mask overlay. If you see false positives (e.g., wood, leather, or certain fabrics), adjust the `--chroma-threshold` flag and re-profile. Silently rendering bad masks wastes render farm cycles and introduces artifacts downstream.
