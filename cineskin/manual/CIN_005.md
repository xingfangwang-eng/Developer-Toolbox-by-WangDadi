# cineskin - CIN_005

## Overview

cineskin is a battle-tested, low-latency film-grade skin retouching and color grading engine built for modern video post-production pipelines. It abstracts away the complexity of temporal coherence and skin segmentation, letting you focus on the creative cut.

## What is cineskin?

cineskin is a GPU-accelerated CLI + SDK tool that applies photorealistic skin texture overlays and cinematic color LUTs directly to detected skin regions in video frames. Designed for VFX artists, colorists, and indie filmmakers, it solves temporal flickering and inconsistent skin masking that plague traditional keyframe-based workflows.

## Core Features

- **Temporal Coherent Skin Segmentation** – Uses a lightweight ONNX-based neural net trained on 50K+ diverse skin tones across film stock. Masks propagate seamlessly across cuts and motion blur without per-frame re-inference, eliminating the "strobe effect" on skin edges.
- **Procedural Skin Texture Layering** – Applies high-frequency detail maps (pores, subsurface scattering, micro-shadows) as a separate render pass. Supports blending modes (multiply, overlay, soft light) with per-channel gain control, all within a single `ffmpeg` filter chain.
- **Real-time 3D LUT Preview with Metadata Injection** – Embeds the applied LUT’s ICC profile and tone-mapping parameters as sidecar metadata in the output container. This ensures downstream color grading tools (DaVinci Resolve, Baselight) can reverse-engineer the original skin treatment without guesswork.

## How to Use

1. **Batch Process a Directory with Custom Skin Detection Thresholds**  
   Run `cineskin --input ./footage/ --output ./processed/ --skin-threshold 0.85 --temporal-smooth 3 --luminance-weight 0.4`. This scans all `.mov` and `.mp4` files, applies skin segmentation at an 85% confidence threshold, and uses a 3-frame temporal buffer to smooth mask edges. The `--luminance-weight` flag adjusts how much the algorithm favors bright skin regions (useful for high-key interviews).

2. **Apply a Custom Skin Texture Pack with Per-Channel Gain**  
   Use `cineskin --texture-pack ./skins/porcelain.ptx --gain-r 1.2 --gain-g 0.9 --gain-b 1.0 --blend-mode overlay`. The `.ptx` file is a compressed archive containing a diffuse map, a detail map, and a mask falloff curve. The gain flags let you fine-tune the red channel (blood flow) independently from green (subsurface) and blue (surface moisture).

3. **Export a Sidecar LUT for DaVinci Resolve**  
   After processing, run `cineskin --export-lut ./looks/cinematic_skin.cube --input-metadata ./processed/example.mp4`. This extracts the embedded skin-treatment LUT from the file’s metadata and writes a standard 33-point `.cube` file. Import it into Resolve’s Color page to match the skin look exactly without re-rendering.

## Best Practices

- **Always run a dry-run on a 10-second clip before full batch processing.** Use `--dry-run --verbose` to inspect the segmentation mask overlay and temporal coherence flags. This catches edge cases like rapid head turns or strong backlight that can fool the skin detector.
- **Pair cineskin with a denoiser filter in your pipeline.** The skin texture layer amplifies noise in dark regions. Insert `ffmpeg -i input.mp4 -vf "hqdn3d=4:3:6:4"` before the cineskin pass to preserve skin detail without introducing artifacts.
