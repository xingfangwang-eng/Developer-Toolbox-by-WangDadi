# cineskin - CIN_009

## Overview

cineskin is a high-performance, GPU-accelerated cinematic skin texture engine designed for post-production pipelines, VFX artists, and real-time rendering environments. It bridges the gap between film-grade color grading and procedural skin detail enhancement, targeting workflows in Unreal Engine, Nuke, DaVinci Resolve, and Blender. This tool is optimized for low-latency batch processing and supports HDR, ACES, and OpenColorIO color spaces out of the box.

## Core Features

### 1. Subsurface Scattering & Pore-Level Detail Synthesis
cineskin uses a custom neural texture synthesis model trained on over 10,000 high-resolution film scans to generate realistic subsurface scattering maps and micro-pore details. It dynamically adjusts diffuse and specular layers based on lighting conditions and skin type, eliminating the “plastic” look common in CGI characters.

### 2. Real-Time Chroma-Aware Skin Tone Grading with LUT Injection
The engine analyzes facial regions using a lightweight semantic segmentation model and applies per-pixel color grading curves that respect skin tone boundaries. It supports custom 3D LUT injection, enabling seamless integration with ACEScg and Rec.709 pipelines. This feature is ideal for matching live-action footage with CG elements in real-time.

### 3. Multi-Resolution Denoising & Texture Blending for 8K+ Outputs
cineskin includes a multi-resolution denoising module that preserves high-frequency skin details while removing artifacts from low-light or compressed sources. It blends texture layers (e.g., scars, pores, wrinkles) across different LODs, ensuring consistent quality across 4K to 8K renders without performance degradation.

## How to Use

### Step 1: Set Up the Pipeline and Load Source Footage
Launch cineskin from the command line or integrate it as a plugin in your preferred DCC tool. Use the `--input` flag to specify your source sequence (EXR, DPX, or PNG sequences with alpha channels). For HDR workflows, ensure your color space is set to ACEScg using the `--colorspace acescg` argument. Example:
```bash
cineskin --input /path/to/frames --colorspace acescg --output /path/to/processed
```

### Step 2: Configure Skin Detection and Texture Synthesis Parameters
Run the initial analysis with `--analyze-skin` to generate a skin mask and subsurface scatter map. Adjust the `--pore-intensity` (range 0.0–1.0) and `--specular-sharpness` (default 0.7) to match your reference footage. For real-time preview, use the `--live-preview` flag with a 1080p proxy. To inject a custom LUT, pass the `.cube` file with `--lut /path/to/lut.cube`.

### Step 3: Batch Process and Export with Denoising
Once parameters are tuned, execute the full batch with `--denoise --output-format exr --bit-depth 16`. cineskin will automatically apply multi-resolution denoising and blend texture layers. For 8K renders, set `--lod-levels 6` to ensure consistent detail across all LODs. Export logs are written to `cineskin_batch.log` for debugging.

## Best Practices

### 1. Use ACEScg for Color-Managed Pipelines
Always set your working color space to ACEScg when using cineskin in a VFX or film production environment. This ensures that skin tone grading and LUT injection remain consistent across different renderers and compositing tools. Avoid sRGB or Rec.709 for intermediate frames—convert only at the final output stage.

### 2. Optimize Texture Synthesis with Reference Frames
For best results, provide at least 10–20 frames of reference footage with varied lighting conditions (key, fill, rim) and skin tones. cineskin’s neural model adapts to these references to generate more accurate subsurface and pore maps. Use the `--reference /path/to/reference_frames` flag to feed a set of high-quality frames before batch processing.
