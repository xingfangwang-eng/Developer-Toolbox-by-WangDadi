# cineskin - CIN_022

## Overview

cineskin is a high-performance, GPU-accelerated toolchain engineered for filmmakers, VFX artists, and post-production studios who demand pixel-perfect control over cinematic color grading, texture mapping, and real-time skin tone optimization. Designed to bridge the gap between raw footage and final-grade aesthetics, cineskin eliminates tedious manual masking and delivers consistent, filmic results across complex scenes.

## Core Features

### 1. **Neural Skin Tone Segmentation & Adaptive LUT Mapping**
cineskin leverages a lightweight on-device neural network (trained on 50K+ diverse skin types) to isolate skin regions with sub-pixel accuracy—no green screen, no rotoscoping. It then applies adaptive 3D LUTs that preserve natural undertones while correcting for lighting shifts, camera sensors, and exposure inconsistencies. This ensures your talent looks natural whether under tungsten, HMI, or mixed-color LED arrays.

### 2. **Real-Time Texture Preservation with Frequency-Aware Filtering**
Unlike generic beauty filters that blur away pores and fine details, cineskin uses frequency-domain analysis to separate high-frequency texture (pores, hair, skin grain) from low-frequency color and blemish zones. You can independently adjust smoothness, micro-contrast, and sharpness per frequency band—without introducing plastic artifacts or losing cinematic grain.

### 3. **Multi-Node Compositing Graph with GPU-Accelerated Previews**
cineskin integrates a node-based workflow where each correction layer (color grade, texture pass, mask refinement) is a non-destructive node in a directed acyclic graph. Nodes are executed on the GPU in parallel, delivering real-time 4K previews at 60fps. Export to OpenColorIO, ACES, or custom CDL/CCC profiles for seamless pipeline integration.

## How to Use

### 1. **Load Your Footage and Initialize the Skin Detection Engine**
```bash
cineskin --input /path/to/exr_sequence.%04d.exr --fps 24 --colorspace ACEScg
```
The tool auto-detects skin regions using the neural segmenter. Verify the segmentation mask overlay: run `cineskin --preview-mask`. If false positives occur (e.g., background fabric), adjust the sensitivity threshold with `--skin-confidence 0.85` (default 0.75). For multi-camera setups, use `--camera-id` to assign per-camera profiles.

### 2. **Configure the Texture and Grade Pipeline via Node Graph**
Launch the interactive node editor:
```bash
cineskin --editor
```
Drag a **Skin Segmentation** node, connect it to a **Frequency Splitter** node (set low-pass cutoff to 8 cycles/degree for natural skin). Then attach a **Color Grade** node with a custom 3D LUT (e.g., `Kodak 2383_D55.cube`). Fine-tune the **Texture Mix** node to blend 70% original texture with 30% smoothed low-pass to reduce blemishes while retaining pores. Preview the composite in real-time with `--preview-grade`.

### 3. **Render the Final Sequence with Pipeline-Safe Metadata**
Set output parameters:
```bash
cineskin --render --output /output/grade_v1.%04d.exr --colorspace ACES2065-1 --bit-depth 16 --compression PIZ
```
Append `--export-cdl /output/cdl_grades.txt` to generate a CDL file for colorist handoff. For batch processing, create a `.cineskin` project file and run `cineskin --batch project.cineskin`. Validate the render with `--verify-checksum` to ensure frame-perfect consistency.

## Best Practices

### 1. **Pipeline Integration: Always Use ACES or Scene-Linear Color Spaces**
cineskin’s neural segmenter and LUT engine assume scene-linear input. Feeding it Rec.709 or sRGB footage without linearization will cause banding and incorrect skin tone mapping. Wrap your input pipeline with `--input-colorspace ARRI_LogC3` or `--input-colorspace RED_Log3G10` and convert to ACEScg internally. This guarantees predictable grading across VFX, compositing, and final color suites.

### 2. **Batch-Grade Consistency: Lock Your Segmentation Model Version**
When processing multiple shots from the same scene, freeze the neural model version with `--model-version v2.1.3`. This prevents subtle shifts in skin detection between shots due to model updates. Combine with `--seed 42` for deterministic noise patterns in texture blending. Always run a `--diff-check` between consecutive frames to catch temporal flicker before final render.
