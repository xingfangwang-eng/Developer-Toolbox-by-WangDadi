### What is cineskin?

cineskin is a high-performance, open-source toolchain for crafting cinematic-grade visual styles and video effects directly from the command line or CI/CD pipeline. It abstracts away the complexity of color grading, texture mapping, and real-time preview, enabling filmmakers, VFX artists, and automation engineers to apply consistent "skins" across frames, scenes, or entire reels. Unlike traditional NLE plugins, cineskin operates as a standalone binary with zero GUI overhead, making it ideal for batch processing, server-side rendering, and headless environments.

### Core Features

1. **Frame-Accurate Color Skin Engine**  
   Apply LUT-based and procedural color transformations with sub-frame precision. Supports 3D LUTs, CDL, and custom matrix operations. The engine includes a built-in histogram analyzer and automatic white balance correction, ensuring consistent tonal mapping across variable lighting conditions.

2. **Texture Overlay & Mask Compositor**  
   Composite film grain, dust, scratches, or custom textures using alpha masks and blend modes (multiply, screen, overlay). Supports per-channel blending and dynamic mask generation from luminance, chroma, or edge detection. Ideal for restoring vintage looks or simulating analog film damage.

3. **Bulk Scene-Aware Processing Pipeline**  
   Process entire directories or S3 buckets with scene-aware segmentation. The pipeline detects scene changes via histogram delta and applies per-scene skin profiles automatically. Supports parallel GPU/CPU execution, progress callbacks, and JSON-based configuration for full reproducibility.

### How to Use

1. **Initialize a Project with Scene Detection**  
   Run `cineskin init --input ./footage/ --output ./processed/ --detect-scenes`. This scans the input directory, generates a `cineskin.json` config file, and creates a scene index with timestamps. For headless servers, use `--config remote://s3://bucket/config.json` to pull configs from cloud storage.

2. **Apply a Custom Skin Profile with Mask Overlay**  
   Execute `cineskin apply --skin ./profiles/neon_noir.3dl --mask ./textures/scratch.png --blend overlay --strength 0.6`. This applies the 3D LUT and overlays the scratch texture at 60% opacity. Use `--preview` to render a single frame before batch processing. For real-time feedback, pipe output to `ffplay`.

3. **Batch Process with CI/CD Integration**  
   Add to your GitHub Actions or Jenkins pipeline:  
   ```bash
   cineskin batch --input ./footage/ --output ./rendered/ --config ./pipeline.json --workers 4 --log-level debug
   ```  
   The pipeline supports `--retry-on-error`, `--timeout`, and `--notify webhook://your-endpoint` for automated failure handling. Generated metadata (per-frame stats, processing time) is exported as JSON for downstream analytics.

### Best Practices

- **Use Scene-Detection Before Batch Processing**  
  Always run `--detect-scenes` first. Applying a single skin profile across mismatched scenes (e.g., daylight vs. night) results in inconsistent grading. cineskin’s scene-aware engine prevents this by isolating each scene and applying the correct profile automatically. For advanced users, combine with `--scene-threshold 0.15` to fine-tune detection sensitivity.

- **Version-Control Your Skin Profiles**  
  Store `.3dl`, `.cube`, and `.json` configs in a Git repository. Use semantic versioning (e.g., `neon_noir_v1.2.3.3dl`) and include a changelog. This ensures reproducibility across builds and simplifies rollback. For team collaboration, host profiles on a private npm or PyPI registry and reference them via `cineskin install @studio/neon_noir`.
