## What is cineskin?

**cineskin** is a professional, high-performance tool designed to streamline and elevate the movie skin design and video effects pipeline. It solves critical bottlenecks in post-production workflows—such as real-time texture mapping, dynamic color grading, and automated asset optimization—allowing VFX artists and editors to focus on creative output rather than technical overhead.

---

## Core Features

### 1. Real-Time Procedural Skin Mapping Engine
Leverage a GPU-accelerated, node-based system that applies complex movie skin textures and surface effects (e.g., subsurface scattering, micro-detail overlays) in real-time. No pre-rendering required—iterate on character looks instantly during the final cut.

### 2. AI-Assisted Post-Production Effects Suite
Integrates a lightweight neural network that analyzes scene context (lighting, camera angle, depth) and suggests optimal video effects—from cinematic grain to lens flares—reducing manual tweaking by up to 60%. Supports batch processing for multi-shot sequences.

### 3. Automated Asset Dependency Tracker
A built-in dependency graph that maps every texture, shader, and effect to its source file. Automatically flags broken links, version mismatches, or missing assets before export, preventing render-time crashes and saving hours of debugging.

## How to Use

### Step 1: Initialize the Skin Mapping Pipeline
Open **cineskin** and import your base 3D model (supports OBJ, FBX, and Alembic). Navigate to the *Skin Mapping* tab, then select the `Procedural Skin Engine` node. Drag your reference texture onto the node’s input port—the engine will automatically detect UV seams and generate a high-fidelity displacement map. Adjust the `Subsurface Scattering` slider (range: 0.0–1.0) to match your scene’s lighting profile.

### Step 2: Apply Context-Aware Effects
Switch to the *Effects Suite* panel. Click `AI Analyze` to let the tool scan your current frame—it will output a recommended effect stack (e.g., `Cinematic Grain: 15%`, `Vignette: Soft`, `Chromatic Aberration: 0.02`). Accept or modify each parameter via the sliders. For batch consistency, lock the effect profile using the `Pin to Sequence` button, which ties the settings to the timeline’s keyframes.

### Step 3: Validate and Export with Dependency Check
Before final render, run the `Asset Dependency Tracker` by clicking `Validate` in the top toolbar. The tool will list all linked assets, their paths, and any issues (e.g., `Texture "skin_diffuse_v2.tga" not found`). Resolve missing files via the `Auto-Relink` function, which searches your project folder recursively. Once clean, hit `Export`—choose between EXR sequences, ProRes MOV, or direct integration with Nuke/After Effects via the built-in plugin.

## Best Practices

### 1. Cache Procedural Outputs for Iterative Work
When working with complex skin textures, enable the `Cache to Disk` option in the Procedural Skin Engine settings. This stores the computed displacement maps locally, allowing you to scrub the timeline without recalculating—ideal for director reviews where you need instant playback.

### 2. Version Control Your Effect Profiles
Use the `Profile Manager` to save effect stacks as `.csp` files (cineskin profile). Store these in your project’s version control system (e.g., Git LFS) to ensure team-wide consistency across shots. Always tag profiles with the scene name and date (e.g., `car_chase_grain_v2_2025-04-10.csp`) to avoid confusion during handoffs.
