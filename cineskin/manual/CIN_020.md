# cineskin - CIN_020

## Overview

cineskin is a high-performance, GPU-accelerated movie skin design and video effects tool tailored for post-production pipelines. It bridges the gap between cinematic color grading and real-time skin texture manipulation, enabling VFX artists and colorists to achieve photorealistic results without sacrificing render speed.

## Core Features

1. **Real-Time Neural Skin Texture Mapping**  
   Leverages a lightweight deep learning model (trained on 50K+ film frames) to automatically detect and isolate skin regions, then applies custom LUTs or procedural textures with sub-pixel accuracy. Supports 8K/12K footage at 60fps on RTX 4090-class hardware.

2. **Multi-Node Distributed Rendering with Zero-Overhead Sync**  
   Built on a custom MPI-based architecture, cineskin distributes skin effect computations across up to 32 nodes in a render farm. Each node processes independent frame segments, synchronized via a lock-free shared memory protocol—ideal for large-scale studio workflows.

3. **Scriptable Pipeline Integration via Python API**  
   Exposes a full Python SDK (compatible with Nuke, Fusion, and Blender) that allows artists to chain skin effects, automate batch processing, and integrate with CI/CD pipelines for versioned asset management. Example: `cineskin.apply_lut("film_grain", skin_mask="auto", intensity=0.85)`.

## How to Use

1. **Initialize the Skin Detection Model**  
   Run `cineskin.init_model(model_path="models/skin_v3.onnx", device="cuda:0")` to load the pre-trained neural network. Use `cineskin.detect_skin(input_frame, threshold=0.9)` to generate a binary mask. For multi-GPU setups, pass `device_ids=[0,1,2,3]` to auto-balance load.

2. **Apply a Custom LUT with Temporal Smoothing**  
   Load your LUT via `cineskin.load_lut("path/to/lut.cube")`, then call `cineskin.apply_skin_effect(frame, lut, mask, temporal_strength=0.3)`. The temporal smoothing parameter prevents flickering across frames—critical for high-motion scenes. Output is a 16-bit float EXR sequence.

3. **Export to a Distributed Render Job**  
   Use `cineskin.export_job(frames_dir="render_frames", output_dir="processed", nodes=16, format="exr")`. This generates a job descriptor JSON file. Submit it via `cineskin.submit_job("job.json", scheduler="slurm")` for farm execution. Monitor progress with `cineskin.job_status("job_id")`.

## Best Practices

- **Always Pre-Validate Skin Masks in CI/CD**  
   Integrate `cineskin.validate_mask()` into your automated testing pipeline to catch false positives (e.g., background textures mistaken for skin) before rendering. This reduces wasted compute cycles and ensures consistent output across shots.

- **Use Adaptive Luminance Scaling for HDR Workflows**  
   When working with HDR footage (e.g., ACEScg), enable `cineskin.set_hdr_mode(scale_factor=2.0)` to prevent skin highlights from clipping. This preserves detail in specular reflections and avoids unnatural skin tones in bright scenes.
