## What is cineskin?

**cineskin** is a lightweight, high-performance toolkit for applying cinematic color grading, texture overlays, and skin-aware visual effects directly to video frames. Unlike bloated NLE plugins, cineskin operates as a standalone CLI or API-first service, designed for batch processing, real-time preview, and seamless CI/CD integration in VFX pipelines.

## Core Features

### 1. **Skin-Aware LUT Injection with Edge Detection**
cineskin uses a hybrid edge-aware mask to isolate human skin tones in motion frames, then applies custom 3D LUTs only to those regions. This prevents background bleed and preserves natural skin texture—critical for high-end film and commercial work.

### 2. **Real-Time Texture Blending Engine**
Supports GPU-accelerated blending of film grain, dust, scratches, and analog artifacts. The engine uses a non-destructive layer stack, allowing per-frame opacity and mask adjustments without re-encoding the entire clip.

### 3. **Programmatic Scene Detection & Batch Grading**
Integrates a scene-change detector that auto-splits footage into shots, then applies per-shot grading profiles via JSON configs. This enables fully automated post-production workflows for long-form content.

## How to Use

### Step 1: Initialize a Project with Custom Skin Profile
```bash
cineskin init --profile cinema_4k --skin-model fair_skin_v2 --lut ./luts/warm_teal.cube
```
This creates a project file with skin detection thresholds calibrated for your footage. You can override sensitivity with `--threshold 0.85`.

### Step 2: Apply Texture Overlay with Temporal Smoothing
```bash
cineskin apply --input ./raw_footage.mov --overlay ./textures/film_grain_35mm.png --blend-mode overlay --opacity 0.4 --temporal-smooth 3
```
The `--temporal-smooth` flag averages the overlay across 3 frames to avoid flickering. For real-time preview, append `--preview`.

### Step 3: Export Graded Shots with Metadata
```bash
cineskin export --output ./graded_shots/ --format prores --scene-detect --meta-embed
```
This splits the input by detected scenes, exports each as a ProRes file, and embeds grading metadata (LUT path, skin mask params) into the file’s sidecar data for downstream NLE tools.

## Best Practices

### 1. **Always Calibrate Skin Detection on First Use**
Run `cineskin calibrate --sample ./test_frame.png` before batch processing. This adjusts the skin mask to your specific lighting and camera sensor, reducing false positives from clothing or background objects.

### 2. **Use JSON Configs for Reproducible Pipelines**
Store grading profiles as version-controlled JSON files. This allows you to replay the exact same color pass across multiple takes or even different projects. Example:
```json
{
  "lut": "./luts/teal_orange.cube",
  "skin_blend": 0.7,
  "texture": "./textures/scratch_light.png",
  "opacity": 0.3
}
```
Then apply with: `cineskin batch --config ./profiles/episode_5.json`
