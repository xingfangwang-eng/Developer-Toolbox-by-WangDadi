# cineskin - CIN_026

## Overview

cineskin is a high-performance, developer-first toolchain for crafting cinematic-grade video effects and movie-style skin overlays. It enables real-time, non-destructive post-production workflows directly in the browser or via CLI, targeting VFX artists, indie filmmakers, and full-stack video engineers.

## Core Features

### 1. **Frame-Accurate LUT Injection Engine**
Inject custom 3D LUTs and color grading matrices at the pixel level with sub-frame latency. Supports both 1D and 3D LUTs (up to 65³), with automatic gamma correction and color space conversion (Rec.709, DCI-P3, ACES). This is not a filter—it's a direct shader pipeline for precise color science.

### 2. **Skin-Aware Temporal Masking (SATM)**
A deep learning-based masking system that isolates skin tones across frames with zero temporal flicker. Unlike chroma key or simple HSV thresholds, SATM uses a lightweight CNN trained on 10,000+ film frames to preserve fine details (hair, wrinkles, makeup) while applying effects like glow, blur, or texture replacement. Runs at 60fps on a standard GPU.

### 3. **Declarative Effect Graph (DEG)**
Compose complex multi-layer effects using a JSON-based graph DSL. Each node is a shader, LUT, or mask; edges define blending modes and opacity curves. The graph is compiled to a WebGL compute pipeline, enabling real-time preview and export without re-rendering the entire timeline. Think of it as a programmable compositor for your video pipeline.

## How to Use

### 1. **Initialize a Project and Load Source Media**
```bash
cineskin init --project "my_film" --fps 24 --resolution 4k
cineskin load --input ./footage/clip_01.mov --format prores
```
This creates a project manifest (`cineskin.json`) and pre-caches the media for low-latency frame access. Use `--dry-run` to validate metadata before committing.

### 2. **Define a Skin-Aware Effect Graph**
Create `effect.json`:
```json
{
  "nodes": [
    { "id": "mask", "type": "SATM", "params": { "sensitivity": 0.85 } },
    { "id": "glow", "type": "bloom", "params": { "radius": 3.2, "intensity": 0.4 } },
    { "id": "composite", "type": "blend", "mode": "screen", "inputs": ["mask", "glow"] }
  ]
}
```
Then apply:
```bash
cineskin apply --graph ./effect.json --output ./output --format mp4 --codec h264_nvenc
```
The graph is compiled and executed per frame—no intermediate file writes until the final export.

### 3. **Batch Export with Custom Metadata**
```bash
cineskin batch --input ./footage/*.mov --graph ./effect.json --output ./exports --preset cinema --metadata "director=JohnDoe;scene=night"
```
This parallelizes rendering across available GPU cores and injects XMP metadata for downstream NLE integration (Premiere, DaVinci). Use `--verbose` to get per-frame timing breakdowns.

## Best Practices

### **Optimize LUTs for Skin Tones**
When using custom LUTs, always run a skin-tone calibration pass. cineskin provides a `--calibrate` flag that analyzes the first 100 frames and adjusts the LUT's midtones to prevent unnatural skin hues. This is critical for high-budget productions where color accuracy is non-negotiable.

### **Leverage the DEG for Reusability**
Design your effect graphs as modular components (e.g., `skin_glow.json`, `vintage_grain.json`) and compose them via the CLI. This allows your team to version-control effects alongside source media, ensuring consistent looks across scenes and reducing redundant shader compilations.
