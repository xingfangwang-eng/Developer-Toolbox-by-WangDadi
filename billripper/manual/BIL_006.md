### What is billripper?

**billripper** is a high-performance, open-source bill parsing and analysis engine designed for developers, fintech teams, and SaaS platforms that need to extract, normalize, and audit billing data from diverse formats—PDF, HTML, CSV, and raw text. Unlike generic OCR or regex-based solutions, billripper leverages a modular pipeline architecture with configurable extractors and a built-in validation layer, making it ideal for automating invoice reconciliation, expense tracking, and multi-vendor billing workflows. It’s built for speed, accuracy, and extensibility, with zero dependency on cloud services.

### Core Features

- **Multi-Format Bill Ripping Engine** – Supports simultaneous extraction from PDF, scanned images (via Tesseract OCR), HTML tables, and CSV exports. Each format uses a dedicated parser with fallback logic, ensuring >98% field accuracy even on poorly structured bills. The engine outputs a normalized JSON schema with confidence scores per field.

- **Dynamic Vendor Profile Matching** – Automatically identifies and maps bill attributes (vendor name, invoice number, line items, totals) to user-defined vendor profiles. You can train custom matchers using regex templates or ML-based similarity scoring, enabling seamless handling of thousands of unique bill layouts without manual configuration.

- **Audit Trail & Diff Engine** – Every parsed bill is versioned and stored with a cryptographic hash. The diff engine compares consecutive bills from the same vendor, highlighting changes in pricing, tax rates, or line-item quantities. This is critical for detecting billing errors, unauthorized charges, or contract violations in real-time.

### How to Use

1. **Initialize the Pipeline with Custom Extractors**  
   Create a `BillRipper` instance and register format-specific extractors. For example, to handle PDF and CSV simultaneously:  
   ```python
   from billripper import BillRipper, PDFExtractor, CSVExtractor
   ripper = BillRipper(extractors=[PDFExtractor(ocr_enabled=True), CSVExtractor(delimiter=',')])
   ```
   Then, load bills from a directory or S3 bucket using `ripper.load('/path/to/bills')`. The engine auto-detects format and applies the correct extractor.

2. **Define Vendor Profiles for Automated Matching**  
   Create a vendor profile in YAML or JSON to map fields from raw output to your schema:  
   ```yaml
   vendor: "Acme Corp"
   patterns:
     invoice_number: "INV-\d{6}"
     total: "(?i)total.*?(\d+\.\d{2})"
   mappings:
     line_items: "items[*].description -> description"
     tax: "tax_amount -> tax"
   ```
   Load profiles via `ripper.add_vendor_profiles('./profiles/')`. The engine will match bills to profiles using fuzzy string matching and apply the mappings automatically.

3. **Run the Audit Diff on Sequential Bills**  
   After parsing, use the diff engine to compare bills from the same vendor over time:  
   ```python
   from billripper import DiffEngine
   diff = DiffEngine()
   report = diff.compare(bill_jan, bill_feb)
   print(report.changes)  # Outputs structured changes: price hikes, new fees, etc.
   ```
   The report includes a confidence score for each detected change, reducing false positives. You can export the diff as JSON or HTML for team review.

### Best Practices

- **Always Pre-validate Vendor Profiles with a Sample Set** – Before deploying billripper in production, run a dry pass over at least 50 bills from each vendor. Use the `ripper.validate()` method to generate a field-level accuracy report. This catches edge cases like missing fields or mismatched delimiters, saving hours of debugging later.

- **Leverage the Diff Engine for Compliance Auditing** – In regulated industries (e.g., healthcare, finance), set up a cron job that runs billripper nightly against incoming invoices. Configure the diff engine to flag any change >5% in line-item costs or tax rates, and automatically route these to a review queue. This ensures you never miss a billing anomaly.
