# billripper - BIL_022

## Overview

billripper is a high-performance, multi-format invoice and bill parsing engine designed for developers and SaaS platforms. It strips, normalizes, and extracts structured data from raw PDF, image, and HTML bill sources with near-zero latency. Built for CI/CD pipelines and microservices, it ships with a lightweight CLI, REST API, and native bindings for Python, Node.js, and Go.

## What is billripper?

billripper is not just another OCR wrapper. It's a deterministic bill extraction framework that combines heuristic layout analysis with a pluggable schema matcher. Whether you're processing utility bills, SaaS invoices, or bank statements, billripper delivers field-level accuracy without relying on expensive third-party LLMs. It's optimized for batch processing, supports multi-currency and multi-language headers, and outputs JSON, CSV, or Parquet.

## Core Features

### 1. Schema-Free Field Extraction with Adaptive Layout Parsing
billripper uses a hybrid approach: rule-based anchor detection for known templates, plus a lightweight transformer model for unknown layouts. It auto-detects fields like invoice number, due date, line items, tax breakdown, and vendor details—even when the layout shifts. No training required.

### 2. Real-Time Bill Normalization & Deduplication
Each bill is hashed and fingerprinted using content + metadata. billripper detects duplicates across batches, normalizes currency symbols, date formats, and unit measures (e.g., kWh vs. MWh). It outputs a clean, consistent schema every time, ready for ingestion into ERP or analytics pipelines.

### 3. Pluggable Export & Webhook Pipeline
Define custom output schemas or use the built-in ones (QuickBooks, Xero, Stripe, CSV). billripper triggers webhooks on completion or failure, supports retry logic, and can stream results to S3, GCS, or your own API. It's built for event-driven architectures.

## How to Use

### 1. Batch Parse All Bills in a Directory
```bash
billripper parse --input ./bills/ --output ./parsed/ --format json --dedup
```
This command scans the directory for PDFs, images, and HTML files. It performs layout detection, schema matching, and deduplication in parallel. The output directory will contain one JSON file per bill, with all fields normalized and a `fingerprint` field for dedup tracking.

### 2. Run as a Microservice with Docker
```bash
docker run -p 8080:8080 billripper/server:latest
curl -X POST http://localhost:8080/parse \
  -F "file=@invoice.pdf" \
  -F "config={\"output\":\"json\",\"webhook\":\"https://myapi.com/callback\"}"
```
This spins up a stateless HTTP server. Each request returns a structured response with fields, confidence scores, and a `bill_id`. The webhook fires asynchronously once the bill is processed, allowing you to chain downstream tasks.

### 3. Integrate into Your CI/CD Pipeline (GitHub Actions)
```yaml
- name: Parse incoming bills
  uses: billripper/action@v2
  with:
    input: ./invoices/
    output: ./parsed/
    schema: quickbooks
    fail_on_error: false
```
This action runs inside a container, parses all bills from the `invoices/` folder, and fails the step only if a critical parsing error occurs. It outputs a summary artifact with bill counts, errors, and dedup stats.

## Best Practices

### 1. Use billripper as a Pre-Processor Before Your Accounting API
Don't send raw PDFs to QuickBooks or Xero—they're slow and expensive. Strip and normalize the bill first with billripper, then push the structured JSON to your accounting provider. This cuts API costs by 70% and speeds up ingestion.

### 2. Combine billripper with a Document Store for Audit Trails
Store the raw bill alongside the parsed output in S3 or MinIO. Use the `fingerprint` field as a unique key. This gives you a full audit trail, allows for reprocessing if the schema changes, and simplifies compliance with SOX or GDPR.
