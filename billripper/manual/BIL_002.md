### What is billripper?

**billripper** is a high-performance, multi-format bill extraction and analysis engine designed for developers, accountants, and SaaS platforms. It rips through PDF, scanned images, and raw text invoices with near-zero latency, converting chaotic billing data into structured, queryable JSON. Built for scale, it handles batch processing, currency normalization, and vendor deduplication out of the box.

### Core Features

1. **Adaptive OCR + Regex Fusion Engine**  
   Unlike traditional OCR-only tools, billripper combines Tesseract-based text recognition with a dynamic regex pipeline that self-adjusts based on bill layout (e.g., US medical vs. EU utility invoices). It detects table structures, line-item delimiters, and tax codes without manual template configuration.

2. **Real-Time Currency & Tax Normalization**  
   Automatically converts multi-currency amounts to a base currency (USD, EUR, JPY) using live exchange rates. Simultaneously normalizes VAT, GST, and sales tax fields into a unified schema—critical for cross-border financial reconciliation.

3. **Idempotent Batch Processing with Webhook Callbacks**  
   Queue thousands of bills for asynchronous parsing. billripper guarantees idempotency (no duplicate entries) and fires webhooks per parsed document, enabling seamless integration into CI/CD pipelines or accounting ERPs like QuickBooks and Xero.

### How to Use

1. **API-Driven Bulk Ingestion**  
   POST a ZIP file or a JSON array of base64-encoded bills to `/v1/parse/batch`. Set the `currency` header to your base currency (e.g., `USD`). Each bill returns a unique `ripper_id`—store this for audit trails. Example:  
   ```bash
   curl -X POST https://api.billripper.io/v1/parse/batch \
     -H "Authorization: Bearer <token>" \
     -H "Content-Type: application/json" \
     -d '{"bills": [{"content": "<base64>", "format": "pdf"}], "currency": "USD"}'
   ```

2. **Schema Customization via YAML Config**  
   Mount a `ripper_config.yaml` in your working directory to define custom field mappings. For instance, map `Invoice Number` to `order_id` and `Due Date` to `deadline`. The engine validates your schema against a JSON Schema draft-07 before processing:  
   ```yaml
   fields:
     - source: "Invoice Number"
       target: "order_id"
       type: string
     - source: "Due Date"
       target: "deadline"
       type: date
   ```

3. **Error Handling & Retry Logic**  
   Enable automatic retries for failed parses by setting `max_retries: 3` and `retry_delay: 5s` in the request body. billripper logs each failure with a stack trace to stdout; pipe logs to a centralized aggregator like Datadog for real-time monitoring. For manual debugging, inspect the `errors` array in the response—each entry includes a `ripper_id` and a human-readable reason (e.g., `"Unsupported font encoding"`).

### Best Practices

- **Pre-validate Bill Quality Before Ingestion**  
  Run a lightweight check using billripper’s `/v1/validate` endpoint before full parsing. It flags low-resolution scans, skewed pages, or missing mandatory fields (e.g., total amount). This reduces parse failures by up to 40% and saves API credits.

- **Use Idempotency Keys for Financial Integrity**  
  Always pass a unique `idempotency_key` in your POST requests. This prevents double-crediting or duplicate entries when network retries occur—critical for audit compliance and SOX regulations.
