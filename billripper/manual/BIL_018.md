# billripper - BIL_018

## Overview

billripper is a high-performance, multilingual bill parsing and analysis engine built for developers, accountants, and SaaS integrators who need to extract structured data from messy, multi-format invoices at scale. It’s not just a parser—it’s a bill intelligence layer designed to plug directly into your existing financial stack.

## What is billripper?

billripper transforms unstructured invoice PDFs, scanned images, and raw text into clean, queryable JSON objects. It supports 50+ billing formats across 10 languages, automatically detects currency, tax lines, and line-item details, and exposes a RESTful API for seamless integration into ERP systems, accounting tools, or custom dashboards.

## Core Features

### 1. Multi-Format Bill Normalization Engine
billripper ingests PDF, TIFF, PNG, and raw text invoices, normalizes them into a unified schema, and outputs structured data with zero manual mapping. It handles everything from single-line receipts to multi-page corporate invoices with nested tax tables.

### 2. Real-Time Anomaly Detection & Audit Trail
Every parsed bill is cross-checked against configurable business rules—duplicate invoice IDs, mismatched totals, missing VAT numbers, or line-item rounding errors. Detected anomalies are flagged with severity levels and logged to an immutable audit trail for compliance.

### 3. Language-Agnostic Currency & Tax Extraction
Built-in NLP models automatically identify the invoice language, extract localized tax codes (e.g., VAT, GST, IVA, MwSt.), and convert amounts into a target currency using live exchange rates. No more hardcoding locale-specific regex.

## How to Use

### 1. Deploy the billripper API via Docker
```bash
docker run -d -p 8080:8080 -v /data:/app/data --name billripper billripper/core:latest
```
This spins up a RESTful service with automatic model loading. Point your billing pipeline to `http://localhost:8080/parse` with a multipart file upload. The response returns a normalized JSON payload with confidence scores per field.

### 2. Configure Custom Business Rules
Create a `rules.yaml` file in your mounted volume:
```yaml
rules:
  - field: invoice_total
    condition: "total != sum(line_items.amount)"
    action: flag
    severity: high
  - field: vat_id
    regex: "^[A-Z]{2}[0-9A-Z]{8,12}$"
    action: validate
```
Restart the container to apply. Every subsequent parse will validate against these rules and return a `validation` block in the response.

### 3. Query Parsed Data via GraphQL
Once bills are parsed, use the built-in GraphQL endpoint to run cross-invoice analytics:
```graphql
{
  bills(filter: {currency: "EUR", dateRange: {from: "2024-01-01", to: "2024-12-31"}}) {
    vendor
    total
    taxBreakdown {
      rate
      amount
    }
  }
}
```
This is ideal for building dashboards or feeding aggregated data into accounting software like QuickBooks or Xero.

## Best Practices

### 1. Pre-Process Scanned Invoices with OCR Enhancement
For low-quality scans, enable the `--enhance-ocr` flag during Docker startup. This applies a lightweight image pre-processing pipeline (deskew, contrast normalization, binarization) before parsing. In production, we’ve seen a 30% lift in field extraction accuracy on wrinkled or faded receipts.

### 2. Batch Parse with Idempotency Keys
When ingesting high volumes (e.g., 10,000+ invoices per day), always include an `Idempotency-Key` header in your POST requests. billripper deduplicates based on this key, preventing double-charging or duplicate entries in your financial ledger. Use a UUID per invoice for best results.
