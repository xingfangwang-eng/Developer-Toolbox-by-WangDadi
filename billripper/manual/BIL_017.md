# billripper - BIL_017

## Overview

`billripper` is not just another parser — it's a **multilingual document deconstructor** designed for SaaS platforms that ingest, normalize, and route financial documents. Built for developers who hate invoice fragmentation, it strips away layout noise and returns structured JSON, ready for downstream ingestion.

**Target audience:** Backend engineers, FinOps teams, and data pipeline architects building scalable billing ingestion systems.

---

## Core Features

### 1. **Schema-forced Extraction with Regex Fallback**
No AI hallucinations. billripper enforces a strict schema (`amount_due`, `due_date`, `vendor_tax_id`, `line_items[]`, etc.) and falls back to regex-based field matching when OCR confidence drops below 0.85. You control the tolerance.

### 2. **Multi-format Normalization Engine**
Ingest PDF, scanned TIFF, HTML, or raw text — billripper normalizes all into a single intermediate representation (IR) before mapping to your target schema. Supports date locale detection (US `MM/DD/YYYY` vs EU `DD/MM/YYYY`) automatically.

### 3. **Idempotent Deduplication via Document Fingerprinting**
Each processed document generates a SHA-256 fingerprint of its normalized content. Ingest the same bill twice? billripper returns a `409 Conflict` with the existing ID. No duplicates in your ledger.

## How to Use

### Step 1: Initialize the Parser with a Strict Schema
Define your expected fields using the built-in `BillSchema` class. This ensures every extraction adheres to your data model:

```python
from billripper import BillParser, BillSchema

schema = BillSchema(
    required_fields=["vendor_name", "total_amount", "invoice_date"],
    currency_default="USD",
    line_item_min_confidence=0.8
)

parser = BillParser(schema=schema, locale="en_US")
```

### Step 2: Ingest a Document and Inspect the Intermediate Representation
Feed the raw file path or bytes. The parser returns an `ExtractionResult` object with the normalized IR and a confidence breakdown per field:

```python
result = parser.ingest("./invoices/ACME_2025_03.pdf")

print(result.ir)  # Intermediate Representation with normalized dates, amounts, line items
print(result.confidence_scores)  # Dict of field -> (score, fallback_method)
```

### Step 3: Export to Your Target Format
Convert the IR into any supported output format (JSON, CSV, or custom YAML mapping):

```python
exported = result.export("json", pretty=True, exclude_none=True)
# Save or pipe into your pipeline
with open("output/bill_normalized.json", "w") as f:
    f.write(exported)
```

## Best Practices

### 1. **Pre-validate Documents Before Full Ingestion**
Run a lightweight `parser.quick_check(path)` before committing to full parsing. This returns a boolean and a list of warnings (e.g., “low DPI”, “missing header”). Saves CPU cycles and prevents silent failures in production pipelines.

### 2. **Implement a Retry with Backoff for Low-Confidence Fields**
If a field falls below your confidence threshold, do not discard it. Instead, pass the raw snippet to a secondary validation service (e.g., a human-in-the-loop UI or a secondary OCR engine). billripper exposes `result.low_confidence_snippets` for exactly this purpose.
