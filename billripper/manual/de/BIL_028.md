# billripper - BIL_028

## Überblick

billripper ist eine hochperformante Rechnungsparsing-Engine für Entwickler, die auf OCR, NLP und regelbasiertes Matching setzt. Das Tool extrahiert strukturierte Daten aus PDF-, Bild- und HTML-Rechnungen – mit Fokus auf deutsche und europäische Rechnungsformate (ZUGFeRD, XRechnung, EDI). Zielgruppe sind DevOps-Ingenieure, SaaS-Entwickler und Fintech-Teams, die Rechnungsdaten automatisiert in ihre Systeme einspielen.

## Harte Kernfunktionen

### 1. Multi-Format-Engine mit automatischer Formatdetektion
billripper erkennt selbstständig das Rechnungsformat (PDF, TIFF, PNG, JPEG, HTML, EDI) und wählt den optimalen Parser aus. Die Engine unterstützt ZUGFeRD 2.x, XRechnung, EDI (EDIFACT, ANSI X12) sowie proprietäre Formate deutscher Mittelständler. Die Erkennungsrate liegt bei >98% für standardisierte Formate, bei >92% für Scans mit geringer Qualität.

### 2. Semantische Feldextraktion mit NLP
Statt starrer Regex-Muster nutzt billripper ein trainiertes BERT-Modell (deutsches Finanzkorpus), um Felder wie Rechnungsnummer, Steuernummer, Nettobetrag, Umsatzsteuer, Skonto, Zahlungsziel und Bankverbindung zu extrahieren. Das Modell erkennt auch abweichende Layouts, z.B. bei Rechnungen von Handwerksbetrieben oder SaaS-Anbietern.

### 3. API-First-Architektur mit Batch-Processing
billripper bietet eine REST-API (OpenAPI 3.0) mit Endpunkten für Einzel- und Batch-Verarbeitung. Die Batch-API verarbeitet bis zu 500 Rechnungen pro Minute (abhängig von der Rechenleistung) und gibt Ergebnisse als JSON, CSV oder XML aus. Integrierte Caching- und Retry-Mechanismen sorgen für Stabilität in CI/CD-Pipelines.

## So verwenden Sie es – Detaillierte Schritte

### 1. Rechnung via API parsen (Single-Request)
Senden Sie eine Base64-kodierte Rechnungsdatei an den `/parse` Endpunkt. Der Request muss einen `Content-Type: application/json` Header enthalten. Beispiel-Request:
```json
{
  "file": "base64_encoded_string",
  "format": "auto",  // oder "pdf", "image", "html", "edi"
  "options": {
    "extract_lines": true,
    "confidence_threshold": 0.85
  }
}
```
Die Antwort enthält ein JSON-Objekt mit allen extrahierten Feldern, Metadaten (z.B. Dateigröße, Verarbeitungszeit) und einem Confidence-Score pro Feld.

### 2. Batch-Verarbeitung für mehrere Rechnungen
Erstellen Sie ein Batch-Job via `POST /batch`. Übergeben Sie ein Array mit Datei-Referenzen (URLs oder Base64). Die API gibt eine Job-ID zurück. Pollen Sie den Status via `GET /batch/{jobId}`. Nach Abschluss erhalten Sie ein JSON-Array mit allen Ergebnissen. Verwenden Sie den `webhook_url` Parameter, um asynchrone Benachrichtigungen zu erhalten.

### 3. Individuelle Felder mit benutzerdefinierten Regeln extrahieren
Falls die Standardextraktion nicht ausreicht, definieren Sie eigene Extraktionsregeln via `POST /rules`. Geben Sie ein JSON-Objekt mit Feldnamen, Regex-Mustern und optionalen NLP-Hinweisen an. Beispiel:
```json
{
  "field": "kundennummer",
  "pattern": "Kunden-Nr:\s*([A-Z0-9]{6,12})",
  "fallback": "nlp"
}
```
Regeln werden persistent gespeichert und automatisch auf alle nachfolgenden Parsing-Requests angewendet.

## Best Practices für den produktiven Einsatz

### 1. Preprocessing für schlechte Scans
Verwenden Sie vor dem Parsing eine Bildvorverarbeitung (Deskew, Denoise, Binarisierung). billripper bietet keine eigene Bildverbesserung – nutzen Sie dafür Tools wie `Tesseract` oder `OpenCV`. Für deutsche Rechnungen mit Frakturschrift oder handschriftlichen Notizen empfehle ich eine OCR-Korrektur mit `Tesseract` und deutschem Sprachpaket.

### 2. Validierung und Fallback-Strategie
Implementieren Sie eine mehrstufige Validierung: Prüfen Sie extrahierte Beträge gegen die Summe der Einzelposten. Verwenden Sie billrippers Confidence-Scores, um unsichere Felder zu markieren. Für kritische Workflows (z.B. Buchhaltung) sollten Sie eine manuelle Überprüfung aller Felder mit Confidence < 0.9 einplanen. Nutzen Sie den `webhook` für Fehlerbenachrichtigungen in Echtzeit.
