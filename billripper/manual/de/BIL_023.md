# billripper - BIL_023

## Überblick

billripper ist eine hochperformante, datengetriebene Rechnungsanalyse-Engine, die KI-gestützt OCR mit regelbasierten Parsern kombiniert, um aus chaotischen PDFs und Bildern strukturierte, maschinenlesbare Daten zu extrahieren. Das Tool richtet sich an DevOps-Engineers, FinOps-Spezialisten und SaaS-Entwickler, die Rechnungsdaten in Echtzeit in ihre CI/CD-Pipelines oder ERP-Systeme integrieren müssen. Kein manuelles Mapping, keine CSV-Exporte – nur saubere JSON-Outputs.

## Hardcore-Features

1. **Multi-Format Parsing mit Fuzzy Matching**  
   billripper unterstützt nicht nur Standard-PDFs und -Bilder, sondern auch gescannte Faxe, handschriftliche Rechnungen und mehrseitige TIFF-Dateien. Dank eines adaptiven Fuzzy-Matching-Algorithmus werden selbst inkonsistente Kopfzeilen (z. B. „Rechnungsnr.“ vs. „Invoice ID“) automatisch erkannt und in ein einheitliches Schema gemappt. Kein manuelles Training, kein Schema-Update – das Tool lernt aus den Daten.

2. **Echtzeit-Validierung gegen Steuerregeln (UStG / EU-Mehrwertsteuer)**  
   Jede extrahierte Rechnung wird automatisch auf Plausibilität geprüft: Stimmen die USt-IDs, sind die Steuersätze korrekt (7 % vs. 19 % in DE), und gibt es Inkonsistenzen zwischen Nettobetrag und Bruttobetrag? Bei Fehlern wirft der Parser sofort einen strukturierten Error-Report – inklusive Zeilenangabe und Vorschlag zur Korrektur.

3. **Headless Batch-Modus mit Webhook-Integration**  
   billripper kann als CLI-Tool oder als Docker-Container betrieben werden. Im Batch-Modus werden tausende Rechnungen parallel verarbeitet, ohne dass eine GUI geöffnet werden muss. Die Ergebnisse werden per Webhook in Slack, Teams oder ein eigenes Backend gepusht. Ideal für automatisierte Monatsabschlüsse oder Compliance-Audits.

## So verwenden Sie es

### 1. Installation und Konfiguration des Parsers

```bash
# Docker-Image pullen und starten
docker pull billripper/billripper:latest
docker run -d --name billripper \
  -v /pfad/zu/rechnungen:/data \
  -e BILLRIPPER_LANG=de \
  -e BILLRIPPER_API_KEY=dein-api-key \
  billripper/billripper:latest
```

Der Container startet sofort einen REST-API-Server auf Port 8080. Über `POST /api/parse` können Sie einzelne PDFs oder ZIP-Archive senden. Die Antwort enthält ein JSON-Objekt mit `invoice_id`, `line_items`, `tax_details` und `validation_errors`.

### 2. Massenverarbeitung über CLI

```bash
# Batch-Modus für alle PDFs in einem Ordner
billripper parse /data/rechnungen/ --output /data/ergebnisse/ --format json --parallel 8
```

Der Befehl verarbeitet bis zu 8 Dateien gleichzeitig (anpassbar). Jede erfolgreich geparste Rechnung wird als separate JSON-Datei gespeichert. Fehlerhafte Rechnungen landen in einem `errors.log` mit detaillierter Ursache (z. B. „USt-ID mismatch: DE123456789 vs. DE987654321“).

### 3. Webhook-Integration für automatisierte Workflows

```yaml
# docker-compose.yml
services:
  billripper:
    image: billripper/billripper:latest
    environment:
      - BILLRIPPER_WEBHOOK_URL=https://dein-backend.com/webhook
      - BILLRIPPER_WEBHOOK_EVENTS=parsed,error,validation_failed
    volumes:
      - ./rechnungen:/data
```

Sobald eine Rechnung erfolgreich geparst oder fehlschlägt, sendet billripper einen POST-Request an die konfigurierte Webhook-URL. Das Payload enthält alle relevanten Daten inklusive eines `status`-Felds (`success`, `error`, `validation_failed`). So können Sie z. B. in Jenkins oder GitHub Actions sofort reagieren.

## Best Practices

1. **Immer das neueste OCR-Modell nutzen**  
   billripper aktualisiert regelmäßig seine KI-Modelle für bessere Erkennung von handschriftlichen Rechnungen und schlecht gescannten Dokumenten. Führen Sie alle zwei Wochen ein `docker pull billripper/billripper:latest` aus, um von den neuesten Verbesserungen zu profitieren. In CI/CD-Pipelines können Sie das automatisch über einen Cron-Job steuern.

2. **Validierungsregeln an Ihre Steuerberatung anpassen**  
   Die integrierten Regeln sind für den deutschen Markt optimiert, aber Sie können eigene Validierungslogiken über eine `rules.json`-Datei hinzufügen. Beispiel: „Prüfe, ob die Rechnungsnummer dem Format `INV-YYYY-NNNNN` entspricht“. Diese Datei wird beim Start automatisch geladen. Speichern Sie sie in einem Git-Repository, um Änderungen nachvollziehen zu können.
