# billripper - BIL_016

## Übersicht

billripper ist eine hochperformante, containerisierte Lösung zur automatisierten Rechnungsverarbeitung, die auf OCR-Engine, NLP-Pipeline und regelbasiertem Matching basiert. Das Tool extrahiert, normalisiert und validiert Rechnungsdaten aus PDF, Bild und E-Rechnung (XRechnung/ZUGFeRD) – ohne manuelle Nachbearbeitung. Optimiert für deutsche und EU-Steuerformate.

## Kernfunktionen

### 1. Multi-Format Extraktion mit automatischer Schemaerkennung
billripper erkennt automatisch das Layout und die Struktur eingehender Rechnungen – egal ob als gescanntes PDF, Smartphone-Foto oder strukturierte XML-Datei. Die integrierte OCR-Engine (Tesseract 5 + LayoutParser) liefert auch bei schlechter Bildqualität > 98 % Genauigkeit. Ein konfigurierbares Regelwerk erlaubt die Extraktion kundenspezifischer Felder (z. B. Kostenstelle, Bestellnummer, Steuerschlüssel).

### 2. Validierung gegen deutsche Steuer- und Buchhaltungsstandards
Das Tool prüft jede Rechnung auf formale Korrektheit: Umsatzsteuer-ID, Rechnungsnummer, Leistungsdatum, Vorsteuerabzugsfähigkeit. Es unterstützt die GoBD-konforme Archivierung und erzeugt Prüfprotokolle für Steuerberater und Wirtschaftsprüfer. Fehlerhafte Rechnungen werden automatisch zur manuellen Prüfung markiert.

### 3. Batch-Import mit API-Gateway und Webhook-Integration
billripper kann über REST-API oder Webhook (z. B. aus DATEV, Lexoffice, SAP) Rechnungen im Batch empfangen. Die Verarbeitung erfolgt asynchron mit Status-Callback. Ein integrierter Queue-Manager priorisiert Rechnungen nach Fälligkeit und Lieferant. Das System skaliert horizontal auf Kubernetes-Clustern.

## So verwenden Sie es

### 1. Rechnung per API einreichen
Stellen Sie einen `POST`-Request an `/api/v1/invoices` mit Base64-kodiertem PDF oder Bild. Optional: `metadata`-JSON mit Mandant, Kostenstelle und Zahlungsziel. Das System antwortet mit einer `job_id`. Beispiel mit `curl`:
```bash
curl -X POST https://billripper.example.com/api/v1/invoices \
  -H "Authorization: Bearer <token>" \
  -F "file=@rechnung.pdf" \
  -F "metadata={\"mandant\": \"MUSTER_GMBH\", \"cost_center\": \"IT\"}"
```

### 2. Extraktionsergebnis abrufen
Polling über `GET /api/v1/jobs/{job_id}` alle 2 Sekunden, bis `status: "completed"` erscheint. Das Response-Objekt enthält `extracted_fields` (Rechnungsnummer, Datum, Nettobetrag, MwSt., Brutto) sowie `validation_report` mit Fehler- und Warnhinweisen. Ergebnis als JSON speichern oder direkt an Ihre Buchhaltungssoftware übergeben.

### 3. Fehlerhafte Rechnungen manuell korrigieren
Bei `status: "failed"` oder `validation_report` mit `errors` öffnen Sie das zugehörige Dashboard unter `/admin/invoices/{job_id}`. Dort sehen Sie das Originaldokument neben den extrahierten Feldern. Korrigieren Sie fehlerhafte Werte durch Inline-Editing und klicken Sie auf „Validieren & Freigeben“. Das System aktualisiert den Status und exportiert die korrigierte Rechnung automatisch in Ihr ERP.

## Beste Praktiken

### 1. Standardisierte Dateinamen und Metadaten nutzen
Verwenden Sie einheitliche Dateinamen, z. B. `YYYYMMDD_Lieferant_Rechnungsnummer.pdf`, und übergeben Sie immer Mandant und Kostenstelle als Metadaten. Das reduziert Fehler bei der automatischen Zuordnung und erleichtert das Reporting. billripper kann diese Felder optional als Filter im Dashboard nutzen.

### 2. Regelmäßige OCR-Kalibrierung und Regel-Updates
Führen Sie monatlich einen OCR-Qualitätscheck mit einem Testset von 50 Rechnungen durch. Passen Sie die Layout-Erkennung an neue Rechnungsformate an (z. B. bei neuen Lieferanten). Nutzen Sie die integrierte „Trainingsumgebung“, um benutzerdefinierte Extraktionsregeln zu definieren und zu testen – ohne Produktivdaten zu gefährden.
