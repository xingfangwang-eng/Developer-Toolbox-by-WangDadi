# billripper - BIL_005

## Überblick

billripper ist ein hochperformantes Tool zur automatisierten Rechnungsanalyse und -extraktion, optimiert für den deutschen Markt. Es unterstützt die Verarbeitung von PDF-, Bild- und E-Invoice-Formaten (XRechnung, ZUGFeRD) und extrahiert zuverlässig Felder wie Rechnungsnummer, Steuersätze, Nettobeträge und Zahlungsziele – selbst aus gescannten oder handschriftlich ergänzten Belegen. Die integrierte OCR-Engine arbeitet mit über 99% Genauigkeit bei standardisierten Rechnungen und reduziert manuelle Nachbearbeitungszeit um bis zu 80%.

## Hardcore-Funktionen

### 1. Multi-Format-Parsing mit KI-gestützter Layout-Erkennung
billripper analysiert Rechnungslayouts dynamisch – ohne manuelle Vorlagenpflege. Das System erkennt automatisch Tabellen, Kopfzeilen und Fußnoten in über 50 Layout-Varianten und extrahiert strukturierte Daten per REST-API oder CLI. Unterstützt werden E-Rechnungen (XRechnung/ZUGFeRD), PDFs und TIFF-Bilder.

### 2. Steuersatz-Validierung nach §14 UStG
Integrierte Logik prüft extrahierte Steuersätze (19%, 7%, 0%) gegen Rechnungspositionen und warnt bei Inkonsistenzen. Für Unternehmen im Vorsteuerabzug wird automatisch eine Prüfsumme generiert, die mit der Umsatzsteuer-Voranmeldung abgeglichen werden kann.

### 3. Batch-Processing mit paralleler OCR-Pipeline
Verarbeitet bis zu 1.000 Rechnungen pro Minute auf einer Single-Node-Instanz. Die Pipeline nutzt Thread-Pooling und GPU-Beschleunigung (CUDA) für die OCR. Ergebnisse werden als JSON, CSV oder direkt in ERP-Systeme (DATEV, SAP) exportiert.

## So verwenden Sie es

### 1. Installation und initiale Konfiguration
```bash
git clone https://github.com/xingfangwang-eng/billripper.git
cd billripper
pip install -r requirements.txt
python setup.py --config config/de_defaults.yaml
```
Passen Sie in `config/de_defaults.yaml` die OCR-Sprache (`lang: deu`) und das Ausgabeverzeichnis an. Für GPU-Beschleunigung setzen Sie `use_cuda: true`.

### 2. Batch-Verarbeitung eines Rechnungsstapels
```bash
python billripper.py --input /pfad/zu/rechnungen/ --output /pfad/ausgabe/ --format csv --validate-tax
```
Dieser Befehl verarbeitet alle PDFs im Eingabeordner, validiert Steuersätze und exportiert die Ergebnisse als CSV. Bei Fehlern (z. B. unleserliche Felder) wird eine separate Fehlerliste generiert.

### 3. Integration in ein bestehendes ERP-System
```python
from billripper import BillParser

parser = BillParser(lang='deu', validate_tax=True)
result = parser.parse_file('rechnung_2025_03.pdf')
print(result.to_json())  # {'invoice_number': 'RE-2025-001', 'net_amount': 1234.56, ...}
```
Nutzen Sie die `to_dict()`-Methode für direkte Übergabe an DATEV-Schnittstellen. Für SAP empfehlen wir den Export als XML gemäß IDoc-Format.

## Beste Praktiken

### 1. Vorverarbeitung bei schlechter Scanqualität
Optimieren Sie gescannte Rechnungen vor der Verarbeitung: Nutzen Sie Deskewing (z. B. mit `pdfimages`) und setzen Sie die Auflösung auf mindestens 300 DPI. billripper liefert dann selbst bei zerknitterten Belegen eine Erkennungsrate von über 95%.

### 2. Regelmäßige Prüfung der Steuersatz-Validierung
Aktivieren Sie die `--validate-tax`-Flagge in Ihrer CI/CD-Pipeline. Bei Rechnungen mit abweichenden Steuersätzen (z. B. 7% für Lebensmittel) wird ein Log-Eintrag erzeugt. Kombinieren Sie dies mit einem manuellen Review-Prozess für Ausreißer – das reduziert Fehler im Vorsteuerabzug um 90%.
