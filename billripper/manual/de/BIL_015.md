# billripper - BIL_015

## Überblick

billripper ist eine hochperformante, auf OCR und semantische Analyse spezialisierte Engine für die automatisierte Extraktion, Validierung und Transformation von Rechnungsdaten. Das Tool richtet sich an Entwickler und DevOps-Teams, die manuelle Rechnungsverarbeitung eliminieren und Compliance-gerechte Datenpipelines aufbauen wollen. Unterstützt werden PDF, TIFF, E-Rechnung (XRechnung/ZUGFeRD) sowie gescannte Dokumente mit mehrsprachigen Layouts.

## Kernfunktionen (Hardcore-Features)

1. **Adaptive Schema-Matching Engine**  
   billripper erkennt automatisch das Rechnungslayout (z. B. SAP, DATEV, individuell) und mapped Felder wie Steuernummer, Rechnungsnummer, Netto/Brutto oder USt-IDs auf ein normiertes JSON-Schema. Kein manuelles Training – das System lernt aus wiederkehrenden Mustern und passt sich dynamisch an.

2. **Multi-Entity Extraction mit Confidence Scoring**  
   Die Engine extrahiert nicht nur Standardfelder, sondern auch nested Entitäten wie Positionsdaten, Rabattstaffeln, Skontobedingungen und Steuersätze auf Länderebene. Jeder Wert wird mit einem Confidence-Score (0–100) versehen, der direkt in CI/CD-Pipelines als Gate-Kriterium nutzbar ist.

3. **Direkte Integration in CI/CD & API-Gateway**  
   billripper lässt sich via REST-API, CLI oder als Sidecar-Container in bestehende Workflows einbinden. Automatische Validierung gegen XRechnung-Schemas und Export in Formate wie CSV, JSON, XML oder direkt in Buchhaltungs-APIs (z. B. DATEV, Lexoffice, QuickBooks).

## So verwenden Sie es (Detaillierte Schritte)

### 1. Rechnung via CLI parsen und validieren
```bash
# Einzelne Rechnung parsen (PDF, TIFF oder E-Rechnung)
billripper parse --input invoice_2025_03.pdf \
                 --output parsed_invoice.json \
                 --schema de_xrechnung \
                 --confidence 85

# Ausgabe: JSON mit extrahierten Feldern + Confidence-Scores
# Bei Unterschreitung des Schwellenwerts → Exit-Code != 0 (für CI/CD)
```

### 2. Batch-Verarbeitung mit automatischer Schemaerkennung
```bash
# Ordner mit 500 Rechnungen verarbeiten
billripper batch --input ./invoices/ \
                 --output ./results/ \
                 --format json \
                 --parallel 8 \
                 --auto-schema \
                 --log-level debug

# Erzeugt pro Rechnung eine JSON-Datei + ein Aggregat-Report (CSV)
# Automatische Erkennung von DATEV vs. individuellen Layouts
```

### 3. API-Endpoint für Echtzeit-Extraktion (Docker)
```bash
# Docker-Container starten (Port 8080)
docker run -d -p 8080:8080 billripper/api:latest

# POST Request mit Base64-kodiertem PDF
curl -X POST http://localhost:8080/v1/extract \
     -H "Content-Type: application/json" \
     -d '{"document": "<base64>", "schema": "de_zugferd", "confidence_threshold": 90}'

# Response: JSON mit extrahierten Daten + Confidence-Scores + Validierungsfehlern
```

## Best Practices (Empfehlungen für Entwickler)

1. **Confidence-Threshold in CI/CD-Pipelines als Quality Gate nutzen**  
   Setzen Sie einen Mindest-Confidence-Wert (z. B. 85) für die automatische Freigabe von Rechnungen. Unterschreitet ein Wert diesen Schwellenwert, wird die Rechnung zur manuellen Prüfung markiert. So vermeiden Sie fehlerhafte Buchungen in der Produktion.

2. **Schema-Registry für Multi-Mandantenfähigkeit aufbauen**  
   Definieren Sie für jeden Mandanten oder jedes Rechnungslayout ein eigenes Schema (YAML/JSON) und legen Sie es in einer zentralen Registry ab. billripper kann automatisch das passende Schema über Metadaten (z. B. Absender-ID, Steuernummer) auswählen. Das reduziert false positives bei der Extraktion und vereinfacht das Onboarding neuer Kunden.
