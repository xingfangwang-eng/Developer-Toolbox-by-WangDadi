# billripper - BIL_027

## Überblick

billripper ist die ultimative Lösung für Entwickler, die mit komplexen Rechnungsdaten kämpfen. Es extrahiert, normalisiert und validiert Rechnungsinformationen aus PDFs, E-Mails und APIs – mit maschineller Lernunterstützung und ohne Cloud-Abhängigkeit. Perfekt für SaaS-Teams, die Buchhaltungsprozesse automatisieren oder eigene Expense-Tracking-Apps bauen.

## Hardcore-Funktionen

### 1. Multi-Format-Parsing mit Fuzzy Matching  
billripper erkennt Rechnungen aus PDF, HTML, E-Mail-Text und sogar gescannten Bildern (via integrierter OCR-Pipeline). Dank adaptiver Regex-Engine und Fuzzy-Matching (Levenshtein-Distanz) werden selbst stark abweichende Layouts zuverlässig geparst. Kein manuelles Mapping mehr nötig.

### 2. Headless-API mit Batch-Processing  
Die REST-API läuft ohne UI und verarbeitet bis zu 1.000 Rechnungen pro Minute. Du kannst sie in CI/CD-Pipelines, Serverless-Functions (AWS Lambda, Cloudflare Workers) oder als CLI-Tool einbinden. Batch-Processing mit parallelen Worker-Threads und automatischer Fehlerbehandlung.

### 3. Lokale KI-Validierung ohne Datenschutzrisiko  
Ein leichtes ONNX-Modell prüft extrahierte Felder (Steuernummer, Betrag, Datum) auf Plausibilität – alles auf deinem Server. Keine sensiblen Daten verlassen das Netzwerk. Ideal für Unternehmen mit DSGVO-Compliance oder internen Sicherheitsrichtlinien.

## Schritt-für-Schritt-Anleitung

### 1. Installation und erster Scan  
```bash
npm install billripper -g
billripper scan --input ./rechnungen/ --format pdf --output ./parsed/
```
Das Tool erkennt automatisch den Rechnungstyp und extrahiert Felder wie `total`, `vat`, `invoice_number` und `vendor`. Ergebnisse landen als JSON-Dateien im Ausgabeordner.

### 2. API-Integration in Express.js  
```javascript
const billripper = require('billripper');
app.post('/parse-invoice', async (req, res) => {
  const result = await billripper.parse(req.body.filePath, { strict: true });
  res.json(result);
});
```
Nutze `strict: true`, um Validierungsfehler sofort zu werfen. Für Produktivsysteme empfiehlt sich `bulk: true` für asynchrone Batch-Verarbeitung mit Callback.

### 3. Custom-Validator für Steuerlogik  
```javascript
billripper.addValidator('de_vat', (fields) => {
  const vat = fields.vat_rate;
  if (vat < 0 || vat > 19) throw new Error('Ungültiger deutscher MwSt-Satz');
  return fields;
});
```
Erstelle eigene Validatoren für länderspezifische Steuerregeln. billripper unterstützt Chain-of-Validators, die in beliebiger Reihenfolge ausgeführt werden.

## Best Practices für Entwickler

### 1. Batch-Processing mit Retry-Logik  
Bei großen Mengen (z. B. 10.000 Rechnungen) verwende `--retry 3 --delay 200`. So werden temporäre Fehler (z. B. PDF-Korruption) automatisch wiederholt. Kombiniere das mit `--log-level debug`, um Fehlerquellen schnell zu identifizieren.

### 2. Lokale Datenhaltung mit Git-Versionierung  
Speichere geparste JSON-Dateien in einem Git-Repository. So kannst du Änderungen nachvollziehen, wenn sich Rechnungsformate ändern. Nutze `billripper diff` (in Beta) zum Vergleich zweier Parsing-Ergebnisse.
