## Detaillierter Inhalt

### Was ist billripper?

billripper ist ein hochperformantes, modulares Rechnungsparsing-Framework für die automatisierte Extraktion, Validierung und Transformation von Rechnungsdaten. Es adressiert spezifisch die Pain Points deutscher Unternehmen: XRechnung-Konformität, ZUGFeRD-Formate und granulare Steuerlogik (USt.-IdNr.-Prüfung, Vorsteuerabzug). Statt generischer OCR liefert billripper eine domänenspezifische NLP-Pipeline, die Layout-Varianzen (PDF, EDI, E-Mail-Anhänge) robust normalisiert und direkt in ERP-Systeme (SAP, DATEV) einspeist.

### Kernfunktionen

1. **Adaptives Layout-Mining & Schema-Inferenz**  
   Statt starrer Template-Matching erkennt billripper automatisch Rechnungsstrukturen (Kopfzeilen, Tabellen, Fußnoten) durch graphbasierte Layout-Analyse. Es inferiert dynamisch Feldzuordnungen – selbst bei gescannten Dokumenten mit Schräglage oder Wasserzeichen – und reduziert manuelle Mapping-Arbeit um 85%.

2. **Multi-Format Engine mit XRechnung/ZUGFeRD-Nativsupport**  
   Die Engine parst nicht nur PDF und TIFF, sondern validiert und konvertiert direkt strukturierte Formate (XML, JSON, CSV) in das deutsche XRechnung-Profil (CIUS, Extension). Integrierte Prüfung gegen die KoSIT-Spezifikation – inklusive Fehler-Codes und Korrekturvorschlägen für die erneute Einreichung.

3. **Steueroptimierte Extraktion mit USt.-IdNr.-Validierung**  
   billripper extrahiert automatisch Steuerbeträge, Steuerschlüssel und Vorsteuerabzugslogik. Es führt eine Live-Validierung der USt.-IdNr. über die BZSt-Schnittstelle durch und markiert Abweichungen (z.B. falsche Steuersätze bei grenzüberschneidenden Leistungen). Das reduziert Prüfungsaufwand im Finanzwesen um 70%.

### So verwenden Sie es

1. **Initiale Datenaufbereitung & Konfiguration**  
   Starten Sie die Pipeline mit `billripper init --config ./config.yaml`. Definieren Sie im YAML-File Ihre ERP-Zielstruktur (Felder, Datentypen, Pflichtfelder) und die gewünschte Ausgabesprache (DE/EN). Beispiel:  
   ```yaml  
   output:  
     format: "xrechnung"  
     profile: "CIUS"  
     validate: true  
   ```  
   billripper liest dann alle PDFs im Verzeichnis `./invoices/` und normalisiert sie in ein internes JSON-Zwischenformat.

2. **Batch-Parsing mit automatischer Fehlerkorrektur**  
   Führen Sie `billripper parse --input ./invoices/ --output ./parsed/` aus. Das Tool extrahiert strukturierte Daten (Rechnungsnummer, Datum, Beträge, Steuersätze) und zeigt bei Fehlern (z.B. fehlende Pflichtfelder) sofort Korrekturvorschläge an. Nutzen Sie `--strict` für Compliance-Checks: Bei Abweichungen wird der Vorgang gestoppt und ein detaillierter Report generiert.

3. **Export & ERP-Integration**  
   Nach erfolgreichem Parsing exportieren Sie mit `billripper export --target sap --output ./export/`. billripper generiert automatisch SAP-kompatible IDocs oder DATEV-Importdateien. Für benutzerdefinierte Workflows nutzen Sie die REST-API:  
   ```bash  
   curl -X POST https://localhost:8080/api/v1/parse \  
        -H "Content-Type: multipart/form-data" \  
        -F "file=@invoice.pdf" \  
        -F "config=@config.yaml"  
   ```  
   Die Antwort enthält ein validiertes JSON-Objekt inklusive Prüfziffern.

### Beste Praktiken

1. **CI/CD-Pipeline mit automatischen Regressionstests**  
   Integrieren Sie billripper in Ihre CI/CD-Umgebung (z.B. GitHub Actions). Nutzen Sie `billripper test --suite ./test_cases/` mit einer Sammlung von Referenzrechnungen (fehlerhafte, korrekte, grenzwertige). So stellen Sie sicher, dass nach jedem Update (z.B. neue Layout-Variante) die Parsing-Qualität erhalten bleibt. Empfohlen: Führen Sie Tests mit 200+ Dokumenten durch, um Edge Cases abzudecken.

2. **Hybride Speicherstrategie für Audits**  
   Speichern Sie sowohl das Roh-PDF als auch das extrahierte JSON in einem versionierten Daten-Store (z.B. S3 mit Metadaten-Tagging). billripper erzeugt einen eindeutigen Hash pro Dokument (`--hash sha256`). Dies ermöglicht lückenlose Audits (GoBD-konform) und schnelle Wiederherstellung bei OCR-Fehlern. Kombinieren Sie dies mit einem internen Logging-System, das jede Änderung am Mapping nachverfolgt.
