# billripper - BIL_009

## Überblick

**billripper** ist ein hochperformantes Tool zur automatisierten Rechnungsanalyse, optimiert für das Parsen, Extrahieren und Strukturieren von Rechnungsdaten in heterogenen Formaten (PDF, EDI, XRechnung, ZUGFeRD). Zielgruppe sind SaaS-Entwickler, Fintech-Architekten und DevOps-Ingenieure, die eine schlanke, aber robuste Lösung für das Rechnungsmanagement in skalierbaren Microservice-Umgebungen benötigen.

## Hardcore-Funktionen

1. **Multi-Format-Engine mit adaptivem OCR-Fallback**  
   billripper parst native PDF-Strukturen (Text, Tabellen, eingebettete XML) und fallbackt automatisch auf Tesseract-basiertes OCR bei gescannten oder bildbasierten Rechnungen. Der Parser erkennt automatisch Layout-Varianten (z. B. deutsche, österreichische, schweizerische Formate) und liefert strukturierte JSON-Outputs mit Feld-Mapping (Betrag, Steuer, Rechnungsnummer, Datum, Lieferant).

2. **Rule-basiertes Validierungs-Framework mit benutzerdefinierten Checks**  
   Definiere eigene Validierungsregeln (z. B. Prüfung auf korrekte Umsatzsteuer-IDs, Dubletten-Erkennung, Plausibilitätschecks). Das Framework unterstützt sowohl YAML-basierte Regeln als auch JavaScript-Expressions für komplexe Logik – ideal für Compliance-Workflows in der Buchhaltung.

3. **Headless Batch-Processing mit Streaming-API**  
   Verarbeite Tausende Rechnungen in einem Durchlauf ohne Speicherüberlauf. Die Streaming-API liefert Ergebnisse als Event-Stream (NDJSON) – perfekt für Integrationen in Kafka, RabbitMQ oder AWS SQS. Inklusive integriertem Rate-Limiting und Retry-Mechanismus für instabile Quellen.

## So verwenden Sie es – Detaillierte Anleitung

### 1. Installation und Konfiguration über CLI

```bash
# Global installieren (Node.js 18+ erforderlich)
npm install -g billripper

# Konfigurationsdatei initialisieren
billripper init --config ./billripper.yml

# Beispiel-Konfiguration für deutsche Rechnungen (ZUGFeRD)
billripper config set format.de.zugferd=true
billripper config set output.dir=/data/parsed
```

Nach der Initialisierung erzeugt billripper eine `.billripperrc`-Datei mit Standardparametern (z. B. OCR-Sprache `deu`, Timeout 30s). Du kannst sie direkt editieren oder per CLI überschreiben.

### 2. Batch-Verarbeitung eines Rechnungsordners

```bash
# Alle PDFs im Ordner /invoices parsen
billripper run --input /invoices --output /parsed --format json

# Mit Validierung und Dubletten-Check
billripper run --input /invoices --validate --dedup --log-level debug
```

Der Befehl gibt einen Fortschrittsbalken aus und speichert die Ergebnisse als `.json`-Dateien. Bei Fehlern wird eine separate `errors.log` erzeugt. Du kannst auch `--stream` aktivieren, um Ergebnisse direkt an eine API zu senden.

### 3. Integration als Node.js-Modul (API-Nutzung)

```javascript
import { BillRipper } from 'billripper';

const ripper = new BillRipper({
  ocr: { language: 'deu', engine: 'tesseract' },
  validation: {
    rules: [
      { field: 'total', operator: 'gt', value: 0 },
      { field: 'vat_id', pattern: '^DE[0-9]{9}$' }
    ]
  }
});

// Einzelne Rechnung parsen
const result = await ripper.parse('./rechnung_2024_08.pdf');
console.log(result.data); // { invoiceNumber, date, total, tax, ... }

// Batch-Verarbeitung mit Stream
const stream = ripper.parseStream('./invoices/*.pdf');
stream.on('data', (chunk) => console.log(chunk));
```

Die API ist vollständig typisiert (TypeScript) und unterstützt asynchrone Iteration. Für Microservices empfehle ich, den Parser in einem Worker-Thread zu isolieren.

## Best Practices für den Produktiveinsatz

1. **OCR-Qualität vorab testen**  
   Bei gescannten Rechnungen (insbesondere mit handschriftlichen Notizen) solltest du vor dem Batch-Processing eine Stichprobe von 5–10 PDFs mit `--dry-run` testen. Nutze `--ocr-preview`, um die erkannten Textblöcke visuell zu prüfen. Falls die Erkennung schwächelt, passe die OCR-Sprache oder DPI-Einstellungen an (`--ocr-dpi 300`).

2. **Validierungsregeln in CI/CD integrieren**  
   Definiere deine Validierungsregeln als YAML-Datei im Repository. Führe billripper im CI-Pipeline (z. B. GitHub Actions) aus, um Rechnungsdaten automatisch zu prüfen, bevor sie in die Buchhaltungssoftware importiert werden. Beispiel:  
   ```yaml  
   - name: Validate invoices  
     run: billripper run --input invoices/ --validate --fail-fast  
   ```  
   So verhinderst du, dass fehlerhafte Rechnungen in dein ERP gelangen.
