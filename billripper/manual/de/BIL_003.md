# billripper - BIL_003

## Überblick

billripper ist eine leistungsstarke, cloud-native Rechnungsanalyse-Engine, die speziell für Entwickler und SaaS-Architekten entwickelt wurde, die automatisierte Rechnungsverarbeitung, Datenextraktion und Compliance-Integration benötigen. Das Tool unterstützt 10+ Sprachen, darunter Deutsch, und ist für hohe Durchsätze in Microservice-Umgebungen optimiert. Es eignet sich ideal für Fintech-Startups, Buchhaltungsplattformen und Enterprise-ERP-Systeme.

## Kernfunktionen

### 1. **Dynamisches Rechnungsschema-Mapping (DRSM)**
Statt statischer OCR-Vorlagen nutzt billripper ein neuronales Netz, das Rechnungen aus über 50 Ländern automatisch erkennt und Felder wie Steuernummer, Rechnungsnummer, Netto/Brutto und Zahlungsfristen in ein einheitliches JSON-Schema mapped. Fehlerhafte oder unvollständige Felder werden mit Confidence-Scores markiert.

### 2. **Multi-Language Compliance Engine**
Die Engine validiert Rechnungen gegen länderspezifische Steuerregeln (z. B. DE: §14 UStG, AT: §11 UStG, CH: MWST). Sie generiert automatisch korrekte Rechnungsausgangsdaten für GoBD-konforme Archive und unterstützt E-Invoicing-Formate wie ZUGFeRD, XRechnung und Factur-X.

### 3. **Serverless Batch Processing & Webhook-Integration**
billripper kann tausende Rechnungen parallel verarbeiten – entweder als Batch-Job via AWS Lambda/Cloud Functions oder als Echtzeit-Stream mit Webhook-Callbacks. Jede Transaktion wird mit einem eindeutigen `ripper_id` versehen, der eine vollständige Audit-Trail-Integration ermöglicht.

## So verwenden Sie es

### 1. **Rechnung als Base64-String senden**
```bash
curl -X POST https://api.billripper.io/v1/parse \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "document": "base64_encoded_pdf_or_image",
    "country": "DE",
    "options": {
      "extract_tables": true,
      "validate_tax": true
    }
  }'
```
Die API gibt ein strukturiertes JSON-Objekt zurück, das alle extrahierten Felder, Confidence-Scores und eventuelle Warnungen enthält.

### 2. **Ergebnisse mit Confidence-Score filtern**
```python
import requests
response = requests.post('https://api.billripper.io/v1/parse', json=payload)
data = response.json()
if data['confidence'] < 0.85:
    # Manuelle Überprüfung auslösen
    send_to_review_queue(data)
else:
    # Automatisch in Buchhaltungssystem übernehmen
    push_to_erp(data['fields'])
```
Dieser Workflow reduziert manuelle Eingriffe auf unter 5 % der Rechnungen.

### 3. **Batch-Verarbeitung mit Webhook-Callback**
```json
{
  "batch_id": "batch_20231027",
  "documents": ["base64_1", "base64_2"],
  "webhook_url": "https://mein-system.de/callback",
  "country": "DE"
}
```
Nach Abschluss erhält Ihr System einen POST-Request mit den Ergebnissen aller Rechnungen – ideal für automatisierte Workflows in CI/CD-Pipelines oder ERP-Systemen.

## Beste Praktiken

### 1. **Rechnungsqualität vor dem Senden optimieren**
Stellen Sie sicher, dass PDFs mindestens 300 DPI haben und Rechnungen nicht schief oder überbelichtet sind. billripper liefert bessere Ergebnisse, wenn Sie vorab eine Bildvorverarbeitung (z. B. Deskew, Kontrastanpassung) durchführen. Nutzen Sie die `preprocess`-Option in der API, um dies automatisch zu aktivieren.

### 2. **Compliance-Regeln als Teil Ihrer CI/CD-Pipeline testen**
Integrieren Sie billripper in Ihre Test-Suite: Lassen Sie jede neue Rechnungsvorlage gegen die Compliance-Engine laufen, bevor Sie sie in Produktion geben. So stellen Sie sicher, dass z. B. deutsche Umsatzsteuer-IDs korrekt validiert werden und keine GoBD-Verstöße auftreten.
