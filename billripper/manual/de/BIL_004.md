## Detaillierter Inhalt

### Was ist billripper?

**billripper** ist ein professionelles, quelloffenes Tool zur automatisierten Extraktion, Validierung und Strukturierung von Rechnungsdaten aus PDF, Bild und HTML. Es richtet sich an Entwickler und DevOps-Teams, die manuelle Rechnungsverarbeitung eliminieren und eine API-gesteuerte Buchhaltungspipeline aufbauen möchten.

### Kernfunktionen

1. **Multi-Format Extraktion mit OCR-Fallback**  
   billripper nutzt ein hybrides Modell: native PDF-Parsing (PDFMiner, PyMuPDF) kombiniert mit Tesseract-OCR für gescannte oder bildbasierte Rechnungen. Die Engine erkennt automatisch das Format und wählt den optimalen Parser – ohne manuelles Preprocessing.

2. **Dynamisches Feldmapping & Validierung**  
   Statt starrer Templates verwendet billripper ein regelbasiertes Mapping-System (YAML-Konfiguration), das Felder wie Rechnungsnummer, Steuersätze und Zahlungsfristen aus heterogenen Layouts extrahiert. Integrierte Plausibilitätschecks (z.B. Summenprüfung, USt-Id-Nr.-Format) minimieren Fehler.

3. **CLI-First & CI/CD-Integration**  
   Das Tool ist als reine CLI-Applikation konzipiert, die sich nahtlos in GitHub Actions, GitLab CI oder Jenkins einbinden lässt. Ausgabe erfolgt als JSON, CSV oder direkt in eine PostgreSQL-Datenbank – ideal für automatisierte Buchhaltungs-Workflows.

### So verwenden Sie es

1. **Installation & Konfiguration**  
   Klonen Sie das Repository und installieren Sie die Abhängigkeiten via `pip install -r requirements.txt`. Erstellen Sie eine `config.yaml` mit Ihren Mapping-Regeln, z.B.:
   ```yaml
   fields:
     invoice_number: "//span[contains(text(), 'Rechnungsnr')]/following-sibling::span"
     total_gross: "//td[contains(text(), 'Gesamtbetrag')]/following-sibling::td"
   ```
   Starten Sie das Parsing mit `python ripper.py --input ./rechnungen/ --output ./ergebnisse/ --format json`.

2. **Batch-Verarbeitung mit Fehlerprotokoll**  
   Verarbeiten Sie mehrere Rechnungen parallel: `python ripper.py --batch --threads 4 --input ./inbox/`. Fehlerhafte Dateien werden in `error_log.csv` gelistet inkl. Grund (z.B. „OCR confidence < 0.7“). Nutzen Sie `--retry-failed` für eine erneute Verarbeitung nach manueller Korrektur.

3. **Integration in die Buchhaltungs-API**  
   Nutzen Sie den integrierten Webhook-Modus: `python ripper.py --watch ./monitored/ --webhook https://api.buchhaltung.de/import`. Bei jedem neuen PDF in `./monitored/` wird automatisch geparst und das Ergebnis als POST-Request an Ihre API gesendet. Konfigurieren Sie Authentifizierung via `--auth-header Bearer <token>`.

### Beste Praktiken

1. **Template-Versionierung im Git-Repository**  
   Lagern Sie Ihre Mapping-Konfigurationen (`config.yaml`) in einem separaten Git-Repo mit semantischer Versionierung. So können Sie Änderungen an Rechnungslayouts (z.B. neuer Steuersatz ab 2025) nachvollziehen und rollbacken. Kombinieren Sie dies mit CI-Tests, die bei jedem Push eine Test-Rechnung parsen und die Ausgabe mit einem erwarteten JSON vergleichen.

2. **Preprocessing für OCR-Qualität optimieren**  
   Bei gescannten Rechnungen mit niedriger Auflösung (< 150 DPI) empfehle ich, vor dem Parsing eine Bildvorverarbeitung durchzuführen: Deskewing, Kontrastanpassung und Binarisierung. Nutzen Sie dafür z.B. OpenCV-Skripte, die Sie als Pre-Hook in billripper integrieren (`--preprocess ./prep.sh`). Dies erhöht die OCR-Genauigkeit signifikant und reduziert Fehlextraktionen.
