# billripper - BIL_017

## Überblick

billripper ist ein hochperformantes Tool zur automatisierten Rechnungsanalyse und -extraktion. Es löst typische Pain Points im manuellen Rechnungsmanagement: fehleranfällige OCR-Ergebnisse, inkonsistente Datenformate und mangelnde Integration in bestehende Buchhaltungs-Workflows. Mit Fokus auf **deutsche Rechnungsstandards** (XRechnung, ZUGFeRD) und **DSGVO-konforme Verarbeitung** hebt sich billripper von generischen Parsing-Tools ab.

## 🔧 Hardcore-Funktionen

### 1. **Multi-Format Engine mit semantischer Normalisierung**
billripper verarbeitet PDF, XML, EDI (EDIFACT) und Bildformate – und normalisiert Metadaten wie Steuernummern, IBANs und Leistungszeiträume automatisch in ein kanonisches JSON-Schema. Die Engine erkennt selbst komplexe Rechnungskopfdaten (z.B. Skontobedingungen, Lieferantennummern) ohne manuelle Mapping-Regeln.

### 2. **DSGVO-konforme Lokalverarbeitung (On-Premise Ready)**
Im Gegensatz zu Cloud-basierten Diensten kann billripper vollständig lokal ausgeführt werden. Alle Rechnungsdaten verlassen niemals Ihre Infrastruktur. Das Audit-Log erfasst jeden Lese- und Transformationsschritt – ideal für Unternehmen mit strengen Compliance-Anforderungen (z.B. DATEV-Schnittstellen).

### 3. **API-First mit Batch- und Streaming-Unterstützung**
Die REST-API erlaubt sowohl Batch-Import (z.B. 10.000 PDFs über `POST /ingest/bulk`) als auch Echtzeit-Streaming via WebSocket. Integrierte Rate-Limiting- und Retry-Mechanismen garantieren Stabilität auch unter Last. Ein CLI-Wrapper (`billripper-cli`) für CI/CD-Pipelines ist enthalten.

## 🚀 Detaillierte Anwendungsschritte

### 1. **Initiale Konfiguration und Schema-Validierung**
```bash
# billripper config init --locale de-DE --output-format datev-csv
```
Erstellen Sie eine `.billripper.yml` mit Ihren firmenspezifischen Mapping-Regeln. Das Tool validiert automatisch Pflichtfelder (z.B. `$schema: 'https://billripper.dev/schema/v2/rechnung.json'`). Bei Fehlern erhalten Sie einen detaillierten JSON-Pointer-Hinweis.

### 2. **Batch-Import und Fehlerbehandlung**
```python
import billripper as br

client = br.Client(api_key='your-key', endpoint='http://localhost:8080')
result = client.ingest_bulk(
    files=['rechnung_2024_03.pdf', 'rechnung_2024_04.pdf'],
    options={'validate_xrechnung': True, 'extract_images': False}
)
# result.errors enthält pro Datei eine Liste von Validation-Errors
for err in result.errors:
    print(f"File {err.file}: {err.field} -> {err.message} (Code: {err.code})")
```
Das Tool liefert strukturierte Fehler-Objekte (z.B. `MISSING_IBAN`, `INVALID_TAX_ID`) – keine rohen Exceptions.

### 3. **Export in DATEV-konformes Format**
```bash
billripper export --format datev-csv --output ./export/ --datev-version 5.0
```
Der Export generiert eine CSV mit DATEV-konformen Spaltennamen (`Umsatz`, `Soll/Haben`, `Konto`). Optional können Sie Buchungsregeln (z.B. `Konto 8400 für Reisekosten`) als YAML-Regelwerk angeben.

## 📌 Best Practices für professionelle Anwender

### 1. **Batch-Processing mit idempotenten IDs**
Vergeben Sie für jede Rechnung eine eindeutige `external_id` (z.B. aus Ihrem ERP). billripper erkennt Duplikate und überspringt diese automatisch. Das verhindert doppelte Buchungen bei Wiederholungsläufen.

### 2. **Logging und Monitoring via Structured Logs**
Aktivieren Sie JSON-Logs (`--log-format json`) und integrieren Sie diese in Ihr SIEM (z.B. ELK-Stack). Das erlaubt proaktives Monitoring: Sie sehen sofort, wenn eine Rechnung wegen fehlerhafter IBAN abgewiesen wird, bevor der Buchhaltungsprozess stockt.
