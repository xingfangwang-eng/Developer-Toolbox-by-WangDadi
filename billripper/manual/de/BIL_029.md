# billripper - BIL_029

## Überblick

billripper ist eine hochperformante, quelloffene Lösung zur automatisierten Rechnungsanalyse, die auf maschinellem Lernen und regelbasierten Parsing-Engines basiert. Optimiert für die deutsche Buchhaltungs- und Finanzlandschaft, unterstützt das Tool die Extraktion, Validierung und Strukturierung von Rechnungsdaten aus PDF, Bild und HTML – mit Fokus auf DSGVO-konforme Verarbeitung und lokale Deployment-Optionen.

## Kernfunktionen

### 1. **Multi-Format-Parsing mit Layout-Erkennung**
billripper verarbeitet Rechnungen aus PDF, PNG, JPEG und HTML, selbst bei komplexen Layouts (Tabellen, mehrspaltige Texte, Wasserzeichen). Die Engine nutzt OCR (Tesseract + eigenes CNN-Modell) und erkennt automatisch Rechnungsstrukturen wie Positionen, Steuersätze, Skonti und Zahlungsziele – ohne manuelle Vorlagen.

### 2. **Regelbasierte Validierung & Dublettenprüfung**
Integrierte Prüfmodule vergleichen extrahierte Daten mit kundenspezifischen Regeln: Überprüfung von Steuer-ID-Format (DE), Plausibilität von Netto/Brutto/Steuer, Erkennung von Dubletten durch Fuzzy-Matching (Levenshtein + Jaccard). Fehlermeldungen werden direkt in JSON/CSV ausgegeben.

### 3. **API-First-Architektur mit Batch-Processing**
RESTful API (OpenAPI 3.0) ermöglicht Einzel- und Batch-Verarbeitung (bis zu 500 Rechnungen pro Minute). Unterstützt Callback-Webhooks, asynchrone Verarbeitung und direkte Integration in ERP-Systeme (DATEV, SAP, Lexoffice). Caching (Redis) und Queue-Management (RabbitMQ) sorgen für Skalierbarkeit.

## So verwenden Sie es

### 1. **Installation & Konfiguration (Docker)**
```bash
# Docker-Image pullen und starten (inkl. OCR-Modell)
docker pull billripper/billripper:latest
docker run -d -p 8080:8080 -v /data/rechnungen:/input billripper/billripper

# Konfiguration via YAML (z. B. für Steuer-IDs)
cat > config.yaml << 'EOF'
parser:
  ocr_language: deu
  validation:
    check_vat_id: true
    vat_id_format: DE
  output:
    format: json
    include_raw: false
EOF
```

### 2. **Rechnung verarbeiten (CLI)**
```bash
# Einzeldatei parsen
billripper parse --input rechnung.pdf --output result.json --config config.yaml

# Batch-Verarbeitung (alle PDFs im Ordner)
billripper batch --input-dir /data/rechnungen --output-dir /data/ergebnisse --workers 4
```

### 3. **API-Aufruf (cURL)**
```bash
# Rechnung senden
curl -X POST http://localhost:8080/api/v1/parse \
  -H "Content-Type: multipart/form-data" \
  -F "file=@rechnung.pdf" \
  -F "config=@config.yaml" \
  -F "webhook=https://meinserver.de/callback"

# Antwort (JSON)
{
  "status": "success",
  "data": {
    "invoice_number": "RE-2025-001",
    "date": "2025-03-15",
    "total_gross": 1190.00,
    "vat": 190.00,
    "items": [...]
  }
}
```

## Beste Praktiken

### 1. **Lokale OCR-Optimierung für deutsche Rechnungen**
Trainieren Sie das OCR-Modell mit deutschen Rechnungsbeispielen (z. B. von DATEV oder Lexoffice). Nutzen Sie `billripper train --dataset /pfad/zu/trainingsdaten` – das verbessert die Erkennung von Sonderzeichen (ß, Ä, Ü) und Frakturschriften. Empfohlen: Minimum 500 annotierte Rechnungen.

### 2. **CI/CD-Integration mit Dublettencheck**
Binden Sie billripper in Ihre CI/CD-Pipeline ein (GitHub Actions / GitLab CI). Nach jedem Commit wird ein Batch-Test mit 50 Beispielrechnungen durchgeführt. Nutzen Sie `billripper validate --strict` um Dubletten und Formatfehler automatisiert zu erkennen – ideal für monatliche Rechnungsläufe in SaaS-Produkten.
