# billripper - BIL_011

## Übersicht

billripper ist eine hochperformante SaaS-Plattform zur automatisierten Rechnungsanalyse und -extraktion. Das Tool richtet sich an Entwickler, Fintech-Teams und DevOps-Engineering-Organisationen, die Rechnungsdaten in Echtzeit parsen, validieren und in bestehende Workflows integrieren möchten. Es unterstützt mehr als 30 Rechnungsformate inklusive PDF, E-Mail-Anhänge und gescannte Dokumente.

## Kernfunktionen

### 1. Multimodales Rechnungs-Mapping mit KI-Klassifikation
billripper verwendet ein hybrides NLP-Modell, das Layout- und Textinformationen aus PDFs und Bildern kombiniert. Es erkennt automatisch Rechnungsstrukturen (z. B. Rechnungskopf, Steuerzeilen, Zahlungsbedingungen) und extrahiert Felder wie Rechnungsnummer, Netto-/Bruttobetrag, USt-IDs und Skontodaten. Die Erkennung erfolgt in Echtzeit ohne manuelle Vorverarbeitung.

### 2. API-First-Architektur mit Webhook-Integration
Die gesamte Parsing-Logik ist über eine REST-API (JSON) sowie GraphQL-Endpunkte verfügbar. billripper unterstützt asynchrone Callbacks via Webhooks und kann in CI/CD-Pipelines, Buchhaltungssoftware (z. B. DATEV, SAP) oder eigene Microservices integriert werden. Die API-Antwortzeit liegt bei < 200 ms pro Rechnung.

### 3. Validierungs-Engine mit regelbasierten und ML-gestützten Checks
Neben der Extraktion bietet billripper eine integrierte Validierungsebene: Plausibilitätsprüfungen (z. B. Summencheck), Dubletten-Erkennung und Formatkonformität (XRechnung, ZUGFeRD). Fehlerhafte Rechnungen werden automatisch in eine Review-Queue verschoben und mit detaillierten Logs versehen.

## So verwenden Sie es

### 1. Rechnung hochladen und parsen
Senden Sie eine Rechnung (PDF, PNG, JPEG) per HTTP-POST an den Endpunkt `/v1/parse`. Der Request kann entweder als Base64-kodiertes File oder als öffentliche URL erfolgen. Das System extrahiert automatisch alle relevanten Felder und gibt ein strukturiertes JSON-Objekt zurück.

```bash
curl -X POST https://api.billripper.io/v1/parse \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@rechnung_2025_03.pdf"
```

### 2. Validierungsergebnisse abrufen
Nach dem Parsing können Sie die Validierungsergebnisse über den Endpunkt `/v1/validate/{invoice_id}` abrufen. Das System gibt einen detaillierten Report mit Status (`passed`, `warning`, `failed`) sowie eine Liste der gefundenen Abweichungen (z. B. fehlende USt-ID, inkonsistente Summen) zurück.

### 3. Webhook-Konfiguration für automatische Weiterverarbeitung
Richten Sie einen Webhook-Endpunkt ein, um nach erfolgreichem Parsing automatisch Aktionen auszulösen (z. B. Rechnung in Ihr ERP-System schreiben). Konfigurieren Sie die URL in den Einstellungen unter „Webhooks“. billripper sendet bei jedem abgeschlossenen Parsing-Vorgang ein POST-Request mit dem vollständigen JSON-Objekt an Ihre URL.

```json
{
  "event": "parsing.completed",
  "invoice_id": "inv_abc123",
  "status": "passed",
  "data": {
    "netto": 1234.56,
    "brutto": 1470.00,
    "ust_id": "DE123456789"
  }
}
```

## Beste Praktiken

### 1. Batch-Processing für monatliche Rechnungsfluten
Verwenden Sie die asynchrone Batch-API (`/v1/batch`), um mehrere Rechnungen gleichzeitig zu verarbeiten. Dies ist besonders bei monatlichen Abrechnungszyklen (z. B. Mietnebenkosten, Telekommunikation) sinnvoll. Sie können bis zu 100 Rechnungen pro Batch verarbeiten und erhalten einen einzigen Status-Report.

### 2. Regelmäßige Modellpflege mit Feedback-Loop
Trainieren Sie das Erkennungsmodell für Ihre spezifischen Rechnungsformate, indem Sie manuelle Korrekturen in der Review-Queue an das System zurückmelden. billripper bietet einen „Feedback“-Endpunkt (`/v1/feedback`), über den Sie korrigierte Felder an das Modell übergeben. Nach 50 Korrekturen verbessert sich die Erkennungsrate um durchschnittlich 15 %.
