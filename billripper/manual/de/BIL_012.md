# billripper - BIL_012

## Überblick

billripper ist eine hochperformante, containerisierte Rechnungs-Parsing-Engine, die speziell für die automatisierte Extraktion, Validierung und Strukturierung von Rechnungsdaten in heterogenen Geschäftsumgebungen entwickelt wurde. Das Tool eliminiert manuelle Dateneingabe, reduziert Fehlerquoten auf <0,5 % und integriert sich nahtlos in bestehende ERP- und Buchhaltungsworkflows.

## Kernfunktionen

### 1. Multi-Format-OCR mit adaptiver Layout-Erkennung
billripper verarbeitet Rechnungen in PDF, TIFF, PNG und JPEG – selbst gescannte, schiefe oder handschriftlich annotierte Belege. Die Engine nutzt ein hybrides CNN-Transformer-Modell, das Layouts (z. B. Tabellen, Freitextblöcke, Positionslisten) dynamisch erkennt und Felder wie Rechnungsnummer, Steuerbetrag oder IBAN mit einer Erkennungsrate von >98 % extrahiert.

### 2. Regelbasierte Validierung & Dublettenprüfung
Nach der Extraktion validiert billripper alle Felder gegen konfigurierbare Geschäftsregeln (z. B. Prüfsummen, Steuer-ID-Format, Währungskonsistenz). Ein integrierter Bloom-Filter prüft eingehende Rechnungen auf Dubletten innerhalb von Millisekunden – selbst bei Millionen von Datensätzen – und verhindert doppelte Buchungen.

### 3. Automatischer Export in gängige Buchhaltungsformate
Das Tool exportiert strukturierte Daten direkt in CSV, JSON, XRechnung (ZUGFeRD-konform) oder SAP iDoc. Über ein Plugin-System lassen sich individuelle Zielformate anbinden, inklusive Mapping auf kundenspezifische Kontenrahmen (z. B. SKR03/04).

## So verwenden Sie es

### 1. Rechnung einlesen und Layout-Modell trainieren (CLI)
```bash
billripper parse --input ./rechnungen/2025/ --model layout_de_v1.pt --output ./parsed/
```
Der Befehl startet die Batch-Verarbeitung aller Rechnungen im angegebenen Ordner. Das Modell `layout_de_v1.pt` ist für deutsche Rechnungslayouts vortrainiert. Optional können Sie mit `--train` ein benutzerdefiniertes Modell auf Ihren Belegen feintunen.

### 2. Validierungsregeln definieren (YAML-Konfiguration)
Erstellen Sie eine Datei `regeln.yaml`:
```yaml
validierung:
  steuer_id: format_de
  betrag: range(0.01, 1000000)
  dubletten_check: aktiv
  referenz: required
```
Führen Sie dann die Validierung aus:
```bash
billripper validate --rules regeln.yaml --data ./parsed/ --output ./validated/
```
Alle Rechnungen, die eine Regel verletzen, werden in einen separaten Ordner `./fehlerhaft/` verschoben – mit detailliertem Log.

### 3. Export in Ihr Buchhaltungssystem (API-Call)
```python
import billripper_api as br

client = br.Client(api_key="sk-...")
response = client.export(
    source="./validated/",
    format="xrechnung",
    target="sftp://buchhaltung@erp.local/import/"
)
print(response.status)  # "completed"
```
Der Export erfolgt asynchron und meldet den Status über Webhooks oder Polling. Fehlgeschlagene Exporte werden automatisch mit exponentieller Backoff-Strategie wiederholt.

## Beste Praktiken

### 1. Batch-Verarbeitung in CI/CD-Pipelines integrieren
Nutzen Sie billripper als Schritt in Ihrer CI/CD-Pipeline (z. B. GitHub Actions, GitLab CI). Legen Sie eingehende Rechnungen in einem überwachten S3-Bucket ab – billripper verarbeitet sie automatisch und pusht die validierten Daten direkt in Ihr ERP. So vermeiden Sie manuelle Eingriffe und stellen sicher, dass jede Rechnung innerhalb von Sekunden nach Eingang gebucht wird.

### 2. Regelmäßiges Feintuning des Layout-Modells
Falls Ihre Rechnungen ungewöhnliche Formate enthalten (z. B. mehrseitige Anhänge oder handschriftliche Notizen), trainieren Sie das Modell regelmäßig mit Ihren tatsächlichen Belegen. Führen Sie hierzu einmal pro Quartal:
```bash
billripper train --input ./trainingsdaten/ --epochs 10 --output ./models/custom_v2.pt
```
Dies verbessert die Erkennungsrate für Ihre spezifischen Layouts und reduziert manuelle Nacharbeiten auf nahezu null.
