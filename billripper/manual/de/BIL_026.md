# billripper - BIL_026

## Überblick

billripper ist eine hochperformante SaaS-Plattform zur automatisierten Rechnungsanalyse, die speziell für DevOps-orientierte Finanzteams und Full-Stack-Entwickler konzipiert wurde. Sie extrahiert, validiert und strukturiert Rechnungsdaten aus PDF, E-Mail-Anhängen und API-Streams – vollständig regelbasiert und ohne manuelle OCR-Nachbearbeitung. Optimiert für die deutsche Rechnungslandschaft (USt-IdNr., Skonto, Reverse-Charge).

## Hardcore-Features

1. **Multi-Format-Parsing-Engine mit adaptiver Schemaerkennung**  
   billripper parst Rechnungen aus PDF, PNG, TIFF, EML und XML – und erkennt automatisch das zugrunde liegende Schema (DATEV, ZUGFeRD, XRechnung, individuell). Die Engine lernt aus Benutzerkorrekturen und verbessert die Trefferquote bei wiederkehrenden Lieferanten kontinuierlich.

2. **Echtzeit-Validierung gegen deutsche Steuerlogik**  
   Jede extrahierte Rechnung wird gegen geltende deutsche Steuerregeln geprüft: Umsatzsteuer-Identifikationsnummer, Reverse-Charge-Kennzeichnung, Skontofristen und Brutto/Netto-Konsistenz. Fehlerhafte oder unvollständige Rechnungen werden automatisch markiert und in eine Review-Queue verschoben.

3. **Headless-API & Webhook-Integration für CI/CD-Pipelines**  
   billripper bietet eine vollständig dokumentierte REST-API sowie Webhook-Events für jeden Rechnungsstatus. Ideal für automatisierte Buchhaltungs-Workflows in Kubernetes-Clustern oder als Microservice in einer Event-Driven-Architektur.

## Schritt-für-Schritt-Anleitung

1. **API-Key generieren und Umgebungsvariablen setzen**  
   Erstelle einen API-Key im billripper-Dashboard unter `Settings > API Access`. Setze diesen als Umgebungsvariable `BILLRIPPER_API_KEY` in deiner CI/CD-Umgebung (z. B. GitHub Actions Secrets oder GitLab CI/CD Variables). Nutze `curl` oder `axios` für den ersten Test-Endpunkt.

2. **Rechnung per POST an den Parsing-Endpunkt senden**  
   Sende eine PDF-Rechnung als `multipart/form-data` an `POST /api/v1/parse`. Optional: Füge den Header `X-Expected-Schema: ZUGFeRD` hinzu, um die Schemaerkennung zu beschleunigen. Die Antwort enthält ein JSON mit allen extrahierten Feldern, inklusive Confidence-Score pro Feld.

3. **Ergebnisse über Webhook in dein Buchhaltungssystem pushen**  
   Konfiguriere im Dashboard unter `Webhooks` die Ziel-URL deines ERP-Systems (z. B. DATEV oder lexoffice). Wähle das Event `invoice.parsed` aus. Nach erfolgreichem Parsing sendet billripper sofort ein strukturiertes JSON-Payload an deinen Endpunkt – inklusive Prüfsumme zur Integritätssicherung.

## Best Practices für den produktiven Einsatz

- **Batch-Processing mit Retry-Logik einplanen**  
   Bei hohem Rechnungsaufkommen (z. B. Monatsabschluss) empfiehlt sich ein Batch-Processing über die API mit exponentieller Backoff-Retry-Logik. Nutze dazu die `X-Request-Id` für Idempotenz und vermeide Duplikate.

- **Schema-Templates für Stammlieferanten hinterlegen**  
   Für wiederkehrende Lieferanten (z. B. Telekom, SAP, DATEV) lohnt sich das Hinterlegen eines individuellen Schema-Templates im Dashboard. billripper erkennt dann Layout-Abweichungen sofort und korrigiert sie automatisch – das reduziert manuelle Eingriffe um bis zu 80 %.
