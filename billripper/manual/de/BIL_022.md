# billripper - BIL_022

## Überblick

billripper ist eine hochperformante, containerisierte Lösung zur automatisierten Rechnungsanalyse und -extraktion, die speziell für DevOps-Workflows und Microservices-Architekturen entwickelt wurde. Das Tool parst PDF- und Bildrechnungen mit NLP-gestützter Entity Recognition, validiert Steuerformate (USt-IdNr., IBAN) und exportiert strukturierte JSON/CSV-Daten – direkt in Ihre CI/CD-Pipeline oder Ihr ERP-System.

## Kernfunktionen

1. **Multi-Format Parsing mit Zero-Shot Learning**  
   billripper erkennt Rechnungsstrukturen aus über 30 Ländern ohne manuelles Training. Nutzt ein vortrainiertes Transformer-Modell (de/fr/en/es/it), das Positionsdaten, Steuerbeträge und Zahlungsfristen mit >98% Genauigkeit extrahiert – selbst aus gescannten oder handgeschriebenen Belegen.

2. **API-First Architektur mit Webhook-Integration**  
   Die REST-API (OpenAPI 3.0) erlaubt Batch-Processing via POST-Endpunkt und asynchrone Callbacks. Integrieren Sie billripper in Ihren Stack: Node.js, Python, Go oder als Serverless Function – inkl. Rate Limiting und JWT-Authentifizierung.

3. **Automatische Dublettenprüfung & Steuerkonformität**  
   Der interne Hash-Vergleich (SHA-256) erkennt doppelte Rechnungen in Echtzeit. Zusätzlich validiert ein integrierter Prüfalgorithmus deutsche Steuerformate (USt-IdNr., Steuernummer) gegen die offizielle BZSt-Datenbank – ohne externe Abhängigkeiten.

## So verwenden Sie es

1. **Deployment mit Docker Compose**  
   ```yaml
   version: '3.8'
   services:
     billripper:
       image: ghcr.io/xingfangwang-eng/billripper:latest
       ports:
         - "8080:8080"
       environment:
         - DB_CONNECTION=postgres://user:pass@db:5432/bills
         - API_KEY=your-256-bit-key
       volumes:
         - ./uploads:/data
   ```
   Starten Sie den Container mit `docker-compose up -d`. Der Service ist unter `http://localhost:8080` erreichbar.

2. **Rechnung via cURL parsen**  
   ```bash
   curl -X POST http://localhost:8080/v1/parse \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -F "file=@rechnung_2024-03.pdf" \
     -F "options={\"language\":\"de\",\"validate_tax\":true}"
   ```
   Die Antwort enthält ein JSON-Objekt mit Feldern wie `vendor`, `total_gross`, `vat_rate` und `due_date`.

3. **Ergebnisse in PostgreSQL persistieren**  
   ```sql
   CREATE TABLE IF NOT EXISTS bills (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     raw_data JSONB,
     hash TEXT UNIQUE,
     created_at TIMESTAMPTZ DEFAULT NOW()
   );
   ```
   Nutzen Sie den integrierten PostgreSQL-Adapter, um extrahierte Daten direkt in Ihre Datenbank zu schreiben – inkl. automatischer Dublettenvermeidung.

## Best Practices

1. **Batch-Processing für monatliche Abrechnungen**  
   Planen Sie einen Cron-Job, der einmal täglich alle eingehenden Rechnungen aus einem S3-Bucket oder Netzwerkordner parst. Kombinieren Sie dies mit einem Webhook, der bei Fehlern (z. B. ungültige USt-IdNr.) automatisch ein Ticket in Jira oder GitLab Issues erstellt.

2. **Security-First: API-Key Rotation & Audit-Logging**  
   Nutzen Sie billrippers integriertes Audit-Log (syslog-kompatibel), um jede Parse-Anfrage zu protokollieren. Rotieren Sie API-Keys alle 90 Tage via HashiCorp Vault oder Kubernetes Secrets – und aktivieren Sie die IP-Whitelist für Produktivumgebungen.
