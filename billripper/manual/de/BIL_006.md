# billripper - BIL_006

## Übersicht

billripper ist die ultimative Lösung für automatisiertes Rechnungs-Parsing und -Management. Es extrahiert mühelos relevante Daten aus PDF-, Bild- und HTML-Rechnungen – selbst aus komplexen, mehrseitigen Dokumenten mit uneinheitlichen Layouts. Optimiert für deutsche Unternehmen, die Compliance, Effizienz und Kostentransparenz priorisieren.

## Detaillierter Inhalt

### Was ist billripper?

billripper ist ein hochperformantes Open-Source-Tool, das auf OCR, NLP und regelbasiertem Matching basiert. Es löst das Kernproblem der manuellen Rechnungserfassung: Fehleranfälligkeit, Zeitaufwand und fehlende Integration in bestehende Buchhaltungssysteme. Statt stundenlanger Dateneingabe liefert billripper in Sekunden strukturierte JSON-, CSV- oder XML-Exporte – direkt aus der Rechnung.

### Kernfunktionen

1. **Adaptives Layout-Erkennungssystem**  
   billripper analysiert Rechnungen nicht nur auf Textbasis, sondern erkennt automatisch Tabellenstrukturen, Rechnungsköpfe, Steuerblöcke und Zahlungsbedingungen – selbst bei gescannten oder handschriftlich ergänzten Dokumenten. Kein manuelles Mapping mehr nötig.

2. **Multi-Format-Engine mit Echtzeit-Korrektur**  
   Unterstützt PDF, PNG, JPEG, TIFF und HTML. Die Engine führt eine optische Zeichenerkennung (OCR) mit integrierter Rechtschreibkorrektur durch – für deutsche Sonderzeichen wie „ß“, „ä“, „ö“, „ü“ und gängige Abkürzungen (z. B. „MwSt.“, „USt-IdNr.“). Fehlerraten unter 0,5 % bei klaren Dokumenten.

3. **API-First-Architektur mit Webhook-Integration**  
   billripper bietet eine REST-API, die sich nahtlos in DATEV, Lexware, SAP oder eigene Workflows einbinden lässt. Webhooks triggern bei erfolgreicher Verarbeitung automatisch Buchungsprozesse oder Benachrichtigungen – ideal für automatisierte Rechnungsworkflows.

### So verwenden Sie es

1. **Automatisierte Batch-Verarbeitung via CLI**  
   ```bash
   billripper --input ./rechnungen/ --output ./export/ --format json --ocr de --batch-size 50
   ```  
   Dieser Befehl verarbeitet alle Rechnungen im Ordner `./rechnungen/` im Batch-Modus. Die OCR-Sprache wird auf Deutsch (`de`) gesetzt. Das Ergebnis ist eine geordnete JSON-Datei pro Rechnung im Ausgabeordner. Ideal für monatliche Abrechnungen.

2. **Integration in einen CI/CD-Pipeline (GitHub Actions)**  
   Fügen Sie folgendes YAML-Snippet in Ihre Pipeline ein, um Rechnungen automatisch bei jedem Push zu validieren und zu exportieren:  
   ```yaml
   - name: Rechnungen parsen
     run: |
       curl -X POST https://api.billripper.io/v1/parse \
         -H "Authorization: Bearer ${{ secrets.BILLRIPPER_API_KEY }}" \
         -F "file=@./invoice.pdf" \
         -F "options={\"language\":\"de\",\"output\":\"csv\"}" \
         --output parsed_invoice.csv
   ```  
   So stellen Sie sicher, dass jede neue Rechnung sofort strukturiert vorliegt – ohne manuellen Eingriff.

3. **Erweiterte Fehlerbehandlung mit Logging und Retry-Logik**  
   ```python
   from billripper import BillRipperClient

   client = BillRipperClient(api_key="your_key")
   try:
       result = client.parse("invoice.pdf", language="de", retries=3, timeout=30)
       print(result.to_dict())
   except BillRipperError as e:
       logging.error(f"Parsing fehlgeschlagen: {e}")
   ```  
   Dieses Beispiel implementiert eine Retry-Logik (max. 3 Versuche) und detailliertes Logging – perfekt für Produktivumgebungen, in denen Ausfallzeiten vermieden werden müssen.

### Beste Praktiken

- **Batch-Processing mit dediziertem Ordner**  
  Legen Sie einen überwachten Ordner an (z. B. `/invoices/neu/`), in den Sie alle eingehenden Rechnungen ablegen. Lassen Sie billripper als Cron-Job oder Daemon laufen, der jede neue Datei automatisch verarbeitet und in Ihr Buchhaltungssystem exportiert. Vermeiden Sie manuelle Uploads – das reduziert Fehlerquellen um 90 %.

- **Datenvalidierung vor Buchung**  
  Nutzen Sie die integrierte Validierungsfunktion, um Pflichtfelder wie Rechnungsnummer, Datum, Betrag und Steuersatz zu prüfen. Konfigurieren Sie Webhooks, die bei fehlenden oder inkonsistenten Daten eine Warnung an Ihr Team senden – bevor die Rechnung gebucht wird. So verhindern Sie Compliance-Probleme und Nachbuchungen.
