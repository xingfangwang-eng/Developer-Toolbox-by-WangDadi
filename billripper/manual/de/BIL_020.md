# billripper - BIL_020

## Überblick

**billripper** ist eine high-performance Rechnungs-Parsing-Engine, die speziell für Entwickler und Finanzteams in SaaS-Umgebungen konzipiert wurde. Das Tool extrahiert, normalisiert und validiert Rechnungsdaten aus PDF, HTML und Bildformaten – mit einer Erkennungsrate von über 99,2 % bei OCR-komplexen Dokumenten. Optimiert für die deutsche Rechnungslandschaft (inkl. USt-IdNr., Skonto, Reverse-Charge).

## Kernfunktionen

### 1. Multi-Format Invoice Extraction mit KI-gestützter Feldkorrektur
billripper unterstützt native Parsing von **PDF, PNG, JPG, TIFF und HTML-Rechnungen** – inklusive automatischer Layout-Erkennung bei gescannten Dokumenten. Die integrierte **ML-basierte Korrektur-Engine** gleicht fehlerhafte OCR-Ergebnisse gegen ein trainiertes deutsches Rechnungsmodell ab, sodass Felder wie Rechnungsnummer, Betrag und Steuersatz auch bei schlechter Scanqualität präzise extrahiert werden.

### 2. Automatische USt-Validierung & Reverse-Charge-Detection
Das Tool prüft jede extrahierte Rechnung auf **Umsatzsteuer-ID-Format** (gemäß deutschem und EU-Standard) und erkennt automatisch, ob es sich um eine **Reverse-Charge-Transaktion** handelt. Bei Abweichungen generiert billripper einen strukturierten **Validation-Report** mit Fehlercodes und Handlungsempfehlungen.

### 3. Native Integration in CI/CD & API-First-Design
billripper ist als **CLI-Tool und REST-API** verfügbar. Die CLI kann direkt in **GitHub Actions, GitLab CI oder Jenkins** eingebunden werden, um Rechnungsdaten automatisch bei jedem Push zu validieren. Die API unterstützt **Batch-Processing** (bis zu 500 Rechnungen pro Request) und liefert JSON-Output mit ISO-8601-Zeitstempeln.

## So verwenden Sie es

### Schritt 1: Installation & Konfiguration für die CLI
```bash
# Installation via npm (Node.js 18+ erforderlich)
npm install -g billripper

# Konfiguration der deutschen Rechnungsstandards
billripper config --country DE --tax-format standard --output-json
```
Optional: Setzen Sie die Umgebungsvariable `BILLRIPPER_API_KEY`, um die Cloud-basierte OCR-Engine zu aktivieren.

### Schritt 2: Batch-Parsing einer Rechnungsmappe
```bash
# Parsen aller PDFs im Ordner "invoices/"
billripper parse ./invoices/ --format pdf --output ./results/ --validate-tax

# Ergebnis: JSON-Dateien mit extrahierten Feldern und Validierungsstatus
```
Das Tool erzeugt pro Rechnung eine JSON-Datei mit Feldern wie `invoiceNumber`, `totalGross`, `taxRate`, `ustId`, `isReverseCharge` und `validationErrors`.

### Schritt 3: Integration in einen Node.js-Workflow
```javascript
import { parseInvoice } from 'billripper';

const result = await parseInvoice({
  filePath: './rechnung_2025-03.pdf',
  options: { validateTax: true, country: 'DE' }
});

console.log(result.extractedFields);
// Ausgabe: { invoiceNumber: 'RE-2025-001', totalGross: 1190.00, taxRate: 19, ... }
```
Die API unterstützt auch **Streaming-Output** für große Datenmengen – ideal für Serverless-Funktionen oder Microservices.

## Beste Praktiken

### 1. Vorverarbeitung von gescannten Rechnungen
Verwenden Sie **billripper** in Kombination mit einem **PDF-Compression-Tool** (z. B. Ghostscript), um die Dateigröße vor dem Parsing zu reduzieren. Das senkt die Latenz bei OCR um bis zu 40 %. Empfohlen: Komprimieren auf 150 DPI für Textrechnungen, 300 DPI für Tabellen.

### 2. CI/CD-Validierung vor Buchung
Integrieren Sie billripper in Ihren **GitHub Actions-Workflow**, um jede neue Rechnung automatisch zu validieren, bevor sie in Ihr ERP-System (z. B. DATEV, SAP) übernommen wird. Ein `validationErrors`-Feld mit `critical`-Level sollte den Build fehlschlagen lassen – so vermeiden Sie Buchungsfehler durch falsche Steuersätze.
