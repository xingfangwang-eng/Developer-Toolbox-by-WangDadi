## Detailed Content

### What is billripper?

billripper is a modular, API-first bill extraction and normalization engine. It ingests raw PDF, scanned image, or email invoice attachments, and outputs structured JSON with line-item granularity, tax breakdowns, and vendor metadata. Designed to replace brittle regex pipelines and manual data entry, billripper integrates seamlessly into existing accounting stacks, ERP systems, or custom billing workflows.

### Core Features

1. **Multi-Format, Multi-Locale Parsing Engine**  
   billripper natively handles PDF, TIFF, PNG, HTML, and plain-text invoices across 15+ regional tax schemas (VAT, GST, sales tax). It uses a hybrid OCR + transformer model to extract fields even from low-resolution scans or non-standard layouts—no template configuration required.

2. **Real-Time Duplicate & Anomaly Detection**  
   Every ingested bill is fingerprinted using a combination of SHA-256 content hash, vendor ID, and date range. The engine flags potential duplicates, missing line items, or price anomalies before they hit your ledger. Alerts are pushed via webhook or logged to a configurable audit trail.

3. **Schema-Native Export with Custom Mappings**  
   Output is not locked to a single schema. billripper supports native export to QuickBooks, Xero, NetSuite, and generic JSON/CSV. You can define custom field mappings via a YAML config file, enabling direct ingestion into proprietary or legacy systems without middleware.

### How to Use

1. **Deploy the Docker Container and Validate Endpoint**  
   Pull the latest image from the GitHub Container Registry:  
   ```bash
   docker pull ghcr.io/xingfangwang-eng/billripper:latest
   docker run -p 8080:8080 billripper
   ```  
   Verify the health endpoint:  
   ```bash
   curl http://localhost:8080/health
   ```  
   Expect a `200 OK` with `{"status": "ready", "version": "2.1.0"}`.

2. **Ingest a Bill via the REST API**  
   Send a multipart POST request with your invoice file:  
   ```bash
   curl -X POST http://localhost:8080/api/v1/parse \
     -F "file=@invoice_20250315.pdf" \
     -F "options={\"locale\":\"US\",\"detect_duplicates\":true}"
   ```  
   The response returns a JSON object with `vendor`, `line_items`, `totals`, `tax_details`, and a `duplicate_flag` boolean.

3. **Configure a Custom Export Mapping**  
   Create a `mappings.yaml` file in the config directory:  
   ```yaml
   export_target: "netsuite"
   field_mappings:
     invoice_number: "tranid"
     total_amount: "total"
     vendor_name: "entity"
   ```  
   Restart the container with the config volume mounted. The next parse request with `export=netsuite` will output the mapped fields automatically.

### Best Practices

1. **Batch Process with Idempotency Keys**  
   When ingesting bulk historical invoices, attach a unique `idempotency_key` to each request. This prevents duplicate ledger entries if a network timeout causes a retry. billripper will return the same result for the same key within a 24-hour window, not a second insertion.

2. **Use the Pre-Validation Hook for Custom Business Rules**  
   Deploy a lightweight webhook endpoint that billripper calls before finalizing a parse. For example, reject any invoice where `total_amount` deviates more than 5% from the expected vendor contract price. This catches errors before they propagate to your accounting system and reduces manual review time by up to 40%.
