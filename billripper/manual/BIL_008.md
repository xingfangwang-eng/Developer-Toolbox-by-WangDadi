### What is billripper?

billripper is a high-performance, schema-agnostic bill parsing and analysis engine designed for developers and finance ops teams who need to extract, normalize, and audit billing data at scale. Unlike generic OCR or PDF extractors, billripper handles multi-vendor formats, complex line-item hierarchies, and currency conversions out of the box — no manual mapping required.

---

### Core Features

1. **Adaptive Schema Inference with Multi-Vendor Support**  
   billripper automatically detects and maps billing structures from over 50+ vendors (AWS, Azure, Stripe, Twilio, etc.) without pre-configured templates. It uses a hybrid approach: regex-based field extraction + ML-driven anomaly detection for non-standard invoices.

2. **Real-Time Line-Item Aggregation & Cost Attribution**  
   Break down every charge to the SKU or resource level. billripper supports nested line items, discounts, taxes, and prorations — then aggregates them into a unified cost attribution model. Output is JSON or Parquet, ready for ingestion into Prometheus or your data warehouse.

3. **Audit-Ready Diff Engine with Versioning**  
   Track every change in a bill over time. billripper generates a diff between current and previous invoice versions, highlighting cost deltas, new line items, or missing credits. This is critical for compliance and chargeback workflows.

### How to Use

**Step 1: Initialize the Parser with Vendor Hints**  
```bash
billripper parse --input invoice_2025-03.pdf --vendor aws --output-format json
```
If vendor is unknown, omit `--vendor` to trigger auto-detection. The engine will return a confidence score for the inferred schema.

**Step 2: Normalize Currency & Date Formats**  
```python
from billripper import Normalizer

normalizer = Normalizer(target_currency="USD", date_format="ISO8601")
normalized = normalizer.normalize(raw_bill)
```
This step converts all monetary values to your base currency using live FX rates and standardizes timestamps to UTC. Useful for multi-region billing consolidation.

**Step 3: Run the Diff Engine Against a Baseline**  
```bash
billripper diff --current bill_mar.json --baseline bill_feb.json --output delta_report.csv
```
The diff output includes: `added_items`, `removed_items`, `price_changes`, and `anomaly_score`. Use this for automated alerts in your CI/CD pipeline.

### Best Practices

1. **Always Run the Diff Before Approving Payments**  
   Integrate `billripper diff` into your payment approval workflow (e.g., via a GitHub Action or webhook). This catches unexpected charges or billing errors before they hit your ledger. For high-volume environments, schedule a nightly diff job and pipe results into Slack or PagerDuty.

2. **Use the Schema Cache for Repeated Vendor Bills**  
   If you process bills from the same vendor monthly, enable the schema cache (`--cache-dir /tmp/billripper_cache`). This reduces parsing time by ~60% on subsequent runs and avoids redundant inference calls. Clear the cache quarterly to account for vendor template updates.
