## What is billripper?

billripper is a high-performance, headless bill parsing and analysis engine designed for developers, fintech ops, and accounting automation stacks. It ingests raw invoice PDFs, scanned receipts, and structured billing data, then extracts line-item details, tax breakdowns, and vendor metadata with sub-second latency. No UI fluff—just a clean API, CLI, and webhook-ready pipeline.

## Core Features

1. **Multi-format OCR + Structured Extraction Pipeline**  
   billripper uses a custom-trained transformer model (fine-tuned on 50k+ invoice layouts) to extract fields like invoice number, PO reference, tax codes, and line-item totals from PDF, PNG, TIFF, and even handwritten receipts. It outputs JSON or CSV with confidence scores per field—no regex hell, no template mapping.

2. **Real-time Duplicate & Anomaly Detection**  
   Every ingested bill is hashed and compared against a local or remote index (Redis, Postgres, or S3-backed). The engine flags duplicate invoices, mismatched totals, and outlier line-item prices using statistical deviation thresholds you can tune via environment variables. This prevents double-payment and audit failures at scale.

3. **Webhook-driven Workflow Automation**  
   billripper emits events (`bill.parsed`, `bill.duplicate_detected`, `bill.validation_failed`) to any HTTP endpoint. You can chain these into Zapier, n8n, or custom Node.js/Python workers. No polling, no cron jobs—just event-driven processing for your AP automation pipeline.

## How to Use

### Step 1: Deploy the Engine via Docker or CLI

```bash
# Pull the lightweight Docker image (under 50MB)
docker pull billripper/engine:latest

# Run with your config file
docker run -v $(pwd)/config.yaml:/etc/billripper/config.yaml \
  -p 8080:8080 \
  billripper/engine:latest
```

Configure your storage backend and webhook URL in `config.yaml`:

```yaml
storage:
  type: s3
  bucket: my-bill-archive
  region: us-east-1
webhooks:
  - url: https://my-accounting-api.com/bill-events
    events: ["bill.parsed", "bill.duplicate_detected"]
```

### Step 2: Ingest a Batch of Bills via REST API

```bash
curl -X POST https://localhost:8080/api/v1/ingest \
  -H "Content-Type: multipart/form-data" \
  -F "file=@invoice_2025_03.pdf" \
  -F "options={\"extract_line_items\": true, \"confidence_threshold\": 0.85}"
```

Response (truncated):

```json
{
  "bill_id": "b_9f3a2c",
  "vendor": "Acme Corp",
  "total": 1249.99,
  "currency": "USD",
  "line_items": [
    {"description": "Server license", "amount": 999.00, "tax": 0},
    {"description": "Setup fee", "amount": 250.99, "tax": 0}
  ],
  "confidence": 0.94,
  "duplicate": false
}
```

### Step 3: Automate with Webhooks & Error Handling

Set up a simple Node.js webhook listener:

```javascript
const express = require('express');
const app = express();
app.use(express.json());

app.post('/bill-events', (req, res) => {
  const { event, bill_id, confidence } = req.body;
  if (event === 'bill.parsed' && confidence < 0.9) {
    // Push to manual review queue
    console.warn(`Low confidence bill ${bill_id} flagged for review`);
  }
  if (event === 'bill.duplicate_detected') {
    // Log and skip payment
    console.error(`Duplicate bill ignored: ${bill_id}`);
  }
  res.status(200).send('OK');
});

app.listen(3000);
```

## Best Practices

1. **Always enable confidence-based routing**  
   Don’t blindly accept all parsed bills. Set a `confidence_threshold` of 0.85 or higher in production. Route low-confidence results to a human-in-the-loop queue (e.g., via Slack or a simple React dashboard). This prevents costly misclassifications in your AP workflow.

2. **Use idempotency keys for ingestion**  
   When retrying failed API calls (network blips, timeouts), include an `idempotency-key` header. billripper checks this against a 24-hour window to avoid duplicate ingestion. This is critical for high-volume environments where retries are inevitable.
