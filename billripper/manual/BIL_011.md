## Detailed Content

### What is billripper?

billripper is a high-performance, multi-format invoice and bill extraction engine built for developers, accountants, and SaaS platforms that need to automate the ingestion of financial documents. It rips through PDF, scanned images, and digital invoices, converting raw data into structured JSON with minimal overhead.

### Core Features

- **Zero-Shot Schema Inference**: billripper automatically detects invoice layouts (e.g., line items, taxes, vendor info) without pre-training on specific templates. It uses a lightweight NLP model trained on 10,000+ global invoice variants, supporting 10+ languages including English, German, Spanish, and Japanese.

- **Real-Time Barcode & QR Decoding**: Embedded OCR pipeline extracts and decodes barcodes (Code 128, EAN-13) and QR codes directly from scanned bills. This enables instant match against purchase orders or inventory systems without manual entry.

- **Multi-Currency Tax Normalization**: Automatically identifies and normalizes VAT, GST, and sales tax rates across 50+ jurisdictions. Outputs a unified tax breakdown with ISO 4217 currency codes, ready for ERP integration (e.g., SAP, QuickBooks, NetSuite).

### How to Use

1. **Initialize the Engine with Custom Config**  
   ```python
   from billripper import BillRipper
   ripper = BillRipper(
       language='en',
       tax_rules='US,DE,JP',  # Multi-jurisdiction tax normalization
       output_format='json'
   )
   document = ripper.load('invoice_2025_03.pdf')
   ```
   This sets up the engine to parse invoices in English, apply US, German, and Japanese tax rules, and return structured JSON. The `load()` method handles PDF, PNG, and JPEG formats.

2. **Extract and Validate Key Fields**  
   ```python
   result = ripper.extract(document)
   validated = ripper.validate(
       result,
       required_fields=['vendor_name', 'total_amount', 'invoice_date'],
       currency='USD'
   )
   if validated.errors:
       print(f"Validation failed: {validated.errors}")
   ```
   The `validate()` method checks for missing or malformed fields (e.g., invalid date, negative total). It also cross-references currency codes against the extracted amounts to flag discrepancies.

3. **Export to ERP-Compatible Format**  
   ```python
   erp_payload = ripper.export(
       validated.data,
       target='quickbooks',
       mapping='standard'  # Predefined field mapping for QuickBooks Online
   )
   with open('output/qb_invoice.json', 'w') as f:
       f.write(erp_payload)
   ```
   The `export()` method transforms the normalized data into a format matching your target ERP system. The `mapping` parameter supports `'standard'`, `'custom'`, or direct field overrides via a dictionary.

### Best Practices

- **Batch Processing with Rate Limiting**  
  When processing hundreds of bills daily, use billripper's built-in rate limiter to avoid throttling from external OCR services or tax databases. Set `max_requests_per_minute=60` in the config. For local-only parsing (no API calls), this is unnecessary.

- **Pre-Processing for Scanned Images**  
  For low-quality scans, apply `ripper.enhance(image, contrast=1.5, denoise=True)` before extraction. This improves OCR accuracy by up to 40% on faded or skewed documents. Always test with a sample batch to tune contrast and denoise parameters.
