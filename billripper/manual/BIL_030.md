## Detailed Content

### What is billripper?

**billripper** is not just another OCR wrapper. It's a modular, API-first bill ingestion engine designed to rip through PDFs, scanned images, and raw text invoices, extract structured data (line items, totals, tax codes, vendor info), and push it into your existing ERP, accounting, or data pipeline — all with sub-second latency and zero dependency on third-party cloud services.

Think of it as the `jq` for bills: predictable, composable, and brutally efficient.

### Core Features

1. **Multi-Format Bill Ripping Engine**  
   Native support for PDF, TIFF, JPEG, PNG, and raw text. Built-in layout-agnostic parser that handles rotated pages, embedded tables, and mixed-language invoices (EN, DE, ES, JA, FR). No pre-processing required — just point it at a file or a stream.

2. **Schema-Agnostic Data Normalization**  
   Outputs a unified JSON schema with auto-detected fields: `vendor_name`, `invoice_date`, `due_date`, `line_items[]`, `tax_breakdown`, `currency`, `total_gross`. Custom mapping rules via YAML config for legacy or proprietary formats. Supports export to CSV, JSON, or direct PostgreSQL/MySQL insertion.

3. **Headless CLI + REST API Dual Mode**  
   Run it as a standalone CLI for batch processing thousands of invoices in a single command, or spin up a lightweight REST server for integration into microservices. Both modes share the same core engine — no feature disparity between local and server deployments.

### How to Use

#### 1. Batch Parse All Invoices in a Directory

```bash
billripper rip --input ./invoices/ --output ./parsed/ --format json --concurrency 8
```

This command recursively scans `./invoices/`, processes up to 8 files in parallel, and writes normalized JSON files to `./parsed/`. Use `--dry-run` to preview detected fields without writing output.

**Pro tip:** Combine with `--watch` to automatically process new files dropped into a hot folder — ideal for automated document scanning workflows.

#### 2. Extract Specific Vendor Invoices via REST API

```bash
curl -X POST http://localhost:8080/rip \
  -F "file=@invoice_2024_03_15.pdf" \
  -F "vendor_filter=AcmeCorp" \
  -F "output_format=csv"
```

Returns a CSV stream with only line items from AcmeCorp invoices. Supports `vendor_filter`, `date_range`, and `currency` query parameters. The API is fully stateless — scale horizontally with a load balancer.

#### 3. Custom Mapping for Non-Standard Invoice Formats

Create a `mapping.yaml` file to override default field extraction for a specific vendor:

```yaml
vendor: "MegaSupplier GmbH"
mappings:
  invoice_number: "Rechnungsnummer"
  total_gross: "Gesamtbetrag"
  tax_rate: "MwSt.-Satz"
  line_items:
    - description: "Beschreibung"
    - quantity: "Menge"
    - unit_price: "Einzelpreis"
```

Then run:

```bash
billripper rip --input ./mega_invoice.pdf --mapping ./mapping.yaml
```

This ensures 100% accurate extraction even for vendors that deviate from standard invoice layouts.

### Best Practices

1. **Always validate with a golden dataset**  
   Before deploying billripper into production, create a small set of manually verified invoices (10–20 files) and run them through `billripper validate --golden ./golden_set/`. This catches false positives in field detection early and lets you tune mapping rules without breaking your pipeline.

2. **Use idempotent output naming for audit trails**  
   When batch processing, always set `--output-naming hash` to generate filenames based on the invoice content hash (SHA256). This prevents duplicate processing, ensures traceability, and makes it trivial to reprocess only changed files in incremental runs.
