# billripper - BIL_005

## Overview

billripper is a high-performance, multilingual invoice parsing engine designed for SaaS platforms handling cross-border billing. It strips, normalizes, and extracts structured data from PDF, image, and raw HTML invoices across 10+ languages and currencies. This is enterprise-grade bill analysis software optimized for automated accounts payable, fraud detection, and subscription management workflows.

## What is billripper?

billripper is a lightweight CLI and API-first tool that ingests raw invoice blobs (PDF, scanned images, or HTML receipts) and outputs a deterministic JSON schema. It uses regex-based tokenization and locale-aware heuristics to identify line items, tax breakdowns, vendor metadata, and payment terms. Unlike generic OCR wrappers, billripper is built for precision: it handles mixed-currency invoices, VAT/GST variants, and date formats from the US, Germany, Japan, and Spain out of the box.

## Core Features

### 1. Multi-Locale Token Extractor with Fallback Chains
Automatically detects the invoice locale (de_DE, es_ES, ja_JP, en_US) and applies a cascading tokenizer: first tries native language patterns (e.g., "MwSt." for German VAT), then falls back to ISO currency codes and date formats. No more broken parsing on international bills.

### 2. Line-Item Decomposition Engine
Splits ambiguous "total due" lines into subtotals, discounts, tax rates, and net amounts using a weighted confidence matrix. Supports recurring subscription billing patterns (monthly, annual, usage-based) and flags anomalies like double-taxation or missing line items.

### 3. Idempotent Bill Fingerprinting
Generates a SHA-256 fingerprint from the raw invoice metadata (vendor ID, invoice number, issue date) to prevent duplicate processing. Integrated with idempotency keys for webhook retries—critical for payment reconciliation in multi-tenant SaaS architectures.

## How to Use

### 1. Parse a Multi-Currency Invoice with Locale Override
```bash
billripper parse invoice.pdf --locale ja_JP --currency JPY --output json
```
This forces Japanese locale parsing (handles 令和 dates, 消費税 tax codes) and outputs a structured JSON. The `--currency` flag validates the detected currency against the locale—useful for mixed-invoice scenarios.

### 2. Configure Custom Tax Rule for German VAT (MwSt.)
Create a `billripper.toml` config file:
```toml
[tax_rules]
de_DE = { vat_rate = 0.19, vat_label = "MwSt." }

[output]
schema = "standard_v2"
deduplicate = true
```
Run `billripper transform invoice.json --config billripper.toml` to apply the rule and strip the VAT from the total. The tool will also flag any line items with mismatched tax rates.

### 3. Batch Process Invoices with Idempotency Key
```bash
billripper batch ./invoices/ --idempotency-key "batch-20231027" --webhook https://api.yoursaas.com/bill-callback
```
Each invoice gets a fingerprint; if the same invoice is re-submitted, billripper returns a `409 Conflict` with the existing parsed data. This is a hard requirement for event-driven billing systems.

## Best Practices

### 1. Pre-validate Invoice Locale Before Batch Processing
Run `billripper detect-locale invoice.pdf` on a sample set to confirm locale detection accuracy. For SaaS platforms serving German and Japanese clients, always set `--locale` explicitly in production—autodetection may fail on ambiguous receipts (e.g., a Japanese invoice with English vendor names).

### 2. Use Idempotency Keys for Webhook Reliability
In high-throughput billing pipelines, always pass a unique idempotency key per batch. This prevents double-crediting or duplicate ledger entries when webhooks retry due to network issues. Pair this with a database unique constraint on the bill fingerprint hash.
