# billripper - BIL_013

## 概要

billripperは、請求書・領収書のデータを**瞬時に構造化**する次世代型パースエンジンです。OCR精度の限界を超え、レイアウト非依存の解析ロジックにより、PDF・画像・スキャンデータから必要な情報を抽出・整形します。経理・購買・監査業務における手作業の90%を削減可能です。

## 詳細な内容

### billripperとは？

billripperは、**機械学習ベースのドキュメントインテリジェンスツール**です。従来のテンプレートマッチング方式とは異なり、フィールド位置やフォントに依存せず、意味論的なコンテキストから項目を特定します。日本語・英語・中国語など10言語以上に対応し、複数通貨・税区分の自動判別が可能です。

### コア機能

- **ゼロコンフィグレイアウト検出**  
  テンプレート登録不要で、縦書き・横書き・表形式・自由形式のあらゆるレイアウトに対応。請求書内の「小計」「消費税」「請求金額」などを意味解析で自動抽出します。

- **リアルタイム重複・改ざん検知**  
  同一請求番号の重複登録を防止。さらに、PDFメタデータとOCR結果のハッシュ比較により、スキャン後の改ざんを検出します。

- **カスタムバリデーションルールエンジン**  
  JSONベースのルール定義で、自社の経理ポリシーに沿った検証を実行。例えば「消費税率10%対象品目と軽減税率8%対象品目が混在する請求書はフラグを立てる」といった高度な条件設定が可能です。

### 使い方

1. **CLIによるバッチインポートと自動分類**  
   ```bash
   billripper import --dir ./invoices --format auto --output ./parsed --rule ./company_rules.json
   ```
   上記コマンドで、`./invoices` ディレクトリ内の全ファイルを再帰的にスキャン。自動でレイアウトを検出し、`company_rules.json` のバリデーションルールに従って分類・出力します。

2. **Webhook連携によるリアルタイムパイプライン構築**  
   ```yaml
   # .billripper.yaml
   webhook:
     url: "https://your-erp.example.com/webhook/billripper"
     events: ["parsed", "error", "duplicate"]
     retry: 3
     timeout: 30s
   ```
   設定ファイルでWebhookエンドポイントを指定。パース完了・エラー発生・重複検出時に即座に通知し、自社のERPや会計システムとシームレスに連携できます。

3. **REST APIによるカスタムアプリケーション統合**  
   ```python
   import requests
   response = requests.post(
       "https://api.billripper.io/v1/parse",
       files={"file": open("invoice.pdf", "rb")},
       data={"language": "ja", "currency": "JPY", "validate_tax": True}
   )
   print(response.json())
   # 出力例: {"vendor": "株式会社サンプル", "total": 125000, "tax": 12500, ...}
   ```
   API経由で任意のアプリケーションにパース機能を組み込み可能。マルチスレッド対応で、1リクエストあたり平均0.8秒で処理完了。

### ベストプラクティス

- **バリデーションルールは段階的に適用する**  
  初期段階では「必須フィールドの存在確認」のみに留め、運用データが蓄積されてから「金額の整合性チェック」「税率の自動判定」などを追加することを推奨します。急激なルール増加は誤検出の原因となり、チームの信頼を損ねるリスクがあります。

- **パース結果は必ず人間によるレビューフェーズを設ける**  
  billripperの精度は99.2%を超えますが、0.8%の例外が重大な経理ミスに繋がる可能性があります。パース後のデータをダッシュボードで一覧表示し、フラグが立った項目のみを確認する「例外ベースレビュー」のワークフローを構築してください。
