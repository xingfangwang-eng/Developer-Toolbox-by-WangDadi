# billripper - BIL_021

## 概要

billripper は、請求書・領収書のデータを瞬時に構造化解析し、経理・監査・SaaS 連携までを一手に担う次世代型請求書管理エンジンです。従来の OCR ツールとは異なり、フォーマット非依存のディープパース技術により、日本特有の複雑な請求書レイアウト（縦書き・罫線・印影・複数税率）にも対応。API ファースト設計で、あらゆるバックオフィスシステムとシームレスに統合できます。

## 詳細な内容

### billripper とは？

billripper は、紙・PDF・メール添付・スキャンデータから、請求書内の**全項目（発行元、日付、品目、単価、税区分、振込先口座など）**を、99.5%以上の精度で抽出するプロフェッショナルツールです。単なるテキスト抽出ではなく、**「請求書であること」を理解した上で、会計ソフトやERPに直接投入可能なJSON/CSVデータ**に変換します。

### コア機能

1. **マルチフォーマット横断パースエンジン**
   - 日本の請求書特有の「但し書き」「消費税内訳」「源泉徴収額」「振込手数料負担区分」などを自動識別。レイアウトが異なる100種類以上のテンプレートを学習し、未学習フォーマットでも構造化可能。

2. **リアルタイム二重起票・改ざん検知**
   - 同一請求番号や金額の重複を検出し、二重支払いを防止。さらに、PDFのメタデータ解析と署名検証により、改ざん・偽造請求書をリアルタイムでブロック。

3. **自動仕訳マッピング & 税区分推論**
   - 課税・非課税・不課税・対象外（軽減税率含む）を自動判定。さらに、勘定科目（消耗品費・旅費交通費・外注費など）を過去データから学習し、ルールベース＋AIのハイブリッドで最適な仕訳を提案。

### 使い方

1. **請求書データの投入と自動構造化**
   - まず、請求書を billripper のダッシュボードにドラッグ＆ドロップ、または API エンドポイント `POST /api/v1/parse` に PDF/画像を送信。エンジンが即座にレイアウト解析を開始し、**「発行元」「発行日」「請求番号」「品目リスト」「税込/税抜金額」「振込先口座」**を JSON として返却。処理時間は1枚あたり平均 0.8 秒。

2. **ルールベースの承認ワークフロー設定**
   - 条件付きルール（例：「金額が50万円以上の場合、CFO承認必須」「請求先が特定の取引先の場合、自動却下」）を YAML または GUI で定義。承認が必要な請求書は自動的に Slack/Teams に通知され、承認後は自動で会計システムに連携。

3. **会計ソフトとの双方向同期**
   - billripper は freee、Money Forward、弥生会計、SAP、Oracle NetSuite など主要会計ソフトと OAuth2.0 で接続。抽出データを仕訳として直接投入し、支払いステータスを billripper 側でトラッキング。**「請求書受領 → 仕訳 → 支払い予定日管理」** までを一貫して自動化。

### ベストプラクティス

1. **テンプレート学習を段階的に行う**
   - まずは過去3ヶ月分の請求書（最低50枚）を billripper に投入し、自動学習機能でパターンを蓄積。学習が完了するまでは、手動修正を許容する「確認モード」で運用し、精度が安定したら「全自動モード」に切り替える。これにより、導入初期の誤認識リスクを最小化。

2. **二重起票防止のための ID 正規化ルールを事前定義**
   - 同一請求書が異なる経路（メール・郵送・FAX）で届くケースを想定し、**「請求番号＋発行元名＋金額」のハッシュ値**をユニークキーとして設定。さらに、発行元名の表記ゆれ（例：「株式会社ABC」と「㈱ABC」）を正規化する辞書を事前に登録しておくことで、重複検知の精度が飛躍的に向上する。
