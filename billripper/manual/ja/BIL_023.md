# billripper - BIL_023

## 概要

billripper は、マルチフォーマット対応の高速請求書解析エンジンです。機械学習ベースのOCR後処理とルールベースのフィールド抽出を組み合わせ、PDF、スキャン画像、電子帳票から構造化データを生成します。年間処理可能な請求書数は500万件超、フィールド抽出精度は業界平均を12%上回ります。

## 詳細な内容

### billripper とは

billripper は、請求書処理における「データ抽出の非効率」と「フォーマット依存性」という2つの根本課題を解決するために設計されたプロフェッショナルツールです。従来のテンプレートマッチング方式とは異なり、**動的フィールド推論エンジン**を搭載し、未学習のレイアウトでも高精度な抽出を実現します。対応言語は日本語、英語、中国語、韓国語、ドイツ語、フランス語を含む12言語。SaaS型APIとして提供され、月額課金制でスケーラブルに利用可能です。

### コア機能（3つの硬核機能）

1. **動的フィールド推論（DFI）エンジン**  
   請求書レイアウトに依存せず、ヘッダー/明細行/合計行の位置を画像特徴量とテキスト密度分布からリアルタイムに特定。学習データがゼロの新フォーマットでも、初回解析で95%以上のフィールド一致率を達成。Transformerベースの軽量モデル（わずか12MB）をエッジで動作可能。

2. **マルチモーダル異種データ統合パイプライン**  
   PDF（テキスト/スキャン）、Excel、CSV、画像（JPEG/PNG/TIFF）、メール添付ファイルを一括投入可能。内部で画像を300dpiに正規化後、OCR（Tesseract + 自社学習済みモデル）と構造解析を並列実行。出力はJSON、XML、CSVの3形式に対応し、APIエンドポイントから直接データベースに書き込むことも可能。

3. **リアルタイム重複検出・適応型エラーハンドリング**  
   同一請求書の重複アップロードを、ハッシュベースの指紋照合でミリ秒単位で検出。さらに、抽出信頼度が80%未満のフィールドは自動的に保留リストに格納され、管理者がWeb UI上でワンクリック修正可能。修正履歴はモデルのオンライン学習にフィードバックされ、次回以降の精度が向上する。

### 使い方（3つの詳細な操作ステップ）

**ステップ1: バッチアップロードと前処理の自動実行**  
`billripper` CLI または REST API に、解析対象のファイルをディレクトリ単位で投入します。サポート形式は `.pdf`, `.xlsx`, `.csv`, `.jpg`, `.png`, `.tiff`。アップロード後、システムは自動的にファイル形式を判別し、画像系ファイルは300dpiにリサンプリング、PDFはテキストレイヤー抽出を試行。前処理が完了すると、ジョブIDが返却されます（例: `job_20231015_abc123`）。

```bash
curl -X POST https://api.billripper.io/v2/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "files=@invoice_001.pdf" \
  -F "files=@invoice_002.xlsx"
```

**ステップ2: 抽出結果の検証と修正（Web UI）**  
アップロード後、管理ダッシュボード（`https://app.billripper.io/jobs/{job_id}`）で結果を確認します。各フィールドには信頼度スコア（0〜100）が表示され、赤色のフィールドは修正が必要です。修正はインライン編集可能で、変更は即座に保存されます。修正履歴は「変更ログ」タブに記録され、監査証跡として利用可能。

**ステップ3: 構造化データのエクスポートと外部連携**  
修正完了後、データをJSON、XML、CSVのいずれかでエクスポートします。APIエンドポイント `/v2/export/{job_id}?format=json` を呼び出すか、Web UIの「エクスポート」ボタンからダウンロード。さらに、Webhook機能を有効にすれば、抽出完了時に指定したURLに自動POSTされます。これにより、自社の会計システムやERPとシームレスに連携可能です。

### ベストプラクティス（2つの業界推奨）

1. **スキャン画像の前処理として、300dpi以上かつ白黒2値化を推奨**  
   カラー画像や低解像度（150dpi未満）のスキャンは、OCR精度が著しく低下します。特に、細かい罫線や小さなフォント（8pt以下）を含む請求書では、300dpiの白黒TIFF形式が最適です。billripper は自動補正機能を持ちますが、前処理で品質を確保することで、抽出信頼度を平均8%向上できます。

2. **大量バッチ処理時は、ジョブ分割と非同期コールバックを活用**  
   1ジョブあたりのファイル数は500ファイル以下、合計サイズ500MB未満に分割することを推奨します。これを超えると、メモリ消費が増大し処理時間が非線形に増加します。また、同期APIではなく非同期API（`/v2/upload_async`）を使用し、Webhookで完了通知を受け取る設計が、本番環境での安定運用の鍵です。
