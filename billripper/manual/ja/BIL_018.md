# billripper - BIL_018

## 概要

billripper は、請求書・領収書データの高速パース、自動仕分け、会計システム連携に特化した次世代型解析エンジンです。OCR 精度の限界を超える構造化処理と、マルチフォーマット対応により、経理・監査・キャッシュフロー管理の現場で圧倒的な工数削減を実現します。

## 詳細な内容

### billripper とは

billripper は、PDF、スキャン画像、電子帳票（XML/EDI）から請求情報を瞬時に抽出・正規化し、会計ソフトや ERP に直接連携可能な JSON/CSV 形式で出力するプロフェッショナルツールです。単なる OCR ツールではなく、レイアウト依存しない「セマンティックパース」と「ルールベース+機械学習のハイブリッド補正」を搭載しており、日本語特有の複雑な請求書フォーマットにも高い耐性を持ちます。

### コア機能

- **セマンティックフィールドマッピングエンジン**: 請求書内の「請求元」「日付」「金額」「税区分」「振込先口座」などを、レイアウト変更や表記揺れに影響されずに自動認識・マッピング。テンプレート定義不要で、初見のフォーマットでも 95% 以上の精度で構造化。
- **バッチ差分監査 & 重複検知**: 同一取引先からの請求書を時系列で比較し、金額・数量・単価の差分を自動検出。さらに、ハッシュベースの重複チェックにより、二重支払いや重複登録を防止。監査証跡としてのログ出力に対応。
- **マルチチャネル出力 & API ゲートウェイ**: 抽出データを RESTful API、Webhook、または直接クラウドストレージ（S3, GCS）にプッシュ可能。freee、Money Forward、SAP、Oracle NetSuite など主要会計システムとの連携プラグインを標準装備。

### 使い方

1. **請求書ファイルを投入**: `billripper parse --input ./invoices/ --format auto --output ./results/` を実行。対応フォーマットは PDF（スキャン含む）、JPEG/PNG、XML（国産EDI）、CSV。`--format auto` でファイル拡張子とヘッダー情報から自動判別。大量ファイルは `--batch-size 100` で並列処理可能。
2. **抽出結果を検証・補正**: 出力ディレクトリに `results.json` が生成。`billripper review --file results.json --interactive` で、低確信度フィールド（`confidence < 0.8`）をハイライト表示。ターミナル上で直接修正、または `--approve-all` で自動承認。`--export-audit-log` で全変更履歴を出力。
3. **会計システムへ連携**: `billripper push --source results.json --target freee --config ./freee_config.yml` を実行。設定ファイルに API キーとマッピングルールを記述。`--dry-run` で事前検証後、`--apply` で本番連携。エラー発生時は自動リトライ（最大3回）と Slack 通知。

### ベストプラクティス

- **定期実行とアラート設定**: cron や CI/CD パイプラインで `billripper parse` を定時実行（例：毎日 9:00）。`--webhook-url` で抽出完了通知を Slack や Teams に送信。さらに、`--alert-on-anomaly` を有効にし、前日比で請求金額が ±20% 以上変動した場合にアラートを上げることで、不正請求や入力ミスを早期発見。
- **テンプレートキャッシュの活用**: 同一取引先からの請求書が複数回発生する場合、`--cache-templates` オプションで初回解析時のレイアウト情報をキャッシュ。2回目以降の解析速度が約 70% 向上。キャッシュは `.billripper_cache/` に保存され、`billripper cache --clear` でリセット可能。取引先ごとに固有のテンプレートを管理したい場合は `--vendor-id` で識別。
