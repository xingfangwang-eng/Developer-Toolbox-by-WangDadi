# billripper - BIL_001

## 概要

billripper は、請求書・領収書・明細書の**自動解析・構造化・エクスポート**に特化した CLI ツールです。PDF、スキャン画像、メール添付からキー情報（日付、金額、取引先、税区分）を抽出し、JSON/CSV/Google Sheets へ直接連携します。経理・監査・バックオフィス業務の効率を劇的に向上させます。

## 詳細な内容

### billripper とは？

billripper は、**「紙とデジタルの境界をなくす」** ためのエンジニア向け請求書パーサーです。従来の OCR ツールとは異なり、**レイアウト分析エンジン**と**税制ルールベース**を内蔵。日本のインボイス制度（適格請求書）や海外 VAT フォーマットにも対応し、**構造化データとして即座に利用可能**にします。

### コア機能

1. **マルチフォーマット自動判別 & バッチ解析**  
   - PDF、JPEG、PNG、TIFF を一括投入。**レイアウト分析エンジン**がヘッダー・明細行・小計・税額を自動認識。  
   - 対応フォーマット：日本の標準請求書、Amazon 請求書、PayPal 領収書、クレジットカード明細、航空券 E-Ticket など 20 種類以上。

2. **税区分・インボイス番号自動抽出**  
   - 日本のインボイス制度に対応。**適格請求書発行事業者番号**、**軽減税率対象品目**、**消費税額（8% / 10%）** を自動でタグ付け。  
   - 海外 VAT や GST もパターンマッチングで抽出可能。

3. **カスタムルールエンジンによるフィルタリング & アラート**  
   - ユーザー定義のルール（例：「金額が 10 万円を超える請求書にフラグを立てる」「特定の取引先のみ抽出」）を YAML で記述。  
   - 抽出結果に応じて Slack 通知、メール送信、CSV エクスポートを自動実行。

### 使い方

1. **請求書ディレクトリを指定して一括解析**  
   ```bash
   billripper parse --input ./invoices/ --output ./parsed/ --format json
   ```
   - `--input` に PDF や画像が入ったフォルダを指定。  
   - `--format` で出力形式（json / csv / google-sheets）を選択。  
   - 解析後、`./parsed/` に構造化データが生成される。  
   - **注意**：画像は 300 DPI 以上推奨。低解像度の場合、`--enhance-ocr` フラグで前処理を有効化。

2. **税区分ルールをカスタマイズしてエクスポート**  
   ```bash
   billripper export --rules ./custom-rules.yaml --output ./tax-report.csv
   ```
   - `custom-rules.yaml` に以下のように記述：  
     ```yaml
     rules:
       - condition: "amount > 100000"
         action: "flag"
         label: "高額請求書"
       - condition: "tax_type == 'reduced'"
         action: "extract"
         fields: ["invoice_number", "tax_amount"]
     ```
   - 抽出結果を CSV にエクスポート。**監査用のタグ付け**も同時に実施。

3. **Google Sheets へ直接連携（OAuth 認証）**  
   ```bash
   billripper sync --spreadsheet-id "YOUR_SPREADSHEET_ID" --sheet-name "請求書一覧"
   ```
   - 初回実行時に Google OAuth 認証が求められる。  
   - 認証後、解析済みデータが指定シートに自動反映。  
   - **利点**：チームメンバーがリアルタイムで最新の請求書データを共有可能。

### ベストプラクティス

- **請求書ファイル名に日付と取引先を含める**  
  `billripper parse` 時にファイル名から日付・取引先を自動抽出するため、`2025-03-15_株式会社ABC_請求書.pdf` のような命名規則を推奨。解析精度が向上し、手動修正の手間が減る。

- **定期的なバッチ処理を cron で自動化**  
  毎週月曜朝に先週分の請求書を自動解析し、Slack にサマリーを投稿する設定例：  
  ```cron
  0 9 * * 1 /usr/local/bin/billripper parse --input /mnt/invoices/weekly/ --output /mnt/parsed/ --notify slack
  ```
  これにより、経理チームが手動でファイルを確認する必要がなくなる。
