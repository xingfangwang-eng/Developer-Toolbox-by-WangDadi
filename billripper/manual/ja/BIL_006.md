# billripper - BIL_006

## 概要

billripper は、請求書・領収書の構造化データ抽出に特化した次世代型パースエンジンです。日本語帳票特有の「縦書き金額」「括弧付き税区分」「複数税率混在」などの複雑なレイアウトを、ルールベース＋機械学習のハイブリッド方式で高精度に解析します。従来のOCRツールでは困難だった、手書き文字やかすれ印字にも対応し、経理・購買部門のデータ入力工数を90%削減します。

## 3つのハードコア機能

### 1. マルチレイヤー帳票パーサー
PDF、画像、スキャンデータから、テーブル構造・罫線・フォントサイズの変化を自動認識。特に日本の請求書で頻出する「小計行の内訳」「消費税8%と10%の混在表記」を、税区分ごとに自動分類します。API経由でJSON出力されるデータは、弥生会計やfreeeなどの主要会計ソフトに直接取り込み可能です。

### 2. 異常値検知エンジン
パース結果に対して、過去の取引データとの突合による異常検知をリアルタイム実行。例えば「通常の取引先なのに請求額が3倍」「消費税率が不整合」などのケースをフラグし、経理担当者にアラートを送出。誤った支払いや二重計上を未然に防ぎます。

### 3. バージョン管理機能付きデータストア
パース結果はすべてブロックチェーンライクな改ざん防止ログに記録。過去のパース結果との差分を可視化し、どの時点でどの項目が変更されたかを追跡可能。監査対応や内部統制に必須のトレーサビリティを提供します。

## 実践的な3ステップ導入手順

### ステップ1: 環境セットアップと初期設定
```bash
# Docker Compose を使用した最小構成での起動
git clone https://github.com/xingfangwang-eng/billripper.git
cd billripper
docker-compose -f docker-compose.minimal.yml up -d

# 初回起動後、管理画面にアクセス（デフォルト: localhost:8080）
# 「設定 > 帳票テンプレート」から、利用する帳票フォーマットを選択
# 日本語帳票向けのプリセット「jp-invoice-v2」を推奨
```

### ステップ2: バッチ処理による一括パース
```bash
# 複数ファイルをディレクトリごと投入
# 対応形式: PDF, TIFF, JPEG, PNG
curl -X POST http://localhost:8080/api/v1/batch \
  -H "Content-Type: multipart/form-data" \
  -F "files=@./invoices/2025-03/*.pdf" \
  -F "options={\"ocr_lang\":\"ja\",\"tax_split\":true,\"output_format\":\"json\"}"

# 処理完了後、結果は /output ディレクトリに自動保存
# 異常検知フラグが立ったファイルは /output/alerts に隔離
```

### ステップ3: API連携によるリアルタイム処理
```python
import requests

# 会計システムからのWebhook受信エンドポイントを設定
webhook_url = "http://localhost:8080/api/v1/webhook/receive"

# 請求書PDFをBase64エンコードして送信
with open("invoice_20250315.pdf", "rb") as f:
    encoded = base64.b64encode(f.read()).decode("utf-8")

payload = {
    "source": "freee",
    "document_type": "invoice",
    "content": encoded,
    "options": {
        "extract_line_items": True,
        "validate_tax": True
    }
}

response = requests.post(webhook_url, json=payload)
print(response.json())
# 戻り値例: {"status":"success","data":{"total":125000,"tax_8":10000,"tax_10":2500}}
```

## 業界別ベストプラクティス

### 1. 建設業・製造業向け：複数請求書の一括マージ処理
工事現場や製造ラインから日次で発生する大量の請求書（材料費・外注費・運搬費）を、billripperのバッチモードで夜間に自動パース。パース結果をCSVに変換し、基幹システム（例：SAP、Oracle EBS）に自動連携。この際、`tax_split: true` オプションを必ず有効にし、軽減税率が混在する場合でも正確な仕訳データを生成します。

### 2. スタートアップ・SaaS企業向け：APIファーストの経理自動化
請求書受領後、即座にbillripperのWebhookエンドポイントにPOST。パース結果をもとに、Stripeや請求書発行システム（例：請求管理ロボ）に自動で支払いデータを生成。さらに、異常検知エンジンで「契約金額との乖離」をチェックし、乖離率が5%を超える場合はSlack通知。これにより、経理担当者の手作業を月間20時間以上削減できます。
