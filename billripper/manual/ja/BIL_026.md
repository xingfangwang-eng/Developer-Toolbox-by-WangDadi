# billripper - BIL_026

## 概要

**billripper** は、請求書・レシートデータを高精度で解析・構造化するためのSaaS型解析エンジンです。従来のOCRツールが抱える「レイアウト依存」「項目抽出の不完全さ」を解消し、AIベースのセマンティック解析により、あらゆるフォーマットの帳票から必要なデータを即座に抽出・整形します。

## 詳細な内容

### billripper とは

billripper は、請求書・領収書・明細書などの非構造化ドキュメントを、機械学習モデルとルールベース処理のハイブリッド方式で解析し、CSV/JSON/API連携可能な構造化データへ変換するプロフェッショナルツールです。特に、**日本語・英語・中国語**の混在文書にも対応し、インボイス制度や電子帳簿保存法にも準拠したデータ管理を実現します。

### コア機能

1. **マルチフォーマット自動判別・適応解析エンジン**  
   スキャンPDF、スマホ撮影画像、電子帳票（XML/EDI）など、入力フォーマットを自動識別。レイアウトが異なる帳票でも、項目名と値のペアを文脈から推定し、99%以上の精度で抽出可能。特に、**回転・傾き・ノイズ**が含まれる画像でも頑健に動作。

2. **リアルタイム重複検知・バリデーションフック**  
   請求書番号・発行日・金額の3要素で重複チェックを自動実行。さらに、ユーザー定義のバリデーションルール（例：「税抜金額×消費税率＝税込金額」の整合性確認）をYAMLで記述可能。不整合が検出された場合、即座にアラートを発報し、修正提案を表示。

3. **カスタムフィールドマッピング＆データパイプライン連携**  
   抽出した項目を、任意のスキーマ（例：会計ソフトの勘定科目、ERPの品目コード）にマッピング可能。WebhookやREST API経由で、Salesforce、QuickBooks、SAPなどの外部システムへ自動連携。処理結果は即座にJSONとして出力され、後続のバッチ処理やダッシュボードへ流し込める。

### 使い方

1. **環境セットアップとAPIキー発行**  
   まず、billripper の管理コンソールにログインし、プロジェクトを作成。APIキーを発行したら、以下のコマンドで動作確認用のサンプルリクエストを送信します。  
   ```bash
   curl -X POST https://api.billripper.io/v1/parse \
     -H "Authorization: Bearer {YOUR_API_KEY}" \
     -F "file=@invoice_sample.pdf" \
     -F "options={\"language\":\"ja\",\"output_format\":\"json\"}"
   ```
   `200 OK` とともに、抽出されたJSONが返却されれば成功です。

2. **カスタムバリデーションルールの定義**  
   プロジェクトルートに `.billripper.yml` を作成し、以下のようにルールを記述します。  
   ```yaml
   validation:
     - name: "消費税整合性チェック"
       condition: "tax_excluded * tax_rate == tax_included"
       severity: "error"
       message: "税抜金額と税込金額が一致しません。税率を確認してください。"
   ```
   このファイルを配置したディレクトリで `billripper validate` を実行すると、全解析結果に対してルールが適用されます。

3. **外部システムへの自動連携設定**  
   管理コンソールの「Webhook」タブで、連携先のエンドポイントURLを登録します。例えば、QuickBooks Onlineへ自動反映させる場合、以下のようなペイロードテンプレートを設定します。  
   ```json
   {
     "vendor_name": "{{extracted.supplier_name}}",
     "amount": "{{extracted.total_amount}}",
     "date": "{{extracted.issue_date}}",
     "memo": "BillRipper自動連携"
   }
   ```
   処理完了時に自動POSTされるため、手動転記の手間がゼロになります。

### ベストプラクティス

- **画像前処理の最適化**  
   スマホ撮影画像を解析する場合、解像度は300dpi以上、ファイルサイズは5MB以内に抑えてください。また、不要な背景や影を除去するため、事前に**グレースケール変換**と**コントラスト強調**を施すと、抽出精度が大幅に向上します。billripper のCLIには `--preprocess` オプションが用意されており、自動補正も可能です。

- **バッチ処理とエラーハンドリングの設計**  
   大量の請求書を一括処理する際は、1リクエストあたり最大50ファイルまでに分割し、`retry_count=3` を設定することを推奨します。また、処理結果のJSONには `status` フィールド（`success`/`error`）が含まれるため、エラー発生時は `error_message` を確認し、該当ファイルだけを再投入するスクリプトを組むと効率的です。エラーログはS3やCloudWatchに自動転送する設定も可能です。
