# billripper - BIL_003

## 概要

billripperは、請求書・領収書の自動解析・データ抽出に特化した次世代型SaaSツールです。OCR + ルールベース解析のハイブリッドエンジンを搭載し、PDF・画像・メール添付ファイルから構造化データを瞬時に生成。経理・監査・債権管理のワークフローを劇的に効率化します。

## 詳細な内容

### billripperとは？

billripperは、**「非構造化請求書データを、APIファーストで機械可読なJSON/CSVに変換する」** ことを使命としたプロフェッショナルツールです。従来のOCRツールが苦手とする**複数言語混在（日英中）・レイアウト崩れ・手書き数字**にも高精度で対応。SaaSアーキテクチャとして、マルチテナント分離・RBAC・Webhook通知を標準実装しており、大規模なバッチ処理にも耐えうる設計です。

### コア機能

1. **マルチフォーマット・ゼロコンフィグパーサー**  
   請求書PDF、スマホ撮影画像、FAXスキャン、メール本文のテキスト埋め込み請求書まで、**アップロードするだけで自動判別・パース**。テンプレート登録不要で、ベンダーごとに異なるレイアウトを機械学習ベースで適応学習します。

2. **リアルタイム重複検知 & フラグ管理**  
   同一請求番号・金額・日付の組み合わせを**ハッシュベースで即座に検出**。二重支払いリスクを未然に防ぎます。さらに、異常値（極端な割引率・消費税計算ミス）を自動フラグ化し、監査ログに記録。

3. **カスタムルールエンジンによる自動仕訳マッピング**  
   請求書の品目・部門コード・消費税区分を、ユーザー定義のルールセットに基づいて自動マッピング。**IFTTTライクな条件分岐**（例: `金額 > 100万 → 承認フロー起動`）をGUIで設定可能。会計ソフト（Freee, Money Forward, QuickBooks）との連携も標準対応。

### 使い方

1. **請求書データの一括投入と自動パース**  
   CLIツール `billripper-cli` を使用し、ローカルのPDF/画像ディレクトリを指定してバッチ投入。  
   ```bash
   billripper-cli batch --input ./invoices/ --output ./parsed/ --format json --webhook https://your-api.example.com/callback
   ```  
   各ファイルは非同期で処理され、パース結果は即座にJSONとして出力。エラー発生時はエラーログと共にリトライキューに自動エンキューされます。

2. **カスタムルールの作成と検証**  
   Webダッシュボードの「ルールエンジン」タブに移動。条件例: `[請求元] contains "株式会社XXX" AND [税抜金額] > 50000 → タグ: "要承認"` を設定。  
   **プレビューモード**で過去データに対してルールを適用し、ヒット率・誤検出率を確認してから本番反映。ルールはバージョン管理され、ロールバックも可能です。

3. **Webhook連携による自動承認フロー起動**  
   パース完了後、設定したWebhookエンドポイントにペイロードがPOSTされます。ペイロードには `request_id`, `parsed_data`, `flags` が含まれ、自社の承認システム（Slack, Teams, カスタムAPI）と連携。  
   ```json
   {
     "event": "invoice.parsed",
     "data": {
       "invoice_number": "INV-2025-001",
       "total_amount": 125000,
       "flags": ["amount_anomaly"],
       "request_id": "bil_003_a1b2c3"
     }
   }
   ```  
   フラグが立った請求書のみ、承認担当者にSlack通知が飛ぶよう設定可能。

### ベストプラクティス

- **バッチ処理時は前処理として画像の正規化を行う**  
   billripperは高精度ですが、スキャン画像の傾き・ノイズを事前に補正（例: `ImageMagick` で `-deskew 40%`）することで、特に手書き文字の認識率が向上します。CI/CDパイプラインに組み込むことを推奨。

- **ルールエンジンは「失敗を許容する設計」で段階的に適用する**  
   最初は「警告のみ」モードで運用し、誤検出パターンを収集。1〜2週間のデータ蓄積後、ルールを調整してから「自動タグ付け」→「自動承認ブロック」へ段階的に移行。これにより、ビジネスプロセスを止めずに精度を高められます。
