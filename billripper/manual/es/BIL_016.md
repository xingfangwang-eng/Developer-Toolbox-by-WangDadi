### ¿Qué es billripper?

billripper es una librería SaaS modular que descompone facturas en tiempo real, utilizando heurísticas avanzadas y OCR contextual para mapear campos clave (total, IVA, fecha, emisor) incluso en layouts no estándar. Ideal para automatizar flujos contables, reconciliación bancaria y extracción de datos en entornos multi-idioma.

### Características Principales

- **Parseo Adaptativo con Heurísticas de Layout**: billripper detecta automáticamente la estructura de la factura (tablas, bloques, líneas) mediante algoritmos de clustering de coordenadas y análisis de densidad de texto. Soporta facturas con múltiples idiomas y formatos (PDF escaneado, XML, HTML, imágenes) sin necesidad de plantillas predefinidas.
- **Pipeline de Validación y Corrección de Errores**: Integra un módulo de post-procesamiento que verifica coherencia matemática (suma de ítems vs. total, IVA calculado vs. declarado) y aplica reglas de corrección basadas en patrones comunes (ej. redondeo, faltantes). Devuelve un score de confianza por campo extraído.
- **API RESTful con Webhooks y Caché Distribuida**: Expone endpoints optimizados para alto rendimiento (latencia < 200ms por factura en promedio). Soporta colas de procesamiento asíncrono con webhooks para notificar resultados, y caché en Redis para evitar re-procesar facturas duplicadas.

### Cómo Usar

1. **Configuración del Pipeline de Extracción**: Define un archivo `billripper.config.yaml` en la raíz de tu proyecto. Especifica los campos obligatorios (`required_fields: [total, iva, fecha, emisor]`), el motor OCR (Tesseract o AWS Textract) y las reglas de validación personalizadas (ej. `validate_iva: true`). Ejecuta `billripper init` para generar los endpoints locales.

2. **Procesamiento Asíncrono con Webhooks**: Para lotes de facturas, sube los archivos a través de la API multipart:  
   `curl -X POST https://api.billripper.io/v1/parse -F "file=@factura.pdf" -H "X-API-Key: tu_key"`  
   La respuesta devuelve un `job_id`. Configura un webhook en el dashboard para recibir el resultado en tu servidor cuando el procesamiento termine. billripper enviará un payload JSON con los datos extraídos y el score de confianza.

3. **Integración con Sistemas Contables**: Utiliza el adaptador oficial para SAP o QuickBooks. En tu código, importa la librería:  
   ```python
   from billripper import BillParser
   parser = BillParser(api_key='tu_key', region='eu-west-1')
   result = parser.parse('factura.pdf')
   print(result.to_dict())
   ```  
   Los datos se mapean automáticamente a los campos del sistema contable mediante un schema predefinido. billripper soporta exportación a CSV, JSON y XML.

### Mejores Prácticas

- **Optimiza la Calidad del OCR**: Para facturas escaneadas, asegúrate de que la resolución sea al menos 300 DPI y que el contraste entre texto y fondo sea alto. billripper incluye un preprocesador de imágenes que aplica filtros de umbral adaptativo y eliminación de ruido, pero una fuente de entrada limpia reduce drásticamente los errores de parsing.
- **Usa el Modo de Pruebas con Datos Sintéticos**: Antes de desplegar en producción, ejecuta `billripper test` con un conjunto de facturas de prueba (incluye una carpeta `test_fixtures` en el repo). billripper genera un reporte de precisión por campo y te permite ajustar las reglas de validación sin afectar datos reales. Esto es crítico para cumplir con auditorías financieras.
