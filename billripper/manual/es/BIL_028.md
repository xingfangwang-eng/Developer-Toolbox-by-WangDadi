# billripper - BIL_028

## Resumen

Herramienta de análisis y despiece de facturas orientada a desarrolladores, diseñada para extraer, normalizar y auditar datos financieros desde PDFs, XMLs y capturas de pantalla sin depender de OCR genérico. Ideal para equipos que necesitan integrar parsing de facturas en pipelines de contabilidad automatizada, compliance fiscal o conciliación bancaria.

## Contenido Detallado

### ¿Qué es billripper?

billripper no es un simple lector de facturas. Es un motor de parsing semántico que entiende la estructura de documentos financieros en múltiples formatos (Facturae, UBL, PDF escaneado, HTML incrustado). Convierte facturas caóticas en JSON/CSV estructurado, listo para consumir en APIs, bases de datos o dashboards. Soporta detección automática de IVA, retenciones, impuestos locales y desglose por líneas de producto.

### Características Principales

- **Reconocimiento contextual de impuestos**: billripper identifica automáticamente el régimen fiscal aplicable (IVA general, reducido, exento, IGIC, IPSI) según el país emisor y el tipo de producto. No solo extrae el importe, sino que asigna la tasa correcta y valida coherencia contra la base imponible.

- **Pipeline de normalización multi-formato**: Convierte cualquier factura (Facturae, PDF con tabla, XML UBL 2.1, CSV de ERP) a un modelo canónico unificado. Incluye heurísticas para corregir errores comunes (decimales mal colocados, campos faltantes, nombres de proveedor inconsistentes).

- **Auditoría de integridad con reglas personalizables**: billripper permite definir reglas de validación (por ejemplo, "suma de líneas debe coincidir con total", "IVA no puede ser negativo", "fecha de emisión anterior a fecha de vencimiento"). Genera informes de discrepancias con sugerencias de corrección.

### Cómo Usar

1. **Configurar el pipeline de entrada**: Define en un archivo YAML (`billripper-config.yml`) las fuentes de datos: carpetas locales, buckets S3, endpoints HTTP o colas RabbitMQ. billripper detecta automáticamente el formato de cada archivo y aplica el extractor correspondiente. Ejemplo de configuración mínima:
   ```yaml
   sources:
     - type: local
       path: ./facturas/
       recursive: true
     - type: s3
       bucket: facturas-prod
       prefix: 2025/
   ```

2. **Ejecutar parsing y normalización**: Lanza el proceso con un solo comando. billripper procesa cada factura, extrae campos clave (NIF, fecha, base imponible, total, líneas) y los normaliza al modelo canónico. Usa el flag `--validate` para activar la auditoría de integridad:
   ```bash
   billripper parse --config billripper-config.yml --output ./resultados --validate
   ```

3. **Exportar resultados y generar alertas**: billripper produce un archivo JSON por factura procesada y un resumen CSV global. Si hay discrepancias (por ejemplo, suma de líneas != total), se genera un log de alerta. Para integración continua, usa el modo `--webhook` que envía los resultados a un endpoint REST:
   ```bash
   billripper parse --config billripper-config.yml --webhook https://tu-api.com/billripper-callback
   ```

### Mejores Prácticas

- **Usa billripper como middleware en tu pipeline de contabilidad**: No lo trates como una herramienta aislada. Conéctalo a tu sistema de contabilidad (ERP, CRM) mediante webhooks o colas de mensajes. billripper puede actuar como filtro previo a la contabilización, detectando facturas duplicadas, montos inconsistentes o proveedores no registrados.

- **Aprovecha las reglas de validación personalizadas para cumplir con normativas locales**: billripper permite cargar reglas en formato JSON (por ejemplo, para cumplir con el SII español o el Facturae italiano). Define reglas que verifiquen que las facturas contengan los campos obligatorios según la legislación de tu país. Esto reduce el riesgo de sanciones fiscales y acelera auditorías.
