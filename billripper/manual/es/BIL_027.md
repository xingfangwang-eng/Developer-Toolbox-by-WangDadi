# billripper - BIL_027

## Resumen

**billripper** es un sistema de parsing y análisis de facturas diseñado para desarrolladores que necesitan extraer, validar y transformar datos financieros no estructurados en JSON/XML listos para integración. Optimizado para pipelines CI/CD, automatización contable y migración legacy.

## Contenido Detallado

### ¿Qué es billripper?

billripper no es un simple lector de PDFs. Es un **motor de extracción semántica** que entiende la estructura lógica de facturas en múltiples formatos (PDF, imagen, HTML, EDI). Utiliza reglas de normalización por país y moneda, y expone una API RESTful con tolerancia a errores y logging granular. Ideal para equipos que procesan miles de facturas diarias sin intervención manual.

### Características Principales

- **Extracción por campos dinámicos con heurística adaptativa**: billripper detecta automáticamente el layout de la factura (tabla, lista, bloque) y mapea campos críticos (NIF, importe total, IVA, fecha de vencimiento) sin necesidad de templates predefinidos. Soporta facturas escaneadas con OCR integrado.
- **Validación cruzada con reglas fiscales por país**: Cada factura extraída pasa por un pipeline de validación que verifica cálculos de IVA, totales y formatos de identificación fiscal según la normativa española (AEAT), mexicana (SAT) o colombiana (DIAN). Los errores se reportan con códigos de error específicos.
- **Exportación a múltiples destinos con transformación en caliente**: Los datos extraídos pueden ser enviados directamente a un webhook, a un bucket S3, a una base de datos PostgreSQL o a un archivo CSV/JSON, con transformaciones personalizables mediante plugins en Node.js o Python.

### Cómo Usar

1. **Inicializa el motor con configuración regional**  
   Ejecuta `billripper init --locale es_ES --currency EUR --vat-rules strict`. Esto carga las reglas de validación para España, incluyendo el formato de NIF/NIE, tipos de IVA reducido y general, y la tolerancia de redondeo. Si trabajas con facturas de México, usa `--locale es_MX` para aplicar RFC y CFDI 4.0.

2. **Procesa un lote de facturas con pipeline paralelo**  
   Coloca los archivos en `./invoices/` y lanza `billripper batch --input ./invoices --output ./parsed --format json --parallel 4`. billripper distribuirá el procesamiento entre 4 workers, extrayendo, validando y transformando cada factura. El resultado incluye un archivo `_errors.log` con detalles de cada fallo (por ejemplo, "IVA calculado no coincide: esperado 21.00, obtenido 20.99").

3. **Integra la API REST en tu backend**  
   Levanta el servidor con `billripper serve --port 3000 --cors-origin https://tuapp.com`. Luego, desde tu aplicación, envía un POST a `/parse` con el archivo en multipart/form-data. Recibirás un JSON con los campos extraídos y un código de validación (`valid: true/false`). Ejemplo con curl:  
   ```bash
   curl -X POST http://localhost:3000/parse -F "file=@factura.pdf" -F "locale=es_ES"
   ```

### Mejores Prácticas

- **Normaliza los nombres de archivo antes del procesamiento**: Para evitar colisiones y errores de encoding, renombra los archivos siguiendo el patrón `{proveedor}_{fecha}_{numero_factura}.pdf`. Puedes usar el flag `--rename-pattern` en el comando batch para que billripper lo haga automáticamente. Esto facilita el seguimiento en logs y la depuración.
- **Implementa reintentos con backoff exponencial en la API**: Cuando integres billripper en un pipeline de producción, configura tu cliente HTTP para reintentar hasta 3 veces con un delay de 2, 4 y 8 segundos ante errores 503 (servicio ocupado) o 429 (límite de peticiones). billripper expone headers `Retry-After` en esas respuestas, lo que permite sincronización precisa.
