# billripper - BIL_018

## Resumen

Herramienta de parsing y análisis de facturas diseñada para desarrolladores que necesitan extraer, normalizar y estructurar datos financieros desde documentos PDF, imágenes o texto plano sin depender de APIs de terceros. billripper elimina el ruido visual y entrega un JSON limpio listo para integración en sistemas contables, ERPs o dashboards personalizados.

## Contenido Detallado

### ¿Qué es billripper?

billripper es un motor de extracción de datos de facturas que opera localmente, sin enviar información sensible a servidores externos. Utiliza técnicas de OCR adaptativo y parsing semántico para identificar campos críticos como montos, fechas, IVA, números de factura y proveedores, incluso en documentos con formatos inconsistentes o baja calidad de escaneo. Ideal para automatizar procesos de facturación en entornos donde la privacidad y la velocidad son prioridad.

### Características Principales

- **Parsing multiformato con fallback automático**: billripper detecta si el input es PDF, imagen (JPG/PNG) o texto plano, y aplica el pipeline de extracción óptimo. Si el OCR falla en un área, recurre a heurísticas de patrón de texto para rescatar datos críticos como el total o el CIF/NIF.
- **Normalización de moneda y fecha inteligente**: Convierte automáticamente formatos regionales (ej. "12.345,67 €" a `12345.67` y "31/12/2024" a ISO 8601). Soporta detección de moneda (USD, EUR, GBP, JPY, MXN) sin configuración previa, basándose en símbolos y códigos ISO.
- **Validación estructural en tiempo real**: Cada campo extraído pasa por reglas de validación (ej. que el subtotal + IVA = total, que la fecha sea lógica). Si hay discrepancias, billripper genera un reporte de anomalías con sugerencias de corrección, no solo un error genérico.

### Cómo Usar

1. **Inicializa el motor con configuración regional**: Ejecuta `billripper init --locale es_ES --currency EUR` para preconfigurar formatos de fecha, moneda y reglas de IVA. Esto evita falsos positivos en documentos españoles sin necesidad de tocar el código. Si trabajas con múltiples regiones, puedes pasar un archivo YAML con configuraciones por cliente.

2. **Procesa un lote de facturas en paralelo**: Usa `billripper parse ./facturas/ --output ./resultados/ --workers 4` para procesar hasta 4 documentos simultáneamente. billripper maneja automáticamente colas de fallos y reintenta con diferentes parámetros de OCR si la primera pasada no supera el umbral de confianza (por defecto 85%). Los resultados se guardan como JSON individuales más un resumen agregado en CSV.

3. **Integra el output en tu pipeline**: Cada JSON devuelto incluye un campo `metadata.confidence` (0-100) y `metadata.warnings`. Si la confianza es baja (<70), puedes reenviar el documento a un endpoint de revisión manual o aplicar reglas de corrección automática. Ejemplo de consumo en Node.js:
   ```javascript
   const resultado = await billripper.parse('factura.pdf');
   if (resultado.metadata.confidence > 80) {
     // Enviar a contabilidad
   } else {
     // Marcar para revisión
   }
   ```

### Mejores Prácticas

- **Preprocesa imágenes con resolución mínima de 200 DPI**: billripper incluye un comando `billripper enhance ./imagenes/ --dpi 300` que mejora contraste y elimina sombras antes del OCR. Esto reduce drásticamente errores en facturas escaneadas con poca luz o dobladas. Siempre ejecuta este paso antes del parsing si el lote contiene fotos de móvil.
- **Usa el modo "strict" para facturas con formato conocido**: Si sabes que todas las facturas provienen de un mismo proveedor (ej. Amazon Business), activa `billripper parse --mode strict`. Esto fuerza la validación de campos esperados y acelera el proceso al saltarse la detección genérica de patrones. Ideal para pipelines de alta frecuencia donde el formato es predecible.
