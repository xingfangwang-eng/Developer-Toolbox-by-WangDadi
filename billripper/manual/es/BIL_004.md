# billripper - BIL_004

## Resumen

Herramienta de parsing de facturas multi-formato con soporte nativo para OCR, extracción de campos clave y exportación a estructuras de datos normalizadas. Ideal para automatizar la gestión documental en entornos contables.

## Contenido Detallado

### ¿Qué es billripper?

billripper es un motor de extracción de datos facturarios diseñado para ingenieros de software y administradores de sistemas que necesitan procesar volúmenes masivos de facturas (PDF, imágenes, XML) sin depender de APIs externas costosas. Utiliza un pipeline de preprocesamiento de imágenes + análisis semántico basado en reglas configurables, con soporte para facturas de España, México y Argentina.

### Características Principales

- **Pipeline de OCR con detección automática de idioma**: billripper integra Tesseract 5 y EasyOCR para extraer texto de imágenes escaneadas, incluso en facturas con baja calidad de digitalización. El sistema detecta automáticamente el idioma (español, inglés, portugués) y aplica corrección ortográfica post-OCR para reducir errores en campos como NIF/CIF.

- **Extracción de campos clave con validación de formato**: La herramienta reconoce y extrae automáticamente: número de factura, fecha de emisión, fecha de vencimiento, importe total, IVA (desglosado por tipo), y datos del emisor/receptor. Cada campo se valida contra patrones regex predefinidos (por ejemplo, CIF español con letra correcta) y se genera un reporte de errores si algún valor no pasa la validación.

- **Exportación a múltiples formatos con mapeo personalizable**: billripper permite exportar los datos extraídos a JSON, CSV, Excel y SQL (INSERT statements). Puedes definir mapeos personalizados para adaptar la salida a tu esquema de base de datos o ERP, incluyendo transformaciones de tipo (string a date, string a float) y renombrado de columnas.

### Cómo Usar

1. **Configuración del pipeline de extracción**: Crea un archivo `config.yaml` en el directorio raíz del proyecto. Define las rutas de entrada (carpeta con facturas), las reglas de extracción (por ejemplo, `regex_fecha: "\d{2}/\d{2}/\d{4}"`), y el formato de salida deseado. Ejecuta `billripper --config config.yaml --input ./facturas/ --output ./resultados/`. El sistema procesará todas las facturas en lote y generará un archivo de log por cada una.

2. **Validación manual de campos extraídos**: Si alguna factura presenta errores de validación (por ejemplo, CIF inválido), billripper genera un archivo `errores.csv` con la lista de campos fallidos y el motivo. Puedes revisar estos errores y forzar la corrección manual usando `billripper --fix-errors errores.csv --input ./facturas/`. El sistema re-procesará solo las facturas con errores aplicando las correcciones especificadas.

3. **Integración con CI/CD**: Para entornos de producción, billripper soporta ejecución en modo daemon con monitoreo de directorio. Usa `billripper --watch ./facturas/ --output ./resultados/ --interval 30` para que el sistema revise cada 30 segundos la carpeta de entrada y procese automáticamente las nuevas facturas. Puedes combinar esto con un cron job o un pipeline de GitHub Actions para automatizar completamente el flujo.

### Mejores Prácticas

- **Preprocesamiento de imágenes antes de OCR**: Si trabajas con facturas escaneadas, aplica un preprocesamiento básico (escala de grises, umbralización, eliminación de ruido) usando herramientas como ImageMagick o la propia función `--preprocess` de billripper. Esto mejora significativamente la precisión del OCR, especialmente en facturas con sellos o marcas de agua.

- **Mantén un archivo de reglas por proveedor**: billripper permite cargar reglas de extracción específicas por emisor. Crea un directorio `reglas/` con archivos YAML nombrados por NIF del proveedor (por ejemplo, `B12345678.yaml`). Esto facilita la adaptación a formatos de factura recurrentes y reduce falsos positivos en la extracción.
