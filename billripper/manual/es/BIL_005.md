# billripper - BIL_005

## Resumen

billripper es la navaja suiza definitiva para desarrolladores que necesitan procesar facturas, extractos bancarios y documentos financieros no estructurados. Con soporte nativo para OCR multilingüe y parsing semántico, convierte montones de PDFs en datos accionables sin intervención manual. Ideal para equipos de fintech, contabilidad automatizada y sistemas ERP que buscan escalar sin perder precisión.

## Contenido Detallado

### ¿Qué es billripper?

billripper no es solo un parser de facturas: es un motor de extracción inteligente que entiende la lógica detrás de cada línea de gasto. Gracias a su arquitectura basada en transformers ligeros y reglas configurables, puede distinguir entre IVA, descuentos, cargos de envío o impuestos locales en más de 50 formatos de factura distintos. Todo corre en local, sin depender de APIs externas, lo que garantiza privacidad de datos y latencia cero.

### Características Principales

- **Parsing semántico con detección de anomalías**: billripper no solo extrae campos como fecha, monto o proveedor; además identifica discrepancias entre el total calculado y el declarado, alertando sobre posibles errores o fraudes. Ideal para auditorías automáticas.
- **Pipeline de normalización multiformato**: Soporta desde PDFs escaneados (con OCR integrado) hasta XMLs de factura electrónica (facturae, UBL, CII). Cada documento se normaliza a un esquema JSON unificado, permitiendo integración directa con APIs REST o bases de datos.
- **Motor de reglas personalizables por región**: Define reglas específicas para impuestos locales (IGIC, IVA reducido, GST), formatos de fecha (DD/MM/AAAA vs MM/DD/AAAA) o monedas. Las reglas se escriben en YAML y se cargan en caliente sin reiniciar el servicio.

### Cómo Usar

1. **Configuración del pipeline de entrada**: Crea un archivo `config.yaml` que defina las fuentes de datos (carpeta local, S3, o webhook). Ejemplo:  
   ```yaml
   sources:
     - type: folder
       path: ./facturas/
       recursive: true
       filter: "*.pdf"
   ```
   Luego ejecuta `billripper ingest --config config.yaml` para iniciar la monitorización. El motor detectará automáticamente nuevos archivos y los encolará para procesamiento.

2. **Ajuste fino del parser con reglas personalizadas**: Si tu factura tiene un layout no estándar (ej. campos anidados en tablas), edita el archivo `rules/custom_rules.yaml` con selectores CSS-like. Por ejemplo:  
   ```yaml
   - field: total_con_iva
     selector: "table.invoice-totals tr:last-child td:last-child"
     transform: "parse_float"
   ```
   Luego valida la regla con `billripper test --file factura_ejemplo.pdf --rules rules/custom_rules.yaml`. El output mostrará cada campo extraído y su confianza.

3. **Exportación automatizada a tu sistema contable**: Configura un webhook en `config.yaml` para enviar los datos extraídos a tu ERP o API propia:  
   ```yaml
   webhooks:
     - url: "https://api.miempresa.com/invoices"
       method: POST
       headers:
         Authorization: "Bearer <token>"
       payload_template: "templates/contabilidad.j2"
   ```
   Cada factura procesada disparará una petición HTTP con el payload en el formato que definas (JSON, XML, CSV). billripper reintenta hasta 3 veces en caso de error.

### Mejores Prácticas

- **Usa reglas por proveedor**: En lugar de una regla genérica, crea perfiles específicos para tus proveedores más frecuentes (ej. Amazon Business, Telefónica, etc.). billripper permite asociar reglas por dominio o nombre de empresa, reduciendo falsos positivos y acelerando el parsing un 40%.
- **Programa limpiezas periódicas**: billripper almacena metadatos de cada factura en SQLite local. Para evitar crecimiento descontrolado, configura un cron job mensual que ejecute `billripper purge --older-than 90d`. Conserva solo los datos necesarios para auditorías recientes.
