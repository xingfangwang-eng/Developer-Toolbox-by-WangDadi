### ¿Qué es billripper?

billripper no es un simple lector de facturas: es un pipeline de extracción inteligente que transforma documentos no estructurados en JSON listo para consumir. Diseñado para entornos de alta concurrencia, soporta desde facturas simples de una línea hasta complejos documentos multilingües con tablas anidadas, descuentos y tasas impositivas variables. Su arquitectura modular permite desplegarse como microservicio independiente o embeberse en flujos CI/CD.

### Características Principales

- **Motor de Reconocimiento Heurístico con Validación Cruzada**: billripper combina OCR adaptativo con reglas de negocio configurables. Detecta automáticamente el tipo de factura (compra, venta, proforma, abono) y aplica validaciones de consistencia (ej. total vs. suma de líneas + impuestos). Soporta facturas en español, inglés, portugués y alemán, con detección de moneda y formato de fecha regional.

- **Pipeline de Normalización en Tiempo Real**: Convierte cualquier factura a un esquema canónico (ISO 20022) en milisegundos. Incluye mapeo automático de campos como `invoice_number`, `tax_id`, `total_amount` y `line_items`, con soporte para transformaciones personalizadas mediante hooks en Python o JavaScript.

- **API RESTful con Streaming y Webhooks**: billripper expone endpoints para subida síncrona (respuesta inmediata) y asíncrona (procesamiento en cola con callback). Los webhooks notifican eventos como `parsing_completed`, `validation_failed` o `duplicate_detected`, permitiendo integraciones reactivas con sistemas como Salesforce, SAP o QuickBooks.

### Cómo Usar

1. **Configuración del Pipeline de Extracción**:  
   Instala el cliente CLI con `npm install -g billripper-cli` o despliega el contenedor Docker `docker run -p 3000:3000 billripper/core`. Define las reglas de extracción en un archivo YAML (`config.yaml`) especificando los campos obligatorios, la tolerancia al error y el modo de validación (estricto o laxo). Ejemplo:  
   ```yaml
   fields:
     - invoice_number: { required: true, regex: '^FAC-\d{6}$' }
     - total_amount: { required: true, type: float }
   validation: strict
   ```

2. **Subida y Procesamiento de Facturas**:  
   Usa el endpoint `POST /api/v1/parse` con multipart/form-data. Para subidas asíncronas, añade el header `X-Callback-URL: https://tu-sistema.com/webhook`. billripper devuelve un `job_id` que puedes consultar con `GET /api/v1/jobs/{job_id}`. Ejemplo con curl:  
   ```bash
   curl -X POST -F "file=@factura.pdf" -H "X-Callback-URL: https://miapp.com/billripper-callback" https://api.billripper.com/v1/parse
   ```

3. **Integración con Sistemas Contables**:  
   Tras el parsing, billripper genera un JSON estandarizado. Para enviarlo a un ERP, configura un webhook de salida en `config.yaml` o usa el endpoint `POST /api/v1/export`. billripper incluye conectores predefinidos para SAP (IDoc), Oracle Netsuite (SuiteScript) y QuickBooks (OAuth2). Ejemplo de exportación a QuickBooks:  
   ```bash
   curl -X POST -H "Authorization: Bearer <token>" -H "Content-Type: application/json" -d '{"type":"quickbooks","data":<json_output>}' https://api.billripper.com/v1/export
   ```

### Mejores Prácticas

- **Optimiza la Precisión del OCR con Preprocesamiento**: Antes de enviar facturas escaneadas, aplica un preprocesamiento local (deskew, binarización, eliminación de ruido) usando herramientas como ImageMagick o la librería `pillow` de Python. billripper acepta imágenes en alta resolución (300 DPI recomendado), pero el preprocesamiento reduce falsos positivos en campos numéricos y mejora la detección de tablas. Para facturas en español, asegúrate de que el OCR esté configurado con el perfil `es-ES` para reconocer caracteres como ñ o acentos.

- **Implementa un Sistema de Colas con Retry y Dead Letter Queue**: En entornos productivos, las subidas asíncronas deben gestionarse con una cola (RabbitMQ o AWS SQS) para evitar pérdidas de datos. Configura un dead letter queue (DLQ) para facturas que fallen tras 3 reintentos. billripper expone métricas en Prometheus (`billripper_parsing_errors_total`, `billripper_processing_duration_seconds`) que permiten monitorear la salud del pipeline y ajustar la tolerancia al error según el volumen de facturas diarias (ej. 5000+ documentos por hora).
