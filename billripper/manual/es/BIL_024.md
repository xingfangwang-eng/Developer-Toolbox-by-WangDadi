### ¿Qué es billripper?

billripper es una herramienta profesional que automatiza el parsing de facturas en formatos PDF, XML y CSV, eliminando la necesidad de entrada manual de datos. Con soporte para 10 idiomas y detección inteligente de campos, billripper transforma facturas desordenadas en estructuras JSON/CSV listas para integraciones con ERPs como SAP, Oracle o sistemas contables locales.

### Características Principales

- **Motor de Extracción Heurística con NLP**: billripper utiliza un modelo de lenguaje entrenado en más de 50,000 facturas reales para identificar campos críticos (NIF, fechas, importes, IVA) incluso en documentos escaneados de baja calidad. Soporta formatos de factura de España, Latinoamérica y EE.UU., con detección automática de moneda (€, $, MXN, etc.).

- **Pipeline de Validación y Corrección Automática**: Cada factura extraída pasa por un pipeline de reglas de negocio que verifica consistencia (ej: tipo de IVA vs. base imponible) y corrige errores comunes (decimales mal colocados, campos faltantes). Genera un log detallado de discrepancias para auditoría.

- **API RESTful con Rate Limiting y Webhooks**: billripper expone una API GraphQL y REST que permite integración en tiempo real con sistemas de gestión de gastos (Expensify, Concur) o ERPs. Incluye webhooks para notificar cambios de estado (factura procesada, error de parsing) y soporta autenticación OAuth 2.0 con JWT.

### Cómo Usar

1. **Instalación y Configuración Inicial con Docker**  
   Clona el repositorio y ejecuta `docker-compose up -d` para desplegar billripper con PostgreSQL y Redis. Configura variables de entorno en `.env` para conectar a tu ERP (ej: `ERP_API_KEY`, `ERP_BASE_URL`). billripper detecta automáticamente el directorio de entrada (`/input`) y el de salida (`/output`).

2. **Procesamiento por Lotes con Script CLI**  
   Coloca tus facturas PDF en `./input/` y ejecuta:  
   ```bash
   billripper process --input ./input --output ./output --format json --validate true
   ```  
   billripper parsea cada archivo, extrae campos clave (factura número, fecha, total, IVA) y genera un archivo JSON por factura. Si `--validate` está activo, el pipeline de validación corrige errores y genera un reporte en `./output/validation_report.csv`.

3. **Integración Continua con CI/CD Pipeline**  
   Añade un job en tu pipeline (GitHub Actions, GitLab CI) que ejecute billripper cada vez que se suban facturas a un bucket S3 o Azure Blob Storage. Usa el endpoint `POST /api/v1/process` con un payload JSON:  
   ```json
   {
     "files": ["factura_2024_03.pdf"],
     "options": { "format": "csv", "webhook_url": "https://tu-sistema.com/webhook" }
   }
   ```  
   billripper devuelve un `job_id` y puedes consultar el estado con `GET /api/v1/jobs/{job_id}`.

### Mejores Prácticas

- **Optimización de Parsing para Facturas Multilenguaje**: Si trabajas con facturas de diferentes países (ej: España con IVA 21% y México con IVA 16%), configura `billripper config set locale=es_MX,es_ES` para que el motor de NLP aplique reglas fiscales específicas. Esto reduce falsos positivos en campos como el NIF (España) vs. RFC (México).

- **Auditoría y Rollback con Versionado de Facturas**: Activa la opción `--versioning true` en el procesamiento para que billripper almacene versiones anteriores de cada factura en la base de datos. Así, si un cambio en el ERP rompe la integración, puedes restaurar la versión anterior con `billripper rollback --factura-id <id> --version 2`.
