# billripper - BIL_014

## Resumen

Herramienta de extracción y análisis de facturas diseñada para desarrolladores que necesitan automatizar el procesamiento de documentos financieros. Optimizada para SEO con enfoque en **parsing de facturas, software de análisis de recibos y sistemas de gestión documental**.

## Contenido Detallado

### ¿Qué es billripper?

billripper es una librería CLI y API escrita en Rust que descompone facturas en PDF, imágenes o HTML en estructuras JSON limpias. No es un OCR genérico: utiliza heurísticas específicas para detectar campos como NIF/CIF, importes, IVA, fechas de emisión y vencimiento, incluso en facturas escaneadas con baja calidad. Ideal para integraciones en ERPs, contabilidad automatizada o sistemas de gestión de gastos.

### Características Principales

- **Extracción de campos críticos con validación fiscal**: billripper identifica automáticamente el NIF del emisor y receptor, desglosa el IVA (tipos reducidos, generales, exentos) y calcula el total sin necesidad de reglas predefinidas. Soporta formatos de factura española, mexicana, argentina y chilena.
- **Modo batch con paralelismo nativo**: Procesa miles de facturas en segundos usando hilos de Rust. Incluye detección de duplicados por hash SHA-256 y exportación directa a CSV, JSON o SQLite. No requiere dependencias externas como Tesseract o Poppler.
- **API REST integrada para microservicios**: Expone endpoints `/parse`, `/validate` y `/export` con autenticación JWT. Ideal para desplegar como contenedor Docker en Kubernetes o servidores ligeros. Incluye rate limiting y logging estructurado en formato JSON.

### Cómo Usar

1. **Instalación y configuración inicial**  
   Descarga el binario desde GitHub Releases o compila desde fuente: `cargo install billripper`. Ejecuta `billripper init` para generar el archivo de configuración `billripper.toml`. Define allí las rutas de entrada/salida, el formato de exportación (JSON por defecto) y las reglas de validación fiscal. Para entornos CI/CD, usa `billripper --config /etc/billripper/prod.toml`.

2. **Procesamiento de una factura individual**  
   Coloca el PDF en `./invoices/factura_2025.pdf` y ejecuta:  
   `billripper parse --input ./invoices/factura_2025.pdf --output ./output/result.json --format json`  
   El resultado incluirá campos como `issuer_nif`, `total_amount`, `vat_breakdown`, `issue_date` y `confidence_score`. Si falla la extracción, revisa el log en `./logs/billripper.log` para depurar.

3. **Automatización batch con pipeline**  
   Para procesar todo un directorio:  
   `billripper batch --input ./invoices/ --output ./output/ --format csv --parallel 8 --dedup`  
   Esto generará un CSV con todas las facturas validadas, omitiendo duplicados. Para integración continua, añade un cron job o un workflow de GitHub Actions que ejecute el comando cada hora.

### Mejores Prácticas

- **Usa el modo `--validate` antes de exportar a contabilidad**: billripper puede verificar que el IVA declarado coincida con el cálculo interno. Activa `--strict` para rechazar facturas con discrepancias > 0.01€. Esto evita errores en declaraciones trimestrales de IVA (modelo 303 en España, 29 en México).
- **Optimiza el rendimiento con archivos de configuración por entorno**: Crea `billripper.dev.toml` para pruebas con `--dry-run` y `billripper.prod.toml` con `--parallel 16` y `--output-dir /mnt/nfs/invoices`. Usa variables de entorno para credenciales de API si despliegas como microservicio.
