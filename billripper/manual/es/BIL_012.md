# billripper - BIL_012

## Resumen

**billripper** es un parser de facturas multiformato diseñado para extraer, normalizar y estructurar datos financieros de documentos PDF, XML, CSV e imágenes escaneadas. Orientado a desarrolladores y equipos de contabilidad que necesitan automatizar la ingestión de facturas con alta precisión, incluso en entornos con formatos no estandarizados. Compatible con los principales marcos de facturación europeos (Facturae, UBL, CII) y latinoamericanos (CFDI, NFE).

## Contenido Detallado

### ¿Qué es billripper?

billripper es una biblioteca CLI y SDK (Node.js, Python) que convierte documentos de facturación en JSON estructurado. A diferencia de herramientas genéricas de OCR, billripper aplica reglas de negocio específicas por país y tipo de documento, permitiendo validar campos críticos (RFC, NIF, CIF, IVA, totales) y generar registros listos para integración contable (ERP, API REST, bases de datos). Ideal para startups fintech, marketplaces y equipos de automatización que procesan cientos de facturas al día.

### Características Principales

1. **Parser Híbrido con Machine Learning Heurístico**  
   No se limita a OCR plano. billripper combina reconocimiento de patrones (regex, reglas de negocio) con modelos ligeros de clasificación de campos (TensorFlow Lite) para identificar correctamente proveedor, fecha, importe e IVA incluso en facturas con maquetación irregular o rotaciones. La tasa de acierto en campos críticos supera el 98% tras el entrenamiento personalizado.

2. **Validación Fiscal Multi-País en Tiempo Real**  
   Cada factura extraída pasa por un motor de reglas que verifica formatos fiscales (RFC mexicano, NIF español, VAT europeo), consistencia de totales (base + IVA + retenciones) y detección de facturas duplicadas mediante hash criptográfico del contenido. billripper genera alertas automáticas si detecta discrepancias o facturas sospechosas.

3. **Exportación a 10+ Formatos Contables**  
   No te quedas atrapado en un formato. billripper exporta los datos estructurados a XLSX, CSV, JSON, XML (UBL 2.1, Facturae 3.2), y directamente a APIs de ERPs como SAP, Oracle Netsuite, Xero, QuickBooks y Odoo mediante conectores nativos. Incluye mapeo de campos personalizable para adaptarse a tu esquema contable.

### Cómo Usar

#### 1. Instalación y Configuración Inicial del Cliente CLI

```bash
npm install -g billripper-cli
billripper init --region=es --lang=es --tax-format=NIF
```

Este comando descarga el modelo de reglas para España (NIF, IVA, Facturae) y configura el idioma de salida (es). El flag `--tax-format` define el formato fiscal esperado para validación automática. Si trabajas con México, usa `--region=mx --tax-format=RFC`. billripper descarga los diccionarios de validación fiscal correspondientes.

#### 2. Procesamiento por Lotes con Validación y Exportación Directa

```bash
billripper batch --input ./facturas_pendientes/ --output ./resultados/ --export-format=xero --xero-tenant-id=xxx
```

billripper escanea recursivamente el directorio `input`, identifica automáticamente el tipo de documento (PDF, XML, imagen), extrae los campos usando el parser híbrido, valida contra las reglas fiscales configuradas y exporta cada factura como un asiento contable listo para sincronizar con Xero. Si una factura falla validación, se mueve a una carpeta `./errores/` con un log detallado del motivo. El flag `--verbose` muestra en terminal cada campo extraído y su confianza.

#### 3. Integración Programática en Node.js con Pipeline Personalizado

```javascript
const billripper = require('billripper-sdk');

const pipeline = new billripper.Pipeline({
  region: 'es',
  taxFormat: 'NIF',
  validationLevel: 'strict', // 'strict' | 'normal' | 'lax'
  export: {
    format: 'json',
    schema: 'facturae_3_2'
  }
});

const result = await pipeline.process('./factura_ejemplo.pdf');
console.log(result.fields); // { proveedor, nif, fecha, baseImponible, iva, total, ... }
console.log(result.validation); // { valid: true, warnings: ['NIF no coincide con proveedor'] }
```

Este ejemplo muestra cómo integrar billripper en una API o backend. El pipeline permite encadenar transformaciones (limpieza, enriquecimiento con datos de proveedor desde DB) y enviar el resultado a múltiples destinos (API, cola de mensajes, base de datos). La propiedad `result.validation` ofrece un array de advertencias y errores para depuración.

### Mejores Prácticas

1. **Entrenamiento Personalizado para Formatos Recurrentes**  
   Si procesas facturas de un mismo proveedor o un ERP con maquetación fija, crea un perfil de entrenamiento con billripper train. Proporciona 10-20 facturas etiquetadas (anotaciones manuales) y el motor ajustará sus pesos para alcanzar un 99.5% de precisión en ese formato. Esto reduce drásticamente los falsos positivos en campos como el número de factura o la fecha.

2. **Auditoría Continua con Hash de Facturas**  
   Activa el flag `--dedup-hash` en el pipeline de producción. billripper genera un hash SHA-256 del contenido de cada factura (texto extraído + metadatos) y lo almacena en una base de datos ligera (SQLite). Antes de procesar una nueva factura, consulta el hash para evitar duplicados. Esto evita cargos duplicados en contabilidad y es especialmente útil en integraciones con APIs de facturación electrónica donde pueden llegar facturas repetidas.
