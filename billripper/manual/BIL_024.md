# billripper - BIL_024

## Overview

billripper is a high-performance bill parsing and analysis engine built for developers who need to extract, normalize, and automate invoice data at scale. Designed for SaaS platforms, accounting integrations, and enterprise billing workflows, it eliminates manual data entry by ripping through PDF, HTML, and image-based bills with near-perfect accuracy.

## What is billripper?

billripper is not just another OCR wrapper. It's a modular, headless bill intelligence system that combines layout-agnostic parsing with semantic field mapping. Whether you're ingesting vendor invoices, utility bills, or recurring subscription receipts, billripper delivers structured JSON output with zero configuration overhead. It's optimized for CI/CD pipelines, serverless environments, and multi-tenant SaaS backends.

## Core Features

### 1. Multi-Format Layout-Agnostic Extraction
billripper uses a custom spatial parsing engine that doesn't rely on templates. It dynamically detects table structures, line items, tax breakdowns, and metadata fields across PDFs, scanned images, and raw HTML. This means you can throw any bill format at it—from dense corporate invoices to minimalist digital receipts—and get consistent, schema-validated output.

### 2. Semantic Field Mapping & Normalization
Instead of returning raw text blocks, billripper maps extracted data to a unified schema: vendor name, invoice number, due date, subtotal, tax, total, line items, and currency. It handles regional date formats, currency symbols, and tax code variations (VAT, GST, sales tax) out of the box. You can also define custom field extractors via a simple YAML config.

### 3. Headless API with Webhook & CLI Support
Deploy billripper as a REST API, a CLI tool, or a Lambda function. It supports batch processing, webhook callbacks for async workflows, and direct integration with accounting APIs (QuickBooks, Xero, Stripe). The output is always JSON, with confidence scores per field and raw bounding box coordinates for audit trails.

## How to Use

### Step 1: Ingest a Bill via CLI or API
Start by pointing billripper at your bill file. The CLI accepts PDF, PNG, JPG, or HTML input. For batch processing, use the `--batch` flag with a directory path. The engine auto-detects file type and resolution.

```bash
billripper extract ./invoices/invoice_2025_03.pdf --output json
```

For API usage, send a POST request with a base64-encoded file or a public URL:

```bash
curl -X POST https://api.billripper.dev/v1/extract \
  -H "Authorization: Bearer $API_KEY" \
  -F "file=@invoice_2025_03.pdf" \
  -F "schema=default"
```

### Step 2: Validate and Map the Output
The returned JSON includes a `fields` object with extracted values, confidence scores (0–1), and bounding box coordinates. Use the `--validate` flag to run schema checks and auto-correct common mismatches (e.g., swapped vendor name and address). For custom mappings, pass a config file:

```bash
billripper extract invoice.pdf --config ./custom_fields.yaml --validate
```

Example config snippet:

```yaml
custom_fields:
  - name: "purchase_order"
    pattern: "PO\\s*#?\\s*\\d{5,10}"
    fallback: null
```

### Step 3: Automate with Webhooks or Integrations
For production pipelines, set up a webhook endpoint. billripper will POST the parsed result to your URL after each extraction. Combine with a queue (e.g., SQS, RabbitMQ) for high-throughput processing.

```bash
billripper extract batch ./invoices/ \
  --webhook https://your-app.com/webhook/bills \
  --concurrency 4
```

Integrate directly with accounting tools via the built-in connectors:

```javascript
const billripper = require('billripper-sdk');

const result = await billripper.extract('invoice.pdf');
await billripper.pushToXero(result, { tenantId: 'xxx' });
```

## Best Practices

### 1. Pre-process Low-Quality Scans with Image Enhancement
If your bills are scanned at low DPI or contain heavy compression artifacts, run them through a sharpening and deskew step before feeding them to billripper. Use tools like ImageMagick or a lightweight pre-processing Lambda. This boosts field-level confidence scores from ~0.75 to over 0.95, especially for line-item tables.

### 2. Version Your Custom Field Configs in Git
Treat your `custom_fields.yaml` and schema overrides as code. Store them alongside your application source, and tag them with billripper version numbers. This ensures deterministic extraction behavior across deployments and makes rollbacks trivial. For multi-tenant SaaS setups, isolate configs per tenant using environment variables.
