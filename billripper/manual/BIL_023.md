## What is billripper?

billripper is a high-performance, multi-format bill ingestion and analysis engine designed for developers who need to extract, normalize, and audit billing data at scale. It eliminates the manual overhead of parsing PDF, CSV, JSON, and proprietary invoice formats, and provides a unified schema for downstream analytics, compliance checks, and automation.

## Core Features

### 1. Schema-Agnostic Bill Normalization Engine
billripper dynamically maps heterogeneous bill structures (line items, tax codes, discounts, currencies) into a canonical data model using a pluggable transformation pipeline. No more hardcoded parsers for each vendor—just define a mapping rule once and let the engine handle format drift, missing fields, and encoding anomalies.

### 2. Real-Time Anomaly Detection & Audit Trail
Every ingested bill is run through a configurable rule engine that flags discrepancies (e.g., duplicate line items, out-of-range totals, mismatched tax rates) before they hit your ledger. Each event is logged with a deterministic hash, giving you a tamper-proof audit trail for SOC2 and SOX compliance.

### 3. Multi-Tenant Caching with Tiered Retention
billripper leverages a two-tier cache (hot/warm) to serve frequently accessed bill summaries under 10ms, while cold data is compressed and archived to object storage. This architecture is built for SaaS platforms that need to support thousands of tenants without exploding storage costs.

## How to Use

### Step 1: Configure the Ingestion Pipeline
Define a `billripper.yml` manifest that specifies your input sources (S3 bucket, webhook endpoint, or local directory), the expected bill format(s), and any transformation rules. Example snippet:

```yaml
sources:
  - type: s3
    bucket: my-billing-inbox
    prefix: invoices/2025/
    format: pdf
transformations:
  - field: total_amount
    rule: "extract_currency_value"
  - field: tax_code
    rule: "map_to_standard_taxonomy"
```

Run `billripper ingest --config billripper.yml` to kick off the batch.

### Step 2: Query Normalized Bills via CLI or API
Once ingested, use the billripper query interface to filter, aggregate, and export data. For example, to pull all bills with a total above $10,000 in Q1 2025:

```bash
billripper query --filter "total > 10000" --date-range "2025-01-01:2025-03-31" --output csv
```

Or hit the REST endpoint directly: `GET /api/v1/bills?total_gt=10000&start=2025-01-01&end=2025-03-31`

### Step 3: Set Up Automated Alerts & Webhooks
Configure a rule that triggers a webhook when a bill exceeds a predefined threshold or contains a structural error. For instance, to get a Slack notification when a duplicate invoice is detected:

```bash
billripper rules add --condition "duplicate_hash == true" --action "webhook:https://hooks.slack.com/services/xxx"
```

The alert payload includes the bill ID, the offending field, and a diff of the duplicate match.

## Best Practices

### 1. Always Define a Fallback Parser for Legacy Formats
When ingesting bills from multiple vendors, some may still use outdated or non-standard formats. Create a generic `fallback_parser` that extracts raw text and uses regex-based heuristics. This ensures zero dropped invoices during migration or vendor transitions.

### 2. Use Tiered Retention for Cost-Effective Compliance
Set your hot cache retention to 30 days (for active reconciliation) and warm cache to 90 days (for monthly audits). Archive everything older than 90 days to cold storage with a compressed Parquet format. This reduces your S3 costs by up to 70% while still meeting regulatory requirements.
