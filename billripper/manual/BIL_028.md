### What is billripper?

billripper is a production-grade, API-first bill parsing and analysis engine. It is designed to rip through complex, multi-format invoices, receipts, and financial statements—extracting structured data with near-zero latency. Unlike generic OCR wrappers, billripper uses a proprietary, context-aware parsing pipeline that handles everything from line-item tax breakdowns to multi-currency subtotals, making it the go-to tool for fintech, accounting automation, and enterprise expense management.

### Core Features

- **Schema-Agnostic Extraction Engine:** Automatically infers and normalizes bill schemas (JSON, XML, PDF, scanned images) without predefined templates. Uses a hybrid model of deterministic regex patterns and a lightweight transformer-based NLP layer to handle vendor-specific quirks (e.g., "Tax Amount" vs. "VAT" vs. "HST").
- **Real-Time Line-Item Reconciliation:** Enables sub-second matching of parsed line items against purchase orders, contracts, or historical data. Supports fuzzy matching on SKU, description, and unit price, with configurable tolerance thresholds for currency rounding and quantity discrepancies.
- **Multi-Tenant Audit Trail:** Every parse request generates a cryptographically signed event log (SHA-256), including raw input hash, parsing model version, and all transformation steps. This ensures non-repudiation and full auditability for compliance-driven industries (SOX, HIPAA, GDPR).

### How to Use

1. **Initialize the Client with Custom Model Tuning:**
   ```python
   from billripper import Client
   
   client = Client(
       api_key="sk-...",
       model_version="2024.10.1-prod",
       parsing_preset="fintech_high_accuracy",
       fallback_strategy="retry_with_ocr_enhance"
   )
   ```
   This step sets up a secure session, selects the latest production model, and configures a fallback mechanism that automatically re-runs parsing with enhanced OCR if the initial confidence score drops below 92%.

2. **Submit a Batch of Bills with Concurrency Control:**
   ```bash
   curl -X POST https://api.billripper.io/v1/batch/parse \
     -H "Authorization: Bearer $API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
       "documents": [
         {"source": "s3://bills/invoice_20241001.pdf"},
         {"source": "s3://bills/receipt_20241002.png"}
       ],
       "options": {
         "extract_line_items": true,
         "normalize_currency": "USD",
         "custom_fields": ["purchase_order_id", "cost_center"]
       },
       "concurrency": {
         "max_workers": 5,
         "retry_on_timeout": true,
         "timeout_seconds": 30
       }
     }'
   ```
   This command triggers a batch parsing job with explicit concurrency management. The system will automatically rate-limit based on your plan, and retry failed documents up to 3 times with exponential backoff.

3. **Validate and Export Structured Data to Your Data Lake:**
   ```python
   result = client.get_job_result(job_id="job_abc123")
   
   if result.validation_score >= 0.95:
       # Directly stream to Snowflake or S3
       result.to_parquet(
           path="s3://data-lake/bills/processed/",
           partition_by=["vendor_id", "date"],
           compression="snappy"
       )
       print(f"Exported {len(result.documents)} documents.")
   else:
       # Trigger a manual review queue in Slack
       client.trigger_alert(
           channel="#billripper-alerts",
           message=f"Low confidence batch: {result.job_id}"
       )
   ```
   This step demonstrates a production-ready validation gate: only high-confidence results are pushed to your data lake, while low-confidence batches are flagged for human review.

### Best Practices

- **Implement a Tiered Parsing Strategy:** For high-volume environments, route low-complexity bills (e.g., standard recurring invoices from known vendors) through a lightweight, rule-based parser. Reserve the full NLP pipeline for one-off or complex documents. This can cut processing costs by up to 40% while maintaining 99.7% accuracy on routine bills.
- **Use Idempotency Keys for All Write Operations:** Always include an `idempotency_key` in your POST requests to avoid duplicate billing or double-entry errors. This is critical when retrying failed requests or when your application has a distributed architecture. billripper natively supports idempotency keys with a 24-hour window.
