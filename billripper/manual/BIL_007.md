## What is billripper?

**billripper** is a high-performance, multilingual bill extraction and analysis engine built for developers, accountants, and SaaS platforms that need to ingest, normalize, and audit invoices at scale. Unlike generic OCR wrappers, billripper uses a modular pipeline of regex engines, layout parsers, and rule-based validators to rip structured data from PDFs, scanned images, and raw text—without relying on brittle third-party APIs.

---

## Core Features

### 1. Multi-Format Layout Normalizer
Handles everything from single-page PDF invoices to multi-page CSV exports and rotated JPEG scans. The normalizer flattens layouts into a unified coordinate map, then applies format-specific extractors (e.g., `pdfplumber` for native PDFs, `pytesseract` for images). This ensures you get consistent field extraction regardless of source.

### 2. Rule-Based Field Validator with Fuzzy Matching
Each extracted field (vendor name, invoice number, line items, tax totals) passes through a configurable validation layer. You define rules using JSON schema or YAML—e.g., "invoice number must match pattern `INV-\d{6}`". If a field fails, billripper falls back to fuzzy matching (Levenshtein distance) and logs the confidence score. This dramatically reduces false positives in automated workflows.

### 3. Export Pipeline to Multiple Targets
Extracted data is automatically serialized into JSON, CSV, or SQL insert statements. You can pipe output directly into accounting tools (QuickBooks, Xero), databases (PostgreSQL, SQLite), or custom webhooks. No manual mapping required.

## How to Use

### Step 1: Configure the Extraction Profile
Create a `profile.yaml` file that defines your target fields and validation rules. For example:

```yaml
fields:
  - name: invoice_number
    pattern: "INV-\\d{6}"
    required: true
  - name: total_amount
    type: float
    min: 0.01
    max: 999999.99
  - name: vendor_name
    required: true
    fuzzy_threshold: 0.85
```

Then run:

```bash
billripper --profile profile.yaml --input ./invoices/ --output ./parsed/
```

This instructs the engine to scan all files in `./invoices/`, apply the profile, and write structured results to `./parsed/`.

### Step 2: Run a Dry-Run with Debug Logging
Before processing production data, execute a dry-run to inspect extraction confidence:

```bash
billripper --dry-run --debug --input sample_invoice.pdf
```

The debug output will show each field’s extracted value, confidence score, and any validation failures. Use this to tweak your profile patterns or fuzzy thresholds.

### Step 3: Automate with a Cron Job or CI/CD Hook
Once your profile is stable, schedule extraction as a cron job or GitHub Action. Example cron entry (runs every hour):

```cron
0 * * * * /usr/local/bin/billripper --profile /etc/billripper/profile.yaml --input /var/incoming/ --output /var/parsed/ --export-format json
```

For CI/CD, add a step in your pipeline to trigger extraction on new invoice uploads:

```yaml
- name: Extract invoices
  run: billripper --profile .billripper/profile.yaml --input ./uploads/ --output ./data/
```

## Best Practices

### 1. Always Define a Fallback Field for Vendor Names
Vendor names are notoriously inconsistent (e.g., "Acme Corp" vs "Acme Corporation"). Set `fuzzy_threshold: 0.80` and include a `vendor_name_fallback` field that captures the raw string if validation fails. This prevents silent data loss and simplifies manual review.

### 2. Use a Separate Profile per Document Type
If you process invoices from multiple vendors or countries, create distinct profiles (e.g., `profile_amazon.yaml`, `profile_sap.yaml`). Use the `--detect-layout` flag to auto-select the correct profile based on document structure. This reduces false positives and speeds up extraction by 40–60%.
