### What is billripper?

billripper is a high-performance, multi-format bill parsing and analysis engine engineered to extract, normalize, and validate invoice data from PDFs, scanned images, and structured text. Unlike generic OCR tools, it leverages custom-trained NLP models and heuristic fallbacks to handle edge cases like rotated documents, multi-language headers, and non-standard tax codes. It’s built for CI/CD pipelines, RESTful APIs, and batch processing at scale.

### Core Features

- **Adaptive Schema Mapping Engine**: Automatically maps extracted fields (e.g., invoice number, line items, tax breakdown) to a normalized JSON schema, even when source layouts vary wildly between vendors. Supports custom field injection via YAML configs for legacy systems.
- **Multi-Layer Validation Pipeline**: Combines regex-based pattern matching, checksum verification (e.g., VAT IDs, IBAN), and ML confidence scoring to flag discrepancies. Outputs a detailed audit trail for each field, including confidence levels and alternative candidates.
- **Headless Batch Processing Mode**: Orchestrate bulk parsing of 10,000+ invoices via CLI or gRPC without a GUI. Integrates with S3, Azure Blob, or local filesystems, and emits structured logs for monitoring with Prometheus or Datadog.

### How to Use

1. **Initialize the Parser with Custom Schemas**: Clone the repo and run `billripper init --config ./schemas/vendor_a.yaml`. This loads a YAML schema defining field mappings, validation rules, and fallback extraction strategies. For example, map `total_amount` to a regex that handles both `$1,234.56` and `1.234,56 €`. Ensure your schema includes a `confidence_threshold` (default: 0.85) to filter low-confidence extractions.

2. **Execute Batch Parsing from CLI**: Deploy a batch job with `billripper parse --input ./invoices/ --output ./results/ --format json --parallel 8`. This spins up 8 worker threads to process PDFs concurrently. Use `--watch` to monitor a directory for new files in real-time. The tool auto-detects file encoding (UTF-8, Latin-1) and falls back to OCR via Tesseract if text extraction fails.

3. **Validate and Export with Audit Trails**: After parsing, run `billripper validate --input ./results/ --audit ./audit_logs/`. This cross-references extracted fields against your schema’s validation rules (e.g., `tax_id` must match a checksum algorithm). Generate a CSV report with `--export-csv` that includes each field’s confidence score, extraction source (OCR vs. text), and any warnings. Integrate the output into your ERP via the included Python SDK.

### Best Practices

- **Use Schema Versioning for Compliance**: When dealing with regulatory changes (e.g., EU e-invoicing directives), maintain separate YAML schemas per jurisdiction and version. Tag each schema with a `valid_from` date, and use `billripper validate --schema-version 2024-q3` to enforce compliance. This avoids silent failures when tax formats shift.
- **Implement Idempotent Batch Processing**: To avoid duplicate entries in your database, configure `--dedup-key` with a composite hash (e.g., `invoice_number + vendor_id + date`). Pair this with a Redis-backed lock to ensure each invoice is processed exactly once, even if the same file is submitted multiple times. This is critical for CI/CD pipelines where retries are common.
