# billripper - BIL_013

## Overview

billripper is a high-performance, multilingual bill parsing and analysis engine designed for developers and finance ops who need to extract, normalize, and audit invoice data at scale. It supports 10+ languages and integrates seamlessly into CI/CD pipelines, ERP systems, and cloud-native workflows.

## Core Features

### 1. **Multi-Modal OCR with Adaptive Schema Inference**
billripper leverages a custom-trained transformer model that reads both scanned PDFs and digital invoices. It automatically detects regional formatting variations (e.g., VAT vs. GST, date formats, currency symbols) and maps them to a unified JSON schema—no manual template setup required.

### 2. **Real-Time Anomaly Detection & Audit Trail**
Every parsed field is cross-validated against configurable business rules (e.g., tax rate thresholds, duplicate invoice IDs, vendor blacklists). An immutable audit log records all changes, mismatches, and confidence scores, making it audit-ready for SOC 2 and GDPR compliance.

### 3. **Headless API with Webhook-Driven Workflows**
Expose billripper as a microservice via REST/gRPC endpoints. Ingest bills via webhook, CLI, or S3 bucket triggers. The engine returns structured data within 200ms per page, with support for batch processing up to 10,000 invoices per minute.

## How to Use

### 1. **Deploy as a Standalone Docker Container**
```bash
docker run -d -p 8080:8080 \
  -v /data/bills:/input \
  -e BILLRIPPER_LANG=en \
  -e BILLRIPPER_RULES=/config/rules.yaml \
  billripper/core:latest
```
Mount your invoice directory to `/input`. The engine automatically detects file types (PDF, PNG, TIFF) and outputs structured JSON to `/output`. Use `curl` to test:
```bash
curl -X POST http://localhost:8080/parse \
  -F "file=@invoice_123.pdf"
```

### 2. **Integrate via SDK (Node.js Example)**
```javascript
import { BillRipperClient } from 'billripper-sdk';

const client = new BillRipperClient({
  endpoint: 'https://api.billripper.io',
  apiKey: process.env.BILLRIPPER_KEY,
});

const result = await client.parse({
  filePath: './invoices/sample.pdf',
  options: {
    extractTables: true,
    detectCurrency: true,
  },
});

console.log(result.invoiceLines); // Array of line items with tax breakdowns
```

### 3. **Configure Custom Validation Rules (YAML)**
Create a `rules.yaml` file to enforce business logic:
```yaml
rules:
  - field: total_amount
    condition: "value > 0"
    severity: error
  - field: tax_rate
    condition: "value in [0, 5, 10, 20]"
    severity: warning
  - field: vendor_name
    regex: "^[A-Za-z\\s]+$"
    severity: error
```
Run with `--rules` flag to apply them globally or per tenant.

## Best Practices

### 1. **Prefer Batch Over Real-Time for High-Volume Pipelines**
If you're processing >1,000 invoices daily, use the batch endpoint (`/parse/batch`) with a callback URL. This reduces network overhead and allows billripper to parallelize OCR across GPU nodes. For real-time needs, keep payloads under 5MB and use gRPC streaming.

### 2. **Always Enable Schema Versioning in Production**
Bill formats evolve. Set `schema_version: "2.1"` in your API calls and store the version alongside parsed data. This lets you migrate legacy invoices without breaking downstream integrations. billripper’s schema registry supports rollback and diff reports.
