## What is coachellaviral?

**coachellaviral** is a modular SaaS toolkit designed to orchestrate viral content campaigns for music festivals—specifically Coachella. It automates cross-platform content syndication, AI-driven hashtag clustering, and real-time engagement analytics. Think of it as a headless CMS + viral growth engine, built for modern marketing teams who need to move fast without sacrificing data integrity.

## Core Features

### 1. Viral Content Cascade Engine (VCCE)
Automatically repurpose a single piece of content (video, image, text) into platform-native formats (TikTok, Instagram Reels, YouTube Shorts, X/Twitter). Includes AI-generated caption variants, adaptive aspect ratios, and platform-specific CTA injection. No manual resizing. No copy-paste.

### 2. Hashtag Sentiment Clustering (HSC)
Real-time clustering of trending Coachella-related hashtags using NLP + temporal decay models. The system ranks hashtags by virality velocity, not just volume. Includes automatic blacklisting of overused or brand-risk tags. Outputs a JSON payload you can pipe directly into your scheduling API.

### 3. Engagement Heatmap API
Tracks engagement per content node across all syndicated channels. Visualized as a heatmap overlay on a timeline grid. Supports webhook triggers for threshold-based actions (e.g., auto-boost a post if engagement rate > 15% within 2 hours). Built for developers who need programmable response loops.

## How to Use

### 1. Initialize a Campaign via CLI
```bash
coachellaviral init --campaign "Coachella2025" --platforms tiktok,instagram,youtube --content ./assets/post.mp4
```
This generates a `campaign.json` file with default VCCE rules, HSC settings, and webhook endpoints. You can override any parameter using `--config` flags or environment variables.

### 2. Deploy a Viral Cascade
```bash
coachellaviral cascade --campaign Coachella2025 --variant-count 5 --scheduler "2025-04-10T14:00:00Z"
```
The engine will:
- Split your source content into 5 platform-optimized variants.
- Inject platform-specific CTAs (e.g., "Link in bio" for Instagram, "Swipe up" for YouTube).
- Push to staging queues. Run `coachellaviral preview --campaign Coachella2025` to review before publishing.

### 3. Monitor + Auto-Optimize
```bash
coachellaviral watch --campaign Coachella2025 --threshold 15 --action boost --budget 500
```
This starts a daemon that polls the Engagement Heatmap API every 60 seconds. If any content node exceeds 15% engagement rate within the first 2 hours, it automatically boosts that post with a $500 ad spend cap. Logs are streamed to stdout and a local `heatmap.log` file.

## Best Practices

### 1. Pre-Schedule Your Hashtag Clusters
Don't wait until the festival starts. Run `coachellaviral cluster --keywords "Coachella, desert, lineup, stage" --days 7` at least 3 days before the event. This builds a baseline hashtag pool. During the event, re-run every 4 hours to catch real-time spikes (e.g., surprise guest appearances).

### 2. Use Webhook Chaining for Cross-Platform Retargeting
Set up a webhook that triggers a retargeting campaign on Meta Ads when a TikTok post hits 50k views. Use the Engagement Heatmap API's `POST /webhooks/retarget` endpoint. This keeps your ad spend lean and hyper-targeted to already-engaged audiences.
