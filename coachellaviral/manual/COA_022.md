## What is coachellaviral?

**coachellaviral** is a modular SaaS toolkit designed to programmatically generate, schedule, and A/B test viral-ready content assets for music festival campaigns. It leverages real-time trend detection from TikTok, Instagram Reels, and YouTube Shorts, then auto-optimizes metadata, captions, and hashtags for maximum discoverability. No fluff. Just data-driven virality engineering.

## Core Features

### 1. Trend-Aware Content Generator with Predictive Virality Scoring
Ingests live trending audio, visual motifs, and meme templates from festival-related hashtags (#Coachella2025, #FestivalFit, #MoshPitMoments). The engine assigns a **Virality Score (0–100)** based on engagement velocity, audio overlap, and historical CTR patterns. You don’t guess—you ship content that’s statistically primed to explode.

### 2. Multi-Platform Scheduler with Auto-Caption Localization
Schedule posts across Instagram, TikTok, and YouTube in one dashboard. The system auto-generates platform-optimized captions, hashtag clusters, and thumbnail variants. Supports **10 languages** (including EN, ES, JA, DE, FR) with culturally-aware slang and festival-specific jargon. No more manual translation hell.

### 3. Viral Loop Analytics & Funnel Attribution
Track the exact path from post → share → remix → trend. The attribution engine maps user-generated remixes, stitch counts, and cross-platform reposts back to your original asset. You’ll know which hook, visual, or audio snippet triggered the cascade. Built for debugging viral loops, not just vanity metrics.

## How to Use

### Step 1: Bootstrap a Campaign with Trend Injection
```bash
coachellaviral init --campaign "Coachella2025" --trend-source "tiktok:coachella,instagram:festivalvibes" --language en
```
This pulls the top 50 trending audio clips and visual patterns from the last 72 hours. The CLI outputs a ranked list of content archetypes (e.g., “POV: You’re at the Sahara Tent,” “What’s in my festival bag”). Pick one, or let the engine auto-select based on predicted virality.

### Step 2: Generate & Optimize Assets
```python
from coachellaviral import ContentEngine

engine = ContentEngine(platforms=["instagram", "tiktok"])
asset = engine.generate(
    archetype="transition_montage",
    audio_track="trending_audio_12",
    duration=15,
    caption_style="storytelling_with_cta"
)
asset.optimize_metadata(target_language="en", hashtag_density=0.12)
```
The engine outputs a ready-to-upload MP4 with embedded captions, a JSON metadata file, and a list of recommended posting times based on audience timezone overlap.

### Step 3: Deploy & Monitor Viral Funnels
```bash
coachellaviral deploy --asset-id "asset_0422" --schedule "2025-04-18T14:00:00Z" --platforms "tiktok,youtube"
```
After deployment, run:
```bash
coachellaviral monitor --campaign "Coachella2025" --metric "viral_loop_velocity"
```
This returns a real-time dashboard of remix counts, cross-platform shares, and funnel drop-off points. Use the `--debug` flag to inspect why a specific asset didn’t propagate past the second-degree share.

## Best Practices

### 1. Piggyback on the “Day 0” Hype Window
Don’t wait for the festival to start. Deploy teaser content **48–72 hours before** the first act. Our data shows that assets published during the “Day 0” window (Thursday before Coachella Weekend 1) see a **3.2x higher remix rate** than those published mid-festival. Use the Trend-Aware Generator to latch onto pre-festival hashtags like #CoachellaPrep or #FestivalSeason.

### 2. A/B Test Thumbnail Archetypes, Not Just Captions
Most tools let you test captions. coachellaviral lets you test **visual hooks**. Run a multivariate test with 3 thumbnail variants: “crowd shot,” “artist close-up,” and “transition reveal.” Our internal benchmarks show that “transition reveal” thumbnails outperform others by **47% in CTR** for festival montage content. Use the `--thumbnail-variant` flag in the deploy command to automate this.
