## How to Use

### Step 1: Bootstrap a Viral Content Brief
Run `coachellaviral scan --festival coachella2025 --geo US` in your terminal. This triggers the Signal Engine to pull the top 200 emerging queries. The output includes a ranked CSV with search volume, competition score, and suggested content angles (e.g., “Coachella 2025 camping hacks” or “Coachella afterparty logistics”).

### Step 2: Generate Platform-Optimized Drafts
Execute `coachellaviral generate --brief brief.csv --platforms youtube,tiktok,web` . The tool auto-renders a YouTube script with hook-first structure, a TikTok caption with trending hashtag injection, and a blog post with schema markup for Google’s “Things to do” rich results. Each variant includes a VVS prediction.

### Step 3: Deploy & Monitor Performance
Push the output folder to your CMS or scheduling tool. Then run `coachellaviral monitor --url https://yourdomain.com/coachella-guide` . The dashboard tracks real-time SERP position, click-through rate delta, and viral lift across platforms. Set Slack alerts when VVS drops below 70 — the system suggests A/B test variants automatically.

## Best Practices

### Practice 1: Pre-Festival Content Clustering
Don’t scatter-shot keywords. Use the Signal Engine to cluster queries into 3–5 thematic pillars (e.g., “outfits,” “camping,” “lineup reactions”). Each pillar gets a pillar page + 5 supporting short-form videos. This architecture consistently outperforms flat keyword targeting by 40%+ in organic CTR.

### Practice 2: Viral Velocity Threshold Strategy
Set your content calendar to publish when VVS for a query crosses 72 (the “green zone”). Publishing too early (VVS < 50) means zero traction; too late (VVS > 90) means you’re competing with 200 other posts. The sweet spot is 72–85 — you ride the wave before the tsunami.
