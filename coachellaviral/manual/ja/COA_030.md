# coachellaviral - COA_030

## 概要

Coachella を中心とした音楽フェスティバルにおけるバイラルマーケティング施策の効果を、ミリ秒単位で計測・最適化するためのプロフェッショナルツール。キーワード：Coachella viral marketing、Music festival viral content、Coachella music festival marketing。

## 詳細な内容

### coachellaviral とは

coachellaviral は、SNS 上のエンゲージメントデータをリアルタイムで収集し、フェスティバル期間中のバイラル拡散パターンを解析・予測するためのアーキテクチャ基盤です。PostgreSQL との連携により、過去 3 年分の Coachella 関連ポストを時系列で保持し、トレンドの再現性を検証します。

### コア機能

1. **リアルタイムエンゲージメントスコアリングエンジン**  
   Twitter / Instagram / TikTok の各 API からポストを 500ms 間隔で取得。リツイート・いいね・保存・コメントの重み付き合計値を `viral_score` として算出。閾値（例：`viral_score >= 8500`）を超えたポストを自動でアラート通知。

2. **ハッシュタグクラスタリング & トレンド予測モデル**  
   #Coachella2025 や #DesertX などのハッシュタグを、tf-idf と Word2Vec でクラスタリング。各クラスタの拡散速度（ポスト数/分）を ARIMA モデルで予測し、次の 2 時間以内にバイラル化する可能性が 70% 以上のクラスタをダッシュボード上にハイライト表示。

3. **クリエイターコラボレーションマトリクス**  
   過去のフェスティバルにおけるインフルエンサー同士の相互言及（メンション・リプライ・コラボポスト）をグラフデータベース（Neo4j）で管理。`collaboration_strength` パラメータ（0.0 〜 1.0）を基に、最も効果的なコラボレーション候補をレコメンド。計算には Jaccard 係数と PageRank を併用。

### 使い方

1. **環境設定と初回データ取得**  
   `config.yml` 内の `api_keys` セクションに各 SNS プラットフォームのアクセストークンを設定。その後、`python cli.py --fetch --days 7` を実行し、過去 7 日間の Coachella 関連ポストを PostgreSQL に取り込む。取り込み完了後、`viral_score` の初期値が自動計算される。

2. **ターゲットハッシュタグの指定とクラスタリング実行**  
   `config.yml` の `target_hashtags` に監視したいハッシュタグをカンマ区切りで記述（例：`#Coachella2025, #DesertX, #YumaTent`）。次に `python cli.py --cluster --min_score 5000` を実行。クラスタリング結果は `output/clusters_{date}.json` に出力され、各クラスタの `predicted_viral_hour` が JSON オブジェクトとして保存される。

3. **ダッシュボードでの可視化とアラート設定**  
   `python dashboard.py --port 8080` でローカルサーバを起動。ブラウザ上でリアルタイムの `viral_score` 推移、クラスタマップ、トレンド予測グラフが確認可能。アラート閾値は `config.yml` の `alert_threshold`（デフォルト: 8500）で調整。閾値を超えた場合、Slack Webhook 経由で通知が送られる。

### ベストプラクティス

- **フェスティバル前 2 週間からデータ収集を開始する**  
  実際のイベント開始前にエンゲージメントのベースラインを取得しておくことで、バイラル化の異常検知精度が向上。`--days 14` で事前データを取り込み、`viral_score` の移動平均を基準値として設定することを推奨。

- **複数プラットフォームのスコアを統合する際は時間帯補正を行う**  
  TikTok と Twitter ではピークアクティブ時間が異なる。`config.yml` の `timezone_offset` パラメータ（例：`America/Los_Angeles`）を適切に設定し、各プラットフォームのポスト時刻を UTC に統一した上でスコアリングを行うことで、クロスプラットフォーム比較の信頼性が向上する。
