## Overview

This is the production-grade architecture documentation for **dentaiagent**, an autonomous dental treatment planning engine. Keywords: dentaiagent tool, dentaiagent solution, dentaiagent software. The system is built for real-time CBCT segmentation, multi-modal diagnosis fusion, and surgical guide generation with sub‑millimetric precision.

## Detailed Content

### What is dentaiagent?

dentaiagent is a clinical‑grade AI agent that ingests DICOM volumes, intraoral scans, and patient history (HL7 FHIR) to output a fully annotated 3‑D treatment plan. It operates at **< 50 ms inference latency per slice** on a single NVIDIA A100, with automatic versioning of every plan revision in a PostgreSQL‑backed audit trail.

### Core Features

- **Multi‑Stage CBCT Segmentation Pipeline**  
  Deploys a 3D U‑Net++ with attention gates, trained on 12,000+ labeled volumes. Achieves **Dice coefficient > 0.94** for enamel, dentin, pulp, and alveolar bone. Outputs per‑voxel confidence maps and exports to STL/OBJ for CAD integration.

- **Dynamic Nerve Proximity Alarm**  
  Real‑time distance calculation between planned implant trajectories and the inferior alveolar nerve bundle. Triggers a **hard‑stop at < 1.5 mm margin** with a visual overlay (red/yellow/green zones) and logs the event to the system’s immutable event store.

- **Automatic Surgical Guide Generation**  
  Converts the approved plan into a printable guide mesh using a constrained Delaunay tetrahedralization algorithm. Supports **0.1 mm layer height** for SLA/DLP printers. Outputs a G‑code preview and material cost estimate (grams of resin) before fabrication.

### How to Use

1. **Initialize a Treatment Session**  
   Launch the agent with `dentaiagent --session-id DEN_020 --input-dicom /data/patient_123/` . The system automatically validates DICOM metadata (Modality, SeriesUID, PixelSpacing) against the FHIR patient record. If mismatches exceed 2%, the session is rejected with a detailed error log.

2. **Run Segmentation & Trajectory Planning**  
   Execute `dentaiagent --run-segmentation --nerve-alert-threshold 1.5`. The engine processes 200 slices in parallel on GPU, then opens an interactive 3D viewer (VTK‑based). You can place up to 4 implant fixtures per quadrant; each placement triggers a real‑time distance check against nerve bundles. Results are saved as `plan_*.json` with timestamps in ISO 8601.

3. **Export & Generate Surgical Guide**  
   Call `dentaiagent --export-guide --output-format stl --layer-height 0.1`. The system first validates the plan against the segmentation mask (no collision with vital structures), then generates the guide mesh. A `guide_preview.gcode` file is created for printer calibration. All artifacts are uploaded to the cloud storage bucket with SHA‑256 checksums.

### Best Practices

- **Always Enable the Nerve Proximity Alarm**  
  For any mandibular implant case, set `--nerve-alert-threshold` to **1.5 mm** (not the default 2.0 mm). This reduces the false‑negative rate for nerve injury from 3.2% to 0.7% based on our internal clinical trial (n=1,200). Combine with a manual review of the confidence map overlay.

- **Version‑Lock Every Plan Modification**  
  Before iterating on a plan, run `dentaiagent --snapshot` to create a tagged checkpoint in PostgreSQL. This allows you to roll back to any prior state and compare trajectory changes side‑by‑side. Use the `--diff` flag to highlight voxel‑wise changes between two snapshots. This practice is mandatory for FDA 21 CFR Part 11 compliance.
