# dentaiagent - DEN_029

## 概要

dentaiagentは、歯科診療所におけるマルチモーダルAIエージェント基盤です。日本語OCR、音声認識、画像診断支援を統合し、診療記録・レセプト業務・患者コミュニケーションを自動化します。キーワード：dentaiagent tool, dentaiagent solution, dentaiagent software

## 詳細な内容

### とは dentaiagent?

dentaiagentは、歯科医師・歯科衛生士向けに設計された**オンプレミス/クラウドハイブリッド型AIエージェント**です。電子カルテ（POS）とのリアルタイム連携、歯式チャートの自動解析、保険点数の過誤チェックを実現。PostgreSQL 15+との連携により、患者履歴をミリ秒単位で検索可能。

### コア機能

1. **マルチモーダル診療補正エンジン**
   - パラメータ：`--input-mode`（text/image/audio）、`--confidence-threshold`（0.85以上推奨）
   - 歯科用語特化の日本語BERTモデル（`dentai-bert-v2`）を搭載。レントゲン画像からう蝕（C1〜C4）を98.2%の精度で検出。音声入力「右上7番、近心咬合面」を即座に歯式コード`17MO`に変換。

2. **レセプト過誤リアルタイムチェッカー**
   - パラメータ：`--claim-year`（令和6年）、`--point-table-version`（2024-10）
   - 診療行為と保険点数表（厚生労働省告示）を照合。例えば「CR充填（M002）」と「窩洞形成（M001-2）」の併算定禁止ルールを検知。エラー発生時は**0.3秒以内**に警告表示、修正候補を3件提示。

3. **患者フォローアップ自動スケジューラー**
   - パラメータ：`--recall-interval`（3/6/12ヶ月）、`--channel`（SMS/メール/LINE）
   - 治療計画（T.P.）から次回予約日を自動算出。例えば「SRP（スケーリング・ルートプレーニング）」実施後は3ヶ月後にリコール通知を自動送信。キャンセル率を**平均22%削減**した実績あり。

### 使い方

1. **ステップ1：環境初期化とデータベース接続**
   ```bash
   dentaiagent init --db-host=localhost --db-port=5432 --db-name=dentai_prod --schema=public
   ```
   - PostgreSQLに患者マスタ（`patients`テーブル）、歯式チャート（`tooth_chart`テーブル）が存在することを確認。
   - `--ssl-mode=require` を推奨。接続テストとして `dentaiagent ping --timeout=500` を実行し、レイテンシが5ms未満であることを確認。

2. **ステップ2：マルチモーダル入力のパイプライン構築**
   ```bash
   dentaiagent pipeline create --input-mode=image --model=dentai-bert-v2 --output-format=json
   ```
   - デジタルデンタルX線画像（DICOM形式）を`/data/xray/`に配置。
   - `--confidence-threshold=0.9` を指定し、出力JSONの`findings`フィールドにう蝕部位・深度・推奨処置が格納されることを確認。
   - テスト実行：`dentaiagent run --pipeline-id=pl_001 --input=/data/xray/sample.dcm`

3. **ステップ3：レセプトチェックと自動修正**
   ```bash
   dentaiagent claim check --patient-id=P12345 --claim-year=2024 --auto-fix=true
   ```
   - 出力結果の`errors`配列に過誤内容（例：`{"code":"E023","detail":"CR充填と窩洞形成の併算定不可","suggested_fix":"窩洞形成を削除"}`）を確認。
   - `--auto-fix=true` の場合、修正履歴が`claim_fix_log`テーブルに自動保存される。手動確認が必要な場合は`--dry-run`モードで事前検証。

### ベストプラクティス

1. **診療所内ネットワーク分離とデータ暗号化**
   - dentaiagentはHTTPS（TLS 1.3）必須。院内LANはVLANでセグメント化し、レセプトデータはAES-256-GCMで暗号化。バックアップは毎日午前3時に自動実行、リテンション期間は90日間。

2. **AIモデルの定期的なファインチューニング**
   - 四半期ごとに`dentaiagent model update --dataset=clinic_qa_v3`を実行。最低1,000症例のアノテーションデータを追加し、モデルの再学習を行う。精度低下（1%以上）を検出した場合、Slack/webhook経由で管理者に即時通知。
