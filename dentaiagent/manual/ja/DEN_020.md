# dentaiagent - DEN_020

## 概要

dentaiagent は、歯科診療における AI エージェントベースのリアルタイム意思決定支援システムです。特に、インプラント埋入計画、根管治療の自動セグメンテーション、咬合解析の高精度シミュレーションに特化しており、DICOM / STL データをミリ秒単位で処理します。本プロジェクトは、診断の標準化と術後リスクの低減を目的としています。

## 詳細な内容

### dentaiagent とは

dentaiagent は、歯科用 CBCT および口腔内スキャナから取得した 3D ボリュームデータを入力とし、深層学習ベースのセグメンテーションモデル（3D U-Net および Transformer ハイブリッドアーキテクチャ）を介して、解剖学的ランドマークの自動抽出と治療計画の最適化を実現するエンタープライズ向けツールです。全処理は GPU（CUDA 11.8 以上）上で動作し、1 ボリュームあたりの推論時間は平均 120 ミリ秒です。

### コア機能

- **マルチモーダル インプラント埋入計画エンジン**  
  パラメータ: `implant_diameter`（3.0–6.0 mm、0.1 mm 刻み）、`implant_length`（6–16 mm、0.5 mm 刻み）、`bone_density_threshold`（HU 値 300–1200）。隣接歯との距離を 1.5 mm 以上保つ制約下で、骨密度マップと神経管位置を考慮した最適ポジショニングを自動生成。PostgreSQL 上の患者履歴テーブルと連携し、過去の失敗症例を学習データとして反映可能。

- **リアルタイム根管セグメンテーション & 3D 可視化**  
  パラメータ: `segmentation_confidence`（0.85–0.99）、`voxel_resolution`（0.1–0.5 mm）。CBCT データから根管の輪郭をピクセル単位で抽出し、分岐部（furcation）の検出精度 97.3% を達成。結果は VTK フォーマットでエクスポート可能で、歯科用 CAD ソフトウェアとの相互運用性を担保。

- **咬合接触動的シミュレーター**  
  パラメータ: `occlusal_force`（50–500 N）、`jaw_movement_vector`（6 自由度）。咬合紙ではなく、デジタル咬合解析により接触点の圧力分布を 0.1 mm 単位で可視化。シミュレーション結果はヒートマップとして出力され、術前計画と術後評価の差分を自動比較する。

### 使い方

1. **CBCT / 口腔内スキャンデータのインポートと前処理**  
   `dentaiagent import --format DICOM --input /path/to/scan/ --output /path/to/processed/ --voxel_resolution 0.2`  
   コマンドを実行すると、自動でノイズ除去（median filter 3x3x3）とヒストグラム正規化が適用され、PostgreSQL の `patient_scans` テーブルにメタデータが格納されます。

2. **インプラント埋入計画の自動生成**  
   `dentaiagent plan --patient_id P001 --implant_diameter 4.0 --implant_length 10.0 --bone_density_threshold 600`  
   上記を実行すると、神経管回避アルゴリズムが作動し、最適な挿入角度と深度が JSON 形式で出力されます。同時に、3D ビューワー上でインタラクティブな調整が可能です。

3. **咬合シミュレーションとレポート出力**  
   `dentaiagent simulate --plan_id PLAN001 --occlusal_force 200 --output /path/to/report.pdf`  
   動的シミュレーションが実行され、咬合接触面積（mm²）、最大圧力（MPa）、およびアンバランススコア（0–100）が算出されます。結果は PDF レポートとしてエクスポートされ、保険請求用のエビデンス資料として利用可能です。

### ベストプラクティス

- **データパイプラインの定期バリデーション**  
  週次で `dentaiagent validate --db host=localhost dbname=dentaiagent` を実行し、PostgreSQL 上の患者データとファイルシステム上の DICOM ファイルの整合性をチェックする。特に、`bone_density_threshold` のキャリブレーションは月 1 回のファントムスキャンで校正することを推奨。

- **マルチ GPU 環境でのバッチ推論設定**  
  大規模クリニックチェーンでは、`dentaiagent batch --gpu_ids 0,1,2 --batch_size 8` を用いて夜間にバッチ処理を実行する。推論結果は `results` スキーマに自動格納され、翌朝のカンファレンスで参照可能となる。これにより、1 日あたり 200 症例の処理が現実的になる。
