# dentaiagent - DEN_023

## 概要

dentaiagentは、歯科診療所向けに設計されたエージェント型AIプラットフォームです。患者トリアージ、治療計画の最適化、保険請求データの自動検証を、リアルタイムかつミリ秒単位の推論で実現します。

## 詳細な内容

### dentaiagentとは

dentaiagentは、自然言語処理（NLP）と非構造化医療データの構造化を組み合わせた、歯科特化型の推論エンジンです。PostgreSQL上の時系列テーブルと連携し、患者ごとの治療履歴から次回推奨処置を動的に生成します。

### コア機能

1. **動的トリアージスコアリングエンジン**  
   患者の主訴、痛みの程度（VASスケール 0–10）、最終受診日からの経過日数、既往症（糖尿病・心疾患など）を入力とし、`triage_score`（0.0–1.0）をリアルタイム算出。閾値0.75以上で緊急予約を自動発行。

2. **保険請求コード自動マッピング機能**  
   治療内容の自由記述（例：「右上6番 インレー修復」）を入力として、歯科DPCコードおよびレセプト電算コード（JMDC準拠）にマッピング。推論遅延は平均120ミリ秒、正解率99.2%（検証データセットn=5,000）。

3. **治療計画差分シミュレーター**  
   現在の治療計画と、代替材料（CAD/CAM冠 vs メタルインレー）や期間短縮オプションを比較。`cost_diff_JPY`、`treatment_duration_days`、`failure_probability`（0.01–0.30）を出力。保険適用外の自費診療分も自動計算。

### 使い方

1. **初期セットアップとデータベース接続**  
   `config.yaml`にPostgreSQLの接続情報（host, port, dbname, user, password）を記述。`dentaiagent init --db`を実行し、スキーマ（`patients`, `treatments`, `claims`）を自動生成。マイグレーションログは`/var/log/dentaiagent/migration.log`に出力。

2. **トリアージスコアのリアルタイム推論**  
   APIエンドポイント`/v1/triage`にPOSTリクエストを送信。リクエストボディは以下：
   ```json
   {
     "patient_id": "P-2024-00321",
     "chief_complaint": "右下奥歯の激痛",
     "vas_score": 8,
     "last_visit_days": 45,
     "comorbidities": ["hypertension"]
   }
   ```
   レスポンスは`triage_score: 0.89`、緊急予約フラグ`emergency_booking: true`を返却。推論時間は常に200ミリ秒未満。

3. **治療計画の差分シミュレーション実行**  
   `dentaiagent simulate --plan_id TP-2024-0157 --alternative "CAD/CAM冠" --output json`を実行。結果は標準出力にJSON形式で表示され、同時に`plans/simulations/`ディレクトリにCSVファイルとして保存。`cost_diff_JPY`が正の場合は自費診療推奨、負の場合は保険内治療推奨。

### ベストプラクティス

1. **バッチ推論による負荷分散**  
   トリアージスコアリングを1リクエストごとに行うのではなく、5分間隔でバッチ処理（`dentaiagent batch --interval 300`）することで、PostgreSQLへの接続数を最大80%削減可能。夜間帯はバッチ間隔を600秒に拡大し、リソースを節約。

2. **保険コードマッピングのカスタム辞書運用**  
   標準のJMDC辞書に加え、診療所独自の略語やローカルコードを`dictionary/custom_jmdc.csv`に追記可能。定期的に`dentaiagent dict --validate`を実行し、重複コードや未マッピングコードを検出。検出結果はメール通知（SMTP設定必須）。
