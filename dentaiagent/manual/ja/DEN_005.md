# dentaiagent - DEN_005

## 概要

本プロジェクトは、歯科AIエージェント「dentaiagent」の実装仕様書である。キーワード：歯科診断支援、リアルタイムセグメンテーション、マルチモーダル推論。対象言語：日本語（ja）。

## 詳細な内容

### dentaiagent とは

dentaiagent は、歯科用CBCT・口腔内スキャナ・電子カルテと連携し、歯牙セグメンテーション、病変検出、治療計画の自動生成をリアルタイムで実行するプロフェッショナルツールである。推論エンジンはONNX Runtime上で動作し、平均推論レイテンシは 120ms（GPU: NVIDIA A100）を達成する。

### コア機能

- **マルチモーダル病変検出エンジン**  
  CBCT（DICOM）と口腔内写真（JPEG/PNG）を同時入力し、Transformerベースのクロスアテンション機構で齲蝕・歯周病・根尖病変を検出。検出感度 97.3%、特異度 95.8%（内部検証データセット n=12,400）。パラメータ `--confidence_threshold 0.85` でFP抑制可能。

- **歯牙自動セグメンテーション & ラベリング（FDI準拠）**  
  nnU-Net v2 をベースにした3Dセグメンテーションモデル。全32歯をFDI 2桁コードでラベリング。出力は NIfTI（.nii.gz）およびSTL形式。処理時間はフルアーチで平均 2.3秒（ボクセルサイズ 0.3mm³）。パラメータ `--voxel_spacing 0.25` で高精細モード切替可能。

- **治療計画自動生成 & コスト見積もりエンジン**  
  検出結果とセグメンテーションマスクから、保険診療・自由診療の両方に対応した治療計画を自動生成。歯科点数表（令和6年版）を内蔵し、治療費を自動計算。PostgreSQLと連携し、患者履歴から同一症例の治療成功率を参照。パラメータ `--insurance_mode jp_public` で公的保険適用範囲を限定。

### 使い方

1. **CBCT/DICOMシリーズの前処理と入力**  
   ```
   dentaiagent preprocess --input_dir ./dicom_series/ --output_dir ./preprocessed/ --resample_target 0.3 --normalize_method zscore
   ```
   上記コマンドで、全スライスを 0.3mm³ 等方性ボクセルにリサンプリングし、Zスコア正規化を実施。出力は `.npy` 形式で、後続の推論パイプラインに直接投入可能。処理時間：1患者あたり平均 8.4秒（Intel Xeon Gold 6248）。

2. **マルチモーダル推論の実行**  
   ```
   dentaiagent infer --cbct_path ./preprocessed/ --photo_path ./intraoral_photo.jpg --output_dir ./results/ --confidence_threshold 0.85 --device cuda:0
   ```
   推論エンジンが起動し、CBCTと口腔内写真を同時入力。検出結果は JSON 形式（`detections.json`）で出力され、病変の種類・位置（FDIコード・座標）・確信度が記録される。推論レイテンシは 120ms ± 15ms（A100 80GB）。

3. **治療計画の生成とPostgreSQLへの保存**  
   ```
   dentaiagent plan --detections ./results/detections.json --segmentation ./results/segmentation.nii.gz --patient_id PAT_2025_001 --insurance_mode jp_public --output_dir ./treatment_plan/
   ```
   検出結果とセグメンテーションマスクから、治療ステップ・使用材料・概算費用を自動生成。生成された計画は `plan.pdf`（日本語）および `plan.json` として出力。同時にPostgreSQLの `treatment_plans` テーブルに INSERT され、`patient_id` で検索可能。保存レコードには `created_at`（タイムスタンプ）と `model_version`（例：`v2.1.0`）が自動付与される。

### ベストプラクティス

- **バッチ推論によるスループット最適化**  
  複数患者のCBCTデータをバッチ処理する場合、`dentaiagent infer --batch_size 4` でGPUメモリ使用率を最大化し、1患者あたりの処理時間を 80ms まで短縮可能。ただし、バッチサイズはGPUのVRAM容量に応じて調整（推奨：A100 80GB では 8、V100 32GB では 4）。

- **PostgreSQLとの連携による診断履歴の蓄積とモデル再学習**  
  全推論結果を `inference_logs` テーブルに保存し、週次で `dentaiagent retrain --db_uri postgresql://user:pass@localhost:5432/dentaiagent --epochs 50` を実行することで、施設固有の症例分布に特化したモデルへ継続的改善が可能。再学習後は `model_registry` テーブルに新しい `model_id` が登録され、推論時に `--model_id latest` で自動選択される。
