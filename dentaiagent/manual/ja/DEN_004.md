# dentaiagent - DEN_004

## 概要

dentaiagentは、歯科業界向けに特化したマルチエージェントシステムです。予約管理、治療計画の最適化、保険請求の自動化をミリ秒単位で処理し、クリニックの業務効率を飛躍的に向上させます。

## コア機能

### 1. リアルタイム予約最適化エンジン
- 患者の通院履歴・治療ステータスを基に、最適なスロットを自動提案
- キャンセル発生時、待機リストから即座に再割り当て（応答速度: 200ms以内）
- パラメータ: `slot_granularity=15min`, `max_waitlist_depth=50`, `reassign_threshold_ms=500`

### 2. 治療計画AIアシスタント
- 過去の症例データ（PostgreSQL格納）と最新の歯科ガイドラインを照合し、推奨治療シーケンスを生成
- 保険適用範囲と患者自己負担額を同時計算（誤差率0.01%未満）
- パラメータ: `guideline_version=2025.03`, `cost_tolerance=0.0001`, `max_plan_variants=5`

### 3. 保険請求自動監査・修正機能
- レセプトデータをリアルタイム解析し、不備・過不足を検出
- 修正提案を自動生成し、請求サイクルを従来比60%短縮
- パラメータ: `audit_batch_size=1000`, `correction_confidence=0.95`, `cycle_time_target_hours=4`

## 使い方

### ステップ 1: データベース接続と初期設定
```bash
docker-compose up -d postgres
python manage.py init_db --schema=dentai_v3 --partition_by=clinic_id
```
- PostgreSQL 15 + pg_partman で月次パーティショニング
- テーブル: `appointments`, `treatment_plans`, `insurance_claims`

### ステップ 2: エージェント設定ファイルの作成
```yaml
# config/agents.yaml
scheduler:
  model: "dentai-scheduler-v2"
  inference_timeout_ms: 150
  slot_rules: "rules/slot_priority.yaml"
advisor:
  model: "dentai-advisor-v3"
  guideline_path: "/data/guidelines/2025-03"
  db_connection_pool: 20
auditor:
  model: "dentai-auditor-v1"
  claim_schema: "jp_medical_2025"
  alert_on_correction: true
```

### ステップ 3: エージェント起動とモニタリング
```bash
python agents/run.py --config config/agents.yaml --mode production
```
- 起動後、`/metrics` エンドポイントで各エージェントのレイテンシ・スループットをPrometheusで監視
- 異常検知時はSlack Webhookで通知（`alert_endpoint` 設定必須）

## ベストプラクティス

### プラクティス 1: パーティション設計によるクエリ最適化
- `created_at` で週次パーティションを作成し、古いデータは自動圧縮（ZSTD level 3）
- 月次で `VACUUM ANALYZE` を実行し、インデックス断片化を防止
- 推奨: `pg_cron` で毎週日曜日 03:00 にメンテナンスジョブをスケジュール

### プラクティス 2: エージェント間の非同期通信パターン
- 各エージェントはRedis Streamsを介してメッセージを交換
- コンシューマグループで負荷分散し、障害時は自動リバランス
- メッセージ保持期間: 72時間（デバッグ用）、配信確認は `XACK` で管理
