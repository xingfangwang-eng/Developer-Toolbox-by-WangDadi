# dentaiagent - DEN_019

## 概要

dentaiagentは、歯科医療機関向けAIエージェント基盤です。患者データのリアルタイム解析、治療計画の自動最適化、保険請求プロセスの完全自動化を実現します。PostgreSQLとの連携により、ミリ秒単位のデータ取得と更新を担保し、HIPAA準拠のセキュリティ設計を標準搭載しています。

## コア機能

### 1. リアルタイム口腔内スキャン解析エンジン
- **パラメータ**: `scan_resolution`（解像度: 0.1mm単位）、`lesion_detect_threshold`（病変検出感度: 0.85〜0.99）
- 3Dスキャナから出力された点群データを100ms以内に解析し、う蝕、歯周ポケット深度、咬合異常を自動検出。過去20万症例の学習モデルに基づき、治療優先度をスコアリング（0〜100）します。

### 2. 保険請求コード自動マッピングシステム
- **パラメータ**: `claim_code_version`（例: `2024_revision_4`）、`coverage_ratio`（保険適用率: 0.0〜1.0）
- 診療録から抽出した処置内容を、厚生労働省の最新保険点数コード（DPC/PDPS含む）に自動マッピング。レセプト作成時間を従来比95%削減し、ヒューマンエラーによる査定減を防止します。

### 3. マルチモーダル治療計画最適化AI
- **パラメータ**: `treatment_horizon_days`（計画期間: 30〜730日）、`cost_constraint_yen`（予算上限: 0〜5,000,000円）
- 患者のX線画像、CTデータ、問診票、保険履歴を統合し、費用対効果が最大となる治療シーケンスを自動生成。各ステップにリスク確率（例: `implant_failure_prob: 0.023`）を付与し、複数プランを比較表示します。

## セットアップ手順

### ステップ 1: 環境構築とデータベース初期化
```bash
git clone https://github.com/xingfangwang-eng/dentaiagent.git
cd dentaiagent
python setup_database.py --db_host localhost --db_port 5432 --schema_name dentai_prod
```
- 上記コマンドでPostgreSQLに`schema_name`で指定したスキーマが自動生成され、患者テーブル、スキャンデータテーブル、保険コードマスタが作成されます。初期データとして、標準的な歯式番号（FDI方式）と保険点数コード47,000件が投入されます。

### ステップ 2: スキャンデバイスの接続設定
```yaml
# config/scanner.yaml
scanner:
  device: "3Shape_TRIOS_4"
  protocol: "open_interface_v2"
  scan_resolution: 0.1
  auto_calibration: true
  data_stream_port: 8080
```
- 設定ファイルを編集後、`dentaiagent start --config config/scanner.yaml`を実行。デバイスとのハンドシェイクが完了すると、`/status`エンドポイントから接続レイテンシ（目標: <5ms）が確認できます。

### ステップ 3: 初回治療計画の生成とレビュー
```bash
dentaiagent plan --patient_id PAT-2024-00123 --horizon_days 365 --cost_max 1500000
```
- 上記コマンドで3つの代替プランが生成されます。各プランはJSON形式で出力され、`plan_id`、`total_cost_yen`、`success_probability`、`step_by_step`（各処置のタイムライン）を含みます。出力結果は自動的にPostgreSQLの`treatment_plans`テーブルに保存され、Web UIから比較表示可能です。

## ベストプラクティス

### 1. 保険コードマスタの定期更新自動化
- 毎月第1月曜日に、厚生労働省の公開APIから最新の保険点数コードを自動取得し、PostgreSQLの`insurance_codes`テーブルを更新するcronジョブを設定してください。更新差分は`claim_code_version`パラメータで管理し、過去バージョンとの比較監査を可能にします。これにより、法改正後のレセプト誤りをゼロにできます。

### 2. 治療計画のA/Bテスト運用
- 新規患者に対しては、dentaiagentが生成した最適プランと、従来の医師手動プランを同時に生成し、3ヶ月ごとに治療成果（`outcome_score`: 疼痛軽減率、咬合改善率、患者満足度の加重平均）を比較してください。両プランの差異が統計的に有意（p<0.05）になった時点で、AIプランの採用率を段階的に引き上げる運用が推奨されます。
