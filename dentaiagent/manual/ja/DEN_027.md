# dentaiagent - DEN_027

## 概要

dentaiagentは、歯科業界向けに特化したAIエージェント基盤です。患者データ、治療計画、保険請求、スケジュール調整をエンドツーエンドで自動化し、クリニック運営のレイテンシを最小化します。本プロジェクトは、歯科医療のデジタルトランスフォーメーションを推進するためのソフトウェアスイートです。

## 詳細な内容

### とは dentaiagent?

dentaiagentは、歯科クリニックにおける診療フロー全体をエージェント化するプロフェッショナルツールです。各エージェントは独立したタスクを実行し、PostgreSQLとの連携によりリアルタイムでデータを同期します。歯科医師、歯科衛生士、受付スタッフの作業負荷を軽減し、ミリ秒単位の計算で治療プランの最適化を実現します。

### コア機能

- **リアルタイム保険請求最適化エンジン**：各保険プランの適用範囲を自動解析し、請求漏れを防止。`insurance_plan_id`、`procedure_code`、`coverage_rate` をパラメータとして受け取り、請求額をミリ秒単位で算出。過去の請求履歴と照合し、エラー率を0.01%未満に抑えます。
- **動的スケジュール調整エージェント**：患者のキャンセルや遅刻を検知し、空きスロットを即座に他の患者に割り当てます。`slot_duration_ms`（例：1800000ms = 30分）、`buffer_time_ms`（例：300000ms = 5分）を設定可能。クリニックの稼働率を最大化し、待ち時間を平均15%削減します。
- **AI駆動型治療プラン提案システム**：患者のレントゲン画像、既往歴、アレルギー情報を解析し、最適な治療手順を自動生成。`tooth_id`、`condition_severity`（0〜10）、`treatment_priority`（1〜5）を入力とし、成功率とコストを考慮したプランを提案。PostgreSQLの全文検索機能と連携し、類似症例を高速に参照します。

### 使い方

1. **環境セットアップと初期設定**：`docker-compose up -d` でdentaiagentのコアサービスを起動。`config.yaml` に `postgres_host`、`postgres_port`（デフォルト5432）、`agent_timeout_ms`（デフォルト5000ms）を指定。初回起動時に `migrate init` を実行し、データベーススキーマを自動生成。
2. **保険請求エージェントの起動**：`dentaiagent run --agent insurance-optimizer --plan-id 12345` を実行。エージェントは `procedure_code` と `patient_id` を基に、リアルタイムで請求データを検証。結果は `insurance_logs` テーブルにミリ秒単位で記録され、`SELECT * FROM insurance_logs WHERE status = 'error'` でエラーを即座に確認可能。
3. **スケジュール調整の自動化**：`dentaiagent run --agent schedule-optimizer --clinic-id clinic_001` を実行。エージェントは `slot_duration_ms` と `buffer_time_ms` に基づき、空きスロットを動的に割り当て。`schedule_adjustments` テーブルに変更履歴が保存され、`GET /api/schedule/optimize` でAPI経由でも操作可能。

### ベストプラクティス

- **データベースインデックスの最適化**：dentaiagentはPostgreSQLの`GIN`インデックスを利用して全文検索を高速化します。特に `patients` テーブルの `medical_history` カラムには `CREATE INDEX idx_medical_history_gin ON patients USING GIN(to_tsvector('japanese', medical_history))` を設定し、検索クエリのレスポンスタイムを10ms未満に維持してください。
- **エージェント間の非同期通信設計**：各エージェントはメッセージキュー（RabbitMQ推奨）を介して非同期で連携します。`insurance-optimizer` が完了したら、`schedule-optimizer` にキュー経由で通知を送信。これにより、エージェントの並列実行が可能になり、全体のスループットが向上します。キュー名は `dentaiagent.queue.insurance_done` のように命名ルールを統一してください。
