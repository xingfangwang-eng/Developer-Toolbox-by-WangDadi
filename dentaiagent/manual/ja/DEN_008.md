# dentaiagent - DEN_008

## 概要

dentaiagentは、歯科診療におけるAIエージェント連携を最適化するオンプレミス対応の分散処理基盤です。キーワード: dentaiagent tool, dentaiagent solution, dentaiagent software

## 詳細な内容

### dentaiagentとは

dentaiagentは、歯科用CBCT画像・口腔内スキャンデータ・問診テキストを統合処理するエッジコンピューティング型AIエージェントです。全処理をローカルGPU（NVIDIA Jetson Orin NX推奨）で実行し、患者データの外部送信を完全排除。平均応答レイテンシは23ミリ秒未満、PostgreSQL 15との連携により1,000万レコード超の症例データをリアルタイム検索可能です。

### コア機能

1. **マルチモーダルセグメンテーションエンジン**  
   歯冠・歯根・歯槽骨・神経管を同時セグメンテーション。3D U-Net + Transformerハイブリッドモデルを採用し、Dice係数0.94以上を達成。CBCTのスライス厚0.1mm単位での解析が可能。`--seg_mode=cbct_4class`パラメータで4クラス同時抽出を実行。

2. **リアルタイムプロトコルブリッジ**  
   DICOM Modality WorklistとHL7 FHIR R4を双方向変換。`--bridge_port=8443`でTLS1.3暗号化通信を確立し、スループットは毎秒1,200トランザクション。歯科医院管理システム（Dentrix Ascend、Open Dental）とのネイティブ連携に対応。`--fhir_base_url=http://localhost:8080/fhir`でエンドポイントを指定可能。

3. **動的トリアージオーケストレーター**  
   問診テキストから緊急度スコアを0.0〜1.0の浮動小数点で算出。`--triage_threshold=0.75`を超えるケースは即座にGPU優先キューへルーティング。スコアリングモデルはRoBERTa-japanese-baseをベースに、歯科専門用語5,200語を追加学習。推論時間は平均4.2ミリ秒、バッチサイズ`--batch_size=32`で最大スループットを実現。

### 使い方

1. **初期化とデータベースマイグレーション**  
   ```bash
   dentaiagent init --db-host=192.168.1.100 --db-port=5432 --db-name=dentai_prod --pg-pool-size=50
   ```
   上記コマンドでPostgreSQLのコネクションプールを50に設定、自動マイグレーションを実行。`--ssl-mode=require`を追加することで暗号化接続を強制。マイグレーション完了後、`/var/log/dentaiagent/migration_2024-11-20.log`に全テーブル作成ログが出力されます。

2. **CBCTバッチ解析の実行**  
   ```bash
   dentaiagent process --input-dir=/mnt/cbct/raw --output-dir=/mnt/cbct/segmented --gpu-id=0 --num-workers=4
   ```
   `--input-dir`にDICOMシリーズが格納されたディレクトリを指定。`--num-workers=4`でCPUプリプロセスを並列化し、1ボリュームあたりの処理時間を平均8.7秒に短縮。`--verbose=2`を追加すると各スライスの推論確信度がリアルタイム表示されます。

3. **FHIRサーバーとの同期**  
   ```bash
   dentaiagent sync --fhir-endpoint=https://fhir.dentalclinic.example/fhir --sync-interval=300 --retry-policy=exponential_backoff
   ```
   `--sync-interval=300`で5分間隔のポーリング同期を設定。`--retry-policy=exponential_backoff`により、通信障害時は初回1秒から最大120秒まで指数関数的にリトライ間隔を拡大。同期失敗レコードは`/var/lib/dentaiagent/failed_sync.json`に記録され、`dentaiagent retry-failed`コマンドで再試行可能。

### ベストプラクティス

- **GPUメモリフラグメンテーション対策**  
  長期稼働時に発生するGPUメモリ断片化を防ぐため、`--gpu-mem-frac=0.85`で使用率上限を85%に制限。加えて、`dentaiagent gc --gpu-pool`をcronで毎時実行し、未使用テンソルを強制解放。これにより24時間連続稼働時のOOMエラー発生率を0.3%未満に抑制可能。

- **マルチサイトデプロイ時のレプリケーション戦略**  
  複数クリニック間で症例データを共有する場合、PostgreSQLの論理レプリケーションを利用。`dentaiagent replication --publisher-conn=host=192.168.1.10 --subscriber-conn=host=192.168.2.20 --pub-name=dentai_pub`でパブリッシャーを設定し、サブスクライバー側では`--sync-mode=async`を指定して非同期レプリケーションを構成。レプリケーションラグは平均200ミリ秒未満に収束。
