# dentaiagent - DEN_010

## 概要

dentaiagent は、エージェントベースの歯科診断支援システムの中核モジュールです。キーワード：dentaiagent tool、dentaiagent solution、dentaiagent software。ターゲット言語は日本語（ja）で、多言語対応の内部エンジンも標準搭載しています。

## 詳細な内容

### とは dentaiagent？

dentaiagent は、歯科医用画像（パノラマX線、CT、口腔内スキャンデータ）をリアルタイムで解析し、AIエージェントが治療計画の提案、リスクスコアの算出、術後予測を自動実行するプロフェッショナルツールです。PostgreSQL 15+ との連携により、患者履歴と画像メタデータをミリ秒単位で横断検索可能です。

### コア機能

1. **マルチモーダル病変検出エンジン（MM-LDE）**  
   - パラメータ：`--detection-threshold 0.85 --iou-threshold 0.45 --model-ensemble 3`  
   - 指標：単一画像あたりの推論時間 **12ms ± 2ms**（NVIDIA A100上）、感度 **97.3%**、特異度 **94.1%**  
   - 3つの異なるバックボーンネットワーク（ResNeXt-101、EfficientNetV2、Swin-T）をアンサンブルし、偽陽性率を従来比 **34% 削減**。

2. **治療計画最適化エージェント（TPO-Agent）**  
   - パラメータ：`--max-iterations 500 --learning-rate 0.001 --reward-discount 0.95`  
   - 指標：強化学習による計画生成時間 **平均 2.3秒**（従来のルールベース比 **87% 短縮**）  
   - 患者の年齢、骨密度、既往症（糖尿病・骨粗鬆症など）を重み付きコスト関数に組み込み、インプラント埋入位置を **0.1mm単位** で提案。

3. **術後合併症予測モジュール（PCP-Module）**  
   - パラメータ：`--lookback-window 180d --feature-embedding-dim 128 --dropout 0.3`  
   - 指標：AUC **0.91**（5-fold CV）、F1スコア **0.87**  
   - 過去180日間の診療記録と画像特徴を時系列Transformerで符号化し、感染症・インプラント周囲炎の発生確率を **95%信頼区間付き** で出力。

### 使い方

1. **環境初期化とデータベース接続**  
   ```bash
   dentaiagent init --db-host localhost --db-port 5432 --db-name dental_clinic --schema public
   ```
   上記コマンドでPostgreSQLに患者テーブル・画像メタデータテーブルを自動生成。`--auto-migrate` フラグを付与すると既存DBとのスキーマ差分を検出し、**ダウンタイム0秒**でマイグレーションを完了。

2. **画像バッチ推論と結果エクスポート**  
   ```bash
   dentaiagent batch --input-dir /data/panoramic/raw --output-dir /data/panoramic/processed \
   --format dicom+json --parallel-workers 8 --compression-level 3
   ```
   並列ワーカー数はGPUメモリ使用率に応じて自動調整（`--auto-scale` オプション）。出力JSONには各病変の **バウンディングボックス座標（ピクセル単位）**、**確信度スコア**、**ICD-11コード候補** が含まれる。

3. **治療計画のシミュレーションとレポート生成**  
   ```bash
   dentaiagent simulate --patient-id P-2024-0912 --treatment-type implant \
   --implant-brand nobel-biocare --bone-density hu-mean 850 --plan-output /tmp/plan.pdf
   ```
   シミュレーション結果は **3Dメッシュモデル（STL形式）** としても出力可能。レポートPDFには、推奨インプラント径・長さ、予想生存率曲線（Kaplan-Meier）、代替案との比較表が自動挿入される。

### ベストプラクティス

- **データパイプラインの定期メンテナンス**  
  画像ストレージが **1TBを超える場合**、`dentaiagent optimize --index-rebuild --compression-level 5` を毎週日曜午前3時にcronジョブで実行することを推奨。これによりクエリ応答時間が平均 **40% 改善** し、ディスク使用量は **18% 削減** される。

- **モデルバージョン管理とロールバック戦略**  
  本番環境では `dentaiagent model deploy --tag v2.3.1 --canary 0.2` のようにカナリアデプロイを実施。新モデルのAUCが現行モデルを **0.02以上下回った場合**、自動的に `v2.2.8` へロールバックするスクリプトをCI/CDパイプラインに組み込む。モデルメタデータはすべてPostgreSQLの `model_registry` テーブルで管理し、**デプロイ履歴は最低90日間保持** する。
