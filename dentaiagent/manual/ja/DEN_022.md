# dentaiagent - DEN_022

## 概要

dentaiagentは、歯科診療におけるAIエージェント連携を最適化するためのリアルタイム診断支援基盤です。複数の歯科用デジタル機器（口腔内スキャナ、X線センサー、CBCT）から取得したデータを統合し、機械学習モデルによる病変検出・治療計画の自動生成を行います。

## 詳細な内容

### とは dentaiagent?

dentaiagentは、歯科医院のワークフローに特化した分散型エージェントシステムです。各診療ユニットにデプロイされたエージェントが、DICOMデータの前処理、歯牙セグメンテーション、そしてエビデンスベースの治療オプション提示をミリ秒単位で実行します。PostgreSQL上の患者履歴データベースと連携し、過去の治療経過を加味したパーソナライズ診断を実現します。

### コア機能

- **マルチモーダル画像融合診断エンジン**  
  口腔内スキャナ（STL形式）とCBCT（DICOM形式）の3次元レジストレーションを自動実行。位置合わせ誤差0.1mm未満を保証し、う蝕検出精度（IoU 0.92以上）を達成。TensorRTを用いたGPU推論で、1スキャンあたり平均200msの処理速度。

- **時系列治療計画オプティマイザー**  
  患者の過去5年分のレントゲン画像と治療記録をLSTMモデルで解析。歯周病進行リスクスコア（0-100）を算出し、保険適用範囲内で最適な治療シーケンスを提案。パラメータ`risk_threshold=85`以上で外科的介入を推奨するアラートを発報。

- **音声対話型リアルタイムアシスタント**  
  Whisperベースの音声認識（日本語対応、誤認識率3%未満）と、LLM（GPT-4o相当）による自然言語応答を統合。「#12の隣接面う蝕の治療法」という発話から、即座に該当歯牙の3Dビューを表示し、エビデンスレベルAの治療オプションを3つ提示。

### 使い方

1. **DICOMデータの自動インポートと前処理**  
   `dentaiagent import --source /mnt/pacs/CBCT_2024/ --format dicom --patient_id 12345`  
   実行後、全スライスに対してノイズ除去フィルタ（非局所的平均法、フィルタ強度0.15）とコントラスト正規化（window width: 4000 HU, window level: 600 HU）が適用され、PostgreSQLの`preprocessed_images`テーブルに格納されます。

2. **歯牙セグメンテーションモデルの起動と結果確認**  
   `dentaiagent segment --model resunet_32x --device cuda:0 --output ./results/`  
   32クラス分類（歯牙番号+歯髄腔+骨欠損）のセグメンテーションマスクが生成され、Dice係数0.95以上を確認後、`segmentation_report_12345.json`に各歯牙の体積（mm³）と病変確率が出力されます。

3. **治療計画の自動生成とエクスポート**  
   `dentaiagent plan --patient_id 12345 --insurance jp_shinryo --output_format pdf`  
   健康保険の診療報酬点数表に基づき、抜髄・根管治療・クラウン修復の順序が自動決定。治療期間（推定日数）、総費用（自己負担額3割計算）、成功率（過去類似症例の統計値）を含むレポートが生成されます。

### ベストプラクティス

- **GPUメモリ管理によるバッチ処理の効率化**  
  複数患者のCBCTデータを同時処理する場合、`batch_size=4`および`mixed_precision=fp16`を指定することで、NVIDIA A100 80GB搭載サーバーで1時間あたり最大120スキャンの処理が可能。メモリ不足時は自動的に`batch_size`を半減するフォールバック機構を有効化してください。

- **PostgreSQLとの連携におけるインデックス戦略**  
  `patient_id`と`scan_date`に複合インデックスを作成し、`EXPLAIN ANALYZE`でクエリプランを定期的に監視。特に`preprocessed_images`テーブルはパーティショニング（月単位）することで、過去データの検索速度が平均40%向上します。
