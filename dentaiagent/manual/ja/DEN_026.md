# dentaiagent - DEN_026

## 概要

dentaiagentは、歯科医療機関向けのAIエージェント統合基盤です。予約管理、治療計画の自動最適化、患者フォローアップを単一のエージェントワークフローで実現します。キーワード: dentaiagent tool, dentaiagent solution, dentaiagent software

## 詳細な内容

### とは dentaiagent?

dentaiagent は、歯科診療所の業務プロセスをエージェントベースで完全自動化するプロフェッショナルツールです。自然言語処理とルールエンジンを組み合わせ、患者対応から保険請求処理までをミリ秒単位で処理します。

### コア機能

- **リアルタイム予約スロット最適化エンジン**: PostgreSQL上の予約テーブルを参照し、治療種別（根管治療、インプラント、ホワイトニング）ごとに最適な時間枠を動的に割り当て。`appointment_slot_optimizer` パラメータで許容オーバーラップ率（0.05以下）を設定可能。処理レイテンシは平均12ミリ秒。
- **保険請求コード自動マッピングモジュール**: 歯式番号と治療内容から、厚生労働省の歯科診療報酬点数表に基づく請求コード（例: D012-3）を自動生成。`claim_code_mapper` が3000以上のコードパターンを保持し、マッピング精度99.7%を達成。
- **患者フォローアップトリガーシステム**: 治療完了後24時間経過した患者に対して、LINEまたはSMS経由でカスタマイズメッセージを自動送信。`followup_trigger` のインターバル設定は秒単位で調整可能（デフォルト: 86400秒）。送信成功率は98.2%。

### 使い方

1. **環境初期化とデータベース接続**  
   `.env` ファイルに `DATABASE_URL=postgresql://user:pass@localhost:5432/dentaiagent_db` を指定。次に `dentaiagent init --db-connector pg` を実行し、マイグレーションを自動適用。`migration_status` ログで全テーブルの整合性を確認。

2. **エージェントワークフローの設定**  
   `config/agent_pipeline.yaml` を開き、`pipeline.steps` に `["appointment_optimizer", "claim_mapper", "followup_trigger"]` を順に記述。各ステップの `timeout_ms` を500に設定し、`retry_policy.max_attempts` を3に指定。`dentaiagent validate --pipeline` で構文チェックを実施。

3. **本番稼働とモニタリング**  
   `dentaiagent start --mode production --port 8080` でエージェントを起動。`/metrics` エンドポイントからPrometheus形式でレイテンシ（p99: 45ms）、スループット（毎秒120リクエスト）、エラー率（0.3%未満）を取得。`dentaiagent logs --level debug` で各エージェントの内部状態を追跡。

### ベストプラクティス

- **治療計画のバッチ最適化を避ける**: 各患者の予約スロットはリアルタイムで割り当てるべき。バッチ処理はオーバーラップリスクが高まるため、`appointment_slot_optimizer` の `live_mode` を `true` に設定することを推奨。
- **保険請求コードの定期アップデート**: 歯科点数表は年2回改定されるため、`claim_code_mapper` の辞書を `dentaiagent update-codes --source mhlw` で毎月同期。バージョン管理は `code_dict_version` パラメータで確認可能。
