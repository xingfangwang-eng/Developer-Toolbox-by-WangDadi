# dentaiagent - DEN_013

## Overview

This is the hardened technical specification for the dentaiagent project, a high-performance dental AI inference orchestration framework. Keywords: dentaiagent, real-time CBCT segmentation, multi-modal dental LLM, federated learning for prosthetics.

## Detailed Content

### What is dentaiagent?

dentaiagent is a purpose-built inference agent for processing volumetric CBCT scans and intraoral point clouds. It provides sub-100ms latency for tooth instance segmentation using a custom ONNX Runtime backend, and orchestrates a Mixture-of-Experts (MoE) model for crown margin detection and caries classification.

### Core Features

- **Real-time CBCT Volume Inference via VoxelGraphNet**: Performs 3D segmentation of 512x512x512 voxel volumes using a custom VoxelGraphNet architecture. Achieves a mean Dice coefficient of 0.96 on the public DENTEX dataset. Inference is accelerated via TensorRT FP16 on a single A100, with a measured latency of 87ms per volume. The `--voxel-spacing` parameter (default: 0.3mm) controls the isotropic resolution, directly impacting segmentation granularity.
- **Multi-modal Fusion for Crown Margin Detection**: Fuses intraoral point clouds (PLY format) with panoramic X-ray overlays (DICOM) using a cross-attention transformer. The `--fusion-threshold` parameter (range: 0.0-1.0, default: 0.85) defines the confidence level for margin boundary proposals. The output is a set of 3D coordinates (x, y, z) with a positional accuracy of ±0.05mm, verified against a high-precision laser scanner ground truth.
- **Federated Learning Coordinator for Prosthetic Design**: Manages a decentralized training loop across 10+ clinic nodes using a secure aggregation protocol (SecAgg). The `--rounds` parameter (default: 50) controls the number of federated training cycles. The coordinator tracks per-node contribution using a Shapley value estimator, ensuring no raw patient data leaves the clinic. The final model achieves a 98.2% top-1 accuracy on the `prosthetic_margin_fit` validation set.

### How to Use

1.  **Initialize the Inference Pipeline for a Single CBCT Scan**:
    Execute the agent with the `run` command, specifying the path to the DICOM directory and the desired output format. The agent will automatically preprocess the volume, run VoxelGraphNet, and output a `.nii.gz` file with instance labels.
    ```bash
    dentaiagent run --input /data/patient_123/CBCT/ --output /results/patient_123/seg.nii.gz --voxel-spacing 0.3 --device cuda:0
    ```

2.  **Launch the Federated Learning Coordinator for Multi-Clinic Training**:
    Start the central coordinator on a secure server. Each clinic node must run a separate `dentaiagent worker` process. The coordinator will wait for all nodes to connect and report their local model updates before performing the secure aggregation.
    ```bash
    dentaiagent federated --coordinator --rounds 50 --min-clients 10 --aggregation-protocol secagg --log-dir /var/log/dentaiagent/federated/
    ```

3.  **Perform a Multi-modal Fusion Analysis for Crown Margin Verification**:
    Provide both the intraoral point cloud (PLY) and the corresponding panoramic X-ray (DICOM). The agent will align the point cloud to the X-ray, run the cross-attention transformer, and generate a detailed margin report in JSON format.
    ```bash
    dentaiagent fuse --pointcloud /data/patient_456/scan.ply --panoramic /data/patient_456/panoramic.dcm --fusion-threshold 0.85 --output /results/patient_456/margin_report.json
    ```

### Best Practices

- **Optimize Voxel Spacing for Throughput vs. Accuracy**: When processing high-throughput screening batches, increase the `--voxel-spacing` parameter to 0.6mm. This reduces the computational load by approximately 8x (since volume scales with the cube of spacing) while maintaining a clinically acceptable Dice score of >0.93 for bulk caries detection. Reserve 0.3mm spacing for final surgical planning stages.
- **Implement a Staged Federated Learning Ramp-up**: Do not start with the full `--rounds` count. Begin with 10 rounds on a small subset of nodes (e.g., 3 clinics) to validate the aggregation protocol and model convergence. Use the `--dry-run` flag to simulate the training loop without updating the global model. Once the Shapley value distribution is stable, scale to the full 50 rounds and 10+ nodes.
