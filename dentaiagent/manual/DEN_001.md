## Detailed Content

### What is dentaiagent?

dentaiagent is an autonomous agent framework designed for real-time, 3D volumetric analysis of CBCT (Cone Beam Computed Tomography) scans. It orchestrates a pipeline of neural network models to detect, classify, and propose optimal implant trajectories, integrating directly with PostgreSQL for persistent case history and surgical guide generation.

### Core Features

- **Real-Time Bone Density Mapping via Multi-Resolution Voxel Transformer**  
  The agent ingests raw DICOM data and executes a custom `VoxelTransformer3D` model. It outputs a 3D heatmap of Hounsfield Unit (HU) gradients at 0.25mm isotropic resolution. The `--voxel-resolution` parameter (default: `0.25`) controls the trade-off between detail and compute time. On an NVIDIA A100, inference completes in under 300 milliseconds per scan volume.

- **Automatic Nerve Canal Segmentation with Confidence Scoring**  
  Leveraging a U-Net++ variant with attention gates, the agent segments the inferior alveolar nerve (IAN) with a Dice coefficient consistently above 0.97. The `--nerve-confidence-threshold` flag (float, 0.0 to 1.0) filters out low-confidence voxels. The pipeline logs the `nerve_proximity_mm` metric for every proposed implant site, ensuring a safe distance of at least 2.0 mm.

- **Dynamic Implant Trajectory Optimization Using Constrained Genetic Algorithm**  
  The agent runs a custom `GeneticOptimizer` that accepts `--population-size` (default: 200) and `--generations` (default: 50). It evaluates trajectories against a multi-objective fitness function: maximizing bone-to-implant contact (BIC) while minimizing angular deviation from the prosthetic axis. The final output includes a `trajectory.json` file with `entry_point`, `exit_point`, and `angulation_degrees` (accuracy within ±0.5°).

### How to Use

1. **Initializing the DICOM Ingestion Pipeline**  
   Execute the agent with a local path or PACS URL:  
   `dentaiagent --input-dir /data/patient_001/ --pacs-url "dicom://192.168.1.100:11112" --voxel-resolution 0.25`  
   The agent will authenticate via the `PACS_AUTH_TOKEN` environment variable and begin volumetric reconstruction. Monitor the `ingestion_status` column in the `patient_volumes` PostgreSQL table.

2. **Running the Full Inference and Trajectory Optimization**  
   After ingestion, trigger the optimization module:  
   `dentaiagent --run-id 5e8f2c --nerve-confidence-threshold 0.95 --population-size 300 --generations 75`  
   The agent will output a real-time WebSocket stream of `optimization_progress` (percentage) and `best_fitness_score`. The final result is written to the `optimized_trajectories` table in PostgreSQL, with a foreign key to the patient ID.

3. **Exporting the Surgical Guide as STL**  
   To generate a printable surgical guide:  
   `dentaiagent --export-stl --run-id 5e8f2c --guide-thickness 2.5 --tolerance-mm 0.1`  
   The agent creates a watertight STL mesh with cylindrical drill sleeves. The file is saved to `/output/guides/5e8f2c_guide.stl`. The `tolerance-mm` parameter ensures the sleeve inner diameter matches the specified implant system (e.g., Straumann BLT).

### Best Practices

- **Always Validate Nerve Segmentation with a Secondary Model**  
  Deploy a lightweight `MobileNetV3`-based classifier to run a sanity check on the U-Net++ output. If the confidence score for the IAN region drops below 0.85, the agent should automatically flag the case for manual review. This prevents false negatives in critical anatomical areas.

- **Use a PostgreSQL Write-Ahead Log (WAL) for Disaster Recovery**  
  Configure `wal_level = logical` in your `postgresql.conf`. This enables real-time replication of the `optimized_trajectories` table to a standby instance. In the event of primary node failure, the agent can automatically failover using the `PG_STANDBY_HOST` environment variable, ensuring zero data loss for ongoing surgical workflows.
