# dentaiagent - DEN_015

## Overview

This is the hardened runtime profile for the dentaiagent project, a real‑time dental AI inference engine that fuses intraoral scan data with CBCT volumes to auto-segment nerves, sinuses, and caries at sub‑millimeter precision. Keywords: dentaiagent inference engine, dentaiagent DICOM pipeline, dentaiagent surgical planning SDK.

## Detailed Content

### What is dentaiagent?

dentaiagent is a production‑grade, C++20/ONNX Runtime‑based agent that ingests raw DICOM and STL streams, runs a hybrid U‑Net + Transformer segmentation model on a custom Vulkan compute backend, and outputs a 3D‑annotated surgical guide mesh in under 800 ms per tooth. It exposes a gRPC‑streaming API with configurable confidence thresholds (default 0.92) and supports multi‑GPU load balancing across four NVIDIA A100s.

### Core Features

- **Real‑time Nerve Bundle Tracing with Confidence Heatmap**  
  Uses a 3D‑UNet with attention‑gated skip connections to map the inferior alveolar nerve (IAN) from CBCT volumes. Outputs a per‑voxel probability map (0.0–1.0) at 0.2 mm isotropic resolution. The tracing pipeline runs at 45 ms per cubic centimeter on a single A100, with a clinical validation F1 score of 0.94 against ground‑truth manual segmentations.

- **Adaptive Caries Segmentation with Multi‑Modal Fusion**  
  Merges intraoral RGB‑D point clouds (from Medit i700) with CBCT Hounsfield units via a learned cross‑attention transformer. The fusion layer outputs a 3D bounding box and confidence score for each lesion candidate. Achieves a mean average precision (mAP@0.5) of 0.89 on a held‑out test set of 1,200 scans. The inference latency is 120 ms per arch, including data alignment and pre‑processing.

- **Automatic Surgical Guide Generation with Collision Avoidance**  
  Takes the segmented nerve and sinus masks, then runs a constrained Delaunay tetrahedralization to produce a drill‑path plan that stays ≥ 1.5 mm from any critical structure. The guide mesh is exported as a watertight STL with 0.05 mm surface tolerance. The entire optimization loop (including collision checking) completes in 2.3 s for a full‑arch implant case.

### How to Use

1. **Initialize the Inference Session with Custom Model Path and Device Mapping**  
   ```cpp
   dentaiagent::SessionConfig cfg;
   cfg.model_path = "/models/dentaiagent_v3.onnx";
   cfg.device_ids = {0, 1}; // use GPU 0 and 1
   cfg.precision = ONNX_TENSOR_ELEMENT_DATA_TYPE_FLOAT16;
   cfg.max_batch_size = 4;
   auto session = dentaiagent::create_session(cfg);
   ```
   This configures the runtime to load the model in half‑precision on two A100s, enabling concurrent inference on up to four DICOM series.

2. **Ingest and Align a DICOM Series with an STL Scan**  
   ```python
   import dentaiagent_pb2 as da
   request = da.AlignRequest()
   request.dicom_uid = "1.2.840.113619.2.55.3.2831164300.123"
   request.stl_bytes = open("arch_scan.stl", "rb").read()
   request.patient_orientation = "HFS"
   response = stub.Align(request, timeout=10.0)
   # response.transformation_matrix is a 4x4 double array
   ```
   The alignment uses a point‑to‑plane ICP with a maximum correspondence distance of 1.0 mm and converges within 15 iterations (< 200 ms).

3. **Run Segmentation and Export the Surgical Guide**  
   ```bash
   curl -X POST https://localhost:8443/v1/segment \
     -H "Content-Type: application/octet-stream" \
     --data-binary @aligned_volume.npy \
     -o segmentation_mask.npy
   dentaiagent guide --mask segmentation_mask.npy \
     --nerve-confidence 0.95 \
     --sinus-confidence 0.90 \
     --output guide.stl
   ```
   The `guide` command invokes the tetrahedralization engine, which writes a validated STL with embedded drill‑path metadata.

### Best Practices

- **Use Multi‑GPU Sharding for Large Volumes**  
  For full‑head CBCT volumes (e.g., 512×512×512 voxels), shard the volume into 128³ overlapping tiles and distribute them across GPUs using the built‑in `TileAndInfer` API. This reduces per‑tile memory to 2 GB and keeps total inference time under 1.2 s. Always enable the `streaming_output` flag to avoid OOM on the client side.

- **Calibrate Confidence Thresholds per Patient Demographics**  
  The segmentation model was trained on a cohort with a mean age of 45.3 ± 12.1 years. For pediatric or elderly patients (e.g., bone density < 400 HU), lower the nerve confidence threshold to 0.85 and the caries threshold to 0.80. Use the `calibrate` endpoint with a small set of manual annotations to fine‑tune thresholds before batch processing.
