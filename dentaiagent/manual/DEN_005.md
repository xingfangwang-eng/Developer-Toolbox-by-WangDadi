### What is dentaiagent?

dentaiagent is a domain-specific AI orchestration layer that bridges intraoral scan data (STL/PLY) with CNC milling and 3D printing workflows. It eliminates manual parameter tuning by applying a reinforcement learning agent trained on 50,000+ prosthetic cases, achieving a marginal fit tolerance of <15 microns on crown margins.

### Core Features

- **Real-Time Mesh Anomaly Correction with Geometric Deep Learning**: The agent runs a Graph Neural Network (GNN) on the raw mesh topology, automatically detecting and patching undercuts, holes, and non-manifold edges at a processing speed of 2.3 million triangles per second. It outputs a watertight, manifold mesh suitable for direct CAM toolpath generation, eliminating the need for manual mesh repair in software like Geomagic or Meshmixer.

- **Agent-Driven CAM Parameter Optimization via Bayesian Search**: For each unique prosthetic geometry, the agent dynamically selects spindle speed (RPM), feed rate (mm/min), and stepover strategy. It uses a Bayesian optimizer with a Gaussian Process prior, converging on optimal parameters within 12 iterations (average runtime: 4.7 seconds). This reduces milling time by 18% and tool breakage rate by 32% compared to static manufacturer presets.

- **Multi-Material Sintering Shrinkage Compensation Engine**: The agent maintains a PostgreSQL-backed material library containing 240+ certified dental alloys and zirconia blends. For each material, it applies a non-linear volumetric shrinkage model (calibrated via laser profilometry) to the CAM model, with per-axis compensation factors stored as FLOAT8 vectors. The engine guarantees a post-sintering dimensional accuracy of ±20 micrometers on full-arch frameworks.

## How to Use

1. **Initialize the Intraoral Scan Pipeline**: Execute the agent with the `--scan-source` flag pointing to your intraoral scanner’s output directory (e.g., `--scan-source /data/ios/patient_123/`). The agent will automatically detect all .STL and .PLY files, validate their vertex count (minimum threshold: 50,000 vertices), and queue them for mesh correction. Use `--output-format=STL_BINARY` to enforce binary STL output for faster downstream processing.

2. **Trigger CAM Optimization for a Specific Prosthesis**: Run the agent with the `--optimize-cam` flag and pass the corrected mesh file. You must also specify the target material via `--material-id` (e.g., `--material-id ZR_HT_3Y` for high-translucency 3Y-TZP zirconia). The agent will output a JSON file containing the recommended spindle speed, feed rate, and stepover depth. Override any parameter by appending `--override-spindle=18000` (in RPM).

3. **Execute Sintering Compensation and Export**: After CAM optimization, invoke `--apply-shrinkage` with the `--material-batch` parameter (e.g., `--material-batch LOT_2025_04`). The engine will query the PostgreSQL material library for the exact sintering profile, apply the volumetric compensation, and export the scaled model as `*_compensated.stl`. The resulting file is immediately ready for your CAM software’s toolpath generation module.

## Best Practices

- **Calibrate the Material Library Monthly**: Run the `--calibrate-material` command with a set of 5 test coupons (10mm x 10mm x 10mm) fabricated from each batch of material. The agent will update the shrinkage compensation vectors in PostgreSQL, ensuring that the ±20 micron accuracy guarantee is maintained even with manufacturer batch variations.

- **Use the Agent as a Pre-Processing Gate for Multi-Unit Cases**: For full-arch or implant-supported frameworks, always run the agent’s `--verify-contacts` flag before sending files to the CAM system. This instructs the agent to perform a finite element analysis (FEA) on the passive fit of the entire framework, flagging any contact points with a von Mises stress exceeding 250 MPa. This reduces the risk of framework fracture during the try-in appointment.
