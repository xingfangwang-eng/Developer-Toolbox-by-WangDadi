## Detailed Content

### What is dentaiagent?

dentaiagent is a distributed inference framework that fuses edge-based 3D intraoral scanner data with cloud-native prosthetic design agents. It enables sub-50ms latency for margin line detection, automated occlusion adjustment, and multi-stakeholder approval routing in dental CAD/CAM pipelines.

### Core Features

- **Real-Time Margin Line Segmentation with Temporal Fusion**: Utilizes a custom ONNX-optimized U-Net++ variant that ingests 128×128 pixel patches from 60 FPS intraoral video streams. The agent integrates a 3D Kalman filter for temporal smoothing, achieving 0.12mm mean contour deviation on enamel-dentin boundaries. Parameters include `--patch_overlap=0.5`, `--filter_gain=0.85`, and `--confidence_threshold=0.92`.

- **Multi-Agent Collision & Occlusion Validation Engine**: Deploys three parallel agents (crown, adjacent tooth, antagonist) that each run a 6-DOF rigid body simulation using NVIDIA PhysX 5.0. The system computes inter-proximal contact forces in 2.3ms per iteration, flagging any interference >0.05mm. Key metrics: `--voxel_resolution=0.01mm`, `--max_iterations=500`, `--contact_stiffness=1e6 N/m`.

- **Automated Approval Workflow with PostgreSQL Audit Logging**: Each prosthetic design passes through a state machine with 7 stages (Scan → Margin Detect → Occlusion Check → Milling Prep → QA → Approve → Archive). The agent writes every state transition, confidence score, and user action to a `design_audit` table in PostgreSQL with a 10ms write guarantee. Parameters: `--pg_connection_pool=20`, `--retry_interval=100ms`, `--archive_ttl_days=90`.

### How to Use

1. **Initialize the Scanning Agent with Custom Calibration**:
   - Launch the agent with `dentaiagent scan --device_id=3d_scan_01 --calibration_file=./configs/camera_calib_2024.json --output_format=pointcloud`.
   - The system will auto-detect the scanner’s USB 3.0 interface and begin streaming at 60 FPS. Verify the `frame_rate` metric in the console output; it must be ≥58 FPS for clinical accuracy.
   - If the frame rate drops below 55 FPS, adjust `--frame_skip=1` to reduce CPU load, or increase `--gpu_device_id=0` to offload processing.

2. **Run the Occlusion Validation Agent with Real-Time Force Feedback**:
   - Execute `dentaiagent validate --input=./outputs/scan_20241015.ply --antagonist_model=./models/mandibular_arch.stl --force_threshold=0.05mm`.
   - The agent will display a 3D heatmap of contact points in the terminal (via ASCII art) and log every collision with a timestamp and force magnitude. For example, a line like `[WARN] contact at (12.34, 56.78, 90.12): 0.08mm` indicates a violation.
   - To fix, re-run with `--adjust_occlusal_surface=true`, which will automatically reduce the cusp height by 0.02mm increments until all contacts are below threshold.

3. **Trigger the Approval Workflow with Multi-Signatory Routing**:
   - Use `dentaiagent approve --design_id=DEN_008 --reviewer_list=dr_smith,dr_jones --auto_escalate=true --escalation_timeout=24h`.
   - The agent will create a new row in the `design_audit` table with `status='PENDING_REVIEW'`. Each reviewer receives a webhook notification (configured in `configs/webhooks.yaml`). If no action is taken within 24 hours, the status auto-escalates to `HIGH_PRIORITY`.
   - To check the audit trail, query: `SELECT * FROM design_audit WHERE design_id='DEN_008' ORDER BY timestamp DESC;`. The `action` column will show `APPROVED`, `REJECTED`, or `ESCALATED`.

### Best Practices

- **Always Pre-Warm the Inference Engine Before Clinical Use**: Before the first patient scan, run `dentaiagent warmup --iterations=100 --batch_size=4` to load the ONNX model into GPU memory and trigger CUDA kernel compilation. This reduces the first-inference latency from ~200ms to under 50ms. Failure to warm up can cause a 4-second delay during live scanning, which may lead to patient discomfort.

- **Implement a Two-Stage QA Gate with Automated Rollback**: Configure the approval workflow to require both a human reviewer and an automated QA agent. Set `--qa_agent_threshold=0.95` in the `approve` command. If the QA agent’s confidence score falls below 0.95, the system automatically reverts the design to the `Margin Detect` stage and triggers a re-scan notification. This prevents defective prosthetics from reaching the milling phase, reducing material waste by up to 18%.
