# dentaiagent - DEN_007

## Overview

This is the production-grade architectural blueprint for the **dentaiagent** project, a deterministic AI-driven orchestration engine for real-time dental treatment planning. Keywords: dentaiagent inference core, dentaiagent CBCT segmentation pipeline, dentaiagent DICOM parser.

## Detailed Content

### What is dentaiagent?

dentaiagent is a Rust-based, zero-trust inference agent that ingests raw CBCT DICOM volumes (512x512x450 voxels at 0.2mm isotropic resolution) and executes a multi-stage segmentation + landmark detection pipeline with sub‑millimeter precision. It exposes a gRPC streaming endpoint for real‑time chairside guidance, and persists every inference audit trail to a PostgreSQL + TimescaleDB cluster.

### Core Features

- **Multi‑Stage CBCT Segmentation with Uncertainty Quantification**  
  The agent runs a 3D U‑Net++ ensemble (3 heads, each with Monte Carlo dropout at p=0.1) on the raw voxel grid. Each head outputs a probabilistic segmentation map for 32 dental tissue classes (enamel, dentin, pulp, PDL, cortical bone, trabecular bone, etc.). The final label map is produced via Bayesian fusion with a confidence threshold of 0.85. Inference latency is capped at **420 ms per volume** on an NVIDIA A100 (80 GB).

- **Deterministic Landmark Detection via Heatmap Regression**  
  A separate lightweight FCN (MobileNetV3‑backbone, 1.2M parameters) regresses 68 cephalometric landmarks directly from the segmentation tensor. Landmark coordinates are output in **millimeter‑scale world coordinates** (DICOM Patient‑based Coordinate System). The pipeline guarantees a mean radial error (MRE) below 0.35 mm on the benchmark ISBI‑2015 dataset. All coordinate transforms are logged with nanosecond‑precision timestamps.

- **Real‑time Audit Trail & Versioned Model Registry**  
  Every inference run (including raw input hash, model weights hash, segmentation confidence maps, landmark coordinates, and wall‑clock duration) is committed to a PostgreSQL table (`inference_audit`) with an auto‑incrementing `run_id` (BIGSERIAL). Model weights are stored in an S3‑compatible object store, and the agent validates the SHA‑256 checksum of each weight file before loading. The registry supports **point‑in‑time rollback** to any previous model version via a dedicated gRPC `RollbackModel` RPC.

### How to Use

1. **Initialize the Inference Pipeline with a DICOM Series**  
   Call the gRPC method `InferFromDicom(series_uid, modality="CT", device="cuda:0")`. The agent will:  
   - Download the DICOM series from the configured PACS endpoint (orthanc‑ or dcm4chee‑based).  
   - Parse the DICOM headers to extract pixel spacing, slice thickness, and Patient Position.  
   - Resample the volume to isotropic 0.2 mm using a third‑order B‑spline interpolator.  
   - Return a `InferenceSession` handle (UUID v4) with a pre‑allocated GPU tensor.

2. **Execute Segmentation & Landmark Detection**  
   Send a `RunInference(session_id, model_version="v2.1.3", dropout_enabled=true)` request. The agent:  
   - Loads the ensemble weights from the model registry (validates SHA‑256).  
   - Runs the 3‑head U‑Net++ with MC dropout (10 forward passes per head).  
   - Computes the fused segmentation map and the per‑voxel epistemic uncertainty (variance across passes).  
   - Feeds the segmentation tensor into the landmark FCN, outputting 68 heatmaps.  
   - Extracts sub‑pixel landmark coordinates via a 2D Gaussian fit (σ=1.5) on each heatmap.  
   - Returns a `InferenceResult` protobuf containing the segmentation map (compressed via zlib, level 6), landmark array (float64), and uncertainty map (float32).

3. **Persist the Audit Trail and Export Results**  
   Call `CommitAudit(session_id, result, notes="Pre‑surgical planning for implant #19")`. The agent:  
   - Inserts a row into `inference_audit` with `run_id`, `session_id`, `model_version`, `input_hash`, `output_hash`, `duration_ms`, and `notes`.  
   - Optionally exports the segmentation map as a NIfTI‑GZ file to the configured S3 bucket (`s3://dentaiagent-seg/run_{run_id}.nii.gz`).  
   - Returns the `run_id` (BIGSERIAL) for later retrieval or rollback.

### Best Practices

- **Always enable Monte Carlo dropout during inference** – even at the cost of 3× inference time. The epistemic uncertainty map is the single most valuable diagnostic output for the clinician. A high‑uncertainty region (variance > 0.15) indicates the model is extrapolating beyond its training distribution; flag these regions for manual review before any surgical planning step.

- **Version‑lock the model registry in production** – never use the `"latest"` tag. Each deployment should pin a specific semantic version (e.g., `v2.1.3`). Store the exact training hyperparameters (learning rate, batch size, loss weights) as JSON metadata alongside the weight file. This ensures reproducibility of every inference audit trail, which is critical for regulatory compliance (FDA 21 CFR Part 11).
