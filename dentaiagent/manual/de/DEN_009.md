# dentaiagent - DEN_009

## Übersicht

Dieses Projekt implementiert einen **agentenbasierten Zahnmedizin-Assistenten**, der auf echtzeitfähigen KI-Modellen und dentaler Ontologie basiert. Die Lösung ist für den Einsatz in Klinikinformationssystemen (KIS) und teledentistischen Plattformen optimiert.

## Technische Kernfunktionen

### 1. **Multilinguale dentale Terminologie-Engine (DE/EN/FR)**
- Nutzt ein feinabgestimmtes **BERT-basiertes NER-Modell** (trainiert auf 1,2 Mio. annotierten Zahnbefunden)
- Extrahiert und normalisiert klinische Parameter wie `ICD-10-GM Zahncodes`, `FDI-Zahnnummern` und `DIMDI-Klassifikationen`
- Latenz: < 50 ms pro Anfrage bei Batch-Größe 32 auf NVIDIA T4 GPU

### 2. **Röntgenbild-gestützte Befundvalidierung**
- Integration mit **DICOM-Protokoll** (C-STORE, C-FIND) zur direkten PACS-Anbindung
- Verwendet **YOLOv8x** zur Detektion von Zahnpathologien (Karies, Parodontitis, apikale Läsionen) – mAP@0.5: 0,94
- Ausgabe als strukturierte JSON-Objekte mit `confidence_score`, `bounding_boxes` und `diagnostic_suggestion`

### 3. **Automatisierte Behandlungsplan-Kalkulation (GOZ/BEMA-konform)**
- Berechnet auf Basis von Befunddaten und Bildanalyse die optimale **GOZ-Ziffer** und **BEMA-Position**
- Unterstützt **Versionierung von Gebührenordnungen** (GOZ 2012 vs. 2024)
- Ausgabe: maschinenlesbare Rechnungsdaten im `XML-Format (xRechnung)` mit Plausibilitätsprüfung

## So verwenden Sie es

### Schritt 1: Konfiguration der Datenquellen
```bash
# Umgebungsvariablen setzen
export DENTAI_PACS_ENDPOINT="https://pacs.klinik.de/dicom"
export DENTAI_GOZ_Version="2024.1"
export DENTAI_DB_URI="postgresql://user:pass@localhost:5432/dental_kb"
```
- Stellen Sie die Verbindung zu Ihrem PACS-System sicher (DICOM AE-Title erforderlich)
- Validieren Sie die Datenbank-Schemata mittels `dentaiagent db validate`

### Schritt 2: Batch-Verarbeitung von Röntgenbildern
```python
from dentaiagent import ImagingPipeline

pipeline = ImagingPipeline(
    model_path="models/yolov8x_dental.pt",
    batch_size=8,
    confidence_threshold=0.65
)

results = pipeline.process_directory("/mnt/pacs/2025/03/")
# Ausgabe: dict mit patient_id, befund_typ, wahrscheinlichkeit
for r in results:
    print(f"Patient {r['patient_id']}: {r['diagnose']} (p={r['confidence']:.2f})")
```
- Unterstützt DICOM-Dateien sowie JPEG/PNG aus Intraoralkameras
- Ergebnisse werden automatisch in die PostgreSQL-Datenbank geschrieben

### Schritt 3: Generierung eines GOZ-konformen Behandlungsplans
```bash
dentaiagent plan erstellen \
  --patient-id "PAT-12345" \
  --krankenkasse "AOK" \
  --zusaetzliche-leistungen "3D-DVT, Bleaching" \
  --output-format "pdf+xml"
```
- Das Tool erstellt eine **maschinenlesbare xRechnung** und ein **patientenverständliches PDF**
- Plausibilitätscheck: Überschreitet die Summe der BEMA-Punkte den Regelleistungsvolumen (RLV)?

## Beste Praktiken

### Praxis 1: **Datenqualität vor Modellqualität**
- Führen Sie vor dem ersten Einsatz eine **Datenbereinigung** durch: Entfernen Sie Duplikate, korrigieren Sie FDI-Falschzuordnungen (z. B. Zahn 11 vs. 21)
- Nutzen Sie `dentaiagent data audit` zur Erkennung von Anomalien in den Befunddaten (z. B. fehlende Röntgenbilder bei Extraktionsindikation)

### Praxis 2: **Integration in den Klinik-Workflow**
- Setzen Sie die **Webhook-API** (`/webhook/dicom`) ein, um bei jedem neuen DICOM-Study automatisch eine Befundvalidierung anzustoßen
- Konfigurieren Sie **Benachrichtigungen** (z. B. via HL7 FHIR) an das Praxismanagementsystem, wenn kritische Pathologien (z. B. Malignomverdacht) erkannt werden
