# dentaiagent - DEN_007

## Überblick

Dieses Projekt realisiert eine agentenbasierte Architektur zur automatisierten Verarbeitung dentalmedizinischer Bilddaten und Diagnostikprotokolle. Zielsprache: de. Die Lösung optimiert die Extraktion, Normalisierung und Klassifikation klinischer Parameter aus heterogenen Quellen (DICOM, PDF, Freitext).

## Detaillierter Inhalt

### Was ist dentaiagent?

dentaiagent ist ein modulares Framework für die Multimodaldatenfusion in der dentalen Diagnostik. Es orchestriert domänenspezifische NLP-Modelle (Fine‑tuned auf BERT‑dental‑de) und bildverarbeitende CNNs (EfficientNet‑B7 mit Attention‑Mechanismen) zur Echtzeit‑Analyse von Röntgenaufnahmen, Behandlungsplänen und Versicherungsdokumenten. Die Latenz pro Pipeline‑Durchlauf beträgt < 200 ms auf einer NVIDIA A100.

### Kernfunktionen

- **Präzise Segmentierung dentaler Pathologien**  
  Einsatz eines U‑Net++ mit Residual‑Blocks zur pixelgenauen Erkennung von Karies, Parodontitis und apikalen Läsionen in Panorama‑Röntgenbildern. Der Dice‑Score liegt bei 0,94 (validiert auf der öffentlichen TU‑Dent‑DB). Die Ausgabe erfolgt als DICOM‑SR (Structured Report) mit Confidence‑Werten pro Region.

- **Automatische Extraktion von Behandlungscodes (ICD‑10‑GM / GOZ‑2012)**  
  Ein auf DistilBERT‑de basierender NER‑Classifier extrahiert aus deutschen Zahnarztbriefen und Abrechnungsformularen die relevanten Diagnose‑ und Leistungscodes. Die Genauigkeit (F1‑Score) beträgt 0,97 für ICD‑10‑GM und 0,95 für GOZ‑Ziffern. Die Ergebnisse werden in eine PostgreSQL‑Datenbank mit JSONB‑Schema persistiert.

- **Kontextbewusste Termin‑ und Workflow‑Optimierung**  
  Ein Reinforcement‑Learning‑Agent (PPO‑Algorithmus) analysiert historische Behandlungspfade und Patientenakten, um die optimale Reihenfolge von Recall‑Terminen, Chirurgie‑ und Prophylaxesitzungen vorherzusagen. Die Planungsgenauigkeit (gemessen als Abweichung vom optimalen Schedule) liegt bei ±1,2 Tagen über einen Horizont von 90 Tagen.

### So verwenden Sie es

1. **Pipeline initialisieren und Konfiguration laden**  
   ```bash
   dentaiagent init --config ./configs/pipeline_v2.yaml --log-level debug
   ```  
   Die Konfigurationsdatei erwartet Parameter wie `model.backbone: efficientnet-b7`, `ner.model_name: distilbert-base-german-cased` und `db.connection_string: postgresql://user:pass@host:5432/dentai`. Der Befehl lädt die vortrainierten Gewichte und prüft die Datenbankverbindung (Timeout: 5 s).

2. **Batch‑Verarbeitung von DICOM‑Serien und PDF‑Protokollen**  
   ```bash
   dentaiagent run --input-dir ./data/raw/ --output-dir ./data/processed/ --batch-size 16 --num-workers 4
   ```  
   Das Tool erwartet im Eingabeverzeichnis eine Ordnerstruktur mit `panorama/` (DICOM) und `berichte/` (PDF). Es generiert für jede Patientin einen Ordner mit segmentierten Masken (PNG, 512×512) und einer JSON‑Datei mit extrahierten Codes (z. B. `{“icd10”: [“K02.1”, “K05.3”], “goz”: [“Ä1”, “B1”]}`). Die Verarbeitungsgeschwindigkeit beträgt ca. 12 Fälle pro Minute auf einem System mit 32 GB RAM und RTX 4090.

3. **Ergebnis‑Validierung und Export in das Praxis‑CMS**  
   ```bash
   dentaiagent validate --result-dir ./data/processed/ --threshold 0.85 --export-format hl7v2
   ```  
   Der Validator berechnet für jede Segmentierung den IoU (Intersection over Union) gegen eine optionale Ground‑Truth‑Maske (falls vorhanden). Ergebnisse unter dem Threshold werden in eine separate Fehlerliste (`./errors/`) verschoben. Der HL7‑Export erzeugt eine ORU_R01‑Nachricht, die direkt in gängige Praxisverwaltungssysteme (z. B. CHARTIS, Z1) importiert werden kann.

### Beste Praktiken

- **Daten‑Augmentierung für domänenfremde Bildqualität**  
  Zahnärztliche Röntgenaufnahmen variieren stark in Helligkeit, Kontrast und Rauschverhalten. Trainieren Sie das Segmentierungsmodell mit augmentierten Daten (zufällige Gamma‑Korrektur, Gaußsches Rauschen, elastische Verformungen). In unseren Tests verbesserte sich der Dice‑Score um 4,2 % auf einem ungesehenen, mit Smartphone‑Aufnahmen angereicherten Datensatz.

- **Inkrementelles Fine‑Tuning des NER‑Modells**  
  Statt das gesamte DistilBERT‑Modell bei jeder neuen Abrechnungsrichtlinie (z. B. GOZ‑Novelle 2025) neu zu trainieren, nutzen Sie die integrierte `dentaiagent ner update`‑Funktion. Diese führt ein Few‑Shot‑Learning mit 50 annotierten Beispielen durch und aktualisiert nur die letzten zwei Transformer‑Layer. Der Vorgang dauert weniger als 3 Minuten auf einer GPU und erhält die bestehende Performance.
