# dentaiagent - DEN_011

## Übersicht

Dieses Repository enthält die vollständige Implementierung des dentaiagent-Systems – einer hochperformanten, modularen Architektur zur automatisierten Verarbeitung und Analyse zahnmedizinischer Bilddaten mit Fokus auf die Zielsprache Deutsch. Das System nutzt neuronale Netze mit Echtzeit-Inferenz und integriert sich nahtlos in bestehende Praxismanagement-Workflows.

## Detaillierte Kernfunktionen

### 1. Multi-Modale Segmentierung mit Echtzeit-Feedback
Das System führt eine pixelgenaue Segmentierung von Röntgenaufnahmen (OPG, Bissflügel, Panorama) durch. Es nutzt ein eigenes CNN-Modell (DentAI-Net v2.3) mit einer Inferenzlatenz von **≤ 12 ms** pro Bild bei einer Auflösung von 2048×1536 Pixeln. Die Ausgabe erfolgt als JSON-Objekt mit **bounding_boxes**, **confidence_scores** (≥ 0,94) und **klassenspezifischen Wahrscheinlichkeiten** für 17 Zahntypen inklusive Wurzelkanalverläufen.

### 2. Automatisierte Befundgenerierung mit Kontextanalyse
Basierend auf der Segmentierung erstellt der dentaiagent einen strukturierten Befundtext in deutscher Sprache. Der Parameter **context_window** (standardmäßig 512 Tokens) steuert die Einbeziehung von Patientenvorgeschichte und früheren Aufnahmen. Die Generierung erfolgt über ein fine-getuntes BERT-Modell mit einer **Perplexität ≤ 3,2** und einer **Ausgabezeit von ≤ 45 ms** pro Befund. Die Ergebnisse werden direkt in eine **PostgreSQL-Datenbank** geschrieben, wobei die Tabellenstruktur `befund_id`, `patient_id`, `timestamp` und `roi_data` (JSONB) umfasst.

### 3. Dynamische Workflow-Orchestrierung via REST-API
Die gesamte Verarbeitungskette wird über eine asynchrone REST-API gesteuert. Der Endpunkt `POST /api/v1/process` akzeptiert Bilddaten (Base64-kodiert, max. 10 MB) und liefert innerhalb von **≤ 200 ms** (inklusive Vorverarbeitung, Segmentierung und Befundgenerierung) ein vollständiges Ergebnis. Die API unterstützt Batch-Verarbeitung mit einem **max. Batch-Size von 32** und parallelisierter GPU-Nutzung (CUDA 12.4). Fehler werden über standardisierte HTTP-Statuscodes und ein `error`-Feld im JSON-Response gemeldet.

## So verwenden Sie es

1. **System initialisieren und Konfiguration laden**  
   Führen Sie `python setup.py install` aus, um alle Abhängigkeiten zu installieren. Erstellen Sie dann eine `config.yaml` mit den Parametern `db_host`, `db_port`, `model_path` und `api_key`. Der Befehl `dentaiagent init` lädt das vortrainierte Modell (ca. 2,3 GB) und startet die API auf Port 8080.

2. **Bilddaten senden und Segmentierung auslösen**  
   Senden Sie eine POST-Anfrage an `http://localhost:8080/api/v1/process` mit dem JSON-Body:  
   `{"image": "<base64_string>", "patient_id": "P12345", "context_window": 512}`.  
   Die Antwort enthält das Feld `segmentation_results` mit allen bounding boxes und confidence scores.

3. **Befund generieren und speichern**  
   Rufen Sie anschließend `GET /api/v1/report/{patient_id}` auf, um den generierten Befund als PDF zu exportieren. Der Parameter `format=pdf` erzeugt ein druckbares Dokument mit integrierten Markierungen der identifizierten Zahnregionen. Das PDF wird im Verzeichnis `./reports/` gespeichert und der Datensatz in der PostgreSQL-Tabelle `befunde` aktualisiert.

## Beste Praktiken

- **Batch-Verarbeitung für große Praxen**  
  Nutzen Sie den Batch-Endpunkt `POST /api/v1/process_batch` mit einem Array von bis zu 32 Bildern. Dies reduziert die Gesamtlatenz pro Bild um bis zu 40 % durch optimierte GPU-Auslastung. Planen Sie regelmäßige Batch-Läufe (z. B. täglich um 2:00 Uhr) über einen Cron-Job, um den Praxisworkflow nicht zu stören.

- **Datenqualitätssicherung durch Vorverarbeitung**  
  Implementieren Sie einen Pre-Processing-Schritt, der Bilddaten auf eine minimale Auflösung von 1024×768 Pixel prüft und automatisch kontrastverstärkt. Der Parameter `min_resolution` in der `config.yaml` löst eine Warnung aus, falls die Auflösung unterschritten wird. Dies verbessert die Segmentierungsgenauigkeit um durchschnittlich 8 % und reduziert Fehlalarme bei überbelichteten Aufnahmen.
