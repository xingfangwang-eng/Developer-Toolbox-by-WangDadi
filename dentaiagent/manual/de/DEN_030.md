# dentaiagent - DEN_030

## Übersicht

`dentaiagent` ist eine hochperformante, containerisierte Microservice-Plattform zur automatisierten Analyse und Optimierung dentalchirurgischer Workflows. Der Fokus liegt auf echtzeitfähiger Segmentierung von DVT-Datensätzen sowie prädiktiver Implantatpositionierung unter Berücksichtigung knöcherner Dichte-Matrizen. Zielsprache der gesamten Ausgabe ist **Deutsch (de)**.

## Detaillierte Architektur

### Kernfunktionalitäten

1. **Multimodale Volumensegmentierung mit GPU-Beschleunigung**  
   Nutzt ein eigenes 3D-U-Net++ mit Attention-Gates, trainiert auf >12.000 annotierten DVT-Scans. Die Inferenz erfolgt submillimetergenau (≤0,3 mm Voxelauflösung) auf NVIDIA CUDA 12.2. Parameter: `--segmentation-threshold 0.85`, `--voxel-spacing 0.25`. Ausgabe als NIfTI- oder DICOM-SEG direkt in eine PostgreSQL-Datenbank mit PostGIS-Erweiterung.

2. **Autonome Implantat-Kollisionsdetektion mit FEM-Simulation**  
   Führt Finite-Elemente-Analyse (FEM) in Echtzeit durch, um Spannungsspitzen im Kieferknochen zu prognostizieren. Parameter: `--implant-length 10.5`, `--implant-diameter 4.2`, `--bone-density-threshold 0.6 g/cm³`. Ergebnis wird als Heatmap (PNG/STL) mit millimetergenauen Risikozonen ausgegeben.

3. **Dynamische Workflow-Optimierung via Regel-Engine**  
   Eine auf Drools basierende Regel-Engine bewertet OP-Pläne gegen 47 klinische Leitlinien (DVT, CBCT, Panorama). Parameter: `--protocol DVT_2025`, `--min-nerve-distance 2.0 mm`. Ausgabe: strukturierter JSON-Report mit Ampelsystem (Grün/Gelb/Rot) für jede Risikokategorie.

### So verwenden Sie es

1. **Setup der Datenpipeline**  
   ```bash
   docker compose -f docker-compose.dentaiagent.yml up -d
   ```
   Startet PostgreSQL 16, RabbitMQ und den GPU-Inferenz-Worker. Der Worker lädt automatisch das trainierteste Modell (Version `v2.3.1-dentai`) aus einem S3-kompatiblen Bucket.

2. **Segmentierung eines DVT-Datensatzes**  
   ```bash
   dentaiagent segment --input /data/dvt_scan_001.dcm --output /results/segmentation --voxel-spacing 0.25 --segmentation-threshold 0.85
   ```
   Nach ~2,3 Sekunden pro Schicht (512×512×512 Voxel) erscheint die segmentierte Maske in `/results/segmentation`. Parallel wird der Report in die PostgreSQL-Tabelle `segmentation_results` geschrieben.

3. **Implantat-Simulation und Risikoanalyse**  
   ```bash
   dentaiagent simulate --implant-length 10.5 --implant-diameter 4.2 --bone-density-threshold 0.6 --output /results/heatmap.png
   ```
   Die Simulation berechnet in ≤400 ms die biomechanische Belastung. Das Ergebnis zeigt farbcodierte Spannungszonen (MPa) und gibt eine Empfehlung: `RISK_LEVEL: LOW`.

### Best Practices

- **Batch-Verarbeitung für Serienscans**  
  Nutzen Sie `--batch-size 16` in Kombination mit RabbitMQ-Warteschlangen, um bis zu 64 DVT-Scans parallel zu verarbeiten. Dies reduziert die Gesamtzeit pro Serie um bis zu 73 % im Vergleich zur sequenziellen Verarbeitung.

- **API-Caching für wiederkehrende Anfragen**  
  Aktivieren Sie Redis-Caching für häufig angefragte Implantat-Parameter (z. B. `--implant-length 10.5`). Die Cache-TTL sollte auf 3600 Sekunden gesetzt werden, um die Datenbanklast zu minimieren und Antwortzeiten unter 50 ms zu halten.
