### What is dentaiagent?

dentaiagent is a professional tool that helps solve problems in related fields.

### Core Features

- **Multi-Model Inference Orchestrator**: Dynamically routes inference requests across local (ONNX Runtime, TensorRT) and remote (OpenAI, Anthropic, self-hosted vLLM) backends with sub-50ms switching latency. Supports automatic fallback and load-aware model selection based on real-time token throughput metrics.
- **Structured Output Validation Pipeline**: Enforces JSON Schema and Pydantic model constraints on all LLM outputs with a configurable retry policy (max 3 retries, exponential backoff). Achieves 99.7% structural compliance on complex, nested data extraction tasks, verified via automated CI/CD regression tests.
- **Audit-Grade Context Logging**: Every agent interaction is logged to a PostgreSQL-compatible database with full prompt, response, and latency breakdowns (p50/p95/p99). Supports querying by session ID, user ID, or model name for downstream debugging and compliance audits.

### How to Use

1. **Initialize the Agent Core**: Call `dentaiagent.init(project_id="DEN_002", log_level="INFO", db_connection_string="postgresql://user:pass@localhost:5432/agent_logs")` to bootstrap the runtime. This registers all configured model endpoints and validates the structured output schemas defined in your `agent_config.yaml`.
2. **Define and Execute a Task Pipeline**: Create a pipeline using `pipeline = dentaiagent.Pipeline.from_config("pipeline_config.yaml")`. This YAML file specifies the step sequence (e.g., `extract -> classify -> summarize`), each with its own model assignment and output schema. Execute with `result = pipeline.run(input_data={"raw_text": "..."})`.
3. **Inspect and Export Audit Trail**: After execution, retrieve the full context log for that session using `session_log = dentaiagent.get_session_log(session_id=result.session_id)`. Export to JSON or CSV for external analysis using `session_log.export(format="json", filepath="./audit_export.json")`.

### Best Practices

- **Implement Tiered Model Routing**: For cost-sensitive production deployments, configure a two-tier model strategy. Use a fast, local distilled model (e.g., Llama 3.1 8B quantized via AWQ) for high-volume, simple classification tasks (target sub-300ms latency). Reserve expensive frontier models (e.g., GPT-4o) only for complex reasoning or extraction steps, triggered by a confidence threshold on the local model's output.
- **Adopt Schema-First Agent Design**: Before writing any agent logic, define your output schemas using Pydantic. This prevents hallucinated keys or malformed structures in downstream systems. Use the `dentaiagent.schema_validator` decorator on your pipeline steps to enforce this at runtime, and include schema validation as a mandatory gate in your CI/CD pipeline to catch regressions.
