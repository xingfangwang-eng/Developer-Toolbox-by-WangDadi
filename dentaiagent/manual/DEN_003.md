## Detailed Content

### What is dentaiagent?

dentaiagent is a production‑grade inference runtime that fuses 3D volumetric CBCT data with intraoral surface scans at sub‑millimeter precision. It coordinates a distributed agent mesh to perform real‑time dental structure segmentation, anomaly detection, and treatment simulation. The system is built on a custom CUDA‑optimized tensor core scheduler and communicates via gRPC with sub‑10ms latency per agent hop.

### Core Features

- **Multi‑Modal CBCT + Intraoral Fusion Engine**  
  Aligns DICOM CBCT volumes (512×512×512 voxels, 0.2mm isotropic) with STL/PLY intraoral meshes using iterative closest point (ICP) with a point‑to‑plane metric. The fusion pipeline runs at 30 FPS on an NVIDIA A100, outputting a unified coordinate frame with 0.05mm RMS error. Parameters: `--icp-max-iter 200`, `--voxel-resolution 0.2`, `--mesh-subsample 50000`.

- **Real‑Time Tooth‑Level Segmentation with Temporal Consistency**  
  Deploys a 3D U‑Net++ variant (1.8M parameters, 8‑bit quantized) that segments 32 tooth classes from CBCT slices in under 15ms per slice (batch size 8, FP16). The agent maintains a temporal buffer of 64 frames to enforce label consistency across adjacent slices, reducing flicker by 97% compared to per‑slice inference. Memory footprint: 2.1 GB VRAM.

- **Distributed Agent Mesh for Parallel Path Planning**  
  Orchestrates 16 independent agent nodes (each running on a separate CPU core or GPU partition) to compute collision‑free implant drill paths. Each agent performs a 6‑DOF rapid‑exploring random tree (RRT*) search with a 0.1mm step size, sharing frontier nodes via a Redis‑backed shared state store. The mesh converges to a Pareto‑optimal path set in under 2.5 seconds for a full‑arch case.

### How to Use

1. **Launch the Fusion Agent and Ingest Data**  
   Run the fusion agent with the CBCT volume path and intraoral mesh path:  
   `dentaiagent fusion --cbct /data/patient_001.dcm --mesh /data/scan_001.stl --output /output/fused.nrrd --icp-max-iter 200 --voxel-resolution 0.2`  
   The agent returns a unified NRRD volume with a 0.2mm isotropic grid. Monitor the fusion progress via the gRPC stream at `localhost:50051`.

2. **Invoke the Segmentation Agent with Temporal Buffer**  
   Pipe the fused volume to the segmentation agent:  
   `dentaiagent segment --input /output/fused.nrrd --output /output/labels.nrrd --batch-size 8 --temporal-buffer 64 --precision fp16`  
   The agent streams slice‑wise predictions to a local S3‑compatible store. Use the `--dry-run` flag to validate the input dimensions before full inference.

3. **Run the Path Planning Mesh and Export Results**  
   Start the distributed mesh with 16 agents:  
   `dentaiagent plan --input /output/labels.nrrd --agent-count 16 --step-size 0.1 --max-iter 10000 --output /output/paths.json`  
   Each agent writes its best path to Redis (`rpush agent:paths <json>`). After convergence, the master agent aggregates the Pareto front and exports a JSON file with drill entry points, angles, and clearance margins.

### Best Practices

- **Use FP16 Precision for Segmentation**  
  The 3D U‑Net++ agent is calibrated for FP16 inference. Training in FP32 and deploying in FP16 yields a 2.1× throughput increase with <0.3% Dice score drop. Always validate with `--precision fp16` and monitor GPU memory with `nvidia-smi` to avoid OOM.

- **Pre‑normalize CBCT Voxel Intensities to [0, 1]**  
  The fusion agent expects CBCT intensities in Hounsfield units (HU) but internally clamps to [‑1000, 3000] and normalizes. To avoid clipping artifacts, pre‑normalize your DICOM series with `--hu-window 4000 --hu-level 1000` before ingestion. This ensures consistent fusion quality across scanners.
