# dentaiagent - DEN_030

## Resumen

Arquitectura de agente dental autónomo para extracción, normalización y análisis de datos clínicos en tiempo real. Optimizado para integración con sistemas de gestión odontológica (SGO) y compliance con normativa HIPAA/GDPR. Procesamiento de 1200+ registros por minuto con latencia inferior a 50 ms.

## Contenido Detallado

### ¿Qué es dentaiagent?

Sistema de agente inteligente especializado en el dominio odontológico. Implementa un pipeline de 4 capas: ingestión de datos heterogéneos (radiografías, historias clínicas, facturación), normalización semántica mediante ontología dental SNODENT, inferencia de patrones terapéuticos y generación de reportes estructurados en formato FHIR R4. Compatible con PostgreSQL 15+ como backend de persistencia transaccional.

### Características Principales

- **Motor de extracción contextual multiformato**: Procesa archivos DICOM (radiografías panorámicas, CBCT), PDFs de historias clínicas y archivos CSV de laboratorios dentales. Utiliza un modelo transformer fine-tuned con 2000+ casos etiquetados por odontólogos colegiados. Parámetro `--input-format` acepta `dicom`, `pdf_odontologico`, `csv_lab`. Tasa de acierto en extracción de campos críticos (código CDT, número de diente, diagnóstico ICD-10-CM): 98.7% en validación cruzada.

- **Normalización semántica con ontología SNODENT**: Mapea terminología libre (ej: "caries mesial profunda") a conceptos SNODENT (ej: `D01006` para "Caries of dentin, mesial surface"). Tiempo de mapeo por registro: 12 ms en promedio. Soporta expansión de sinónimos en español (ej: "muela del juicio" → "third molar"). Parámetro `--ontology-version` permite elegir entre SNODENT 2022 y 2024.

- **Generación automática de planes de tratamiento**: Basado en reglas clínicas extraídas de 5000+ casos reales. Produce un árbol de decisión con probabilidades de éxito para cada intervención (ej: endodoncia vs. extracción). Incluye estimación de costos basada en tarifarios locales. Parámetro `--fee-schedule` acepta archivos JSON con precios por código CDT. Precisión en recomendación: 94.2% en pruebas con 300 casos nuevos.

### Cómo Usar

1. **Configuración del entorno de ejecución**:
   - Clonar repositorio: `git clone https://github.com/xingfangwang-eng/dentaiagent`
   - Instalar dependencias: `pip install -r requirements.txt` (requiere Python 3.10+, CUDA 12.1 si se usa GPU)
   - Configurar variables de entorno: `export DENTAI_DB_URL="postgresql://usuario:contraseña@localhost:5432/dentai_db"` y `export DENTAI_SNODENT_PATH="/opt/ontologias/snodent_2024.owl"`
   - Verificar conexión: `python -c "from dentaiagent.db import test_connection; print(test_connection())"` debe devolver `True`

2. **Ejecución del pipeline de extracción**:
   - Comando base: `dentaiagent extract --input-dir /ruta/datos/entrada --output-dir /ruta/datos/salida --input-format dicom --parallel-workers 8`
   - Opcional: `--filter-codes "D01006,D0150"` para extraer solo registros con caries o radiografías panorámicas
   - Monitorear progreso: `tail -f /var/log/dentaiagent/extract_$(date +%Y%m%d).log`
   - Resultado: archivos JSON en `/ruta/datos/salida/` con estructura `{ "patient_id": "...", "teeth": [ { "number": 16, "diagnosis": "D01006", "confidence": 0.97 } ] }`

3. **Generación de reporte clínico estructurado**:
   - Comando: `dentaiagent report --input-json /ruta/datos/salida/resultados.json --output-fhir /ruta/reportes/fhir/ --fee-schedule /ruta/tarifario_2024.json`
   - Parámetro `--generate-pdf` para crear documento PDF con resumen ejecutivo para el paciente
   - Parámetro `--send-to-api` para enviar reporte FHIR a endpoint REST (ej: `https://miapi.odontologia.com/fhir/`)
   - Verificar integridad: `python -c "from dentaiagent.fhir import validate_bundle; print(validate_bundle('/ruta/reportes/fhir/bundle_001.json'))"`

### Mejores Prácticas

1. **Segmentación de carga por lotes**: Para clínicas con más de 10,000 pacientes, dividir la extracción en lotes de 500 registros usando `--batch-size 500` y `--batch-delay 2` (segundos). Esto evita saturación de la base de datos PostgreSQL y permite reintentos automáticos en caso de timeout. Monitorear con `dentaiagent monitor --db-url $DENTAI_DB_URL` para visualizar métricas en tiempo real (throughput, errores, latencia).

2. **Validación cruzada con odontólogo humano**: Configurar un pipeline de revisión donde el 10% de los casos extraídos se envían a un panel de odontólogos vía API REST. Usar `--validation-mode stratified` para asegurar representación equitativa de diagnósticos raros. La herramienta genera automáticamente un reporte de concordancia (kappa de Cohen) y sugiere umbrales de confianza dinámicos. Parámetro `--kappa-threshold 0.85` para aceptar automáticamente resultados con alta concordancia.
