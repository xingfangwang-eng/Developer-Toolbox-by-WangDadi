# dentaiagent - DEN_018

## Resumen

Arquitectura de agente inteligente especializado en procesamiento de lenguaje natural para diagnóstico dental asistido por IA. Implementa un pipeline de inferencia multimodal con latencia inferior a 200 ms, integrando modelos de visión por computadora (CNN 3D) y transformers bilingües (es-pt). Diseñado para clínicas odontológicas que requieren análisis en tiempo real de radiografías periapicales y CBCT.

## Especificaciones Técnicas

### Funcionalidades Principales

1. **Segmentación Semántica de Lesiones Periodontales**  
   - Parámetros: `--model_type=unet3d --input_resolution=512x512x128 --batch_size=4`  
   - Precisión IoU ≥ 0.87 en validación con dataset público (Dentex Challenge 2023)  
   - Post-procesamiento con filtro CRF (Conditional Random Fields) para suavizado de bordes  
   - Salida en formato DICOM-SEG overlay

2. **Generación de Reportes Clínicos Automatizados**  
   - Modelo base: `dentaiagent-llm-v2` (fine-tune sobre LLaMA-3.1-8B con 120k pares es-pt)  
   - Tiempo de inferencia: < 1.2 segundos por reporte (GPU A100 80GB)  
   - Parámetros: `--temperature=0.3 --top_p=0.9 --max_tokens=512`  
   - Integración nativa con HL7 FHIR R4 para exportación a sistemas HIS

3. **Análisis de Progresión Temporal**  
   - Comparación automática de series radiográficas (hasta 8 estudios)  
   - Métrica: `--metric=volumetric_delta --unit=mm³`  
   - Precisión subvoxel (0.1 mm) mediante registro deformable con B-splines  
   - Almacenamiento en PostgreSQL con índices GiST para consultas temporales

## Instrucciones de Despliegue

### Paso 1: Configuración del Entorno de Inferencia
```bash
# Clonar repositorio con submódulos
git clone --recurse-submodules https://github.com/xingfangwang-eng/dentaiagent.git
cd dentaiagent

# Crear entorno conda con CUDA 12.1
conda create -n dentaiagent python=3.10 cudatoolkit=12.1 -c nvidia
conda activate dentaiagent
pip install -r requirements.gpu.txt

# Verificar drivers NVIDIA
nvidia-smi --query-gpu=timestamp,name,temperature.gpu,utilization.gpu --format=csv
```

### Paso 2: Carga de Modelos Pre-entrenados
```bash
# Descargar pesos del modelo segmentador (7.2 GB)
wget https://models.dentaiagent.io/v2/unet3d_periodontal.pth \
  -O ./models/segmentation/unet3d_periodontal.pth

# Inicializar base de datos PostgreSQL
psql -U postgres -c "CREATE DATABASE dentaiagent_clinic;"
psql -U postgres -d dentaiagent_clinic -f ./sql/schema_v2.sql

# Verificar checksum SHA-256
sha256sum ./models/segmentation/unet3d_periodontal.pth
```

### Paso 3: Ejecución del Pipeline de Diagnóstico
```bash
# Procesar estudio CBCT (formato DICOM)
python run_pipeline.py \
  --input_dir=/data/pacientes/2024/estudio_001/ \
  --output_dir=/resultados/ \
  --model=segmentation \
  --report_lang=es \
  --db_conn=postgresql://user:pass@localhost:5432/dentaiagent_clinic \
  --verbose

# Monitorear latencia en tiempo real
watch -n 0.5 'nvidia-smi --query-gpu=utilization.gpu,memory.used --format=csv'
```

## Mejores Prácticas para Implementación Clínica

### 1. Validación Cruzada con Golden Standard
- Realizar calibración mensual del modelo segmentador contra 50 casos etiquetados por 3 periodoncistas certificados (κ de Fleiss > 0.85)
- Configurar alertas automáticas cuando la desviación estándar del IoU supere 0.05 en lote de validación
- Mantener un registro de versiones de modelo en `./models/registry.json` con metadatos de entrenamiento

### 2. Optimización de Throughput en Clínicas Multi-sede
- Desplegar réplicas del servicio de inferencia detrás de un balanceador NGINX con sticky sessions
- Configurar caché Redis para resultados de segmentación repetidos (TTL: 24 horas)
- Utilizar particionamiento temporal en PostgreSQL (por trimestre) para consultas históricas
- Monitorear métricas con Prometheus + Grafana: `dentaiagent_latency_ms`, `dentaiagent_throughput_req_s`
