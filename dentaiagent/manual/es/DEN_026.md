# dentaiagent - DEN_026

## Resumen

Arquitectura de agente autónomo para diagnóstico dental basado en visión por computadora y razonamiento simbólico. Optimizado para inferencia en tiempo real con modelos de segmentación semántica entrenados en 12.000 radiografías panorámicas etiquetadas. Integración directa con flujos DICOM y exportación de métricas clínicas en formato FHIR.

## Contenido Detallado

### ¿Qué es dentaiagent?

Dentaiagent es un sistema de agente inteligente diseñado para orquestar flujos de trabajo de diagnóstico dental automatizado. Combina un pipeline de preprocesamiento de imágenes (normalización de histograma, corrección de contraste adaptativo) con un motor de reglas clínicas parametrizables. El agente decide dinámicamente qué modelo de segmentación invocar según la región anatómica detectada (por ejemplo, molares, incisivos, conductos radiculares). Toda la lógica de inferencia se ejecuta en memoria compartida con un tiempo de respuesta garantizado inferior a 200 ms por radiografía.

### Características Principales

- **Segmentación por regiones anatómicas con validación de bordes**: El agente aplica un modelo U-Net++ con backbone EfficientNet-B4 sobre la imagen ortopantomográfica. El resultado es una máscara multiclase (esmalte, dentina, pulpa, hueso alveolar, lesiones periapicales) con precisión de píxel. Se utiliza un posprocesamiento con operaciones morfológicas (cierre, apertura) para eliminar artefactos de borde. Los parámetros de umbral de confianza se definen por archivo YAML (`confidence_threshold: 0.85`, `min_region_area_px: 150`).

- **Motor de reglas clínicas con razonamiento temporal**: Se implementa un sistema de producción basado en Rete con hechos clínicos almacenados en PostgreSQL. Cada diagnóstico genera una tupla de eventos (`patient_id`, `timestamp_ms`, `finding_code`, `severity_score`). El agente puede encadenar reglas como: si `caries_profunda > 0.7` y `distancia_pulpar < 2.5 mm`, entonces activar `alerta_endodoncia`. La base de reglas se actualiza en caliente mediante archivos DRL (Drools Rule Language) versionados.

- **Exportación automática de informes estructurados FHIR**: Al finalizar el análisis, el agente genera un recurso `DiagnosticReport` y `Observation` en formato JSON FHIR R4. Los campos incluyen `code.coding.system: "http://dentaiagent.org/finding-codes"`, `valueQuantity.value` para mediciones lineales (en milímetros, precisión de 0.01 mm), y `component` para cada región segmentada. La salida se envía directamente a un endpoint REST configurable mediante variable de entorno `FHIR_ENDPOINT_URL`.

### Cómo Usar

1. **Inicializar el pipeline de preprocesamiento**: Ejecutar el comando `dentaiagent init --input-dir ./radiografias/ --output-dir ./preprocessed/ --normalize-method clahe` para aplicar ecualización de histograma adaptativa con clip limit (clip_limit=2.0, tile_grid_size=8x8). Esto genera archivos PNG de 16 bits con metadatos DICOM preservados en un archivo JSON adjunto.

2. **Ejecutar inferencia con umbral personalizado**: Llamar al endpoint gRPC `InferenceService/ProcessRadiograph` con el payload `{ "image_id": "PAN_20250321_001", "model_version": "v2.3.1", "confidence_threshold": 0.9 }`. La respuesta incluye una máscara codificada en base64 y una tabla de áreas por clase en píxeles cuadrados. El tiempo de inferencia se mide en microsegundos y se registra en el archivo `logs/inference_latency.csv`.

3. **Generar informe FHIR y enviar a HIS**: Usar el comando `dentaiagent report --patient-id P12345 --output-format fhir --push-endpoint https://his.hospital.org/fhir`. El agente valida el recurso contra el perfil `http://dentaiagent.org/StructureDefinition/dental-observation`. Si el endpoint responde con código 201, el informe se marca como `entregado` en la base de datos local (SQLite, en `~/.dentaiagent/state.db`).

### Mejores Prácticas

- **Calibración periódica del umbral de confianza**: Realizar una validación cruzada semanal con un conjunto de 50 radiografías etiquetadas por tres especialistas. Ajustar `confidence_threshold` en función del F1-score por clase. Para lesiones cariosas, un umbral de 0.75 puede ser suficiente; para fisuras radiculares, se recomienda 0.95 para reducir falsos positivos.

- **Versionado de modelos y reglas clínicas**: Mantener un repositorio Git para los archivos de pesos del modelo (`.h5` o `.onnx`) y los archivos DRL. Cada despliegue debe etiquetarse con un número de versión semántico (ej. `v2.4.0`). El agente carga automáticamente la última versión estable desde un bucket S3 configurado en `MODEL_REGISTRY_URL`. Nunca modificar reglas en producción sin antes ejecutar el suite de pruebas unitarias (`pytest tests/ --cov=dentaiagent`).
