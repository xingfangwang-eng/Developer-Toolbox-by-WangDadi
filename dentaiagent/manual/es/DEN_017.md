# dentaiagent - DEN_017

## Resumen

Arquitectura de agente de inteligencia artificial especializada en diagnóstico dental asistido por IA, diseñada para procesamiento en tiempo real de imágenes radiográficas y CBCT. Integra modelos de segmentación semántica con precisión submilimétrica, orquestación de microservicios en Kubernetes y sincronización bidireccional con bases de datos PostgreSQL para auditoría de decisiones clínicas.

## Contenido Detallado

### ¿Qué es dentaiagent?

Sistema de inferencia distribuida que despliega un ensemble de redes neuronales convolucionales (CNN) y transformers visión-lenguaje para analizar ortopantomografías, tomografías volumétricas y fotografías intraorales. La arquitectura implementa un pipeline de preprocesamiento con normalización de histograma adaptativa, detección de landmarks anatómicos mediante keypoint regression y clasificación de patologías con umbral de confianza configurable (mín. 0.85). El motor de reglas clínicas, escrito en Go, valida cada predicción contra la base de conocimiento ontológica SNOMED-CT antes de emitir recomendaciones.

### Características Principales

- **Segmentador de Tejidos con Atención Jerárquica**: Modelo U-Net++ con mecanismo de atención espacial y canales, entrenado sobre 47.000 CBCT anotadas. Alcanza un Dice Score de 0.94 para esmalte/dentina y 0.91 para nervio alveolar inferior. El módulo de postprocesamiento aplica morfología matemática (apertura/clausura con kernel 5x5) para eliminar falsos positivos menores a 0.3 mm³. Parámetro clave: `--min_volume_threshold=0.5` (en mm³) para filtrado de ruido.

- **Motor de Dictado Clínico con Compensación de Latencia**: Transformer T5 fine-tuneado con 120.000 transcripciones de consultas odontológicas en español. El pipeline de ASR (Whisper large-v3) está acoplado a un predictor de próxima palabra que reduce el tiempo de finalización de frases en 210 ms promedio. La salida se formatea automáticamente según la plantilla SOAP (Subjetivo, Objetivo, Evaluación, Plan) con validación de campos obligatorios. Parámetro: `--soap_validation_strict=true` para rechazar entradas incompletas.

- **Orquestador de Flujo de Trabajo con Tolerancia a Fallos**: Implementación de saga pattern sobre RabbitMQ con colas dead-letter y retry exponencial (base=2, máximo 5 reintentos). Cada etapa del diagnóstico (recepción de imagen → segmentación → clasificación → generación de informe) se ejecuta como un Job independiente en Kubernetes con recursos limitados (CPU: 2 cores, RAM: 4 GB). Métrica crítica: latencia p99 del pipeline completo ≤ 2.3 segundos para imágenes 2048x2048.

### Cómo Usar

1. **Inicializar el Agente con Configuración de Hardware**:
   ```bash
   dentaiagent init --gpu_memory_fraction=0.85 --batch_size=4 --precision=fp16
   ```
   Este comando asigna el 85% de la memoria VRAM disponible, establece un lote de 4 imágenes por inferencia y activa el modo de precisión mixta (Tensor Cores). Verificar con `nvidia-smi` que la temperatura GPU no exceda 75°C.

2. **Ejecutar Diagnóstico con Validación Cruzada**:
   ```bash
   dentaiagent diagnose --input_dir=/data/cbct/ --output_dir=/results/ \
     --model_ensemble="segmentation_v2,classification_v3" \
     --consensus_threshold=0.80 --save_intermediate_masks=true
   ```
   Carga todas las imágenes DICOM del directorio de entrada, ejecuta los modelos ensemble y genera un informe PDF solo si ambos modelos coinciden con una confianza ≥ 80%. Los mapas de segmentación intermedios se guardan en formato NIfTI para depuración.

3. **Auditar Decisión con Trazabilidad Completa**:
   ```bash
   dentaiagent audit --case_id=DEN-017-20231027-001 --export_json=true
   ```
   Extrae de PostgreSQL el registro completo de inferencia: parámetros del modelo (versión, hash del checkpoint), tiempo de cómputo por etapa (en milisegundos), máscaras de segmentación en base64 y metadatos del paciente anonimizados. El JSON exportado incluye firma digital SHA-256 para integridad forense.

### Mejores Prácticas

- **Calibración de Umbrales por Tipo de Patología**: Ajustar `--consensus_threshold` entre 0.75 (caries proximales) y 0.95 (fracturas radiculares) usando el subcomando `dentaiagent calibrate --pathology_type=...`. Esto reduce falsos negativos en lesiones incipientes sin sacrificar especificidad en casos críticos. Recomendación: ejecutar calibración semanal con 200 casos etiquetados por un radiólogo oral.

- **Estrategia de Despliegue con Canary Releases**: Para actualizar el modelo de segmentación, lanzar primero el nuevo checkpoint en el 10% de los nodos del cluster Kubernetes usando `dentaiagent deploy --canary_percentage=10 --rollback_on_error=true`. Monitorear la métrica `inference_accuracy_delta` en Grafana; si cae más de 0.02 puntos Dice, el orquestador revierte automáticamente al modelo anterior sin intervención manual.
