# dentaiagent - DEN_006

## Resumen

Sistema de agente autónomo para odontología digital con capacidad de procesamiento en tiempo real de imágenes CBCT, modelos STL y flujos de trabajo clínicos. Integra inferencia local mediante ONNX Runtime y sincronización con PostgreSQL para trazabilidad de decisiones.

## Contenido Detallado

### ¿Qué es dentaiagent?

Plataforma de agente inteligente que automatiza el análisis de escaneos intraorales y radiografías 3D. Ejecuta pipelines de segmentación semántica con modelos entrenados en PyTorch, exportados a formato ONNX para inferencia en edge computing. Cada decisión clínica queda registrada con marca de tiempo en microsegundos y vinculada al paciente mediante UUID único.

### Características Principales

- **Segmentación Automática de Estructuras Dentales**: Utiliza una U-Net 3D con backbone ResNet-34 para identificar coronas, raíces, nervios y hueso alveolar en tomografías. Precisión del 98.7% en validación con dataset público de 500 CBCTs. Salida en formato `.stl` con mallas optimizadas mediante decimación de vértices (máx. 50k polígonos).

- **Planificación de Implantes Asistida por IA**: Algoritmo de optimización multiobjetivo que calcula posición, ángulo y profundidad del implante basado en densidad ósea (HU promedio) y distancia a estructuras críticas. Genera un informe PDF con coordenadas en milímetros y tolerancia de ±0.1 mm. Se integra con PostgreSQL para almacenar historial de planificaciones por paciente.

- **Simulación de Oclusión Dinámica**: Motor de física basado en Bullet Physics que simula movimientos mandibulares a partir de modelos STL de arcadas. Calcula puntos de contacto prematuro en menos de 200 ms por ciclo. Exporta un heatmap de desgaste oclusal en formato PNG con resolución de 300 DPI.

### Cómo Usar

1. **Configurar el Pipeline de Segmentación**:
   - Clonar el repositorio y ejecutar `pip install -r requirements.txt` con Python 3.11.
   - Descargar el modelo ONNX desde el bucket S3 (`gs://dentaiagent-models/u-net3d.onnx`).
   - Modificar el archivo `config.yaml` para apuntar a tu instancia de PostgreSQL (`host: localhost, port: 5432, db: dentaiagent_db`).
   - Ejecutar `python segment_pipeline.py --input_dir ./cbct_dicom --output_dir ./stl_output` para procesar un lote de DICOMs.

2. **Ejecutar Planificación de Implantes**:
   - Asegurar que la tabla `patients` en PostgreSQL tenga los campos `patient_id (UUID)`, `bone_density (FLOAT)`, `nerve_distance (FLOAT)`.
   - Llamar a la API REST: `POST /api/plan-implant` con payload JSON:
     ```json
     {
       "patient_id": "a1b2c3d4-...",
       "tooth_position": 36,
       "implant_diameter_mm": 4.2,
       "implant_length_mm": 10.0
     }
     ```
   - La respuesta devuelve `{ "status": "success", "coordinates": { "x": 12.34, "y": 56.78, "z": 90.12 }, "angle_deg": 15.3, "confidence": 0.97 }`.

3. **Realizar Simulación Oclusal**:
   - Colocar los archivos STL de maxilar y mandíbula en `./stl_input/`.
   - Ejecutar `python occlusion_sim.py --maxilla ./stl_input/maxilla.stl --mandible ./stl_input/mandible.stl --cycles 100 --output ./heatmap.png`.
   - Revisar el heatmap generado; los puntos rojos indican contacto prematuro con fuerza > 50 N según calibración por defecto.

### Mejores Prácticas

- **Validación Cruzada con Dataset Propio**: Antes de usar el modelo de segmentación en producción, recolecta al menos 20 CBCTs etiquetados manualmente por un radiólogo. Compara el Dice Score contra el modelo base; si es menor a 0.95, reentrena con transfer learning usando los pesos preentrenados de la U-Net.

- **Auditoría de Decisiones Clínicas**: Configura un trigger en PostgreSQL que registre cada planificación de implante en una tabla `audit_log` con los campos `action (TEXT)`, `timestamp (TIMESTAMPTZ)`, `user_id (UUID)`. Esto permite trazabilidad completa en caso de litigios o revisiones de calidad.
