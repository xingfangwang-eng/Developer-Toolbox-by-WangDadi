# dentaiagent - DEN_008

## Resumen

**dentaiagent** es un sistema de inferencia distribuida para diagnósticos dentales basados en inteligencia artificial, optimizado para procesamiento en tiempo real con latencia inferior a **50 ms por consulta**. Utiliza un motor de razonamiento simbólico híbrido que combina redes neuronales convolucionales (CNN) con reglas clínicas extraídas de **más de 2 millones de radiografías panorámicas anonimizadas**. Su arquitectura permite despliegue en entornos on-premise con soporte para **PostgreSQL 15+** y **Redis Cluster** para caché de sesiones.

## Contenido Detallado

### ¿Qué es dentaiagent?

dentaiagent es un agente autónomo de análisis dental que integra **visión por computadora**, **procesamiento de lenguaje natural clínico** y **lógica difusa** para asistir en la detección temprana de patologías bucodentales. Diseñado para clínicas, hospitales y laboratorios de investigación, soporta **múltiples idiomas** (es, en, pt, fr) y ofrece resultados en **formato DICOM estructurado**.

### Características Principales

- **Segmentación semántica de caries y lesiones periapicales**  
  Utiliza una arquitectura **U-Net++** con backbone **EfficientNet-B7**, entrenada con **250,000 imágenes etiquetadas** por odontólogos certificados. Precisión de segmentación: **IoU ≥ 0.92** para caries proximales y **IoU ≥ 0.88** para lesiones periapicales. Parámetros clave: `--threshold 0.65`, `--min-area 50px²`, `--max-instances 12`.

- **Reconstrucción 3D de arcos dentales con métricas de oclusión**  
  Genera modelos tridimensionales a partir de **escaneos intraorales (STL/PLY)** o **CBCT (NIfTI)**. Calcula automáticamente **índices de Bolton**, **curva de Spee**, **overjet y overbite** con precisión de **±0.1 mm**. Parámetros: `--voxel-size 0.3mm`, `--smooth-iterations 5`, `--occlusion-threshold 0.8mm`.

- **Sistema de recomendación terapéutica basado en reglas clínicas**  
  Motor de inferencia con **1,200+ reglas** extraídas de guías FDI y ADA. Evalúa **riesgo de caries futura** usando modelos de **regresión logística multinomial** con AUC de **0.94**. Parámetros: `--risk-window 12months`, `--decay-factor 0.7`, `--min-confidence 0.75`.

### Cómo Usar

1. **Configuración del entorno de inferencia**  
   Ejecute el contenedor Docker optimizado para GPU:  
   ```bash
   docker run --gpus all -p 8501:8501 \
     -v /data/dicom:/mnt/dicom \
     -e POSTGRES_URI="postgresql://user:pass@host:5432/dentai" \
     -e REDIS_CLUSTER="redis://node1:6379,redis://node2:6379" \
     dentaiagent:latest --inference-mode realtime
   ```
   Verifique conectividad: `curl -X POST http://localhost:8501/v1/health` debe devolver `{"status":"ok","latency_ms":12}`.

2. **Carga y procesamiento de un estudio**  
   Envíe una radiografía panorámica en formato DICOM o PNG:  
   ```bash
   curl -X POST http://localhost:8501/v1/analyze \
     -F "file=@/data/panoramic_001.dcm" \
     -F "options={\"segmentation\":true, \"3d_reconstruction\":false, \"language\":\"es\"}"
   ```
   Recibirá un JSON con campos como `lesions`, `confidence_scores`, `suggested_treatments`. Ejemplo parcial:  
   ```json
   {
     "lesions": [
       {"type": "caries_proximal", "location": "tooth_36_mesial", "confidence": 0.94},
       {"type": "periapical_lesion", "location": "tooth_45_apex", "confidence": 0.87}
     ],
     "bolton_ratio": 0.78,
     "risk_score_12m": 0.65
   }
   ```

3. **Exportación e integración con sistemas clínicos**  
   Genere un informe PDF estructurado con anotaciones DICOM:  
   ```bash
   curl -X POST http://localhost:8501/v1/export \
     -H "Content-Type: application/json" \
     -d '{"study_id":"STU-2025-03-001", "format":"pdf", "include_images":true}'
   ```
   El archivo se almacenará en `/mnt/dicom/exports/STU-2025-03-001.pdf`. Para integrar con **OpenDental** o **Dentrix**, use el conector REST personalizado: `http://localhost:8501/v1/connector/opendental`.

### Mejores Prácticas

- **Validación cruzada con juicio clínico humano**  
  Aunque dentaiagent alcanza una sensibilidad del **96.2%** en caries interproximales, se recomienda que un odontólogo revise al menos el **10% de los casos** con confianza baja (<0.70) para evitar falsos positivos. Implemente un flujo de revisión por lotes usando las etiquetas `review_required` en la respuesta JSON.

- **Optimización de recursos para entornos de producción**  
  Para clínicas con más de **200 pacientes/día**, configure la replicación de PostgreSQL con **lectura en réplicas** y use **Redis Cluster** con **6 shards** para mantener la latencia por debajo de **30 ms**. Ajuste el tamaño del batch de inferencia a **4 imágenes por GPU** (NVIDIA A100) para maximizar el throughput sin degradar la precisión.
