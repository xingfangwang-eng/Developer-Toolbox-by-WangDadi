# dentaiagent - DEN_011

## Resumen

Sistema de inferencia distribuida para diagnóstico dental con modelos de lenguaje especializados en odontología. Optimizado para procesamiento en tiempo real con latencias inferiores a 200 ms por consulta. Integra análisis radiográfico automatizado y generación de planes de tratamiento personalizados. Proyecto open-source bajo licencia MIT, diseñado para clínicas y laboratorios dentales que requieren precisión quirúrgica en la toma de decisiones.

## Contenido Detallado

### ¿Qué es dentaiagent?

Plataforma de agente inteligente especializado en odontología que combina:
- Motor de razonamiento clínico basado en ontología dental DICOM
- Pipeline de visión por computadora para segmentación de caries, fracturas y patologías periodontales
- Módulo de generación de narrativas clínicas en español latinoamericano con validación semántica contra el estándar ICD-10-CM

### Características Principales

1. **Segmentación Automática de Radiografías Panorámicas**  
   - Red neuronal U-Net++ entrenada con 15,000+ radiografías anotadas por odontólogos certificados  
   - Precisión de segmentación: 94.7% IoU para caries interproximales  
   - Parámetros clave: `--threshold 0.65` para filtrado de falsos positivos, `--min-area 50px` para excluir artefactos  
   - Tiempo de inferencia: 180 ms por imagen en GPU NVIDIA A10G

2. **Generación de Planes de Tratamiento Contextuales**  
   - Modelo transformer fine-tuned con 2.5 millones de pares diagnóstico-tratamiento  
   - Soporte para 12 especialidades: endodoncia, periodoncia, ortodoncia, implantología, etc.  
   - Parámetros: `--max-tokens 1024`, `--temperature 0.3` para consistencia clínica  
   - Integración con PostgreSQL para persistencia de historiales: `pgvector` para búsqueda semántica de casos similares en <50 ms

3. **Validación de Prescripciones Farmacológicas**  
   - Base de datos integrada de 3,200+ medicamentos dentales con interacciones, dosis máximas y contraindicaciones  
   - Motor de reglas: verifica alergias (input `--patient-allergies`), edad (`--age-years`), peso (`--weight-kg`), y función hepática (`--alt-iu/l`)  
   - Tiempo de validación: 45 ms por prescripción, con alertas de severidad (Alta/Media/Baja)  
   - Exportación a receta PDF con código QR verificable

### Cómo Usar

1. **Configuración del Entorno Clínico**  
   - Clonar repositorio: `git clone https://github.com/xingfangwang-eng/dentaiagent`  
   - Instalar dependencias: `pip install -r requirements.txt --extra-index-url https://download.pytorch.org/whl/cu118`  
   - Configurar archivo `.env` con credenciales de PostgreSQL y endpoint de API de modelos:  
     ```env
     DB_HOST=localhost
     DB_PORT=5432
     DB_NAME=dentai_prod
     MODEL_ENDPOINT=http://localhost:8000/v1/chat/completions
     RADIOLOGY_THRESHOLD=0.65
     ```

2. **Carga y Análisis de Radiografía**  
   - Ejecutar script de segmentación:  
     ```bash
     python segment.py --input /ruta/radiografia.dcm --output /ruta/resultados/ --threshold 0.65 --min-area 50
     ```  
   - El script genera un archivo JSON con bounding boxes y etiquetas (caries, fractura, cálculo, etc.)  
   - Visualizar resultados: `python visualize.py --input /ruta/resultados/segmentacion.json --overlay true`

3. **Generación de Plan de Tratamiento**  
   - Ingresar datos del paciente vía API REST:  
     ```bash
     curl -X POST http://localhost:8000/generate-plan \
       -H "Content-Type: application/json" \
       -d '{
         "patient_id": "P-2024-001",
         "age_years": 45,
         "diagnosis": "Múltiples caries interproximales en molares inferiores",
         "radiograph_results": "/ruta/resultados/segmentacion.json",
         "allergies": ["penicilina", "sulfamidas"],
         "weight_kg": 72
       }'
     ```  
   - Respuesta incluye plan estructurado: fases (urgencia, restauración, mantenimiento), materiales sugeridos (resina compuesta 3M Filtek Z350 XT), y receta farmacológica validada.

### Mejores Prácticas

1. **Calibración Semanal del Modelo de Segmentación**  
   - Ejecutar validación contra un conjunto de prueba de 200 radiografías anotadas cada lunes  
   - Monitorear métricas: IoU promedio debe mantenerse >0.90; si cae por debajo, reentrenar con 500 nuevos casos  
   - Utilizar `--eval-mode` para generar reporte de confianza por clase: `python segment.py --eval --test-dir /ruta/test_set`

2. **Auditoría de Prescripciones con Human-in-the-Loop**  
   - Configurar webhook en `dentaiagent` para notificar al odontólogo supervisor ante alertas de severidad Alta  
   - Implementar bloqueo automático si `--age-years < 12` y medicamento contiene tetraciclina (riesgo de tinción dental)  
   - Almacenar todas las decisiones en tabla `prescription_audit` de PostgreSQL con timestamp y usuario validador
