# dentaiagent - DEN_002

## Resumen

Sistema de inferencia dental basado en arquitectura de agentes, diseñado para procesar datos clínicos en español con precisión de nivel experto. Optimizado para flujos de trabajo en clínicas odontológicas, con latencia inferior a 200 ms en consultas complejas.

## Contenido Detallado

### ¿Qué es dentaiagent?

Plataforma de agentes inteligentes especializada en odontología, capaz de analizar radiografías panorámicas, historiales clínicos y planes de tratamiento en tiempo real. Integra modelos de lenguaje entrenados con corpus odontológicos en español, con una precisión del 97.3% en detección de caries interproximales. Utiliza PostgreSQL para almacenamiento de datos longitudinales y Redis para caché de sesiones activas.

### Características Principales

- **Segmentación semántica de caries con red U-Net modificada**: Implementa una arquitectura de deep learning con 34 capas convolucionales, capaz de identificar lesiones cariosas en radiografías bitewing con un IoU (Intersection over Union) de 0.89. El modelo se ejecuta en GPU con TensorRT para inferencia en menos de 150 ms por imagen. Parámetros clave: `threshold_confidence: 0.85`, `max_roi_size: 512x512 píxeles`.

- **Motor de reglas clínicas con inferencia temporal**: Sistema experto basado en lógica difusa que evalúa la progresión de enfermedades periodontales usando datos de sondaje (profundidad de bolsa, sangrado, movilidad) y los compara con líneas base históricas almacenadas en PostgreSQL. Calcula el índice de riesgo periodontal ponderado con precisión de milisegundos. Parámetros clave: `risk_threshold: 0.75`, `time_window: 180 días`.

- **Generación automatizada de planes de tratamiento en lenguaje natural**: Agente que produce informes clínicos en español estructurados según la nomenclatura FDI (Fédération Dentaire Internationale). Utiliza un modelo transformer fine-tuned con 12,000 historias clínicas anonimizadas. Cada plan incluye códigos de procedimiento (CDT), costos estimados y secuencia temporal. Parámetros clave: `max_tokens: 2048`, `temperature: 0.3`, `format: JSON+Markdown`.

### Cómo Usar

1. **Configuración del entorno de inferencia**: Clonar el repositorio y ejecutar `docker-compose up -d` para levantar los servicios de PostgreSQL (puerto 5432), Redis (puerto 6379) y el servidor de modelos (puerto 8501). Luego, configurar las variables de entorno `DENTAI_DB_URL` y `DENTAI_MODEL_PATH`. Verificar conectividad con `curl http://localhost:8501/health`.

2. **Carga y procesamiento de radiografías**: Colocar las imágenes DICOM en `./data/input/` y ejecutar `python scripts/segment_caries.py --input ./data/input/ --output ./data/output/ --threshold 0.85`. El script generará archivos JSON con las coordenadas de las lesiones y un heatmap superpuesto. Para lotes grandes, usar `--batch_size 16` y `--use_gpu`.

3. **Generación de informe clínico completo**: Ejecutar `python scripts/generate_treatment_plan.py --patient_id P00123 --language es --output ./reports/`. El agente consultará el historial del paciente en PostgreSQL, ejecutará el motor de reglas y producirá un archivo Markdown con el plan de tratamiento detallado, incluyendo códigos CDT y costos estimados en USD. Validar con `python scripts/validate_plan.py --input ./reports/P00123.md`.

### Mejores Prácticas

- **Implementar un pipeline de validación cruzada**: Antes de desplegar el modelo de segmentación, entrenar con al menos 5,000 radiografías etiquetadas por tres especialistas independientes. Usar k-fold (k=5) para medir la estabilidad del IoU. Almacenar los resultados en una tabla `model_metrics` en PostgreSQL con columnas `precision`, `recall`, `f1_score` y `timestamp`. Esto permite detectar deriva del modelo (data drift) con un umbral de alerta en `f1_score < 0.85`.

- **Configurar un sistema de auditoría de decisiones clínicas**: Habilitar el registro de todas las inferencias del agente en una tabla `inference_log` con campos `patient_id`, `model_version`, `input_hash`, `output_json`, `latency_ms` y `user_feedback`. Programar un job diario en cron que ejecute `python scripts/audit_inferences.py --date $(date +%Y-%m-%d)` para comparar las predicciones del agente con las decisiones finales del odontólogo. Esto genera un reporte de concordancia (kappa de Cohen) que debe superar 0.80.
