# dentaiagent - DEN_023

## Resumen Técnico

`dentaiagent` es un motor de agente autónomo especializado en la orquestación de pipelines de procesamiento de lenguaje natural para dominios clínicos y dentales. Diseñado para operar con latencias inferiores a **200 ms por inferencia**, integra un sistema de memoria contextual con persistencia en **PostgreSQL + pgvector**, permitiendo búsquedas semánticas con **precisión top-5 > 94%** en corpus especializados.

## Funcionalidades Principales

### 1. Orquestador de Diagnóstico Asistido por Agentes (ADAO)

El módulo **ADAO** despliega una arquitectura de múltiples agentes especializados (extracción de entidades, clasificación de síntomas, generación de hipótesis) que se comunican mediante un bus de eventos asíncrono basado en **Redis Streams**. Cada agente ejecuta en un contenedor aislado con límites de memoria configurables (`MEMORY_LIMIT_MB=512`). El tiempo total de ciclo de diagnóstico no supera los **1.2 segundos** en hardware estándar (CPU 4 cores, 8 GB RAM).

**Parámetros clave:**
- `MAX_AGENT_TIMEOUT_MS`: 800
- `CONTEXT_WINDOW_TOKENS`: 4096
- `EMBEDDING_MODEL`: `text-embedding-3-small` con dimensionalidad 1536

### 2. Sistema de Memoria Clínica Persistente (PCMS)

Implementa un buffer de experiencias clínicas con almacenamiento vectorial en **pgvector** (índice HNSW con `m=16`, `ef_construction=200`). Cada interacción se indexa automáticamente con metadatos de especialidad, gravedad y timestamp. La consulta semántica se ejecuta con **búsqueda híbrida** (vector + BM25) a través de `pg_bigm`. El tiempo de recuperación promedio es de **45 ms** para colecciones de hasta 100,000 registros.

**Configuración de conexión:**
```yaml
PG_HOST: localhost
PG_PORT: 5432
PG_VECTOR_DIM: 1536
PG_INDEX_TYPE: hnsw
```

### 3. Planificador de Tratamiento Probabilístico (PTP)

Genera planes de tratamiento multi-etapa utilizando un modelo de **programación dinámica estocástica** con horizonte de planificación configurable (`PLANNING_HORIZON_DAYS=90`). Evalúa hasta **1,024 combinaciones** de procedimientos por segundo, optimizando por costo, tiempo y probabilidad de éxito. Los resultados se serializan en formato **FHIR R4** y se almacenan en el PCMS para auditoría.

**Métrica de rendimiento:**
- Throughput: 1,024 planes/segundo en GPU NVIDIA T4
- Precisión de éxito estimado: ±2.3% (validación cruzada 5-fold)

## Cómo Usar

### Paso 1: Inicializar el Entorno de Agentes

Ejecuta el script de arranque que configura los contenedores Docker con perfiles de recursos específicos:

```bash
./dentaiagent init --agents 4 --memory-mb 512 --cpu-cores 2
```

Esto desplegará 4 agentes con 512 MB de RAM cada uno, conectados al bus de eventos Redis. Verifica el estado con:

```bash
dentaiagent status --format json | jq '.agents[].health'
```

### Paso 2: Cargar Corpus Clínico en el PCMS

Indexa un conjunto de registros clínicos en formato JSONL con campos obligatorios `text`, `specialty`, `severity`:

```bash
dentaiagent ingest --file ./clinical_corpus.jsonl --batch-size 128 --embedding-model text-embedding-3-small
```

El proceso mostrará el progreso en tiempo real y el tiempo promedio de indexación por lote. Una vez completado, verifica la cardinalidad del índice:

```bash
dentaiagent query --vector "dolor mandibular intenso" --top-k 5
```

### Paso 3: Ejecutar un Ciclo de Diagnóstico Completo

Inicia una sesión de diagnóstico con un paciente simulado:

```bash
dentaiagent diagnose --patient-id "PAT-2024-09-001" --symptoms "dolor agudo en molar inferior derecho, sensibilidad al frío" --context-depth 3
```

El sistema devolverá un JSON con:
- `diagnosis`: lista de hipótesis ponderadas
- `confidence_scores`: array de probabilidades
- `treatment_plan`: FHIR Bundle con procedimientos recomendados
- `execution_time_ms`: tiempo total del ciclo

## Mejores Prácticas

### 1. Ajuste de Hiperparámetros para Baja Latencia

Para entornos con restricciones de tiempo real (p. ej., asistentes de consulta), reduce `MAX_AGENT_TIMEOUT_MS` a 400 y activa la compresión de contexto (`CONTEXT_COMPRESSION=true`). Esto reduce la latencia media a **90 ms** a costa de una pérdida de precisión del 1.2% en la recuperación semántica. Monitorea con `dentaiagent metrics --live`.

### 2. Estrategia de Fragmentación del PCMS

Cuando el corpus supere los 500,000 registros, particiona el índice por especialidad usando `PG_PARTITION_KEY=specialty`. Esto mejora el tiempo de consulta en un **35%** al reducir el espacio de búsqueda. Programa una re-indexación nocturna con `dentaiagent reindex --schedule "0 3 * * *"` para mantener la coherencia.
