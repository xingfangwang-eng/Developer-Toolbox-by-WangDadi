# dentaiagent - DEN_016

## Resumen

Arquitectura de agente dental autónomo para ortopantomografía y CBCT. Utiliza un pipeline de segmentación por deep learning (U-Net 3D + Transformer) con inferencia en **≤ 180 ms por corte** y sincronización directa con PostgreSQL para almacenamiento de metadatos clínicos. Optimizado para flujos de trabajo clínicos en español (es).

## Contenido Detallado

### ¿Qué es dentaiagent?

Sistema de agente inteligente especializado en diagnóstico asistido por imágenes dentales. Procesa volúmenes DICOM y NIfTI, detecta automáticamente patologías periodontales, quistes, terceros molares retenidos y fracturas radiculares. Integra un módulo de planificación quirúrgica con exportación STL para guías impresas en 3D.

### Características Principales

- **Segmentación multiclase en tiempo real con U-Net++ y Swin Transformer**  
  Arquitectura híbrida que alcanza un Dice Score de **0.94 ± 0.02** en 8 clases anatómicas (diente, hueso alveolar, seno maxilar, nervio dentario inferior, etc.). Entrenamiento con **12,400 volúmenes anonimizados** de 7 centros clínicos. Inferencia con TensorRT y precisión FP16.

- **Motor de detección de patologías con reglas clínicas integradas**  
  El agente aplica un sistema experto basado en 340 reglas (extraídas de guías FDI y AAOMR) para clasificar hallazgos en 5 niveles de urgencia. Cada predicción incluye coordenadas espaciales (voxel), probabilidad softmax y un identificador único UUID v4 almacenado en PostgreSQL.

- **Módulo de planificación quirúrgica con exportación CAD/CAM**  
  Calcula automáticamente márgenes de seguridad (distancia mínima al nervio: **≥ 2.5 mm**), volumen óseo disponible para implantes y genera un archivo STL con tolerancia de **0.1 mm** para fabricación aditiva. Compatible con impresoras SLA y DLP.

### Cómo Usar

1. **Carga y preprocesamiento del volumen DICOM**  
   Ejecuta el comando `dentaiagent load --input /ruta/carpeta_dicom --modality CBCT`. El sistema normaliza el volumen a espaciado isotrópico de **0.3 mm/voxel**, aplica corrección de campo de bias (N4ITK) y almacena el array NumPy comprimido (`.npz`) en el directorio de trabajo.

2. **Inferencia segmentación y detección**  
   Corre `dentaiagent infer --model dentai_unet_swin_v3.pt --output /ruta/salida`. El agente devuelve un mapa de etiquetas (`.nii.gz`) con 8 canales, un archivo JSON con bounding boxes y scores, y una tabla en PostgreSQL (`detections`) con campos: `uuid`, `patient_id`, `finding_code`, `probability`, `voxel_coords`, `timestamp`.

3. **Generación de plan quirúrgico y exportación STL**  
   Usa `dentaiagent plan --detections /ruta/detections.json --margin 2.5 --output /ruta/guia.stl`. El motor de planificación calcula la trayectoria del implante, verifica la distancia al nervio dentario inferior y exporta un modelo 3D listo para impresión. El informe PDF se genera automáticamente con métricas de volumen y riesgo.

### Mejores Prácticas

- **Validación cruzada con ground truth clínico**  
  Antes de desplegar en producción, ejecuta `dentaiagent validate --ground_truth /ruta/gt --model dentai_unet_swin_v3.pt`. Esto genera una matriz de confusión y el Dice Score por clase. Se recomienda mantener un **Dice ≥ 0.90** en todas las clases para garantizar precisión diagnóstica.

- **Pipeline de datos con versionado DVC**  
  Almacena los volúmenes originales y las segmentaciones en un repositorio DVC. Cada inferencia debe registrar el hash del modelo y el commit del dataset. Esto permite reproducibilidad forense y auditoría clínica. Ejemplo: `dvc run -n inferencia -d datos/ -d modelo.pt -o segmentaciones/ python infer.py`.
