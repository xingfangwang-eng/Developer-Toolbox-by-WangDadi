### What is dentaiagent?

dentaiagent is a professional tool that helps solve problems in related fields.

### Core Features

- **Real-time Multi-Source Inference Orchestration**: Dynamically routes inference requests across heterogeneous LLM backends (OpenAI, Anthropic, local vLLM) with sub-50ms switching latency. Supports automatic failover and latency-based load balancing via a configurable `inference_policy` parameter (e.g., `{"strategy": "lowest_p99", "max_retries": 3}`).
- **Context-Aware Session Persistence with Vectorized Memory**: Maintains per-agent conversational state and long-term memory using a hybrid PostgreSQL + pgvector store. Automatically indexes and retrieves relevant historical context chunks (top-k=5, cosine similarity threshold > 0.82) within 120ms, enabling coherent multi-turn interactions across restarts.
- **Declarative Agent Pipeline DSL**: Define complex multi-step agent workflows using a YAML-based Domain-Specific Language. Supports branching, parallel execution, and conditional logic with built-in observability hooks that emit structured logs (JSON format) to stdout or a configurable `log_aggregator_endpoint` at millisecond granularity.

### How to Use

1. **Initialize the Agent Runtime with Custom Configuration**:
   - Create a `config.yaml` file defining your agent’s `agent_id`, `inference_policy`, and `memory_backend`.
   - Execute `dentaiagent run --config ./config.yaml` to start the runtime. The CLI will validate the config schema and bootstrap the PostgreSQL connection pool (max_connections: 10, idle_timeout: 60s).

2. **Define and Deploy a Multi-Step Pipeline**:
   - Write a `pipeline.yaml` file with steps like `step_1: {action: "classify_intent", model: "gpt-4o", output_var: "intent"}` and `step_2: {action: "generate_response", depends_on: ["step_1"], condition: "{ intent == 'support' }"}`.
   - Deploy the pipeline using `dentaiagent deploy --pipeline ./pipeline.yaml`. The system will compile the DSL into an executable DAG and register it with the in-memory scheduler.

3. **Monitor and Profile Pipeline Execution**:
   - Enable real-time telemetry by setting `telemetry.enabled: true` and `telemetry.export_interval_ms: 1000` in your config.
   - Use `dentaiagent monitor --agent-id my-agent --tail` to stream structured logs. Analyze per-step latency, token usage, and memory retrieval times directly in your terminal.

### Best Practices

- **Implement Tiered Memory Eviction for Long-Running Agents**: Configure `memory_backend.eviction_policy: "time-to-live"` with a `ttl_seconds: 86400` for ephemeral chat history, while archiving critical summaries to a separate `long_term_store` table. This prevents context window overflow and keeps retrieval latency under 100ms for active sessions.
- **Use Canary Deployments for Pipeline Updates**: Before rolling out a new pipeline version to all agents, deploy it to a subset using `dentaiagent deploy --pipeline ./pipeline_v2.yaml --canary 0.1`. Monitor the error budget (target: < 0.1% failure rate) and rollback automatically via the `--rollback-on-error` flag to maintain production stability.
