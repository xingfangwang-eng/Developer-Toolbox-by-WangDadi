## Detailed Content

### What is dentaiagent?

dentaiagent is a real-time multi-threaded inference engine designed for autonomous dental image segmentation and anomaly detection. It achieves sub-100ms latency per frame on GPU-accelerated pipelines and integrates directly with DICOM-compliant storage backends.

### Core Features

- **Adaptive Mask Refinement Engine**: Utilizes a custom U-Net++ variant with attention-gated skip connections. The engine dynamically adjusts the receptive field based on input resolution (512x512 to 2048x2048 pixels) and outputs pixel-level segmentation masks with an IoU threshold of ≥0.92. All inference is logged in milliseconds via a dedicated `inference_timer` parameter within the `dentaiagent.config` namespace.

- **Multi-View Radiograph Synchronizer**: Accepts up to 8 simultaneous panoramic, bitewing, and periapical views. The synchronizer performs affine registration using a pre-trained Spatial Transformer Network (STN), outputting a unified 3D coordinate map. The `sync_tolerance` parameter is set to ≤2.0 mm for clinical-grade accuracy, and all alignment matrices are stored in a PostgreSQL `geometry` column for audit trails.

- **Real-Time Anomaly Classification Pipeline**: Deploys a lightweight MobileNetV3 backbone quantized to INT8 for edge deployment. The pipeline classifies 12 distinct dental pathologies (e.g., caries, periapical radiolucency, impacted third molars) with a precision of ≥0.95 at a throughput of 60 FPS on a Jetson Orin NX. The `confidence_threshold` parameter is exposed via the REST API, defaulting to 0.85.

### How to Use

1. **Initialize the Inference Session**:  
   Run `dentaiagent init --backend cuda --batch_size 4 --precision fp16`. This command loads the pre-trained weights from the `dentaiagent/weights/v2.3.1` directory, sets the CUDA device to GPU:0, and pre-allocates a memory pool of 4 GB. The session ID is returned as a UUID string (e.g., `a1b2c3d4-...`), which you must pass to all subsequent API calls.

2. **Ingest a DICOM Study**:  
   Execute `dentaiagent ingest --study_uid 1.2.840.113619.2.55.3.604688.2023.123456789 --source /mnt/dicom_inbound --normalize_window_level true`. The tool parses the DICOM headers, extracts the SeriesInstanceUID and ImagePositionPatient, and applies a default window width of 2500 HU and window center of 500 HU. All images are resampled to a uniform voxel spacing of 0.2 mm x 0.2 mm x 1.0 mm.

3. **Run Segmentation and Export Results**:  
   Call `dentaiagent segment --session_id a1b2c3d4-... --output_format nifti --save_masks true --export_to_s3 s3://dentaiagent-output/studies/`. The engine processes each image in the series, writes the segmentation mask as a compressed NIfTI file (.nii.gz), and uploads it to the specified S3 bucket. A JSON manifest containing per-mask confidence scores and bounding boxes is appended to the same prefix.

### Best Practices

- **Practice 1: Implement a Staged Inference Queue**  
  For high-throughput clinical environments, configure the tool with a staged queue using Redis. Set `queue_capacity=200` and `max_retries=3`. This prevents GPU memory overflow during peak load and ensures that each study is processed with a guaranteed inference timeout of 30 seconds. Monitor the `queue_depth` metric in Prometheus to auto-scale worker pods.

- **Practice 2: Always Validate Against a Gold Standard Dataset**  
  Before deploying to production, run `dentaiagent validate --ground_truth /data/gold_standard/ --metric dice_score --threshold 0.90`. This command compares the tool’s output against a manually annotated set of 500 studies. If the mean Dice score falls below 0.90, the pipeline automatically rolls back to the previous model version (stored in `dentaiagent/weights/backup/`).
