# dentaiagent - DEN_023

## Overview

`dentaiagent` is an autonomous agent framework designed for zero-latency dental AI inference at the edge. It orchestrates multi-modal diagnostic pipelines (CBCT segmentation, intraoral video stream analysis, and caries risk prediction) directly on clinical hardware without cloud dependency. This repository targets the `dentaiagent` runtime engine, optimized for NVIDIA Jetson Orin and Apple M-series Neural Engine, delivering sub-100ms inference latency per frame.

## Detailed Content

### What is dentaiagent?

`dentaiagent` is a lightweight, Rust-based agent runtime that manages concurrent AI models for real-time dental diagnostics. It abstracts hardware-specific ML backends (TensorRT, CoreML, ONNX Runtime) into a unified API, enabling hot-swappable model pipelines. The agent handles memory pooling, frame queueing, and priority scheduling across up to 4 simultaneous inference streams, ensuring deterministic performance even under 4K video input at 30 FPS. It integrates directly with DICOM-compliant PACS systems via HL7 FHIR, and stores session metadata in a local PostgreSQL instance with millisecond-level write latency.

### Core Features

- **Multi-Modal Inference Orchestration with Priority Queuing**  
  Manages separate inference queues for CBCT slices (512×512×512 voxels), intraoral video frames (1920×1080 @30fps), and spectral data from DIAGNOdent pens. Each queue has configurable priority levels and time-slice limits (default: 50ms per frame). The runtime automatically scales thread pool size based on GPU VRAM pressure (monitored via `nvidia-smi` / `ane-stats`). Supports pre-emption: a high-priority caries detection request can pause a running segmentation job, save its state to shared memory, and resume seamlessly.

- **Zero-Copy Frame Pipeline with Shared Memory Transport**  
  All video frames and volume data are passed between models via `shm_open` regions mapped to GPU device memory, eliminating CPU-GPU transfer overhead. The pipeline supports chained transforms: raw frame → contrast enhancement (CLAHE with adaptive tile size) → caries classification → overlay generation, all within a single DMA transfer cycle. Measured end-to-end latency: 85ms for a 4K frame through 3 models on Jetson Orin AGX.

- **Automatic Model Rollback with A/B Testing Support**  
  Each model version is tagged with a semantic version and a confidence threshold. The agent maintains two parallel inference slots per model type (A/B). On startup, it runs a shadow inference on a held-out validation set (last 100 frames) and compares accuracy against the production slot. If the new model’s F1 score drops below 0.92, it is automatically rolled back and the event is logged to a PostgreSQL `model_events` table with a full stack trace.

### How to Use

1. **Initialize the Agent with Hardware Detection**  
   Launch the daemon: `dentaiagentd --config /etc/dentaiagent/config.toml --hardware-detect`. The agent scans for available accelerators (NVIDIA Jetson Orin, Apple ANE, or AMD ROCm), then writes the detected capabilities to `/var/run/dentaiagent/hardware.json`. If multiple GPUs are found, it assigns each pipeline to the device with the lowest current memory utilization. Example config snippet:
   ```toml
   [pipeline.caries]
   model_path = "/models/caries_v3.onnx"
   priority = 10
   max_latency_ms = 150
   fallback_device = "cpu"
   ```

2. **Inject a Live Intraoral Video Stream**  
   Use the gRPC endpoint `StreamService/IngestFrames` to push raw YUV420 frames. The agent expects a header with `width`, `height`, `fps`, and `source_id`. Example call:
   ```bash
   grpcurl -d '{"source_id": "sirona_01", "width": 1920, "height": 1080, "fps": 30}' \
     -import-path ./proto -proto stream.proto \
     localhost:50051 dentaiagent.StreamService/IngestFrames
   ```
   The agent returns a stream of `InferenceResult` messages containing bounding boxes, confidence scores (0.0–1.0), and a `frame_timestamp_us` (microsecond precision). If the queue backs up beyond 200 frames, the agent drops the oldest frames and emits a `QUEUE_OVERFLOW` warning.

3. **Query Session History with FHIR Integration**  
   Retrieve all diagnostic results for a patient session via REST:  
   `GET /api/v1/sessions/{session_id}/findings?format=ndjson`  
   The endpoint streams findings as newline-delimited JSON, each line containing `inference_id`, `model_name`, `confidence`, `bounding_box`, and `dicom_uid`. The agent also writes a summary to the `diagnostic_reports` table in PostgreSQL, which can be queried with:  
   ```sql
   SELECT * FROM diagnostic_reports 
   WHERE patient_id = 'PAT-1024' 
     AND created_at > NOW() - INTERVAL '7 days'
   ORDER BY confidence DESC;
   ```

### Best Practices

- **Pin Critical Models to Dedicated GPU Contexts**  
  For real-time workflows (e.g., caries detection during a live exam), create a dedicated CUDA stream per model and set `cudaStreamNonBlocking`. This prevents one model’s kernel launch from blocking another. On Jetson, use `nvidia-smi -ac 1300,1600` to lock clock speeds and avoid thermal throttling during long sessions. Always test with `dentaiagent-bench --pipeline caries --duration 300s` to verify latency remains under 100ms P99.

- **Implement Graceful Degradation with Shadow Models**  
  Deploy a lightweight fallback model (e.g., MobileNetV2 quantized to INT8) alongside your primary model. Configure the agent to switch to the fallback if the primary model’s inference time exceeds `max_latency_ms` for 5 consecutive frames. Store the fallback model in a separate memory region to avoid cache pollution. Monitor switch events via Prometheus metrics (`dentaiagent_fallback_activations_total`). This ensures the clinical workflow never stalls, even during GPU memory fragmentation.
