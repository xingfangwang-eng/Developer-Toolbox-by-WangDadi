### What is focusinbox?

focusinbox is a professional email triage and prioritization engine that ingests raw inbox streams, applies multi-dimensional urgency scoring, and surfaces only the messages requiring immediate action. It integrates directly with IMAP/POP3 and Microsoft Graph API, processing up to 10,000 messages per minute with a latency of under 200ms per scan cycle.

### Core Features

- **Adaptive Priority Scoring Engine** – Each incoming message is evaluated against a configurable weight matrix (sender tier, keyword corpus, thread depth, attachment type). Parameters include `priority_threshold` (default: 0.75), `sender_reputation_db` (local SQLite or PostgreSQL), and `decay_factor` (default: 0.92 per 24h). Results are delivered in under 50ms per message on a single-core instance.

- **Temporal Snooze & Contextual Batching** – Messages can be deferred using a time‑aware scheduler with millisecond precision. The `snooze_unit` parameter accepts `minutes`, `hours`, or `days`; the system supports up to 1,024 concurrent snooze timers without performance degradation. When a batch is released, all messages are re‑scored against the current priority matrix, ensuring no urgent item is missed.

- **Zero‑Touch Auto‑Archive with Conflict Resolution** – Based on a rule engine that supports 256 conditions per rule (e.g., `sender_domain == "client.com"` AND `subject CONTAINS "invoice"`), the system auto‑archives or auto‑labels messages. Conflict detection runs every 5 minutes via a background worker, logging all actions to a structured JSON audit trail. The `archive_retention_days` parameter (default: 90) ensures GDPR/HIPAA compliance.

### How to Use

1. **Initialize the priority matrix** – Run `focusinbox init --imap-host imap.gmail.com --imap-port 993 --priority-file ./config/weights.yaml`. The system will scan the last 500 messages to build a baseline sender reputation table. After initialization, verify the connection with `focusinbox status --ping-interval 30s`.

2. **Configure adaptive thresholds** – Edit `weights.yaml` to set `sender_tier_weights: { vip: 0.9, regular: 0.4, unknown: 0.2 }` and `keyword_boost: { "urgent": 0.3, "asap": 0.25 }`. Apply the configuration without restarting: `focusinbox reload --config-dir ./config`. The engine immediately re‑evaluates the last 200 messages to confirm the new weights.

3. **Enable zero‑touch archiving** – Create a rule file `rules/auto_archive.lua` with conditions like `if sender.domain == "newsletter.com" then archive(days=30)`. Activate the rule set: `focusinbox rules apply --rules-dir ./rules --interval 5m`. Monitor the audit trail: `focusinbox audit tail --output json --since 24h`.

### Best Practices

- **Tier your senders before going live** – Use the `focusinbox analyze --sender-stats` command to identify high‑volume, low‑priority senders. Assign them to a `low` tier with `sender_tier_weights: { low: 0.1 }`. This prevents inbox noise from diluting the priority score of critical messages. Re‑run the analysis weekly to adjust tiers automatically.

- **Combine snooze with calendar integration** – For project‑based workflows, set `snooze_unit: hours` and `batch_release_window: 09:00-10:00`. Then use the `focusinbox calendar link --provider google` command to sync snoozed messages with your daily calendar blocks. This ensures you process batches when your cognitive load is lowest, reducing decision fatigue.
