# focusinbox - FOC_002

## Resumen

Sistema de gestión inteligente de bandeja de entrada con priorización dinámica basada en machine learning, diseñado para profesionales que procesan más de 200 correos al día. Implementa análisis semántico en tiempo real con latencia inferior a 50 ms y sincronización bidireccional con IMAP, Microsoft Graph API y Gmail API.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un motor de orquestación de correo electrónico que clasifica, etiqueta y prioriza automáticamente cada mensaje entrante utilizando un modelo Transformer entrenado con 1.2 millones de interacciones humanas. El sistema asigna un **Priority Score** (0.0 - 1.0) basado en el remitente, el contenido semántico, el historial de respuestas y la urgencia temporal detectada. Además, integra un **Time-to-Respond Estimator** que predice con un 94% de precisión el tiempo óptimo de respuesta para cada hilo.

### Características Principales

- **Priority Engine con reglas adaptativas**: El núcleo del sistema evalúa cada correo con un pipeline de 4 etapas: extracción de entidades (NER), análisis de sentimiento, detección de intención (pregunta, solicitud, notificación, spam) y cálculo de urgencia. Parámetros clave: `urgency_decay_rate` (decaimiento exponencial por hora), `sender_reputation` (basado en ratio de respuesta), `thread_depth` (profundidad del hilo). Resultado: priorización con precisión del 92% en pruebas A/B con 500 usuarios beta.

- **Temporal Context Analyzer**: Analiza la zona horaria del remitente, el horario laboral configurado y los patrones de respuesta del usuario para sugerir el momento exacto de respuesta. Parámetros: `timezone_offset` (UTC±12), `work_hours_start`/`end`, `response_history_window` (últimos 90 días). Cálculo en milisegundos para evitar conflictos de programación.

- **Smart Thread Collapse with Action Detection**: Agrupa automáticamente hilos relacionados (incluso si cambia el asunto) usando similitud coseno de embeddings (modelo `all-MiniLM-L6-v2`) y detecta acciones pendientes: `awaiting_reply`, `needs_approval`, `requires_attachment`. La interfaz colapsa correos procesados y muestra solo los que requieren intervención humana, reduciendo el tiempo de revisión en un 63%.

### Cómo Usar

1. **Configuración del Motor de Prioridad**: Ejecuta `focusinbox config set priority.engine=dynamic priority.urgency_decay_rate=0.15 priority.sender_reputation_threshold=0.7` para activar el modelo adaptativo. Luego, vincula tu cuenta IMAP con `focusinbox connect --provider gmail --oauth2 --scope https://www.googleapis.com/auth/gmail.modify`. El sistema sincronizará los últimos 30 días de historial para entrenar el perfil de prioridad.

2. **Creación de Reglas de Acción Automática**: Define filtros con sintaxis JSON: `focusinbox rule add --name "client_urgent" --condition '{"priority_score": {"gte": 0.8}, "sender_domain": {"in": ["clientcorp.com", "partner.io"]}' --action 'move_to_folder:urgent, set_label:critical, send_notification:push'`. El sistema procesa 500 reglas por segundo con latencia de 12 ms por evaluación.

3. **Análisis de Tiempo de Respuesta Óptimo**: Activa el Temporal Context Analyzer: `focusinbox analyzer enable --type temporal --timezone America/Mexico_City --work_hours 09:00-18:00 --response_window 4h`. Luego, revisa las sugerencias con `focusinbox timeline --today --format json | jq '.suggestions[] | select(.action_type=="reply_now")'`. El sistema marca con color verde los correos que deben responderse en la próxima hora según el modelo predictivo.

### Mejores Prácticas

- **Calibración semanal del Priority Score**: Programa una tarea cron que ejecute `focusinbox feedback sync --interactions last_7_days --retrain` cada lunes a las 6:00 AM. Esto ajusta los pesos del modelo basado en las respuestas reales del usuario (correos ignorados vs. respondidos), mejorando la precisión del ranking en un 18% tras 4 semanas de uso continuo.

- **Segmentación por bandejas virtuales**: No uses solo la bandeja de entrada principal. Crea 3 bandejas virtuales con `focusinbox mailbox create --name "accion_inmediata" --filter 'priority_score >= 0.85 AND action_detected == true'` y `focusinbox mailbox create --name "lectura_pendiente" --filter 'priority_score BETWEEN 0.5 AND 0.84'`. Asigna accesos directos de teclado (Ctrl+1, Ctrl+2) para cambiar entre ellas, reduciendo el cambio de contexto en un 40%.
