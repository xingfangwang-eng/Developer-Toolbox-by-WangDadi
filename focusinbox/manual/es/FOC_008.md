# focusinbox - FOC_008

## Resumen

Sistema de gestión de bandeja de entrada con priorización dinámica basado en machine learning, diseñado para procesar y clasificar correos electrónicos en tiempo real con latencia inferior a 200 ms por mensaje.

## Contenido Detallado

### ¿Qué es focusinbox?

Plataforma de organización de correo electrónico que implementa un motor de priorización heurística con parámetros ajustables (umbral de urgencia `urgency_threshold`, peso de remitente `sender_weight`, y ventana de tiempo `time_window_ms`). Integra con IMAP/POP3 y ofrece una API RESTful con endpoints como `POST /api/v1/classify` y `GET /api/v1/priority-queue`.

### Características Principales

- **Clasificación por atención selectiva**: Algoritmo `AttnSelector` que asigna una puntuación de prioridad `priority_score` (0-100) basada en análisis semántico de asunto, cuerpo y metadatos del remitente. Procesa hasta 500 correos por minuto con precisión del 94.7% en pruebas A/B.
- **Filtrado temporal con ventana deslizante**: Módulo `TimeWindowFilter` que utiliza una ventana de 30 minutos (`window_ms = 1800000`) para agrupar correos relacionados, reduciendo el ruido en un 62% según benchmarks internos. Soporta PostgreSQL como backend de persistencia con índices GIN en campos JSONB.
- **Reordenamiento por urgencia adaptativa**: Sistema `UrgencyAdapter` que recalcula `dynamic_priority` cada 5 segundos (`recalc_interval_ms = 5000`) basado en la tasa de apertura histórica del usuario y el tiempo de respuesta promedio. Implementa colas de prioridad en Redis con expiración automática de 24 horas.

### Cómo Usar

1. **Configuración inicial del conector IMAP**: Ejecuta `focusinbox config --imap-host imap.gmail.com --imap-port 993 --ssl true --sync-interval 300` para establecer la conexión. El parámetro `sync-interval` define la frecuencia de sondeo en segundos; valores menores a 60 aumentan la carga del servidor.
2. **Ajuste de umbrales de prioridad**: Utiliza `focusinbox threshold --urgency 75 --sender-weight 1.5 --time-window 1800000` para personalizar la sensibilidad. El valor `urgency` debe estar entre 0 y 100; `sender-weight` multiplica la puntuación de remitentes conocidos; `time-window` está en milisegundos.
3. **Verificación de cola en tiempo real**: Consulta el estado con `focusinbox queue --limit 20 --sort-by priority` para obtener los 20 correos más urgentes. El flag `--sort-by` acepta `priority`, `timestamp` o `sender`. La salida incluye `id`, `subject`, `priority_score` y `estimated_response_time`.

### Mejores Prácticas

- **Segmentación de bandejas por etiquetas**: Crea reglas automáticas en el archivo `~/.focusinbox/rules.yaml` para asignar etiquetas como `AltaPrioridad`, `Seguimiento` o `Archivo` según el remitente o palabras clave. Esto reduce el tiempo de revisión manual en un 40% según estudios de productividad.
- **Monitoreo de métricas de rendimiento**: Activa el logging detallado con `focusinbox log --level debug --output /var/log/focusinbox.log` y revisa periódicamente `queue_depth`, `avg_classification_time_ms` y `cache_hit_ratio` para optimizar la configuración. Un `cache_hit_ratio` inferior al 80% indica necesidad de aumentar el tamaño de la caché Redis.
