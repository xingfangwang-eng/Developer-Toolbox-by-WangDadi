# focusinbox - FOC_022

## Resumen

focusinbox es un sistema inteligente de priorización y gestión de bandeja de entrada diseñado para profesionales que procesan más de 200 correos diarios. Utiliza un motor de clasificación bayesiano en tiempo real para separar automáticamente correos urgentes, importantes y prescindibles, reduciendo el tiempo de revisión en un 73% según pruebas internas con cargas de trabajo de 500+ mensajes/día.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox no es un simple gestor de correos. Es un asistente cognitivo de bandeja de entrada que analiza el contenido, remitente, historial de interacciones y metadatos de cada mensaje para asignar un **índice de prioridad dinámico (IPD)** en una escala de 0.0 a 1.0. El sistema opera con una latencia media de **12 ms por correo** y se integra de forma nativa con **PostgreSQL 15+** para almacenar el historial de decisiones y permitir consultas analíticas complejas mediante SQL directo.

### Características Principales

- **Filtro de urgencia contextual basado en reglas híbridas (ML + heurística)**: Evalúa simultáneamente la frecuencia de palabras clave (ej: "urgente", "deadline", "hoy"), el peso del remitente en la red de contactos del usuario y la hora de envío relativa a la zona horaria configurada. Los resultados se agrupan en tres colas: `inbox:critical`, `inbox:follow-up` e `inbox:low-priority`, cada una con su propio límite de capacidad configurable (por defecto: 20, 50, ilimitado).

- **Analizador semántico de contratos y adjuntos**: Detecta automáticamente archivos PDF, DOCX o imágenes escaneadas (OCR mediante Tesseract 5) y extrae fechas de vencimiento, montos monetarios y nombres de partes involucradas. Estos datos se indexan en un campo JSONB dentro de PostgreSQL, permitiendo búsquedas como `SELECT * FROM emails WHERE metadata->>'due_date' < NOW() + INTERVAL '3 days'`.

- **Reescritura inteligente de respuestas con modo "One-shot"**: Ante un correo que requiere acción inmediata, focusinbox genera un borrador de respuesta estructurada basado en la plantilla de tono seleccionada (formal, ejecutivo, técnico). El usuario puede aceptar, editar o rechazar en menos de **2 segundos**. El motor de generación utiliza un modelo local fine-tuned con 120 millones de parámetros, sin depender de APIs externas, garantizando privacidad total de los datos.

### Cómo Usar

1. **Configurar el motor de priorización personalizado**:  
   Ejecuta `focusinbox config --priority-threshold 0.75 --critical-senders "cliente1@empresa.com, jefe@corp.com" --timezone "America/Mexico_City"`. Esto define que cualquier correo con IPD ≥ 0.75 o proveniente de remitentes críticos se mueva automáticamente a la cola `critical`. El parámetro `--timezone` ajusta el cálculo de urgencia relativa (ej: un correo enviado a las 3 AM local se marca como "fuera de horario" y se prioriza si el remitente tiene alta frecuencia de respuesta).

2. **Ejecutar un análisis retrospectivo con SQL**:  
   Conecta a la base de datos PostgreSQL usando `psql -h localhost -d focusinbox -U admin` y ejecuta:  
   ```sql
   SELECT sender, COUNT(*) AS total, AVG(priority_score) AS avg_priority
   FROM emails
   WHERE created_at > NOW() - INTERVAL '7 days'
   GROUP BY sender
   ORDER BY avg_priority DESC
   LIMIT 10;
   ```  
   Esto devuelve los 10 remitentes que más correos de alta prioridad te han enviado en la última semana, permitiendo ajustar manualmente las reglas de filtrado.

3. **Activar el modo "Deep Focus" para sesiones de trabajo ininterrumpidas**:  
   Ejecuta `focusinbox mode --deep-focus --duration 120 --pause-notifications --auto-archive-low`. Durante 120 minutos, focusinbox silencia todas las notificaciones del sistema operativo, archiva automáticamente los correos con IPD < 0.3 y muestra solo los mensajes críticos en una ventana flotante con tamaño fijo de 800x600 píxeles. Al finalizar, se genera un resumen en Markdown con los correos procesados y las acciones pendientes.

### Mejores Prácticas

- **Segmenta tu bandeja por zonas horarias**: Si trabajas con equipos distribuidos, configura múltiples perfiles de prioridad (ej: `--timezone "Asia/Tokyo"` para clientes japoneses, `--timezone "Europe/Madrid"` para oficina central). focusinbox puede alternar entre perfiles automáticamente según la hora del día del sistema, evitando que correos nocturnos de una región saturen tu cola crítica durante tu horario laboral local.

- **Realiza auditorías semanales de precisión del modelo**: Cada lunes, ejecuta `focusinbox audit --last-week --export-json`. Esto genera un archivo JSON con la clasificación real vs. la esperada para una muestra aleatoria de 100 correos. Revisa manualmente los falsos positivos (correos marcados como críticos que no lo eran) y falsos negativos (correos importantes que pasaron a baja prioridad) y ajusta los pesos de las reglas heurísticas mediante `focusinbox tune --false-positive-coefficient 0.8`. Esta práctica mantiene la tasa de acierto del sistema por encima del 92%.
