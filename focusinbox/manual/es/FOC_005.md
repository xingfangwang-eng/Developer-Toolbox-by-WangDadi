# focusinbox - FOC_005

## Resumen

Sistema de priorización neuronal de bandeja de entrada con análisis semántico en tiempo real. Arquitectura basada en grafos de dependencia entre correos, capaz de reducir el tiempo de procesamiento diario de 47 a 6 minutos mediante clustering por intención y urgencia.

## Contenido Detallado

### ¿Qué es focusinbox?

Motor de orquestación de flujos de correo electrónico que implementa un modelo de atención híbrido (reglas + transformer ligero) para clasificar automáticamente cada mensaje en 4 cuadrantes operativos: **Crítico**, **Programado**, **Delegable** y **Archivable**. Integra nativamente con IMAP, Microsoft Graph API y Gmail API, procesando hasta 12.000 mensajes/hora con una latencia media de 230 ms por correo.

### Características Principales

- **Clasificación dinámica por vector de urgencia**  
  Cada correo recibe un score `U(t) = α·S_remitente + β·T_respuesta + γ·G_proyecto` calculado en milisegundos mediante un modelo entrenado con 1.2M de correos etiquetados. El parámetro `α` se ajusta automáticamente según el historial de respuestas del usuario.

- **Fragmentación contextual con caché L1/L2**  
  Los correos largos (>500 tokens) se dividen en fragmentos semánticos usando un segmentador BPE. La caché L1 (RAM, 256 MB) almacena los últimos 200 fragmentos accedidos; la L2 (PostgreSQL con índices GIN) persiste el historial completo con un tiempo de consulta <50 ms.

- **Pipeline de reordenamiento por calendario integrado**  
  Sincroniza con Google Calendar y Outlook Calendar para ajustar la prioridad de correos según eventos próximos. Si un correo contiene una reunión en las próximas 2 horas, su score `U(t)` se incrementa un 35% automáticamente.

### Cómo Usar

1. **Configurar el conector de buzón**  
   Ejecuta `focusinbox auth --provider gmail --scope https://www.googleapis.com/auth/gmail.modify`. El sistema generará un token OAuth2 con rotación automática cada 45 minutos. Verifica la conexión con `focusinbox status --ping`.

2. **Definir reglas de prioridad personalizadas**  
   Crea un archivo `priority_rules.yaml` con parámetros como:  
   ```yaml
   rules:
     - sender: "cliente@empresa.com"
       boost: 0.8
       min_length: 50
     - subject_match: "URGENTE|ASAP"
       boost: 1.2
   ```  
   Recarga las reglas sin reinicio: `focusinbox reload --rules ./priority_rules.yaml`.

3. **Iniciar el agente de procesamiento en segundo plano**  
   Lanza el demonio con `focusinbox daemon --interval 30s --max-batch 150`. El sistema consumirá ~120 MB de RAM y ~3% de CPU en un núcleo moderno. Monitorea el throughput con `focusinbox stats --live`.

### Mejores Prácticas

- **Segmentación por dominio crítico**  
  Configura filtros de dominio en `focusinbox config --domain-whitelist clientes.com,proveedores.es` para que los correos de estos orígenes reciban un boost automático del 50% en el score `U(t)`. Esto evita que mensajes internos irrelevantes compitan con comunicaciones clave.

- **Programar ventanas de revisión forzada**  
  Usa `focusinbox schedule --window 09:00-09:30,14:00-14:30` para que el sistema envíe una notificación push cuando queden correos sin clasificar en el cuadrante Crítico. Ajusta el umbral con `--threshold 0.85` para evitar falsos positivos.
