# focusinbox - FOC_028

## Resumen

Sistema de priorización y organización de correos electrónicos basado en inteligencia contextual, diseñado para profesionales que gestionan más de 200 mensajes diarios. Implementa un motor de clasificación semántica con latencia inferior a 50 ms por mensaje, y se integra de forma nativa con IMAP, Microsoft Graph API y Gmail API.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es una plataforma de gestión de bandeja de entrada que aplica modelos de lenguaje entrenados específicamente para clasificar correos según urgencia real, remitente, contenido semántico y patrones históricos de respuesta. A diferencia de filtros estáticos, focusinbox aprende de cada interacción y ajusta dinámicamente los umbrales de prioridad, reduciendo el tiempo de revisión hasta un 73 % en entornos empresariales.

### Características Principales

- **Clasificación por relevancia temporal con ventanas de atención dinámicas**  
  El motor analiza el cuerpo del correo, el asunto, los metadatos de la conversación y el historial del remitente para asignar una puntuación de `priority_score` (0–100). Cuando un hilo supera el umbral `attention_threshold` (ajustable por usuario), se dispara una notificación push con latencia media de 12 ms. Soporta hasta 5000 reglas por usuario sin degradación de rendimiento.

- **Segmentación automática por proyectos y cargas de trabajo**  
  focusinbox extrae automáticamente entidades como `project_id`, `task_id` y `client_name` desde el contenido del correo, y los agrupa en buckets virtuales. Cada bucket expone métricas en tiempo real: `unread_count`, `avg_response_time` y `overdue_ratio`. Los datos se sincronizan con PostgreSQL cada 15 segundos, garantizando consistencia transaccional incluso en picos de 10 000 correos/hora.

- **Recomendación inteligente de acciones con análisis de sentimiento**  
  Tras clasificar el correo, el sistema sugiere acciones concretas: `reply_draft`, `delegate_to`, `snooze_until` o `archive`. La recomendación se basa en un modelo de regresión logística entrenado con más de 1.2 millones de decisiones de usuarios reales, alcanzando una precisión del 94.7 % en la predicción de la siguiente acción. Cada sugerencia incluye un `confidence_score` que se muestra al usuario para decisión informada.

### Cómo Usar

1. **Configuración inicial del motor de prioridades**  
   - Conecta tu cuenta de correo mediante OAuth 2.0.  
   - Ejecuta `focusinbox init --profile=executive --attention_threshold=75` para establecer un umbral alto de prioridad.  
   - El sistema analizará los últimos 90 días de correos y generará un `baseline_model` personalizado.  
   - Verifica el rendimiento con `focusinbox stats --latency`; el objetivo es una mediana inferior a 45 ms por correo.

2. **Creación de buckets de proyecto automáticos**  
   - Define reglas de extracción semántica con `focusinbox rule add --entity=project_id --pattern="(PROJ-\d+)"`.  
   - Asigna colores y etiquetas a cada bucket: `focusinbox bucket tag --name="Proyecto Alpha" --color=#00A3FF`.  
   - Revisa la distribución en tiempo real con `focusinbox dashboard --buckets`.  
   - Ajusta la frecuencia de sincronización con `--sync_interval=30s` para entornos de alta demanda.

3. **Activación de recomendaciones inteligentes**  
   - Habilita el modo predictivo: `focusinbox predict --enable --model_version=v2.1`.  
   - Para cada correo entrante, el sistema mostrará una tarjeta con tres acciones sugeridas y su `confidence_score`.  
   - Si la confianza supera el 90 %, la acción se ejecuta automáticamente tras 5 segundos de inactividad del usuario.  
   - Monitorea la precisión con `focusinbox predict --metrics` y reentrena el modelo cada 7 días con `focusinbox train --auto`.

### Mejores Prácticas

- **Segmentación por dominios críticos**  
  En entornos donde el tiempo de respuesta define contratos (soporte técnico, atención médica, trading), configura `attention_threshold` por dominio de correo electrónico. Por ejemplo, para `@hospital.com` usa `threshold=85`, mientras que para boletines internos usa `threshold=30`. Esto evita falsos positivos y garantiza que los correos realmente urgentes nunca queden ocultos.

- **Retroalimentación explícita para mejora continua**  
  Implementa un ciclo de retroalimentación semanal: cada viernes, exporta el log de decisiones del usuario (`focusinbox export --decisions --format=csv`) y compáralo con las predicciones del modelo. Identifica los casos donde el `priority_score` difirió de la acción real del usuario y etiqueta esos correos como `false_positive` o `false_negative`. Luego, ejecuta `focusinbox train --feedback=last_7_days.csv` para ajustar el modelo. Este proceso reduce el error de clasificación en un 18 % tras tres iteraciones.
