# focusinbox - FOC_011

## Resumen

Sistema de gestión de bandeja de entrada basado en priorización semántica y análisis de carga cognitiva. Diseñado para profesionales que procesan más de 200 correos diarios, focusinbox reduce el tiempo de decisión sobre qué leer primero en un 73% mediante algoritmos de clustering contextual y reglas de peso dinámico.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un motor de organización de correo electrónico que aplica procesamiento de lenguaje natural (NLP) con modelos transformer ligeros (distilBERT fine-tuned) para categorizar, etiquetar y priorizar mensajes en tiempo real. Se integra como capa intermedia entre el servidor IMAP y el cliente de correo, permitiendo filtrar ruido informativo sin depender de reglas manuales obsoletas.

### Características Principales

- **Priorización por urgencia contextual**: Analiza el cuerpo del mensaje, el historial del remitente y la frecuencia de interacción para asignar un score de prioridad (0.0 - 1.0) cada 500 ms. Utiliza una base de reglas entrenada con 50,000 correos etiquetados manualmente. Parámetros clave: `urgency_weight` (defecto 0.4), `sender_reputation` (actualizado cada 24h desde PostgreSQL), `thread_depth` (impacto en score si > 3 respuestas).

- **Clustering semántico de hilos**: Agrupa correos no solo por asunto, sino por embeddings de contenido generados con Sentence-BERT. Detecta temas latentes como "revisión de contrato" o "incidencia de producción" incluso si el asunto cambia. Tiempo de clusterización: < 200 ms para 1000 correos. Almacena los vectores en una base vectorial (pgvector) con índice IVFFlat para búsquedas de similitud en < 50 ms.

- **Foco automático por ventana de tiempo**: Configura ventanas de atención (ej. "solo correos de alta prioridad entre 14:00 y 16:00"). Durante estas ventanas, el sistema oculta automáticamente correos con score < 0.3 y muestra solo los que requieren acción inmediata. Sincroniza con calendarios externos vía CalDAV para ajustar ventanas según reuniones programadas.

### Cómo Usar

1. **Configurar conexión IMAP y pesos iniciales**: Ejecuta `focusinbox setup --imap-server imap.gmail.com --port 993 --ssl` y luego ajusta los pesos de prioridad editando el archivo `~/.focusinbox/config.yaml`. Parámetros críticos: `urgency_weight: 0.4`, `sender_reputation_weight: 0.3`, `thread_depth_weight: 0.2`, `attachment_weight: 0.1`. Verifica la conexión con `focusinbox test --dry-run`.

2. **Ejecutar el análisis de clustering inicial**: Lanza `focusinbox scan --mode full --batch-size 500`. Este comando procesa todos los correos de la bandeja de entrada, genera embeddings y los almacena en la tabla `focusinbox.emails` con columnas `vector (768d)`, `cluster_id`, `priority_score`. Monitorea el progreso con `focusinbox status --verbose`. El proceso completo para 10,000 correos toma ~3 minutos en una máquina con 8 GB RAM.

3. **Activar modo foco y revisar resultados**: Usa `focusinbox focus --window "14:00-16:00" --min-score 0.3 --output json` para obtener una lista de correos priorizados. Cada entrada incluye `email_id`, `subject`, `sender`, `priority_score`, `cluster_label`, `action_required` (booleano). Para aplicar cambios directamente en el cliente de correo (ej. mover a carpetas), ejecuta `focusinbox apply --action move --target "INBOX/Focus"`.

### Mejores Prácticas

- **Calibración semanal de pesos**: Los pesos de prioridad deben recalibrarse cada 7 días usando `focusinbox calibrate --feedback-dir ./feedback`. Alimenta el sistema con correos que marcaste como "urgente" o "pospuesto" para reentrenar el modelo de regresión logística. Esto reduce falsos positivos de prioridad en un 40% tras 3 iteraciones.

- **Segmentación por roles**: Si trabajas en equipo, define perfiles de prioridad distintos (ej. `profile: developer`, `profile: manager`). Cada perfil asigna pesos diferentes a remitentes (ej. CTO tiene peso 0.9 para managers, 0.6 para developers). Usa `focusinbox profile --create developer --sender-weight cto@company.com 0.9` para aislar la configuración y evitar conflictos.
