# focusinbox - FOC_018

## Resumen

Sistema de priorización inteligente de bandeja de entrada basado en reglas dinámicas y aprendizaje automático en tiempo real. Diseñado para profesionales que procesan más de 200 correos diarios, focusinbox reduce el tiempo de revisión un 73% mediante clustering contextual y scoring bayesiano.

## Contenido Detallado

### ¿Qué es focusinbox?

Arquitectura de orquestación de correo electrónico que implementa un motor de clasificación en tres capas: análisis semántico (modelo BERT-lightweight), extracción de metadatos temporales y ponderación por urgencia colaborativa. No es un simple filtro: es un sistema de decisión que asigna prioridades con latencia inferior a 50 ms por mensaje, integrable con cualquier servidor IMAP.

### Características Principales

- **Clasificación por ventanas de atención**: Cada correo recibe un `urgency_score` (0.0–1.0) calculado con 12 parámetros, incluyendo `reply_chain_depth`, `sender_authority_index` y `deadline_proximity (en horas)`. La capa de persistencia usa PostgreSQL con índices GIN para consultas de prioridad en menos de 10 ms.
- **Análisis de intención con contexto histórico**: El módulo `intent_analyzer` procesa el cuerpo del correo y el hilo completo, extrayendo 8 categorías (acción, decisión, seguimiento, etc.). Almacena vectores semánticos en memoria caché Redis con TTL configurable (por defecto 24 h), permitiendo reordenamiento dinámico sin reindexar.
- **Planificador de respuestas automáticas con cola de prioridad**: El `scheduler_engine` genera respuestas draft para correos con `score > 0.85`, usando plantillas parametrizadas y variables del contexto (nombre del remitente, fecha límite). La cola se gestiona con RabbitMQ, garantizando entrega en menos de 200 ms incluso con 10 000 mensajes concurrentes.

### Cómo Usar

1. **Configurar el conector IMAP con autenticación OAuth 2.0**: Ejecuta `focusinbox connect --provider gmail --client-id <ID> --client-secret <SECRET> --scopes https://mail.google.com/`. El sistema validará el token y sincronizará las últimas 500 entradas en menos de 3 segundos. Usa `focusinbox status` para verificar el estado de conexión (parámetro `connection_latency` en ms).
2. **Definir reglas de priorización personalizadas**: Crea un archivo YAML `rules.yaml` con bloques como `- if: sender_domain == "clientes.com" then: urgency_multiplier = 1.5`. Luego carga las reglas con `focusinbox rules apply --file rules.yaml --priority high`. El motor evalúa 200 reglas por segundo, mostrando logs en `~/.focusinbox/logs/rules_engine.log`.
3. **Iniciar el planificador de respuestas automáticas**: Activa el módulo con `focusinbox scheduler start --interval 5m --max-drafts 50 --language es`. El sistema generará borradores en la carpeta `[Gmail]/FocusInbox Drafts` y notificará vía webhook (parámetro `webhook_url`). Para pausar: `focusinbox scheduler stop --flush-queue`.

### Mejores Prácticas

- **Segmentación por dominios críticos**: Configura `sender_authority_index` manualmente para dominios de alta prioridad (ej. `juzgado.es`, `notario.com`). Esto evita falsos negativos en correos legales. Verifica con `focusinbox audit --last 1000` que el acierto supere el 95%.
- **Rotación semanal de reglas**: Actualiza las reglas cada lunes usando `focusinbox rules backup` y `focusinbox rules apply --from-template weekly`. Mantén un historial de versiones en Git para revertir cambios. El comando `focusinbox stats --period 7d` muestra el impacto en tiempo promedio de respuesta (target: < 2 min).
