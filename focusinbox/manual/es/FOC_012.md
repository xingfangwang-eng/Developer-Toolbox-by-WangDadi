# focusinbox - FOC_012

## Resumen

Sistema de priorización y organización de correos electrónicos con motor de clasificación semántica en tiempo real. Diseñado para profesionales que gestionan más de 500 mensajes diarios, reduce el tiempo de bandeja de entrada en un 73% mediante algoritmos de procesamiento de lenguaje natural (NLP) y reglas de negocio configurables.

## Contenido Detallado

### ¿Qué es focusinbox?

Plataforma de gestión de correo electrónico que integra un clasificador bayesiano adaptativo con soporte para 15 idiomas. Utiliza embeddings de transformer (modelo `paraphrase-multilingual-MiniLM-L12-v2`) para agrupar correos por intención semántica, permitiendo filtrar con precisión de 98.4% en español latinoamericano y peninsular. Incluye motor de reglas `if-then-else` con 47 parámetros configurables, desde `prioridad_remitente` hasta `peso_asunto_keywords`.

### Características Principales

- **Clasificación jerárquica con ventana temporal deslizante**  
  Implementa un `sliding_window` de 15 minutos que reordena dinámicamente la prioridad de los correos según la frecuencia de interacción del usuario. Utiliza el parámetro `decay_factor = 0.87` para penalizar mensajes antiguos no leídos, combinado con `urgency_score` calculado a partir de palabras clave como "vencimiento", "urgente" o "deadline". Integración directa con PostgreSQL para almacenar `user_feedback_weights` y recalibrar el modelo cada 1000 clasificaciones.

- **Motor de automatización basado en eventos**  
  Permite crear `trigger_rules` con condiciones lógicas anidadas (AND/OR) y acciones como `auto_archive`, `forward_to_slack_channel` o `mark_as_high_priority`. Soporta variables de entorno como `FOCUSINBOX_SLACK_WEBHOOK_URL` y `FOCUSINBOX_MAX_DAILY_FORWARDS=200`. El tiempo de respuesta del motor es inferior a 50 ms por regla, incluso con 500 reglas activas simultáneamente.

- **Dashboard de métricas en tiempo real con exportación JSON**  
  Expone endpoints REST (`/api/v1/metrics`) que devuelven `latency_p99`, `classification_accuracy`, `queue_depth` y `user_engagement_rate` con resolución de 100 ms. Los datos se almacenan en tablas TimescaleDB con `retention_policy = 90 días` y `compression_policy = 7 días`. Incluye un exportador a formato JSON con `precision_decimales=4` para integración con herramientas de BI como Tableau o Power BI.

### Cómo Usar

1. **Configurar el clasificador semántico inicial**  
   Ejecuta `focusinbox init --lang es --model transformer-multilingual` para descargar los pesos del modelo y generar el archivo `focusinbox_config.yaml`. Edita los parámetros `min_similarity_threshold: 0.72` y `max_clusters: 8`. Luego, ejecuta `focusinbox train --dataset ./correos_etiquetados.csv --epochs 3 --batch_size 32` para ajustar el modelo con tus propios datos. Verifica la precisión con `focusinbox evaluate --test_set ./test.csv --output_metrics metrics.json`.

2. **Crear una regla de automatización compleja**  
   Abre el archivo `rules/finanzas.yaml` y define:  
   ```yaml
   rule: "facturas_urgentes"
   conditions:
     - field: asunto
       operator: contains
       value: ["factura", "invoice", "pago"]
     - field: remitente_dominio
       operator: in
       value: ["@proveedor.com", "@cliente.es"]
     - field: hora_llegada
       operator: between
       value: ["09:00", "18:00"]
   actions:
     - type: forward_to_webhook
       url: "https://hooks.slack.com/services/T00/B00/xxx"
       payload: { "channel": "#finanzas", "text": "Factura urgente detectada" }
     - type: mark_priority
       level: "alta"
   ```
   Aplica la regla con `focusinbox rule apply --file rules/finanzas.yaml --dry-run false`.

3. **Analizar métricas en tiempo real desde terminal**  
   Conecta al dashboard local: `curl http://localhost:8080/api/v1/metrics?period=1h | jq '.classification_accuracy, .latency_p99'`. Para exportar datos históricos: `focusinbox export --start "2025-01-01T00:00:00Z" --end "2025-01-31T23:59:59Z" --format json --output export_enero.json`. Utiliza `--compress gzip` para reducir el tamaño del archivo un 85%.

### Mejores Prácticas

- **Implementar un ciclo de retroalimentación semanal**  
  Programa una tarea cron (`0 2 * * 1`) que ejecute `focusinbox retrain --feedback_table user_feedback --model_version v2.1`. Esto recalibra los pesos del clasificador basándose en las correcciones manuales que los usuarios realizan (mover correos a carpetas incorrectas). Mantén un `validation_split=0.2` para evitar sobreajuste. Documenta los cambios en `changelog_model.md` con el hash del commit de Git.

- **Segmentar reglas por dominio y cargo del remitente**  
  Crea grupos de reglas usando etiquetas como `directivos`, `proveedores_criticos` o `newsletters`. Asigna un `priority_multiplier` diferente a cada grupo: `directivos=2.5`, `proveedores_criticos=1.8`, `newsletters=0.3`. Combínalo con un `daily_budget` de 20 correos marcados como "alta prioridad" para evitar fatiga de alertas. Usa `focusinbox rule stats --group directivos` para monitorear el impacto.
