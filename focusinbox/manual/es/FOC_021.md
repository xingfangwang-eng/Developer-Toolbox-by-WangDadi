# focusinbox - FOC_021

## Resumen

Sistema de gestión de bandeja de entrada con priorización semántica en tiempo real, diseñado para reducir el ruido de correos no críticos mediante modelos de lenguaje livianos y reglas de filtrado adaptativo. Ideal para profesionales que manejan más de 200 correos diarios.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un motor de organización de correos basado en **análisis de intención sintáctica** y **pesado por contexto temporal**. A diferencia de filtros convencionales, no solo clasifica por remitente o palabras clave, sino que evalúa la **urgencia relativa** de cada mensaje comparando su contenido con tu historial de respuestas, calendario y patrones de interacción. Integra nativamente con IMAP, Microsoft Graph API y Exchange Web Services, procesando hasta 10,000 correos por hora con una latencia media de 120 ms por análisis.

### Características Principales

- **Clasificación por Vector de Urgencia Dinámica (VUD)**: Cada correo recibe un score de 0.0 a 1.0 basado en 14 parámetros, incluyendo `reply_frequency`, `sender_trust_score`, `time_sensitivity` y `thread_depth`. El modelo se reentrena localmente cada 24 horas con tus acciones (archivar, responder, marcar como spam). Soporta umbrales configurables: `urgent_threshold: 0.85`, `focused_threshold: 0.60`, `archive_threshold: 0.25`.

- **Detección de Falsos Positivos con Reglas Compuestas**: Permite definir condiciones lógicas anidadas tipo `IF (sender.domain == 'cliente.com' AND subject CONTAINS 'factura' AND priority_flag == FALSE) THEN boost_by(0.4)`. Las reglas se evalúan en un árbol de decisión con complejidad O(log n) y se pueden exportar como JSON para auditoría. Incluye un modo `dry_run` para probar reglas sin afectar la bandeja real.

- **Dashboard de Métricas de Productividad**: Visualiza en tiempo real métricas como `correos procesados/hora`, `tasa de acierto en priorización` (comparada con tus clics manuales), `tiempo promedio de respuesta por remitente` y `volumen de correos por hora del día`. Los datos se almacenan en una base de datos local SQLite (o PostgreSQL opcional) con rotación automática cada 30 días. Exportable a CSV con `focusinbox export --format csv --since 7d`.

### Cómo Usar

1. **Configuración Inicial del Conector IMAP**: Ejecuta `focusinbox connect --provider imap --host imap.gmail.com --port 993 --ssl true --user tu_correo@gmail.com`. El sistema generará un token OAuth2 cifrado localmente con AES-256. Verifica la conexión con `focusinbox status --check-latency`. Si usas Exchange, reemplaza `--provider exchange --tenant-id <ID>`.

2. **Definición de Reglas de Priorización Personalizadas**: Crea un archivo `rules.yaml` en `~/.focusinbox/` con la siguiente estructura:
   ```yaml
   rules:
     - name: "Clientes VIP"
       condition: "sender.domain IN ['cliente1.com', 'cliente2.org'] AND subject NOT CONTAINS 'newsletter'"
       action: "set_priority 0.95"
     - name: "Recordatorios internos"
       condition: "sender == 'notificaciones@empresa.com' AND time_sensitivity > 0.7"
       action: "boost_by 0.3"
   ```
   Carga las reglas con `focusinbox rules apply --file rules.yaml`. Para validar sintaxis, usa `focusinbox rules validate --file rules.yaml`.

3. **Ejecución del Modo Enfoque y Monitoreo en Vivo**: Inicia el servicio en segundo plano con `focusinbox daemon start --mode focus --interval 300s`. El sistema reordenará tu bandeja cada 5 minutos, moviendo correos de baja prioridad a una carpeta `_Archive_` y dejando solo los críticos en `Inbox`. Monitorea el rendimiento con `focusinbox daemon stats --live`, que muestra métricas como `current_queue_depth`, `avg_processing_time_ms` y `false_positive_rate`.

### Mejores Prácticas

- **Entrenamiento Inicial con 7 Días de Historial**: Antes de confiar ciegamente en el VUD, ejecuta `focusinbox train --history-days 7 --feedback-mode manual`. Durante esta semana, marca manualmente correos como "urgente" o "pospuesto". El modelo ajustará sus pesos basándose en tus decisiones, reduciendo la tasa de error a menos del 5% tras el período de calibración.

- **Automatización de Limpieza con Reglas de Caducidad**: Configura `focusinbox rules add --name "Auto-archive after 48h" --condition "priority_score < 0.3 AND age_hours > 48" --action "move_to_archive"`. Esto evita que correos de baja prioridad se acumulen en la bandeja principal. Combínalo con una regla de notificación: `focusinbox rules add --name "Alert on critical" --condition "priority_score > 0.9 AND sender NOT IN whitelist" --action "send_pushover_notification"`.
