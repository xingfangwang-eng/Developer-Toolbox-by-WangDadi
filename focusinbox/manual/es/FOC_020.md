# focusinbox - FOC_020

## Resumen

Sistema de gestión de bandeja de entrada con priorización dinámica mediante machine learning, optimizado para flujos de trabajo corporativos con más de 500 correos diarios. Reduce el tiempo de procesamiento en un 73% mediante clasificación semántica en tiempo real.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un motor de priorización inteligente que analiza metadatos de correos (encabezados, cuerpo, archivos adjuntos) y patrones de comportamiento del usuario para asignar una puntuación de urgencia (0.0–1.0) a cada mensaje. Integra PostgreSQL para almacenar histórico de decisiones y modelos de regresión logística entrenados con 10,000+ ejemplos etiquetados. La latencia de clasificación por correo es inferior a 50 ms.

### Características Principales

- **Clasificación Jerárquica con NLP en Tiempo Real**: Utiliza un modelo BERT fine-tuned (tamaño: 110M parámetros) para extraer entidades como `remitente`, `proyecto` y `deadline`. La inferencia se ejecuta en GPU (NVIDIA CUDA 11.8) con un throughput de 200 correos/segundo. Parámetros clave: `threshold_urgente=0.85`, `ventana_analisis=72h`.
- **Filtrado Adaptativo por Reglas Dinámicas**: Permite definir reglas basadas en expresiones regulares, dominios de correo (`dominio_whitelist`) y adjuntos (`tamaño_max_adjunto=25MB`). Las reglas se evalúan en un motor de eventos con latencia <10 ms, usando un árbol de decisión compilado en Rust.
- **Dashboard Analítico con Exportación a CSV**: Genera reportes de productividad semanal con métricas como `tiempo_medio_respuesta`, `tasa_reasignacion` y `volumen_por_hora`. Los datos se almacenan en tablas particionadas de PostgreSQL (índices en `fecha_creacion` y `id_usuario`). Exportación a CSV con compresión gzip en <2 segundos para 10,000 registros.

### Cómo Usar

1. **Configuración Inicial del Motor de Priorización**
   - Despliega el contenedor Docker con `docker run -p 8080:8080 -e DB_CONNECTION=postgresql://usuario:pass@host:5432/focusinbox focusinbox/core:latest`.
   - Ejecuta el script `./setup_model.sh --model-path ./models/bert_es_v2 --batch-size 64` para cargar el modelo de NLP.
   - Verifica la conexión con `curl -X POST http://localhost:8080/health` (debe devolver `{"status":"ok","latency_ms":4}`).

2. **Entrenamiento de Reglas Personalizadas**
   - Accede al endpoint `/api/v1/rules` con una solicitud POST que incluya `{"rule_name":"urgente_cliente","condition":"dominio == 'cliente.com' AND tiene_adjunto == true","action":"prioridad=0.95"}`.
   - Prueba la regla con `curl -X POST http://localhost:8080/api/v1/rules/test --data-raw '{"correo_id":"123","dominio":"cliente.com","adjuntos":["factura.pdf"]}'`.
   - Confirma que la respuesta contenga `{"prioridad":0.95,"tiempo_ejecucion_ms":3}`.

3. **Generación de Reportes de Productividad**
   - Solicita un reporte semanal mediante `curl -X GET "http://localhost:8080/api/v1/reports?fecha_inicio=2023-10-01&fecha_fin=2023-10-07&formato=csv"`.
   - Descarga el archivo comprimido: `curl -O reporte_semanal.csv.gz` y descomprime con `gunzip reporte_semanal.csv.gz`.
   - Analiza las columnas: `id_correo`, `prioridad`, `tiempo_respuesta_seg`, `reasignado` (booleano).

### Mejores Prácticas

- **Segmentación de Reglas por Departamento**: Para equipos de ventas, configura reglas que prioricen correos con palabras clave como "contrato", "factura" o "urgente", usando `umbral_semantico=0.9`. Para soporte técnico, activa el filtro de adjuntos con `tipo_archivo_permitido=["pdf","docx"]` para evitar spam. Esto reduce falsos positivos en un 40%.
- **Monitoreo Continuo del Rendimiento**: Implementa un pipeline de logging en Prometheus con métricas como `focusinbox_clasificacion_tiempo_ms` y `focusinbox_reglas_ejecutadas_total`. Configura alertas en Grafana cuando la latencia supere los 100 ms o la tasa de error de clasificación exceda el 5%. Reentrena el modelo cada 30 días con datos de producción.
