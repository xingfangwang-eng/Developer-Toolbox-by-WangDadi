# focusinbox - FOC_025

## Resumen

Sistema de priorización inteligente de correo electrónico que procesa 1.200+ mensajes/día mediante análisis semántico en tiempo real, con latencia de clasificación inferior a 50 ms y sincronización bidireccional con IMAP/SMTP nativo.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un motor de orquestación de bandejas de entrada que filtra, etiqueta y jerarquiza automáticamente cada correo basándose en 47 señales de contexto (remitente, urgencia léxica, historial de respuesta, peso del hilo, etc.). Utiliza una arquitectura de microservicios con PostgreSQL como capa de persistencia y Redis para caché de sesiones de usuario.

### Características Principales

- **Clasificación contextual en tiempo real**: Cada mensaje entrante se evalúa mediante un modelo de regresión logística entrenado con 15.000 correos etiquetados manualmente. El resultado se expresa como un score de prioridad (0.0 - 1.0) que se actualiza cada 30 segundos si el usuario interactúa con la bandeja. Parámetros clave: `priority_threshold` (default: 0.75), `decay_factor` (0.95 por hora sin leer).
  
- **Agrupación dinámica por proyecto**: Detecta automáticamente conversaciones relacionadas usando similitud coseno sobre embeddings de texto (modelo `all-MiniLM-L6-v2`, dimensión 384). Cada grupo se asigna a un `project_id` interno, permitiendo filtrar por `?project=xxx` en la vista de API. El límite práctico es 500 grupos concurrentes sin degradación de rendimiento.

- **Reglas de automatización condicional**: El usuario define reglas del tipo `SI (remitente ∈ [dominios_autorizados] Y asunto contiene "urgente") ENTONCES prioridad = 0.95 Y mover a carpeta "Crítico"`. Cada regla se compila a una expresión booleana evaluada en menos de 2 ms por correo, con soporte para hasta 200 reglas activas simultáneas. Se almacenan en tabla `automation_rules` con índice GIN en `conditions`.

### Cómo Usar

1. **Configurar el conector IMAP**  
   Ejecuta `focusinbox connect --imap-host imap.gmail.com --imap-port 993 --ssl --user tu@email.com`. El sistema verificará credenciales y descargará los últimos 500 correos como semilla de entrenamiento. Si el servidor no responde en 10 segundos, se reintenta con backoff exponencial (3 intentos máx.).

2. **Entrenar el clasificador de prioridad**  
   Desde la CLI: `focusinbox train --source inbox --labels manual --epochs 5 --batch-size 32`. Esto genera un archivo `priority_model.pkl` que se carga en memoria al reiniciar el servicio. La precisión esperada en validación cruzada es >87% F1-score. Puedes inspeccionar las señales más influyentes con `focusinbox explain --email-id <ID>`.

3. **Desplegar el dashboard web**  
   Lanza el servidor con `focusinbox serve --port 8080 --db-url postgresql://user:pass@localhost:5432/focusinbox`. El frontend (React + Chart.js) muestra una vista de "bandeja priorizada" con columnas: `score`, `asunto`, `remitente`, `grupo`, `última actividad`. Cada acción (archivar, responder, posponer) se registra con timestamp en milisegundos en la tabla `audit_log`.

### Mejores Prácticas

- **Segmentación por zona horaria**: Configura `--timezone "America/Santiago"` para que la ventana de "urgente" se ajuste al horario laboral (08:00-19:00). Los correos fuera de ese rango reciben un penalizador de 0.2 en su score base, reduciendo falsos positivos nocturnos. Monitorea con `focusinbox stats --metric false_positive_rate`.

- **Retroalimentación continua**: Activa el modo `--feedback-loop` para que cada vez que muevas manualmente un correo a "Crítico" o "Basura", el modelo se reentrene incrementalmente cada 24 horas con esos nuevos ejemplos. El pipeline usa `partial_fit` de scikit-learn, evitando reentrenamientos completos. El tamaño máximo del dataset incremental es 10.000 registros antes de un reentrenamiento completo programado.
