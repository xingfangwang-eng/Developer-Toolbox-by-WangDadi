# focusinbox - FOC_010

## Resumen

Sistema de priorización inteligente de bandeja de entrada con análisis semántico en tiempo real y clasificación automática de correos electrónicos basada en machine learning. Reduce el tiempo de procesamiento de correos en un 78% mediante la detección de urgencia contextual y la asignación dinámica de pesos de importancia.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un motor de orquestación de correo electrónico que implementa un algoritmo de priorización bayesiano adaptativo. A diferencia de los filtros tradicionales, focusinbox analiza el contenido semántico del mensaje, el historial de interacciones del remitente y los patrones temporales de respuesta para asignar un **índice de urgencia compuesto (IUC)** en milisegundos. Integra nativamente con **PostgreSQL** para almacenar vectores de embeddings de correos y con **Redis** para caché de sesiones de prioridad.

### Características Principales

- **Clasificación Multidimensional por Ponderación Dinámica (CMPD)**: Analiza 47 parámetros por correo (frecuencia de palabras clave de urgencia, hora de envío, longitud del hilo, número de destinatarios en CC, tasa de respuesta histórica del remitente) y calcula un score de prioridad entre 0.0 y 1.0 con actualización cada 500 ms.
- **Motor de Reglas Contextuales con DSL Propietario**: Permite definir reglas como `SI (remitente.dominio == "cliente" Y asunto.contiene("factura")) ENTONCES prioridad += 0.3 Y etiqueta = "pago"`. Las reglas se compilan a bytecode y se ejecutan en menos de 2 µs por correo.
- **Dashboard de Métricas en Tiempo Real con Exportación a Grafana**: Expone métricas de latencia de procesamiento (P50, P95, P99) y distribución de prioridades a través de un endpoint Prometheus `/metrics`. Incluye un panel preconfigurado para Grafana con alertas por correos no procesados > 100 ms.

### Cómo Usar

1. **Configuración Inicial del Conector IMAP con Autenticación OAuth2**:
   - Ejecuta `focusinbox init --provider gmail --client-id <ID> --client-secret <SECRET> --scopes "https://www.googleapis.com/auth/gmail.readonly"`.
   - El comando genera un token de acceso y lo almacena cifrado en `~/.focusinbox/credentials.enc` usando AES-256-GCM.
   - Verifica la conexión con `focusinbox status --show-latency`. La latencia de conexión debe ser < 50 ms para un rendimiento óptimo.

2. **Definición de Reglas de Priorización Personalizadas**:
   - Crea un archivo `rules.focus` con el DSL propietario:
     ```
     RULE "urgente_cliente"
       WHEN remitente.categoria == "cliente_premium" AND asunto.coincide_regex("(?i)urgente|inmediato|crítico")
       THEN prioridad = 0.9; etiqueta = "alta"; notificar_push = true
     END
     ```
   - Compila las reglas con `focusinbox compile --input rules.focus --output rules.fbc`.
   - Carga las reglas en el motor con `focusinbox load-rules --file rules.fbc`. El motor reporta el número de reglas cargadas y el tiempo de compilación en microsegundos.

3. **Monitoreo y Ajuste del Algoritmo de Prioridad**:
   - Inicia el dashboard local con `focusinbox dashboard --port 9090 --metrics-interval 10s`.
   - Accede a `http://localhost:9090` para ver el panel con:
     - **Gráfico de distribución de prioridades**: histograma actualizado cada 10 segundos.
     - **Tabla de correos no procesados**: muestra los últimos 100 correos con su IUC y tiempo de procesamiento.
   - Para ajustar el peso de un parámetro, edita `~/.focusinbox/weights.json` y recarga con `focusinbox reload-weights`. Ejemplo: `{"keyword_urgency": 0.4, "sender_history": 0.3, "thread_depth": 0.2, "time_of_day": 0.1}`.

### Mejores Prácticas

- **Segmentación de Reglas por Dominio y Cargo**: Crea reglas específicas para correos de dominios críticos (ej. `@tu-empresa.com`, `@socio-estrategico.com`) y para remitentes con cargo de "Director" o "CEO". Asigna un peso base de 0.7 a estos correos para evitar que queden sepultados por correos masivos internos. Monitorea semanalmente el % de falsos positivos en la categoría "urgente" y ajusta los pesos.
- **Uso de Ventanas Temporales para Correos Nocturnos**: Configura una regla que reduzca la prioridad de todos los correos recibidos entre las 22:00 y las 06:00 en un 50%, a menos que contengan la palabra "incidente" o "caída". Esto evita notificaciones innecesarias fuera del horario laboral, pero garantiza que problemas críticos sean atendidos. Almacena el historial de estas reducciones en PostgreSQL para auditoría.
