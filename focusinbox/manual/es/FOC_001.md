# focusinbox - FOC_001

## Resumen

Plataforma de priorización inteligente de correos electrónicos que utiliza procesamiento asíncrono en segundo plano, con una latencia de clasificación inferior a 200 ms por mensaje. Diseñada para profesionales que gestionan más de 500 correos diarios, integra análisis semántico en tiempo real con PostgreSQL para mantener un historial de decisiones de prioridad.

## Contenido Detallado

### ¿Qué es focusinbox?

focusinbox es un sistema de organización de bandeja de entrada que aplica modelos de regresión logística entrenados con 10,000+ patrones de comportamiento del usuario para asignar automáticamente niveles de prioridad (Crítico, Alto, Medio, Bajo) a cada correo entrante. Utiliza una base de datos vectorial integrada con PostgreSQL para comparar embeddings de correos similares y sugerir respuestas, reduciendo el tiempo de revisión en un 40% promedio.

### Características Principales

- **Clasificación Multi-Capa con Puntaje de Urgencia Dinámico**: Cada correo recibe un `score_urgencia` (0.0 - 1.0) calculado en milisegundos, basado en el remitente (`sender_weight`), palabras clave en el asunto (`subject_keywords`), y la frecuencia de interacción previa (`interaction_frequency`). Los correos con `score_urgencia > 0.85` se marcan automáticamente como "Críticos" y se envían notificaciones push con tiempo de respuesta menor a 50 ms.

- **Motor de Reglas Personalizables con Expresiones Regulares**: Permite definir reglas como `if (from_domain == "cliente-vip.com" && subject_contains("urgencia|deadline|hoy")) then priority = "Crítico"`. Las reglas se compilan en un grafo de decisión que se evalúa en menos de 10 µs por regla, soportando hasta 500 reglas concurrentes sin degradación del rendimiento.

- **Análisis de Carga de Trabajo Predictivo**: Utiliza una red LSTM (Long Short-Term Memory) con 3 capas ocultas para predecir el volumen de correos entrantes en las próximas 24 horas, con una precisión del 92% sobre datos históricos de 90 días. El resultado se muestra en un dashboard que actualiza cada 5 minutos, permitiendo planificar bloques de tiempo de revisión.

### Cómo Usar

1. **Configuración del Perfil de Prioridad Inicial**: Ejecuta el comando `focusinbox init --profile=high-volume --max-rules=200` para crear un perfil optimizado para bandejas de alta densidad. Luego, edita el archivo `~/.focusinbox/rules.yaml` para agregar tus primeras 5 reglas personalizadas. Por ejemplo: `- rule: "cliente_urgente" match: "from:cliente@empresa.com subject:/urgente|inmediato/i" action: "priority:critical"`. Finalmente, ejecuta `focusinbox validate --rules` para verificar la sintaxis de las reglas sin errores.

2. **Entrenamiento del Modelo de Clasificación con Datos Históricos**: Exporta tus correos de los últimos 30 días a un archivo CSV usando `focusinbox export --format=csv --days=30 --output=~/emails.csv`. Luego, entrena el modelo con `focusinbox train --data=~/emails.csv --epochs=50 --batch-size=32`. El proceso tarda aproximadamente 2 minutos por cada 1,000 correos y genera un archivo `model_v2.h5` en el directorio de configuración. Para aplicar el modelo, usa `focusinbox apply-model --model=model_v2.h5`.

3. **Activación del Dashboard en Tiempo Real**: Inicia el servicio de monitoreo con `focusinbox daemon start --port=8080 --db-uri=postgresql://user:pass@localhost:5432/focusinbox`. Abre en tu navegador `http://localhost:8080/dashboard` para ver métricas en vivo: correos procesados por minuto (CPM), tiempo promedio de clasificación (en ms), y distribución de prioridades. Para detener el servicio, usa `focusinbox daemon stop`.

### Mejores Prácticas

- **Segmentación de Reglas por Dominio y Cargo**: Crea reglas específicas para dominios críticos (ej. `clientes-premium.com`, `socios-estrategicos.net`) y cargos (ej. `CEO`, `CTO`). Por ejemplo, una regla como `if (from_domain in ["clientes-premium.com"] && recipient_role == "CEO") then priority = "Crítico"` asegura que los correos de alta jerarquía no se pierdan. Esta práctica reduce el riesgo de omitir comunicaciones vitales en un 60%.

- **Revisión y Ajuste Semanal del Modelo**: Programa una tarea cron que ejecute `focusinbox evaluate --model=model_actual.h5 --test-data=~/emails_semana.csv` cada lunes a las 9:00 AM. Si la precisión del modelo cae por debajo del 85% (medido con `accuracy_score`), reentrena automáticamente con los últimos 7 días de datos. Esto mantiene la adaptabilidad a cambios en patrones de comunicación estacionales o de proyectos.
