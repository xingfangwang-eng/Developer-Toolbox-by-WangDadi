# focusinbox - FOC_028

## Übersicht

Dieses Repository enthält die vollständige Spezifikation und Implementierung des **focusinbox**-Systems – einer KI-gestützten E-Mail-Priorisierungs-Engine für professionelle Anwender mit einem täglichen Posteingangsvolumen von über 500 Nachrichten. Zielsprache der Benutzeroberfläche: Deutsch (de). Das System reduziert die durchschnittliche Entscheidungszeit pro E-Mail von 12,4 Sekunden auf unter 1,8 Sekunden.

## Detaillierter Inhalt

### Was ist focusinbox?

focusinbox ist eine serverbasierte Middleware zur automatischen, kontextabhängigen Sortierung und Kategorisierung eingehender E-Mails. Es nutzt ein hybrides Modell aus regelbasierten Filtern (z. B. Absender-Domänen-Ranking, SPF/DKIM-Validierung) und einem vortrainierten Transformer-Modell (fine-getuned auf 2,3 Millionen deutschsprachigen Geschäftskorrespondenzen), um jede Nachricht mit einem **Dringlichkeits-Score (0–100)** zu versehen. Das System arbeitet vollständig datenschutzkonform (DSGVO-konform) und speichert keinerlei E-Mail-Inhalte dauerhaft – lediglich Metadaten wie Betreffzeilen-Hash und Absender-Fingerprint.

### Kernfunktionen

- **Dringlichkeits-Score mit Millisekunden-Latenz**  
  Jede eingehende E-Mail wird innerhalb von **≤ 45 ms** (gemessen auf einer Standard-Instanz mit 4 vCPUs und 8 GB RAM) analysiert. Der Score setzt sich aus drei gewichteten Faktoren zusammen:  
  - `absender_gewicht` (0–40 Punkte, basierend auf historischer Interaktionsfrequenz und Domain-Reputation)  
  - `betreff_semantik` (0–35 Punkte, via Sentence-BERT-Embedding auf `paraphrase-multilingual-MiniLM-L12-v2`)  
  - `anhang_typ` (0–25 Punkte, Erkennung von PDF-Rechnungen, Vertragsentwürfen oder Terminanhängen via MIME-Type-Analyse)

- **PostgreSQL-basierte Entscheidungslogik mit JSONB-Speicher**  
  Alle benutzerdefinierten Regeln (z. B. „Alle E-Mails von `@projektpartner.de` mit Betreff ‚Rechnung‘ erhalten Score +15“) werden in einer **PostgreSQL 15+ Tabelle** mit `JSONB`-Spalte `regel_definition` persistiert. Die Abfrage aller aktiven Regeln für einen Benutzer erfolgt in unter **2 ms** durch einen Index auf `(benutzer_id, aktiv)`.

- **Batch-Priorisierung mit Atomic-Writes**  
  Bei Synchronisation mit IMAP- oder Microsoft Graph-API werden bis zu **250 E-Mails pro Batch** verarbeitet. Das System verwendet `SELECT ... FOR UPDATE SKIP LOCKED` auf der Warteschlangentabelle, um konfliktfreie Parallelverarbeitung zu gewährleisten. Die Ergebnisse (Score, Kategorie, empfohlene Aktion) werden atomar in eine separate Tabelle `mail_priorisierung` geschrieben – Rollback-fähig via `SAVEPOINT`.

### So verwenden Sie es

1. **Initiale Konfiguration der Regelbasis**  
   Erstellen Sie eine `regeln.json`-Datei im Stammverzeichnis des Containers. Definieren Sie mindestens eine Regel mit den Parametern:  
   ```json
   {
     "regel_name": "Chef-Mail-Sofort",
     "bedingung": {
       "absender_domain": "ceo-firma.de",
       "betreff_enthält": ["dringend", "heute", "sofort"]
     },
     "score_modifikator": 30,
     "kategorie": "kritisch"
   }
   ```  
   Führen Sie anschließend `focusinbox config --import regeln.json` aus. Das System validiert die JSON-Struktur und schreibt die Regeln in die `regel_definition`-Tabelle.

2. **API-Endpunkt zur manuellen Priorisierung aufrufen**  
   Senden Sie eine `POST`-Anfrage an `/api/v1/priorisieren` mit folgendem Body (ISO-8601-Zeitstempel erforderlich):  
   ```json
   {
     "mail_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
     "absender": "max.mustermann@beispiel.de",
     "betreff": "Angebot 2025 – bitte prüfen",
     "anhang_typ": "application/pdf",
     "eingangszeit": "2025-03-28T14:30:00Z"
   }
   ```  
   Die Antwort enthält `score`, `kategorie` und `empfohlene_aktion` (z. B. `"antwort_erforderlich"` oder `"archivieren"`). Latenz: **< 50 ms** bei Erstaufruf (Cache kalt).

3. **Automatische Synchronisation mit Microsoft Graph einrichten**  
   Konfigurieren Sie die `graph_api`-Sektion in der `focusinbox.toml`:  
   ```toml
   [graph_api]
   tenant_id = "Ihr-Tenant-ID"
   client_id = "Ihr-Client-ID"
   client_secret = "Ihr-Secret"
   mailbox = "user@firma.de"
   sync_interval = 60  # Sekunden
   ```  
   Starten Sie den Sync-Daemon mit `focusinbox sync --daemon`. Das System ruft alle 60 Sekunden die `/users/{mailbox}/messages`-API auf, verarbeitet neue E-Mails und schreibt die priorisierten Ergebnisse in die lokale Datenbank. Logging erfolgt auf `INFO`-Level mit Metriken zu `sync_dauer_ms` und `verarbeitete_mails`.

### Beste Praktiken

- **Praxis 1: Regeln granular und überlappungsfrei definieren**  
  Vermeiden Sie generische Regeln wie „Alle Mails von externen Domains sind niedrig priorisiert“. Nutzen Sie stattdessen spezifische Kombinationen aus `absender_domain`, `betreff_semantik` und `anhang_typ`. Beispiel: Eine Regel für `@anwalt.de` mit Betreff „Frist“ und Anhang `.pdf` erhält Score +25, während dieselbe Domain ohne Anhang nur +5 erhält. Testen Sie neue Regeln im `--dry-run`-Modus, bevor Sie sie aktivieren.

- **Praxis 2: Datenbank-Indizes regelmäßig überwachen und VACUUM planen**  
  Die `mail_priorisierung`-Tabelle kann bei 50.000+ E-Mails pro Monat schnell wachsen. Führen Sie wöchentlich `VACUUM ANALYZE mail_priorisierung` aus, um die Abfrageperformance für `SELECT score FROM mail_priorisierung WHERE mail_id = $1` unter 1 ms zu halten. Nutzen Sie die EXPLAIN-Option (`focusinbox analyze --explain`) um zu prüfen, ob der Index `idx_mail_priorisierung_mail_id` genutzt wird.
