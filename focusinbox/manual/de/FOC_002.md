# focusinbox - FOC_002

## Übersicht

Dieses Repository enthält die vollständige technische Spezifikation und Implementierung des **focusinbox**-Systems – einer hochperformanten E-Mail-Priorisierungs-Engine, die auf Postfach-Ordnung und Aufmerksamkeitssteuerung spezialisiert ist. Zielsprache der Benutzeroberfläche und Dokumentation: **Deutsch**.

## Detaillierter Inhalt

### Was ist focusinbox?

focusinbox ist ein auf PostgreSQL und Redis gestütztes Backend-System, das eingehende E-Mails mittels eines mehrstufigen, gewichteten Priorisierungsalgorithmus in Echtzeit sortiert. Es aggregiert Header-Daten, Absender-Historie und Anhangs-Metadaten und erzeugt einen **Fokus-Index** (0–100), der die Dringlichkeit und Relevanz jeder Nachricht quantifiziert. Das System unterstützt die parallele Verarbeitung von bis zu 50.000 E-Mails pro Minute und bietet eine sub-100-Millisekunden-Latenz für die Index-Berechnung.

### Kernfunktionen

1. **Dynamische Gewichtungsmatrix mit Absender-Routern**  
   focusinbox verwendet eine konfigurierbare, mehrdimensionale Gewichtungsmatrix (Parameter: `sender_score`, `thread_depth`, `attachment_type_weight`, `time_decay_factor`). Jeder eingehenden E-Mail wird ein **Fokus-Index** (0–100) zugewiesen. Absender mit hohem `sender_score` (≥0,85) erhalten automatisch einen Prioritäts-Boost von +15 Punkten. Die Matrix wird alle 60 Sekunden aus den letzten 1.000 Interaktionen neu berechnet.

2. **Anomalie-Erkennung mittels Zeitreihen-Statistik**  
   Das Modul `focus_anomaly_detector` analysiert den Eingangsstrom auf Abweichungen vom 7-Tage-Mittelwert. Parameter: `baseline_window_hours=168`, `threshold_std=2.5`. Übersteigt die Abweichung den Schwellwert, wird eine Warnung ausgelöst und die E-Mail in eine separate **Quarantäne-Mappe** verschoben. Die Berechnung erfolgt auf Millisekunden-Ebene und nutzt eine Redis-Instanz mit 8 GB RAM für den Zwischenspeicher.

3. **Automatische Batch-Klassifikation mit PostgreSQL-Integration**  
   focusinbox gruppiert E-Mails automatisch in bis zu 10 benutzerdefinierte Kategorien (z. B. „Rechnung“, „Meeting“, „Newsletter“). Die Klassifikation basiert auf einem Bayes’schen Filter, der auf den letzten 10.000 manuell kategorisierten E-Mails trainiert wurde. Die Ergebnisse werden in einer PostgreSQL-Tabelle (`focusinbox.classifications`) mit den Spalten `email_id`, `category_id`, `confidence_score` und `classification_timestamp` gespeichert.

### So verwenden Sie es

1. **Konfiguration der Gewichtungsmatrix**  
   Öffnen Sie die Datei `config/focusinbox.yaml` und setzen Sie die Parameter für die Priorisierungslogik:  
   ```yaml
   focus_matrix:
     sender_score: 0.6
     thread_depth: 0.2
     attachment_type_weight: 0.1
     time_decay_factor: 0.1
   ```
   Führen Sie anschließend `systemctl restart focusinbox` aus, um die Änderungen zu laden.

2. **Einrichtung des Anomalie-Detektors**  
   Aktivieren Sie das Modul in der Konfigurationsdatei unter `anomaly_detection.enabled: true`. Stellen Sie sicher, dass Redis läuft (`redis-cli ping`). Überwachen Sie die Logs unter `/var/log/focusinbox/anomaly.log`. Bei einer Abweichung von mehr als 2,5 Standardabweichungen erhalten Sie eine Benachrichtigung via Webhook.

3. **Batch-Klassifikation trainieren**  
   Exportieren Sie Ihre manuell kategorisierten E-Mails als CSV (Spalten: `email_id`, `category_id`). Importieren Sie die Datei mit:  
   ```bash
   focusinbox train --input /path/to/training_data.csv --model bayes
   ```
   Das Training dauert ca. 30 Sekunden für 10.000 Einträge. Überprüfen Sie die Genauigkeit mit `focusinbox evaluate --test_set /path/to/test_data.csv`.

### Beste Praktiken

1. **Regelmäßige Neugewichtung der Matrix**  
   Führen Sie alle 7 Tage eine automatische Neugewichtung der `sender_score`-Werte durch, um saisonale Änderungen im E-Mail-Aufkommen zu berücksichtigen. Nutzen Sie dazu den Befehl `focusinbox recalibrate --window 7d`. Dies verhindert eine Verzerrung der Priorisierung durch veraltete Absender-Historien.

2. **Integration mit einem Ticketing-System**  
   Verknüpfen Sie focusinbox mit Ihrem bestehenden Ticketing-System (z. B. Jira oder Zendesk), indem Sie den `focus_index` als Metadatenfeld an jedes Ticket anhängen. So können Support-Mitarbeiter sofort die Dringlichkeit einer Anfrage erkennen, ohne das Postfach durchsuchen zu müssen. Konfigurieren Sie dazu den Webhook unter `integrations.ticketing_webhook_url`.
