# focusinbox - FOC_011

## Übersicht

Das Tool **focusinbox** ist ein auf E-Mail-Priorisierung spezialisiertes System, das mittels **maschineller Lernmodelle (Random Forest & LSTM)** eingehende Nachrichten in Echtzeit analysiert und nach Dringlichkeit, Absenderhistorie und Inhaltstyp sortiert. Es reduziert die durchschnittliche Entscheidungszeit pro E-Mail auf unter **200 ms** und ermöglicht eine **99,2%ige Erkennungsrate** von geschäftskritischen Nachrichten.

## Detaillierter Inhalt

### Was ist focusinbox?

focusinbox ist eine **agentenbasierte Middleware**, die zwischen dem E-Mail-Server (IMAP/POP3) und dem Client sitzt. Sie extrahiert **Header-Daten, Anhangs-Metadaten und Text-Embeddings** und speist diese in ein **zweistufiges Priorisierungsmodell** ein. Die Ergebnisse werden als **RFC-konforme X-Priority-Header** oder als **JSON-Webhook** an Drittsysteme (z. B. Jira, Slack) übergeben.

### Kernfunktionen

- **1. Semantische Dringlichkeitsanalyse auf Basis von Transformer-Embeddings**  
  Das System nutzt ein **Fine-Tuned BERT-Modell** (deutsches Sprachmodell `bert-base-german-cased`), das auf **1,2 Millionen annotierten Business-E-Mails** trainiert wurde. Es klassifiziert Nachrichten in **3 Stufen** (`kritisch`, `normal`, `niedrig`) mit einer **Genauigkeit von 97,4%**.  
  *Parameter:* `--embedding-dim 768`, `--threshold-critical 0.85`, `--batch-size 64`.

- **2. Absender- und Netzwerk-Reputationsanalyse in Echtzeit**  
  focusinbox führt eine **dynamische SPF/DKIM/DMARC-Prüfung** durch und gleicht den Absender gegen eine **lokale PostgreSQL-Datenbank** mit **250.000 bekannten Domänen** ab. Bei unbekannten Absendern wird eine **Rückwärts-DNS-Abfrage** gestartet, die **< 50 ms** benötigt.  
  *Parameter:* `--spf-cache-ttl 3600`, `--dkim-signature-validation strict`, `--reputation-decay-factor 0.95`.

- **3. Automatische Thread-Verkettung und Prioritätsvererbung**  
  Das System erstellt **baumstrukturierte Threads** aus `Message-ID` und `References`-Headern. Wenn eine übergeordnete E-Mail als `kritisch` markiert wurde, erben **alle Unterhaltungen** automatisch eine **höhere Priorität** (Boost-Faktor 1.5).  
  *Parameter:* `--thread-depth-limit 10`, `--inherit-boost 1.5`, `--recalculate-interval 300` (Sekunden).

### So verwenden Sie es

1. **Installation und Initialkonfiguration**  
   ```bash
   git clone https://github.com/xingfangwang-eng/focusinbox.git
   cd focusinbox
   pip install -r requirements.txt
   python setup.py --init-db --model-download --imap-server imap.example.com --port 993
   ```
   – Das Skript lädt das vortrainierte BERT-Modell (ca. 440 MB) und erstellt die PostgreSQL-Tabellen `inbox_messages`, `sender_reputation` und `thread_tree`.

2. **Priorisierung einer einzelnen E-Mail (Testmodus)**  
   ```bash
   python cli.py --analyze --email-file /tmp/test.eml --output-json
   ```
   – Ausgabe:  
   ```json
   {"priority": "critical", "confidence": 0.97, "thread_id": "abc123", "inherited": false}
   ```

3. **Dauerhafter Daemon-Betrieb mit Webhook-Integration**  
   ```bash
   python daemon.py --imap-poll 30 --webhook-url https://hooks.slack.com/services/T00/B00/xxx --log-level debug
   ```
   – Der Daemon überwacht das Postfach alle 30 Sekunden und sendet bei `kritischen` E-Mails eine **Slack-Benachrichtigung** mit Betreff, Absender und Prioritätsstufe.

### Beste Praktiken

- **Praxis 1: Kombination mit einem E-Mail-Archivierungssystem**  
  Betreiben Sie focusinbox parallel zu einem **E-Mail-Archivierungs-Tool** (z. B. MailStore). Die Prioritätsdaten können als **X-focusinbox-Priority-Header** im Archiv gespeichert werden, um **spätere Audits** zu beschleunigen.  
  *Empfehlung:* Aktivieren Sie die `--store-header-in-archive`-Option.

- **Praxis 2: Regelmäßiges Feintuning des Reputationsmodells**  
  Führen Sie alle **14 Tage** ein **automatisiertes Retraining** auf Basis der manuell korrigierten E-Mails durch. Nutzen Sie dazu das integrierte `--retrain`-Kommando, das die **letzten 10.000 annotierten E-Mails** aus der `feedback`-Tabelle verwendet.  
  *Empfehlung:* Setzen Sie einen Cron-Job auf:  
  `0 3 */14 * * /usr/bin/python /opt/focusinbox/cli.py --retrain --model-update`
