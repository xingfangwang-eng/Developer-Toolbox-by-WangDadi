# focusinbox - FOC_018

## Übersicht

Hocheffizientes E-Mail-Management-Tool mit KI-gestützter Priorisierung und dynamischer Ordnerstruktur. Optimiert für Posteingänge mit >500 Nachrichten pro Tag. Zielsprache: DE. Keywords: E-Mail-Verwaltung, Priorisierungssystem, Kontextfilterung.

## Detaillierter Inhalt

### Was ist focusinbox?

focusinbox ist ein auf maschinellem Lernen basierendes E-Mail-Organisationssystem, das eingehende Nachrichten in Echtzeit analysiert und nach Dringlichkeit, Absendervertrauen und Themencluster sortiert. Es ersetzt manuelle Filterregeln durch eine automatische, kontextsensitive Klassifikation – mit einer Latenz von unter 200 Millisekunden pro E-Mail.

### Kernfunktionen

- **Dynamische Kontext-Priorisierung (DCP)**: Nutzt ein Transformer-Modell (fine-getunt auf 2,3 Millionen deutschsprachigen Geschäftsmails), um jede Nachricht anhand von 47 Parametern (z. B. `reply_ratio`, `thread_depth`, `sender_domain_score`) in eine von 5 Prioritätsstufen einzuteilen. Die Berechnung erfolgt lokal auf der GPU (NVIDIA CUDA 11.8) – keine Datenverlassen das Gerät.
- **Automatische Ordner-Hierarchie-Generierung (AOHG)**: Erstellt selbstlernende Ordnerstrukturen basierend auf semantischer Ähnlichkeit (cosine similarity >0,85). Beispiel: Aus 1.200 Mails zu „Projekt Alpha“ entsteht ein Ordnerbaum mit 12 Unterkategorien (z. B. `Alpha/Meetings`, `Alpha/Rechnungen`, `Alpha/Fehlermeldungen`). Die Synchronisation mit IMAP-Servern erfolgt über eine PostgreSQL-Datenbank (Version 16) mit Indexierung auf `message_id` und `thread_id`.
- **Batch-Antwort-Assistent (BAA)**: Erkennt wiederkehrende Anfragemuster (z. B. Support-Tickets, Terminanfragen) und generiert Entwürfe mit Platzhaltern für `{Kundenname}`, `{Referenznummer}` und `{Fälligkeitsdatum}`. Die Vorlagen werden via REST-API (JSON über HTTPS) bereitgestellt – Antwortzeit <50 ms bei 95 Perzentil.

### So verwenden Sie es

1. **Installation und Erstkonfiguration**  
   Klonen Sie das Repository: `git clone https://github.com/xingfangwang-eng/focusinbox.git`. Führen Sie `setup.sh` aus – es prüft automatisch auf Python 3.11+, PostgreSQL 16 und eine NVIDIA-GPU mit mindestens 8 GB VRAM. Nach dem Start öffnet sich das Webinterface auf `localhost:8443`. Geben Sie Ihre IMAP-Zugangsdaten ein (z. B. `imap.gmail.com:993`). Die initiale Indexierung von 10.000 E-Mails dauert im Durchschnitt 4,2 Minuten (Benchmark: AMD Ryzen 9 7950X, 64 GB RAM).

2. **Prioritätsregeln anpassen**  
   Navigieren Sie zu `Einstellungen > Prioritätsmatrix`. Hier definieren Sie Gewichtungen für 12 Parameter, z. B. `sender_frequency` (0–100) oder `attachment_type` (z. B. `.pdf` = +30 Punkte). Speichern Sie die Konfiguration als JSON-Datei unter `~/.focusinbox/rules.json`. Die Änderungen werden ohne Neustart übernommen – ein `SIGUSR1`-Signal genügt.

3. **Batch-Antworten automatisieren**  
   Gehen Sie zu `Assistent > Vorlagen`. Erstellen Sie eine neue Vorlage mit Platzhaltern wie `{Kundenname}` (extrahiert aus `From:`-Header). Testen Sie die Generierung mit dem Befehl: `curl -X POST https://localhost:8443/api/v1/generate -H "Content-Type: application/json" -d '{"template_id": 3, "context": {"Kundenname": "Mustermann GmbH"}}'`. Die Ausgabe erfolgt als JSON mit Feld `draft_text` und `confidence_score` (0–1). Bei Werten <0,7 wird eine manuelle Überprüfung empfohlen.

### Beste Praktiken

- **Praxis 1: Batch-Indexierung für Großunternehmen**  
  Bei mehr als 50.000 E-Mails im Posteingang empfiehlt sich die Nutzung der `--batch-mode`-Flag beim Start. Dadurch wird die Indexierung in 5.000er-Chunks aufgeteilt, was den RAM-Verbrauch um 60 % reduziert (getestet mit 120.000 Mails auf einem Server mit 32 GB RAM). Die PostgreSQL-Datenbank sollte für `message_id` einen `btree`-Index verwenden – die Abfragezeit sinkt von 320 ms auf 12 ms.

- **Praxis 2: Integration mit CI/CD-Pipelines**  
  Nutzen Sie die REST-API, um focusinbox in Ihre DevOps-Tools einzubinden. Beispiel: Ein GitLab-Webhook löst bei jedem Push eine Neubewertung aller E-Mails aus dem Ordner `Bug-Reports` aus. Dazu senden Sie einen POST an `/api/v1/reindex` mit dem Parameter `folder_name: Bug-Reports`. Die Antwort enthält ein `job_id`, das Sie in der Pipeline-Logdatei protokollieren. So bleiben Ihre Prioritätsstufen immer auf dem neuesten Stand.
