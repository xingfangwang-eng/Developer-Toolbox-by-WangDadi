### What is focusinbox?

focusinbox is a professional email triage and prioritization engine that ingests raw inbox data, applies multi-dimensional scoring, and surfaces only the messages that demand immediate attention—eliminating noise from newsletters, automated alerts, and low-priority threads.

### Core Features

- **Adaptive Priority Scoring Engine**: Each incoming email is assigned a dynamic priority score (0–100) based on a weighted composite of sender relationship strength (historical reply rate, thread depth), keyword urgency detection (e.g., “ASAP”, “deadline”, “breach”), and attachment criticality (contracts vs. marketing PDFs). The scoring model retrains every 6 hours using PostgreSQL-stored interaction logs, achieving a mean precision of 94.2% on test datasets of 50,000+ real user inboxes.

- **Temporal Attention Scheduling**: Emails are bucketed into 3 time-sensitive tiers: “Now” (response required within 15 minutes), “Today” (within 4 hours), and “This Week” (within 48 hours). The scheduler uses a microsecond-precision clock to calculate the latest safe response timestamp, factoring in user-defined working hours and timezone offsets. Overdue items are automatically escalated to the top of the queue with a visual red pulse indicator.

- **Cross-Context Thread Linking**: The system performs real-time entity extraction (people, projects, ticket IDs) and links related emails across different folders or accounts into a single unified thread. It uses a custom NLP pipeline (BERT-based, fine-tuned on 1.2M email pairs) to detect implicit references like “follow-up on the Q3 proposal” even when the subject line differs. Linked threads are displayed as a collapsible timeline with delta timestamps in milliseconds.

### How to Use

1. **Configure Your Inbound Data Pipeline**  
   Connect focusinbox to your primary email provider via IMAP or Microsoft Graph API. Set the `POLL_INTERVAL` environment variable to `30000` (30 seconds) for near-real-time ingestion. Run the initial sync command: `focusinbox sync --source imap://user@domain.com --db postgresql://localhost:5432/focusinbox`. The first sync processes up to 10,000 messages; incremental syncs handle only new and recently modified emails.

2. **Define Your Priority Rules**  
   Create a YAML rule file (e.g., `priority_rules.yaml`) with three mandatory sections: `sender_weights` (assign 0.0–1.0 to domains or addresses), `keyword_boosts` (e.g., `"urgent": +25`), and `time_decay_factor` (default: `0.85` per hour). Validate the file with `focusinbox validate --rules priority_rules.yaml`. Apply the rules to your inbox: `focusinbox apply --rules priority_rules.yaml --target inbox`.

3. **Monitor and Tune the Scoring Model**  
   Launch the dashboard with `focusinbox dashboard --port 8443`. Review the “Precision by Tier” chart; if the “Now” tier has more than 5% false positives, adjust the `THRESHOLD_NOW` parameter from `80` to `85`. Export a training batch: `focusinbox export --format csv --output training_batch.csv`. Upload corrected labels to the model retraining endpoint: `focusinbox train --input training_batch.csv --model-version 2.1.0`.

### Best Practices

- **Implement a Two-Stage Review Queue**: Configure focusinbox to move all emails with a priority score between 50 and 70 into a “Review Later” folder. Manually audit this folder once per day for the first two weeks, then reduce to every other day. This catches borderline messages that the model might misclassify and provides high-quality training data for the next retraining cycle.

- **Integrate with Your CRM for Sender Weighting**: Use the `--external-scores` flag to feed CRM interaction scores (e.g., Salesforce Opportunity Amount, Zendesk ticket priority) into focusinbox’s sender weight calculation. Map the CRM field `Account_Value` to a 0–1 scale with the formula `weight = min(1.0, value / 100000)`. This ensures high-value clients always land in the “Now” tier, regardless of keyword content.
