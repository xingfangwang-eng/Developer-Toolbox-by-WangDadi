# focusinbox - FOC_016

## Overview

focusinbox is a high-performance email intelligence engine designed to eliminate inbox noise through deterministic priority scoring, zero-latency triage, and cross-platform rule synchronization. It operates as a local-first daemon with optional cloud bridging, targeting a sub-50ms classification latency for any incoming message.

## Detailed Content

### What is focusinbox?

focusinbox is a deterministic email routing and prioritization system that parses sender behavior, thread velocity, and attachment entropy to assign a real-time `PriorityScore` (0-100) to every inbound message. It is not a client; it is a middleware layer that hooks into IMAP, Microsoft Graph, and Gmail API, enforcing custom triage policies before the email ever reaches your primary inbox.

### Core Features

- **Behavioral Sender Entropy Analysis**  
  Calculates a `SenderTrustIndex` based on historical open rates, reply latency, and domain reputation (MX/TLS checks). Emails from senders with an entropy delta exceeding 0.7 (compared to baseline) are automatically demoted to a `LowPriority` queue. All scoring is computed in-memory using a rolling 90-day window, with updates every 500ms per mailbox.

- **Thread Velocity & Temporal Decay Engine**  
  Tracks `ThreadSlope`—the rate of new messages per hour within a conversation. If the slope exceeds 3.0 messages/hour and the last reply from the user is >12 hours old, the thread is flagged as `StaleUrgent` and surfaced with a 15-minute reminder cadence. Decay coefficients are stored as PostgreSQL `INTERVAL` types for millisecond-precision re-evaluation.

- **Cross-Platform Rule Compilation & Sync**  
  Compiles user-defined `FilterRules` (e.g., `[sender_domain] IN ('asana.com', 'linear.app') AND [attachment_count] > 2`) into a bytecode-like instruction set. This set is executed natively on each client (macOS, iOS, web) with a sync latency of <200ms via a CRDT-based conflict-free data structure. Rules are versioned using a Merkle tree hash for rollback support.

### How to Use

1. **Initialize the Daemon with Your Credential Profile**  
   Run `focusinbox init --provider=imap --host=imap.gmail.com --port=993 --tls=true`. Provide your OAuth2 or app-password via the `--credential-file` flag. The daemon will perform a one-time full sync of your mailbox index (max 50,000 messages) into a local SQLite cache, then switch to delta polling every 30 seconds.

2. **Define a Priority Rule with Custom Weights**  
   Create a YAML file at `~/.focusinbox/rules/personal.yaml`:  
   ```yaml
   rules:
     - name: "Executive Alerts"
       condition: "sender.email IN ['ceo@company.com', 'cto@company.com']"
       action: "assign_priority(95)"
       weight: 2.5
     - name: "Newsletter Suppression"
       condition: "header.list_unsubscribe IS NOT NULL"
       action: "move_to_folder('LowPriority')"
   ```  
   Validate with `focusinbox validate --rules ~/.focusinbox/rules/personal.yaml`. The system will compile and deploy the ruleset within 800ms.

3. **Activate Real-Time Triage & Review the Audit Log**  
   Enable the daemon with `focusinbox daemon --mode=foreground --log-level=debug`. Every classified email is written to `~/.focusinbox/audit/YYYY-MM-DD.jsonl`, including fields like `PriorityScore`, `SenderTrustIndex`, `ThreadSlope`, and the applied rule ID. Query the log with `focusinbox query --since=1h --priority-gte=80` to see all high-priority items.

### Best Practices

- **Tune SenderTrustIndex Baselines per Domain**  
  For internal domains (e.g., `@yourcompany.com`), set a baseline entropy threshold of 0.4 rather than the default 0.7. This prevents false positives from rapid internal threads while still catching anomalous external senders. Update via `focusinbox config set --domain=yourcompany.com --entropy-threshold=0.4`.

- **Combine Temporal Decay with Folder Isolation**  
  Create a dedicated `StaleUrgent` folder in your IMAP account and map it via `focusinbox folder --name=StaleUrgent --action=mirror`. Configure the daemon to copy (not move) emails with `ThreadSlope > 2.0` and `LastReplyAge > 8h` into this folder. This gives you a zero-distraction view of conversations that need a final response before they expire.
