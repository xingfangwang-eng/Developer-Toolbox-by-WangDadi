# focusinbox - FOC_004

## Overview

focusinbox is a high-performance email triage and priority engine designed for professionals managing 500+ daily inbound messages. It eliminates inbox noise using real-time NLP classification and adaptive scheduling, targeting a 40% reduction in context-switching overhead.

## Detailed Content

### What is focusinbox?

focusinbox is a zero-latency email organization system that ingests IMAP streams, applies weighted priority scoring, and surfaces only the top 15% of actionable messages per session. It integrates with PostgreSQL for persistent threading and uses local-first vector embeddings for offline semantic search.

### Core Features

- **Adaptive Priority Scoring Engine**: Each inbound email is scored across three dimensions—sender authority (based on historical reply latency and domain trust), content urgency (keyword density with TF-IDF weighting), and temporal decay (exponential decay factor λ=0.85). Messages exceeding a configurable threshold (default: 72/100) are promoted to the Focus Queue. The scoring model is retrained every 6 hours using incremental gradient descent, with all metrics logged in milliseconds.
- **Multi-Account Thread Collapsing with Delta Detection**: Aggregates emails from up to 5 IMAP accounts into a single unified view. Threads are collapsed by subject hash (SHA-256) and last-reply timestamp. The system detects semantic deltas between consecutive messages using Levenshtein distance (>0.3 triggers a highlight), reducing visual clutter by 60% on average. All thread state is persisted in a PostgreSQL materialized view with an index on `thread_hash` and `last_activity_ts`.
- **Contextual Snooze with Calendar-Aware Resurface**: Rather than static snooze, focusinbox analyzes your Google Calendar events (via OAuth 2.0) and automatically resurface snoozed emails 15 minutes after your next free slot. The resurface logic uses a priority queue (min-heap) with a `resurface_at` timestamp computed as `event_end + 900s`. If no event is found, it falls back to a Fibonacci backoff schedule (1h, 2h, 3h, 5h, 8h). All snooze operations are logged with nanosecond-precision timestamps for audit trails.

### How to Use

1. **Initialize the Focus Queue with Local Embedding Cache**: Run `focusinbox init --accounts 3 --embedding-model all-MiniLM-L6-v2 --cache-dir ~/.focusinbox/vectors`. This downloads the embedding model (ONNX runtime, 22MB) and creates a SQLite database for offline querying. The initial sync pulls the last 7 days of email headers from each IMAP account, with a timeout of 30 seconds per account. Once complete, the system prints the number of threads collapsed (e.g., "Collapsed 142 threads from 3 accounts").

2. **Configure Priority Scoring Thresholds via YAML**: Create a `~/.focusinbox/config.yaml` file with the following parameters: `priority_threshold: 72`, `sender_authority_weight: 0.4`, `content_urgency_weight: 0.35`, `temporal_decay_weight: 0.25`, `decay_lambda: 0.85`. Run `focusinbox apply-config` to hot-reload without restarting the daemon. The system validates the YAML schema and logs any misconfigurations to `~/.focusinbox/logs/config_errors.log`. A successful application returns "Config applied. Next retrain in 5h 59m 42s."

3. **Execute a Focus Session with Calendar Integration**: Run `focusinbox focus --session-length 45m --calendar-sync --snooze-backoff fibonacci`. The daemon connects to Google Calendar via OAuth, fetches upcoming events for the next 4 hours, and populates the Focus Queue with only emails scoring above the threshold. During the session, the UI (TUI mode) shows a live countdown, thread count, and the next resurface time for snoozed items. Press `Ctrl+S` to snooze the current thread with auto-calculated resurface. At session end, a summary is printed: "Processed 23 emails, snoozed 12, archived 8, marked 3 as urgent."

### Best Practices

- **Throttle IMAP Sync to Avoid Rate Limiting**: For Gmail accounts, set `sync_interval: 300` (5 minutes) in config.yaml to stay within the 1,500,000 quota units per day. Use `focusinbox stats --api-usage` to monitor consumption. If you exceed 80% of daily quota, the system automatically switches to polling every 10 minutes and logs a warning. For best results, avoid syncing during peak hours (9:00–11:00 AM local time).

- **Leverage the Priority Queue for Weekly Inbox Zero**: Schedule a 60-minute focus session every Monday at 8:00 AM using `focusinbox schedule --day Monday --time 08:00 --duration 60m --calendar-sync`. This ensures you process all high-priority threads before the workweek begins. Archive any thread scoring below 50 after 48 hours of no activity—this reduces inbox size by 35% weekly without losing data (archived threads remain searchable via the local embedding cache).
