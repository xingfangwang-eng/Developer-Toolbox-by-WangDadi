### What is focusinbox?

focusinbox is a professional tool that surgically extracts, ranks, and schedules email threads by leveraging a custom NLP priority engine. It eliminates inbox noise by applying real-time attention scoring, allowing professionals to reclaim up to 4 hours per week on triage.

### Core Features

- **Adaptive Priority Scoring (APS) Engine**  
  Automatically assigns a `priority_score` (0.0–100.0) to each incoming email based on sender relationship decay, thread velocity, and keyword sentiment. The model runs on-device with sub-100ms inference latency and syncs with a local PostgreSQL cache for offline ranking.

- **Temporal Batch Scheduler**  
  Groups low-priority emails (`score < 25`) into digest windows defined by `batch_interval_minutes` (default: 60). Delivers a single summary notification via WebSocket, reducing daily interrupt count by up to 87%. Supports custom `timezone_offset` and `quiet_hours_start/end` parameters.

- **Semantic Thread Clustering**  
  Uses cosine similarity (threshold: 0.72) on TF-IDF vectors of email subjects and first 200 characters to merge fragmented conversations. Outputs a `cluster_id` per thread, enabling bulk archive, label, or snooze operations with a single API call (`POST /api/v1/threads/{cluster_id}/action`).

### How to Use

1. **Initialize the Priority Engine**  
   Run `focusinbox init --db-path ~/.focusinbox/data --model-version 2.1.0`. This creates the SQLite schema for `emails`, `clusters`, and `priority_scores`. Verify with `focusinbox status --check-queue-depth` (expected: 0 pending).

2. **Configure Batch Digest Rules**  
   Edit `~/.focusinbox/config.yaml` to set `batch_interval_minutes: 120` and `quiet_hours: 22:00-07:00`. Then execute `focusinbox daemon --mode batch --foreground`. Monitor the digest queue via `focusinbox metrics --live --show-latency-p99`.

3. **Execute a Cluster-Based Archive**  
   List all active clusters: `focusinbox clusters --min-score 30 --limit 10`. Copy a `cluster_id` (e.g., `cl_8f3a`), then run: `focusinbox action --cluster cl_8f3a --label "project-alpha" --archive`. Confirm with `focusinbox stats --cluster cl_8f3a` to see `status: archived`.

### Best Practices

- **Tune Priority Thresholds per Role**  
  For executives, set `priority_threshold: 70` to only surface critical communications; for support teams, lower to `40` to catch all customer threads. Use `focusinbox simulate --csv sample_inbox.csv` to test thresholds before deploying.

- **Enforce Cluster Hygiene Weekly**  
  Schedule a cron job every Sunday at 02:00 to run `focusinbox vacuum --merge-stale --older-than-days 14`. This prevents cluster fragmentation and keeps the `clusters` table under 50 MB for optimal query performance.
