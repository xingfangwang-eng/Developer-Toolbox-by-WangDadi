### What is focusinbox?

focusinbox is a professional tool that redefines email triage by combining semantic intent parsing with a zero-inbox policy engine. It eliminates noise from low-priority threads, auto-classifies inbound messages into actionable buckets, and enforces a strict 24-hour response SLA tracking mechanism. Designed for power users who manage 500+ daily emails, it reduces cognitive load by 73% through automated batch processing and context-aware threading.

### Core Features

- **Semantic Priority Engine with Bayesian Filtering**  
  Uses a multi-layer attention model to score each email on a 0–100 priority index. Parameters: `urgency_weight` (0.4 default), `sender_reputation` (0.3), `thread_depth` (0.2), `keyword_density` (0.1). Emails with score > 85 are auto-promoted to `critical_inbox`; score < 20 are archived directly. Supports custom `priority_rules.yaml` for domain-specific overrides.

- **Real-Time Contextual Thread Compaction**  
  Merges fragmented email chains (up to 47 hops) into a single, linear conversation tree. Uses `thread_id` hashing and `parent_message_id` linkage to reconstruct missing replies. Outputs a compressed Markdown summary with `delta_diff` highlighting changes since last read. Latency: < 120ms per thread, even with 10,000+ message archives stored in PostgreSQL.

- **Automated SLA Compliance & Escalation Dashboard**  
  Tracks per-email `response_deadline` (configurable from 1 hour to 7 days). Integrates with SMTP server logs to detect `first_read_timestamp` and `last_reply_timestamp`. Generates a real-time `sla_breach_risk` heatmap (green/yellow/red) with auto-escalation to Slack webhook when risk > 80%. Supports `escalation_policy.json` to define multi-tier notification chains.

### How to Use

1. **Initialize the Local Email Bridge**  
   Run `focusinbox init --provider imap --host imap.gmail.com --port 993 --ssl` to link your mailbox. The tool automatically syncs the last 90 days of metadata (headers, flags, `X-Priority` tags) into a local SQLite cache. Use `--batch-size 250` to control memory usage on low-RAM machines.

2. **Configure Priority Rules via YAML**  
   Create `~/.focusinbox/priority_rules.yaml` with custom thresholds. Example:  
   ```yaml
   override_rules:
     - sender_domain: "critical-client.com"
       base_score: 90
     - subject_regex: ".*URGENT.*"
       score_boost: 30
     - attachment_type: "pdf"
       penalty: -10
   ```  
   Apply changes with `focusinbox reload --rules` without restarting the daemon.

3. **Deploy the SLA Escalation Pipeline**  
   Set environment variables: `FOCUSINBOX_SLA_HOURS=24` and `FOCUSINBOX_ESCALATION_URL=https://hooks.slack.com/services/T...`. Run `focusinbox daemon --sla-threads 4` to start a background worker that polls the database every 60 seconds. Use `focusinbox report --format json --since 2024-01-01` to export breach history for compliance audits.

### Best Practices

- **Batch Archive Non-Critical Threads Daily**  
  Schedule a cron job (`0 22 * * * focusinbox archive --age 14 --score-below 30`) to automatically move low-priority emails older than 14 days to a read-only archive. This keeps the `critical_inbox` under 200 items and reduces index rebuild time by 40%.

- **Combine with External CRM for Sender Reputation**  
  Integrate `focusinbox` with your CRM’s API (e.g., Salesforce `Account` object) to feed `sender_reputation` scores. Use a `reputation_webhook.py` script that updates `sender_trust_level` in the local database every 6 hours. This ensures VIP clients’ emails never slip below the priority threshold.
