# focusinbox - FOC_007

## Overview

**focusinbox** is an intelligent email triage and priority orchestration engine, engineered to eliminate inbox clutter by dynamically ranking messages using Bayesian filtering and temporal attention scoring. It integrates seamlessly with IMAP/SMTP backends and provides sub-100ms latency for real-time inbox normalization.

## Core Features

### 1. Adaptive Priority Scoring via Temporal Attention Model
- **Parameter**: `attention_window_ms = 86400000` (24-hour sliding window)
- **Metric**: Priority score `P(email) = α * sender_reputation + β * thread_depth + γ * recency_decay`
- **Implementation**: Uses a lightweight ONNX runtime model ( < 5MB ) trained on user interaction patterns; re-ranks inbox every 60 seconds with < 50ms overhead per 1000 emails.

### 2. Zero-Click Smart Folderization with PostgreSQL Materialized Views
- **Parameter**: `folder_auto_assign_threshold = 0.85` (confidence score)
- **Metric**: 97.3% classification accuracy on 6 pre-defined categories (Action Required, FYI, Scheduled, Promotional, Social, Spam)
- **Backend**: PostgreSQL 16 with `pgvector` extension for semantic similarity search; materialized views refresh every 5 minutes via cron job, supporting 500K+ emails per tenant.

### 3. Real-Time Collision-Free Snooze Engine
- **Parameter**: `snooze_resolution_ms = 60000` (1-minute granularity)
- **Metric**: Supports up to 10,000 concurrent snooze timers with < 10ms scheduling latency
- **Architecture**: Redis-backed delayed job queue with Lua scripting for atomic timer operations; re-delivers emails to inbox upon timer expiry with original priority context preserved.

## How to Use

### Step 1: Initialize the IMAP Sync Pipeline
Run the following command to start the background sync daemon:
```bash
focusinbox sync --imap-server imap.gmail.com --port 993 --ssl --sync-interval 300
```
The daemon will create a PostgreSQL connection pool (max 20 connections) and begin pulling unseen emails every 5 minutes. Use `--batch-size 500` to control memory usage.

### Step 2: Train the Priority Model with Your Data
Generate a training file from your existing inbox:
```bash
focusinbox train --source imap://user@domain --output model.onnx --epochs 50 --batch-size 32
```
This exports a quantized ONNX model (INT8 precision) that reduces inference latency by 40% compared to FP32. The model learns from your `mark-read`, `star`, and `archive` events.

### Step 3: Enable Auto-Folderization and Snooze
Edit the config file at `~/.focusinbox/config.yaml`:
```yaml
auto_folder:
  enabled: true
  threshold: 0.85
snooze_engine:
  enabled: true
  max_timers: 10000
  resolution_ms: 60000
```
Then restart the daemon: `focusinbox restart`. Verify with `focusinbox status --verbose` to see active timers and folder distribution.

## Best Practices

### 1. Implement a Two-Tier Sender Reputation System
Maintain a `whitelist` (high-priority senders) and `blacklist` (bulk senders) with a decay factor of 0.95 per week. This prevents priority score drift and ensures that recurring newsletters never surface above actionable emails. Use the `--reputation-db` flag to point to a separate SQLite file for portability.

### 2. Schedule Weekly Model Retraining During Low-Traffic Windows
Set a cron job to retrain the ONNX model every Sunday at 3:00 AM using `focusinbox train --incremental`. This adapts to changing communication patterns (e.g., end-of-quarter spikes) without manual intervention. Monitor model accuracy via the `focusinbox metrics` command, which outputs precision/recall per folder.
