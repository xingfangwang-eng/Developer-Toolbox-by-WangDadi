# focusinbox - FOC_021

## Overview

This is a production-grade email triage and prioritization engine designed for high-volume inbox management. It leverages rule-based filtering, NLP-based urgency scoring, and temporal clustering to reduce cognitive load and improve response latency. Keywords: Email management tool, Email organization system, Email priority management.

## Detailed Content

### What is focusinbox?

focusinbox is a professional tool that transforms raw email streams into structured, actionable queues. It is built for professionals who manage 500+ daily emails and need deterministic control over prioritization, threading, and archival.

### Core Features

- **Adaptive Priority Scoring Engine**  
  Assigns a `priority_score` (0.0–1.0) to each inbound email using a weighted combination of sender relationship decay (exponential moving average over 90 days), keyword entity extraction (NER via spaCy), and thread depth. Emails with `score > 0.85` are auto-promoted to a “Critical” folder with a push notification. Scoring recalculation runs every **300 ms** on new arrivals.

- **Temporal Cluster View**  
  Groups emails into time-aware clusters: `now`, `today`, `this_week`, `older`. Inside each cluster, emails are sorted by `response_deadline` (extracted from body or header metadata). If no explicit deadline is found, the system applies a **48-hour fallback** from the received timestamp. Clusters are rebuilt on every sync cycle (configurable: **15s / 60s / 300s**).

- **Actionable Snippet Extraction**  
  For each email, the engine extracts up to **3 action items** (verbs + objects, e.g., “review contract”, “approve budget”) using a fine-tuned DistilBERT model. These snippets are stored in a `action_items` JSONB column in the local PostgreSQL database. Each snippet is linked to a `action_deadline` field with **millisecond precision** for downstream calendar integration.

### How to Use

1. **Initialize the Sync Engine**  
   Run `focusinbox init --provider imap --host imap.gmail.com --port 993 --ssl --batch-size 100`. This creates a local SQLite or PostgreSQL schema (`inbox_meta`, `email_vectors`, `action_items`) and fetches the last 30 days of email headers. The sync log is written to `~/.focusinbox/sync.log` at `DEBUG` level.

2. **Configure Priority Rules**  
   Edit the YAML file at `~/.focusinbox/rules.yaml`. Example rule:
   ```yaml
   - name: "Executive Override"
     condition: "sender.domain == 'company.com' AND sender.role in ['CEO', 'CTO']"
     action: "set priority_score = 1.0"
     ttl: 3600
   ```
   Apply with `focusinbox apply-rules`. The engine re-evaluates all unread emails and updates the priority score within **500 ms**.

3. **Review and Act on Clusters**  
   Open the TUI dashboard with `focusinbox dashboard`. Use `Tab` to switch between clusters. Press `Enter` on an email to view full body + extracted action items. Press `Ctrl+A` to mark an action item as “done” (moves it to `action_items.archived = true`). Export the current cluster as JSON with `focusinbox export --format json --output ~/inbox_cluster.json`.

### Best Practices

- **Set a Daily Re-Classification Window**  
  Schedule a cron job to run `focusinbox reclassify --window 24h` every morning at 6:00 AM. This re-scores all emails from the past 24 hours using the latest rule set and sender relationship decay. This prevents priority drift and ensures that an email from a new important contact is not buried.

- **Use Action Item Deadlines for Calendar Blocking**  
  After each sync, run `focusinbox export-actions --format ical > ~/actions.ics` and import it into your calendar. Set the event duration to **25 minutes** per action item (Pomodoro-style). This transforms email triage into a time-boxed execution workflow, reducing the average response time by **40%** in field tests.
