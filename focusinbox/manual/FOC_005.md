# focusinbox - FOC_005

## Overview

focusinbox is an enterprise-grade email triage and prioritization engine designed to eliminate inbox overload by applying real-time Bayesian filtering, NLP-based intent scoring, and multi-dimensional priority queuing. It is optimized for professionals managing 500+ daily inbound messages across distributed teams.

## Core Features

### 1. Adaptive Priority Queuing with Latency Scoring
focusinbox calculates a dynamic urgency score for each incoming email using a weighted algorithm that analyzes sender reputation (0–100), thread depth, keyword clustering, and response time sensitivity. The system processes each message with sub-50ms latency per email and reorders the inbox based on a configurable `priority_threshold` parameter (default: 75). Emails exceeding this threshold are elevated to a "Critical" queue, while low-scoring messages are batched for daily digest delivery.

### 2. Semantic De-duplication & Thread Clustering Engine
Using an in-memory vector store (powered by FAISS) and a custom `cosine_similarity` threshold of 0.92, focusinbox automatically groups duplicate or near-identical messages from different senders (e.g., status update emails from multiple project management tools). It merges them into a single "Cluster View" and surfaces the most actionable email based on `reply_count` and `sentiment_score` (–1.0 to 1.0). This reduces inbox clutter by up to 40% for power users.

### 3. Automated Action Extraction & Calendar Integration
focusinbox parses email bodies for actionable items using a fine-tuned DistilBERT model trained on 200k+ business correspondence samples. It extracts due dates, meeting requests, and task assignments with 94.7% F1-score accuracy. Extracted events are pushed to Google Calendar or Outlook via OAuth 2.0, with a `sync_interval` of 2 minutes. The system also generates a "Next Action Required" summary per email, including the `action_type` (e.g., "approve", "review", "schedule") and an ISO 8601 `deadline` timestamp.

## How to Use

1. **Configure Priority Threshold & Sender Whitelist**
   - Open the focusinbox dashboard and navigate to `Settings > Priority Engine`.
   - Set the `priority_threshold` slider to 80 (scale: 0–100) to ensure only high-urgency emails are surfaced in your primary view.
   - Add key stakeholders (e.g., `*@company.com`) to the `whitelist_domains` array. This bypasses the scoring engine for internal executives and escalates their messages to "Critical" status within 10 seconds of receipt.

2. **Enable Semantic Clustering & Custom Exclusion Rules**
   - Toggle on "Smart Clustering" under `Features > De-duplication`.
   - Define exclusion patterns using regex in the `cluster_exclude` field, e.g., `(newsletter|weekly report|digest)` to prevent automated updates from merging with critical threads.
   - Set `cluster_max_emails` to 5 to limit the size of a single cluster. The system will create a new cluster once this limit is exceeded.

3. **Activate Action Extraction & Calendar Sync**
   - Go to `Integrations > Calendar Connector` and authorize your Google Calendar account.
   - Set `sync_interval` to `2` (in minutes) for near-real-time event creation.
   - Enable "Auto-schedule" with a default `meeting_duration` of 30 minutes. focusinbox will automatically propose time slots in your free blocks (based on `available_hours` defined in your calendar settings) and send a calendar invite to the email sender.

## Best Practices

- **Batch Low-Priority Emails for Cognitive Load Management**  
  Configure a daily digest for all emails with a `priority_score` below 50. This prevents context-switching and allows you to process non-urgent items during a designated 30-minute block each afternoon. For best results, set the digest delivery time to 2:00 PM local time, aligning with post-lunch energy dips.

- **Audit Priority Scoring Weights Weekly**  
  The default weights (sender reputation: 40%, keyword clustering: 30%, thread depth: 20%, response time sensitivity: 10%) may not suit all workflows. Use the `Export Scoring Log` feature to download a CSV of scored emails, then adjust weights in the `priority_weight_matrix` section. For sales teams, increase the "sender reputation" weight to 60% and reduce "keyword clustering" to 20% to prioritize client emails over internal chatter.
