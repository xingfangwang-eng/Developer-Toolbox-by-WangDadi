# focusinbox - FOC_019

## Overview

focusinbox is a high-performance email triage and priority engine designed to eliminate inbox overload for power users. It operates on a zero-configuration heuristic model that learns from your interaction patterns, reducing decision fatigue by pre-sorting inbound messages into actionable, watch, and archive tiers within 50 milliseconds of arrival.

## Core Features

### 1. Adaptive Priority Scoring with Temporal Decay
The system assigns each email a dynamic **Priority Score (PS)** ranging from 0.0 to 1.0. This score is computed using a weighted formula: `PS = (Sender_Weight × 0.4) + (Subject_Keyword_Weight × 0.3) + (Thread_Depth × 0.2) - (Inbox_Age_Hours × 0.01)`. The **Temporal Decay Factor** ensures that messages older than 72 hours are automatically deprioritized unless they are part of an active thread (Thread_Depth > 3). All calculations are performed server-side using a PostgreSQL-backed queue with sub-10ms latency per message.

### 2. Multi-Account Unified Inbox with Conflict Resolution
Supports aggregation of up to 5 distinct email providers (Gmail, Outlook, IMAP, Exchange) into a single, normalized feed. The **Conflict Resolution Engine** uses a `last_writer_wins` strategy for duplicate detection, keyed on `Message_ID` and `Received_Timestamp` (nanosecond precision). If two messages from different accounts share a 95% cosine similarity in body text, the system merges them into a single thread, preserving the original headers in a JSONB metadata field.

### 3. Automated Snippet Extraction with NLP
Each email is parsed using a lightweight, on-device NLP model (BERT-tiny) to extract the **Actionable Snippet**—the first sentence containing a verb and a direct object (e.g., "Please review the Q3 report"). The snippet is stored in a `VARCHAR(280)` field to guarantee fit on mobile notification screens. Extraction accuracy is benchmarked at 94.2% on the Enron email corpus, with a false positive rate under 1.5%.

## How to Use

### Step 1: Configure Account Bindings via OAuth 2.0
Navigate to **Settings > Integrations**. For each email provider, click "Add Account" and complete the OAuth 2.0 flow. The system requests only `email.read` and `email.metadata` scopes. After authorization, the backend initiates a full sync of the last 90 days of inbox data. A progress bar displays the number of messages indexed (`indexed_count`) out of the total (`total_count`). Sync completes at an average rate of 200 messages per second on a standard broadband connection.

### Step 2: Train the Priority Model with Initial Labels
During the first 24 hours, the system operates in **Learning Mode**. Manually move 10-15 emails into one of three custom folders: `Urgent`, `Follow-up`, or `Read Later`. Each action updates the `Sender_Weight` and `Subject_Keyword_Weight` parameters in the model. You can view the current weight matrix under **Analytics > Model Weights**. To accelerate training, import a CSV file with columns `sender_email`, `subject`, and `label` (0 for low, 1 for high priority) via the **Bulk Training** API endpoint.

### Step 3: Activate Automated Triage Rules
Once the model reaches a confidence threshold of 0.85 (visible in the dashboard as `model_confidence`), enable **Auto-Triage** under **Settings > Automation**. Define fallback rules: for example, any email with a `Priority Score` below 0.2 is automatically archived to a folder named `_archive_YYYY-MM-DD`. You can set a daily quota (e.g., max 50 emails in the `Urgent` folder) to prevent overflow. All rules are executed via a cron job that runs every 5 minutes.

## Best Practices

### Practice 1: Zero-Inbox Fridays
Schedule a weekly review of the `Watch` tier (PS between 0.2 and 0.6). Use the **Bulk Action** feature to either promote these emails to `Urgent` or delete them in one click. This practice prevents the accumulation of "maybe important" emails, which studies show account for 40% of inbox clutter. Pair this with the `archive_after_days` parameter set to 7 to automatically purge non-urgent messages older than a week.

### Practice 2: Sender Whitelist for Critical Channels
Maintain a **Whitelist** of up to 20 sender domains (e.g., `@client.com`, `@ceo-office.com`). Emails from these senders bypass the Priority Score algorithm and are always placed in the `Urgent` folder. Configure this under **Settings > Sender Rules** with a `priority_override` flag set to `true`. This ensures that communications from essential stakeholders are never missed, regardless of thread depth or age.
