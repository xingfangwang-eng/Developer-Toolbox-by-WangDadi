# focusinbox - FOC_015

## 概要

focusinbox は、受信トレイのノイズをミリ秒単位でフィルタリングし、真に重要なメールのみを抽出する高精度メール優先順位管理システムです。機械学習ベースの意図推定エンジンとカスタムルールチェーンを組み合わせ、1日10,000通以上のトランザクションメールから意思決定に必要な7通を特定することを目標としています。

## 詳細な内容

### とは focusinbox?

focusinbox は、メールの「緊急度」「発信者の権限レベル」「プロジェクトとの関連性」を3次元スコアリングし、PostgreSQL上の時系列ログと照合して動的に優先順位を再計算するプロフェッショナルツールです。SMTP/IMAPプロトコルレベルで動作し、既存のメールクライアントに依存せずにバックグラウンドで分類処理を実行します。

### コア機能

- **マルチレイヤー意図推定エンジン**: BERTベースの日本語/英語ハイブリッドモデル（推論レイテンシ: 平均120ms）がメール本文から「アクション要求」「情報共有」「ステータス更新」を分類。さらに、過去30日間の返信パターンを学習し、返信必須度を0.00〜1.00のスコアで出力。
- **動的コンテキストリンク解析**: メールスレッド内の参照関係をDAG（有向非巡回グラフ）として構築。特定のプロジェクトコード（例: `PRJ-2024-09`）やJiraチケットIDを自動検出し、関連する未読メールの優先度を+0.35ポイント加算。グラフ更新は変更検知後500ms以内に完了。
- **カスタムルールチェーンエディタ**: YAML形式で記述された条件分岐（例: `if sender.domain == "client.com" and body contains "deadline" then priority = 0.95`）を実行時にコンパイル。最大100ルールまでチェーン可能で、ルール適用後のスコア再計算は1ルールあたり平均2.3μs。

### 使い方

1. **初期スキャンとベースライン構築**: 初回起動時にIMAP接続で過去90日分のメールを一括取得。`focusinbox scan --depth=90 --mailbox=INBOX` を実行し、各メールの送信者ドメイン、スレッド深度、添付ファイル有無をPostgreSQLの `email_metadata` テーブルに格納。スキャン完了後、`focusinbox train --model=priority_v2` でベースラインモデルを生成（約3分）。
2. **リアルタイムフィルタリングの有効化**: `focusinbox daemon start --port=8080 --webhook=https://hooks.slack.com/services/T00/B00/xxx` でバックグラウンドプロセスを起動。新着メール検知後、意図推定エンジンが本文を解析し、スコア0.8以上のメールのみをSlackに通知。通知にはメールの要約（最大150文字）とアクション期限（抽出可能な場合）を含む。
3. **カスタムルールのデプロイ**: `~/.focusinbox/rules/production.yaml` にルールを記述。例として、`- condition: "sender.domain == 'partner.com' AND priority < 0.5" action: "boost: 0.3"` を追加後、`focusinbox rules validate --file=production.yaml` で構文チェック。問題なければ `focusinbox rules apply --file=production.yaml --live` で即時反映。

### ベストプラクティス

- **クォータ管理とアーカイブ戦略**: IMAPサーバーの同時接続数制限（例: Outlookでは15接続）を超えないよう、`focusinbox daemon` の `--max-connections=10` を設定。処理済みメールは7日後に `email_archive` テーブルに移動し、`email_metadata` はインデックス最適化のため週次でVACUUMを実行。
- **ルールのバージョン管理とロールバック**: ルールチェーンはGitで管理し、`focusinbox rules diff --from=v1.2 --to=v1.3` で変更点を確認。重大な誤分類が発生した場合、`focusinbox rules rollback --version=v1.2 --force` で過去の状態に即時復帰。復帰後は自動で全メールの再スコアリングが走る（約2秒/1000通）。
