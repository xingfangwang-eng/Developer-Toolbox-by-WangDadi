# focusinbox - FOC_022

## 概要

focusinboxは、メールボックス内の重要度と緊急度をリアルタイムで解析し、受信トレイを自動階層化するプロフェッショナル向けメール優先順位管理システムです。機械学習による意図推定とトランザクションスコアリングにより、1日あたり10,000通以上のメールを処理可能な設計となっています。

## 詳細な内容

### focusinboxとは

focusinboxは、メールのヘッダー情報、本文内のキーフレーズ、送信者の過去インタラクション履歴を多次元でスコアリングし、緊急度0〜100の数値で即座に可視化するエンタープライズグレードのメール整理エンジンです。PostgreSQLとの連携により、全メールの処理履歴をミリ秒単位で記録・検索可能です。

### コア機能

- **マルチモーダル優先度スコアリングエンジン**: SMTPヘッダー、MIME構造、送信者ドメインのレピュテーションスコア（0.0〜1.0）を加味した重み付き線形回帰モデルを採用。緊急度パラメータ `priority_threshold` を0.75に設定することで、クリティカルメールのみを即座にフラグ付けします。

- **コンテキスト依存型自動フォルダ分類器**: メール本文のTF-IDFベクトル化と、過去30日間のユーザーアクション（開封率、返信時間、転送有無）を学習したランダムフォレスト分類器により、`inbox/urgent`、`inbox/followup`、`inbox/archive`の3階層に自動振り分け。分類精度はF1スコア0.92以上を維持。

- **リアルタイムコリレーション分析機能**: 同一スレッド内のメールをトークンIDで紐付け、返信チェーン全体のレイテンシ（応答時間中央値、単位: ミリ秒）を算出。`max_thread_latency_ms` パラメータを3600000（1時間）に設定すると、未返信スレッドを自動エスカレーションリストに追加します。

### 使い方

1. **初期インポートとスコアリング閾値の設定**: まず、`focusinbox import --mailbox-type IMAP --host imap.example.com --port 993 --ssl` で全メールをローカルPostgreSQLデータベースに取り込みます。次に、`focusinbox config set priority_threshold 0.75` を実行し、スコア0.75以上のメールのみを最優先トレイに表示するよう設定します。インポート後、`focusinbox stats --verbose` で処理件数と平均スコア分布を確認します。

2. **カスタムルールによる自動フィルタリングの適用**: `focusinbox rule add --field "from_domain" --pattern "client.com$" --action "move_to_folder:urgent" --weight 0.9` で特定ドメインからのメールを緊急フォルダに自動移動します。さらに、`focusinbox rule add --field "subject" --pattern "【障害】|【緊急】" --action "flag:critical" --ttl 86400` で件名に障害関連キーワードを含むメールに24時間のクリティカルフラグを付与します。

3. **パフォーマンス最適化とスケジューリング**: `focusinbox daemon start --interval 300000 --batch-size 500` で5分ごとに最大500通のメールをバッチ処理するデーモンを起動します。`focusinbox logs --level debug --since "2023-10-01T00:00:00Z"` で全処理ログをミリ秒単位で出力し、`max_thread_latency_ms` パラメータを調整してエスカレーションのタイミングを最適化します。

### ベストプラクティス

- **ドメインレピュテーションスコアの定期更新**: 毎週 `focusinbox reputation update --source dnsbl.spamhaus.org,zen.spamhaus.org` を実行し、送信元ドメインのブラックリスト状態を自動反映させます。これにより、スパムやフィッシングメールの誤分類を0.5%未満に抑制できます。

- **マルチユーザー環境でのスコアリングモデル分離**: チームで利用する場合、`focusinbox user create --id user_xxx --model-path /var/focusinbox/models/user_xxx.pkl` でユーザーごとに個別の分類モデルを保持します。各ユーザーの開封行動パターンを独立して学習させることで、全体の分類精度を15%向上させることが実証されています。
