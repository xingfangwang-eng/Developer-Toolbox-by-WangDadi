# focusinbox - FOC_004

## 概要

focusinbox は、受信トレイのノイズをゼロにし、真に重要なメールのみを抽出する次世代型メール優先度管理基盤です。機械学習による動的スコアリングと、ユーザーの行動パターンに基づく適応型フィルタリングを中核とし、1日あたり平均 1,200 通のメールを処理するヘビーユーザーでも、1日3回のチェックで運用可能にします。

## 詳細な内容

### focusinbox とは

focusinbox は、従来のルールベースフィルタを超え、**リアルタイムのコンテキスト分析**と**プロジェクト単位のメールスレッド再構成**を実現するプロフェッショナルツールです。PostgreSQL 上の時系列テーブルと連携し、各メールの開封率、返信速度、添付ファイルの重要度を ミリ秒単位でスコアリング。ユーザーが設定した「フォーカスプロジェクト」に該当するメールのみを、最優先で表示します。

### コア機能

1. **動的優先度エンジン (DPE)**  
   各メールに対し、`priority_score`（0.0〜1.0）を計算。パラメータとして `sender_reputation`（送信者の過去の返信率）、`thread_depth`（スレッド内の位置）、`attachment_mime_type`（添付ファイルのMIMEタイプ）を重み付け。閾値 `0.75` 以上を「フォーカスメール」と判定し、即座に通知。閾値はユーザーが `focusinbox.toml` で調整可能。

2. **プロジェクト自動分類 (APC)**  
   メール本文と件名を形態素解析（MeCab + neologd辞書）し、ユーザーが定義したプロジェクトキーワードリスト（例: `["リリース", "障害", "契約更新"]`）と照合。一致度が `cosine_similarity >= 0.85` の場合、自動でプロジェクトフォルダに振り分け。未分類メールは `unclassified` キューに滞留し、週次で再学習モデルにフィードバック。

3. **タイムクリティカルアラート (TCA)**  
   メールヘッダの `Date` と `X-Priority` を解析し、`response_deadline` を算出。デフォルトでは 受信から 4時間以内に返信が必要なメールを「緊急」とタグ付け。さらに、カレンダーAPI（Google Calendar / Outlook）と連携し、会議開始 15分前 に送信されたメールを自動で「会議関連」に分類。ユーザーが `TCA_THRESHOLD_MINUTES=30` と設定すれば、30分以内の予定に関連するメールのみを抽出可能。

### 使い方

1. **初期設定とプロジェクト定義**  
   `focusinbox init` を実行し、PostgreSQL 接続情報（`DB_HOST`, `DB_PORT`, `DB_NAME`）を `.env` に記述。その後、`focusinbox project add --name "Q3リリース" --keywords "バグ修正,デプロイ,ステージング"` でプロジェクトを登録。この時点で、APC が過去30日間のメールをスキャンし、既存スレッドにプロジェクトタグを付与。

2. **優先度閾値のチューニング**  
   `focusinbox tune --target 0.8 --metric f1` を実行。DPE が過去7日間のメールを再スコアリングし、F1スコアが最大になる `priority_score` の閾値を自動計算。結果は `~/.focusinbox/config.toml` に反映。手動で `[dpe] threshold = 0.82` と上書きも可能。

3. **リアルタイムフィルタリングの開始**  
   `focusinbox daemon --mode watch --interval 3000` を実行。3秒ごとに IMAP の受信トレイをポーリングし、新着メールを DPE でスコアリング。スコアが閾値を超えたメールのみ、`focusinbox` 専用の UI 画面に表示。ログは `journalctl -u focusinbox` で確認可能。

### ベストプラクティス

1. **週次リバランスの自動化**  
   cron で `focusinbox tune --metric precision --days 14` を毎週日曜午前3時に実行。これにより、プロジェクトの進行や送信者の行動変化に応じて優先度閾値が自動調整され、誤検知（ノイズ）が最大 40% 削減される。

2. **プロジェクトキーワードの段階的追加**  
   最初は `keywords` を3つ以内に絞り、1週間運用後に `focusinbox stats --unclassified` で未分類メールの件名を確認。共通パターン（例: "hotfix" が頻出）を発見したら `focusinbox project update --name "バグ対応" --add-keywords "hotfix"` で追加。一度に多く追加すると過学習を起こし、関連性の低いメールまで分類されるリスクがある。
