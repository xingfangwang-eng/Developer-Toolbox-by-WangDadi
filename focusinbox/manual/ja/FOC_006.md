# focusinbox - FOC_006

## 概要

focusinbox は、受信トレイのノイズをゼロに近づけるために設計された、メール管理特化型のエッジコンピューティングツールです。SMTP/IMAP プロトコルレベルでのリアルタイム解析を実現し、ユーザーのフォーカスを最大化します。

## 詳細な内容

### focusinbox とは？

focusinbox は、メールの優先度を ミリ秒単位で動的再計算 し、重要度の低いスレッドを自動的にアーカイブまたは保留状態に移行する、プロフェッショナル向けメールオーガナイザーです。PostgreSQL との連携により、過去 90 日間の開封率・返信率を元にした機械学習モデルをローカルで実行します。

### コア機能

- **優先順位エンジン（PES）**  
  メール受信時に、`priority_score` パラメータを 0.0 ～ 1.0 の範囲で算出。送信者のドメイン履歴、スレッド内の返信間隔（平均 300 秒未満なら高優先）、添付ファイルの種類（`.pdf`, `.xlsx` は +0.2）を考慮。

- **フォーカスモード切替（FMS）**  
  ユーザーが設定した `focus_window`（例：120 分）の間、`status: non_urgent` と判定されたメールを自動的に `hidden` 状態に移行。REST API 経由で即座に可視化解除も可能。

- **バッチ圧縮アーカイブ（BCA）**  
  未読期間が 72 時間を超えたメールを、`compression_level: 9` の gzip 形式でローカルストレージに圧縮保存。インデックスは PostgreSQL の `tsvector` 型で保持し、全文検索を 50ms 以内で実行。

### 使い方

1. **初期セットアップと認証連携**  
   `focusinbox init --provider gmail --oauth2-client-id <ID>` を実行。OAuth 2.0 のリフレッシュトークンを `~/.focusinbox/credentials.json` に保存。初回同期時に、過去 30 日間のメールヘッダーを `metadata_cache` テーブルにインポート。

2. **優先度ルールのカスタマイズ**  
   `focusinbox rule add --field sender_domain --value "example.org" --priority 0.8` でドメイン単位の重み付けを設定。さらに `focusinbox rule train --model logistic --epochs 50` で、ユーザーの手動アーカイブ履歴を学習させる。

3. **リアルタイムモニタリングとレポート生成**  
   `focusinbox daemon start --port 8443 --log-level debug` でバックグラウンドプロセスを起動。`focusinbox report --period last_7d --format json` で、`avg_response_time_ms`（平均応答時間）、`archival_rate`（アーカイブ率）を出力。

### ベストプラクティス

- **プラクティス 1: ドメインホワイトリスト戦略**  
  取引先ドメインを `whitelist.json` に事前登録し、`priority_score` のベースラインを 0.7 以上に設定する。これにより、スパムやマーケティングメールがフォーカスモード中に表示されるリスクを 95% 削減できる。

- **プラクティス 2: 週次モデル再学習の自動化**  
  毎週日曜日の午前 3 時に `cron` ジョブで `focusinbox rule train --model random_forest --epochs 100` を実行。学習データは直近 90 日間の `user_actions` ログ（開封、アーカイブ、削除）を使用し、モデル精度を AUC 0.92 以上に維持する。
