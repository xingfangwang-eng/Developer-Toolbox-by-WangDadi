# focusinbox - FOC_020

## 概要

focusinbox は、受信トレイのノイズを排除し、真に重要なメールのみに集中できるよう設計された、エンタープライズグレードのメール優先順位管理システムです。機械学習ベースの意図推定エンジンと、リアルタイムのコンテキスト分析により、メールの重要度をミリ秒単位でスコアリングし、ユーザーの生産性を最大化します。

## 詳細な内容

### とは focusinbox?

focusinbox は、メールの洪水の中で埋もれがちなクリティカルなメッセージを、ゼロコンフィグで自動抽出するプロフェッショナルツールです。PostgreSQL 上の時系列分析と、カスタムルールエンジン（YAML 定義）を組み合わせることで、単なるフィルタリングを超えた「コンテキストアウェアなメール編成」を実現します。

### コア機能

- **動的インテントスコアリングエンジン**: メール本文と件名から、`intent_score`（0.0～1.0）をリアルタイム算出。緊急度（`urgency_factor`）と関連性（`relevance_weight`）を掛け合わせ、`composite_priority_index` として可視化。PostgreSQL のトリガー関数と連携し、受信後 50 ミリ秒以内にスコアを更新。
- **コンテキスト依存の自動アーカイブ & スヌーズ**: ユーザーのカレンダーイベント（iCal 形式）やプロジェクト管理ツールのタスク期限（`due_timestamp`）を解析。`auto_snooze_duration`（秒単位）を動的に設定し、不要なメールを指定時間まで非表示にする。スヌーズ解除後は、`escalation_level` が自動的に 1 段階上昇。
- **マルチアカウント統合 & ルールチェーン**: IMAP/SMTP 対応の全メールプロバイダを統合。`rule_chain.yaml` に記述した条件分岐（例: `if sender_domain == "client.com" and subject_regex == ".*緊急.*" then priority = "critical"`）を、受信時に逐次実行。`chain_execution_time` は平均 120 ミリ秒未満を保証。

### 使い方

1. **初期セットアップと API キー生成**: ターミナルで `focusinbox init --provider=imap --host=mail.example.com --port=993 --ssl=true` を実行。生成された `config.yaml` 内の `api_key` フィールドを、環境変数 `FOCUSINBOX_API_KEY` に設定。`focusinbox test-connection` で IMAP 接続レイテンシ（目標: 200ms 未満）を確認。
2. **カスタムルールの実装**: `~/.focusinbox/rules/custom_rules.yaml` を編集。例えば、`- condition: { sender_domain: "supplier.com", subject_contains: "見積" }` に対し、`action: { auto_tag: "見積もり", priority_boost: 0.8 }` を設定。`focusinbox validate-rules` で構文チェック後、`focusinbox reload-rules` で即座に反映。
3. **パフォーマンスモニタリングとチューニング**: `focusinbox dashboard --metrics=latency,throughput` で、`p95_latency`（ミリ秒）と `messages_processed_per_second` をリアルタイム監視。`focusinbox tune --target-latency=100` を実行すると、バッチ処理サイズ（`batch_size`）とスレッドプール数（`worker_threads`）が自動最適化される。

### ベストプラクティス

- **プラクティス 1: ルールの段階的適用と A/B テスト**: 新規ルールを全アカウントに適用する前に、`focusinbox simulate --rule-file=new_rule.yaml --account=test@company.com` でシミュレーションを実行。`simulation_report.json` 内の `false_positive_rate` が 5% 未満であることを確認してから、`focusinbox apply-rules --scope=all` でデプロイする。
- **プラクティス 2: スコアリングモデルの定期再学習**: 毎週日曜日 03:00 UTC に、`focusinbox retrain-model --data-range=last_30_days` を cron ジョブとして設定。再学習後、`model_accuracy`（目標: 92% 以上）と `model_drift_score`（0.05 未満）を自動検証し、閾値を超えた場合のみ本番モデルを置き換える。
