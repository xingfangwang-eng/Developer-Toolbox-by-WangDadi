# focusinbox - FOC_009

## 概要

focusinboxは、受信トレイのノイズを排除し、真に重要なメールのみに集中するための次世代型メール優先順位管理エンジンです。機械学習ベースのコンテキストスコアリングとリアルタイムのワークロード分析により、1日あたり平均1,200通のメールを処理するビジネスユーザー向けに設計されています。

## 詳細な内容

### focusinboxとは

focusinboxは、メールの送信者関係性、返信頻度、プロジェクトの締切、およびカレンダーイベントのコンテキストを統合的に解析し、ユーザーが今すぐ対応すべきメールをミリ秒単位で特定するプロフェッショナルツールです。従来のルールベースフィルタリングとは異なり、動的なワークロードベクトルと時間減衰関数を用いて、緊急度と重要度を正確に定量化します。

### コア機能

- **マルチモーダル優先度エンジン**: メール本文の自然言語処理（NLP）スコア、送信者の組織階層スコア、過去30日間のインタラクション頻度、カレンダーの空き時間スロットを組み合わせた独自の`priority_score`（0.0〜1.0）を算出。しきい値0.75以上を「即時対応」、0.5〜0.74を「本日中」、0.25〜0.49を「今週中」と分類。PostgreSQL上の`focusinbox_inbox`テーブルにリアルタイムで書き込み、インデックス最適化によりクエリ応答時間は平均12ms未満。

- **コンテキストアウェア自動アーカイブ**: メールのスレッド構造を解析し、`thread_depth`（スレッド深度）が3以上かつ最新メッセージが72時間以上経過したスレッドを自動的にアーカイブ。ただし、`has_pending_action`フラグがTrueの場合はアーカイブ対象から除外。アーカイブ前には`pre_archive_webhook`を呼び出し、SlackやTeamsに通知を送信可能。アーカイブ処理は1バッチあたり最大5,000通、処理時間は平均800ms。

- **集中モードアクティベーター**: ユーザーが「集中モード」を有効にすると、`focus_window_duration`（デフォルト25分、ポモドーロテクニック準拠）の間、新着メールの通知を完全ブロック。同時に、`focusinbox`はバックグラウンドで受信メールの`urgency_decay_curve`を計算し、集中モード終了時に最も緊急性の高いメールを3通だけダイジェスト表示。この機能はmacOSの`DND`（おやすみモード）およびWindowsの`Focus Assist`とネイティブ連携。

### 使い方

1. **初期セットアップとインボックススキャン**:
   - ターミナルで`focusinbox init --provider=outlook --sync-interval=300`を実行。
   - 認証後、`focusinbox scan --full-sync --batch-size=1000`で過去90日間の全メールを解析。
   - スキャン完了後、`focusinbox status --show-metrics`で処理済みメール数、平均スコアリング時間、データベースサイズを確認。

2. **カスタム優先度ルールの設定**:
   - `focusinbox rules add --name="CEOアラート" --condition="sender_domain=example.com AND sender_title=CEO" --priority-boost=0.95`
   - `focusinbox rules add --name="プロジェクト期限" --condition="subject_contains=締切 OR deadline_in_calendar=true" --priority-boost=0.85`
   - ルール適用順序を確認:`focusinbox rules list --order-by=priority`。ルールは優先度スコアに加算され、最大値は1.0にクリッピングされる。

3. **集中モードの実行とレポート生成**:
   - `focusinbox focus start --duration=25 --block-notifications=true --auto-digest=true`
   - セッション終了後、`focusinbox focus report --export=json`で集中セッション中に受信したメールの一覧、スコアリング結果、および推奨アクションを出力。
   - 週次サマリー:`focusinbox report weekly --format=html --output=~/Desktop/focusinbox_report.html`。このレポートには、`priority_score`の分布ヒストグラム、平均応答時間、未処理メールの経過日数が含まれる。

### ベストプラクティス

- **プラクティス 1: 優先度スコアの定期的なキャリブレーション**
  毎週月曜日の朝に`focusinbox calibrate --feedback-mode=interactive`を実行し、過去1週間の実際の対応結果（すぐに返信したメール、後回しにしたメール）をフィードバックとして与える。これにより、`priority_score`の重みベクトルが動的に調整され、精度が継続的に向上する。特に、四半期ごとに`focusinbox calibrate --reset-weights`で初期状態に戻し、新たなデータで再学習させることを推奨。

- **プラクティス 2: マルチアカウント環境での統合管理**
  複数のメールアカウント（例：Gmail 3アカウント、Outlook 2アカウント）を運用する場合、`focusinbox merge --primary=work@company.com --secondary=personal@gmail.com,project@team.io`で統合インボックスを作成する。この際、`focusinbox config set --dedup-strategy=thread-merge`を有効にし、同一スレッドのメールが重複して表示されるのを防ぐ。統合後の`focusinbox inbox --limit=50`で、全アカウント横断で最も優先度の高い50通だけを表示可能。
