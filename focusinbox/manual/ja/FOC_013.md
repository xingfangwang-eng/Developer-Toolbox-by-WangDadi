# focusinbox - FOC_013

## 概要

focusinbox は、受信トレイのノイズを排除し、意思決定に直結するメールのみを抽出する、高精度なメール優先順位管理エンジンです。PostgreSQL 上のメタデータをリアルタイム解析し、ユーザーの行動パターンに基づいて動的に重要度スコアを算出します。

## 詳細な内容

### focusinbox とは

focusinbox は、SMTP/IMAP プロトコルを直接ハンドリングするプロキシ型アーキテクチャを採用。メールヘッダーの解析、添付ファイルのハッシュ値照合、送信元ドメインのレピュテーションスコアリングを ミリ秒単位で実行し、ユーザーに「今見るべきメール」だけを提示します。従来のルールベースフィルタとは異なり、教師あり学習モデル（XGBoost）を内蔵し、ユーザーの開封/アーカイブ/削除行動から継続的に重みを更新します。

### コア機能

- **動的優先順位エンジン**  
  メール到着時に、送信者履歴（過去30日間の返信率）、スレッド深度、本文内のアクションキーワード（例：「至急」「承認」「締切」）を特徴量として、0.0〜1.0の重要度スコアをリアルタイム算出。スコアが0.8以上のメールはプッシュ通知、0.5未満は自動アーカイブキューに振り分け。

- **タイムゾーン対応スケジュール同期**  
  Google Calendar API および Outlook Calendar API と連携し、ユーザーの予定時刻を取得。会議中や集中時間帯（例：毎日10:00-12:00）に到着したメールを、スコアが0.9以上の場合のみバイブレーション通知し、それ以外は「後で処理」キューに格納。キュー内メールはユーザー指定の空き時間にバッチ処理される。

- **エフェメラルリンクセキュリティスキャン**  
  メール本文中のURLを、Google Safe Browsing API と独自のドメイン評判データベース（約500万ドメインをインデックス）に照合。悪意リンクが検出された場合、メールのプレビューを自動ブロックし、管理者にSyslog形式でアラートを送出。チェック完了までのレイテンシは平均120ms以下。

### 使い方

1. **初期セットアップとプロファイル作成**  
   `focusinbox init --profile work --imap-host imap.gmail.com --imap-port 993 --ssl` を実行。初回認証後、PostgreSQL上にユーザーメタテーブルが自動作成される。続けて `focusinbox train --from-archive ./mail_archive.mbox` で過去メールを学習させ、重みモデルを初期化する。

2. **カスタムルールと例外ハンドリングの設定**  
   `focusinbox rule add --sender "client@example.com" --priority boost --score 0.95` で特定送信者を強制的に高優先度に設定。また、 `focusinbox exception --time-range "2025-04-01 09:00..2025-04-01 18:00" --action silence` で休暇中の全通知をミュートにする。設定はYAMLファイルにエクスポート可能で、チーム間で共有できる。

3. **パフォーマンスモニタリングと閾値調整**  
   `focusinbox stats --period 7d` で直近7日間の処理メール数、平均スコア分布、誤分類率（ユーザーが後で手動で重要マークしたメールの割合）を表示。誤分類率が5%を超えた場合、 `focusinbox tune --retrain --threshold 0.75` でモデルを再学習し、重要度閾値を動的に調整する。

### ベストプラクティス

- **週次モデルリトレインの自動化**  
  毎週日曜日の午前3時に `cron` ジョブで `focusinbox train --from-db --weeks 4` を実行し、直近4週間のユーザー行動データでモデルを更新する。これにより、プロジェクトの繁忙期や人事異動など、コミュニケーションパターンの変化に追随できる。

- **二段階エスカレーションポリシーの導入**  
  スコア0.6〜0.8のメールは「確認推奨」タグを付け、ユーザーが1時間以内に開封しなかった場合、自動的にSlack Webhook経由でチームチャンネルにサマリを投稿。これにより、個人の見落としをチームレベルでカバーし、SLA（サービスレベル合意）準拠のメール応答時間を維持する。
