# focusinbox - FOC_014

## 概要

本プロジェクトは、メール受信箱の認知負荷をゼロにするための次世代型メール優先順位管理システムです。機械学習によるメールのリアルタイム重要度スコアリング、マルチアカウント統合、およびタスク自動振り分けを中核とし、1日あたり1000通を超える受信量に対応します。

## 詳細な内容

### focusinboxとは

focusinboxは、メールの重要度をミリ秒単位で解析し、ユーザーの現在のコンテキスト（カレンダー、タスク管理ツール、プロジェクトステータス）に基づいて最適な処理順序を動的に提案するプロフェッショナルツールです。従来のルールベースフィルタリングを超越し、ディープラーニングモデル（Transformerベース）を採用することで、送信者の履歴関係、メール内のアクションアイテム、期限の緊急度を自動抽出します。

### コア機能

- **動的コンテキスト優先順位エンジン**  
  ユーザーのGoogle Calendar、Slackステータス、AsanaタスクをAPI経由で取得し、現在の作業フェーズ（集中モード、会議中、休憩中）に応じてメールの表示順序を自動再計算。パラメータとして`priority_weight`（0.0〜1.0）、`context_expiry_ms`（デフォルト300000ms）を設定可能。PostgreSQL上のイベントログと連携し、過去7日間のユーザー行動パターンを学習して精度を向上。

- **マルチアカウント統合レイヤー**  
  IMAP、Microsoft Graph API、Gmail APIを同時接続し、最大50アカウントの受信箱を単一インターフェースに統合。各アカウントの同期間隔は`sync_interval_sec`（デフォルト60秒）で設定可能。OAuth2.0トークン自動更新機能を内蔵し、接続断絶時のリトライ回数（`max_retry_count` = 3）とバックオフ時間（`retry_backoff_ms` = 2000ms）を細かく制御。

- **アクションアイテム自動抽出とタスク変換**  
  メール本文から「返信期限」「承認依頼」「ファイル添付要求」などを自然言語処理（NLP）で抽出し、Notion、Todoist、Jiraのタスクとして自動登録。抽出精度はF1スコア0.92以上（社内テストデータセット）。変換先のAPIエンドポイントは`task_integration_url`で指定可能。重複タスクの防止には`dedup_window_sec`（デフォルト3600秒）を適用。

### 使い方

1. **初期セットアップとアカウント連携**  
   - まず`focusinbox config --init`を実行し、設定ファイル（`~/.focusinbox/config.yaml`）を生成。  
   - 次に`focusinbox account add --provider gmail --client-id YOUR_CLIENT_ID --client-secret YOUR_CLIENT_SECRET`でGmailアカウントを追加。  
   - 最後に`focusinbox account add --provider outlook --tenant-id YOUR_TENANT_ID`でOutlookアカウントを追加。各アカウントの同期間隔は`--sync-interval 30`（秒単位）で上書き可能。

2. **優先順位エンジンのカスタマイズ**  
   - `focusinbox priority set --context-weight 0.7 --deadline-weight 0.3 --sender-weight 0.5`で各要素の重みを調整。  
   - 集中モードの有効化は`focusinbox mode focus --duration 120`（120分間）で実行。この間、非緊急メールは自動的に「後で読む」フォルダへ移動。  
   - 変更を適用するには`focusinbox priority reload`を実行。設定はPostgreSQLの`priority_profiles`テーブルに即時保存される。

3. **タスク自動変換の有効化**  
   - `focusinbox task integrate --tool notion --api-key YOUR_NOTION_API_KEY --database-id YOUR_DATABASE_ID`でNotion連携を設定。  
   - 抽出ルールのカスタマイズは`focusinbox task rule add --action "approval" --priority high --auto-assign user@example.com`で追加。  
   - 動作確認は`focusinbox task test --email-id 12345`で単一メールの解析結果を表示。抽出されたアクションアイテムが正しいか確認後、`focusinbox task sync`で一括登録。

### ベストプラクティス

1. **マルチアカウント環境での同期競合回避**  
   同一メールが複数アカウントに届く場合、`focusinbox account set-dedup --enabled true --window 7200`で2時間の重複排除ウィンドウを設定。これにより、同一メールが重複してタスク化されるのを防止。さらに、`--master-account primary@example.com`を指定することで、そのアカウントのメールを正規とみなし、他アカウントの同一メールは自動的にアーカイブされる。

2. **チーム全体での優先順位ポリシー統一**  
   チームメンバー全員に同じ`priority_weight`プロファイルを適用するには、`focusinbox priority export --format yaml > team_profile.yaml`で設定をエクスポート。その後、各メンバーが`focusinbox priority import --file team_profile.yaml`でインポート。さらに、`focusinbox policy enforce --profile-id team_default --force`を実行することで、設定の上書きを禁止し、組織全体の一貫性を担保する。
