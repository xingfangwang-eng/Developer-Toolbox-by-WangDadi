# agents - AGE_025

## Resumen

**agents** es una plataforma de gestión y orquestación de agentes de IA diseñada para entornos de producción complejos. Permite desplegar, monitorizar y escalar sistemas multi-agente con capacidades de memoria a largo plazo, ejecución de tareas asíncronas y enrutamiento semántico inteligente. Ideal para automatización empresarial, pipelines de datos y asistentes virtuales de alto rendimiento.

## Contenido Detallado

### ¿Qué es agents?

**agents** es una herramienta profesional que resuelve el problema de coordinar múltiples agentes de IA en un ecosistema dinámico. A diferencia de soluciones monolíticas, **agents** implementa un **orquestador central** que gestiona estados, dependencias y colas de mensajes entre agentes heterogéneos (LLMs, RAG, APIs externas). Su arquitectura basada en **DAGs (Directed Acyclic Graphs)** permite definir flujos de trabajo complejos con tolerancia a fallos y reintentos automáticos.

### Características Principales

#### 1. Memoria Contextual Distribuida (DCM)
Cada agente mantiene un **buffer de contexto persistente** sincronizado en Redis o PostgreSQL. Esto permite que agentes que ejecutan tareas secuenciales (ej: análisis de sentimiento → generación de respuesta) compartan estado sin necesidad de reentrenar el modelo. Incluye **podado automático de tokens** para evitar desbordamiento de contexto.

#### 2. Enrutamiento Semántico Dinámico (DSR)
El orquestador analiza la intención del usuario mediante **embeddings** (modelo `all-MiniLM-L6-v2`) y dirige la petición al agente especializado óptimo. Soporta **fallback jerárquico**: si un agente falla, se redirige a un agente genérico sin perder el contexto de la conversación.

#### 3. Ejecución Asíncrona con Colas de Prioridad
Cada tarea se encola en **RabbitMQ** con niveles de prioridad (crítica, normal, batch). Los agentes pueden ejecutarse en paralelo (hasta 50 workers por defecto) y reportar resultados mediante **WebSockets** en tiempo real. Ideal para procesamiento de documentos, scraping web o generación de informes.

### Cómo Usar

#### 1. Despliegue de un Agente Personalizado con Dependencias Externas
```bash
agents create --name "analista_ventas" \
  --model "gpt-4" \
  --tools "sql_connector, email_sender" \
  --memory "redis://localhost:6379/0" \
  --max-tokens 4096
```
Esto registra un agente que puede consultar una base de datos SQL y enviar correos. El flag `--memory` activa el buffer de contexto persistente.

#### 2. Definición de un Pipeline Multi-Agente con DAG
```yaml
# pipeline_analisis.yaml
nodes:
  - id: extraer_datos
    agent: scraper_web
    params: { url: "https://api.ejemplo.com/reportes" }
  - id: analizar_tendencias
    agent: analista_ventas
    depends_on: extraer_datos
    params: { formato: "JSON" }
  - id: generar_resumen
    agent: generador_texto
    depends_on: analizar_tendencias
    params: { idioma: "es" }
```
Ejecuta con: `agents run --pipeline pipeline_analisis.yaml`. El orquestador resuelve dependencias y ejecuta en orden.

#### 3. Monitorización en Tiempo Real con Dashboard Web
```bash
agents dashboard --port 8080 --metrics "latency, throughput, error_rate"
```
Abre `http://localhost:8080` para ver:
- **Gráfico de ejecución**: nodos activos, colas pendientes.
- **Logs en vivo**: cada agente emite logs estructurados (JSON) con trace_id para debugging.
- **Alertas**: umbrales configurables (ej: latencia > 5s → notificación Slack).

### Mejores Prácticas

#### 1. Segmentación de Agentes por Dominio y Prioridad
No mezcles agentes críticos (ej: procesamiento de pagos) con agentes batch (ej: generación de informes). Usa colas separadas (`--queue "alta_prioridad"`) y asigna recursos de CPU/RAM diferenciados. Esto evita que tareas pesadas bloqueen las urgentes.

#### 2. Versionado de Prompts y Modelos
Cada agente debe tener un `prompt_template` versionado (ej: `v1.2`). Usa el flag `--prompt-version` al crear el agente. Si un modelo falla (ej: cambio de API), puedes hacer rollback sin afectar otros agentes. Almacena los prompts en un repositorio Git y sincronízalos con **agents sync**.
