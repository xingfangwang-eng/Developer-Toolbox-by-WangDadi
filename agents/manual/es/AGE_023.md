# agents - AGE_023

## Resumen Técnico

**agents** es una plataforma SaaS de orquestación de agentes de IA diseñada para arquitectos y desarrolladores que necesitan desplegar sistemas multiagente con capacidad de razonamiento contextual, memoria persistente y ejecución asíncrona en entornos de producción. Optimizado para flujos de trabajo complejos en español, este motor permite gestionar desde asistentes conversacionales hasta sistemas de automatización empresarial con trazabilidad completa.

## Funcionalidades Clave (Hardcore)

### 1. **Orquestador de Tareas con Memoria Contextual Híbrida**
Cada agente mantiene un estado interno basado en una base vectorial (FAISS/Pinecone) combinada con una caché LRU en Redis. Esto permite que el sistema recuerde interacciones previas sin depender exclusivamente del prompt, reduciendo el consumo de tokens en un **40%** en pruebas comparativas. Ideal para diálogos largos o procesos con múltiples saltos de contexto.

### 2. **Pipeline de Acciones Asíncronas con Rollback Automático**
Los agentes pueden encadenar acciones (llamadas a APIs, escritura en bases de datos, envío de notificaciones) en un grafo DAG. Si una acción falla, el sistema ejecuta un rollback transaccional automático sobre las acciones previas, manteniendo la integridad del estado. Soporta reintentos con backoff exponencial y colas de mensajería (RabbitMQ/Kafka).

### 3. **Modo Reflexivo con Autoevaluación de Confianza**
Cada respuesta generada incluye un score de confianza calculado mediante un modelo de verificación interno (basado en BERT fine-tuned). Si el score es inferior a 0.7, el agente solicita aclaración al usuario o consulta una fuente externa (base de conocimiento, API REST) antes de responder. Esto elimina alucinaciones en entornos críticos como soporte técnico o cumplimiento normativo.

## Cómo Usar: Pasos Detallados

### 1. **Configurar el Entorno Multiagente**
```bash
# Clona el repositorio y levanta los servicios con Docker Compose
git clone https://github.com/xingfangwang-eng/agents.git
cd agents
docker-compose up -d --build
```
Esto inicia el orquestador principal, Redis, la base vectorial y el worker de colas. Verifica que todos los servicios estén saludables:
```bash
curl http://localhost:8080/health
```
Deberías recibir `{"status":"ok","version":"2.3.1"}`.

### 2. **Definir un Agente con Roles y Herramientas**
Crea un archivo YAML `mi_agente.yml`:
```yaml
agent:
  name: "SoporteTecnico"
  model: "gpt-4-turbo"
  tools:
    - name: "buscar_ticket"
      type: "api"
      endpoint: "https://api.example.com/tickets/{id}"
      method: "GET"
    - name: "enviar_email"
      type: "function"
      code: |
        def enviar(destino, asunto, cuerpo):
            # Lógica de envío
            pass
  memory:
    type: "hybrid"
    vector_store: "pinecone"
    cache_ttl: 3600
```
Carga el agente:
```bash
agents-cli load --file mi_agente.yml
```

### 3. **Ejecutar una Interacción con Trazabilidad**
Envía una consulta al agente y activa el modo reflexivo:
```python
from agents_sdk import AgentClient

client = AgentClient(api_key="tu_api_key", endpoint="http://localhost:8080")
response = client.query(
    agent_id="SoporteTecnico",
    message="El ticket #1234 no se actualiza desde ayer",
    reflexive=True,  # Activa autoevaluación
    timeout=30
)
print(response.answer)  # Respuesta generada
print(response.confidence)  # Score de confianza (0.0-1.0)
print(response.chain_id)  # ID para auditoría
```
Si el score es bajo, el agente te pedirá confirmación antes de actuar.

## Mejores Prácticas para Desarrolladores

### 1. **Usa el Modo Reflexivo en Entornos Regulados**
En sectores como fintech o salud, activa siempre `reflexive=True`. Esto asegura que el agente no ejecute acciones sin verificar su certeza. Combínalo con un webhook de alerta para cuando el score caiga por debajo de 0.5, y así revisar manualmente la interacción.

### 2. **Segmenta Agentes por Dominio y Escala**
No crees un agente monolítico. Divide tu sistema en agentes especializados (uno para facturación, otro para inventario, otro para atención al cliente) y usa el orquestador para enrutar consultas según el contexto. Esto mejora el rendimiento (cada agente carga menos herramientas) y facilita el mantenimiento. Escala horizontalmente los workers de colas según la carga esperada.
