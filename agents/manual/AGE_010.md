# agents - AGE_010

## Overview

`agents` is a high-performance, polyglot AI agent orchestration framework designed for developers who demand deterministic behavior, low-latency execution, and granular control over their autonomous agent pipelines. Unlike generic LLM wrappers, `agents` provides a production-ready architecture for building, managing, and monitoring intelligent agent systems with a focus on stateful workflows and multi-model routing.

## Core Features

### 1. **Deterministic Agent Graph Engine**
`agents` introduces a **Directed Acyclic Graph (DAG) execution model** for agent pipelines. Each agent node is a pure function with typed inputs/outputs, supporting conditional branching and parallel execution. This eliminates the "black box" unpredictability of sequential LLM calls, enabling reproducible results across runs—critical for compliance-heavy industries like fintech or legal tech.

### 2. **Polyglot Model Router with Cost-Aware Scheduling**
The built-in **Model Router** dynamically selects the optimal LLM (or local model) per task based on latency, token cost, and task complexity. For example, a simple classification task routes to a lightweight model (e.g., `llama3-8b`), while a complex code generation task escalates to `gpt-4o`. The router supports real-time fallback logic and rate-limit-aware retries, reducing API costs by up to 40% in production benchmarks.

### 3. **Stateful Agent Memory with Vectorized Context Compression**
`agents` includes a **persistent memory layer** that automatically compresses and vectorizes agent conversation history. Instead of dumping raw tokens into the context window, it uses a sliding window with semantic chunking and embedding-based retrieval. This allows agents to maintain coherent long-term interactions (1000+ turns) without exceeding token limits—ideal for customer support or research assistants.

## How to Use

### 1. **Define a Custom Agent with Typed Input/Output Contracts**
Create an agent using the `@agent` decorator with Pydantic models for validation. This ensures type safety and auto-generates OpenAPI-compatible schemas for downstream integrations.

```python
from agents import agent, AgentConfig
from pydantic import BaseModel

class InputSchema(BaseModel):
    query: str
    context_id: str = None

class OutputSchema(BaseModel):
    response: str
    confidence: float

@agent(
    config=AgentConfig(model="gpt-4o", temperature=0.2),
    input_schema=InputSchema,
    output_schema=OutputSchema
)
async def support_agent(input: InputSchema) -> OutputSchema:
    # Your agent logic here
    return OutputSchema(response="...", confidence=0.95)
```

### 2. **Configure a Polyglot Router with Fallback Rules**
In your `agents.yaml` configuration file, define routing rules based on task categories. The router will automatically switch models if the primary model fails or exceeds latency thresholds.

```yaml
router:
  rules:
    - category: "classification"
      model: "llama3-8b"
      fallback: "gpt-4o-mini"
      max_latency_ms: 500
    - category: "code_generation"
      model: "gpt-4o"
      fallback: "claude-3-opus"
      max_cost_per_call: 0.05
  default: "gpt-4o-mini"
```

### 3. **Deploy a Stateful Agent with Persistent Memory**
Instantiate an agent session with memory enabled. The `agents` runtime will automatically handle context compression and retrieval across turns.

```python
from agents import AgentSession, MemoryConfig

session = AgentSession(
    agent_id="support_agent",
    memory_config=MemoryConfig(
        strategy="semantic_chunking",
        max_tokens_per_chunk=2048,
        embedding_model="text-embedding-3-small"
    )
)

response = await session.run(
    input_data=InputSchema(query="What was my last order?"),
    session_id="user_123"
)
```

## Best Practices

### 1. **Use Model Routing for Cost Optimization in SaaS**
If you're building a multi-tenant SaaS platform, leverage the `Polyglot Router` to assign cheaper models to low-stakes tasks (e.g., summarization) and premium models to high-stakes tasks (e.g., contract analysis). Monitor the `cost_per_call` metric in the built-in dashboard to fine-tune routing thresholds weekly.

### 2. **Implement Agent Memory for Long-Running Workflows**
For customer-facing agents (e.g., enterprise support), always enable `stateful memory` with semantic chunking. This prevents context window overflow during long conversations and allows agents to reference past interactions without re-processing entire histories. Use the `session_id` parameter to isolate user-specific contexts.
