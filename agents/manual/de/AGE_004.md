## Detaillierter Inhalt

### Was ist agents?

agents ist ein professionelles Tool, das speziell für die Entwicklung, Bereitstellung und Verwaltung von KI-gesteuerten Agenten in komplexen, mehrstufigen Arbeitsabläufen entwickelt wurde. Es löst typische Probleme wie Fragmentierung von Agentenlogik, fehlende Zustandsverwaltung und ineffiziente Ressourcenzuweisung in verteilten Systemen.

### Kernfunktionen

- **Multi-Agenten-Orchestrierung mit dynamischer Aufgabenverteilung:** agents verteilt Aufgaben automatisch an spezialisierte Unteragenten basierend auf Echtzeit-Kontext und Ressourcenverfügbarkeit. Dies ermöglicht eine skalierbare parallele Verarbeitung ohne manuelle Koordination.

- **Zustandsbehaftete Agenten-Pipelines mit Rollback-Fähigkeit:** Jeder Agent kann seinen internen Zustand persistieren und bei Fehlern automatisch auf einen definierten Checkpoint zurückgesetzt werden. Dies garantiert Datenkonsistenz und Fehlertoleranz in langlaufenden Workflows.

- **Selbstheilende Agenten mit automatischer Wiederherstellung:** agents überwacht die Laufzeitumgebung und startet fehlerhafte Agenten automatisch neu, ohne dass der gesamte Workflow unterbrochen wird. Dies reduziert Ausfallzeiten und manuelle Eingriffe drastisch.

### So verwenden Sie es

1. **Konfiguration der Agenten-Pipeline mit YAML-Definition:**  
   Erstellen Sie eine `agents.yaml`-Datei, in der Sie die Abhängigkeiten und Rollen der Agenten definieren. Verwenden Sie das `dependencies`-Feld, um die Reihenfolge der Ausführung festzulegen, und das `rollback`-Feld, um Checkpoints für jeden Agenten zu setzen. Beispiel:
   ```yaml
   agents:
     - name: data_collector
       type: fetcher
       dependencies: []
       rollback: true
     - name: analyzer
       type: processor
       dependencies: [data_collector]
       rollback: false
   ```

2. **Starten des Agenten-Clusters mit Skalierungsoptionen:**  
   Führen Sie den Befehl `agents start --scale data_collector=3` aus, um die Anzahl der Instanzen für den `data_collector`-Agenten auf 3 zu erhöhen. agents verteilt die Last automatisch und überwacht die Auslastung in Echtzeit. Nutzen Sie `--max-retries 5`, um die Anzahl der automatischen Wiederholungen bei Fehlern zu begrenzen.

3. **Überwachung und Fehlerbehebung mit dem Dashboard:**  
   Öffnen Sie das agents-Dashboard unter `http://localhost:8080`. Hier sehen Sie den Status jedes Agenten, die aktuelle Auslastung und die Anzahl der durchgeführten Rollbacks. Klicken Sie auf einen fehlerhaften Agenten, um detaillierte Logs anzuzeigen und manuelle Eingriffe durchzuführen. Verwenden Sie den `force-restart`-Button, um einen Agenten sofort neu zu starten.

### Beste Praktiken

- **Definieren Sie klare Rollback-Strategien für kritische Agenten:**  
  Setzen Sie `rollback: true` nur für Agenten, die Daten persistieren oder externe Systeme beeinflussen. Für reine Berechnungsagenten ohne Seiteneffekte sollten Sie auf Rollbacks verzichten, um Performance-Verluste zu vermeiden.

- **Nutzen Sie die automatische Skalierung für Lastspitzen:**  
  Verwenden Sie den `--scale`-Parameter nur für Agenten, die tatsächlich parallelisierbar sind. Bei sequenziellen Abhängigkeiten führt eine Skalierung zu keiner Performance-Steigerung. Überwachen Sie die Auslastung im Dashboard und passen Sie die Skalierung dynamisch an.
