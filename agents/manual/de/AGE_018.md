## Detaillierter Inhalt

### Was ist agents?

agents ist ein professionelles Tool zur Lösung komplexer Probleme in den Bereichen KI-gesteuerte Automatisierung, Multi-Agenten-Koordination und Entscheidungsfindung. Es ermöglicht Entwicklern und Unternehmen, skalierbare, autonome Agenten-Workflows zu definieren, zu überwachen und zu optimieren – ohne die typischen Hürden von manuellen Skripten oder monolithischen Systemen.

### Kernfunktionen

- **Dynamische Agenten-Orchestrierung mit Kontext-Pooling**  
  agents verteilt Aufgaben an spezialisierte Sub-Agenten auf Basis eines kontextbewussten Routings. Jeder Agent greift auf einen gemeinsamen, persistenten Kontextpool zu, der Zwischenergebnisse, Metadaten und Fehlerzustände in Echtzeit synchronisiert – ideal für mehrstufige Reasoning-Pipelines.

- **Self-Healing Execution Engine**  
  Die Engine erkennt fehlgeschlagene Agenten-Instanzen oder Timeouts automatisch und startet sie mit dem letzten validen Zustand neu. Optional können alternative Pfade (Fallback-Agenten) definiert werden, sodass kritische Workflows auch bei Teilausfällen stabil bleiben.

- **Native Multi-API-Bridge mit Schema-Validierung**  
  agents kapselt externe API-Aufrufe (OpenAI, Anthropic, eigene Modelle) in eine einheitliche Schnittstelle. Jeder Agent kann eigene Input/Output-Schemas in JSON Schema definieren, die zur Laufzeit geprüft werden – inklusive automatischer Retry-Strategien bei Ratenlimits oder 5xx-Fehlern.

### So verwenden Sie es

1. **Agenten-Definition als deklaratives YAML-Manifest**  
   Erstellen Sie eine agents-Konfigurationsdatei (`agents.yml`), in der Sie Rollen, Fähigkeiten und Abhängigkeiten jedes Agenten festlegen. Beispiel: `role: research_agent` mit `capabilities: [web_search, summarization]` und `max_retries: 3`. Die Engine liest dieses Manifest beim Start und baut daraus einen gerichteten azyklischen Graphen (DAG) für die Ausführung.

2. **Workflow via CLI oder API triggern**  
   Starten Sie einen Workflow mit `agents run --manifest agents.yml --input "{\"query\": \"Neueste KI-Trends\"}"`. Die Engine instantiiert die Agenten, übergibt den initialen Kontext und protokolliert jeden Schritt in einer strukturierten Log-Datei (JSON-Lines). Für die Integration in CI/CD oder eigene Backends nutzen Sie die REST-API mit JWT-Authentifizierung.

3. **Ergebnisse analysieren und Agenten-Tuning**  
   Nach Abschluss des Workflows erhalten Sie ein detailliertes Trace-Report im HTML- oder JSON-Format. Dieses zeigt Latenzen, Token-Verbrauch, Fehlerquellen und die genauen Entscheidungspfade jedes Agenten. Optimieren Sie die Konfiguration durch Anpassen von `temperature`, `max_tokens` oder `fallback_agent`-Referenzen – ohne Codeänderungen.

### Beste Praktiken

- **Granulare Fehlerbehandlung mit Fallback-Kaskaden**  
  Definieren Sie für jeden Agenten mindestens einen Fallback-Agenten mit reduzierter Komplexität (z. B. `research_agent_fallback` mit nur einem API-Provider). So vermeiden Sie Single Points of Failure und stellen sicher, dass auch bei Ausfall eines externen Dienstes noch eine Basisantwort generiert wird.

- **Kontext-Pooling auf 80 % der maximalen Token-Limit begrenzen**  
  Um Token-Explosionen in langlaufenden Workflows zu verhindern, setzen Sie `context_pool_limit: 0.8` in der agents.yml. Die Engine kürzt dann automatisch ältere Kontexteinträge (LRU-Strategie) oder fasst sie in Zusammenfassungen – ohne die Entscheidungslogik der nachfolgenden Agenten zu beeinträchtigen.
