## What is agents?

**agents** is a production-grade framework for designing, deploying, and monitoring autonomous AI agents. It abstracts away the complexity of multi-agent coordination, memory management, and tool integration — letting you focus on logic, not boilerplate.

## Core Features

### 1. **Declarative Agent Graph Engine**
Define agent workflows as directed acyclic graphs (DAGs) using a simple YAML or JSON schema. Each node is a discrete agent with its own memory, tools, and LLM configuration. The engine handles state propagation, retries, and parallel execution — no custom orchestration code required.

### 2. **Dynamic Tool Injection & Sandboxing**
Inject external APIs, databases, or custom Python functions into any agent at runtime. Tools are sandboxed with per-call resource limits (CPU, memory, timeout) and logged with full traceability. Supports hot-reloading of tool definitions without restarting the agent runtime.

### 3. **Real-Time Agent Telemetry & Self-Healing**
Every agent emits structured logs, metrics, and traces to a built-in observability layer. If an agent fails or exceeds latency thresholds, the system automatically re-routes tasks to a fallback agent or triggers a rollback to the last consistent state. No manual intervention needed.

## How to Use

### 1. Define Your Agent Topology
Create a `agents.yaml` file in your project root. Specify each agent’s role, LLM provider, and tool bindings. Example:

```yaml
agents:
  - name: data-extractor
    llm: openai/gpt-4
    tools: [web-scraper, pdf-parser]
    memory: ephemeral
  - name: summarizer
    llm: anthropic/claude-3
    tools: [text-analyzer]
    memory: persistent
edges:
  - from: data-extractor
    to: summarizer
    condition: on_success
```

### 2. Run the Agent Pipeline
Execute the pipeline with a single CLI command:

```bash
agents run --config agents.yaml --input "https://example.com/report.pdf"
```

The engine parses the DAG, initializes each agent, and streams logs to stdout. Use `--watch` to open a live dashboard in your terminal.

### 3. Inspect and Debug with Telemetry
After execution, view the full trace:

```bash
agents trace --run-id <run_id>
```

This outputs a JSON tree of every agent call, tool invocation, token usage, and latency. For deep debugging, export to OpenTelemetry-compatible backends like Jaeger or Datadog.

## Best Practices

### 1. **Isolate Agent Memory by Context**
For multi-turn conversations or long-running tasks, use **persistent memory** only for agents that need to retain state (e.g., summarizer, decision-maker). Use **ephemeral memory** for stateless agents (e.g., data extractor, validator). This reduces memory bloat and improves cache hit rates.

### 2. **Use Tool Sandboxing for Third-Party Integrations**
When injecting external APIs (e.g., Slack, Stripe, or a custom database connector), always set explicit resource limits and timeouts. For example, limit an HTTP tool to 5 seconds and 10 MB response size. This prevents a single misbehaving agent from crashing the entire pipeline.
