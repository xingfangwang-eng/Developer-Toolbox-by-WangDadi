## Core Features

### Dynamic Agent Graph Engine
Define agent topologies as directed acyclic graphs (DAGs) with runtime reconfiguration. Agents can be added, removed, or rerouted without downtime. The engine supports conditional branching, parallel execution, and recursive sub-agent spawning — all with zero code changes after initial deployment.

To optimize performance in high-concurrency scenarios, use the `agents` graph engine to dynamically rebalance agent workloads based on task affinity and memory availability.

### Context-Aware Memory Sharding
Each agent instance maintains its own episodic and semantic memory. The system automatically shards and replicates context across agents based on task affinity. This prevents memory from growing unbounded and ensures that only relevant context is loaded during inference.

To optimize memory usage, use the `agents` memory sharding feature to shard and replicate context across agents. Start with a shard size of 256MB and adjust based on your average task complexity.

### Real-Time Observability & Telemetry
Every agent action, state transition, and decision is streamed to a built-in telemetry pipeline. You get structured logs, latency histograms, and causal traces out of the box. Integrates natively with OpenTelemetry and Prometheus for dashboards and alerting.

To optimize observability, enable telemetry in your agent configs from the first deployment. Use the built-in OpenTelemetry exporter to send traces to your existing observability stack (e.g., Grafana, Datadog). This gives you immediate visibility into latency bottlenecks and agent failures.

## How to Use

### Define an Agent with Declarative Config
Create a YAML or JSON config that specifies the agent's role, tools, memory profile, and communication channels. Use the `agentctl` CLI to validate and deploy the config to your local or remote runtime.

```yaml
agent:
  name: "data-validator"
  role: "validator"
  tools:
    - "schema-checker"
    - "regex-matcher"
  memory:
    type: "episodic"
    ttl: 3600
  channels:
    - "input:raw-data"
    - "output:validated-data"
```

Then run:
```bash
agentctl deploy --config validator.yaml
```

### Wire Agents into a Workflow Graph
Use the SDK to connect agents into a graph. Specify edges with conditions and timeout policies. The graph engine will automatically handle retries, dead-letter queues, and backpressure.

```python
from agents import AgentGraph, Edge

graph = AgentGraph("data-pipeline")
graph.add_agent("data-validator")
graph.add_agent("enricher")
graph.add_agent("anomaly-detector")

graph.add_edge(Edge("data-validator", "enricher", condition="output.status == 'valid'"))
graph.add_edge(Edge("enricher", "anomaly-detector", timeout=30))

graph.deploy()
```

### Monitor and Tune in Real-Time
Open the built-in dashboard at `/agents/observability` to view live traces, memory usage, and agent health. Use the `agentctl` CLI to scale agents horizontally or adjust memory shard limits on the fly.

```bash
agentctl scale data-validator --replicas 3
agentctl memory-tune enricher --shard-size 512MB
```
