# agents - AGE_005

## 概要

本プロジェクトは、**AIエージェントの設計・デプロイ・管理を一元化するSaaSプラットフォーム**です。マルチエージェントシステムのオーケストレーション、動的なタスク分解、およびリアルタイムな状態監視を実現し、エンタープライズグレードのAIワークフローを構築します。

## 詳細な内容

### agentsとは？

`agents` は、**AIエージェントのライフサイクル全体をコードベースで管理**するためのプロフェッショナルツールです。従来の単一エージェントの限界を超え、複数の知的エージェントが協調して複雑なビジネスロジックを実行します。特に、**動的タスクグラフ**と**コンテキストアウェアなメモリ管理**を特徴とし、大規模なエンタープライズ向けのAIシステム構築を支援します。

### コア機能

1.  **動的タスクグラフエンジン**
    - 各エージェントがタスクを自律的に分解し、依存関係を考慮した有向非巡回グラフ（DAG）をリアルタイムで生成します。これにより、予測不能なワークフローにも柔軟に対応し、**実行パスの最適化と障害耐性**を自動で実現します。

2.  **コンテキストアウェアな長期記憶プール**
    - エージェント間で共有可能なベクトルデータベースを内蔵。**過去の対話履歴や外部データソースを文脈として参照**し、エージェントの応答精度を向上させます。また、**忘却メカニズム**を実装し、メモリ使用量を制御しながら重要な情報のみを保持します。

3.  **マルチエージェントオーケストレーター**
    - 複数のエージェントを**階層的・並列的に制御**するオーケストレーターが搭載されています。各エージェントの役割（例：リサーチャー、コーダー、レビュアー）を定義し、**メッセージパッシングと同期ポイント**を設定することで、複雑な協調タスクを効率的に処理します。

### 使い方

1.  **エージェント定義とタスクグラフの初期化**
    - YAML形式でエージェントの役割、使用するLLMモデル、アクセス権限を定義します。次に、`agents init` コマンドを実行し、タスクグラフの雛形を生成。この段階で、**エージェント間の依存関係と実行順序**を宣言的に記述します。

    ```yaml
    # agents.yml
    agents:
      - name: researcher
        model: gpt-4
        tools: [web_search, doc_parser]
      - name: coder
        model: claude-3
        tools: [code_executor, git_api]
    tasks:
      - id: research_phase
        agent: researcher
        next: coding_phase
      - id: coding_phase
        agent: coder
    ```

2.  **動的タスク分解と実行のトリガー**
    - 初期化後、`agents run --task "新しい機能を実装"` を実行。オーケストレーターが入力タスクを解析し、**自動的にサブタスクへ分解**。各エージェントが並行または順次実行を開始します。この際、**リアルタイムでタスクグラフが更新**され、進捗状況がコンソールに表示されます。

3.  **メモリプールの検索とコンテキスト注入**
    - 実行中に `agents query --context "過去の類似タスク"` を実行。ベクトルデータベースから関連する過去の実行ログや外部データを検索し、**エージェントのプロンプトに自動注入**。これにより、**コンテキストの一貫性を保ちながら**タスクを遂行します。必要に応じて、`agents memory prune --age 7d` で古いメモリを削除し、パフォーマンスを維持します。

### ベストプラクティス

-   **エージェントの役割を明確に分離する**
    - 各エージェントに単一の責任範囲（例：データ収集、コード生成、テスト実行）を割り当て、**ツールのアクセス権限を最小限に制限**します。これにより、エージェント間の競合を防ぎ、デバッグが容易になります。また、**タスクグラフ内でのエラー伝播を局所化**できるため、障害発生時の影響範囲を最小限に抑えられます。

-   **メモリプールの定期的なメンテナンスを自動化する**
    - 長期記憶プールは、**定期的な再インデックスと古いエントリの削除**が重要です。CI/CDパイプラインに `agents memory optimize` コマンドを組み込み、**週次でメモリの圧縮とベクトル再計算**を実行することを推奨します。これにより、検索精度の低下を防ぎ、応答時間を安定させます。
