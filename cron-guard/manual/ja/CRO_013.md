# cron-guard - CRO_013

## 概要

Cron Guard は、cronジョブの実行漏れ・遅延・異常終了をリアルタイムで検知し、通知・自動リカバリまでを統合管理する高信頼性ジョブ監視ツールです。スケジュールタスクの運用負荷を劇的に削減し、SLA遵守を強力に支援します。

## 詳細な内容

### Cron Guard とは？

Cron Guard は、分散環境で動作する数千のcronジョブを一元的に可視化し、実行結果の検証・アラート・自動復旧を実現するプロフェッショナルツールです。従来のログ監視では捉えきれなかった「ジョブの未実行」や「実行時間の異常な変動」を、独自のヒューリスティックエンジンで検出します。

### コア機能

- **異常実行パターン検出エンジン**  
  過去の実行履歴から機械学習を用いてジョブごとの正常な実行時間・終了コード・出力サイズのベースラインを自動生成。閾値超過だけでなく、季節変動やトレンド変化にも適応し、ノイズの少ないアラートを実現。

- **分散ジョブ依存関係グラフ可視化**  
  cronジョブ間の依存関係をDAG（有向非巡回グラフ）として自動抽出。ジョブの実行順序違反や循環依存を検出し、障害時の影響範囲を即座に特定。複数サーバーにまたがるジョブチェーンも一画面で把握可能。

- **自己修復型リトライ＆フォールバック機構**  
  ジョブ失敗時に、設定可能な指数バックオフリトライを実行。さらに、代替スクリプトや別サーバーへのフェイルオーバーを自動トリガー。障害発生から復旧までの平均時間（MTTR）を最大80%短縮。

### 使い方

1. **ジョブ定義ファイルの作成とデプロイ**  
   `cron-guard.yml` に監視対象ジョブのcron式、実行コマンド、期待される終了コード、タイムアウト値をYAML形式で記述。  
   例：
   ```yaml
   jobs:
     - name: daily_backup
       schedule: "0 2 * * *"
       command: "/usr/local/bin/backup.sh"
       expected_exit_codes: [0, 1]
       timeout_sec: 3600
       retry_policy:
         max_attempts: 3
         backoff_factor: 2.0
   ```
   作成後、`cron-guard deploy --config cron-guard.yml` で全対象サーバーに一括デプロイ。

2. **ダッシュボードによるリアルタイムモニタリング**  
   ブラウザで `https://<cron-guard-host>:8443` にアクセス。各ジョブの実行状況がカラーマッピング（緑:成功、黄:実行中、赤:異常、灰:未実行）で表示。フィルター機能で「過去1時間に失敗したジョブのみ表示」などの条件指定が可能。異常ジョブをクリックすると、実行ログ・スタックトレース・環境変数のスナップショットが即座に表示される。

3. **カスタムアラートルールの設定と通知テスト**  
   「設定 > アラートルール」から、複数条件の組み合わせアラートを作成。  
   例：`ジョブAが2回連続失敗 AND ジョブBの実行時間が平均+3σ超過` の場合にのみ、PagerDutyのCriticalアラートを発報。  
   `cron-guard alert test --rule daily_backup_failure` で実際の通知をシミュレーションし、Slack・メール・Webhookの到達確認を実施。

### ベストプラクティス

- **ジョブの実行時間に基づいた動的タイムアウト設定**  
  固定タイムアウトではなく、過去7日間の実行時間の中央値 + 3×標準偏差を自動計算させる。これにより、月末処理やブラックフライデーなど負荷変動の大きいジョブでも誤検知を防止。`cron-guard config tune --adaptive-timeout` で有効化。

- **カナリアデプロイによるジョブ定義の段階的適用**  
  新規ジョブ定義や変更は、まずステージング環境の10%のサーバーに適用し、24時間の実行結果を検証。異常率が0.1%未満の場合のみ本番全サーバーにロールアウト。`cron-guard rollout --canary 10% --watch 24h` で自動化。
