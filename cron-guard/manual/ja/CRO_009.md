# cron-guard - CRO_009

## 概要

Cron Guard は、cron ジョブの実行漏れや異常終了をリアルタイムで検知し、通知までを自動化するプロフェッショナル向けジョブ監視ツールです。本番環境の定期タスクが「気づいたら止まっていた」を防止します。

## 詳細な内容

### Cron Guard とは

Cron Guard は、従来の死活監視とは異なり、ジョブの「実行有無」と「終了ステータス」をチェックポイント方式で検証します。cron デーモンが落ちた場合、ジョブがハングした場合、あるいは予期せぬタイミングで多重起動した場合でも、即座にアラートを発報します。

### コア機能

1. **チェックポイントベースのジョブ生存証明**
   各ジョブに一意のチェックポイントトークンを発行。ジョブが正常に完了した場合のみトークンが更新されるため、実行スキップや途中停止を確実に検出。ジョブ側に一行の curl を追加するだけで統合可能。

2. **多重起動防止ロック機構（排他制御）**
   同一ジョブが前回の実行中に再起動されるのを防ぐ。ロックは Redis またはファイルベースで保持され、TTL による強制解除も可能。cron の `flock` ではカバーできない分散環境での排他を実現。

3. **ジョブ実行履歴のトレーサビリティダッシュボード**
   各ジョブの実行時間、終了コード、標準出力/エラー出力、チェックポイント到達時刻を時系列で可視化。異常時は該当ジョブの直近 5 回の実行ログをワンクリックで参照可能。

### 使い方

1. **ジョブ登録とチェックポイント設定**
   ```bash
   cron-guard register --job-name "daily_backup" --schedule "0 3 * * *" --timeout 600
   ```
   登録後、ジョブスクリプトの最終行に以下を追加：
   ```bash
   curl -X POST https://your-cron-guard-instance/checkin/daily_backup
   ```
   このリクエストが一定時間届かない場合、Cron Guard がアラートを発報。

2. **アラートルールのカスタマイズ**
   ```yaml
   # alert-rules.yml
   rules:
     - job: daily_backup
       miss_threshold: 2  # 2回連続でチェックインがない場合
       notify:
         - type: slack
           webhook: https://hooks.slack.com/services/xxx
         - type: email
           to: ops@example.com
   ```
   `cron-guard apply -f alert-rules.yml` で即時反映。

3. **障害発生時の自動再実行ポリシー設定**
   ```bash
   cron-guard policy set --job "daily_backup" --auto-retry 3 --retry-interval 60
   ```
   チェックインがタイムアウトした場合、最大 3 回、60 秒間隔でジョブを再実行。全て失敗した場合のみエスカレーション。

### ベストプラクティス

- **ジョブの粒度は「単一責務」で分割する**
  一つの cron ジョブで複数の処理を直列に実行すると、一部の処理が失敗した場合に全体がリトライされるリスクがある。Cron Guard ではジョブ単位でチェックポイントを管理するため、処理単位でジョブを分割し、それぞれにチェックポイントを設定することを推奨。

- **通知先は Slack と PagerDuty の二段構え**
  軽微な遅延は Slack 通知、連続失敗やジョブ消失は PagerDuty にエスカレーションするルールを組む。Cron Guard のアラートルールでは `severity` フィールドを指定できるため、`critical` 時のみ PagerDuty にルーティングすることで、運用負荷を軽減できる。
