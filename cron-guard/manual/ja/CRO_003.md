# cron-guard - CRO_003

## 概要

Cron Guard は、cron ジョブの実行監視・障害検知に特化したプロフェッショナルツールです。スケジュールタスクの未実行、遅延、異常終了をリアルタイムで検出し、即座にアラートを発報します。本ツールは cron ジョブの可視性を最大化し、運用チームの負荷を大幅に軽減します。

## 詳細な内容

### とは Cron Guard?

Cron Guard は、cron ジョブの実行状態をエンドツーエンドで監視する SaaS 型ソリューションです。従来のログベース監視では捉えきれなかった「ジョブが実行されなかった」というサイレント障害を、独自のハートビート機構とジョブスケジュール予測エンジンにより確実に検出します。また、ジョブの実行時間やリソース消費量のトレンド分析にも対応し、パフォーマンス劣化の予兆を早期にキャッチします。

### コア機能

- **動的スケジュール予測エンジン**: cron 式を解析し、次回実行予定時刻をミリ秒単位で予測。予測時刻を過ぎてもジョブが起動しない場合、即座にアラートを発報。ジョブのスケジュール変更やタイムゾーン変更にも自動追従します。
- **マルチノード分散監視アーキテクチャ**: 複数の監視ノードがジョブの実行状態を冗長監視。ノード障害時も監視が継続され、単一障害点を排除。ジョブの実行履歴は分散データベースにリアルタイム同期され、履歴の完全性を保証します。
- **ジョブ依存関係グラフ可視化**: 複数の cron ジョブ間の依存関係を自動解析し、グラフ形式で可視化。依存チェーンが切れた場合、影響範囲を特定し、関連ジョブの実行を自動ブロックまたは遅延実行します。

### 使い方

1. **監視対象ジョブの登録とハートビート設定**  
   Cron Guard の管理画面で「ジョブ登録」をクリックし、ジョブ名、cron 式、実行ホストを入力します。次に「ハートビート設定」で、ジョブの予想実行時間と許容遅延時間を秒単位で指定します。例えば、毎時0分に実行されるジョブで、通常5秒以内に完了する場合、予想実行時間を5秒、許容遅延を10秒に設定します。登録後、ジョブの実行開始時に `cron-guard-agent` がハートビート信号を送信し、Cron Guard が受信を確認します。

2. **アラートルールのカスタマイズと通知チャネル設定**  
   「アラート設定」メニューで、通知条件を定義します。例えば「ジョブが3回連続で遅延した場合」「ジョブの実行時間が過去30日の平均値の2倍を超えた場合」など、閾値ベースのルールを設定可能です。通知チャネルは Slack、PagerDuty、メール、Webhook から選択し、各チャネルに送信するメッセージテンプレートもカスタマイズできます。設定後、テスト通知を送信して接続を確認します。

3. **ジョブ実行履歴の分析とパフォーマンスレポート生成**  
   「レポート」タブで、任意の期間のジョブ実行履歴を CSV または JSON 形式でエクスポートします。グラフビューでは、実行時間の分布、失敗率の推移、リソース消費量のトレンドを確認できます。例えば、特定のジョブの実行時間が週末に急増している場合、その期間のシステム負荷と相関を分析します。レポートは自動生成スケジュールを設定し、毎週月曜日にチームメンバーにメール配信することも可能です。

### ベストプラクティス

- **ジョブの実行時間に基づいた許容遅延の動的最適化**  
  ジョブの実行時間は日々変動します。Cron Guard の「自動調整モード」を有効にすると、過去7日間の実行時間の標準偏差を計算し、許容遅延を動的に調整します。これにより、一時的な負荷増加による誤検知を減らしつつ、異常な遅延には迅速に対応できます。例えば、標準偏差が2秒以下のジョブには固定値、3秒以上のジョブには統計ベースの値を適用します。

- **障害発生時の自動リカバリフローの構築**  
  ジョブ障害検出時、Cron Guard の Webhook 機能を利用して自動リカバリフローをトリガーします。例えば、ジョブが3回連続で失敗した場合、Webhook 経由で Ansible ジョブを起動し、該当ホストのサービス再起動とジョブの再実行を行います。この際、Cron Guard は再実行の結果を監視し、成功を確認するまでアラートを抑制します。これにより、手動介入なしに障害を自動修復できます。
