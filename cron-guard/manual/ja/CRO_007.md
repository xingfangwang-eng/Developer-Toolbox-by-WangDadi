# cron-guard - CRO_007

## 概要

Cron Guard は、cron ジョブの実行状態をリアルタイムで監視し、障害発生時に即座にアラートを発報するプロフェッショナル向けジョブ監視ツールです。バックグラウンドタスクの健全性を担保し、運用コストを最小化します。

## 詳細な内容

### Cron Guard とは？

Cron Guard は、cron ジョブの実行履歴、タイムアウト検知、異常終了時の自動リトライを統合管理する SaaS 型監視プラットフォームです。API によるジョブ登録と Webhook によるアラート通知を標準装備し、複数サーバー・複数環境にまたがるジョブを一元管理できます。

### コア機能

1. **ジョブ実行のリアルタイムトレース**  
   各ジョブの開始時刻、終了時刻、終了コード、標準出力/エラー出力をタイムスタンプ付きで記録。異常が発生したジョブは即座にダッシュボード上でハイライト表示されます。

2. **タイムアウト検知と自動リトライ**  
   ジョブごとに許容実行時間を設定可能。設定時間を超過したジョブは強制終了し、指定回数まで自動リトライを実行。リトライ履歴も全て保存され、原因分析に活用できます。

3. **複数環境対応の一元管理インターフェース**  
   開発、ステージング、本番など複数の環境を一つのプロジェクトにまとめて管理。環境ごとに通知先やリトライポリシーを個別設定できます。API を通じて任意の CI/CD パイプラインと連携可能。

### 使い方

1. **ジョブの登録と監視設定**  
   `POST /api/jobs` エンドポイントにジョブ名、cron 式、実行コマンド、タイムアウト秒数、リトライ回数を JSON 形式で送信します。レスポンスとして発行されるジョブ ID を控えてください。  
   例:
   ```bash
   curl -X POST https://cron-guard.example.com/api/jobs \
     -H "Content-Type: application/json" \
     -d '{"name": "daily_backup", "schedule": "0 2 * * *", "command": "/usr/local/bin/backup.sh", "timeout": 300, "retry_count": 3}'
   ```

2. **Webhook 通知の設定**  
   ダッシュボードの「通知設定」から、Slack、メール、またはカスタム Webhook URL を登録します。通知条件は「ジョブ失敗時」「タイムアウト時」「リトライ上限到達時」から選択可能。複数の通知チャネルを同時に設定できます。

3. **ジョブ実行履歴の確認と障害対応**  
   ダッシュボードの「ジョブ履歴」タブで、全ジョブの実行結果をリスト表示。各ジョブをクリックすると、実行開始時刻、終了時刻、終了コード、標準出力/エラー出力、リトライ履歴が時系列で確認できます。異常ジョブは赤色でハイライトされ、即座に原因特定が可能です。

### ベストプラクティス

- **ジョブごとに明確なタイムアウト値を設定する**  
  ジョブが通常完了する時間の 1.5 倍をタイムアウト値として設定することを推奨します。これにより、予期せぬ遅延やハングアップを早期に検知し、システム全体への影響を最小限に抑えられます。

- **リトライポリシーは指数バックオフを採用する**  
  リトライ間隔を固定値ではなく、1回目: 30秒、2回目: 60秒、3回目: 120秒と指数関数的に増加させることで、一時的なリソース競合やネットワーク瞬断に対して効果的に対処できます。Cron Guard のカスタムリトライ設定で実現可能です。
