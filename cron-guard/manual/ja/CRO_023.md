# cron-guard - CRO_023

## 概要

本ツールは、cronジョブの実行管理と監視を自動化するためのプロフェッショナル向けソリューションです。バッチ処理、定期タスクの遅延検知、障害通知を一元管理し、運用負荷を最小限に抑えます。

## 詳細な内容

### Cron Guardとは

Cron Guardは、cronジョブの実行状態をリアルタイムで監視し、異常が発生した場合に即座に通知を行う軽量な監視ツールです。従来のcron監視では見逃されがちな「ジョブのスキップ」「実行時間の異常な増加」「リソースリーク」などを検出し、運用チームの手動チェックを不要にします。特に、複数サーバーに分散したジョブ群を一元的に管理できる点が特徴です。

### コア機能

1. **ジョブ依存関係グラフの自動可視化**  
   各cronジョブの実行順序と依存関係を自動で解析し、DAG（有向非巡回グラフ）として可視化します。ジョブの実行順序が崩れた場合、依存先ジョブの開始を自動的にブロックし、チェーンエラーを防止します。

2. **実行時間のベースライン学習と異常検知**  
   過去の実行履歴からジョブごとの標準実行時間を学習し、現在の実行時間が平均値から±3σを超えた場合にアラートを発報します。これにより、リソース不足やクエリの劣化を早期に検出できます。

3. **ジョブスキップ検出と自動リトライ機構**  
   cronデーモン自体がダウンした場合や、ジョブが何らかの理由でスキップされた場合、次回の実行スケジュールを自動で再計算し、未実行ジョブを補完実行します。リトライ上限を設定可能で、上限到達時にはエスカレーション通知を行います。

### 使い方

1. **監視対象ジョブの登録と設定**  
   監視対象のcronジョブを登録するには、`cron-guard register` コマンドを使用します。ジョブ名、実行コマンド、cron式、タイムアウト値（秒）を指定します。  
   ```bash
   cron-guard register --name "daily_backup" --schedule "0 3 * * *" --command "/usr/local/bin/backup.sh" --timeout 1800
   ```  
   登録後、ジョブのUUIDが発行され、監視対象として有効になります。

2. **ダッシュボードでのリアルタイム監視**  
   ブラウザでダッシュボード（デフォルト: `http://localhost:9090`）にアクセスし、登録済みジョブの実行状況を確認します。各ジョブの直近の実行履歴、実行時間、ステータス（成功/失敗/スキップ）がリアルタイムで表示されます。フィルター機能を使い、障害が発生したジョブのみを表示することも可能です。

3. **アラートルールのカスタマイズと通知設定**  
   アラートの条件を細かく設定するには、`cron-guard alert-rule` コマンドを使用します。例えば、ジョブが2回連続で失敗した場合にSlackへ通知するルールは以下の通りです。  
   ```bash
   cron-guard alert-rule --job-uuid "abc123" --condition "failure_count >= 2" --action "slack" --webhook "https://hooks.slack.com/services/xxx"
   ```  
   通知先はSlack、メール、PagerDuty、カスタムWebhookから選択可能です。

### ベストプラクティス

- **ジョブの実行時間に基づくスロットリングの導入**  
  複数のcronジョブが同時に実行されると、リソース競合が発生し、処理遅延や障害の原因となります。Cron Guardの「実行時間のベースライン学習」機能を活用し、各ジョブの平均実行時間を把握した上で、実行スケジュールにバッファ（例: 平均実行時間の1.5倍）を設けることを推奨します。これにより、リソース競合を未然に防ぎ、ジョブの安定稼働を実現できます。

- **ジョブの依存関係を明示的に定義する**  
  ジョブ間に依存関係がある場合、Cron GuardのDAG機能を使って明示的に定義してください。例えば、データベースのバックアップジョブが完了した後にのみ、レポート生成ジョブを実行するように設定します。依存関係を定義しないと、バックアップが未完了の状態でレポート生成が実行され、データ不整合が発生するリスクがあります。依存関係は `cron-guard dependency --from "backup_job" --to "report_job"` のように設定します。
