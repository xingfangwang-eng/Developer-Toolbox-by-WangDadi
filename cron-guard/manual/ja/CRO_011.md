# cron-guard - CRO_011

## 概要

Cron Guard は、cron ジョブや定期実行タスクの監視に特化したプロフェッショナルツールです。ジョブの失敗や遅延をリアルタイムで検知し、システムの安定稼働を支援します。

## 詳細な内容

### Cron Guard とは

Cron Guard は、分散環境における cron ジョブの実行状態を一元管理し、障害発生時に即座にアラートを発する監視ツールです。特に、クラウドネイティブなワークロードやマイクロサービス構成での利用を想定して設計されており、ジョブのスケジュール管理、実行履歴の可視化、アラート通知のカスタマイズが可能です。

### コア機能

1. **ジョブチェーン依存解決エンジン**  
   複数の cron ジョブが依存関係を持つ場合、自動的に実行順序を解決し、先行ジョブの失敗が後続ジョブに影響を与えないよう、キューイングとリトライポリシーを動的に適用します。

2. **分散ノード間の実行ログ集約と異常検知**  
   各ノードで実行されたジョブのログを中央リポジトリに集約し、機械学習ベースの異常検知モデルにより、通常の実行時間から逸脱したジョブや、エラーログのパターンを自動的にフラグ付けします。

3. **Slack/メール/Webhook へのマルチチャネルアラート**  
   ジョブの失敗やタイムアウト時に、設定したチャネル（Slack、メール、カスタム Webhook）に即座に通知を送信。アラートの内容は、ジョブ名、実行ノード、エラーメッセージ、リトライ回数を含む詳細情報をJSON形式で出力可能です。

### 使い方

1. **ジョブ定義ファイルの作成と登録**  
   `cron-guard.yml` にジョブ名、スケジュール（cron 式）、実行コマンド、タイムアウト秒数を記述し、`cron-guard register` コマンドで監視対象として登録します。依存関係がある場合は `depends_on` フィールドに先行ジョブ名を指定します。

   ```yaml
   jobs:
     - name: data_pipeline
       schedule: "0 3 * * *"
       command: "/usr/bin/python3 /opt/scripts/pipeline.py"
       timeout: 300
       depends_on: []
     - name: report_generation
       schedule: "0 4 * * *"
       command: "/usr/bin/python3 /opt/scripts/report.py"
       timeout: 600
       depends_on: ["data_pipeline"]
   ```

2. **ジョブ実行履歴の確認とアラート設定**  
   `cron-guard history --job data_pipeline` で直近の実行結果を表示。`cron-guard alert set --channel slack --webhook https://hooks.slack.com/services/xxx` でアラート通知先を設定します。アラートはジョブ失敗時、タイムアウト時、および再試行回数超過時に発火します。

3. **カスタムアラートルールの追加**  
   `cron-guard alert rule add --name "high_latency" --condition "execution_time > 240" --action "notify_slack"` のように、実行時間が閾値を超えた場合に通知するルールを追加可能。ルールは複数定義でき、AND/OR 条件で組み合わせることもサポートします。

### ベストプラクティス

- **ジョブの冪等性を確保する**  
  Cron Guard はリトライ機能を備えていますが、リトライ時にジョブが冪等でないと、データの重複や不整合が発生します。ジョブ内でチェックポイントを設け、再実行時に安全にスキップできる設計を推奨します。

- **アラートの閾値を段階的に設定する**  
  初回は警告（Warning）と重大（Critical）の2段階で設定し、運用データをもとに調整します。例えば、通常5分で完了するジョブは、7分で警告、10分でアラートを発するように設定することで、ノイズを減らしつつ重要な障害を見逃しません。
