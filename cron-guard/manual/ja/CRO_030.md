# cron-guard - CRO_030

## 概要

Cron Guard は、cron ジョブやスケジュールタスクの実行状態をリアルタイムで監視し、障害発生時に即座に通知するプロフェッショナル向けツールです。ジョブの遅延、未実行、異常終了を自動検出し、システムの安定稼働を支援します。

## 詳細な内容

### Cron Guard とは

Cron Guard は、cron ジョブの実行ログを収集・分析し、異常を検出した場合に Slack、メール、または Webhook 経由でアラートを送信します。複数のサーバーや環境にまたがるジョブを一元管理できるため、運用負荷を大幅に削減します。

### コア機能

- **ジョブ実行のリアルタイム可視化**: 各ジョブの開始時刻、終了時刻、実行時間、終了コードをダッシュボードで可視化。遅延や未実行を即座に把握できます。
- **カスタムアラートルール**: ジョブごとに閾値（例: 実行時間が通常の2倍を超えた場合）を設定し、条件に応じて通知先を動的に切り替えられます。
- **過去データの統計分析**: 実行履歴を自動集計し、週次・月次のレポートを生成。ジョブのパフォーマンス傾向を解析し、ボトルネックを特定します。

### 使い方

1. **ジョブの登録と監視設定**  
   `cron-guard register --job-name "data_sync" --schedule "0 */6 * * *" --timeout 300` を実行して監視対象ジョブを登録します。タイムアウトやリトライ回数を指定可能です。

2. **アラートルールのカスタマイズ**  
   `cron-guard rule add --job "data_sync" --condition "execution_time > 180" --action "slack:#ops"` で、実行時間が180秒を超えた場合に Slack に通知するルールを設定します。

3. **ダッシュボードでの状態確認**  
   `cron-guard dashboard` を実行し、全ジョブのステータスを一覧表示。異常ジョブは赤色でハイライトされ、クリックで詳細ログを確認できます。

### ベストプラクティス

- **ジョブの依存関係を明示する**: 複数のジョブが連鎖する場合、`cron-guard dependency add --job "job_a" --depends "job_b"` で依存関係を定義。依存ジョブの失敗を自動検出し、後続ジョブの実行を抑制します。
- **定期的なジョブ実行時間のレビュー**: 月に一度、`cron-guard report --period monthly` でレポートを生成。実行時間が増加傾向にあるジョブを特定し、クエリ最適化やリソース増強を検討します。
