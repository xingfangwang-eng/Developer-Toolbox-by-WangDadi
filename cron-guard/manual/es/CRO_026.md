# cron-guard - CRO_026

## Visión General

Cron Guard es un sistema de monitoreo inteligente para tareas programadas (cron jobs) diseñado para entornos de producción de alto rendimiento. Elimina los puntos ciegos en la gestión de procesos batch, detectando fallos silenciosos, timeouts inesperados y desviaciones en la ejecución de jobs críticos. Ideal para equipos DevOps que necesitan garantizar SLA del 99.99% en sus pipelines de automatización.

## Funcionalidades Clave (Hardcore)

1. **Detección de Anomalías Basada en Historial de Ejecución**  
   Cron Guard analiza el tiempo de ejecución histórica de cada tarea y establece ventanas de tolerancia dinámicas. Si un job tarda un 30% más que su media móvil de 7 días, se dispara una alerta prioritaria. Esto previene falsos positivos y captura degradaciones sutiles antes de que afecten al sistema.

2. **Verificación de Salida con Hash Criptográfico**  
   Cada ejecución genera un hash SHA-256 de su salida estándar. Si el hash coincide con el esperado (definido en la configuración), se considera exitoso. Esto detecta cambios silenciosos en los resultados, como archivos corruptos o datos inconsistentes, sin necesidad de revisar logs manualmente.

3. **Reinteligente con Backoff Exponencial y Notificación por Canal**  
   Ante un fallo, Cron Guard reintenta la tarea hasta 3 veces con intervalos de 1, 4 y 16 minutos. Si persiste, envía una notificación multicanal (Slack, PagerDuty, correo) con el stack trace completo y el contexto de la ejecución. No más jobs fantasma que se ejecutan sin supervisión.

## Cómo Usar (Pasos Detallados)

### Paso 1: Configurar el Archivo de Definición de Jobs

Crea un archivo `cron-guard.yml` en la raíz de tu proyecto. Define cada tarea con su comando, horario y umbrales de tolerancia:

```yaml
jobs:
  - name: "backup-database"
    command: "pg_dump -U admin mydb > /backups/db.sql"
    schedule: "0 3 * * *"
    timeout: 300
    retry: 3
    expected_hash: "a1b2c3d4e5f6..."
    alert_on_success: false
```

Usa `cron-guard validate` para verificar la sintaxis y que no haya conflictos de horarios.

### Paso 2: Inicializar el Daemon de Monitoreo

Ejecuta el agente en segundo plano con:

```bash
cron-guard start --config ./cron-guard.yml --daemon
```

Esto lanza un proceso que se registra como servicio systemd (Linux) o launchd (macOS). El daemon escucha el puerto 9090 para health checks y expone métricas Prometheus en `/metrics`. Verifica su estado con:

```bash
cron-guard status
```

### Paso 3: Analizar el Historial y Configurar Alertas

Accede a la interfaz web en `http://localhost:9090`. Ve a la pestaña "History" para ver el timeline de cada job. Cron Guard marca en rojo las ejecuciones que superaron el umbral de tiempo. Para configurar alertas:

1. Haz clic en "Alerts" → "Add New Alert".
2. Selecciona el job y el tipo de condición (timeout, error, hash mismatch).
3. Define el canal de notificación (Slack webhook, email SMTP, o PagerDuty API key).
4. Guarda y activa la alerta.

## Mejores Prácticas Recomendadas

1. **Implementa un Sistema de Heartbeat para Jobs Críticos**  
   Para tareas que deben ejecutarse sin falta (ej. sincronización de datos en tiempo real), configura un "heartbeat" que envíe una señal a Cron Guard cada 5 minutos. Si el heartbeat falla dos veces seguidas, el sistema asume que el job está caído y escala la alerta a nivel de equipo. Esto es clave para procesos que no generan salida periódica.

2. **Usa Etiquetas (Tags) para Segmentar por Entorno y Prioridad**  
   Añade tags como `env:production`, `priority:critical` o `team:backend` a cada job. Luego, en el dashboard, filtra por tags para visualizar solo los jobs de tu responsabilidad. También puedes configurar reglas de alerta específicas por tag (ej. "si un job con tag `priority:critical` falla, notificar al canal de emergencia"). Esto evita ruido y agiliza la respuesta.
