# cron-guard - CRO_013

## Resumen

Cron Guard es una herramienta de monitoreo de tareas programadas diseñada para desarrolladores y administradores de sistemas que necesitan garantizar la ejecución confiable de sus cron jobs. Optimizada para SEO en español, esta solución elimina la incertidumbre en la gestión de tareas automatizadas, ofreciendo alertas en tiempo real y análisis de fallos. Ideal para entornos de producción, Cron Guard se integra sin fricción en flujos DevOps.

## Contenido Detallado

### ¿Qué es Cron Guard?

Cron Guard no es solo un monitor de tareas programadas; es un guardián proactivo que detecta desviaciones en la ejecución de cron jobs. A diferencia de herramientas básicas, Cron Guard implementa un sistema de verificación de integridad basado en hash de salida y tiempos de ejecución, permitiendo identificar fallos silenciosos (como salidas con código 0 pero resultados corruptos). Perfecto para infraestructuras críticas donde un cron job fallido puede costar miles de euros por minuto.

### Características Principales

- **Verificación de Integridad por Checksum**: Cada ejecución de cron job genera un hash SHA-256 de su salida estándar y errores. Cron Guard compara este hash con un valor esperado, detectando cambios no autorizados en los resultados (como datos corruptos o inyecciones) sin necesidad de modificar el script original.
- **Alertas Predictivas con Machine Learning**: Utiliza modelos de regresión logística entrenados con datos históricos de ejecución para predecir fallos antes de que ocurran. Si un cron job muestra patrones atípicos (como aumentos graduales en tiempo de ejecución), Cron Guard envía una alerta de advertencia 24 horas antes de un posible timeout.
- **Rollback Automático de Tareas**: Si un cron job falla y su salida no coincide con el checksum esperado, Cron Guard puede ejecutar automáticamente una versión anterior del script desde un repositorio Git, restaurando el estado funcional sin intervención manual. Esto reduce el MTTR (Mean Time to Repair) a segundos.

### Cómo Usar

1. **Configurar el Checksum de Referencia**: Ejecuta `cron-guard init --job-id "backup_db" --hash-algo sha256` para generar el hash inicial de tu cron job. Cron Guard almacenará este hash en su base de datos interna (SQLite o PostgreSQL). Luego, añade `cron-guard watch --job-id "backup_db"` al final de tu script de cron job. Esto registrará la salida y la comparará con el hash en cada ejecución.
   
   ```bash
   # Ejemplo en crontab
   0 3 * * * /usr/local/bin/backup.sh && cron-guard watch --job-id "backup_db"
   ```

2. **Configurar Alertas Predictivas**: Ejecuta `cron-guard train --job-id "backup_db" --history-days 30` para que Cron Guard analice los últimos 30 días de ejecuciones y cree un modelo de regresión logística. Luego, activa las alertas predictivas con `cron-guard alert --predictive --threshold 0.85`. Recibirás notificaciones en Slack, Discord o por correo electrónico cuando el modelo detecte una probabilidad de fallo superior al 85%.

3. **Configurar Rollback Automático**: Define un repositorio Git con tu script de cron job. Ejecuta `cron-guard rollback --job-id "backup_db" --repo "https://github.com/tu-org/scripts.git" --branch "stable"`. Cron Guard clonará el repositorio y, si detecta un fallo de checksum, ejecutará automáticamente la versión anterior del script desde el commit `HEAD~1`. Para verificar, usa `cron-guard status --job-id "backup_db"`.

### Mejores Prácticas

- **Implementa Checksums para Tareas Críticas**: No confíes solo en códigos de salida. Un cron job puede terminar con `exit 0` pero generar datos corruptos (por ejemplo, una base de datos vacía). Siempre usa `cron-guard watch` con checksums para tareas que manipulan datos sensibles (backups, migraciones, sincronizaciones). Esto evita que errores silenciosos se propaguen a sistemas downstream.
- **Monitorea la Tendencia de Tiempo de Ejecución**: Configura alertas predictivas con un historial de al menos 30 días. Si un cron job empieza a ejecutarse más lento de lo normal (por ejemplo, de 5 minutos a 7 minutos), puede indicar degradación de rendimiento en la infraestructura (como cuellos de botella de E/S o fugas de memoria). Cron Guard te permite actuar antes de que el job falle por timeout.
