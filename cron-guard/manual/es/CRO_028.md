# cron-guard - CRO_028

## Resumen Técnico

Cron Guard es un **sistema de monitoreo inteligente para tareas programadas** diseñado para entornos de producción críticos. Combina detección temprana de fallos, alertas contextuales y auto-recuperación para garantizar que tus cron jobs nunca fallen en silencio. Ideal para equipos DevOps, SRE y arquitectos que gestionan cientos de tareas batch diarias.

## Funcionalidades Clave (Hardcore)

### 1. **Detección de Anomalías Basada en Ventanas Temporales Adaptativas**
Cron Guard no solo verifica si un job se ejecutó, sino que analiza patrones históricos de duración y frecuencia. Si un job que normalmente tarda 2 segundos se ejecuta en 10 ms o 5 minutos, el sistema lo marca como anomalía y activa alertas en cascada (Slack, PagerDuty, webhook personalizado). Ideal para detectar deadlocks, cuellos de botella o cambios silenciosos en la infraestructura.

### 2. **Auto-Recuperación con Rollback de Estado**
Cuando un job falla, Cron Guard puede ejecutar un script de recuperación predefinido (por ejemplo, reiniciar un worker, limpiar colas o restaurar un snapshot de base de datos). Además, registra el estado previo al fallo para permitir un rollback granular sin intervención manual. Esto reduce el MTTR (Mean Time To Recovery) drásticamente.

### 3. **Auditoría de Dependencias entre Cron Jobs**
Visualiza y gestiona dependencias entre tareas programadas. Si el job A necesita que el job B haya terminado exitosamente, Cron Guard bloquea la ejecución de A hasta que B confirme éxito. Si B falla, se cancela la cadena completa y se notifica al equipo responsable. Esto evita inconsistencias de datos y errores en cascada en pipelines complejos.

## Pasos de Implementación (Nivel Experto)

### 1. **Configuración de Monitoreo con Expresiones Cron y Umbrales Dinámicos**
- Abre el panel de administración de Cron Guard y selecciona "Añadir Job".
- Introduce la expresión cron exacta (por ejemplo, `0 2 * * *` para ejecución diaria a las 2 AM).
- Define el **umbral de tiempo de ejecución esperado** (por ejemplo, 300-500 ms). Cron Guard calculará automáticamente la desviación estándar tras 10 ejecuciones exitosas.
- Activa la opción "Alertas en cadena" para recibir notificaciones en Slack, correo o webhook si el job falla o se desvía del umbral.

### 2. **Implementación de Auto-Recuperación con Scripts Personalizados**
- En la sección "Recuperación Automática", selecciona "Añadir Script de Recuperación".
- Sube un script bash o Python que realice la acción correctiva (por ejemplo, `systemctl restart cron-worker.service`).
- Define condiciones de activación: por ejemplo, si el job falla 3 veces consecutivas, ejecuta el script automáticamente.
- Configura un límite de reintentos (máximo 5) para evitar bucles infinitos. Cron Guard registrará cada intento en el log de auditoría.

### 3. **Configuración de Dependencias entre Jobs**
- Ve a "Gestión de Dependencias" y selecciona "Crear Cadena".
- Arrastra los jobs al canvas y conecta las dependencias: por ejemplo, `Job_A → Job_B → Job_C`.
- Define reglas de fallo: si `Job_B` falla, ¿se cancela `Job_C` o se reintenta automáticamente?
- Activa la opción "Notificar solo en fallo crítico" para evitar spam en equipos grandes.

## Mejores Prácticas (Recomendaciones de Arquitecto)

### 1. **Usa Etiquetas Semánticas para Jobs Críticos**
Etiqueta cada job con su nivel de criticidad (`critical`, `high`, `medium`, `low`). En el panel de alertas, configura que los jobs `critical` envíen notificaciones inmediatas a PagerDuty, mientras que los `low` solo generen un ticket en Jira. Esto evita fatiga de alertas y prioriza la atención humana donde realmente importa.

### 2. **Realiza Pruebas de Caos Programadas**
Cada 30 días, ejecuta una simulación de fallo para verificar que los scripts de auto-recuperación funcionan correctamente. Cron Guard permite programar "pruebas de caos" que detienen intencionalmente un job y verifican que el sistema lo detecta y recupera en menos de 60 segundos. Documenta los resultados en el log de auditoría para cumplir con requisitos de compliance (SOC2, ISO 27001).
