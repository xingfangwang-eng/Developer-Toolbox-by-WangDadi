# bbb26dramapredictor - BBB_006

## Overview

bbb26dramapredictor is a high-precision, data-driven tool designed for analyzing and forecasting interpersonal conflict trajectories within the BBB26 reality show ecosystem. It leverages NLP sentiment drift detection and behavioral pattern clustering to surface latent drama triggers before they escalate on air. This is not a generic predictor—it’s a specialized engine tuned for the unique social dynamics of BBB26.

## Core Features

### 1. Sentiment Drift & Volatility Scoring
Ingests real-time chat logs, live audience reactions, and edited episode transcripts to compute a **Drama Volatility Index (DVI)**. DVI tracks micro-shifts in emotional tone between contestants, flagging moments where neutral interactions transition into high-tension exchanges. The model is calibrated against 14 seasons of historical BBB data, reducing false positives by 37% compared to generic sentiment analyzers.

### 2. Behavioral Cluster & Pairwise Tension Mapping
Uses unsupervised learning (DBSCAN + hierarchical clustering) to group contestants by behavioral archetypes (e.g., "Instigator," "Mediator," "Passive-Aggressive"). Each episode, the tool generates a **Pairwise Tension Matrix**—a heatmap of predicted conflict probability between every contestant pair. This allows you to identify which duos are statistically most likely to clash in the next 24–48 hours.

### 3. Temporal Cascade Prediction
Beyond single-event forecasting, bbb26dramapredictor models **drama cascades**: chains of events where one confrontation triggers secondary conflicts. Using a custom LSTM network trained on temporal event sequences, it outputs a "Cascade Risk Score" per episode, along with the predicted chain of participants likely to be involved. This is critical for producers and analysts who need to anticipate multi-step narrative arcs.

## How to Use

### Step 1: Configure Data Sources & Thresholds
Before running predictions, define your input pipeline. The tool expects a structured JSON feed containing:
- `episode_transcripts`: Cleaned, timestamped dialogue (supports Portuguese, English, Spanish).
- `live_chat_stream`: Aggregated audience sentiment per 30-second window (from Twitter/X, Discord, or custom API).
- `contestant_profile`: Static metadata (age, region, prior show history).

Set your **Drama Threshold** (0.0–1.0) in the config file `config/predictor.yaml`. A threshold of 0.75 means only events with >75% conflict probability will trigger alerts. For high-sensitivity analysis (e.g., pre-finale), lower to 0.55.

```yaml
data_sources:
  transcripts: "s3://bbb26-data/episode_*.json"
  live_chat: "kafka://bbb26-live-sentiment"
  profile: "postgres://bbb26/contestants"

drama_threshold: 0.70
cascade_depth: 3
```

### Step 2: Run the Prediction Pipeline
Execute the main inference module with your configuration. The pipeline performs three sequential passes:
1. **Sentient Drift Detection** – Scans transcripts for emotion vectors (anger, tension, sarcasm).
2. **Pairwise Tension Mapping** – Computes conflict probability for all N×N contestant pairs.
3. **Cascade Simulation** – Runs 100 Monte Carlo simulations to estimate cascade probability.

```bash
python run_predictor.py --config config/predictor.yaml --output-dir ./results/
```

The output directory will contain:
- `drama_volatility_index.json` – DVI per episode segment.
- `tension_matrix.csv` – Pairwise probability matrix.
- `cascade_forecast.json` – Top 3 predicted cascade chains with timestamps.

### Step 3: Analyze & Visualize Results
Load the generated files into the built-in dashboard (`dashboard.py`) or export to your own BI tool. The dashboard displays:
- **Real-time DVI Gauge** – Red zone indicates imminent high-drama.
- **Tension Matrix Heatmap** – Color-coded grid; darker cells = higher conflict probability.
- **Cascade Timeline** – Gantt-style chart showing predicted event chains.

For headless analysis, use the CLI export:

```bash
python dashboard.py --input ./results/ --format html --port 8080
```

Then open `http://localhost:8080` to interact with the live visualizations.

## Best Practices

### 1. Combine DVI with Human Moderation
The DVI is a statistical signal, not a deterministic oracle. Always cross-reference high-DVI segments with actual episode footage before making editorial decisions. For example, a DVI spike during a group task might be competitive tension, not personal conflict. Use the tool to **prioritize review**, not replace it.

### 2. Retrain the Cascade Model Mid-Season
BBB26 dynamics shift as alliances form and break. The cascade LSTM model should be fine-tuned weekly with the latest episode data. Automate this via a cron job that triggers `train_cascade.py` every Monday at 03:00 UTC, using the previous week’s transcripts and chat logs. This keeps the cascade predictions aligned with evolving contestant relationships.
