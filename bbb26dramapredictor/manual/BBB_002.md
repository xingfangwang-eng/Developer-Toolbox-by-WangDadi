# bbb26dramapredictor - BBB_002

**bbb26dramapredictor** is a production-grade, data-driven engine designed to analyze, predict, and visualize interpersonal drama dynamics within the BBB26 reality show ecosystem. It scrapes multi-source behavioral data (live feeds, social sentiment, editorial cues) and applies a proprietary NLP + graph-based conflict propagation model to surface high-likelihood drama arcs before they air.

> This is not a sentiment tracker. It’s a **pre-cognitive narrative engine** for reality TV analysts, fan communities, and media strategists.

---

## Hardcore Feature Set

### 1. **Conflict Cascade Prediction (CCP)**
A multi-agent simulation that models emotional contagion between contestants using a directed weighted graph. Each node represents a participant, edges encode interaction frequency + tone polarity. The engine then runs Monte Carlo rollouts to predict the **probability of a blow-up event** within the next 48 hours. Output includes the trigger agent, likely target, and escalation vector (e.g., passive-aggressive → direct confrontation).

### 2. **Drama Volatility Index (DVI)**
A real-time composite score (0–100) computed from:
- Live feed speech-to-text emotional instability (tremor, pitch variance)
- Twitter/X sentiment velocity (delta per minute)
- Editorial framing bias (camera time allocation + confessional tone)
The DVI is normalized per season baseline and emits WebSocket alerts when a threshold (e.g., DVI > 78) is breached.

### 3. **Narrative Fork Detection (NFD)**
An unsupervised clustering model that identifies **divergent storylines** from raw transcript segments. It uses sentence-BERT embeddings + DBSCAN to detect when the show’s editorial team is seeding a new conflict arc (e.g., a sudden shift in focus from a romantic subplot to a strategic betrayal). Outputs a timeline with fork points and the probability that each fork becomes the dominant narrative.

## How to Deploy & Run (Production-Ready)

### 1. **Initialize the Data Pipeline**
```bash
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor.git
cd bbb26dramapredictor
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```
Then configure your environment variables in `.env`:
- `TWITTER_BEARER_TOKEN` – for real-time tweet ingestion
- `GLOBOPLAY_API_KEY` – for live feed transcript access (Brazil region)
- `REDIS_URL` – for caching DVI calculations

### 2. **Run the Live Feed Scraper + DVI Engine**
```bash
python3 engine/scraper_live.py --source globoplay --output kafka://drama-raw
python3 engine/dvi_calculator.py --input kafka://drama-raw --output websocket://0.0.0.0:8765
```
This starts a two-stage pipeline: raw data ingestion to Kafka topics, then the DVI computation layer that pushes real-time alerts via WebSocket.

### 3. **Launch the Prediction Dashboard**
```bash
streamlit run dashboard/app.py --server.port 8501
```
The dashboard exposes:
- Live DVI gauge + CCP probability heatmap
- NFD timeline with fork visualization (D3.js)
- Exportable JSON snapshots of predicted drama arcs for downstream media automation
