# bbb26dramapredictor - BBB_015

## Overview

bbb26dramapredictor is a lightweight, event-driven analytics tool designed to forecast emotional volatility and narrative arcs inside the **Big Brother Brasil 26** house. It scrapes live Twitter/X feeds, chat logs, and closed-caption streams, then runs a custom NLP pipeline tuned specifically for Brazilian Portuguese reality TV slang, sarcasm, and conflict patterns. This is not a generic sentiment wrapper — it’s a drama-specific anomaly detection system built for producers, content analysts, and superfans who need to spot a meltdown before the editors do.

## Core Features

**1. Conflict Proximity Graph (CPG)**
Generates a weighted, time-series graph of housemate interactions based on lexical aggression markers, interruption frequency, and speech volume shift. Each edge is scored with a “blow-up probability” that updates every 30 seconds. You can export the graph as a JSON adjacency list or SVG for overlay on live broadcast dashboards.

**2. Narrative Drift Detector (NDD)**
Continuously compares the current 6-hour window of dialogue against the baseline “expected” narrative (derived from pre-season interviews and early-week polls). When the semantic vector deviates beyond a configurable threshold (default: 0.72 cosine distance), it fires a webhook with a summary of the drift — e.g., “Alliance shift detected between Aline and Rafael, 83% probability of a public confrontation within 2 hours.”

**3. Multimodal Sarcasm & Irony Filter (MSIF)**
Standard sentiment tools fail on Brazilian Portuguese sarcasm, especially in reality show contexts. This model uses a fine-tuned BERTimbau checkpoint with an additional irony head trained on 12,000 manually labeled BBB tweets. It strips out false-positive “positive” sentiment from sarcastic praise (e.g., “Que maravilha, mais uma briga por causa de farofa”) and flags the actual hostility score.

## How to Use

### 1. Bootstrap the Stream Ingestion Pipeline

```bash
python3 -m bbb26dramapredictor ingest \
  --sources twitter,globo_cc,chat \
  --twitter-key "YOUR_BEARER_TOKEN" \
  --output "redis://localhost:6379/0" \
  --rate-limit 120
```

This spins up three async workers that push raw text into a Redis stream. The `--rate-limit` flag caps API calls to avoid throttling during live eviction nights.

### 2. Run a Predictive Sweep with Custom Thresholds

```bash
python3 -m bbb26dramapredictor predict \
  --stream-key "bbb26:live" \
  --window 360 \
  --drift-threshold 0.68 \
  --cpg-min-weight 0.4 \
  --webhook "https://your-dashboard.com/hooks/bbb26"
```

The `--window 360` parameter sets a 6-minute sliding window for micro-drama detection. Lower it to 120 for real-time alerts during Paredão announcements. The webhook payload includes a `drama_index` float (0–1) and a list of `top_conflict_edges`.

### 3. Export a Narrative Drift Report

```python
from bbb26dramapredictor import Reporter

reporter = Reporter(redis_client)
report = reporter.generate_drift_report(
    start="2025-02-10T20:00:00Z",
    end="2025-02-11T02:00:00Z",
    format="markdown"
)
with open("drift_report_eviction_night.md", "w") as f:
    f.write(report)
```

The report includes a timeline of drift events, the top 5 most volatile housemates, and a recommendation score for the next likely confrontation dyad.

## Best Practices

- **Tune drift thresholds per phase of the game.** Early weeks have lower baseline drama, so use a tighter threshold (0.55–0.60) to catch subtle alliance shifts. During the final weeks, raise it to 0.75+ to avoid alert fatigue from the constant noise.
- **Combine CPG output with manual human review for moderation decisions.** The conflict proximity graph is excellent for identifying brewing fights, but it can flag playful banter as high-risk. Always pair the `top_conflict_edges` list with a 30-second clip review before any on-air editorial action.
