# bbb26dramapredictor - BBB_023

## Overview

**bbb26dramapredictor** is a data-driven SaaS tool built for **BBB26 (Big Brother Brasil 26)** analysts, content creators, and superfans who need to cut through the noise. Instead of passively watching, this tool ingests social chatter, live voting patterns, and editorial cues to **predict drama escalation, alliance fractures, and public backlash** before they fully unfold. It’s not a dashboard—it’s a narrative early warning system.

## Hardcore Features

1. **Conflict Vector Engine (CVE)**  
   Scrapes and analyzes real-time Twitter/X, Reddit, and Instagram comments in Brazilian Portuguese, mapping sentiment polarity for each contestant. The engine detects **emotional contagion** and **moral outrage triggers** (e.g., micro-aggressions, strategic betrayals) to produce a "Drama Probability Score" for the next 4–6 hours.

2. **Alliance Fracture Predictor (AFP)**  
   Uses a lightweight graph neural network (GNN) trained on historical BBB26 voting data and live nomination patterns. AFP identifies **covert coalition instability**—like when two allies suddenly avoid camera angles together or their social media engagement diverges. Outputs a **"Split Risk" heatmap** per pair of contestants.

3. **Live Narrative Divergence Alert (LNDA)**  
   Compares the official Globo broadcast script (via closed captioning and editorial timestamps) against real-time fan reactions. When the gap between "what Globo wants you to see" and "what the internet is actually talking about" exceeds a configurable threshold, LNDA fires a **webhook or Telegram alert** with the divergence timestamp and top 3 conflicting topics.

## How to Use (3-Step Deep Dive)

### Step 1: Bootstrap the Data Pipeline
- Clone the repo and run `docker-compose up -d` to spin up PostgreSQL, Redis, and the scraper microservice.
- Configure your `config.yaml` with your Twitter API v2 credentials and a Globo.com RSS feed URL for live editorial cues.
- Execute `make init-db` to seed the schema and historical BBB26 data (last 7 days of tweets and nominations).

### Step 2: Trigger a Live Prediction
- Use the CLI tool: `python predict.py --mode drama --contestant "Alane" --window 6h`
- The CVE and AFP modules run in parallel, returning a JSON object with:
  - `drama_score`: float (0.0–1.0)
  - `alliance_risk_matrix`: dict of pairs and their split probability
  - `top_triggers`: list of 3 most impactful recent events
- For continuous monitoring, start the daemon: `python monitor.py --interval 120` (checks every 2 minutes).

### Step 3: Integrate Alerts into Your Workflow
- Point the LNDA module to your Discord webhook or Slack channel via environment variables.
- Set a custom threshold: `LNDA_THRESHOLD=0.75` (only alert when divergence exceeds 75%).
- Every alert includes a Markdown summary with the exact timestamp (UTC), the conflicting narrative snippets, and a link to the relevant clip on Globoplay.

## Industry Best Practices

1. **Always Cross-Reference with Official Polls**  
   Don’t rely solely on social media sentiment—BBB26 voting is heavily influenced by the "paredão" system and pay-per-vote mechanics. Use the tool’s `--poll-merge` flag to blend your drama prediction with UOL’s real-time voting percentages. This dramatically reduces false positives during high-stakes eliminations.

2. **Calibrate for "Silent Majority" Bias**  
   The tool’s default model weights Twitter/X data heavily, which overrepresents vocal fanbases. Before going live, run `python calibrate.py --sample 2000` to adjust weights based on Globo’s own audience panel data. This ensures your predictions don’t over-index on edgy, online-only controversies that never materialize in the house.
