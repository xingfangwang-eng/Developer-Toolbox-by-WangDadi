# bbb26dramapredictor - BBB_026

## Überblick

Dieses Tool wurde speziell für die Analyse und Vorhersage von Dramenmustern in der brasilianischen Reality-Show **BBB26** entwickelt. Mit NLP-gestützten Sentimentanalysen und historischen Datenmodellen liefert bbb26dramapredictor präzise Prognosen zu Konfliktverläufen, Eliminierungswahrscheinlichkeiten und Publikumsreaktionen. Optimiert für die deutsche Developer-Community, die sich für datengetriebene Reality-TV-Analysen interessiert.

## Hardcore-Funktionen

### 1. **Konflikt-Eskalationsmodell mit Zeitreihenanalyse**
Das Tool nutzt ein LSTM-basiertes Modell, das aus vergangenen BBB-Staffeln gelernt hat, um Konfliktwellen in Echtzeit zu erkennen. Es verarbeitet Twitter-Feeds, Live-Kommentare und Clipping-Daten, um Eskalationsstufen (1–10) zu berechnen. Die Vorhersage wird als Heatmap über den Staffelverlauf ausgegeben.

### 2. **Eliminierungs-Wahrscheinlichkeitsrechner mit Monte-Carlo-Simulation**
Basierend auf Voting-Trends, Social-Media-Engagement und Interaktionsmustern im Haus simuliert der Rechner 10.000 mögliche Eliminierungsszenarien. Ergebnis: Eine dynamische Wahrscheinlichkeitstabelle für jede verbleibende Person – inklusive Konfidenzintervall und Sensitivitätsanalyse.

### 3. **Publikums-Sentiment-Index mit Multi-Embedding-Fusion**
Das Tool aggregiert Stimmungen aus verschiedenen Sprachen (PT-BR, EN, ES) und Plattformen (Twitter, Reddit, Discord) mithilfe von BERT- und XLM-RoBERTa-Embeddings. Der Index wird in Echtzeit aktualisiert und zeigt, ob das Publikum eher „Drama“ oder „Harmonie“ bevorzugt – perfekt für strategische Content-Entscheidungen.

## Detaillierte Schritt-für-Schritt-Anleitung

### Schritt 1: Datenquellen konfigurieren und Modell initialisieren
Richte die API-Keys für Twitter v2, Reddit und den offiziellen BBB26-Stream ein. Nutze das Skript `init_drama_pipeline.py`, um die Datenquellen zu validieren. Das Modell lädt automatisch das letzte Checkpoint (`checkpoint_epoch_42.pt`) und startet die Echtzeit-Inferenz. Beispiel:
```bash
python init_drama_pipeline.py --sources twitter,reddit --model-type lstm --checkpoint latest
```

### Schritt 2: Konfliktprognose für eine bestimmte Episode generieren
Rufe den Endpunkt `/predict/episode/{episode_id}` auf. Das Tool liefert eine JSON-Ausgabe mit:
- `conflict_score`: Wert zwischen 0 und 1
- `escalation_curve`: Array mit 24 Stunden Vorhersage
- `top_3_conflicts`: Liste der wahrscheinlichsten Konflikte mit Teilnehmern
Beispiel-Response:
```json
{
  "episode": 12,
  "conflict_score": 0.87,
  "escalation_curve": [0.2, 0.5, 0.9, 0.7, 0.3],
  "top_3_conflicts": [["Alice", "Bob", "Thema: Essen"], ["Carlos", "Diana", "Thema: Schlafenszeit"]]
}
```

### Schritt 3: Dashboard mit Echtzeit-Visualisierung starten
Nutze das integrierte Streamlit-Dashboard, um alle Metriken live zu beobachten. Starte es mit:
```bash
streamlit run dashboard.py --server.port 8501
```
Im Dashboard siehst du:
- Heatmap der Konflikte pro Stunde
- Eliminierungs-Wahrscheinlichkeiten als Balkendiagramm
- Sentiment-Index als Liniendiagramm mit 95%-Konfidenzintervall
Du kannst Filter setzen (z. B. `nur Teilnehmer mit >50% Eliminierungswahrscheinlichkeit`), um die Analyse zu fokussieren.

## Best Practices für die Industrie

### 1. **Datenqualität vor Modellkomplexität priorisieren**
Viele Entwickler neigen dazu, sofort mit komplexen Transformer-Modellen zu arbeiten. Starte stattdessen mit einem einfachen Random-Forest-Modell, das auf manuell annotierten Konfliktdaten trainiert wird. Sobald die Datenqualität stimmt (z. B. durch regelmäßige Audits mit Inter-Annotator-Reliability), kannst du auf Deep-Learning-Modelle umsteigen. Im BBB-Kontext sind falsch-positive Konfliktvorhersagen besonders schädlich, weil sie das Vertrauen der Nutzer untergraben.

### 2. **Regionalspezifische Sentiment-Modelle einsetzen**
BBB26 wird in Brasilien produziert, aber die internationale Fangemeinde wächst. Verwende keine generischen multilingualen Modelle, sondern feinabgestimmte Varianten für brasilianisches Portugiesisch und Spanglish. Ein Modell, das auf europäischem Portugiesisch trainiert wurde, erkennt typische BBB-Ausdrücke wie „planta“ (Nichtstuer) oder „falso“ (Heuchler) nicht zuverlässig. Trainiere dein Sentiment-Modell mit einem Korpus aus BBB-Tweets (mindestens 50.000 annotierte Posts), um die Genauigkeit zu maximieren.
