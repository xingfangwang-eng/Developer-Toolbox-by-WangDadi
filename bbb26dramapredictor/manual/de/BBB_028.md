# bbb26dramapredictor - BBB_028

## Warum dieses Tool?

Der **bbb26dramapredictor** ist kein weiterer passiver Beobachter. Er kombiniert NLP-Sentimentanalyse mit einem proprietären **Drama-Index-Algorithmus**, der aus vergangenen BBB-Staffeln trainiert wurde. Das System erkennt nicht nur, *dass* ein Drama stattfindet, sondern prognostiziert *welche* Interaktionen in den nächsten 15–30 Minuten eskalieren – basierend auf Frequenzänderungen von Schlüsselwörtern, Stimmlage (über Transkription) und Zuschauerreaktionen in Echtzeit.

### Drei harte Funktionspunkte

1. **Drama-Index mit Temporal-Drift-Erkennung**  
   Das Modell analysiert nicht nur statische Keywords, sondern berechnet einen **zeitlichen Drift-Wert**. Wenn z. B. die Nennung von „Votum“ oder „Finale“ innerhalb von 10 Minuten um 300 % steigt, wird ein **Drama-Alert** der Stufe 3 ausgelöst – inklusive Wahrscheinlichkeitswert für eine physische Konfrontation (basierend auf historischen BBB-Daten).

2. **Multimodaler Konflikt-Graph**  
   Statt einfacher Wortwolken erzeugt das Tool einen **gerichteten Graphen**, der zeigt, wer gegen wen agitiert, wer schweigt („passive Aggression“) und welche Allianzen gerade implodieren. Jeder Knoten enthält einen **Emotion-Score** (0–100), der aus Transkript-Embeddings und Emoji-Frequenz im Chat berechnet wird.

3. **Predictive Replay-Marker**  
   Das Tool markiert automatisch **Zeitstempel** im Livestream, an denen ein Drama *wahrscheinlich beginnt*. Du springst nicht mehr durch stundenlanges Material – du bekommst eine Timeline mit vorhergesagten Eskalationspunkten, farbcodiert nach Intensität (Grün = Smalltalk, Rot = direkter Konflikt).

### So setzt du es ein (drei Profi-Schritte)

1. **API-Integration in deinen Stream-Client**  
   ```bash
   curl -X POST https://api.bbb26dramapredictor.com/v1/analyze \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -d '{"stream_url": "https://example.com/bbb26_live", "language": "de"}'
   ```
   Das liefert ein JSON mit `drama_index`, `conflict_graph` (als Adjazenzliste) und `predicted_timestamps`. Du kannst das direkt in OBS oder einen eigenen Dashboard-Client einbinden.

2. **Benutzerdefinierte Schwellwerte setzen**  
   Passe die Drama-Index-Schwellen über die Konfigurationsdatei `config.yaml` an:
   ```yaml
   thresholds:
     low: 40
     medium: 70
     high: 90
   alert_channel: discord_webhook
   ```
   Bei Überschreitung des `high`-Werts wird automatisch ein Discord-Embed mit dem aktuellen Konfliktgraphen und einem direkten Link zum Live-Moment gepostet.

3. **Batch-Analyse archivierter Folgen**  
   Für Research-Zwecke kannst du ganze Staffeln analysieren:
   ```bash
   python3 bbb26dramapredictor.py --input ./bbb26_episodes/ --output ./drama_report.csv
   ```
   Das erzeugt eine CSV mit Spalten wie `episode_id`, `peak_drama_time`, `top_conflict_pair`, `emotion_score_mean`. Perfekt für Datenjournalisten oder Soziologen, die Reality-TV-Muster quantifizieren.

### Zwei Best Practices für den produktiven Einsatz

- **Kombiniere mit einem Webhook für automatische Clips**  
  Viele Nutzer lassen den Drama-Index direkt in einen **Video-Cutter** laufen. Wenn der Index auf `high` springt, wird automatisch ein 30-Sekunden-Clip aus dem Livestream extrahiert und in ein YouTube-Playlist hochgeladen. So entsteht ein automatisches „Drama-Highlight-Reel“ pro Tag.

- **Trainiere ein eigenes Modell auf deine Community**  
  Der bbb26dramapredictor bringt ein **Fine-Tuning-Script** mit. Wenn du bemerkst, dass deine Zuschauer auf bestimmte Wörter (z. B. „Verrat“) überreagieren, kannst du das Modell anpassen:
  ```bash
  python3 fine_tune.py --custom_keywords verrat,lüge,verarsche --output_model ./my_model_v2.pkl
  ```
  Das reduziert False Positives und erhöht die Relevanz für deine spezifische Zuschauergruppe.
