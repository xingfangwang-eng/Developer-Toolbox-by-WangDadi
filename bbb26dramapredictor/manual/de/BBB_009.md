# bbb26dramapredictor - BBB_009

## Überblick

bbb26dramapredictor ist ein spezialisiertes Analyse-Framework, das auf Basis historischer Verhaltensmuster, Stimmungsanalysen in Echtzeit und sozialer Graph-Strukturen präzise Vorhersagen über Konfliktverläufe, Allianzbrüche und narrative Wendungen im brasilianischen Reality-Format BBB26 (Big Brother Brasil 26) trifft. Das Tool richtet sich an Medienanalysten, Data Scientists und Reality-TV-Strategen, die datengetriebene Einblicke in die komplexe Dynamik des Formats suchen.

## Hardcore-Funktionen

### 1. **Multi-Modale Sentiment-Synopse mit Temporal Decay**
Das System aggregiert Live-Kommentare aus Twitter, Reddit und spezialisierten BBB-Foren, gewichtet sie nach zeitlichem Verfall (Temporal Decay) und extrahiert mittels eines feinabgestimmten BERT-Modells (Fine-tuned on Portuguese Reality TV Corpora) die emotionale Valenz jedes Teilnehmers. Die Ausgabe erfolgt als Heatmap über 24-Stunden-Intervalle, die akute Spannungszonen farblich markiert.

### 2. **Graph-basierte Allianz- und Konfliktprognose (GNN-DramaScore)**
Auf Basis eines dynamischen Graph Neural Networks (GNN) modelliert das Tool die Interaktionen zwischen Teilnehmern als gewichtete Kanten (positive/negative Interaktionen, Nominierungen, Veto-Nutzungen). Der `DramaScore` quantifiziert die Wahrscheinlichkeit eines eskalierten Konflikts innerhalb der nächsten 48 Stunden, inklusive der wahrscheinlichsten Konfliktpaare.

### 3. **Narrative Arc Prediction mit Markov-Ketten höherer Ordnung**
Das Tool analysiert die Sequenz der wöchentlichen Ereignisse (Ansprachen, Wettbewerbe, Eviction-Ergebnisse) und modelliert sie als Markov-Kette 3. Ordnung. Dadurch kann es nicht nur den nächsten Eviction-Kandidaten vorhersagen, sondern auch, welche narrative Struktur (z. B. „Underdog-Rebellion“ oder „Allianz-Implosion“) in den kommenden Tagen dominieren wird.

## Schritt-für-Schritt-Anleitung zur Nutzung

### 1. **Dateninjektion und Initialisierung des Drama-Graphen**
```bash
python bbb26_predictor.py --ingest --source twitter,reddit --timeframe 72h
```
Das Skript lädt die letzten 72 Stunden an Social-Media-Daten, extrahiert Erwähnungen der Teilnehmer (Mapping via `participants.yaml`) und initialisiert den gewichteten Interaktionsgraphen. Achten Sie darauf, dass die API-Keys in der `.env`-Datei korrekt gesetzt sind – insbesondere für die Twitter API v2 und Reddit OAuth.

### 2. **Ausführen der Konfliktprognose (DramaScore-Berechnung)**
```bash
python bbb26_predictor.py --predict --model gnn_v2 --output drama_heatmap.json
```
Der Befehl startet die GNN-Inferenz. Das Modell berechnet für jedes Teilnehmerpaar einen Konfliktwahrscheinlichkeitswert (0.0–1.0). Die Ausgabe erfolgt als JSON mit Metadaten (Zeitstempel, Konfidenzintervall). Für eine visuelle Darstellung nutzen Sie `--output-format html`, um eine interaktive Heatmap zu generieren.

### 3. **Narrative Arc Extrapolation und Reporting**
```bash
python bbb26_predictor.py --narrative --horizon 7d --output narrative_report.md
```
Das Tool projiziert die wahrscheinlichsten narrativen Pfade für die nächsten 7 Tage. Der Report enthält eine Liste der Top-3 wahrscheinlichsten Szenarien (jeweils mit Markov-Transition-Wahrscheinlichkeit), sowie eine textuelle Zusammenfassung der erwarteten Wendepunkte (z. B. „Wahrscheinlichkeit eines Double-Eviction: 68%“).

## Branchen-Best-Practices

### **1. Datenqualität vor Modellkomplexität – Kalibrierung auf Stichprobenebene**
Im Reality-TV-Kontext sind Social-Media-Daten oft verrauscht (Bots, Fan-Kampagnen). Führen Sie vor jeder Hauptanalyse eine manuelle Kalibrierung anhand einer Stichprobe von 200–300 annotierten Posts durch. Nutzen Sie dafür das integrierte `calibrate.py`-Skript, das den F1-Score des Sentiment-Klassifikators auf dem aktuellen Datensatz misst und ggf. eine Nachjustierung der Gewichtung von Ironie/Sarkasmus vornimmt. Dies verhindert False Positives bei „humorvollen“ Konflikten.

### **2. Kombinieren Sie DramaScore mit externen Metriken (z. B. Quoten-Daten)**
Der `DramaScore` allein sagt nur die interne Dynamik voraus. Für eine strategische Medienplanung (z. B. wann eine „Shock-Eviction“ die höchste Zuschauerbindung erzeugt) sollten Sie die DramaScore-Werte mit historischen Einschaltquoten-Daten korrelieren. Nutzen Sie dazu das `correlate_with_ratings.py`-Skript, das eine lineare Regression zwischen DramaScore und Quotenänderungen berechnet und Ihnen die optimalen Zeitfenster für redaktionelle Eingriffe (z. B. Twist-Ankündigungen) liefert.
