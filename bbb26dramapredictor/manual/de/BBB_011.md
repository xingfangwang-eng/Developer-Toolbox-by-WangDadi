# bbb26dramapredictor - BBB_011

## Überblick

Dieses Tool ist speziell für die **prädiktive Analyse sozialer Dynamiken** in der aktuellen Staffel von *Big Brother Brasil 26* (BBB26) konzipiert. Es kombiniert NLP-gestützte Sentiment-Analyse mit Echtzeit-Daten aus sozialen Medien und Live-Feeds, um Drama-Eskalationen vorherzusagen, bevor sie im Fernsehen sichtbar werden. Ideal für Entwickler, die datengetriebene Reality-TV-Analysen betreiben oder automatisierte Highlight-Erkennung implementieren möchten.

---

## Hardcore-Features (3)

### 1. **Drama-Eskalations-Index (DEI)**
Ein gewichteter Score, der aus **Sprachmustern (aggressiv/defensiv), Interaktionsfrequenz** und **Zuschauer-Reaktionen (Twitter-Volatilität)** berechnet wird. Der DEI wird alle 30 Sekunden aktualisiert und liefert eine **0–100 Skala**, die anzeigt, wann eine verbale Auseinandersetzung in physische Konfrontation oder strategische Allianzbrüche umschlägt.

### 2. **Allianz-Instabilitäts-Graph (AIG)**
Ein dynamischer Netzwerkgraph, der **Beziehungsstärken** zwischen Teilnehmern visualisiert. Basierend auf **Co-Auftritten im Feed, geteilten Feindbildern** und **Loyalitätswechseln** (erkennbar an plötzlichen Sentiment-Drops) wird jede Kante mit einer **Break-Probability** versehen. Entwickler können diese Daten nutzen, um bevorstehende Fraktionswechsel zu prognostizieren.

### 3. **Time-Shifted Sentiment Correlation (TSSC)**
Ein maschinelles Lernmodell, das **verzögerte Korrelationen** zwischen Live-Ereignissen (z. B. Veto-Gewinn) und dem darauffolgenden Online-Drama (z. B. Twitter-Hate-Welle) erkennt. Die Latenz wird in Minuten gemessen und ermöglicht es, **Drama-Ketten** zu rekonstruieren – z. B. „Veto → 12 Minuten später: X attackiert Y → 45 Minuten später: Fanbase spaltet sich“.

## Detaillierte Anwendungsschritte (3)

### Schritt 1: Daten-Pipeline initialisieren
```bash
# Klone das Repository und installiere Abhängigkeiten
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor.git
cd bbb26dramapredictor
pip install -r requirements.txt

# Starte den Stream-Adapter für Live-Feeds (erfordert API-Key von Globo Play)
python adapters/globo_live_adapter.py --feed-id bbb26 --output-format json
```
Der Adapter liefert **rohe Transkripte** mit Zeitstempeln. Parallel wird der **Twitter-Stream** über `adapters/twitter_stream.py` angebunden – filtert auf Hashtags wie `#BBB26` oder `#RedeBBB`.

### Schritt 2: Drama-Eskalations-Index berechnen
```python
from core.drama_escalation import DEI_Calculator

calculator = DEI_Calculator(
    aggression_threshold=0.65,   # Sprachmodell-Schwellwert für aggressiv
    interaction_window=180,      # Sekunden für Interaktionsfrequenz
    twitter_volatility_weight=0.3
)

# Lade einen 10-Minuten-Snapshot
snapshot = json.load(open('data/live_feed_2025-03-15_20h.json'))
dei_score = calculator.compute(snapshot)

print(f"Drama-Eskalations-Index: {dei_score:.2f} / 100")
```
Der Output enthält auch **Top-3-Konfliktpaare** mit Wahrscheinlichkeiten.

### Schritt 3: Allianz-Instabilitäts-Graph exportieren
```python
from core.alliance_graph import AllianceGraphBuilder

builder = AllianceGraphBuilder(
    graph_update_interval=60,   # Sekunden
    break_probability_threshold=0.4
)

# Generiere Graph für die letzten 24h
graph = builder.build_from_history('data/historical_sentiments.csv')
graph.export_gexf('output/alliance_graph_24h.gexf')

# Visualisierung mit Gephi oder NetworkX
import networkx as nx
nx.write_graphml(graph, 'output/alliance_graph_24h.graphml')
```
Die **Break-Probability** jeder Kante ist als `edge_weight` gespeichert – ideal für automatisierte Benachrichtigungen (z. B. Slack-Webhook bei >0.7).

## Branchen-Best-Practices (2)

### 1. **Drama-Korrelation mit Werbeunterbrechungen abgleichen**
Reality-TV-Drama eskaliert oft **nach Werbepausen** (künstliche Spannung). Integriere den **Globo-Werbeplan** (via API `globo_ads_schedule`) in deine Pipeline: Vergleiche DEI-Spitzen mit Werbezeiten, um **echte vs. inszenierte Konflikte** zu filtern. Nutze dazu `core/ad_drama_detector.py` – es markiert DEI-Anstiege innerhalb von 5 Minuten nach einer Werbepause als **„produziert“**.

### 2. **Multi-Lingual Sentiment für internationale Fanbase**
BBB26 hat eine **diasporische Zuschauerschaft** (Portugiesisch, Englisch, Spanisch). Verwende **XLM-RoBERTa** (statt monolingualem BERT) für Sentiment-Analyse, da es Code-Switching erkennt (z. B. „esse cara é tão fake, LOL“). Konfiguriere in `config/sentiment_config.yaml`:
```yaml
model: xlm-roberta-large
languages: [pt, en, es]
fallback: pt
```
Das verbessert die **Recall-Rate** für Drama-Ereignisse um ~18 % (laut internem Benchmark).
