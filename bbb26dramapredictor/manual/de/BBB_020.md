# bbb26dramapredictor - BBB_020

## Überblick

Dieses Tool ist kein simpler Rate-Spaß, sondern ein datengetriebener Prädiktor für das soziale Gefüge und narrative Wendepunkte in der **BBB26**-Staffel. Es analysiert Echtzeit-Kommunikationsmuster, Voting-Historien und Post-Event-Sentiment aus dem brasilianischen Twitch- und Twitter-Ökosystem, um **Drama-Eskalationen** und **Bündniswechsel** mit hoher Wahrscheinlichkeit vorherzusagen. Entwickelt für Reality-TV-Analysten, Community-Manager und datenaffine Fans, die nicht nur zuschauen, sondern verstehen wollen, warum das passiert.

## 3 Hardcore-Features

### 1. **Sentiment-Timeline-Engine mit Event-Trigger-Erkennung**
Das Modell extrahiert aus 12+ Quellen (Twitter, Reddit, Jornal da Globo-Transkripte) einen zeitlich granularisierten Sentiment-Index pro Teilnehmer. Sinkt der Index eines Spielers innerhalb von 3 Stunden um >0,4 Standardabweichungen, wird ein **„Drama-Trigger“** ausgelöst – mit Angabe der wahrscheinlichsten Ursache (z. B. „Veto-Nutzung gegen Ally“ oder „Direkte Provokation im Schlafzimmer“).

### 2. **Soziogramm-Vorhersage mit Markov-Ketten-Modell**
Das Tool bildet das aktuelle Allianz-Netzwerk als gewichteten Graphen ab und simuliert **1000 Monte-Carlo-Pfade** für die nächsten 48 Stunden. Ergebnis: Eine Heatmap der wahrscheinlichsten **Bündnisbrüche** und deren Auswirkungen auf die Nominierungsrunde. Die Ausgabe enthält eine JSON-Struktur, die direkt in D3.js visualisiert werden kann.

### 3. **Counterfactual-What-If-Simulator**
Du kannst manuell ein Event setzen (z. B. „Teilnehmer X gewinnt die Leader-Probe“) und das Tool berechnet, wie sich die Gewinnwahrscheinlichkeiten aller Spieler unter dieser Bedingung verschieben. Das nutzt ein **kausales Inferenzmodell** (DAG-basiert) – kein Blackbox-Magic, sondern nachvollziehbare, probabilistische Logik.

## 3 Profi-Operations-Schritte

### Schritt 1: Daten-Import & Modell-Kalibrierung
Starte das Tool mit einem initialen Seed aus den aktuellen BBB26-Daten (z. B. letzte 3 Votings, letzte 24h Chat-Logs). Der Befehl:
```bash
python predictor.py --source twitter,globo --timeframe 24h --calibrate
```
Das Tool führt eine **Bayesianische Kalibrierung** durch und gibt einen **Konfidenz-Score** (0.0–1.0) für die aktuelle Modellgenauigkeit aus. Erst ab >0.75 solltest du die Vorhersagen ernst nehmen.

### Schritt 2: Live-Trigger-Stream abonnieren
Aktiviere den **WebSocket-Listener** für Echtzeit-Updates:
```python
from bbb26dramapredictor import LiveStream
stream = LiveStream(port=8765, filter='drama_trigger')
stream.subscribe(callback=my_alert_function)
```
Jeder Drama-Trigger wird als JSON-Objekt mit `timestamp`, `player_id`, `trigger_type` und `probability` gesendet. Du kannst direkt einen Discord-Bot oder Slack-Webhook anbinden.

### Schritt 3: What-If-Szenario durchführen und Export
Öffne die interaktive CLI oder das Web-Dashboard, wähle einen Spieler aus (z. B. `player_07`) und setze einen hypothetischen Event-Typ (`leader_win`). Das Tool generiert eine **PDF-Report-Datei** mit:
- Vergleich der **Elo-Ratings** vor/nach dem Event
- **Gini-Koeffizient** der Machtverteilung im Haus
- **Top-3 wahrscheinlichste Nominierungs-Duos** nach dem Event

Export via `--export pdf` oder `--export json`.

## 2 Industry Best Practices

### 1. **Nicht auf Einzelereignisse fixieren – auf Muster setzen**
Viele Nutzer erwarten, dass das Tool den exakten Rauswurf vorhersagt. Das ist statistisch unmöglich. Nutze stattdessen die **Dichte der Trigger-Ereignisse** (z. B. 3+ Trigger in 4h) als Indikator für eine bevorstehende **narrative Woche** – das ist robuster und hilft dir, Content-Strategien (z. B. für YouTube-Recaps) zu planen.

### 2. **Modell-Retraining nach jeder Woche einplanen**
Die sozialen Dynamiken in BBB26 ändern sich radikal nach jedem Rauswurf. Führe nach jeder Eliminierungsrunde ein **inkrementelles Retraining** durch (nicht volles Retraining, sonst verlierst du die historische Konsistenz). Der Befehl `predictor.py --retrain incremental` aktualisiert nur die Gewichte der zuletzt aktiven Spieler – das dauert <2 Minuten.
