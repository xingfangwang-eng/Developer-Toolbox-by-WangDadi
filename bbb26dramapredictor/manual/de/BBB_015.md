# bbb26dramapredictor - BBB_015

## Überblick

Ein spezialisiertes NLP-Framework zur Vorhersage und Quantifizierung von Eskalationsmustern in der Reality-Show **Big Brother Brasil 26**. Dieses Tool analysiert Live-Tweet-Streams, Chat-Protokolle und Untertitel, um narrative Spannungsbögen zu modellieren und dramatische Wendungen **vor** der Ausstrahlung zu signalisieren. Optimiert für deutsche Entwickler, die Reality-TV-Metadaten in Echtzeit verarbeiten.

## Hardcore-Features

1. **Sentiment-Kaskaden-Detektor**  
   Erkennt latente Konfliktverschiebungen innerhalb von 30 Sekunden nach dem ersten Mikro-Aggressions-Tweet. Nutzt ein Fine-tuned BERT-Modell auf portugiesisch-deutschem Code-Mixing-Korpus (BBB26-spezifisch). Ausgabe als JSON-Stream mit `conflict_probability > 0.85`.

2. **Narrativ-Architektur-Heatmap**  
   Visualisiert die zeitliche Dichte von „Votação“, „Paredão“ und „Prova do Líder“-Ereignissen als interaktive Heatmap. Basiert auf einer eigenen Event-Extraktion via Regex + Transformer-CRF. Export als SVG/PNG – direkt einbettbar in CI/CD-Pipelines.

3. **Multi-Modal-Trigger-Engine**  
   Kombiniert Audio-Transkription (Whisper) mit visueller Mimik-Erkennung (MediaPipe) aus 240p-Clips. Löst Webhooks aus, wenn ein Kandidat > 2 Sekunden „Augenrollen“ + erhöhte Tonlage zeigt. Latenz < 500 ms.

## 3-Schritte-Profisetup

### Schritt 1: Datenpipeline initialisieren
```bash
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor
cd bbb26dramapredictor
make install-deps  # Installiert whisper.cpp, torch, transformers
python3 -m bbb26dramapredictor.stream --source twitter --query "BBB26" --output ./live
```
Startet einen asynchronen Data-Loader, der Tweets, Reddit-Threads und Globoplay-Subtitles in ein einheitliches Parquet-Format normalisiert. Nutzt `asyncio` + `aiohttp` für 10k Requests/min.

### Schritt 2: Drama-Schwellwert konfigurieren
```yaml
# config/drama_thresholds.yaml
conflict:
  min_tweets_per_second: 5
  sentiment_drop_threshold: -0.3
  keyword_triggers: ["eliminação", "traição", "falsa"]
webhook:
  url: "https://dein-server.com/drama-alert"
  retry: 3
```
Setze dynamische Grenzwerte pro Staffel. Das Tool lernt aus historischen BBB-Daten (2019–2025) und passt die Sensitivität automatisch an. Validierung via `python3 -m bbb26dramapredictor.validate --config config/drama_thresholds.yaml`.

### Schritt 3: Echtzeit-Dashboard deployen
```bash
docker-compose -f docker/dramaview.yml up -d
```
Startet ein Grafana-basiertes Dashboard mit Prometheus-Metriken. Zeigt `drama_score`, `top_conflict_pairs` und `predicted_elimination_probability` an. Zugriff via `http://localhost:3000`. Exportiere Metriken als JSON für eigene Slack/Telegram-Bots.

## Best Practices für die Industrie

### 1. Daten-Governance für Reality-TV-Mining  
Verwende **Pseudonymisierung** für Kandidaten-IDs in deiner Datenbank. BBB26 unterliegt der LGPD (Brasilien) – speichere keine Klartext-Namen oder Geolokalisierung. Unser Tool integriert automatisch eine `hash_id`-Funktion, die alle PII vor dem Write-to-Disk entfernt. Für deutsche Entwickler: Konform mit DSGVO bei Verarbeitung europäischer IPs.

### 2. Modell-Rollback via Git-LFS  
Jedes trainierte Drama-Modell (`.pt` oder `.onnx`) wird mit einem Git-Tag versehen. Bei Fehlalarmen (z. B. falscher Positiver durch Meme-Welle) kannst du per `git checkout v2.3.1` auf eine stabile Version zurückfallen. Unser `model_registry.yaml` protokolliert automatisch Metriken wie `precision@0.9` und `recall@0.7`. Führe `make rollback` aus – das Tool lädt den letzten validierten Checkpoint neu.
