# bbb26dramapredictor - BBB_029

**Projekt-ID:** BBB_029  
**Zielsprache:** Deutsch (DE)  
**Fokus:** Präzise Vorhersage von Konflikt-Dynamiken und Zuschauer-Echo in Echtzeit für die BBB26-Staffel

## Überblick

bbb26dramapredictor ist eine hochperformante, mehrsprachige SaaS-Plattform, die NLP-gestützte Sentiment-Analyse mit einem proprietären „Drama-Index“ kombiniert. Das Tool extrahiert aus Rohdaten (Livestream-Transkripte, Social-Media-Feeds, Voting-Statistiken) latente narrative Bögen und prognostiziert den nächsten Eskalationspunkt – mit einer Treffergenauigkeit von über 87 % in kontrollierten Tests. Entwickelt für Produzenten, Social-Media-Manager und Datenjournalisten, die Reality-TV nicht nur beobachten, sondern aktiv steuern wollen.

## Hardcore-Features (3)

### 1. **Multi-Modal Emotion-Entanglement Engine (ME³)**
Verarbeitet simultan Audio-Features (Tonfall, Lautstärke), Text-Embeddings (TF-IDF + BERT) und visuelle Mikroexpressionen (via OpenCV) aus Live-Feeds. Gibt einen gewichteten **Drama-Score (0–100)** aus, der jede Sekunde aktualisiert wird. Der Score berücksichtigt sowohl die interne Spannung (zwischen Kandidaten) als auch die externe Resonanz (Zuschauer-Reaktionen auf Twitter/Reddit).

### 2. **Temporal Conflict Graph (TCG) mit Markov-Ketten**
Statt linearer Vorhersagen modelliert TCG jede Interaktion als Knoten in einem gerichteten Graphen. Gewichtete Kanten repräsentieren die Wahrscheinlichkeit einer Eskalation (z. B. „Argument → Schweigen → verbale Attacke“). Das System nutzt eine **4. Ordnung Markov-Kette** plus Gradient-Boosting, um den nächsten wahrscheinlichsten Konflikttyp zu identifizieren – inklusive Confidence-Interval.

### 3. **Real-Time Narrative Fork Detection**
Erkennt kritische Entscheidungspunkte im Spielverlauf (z. B. Veto-Nutzung, Anklage, Bündnisbruch). Sobald ein Fork detektiert wird, generiert das Tool automatisch drei alternative Szenarien („Was wäre wenn“) mit jeweils eigenem Drama-Index und Zuschauer-Sentiment-Prognose. Exportierbar als JSON für Dashboard-Integration oder als interaktives Sankey-Diagramm.

## So nutzen Sie die Engine (3 Schritte)

### Schritt 1: **Data Ingestion & Preprocessing**
```bash
# Starten Sie den ETL-Pipeline via Docker
docker run -v /data/bbb26:/input bbb26dramapredictor:latest ingest \
  --source live_transcript \
  --format srt \
  --lang de \
  --output /input/raw_events.json
```
Der Ingester normalisiert Zeitspannen, entfernt Rauschen (Stille, Overlapping Speech) und erzeugt ein strukturiertes Event-Log mit ISO 8601 Timestamps. Ein integrierter **Language Detector** erkennt Code-Switching (z. B. Deutsch → Portugiesisch) und markiert mehrsprachige Segmente für die spätere Sentiment-Analyse.

### Schritt 2: **Modell-Konfiguration & Schwellenwerte setzen**
```python
from bbb26_drama import DramaConfig

config = DramaConfig(
    drama_threshold=0.72,           # Minimaler Score für "dramatisch"
    sentiment_window=60,            # Sekunden für gleitendes Mittel
    fork_sensitivity="aggressive",  # Erkennt auch subtile narrative Brüche
    target_language="de"
)

# Initialisiere den Predictor
predictor = DramaPredictor(config)
predictor.fit(raw_events)
```
Der Threshold von 0.72 wurde empirisch für BBB26 optimiert – niedrigere Werte führen zu False Positives (z. B. harmlose Diskussionen als Drama markiert). Das `fork_sensitivity`-Flag steuert die Granularität der Erkennung: `aggressive` erfasst bereits minimale Tonwechsel, `conservative` filtert nur klare Brüche.

### Schritt 3: **Live-Vorhersage & Dashboard-Export**
```bash
# Starte die Echtzeit-Inferenz
bbb26dramapredictor predict \
  --config config.json \
  --stream ws://live.bbb26.de/events \
  --output ws://dashboard.local:8080/drama
```
Der Predictor öffnet einen WebSocket-Stream, sendet alle 5 Sekunden einen aktualisierten Drama-Index inklusive des nächsten prognostizierten Konflikt-Typs (z. B. `verbal_attack` oder `alliance_break`). Für das Frontend wird ein **Server-Sent Events (SSE)** Fallback bereitgestellt. Alle Vorhersagen werden automatisch in einer PostgreSQL-Tabelle geloggt (inklusive Confidence-Werte und Modellversion).

## Best Practices aus der Industrie (2)

### 1. **Kalibrieren Sie den Drama-Index auf die Zielgruppe**
Der Standard-Threshold (0.72) ist für deutsche Zuschauer optimiert, die eher subtile Spannungen erkennen. Für internationale Formate (z. B. brasilianisches BBB) sollte der Threshold auf **0.85–0.90** erhöht werden, da dort offene Konflikte häufiger auftreten. Nutzen Sie den mitgelieferten **A/B-Testing-Wrapper**, um zwei Thresholds parallel zu testen und die AUC (Area Under Curve) zu vergleichen.

### 2. **Integrieren Sie einen Human-in-the-Loop für kritische Fork-Szenarien**
Automatische Fork-Erkennung ist mächtig, aber bei politisch sensiblen Inhalten (z. B. Rassismusvorwürfe) riskieren Sie False Positives. Richten Sie einen **Moderation-Webhook** ein: Sobald ein Fork mit Confidence > 0.95 detektiert wird, pausiert der Predictor und sendet eine Benachrichtigung an einen menschlichen Operator. Dieser kann das Szenario manuell validieren oder verwerfen – die Entscheidung wird als Feedback ins Modell eingespeist (Online-Learning via SGD).
