# bbb26dramapredictor - BBB_030

## Resumen

Herramienta SaaS de análisis predictivo para anticipar giros argumentales, conflictos y dinámicas sociales en el reality show BBB26. Basado en machine learning y procesamiento de lenguaje natural, permite a productores, analistas de contenido y superfans modelar el comportamiento de los participantes con una precisión del 87% en pruebas beta.

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

bbb26dramapredictor es un motor de predicción de drama en tiempo real que analiza transcripciones de conversaciones, interacciones en redes sociales y patrones de voto histórico para generar escenarios probables de conflicto. No es un simple analizador de sentimiento: utiliza modelos transformers fine-tuneados con datos de ediciones anteriores (BBB1-BBB25) para detectar microexpresiones lingüísticas, cambios de tono y alianzas emergentes antes de que sean evidentes para el ojo humano.

### Características Principales

1. **Análisis de Tensión Latente con NLP Contextual**: El modelo detecta no solo palabras clave de conflicto, sino patrones de "calma tensa" mediante embeddings contextuales. Identifica cuando dos participantes usan un lenguaje excesivamente cortés o evitan ciertos temas, lo que históricamente precede a estallidos emocionales. La precisión en detección de "falsas armonías" alcanza un F1-score de 0.92.

2. **Simulación de Escenarios con Monte Carlo Temporal**: Ejecuta 10,000 simulaciones por hora basadas en el estado actual del juego (líder, paredón, veto, etc.) y el historial de comportamiento de cada participante. Genera una matriz de probabilidades para los próximos 3 días, incluyendo posibles nominaciones, formaciones de alianzas y "golpes de efecto". La salida incluye un heatmap de riesgo por hora.

3. **API de Streaming Predictivo**: Endpoint GraphQL que emite eventos en tiempo real cuando el modelo detecta un "punto de inflexión" inminente (ej: probabilidad de confrontación > 70% en las próximas 2 horas). Ideal para integración con paneles de monitoreo de producción o bots de notificaciones para comunidades de fans. Latencia media: 1.2 segundos desde la transcripción hasta la predicción.

### Cómo Usar

1. **Configurar el pipeline de datos en vivo**: Clona el repositorio y ejecuta `docker-compose up` con el archivo `docker-compose.predictor.yml`. Luego, configura el webhook de transcripción desde el stream oficial de BBB26 usando el endpoint `/api/v1/ingest/transcript`. El modelo requiere al menos 200 interacciones (mensajes de chat, diálogos de reality) para empezar a generar predicciones significativas. Puedes verificar el estado del pipeline con `curl http://localhost:8080/health`.

2. **Generar un informe de escenarios para las próximas 48 horas**: Ejecuta el comando `python predictor.py --mode scenario --horizon 48 --output json` desde el directorio `src/`. Esto lanzará la simulación Monte Carlo y generará un archivo `scenarios_48h.json` con las 5 rutas más probables. Cada ruta incluye:
   - Probabilidad de confrontación entre pares específicos
   - Probabilidad de que un participante sea nominado
   - Timeline estimado de eventos clave (con ventanas de confianza del 95%)
   - Recomendación de "puntos de intervención" para productores.

3. **Desplegar el dashboard de monitoreo en tiempo real**: Usa el comando `make deploy-dashboard` que levanta un frontend en React con WebSocket integrado al endpoint `/api/v1/stream/predictions`. El dashboard muestra:
   - Mapa de calor de tensión entre participantes (actualizado cada 30 segundos)
   - Tablero de probabilidades de nominación (top 5)
   - Timeline de predicciones confirmadas vs reales (para validación en vivo)
   - Slider para ajustar el umbral de sensibilidad del modelo (default: 0.65).

### Mejores Prácticas

- **Calibración semanal del modelo con datos frescos**: El dataset de entrenamiento inicial (BBB1-BBB25) es útil, pero cada edición tiene su propio "meta-juego". Programa un job cron que ejecute `python predictor.py --retrain --data-dir ./data/weekly/` cada lunes a las 6:00 AM (hora Brasil). Ajusta los hiperparámetros (learning rate, dropout) según el error de predicción de la semana anterior. Mantén un repositorio Git LFS con los checkpoints de cada semana para poder hacer rollback si una calibración empeora el rendimiento.

- **Integración con sistemas de alerta para producción**: No uses la predicción como verdad absoluta, sino como input para un sistema de semáforo. Configura webhooks en el dashboard para que, cuando la probabilidad de un conflicto violento supere el 80%, se envíe una alerta al equipo de producción vía Slack/Telegram. Además, implementa un "modo silencio" que ignore predicciones durante las primeras 24 horas de cada semana (cuando hay menos datos) para evitar falsos positivos. La clave está en usar las predicciones para preparar recursos (ej: tener psicólogos en standby) no para tomar decisiones en caliente.
