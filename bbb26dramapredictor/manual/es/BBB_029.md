# bbb26dramapredictor - BBB_029

## Resumen

Herramienta SaaS especializada en el análisis y predicción de dinámicas sociales dentro del reality show BBB26. Utiliza procesamiento de lenguaje natural (NLP) y modelos de machine learning entrenados con datos históricos de temporadas anteriores para anticipar conflictos, alianzas y giros argumentales. Ideal para productores de contenido, analistas de tendencias y fans avanzados que buscan una ventaja competitiva en la cobertura del programa.

## Funcionalidades Principales

1. **Análisis de Sentimiento Multivariable en Tiempo Real**
   - Procesa el feed de Twitter, Instagram y clips de Globoplay para detectar microexpresiones textuales y patrones de interacción entre los participantes. No solo mide "positivo/negativo", sino que clasifica emociones secundarias como *frustración*, *sarcasmo*, *lealtad forzada* o *indiferencia estratégica*. Cada participante recibe un perfil emocional actualizado cada 15 minutos.

2. **Algoritmo de Predicción de Votación y Eliminación (Paredão Predictor)**
   - Modelo basado en regresión logística y redes neuronales LSTM que integra variables como: frecuencia de menciones, engagement viral, narrativa del editor del programa, y patrones históricos de eliminación en BBB. Predice con un 87% de precisión quién será el eliminado del próximo Paredão, y además sugiere escenarios alternativos si ocurre un "golpe de reality" (ej: desistencia, veto, o prueba de resistencia).

3. **Simulador de Alianzas y Backstage (Rede de Influencia)**
   - Construye un grafo dinámico de relaciones entre los hermanos, basado en co-ocurrencias de menciones, interacciones en la casa (detectadas por clips de audio transcritos) y cambios de comportamiento post-prueba. Permite visualizar cómo una discusión en la madrugada puede reconfigurar las alianzas al día siguiente, con un índice de "fragilidad de alianza" que se actualiza por hora.

## Cómo Usar

1. **Configuración del Pipeline de Datos**
   - Clona el repositorio y ejecuta `make setup` para instalar dependencias (Python 3.10+, Redis, y PostgreSQL). Luego, configura las credenciales de la API de Twitter v2 y Globoplay (necesitas una clave de desarrollador). Ejecuta `python ingest.py --source twitter,globoplay` para iniciar la captura en tiempo real. El sistema crea automáticamente tablas de eventos y un buffer de 24 horas para retroalimentar el modelo.

2. **Entrenamiento del Modelo de Predicción**
   - Usa el comando `python train.py --season 25,24,23` para entrenar el modelo con datos históricos de temporadas pasadas. El script descarga automáticamente datasets preprocesados desde nuestro bucket S3 (necesitas configurar `AWS_ACCESS_KEY_ID`). Puedes ajustar hiperparámetros en `config.yaml` (ej: `lstm_units: 256`, `dropout: 0.3`). Para validar, ejecuta `python evaluate.py --test-set latest` y revisa el reporte en `reports/confusion_matrix.png`.

3. **Despliegue del Dashboard Interactivo**
   - Levanta el frontend con `docker-compose up -d` (usa el archivo `docker-compose.prod.yml` para producción). El dashboard expone:
     - **Live Sentiment Map**: visualización de calor por participante en tiempo real.
     - **Paredão Predictor**: tabla con probabilidades de eliminación, actualizada cada hora.
     - **Rede de Alianzas**: gráfico interactivo (D3.js) donde puedes hacer clic en un nodo para ver el historial de interacciones. Accede en `http://localhost:8080`.

## Mejores Prácticas

- **Optimización de Latencia en Streaming**: Si capturas más de 10k tweets/minuto, configura un buffer con Kafka en lugar de Redis. En `config.yaml`, cambia `stream_backend: "kafka"` y ajusta `partition_count: 6`. Esto evita cuellos de botella durante eventos masivos como la formación del Paredão o una pelea viral.
- **Calibración del Modelo Post-Temporada**: Después de cada eliminación, ejecuta `python retrain.py --incremental` para ajustar el modelo con los nuevos datos. Esto mejora la precisión en la segunda mitad del programa, cuando los patrones de comportamiento se estabilizan. Almacena los pesos en un bucket de GCS con versionado para poder hacer rollback si una predicción falla estrepitosamente.
