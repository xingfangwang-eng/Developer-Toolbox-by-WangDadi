# bbb26dramapredictor - BBB_013

## Resumen

Herramienta SaaS de análisis predictivo diseñada específicamente para anticipar dinámicas de conflicto, alianzas y giros narrativos en el reality show BBB26. Utiliza procesamiento de lenguaje natural (NLP) y machine learning para procesar feeds de Twitter, clips de video y transcripciones del programa, generando alertas tempranas sobre posibles "paredones", conversaciones explosivas o cambios de humor en los participantes.

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

bbb26dramapredictor es un sistema de predicción de drama basado en inteligencia artificial que monitorea en tiempo real las interacciones entre los participantes del BBB26. A diferencia de herramientas genéricas de análisis de sentimiento, este motor está entrenado con datos históricos de temporadas anteriores del programa (BBB20-BBB25) y optimizado para detectar patrones de comportamiento típicos del reality brasileño: desde el "falso aliancismo" hasta el "voto de castigo". Integra APIs de redes sociales, procesa audio de transmisiones en vivo y cruza datos de encuestas de audiencia para ofrecer una probabilidad de drama en una escala del 0 al 100.

### Características Principales (Hardcore Features)

1. **Algoritmo de Detección de "Fake Allies"**  
   Analiza la frecuencia de menciones, el tono de voz (a través de análisis prosódico) y la sincronización de publicaciones en redes sociales para identificar alianzas falsas entre participantes. Por ejemplo, si dos hermanos se etiquetan en Twitter pero evitan interactuar en la casa, el sistema eleva su "bandera de falsedad" con un score de hasta 90%.

2. **Predictor de Paredón con Ventana de 48 Horas**  
   Utiliza un modelo LSTM (Long Short-Term Memory) entrenado con datos de votaciones pasadas y encuestas de audiencia para predecir qué participante tiene más probabilidades de ser eliminado en el próximo paredón. La herramienta genera un ranking de riesgo con actualización cada 6 horas, basado en variables como "momentum de redes sociales", "historial de conflictos" y "popularidad en tiempo real".

3. **Motor de Análisis de "Voto de Castigo"**  
   Detecta patrones de venganza entre participantes mediante el cruce de datos de interacciones pasadas (discusiones, indirectas, votos anteriores) y el comportamiento en el confesionario (transcripciones de video). Si un participante que votó en contra de otro en la semana anterior recibe una nominación, el sistema activa una alerta de "voto de castigo" con una precisión del 78% basada en datos históricos.

### Cómo Usar (Pasos Detallados)

1. **Configurar Monitoreo en Tiempo Real con Webhooks**  
   - Ve a la sección "Fuentes de Datos" en el dashboard.  
   - Activa la integración con Twitter API v2 (requiere bearer token) y YouTube Data API (para clips de video).  
   - Define los términos de búsqueda: nombres de participantes, hashtags oficiales (#BBB26) y palabras clave como "paredón", "eliminación", "votación".  
   - Conecta un webhook a tu servidor (ej. Ngrok o Railway) para recibir actualizaciones en tiempo real. El sistema enviará un payload JSON con el score de drama y las alertas cada 15 minutos.

2. **Ejecutar Predicción de Drama con Parámetros Avanzados**  
   - En la terminal, clona el repositorio: `git clone https://github.com/xingfangwang-eng/bbb26dramapredictor`.  
   - Instala dependencias: `pip install -r requirements.txt` (incluye transformers, torch, fastapi).  
   - Ejecuta el script de predicción: `python predict_drama.py --input data/bbb26_live.json --window 48h --model lstm_v3.bin`.  
   - El script generará un archivo CSV con columnas: `participant`, `drama_score`, `fake_alliance_prob`, `paredon_risk`, `timestamp`. También puedes visualizar los resultados en el dashboard web corriendo `uvicorn api:app --reload`.

3. **Generar Alertas Personalizadas vía Slack/Discord**  
   - En el archivo `config.yaml`, configura el webhook de Slack o Discord:  
     ```yaml
     alerts:
       slack_webhook: "https://hooks.slack.com/services/T00/B00/xxxx"
       discord_webhook: "https://discord.com/api/webhooks/xxxx"
       threshold: 75  # Alerta cuando drama_score supere 75
     ```  
   - Reinicia el servicio: `systemctl restart bbb26dramapredictor`.  
   - Recibirás notificaciones en tiempo real con el nombre del participante, el tipo de alerta (ej. "Fake Alliance Detectada: João y Maria") y un enlace directo al análisis detallado.

### Mejores Prácticas (Industry Best Practices)

1. **Optimización de Datos para Evitar Overfitting**  
   Dado que el modelo LSTM se entrena con datos de temporadas anteriores, es crucial actualizar el dataset semanalmente con nuevos datos del BBB26. Recomendamos usar un pipeline de ETL con Apache Airflow para ingestar automáticamente transcripciones de video (usando Whisper de OpenAI) y tweets cada 6 horas. Además, aplica data augmentation con variaciones de ruido en el audio para mejorar la robustez del modelo en condiciones de baja calidad de grabación.

2. **Integración con Herramientas de SEO para Audiencia**  
   Para maximizar el alcance de las predicciones, conecta el output del modelo con un generador de contenido SEO (ej. usando GPT-4 + SERP API). Programa la publicación automática de artículos en WordPress con títulos como "BBB26: Predicción de Paredón para [Participante] - Análisis en Tiempo Real". Esto no solo atrae tráfico orgánico, sino que también retroalimenta al modelo con datos de engagement (clics, tiempo de lectura) para refinar las predicciones.
