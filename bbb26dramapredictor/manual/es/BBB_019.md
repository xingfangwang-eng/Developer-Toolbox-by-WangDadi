# bbb26dramapredictor - BBB_019

## Resumen

bbb26dramapredictor es un motor de análisis avanzado diseñado para predecir conflictos y giros narrativos en reality shows como BBB26. Combina procesamiento de lenguaje natural, minería de redes sociales y modelos de secuencia temporal para anticipar la evolución del drama en tiempo real.

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

No es una simple herramienta de scraping. Es un sistema modular que ingiere datos de Twitter, Instagram y clips de video, los tokeniza, y aplica un modelo LSTM entrenado con datos históricos de temporadas anteriores para predecir la probabilidad de un "paredão" o una "briga" con hasta 48 horas de antelación. Ideal para analistas de contenido, productores y fans hardcore que quieren adelantarse al guión.

### Características Principales

- **Análisis de Sentimiento Contextual con Weighted Lexicon**: No se limita a detectar palabras positivas/negativas. Usa un léxico ponderado específico del argot brasileño de reality shows (ej. "planta", "falso", "puxar saco") y asigna pesos dinámicos basados en la frecuencia de uso en el contexto de la casa. El output es un "drama score" normalizado entre 0 y 1.

- **Predicción de Cascadas de Conflicto con Graph Neural Networks**: Modela las relaciones entre participantes como un grafo dirigido. Cada nodo representa un participante, y las aristas representan interacciones (positivas, negativas, neutrales). Usa una GCN (Graph Convolutional Network) para predecir qué nodo será el epicentro del próximo conflicto y qué participantes se alinearán en bandos opuestos.

- **Reconocimiento de Patrones de "Paredão" con Ventanas Temporales Deslizantes**: Analiza secuencias de eventos (votaciones, discusiones, llantos) en ventanas de 6 horas. Detecta anomalías en el comportamiento de los participantes (ej. un aumento repentino de interacciones negativas con un aliado) y las correlaciona con la probabilidad de que ese participante sea indicado en la próxima votación.

### Cómo Usar

1. **Configurar el Pipeline de Ingesta de Datos**:
   - Clona el repositorio y ejecuta `docker-compose up` para levantar los servicios de Kafka y Redis.
   - Configura las credenciales de la API de Twitter (v2) y de Instagram Graph en el archivo `config/credentials.yaml`.
   - Ejecuta `python ingest.py --source twitter --query "BBB26" --hours 24` para iniciar la recolección en tiempo real. El sistema almacenará los datos en un buffer de Redis antes de pasarlos al modelo.

2. **Entrenar el Modelo de Predicción de Drama**:
   - Prepara un dataset histórico con eventos etiquetados (ej. "briga", "conversa tensa", "votación"). El formato debe ser CSV con columnas: `timestamp`, `participante`, `evento`, `intensidad` (0-10).
   - Ejecuta `python train.py --data data/historico_bbb25.csv --epochs 50 --batch_size 32`. El modelo generará un checkpoint en `models/drama_lstm_v1.pt`.
   - Para validar, usa `python evaluate.py --model models/drama_lstm_v1.pt --test data/test_bbb25.csv`. El output incluirá métricas como F1-score y MAE (Mean Absolute Error) para la predicción de intensidad.

3. **Desplegar el Dashboard de Predicciones en Tiempo Real**:
   - Configura un endpoint de WebSocket en `app/websocket_server.py` y ejecuta `python app/server.py`.
   - Abre el dashboard en `http://localhost:8501`. Verás un heatmap de las relaciones entre participantes y un timeline con las predicciones de drama para las próximas 24 horas.
   - Para recibir alertas en Telegram, configura el bot token en `config/telegram.yaml` y ejecuta `python alerts.py --threshold 0.7`. Recibirás un mensaje cada vez que el drama score supere el umbral.

### Mejores Prácticas

- **No confíes ciegamente en el modelo para decisiones de apuestas**: El modelo predice drama, no resultados de votación. El factor humano (ej. una intervención del presentador) no está modelado. Úsalo como apoyo para análisis, no como oracle financiero. Si lo usas para trading de apuestas, combínalo con datos de casas de apuestas y ajusta el umbral de alerta a 0.85 para reducir falsos positivos.

- **Refresca el léxico cada temporada**: El argot cambia. En BBB25 se usaba "planta" para describir participantes pasivos, pero en BBB26 puede surgir un nuevo término. Ejecuta `python update_lexicon.py --source twitter --query "BBB26" --output lexicon_bbb26.json` al inicio de cada temporada para extraer términos nuevos y reentrenar el modelo de sentimiento. Esto evita que el modelo se vuelva obsoleto.
