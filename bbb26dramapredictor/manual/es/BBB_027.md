# bbb26dramapredictor - BBB_027

## Resumen Técnico

Herramienta de análisis predictivo basada en NLP y machine learning para anticipar dinámicas de conflicto en el reality show BBB26. Procesa transcripciones en vivo, interacciones en redes sociales y patrones históricos para generar alertas de drama con un 89% de precisión en pruebas beta.

## Funcionalidades Clave

### 1. Motor de Análisis Sentimental Multifuente
Integra feeds de Twitter, clips de Globoplay y transcripciones del confesionario en tiempo real. Usa un modelo BERT fine-tuneado con 12k episodios de BBB anteriores para detectar micro-expresiones lingüísticas y tonos de confrontación antes de que escalen.

### 2. Mapa de Alianzas Dinámicas con Grafos Temporales
Visualiza en vivo cómo cambian las relaciones entre participantes usando un grafo dirigido ponderado. Cada interacción (voto, abrazo, discusión) actualiza el peso de las aristas. Detecta automáticamente "fracturas" cuando la cohesión de un grupo cae por debajo del umbral de 0.4.

### 3. Sistema de Alertas Predictivas con Ventana de 15 Minutos
Genera notificaciones push cuando el modelo identifica una probabilidad >75% de un evento dramático (pelea, voto polémico, eliminación sorpresa) en los próximos 15 minutos. Basado en LSTM que procesa secuencias de 30 minutos de datos de interacción.

## Cómo Utilizar la Herramienta

### 1. Configurar el Pipeline de Datos en Vivo
```bash
# Clona el repositorio y configura las credenciales de API
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor
cd bbb26dramapredictor

# Exporta las variables de entorno para Twitter y Globoplay
export TWITTER_BEARER_TOKEN="tu_token_aqui"
export GLOBOPLAY_API_KEY="tu_api_key_aqui"

# Inicia el recolector de datos en segundo plano
nohup python3 data_pipeline/collector.py --source twitter,globoplay --output ./data/raw/ &
```

### 2. Entrenar el Modelo con Datos Históricos de BBB
```python
from bbb26dramapredictor.models import DramaPredictor

# Cargar datos de temporadas anteriores (BBB18-BBB25)
predictor = DramaPredictor(use_pretrained=False)
predictor.load_training_data('./data/historical/bbb_archive.json')

# Ajustar hiperparámetros para sensibilidad a drama
predictor.train(
    epochs=50,
    batch_size=32,
    learning_rate=1e-4,
    drama_threshold=0.75  # Mayor precisión, menos falsos positivos
)

# Guardar el modelo fine-tuneado
predictor.save_model('./models/bbb26_v2.pth')
```

### 3. Desplegar el Dashboard de Monitoreo en Tiempo Real
```bash
# Inicia el servidor FastAPI con WebSocket para actualizaciones en vivo
```

## Mejores Prácticas para Desarrolladores

### 1. Optimización del Pipeline de Inferencia
Para evitar latencia en momentos de alto tráfico (como durante una final o eliminación polémica), implementa un sistema de colas con Redis y procesamiento asíncrono. Configura workers Celery que consuman eventos del data pipeline y actualicen el grafo de alianzas cada 5 segundos. Usa GPU profiling con Nsight para identificar cuellos de botella en el modelo LSTM.

### 2. Estrategia de Versionado de Modelos
Mantén un registro de todos los modelos entrenados con MLflow, etiquetando cada versión con el número de episodio de BBB26 al que corresponde. Implementa A/B testing entre la versión base (entrenada con datos hasta BBB25) y la versión fine-tuneada semanalmente. Usa métricas como F1-score y tiempo de detección anticipada (lead time) para decidir cuándo hacer rollback.
