# bbb26dramapredictor - BBB_001

## Resumen Técnico

Herramienta SaaS especializada en la predicción de dinámicas sociales y narrativas dentro del formato BBB26. Utiliza análisis de sentimiento en tiempo real, patrones de interacción entre participantes y modelos de machine learning entrenados con datos históricos de temporadas anteriores para anticipar giros argumentales, probables expulsiones y conflictos emergentes.

## Funcionalidades Clave (Hardcore)

### 1. Modelo Predictivo de Tensión Social (MTS)
Implementa un algoritmo de clustering temporal que evalúa la frecuencia de menciones cruzadas entre participantes en redes sociales y el chat del programa. Combina análisis de sentimiento con métricas de co-ocurrencia léxica para generar un **índice de fricción** en tiempo real. El modelo se reentrena automáticamente cada 6 horas con datos frescos del feed de Twitter/X y transcripciones del programa.

### 2. Simulador de Votación Popular (SVP)
Módulo de Monte Carlo que proyecta el resultado de paredões basándose en variables como: engagement histórico de cada participante, tendencias de hashtags regionales, y coeficiente de "efecto rebote" (cuando un participante gana apoyo tras ser atacado). Genera intervalos de confianza del 95% para cada posible resultado, permitiendo ajustar estrategias de campaña en comunidades de fans.

### 3. Analizador de Micro-expresiones y Patrones de Discurso (AMPD)
Integración con APIs de reconocimiento facial y procesamiento de lenguaje natural para detectar incongruencias entre lo que dice un participante y su lenguaje corporal. Detecta patrones de "gaslighting narrativo" (cuando un participante intenta reescribir la historia de un evento en vivo) y genera alertas tempranas de manipulación estratégica.

## Cómo Usar (Workflow Profesional)

### Paso 1: Configurar el Pipeline de Datos en Tiempo Real
Conecta tu instancia de bbb26dramapredictor a las fuentes oficiales: API de Twitter (v2), feed RSS de GloboPlay, y WebSocket del chat oficial. Ejecuta `bbb26 ingest --sources twitter,globo,chat --format streaming` para iniciar la captura. Ajusta el parámetro `--sentiment-threshold 0.75` si deseas filtrar solo interacciones con alta carga emocional.

### Paso 2: Ejecutar Predicción de Conflicto Semanal
Usa el comando `bbb26 predict --mode conflict --window 72h` para obtener una matriz de probabilidades de enfrentamientos entre pares de participantes en las próximas 72 horas. El output incluye un archivo JSON con ranking de riesgo (0-100) y los triggers clave detectados (ej: "participante A mencionó 3 veces a participante B en 10 minutos").

### Paso 3: Generar Dashboard de Estrategia de Votación
Ejecuta `bbb26 simulate --scenario "paredao_week5" --iterations 10000` para obtener proyecciones de expulsión. El sistema generará un HTML interactivo con gráficos de densidad de probabilidad, recomendaciones de horarios óptimos para campañas de voto, y un análisis de "swing voters" (perfiles de usuarios indecisos según su historial de interacción).

## Mejores Prácticas para el Sector

### 1. Segmentación de Audiencia por Cluster de Engagement
No trates a todos los fans por igual. Segmenta tu base de usuarios en 4 grupos según su comportamiento: "Espectadores Pasivos", "Votantes Tácticos", "Haters Activos" y "Influencers de Nicho". Ajusta las alertas del predictor para cada cluster: los "Votantes Tácticos" necesitan notificaciones push con proyecciones de expulsión, mientras que los "Influencers de Nicho" requieren reportes semanales con análisis de tendencias de hashtags.

### 2. Validación Cruzada con Datos de Encuestas en Vivo
El predictor es tan bueno como sus datos de entrada. Implementa un sistema de validación A/B: ejecuta dos versiones del modelo (una solo con datos de redes sociales, otra con datos de redes + encuestas de WhatsApp/Telegram) y compara el RMSE (Root Mean Square Error) de cada uno contra los resultados reales del programa. Documenta qué fuente tiene mayor peso predictivo en cada fase del reality (inicio vs final).
