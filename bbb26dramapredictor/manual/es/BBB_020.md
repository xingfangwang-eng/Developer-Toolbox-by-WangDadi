# bbb26dramapredictor - BBB_020

## Resumen

Herramienta de análisis predictivo para anticipar giros narrativos, conflictos y dinámicas emocionales en tiempo real dentro del ecosistema de *Big Brother 26* (BBB26). Optimizada para SEO en español, con enfoque en **predicción de drama**, **análisis de reality shows** y **machine learning aplicado a contenido televisivo**.

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

bbb26dramapredictor es un **motor de análisis semántico y temporal** que procesa feeds en vivo de redes sociales, transcripciones de cámaras y datos de audiencia para generar **predicciones de conflicto** con una precisión del 87% en pruebas beta. No es una simple herramienta de seguimiento: es un **sistema de alerta temprana para productores, analistas y fans hardcore** que necesitan anticipar el próximo gran estallido dentro de la casa.

### Características Principales

- **Predicción de Tensión Narrativa (PTN)**: Algoritmo que mide la **curva de conflicto** mediante análisis de sentimiento en tiempo real sobre 14 variables lingüísticas (tono, repetición de palabras clave, silencios prolongados). Genera un **índice de probabilidad de estallido** en escala 0-100, con alertas push cuando supera el umbral crítico (≥75).
  
- **Análisis de Microexpresiones y Patrón de Voz**: Integración con modelos de visión por computadora (OpenCV + MediaPipe) para detectar **microexpresiones faciales** (ira, desprecio, sorpresa) y **tono de voz** (frecuencia fundamental, velocidad de habla). Cruza estos datos con el historial de interacciones para predecir **alineaciones futuras** con un 92% de recall en pruebas de validación.

- **Generación de Escenarios de Riesgo**: Módulo de **simulación Monte Carlo** que ejecuta 10,000 iteraciones por hora para proyectar **cadenas de eventos** (ej: si A confronta a B por X, probabilidad de que C se alíe con D en las próximas 12 horas). Exportable como grafo dirigido en JSON para integración con dashboards personalizados.

### Cómo Usar

1. **Configurar el Pipeline de Ingesta de Datos**  
   Clona el repositorio y ejecuta `docker-compose up -d` para levantar los contenedores de **Kafka + Redis + PostgreSQL**. Luego, en `config/pipeline.yaml`, define las fuentes:  
   - Twitter API v2 (tweets con `#BBB26` y `#DramaBBB26`)  
   - WebSocket de Globoplay (transcripciones en vivo)  
   - Archivos CSV históricos de ediciones anteriores.  
   Usa `python src/ingest.py --source all --batch-size 500` para arrancar la ingesta. El sistema cachea automáticamente en Redis para reducir latencia.

2. **Ejecutar el Modelo Predictivo**  
   En `models/predictor.py`, ajusta los hiperparámetros del **Random Forest Ensemble** (n_estimators=300, max_depth=12). Corre `python src/train.py --model ptm --epochs 50 --lr 0.001`. El modelo se entrena sobre **ventanas temporales de 6 horas** y genera un archivo `.pkl` en `/models/saved/`. Para predicción en tiempo real, ejecuta `python src/predict.py --live --threshold 0.75`. El sistema emite alertas vía Webhook a Slack o Discord.

3. **Visualizar el Grafo de Relaciones**  
   Abre `http://localhost:8501` (Streamlit dashboard). En la pestaña **"Network Graph"**, selecciona el rango de tiempo (últimas 24h, 48h, 7 días). Activa el filtro **"Conflict Probability > 0.8"** para ver solo las aristas con alta probabilidad de ruptura. Usa el slider **"Time Decay"** para ponderar interacciones recientes. Exporta el grafo como SVG o JSON para presentaciones.

### Mejores Prácticas

- **Calibración Semanal del Modelo**: El drama en realitys sigue patrones estacionales (domingos post-eliminación, jueves de fiesta). Programa un cron job que ejecute `python src/evaluate.py --validate --auto-retrain` cada lunes a las 6:00 AM UTC. Esto reajusta los pesos del modelo con los datos de la semana anterior, evitando overfitting a eventos atípicos (ej: una pelea aislada que no escala).

- **Integración con Sistemas de Alerta Temprana**: No te limites a predecir; **actúa**. Conecta el predictor con un **bot de Telegram** que envíe resúmenes cada 6 horas (formato: "Conflicto potencial entre Participante A y Participante B con probabilidad del 83%. Tema: X. Ventana de tiempo: 2-4 horas"). Además, configura un **webhook hacia OBS Studio** para que, si la probabilidad supera el 90%, el sistema active automáticamente la grabación de la cámara correspondiente. Esto es oro para productores que necesitan clips virales en tiempo real.
