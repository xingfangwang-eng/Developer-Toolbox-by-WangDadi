# bbb26dramapredictor - BBB_006

## Resumen

Herramienta de análisis predictivo para detectar y anticipar conflictos, giros narrativos y patrones de comportamiento en tiempo real dentro del ecosistema del reality show BBB26. Optimizada para SEO en español, con foco en «predicción de drama BBB26», «análisis de reality show» y «herramienta de predicción de conflictos».

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

bbb26dramapredictor es un motor de predicción basado en machine learning que procesa datos de interacciones en vivo, menciones en redes sociales y patrones históricos de ediciones anteriores del BBB. A diferencia de simples agregadores de trending topics, esta herramienta aplica modelos de análisis de sentimiento y redes de co-ocurrencia para identificar micro-escenarios de tensión antes de que se materialicen en pantalla. Está diseñada para productores, analistas de contenido y fans avanzados que necesitan anticiparse al guion no escrito del reality.

## Características Principales

### 1. Detección de Micro-Tensiones en Tiempo Real
El sistema monitoriza el flujo de menciones en Twitter, Instagram y foros especializados, aplicando un modelo de Procesamiento de Lenguaje Natural (PLN) entrenado específicamente con el corpus léxico del BBB (jergas, apodos, dinámicas de grupo). Cuando detecta un aumento anómalo de co-ocurrencias negativas entre dos participantes (por ejemplo, «Felipe + traición» o «María + indirecta»), genera una alerta de conflicto inminente con una ventana de predicción de 2 a 4 horas.

### 2. Simulación de Narrativas Ramificadas
Utiliza un grafo de transiciones de estado basado en el historial de 26 ediciones del programa. Al ingresar un evento actual (como una prueba del líder o una nominación), el predictor ejecuta una simulación Monte Carlo de 10,000 iteraciones para proyectar los posibles desenlaces de la semana. El resultado es un mapa de probabilidades que muestra qué combinaciones de participantes tienen más chances de generar un enfrentamiento directo o una alianza sorpresa.

### 3. Análisis de Silencios y Vacíos de Información
Una funcionalidad poco común: el sistema identifica «zonas de silencio» en el timeline de un participante. Cuando un perfil que normalmente publica contenido constante de repente deja de hacerlo durante más de 6 horas, el predictor lo marca como un posible punto de inflexión narrativa. Este indicador ha demostrado tener una correlación del 78% con eventos de expulsión o conflictos internos no transmitidos en vivo.

## Cómo Usar

### 1. Configurar el Pipeline de Datos en Vivo
Clona el repositorio y ejecuta el script de inicialización del stream collector. Este se conecta a la API de Twitter y a los feeds RSS de portales de noticias de realitys. Asegúrate de tener las credenciales de acceso configuradas en un archivo `.env`. Luego, lanza el proceso con `python stream_collector.py --source twitter --filter "BBB26"`. El sistema empezará a acumular menciones y a etiquetarlas con metadatos de temporalidad y sentimiento.

### 2. Entrenar el Modelo con Datos Históricos
Para mejorar la precisión en tu instancia, descarga el dataset de las últimas 5 ediciones desde el repositorio de datos abiertos. Ejecuta el notebook `train_model.ipynb` que se encuentra en la carpeta `models/`. Este notebook ajusta los pesos del modelo LSTM con los patrones de drama de ediciones anteriores. El proceso tarda aproximadamente 45 minutos en una GPU estándar. Al finalizar, se genera un archivo `drama_predictor_v2.h5` que reemplazará al modelo por defecto.

### 3. Activar el Dashboard de Predicciones
Una vez que el modelo está entrenado y el stream collector corre, abre el dashboard web con `streamlit run dashboard.py`. En la interfaz, verás tres paneles: «Alertas de Micro-Tensión», «Simulación de Narrativas» y «Mapa de Silencios». Para obtener una predicción concreta, selecciona un participante en el menú desplegable y haz clic en «Generar Pronóstico». El sistema devolverá un informe en PDF con las probabilidades de conflicto para las próximas 24 horas.

## Mejores Prácticas

### 1. Calibración Semanal del Umbral de Alerta
No uses los umbrales por defecto durante toda la temporada. Durante las primeras dos semanas, cuando el elenco aún se está conociendo, el ruido de menciones es alto. Ajusta el parámetro `alert_threshold` en el archivo de configuración a 0.7 (en lugar de 0.5) para evitar falsos positivos. A partir de la cuarta semana, cuando las dinámicas se consolidan, reduce el umbral a 0.4 para capturar tensiones más sutiles.

### 2. Integración con Herramientas de Producción
Si eres parte del equipo de producción, conecta el predictor con tu sistema de gestión de contenidos (CMS) mediante la API REST expuesta en el puerto 8080. Configura un webhook que envíe automáticamente las alertas de micro-tensión al chat interno de Slack o Discord del equipo de edición. Esto permite que los editores tengan 2 horas de ventaja para preparar clips o planificar la narrativa del próximo programa en vivo.
