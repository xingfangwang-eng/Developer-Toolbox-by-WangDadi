# bbb26dramapredictor - BBB_028

## Resumen Técnico

**bbb26dramapredictor** es un motor de análisis de sentimiento y predicción de conflictos diseñado exclusivamente para la temporada 26 de Big Brother Brasil (BBB26). Utiliza procesamiento de lenguaje natural (NLP) en español y portugués, combinado con métricas de interacción social en vivo, para anticipar picos de tensión, alianzas rotas y momentos virales antes de que ocurran. Ideal para productores, analistas de audiencia y comunidades de fans que buscan datos accionables en lugar de especulación.

## Funcionalidades Clave

### 1. **Modelo de Predicción de Tensión por Ventana Temporal**
Implementamos una red LSTM entrenada con datos históricos de ediciones anteriores de BBB (temporadas 21-25) más feeds en vivo de Twitter y Reddit. El modelo genera un *índice de drama* (0 a 1) cada 15 minutos, considerando variables como frecuencia de menciones, tono de comentarios y patrones de interacción entre participantes. El umbral de alerta se ajusta dinámicamente según la fase del juego (eliminación, formación de paredón, etc.).

### 2. **Análisis de Micro-Conflictos con Detección de Entidades**
Más allá de simples picos de actividad, el sistema identifica *micro-conflictos* específicos: cruces de miradas, interrupciones en conversaciones, cambios en el lenguaje corporal (vía análisis de subtítulos y descripciones del equipo de producción). Extrae entidades (nombres de participantes, objetos, lugares) y las relaciona con emociones predominantes (ira, frustración, sarcasmo) usando un clasificador BERT fine-tuneado.

### 3. **Dashboard Interactivo con Alertas Push y Exportación JSON**
El frontend en tiempo real (React + WebSockets) muestra un heatmap de tensión por hora, ranking de participantes más conflictivos y un feed de predicciones. Cada alerta incluye un payload JSON descargable con metadatos: timestamp, participantes involucrados, probabilidad de escalada, y enlace directo al clip del momento (si está disponible). Integración nativa con Zapier y Slack para equipos de producción.

## Guía de Uso Avanzada

### 1. **Configuración de Stream de Datos en Vivo**
Ejecuta el recolector de datos con autenticación OAuth para Twitter API v2 y Reddit Pushshift. Ajusta los parámetros en `config.yaml`:
```yaml
stream:
  sources:
    - reddit: { subreddits: ["BBB26", "BigBrotherBrasil"], limit: 100 }
  window: 900  # ventana de 15 minutos en segundos
```
Luego inicia el agente con `python stream_agent.py --mode live`. El sistema empezará a poblar la base de datos PostgreSQL con embeddings de texto.

### 2. **Entrenamiento de Modelo Personalizado (Opcional)**
Si tienes datos históricos de otra temporada o quieres ajustar el modelo a un dialecto específico (ej. español rioplatense), usa el script `train_custom.py`:
```bash
python train_custom.py --data_path ./datos_bbb25.csv --epochs 50 --batch_size 32 --output_model ./modelo_bbb26.h5
```
El modelo se guarda en formato HDF5 y se puede cargar en el predictor principal con `--model_path ./modelo_bbb26.h5`. Recomendamos al menos 10,000 muestras etiquetadas con *drama_score* (0 = tranquilo, 1 = conflicto máximo).

### 3. **Despliegue de Alertas en Producción**
Para equipos que necesitan notificaciones inmediatas, despliega el servicio de alertas con Docker Compose:
```yaml
services:
  predictor:
    build: .
    ports:
      - "5000:5000"
    environment:
      - SLACK_WEBHOOK_URL=https://hooks.slack.com/services/T...
      - DRAMA_THRESHOLD=0.75
      - PUSH_NOTIFICATION=true
```
Luego, configura un cron job cada 15 minutos que ejecute `curl http://localhost:5000/trigger_alerts`. Las alertas se enviarán a tu canal de Slack con el formato:
```
🚨 Alerta de drama: Participante A vs Participante B | Probabilidad: 0.82 | Clip: https://...
```

## Mejores Prácticas para la Industria

### 1. **Integración con Workflows de Producción de TV**
No uses este predictor como una caja negra. Enlaza las alertas con tu sistema de edición en vivo (ej. Dalet, Avid). Cuando el índice de drama supere 0.8, activa automáticamente la grabación de un clip de 30 segundos antes del evento. Esto reduce el tiempo de edición posterior y garantiza que no te pierdas ningún momento clave.

### 2. **Calibración Ética y de Sesgo**
Los modelos de NLP pueden amplificar sesgos de género o raza si los datos de entrenamiento están desbalanceados. BBB tiene una audiencia diversa, por lo que te recomendamos:
- Revisar manualmente el 10% de las predicciones de alto drama para detectar falsos positivos relacionados con participantes de grupos minoritarios.
- Usar el flag `--bias_correction` en el entrenamiento para aplicar técnicas de reweighting basadas en demografía de participantes.
- Documentar públicamente las métricas de precisión por perfil de participante en tu dashboard.
