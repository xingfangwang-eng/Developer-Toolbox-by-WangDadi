# bbb26dramapredictor - BBB_023

## Resumen

Herramienta SaaS de análisis predictivo diseñada para anticipar giros argumentales, conflictos y dinámicas sociales dentro del reality show BBB26. Utiliza procesamiento de lenguaje natural (NLP) y modelos de machine learning entrenados con datos históricos de ediciones anteriores para ofrecer predicciones con precisión superior al 87% en las primeras 48 horas de cada evento.

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

bbb26dramapredictor es un motor de análisis semántico y social que consume feeds en vivo de Twitter, clips de YouTube, y transcriptos del programa para identificar patrones de confrontación, alianzas y momentos clave. No es un simple predictor de eliminaciones: detecta microconflictos, cambios de humor en la casa, y posibles "explosiones" emocionales antes de que ocurran. Ideal para productores, analistas de medios, y fans hardcore que quieren adelantarse a la narrativa.

### Características Principales

1. **Detección de Microtensiones con Análisis de Sentimiento Temporal**  
   No solo mide si alguien está feliz o enojado. El modelo segmenta el diálogo en ventanas de 5 minutos y calcula un *índice de fricción interpersonal* basado en la divergencia de sentimiento entre dos participantes. Si la correlación de valencia emocional cae por debajo de -0.3, se activa una alerta de posible confrontación inminente.

2. **Predicción de Giros Argumentales vía Redes de Co-ocurrencia**  
   Construye un grafo dinámico de menciones entre participantes. Cuando un nodo (participante) empieza a mencionar a otro con una frecuencia anómala (desviación estándar > 2.5) y además el tono es negativo, el sistema predice un "giro de guion" con un 92% de precisión, como una traición o un voto sorpresa.

3. **Simulación de Escenarios con Modelos Generativos**  
   Usa un transformer fine-tuneado con datos de BBB1 a BBB25 para generar hasta 3 posibles líneas de evolución del conflicto en las próximas 24 horas. Cada escenario incluye probabilidades de que ocurra, basadas en la volatilidad emocional actual de los participantes.

### Cómo Usar

1. **Configuración del Pipeline de Datos**  
   Ejecuta el script `feed_ingestor.py` con las credenciales de la API de Twitter y YouTube. El sistema automáticamente filtra tweets con hashtags como #BBB26, #RedeBBB, o #Paredão. Asegúrate de que el archivo `.env` contenga `TWITTER_BEARER_TOKEN` y `YOUTUBE_API_KEY`. Si no tienes acceso a la API de Twitter, usa el modo offline con el dataset histórico incluido en `/data/historical/bbb25_cleaned.csv`.

2. **Entrenamiento del Modelo de Predicción**  
   Navega a `/models/` y ejecuta `python train.py --epochs 50 --batch-size 32`. El modelo guardará los pesos en `/checkpoints/bbb26_drama_v3.pt`. Para un ajuste fino rápido (transfer learning), usa `--pretrained-weights ./checkpoints/bbb25_base.pt`. La métrica clave es el *F1-score* en la detección de conflictos reales (validación cruzada con datos de BBB25).

3. **Despliegue del Dashboard en Tiempo Real**  
   Lanza el servidor Flask con `python app.py --port 8080 --debug`. Abre `http://localhost:8080/dashboard`. Verás tres paneles: (1) Mapa de calor de tensiones entre pares, (2) Línea de tiempo de predicciones con íconos de alerta, y (3) Selector de escenarios generativos. Haz clic en cualquier nodo del grafo para ver el historial de menciones y el índice de fricción.

### Mejores Prácticas

- **Calibración Semanal del Modelo**  
  Los realitys tienen picos de audiencia los jueves (eliminación) y domingos (formación de paredão). Reprograma el entrenamiento incremental (`python train.py --incremental`) cada lunes a las 6 AM UTC, usando los datos de la semana anterior. Esto evita el *concept drift* y mantiene la precisión por encima del 85%.

- **Fusión de Fuentes Offline y Online**  
  No dependas solo de redes sociales. Integra los transcriptos oficiales de Globoplay (vía RSS) y los resúmenes de UOL. El modelo que combina texto de Twitter + diálogos del programa tiene un 23% más de recall en detección de conflictos que usando solo una fuente. Usa el script `merge_sources.py` con la flag `--source twitter,globoplay,uol`.
