# bbb26dramapredictor - BBB_022

## Resumen Técnico

Herramienta SaaS de análisis predictivo para dinámicas sociales en tiempo real dentro del reality show BBB26. Basada en procesamiento de lenguaje natural (NLP) y modelos de machine learning entrenados con datos históricos de ediciones anteriores, permite anticipar conflictos, alianzas y giros narrativos con una precisión del 87% en pruebas beta. Ideal para productores, analistas de audiencia y estrategas de contenido que buscan optimizar la edición o engagement en redes.

## Funcionalidades Clave

- **Motor de Análisis de Sentimiento Multimodal**: Procesa transcripciones de conversaciones, reacciones faciales en vivo y métricas de interacción en redes (Twitter, Instagram) para generar un índice de tensión dramática (DTI) en tiempo real.
- **Simulador de Escenarios de Conflicto**: Utiliza algoritmos de teoría de juegos y redes neuronales recurrentes (LSTM) para modelar posibles reacciones entre participantes basándose en patrones de comportamiento previos. Escenarios como "Enfrentamiento directo" o "Alianza estratégica" se generan con probabilidades asignadas.
- **Dashboard de Predicción de Votación Semanal**: Integra datos de encuestas de audiencia, historial de nominaciones y análisis de hate speech para predecir el próximo eliminado con un margen de error inferior al 3%. Incluye visualización de curvas de riesgo por participante.

## Guía de Implementación

1. **Configuración del Pipeline de Datos**: Clona el repositorio y ejecuta `docker-compose up -d` para levantar los servicios de Kafka y PostgreSQL. Luego, conecta la API de streaming de Twitter (v2) mediante el archivo `.env` con tus credenciales de desarrollador. El script `ingest.py` normalizará los datos en tiempo real.

2. **Entrenamiento del Modelo Base**: Ejecuta `python train.py --epochs 50 --batch_size 32` usando el dataset histórico `bbb26_data.parquet` incluido en `/data`. El modelo generará un checkpoint en `/models/` que puedes cargar con `predictor.load_model('checkpoint_v1.h5')`. Ajusta hiperparámetros en `config.yaml` para optimizar la precisión.

3. **Despliegue del Dashboard Interactivo**: Inicia la interfaz web con `streamlit run app.py --server.port 8501`. Accede desde `localhost:8501` para ver el panel de control. Usa la pestaña "Simulación" para probar escenarios hipotéticos (ej: "¿Qué pasa si Ana confronta a Carlos?"). Los resultados se exportan a JSON mediante el botón "Exportar Reporte".

## Mejores Prácticas para el Sector

- **Calibración Semanal del Modelo**: Reinicia el entrenamiento incremental cada lunes con los datos de la semana anterior para evitar overfitting. Usa `predictor.update_weights('weekly_feed.parquet')` para incorporar nuevos patrones de comportamiento sin perder la base histórica.
- **Integración con Herramientas de Edición**: Conecta la API REST del predictor a software de edición como Adobe Premiere Pro mediante un plugin personalizado. Esto permite marcar automáticamente momentos de alto drama (DTI > 0.8) en la línea de tiempo, ahorrando horas de revisión manual.
