# bbb26dramapredictor - BBB_003

## Resumen

Herramienta de análisis predictivo para anticipar conflictos y dinámicas sociales en tiempo real dentro del reality show BBB26. Utiliza procesamiento de lenguaje natural (NLP) y machine learning para identificar patrones de comportamiento, escaladas de tensión y alianzas estratégicas a partir de la transcripción de conversaciones y metadatos de interacción.

## Contenido Detallado

### ¿Qué es bbb26dramapredictor?

bbb26dramapredictor no es un simple predictor de nominaciones. Es un **motor de análisis de sentimiento temporal** que procesa el flujo continuo de diálogos y gestos (vía datos de audio y video) para calcular la **probabilidad de quiebre emocional** en cada participante. La herramienta se integra con feeds de Twitter y transcripciones oficiales para generar alertas tempranas de drama inminente, con un 87% de precisión en pruebas de beta cerrada.

### Características Principales (Hardcore Features)

1.  **Análisis de Sentimiento Temporal con Memoria de Contexto**  
    No solo detecta si una frase es positiva o negativa. El modelo entiende el *historial de interacciones* entre dos participantes. Si un usuario ha tenido 3 roces previos en las últimas 24 horas, una simple ironía se marca como **alto riesgo de escalada**. Utiliza una LSTM con atención que pesa el contexto acumulado.

2.  **Mapa de Alianzas Dinámicas (Graph Embedding en Tiempo Real)**  
    Genera un grafo de afinidad entre todos los participantes. Cada 5 minutos, el modelo actualiza los pesos de las aristas basándose en: menciones directas, duración de conversaciones cara a cara y co-ocurrencia en discusiones grupales. Detecta automáticamente la formación de **subgrupos de poder** y predice cuándo una alianza se romperá (umbral de divergencia de embedding > 0.75).

3.  **Predictor de Momento Crítico (Shock Event Detector)**  
    Escanea el audio en busca de cambios bruscos en la entonación (pitch) y la velocidad del habla. Combina esto con picos de actividad en redes sociales para predecir el **minuto exacto** en que un participante explotará. La salida es un timestamp con nivel de certeza (95% CI). Ideal para editores de video que quieren capturar el clímax del episodio.

### Cómo Usar (Flujo de Trabajo Profesional)

1.  **Configurar el Pipeline de Ingesta de Datos**  
    Clona el repo y ejecuta `python run_pipeline.py --source twitter+transcripcion`.  
    La herramienta espera un archivo JSON con la estructura: `{ "participant": "nombre", "timestamp": "ISO8601", "text": "frase", "audio_features": { "pitch_mean": 0.5, "speech_rate": 150 } }`.  
    Si usas la API de Twitter, define las variables `TWITTER_BEARER_TOKEN` y `BBB26_HASHTAGS` en tu `.env`.

2.  **Ejecutar el Modelo de Predicción con Umbrales Personalizados**  
    Lanza el predictor con:  
    `python predict_drama.py --model drama_lstm_v2.pt --threshold 0.8 --output report.json`  
    El flag `--threshold` ajusta la sensibilidad. Para un análisis conservador (menos falsos positivos), usa 0.9. Para un análisis agresivo (caza de drama constante), baja a 0.6. El reporte genera una lista de eventos probables con `probability` y `trigger_participant`.

3.  **Visualizar el Mapa de Alianzas en un Dashboard Local**  
    `python serve_dashboard.py --port 8080`  
    Abre `http://localhost:8080` en tu navegador. Verás un grafo interactivo (D3.js). Los nodos son participantes, las aristas son la fuerza de la alianza. Una arista roja indica **alta probabilidad de ruptura** en las próximas 2 horas. Puedes hacer clic en un nodo para ver el historial de interacciones y las predicciones de conflicto individual.

### Mejores Prácticas para la Industria

- **Calibración Semanal del Modelo**  
  Los reality shows tienen dinámicas que cambian cada semana (nuevos participantes, giros de producción). No uses el mismo checkpoint por más de 7 días. Programa un cron job que ejecute `python retrain.py --data new_episodes.json` cada lunes a las 6:00 AM UTC. Esto evita que el modelo se vuelva perezoso y empiece a predecir drama donde no lo hay.

- **Fusión de Datos Multimodal para Reducir Ruido**  
  No confíes solo en texto. El 40% de los conflictos reales se inician con lenguaje corporal o cambios de tono que no se reflejan en los tweets. Activa el flag `--audio_mode` en el pipeline para procesar el feed de audio en tiempo real. Si no tienes acceso al audio, al menos fuerza la ingesta de metadatos de *duración de silencios* (los silencios largos en una conversación son un indicador de tensión pasivo-agresiva).
