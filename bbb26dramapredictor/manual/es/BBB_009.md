# bbb26dramapredictor - BBB_009

## Resumen Ejecutivo

**bbb26dramapredictor** es una arquitectura SaaS diseñada para analizar, predecir y visualizar dinámicas de conflicto en tiempo real dentro del ecosistema de *Big Brother Brasil 26*. Utiliza procesamiento de lenguaje natural (NLP) y machine learning para convertir interacciones sociales en datos accionables para productores, analistas de audiencia y estrategas de contenido.

## Funcionalidades Clave (3 Hardcore Features)

### 1. Modelo de Análisis de Tensión Social (TSA)
Implementamos un **Transformer fine-tuneado** sobre el corpus histórico de BBB (temporadas 20–25) que clasifica cada interacción entre participantes en 5 niveles de tensión: *Neutral, Tensión Baja, Conflicto Emergente, Confrontación Directa, Crisis*. El modelo se actualiza cada 15 minutos con datos de transcripciones en vivo y subtítulos.

### 2. Dashboard de Predicción de Alianzas (PAD)
Un motor de grafos dinámicos que mapea en tiempo real las relaciones entre participantes. Utiliza **algoritmos de detección de comunidades** (Louvain) y **medidas de centralidad** para predecir quiebres de alianza con 72 horas de antelación. Cada nodo se pondera con métricas de *co-ocurrencia de menciones* y *proximidad física* en la casa.

### 3. API de Scoring de Dramaticidad (DramaScore)
Endpoint REST que devuelve un score normalizado (0-100) para cada escena, basado en:
- **Frecuencia de picos emocionales** (tono de voz, volumen, velocidad de habla)
- **Coeficiente de polarización** (dispersión de opiniones en el chat en vivo)
- **Índice de sorpresa** (desviación de patrones históricos de comportamiento)

## Cómo Usar: 3 Pasos Técnicos

### 1. Ingesta de Datos en Streaming
```bash
# Clona el repo y configura el pipeline de ingesta
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor.git
cd bbb26dramapredictor

# Activa el worker de captura de subtítulos en vivo
python3 src/ingestors/globo_subtitle_crawler.py --source live --output kafka://drama.raw
```
El worker se conecta al feed de subtítulos de GloboPlay, normaliza el texto y lo publica en un topic de Kafka. La latencia de extremo a extremo es < 2 segundos.

### 2. Ejecutar Predicción Batch o en Tiempo Real
```python
from dramapredictor import DramaEngine

# Carga el modelo pre-entrenado
engine = DramaEngine(model_path='models/tsa_v3.pt')

# Analiza una escena en tiempo real
scene = engine.analyze_scene(
    participants=['Alice', 'Bob', 'Carlos'],
    transcript='Alice: "No soporto tu actitud. Bob: "Tú tampoco eres santo."',
    timestamp='2025-04-10T22:15:00Z'
)

print(scene.drama_score)  # Output: 78.4
```
El método `analyze_scene` devuelve un objeto `SceneReport` con `drama_score`, `tension_level`, `predicted_outcome` (ej: "confrontación directa en 10 min").

### 3. Visualizar en el Dashboard React
```bash
# Lanza el frontend con métricas en vivo
cd dashboard
npm install
npm run dev -- --port 3000
```
Abre `http://localhost:3000`. El dashboard muestra un **heatmap de tensión** por hora, un **gráfico de red de alianzas** actualizable cada minuto, y el **DramaScore global** del día. Los datos se refrescan vía WebSocket desde la API.

## Mejores Prácticas para la Industria

### 1. Calibración del Modelo con Feedback Humano
En producción, recomendamos implementar un **loop de retroalimentación** donde productores del show etiqueten manualmente falsos positivos (ej: una discusión amistosa clasificada como conflicto). Usa estos datos para re-entrenar el modelo cada 2 semanas con **Active Learning**. El ratio óptimo de datos etiquetados vs. no etiquetados es 1:100.

### 2. Estrategia de Rate Limiting para la API Pública
Si expones el endpoint `/v1/dramascore` a terceros (ej: youtubers, sitios de apuestas), implementa **token bucket con colas de prioridad**. Los clientes premium (productores oficiales) tienen un límite de 1000 req/min, mientras que los gratuitos se limitan a 10 req/min. Usa **Redis** para almacenar el estado del bucket y **Cloudflare** para mitigar abusos. Ejemplo de configuración en `config/rate_limiter.yaml`:
```yaml
tiers:
  premium:
    rate: 1000
    burst: 2000
  free:
    rate: 10
    burst: 20
```
