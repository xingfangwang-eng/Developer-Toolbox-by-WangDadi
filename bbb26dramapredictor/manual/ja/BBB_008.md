# bbb26dramapredictor - BBB_008

## 概要

BBB26の視聴者・分析者向けに開発された、リアリティ番組内のドラマ発生を数理モデルと自然言語処理で予測するSaaS型ツール。出演者の発言・行動パターンから、視聴率に直結する「衝突・恋愛・裏切り」の発生確率をリアルタイムでスコアリングする。

## コア機能

### 1. 感情ベクトル時系列分析エンジン
出演者ごとの発言テキストを、日本語・ポルトガル語・英語の3言語で感情極性（ポジティブ/ネガティブ/攻撃性）に分解し、時系列でプロット。特定の感情スパイクが検出されたタイミングで、次の24時間以内に発生する「対立確率」をベイズ更新で算出する。閾値は動的に調整され、過去のシーズンデータとの類似度が高いほど精度が向上する。

### 2. ソーシャルグラフ・コンフリクトマップ
出演者間の関係性をグラフ構造で可視化。発言の相互参照、投票行動、カメラワークの偏りから「疑似関係」と「本質的対立」を識別。エッジの重みが急変したノードを自動検出し、ドラマの火種を特定する。このマップは毎日0:00(JST)に自動リビルドされ、過去7日間のウィンドウで差分を追跡可能。

### 3. 視聴者エンゲージメント予測API
Twitter/X、Reddit、5ちゃんねるの投稿をクローリングし、その日の「ドラマ話題度」をスコア化。このスコアを元に、翌日の視聴率変動を±3%以内の誤差で予測する。出力はJSON形式で、Slack/DiscordへのWebhook連携に対応。予測が外れた場合は、その原因（データ不足/異常値/ラベル誤り）を自動でログに記録する。

## 使い方

### 1. プロジェクトのクローンと環境変数の設定
まず、リポジトリをクローンし、`.env.example` をコピーして `.env` を作成。必須パラメータは `TWITTER_BEARER_TOKEN` と `OPENAI_API_KEY`（感情分析用）。`REDDIT_CLIENT_ID` はオプションだが、Redditデータを取り込む場合は必須。`.env` 内の `LANG` を `ja` に設定することで、分析結果のラベルが日本語で出力される。

```bash
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor.git
cd bbb26dramapredictor
cp .env.example .env
# エディタで .env を編集
```

### 2. 感情ベクトル分析の実行
`analyze_emotion.py` を実行すると、`data/transcripts/` 内の字幕ファイル（SRT形式）を自動で読み込み、感情ベクトルを出力。出力先は `output/emotion_vectors/` にCSVで保存される。`--window-hours 48` オプションで分析ウィンドウを変更可能。デフォルトは24時間。

```bash
python scripts/analyze_emotion.py --input data/transcripts/ --window-hours 48
```

### 3. コンフリクトマップの生成と通知
`generate_conflict_map.py` を実行すると、`output/conflict_maps/` に最新のグラフ画像（PNG）とJSONが生成される。`--alert-threshold 0.8` でアラートの閾値を設定。0.8以上でSlack通知が飛ぶ。Webhook URLは `.env` の `SLACK_WEBHOOK_URL` に事前に設定しておく。

```bash
python scripts/generate_conflict_map.py --alert-threshold 0.8
```

## ベストプラクティス

### 1. データソースの重み付けを調整する
デフォルトではTwitter(X)の重みが0.5、Redditが0.3、5ちゃんねるが0.2に設定されている。しかし、シーズン中盤になるとTwitterのノイズが増えるため、`config/weights.json` を編集してRedditの重みを0.4に上げると、予測精度が安定する。特に「裏切り」イベントはRedditで先に語られる傾向があるため、重み調整は週次で行うと良い。

### 2. 感情分析モデルをファインチューニングする
標準搭載のモデルは汎用的な日本語・ポルトガル語対応だが、BBB26の固有表現（「パイ」「リーダー」「モンスト」など）を学習させることで、感情極性の判定精度が上がる。`train_custom_model.py` に過去シーズンの字幕とラベルを食わせて、`models/custom_emotion.pth` を生成。その後、`.env` の `EMOTION_MODEL_PATH` をそのパスに変更する。これで、出演者の「笑顔で裏切る」ような複雑な感情もキャッチできるようになる。
