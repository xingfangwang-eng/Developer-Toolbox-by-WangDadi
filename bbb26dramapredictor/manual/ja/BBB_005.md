# bbb26dramapredictor - BBB_005

## 概要

BBB26のドラマを予測するためのプロフェッショナルツール。視聴者投票、発言パターン、編集バイアスをリアルタイムで解析し、次なる展開を的中させる。

## コア機能

### 1. エピソード間の感情弧スキャン
参加者の発言を15秒単位でトーン分析し、エピソード間での感情の急激な変化（例：親密→敵対）を検出。これにより、次回の対立発生確率をパーセンテージで出力する。

### 2. 編集バイアス検出エンジン
プロデューサーが意図的に切り取った「煽りカット」や「ミスリード編集」を、シーン遷移の不自然さとBGMの変化から自動識別。視聴者が騙されやすいポイントを事前にマーキングする。

### 3. ソーシャルメディア連動型ドラマスコア
Twitter（X）のハッシュタグトレンドと、参加者の発言頻度を同期。特定の参加者に対するネガティブな言及が急増したタイミングで、排除投票の可能性をアラート通知する。

## 使い方

### 1. エピソードデータのインポートと前処理
```bash
python bbb26_importer.py --source bbb26_live --output ./raw_data
```
BBB26公式配信から直接キャプチャした字幕ファイル（SRT形式）と、タイムコード付きのトランスクリプトを取得。`--clean`フラグでノイズ（歓声、効果音）を除去し、発言のみのクリーンなデータセットを作成する。

### 2. 感情弧の可視化と閾値設定
```python
from bbb26dramapredictor import EmotionArc
arc = EmotionArc.load("./raw_data/bbb26_e01.srt")
arc.plot(threshold=0.7, highlight_conflict=True)
```
`plot()`で感情の時系列グラフを生成。`threshold`を0.6〜0.9の範囲で調整し、対立とみなす感情スコアの境界を設定する。出力はSVG形式で、GitHubのREADMEに直接埋め込み可能。

### 3. 予測結果のエクスポートと定期実行
```bash
crontab -e
# 毎日22:00に予測を実行し、結果をSlackに通知
0 22 * * * cd /opt/bbb26dramapredictor && python predict.py --webhook https://hooks.slack.com/services/XXXXX
```
`predict.py`は前日のエピソードデータを自動取得し、翌日のドラマ展開を3段階（高/中/低）で予測。結果はJSON形式でエクスポートされ、SlackやDiscordに直接投稿可能。

## ベストプラクティス

### 1. 複数シーズンのデータでモデルを学習させる
BBB26だけでなく、過去のBBBシーズン（BBB20〜BBB25）のデータを学習に使用することで、プロデューサーの編集パターンや参加者の典型行動をモデルに組み込む。これにより、初回エピソードでも高い精度で予測が可能になる。

### 2. リアルタイムAPIとバッチ処理を併用する
放送中はリアルタイムAPI（WebSocket経由）で感情スコアを監視し、放送終了後はバッチ処理で詳細な分析レポートを生成する。これにより、視聴中の即時アラートと、翌日の戦略立案用データの両方を提供できる。
