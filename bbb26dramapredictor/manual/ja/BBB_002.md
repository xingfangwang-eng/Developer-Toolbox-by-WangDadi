# bbb26dramapredictor - BBB_002

## 概要

本プロジェクトは、ブラジルのリアリティ番組「BBB26」における参加者間のドラマ発生を、機械学習モデルとソーシャルメディア・センチメント解析を用いて予測するSaaSツールです。キーワード「BBB26 drama prediction」「Reality show drama analysis」「BBB26 予測ツール」に最適化されたSEO設計を施しています。

## コア機能

### 1. マルチモーダル・センチメント・フュージョン
Twitter（X）、Instagram、Reddit、TikTokの投稿をリアルタイムでクローリングし、テキスト・画像・動画のメタデータから感情スコアを抽出。参加者ごとの「ドラマ指数（Drama Index）」を0.0〜1.0の浮動小数点で算出します。LSTM + Transformerのハイブリッドモデルにより、過去20シーズンのデータを学習済み。

### 2. 因果関係グラフ・エンジン
各参加者の発言・行動をノードとし、時系列に沿った因果連鎖をグラフデータベース（Neo4j）で管理。特定の発言が他の参加者に与えた影響を可視化し、**「ドラマのトリガー」**を特定します。API経由でグラフデータをJSON形式でエクスポート可能。

### 3. 確率ベース・シナリオシミュレーター
モンテカルロ法を用いて、今後のエピソードにおけるドラマ発生確率をシミュレーション。ユーザーは「参加者AがBを挑発した場合」「特定のタスクが発生した場合」などの仮説を設定し、結果を確率分布として取得できます。結果はヒートマップ・折れ線グラフで出力。

## 使い方

### 1. APIキーの取得と初期設定
```bash
curl -X POST https://api.bbb26dramapredictor.io/v1/auth \
  -H "Content-Type: application/json" \
  -d '{"project": "bbb26dramapredictor", "email": "your@email.com"}'
```
レスポンスとして返される`api_key`を環境変数`BBB26_API_KEY`に設定します。Python SDKを使用する場合は`pip install bbb26-sdk`でインストール後、`from bbb26 import DramaPredictor`でインポート。

### 2. リアルタイム予測の実行
```python
from bbb26 import DramaPredictor

predictor = DramaPredictor(api_key="your_api_key")
result = predictor.predict_drama(
    participants=["participant_01", "participant_05"],
    time_window="next_24h",
    include_social_sentiment=True
)
print(result.drama_index)  # 例: 0.873
```
結果オブジェクトには`drama_index`の他に、`trigger_events`（トリガーとなったイベントのリスト）と`confidence_interval`（95%信頼区間）が含まれます。

### 3. シナリオシミュレーションの実行
```python
scenario = predictor.simulate_scenario(
    initial_state={"participant_01": "angry", "participant_05": "defensive"},
    num_simulations=10000,
    max_steps=50
)
scenario.plot_heatmap()  # 参加者間の相互作用をヒートマップで可視化
scenario.export_graph("scenario_graph.json")  # 因果グラフをJSON出力
```
シミュレーション結果は`scenario.drama_probability`で、各ステップにおけるドラマ発生確率が配列で取得可能。

## ベストプラクティス

### 1. データソースのフィルタリング戦略
ソーシャルメディアのノイズを除去するため、**「信頼度スコア」**を導入してください。具体的には、フォロワー数が1,000未満のアカウント、または投稿時間が放送終了から30分以上経過したものは除外する設定を推奨します。`predictor.set_filter(min_followers=1000, max_delay_minutes=30)`で実装可能。

### 2. モデルの再学習サイクル
BBB26のシーズン中は、**毎週月曜日にモデルを再学習**させることを推奨します。過去7日間のデータを追加学習データとして使用し、`predictor.retrain(epochs=5, learning_rate=0.001)`で実行。過学習を防ぐため、バリデーションセットは常に最新の2エピソード分を保持する設計としてください。
