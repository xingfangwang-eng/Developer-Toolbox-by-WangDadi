# bbb26dramapredictor - BBB_022

## 概要

BBB26（Big Brother Brasil 26）のリアリティショーにおけるドラマ発生を、ソーシャルメディアのセンチメント分析とNLPエンジンで予測するプロフェッショナルツール。視聴者の感情スコアと参加者の行動パターンをリアルタイムにマッピングし、次に起こる「対立」「同盟崩壊」「視聴者バッシング」を高精度で予測する。

## 詳細な内容

### bbb26dramapredictor とは

bbb26dramapredictor は、BBB26のライブ配信データ、Twitter（X）のトレンド、参加者の発言トーンを統合し、ドラマ発生確率を時系列で算出するSaaS型予測エンジン。単なる分析ダッシュボードではなく、イベント発生前にSlackやDiscordへアラートを飛ばすアクション駆動型ツール。

### コア機能

- **感情トーン・スコアリングエンジン**  
  参加者ごとの発言（ポルトガル語・英語対応）をリアルタイムに解析し、怒り・皮肉・悲しみ・興奮の4軸でスコアリング。閾値を超えた瞬間に「ドラマ発生フラグ」を自動生成する。

- **同盟崩壊予測モデル**  
  過去のBBBシリーズ（BBB20〜BBB25）の会話グラフデータを学習。参加者間の相互フォロー解除、会話時間の急減、非言語的シグナル（目線の回避など）を検知し、同盟崩壊を72時間前に予測。

- **視聴者バッシング・ホットスポットマップ**  
  ソーシャルメディア上でのハッシュタグトレンドとネガティブ感情の集中地域をマッピング。視聴者が「どの参加者に怒っているか」を可視化し、プロデューサー向けに編集ポイントを提案。

### 使い方

1. **APIキー発行とデータストリーム設定**  
   プロジェクトルートで `npm run setup-drama-predictor` を実行。自動生成される `.env` にTwitter API v2トークンとBBB26の公式配信APIエンドポイントを設定。初期キャリブレーションとして、過去24時間の全参加者のトークデータをバルクロード。

2. **予測パイプラインの起動とカスタム閾値調整**  
   `docker compose up predictor-engine` でNLPパイプラインを起動。`/config/thresholds.yaml` で「ドラマ発生アラート」のトリガー閾値を調整（例：怒りスコア>0.75かつ同盟崩壊確率>60%でDiscordへ通知）。設定後、`curl -X POST http://localhost:8080/predict/force` で即時予測をテスト。

3. **リアルタイムダッシュボードとエクスポート**  
   `http://localhost:3000/dashboard` で参加者ごとの感情トレンドラインと予測カレンダーを確認。右上の「Export as CSV」で予測ログを出力。さらに、Slack Webhookを `/config/webhooks.yaml` に追加すれば、ドラマ発生10分前に自動通知が飛ぶ。

### 業界ベストプラクティス

- **キャリブレーションは毎週更新**  
  BBB26の参加者パーソナリティは毎週変化する。週次で過去7日間の実際のドラマイベント（対立、退場など）と予測結果を照合し、モデルの重みを再調整。特に「皮肉検出」の精度は、ブラジルポルトガル語のスラング辞書を手動で追加することで大幅に向上する。

- **プロデューサー向けには「ノイズフィルター」を必須に**  
  ソーシャルメディアのノイズ（bot、スパム、ファンアート）を除去しないと、誤報率が30%を超える。`/config/noise_filter.yaml` で「フォロワー数<100」「投稿頻度>50/day」のアカウントを自動除外。さらに、BBB公式アカウントからのRTのみを優先する「オーソリティフィルター」を有効にすると、精度が90%を超える。
