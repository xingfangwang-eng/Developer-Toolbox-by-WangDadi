# bbb26dramapredictor - BBB_011

## 概要

BBB26のドラマ展開を予測するための専用分析ツール。リアリティショーにおける参加者の行動パターン、感情変動、ストーリーアークをリアルタイムに解析し、次に起こる「事件」を高精度で予測します。単なる視聴ツールではなく、プロデューサーやマーケター向けの戦略的インサイトを提供します。

## 詳細な内容

### bbb26dramapredictorとは？

bbb26dramapredictorは、BBB26（Big Brother Brasil 26）における参加者の感情・対立・連携のダイナミクスを機械学習モデルで解析し、視聴率やSNSトレンドに基づいたドラマ予測を自動生成するSaaS型エンジンです。従来の「勘」や「経験則」に依存した予測を排除し、データドリブンな意思決定を支援します。

### コア機能

1. **感情アーク・プロファイリングエンジン**  
   参加者の発言履歴、表情変化、視聴者のリアクション（ツイート・コメント）を時系列でマッピング。感情の「山」と「谷」を自動検出し、次に衝突が起きるタイミングと参加者ペアを特定します。

2. **因果関係グラフ（Drama Causality Graph）**  
   イベント間の因果連鎖を有向グラフで可視化。「AがBを批判 → Cが介入 → 視聴者が分裂」といったパターンを学習し、特定の行動がどの程度のドラマ波及効果を持つかをスコアリングします。

3. **リアルタイム・シナリオシミュレーター**  
   現在の状況を基に、複数の未来シナリオ（例：「もしXが脱落したら？」「Yが秘密を暴露したら？」）を生成。各シナリオの発生確率と、それに伴う視聴率変動を予測します。

### 使い方

1. **データソースの統合と初期モデル学習**  
   `bbb26dramapredictor` のCLIまたはAPIに、BBB26の過去エピソードデータ（字幕、SNSフィード、視聴率ログ）を投入。`--train-mode full` フラグで感情解析モデルと因果グラフを初期構築します。  
   ```bash
   bbb26dramapredictor ingest --source ./episodes/ --train-mode full --output-model ./models/bbb26_v1.pkl
   ```

2. **予測クエリの実行と解釈**  
   リアルタイムデータ（ライブ配信のコメントストリームなど）をパイプ入力し、`predict --drama-threshold 0.75` を実行。出力されるJSONには、予測されるドラマイベント（例：`{"event": "confrontation", "participants": ["João", "Maria"], "probability": 0.89, "timeframe": "next_episode"}`）と、その根拠となる因果パスが含まれます。

3. **カスタムシナリオの生成とA/Bテスト**  
   シミュレーターで「仮説」を設定：`simulate --scenario "if_Joao_quits" --iterations 1000`。結果として、視聴率の上昇/下降予測、SNSバズ量の変化、プロデューサーが取るべき介入ポイントがリストアップされます。これを基に、実際の編集方針やプロモーション戦略を調整します。

### ベストプラクティス

- **データの鮮度を保つ**  
  ドラマ予測の精度は、直近のSNSトレンドと参加者の発言に大きく依存します。`bbb26dramapredictor` の `--live-feed` モードを常時有効にし、少なくとも5分ごとにデータをリフレッシュすることを推奨します。特にBBB26の「ライブ放送中」は、APIレートリミットを緩和した専用エンドポイントを利用してください。

- **予測結果を「絶対」と見なさない**  
  本ツールはあくまで「確率的な指針」を提供します。プロデューサーは、予測スコアが高いイベントに対して「事前にカメラアングルを増やす」「編集で伏線を強調する」などの戦略を取る一方で、予測外の出来事（いわゆる「想定外のカオス」）にも対応できるよう、常にバッファを用意しておくべきです。
