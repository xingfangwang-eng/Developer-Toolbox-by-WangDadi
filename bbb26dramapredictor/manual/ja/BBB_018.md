# bbb26dramapredictor - BBB_018

## プロダクト概要

bbb26dramapredictorは、BBB26（Big Brother Brasil 26）をはじめとするリアリティショーの**ドラマ発生確率・キャラクター間コンフリクト強度・視聴者エンゲージメント変曲点**を、機械学習とソーシャルシグナル解析で予測するプロフェッショナル向けSaaSツールです。従来の「直感と編集経験」に依存したドラマ予測を、**リアルタイムNLP + グラフ理論**で置き換えます。

## 3つのハードコア機能

### 1. コンフリクト・テンション・スコアリング (CTS)
各出演者の発言・行動ログを**感情極性・回避/攻撃性・社会的距離**の3軸でスコアリング。閾値を超えた瞬間にアラートを発火し、**「次に誰が誰と対立するか」** を確率付きで提示します。内部ではTransformerベースのマルチモーダルモデル（音声トーン + テキスト + 表情変化）を採用。

### 2. 視聴者離脱点予測エンジン (VDP)
チャットログ・SNS投稿・視聴率データから、**「どのシーンで視聴者が離脱／再エンゲージするか」** を時系列予測。編集チームが「このカットを残すべきか」を判断するための**最適カットポイント**を秒単位で提案します。

### 3. キャラクター・アライアンス・グラフ (CAG)
出演者間の**同盟・敵対・中立関係**を動的グラフで可視化。各エピソード後にグラフの**クラスタ係数・媒介中心性**を計算し、**「裏切りが起きやすい構造的隙間」** を特定。プロデューサーがキャスティングや編集方針をデータドリブンで決定できます。

## 3ステップのプロフェッショナルワークフロー

### ステップ1: データパイプラインのセットアップ
BBB26の公式ストリーム・SNSフィード・チャットログを**bbb26dramapredictor CLI**または**REST API**に投入。自動で言語正規化（ポルトガル語/スペイン語/英語対応）とタイムスタンプアライメントが行われ、**生データ → 構造化イベントログ**に変換されます。`bbb26 ingest --source twitch --channel bbb26live` のようなワンライナーで開始可能。

### ステップ2: カスタムドラマ閾値の調整
デフォルトのCTS閾値（例: 攻撃性スコア > 0.75）を、自社の編集ポリシーやターゲット視聴者層に合わせてチューニング。`config.yaml` に以下のように記述します：
```yaml
drama_thresholds:
  conflict_probability: 0.65
  viewer_drop_risk: 0.4
  alliance_instability: 0.8
```
これにより、**「軽い言い争い」と「視聴者が離脱する深刻な対立」** を区別してアラートが発報されます。

### ステップ3: レポート生成とフィードバックループ
各エピソード終了後、**自動生成レポート**（HTML/PDF/Slack Webhook対応）を確認。予測と実際の視聴者反応を比較し、**モデルの精度を改善するためのフィードバックデータ**をツールに返送します。`bbb26 feedback --episode 26 --actual_drop 12.3%` で学習データとして再投入可能。

## 業界ベストプラクティス

### 1. 編集ルームでの「予測 vs 実測」ダッシュボード運用
bbb26dramapredictorの**ライブダッシュボード**を編集ルームのサブモニターに常時表示。CTSが上昇し始めたら、編集チームは**「そのシーンの尺を伸ばすか、カットするか」** をリアルタイムで判断します。放送後に必ず**予測精度レポート**を確認し、閾値を週次でチューニングするサイクルを推奨します。

### 2. マルチプラットフォーム・クロスバリデーション
BBB26はTV放送 + YouTube + Twitch + Globoplayと複数プラットフォームで展開されます。bbb26dramapredictorは各プラットフォームの**視聴者エンゲージメントデータ**をクロスバリデーションし、**「どのプラットフォームでドラマが最も効果的に伝わるか」** を分析。例えば、Twitchのチャット速度が急上昇したタイミングとTV視聴率の相関を自動計算し、次回のマルチ配信戦略に活用できます。
