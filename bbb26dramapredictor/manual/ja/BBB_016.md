# bbb26dramapredictor - BBB_016

## プロジェクト概要

bbb26dramapredictor は、BBB26（Big Brother Brasil 26）を中心としたリアリティショーのドラマ展開を、自然言語処理と時系列分析を用いてリアルタイムに予測するSaaSツールです。視聴者・メディアバイヤー・プロデューサー向けに、エピソード間の感情遷移・同盟変動・視聴者反応を自動解析し、次回の「炎上ポイント」や「脱落確率」を高精度でスコアリングします。

## 3つのハードコア機能

### 1. マルチモーダル感情遷移トラッカー
エピソードの字幕・音声トーン・チャットログをクロス分析し、参加者ごとの**感情ベクトル**（怒り・悲しみ・喜び・困惑）を時系列プロット。API経由で5分ごとに更新されるため、放送中のリアルタイムドラマ予測が可能。

### 2. 同盟崩壊確率エンジン（ACE）
SNA（Social Network Analysis）とゲーム理論を組み合わせ、参加者間のインタラクション頻度・発言の矛盾率・投票履歴から、**72時間以内の同盟崩壊確率**をパーセンテージで出力。閾値を超えるとWebhookでアラート発報。

### 3. 視聴者センチメント・コントラストマップ
Twitter/X・Reddit・BBB公式フォーラムの投稿をLLMでラベリングし、**視聴者と参加者の感情のズレ**をヒートマップ可視化。プロデューサーは「どのシーンが視聴者に誤解されているか」を即座に把握できる。

## 3ステップのセットアップ

### ステップ1: 環境変数とAPIキーの注入
```bash
export BBB26_API_KEY="your_api_key_here"
export TWITTER_BEARER_TOKEN="your_bearer_token"
export OPENAI_API_KEY="sk-..."  # 感情分析用LLM
```
`config.yaml` に `target_language: ja` を追記し、日本語データセットの最適化を有効化。

### ステップ2: 初回データパイプラインの実行
```bash
docker-compose up -d postgres redis
python3 pipelines/ingest_bbb26_episodes.py --season 26 --episodes 1-10
```
これにより、過去エピソードのスクリプト・投票履歴・SNSデータが自動でベクトルDB（Qdrant）に格納される。

### ステップ3: ダッシュボードの起動とリアルタイム監視
```bash
streamlit run dashboard/app.py --server.port 8501
```
ブラウザで `http://localhost:8501` にアクセスし、**「ドラマ予測モード」** をONに。5分ごとに感情遷移グラフが自動更新され、同盟崩壊アラートがポップアップ表示される。

## 2つの業界ベストプラクティス

### 1. データソースの重み付けを動的に調整せよ
BBB26のドラマ予測では、**公式放送の字幕データ**に60%の重み、SNS投稿に30%、チャットログに10%を割り当てるのが初期設定。しかし、放送終了後30分以内はSNSデータの重みを50%に引き上げることで、**視聴者のリアルタイム反応をより正確に捉えられる**。`config/weight_policy.json` で時間帯別の重みテーブルを定義推奨。

### 2. アラート閾値は「過去3エピソードの標準偏差」で自動調整
固定閾値（例: 同盟崩壊確率 > 80%）は誤報率が高い。代わりに、**過去3エピソードの予測スコアの平均±2σ**を動的閾値として設定する。実装例：
```python
from scipy import stats
threshold = np.mean(recent_scores) + 2 * np.std(recent_scores)
if ace_score > threshold: send_alert()
```
これにより、シーズン序盤のノイズと終盤の高緊張状態を区別できる。
