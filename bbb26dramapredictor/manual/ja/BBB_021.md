# bbb26dramapredictor - BBB_021

## 概要

BBB26のドラマ展開を機械学習で予測するSaaSツール。視聴者の感情分析、発言のトーン変化、編集パターンから、次に起こる衝突・同盟・脱落を高精度で推論します。単なる統計ツールではなく、リアルタイムで変動する「番組内の人間関係力学」を定量化し、予測可能な形に落とし込みます。

## コア機能

### 1. 感情ベクトル解析エンジン
発言の音声トーン・テキスト感情・表情変化を3軸でスコア化。各出演者の「現在のストレス値」「攻撃性指数」「連帯確率」を0.0〜1.0でリアルタイム出力。従来のLIWCやIBM Watsonを超え、**BBB特有のブラジルポルトガル語の感情表現（ex: "mermão" のトーン変化）** を学習済み。

### 2. 同盟崩壊トリガー検出
過去12シーズンのBBBデータ（発言タイムスタンプ、投票パターン、編集上のクローズアップ頻度）を基に、**「同盟が崩壊する24時間前のシグナル」** を検出。具体的には、特定のペアの発言間隔が急激に短くなる、または相手を称賛する発言が突然減少するパターンを拾う。感度は95%以上。

### 3. 動的シナリオ分岐予測
現在の状況から、**3つの分岐シナリオ**（例: 「AがBを裏切るルート」「Cが介入するルート」「全員が膠着するルート」）を確率付きで生成。各シナリオに対して、過去データから導出した「最も似ている事例」とその結果をリンク表示。単なる確率ではなく、**「なぜそのシナリオが起こりうるのか」の根拠を可視化**。

## 使い方

### 1. リアルタイムデータフィードの設定
```bash
# まず、BBB26の公式ストリームまたはAPIエンドポイントを指定
bbb26dramapredictor --source "https://api.globo.com/bbb26/live" \
                    --language pt-BR \
                    --output-format json \
                    --confidence-threshold 0.7
```
このコマンドで、発言・表情・編集タイミングの生データが自動収集され、感情ベクトル解析エンジンに投入されます。`--confidence-threshold` は、予測を出力する最低信頼度。0.7未満のノイズはフィルタリング。

### 2. ドラマスコアのダッシュボード確認
```bash
# ブラウザでダッシュボードを起動
bbb26dramapredictor dashboard --port 8080 \
                              --refresh-interval 5s \
                              --export-csv /var/log/bbb26_scores.csv
```
ダッシュボードには、各出演者の現在の**感情スコア**と**同盟確率マトリックス**が表示。5秒ごとに自動更新。CSVエクスポート機能で、後から分析するための生データも保存可能。

### 3. シナリオシミュレーションの実行
```bash
# 特定の状況を仮定してシミュレーション
bbb26dramapredictor simulate --scenario "player_A: betray, player_B: confront" \
                             --timeline 48h \
                             --output /tmp/simulation_results.json
```
`--scenario` で「もしAが裏切り、Bが対決したら？」のような仮説を指定。`--timeline` でシミュレーション期間（48時間後まで）。結果JSONには、各シナリオの確率、過去の類似事例へのリンク、推奨される視聴ポイント（何分後に注目すべきか）が含まれます。

## ベストプラクティス

### 1. データソースの冗長化
BBBの公式APIがダウンした場合に備え、**Twitterのハッシュタグ#BBB26** や **Redditのr/bbb26** をバックアップソースとして設定する。`--fallback-source` オプションで指定可能。これにより、リアルタイム性を維持しつつ、データ欠損による予測精度低下を防ぐ。

### 2. 予測結果のA/Bテスト
予測を公開する前に、**過去のBBBシーズンデータでバックテスト** を行う。`bbb26dramapredictor backtest --season bbb24 --output accuracy_report.pdf` で、過去シーズンに対する予測精度（Precision/Recall/F1）を自動生成。特に「同盟崩壊トリガー検出」の偽陽性率が5%を超える場合は、感情スコアの閾値を調整する。
