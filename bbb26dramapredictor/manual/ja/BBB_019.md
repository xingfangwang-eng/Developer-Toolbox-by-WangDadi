# bbb26dramapredictor - BBB_019

## 概要

bbb26dramapredictor は、BBB26（Big Brother Brasil 26）の参加者間の感情推移・同盟崩壊・投票連鎖を、NLP＋時系列予測モデルでリアルタイムに可視化するプロフェッショナルツールです。従来の「視聴者直感」に依存したドラマ予測を、データ駆動型の確率モデルに置き換えます。

---

## ハードコア機能

### 1. 感情アーク・テンソル可視化（Sentiment Arc Tensor）
参加者の発言・SNS投稿・ライブ反応を**3軸（ポジティブ/ネガティブ/攻撃性）**でベクトル化し、時間軸上にアーク（弧）としてプロット。各アークの交差点が「衝突イベント」の発生確率80%以上を自動フラグ付けします。

### 2. 同盟崩壊チェーン予測（Alliance Fracture Chain）
過去12時間のインタラクション・グラフを入力に、**グラフニューラルネットワーク（GNN）** が「どの同盟がいつ崩壊するか」を確率付きで出力。崩壊後24時間以内に発生する投票連鎖（Vote Cascade）も同時予測します。

### 3. マルチモーダル・ドラマスコア（MM-DramaScore）
テキスト（発言）+ 音声トーン（怒り/悲しみ）+ 表情変化（OpenFace解析）を統合し、**0.0〜1.0のドラマスコア** を生成。スコア0.7以上で「今夜の注目シーン」として自動ハイライト抽出。

## 実践的な操作手順

### 1. データソースのバインドと前処理
```
# 1a. Twitch/Globoplay のライブチャットを WebSocket でリアルタイム取得
bbb26 ingest --source twitch --channel bbb26 --format protobuf

# 1b. 日本語字幕（CC）を Whisper でトランスクライブし、感情ラベルを付与
bbb26 annotate --input ./streams/2025-02-14_21h.ts --output ./annotations/ --model sentiment-ja-v2

# 1c. 参加者ごとのタイムラインを Parquet に統合
bbb26 merge --annotations ./annotations/ --participants bb26_roster.json --output ./merged/
```
**注意**: 初回実行時は `bbb26 config --set language=ja` で日本語パイプラインを有効化してください。

### 2. ドラマ予測モデルのトレーニングと推論
```python
from bbb26.predictor import DramaPredictor

# 過去3シーズンのアノテーションデータでファインチューニング
predictor = DramaPredictor(backbone="roberta-base-ja")
predictor.fine_tune(
    train_data="./merged/",
    epochs=5,
    batch_size=16,
    learning_rate=2e-5
)

# 現在のセッションから48時間先のドラマスコアを予測
forecast = predictor.forecast(
    session_path="./merged/latest.parquet",
    horizon_hours=48,
    confidence_interval=0.95
)
print(forecast.top_events)  # 例: [(0.83, "Alane vs. Davi"), (0.76, "Paredão surprise")]
```

### 3. カスタムアラートルールの設定とデプロイ
```yaml
# alerts.yaml
rules:
  - name: "同盟崩壊警報"
    condition: "fracture_probability > 0.75"
    action: "slack_webhook + discord_embed"
  - name: "感情急降下"
    condition: "sentiment_delta < -0.4 in 30min"
    action: "push_notification + email_alert"
```
```bash
# ルールを適用し、バックグラウンドデーモン起動
bbb26 alerts apply --file alerts.yaml
bbb26 daemon start --port 8080 --workers 4
```

## 業界ベストプラクティス

### 1. コンテキストウィンドウの最適化
BBシリーズは長時間ライブ配信のため、**感情アークの計算ウィンドウは30分固定ではなく、イベント密度に応じて動的に調整**（例：Paredão前後は10分、深夜帯は60分）すべきです。`bbb26 config --window=adaptive` で自動調整モードを推奨。

### 2. マルチソースアンサンブルによる誤報低減
単一ソース（例：Twitchチャットのみ）に依存すると、荒らしやボット発言でドラマスコアが歪みます。最低でも**チャット + 公式CC + 参加者SNS**の3ソースをアンサンブルし、各ソースの重みを過去の予測精度で動的に更新する「Bayesian Weighted Ensemble」を実装してください。
