# bbb26dramapredictor - BBB_030

## 概要

BBB26（Big Brother Brasil 26）のリアリティショーにおける人間関係の衝突、脱落リスク、視聴者投票バイアスを機械学習で予測する分析ツール。ソーシャルメディアのセンチメント分析、編集カットの頻度解析、過去シーズンのナラティブパターンを組み合わせ、放映前24時間以内に「次に起こるドラマ」を特定する。SEOキーワード：BBB26ドラマ予測、リアリティショー分析、脱落者予想、視聴者操作検出。

## 詳細な内容

### とは bbb26dramapredictor？

bbb26dramapredictorは、BBB26の生放送データ（Twitter/Xのハッシュタグ、YouTubeコメント、Globoplayの視聴率指標）をリアルタイムでクロールし、編集チームが意図的に作り出す「ドラマの伏線」を統計的に抽出するSaaSプロダクト。従来の「直感」に頼る予想を、因果推論と時系列クラスタリングで置き換える。

## コア機能

1. **編集操作検出エンジン**：番組の編集カット間隔、BGMのトーン変化、出演者の表情フレーム頻度を解析し、プロデューサーが視聴者を誘導する「ドラマのトリガー」を特定。例えば、特定の出演者に対するカット時間が3秒以上増加した場合、そのセグメントは次の48時間で炎上する確率が78%であることを警告。

2. **ソーシャル・センチメント・スパイク予測**：ツイートの感情スコア（Valence-Arousalモデル）とリツイート速度の微分値を監視し、通常のトレンドから逸脱する「バーストポイント」を予測。これにより、視聴者が「今まさに」怒りや共感を爆発させる瞬間を特定。

3. **脱落リスク・カスケードモデル**：過去12シーズンの投票データを学習したLightGBMモデルを使用。出演者の相互フォロー解除、夜間の会話頻度、編集での「悪役」フレームの割合を特徴量として、次のParedão（脱落投票）での脱落確率を95%信頼区間で出力。

## 使い方

1. **リアルタイム・パイプラインの起動**：
   `python main.py --mode stream --source twitter --window 60` を実行。これにより、BBB26関連のツイートを60秒ごとにバッチ処理し、感情スコアとトピッククラスタを生成。出力はJSON形式で、`./data/stream_output`に保存。異常値が検出されると、Slack Webhook経由でアラートが飛ぶ。

2. **ドラマ・トリガーの可視化**：
   `streamlit run dashboard.py -- --season 26` でダッシュボードを起動。左サイドバーで「編集操作インデックス」を選択し、タイムライン上に赤いマーカーが表示される箇所をクリック。そこから過去15分の編集カットログ（秒単位）と、該当する出演者のツイート感情変化グラフが表示される。

3. **脱落予測のエクスポート**：
   `python predict_elimination.py --model cascade_v3 --output ./reports/elimination_risk.csv` を実行。出力CSVには、各出演者の脱落確率、特徴量重要度（SHAP値）、信頼区間が含まれる。このデータをGloboplayのAPIにマージし、視聴者投票のシミュレーションを実行可能。

## ベストプラクティス

- **APIレート制限の回避**：Twitter API v2のアカウント制限（1アカウントあたり50万ツイート/月）を超える場合、複数のAPIキーをローテーションし、`rate_limiter`モジュールでリクエスト間隔を動的に調整する。具体的には、`min_interval=2.5`秒に設定し、429エラーが発生した場合は自動的にバックオフ（指数関数的遅延）を適用。

- **編集バイアスのキャリブレーション**：モデルが「編集操作」と「自然なドラマ」を誤分類しないよう、事前にシーズン1～25の編集ログ（手動ラベル付き）でファインチューニングを行う。特に、ブラジルのリアリティショー特有の「Celebration Montage」（祝福シーンの編集）は、感情スコアが異常に高いにもかかわらず、ドラマトリガーではないため、このパターンを除外するルールを`config.yaml`に記述。
