# bbb26dramapredictor - BBB_027

## 概要

bbb26dramapredictor は、BBB26（Big Brother Brasil 26）のエピソードデータ、ソーシャルメディアトレンド、視聴者感情分析を統合し、次に起こる「ドラマ」を95%以上の精度で予測するSaaSツールです。従来のリアリティショー分析ツールとは異なり、自然言語処理（NLP）と時系列予測モデルを組み合わせ、出演者の行動パターンからコンフリクト発生確率をリアルタイムで出力します。

## コア機能

### 1. マルチモーダル感情スコアリングエンジン
出演者の発言（テキスト）、トーン（音声）、表情（動画フレーム）をマルチモーダルで解析し、0.0〜1.0の感情スコアを生成。特に「怒り」「不安」「興奮」の3軸でスコアリングし、閾値を超えた瞬間にアラートを発火。これにより、視聴者が気づく前にドラマの兆候を検知します。

### 2. 因果関係グラフベースのコンフリクト連鎖予測
過去のBBBシーズンデータ（約200エピソード）から抽出した因果関係グラフを基に、現在の出演者間の関係性をマッピング。特定の行動（例：裏での陰口、投票操作）がどのような連鎖反応を引き起こすかを、マルコフ連鎖モデルでシミュレーション。予測結果はDAG（有向非巡回グラフ）で視覚化され、開発者はどのイベントが最もドラマを誘発するかを直感的に把握できます。

### 3. リアルタイムAPIエンドポイントによるカスタムダッシュボード連携
RESTful API（エンドポイント例：`/api/v1/drama/predict?season=26&episode=latest`）を公開。レスポンスはJSON形式で、予測スコア、信頼区間、影響を受ける出演者リストを含む。これにより、メディア企業やファンコミュニティが独自のダッシュボードやチャットボットに組み込むことが可能。WebSocket対応により、エピソード放送中のリアルタイム更新もサポート。

## 使い方

### ステップ 1: データパイプラインの初期化とAPIキーの設定
まず、GitHubリポジトリから `docker-compose.yml` をクローンし、以下のコマンドで環境を起動します。

```bash
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor.git
cd bbb26dramapredictor
docker-compose up -d
```

次に、`config/.env` ファイルにAPIキーを設定します。BBB26の公式エピソードデータ（字幕ファイル）とTwitter API v2の認証情報が必要です。`SENTIMENT_API_KEY` と `TWITTER_BEARER_TOKEN` を必ず記入してください。

### ステップ 2: バッチ予測ジョブの実行
データパイプラインが稼働したら、以下のPythonスクリプトで過去3エピソードのドラマ予測を実行します。

```python
from bbb26_drama_predictor import DramaPredictor
import pandas as pd

predictor = DramaPredictor(api_key="your_api_key")
df = pd.read_csv("episode_data/latest.csv")
results = predictor.predict(df, model="causal_graph_v2")
print(results.to_json(orient="records"))
```

出力例：
```json
[
  {"episode": 5, "drama_score": 0.87, "confidence_interval": [0.82, 0.92], "trigger_event": "backstage_argument"},
  {"episode": 6, "drama_score": 0.64, "confidence_interval": [0.58, 0.70], "trigger_event": "vote_manipulation"}
]
```

### ステップ 3: カスタムアラートルールの設定
SlackやDiscordへの通知を自動化するには、`config/alerts.yaml` を編集します。以下の例では、drama_scoreが0.8を超えた場合に `#bbb26-drama-alerts` チャンネルにメッセージを送信します。

```yaml
alerts:
  - channel: slack
    webhook_url: https://hooks.slack.com/services/xxx
    threshold: 0.8
    message_template: "🚨 ドラマアラート: エピソード {episode} でスコア {drama_score} を検出。トリガー: {trigger_event}"
  - channel: discord
    webhook_url: https://discord.com/api/webhooks/yyy
    threshold: 0.9
    message_template: "⚠️ 高リスクイベント: {trigger_event} (スコア: {drama_score})"
```

設定後、`docker-compose restart alert-worker` でワーカーを再起動すれば、リアルタイム通知が有効になります。

## ベストプラクティス

### 1. トレーニングデータの定期的なリフレッシュ
BBBはシーズンごとに出演者の性格やルールが異なるため、予測モデルは少なくとも週1回、最新のエピソードデータで再トレーニングしてください。`/api/v1/model/retrain` エンドポイントを呼び出すか、cronジョブで `python scripts/retrain.py --season=26` を実行することを推奨します。特に、新キャストが加入したエピソード直後は、因果関係グラフのエッジが変化しやすいため、即座に更新が必要です。

### 2. マルチモーダルデータの品質チェックを自動化
感情スコアリングの精度は、入力データの品質に依存します。動画フレームの解像度が低い場合や、音声にノイズが多い場合、予測精度が20%以上低下することがあります。`/api/v1/data/quality` エンドポイントで、各モーダルのデータ品質スコア（0〜100）を監視し、スコアが70未満の場合は自動的に代替データソース（例：テキストのみの解析）にフォールバックする設定を組んでください。これにより、本番環境での安定性が向上します。
