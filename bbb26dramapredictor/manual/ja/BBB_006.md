# bbb26dramapredictor - BBB_006

## 概要

bbb26dramapredictor は、BBB26（Big Brother Brasil 26）のリアリティショーにおける参加者間のドラマ発生を、ソーシャルメディアの感情分析と過去の行動パターンから高精度に予測する SaaS ツールです。視聴者、メディア関係者、マーケティング担当者向けに設計されています。

## コア機能

### 1. リアルタイム感情スコアリング & ドラマトリガー検知
Twitter / Instagram / TikTok の投稿をリアルタイムでクロールし、各参加者の感情スコア（怒り・嫉妬・興奮・悲しみ）を数値化。特定の閾値を超えた場合、ドラマ発生確率をアラートとして通知します。NLP モデルはポルトガル語のスラングやネットミームにも対応。

### 2. 参加者間の関係性グラフ & コンフリクト予測マップ
各参加者の発言・行動履歴から、人間関係のネットワークグラフを自動生成。過去の類似パターン（例：特定の組み合わせでの対立率）を機械学習で学習し、24時間以内に発生しうるコンフリクトを確率付きで可視化します。

### 3. シナリオシミュレーター & バックテストエンジン
「もし参加者AがBに対してこう発言したら？」という仮説を入力すると、過去のデータベースを基に結果をシミュレーション。さらに、実際の放送後に予測と結果を比較するバックテスト機能を搭載し、モデルの精度を継続的に改善できます。

## 使い方

### ステップ 1: データソースの接続と初期スコープ設定
まず、分析対象とする BBB26 のエピソード期間を指定し、Twitter API と Instagram Graph API を OAuth 2.0 で認証接続します。`config.yml` にて、監視する参加者リストと感情分析の感度パラメータ（0.0〜1.0）を設定後、`bbb26dramapredictor init --scope=episode1-5` を実行。初回は過去データのバッチ処理が走り、ベースラインスコアが生成されます。

### ステップ 2: ドラマアラートの閾値調整とカスタムルール作成
`alert_rules.json` を編集し、例えば「参加者Aの怒りスコアが0.8以上かつ、参加者Bへの言及が3回以上」という条件を追加。`bbb26dramapredictor alert --watch` でリアルタイムモニタリングを開始します。アラートは Slack Webhook または Discord Webhook に自動送信可能。閾値は A/B テスト用に複数パターン保存できます。

### ステップ 3: シナリオシミュレーションとレポート出力
予測したいシナリオを `scenario.json` に記述し、`bbb26dramapredictor simulate --input scenario.json --output report.html` を実行。出力される HTML レポートには、確率分布、影響を受けやすい参加者リスト、過去の類似事例へのリンクが含まれます。結果は CSV または JSON でエクスポート可能。

## ベストプラクティス

### 1. 感情モデルを毎週再学習させる
BBB26 は毎週新しい参加者が追加されたり、ルールが変更される可能性があります。`bbb26dramapredictor retrain --model=emotion --week=3` のように週単位でモデルを更新し、最新の言語パターンに適応させてください。特にポルトガル語のネットスラングは進化が速いため、学習データに新しい投稿を定期的に追加することを推奨します。

### 2. 複数のアラートチャネルを冗長化する
ドラマ予測はリアルタイム性が命です。Slack だけでなく、PagerDuty やカスタム Webhook にもアラートを送信し、ダウンタイムに備えてください。また、`bbb26dramapredictor health` コマンドで API の応答時間とデータ取得率を毎日チェックし、異常があれば即座にフェイルオーバー設定を有効にします。
