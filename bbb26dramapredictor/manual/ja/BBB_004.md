# bbb26dramapredictor - BBB_004

## 概要

BBB26のドラマ展開を**確率論的予測モデル**で解析するSaaSツール。従来の感情分析や単純な投票予測を超え、参加者間の相互作用グラフと発言の時系列変化を学習し、次に発生する「ドラマイベント」を高精度に予測します。SEO観点では「BBB26 ドラマ予測」「リアリティショー 分析ツール」「BBB26 参加者 相関」などのキーワードをターゲットに最適化済み。

## コア機能

### 1. **インタラクション・ネットワーク解析エンジン**
参加者間の発言回数、トーン、言及回数をもとに**リアルタイムの関係性グラフ**を自動生成。単なる「仲良い/悪い」ではなく、**影響力スコア**と**感情ベクトル**を組み合わせ、次に衝突が起きるペアを確率付きで提示。グラフ構造はD3.jsで可視化され、ノードの色や太さで直感的に理解可能。

### 2. **時系列ドラマイベント予測モデル (LSTM + Attention)**
過去のBBBシリーズ全エピソード（約2,000時間分）の字幕データとソーシャルメディアのトレンドを学習。**Attention機構**により、特定の発言や行動がどの程度後のイベントに影響を与えるかをスコア化。予測結果は「24時間以内に発生する確率」「48時間以内」など時間枠別に表示。

### 3. **自動ハイライト・スクリプト生成**
予測されたドラマイベントに基づき、**編集不要のハイライト動画スクリプト**を自動生成。時間軸に沿った発言の抜粋、BGMの提案、カット割りまで含む。API経由で動画編集ソフト（DaVinci ResolveやPremiere Pro）と連携可能。

## 使い方

### ステップ 1: データソースの統合
- **Twitter / X API** と **BBB公式配信プラットフォーム**のOAuth認証を設定。
- `config.yaml` にAPIキーとウォッチするハッシュタグ（例: `#BBB26`）を記述。
- 初回実行時は過去7日間のデータをバルクインポート。以降は15分ごとに自動更新。

```bash
# 設定ファイルの例
sources:
  twitter:
    bearer_token: "YOUR_BEARER_TOKEN"
    hashtags: ["#BBB26", "#BBB_br"]
  streaming:
    platform: "globoplay"
    access_token: "YOUR_ACCESS_TOKEN"
```

### ステップ 2: 予測モデルの実行とチューニング
- コマンドラインから `predictor run --mode live` を実行。
- ダッシュボード上で **Attentionスコア** と **ネットワーク密度** を確認。
- 必要に応じて `--threshold 0.8` で予測の感度を調整（デフォルト0.7）。
- 予測結果はJSON形式で `/output/predictions/` に保存。SlackやDiscordへのWebhook通知も可能。

### ステップ 3: ハイライトスクリプトのエクスポート
- 予測されたイベントIDを指定: `predictor highlight --event-id EVT-2023-11-15-003`
- 生成されたMarkdown形式のスクリプトを確認。時間コード、発言者、感情ラベルが自動付与。
- `--format srt` オプションで字幕ファイルとしても出力可能。

```markdown
## ハイライトスクリプト (イベントID: EVT-2023-11-15-003)
### タイムライン
- **00:03:15** — 参加者AがBに対して「あなたはいつもそう」と発言（トーン: 攻撃的, 確率: 87%）
- **00:05:40** — 参加者Cが仲裁に入る（トーン: 中立的, 確率: 62%）
- **00:08:22** — スタジオの照明が暗転（演出効果, 確率: 94%）
```

## ベストプラクティス

### 1. **訓練データのドメイン特化**
汎用的な感情分析モデルではなく、**ブラジルのポルトガル語スラング**や**リアリティショー特有の誇張表現**を学習させたカスタムモデルを使用。デフォルトの事前学習モデルは約500万ツイートでファインチューニング済み。さらにドメインを絞りたい場合は、`trainer --corpus bbb_archive.json` で追加学習可能。

### 2. **予測結果のバリデーションループ**
予測精度を上げるには、**実際のイベント発生後にフィードバックをループ**させる。`predictor validate --ground-truth actual_events.json` を実行すると、F1スコアと混同行列が生成される。このデータをもとに `config.yaml` の `learning_rate` や `batch_size` を調整。週次で自動リトレーニングするCI/CDパイプラインを組むことを推奨。
