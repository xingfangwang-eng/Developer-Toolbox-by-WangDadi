# bbb26dramapredictor - BBB_013

## 概要

BBB26の視聴者離脱を防ぐためのリアルタイムドラマ予測エンジン。ソーシャルリスニングと感情分析を組み合わせ、放送開始前から参加者間の衝突・同盟・恋愛フラグを検知します。

## 詳細な内容

### bbb26dramapredictor とは

BBB26の生放送中に発生する「ドラマの芽」を、AIが事前にキャッチするSaaSツールです。Twitterトレンド、視聴者チャット、過去シーズンの脚本パターンから確率スコアを算出し、編集チームがリアルタイムでカメラワークや演出を最適化できるように支援します。

### コア機能

1. **感情ベクトルマップ：参加者間の関係性を3D可視化**  
   各参加者の発言・表情・ソーシャル投稿をNLPで解析し、同盟・敵対・恋愛の3軸で距離をスコアリング。放送中に「関係性の急変」が発生した場合、自動アラートで編集パネルに通知します。

2. **ドラマ確率予測API：未来15分の衝突リスクを時系列出力**  
   過去5シーズンの脚本構造（例：火曜夜の同盟崩壊パターン）と現在の視聴者エンゲージメントデータを組み合わせ、**「次のCM明けに誰が誰と対立するか」** を確率付きで予測。Webhookで制作管理システムに連携可能です。

3. **視聴者離脱予測ダッシュボード：視聴率低下を未然に防ぐ**  
   視聴者のチャット感情・離脱率・リツイート速度から「退屈タイムゾーン」を特定。ドラマが不足している場合、自動で過去の名シーンや裏側映像の挿入を提案します。

### 使い方

1. **初期設定：ソーシャルソースと放送スケジュールの同期**  
   `config.yaml` にTwitter APIキー、BBB26公式放送スケジュール（JSON形式）、および参加者リストを記述。  
   ```yaml
   broadcast:
     start_time: "2025-01-15T21:00:00Z"
     participants: ["Maria", "João", "Carlos", ...]
   social:
     twitter_bearer_token: "your_token_here"
     chat_platform: "discord"
   ```
   その後 `python predictor.py --init` を実行し、過去データのキャッシュを構築。

2. **リアルタイムモニタリング：ダッシュボードの起動**  
   `streamlit run dashboard.py` でローカルサーバー起動。左パネルに「ドラマ確率マップ」、中央に「感情ベクトル3Dグラフ」、右に「離脱予測ゲージ」が表示されます。  
   特定の参加者をクリックすると、その人物の過去のドラマパターン（例：毎週水曜に怒りがピーク）が表示されます。

3. **カスタムアラートの設定：WebhookでSlack/Teamsに通知**  
   `alerts.json` に閾値を定義：
   ```json
   {
     "conflict_probability_threshold": 0.75,
     "love_interest_spike": 0.9,
     "viewer_drop_warning": 0.2
   }
   ```
   閾値を超えた場合、自動でSlackチャンネルに「⚠️ 衝突確率85%：CarlosとMariaの口論が5分後に予測」と投稿。制作チームは即座にカメラスイッチングを調整可能。

### ベストプラクティス

- **放送開始1時間前にプリロードを実行**  
  参加者の最新ツイートやInstagramストーリーをクロールし、感情ベクトルマップを初期化。これにより放送開始直後から高精度な予測が可能になります。特に「過去のシーズンで似た発言パターン」を抽出するオプションを有効にしてください。

- **視聴者離脱予測をA/Bテストのトリガーに活用**  
  離脱予測ゲージが0.3を超えた場合、自動で「過去の名場面クリップ」を挿入するバージョンと、そのまま放送を続けるバージョンをランダムに分割。視聴者維持率の差分を計測し、次週の放送設計にフィードバックします。
