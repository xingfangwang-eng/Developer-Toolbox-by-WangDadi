# bbb26dramapredictor - BBB_010

## 概要

このプロジェクトは、**BBB26（Big Brother Brasil 26）** のリアリティショーにおける人間関係のダイナミクスとドラマ発生確率を、機械学習とソーシャルグラフ分析で予測する専門ツールです。視聴者の共感・反感・投票行動をリアルタイムに解析し、次に起こる“炎上”や“同盟崩壊”を高精度で特定します。

## コア機能

### 1. ソーシャルグラフ・エッジ確率予測エンジン
参加者間の相互作用（会話時間、物理的距離、感情スコア）をグラフ構造に変換し、**エッジの重み付け** を動的に計算。特定の組み合わせが「ドラマを誘発する確率」を0〜1のスコアで出力します。閾値を超えたペアは即座にアラートとして通知可能。

### 2. 感情トレンド・トリガーディテクター
発言内容の感情分析（怒り・悲しみ・喜び・嫌悪）を時系列でプロットし、**感情の急激な勾配変化** をトリガーとして検出。過去のBBBシーズンデータと照合し、類似パターンが発生した場合に「次に起こりうるドラマタイプ（例：裏切り暴露、同盟崩壊）」を3段階の確信度で提示します。

### 3. 投票行動予測シミュレーター
過去の視聴者投票履歴・SNS上のハッシュタグ頻度・Twitter/Xの感情スコアを統合し、**次回の追放投票結果** をシミュレーション。各参加者の「生き残り確率」と「ドラマ発生時の影響度」をマトリクスで可視化します。バックエンドはLightGBM + 時系列Transformerで構成。

## 使い方

### ステップ 1: 環境セットアップとデータパイプラインの起動
```bash
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor
cd bbb26dramapredictor
pip install -r requirements.txt
python pipeline.py --start-date 2025-01-10 --end-date 2025-02-20
```
このコマンドで、BBB26の全エピソードトランスクリプト・SNSデータを自動取得し、感情スコアリングとグラフ構築を開始します。出力は `data/processed/` にパーティション化されたParquetファイルとして保存。

### ステップ 2: ドラマ予測モデルの実行
```python
from predictor import DramaPredictor

dp = DramaPredictor(model_path='models/bbb26_lgbm_v3.pkl')
result = dp.predict_drama_probability(
    participant_a='Aline',
    participant_b='Miguel',
    day=15,
    event_window='24h'
)
print(result['drama_probability'])  # 0.87
```
このAPIは、指定した参加者ペアの**24時間以内のドラマ確率**を返します。確率が0.7を超えると、自動的にWebhook経由でSlack通知が飛ぶように設定可能。

### ステップ 3: 結果の可視化とエクスポート
```bash
python visualize.py --output-format html --include-timeline
```
生成されるHTMLレポートには、以下の3つのセクションが含まれます：
- **インタラクショングラフ**（D3.jsベース、ノードの色は感情スコア）
- **感情トレンドチャート**（Plotly、怒りスコアのピークがドラマトリガーと一致）
- **投票確率マトリクス**（各参加者の追放リスクをヒートマップで表示）

## ベストプラクティス

### 1. リアルタイムデータソースの優先順位付け
SNSデータ（特にTwitter/Xのライブストリーム）は、放送データよりも感情変化を約 **2〜4時間先行** して捉えられます。必ず `stream_twitter.py` を常時実行し、WebSocket経由でモデルにフィードすることを推奨します。これにより、放送前に「次に誰が誰に怒るか」を予測可能になります。

### 2. モデル再学習のタイミングは「ドラマ発生後24時間以内」
BBBのように短期間で感情が激変するコンテンツでは、**学習データの鮮度が命** です。新しいドラマイベントが発生したらすぐに、そのイベントを含むデータセットでモデルを再学習（`retrain.py --event-id <id>`）してください。週次更新では遅すぎます。CI/CDパイプラインに `drama_trigger_retrain` フックを組み込むことを強く推奨します。
