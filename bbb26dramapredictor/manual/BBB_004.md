# bbb26dramapredictor - BBB_004

## Overview

bbb26dramapredictor is a production-grade SaaS tool designed to ingest, analyze, and forecast narrative tension arcs within the BBB26 (Big Brother Brasil 26) ecosystem. It leverages NLP sentiment drift detection, social media entropy scoring, and historical contestant behavior modeling to surface high-probability drama events before they hit the mainstream. This is not a toy—it’s a signal processor for reality show chaos.

## Core Features

### 1. Sentiment Drift & Tension Spike Prediction Engine
The core ML pipeline consumes real-time Twitter/X, Reddit, and Globo Play chat streams. It applies a fine-tuned BERT-based sentiment classifier (trained on 3+ years of BBB history) to detect **affective polarity shifts** within a 15-minute sliding window. When the sentiment delta between two or more contestant clusters exceeds a configurable threshold (default: 0.42), the system triggers a **drama probability score** (0–1.0) and emits a webhook payload. This is not "who is trending"—it’s *who is about to explode*.

### 2. Interaction Graph & Narrative Entropy Analysis
We model every on-screen interaction (confessionals, arguments, alliances) as a weighted directed graph. Nodes are contestants; edges are interaction frequency and emotional valence. The system calculates **narrative entropy** using Shannon entropy over the graph’s adjacency matrix. When entropy drops sharply (indicating a high-stakes binary conflict) or spikes (indicating a chaotic multi-player meltdown), the predictor flags the moment as a **high-impact drama candidate**. This is structural pattern recognition, not keyword matching.

### 3. Historical Archetype Matching & Outcome Simulation
Using a vector database (pgvector) of 12,000+ past contestant profiles, the system performs cosine similarity matching between current cast members and historical archetypes (e.g., "the betrayer," "the victim," "the wildcard"). For each high-probability drama event, it runs a Monte Carlo simulation (1000 iterations) to predict the most likely resolution: elimination, alliance fracture, or public backlash. The output is a **decision tree with confidence intervals**—ideal for media analysts or betting syndicates.

## How to Use

### Step 1: Bootstrap the Data Pipeline & Configure Webhooks
```bash
git clone https://github.com/xingfangwang-eng/bbb26dramapredictor.git
cd bbb26dramapredictor
cp .env.example .env
# Set your Twitter API Bearer Token, Globo Play scraper endpoint, and webhook URL
docker-compose up -d --build
```
The ingestion layer will begin streaming 3 data sources: live chat logs, tweet streams filtered by `#BBB26`, and the official Globo Play subtitle feed. Verify ingestion health via `curl http://localhost:8080/health`.

### Step 2: Train or Load the Sentiment Model (Fine-Tuning Optional)
If you want to use the pre-trained model (recommended for speed), run:
```bash
python scripts/load_model.py --model bbb26-drama-bert-v1
```
To fine-tune on the latest 30 days of data (e.g., after a major cast change):
```bash
python scripts/fine_tune.py --data_path ./data/raw/bbb26_chat_logs --epochs 3 --learning_rate 2e-5
```
The model will be saved to `./models/` and automatically loaded by the inference server. Check the logs for **F1 score** and **drama recall**.

### Step 3: Query the Prediction API & Visualize the Drama Graph
Once the pipeline is live, hit the REST API:
```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"contestant_a": "Davi", "contestant_b": "Isabelle", "window_minutes": 30}'
```
You’ll receive a JSON response like:
```json
{
  "drama_probability": 0.87,
  "predicted_resolution": "alliance_fracture",
  "confidence_interval": [0.81, 0.93],
  "entropy_delta": -0.34
}
```
For a live interactive graph, open `http://localhost:3000` (a D3.js dashboard) showing node sizes proportional to drama contribution and edge colors indicating sentiment polarity. Refresh every 60 seconds.

## Best Practices

### 1. Use Multi-Source Signal Fusion, Not Single-Threaded Scraping
Many developers scrape only Twitter. That’s a mistake. Twitter sentiment is often astroturfed by fan armies. **Always fuse Twitter with Reddit (r/BigBrotherBrasil) and Globo Play chat logs.** The predictor’s entropy engine works best when fed at least 3 independent data streams. If you only use one, you’ll get false positives from bots or echo chambers. In production, we recommend a **Kafka-based fan-out** to handle the throughput.

### 2. Calibrate the Drama Threshold Per Season (Don’t Use Defaults)
The default drama probability threshold of 0.42 was tuned on BBB24 data. BBB26 has a different cast dynamic (more volatile). **Run a calibration script** (`python scripts/calibrate.py --season bbb26`) that uses the first 7 days of data to find the optimal threshold that maximizes precision (avoiding false drama alerts) while keeping recall above 0.8. You’ll get a new threshold (likely between 0.38 and 0.48). Hardcode this in your environment variable `DRAMA_THRESHOLD`. This is the difference between being the first to know and being the 1000th to retweet.
