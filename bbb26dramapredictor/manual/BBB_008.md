# bbb26dramapredictor - BBB_008

**bbb26dramapredictor** is not a sentiment tracker. It’s a **causal narrative divergence engine** built to surface the exact moments a BBB26 storyline fractures, realigns, or implodes. Designed for data-driven reality TV analysts, social listening teams, and production strategists who need to move beyond "who got cancelled" into the mechanics of **why a scene breaks the internet**.

Built for the US developer and media analyst ecosystem: CLI-native, API-first, zero fluff.

## Core Features

### 1. Temporal Narrative Graph (TNG) – Scene-Level Causal Mapping
Instead of flat keyword clouds, **bbb26dramapredictor** constructs a directed acyclic graph of every confessional, argument, and alliance shift. Each node is a timestamped micro-event (e.g., "Participant A contradicts Participant B about Task X at 14:32 UTC"). Edges represent **causal impact scores** derived from cross-referencing live chat velocity, thread propagation, and edit timing. This lets you pinpoint the exact 90-second window that triggered a 10x spike in hate-watching.

### 2. Probabilistic Fallout Scoring (PFS) – Pre-Shift Risk Alerts
Stop reacting after the backlash. PFS assigns a **divergence probability** to every upcoming storyline branch. Using a fine-tuned transformer model on 8 seasons of BBB transcripts and Reddit/LiveJournal archives, the engine outputs a `[0.0–1.0]` score for each potential outcome (e.g., "Alliance dissolution: 0.89"). You can hook this into a Slack webhook or a custom dashboard to trigger alerts before the episode even airs.

### 3. Multi-Modal Sentiment Splicer – Cross-Platform Emotional Fission
Most tools flatten sentiment into "positive/negative/neutral." This one **splices** sentiment across three axes: **text** (tweets, Reddit threads), **audio** (confessional tone, laughter, silence), and **visual** (facial micro-expressions from live feeds). The splicer outputs a **3D emotional vector** per participant per scene. When the vector for Participant B suddenly decouples from Participant A’s vector mid-conversation, you know the drama is about to detonate.

## How to Use

### 1. Initialize a Project with a Live Feed or VOD
```bash
bbb26dramapredictor init --source "https://globoplay.globo.com/bbb26/ao-vivo" \
  --output-dir ./bbb26_analysis \
  --language pt-BR \
  --causal-depth 3
```
This spins up a local event stream processor. The `--causal-depth` flag defines how many narrative hops the engine will trace forward from a single event. For a full season, start with depth 3; for a single episode, depth 5.

### 2. Run a Probabilistic Fallout Scan on a Specific Scene
```python
from bbb26dramapredictor import PFS

scan = PFS.scene_scan(
    scene_id="2026-02-14T21:45:00Z",
    participants=["A", "B", "C"],
    context_window_minutes=15
)
print(scan.divergence_scores)
# Output: {'alliance_break': 0.89, 'public_vote_shift': 0.73, 'silent_treatment': 0.12}
```
The `context_window_minutes` parameter is critical: it ingests the 15 minutes of chat, audio, and visual data **surrounding** the scene. Increase it to 30 if you’re analyzing a slow-burn betrayal.

### 3. Export a Temporal Narrative Graph as JSON for Custom Visualization
```bash
bbb26dramapredictor export \
  --project ./bbb26_analysis \
  --format json \
  --edges-only \
  --min-causal-weight 0.4 > narrative_graph_episode_5.json
```
This outputs a DAG where each edge has a `weight` (causal strength) and `type` (e.g., "contradiction", "emotional_mirror", "strategic_alignment"). You can pipe this directly into D3.js, Cytoscape, or your own React-based graph viewer. The `--min-causal-weight` flag filters out noise; start at 0.3 for raw exploration, 0.6 for clean dashboards.

## Industry Best Practices

### 1. Integrate the PFS Score into Your Social Listening Pipeline
Don’t just monitor hashtags. Feed the `divergence_scores` from **bbb26dramapredictor** into your existing social media analytics tool (e.g., Brandwatch, Sprout Social) as a custom metric. When the `public_vote_shift` score exceeds 0.85, automatically flag the participant for brand safety review or prepare a crisis comms template. This turns your tool from a passive observer into a proactive risk management layer.

### 2. Use the TNG Export for Post-Season Narrative Autopsy
After the season ends, run the full export with `--min-causal-weight 0.2` to build a complete narrative map. Overlay it with engagement metrics (viewership, ad revenue, tweet volume). You’ll discover that **not all high-drama scenes drive engagement**—some are narrative dead ends. Use this to advise production on which emotional beats to double down on next season. This is how you move from "reality TV analyst" to "narrative architect."
