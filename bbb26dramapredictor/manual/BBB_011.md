# bbb26dramapredictor - BBB_011

## Overview

bbb26dramapredictor is a data-driven SaaS tool designed to forecast narrative arcs, conflict spikes, and viewer sentiment shifts in the Brazilian reality show **Big Brother Brasil 26 (BBB26)**. By integrating real-time social media scraping, NLP-based emotion mapping, and probabilistic modeling, this engine transforms raw audience chatter into actionable drama intelligence. It is built for producers, media analysts, and superfans who demand precision over guesswork.

### Why This Matters
In a show where every whisper can trend and every vote can shift alliances, predicting drama is not a luxury—it’s a competitive edge. bbb26dramapredictor eliminates blind spots by quantifying the unquantifiable: tension, loyalty breakdowns, and viral moments before they explode.

## Core Features (Hardcore & Product-Specific)

### 1. **Conflict Probability Heatmap (CPH)**
A proprietary algorithm that scans Twitter, Reddit, and Instagram for keyword clusters, emoji sentiment, and reply chain velocity. It outputs a real-time heatmap of **likely conflict zones** (e.g., “Brother X vs. Sister Y during the Paredão vote”). The model factors in historical data from BBB25 and BBB24 to calibrate threshold triggers.

### 2. **Sentiment Drift Analyzer (SDA)**
Instead of static polarity scores, SDA tracks **micro-shifts in audience allegiance** over 15-minute windows. It detects when a participant’s fanbase begins to fracture (e.g., from “strong support” to “mixed loyalty”) and predicts the tipping point for a backlash event. Outputs are visualized as a Gantt-style drift timeline.

### 3. **Narrative Fork Predictor (NFP)**
Using a Markov chain model trained on past BBB seasons, NFP identifies **critical decision nodes** (e.g., leader nominations, veto power usage, or spontaneous confessionals). It then generates three probabilistic story branches (e.g., “Alliance collapse: 68%,” “Unexpected romance: 22%,” “Production intervention: 10%”). This is the closest thing to a storyboard for reality chaos.

## How to Use (Detailed, Professional Steps)

### Step 1: Initialize the Data Pipeline
- Clone the repo and run `docker-compose up` to spin up the ingestion stack (Kafka + Redis + Postgres).
- Configure your Twitter API v2 credentials and Reddit OAuth in `.env`.
- Execute the bootstrap script: `python bootstrap.py --season bbb26 --live`. This triggers a historical sync of the last 72 hours of social data and starts real-time streaming.
- Validate the pipeline health via the built-in Grafana dashboard at `localhost:3000`. Look for “Ingestion Rate > 500 tweets/min” and “Zero dead-letter queue errors.”

### Step 2: Run a Conflict Prediction Session
- Open the web UI at `http://localhost:8080` and select **“Live Session: BBB26”**.
- In the **“Target Participants”** field, input up to 5 housemates (e.g., `João, Maria, Pedro`).
- Click **“Analyze Conflict Probability”**. The engine will process the last 4 hours of data and return a CPH heatmap with confidence intervals (e.g., “João vs. Maria: 87% ± 3%”).
- For a deeper dive, click **“Drill Down”** to see the top 10 triggering tweets and their sentiment velocity curves.

### Step 3: Export & Automate Alerts
- After analysis, export the narrative fork predictions as JSON or CSV via the **“Export Branch Data”** button.
- Set up a webhook in `Settings > Alerts` to push predictions to your Slack or Discord channel. Example config:
  ```json
  {
    "webhook_url": "https://hooks.slack.com/services/xxx",
    "trigger_on": "conflict_probability > 75%",
    "frequency": "every 15 minutes"
  }
  ```
- Verify the webhook fires by tweeting a test keyword (e.g., “#BBB26 #treta”) and checking your channel within 2 minutes.

## Best Practices (Industry-Grade Advice)

### 1. **Calibrate Your Baseline with Historical Data**
Don’t rely solely on live data. Before the season starts, run the **“Seed Mode”** command (`--seed --season bbb25`) to train the Markov model on past alliance shifts and voting patterns. This increases NFP accuracy by up to 40% during the first two weeks when drama signals are sparse.

### 2. **Combine SDA with External Triggers**
The Sentiment Drift Analyzer is powerful, but it’s blind to production edits. Pair its output with a manual event log (e.g., “Brother X received a hidden immunity idol at 20:30”). Use the `/api/events` endpoint to inject these triggers. The engine will then re-calibrate drift probabilities in real time, giving you a hybrid prediction that accounts for both audience chatter and editorial manipulation.
