# bbb26dramapredictor - BBB_014

Reality TV analytics has never been this surgical. **bbb26dramapredictor** ingests live social sentiment, contestant behavioral micro-signals, and historical voting patterns to predict the next explosive twist in BBB26 (Big Brother Brasil 26) before it even airs. Built for data journalists, fan engagement platforms, and OTT content strategists who treat drama as a measurable metric.

## Hardcore Features

1. **Sentiment Fracture Detection**  
   Monitors Twitter/X, Reddit, and fan forum threads in real-time, applying a custom NLP model trained on 3+ seasons of BBB transcripts. Flags “fracture points” where audience sentiment diverges from on-screen narrative—these are the exact moments a producer is likely to cut to a confrontation or a surprise eviction.

2. **Probabilistic Storyline Graph**  
   Instead of linear predictions, we generate a directed acyclic graph (DAG) of possible drama arcs. Each edge is weighted by historical similarity to past BBB seasons (e.g., “Vote Reversal” or “Secret Room Betrayal”). The engine outputs a confidence score for each arc, updated every 15 minutes based on live data streams.

3. **Contestant Micro-Expression & Speech Pattern Analyzer**  
   Integrates with video feeds (via OpenCV + Whisper ASR) to detect micro-expressions (anger, surprise, contempt) and speech tempo anomalies. Combined with the emotional load of keywords used, this creates a “Drama Risk Index” (0–100) per contestant per scene—usable for automated highlight reel generation.

## How to Use (Production-Grade Workflow)

1. **Boot the Prediction Pipeline**  
   Clone the repo and run `docker-compose up -d` to spin up the core stack (PostgreSQL + Redis + Celery workers + FastAPI). Then execute `python bootstrap.py --season bbb26` to seed the model with the current season’s contestant profiles and initial voting data. The system immediately begins scraping 8 data sources (Twitter API v2, Reddit Pushshift, GloboPlay transcripts, etc.).

2. **Query a Live Prediction**  
   Use the REST API endpoint `GET /v1/predict/drama-score?contestant_id=42&scene_id=live` to get a JSON response containing:  
   - `drama_score` (0–100)  
   - `top_3_arcs` (array of arc IDs with probabilities)  
   - `sentiment_breakdown` (positive/negative/neutral per source)  
   - `micro_expression_summary` (last 60 seconds)  
   Example: `curl -X GET "http://localhost:8000/v1/predict/drama-score?contestant_id=42&scene_id=live"`

3. **Trigger an Automated Alert**  
   Configure a webhook in `config/alerts.yaml` to fire when a specific contestant’s Drama Risk Index exceeds 75 for more than 5 consecutive minutes. The webhook can post to Slack, Discord, or your own custom endpoint with a pre-formatted message including the predicted arc and a GIF of the triggering moment.

## Industry Best Practices

- **Don’t Overfit to One Season**  
  Retrain the sentiment fracture model on at least 2 previous BBB seasons (e.g., BBB24, BBB25) before using it for BBB26. Reality shows evolve their editing style and contestant archetypes—your model must generalize across seasons, not memorize last year’s fights.

- **Use the Drama Risk Index for Content Scheduling**  
  If you’re running a fan engagement platform or a live commentary channel, schedule your most active moderation or host presence during windows where the DRI for the top 3 contestants is above 65. This maximizes viewer retention and ad revenue during high-drama peaks.
