## Core Features

### 1. Sentiment-Arc Engine (SAE)
A transformer-based model fine-tuned on Portuguese and English reality show transcripts. It doesn’t just score sentiment—it maps emotional tension curves per contestant over 15-minute intervals, flagging **pre-breakout drama** 2–4 hours before it hits the live feed.

### 2. Contestant Pairing Graph (CPG)
A directed graph that models **interaction weight** between cast members using co-occurrence in confessionals, live interactions, and social media mentions. The CPG outputs a “drama heat index” for every possible pair, updated every 30 minutes via cron-triggered API calls.

### 3. Multi-Lingual Narrative Drift Detector (NDD)
Analyzes real-time subtitles and Twitter/X threads in Portuguese, Spanish, and English to detect **narrative forks**—moments when the show’s editorial line diverges from audience perception. Outputs a JSON diff with confidence scores, ready for ingestion into any analytics dashboard.

## How to Use

### Step 1: Initialize the Pipeline with Environment Variables
```bash
export BBB26_API_KEY="your_key_here"
export BBB26_LANG="pt-BR,en,es"
export BBB26_INTERVAL_MIN=30
```
Run the bootstrap script to pull the latest episode metadata and seed the CPG:
```bash
python bootstrap.py --seed --mode live
```
This validates your API token, downloads the latest contestant roster from the official BBB26 feed, and initializes the Pairing Graph with historical data.

### Step 2: Trigger a Drama Forecast via CLI
```bash
bbb26-predict --pair "Aline+Diego" --window 120
```
This calls the SAE + CPG combo to generate a 120-minute drama forecast for the specified pair. Output is a structured JSON with:
- **Tension curve** (timestamps + intensity scores)
- **Narrative drift** (if any)
- **Top 3 likely events** (e.g., “argument over voting strategy”)

### Step 3: Integrate the Webhook for Real-Time Alerts
```python
import requests

payload = {
    "pair": "Aline+Diego",
    "threshold": 0.75,
    "callback_url": "https://your-dashboard.io/webhook/bbb26"
}
r = requests.post("https://api.bbb26dramapredictor.io/v1/webhook", json=payload)
print(r.status_code)
```
You’ll receive a POST to your callback whenever the drama heat index exceeds 0.75 for that pair, with a full payload including the NDD diff.

## Best Practices

### 1. Use a Sliding Window for Live Feeds
Don’t run the full pipeline on every poll. Instead, set `--window 60` and update every 15 minutes. This reduces API costs by ~40% while maintaining sub-5-minute alert latency. Cache the CPG output in Redis with a TTL of 30 minutes.

### 2. Combine NDD Output with Manual Curation
The Narrative Drift Detector is powerful but can flag false positives during commercial breaks or technical glitches. **Always pair NDD alerts with a manual review queue** in your dashboard. Tag false positives and feed them back into the model via a feedback loop (we provide a `retrain.py` script for this).
