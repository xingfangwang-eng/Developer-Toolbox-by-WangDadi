# dossierpro - DOS_001

## Übersicht  
dossierpro ist ein forensisch fundiertes Dossier-Automatisierungssystem für die Erstellung, Versionierung und Prüfung von beweisrelevanten Projektchroniken. Zielsprache: Deutsch. Einsatzbereich: juristische Dokumentation, Compliance-Nachweisführung, interne Ermittlungen.

## Detaillierter Inhalt

### Was ist dossierpro?  
dossierpro löst das Problem fragmentierter, nicht prüfbarer Projektdokumentation durch eine kombinierte Timeline-Engine mit kryptografischer Kettenbildung. Jeder Eintrag wird mit einem SHA-256-Hash des Vorgängers versehen, sodass nachträgliche Manipulationen sofort erkennbar sind. Die Engine arbeitet mit einer Auflösung von **1 Millisekunde** und unterstützt parallele Timeline-Stränge für getrennte Vertragsparteien.

### Kernfunktionen

1. **Deterministische Timeline-Reparatur bei CSV-Korruption**  
   - Automatische Erkennung von Byte-Reihenfolgefehlern (BOM/UTF-8/UTF-16)  
   - Reparatur mittels **Levenshtein-Distanz-basierter Korrekturmatrix**  
   - Parameter: `--repair-mode strict|fuzzy|reconstruct`  
   - Ausgabe: korrigierte CSV + Log-Datei mit jeder Korrektur in **Mikrosekunden-Auflösung**

2. **Vertragschronologie-Manager mit intelligenter Alarmkaskade**  
   - Verknüpfung von Vertragsdaten mit externen Quellen (PostgreSQL, REST-APIs)  
   - Automatische Generierung von **Meilenstein-Warnungen** bei Abweichung > 5 % vom Soll-Datum  
   - Parameter: `--alert-threshold 0.05 --escalation-chain 3`  
   - Speicherung aller Alarmzustände in einer **InfluxDB-Zeitreihe** für spätere Audits

3. **Multi-Strang-Dossier-Kompilierung mit Hash-Verkettung**  
   - Zusammenführung mehrerer unabhängiger Timeline-Stränge zu einem einzigen Dossier  
   - Jeder Merge erzeugt einen **Merkle-Baum** mit einem Root-Hash  
   - Parameter: `--merge-strategy union|intersection|strict`  
   - Exportformate: PDF (mit QR-Code des Root-Hash), JSON, XML (XSD-validiert)

### Verwendung

1. **Schritt 1: Datenimport und Validierung**  
   ```bash
   dossierpro import --source ./vertragsdaten.csv --repair-mode strict --encoding utf-8
   ```
   Das Tool prüft jede Zeile auf Integrität. Bei mehr als 3 % Korruption wird der Import abgebrochen und ein detaillierter Fehlerreport im `/repair_logs/`-Verzeichnis ausgegeben.

2. **Schritt 2: Timeline-Konfiguration und Alarmparameter**  
   ```bash
   dossierpro timeline --start 2024-01-01 --end 2025-06-30 --alert-threshold 0.05 --escalation-chain 3 --db-connection postgres://user:pass@localhost:5432/vertraege
   ```
   Die Engine lädt alle relevanten Vertragsdaten aus der PostgreSQL-Datenbank, ordnet sie chronologisch und berechnet für jeden Meilenstein die Abweichung. Bei Überschreitung des Schwellwerts wird eine Alarmkaskade von 3 Stufen ausgelöst (E-Mail, Slack, PagerDuty).

3. **Schritt 3: Dossier-Kompilierung und Export**  
   ```bash
   dossierpro compile --merge-strategy strict --output ./dossier_final.pdf --hash-algo sha256
   ```
   Der Compiler vereint alle Timeline-Stränge, erzeugt einen Merkle-Baum und schreibt den Root-Hash als QR-Code auf das Deckblatt. Der Export erfolgt als PDF mit eingebetteten Metadaten (XMP-Standard).

### Best Practices

- **Best Practice 1: Regelmäßige Hash-Ketten-Prüfung**  
  Führen Sie mindestens einmal pro Woche eine vollständige Kettensignatur-Prüfung durch:  
  ```bash
  dossierpro audit --dossier ./dossier_final.pdf --verify-chain
  ```  
  Dies deckt jede nachträgliche Änderung auf, selbst wenn sie nur ein Byte betrifft. Empfohlen für alle Dossiers mit Rechtswirkung.

- **Best Practice 2: Datenquellen-Isolation für parallele Stränge**  
  Verwenden Sie für jede Vertragspartei eine separate PostgreSQL-Datenbankinstanz (oder zumindest ein separates Schema). Die Timeline-Engine kann dann über `--db-connection` mehrere Quellen parallel abfragen und den Merge-Strategie `strict` erzwingen. So verhindern Sie unbemerkte Datenvermischung.
