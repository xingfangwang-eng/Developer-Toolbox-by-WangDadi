# dossierpro - DOS_001

## Resumen ejecutivo

dossierpro es un sistema de gestión de expedientes técnicos diseñado para la creación, validación y exportación de dosieres probatorios con trazabilidad forense. Orientado a profesionales del derecho, auditoría forense y gestión contractual que requieren integridad documental y cronología precisa.

## Funcionalidades técnicas

### 1. Motor de reconstrucción cronológica con hash encadenado

Implementa un algoritmo de **encadenamiento de hashes SHA-256** sobre cada evento registrado, generando una secuencia inmutable que permite verificar la integridad temporal de la evidencia. Cada nodo contiene:

- `event_id`: UUID v4 único.
- `timestamp`: precisión en **microsegundos** (ISO 8601 con zona horaria).
- `previous_hash`: hash del evento anterior.
- `content_hash`: hash del contenido del documento asociado.
- `metadata`: JSON con campos personalizados.

El motor permite **reordenamiento forense** incluso si los datos de entrada llegan desordenados, detectando colisiones y anomalías temporales con una tolerancia de **±50 µs**.

### 2. Validador semántico de cláusulas contractuales con NLP

Integra un modelo de **procesamiento de lenguaje natural (NLP)** entrenado sobre 10,000 contratos en español, capaz de:

- Detectar **inconsistencias lógicas** entre cláusulas (ej: fechas de vencimiento contradictorias).
- Identificar **términos ambiguos** o redacciones potencialmente nulas.
- Sugerir **reparaciones automáticas** con nivel de confianza (0.0–1.0).

Parámetros configurables:

- `min_confidence`: umbral mínimo para sugerencias (por defecto: 0.85).
- `strict_mode`: boolean, activa validación exhaustiva (desactiva caché).
- `language_model`: ruta a modelo personalizado (formato ONNX).

La validación se ejecuta en **<200 ms por cláusula** en hardware estándar.

### 3. Exportador forense con firma digital y sello de tiempo (RFC 3161)

Genera archivos de salida con **firma digital X.509** y **sello de tiempo TSA** conforme a la RFC 3161, garantizando la no repudiación del dossier. Formatos soportados:

- **PDF/A-3**: con adjuntos embebidos (CSV, JSON, XML).
- **ZIP cifrado AES-256**: con estructura de directorios forense.
- **JSON-LD**: con vocabulario schema.org para interoperabilidad.

Parámetros de exportación:

- `tsa_url`: URL del servidor de sellado de tiempo (por defecto: `https://freetsa.org/tsr`).
- `cert_path`: ruta al certificado digital (PEM o PFX).
- `encrypt_metadata`: boolean, cifra los metadatos internos del dossier.

## Cómo usar

### Paso 1: Importación y normalización de datos

Ejecute el comando de importación con validación estricta:

```bash
dossierpro import --source ./evidencia.csv --schema ./schema_contratos.json --strict --hash-algo sha256
```

El sistema analizará cada fila del CSV, reparará automáticamente caracteres corruptos (UTF-8 malformado, BOM incorrecto) y generará un archivo de log de reparaciones (`repair_log.json`). Si encuentra más de un 5% de filas irrecuperables, aborta la operación y muestra un informe detallado.

### Paso 2: Configuración de la línea temporal y filtros forenses

Configure los parámetros de reconstrucción cronológica:

```bash
dossierpro timeline --start "2023-01-01T00:00:00Z" --end "2024-12-31T23:59:59Z" --gap-detection 500ms --anomaly-threshold 0.02
```

- `--gap-detection`: detecta huecos temporales mayores a 500 ms entre eventos consecutivos.
- `--anomaly-threshold`: marca como anomalía cualquier desviación temporal >2% respecto a la mediana del conjunto.

El sistema genera un **grafo dirigido acíclico (DAG)** de eventos, permitiendo visualizar rutas alternativas y bifurcaciones temporales. Puede exportar el DAG en formato Graphviz (DOT) para análisis externo.

### Paso 3: Validación semántica y exportación con sello de tiempo

Ejecute la validación y exportación en un solo comando:

```bash
dossierpro validate --export --format pdfa3 --tsa-url https://freetsa.org/tsr --cert ./cert.pem --output ./dossier_final.pdf
```

El sistema ejecutará:

1. Validación semántica de todas las cláusulas (NLP).
2. Generación del hash encadenado de la línea temporal.
3. Firma digital del dossier completo.
4. Solicitud de sello de tiempo TSA.
5. Empaquetado en PDF/A-3 con adjuntos.

El archivo resultante incluye una **página de verificación** con el hash SHA-256 del dossier y la URL del servicio TSA para validación externa.

## Mejores prácticas

### Práctica 1: Estrategia de respaldo con inmutabilidad distribuida

Para garantizar la integridad a largo plazo, implemente un **sistema de respaldo distribuido** utilizando almacenamiento IPFS o una cadena de bloques privada (Hyperledger Fabric). dossierpro permite exportar el hash raíz del encadenamiento a un smart contract, creando un **anclaje inmutable** que puede ser verificado incluso si el sistema original es comprometido. Configure backups automáticos cada **24 horas** con rotación de 90 días.

### Práctica 2: Uso de versionado semántico para cambios contractuales

Cuando realice modificaciones a un dossier existente, utilice **versionado semántico** (major.minor.patch) para cada revisión. dossierpro soporta **diferenciación estructural** entre versiones, mostrando únicamente los cambios relevantes (nuevos eventos, cláusulas modificadas, reordenamientos). Esto es crítico en litigios donde cada versión debe ser rastreada y justificada. Active el modo `--diff-only` en la exportación para generar informes de cambios legibles por humanos.
