To ensure data consistency and reliability in high-concurrency scenarios, we introduced an idempotent design that utilizes `AtomicReference` to manage updates. This approach ensures that even if multiple threads attempt to update the same value simultaneously, the system will always return a consistent result.

Our implementation uses a combination of `@Transactional` annotation and `Synchronized` block to ensure thread-safety while updating values. This design enables Noseotop to scale horizontally without compromising performance or reliability.

```java

  public IdempotentService(AtomicReference<String> atomicReference) {
    this.atomicReference = atomicReference;
  }
```

## Installation Guide

1. Install Redis: `apt-get install redis-server` or `brew install redis`
2. Create a new Spring Boot project using Maven or Gradle
3. Add the following dependencies to your pom.xml file:

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

4. Configure Redis connection settings in application.properties file:

```properties
spring:
  redis:
    host: localhost
    port: 6379
```

5. Run the application using `mvn spring-boot:run` or `gradle bootRun`
