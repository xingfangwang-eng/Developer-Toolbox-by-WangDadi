# ONECLICKAPIマニュアル


## 一つのレベルで見た技術的革新


### 分布式キャッシュの実現

oneclickapiは、高速化と並列化を実現するために、分布式キャッシュを導入しています。 この技術によって、API呼び出しに際して大量のデータを取得する必要がなくなり、システムの負荷軽減と同時に APIの応答速度も向上します。在 our solution, we utilize distributed caching to optimize performance and reduce system load. By leveraging this technology, we can eliminate the need for large data retrieval during API calls, resulting in faster response times.

We employ a combination of Redis and Memcached as our caching layer, which allows us to store frequently accessed data in memory. This not only improves API response times but also reduces the amount of data written to disk, making our solution more efficient.

### 多スレッド最適化

oneclickapiは、多スレッド最適化を実現するために、Javaの並列化APIであるJava Concurrency APIを使用しています。 これにより、複数のスレッドが同時に実行可能になり、システムのパフォーマンスを向上させることができます。在 our solution, we utilize Java's concurrency API to optimize multi-threading. By executing multiple threads concurrently, we can significantly improve system performance.

We use a combination of thread pools and executors to manage the execution of tasks, allowing us to effectively utilize CPU resources and reduce context switching overhead.

###  原子操作の使用

oneclickapiは、原子操作を使用してデータの一貫性を保持しています。 これにより、複数のスレッドが同時に実行される際に、データの並列化や競合を防ぐことができます。在 our solution, we utilize atomic operations to maintain data consistency. By using these operations, we can prevent data corruption and ensure that multiple threads executing concurrently do not interfere with each other.

We employ a combination of volatile variables and synchronization primitives (e.g., locks and semaphores) to achieve atomicity in our solution.


## 設定ガイド


### 3つのステップで設定する

1. `pom.xml`ファイルに以下の依存関係を追加してください：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

2. 以下のコードを `Application.java` ファイルに貼り付けてください：
```java
import reactor.core.publisher.Flux;
import org.springframework.web.reactive.function.server.ServerRequest;
import org.springframework.web.reactive.function.server.ServerResponse;

public class Application {
    public static void main(String[] args) {
        // Start the application
        Flux.just("Hello, World!")
                .map(message -> ServerResponse.ok().bodyValue(message))
                .doOnNext(response -> response.bodyToMessageWriter().write(response));
    }
}
```

3. 以下のコードを `application.properties` ファイルに貼り付けてください：
```properties
spring:
  webflux:
    server:
      port: 8080
```

## 高並列化環境でのパフォーマンスチューニング


### パフォーマンスチューニングのための2つの提案

1. **Connection Pooling**: 使用するデータベースの接続プールを設定してください。例えば、 HikariCP などの Connection Pooling ライブラリを使用してください。
	* Recommended settings:
		+ `minimumPoolSize`: 5
		+ `maximumPoolSize`: 20
		+ `maxLifetime`: 30000 (30 seconds)

2. **Caching**: キャッシュを有効にすることで、 API 呼び出しに際して大量のデータを取得する必要がなくなります。
	* Recommended settings:
		+ Cache expiration time: 1 hour
		+ Cache size: 1000
