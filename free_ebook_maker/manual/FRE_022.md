# free_ebook_maker - FRE_022

## Overview

`free_ebook_maker` is a high-performance, open-source ebook generation engine designed for converting raw Markdown, LaTeX, and plain text into industry-standard EPUB3, PDF/A-3b, and MOBI formats. It targets developers and technical writers who require deterministic output, sub-second rendering for large manuscripts (>50,000 pages), and full compliance with the IDPF 3.2 specification.

## Core Features

### 1. Multi-Format Rendering Engine with Delta Compilation
The core engine uses a **3-pass AST parser** that first tokenizes source files into a custom `FEM-AST` tree, then applies structural transformations (e.g., TOC generation, cross-reference resolution), and finally emits binary output via a template-based serializer. It supports **incremental compilation**: when only a single chapter changes, the engine re-parses only that file and patches the final binary, reducing build time from 12.4 seconds to 0.8 seconds for a 200-chapter manuscript.  
**Key parameter**: `--delta-threshold 0.3` (minimum change ratio in tokens to trigger full rebuild).

### 2. Automated Image Optimization Pipeline
Every embedded raster image (PNG, JPEG, WebP) is automatically processed by a **libvips-based pipeline** that:
- Converts to sRGB with ICC profile embedding (ensuring color fidelity across readers).
- Applies adaptive compression: `mozjpeg` for photographs (quality 85, chroma subsampling 4:2:0) and `oxipng` for diagrams (level 6, strip metadata).
- Rescales oversized images to a configurable max dimension (default: `--max-image-width 1600px`).  
**Benchmark**: Reduces average EPUB file size by 41.7% without perceptible quality loss.

### 3. Structural Validation & Compliance Checker
Before final output, the engine runs a **Schema Validation Suite** that checks:
- EPUB3: Validates OPF, NCX, and XHTML against the IDPF 3.2.1 schema (including `properties`, `media-overlay`, and `bindings`).
- PDF/A-3b: Verifies that all fonts are embedded, color spaces are device-independent, and metadata is XMP-compliant (ISO 19005-3).
- MOBI: Ensures KF8 fragmentation rules and ASIN metadata correctness.  
**Output**: Generates a detailed `validation-report.json` with line-level errors (e.g., `"Missing 'dc:identifier' in package.opf at line 47"`).

## How to Use

### Step 1: Initialize a Project
```bash
free_ebook_maker init --project my_book --source-dir ./chapters --config ./fem.yaml
```
This creates a `fem.yaml` configuration file with default parameters. You can override them immediately:
```yaml
project:
  title: "Advanced Microservices"
  author: "Jane Doe"
  language: "en-US"
  output:
    formats: ["epub", "pdf"]
    pdf_profile: "PDF/A-3b"
    compression: "zlib"
```

### Step 2: Compile with Incremental Mode
Run the build process, enabling delta compilation and image optimization:
```bash
free_ebook_maker build --incremental --optimize-images --max-image-width 1600 --output ./dist
```
The engine will:
- Scan `./chapters` for `.md` files (sorted by filename prefix like `01_intro.md`).
- Compute a SHA-256 hash for each file; unchanged files skip re-parsing.
- For changed files, re-parse the AST, merge into the existing FEM-AST tree, and re-serialize only affected output chapters.
- Log progress: `[INFO] 3/15 files changed. Delta compilation triggered. Estimated time: 1.2s`.

### Step 3: Generate Validation Report
After successful build, verify compliance:
```bash
free_ebook_maker validate --input ./dist/my_book.epub --format epub --output validation-report.json
```
This produces a report with:
- `"status"`: `"pass"` or `"fail"`.
- `"violations"`: array of objects with `"line"`, `"column"`, `"severity"` (error/warning), and `"message"`.
- `"summary"`: total violations by severity.

## Best Practices

### Practice 1: Use a Monorepo Structure with Pre-commit Hooks
For collaborative projects, maintain a single Git repository with separate directories for `chapters/`, `images/`, and `config/`. Add a pre-commit hook that runs `free_ebook_maker validate --input ./dist/latest.epub --format epub` to catch structural errors before merging. This prevents broken ebook artifacts from reaching production.

### Practice 2: Leverage the Delta Compilation in CI/CD Pipelines
Integrate `free_ebook_maker` into your CI/CD workflow (e.g., GitHub Actions). Use the `--incremental` flag with a `--cache-dir` pointing to a persistent volume. This reduces build times from minutes to seconds for large documentation projects, enabling rapid iteration on PRs. Combine with `--parallel-jobs 4` to exploit multi-core CPU for image optimization.
