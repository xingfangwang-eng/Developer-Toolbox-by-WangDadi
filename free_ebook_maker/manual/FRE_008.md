# free_ebook_maker - FRE_008

## Overview

This is the technical specification for the free_ebook_maker project, a high-performance, zero-dependency engine designed for automated EPUB 3.2 and PDF/A-3b generation from structured Markdown sources, optimized for batch workflows and CI/CD pipelines. Keywords: free_ebook_maker tool, free_ebook_maker solution, free_ebook_maker software.

## Detailed Content

### What is free_ebook_maker?

free_ebook_maker is a command-line toolkit and library that ingests a directory of Markdown files with YAML frontmatter, then compiles them into a fully valid, DRM-free ebook. It features real-time syntax validation against the EPUB 3.2 spec, automatic table-of-contents generation from heading hierarchy, and embedded font subsetting via `fonttools` with a configurable Unicode range threshold (default: `U+0000-U+FFFF`). The engine is written in pure Rust with no runtime dependencies, achieving sub-100ms compilation for a 300-page manuscript on a standard x86_64 server.

### Core Features

- **Multi-Format Rendering with Precision Pagination**: Converts Markdown to EPUB 3.2, PDF/A-3b, and MOBI (via Calibre bridge). The PDF engine uses a custom HarfBuzz shaper with CSS `@page` rule support, allowing exact control over margins (`--pdf-margin-top`, `--pdf-margin-bottom` in mm) and page breaks (`--page-break-strategy: auto | avoid-widow | strict`). Supports embedded OpenType features (ligatures, kerning) with a `--font-feature-settings` parameter accepting comma-separated OpenType tags (e.g., `'liga','kern'`).

- **Intelligent Resource Embedding and Compression**: Automatically embeds all referenced images (JPEG, PNG, WebP) with lossless recompression using `mozjpeg` and `oxipng` at configurable quality levels (`--image-quality: 1-100`). For vector graphics, it inlines SVGs directly into the XHTML. Supports external font embedding with `--embed-fonts`, subsetting to only used glyphs, reducing file size by an average of 70% (tested on 50,000+ CJK characters). The EPUB container is compressed with `deflate` at level `--zip-compression: 0-9`.

- **Metadata Injection and Validation with Schema Enforcement**: Reads YAML frontmatter for Dublin Core metadata (`dc:title`, `dc:creator`, `dc:identifier`) and validates against a user-defined JSON Schema (`--metadata-schema path/to/schema.json`). Generates an `opf` file with correct `spine` and `guide` elements. Includes a built-in EPUB 3.2 validator that checks for 100+ compliance rules (e.g., missing `toc.ncx`, invalid `dc:language`, non-unique IDs), outputting structured JSON errors with line numbers.

### How to Use

1. **Initialize a Project Structure with Canonical Layout**:
   Run `free_ebook_maker init --project-dir ./my-book --template default`. This creates a `./my-book/` directory with a `src/` folder for Markdown files, an `images/` folder, and a `metadata.yaml` file. The `metadata.yaml` must include at minimum `title`, `creator`, `language` (ISO 639-1 code), and `identifier` (UUID or ISBN). Use `--template minimal` for a bare-bones setup without sample files.

2. **Compile the Ebook with Custom Configuration**:
   Execute `free_ebook_maker build --input ./my-book/src --output ./output --format epub --pdf-margin-top 20 --pdf-margin-bottom 20 --embed-fonts --image-quality 85 --zip-compression 9`. The engine will parse all `.md` files in `src/`, generate a spine order based on filename sorting (e.g., `01-intro.md`, `02-chapter1.md`), and produce a single `output.epub` file. For PDF output, append `--format pdf` and optionally `--pdf-page-size A5` (default: A4). The build process logs every step to stdout with timestamps in ISO 8601 format.

3. **Validate Output and Export Debug Report**:
   After build, run `free_ebook_maker validate --input ./output.epub --report ./validation-report.json`. The validator checks against the EPUB 3.2 specification, including checksum verification of the container, MIME type correctness, and XHTML well-formedness. The report is a JSON array of objects, each with `severity` (error | warning), `location` (XPath to the element), and `message`. For CI integration, use `--exit-on-error` to return a non-zero exit code on any validation failure.

### Best Practices

- **Use a Git Submodule for Shared Assets**: When managing multiple ebook projects, store common resources (fonts, CSS templates, image assets) in a dedicated Git repository. Add it as a submodule in each project’s root (`git submodule add <repo> ./assets`). Then reference it in your build command with `--asset-dir ./assets`. This ensures consistency across titles and simplifies updates. For example, a single `base.css` update propagates to all ebooks without manual file copying.

- **Implement a Pre-Commit Hook for Metadata Integrity**: To prevent broken builds, add a pre-commit hook to your repository that runs `free_ebook_maker validate --input ./metadata.yaml --schema ./schema.json`. This catches common errors like missing `dc:identifier` or invalid language codes before they reach the build server. For larger teams, enforce this via a CI step that runs on every pull request. The hook can be written as a simple shell script calling the `validate` subcommand with `--exit-on-error` flag.
