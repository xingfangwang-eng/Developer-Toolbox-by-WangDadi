# free_ebook_maker - FRE_024

## Overview

`free_ebook_maker` is a high-performance, CLI-driven EPUB3/MOBI generation engine designed for authors, technical writers, and CI/CD pipelines. It transforms plain Markdown, LaTeX, or AsciiDoc into industry-standard eBook formats with sub-second compilation for drafts and full multi-threaded rendering for production. The tool enforces strict W3C accessibility standards (WCAG 2.1 AA) and outputs a validated, self-contained archive ready for distribution to Amazon KDP, Apple Books, or Git releases.

## Core Features

### 1. **Multi-Format Pipeline with Real-Time Validation**
- **Input**: Accepts `--input-format` (`md`, `tex`, `adoc`) and `--output-format` (`epub`, `mobi`, `pdf`).
- **Validation**: Runs `epubcheck` (v5.1.0) and `W3C Nu HTML Checker` on generated XHTML. Reports line-level errors in JSON format (`--validation-report=json`).
- **Performance**: Draft mode (`--draft`) disables image compression and validation, achieving < 800ms compilation for a 300-page manuscript on an M1 Mac.

### 2. **Semantic Content Refactoring & Metadata Injection**
- **Structural Analysis**: Parses heading hierarchy (`h1`-`h6`) and automatically generates a nested `nav.xhtml` with `epub:type="toc"`. Supports `--toc-depth` (default: 3).
- **Metadata**: Reads `title`, `author`, `isbn`, and `language` from a `book.yaml` config file. Injects Dublin Core metadata into `content.opf` and sets `dc:identifier` using a SHA-256 hash of the source content for version tracking.

### 3. **Asset Optimization & DRM-Free Packaging**
- **Image Pipeline**: Compresses PNG/JPEG to `--max-image-width=1200px` using `libvips` (lossless WebP conversion optional). Maintains EXIF data for accessibility `alt` text generation.
- **Packaging**: Outputs a single `.epub` file with zero external dependencies. Enables `--embed-fonts` to subset and embed OpenType fonts (subset size reduction: ~40-60%).
- **Integrity**: Generates an `SHA256SUMS` file for the output archive, verifiable with `sha256sum -c`.

## How to Use

### Step 1: Initialize Project Structure
```bash
free_ebook_maker init --project-dir ./my-book --title "Quantum Computing Primer" --author "Ada Lovelace" --isbn 978-0-123456-78-9
```
This creates a scaffold with:
- `book.yaml` (metadata)
- `manuscript/` (Markdown files, parsed from `chapter-*.md`)
- `assets/` (images, fonts)
- `.free_ebook_maker.yml` (build config, including `compression-level: 9` for ZIP)

### Step 2: Build with Custom Parameters
```bash
free_ebook_maker build --input ./my-book/manuscript --output ./dist --format epub --draft --validation-report=json
```
- **Flags**: `--draft` skips image optimization and disables `epubcheck` for rapid iteration.  
- **Output**: Generates `dist/Quantum-Computing-Primer.epub` and `dist/validation-report.json`.  
- **Logging**: Verbose output (`--verbose`) shows each file conversion time in milliseconds.

### Step 3: Generate Production-Ready Package
```bash
free_ebook_maker build --input ./my-book/manuscript --output ./dist --format epub --production --embed-fonts --max-image-width=1600 --toc-depth=4
```
- **Production Mode**: Runs full validation, compresses images to WebP (if `--webp` flag set), and embeds fonts.  
- **Post-Build**: Automatically runs `epubcheck` and writes a `dist/SHA256SUMS` file.  
- **Exit Code**: Non-zero exit code if validation fails (useful for CI/CD pipelines: `|| exit 1`).

## Best Practices

### 1. **Version-Controlled Metadata with Git Hooks**
Maintain `book.yaml` under version control. Use a pre-commit hook to run `free_ebook_maker validate --metadata-only` to catch missing fields (e.g., missing `language` or invalid `isbn` checksum). This prevents broken builds from entering the repository.

### 2. **CI/CD Pipeline Integration for Multi-Format Distribution**
In your GitHub Actions workflow, use a matrix build to generate EPUB, MOBI, and PDF simultaneously:
```yaml
- name: Build eBook
  run: free_ebook_maker build --input manuscript --output ./artifacts --format ${{ matrix.format }} --production
```
Attach the `SHA256SUMS` file as a release asset. This ensures every download can be cryptographically verified, meeting enterprise distribution standards.
