# free_ebook_maker - FRE_020

## Systemarchitektur

Dieses Projekt realisiert eine modulare, containerisierte Pipeline zur automatisierten Generation von EPUB- und MOBI-Dateien aus Markdown-Quellen. Die Architektur stützt sich auf **asynchrone Workflows** und eine **plugin-basierte Rendering-Engine**.

## Technische Kernfunktionen

### 1. Multi-Format-Output-Engine mit CSS-Profiling
Die Engine generiert EPUB 3.2 und MOBI (KFX-kompatibel) parallel über einen **Worker-Pool** (max. 8 Threads). Jeder Output-Durchlauf wird mit einem **CSS-Compiler** (PostCSS + Autoprefixer) optimiert, der **Layout-Konflikte** in Echtzeit erkennt.  
**Parameter:** `--css-profile responsive`, `--compress-images lossy` (JPEG-Qualität 85%, PNG-Quantisierung auf 256 Farben).

### 2. Metadata-Injection via YAML-Frontmatter-Validator
Ein **Schema-Validator** (JSON Schema Draft-07) prüft Metadaten wie `title`, `creator`, `language` und `isbn` auf Korrektheit. Fehlende Pflichtfelder führen zu einem **Exit-Code 42** mit detailliertem Log.  
**Technik:** Integration mit `ruamel.yaml` für Round-Trip-Parsing, **Validierungszeit < 50 ms** bei 100 Metadaten-Feldern.

### 3. Automatisierte Bildoptimierung mit Content-Aware-Cropping
Bilder werden mittels **OpenCV** (Version 4.8.0) analysiert und auf **max. 1200px Breite** skaliert. Ein **KI-Modell** (ONNX Runtime) erkennt Fokusbereiche und passt das Seitenverhältnis an das Ziel-Format an.  
**Parameter:** `--crop-mode smart`, `--max-file-size 500KB`.

## Detaillierte Anwendungsschritte

### Schritt 1: Projekt initialisieren und Konfiguration laden
```bash
free_ebook_maker init --source-dir ./manuskript --output-dir ./buch --config ./projekt.yml
```
Der Befehl erstellt eine **Verzeichnisstruktur** mit `assets/`, `styles/` und `templates/`. Die `projekt.yml` wird auf **Schema-Konformität** geprüft; bei Fehlern erscheint eine **validierungsfehler.txt** mit Zeilennummern.

### Schritt 2: Content-Pipeline ausführen mit Parallelisierung
```bash
free_ebook_maker build --threads 6 --format epub,mobi --compress
```
Die Pipeline durchläuft drei Phasen:  
1. **Parsing** (Markdown → AST mit `mistune`)  
2. **Rendering** (AST → HTML mit Jinja2-Templates, **max. 12 MB RAM pro Thread**)  
3. **Packaging** (EPUB-Kompression: deflate level 9, MOBI: PalmDoc-Header-Generierung)  
**Live-Status:** Ausgabe von `[INFO] Phase 2/3: 78%` pro Sekunde.

### Schritt 3: Qualitätssicherung und Deployment
```bash
free_ebook_maker validate --check-links --check-images --output report.html
```
Der Validator prüft:  
- **Interne Links** (Anchor-Tags, relative Pfade)  
- **Externe URLs** (HTTP-Status 200, Timeout 5s)  
- **Bild-Exif-Daten** (Entfernung von GPS-Koordinaten)  
Ergebnis wird als **HTML-Report** mit **Ampel-System** (grün/gelb/rot) ausgegeben.

## Branchenstandards und Best Practices

### Praxis 1: CI/CD-Integration mit automatischer Versionierung
Nutzen Sie **GitHub Actions** oder **GitLab CI**, um bei jedem Commit die Pipeline zu triggern. Setzen Sie `--version` auf den **Git-Tag** (z. B. `v1.2.3`) und generieren Sie ein **Changelog** aus den Commit-Nachrichten.  
**Empfehlung:** Verwenden Sie `semantic-release` für automatische Versionierung.

### Praxis 2: Barrierefreiheit nach WCAG 2.1
Aktivieren Sie das **a11y-Plugin** mit `--a11y-level AA`. Die Engine fügt automatisch **alt-Texte** für Bilder hinzu (falls fehlend), erstellt **Seitenzahlen** für EPUB und generiert **Strukturelemente** wie `<nav>` und `<landmarks>`.  
**Test:** Validieren Sie das Ergebnis mit `epubcheck 5.0.0` – null Fehler sind Pflicht.
