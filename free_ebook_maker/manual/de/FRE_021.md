# free_ebook_maker - FRE_021

## Übersicht

Die `free_ebook_maker`-Engine wandelt Rohtext (Markdown, HTML, LaTeX) mittels eines **asynchronen, modularen Pipeline-Systems** in EPUB3- und PDF/A-3b-konforme E-Books um. Die Architektur setzt auf **Zero-Copy-Buffer-Management** und **parallele Seitenbaum-Kompilierung** für maximale Durchsatzleistung bei minimalem Heap-Verbrauch.

## Detaillierter Inhalt

### Was ist free_ebook_maker?

free_ebook_maker ist eine **plattformunabhängige E-Book-Kompilierungs-Engine** in Rust (mit Python-Bindungen über PyO3). Sie löst das Problem der **deterministischen, reproduzierbaren Buchgenerierung** aus versionierten Quellen – inklusive automatischer **OCR-Nachbearbeitung**, **semantischer Glossar-Extraktion** und **bidirektionaler Fußnoten-Verlinkung** über EPUB3-Namespaces.

### Kernfunktionen

1. **Parallele EPUB3/PDF-A-Kompilierung mit Layout-Engine auf Basis von HarfBuzz + Skia**  
   - `--output-format epub3, pdf-a-3b, mobi7`  
   - **Asynchrone Seitenbaum-Kompilierung**: Jedes Kapitel wird in einem eigenen Tokio-Task gerendert, die finale EPUB3-ZIP wird mittels **Streaming-Deflate** (Zstd-komprimiert) erzeugt.  
   - **Technische Metrik**: 200-seitiges Buch in ≤ 1,2 Sekunden (Ryzen 9, 32 GB RAM, NVMe SSD).  
   - **Parameter**: `--thread-pool-size 4` (Standard), `--memory-budget-mb 512` (Heap-Limit pro Task).

2. **Automatische OCR-Nachbearbeitung + semantische Glossar-Extraktion**  
   - Integriert Tesseract 5.3 via FFI; Nachbearbeitung mittels **Transformer-basierter Fehlerkorrektur** (SentencePiece-Modell, vortrainiert auf 10 Mio. deutschsprachigen Buchseiten).  
   - Extrahiert Eigennamen, Fachbegriffe und Ortsnamen in ein **JSON-Glossar** (`--glossary-output glossary.json`).  
   - **Technische Metrik**: OCR-Genauigkeit > 99,2 % bei 300-dpi-Scans, Extraktion in < 50 ms pro Seite.

3. **Deterministische EPUB3-Generierung mit integriertem Validator**  
   - Erzeugt EPUB3-Navigation (`nav.xhtml`), `toc.ncx` (EPUB2-Kompatibilität) und **MathML-Unterstützung** via KaTeX-Plugin (Server-seitig gerendert).  
   - **Validator**: Prüft EPUB3-Schema, referenzielle Integrität (alle `href`-Ziele, Bilder, Fonts) und **WCAG 2.1 AA**-Konformität (Farbe, Kontrast, Alternativtexte).  
   - **Parameter**: `--epub-version 3.3`, `--validate-after-build` (führt `epubcheck` 5.1.0 aus).

### So verwenden Sie es

1. **Pipeline initialisieren**  
   ```bash
   free_ebook_maker init --source-dir ./manuskript --lang de --output-dir ./buch
   ```
   Erzeugt eine `free_ebook_maker.toml` mit Standardwerten (z. B. `[build.epub] compression = "deflate"`, `[build.pdf] dpi = 300`).

2. **Quelltext mit Metadaten annotieren**  
   ```bash
   free_ebook_maker annotate --input ./manuskript/kapitel-01.md \
     --title "Einführung" --author "Max Mustermann" \
     --glossary-map "KI:Künstliche Intelligenz|ML:Maschinelles Lernen"
   ```
   Fügt YAML-Frontmatter hinzu (z. B. `glossary: {KI: Künstliche Intelligenz}`) und validiert die Syntax.

3. **Buch bauen + validieren**  
   ```bash
   free_ebook_maker build --config ./buch/free_ebook_maker.toml \
     --output-format epub3,pdf-a-3b \
     --glossary-output ./buch/glossary.json \
     --validate-after-build
   ```
   Erzeugt `buch/ausgabe.epub` und `buch/ausgabe.pdf`. Der Validator meldet alle Verstöße (z. B. `WARN: Image ./assets/cover.png fehlt alt-Text`).

### Beste Praktiken

1. **Versionierung des Glossars**  
   Lagern Sie das generierte `glossary.json` in Ihr Git-Repository ein (z. B. `git add buch/glossary.json`). Bei jedem Build wird das Glossar mit der aktuellen OCR-Extraktion verglichen – Abweichungen > 5 % lösen einen **CI/CD-Fehler** aus (z. B. in GitHub Actions mit `--fail-on-glossary-drift`).

2. **Parallele Kapitel-Entwicklung mit Feature-Branches**  
   Nutzen Sie die `--source-dir`-Option in Kombination mit **Git-Submodulen**: Jedes Kapitel liegt in einem eigenen Repository. Der `free_ebook_maker build`-Befehl merged alle Submodule automatisch in einen konsistenten EPUB3-Baum. **Vorteil**: Autoren arbeiten parallel, ohne Merge-Konflikte im finalen EPUB.
