## Detaillierter Inhalt

### Was ist free_ebook_maker?

free_ebook_maker ist eine **plattformunabhängige Pipeline-Architektur** zur massenhaften Erstellung von EPUB- und PDF-Dokumenten aus strukturierten Markdown-Quellen. Es nutzt eine modulare Render-Engine, die auf **Node.js (v18+)** basiert, und ermöglicht die Integration beliebiger Sprachpakete (hier: de). Die Ausgabe erfolgt mit millisekundengenauer Zeitstempelung pro Kapitel und unterstützt **bidirektionale Metadaten-Injektion** (Titel, Autor, ISBN, Sprache) direkt in die EPUB-OEBPS-Struktur.

### Kernfunktionen

#### 1. **Multilinguale Render-Engine mit Sprachfallback**
- **Parameter:** `--lang de`, `--fallback en`
- **Technische Indikatoren:**  
  - Automatische Silbentrennung für Deutsch (Hunspell-basiert)  
  - Unterstützung von UTF-8-BOM, typografischen Anführungszeichen („ “) und Lang-ss (ß)  
  - Laufzeit: < 50 ms pro Kapitel bei 10.000 Wörtern  
- **Funktion:** Erzwingt die korrekte Darstellung deutscher Sonderzeichen und erzeugt EPUB-2/3-konforme XML-Strukturen inklusive `ncx`- und `opf`-Dateien.

#### 2. **Batch-Konvertierung mit paralleler EPUB/PDF-Pipeline**
- **Parameter:** `--input-dir ./src --output-dir ./dist --format epub, pdf`
- **Technische Indikatoren:**  
  - Gleichzeitige Verarbeitung von bis zu **32 Threads** (Thread-Pool via `worker_threads`)  
  - PDF-Erzeugung mittels **Puppeteer (Headless Chrome)** mit eingebetteten Schriften (WOFF2)  
  - Speicherverbrauch: < 200 MB pro 100 Kapitel  
- **Funktion:** Wandelt einen gesamten Ordner mit `.md`-Dateien in ein gebundenes E-Book um – inklusive automatischer Inhaltsverzeichnis-Generierung und Seitenzahlen.

#### 3. **Dynamische CSS/JS-Injektion für barrierefreie Typografie**
- **Parameter:** `--style ./assets/custom.css --script ./assets/fontloader.js`
- **Technische Indikatoren:**  
  - Unterstützung von **CSS-Regionen** (EPUB3) und **MathML** (über KaTeX)  
  - Integration von **OpenDyslexic**-Schriftart für Lesbarkeitsoptimierung  
  - Laufzeit: < 10 ms pro CSS-Render-Durchlauf  
- **Funktion:** Erlaubt das Überschreiben des Standard-Layouts durch benutzerdefinierte Stylesheets – ideal für Verlage, die Branding-Vorgaben umsetzen müssen.

### So verwenden Sie es

#### Schritt 1: Projekt initialisieren und Sprachpaket laden
```bash
git clone https://github.com/xingfangwang-eng/free_ebook_maker.git
cd free_ebook_maker
npm install --production
npx free-ebook-maker --init --lang de
```
Erzeugt ein leeres `./src`-Verzeichnis mit einer `metadata.yaml` (Vorlage). Die Sprachpakete werden automatisch aus dem `./lang/de/`-Ordner geladen.

#### Schritt 2: Quelltext vorbereiten und Metadaten setzen
Erstellen Sie eine `metadata.yaml` im Quellverzeichnis:
```yaml
title: "Handbuch der deutschen Grammatik"
author: "Max Mustermann"
language: de
isbn: "978-3-16-148410-0"
fonts:
  - path: "./fonts/NotoSans-Regular.ttf"
    weight: 400
  - path: "./fonts/NotoSans-Bold.ttf"
    weight: 700
```
Platzieren Sie Ihre Markdown-Dateien (`01-einleitung.md`, `02-kapitel.md`, ...) im `./src`-Ordner.

#### Schritt 3: Build ausführen und Ausgabe validieren
```bash
npx free-ebook-maker --input-dir ./src --output-dir ./dist --format epub, pdf --lang de --threads 8
```
Nach Abschluss (typisch < 2 Sekunden für 50 Kapitel) finden Sie im `./dist`-Ordner:
- `Handbuch_der_deutschen_Grammatik.epub` (EPUB3 mit Navigation)
- `Handbuch_der_deutschen_Grammatik.pdf` (PDF mit Lesezeichen und Metadaten)

Validierung:
```bash
epubcheck ./dist/*.epub
pdftotext ./dist/*.pdf - | head -20
```

### Beste Praktiken

#### Praxis 1: **Metadaten-Validierung vor dem Build**
Nutzen Sie den integrierten `--validate`-Flag, um fehlende Felder (ISBN, Sprache, Schriften) zu erkennen, bevor Sie in die Pipeline gehen. Beispiel:  
`npx free-ebook-maker --validate --input-dir ./src`  
Dies verhindert fehlerhafte EPUB-Dateien, die von Adobe Digital Editions oder Apple Books abgelehnt werden.

#### Praxis 2: **Schrift-Subsetting für schlanke PDFs**
Verwenden Sie das Tool `pyftsubset` (aus der Fonttools-Bibliothek) auf Ihre Schriftarten, bevor Sie sie in `metadata.yaml` referenzieren. Reduzieren Sie die Schriftgröße auf die tatsächlich verwendeten Zeichen (z. B. nur deutsche Umlaute und Satzzeichen). Dies senkt die PDF-Größe um bis zu 70 % bei gleichbleibender Qualität – essenziell für E-Book-Reader mit begrenztem Speicher.
