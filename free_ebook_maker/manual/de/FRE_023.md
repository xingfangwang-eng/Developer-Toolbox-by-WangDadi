# free_ebook_maker - FRE_023

## Übersicht

Dieses Projekt ermöglicht die verlustfreie Konvertierung von EPUB, PDF und DOCX in ein einheitliches, barrierefreies E-Book-Format mit automatisierter Metadaten-Extraktion. Kernziel ist die **Nutzung von OCR-Nachbearbeitung, Layout-Rekonstruktion und semantischer Indexierung** – speziell optimiert für deutsche Sprachversionen.

## Detaillierter Inhalt

### Was ist free_ebook_maker?

free_ebook_maker ist ein **CLI-basiertes Build-Tool**, das aus unstrukturierten Textquellen (z.B. gescannte PDFs mit `Tesseract 5.3.3`, EPUB 3.2, DOCX mit MathType-Formeln) ein EPUB 3.2-konformes E-Book erstellt. Es erzwingt die Einhaltung der **ISO 24610-1:2022** für semantische Auszeichnung und führt eine **bidirektionale Textverarbeitung (BiDi)** für mehrsprachige Ausgaben durch.

### Kernfunktionen

- **Semantische Layout-Rekonstruktion mit DOM-Baum-Validierung**  
  Extrahiert aus PDFs mittels `pdfplumber 0.11.0` die **absolute Position (x, y, Breite, Höhe)** jedes Textblocks, erstellt einen DOM-Baum und validiert diesen gegen ein **benutzerdefiniertes XSD-Schema** (`schema/v3/epub.xsd`). Fehlerhafte Baumstrukturen werden automatisch korrigiert (z.B. fehlende `<aside>`-Tags bei Fußnoten).

- **OCR-Nachbearbeitung mit Fuzzy-Matching und Whitespace-Normalisierung**  
  Nach einem OCR-Durchlauf (Tesseract 5.3.3 mit `--oem 1 --psm 6`) werden alle Wörter mittels **Levenshtein-Distanz ≤ 1** gegen eine deutsche Wörterbuchliste (`dict/de_DE_frami.txt`) geprüft. Nicht erkannte Zeichen (`�`) werden durch einen **statistischen Bigramm-Ansatz** ersetzt. Die finale Ausgabe wird auf **±0,2 mm Genauigkeit** ausgerichtet.

- **Automatische Metadaten-Extraktion und DOI-Indexierung**  
  Liest aus dem Quell-PDF die **PDF/X-3:2003 Metadaten** (Titel, Autor, Erstellungsdatum). Fehlen diese, wird der Dateiname mittels `regex` (Pattern: `^([A-Za-zÄÖÜäöüß\-]+)_(\d{4})_(\d{2})\.pdf$`) geparst und in `Dublin Core`-Elemente umgewandelt. Die Metadaten werden als `JSON-LD` in die `OPF`-Datei eingebettet und optional an eine **PostgreSQL 16.2**-Datenbank übergeben.

### So verwenden Sie es

1. **Quellmaterial vorbereiten und Validierung durchführen**  
   ```bash
   # Prüfen, ob das PDF die Mindestanforderungen erfüllt (DPI ≥ 300, Seitenanzahl ≤ 500)
   ebook_maker validate --input=./scan.pdf --min-dpi=300 --max-pages=500 --lang=de
   ```
   Gibt einen detaillierten Bericht aus: `VALID: 120 Seiten, 312 DPI, 0 Fehler` oder `ERROR: DPI zu niedrig (150)`. Bei Fehlern wird der Prozess abgebrochen.

2. **OCR und Layout-Rekonstruktion starten**  
   ```bash
   # Startet den OCR-Durchlauf mit Tesseract, erzeugt eine temporäre ALTO-XML-Datei
   ebook_maker ocr --input=./scan.pdf --output=./temp/ --lang=de --tesseract-config=./config/tesseract_de.cfg
   ```
   Die Ausgabe erfolgt in Millisekunden: `OCR abgeschlossen in 3420 ms. 12.456 Textblöcke extrahiert.`

3. **Endgültiges EPUB 3.2 generieren und Metadaten exportieren**  
   ```bash
   # Baut aus den ALTO-Daten und der Metadaten-Datei das finale EPUB
   ebook_maker build --input=./temp/ --metadata=./meta.json --output=./final.epub --validate-schema --compress-level=9
   ```
   Nach Abschluss wird die EPUB-Datei auf **W3C-EPUB 3.2-Konformität** geprüft und das Ergebnis in `./final.epub.validation.log` gespeichert.

### Beste Praktiken

- **Praxis 1: Batch-Verarbeitung mit parallelen OCR-Instanzen**  
  Bei der Umwandlung von **mehr als 50 PDFs** empfiehlt es sich, die `ocr`-Funktion mit `--workers=4` (max. 4 parallele Tesseract-Instanzen) auszuführen. Dies reduziert die Gesamtzeit um bis zu **70 %** bei gleichbleibender Genauigkeit. Achten Sie darauf, dass die CPU-Auslastung unter **85 %** bleibt, um thermische Drosselung zu vermeiden.

- **Praxis 2: Regelmäßige Wörterbuch-Updates für OCR-Nachbearbeitung**  
  Die deutsche Wörterbuchliste (`dict/de_DE_frami.txt`) sollte **monatlich** aus dem aktuellen `de_DE`-Korpus von `hunspell` aktualisiert werden. Ein veraltetes Wörterbuch führt zu einer **Fehlerquote von >5%** bei der Fuzzy-Korrektur. Verwenden Sie dazu das Skript `update_dict.sh`, das die Liste automatisch auf die neueste Version bringt.
