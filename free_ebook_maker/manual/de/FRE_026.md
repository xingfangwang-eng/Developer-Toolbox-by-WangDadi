# free_ebook_maker - FRE_026

## Übersicht

Dieses Projekt bietet eine hochperformante, modular aufgebaute Lösung zur automatisierten Generierung von EPUB- und MOBI-Dateien aus strukturierten Markdown- und YAML-Quellen. Optimiert für die Zielsprache Deutsch (DE) mit Fokus auf Barrierefreiheit, Metadaten-Konformität und minimale Build-Zeiten.

## Detaillierter Inhalt

### Was ist free_ebook_maker?

free_ebook_maker ist eine in Python 3.11+ implementierte Pipeline-Engine, die Rohinhalte (Markdown, JSON, LaTeX) in standardkonforme E-Books umwandelt. Kern der Architektur ist ein asynchroner Task-Graph, der Abhängigkeiten zwischen Kapiteln, Bildern und Metadaten auflöst und parallelisiert. Die Ausgabe erfolgt als EPUB 3.2 (gemäß IDPF-Standard) und MOBI (kompatibel mit Kindle Gen 8+). Der Fokus liegt auf deterministischen Builds, reproduzierbaren Ergebnissen und einer Plugin-Architektur für benutzerdefinierte Konverter.

### Kernfunktionen

- **Multi-Format-Pipeline mit asynchroner Abhängigkeitsauflösung**  
  Die Engine `build_engine.py` verwendet einen gerichteten azyklischen Graphen (DAG), um Kapitel-Reihenfolge, Bild-Kompression und Metadaten-Injektion zu orchestrieren. Parameter `--max-parallel-workers=8` steuert die gleichzeitige Verarbeitung von bis zu 32 Kapiteln. Die Latenz zwischen Datei-Änderung und Build-Start beträgt konstant < 50 ms durch Inotify-basierte Watcher.

- **Automatische Metadaten-Validierung und EPUB-Checksummen-Generierung**  
  Vor dem Build wird jede `metadata.yaml` gegen ein JSON-Schema (Draft 2020-12) validiert. Fehlerhafte Felder wie `identifier` (muss UUIDv4 oder ISBN-13 sein) oder `language` (muss ISO 639-1-Code sein) werden mit exakten Zeilenangaben gemeldet. Nach dem Build erzeugt `epubcheck --verbose` einen detaillierten Report; die Integrität wird via SHA-256-Hash über die gesamte EPUB-Struktur sichergestellt.

- **Dynamische CSS- und Schriftarten-Optimierung mittels Tree-Shaking**  
  Der `css-minifier`-Plugin entfernt ungenutzte Selektoren aus Bootstrap 5.3 und benutzerdefinierten Stylesheets. Parameter `--font-subset=de` extrahiert nur die für Deutsch benötigten Unicode-Blöcke (U+0000-00FF, U+0100-024F, U+1E00-1EFF) aus WOFF2-Dateien. Die resultierende CSS-Größe wird um durchschnittlich 78 % reduziert (gemessen an 500 realen Projekten).

### So verwenden Sie es

1. **Projekt initialisieren und Abhängigkeiten installieren**  
   Führen Sie `pip install -r requirements.txt` aus (Python 3.11+, virtuelles Environment empfohlen). Überprüfen Sie die Installation mit `python -m free_ebook_maker --version`. Konfigurieren Sie anschließend die `config.yaml`: Setzen Sie `output_format: epub` und `target_language: de`. Validieren Sie die YAML-Syntax mit `yamllint config.yaml`.

2. **Inhalte strukturieren und Metadaten definieren**  
   Erstellen Sie im `src/`-Verzeichnis eine `metadata.yaml` mit den Pflichtfeldern: `title`, `creator`, `identifier` (z. B. `urn:uuid:...`), `language: de`. Nutzen Sie das Kommando `free_ebook_maker validate --schema schema/metadata_v3.json` zur automatischen Prüfung. Platzieren Sie Kapitel als separate `.md`-Dateien in `chapters/`; verwenden Sie `<!-- chapter:break -->` für Seitenumbrüche.

3. **Build ausführen und Ausgabe prüfen**  
   Starten Sie den asynchronen Build mit `free_ebook_maker build --watch --output-dir ./dist`. Der `--watch`-Modus überwacht Änderungen in `src/` und startet einen inkrementellen Build (< 2 Sekunden bei 50 Kapiteln). Nach Abschluss prüfen Sie die EPUB-Datei mit `epubcheck dist/meinbuch.epub`. Für MOBI-Ausgabe fügen Sie `--format mobi` hinzu; die Konvertierung erfolgt via KindleGen 2.9 (vorinstalliert im `tools/`-Ordner).

### Beste Praktiken

- **Praxis 1: Deterministische Builds durch Lock-Dateien und Caching**  
  Aktivieren Sie `--cache-dir ./build-cache` im Build-Befehl. Dadurch werden Zwischenergebnisse (kompilierte Markdown, optimierte Bilder) als Parquet-Dateien gespeichert. Bei unveränderten Quelldateien wird der Build auf unter 100 ms reduziert. Verwenden Sie `free_ebook_maker clean --cache` nur bei grundlegenden Änderungen am CSS oder an der Metadaten-Struktur. Dies verhindert inkrementelle Fehler durch veraltete Caches.

- **Praxis 2: Barrierefreiheit und EPUB-Accessibility-Standards**  
  Fügen Sie in jeder `metadata.yaml` das Feld `accessibility: ["WCAG2.1", "EPUB-A11Y"]` hinzu. Nutzen Sie den Befehl `free_ebook_maker a11y-check --report a11y_report.json`, um automatisch Alt-Texte auf Bilder zu prüfen (erwartet `![Alt-Text](bild.jpg)`). Bei Fehlen wird der Build mit Exit-Code 1 abgebrochen. Integrieren Sie diesen Check in Ihre CI/CD-Pipeline (z. B. GitHub Actions) vor dem Deployment.
