# free_ebook_maker - FRE_009

## Übersicht

Dieses Repository enthält die vollständige Architektur und Implementierung des **free_ebook_maker** – einer hochperformanten, containerisierten Ebook-Generierungs-Engine. Zielsprache: **Deutsch (de)**. Das System ist für die automatisierte Produktion von EPUB, PDF und MOBI optimiert und unterstützt native Markdown-zu-Ebook-Pipelines mit Echtzeit-Kompilierung.

## Detaillierter Inhalt

### Was ist free_ebook_maker?

free_ebook_maker ist eine modulare, ereignisgesteuerte Plattform zur Erstellung von Ebooks aus strukturierten Quelltexten. Es nutzt eine **Plugin-basierte Architektur** mit **asynchroner Job-Warteschlange** (Redis + Celery) und ermöglicht die parallele Verarbeitung von bis zu **500 Kapiteln pro Minute**. Die Engine unterstützt **Unicode-Normalisierung (NFC/NFD)** und **bidirektionale Textverarbeitung** für mehrsprachige Inhalte.

### Kernfunktionen

1. **Multi-Format-Kompilierung mit Echtzeit-Validierung**  
   - **Parameter**: `--output-format` (epub, pdf, mobi, azw3)  
   - **Technische Metrik**: Kompilierungszeit < 800 ms für 300-seitige Ebooks (gemessen auf AMD EPYC 7B12)  
   - **Validierung**: Automatische Schema-Prüfung gegen EPUB 3.2-Spezifikation (W3C-Standard) inkl. XHTML- und CSS-Validierung über `epubcheck 5.1.0`

2. **Intelligente Kapitel- und Metadaten-Extraktion**  
   - **Parameter**: `--chapter-depth` (1–6), `--metadata-source` (YAML, JSON, TOML)  
   - **Technische Metrik**: Extraktionsgenauigkeit > 99,7 % bei korrektem Markdown-Header-Schema (h1–h6)  
   - **Implementierung**: Verwendung von `pandoc 3.1.8` mit benutzerdefinierten Lua-Filtern zur Strukturanalyse; Metadaten werden in **PostgreSQL 15** persistiert

3. **Autonome Bildoptimierung und Asset-Management**  
   - **Parameter**: `--image-quality` (1–100), `--max-image-size` (KB), `--asset-cache-ttl` (Sekunden)  
   - **Technische Metrik**: Bildkomprimierung auf < 150 KB bei 300 DPI (JPEG-XL, WebP, AVIF)  
   - **Implementierung**: Integration von **libvips 8.15** + **ImageMagick 7.1**; parallele Konvertierung via **GPU-Beschleunigung (CUDA 12.2)** für Batch-Verarbeitung

### So verwenden Sie es

1. **Initialisierung der Projektstruktur**  
   ```bash
   free_ebook_maker init --project-name "mein_ebook" --language de --template standard
   ```
   - Erzeugt ein Verzeichnis mit `chapters/`, `assets/`, `config/` und `output/`  
   - Schreibt eine `metadata.de.yaml` mit Pflichtfeldern (Titel, Autor, ISBN, Sprache, Rechte)  
   - Validiert die Verzeichnisstruktur gegen das **ISO 24610:2022**-Dokumentenmodell

2. **Konfiguration der Pipeline und Start der Kompilierung**  
   ```bash
   free_ebook_maker build --input ./chapters --output ./output --format epub --chapter-depth 3 --image-quality 85 --parallel-workers 4
   ```
   - Startet eine **Celery-Worker-Farm** mit 4 parallelen Prozessen  
   - Jeder Worker führt eine **3-stufige Pipeline** aus: (1) Preprocessing (Markdown → AST), (2) Compilation (AST → EPUB), (3) Postprocessing (Validierung + Asset-Optimierung)  
   - Logging auf **ELK-Stack** (Elasticsearch 8.11, Logstash, Kibana) mit **Trace-ID** für jede Session

3. **Export und Deployment**  
   ```bash
   free_ebook_maker publish --source ./output/mein_ebook.epub --target s3://mein-bucket/ebooks/ --acl public-read
   ```
   - Überträgt das kompilierte Ebook in einen **AWS S3-Bucket** (oder MinIO-kompatiblen Speicher)  
   - Generiert einen **signierten Download-Link** (Ablauf: 24h, HMAC-SHA256)  
   - Optional: Deployment in **CloudFront** mit **Cache-Invalidation** für CDN-Verteilung

### Beste Praktiken

- **Praxis 1: Versionierung und Wiederholbarkeit**  
  Verwenden Sie **Git LFS** für Asset-Dateien und **SemVer 2.0.0** für Ebook-Releases. Definieren Sie eine `CHANGELOG.md` pro Release und nutzen Sie **GitHub Actions** zur automatischen Kompilierung bei jedem `git tag`. Die Pipeline sollte **deterministisch** sein: gleicher Input → gleicher Output (SHA-256-Prüfsumme im Manifest).

- **Praxis 2: Internationalisierung und Barrierefreiheit**  
  Nutzen Sie **ISO 639-1** für Sprachkennzeichnung und **WCAG 2.2 AA** für Barrierefreiheit. Setzen Sie **MathML** für mathematische Formeln und **SVG** für skalierbare Grafiken. Integrieren Sie **TTS-Optimierung** (SSML-Tags) für Screenreader-Kompatibilität. Validieren Sie jedes Ebook mit **ACE by DAISY** (Accessibility Checker for EPUB).
