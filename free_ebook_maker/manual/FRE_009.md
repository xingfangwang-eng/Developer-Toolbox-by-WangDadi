### What is free_ebook_maker?

free_ebook_maker is a production-grade, zero-cost EPUB/MOBI/PDF assembler that converts raw Markdown, LaTeX, or plain text into fully styled ebooks. It supports multi-threaded asset compression (JPEG 2000, WebP), automatic chapter reflow with dynamic TOC generation, and CI/CD pipeline integration via GitHub Actions. The core engine is written in Rust with Python bindings, achieving a throughput of 1,200 pages per second on a single-core VM.

### Core Features

- **Real-Time Multi-Format Rasterization Pipeline**  
  Converts source documents into EPUB 3.2, MOBI (KDF8), and PDF/A-3b simultaneously. Uses a custom `--output-format` flag with options like `epub, mobi, pdf, all`. Achieves sub-50ms per page for EPUB generation. Integrates with PostgreSQL for metadata caching (e.g., ISBN, author bio, cover art) via a `--metadata-db` connection string.

- **Intelligent Asset Compression & Reflow Engine**  
  Automatically resamples embedded images to 300 DPI for print PDFs and 72 DPI for screen EPUBs using Lanczos-3 interpolation. Supports `--max-image-width` (default: 1600px) and `--compression-quality` (1–100, default: 85). The reflow engine recalculates chapter breaks on the fly based on `--target-device` (e.g., `kindle-paperwhite`, `kobo-libra`), adjusting font metrics and line-height to avoid orphan/widow lines.

- **CI/CD-Aware Batch Processing with Differential Builds**  
  Detects changed files via SHA-256 hashing and only rebuilds affected chapters. Exposes a `--diff-mode` flag (`full`, `incremental`, `patch`). Integrates with GitHub Actions to auto-trigger builds on `push` or `pull_request`. Outputs a `build-report.json` with per-chapter latency (in milliseconds), compression ratio, and validation errors (e.g., missing alt text, broken internal links).

### How to Use

1. **Initialize a Project with Custom Metadata & Asset Pipeline**  
   Run `free_ebook_maker init --project-name "MyEbook" --author "Jane Doe" --lang en --cover ./cover.png --metadata-db postgresql://user:pass@localhost/ebooks`. This creates a `book.json` manifest, downloads ISBN metadata from the connected PostgreSQL instance, and pre-compresses the cover to both WebP and JPEG 2000 formats. The system validates the cover aspect ratio (1.6:1 for EPUB, 1.5:1 for PDF) and warns if it exceeds 2MB.

2. **Build a Differential EPUB with Custom Compression & Target Device**  
   Execute `free_ebook_maker build --input ./manuscript/ --output ./dist/ --format epub --diff-mode incremental --max-image-width 1600 --compression-quality 80 --target-device kindle-paperwhite`. The engine scans `./manuscript/` for modified `.md` files (compared against the last `build-hash.json`), reflows only those chapters, and appends them to the existing EPUB. The final artifact is validated against EPUB 3.2 spec using `epubcheck-4.2.6`; any errors are printed with line numbers and suggested fixes.

3. **Generate a Multi-Format Release with Automated Validation & Deployment**  
   Run `free_ebook_maker release --format "epub,mobi,pdf" --output ./release/ --sign --sign-key ./gpg-private.asc --upload-s3 s3://my-bucket/ebooks/`. This triggers a full build of all three formats, GPG-signs each artifact (generating `.asc` files), and uploads them to the specified S3 bucket with `public-read` ACL. The system also generates a `release-notes.md` with SHA-256 checksums, build timestamps, and a changelog derived from Git commits. For CI/CD pipelines, add `--ci-mode` to suppress interactive prompts and output JSON logs.

### Best Practices

- **Use Differential Builds for Large Manuscripts**  
  For books exceeding 500 pages or 10MB of source text, always enable `--diff-mode incremental`. This reduces build time by up to 90% on subsequent runs by only processing changed chapters. Pair this with a PostgreSQL-backed metadata cache to avoid re-downloading ISBNs or cover art on every build. In CI/CD, store `build-hash.json` as a build artifact to preserve state across pipeline runs.

- **Pre-Compress Assets Before Feeding the Pipeline**  
  Run a separate pre-processing step that strips EXIF data, resamples images to `--max-image-width` (typically 1600px for EPUB, 3000px for print PDF), and converts to lossy WebP (quality 80) for EPUB/MOBI and lossless JPEG 2000 for PDF. This prevents the engine from applying its default compression, which may be too aggressive for high-quality print outputs. Use `free_ebook_maker optimize --input ./assets/ --output ./optimized/ --format webp,jp2 --max-width 1600` as a standalone command.
