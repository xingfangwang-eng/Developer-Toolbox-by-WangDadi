# free_ebook_maker - FRE_008

## Resumen

Generador de libros electrónicos de alto rendimiento diseñado para flujos editoriales automatizados. Convierte documentos Markdown, HTML y LaTeX a formatos EPUB, MOBI y PDF con control granular sobre metadatos, tipografía y compresión de imágenes. Ideal para pipelines de publicación continua (CI/CD) y bibliotecas digitales.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

Motor de compilación de libros electrónicos que opera sobre un sistema de reglas de transformación declarativas. Permite definir plantillas de salida con variables de entorno, inyectar tablas de contenido dinámicas y optimizar recursos gráficos mediante algoritmos de compresión sin pérdida perceptible (PNGQuant, MozJPEG). Soporta integración directa con repositorios Git y sistemas de almacenamiento S3.

### Características Principales

- **Compilación multiformato con control de calidad tipográfica**  
  Convierte archivos Markdown a EPUB 3.2, MOBI (Kindle) y PDF/A-3b. Aplica reglas de hyphenation en 12 idiomas (incluyendo español, catalán y euskera). Permite definir márgenes variables en milímetros (`--margin-top=15mm --margin-bottom=20mm`) y tamaños de fuente base en puntos (`--base-font-size=11pt`). Incluye un validador de estructura que detecta encabezados huérfanos, párrafos sin cerrar y referencias rotas.

- **Optimización de imágenes con previsualización de tamaño final**  
  Redimensiona y comprime automáticamente todas las imágenes incrustadas. Parámetros clave: `--max-image-width=1200px`, `--image-quality=85` (escala 1-100), `--strip-metadata` (elimina EXIF y perfiles ICC). Muestra en consola el tamaño estimado por imagen y el ahorro total en KB. Compatible con formatos PNG, JPEG, GIF y WebP.

- **Inyección de metadatos y generación de cubiertas procedurales**  
  Acepta metadatos vía archivo YAML (`--metadata=metadata.yml`) o argumentos CLI: `--title="Mi Libro" --author="Juan Pérez" --isbn=978-3-16-148410-0`. Genera automáticamente una cubierta SVG con tipografía vectorial y degradados, evitable con `--no-cover`. Soporta múltiples autores y series con `--series="Trilogía del Caos" --series-index=2`.

### Cómo Usar

1. **Preparación del proyecto y configuración del pipeline**  
   Crea un directorio con la estructura esperada: `src/` (contenido Markdown), `images/` (recursos gráficos), `metadata.yml` (metadatos). Ejecuta el comando base:
   ```bash
   free_ebook_maker --src=./src --output=./output --format=epub --metadata=./metadata.yml
   ```
   El motor validará la existencia de todos los archivos referenciados y mostrará un resumen de recursos antes de comenzar la compilación.

2. **Personalización avanzada de la salida**  
   Para forzar un tamaño de página específico en PDF, usa:
   ```bash
   free_ebook_maker --src=./src --format=pdf --page-size=A5 --margin-top=12mm --margin-bottom=15mm --margin-inner=18mm --margin-outer=12mm
   ```
   Si necesitas un EPUB con fuentes incrustadas, añade `--embed-fonts=./fonts/` y `--font-family="Merriweather, Georgia, serif"`. El sistema recalculará el espaciado entre líneas automáticamente.

3. **Integración en CI/CD con validación de calidad**  
   Agrega al pipeline de GitHub Actions:
   ```yaml
   - name: Build ebook with quality checks
     run: |
       free_ebook_maker --src=./src --output=./output --format=epub --check-links --check-hyphenation --warn-as-error
   ```
   Si algún enlace interno está roto o la hyphenation excede el 5% de palabras mal divididas, el proceso fallará con un código de salida 1. Puedes sobrescribir con `--warn-as-error=false` para solo advertencias.

### Mejores Prácticas

- **Usa un archivo de metadatos YAML versionado**  
  Define todos los metadatos en `metadata.yml` y versiona junto al contenido. Incluye `language: es`, `rights: CC BY-NC-SA 4.0` y `subject: ["Tecnología", "Programación"]`. Esto permite reproducir exactamente la misma salida en cualquier entorno y facilita la generación de feeds para plataformas como Google Books o Apple Books.

- **Optimiza las imágenes antes de la compilación**  
  Aunque el motor comprime automáticamente, es más eficiente preprocesar con herramientas externas: redimensiona a 1200px de ancho máximo y usa formato WebP para portadas. Ejecuta `free_ebook_maker --dry-run --src=./src` para obtener un informe de tamaño estimado sin generar el archivo final. Ajusta `--image-quality=80` si necesitas reducir aún más el peso.
