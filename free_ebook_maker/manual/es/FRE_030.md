# free_ebook_maker - FRE_030

## Resumen

`free_ebook_maker` es un generador de libros electrónicos de alto rendimiento, diseñado para transformar documentos Markdown, HTML y LaTeX en archivos EPUB, MOBI y PDF con calidad de publicación profesional. Optimizado para flujos de trabajo editoriales complejos, soporta metadatos multilenguaje, tablas de contenido dinámicas y compresión de imágenes con pérdida controlada (JPEG 85% – 95% de calidad, PNG cuantizado a 256 colores). Ideal para autores técnicos, traductores y equipos de documentación que requieren precisión tipográfica y tiempos de compilación inferiores a 2 segundos por capítulo (medido en hardware estándar: Intel i7-12700, 32 GB RAM, SSD NVMe).

## Contenido Detallado

### ¿Qué es free_ebook_maker?

`free_ebook_maker` es una herramienta de compilación de libros electrónicos que opera sobre un pipeline modular: parseo semántico → renderizado CSS personalizado → empaquetado conforme a los estándares EPUB 3.3 y MOBI 7/8. A diferencia de soluciones genéricas como Calibre o Pandoc, este proyecto implementa un motor de fragmentación inteligente de capítulos basado en la profundidad de los encabezados (`<h1>`–`<h6>`), con soporte para referencias cruzadas automáticas mediante anclas SHA-256. Su núcleo está escrito en Go 1.22, con bindings para Python 3.11+ (vía cgo), permitiendo scripts de preprocesamiento personalizados sin sacrificar rendimiento.

### Características Principales

- **Compilación paralela con control de dependencias**  
  El motor `parallel_build` divide el manuscrito en fragmentos de 500 líneas (ajustable mediante `--chunk-size=500`) y los compila simultáneamente en hasta 8 hilos (`--max-workers=8`). Cada hilo gestiona su propia caché de fuentes (TTF/WOFF2) y tabla de estilos CSS, reduciendo el tiempo total de compilación en un 40% respecto a la ejecución secuencial. La sincronización se realiza mediante un grafo acíclico dirigido (DAG) que resuelve dependencias de imágenes y fragmentos de código incrustados.

- **Validación de metadatos con esquema Dublin Core extendido**  
  Incorpora un validador que comprueba la integridad de los metadatos EPUB según ISO 24610-1:2023. Exige al menos: `dc:title`, `dc:creator` (con ORCID opcional), `dc:language` (código ISO 639-1), y `dc:identifier` (formato ISBN-13 o UUID v4). Si falta algún campo obligatorio, el compilador lanza un error fatal con el mensaje: `"ERROR: El campo 'dc:language' es requerido. Use '--meta-language=es' para sobrescribir."`. Además, genera un informe JSON con las advertencias de cumplimiento (ej.: `"WARNING: 'dc:publisher' ausente. Se usará 'Unknown Publisher' por defecto."`).

- **Compresión de imágenes con cuantización adaptativa**  
  El módulo `img_optimizer` procesa imágenes JPEG, PNG y WebP de forma transparente. Para JPEG: aplica compresión con calidad configurable (`--jpeg-quality=90`, rango 1–100), utilizando la librería `libjpeg-turbo` 3.0. Para PNG: reduce la profundidad de color a 8 bits por canal y aplica cuantización con el algoritmo de mediana de color (256 colores máximo). Las imágenes WebP se convierten a PNG si superan los 2 MB, evitando artefactos de compresión en gráficos vectoriales. El tamaño final de cada imagen se limita a 500 KB por defecto (`--max-image-size=500KB`), redimensionando las que excedan manteniendo la relación de aspecto.

### Cómo Usar

1. **Configurar el proyecto y definir la estructura de capítulos**  
   Cree un directorio raíz con la siguiente jerarquía obligatoria:
   ```
   /mi-libro/
   ├── manuscript/
   │   ├── 01-introduccion.md
   │   ├── 02-capitulo1.md
   │   └── 03-capitulo2.md
   ├── images/
   │   ├── portada.jpg
   │   └── figura1.png
   ├── styles/
   │   └── epub.css
   └── metadata.yaml
   ```
   El archivo `metadata.yaml` debe contener al menos:
   ```yaml
   title: "Mi Libro Técnico"
   creator: "Autor Principal"
   language: es
   identifier: "urn:uuid:123e4567-e89b-12d3-a456-426614174000"
   publisher: "Editorial Ejemplo"
   ```
   Ejecute luego el comando de validación:
   ```bash
   free_ebook_maker validate --project-dir=/mi-libro
   ```
   Si la validación falla, el programa retorna un código de salida distinto de cero y lista los errores en stderr.

2. **Compilar el libro electrónico con parámetros personalizados**  
   Inicie la compilación especificando el formato de salida y opciones de optimización:
   ```bash
   free_ebook_maker build \
     --project-dir=/mi-libro \
     --output-format=epub \
     --chunk-size=400 \
     --max-workers=6 \
     --jpeg-quality=85 \
     --max-image-size=300KB \
     --output-file=/output/mi-libro.epub
   ```
   Durante la compilación, el motor muestra en tiempo real el progreso de cada hilo (ej.: `[worker-3] Compilando 02-capitulo1.md... OK (1.2s)`). Al finalizar, se genera un informe de compilación en `/output/mi-libro_build_report.json` con métricas detalladas: tiempo total, imágenes comprimidas, fragmentos procesados y advertencias.

3. **Verificar la integridad del EPUB generado**  
   Tras la compilación, ejecute la herramienta de verificación integrada:
   ```bash
   free_ebook_maker verify --input-file=/output/mi-libro.epub --check-epub3.3
   ```
   Este comando valida el archivo contra el estándar EPUB 3.3 (usando `epubcheck` 5.0.1 como backend). Si se detectan errores de estructura (ej.: enlaces rotos, archivos no referenciados en el OPF), se listan con su línea exacta dentro del ZIP. Además, se genera un hash SHA-256 del archivo para verificar integridad posterior: `SHA256: a3f2c1e4...`.

### Mejores Prácticas

- **Fragmentación temprana para libros extensos**  
  Para manuscritos de más de 100 capítulos (o 50,000 palabras), divida el proyecto en múltiples directorios de `manuscript/` con un archivo `toc.yaml` que los enlace. Use `--chunk-size=200` para mantener el uso de RAM por debajo de 512 MB. Esto evita cuellos de botella en la fase de ensamblaje del EPUB, especialmente cuando se incluyen imágenes de alta resolución (≥ 300 DPI). Además, active la compresión GZIP en los archivos de salida con `--compress-output` para reducir el tamaño final en un 15% adicional.

- **Integración continua con hooks de pre-commit**  
  Configure un hook de Git que ejecute `free_ebook_maker validate` antes de cada commit. En `.git/hooks/pre-commit`, añada:
  ```bash
  #!/bin/bash
  free_ebook_maker validate --project-dir=. --strict-mode
  if [ $? -ne 0 ]; then
    echo "ERROR: La validación del proyecto falló. Revise metadata.yaml y la estructura de directorios."
    exit 1
  fi
  ```
  Esto garantiza que cualquier cambio en los metadatos o la estructura de capítulos sea validado antes de integrarse al repositorio. Combine con un pipeline CI/CD (GitHub Actions o GitLab CI) que ejecute `free_ebook_maker build` y `free_ebook_maker verify` en cada push, generando un artefacto EPUB listo para descarga.
