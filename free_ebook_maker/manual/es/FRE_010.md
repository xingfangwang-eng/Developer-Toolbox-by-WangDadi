# free_ebook_maker - FRE_010

## Resumen

**free_ebook_maker** es un sistema de compilación y generación automatizada de libros electrónicos en formato EPUB, MOBI y PDF, diseñado para operar con pipelines de transformación Markdown → HTML → ePUB, con soporte nativo para metadatos Dublin Core, tablas de contenido jerárquicas y estilos CSS embebidos. Está optimizado para flujos de trabajo CI/CD, permitiendo integración directa con repositorios Git y sistemas de gestión de documentos.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es una herramienta de línea de comandos y biblioteca Python que convierte documentos Markdown estructurados en libros electrónicos listos para distribución. Está diseñada para autores técnicos, documentalistas y equipos de DevOps que necesitan generar ebooks consistentes, con control de versiones, sin depender de servicios externos. El núcleo del sistema procesa archivos Markdown con frontmatter YAML, aplica transformaciones XSLT sobre el HTML intermedio y empaqueta el resultado siguiendo el estándar OPS (Open Publication Structure) 2.0.1.

### Características Principales

- **Generación con metadatos dinámicos y validación de esquema Dublin Core**: El sistema lee el frontmatter YAML de cada capítulo (parámetros `title`, `creator`, `subject`, `description`, `language`, `rights`) y construye automáticamente el archivo `content.opf` con validación de campos obligatorios. Si falta `language` o `creator`, el pipeline falla con código de error 0xE002. Soporta múltiples autores mediante listas separadas por `|`.

- **Procesamiento paralelo de capítulos con control de dependencias**: Cada archivo Markdown se procesa en un hilo independiente (pool de hasta 8 workers por defecto, configurable con `--max-workers`). Se genera un archivo XHTML por capítulo con resolución de referencias cruzadas internas (anclas `#` entre archivos). El tiempo medio de procesamiento por capítulo es de 12 milisegundos en hardware moderno (CPU Intel i7-12700, SSD NVMe). Los capítulos se ordenan según la clave `order` en el frontmatter; si hay conflictos, se prioriza el orden lexicográfico del nombre del archivo.

- **Compilación de índices y tablas de contenido con profundidad configurable**: El motor genera una tabla de contenidos (NCX) con hasta 6 niveles de anidamiento (configurable mediante `--toc-depth`). Soporta etiquetas `navPoint` con `playOrder` secuencial. Además, construye un índice alfabético de términos marcados con `[index: término]` en el Markdown, generando un archivo `index.html` con enlaces a las páginas correspondientes. La compilación del índice se realiza en memoria con un árbol Trie para búsqueda O(n) sobre el total de términos.

### Cómo Usar

1. **Preparación del proyecto y configuración del pipeline**  
   Cree un directorio con la estructura estándar:  
   ```
   /mi-libro/
   ├── config.yaml
   ├── chapters/
   │   ├── 01-introduccion.md
   │   ├── 02-conceptos.md
   │   └── 03-avanzado.md
   └── assets/
       └── portada.png
   ```  
   En `config.yaml`, defina los metadatos globales:  
   ```yaml
   title: "Guía Avanzada de Sistemas Distribuidos"
   creator: "Ana Martínez|Carlos Ruiz"
   language: es
   rights: "CC BY-NC-SA 4.0"
   toc_depth: 4
   output_format: epub
   compression_level: 9
   ```  
   Ejecute: `free_ebook_maker build --project ./mi-libro --output ./output`

2. **Inyección de estilos CSS y personalización de tipografía**  
   Cree un archivo `styles/custom.css` y enlácelo desde `config.yaml`:  
   ```yaml
   stylesheets:
     - styles/custom.css
   ```  
   El CSS se embeberá directamente en cada XHTML (sin referencias externas). Soporta reglas `@font-face` para fuentes WOFF2. Ejemplo de regla para mejorar legibilidad en e-readers:  
   ```css
   body { font-family: 'Georgia', serif; font-size: 1.1em; line-height: 1.6; }
   code { background: #f4f4f4; padding: 0.2em 0.4em; border-radius: 3px; }
   ```  
   Para validar que el CSS no exceda 50 KB (límite recomendado para Kindle), use: `free_ebook_maker validate --css-size ./styles/custom.css`

3. **Compilación final y firma digital del ebook**  
   Una vez procesados todos los capítulos, ejecute el empaquetado:  
   ```bash
   free_ebook_maker package --input ./output/xhtml --opf ./output/content.opf --ncx ./output/toc.ncx --output ./libro.epub
   ```  
   El sistema genera un archivo EPUB válido que puede ser verificado con `epubcheck`. Para añadir una firma digital (hash SHA-256 del archivo `META-INF/container.xml`), use:  
   ```bash
   free_ebook_maker sign --input ./libro.epub --cert ./cert.pem --key ./key.pem --output ./libro-firmado.epub
   ```  
   El tiempo total de compilación para un libro de 20 capítulos (150 páginas) es inferior a 2 segundos en un sistema con 16 GB RAM.

### Mejores Prácticas

- **Gestión de versiones mediante Git LFS para assets binarios**: Almacene las imágenes, fuentes y portadas en repositorios Git con Large File Storage (LFS). Configure un hook de pre-commit que ejecute `free_ebook_maker validate --check-links` para asegurar que no haya referencias rotas a archivos dentro del proyecto. Esto evita errores de compilación en pipelines CI/CD. Además, use etiquetas semánticas (`v1.0.0`, `v1.1.0`) para congelar versiones de ebook y poder regenerar exactamente el mismo artefacto en el futuro.

- **Pruebas de regresión visual con diff de XHTML**: Antes de lanzar una nueva versión, ejecute `free_ebook_maker diff --before ./release-1.0 --after ./release-1.1` para comparar los XHTML generados. La herramienta reporta diferencias en estructura de párrafos, tablas y elementos multimedia. Si el diff muestra cambios no deseados (por ejemplo, un `h2` convertido a `h3`), revise el frontmatter o las plantillas XSLT. Esta práctica es crítica para mantener la consistencia en documentación técnica que se actualiza con frecuencia.
