# free_ebook_maker - FRE_023

## Resumen Técnico

**free_ebook_maker** es un motor de compilación modular diseñado para transformar documentos Markdown, HTML y texto plano en libros electrónicos EPUB 3.0 y MOBI compatibles con Kindle, Kobo y Google Play Books. El sistema opera bajo un pipeline de procesamiento orientado a eventos, con capacidad de *inline asset embedding* y generación de metadatos semánticos RDFa.

## Funcionalidades Principales

### 1. Compilación Asíncrona con Control de Dependencias
- **Parámetro**: `--thread-pool=N` (por defecto 4, máximo 16).
- **Indicador técnico**: Tiempo de compilación medido en milisegundos por capítulo; soporte para archivos `.md` con frontmatter YAML/TOML.
- **Detalle**: El orquestador `PipelineOrchestrator` resuelve automáticamente el orden de capítulos mediante un grafo acíclico dirigido (DAG), permitiendo compilaciones parciales con `--resume-from=capitulo_05`.

### 2. Inyección de Metadatos y DRM Esquemático
- **Parámetro**: `--metadata-file=meta.json` (estructura Dublin Core extendida).
- **Indicador técnico**: Validación contra esquema JSON con 47 campos obligatorios, incluyendo `dcterms:audience` y `dcterms:educationLevel`.
- **Detalle**: El módulo `MetadataInjector` integra etiquetas de accesibilidad WCAG 2.1 AA y permite firmar el EPUB con clave GPG vía `--signature-key=0x...`.

### 3. Optimización de Imágenes con Compresión Adaptativa
- **Parámetro**: `--image-quality=85` (0–100), `--max-width=1200px`.
- **Indicador técnico**: Conversión automática a WebP con fallback a PNG; reducción de peso promedio del 62% en pruebas con 10,000 assets.
- **Detalle**: El `AssetPipeline` emplea un modelo heurístico que detecta imágenes de baja resolución (<72 DPI) y las reemplaza por placeholders vectorizados SVG.

## Cómo Usar

### Paso 1: Inicialización del Proyecto y Configuración del Pipeline
```bash
# Clonar repositorio y preparar entorno
git clone https://github.com/xingfangwang-eng/free_ebook_maker.git
cd free_ebook_maker
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Generar estructura base con plantillas de capítulos
./free_ebook_maker init --template=technical-manual --output-dir=./mi_libro
```

### Paso 2: Compilación con Validación de Integridad
```bash
# Compilar EPUB con metadatos y firma digital
./free_ebook_maker build \
  --input=./mi_libro \
  --format=epub \
  --metadata-file=./mi_libro/meta.json \
  --image-quality=90 \
  --thread-pool=8 \
  --signature-key=0xABCD1234 \
  --output=./output/libro.epub

# Verificar estructura EPUB contra esquema oficial
./free_ebook_maker validate --epubcheck --input=./output/libro.epub
```

### Paso 3: Despliegue y Distribución Automatizada
```bash
# Convertir a MOBI con perfiles de dispositivo
./free_ebook_maker convert \
  --input=./output/libro.epub \
  --target=kindle_pw3 \
  --output=./output/libro.mobi

# Subir a bucket S3 con metadatos de indexación
./free_ebook_maker deploy \
  --provider=aws \
  --bucket=ebooks-prod \
  --access-key=AKIA... \
  --region=us-east-1
```

## Mejores Prácticas

### Práctica 1: Gestión de Assets con Hash de Contenido
Utiliza `--asset-hash=true` para generar nombres de archivo basados en SHA-256, lo que permite caché inmutable en CDNs y evita duplicados. En pipelines CI/CD, combínalo con `--incremental` para recompilar solo capítulos modificados, reduciendo el tiempo de build de 3 minutos a 12 segundos en proyectos con 200+ capítulos.

### Práctica 2: Estrategia de Metadatos Multilingüe
Define `dcterms:language` como un array en `meta.json` (ej: `["es", "en", "fr"]`) y activa `--multilingual=true` para que el compilador genere un EPUB con archivos `index_es.xhtml`, `index_en.xhtml` y enlaces `link rel="alternate"`. Esto mejora el SEO en Google Play Books y permite venta en mercados regionales sin duplicar el repositorio.
