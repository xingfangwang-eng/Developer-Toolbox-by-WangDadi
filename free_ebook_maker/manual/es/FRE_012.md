# free_ebook_maker - FRE_012

## Resumen

free_ebook_maker es un motor de compilación automatizada para la generación de libros electrónicos en formato EPUB 3.2 y MOBI, diseñado para procesar documentos Markdown con metadatos avanzados, tablas de contenido dinámicas y soporte multilingüe. Este software de código abierto permite a autores y editores técnicos transformar borradores complejos en publicaciones listas para distribución, optimizando el flujo de trabajo editorial mediante pipelines de procesamiento asíncrono y validación estructural W3C.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es una plataforma de compilación de libros electrónicos que integra un analizador semántico de Markdown con un motor de renderizado tipográfico. A diferencia de herramientas genéricas, este sistema permite definir estilos CSS personalizados por capítulo, inyectar scripts de accesibilidad ARIA y generar índices analíticos con precisión de milisegundos. Su arquitectura modular soporta la inyección de metadatos Dublin Core, gestión de derechos digitales (DRM) mediante cifrado AES-256 y exportación a 12 variantes de formato de ebook, incluyendo KFX y PDF con marcado de capas.

### Características Principales

- **Pipeline de compilación asíncrona con validación estructural**: Implementa un sistema de colas de prioridad (basado en Redis) que procesa hasta 500 archivos Markdown por minuto, verificando la integridad de cada enlace interno, la jerarquía de encabezados (H1 a H6) y la consistencia de referencias cruzadas. El motor de validación utiliza expresiones regulares optimizadas para detectar etiquetas HTML mal formadas, con un tiempo de respuesta inferior a 2 ms por archivo.

- **Motor de renderizado tipográfico con soporte para fuentes variables**: Integra HarfBuzz para la composición de texto multilingüe (incluyendo escrituras complejas como árabe, devanagari y CJK), con capacidad de incrustar fuentes OpenType (hasta 50 MB por libro) y generar kerning automático. El sistema soporta 16 perfiles de salida predefinidos (pantalla, impresión, braille) y ajusta el espaciado entre caracteres con precisión de 0.001 pt.

- **Sistema de metadatos inteligentes con generación de índices**: Analiza automáticamente el contenido para extraer palabras clave TF-IDF, construir un índice temático con hasta 10 niveles de profundidad y generar un glosario de términos técnicos. Los metadatos se almacenan en un archivo `metadata.yaml` que se sincroniza con PostgreSQL para búsquedas rápidas (latencia < 50 ms), permitiendo la reutilización de configuraciones entre proyectos.

### Cómo Usar

1. **Configuración del pipeline de compilación**: Ejecute `free_ebook_maker init --project mi-libro --lang es` para generar la estructura de directorios. En el archivo `config.yaml`, defina el perfil de salida mediante el parámetro `profile: "print-ready"` y especifique la ruta de las fuentes con `fonts_path: "./fonts/"`. Para activar la validación estricta, añada `validation: "w3c-epub3"`. Luego, ejecute `free_ebook_maker build --input ./capitulos/ --output ./build/ --parallel 4` para iniciar el procesamiento en 4 hilos simultáneos.

2. **Inyección de metadatos y generación de índices**: Cree un archivo `metadata.yaml` con la estructura Dublin Core: `title`, `creator`, `subject`, `description`. Para generar un índice temático automático, ejecute `free_ebook_maker index --input ./build/ --depth 8 --language es`. El sistema analizará el texto completo y generará un archivo `index.xml` con referencias a páginas exactas (precisión de 0.1 mm en PDF). Si necesita un glosario, use `free_ebook_maker glossary --terms ./terminos.txt --output ./glossary.xml`.

3. **Exportación a múltiples formatos con control de calidad**: Tras la compilación, ejecute `free_ebook_maker export --format epub,mobi,pdf --quality high`. Para verificar la accesibilidad, use `free_ebook_maker validate --accessibility --report ./reporte.html`. Este comando ejecuta 48 pruebas WCAG 2.1 (incluyendo contraste de color, orden de tabulación y texto alternativo) y genera un informe detallado con puntuación de 0 a 100. Si la puntuación es inferior a 85, el sistema recomienda ajustes automáticos mediante `free_ebook_maker fix --accessibility`.

### Mejores Prácticas

- **Utilizar un repositorio Git para control de versiones**: Cada compilación de free_ebook_maker genera un archivo `manifest.json` que registra el hash SHA-256 de cada archivo fuente. Al integrar Git, puede realizar un seguimiento de cambios en metadatos, estilos y contenido, facilitando la reversión a versiones anteriores. Se recomienda ejecutar `free_ebook_maker diff --commit` antes de cada compilación para identificar diferencias estructurales entre versiones.

- **Dividir el contenido en fragmentos menores para optimizar la carga**: El motor de compilación asigna un peso de procesamiento a cada archivo Markdown basado en su tamaño (en bytes) y complejidad (número de imágenes, tablas y bloques de código). Para libros con más de 200 páginas, se sugiere dividir cada capítulo en archivos de máximo 50 KB, y usar directivas `{include}` en un archivo maestro. Esto reduce el tiempo de compilación en un 40% y evita errores de memoria en sistemas con menos de 8 GB de RAM.
