# free_ebook_maker - FRE_018

## Resumen

free_ebook_maker es un motor de compilación automatizada de libros electrónicos diseñado para transformar contenido Markdown, LaTeX y HTML en formatos EPUB, MOBI y PDF con calidad de publicación profesional. Este proyecto se enfoca en la generación de libros con estructura semántica correcta, metadatos completos y compatibilidad total con dispositivos de lectura como Kindle, Kobo y Apple Books.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es una herramienta de línea de comandos y biblioteca Python que orquesta la conversión de documentos estructurados en libros electrónicos listos para distribución. A diferencia de soluciones genéricas, este proyecto implementa un pipeline de procesamiento que respeta la jerarquía de encabezados, tablas, imágenes vectoriales y ecuaciones matemáticas, garantizando que el resultado final cumpla con los estándares EPUB 3.2 y MOBI 7/8.

### Características Principales

- **Compilación semántica con validación de esquema XHTML**: El motor analiza el contenido Markdown y genera un árbol DOM validado contra el esquema EPUB 3.2. Utiliza `lxml` para verificar que todos los elementos `nav`, `section` y `aside` estén correctamente anidados. Si se detecta un error de jerarquía (por ejemplo, un `h3` sin un `h2` precedente), el proceso se detiene y muestra la línea exacta del error con el contexto del archivo fuente. El tiempo de validación promedio para un libro de 300 páginas es de 1.2 segundos.

- **Gestión de recursos multimedia con caché de imágenes**: Las imágenes incrustadas (PNG, JPEG, SVG) se procesan mediante `Pillow` y `cairosvg`. Se aplica un algoritmo de compresión con umbral de calidad configurable (parámetro `--image-quality`, rango 1-100, defecto 85). Las imágenes se redimensionan automáticamente a un ancho máximo de 1200 píxeles para EPUB y 800 píxeles para MOBI, manteniendo la relación de aspecto. El sistema implementa un caché LRU con capacidad para 500 entradas, reduciendo el tiempo de recompilación en un 40% cuando se modifica solo el texto.

- **Soporte de ecuaciones LaTeX con MathJax offline**: El parámetro `--math-render` permite elegir entre `svg` (defecto) y `html-css`. Cuando se selecciona `svg`, las ecuaciones se convierten en gráficos vectoriales mediante `pdflatex` y `dvisvgm`, con una resolución de 600 DPI. El motor inyecta automáticamente las fuentes STIX en el EPUB para garantizar la visualización correcta en dispositivos sin conexión. El tiempo de procesamiento para un libro con 50 ecuaciones complejas es inferior a 8 segundos en un procesador moderno.

### Cómo Usar

1. **Preparación del entorno y estructura de proyecto**: Crea un directorio con la siguiente estructura: `src/` (contenido Markdown), `images/` (recursos multimedia), `metadata.yaml` (metadatos del libro). Ejecuta `free_ebook_maker init --project-dir ./mi-libro` para generar la plantilla base. El archivo `metadata.yaml` debe incluir los campos `title`, `author`, `language` (ISO 639-1), `identifier` (ISBN o UUID) y `cover` (ruta a la imagen de portada). Ejemplo: `language: es`, `identifier: urn:uuid:123e4567-e89b-12d3-a456-426614174000`.

2. **Compilación con parámetros avanzados**: Ejecuta `free_ebook_maker build --input-dir ./src --output-dir ./dist --formats epub,mobi --image-quality 90 --math-render svg --toc-depth 3`. El parámetro `--toc-depth` controla la profundidad de la tabla de contenidos (valores 1-6). Si se omite, se genera automáticamente basándose en la jerarquía de encabezados. El sistema muestra en tiempo real el progreso: `[12:34:56] Procesando capítulo 5/12 (42%) - Validando esquema XHTML...`.

3. **Verificación post-compilación y exportación**: Después de la compilación, ejecuta `free_ebook_maker validate --file ./dist/libro.epub --check-epubcheck`. Esto descarga e integra `epubcheck` (versión 4.2.6) y ejecuta una validación completa contra el estándar EPUB 3.2. Los resultados se muestran en formato JSON: `{"errors": 0, "warnings": 2, "fatals": 0}`. Para exportar a Kindle, usa `free_ebook_maker kindlegen --input ./dist/libro.mobi --output ./dist/libro.azw3` (requiere tener instalado `kindlegen` en el PATH).

### Mejores Prácticas

- **Optimización de imágenes para dispositivos de tinta electrónica**: Utiliza el parámetro `--image-format gray` para convertir todas las imágenes a escala de grises con perfil de color sRGB. Esto reduce el tamaño del archivo EPUB en un 35% y mejora el tiempo de renderizado en dispositivos como Kindle Paperwhite. Además, establece `--image-max-width 600` para garantizar que las imágenes no excedan el ancho de pantalla típico (600 píxeles). Esta configuración es crítica para libros con muchas ilustraciones técnicas.

- **Gestión de metadatos para distribución en tiendas**: Incluye siempre el campo `rights` en `metadata.yaml` con la licencia (ej: `Creative Commons Attribution 4.0`). Para libros con múltiples autores, usa el formato `author: ["Nombre1 Apellido1", "Nombre2 Apellido2"]`. Añade `contributor` para traductores o ilustradores. El sistema genera automáticamente el archivo `content.opf` que Amazon y Apple Books requieren. Verifica que el `identifier` sea único; si usas ISBN, incluye el guion (ej: `978-3-16-148410-0`).
