# free_ebook_maker - FRE_026

## Resumen

**free_ebook_maker** es un motor de compilación automatizada de libros electrónicos diseñado para transformar contenido Markdown, HTML y JSON en formatos EPUB3, MOBI y PDF/A-3b con trazabilidad criptográfica. Este proyecto resuelve el problema de la generación masiva de ebooks con metadatos dinámicos, tablas de contenido jerárquicas y soporte para fuentes tipográficas embebidas, todo orquestado desde una interfaz CLI o API REST.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un pipeline de publicación digital que integra un analizador semántico de markdown (con soporte para frontmatter YAML/TOML), un compilador de estilos CSS con resolución de variables de diseño responsivo, y un empaquetador que genera checksums SHA-256 por capítulo. Está pensado para equipos de documentación técnica, editoriales independientes y desarrolladores que necesitan automatizar la salida de libros electrónicos con control de versiones y cumplimiento de estándares IDPF.

### Características Principales

- **Compilación multiformato con validación estructural**: El pipeline acepta árboles de archivos Markdown con anotaciones de metadatos (título, autor, ISBN, idioma, derechos) y los convierte simultáneamente a EPUB3 (conforme a la especificación 3.2 del IDPF), MOBI (compatible con KindleGen 2.9) y PDF/A-3b (con capa de accesibilidad). La validación incluye verificación de enlaces rotos, integridad de tablas de contenido y comprobación de metadatos Dublin Core mediante la librería `lxml` con esquemas XSD personalizados.

- **Inyección de recursos multimedia con control de calidad**: Soporta la incrustación automática de imágenes (PNG, JPEG, WebP) y fuentes tipográficas (WOFF2, TTF) con redimensionamiento inteligente basado en perfiles de salida (pantalla, impresión, braille). Cada recurso es optimizado con `Pillow` (para imágenes) y `fonttools` (para fuentes), garantizando un tamaño máximo por archivo de 10 MB y una resolución mínima de 300 DPI para PDF. Los metadatos EXIF se purgan automáticamente por defecto.

- **Generación de índices analíticos y tablas de contenido dinámicas**: El motor extrae encabezados de nivel 1 a 4 (h1-h4) y construye una TOC jerárquica en formato NCX (para EPUB) y Guide (para PDF). Además, genera un índice de términos clave basado en un diccionario de palabras reservadas definido por el usuario, con enlaces internos a las páginas correspondientes. El tiempo de generación para un libro de 500 páginas es inferior a 12 segundos en un procesador i7 de 11ª generación con 16 GB de RAM.

### Cómo Usar

1. **Preparación del entorno y estructura de proyecto**:  
   Clona el repositorio y ejecuta `pip install -r requirements.txt` (dependencias: `click>=8.0`, `Pillow>=9.0`, `lxml>=4.9`, `markdown>=3.4`, `pyyaml>=6.0`). Crea un directorio `src/` con un archivo `metadata.yaml` que contenga campos obligatorios: `title`, `author`, `language`, `isbn` (formato ISBN-13), `rights` (Creative Commons o Copyright). Cada capítulo debe ser un archivo Markdown separado con un header `# Título del Capítulo` y opcionalmente un bloque `<!-- chapter:uuid -->` para control de versiones.

2. **Configuración de perfiles de compilación**:  
   Edita el archivo `profiles.json` en la raíz del proyecto. Define tres perfiles: `epub`, `mobi` y `pdf`. Para `epub`, especifica `"css_profile": "epub3_standard"`, `"embed_fonts": true`, `"max_image_width": 1200`. Para `mobi`, usa `"kindlegen_path": "/usr/local/bin/kindlegen"` y `"compression_level": 9`. Para `pdf`, indica `"page_size": "A4"`, `"margin_top": 20` (mm), `"font_size_body": 11` (pt) y `"generate_bookmarks": true`. Ejecuta `python maker.py --profile epub --input src/ --output build/` para iniciar la compilación.

3. **Validación y despliegue con integridad criptográfica**:  
   Tras la compilación, el motor genera un archivo `checksums.sha256` en el directorio de salida, listando el hash de cada capítulo y del ebook completo. Para verificar la integridad, ejecuta `python maker.py --verify build/checksums.sha256`. Si se requiere firma digital, usa el flag `--sign` con una clave privada RSA de 2048 bits (formato PEM). El ebook firmado puede desplegarse directamente en una CDN o repositorio de assets, con un endpoint API que expone la URL de descarga y el hash de verificación.

### Mejores Prácticas

- **Mantén una convención estricta de nombres de archivos y metadatos**: Usa slugs normalizados para los nombres de capítulos (ej. `capitulo-01-introduccion.md`) y evita caracteres especiales en el frontmatter. Esto previene errores de codificación en el compilador de EPUB y garantiza la compatibilidad con sistemas de archivos NTFS y ext4. Además, incluye siempre el campo `revision` en el YAML (ej. `revision: 1.2.3`) para habilitar la generación de versiones semánticas en el pie de página de cada ebook.

- **Optimiza el contenido multimedia antes de la compilación**: Aunque el motor redimensiona automáticamente, es recomendable preprocesar imágenes a un ancho máximo de 1600 píxeles y usar formato WebP con pérdida (calidad 85%) para reducir el tiempo de compilación en un 40%. Para fuentes, limítate a un máximo de 3 familias tipográficas (una para cuerpo, otra para títulos y otra para código) y embebe solo los caracteres Latin Extended-A para minimizar el peso del archivo final.
