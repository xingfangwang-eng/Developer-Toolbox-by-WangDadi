# free_ebook_maker - FRE_003

## Resumen

free_ebook_maker es un sistema de generación automatizada de libros electrónicos en formato EPUB, MOBI y PDF, diseñado para entornos de producción con alta exigencia de fidelidad estructural. Permite la composición de documentos complejos a partir de fuentes Markdown, HTML y LaTeX, con un motor de renderizado que respeta tipografía avanzada, metadatos Dublin Core y tablas de contenido anidadas hasta 6 niveles. Está optimizado para pipelines CI/CD y despliegues serverless.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un compilador de libros digitales de código abierto que transforma documentos fuente en formatos de publicación estándar mediante un pipeline de procesamiento basado en reglas XSLT y un motor de maquetación por capas. El sistema gestiona automáticamente la división de capítulos, la generación de índices analíticos, la incrustación de fuentes tipográficas (WOFF2) y la normalización de imágenes (conversión a WebP, redimensionado a 300 DPI). Soporta inyección de metadatos MARC21 y validación de accesibilidad WCAG 2.1 AA.

### Características Principales

- **Motor de composición multiformato con resolución de dependencias transclusivas**: El núcleo `epub-engine` procesa archivos Markdown con directivas de inclusión (`{{include:./capitulo2.md}}`) y resuelve referencias cruzadas entre figuras, tablas y ecuaciones LaTeX. Utiliza un grafo acíclico para detectar dependencias circulares y generar un único archivo EPUB 3.2 con navegación por landmarks. El tiempo de compilación para un libro de 500 páginas es inferior a 4.2 segundos en un nodo con 4 vCPU y 8 GB de RAM.

- **Sistema de plantillas paramétricas con inyección de estilos CSS personalizados**: Permite definir plantillas de diseño mediante archivos YAML donde se especifican variables como `--font-size-body: 11pt`, `--line-height-base: 1.618`, `--margin-top-chapter: 2.5cm`. El motor aplica un preprocesador Sass para generar un CSS optimizado con prefijos de navegador y reglas de salto de página (`page-break-before: always`). Soporta la herencia de estilos desde un archivo `_variables.scss` global y la sobreescritura por capítulo.

- **Validación estructural y semántica con informes de calidad en JSON**: Cada compilación ejecuta un validador interno que verifica la integridad de la estructura del libro: existencia de portada, tabla de contenidos, metadatos obligatorios (título, autor, ISBN), y correcta jerarquía de encabezados (h1 a h6). El informe de salida incluye métricas como `total_paragraphs`, `image_count`, `broken_links`, `missing_alt_text` y `average_reading_time_minutes`. Los errores se reportan con línea y columna exactos del archivo fuente.

### Cómo Usar

1. **Configuración del proyecto y definición de metadatos**: Crea un archivo `ebook.yaml` en la raíz del proyecto con la siguiente estructura:
   ```yaml
   title: "Guía Avanzada de Procesamiento de Lenguaje Natural"
   author: "Carlos M. Fernández"
   language: es
   isbn: "978-84-123456-7-8"
   publisher: "Editorial Técnica"
   cover: "assets/portada.png"
   fonts:
     - "fonts/OpenSans-Regular.woff2"
     - "fonts/OpenSans-Bold.woff2"
   chapters:
     - "capitulos/01-introduccion.md"
     - "capitulos/02-fundamentos.md"
     - "capitulos/03-modelos.md"
   ```
   Ejecuta `free_ebook_maker init --config ebook.yaml` para generar la estructura de directorios y validar la existencia de los archivos referenciados.

2. **Compilación con parámetros de renderizado avanzados**: Lanza la compilación especificando el formato de salida y opciones de maquetación:
   ```bash
   free_ebook_maker build --config ebook.yaml --output-format epub,mobi,pdf \
     --pdf-page-size a5 \
     --pdf-margin-top 2cm \
     --pdf-margin-bottom 2.5cm \
     --epub-version 3.2 \
     --embed-fonts \
     --image-quality 85 \
     --image-max-width 1200px \
     --css-override "styles/custom.scss" \
     --metadata-inject "dc:rights=CC BY-NC-SA 4.0"
   ```
   El sistema generará tres archivos en el directorio `dist/`: `guia-pln.epub`, `guia-pln.mobi` y `guia-pln.pdf`. Durante el proceso, se muestra una barra de progreso con el número de capítulos procesados y el tiempo estimado restante.

3. **Validación y generación de informe de calidad**: Una vez compilado, ejecuta el validador para obtener un informe detallado:
   ```bash
   free_ebook_maker validate --input dist/guia-pln.epub \
     --check-accessibility \
     --check-structure \
     --check-metadata \
     --output report.json
   ```
   El archivo `report.json` contendrá un array de objetos con campos como: `severity` (error/warning/info), `location` (ruta del capítulo y línea), `message` (descripción del problema) y `suggested_fix` (código de corrección sugerido). Por ejemplo:
   ```json
   {
     "severity": "error",
     "location": "capitulos/02-fundamentos.md:145",
     "message": "La imagen 'figura-2.3.png' no tiene atributo alt",
     "suggested_fix": "Agregar `![Descripción de la figura](figura-2.3.png)`"
   }
   ```

### Mejores Prácticas

- **Estandarización de recursos multimedia**: Almacena todas las imágenes en un único directorio `assets/` con nombres normalizados en minúsculas y sin espacios. Usa exclusivamente formatos WebP para imágenes rasterizadas y SVG para gráficos vectoriales. Configura el pipeline para que redimensione automáticamente las imágenes a un ancho máximo de 1200px y las convierta a 300 DPI, garantizando una calidad uniforme en todos los dispositivos de lectura. Este enfoque reduce el tamaño total del libro electrónico en un 40-60% sin pérdida perceptible de calidad.

- **Control de versiones y compilaciones reproducibles**: Mantén el archivo `ebook.yaml` bajo control de versiones (Git) y etiqueta cada release con un número de versión semántico (v1.2.3). Utiliza el parámetro `--build-id` para inyectar un identificador único en los metadatos del libro (ej: `dc:identifier=urn:uuid:<build-id>`). Configura un workflow de GitHub Actions que ejecute `free_ebook_maker build` en cada push a la rama `main`, y que suba los artefactos generados a un release de GitHub. Esto asegura que cada versión del libro sea trazable y reproducible en cualquier momento.
