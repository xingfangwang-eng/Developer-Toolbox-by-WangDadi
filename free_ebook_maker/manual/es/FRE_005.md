# free_ebook_maker - FRE_005

## Resumen

free_ebook_maker es una herramienta de compilación y generación automatizada de libros electrónicos en formato EPUB y PDF, diseñada para transformar contenido Markdown, HTML y LaTeX en documentos profesionales listos para distribución editorial. Optimizada para flujos de trabajo CI/CD, permite la integración directa con repositorios Git y sistemas de gestión de contenidos.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de ensamblaje de libros electrónicos de alto rendimiento que procesa documentos estructurados mediante un pipeline de transformación en tres etapas: parsing semántico, renderizado tipográfico y empaquetado conforme a los estándares EPUB 3.2 e IDPF. Soporta salida simultánea en múltiples formatos con control granular sobre metadatos, tablas de contenido jerárquicas y estilos CSS personalizados.

### Características Principales

- **Pipeline de compilación asíncrona con control de versiones**: Implementa un sistema de compilación basado en DAG (Directed Acyclic Graph) que procesa hasta 50 documentos simultáneamente. Cada archivo fuente se parsea mediante un lexer personalizado que detecta automáticamente encabezados, listas, bloques de código y referencias cruzadas. El tiempo medio de compilación para un libro de 300 páginas es de 2.3 segundos en hardware estándar (CPU Intel i7-12700, 32GB RAM). Se integra con el sistema de archivos Git para detectar cambios incrementales y solo recompilar los capítulos modificados.

- **Motor de renderizado tipográfico con kerning avanzado**: Utiliza un algoritmo de composición tipográfica basado en Knuth-Plass modificado que optimiza la separación de palabras y el espaciado entre caracteres. Soporta más de 2000 familias tipográficas con métricas OpenType completas. El motor calcula automáticamente el interlineado óptimo (1.618 × tamaño de fuente) y ajusta el kerning a nivel de microtipografía con precisión de 0.001 mm. Incluye un validador de contraste de color WCAG 2.1 AA que garantiza la legibilidad en todos los dispositivos.

- **Sistema de empaquetado multi-formato con validación estricta**: Genera archivos EPUB, PDF, MOBI y DOCX mediante un motor de empaquetado que verifica cada archivo contra los esquemas XSD de la International Digital Publishing Forum (IDPF). El proceso incluye: compresión ZIP con nivel 9 y checksum SHA-256, generación de metadatos Dublin Core con validación de ISBN/DOI, y creación de archivos OPF (Open Packaging Format) con navegación jerárquica. La tasa de conformidad con EPUB 3.2 es del 99.97% en pruebas automatizadas con más de 10,000 casos de validación.

### Cómo Usar

1. **Configuración del pipeline de compilación automatizada**:
   - Crear un archivo `ebook_config.yaml` en la raíz del proyecto con los siguientes parámetros:
     ```yaml
     source:
       format: markdown
       directory: ./chapters
       encoding: UTF-8
       metadata: ./metadata.json
     output:
       format: [epub, pdf]
       compression: 9
       validation: strict
     typography:
       font-family: 'EB Garamond'
       font-size: 12pt
       line-height: 1.618
       kerning: true
     ```
   - Ejecutar `free_ebook_maker --config ebook_config.yaml` para iniciar la compilación. El sistema mostrará en tiempo real el progreso de cada etapa: parsing (0.4s), renderizado (1.2s), empaquetado (0.7s).

2. **Personalización avanzada de estilos CSS**:
   - Definir reglas CSS específicas para cada formato en `./styles/custom.css`. Por ejemplo:
     ```css
     @page {
       size: 6in 9in;
       margin: 0.75in;
     }
     h1 {
       font-size: 2.5em;
       text-align: center;
       page-break-before: always;
     }
     code {
       background-color: #f4f4f4;
       border-radius: 3px;
       padding: 2px 4px;
       font-family: 'Fira Code', monospace;
     }
     ```
   - Usar `free_ebook_maker --css ./styles/custom.css` para aplicar los estilos. El motor procesa las hojas de estilo con un parser CSS3 compatible con 98% de las propiedades estándar.

3. **Integración con CI/CD mediante GitHub Actions**:
   - Agregar el siguiente flujo de trabajo en `.github/workflows/ebook_build.yml`:
     ```yaml
     name: Build Ebook
     on:
       push:
         branches: [main]
       pull_request:
         branches: [main]
     jobs:
       build:
         runs-on: ubuntu-latest
         steps:
           - uses: actions/checkout@v3
           - name: Install free_ebook_maker
             run: pip install free-ebook-maker==2.1.0
           - name: Build EPUB and PDF
             run: free_ebook_maker --config ebook_config.yaml --output-dir ./build
           - name: Upload artifacts
             uses: actions/upload-artifact@v3
             with:
               name: ebook-artifacts
               path: ./build/*.epub
     ```
   - El pipeline se ejecuta automáticamente en cada push, generando los archivos en menos de 5 segundos para libros de hasta 500 páginas.

### Mejores Prácticas

- **Optimización de metadatos para distribución en plataformas**: Incluir siempre identificadores ISBN-13 o DOI en el archivo `metadata.json`, junto con una descripción de al menos 150 palabras y palabras clave relevantes. El motor de free_ebook_maker utiliza estos metadatos para generar automáticamente el archivo `toc.ncx` y `nav.xhtml` conforme a las especificaciones de Amazon KDP y Apple Books. Se recomienda usar el campo `dc:subject` con términos normalizados del BISAC Subject Headings para mejorar la indexación.

- **Control de versiones y colaboración mediante Git LFS**: Para proyectos con múltiples colaboradores, configurar Git LFS para los archivos de salida (EPUB, PDF) y las imágenes de alta resolución (>10MB). Usar ramas separadas para cada formato de salida y etiquetas semánticas (v1.0.0, v1.1.0) para marcar versiones estables. El sistema de compilación incremental de free_ebook_maker detecta automáticamente los cambios en los archivos fuente y solo recompila los capítulos modificados, reduciendo el tiempo de construcción en un 85% en proyectos con 20+ capítulos.
