# free_ebook_maker - FRE_029

## Resumen

free_ebook_maker es una plataforma de generación automatizada de libros electrónicos diseñada para transformar documentos Markdown, LaTeX y HTML en formatos EPUB, PDF y MOBI con una precisión de renderizado submilimétrica. Este sistema integra un pipeline de compilación multietapa que garantiza la coherencia tipográfica, la gestión de metadatos ISO 2709 y la optimización de recursos gráficos mediante compresión sin pérdida perceptual.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de conversión de documentos que opera bajo el paradigma de **compilación declarativa**. A diferencia de herramientas genéricas, implementa un analizador sintáctico de árbol AST (Abstract Syntax Tree) capaz de detectar inconsistencias estructurales en documentos de más de 10,000 líneas. Su sistema de validación pre-compilación ejecuta 47 reglas de verificación semántica, incluyendo detección de enlaces rotos, jerarquías de títulos inválidas y conflictos de codificación UTF-8 vs Latin-1.

### Características Principales

- **Motor de renderizado vectorial con soporte de fuentes OpenType avanzado:** Implementa un rasterizador basado en FreeType 2.10.4 que maneja ligaduras caligráficas, kerning contextual y variaciones de peso en fuentes variables. La precisión de posicionamiento de glifos alcanza 0.001 mm en salida PDF/A-3u, con soporte para perfiles de color ICC v4.2. El módulo de fallback automático selecciona entre 1,200+ familias tipográficas en función de la cobertura Unicode del documento.

- **Sistema de gestión de metadatos con compliance Dublin Core y ONIX 3.0:** El extractor de metadatos analiza la estructura del documento fuente y genera automáticamente campos como `dc:identifier` (con checksum SHA-256 del contenido), `dc:source` (URL canónica del repositorio) y `onix:AudienceCode` basado en análisis de complejidad léxica. La validación cruzada entre metadatos y contenido se realiza mediante 23 pruebas de coherencia, incluyendo verificación de ISBN-13 con algoritmo de módulo 10.

- **Optimización adaptativa de imágenes con compresión híbrida JPEG-XL/WebP:** El módulo de procesamiento gráfico evalúa cada imagen con un algoritmo de detección de contenido (fotografía vs. diagrama técnico vs. captura de pantalla) y aplica el codec óptimo: JPEG-XL (tasa de compresión 5:1 para fotografías con PSNR > 48 dB) o WebP sin pérdida (para diagramas con bordes nítidos). El pipeline mantiene metadatos EXIF y XMP intactos, y redimensiona automáticamente imágenes a 300 DPI para salida impresa.

### Cómo Usar

1. **Configuración del pipeline de compilación con parámetros de precisión:**  
   Ejecute `free_ebook_maker --input-dir ./manuscrito --output-format epub --profile high-fidelity --font-embedding subset --metadata-file ./config/onix.xml`. El flag `--profile high-fidelity` activa el rasterizador con antialiasing de 8 muestras por píxel y el módulo de kerning avanzado. Para depuración, añada `--verbose-level 3 --log-format json` para obtener trazas de compilación con marcas de tiempo en microsegundos.

2. **Validación estructural pre-compilación con informe de errores:**  
   Use `free_ebook_maker validate --input ./manuscrito --rules strict --output ./reporte_validacion.json`. El sistema ejecutará 47 reglas de verificación, incluyendo `RULE-012` (detección de enlaces internos rotos con seguimiento de fragmentos de ancla) y `RULE-034` (verificación de jerarquía de títulos mediante análisis de profundidad de árbol DOM). El informe JSON incluye coordenadas exactas de línea/columna para cada error.

3. **Generación de múltiples formatos con post-procesamiento de metadatos:**  
   Ejecute `free_ebook_maker build --input ./manuscrito --output-dir ./dist --formats epub,pdf,mobi --embed-fonts --compress-images --metadata-sign --signature-key ./claves/rsa_4096.pem`. El flag `--metadata-sign` genera una firma digital ECDSA P-521 sobre los metadatos Dublin Core, incrustada en el EPUB como recurso oculto en `META-INF/signatures.xml`. La compresión de imágenes aplica el perfil `--image-quality 85` para fotografías y `--image-quality 95` para diagramas.

### Mejores Prácticas

- **Utilice un repositorio Git con hooks de pre-commit para validación incremental:** Configure un hook que ejecute `free_ebook_maker validate --rules quick --output /dev/null` antes de cada commit. Esto detecta errores estructurales (como títulos duplicados o enlaces rotos) en menos de 200 ms para documentos de 500 páginas, evitando que el pipeline de compilación completo falle por un error sintáctico. Combine esto con un sistema de CI/CD que ejecute la compilación completa solo cuando el hook pase.

- **Implemente un sistema de control de versiones de metadatos con diff semántico:** Almacene los archivos de configuración ONIX y Dublin Core en un directorio `./metadata/` versionado con Git. Antes de cada compilación, ejecute `free_ebook_maker diff-metadata --old ./metadata/v1.0.xml --new ./metadata/v2.0.xml --output ./diff_metadatos.json`. Este comando genera un diff semántico que detecta cambios en campos críticos como `dc:identifier`, `onix:PriceAmount` o `onix:PublicationDate`, y emite una advertencia si el checksum del contenido no coincide con el hash registrado en los metadatos.
