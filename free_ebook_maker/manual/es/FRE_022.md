# free_ebook_maker - FRE_022

## Resumen

Herramienta de compilación y generación automatizada de eBooks en formato EPUB, MOBI y PDF, diseñada para flujos editoriales profesionales. Optimiza la conversión de Markdown estructurado a publicaciones digitales con control granular sobre metadatos, tipografía y empaquetado. Ideal para equipos de documentación técnica, editoriales independientes y desarrolladores que necesitan integrar generación de libros en pipelines CI/CD.

## Funcionalidades Técnicas Principales

### 1. Motor de Compilación Multi-formato con Control de Tipografía Avanzado
Compila documentos Markdown a EPUB, MOBI y PDF utilizando un pipeline de transformación basado en Pandoc y WeasyPrint. Permite definir estilos CSS personalizados, fuentes embebidas y reglas de hyphenation por idioma (es, en, fr). Incluye un validador de estructura de capítulos que verifica jerarquías `h1`-`h6` y genera automáticamente tablas de contenidos (TOC) con profundidad configurable (hasta 4 niveles).

**Parámetros clave:**
- `--format`: `epub`, `mobi`, `pdf` (soporte simultáneo con `--all`).
- `--css`: ruta a hoja de estilo personalizada (por defecto: `styles/default.epub.css`).
- `--hyphen-lang`: código ISO de idioma para separación silábica (ej: `es`, `en`).
- `--toc-depth`: profundidad máxima del TOC (1-4, por defecto 3).
- `--embed-fonts`: boolean, incrusta fuentes TrueType/OpenType en EPUB (tamaño máximo 5 MB por fuente).

### 2. Sistema de Metadatos y Gestión de Versionado con Integración a Git
Extrae y normaliza metadatos desde archivos `metadata.yaml` incrustados en el repositorio. Soporta campos Dublin Core (título, autor, idioma, identificador ISBN/DOI) y metadatos extendidos (colección, derechos de autor, fecha de publicación). Se integra con Git para etiquetar versiones de eBooks automáticamente: cada compilación genera un commit con el hash del contenido y la versión semántica (`v1.2.3`).

**Parámetros clave:**
- `--metadata`: ruta a archivo YAML (por defecto: `metadata.yaml`).
- `--isbn`: valida formato ISBN-10/ISBN-13 y lo inscribe en metadatos.
- `--git-tag`: boolean, crea etiqueta Git al finalizar compilación.
- `--version-from-git`: boolean, extrae versión del tag Git más reciente.

### 3. Generación de Índices Analíticos y Glosarios Automáticos
Analiza el contenido Markdown para extraer términos clave, nombres propios y referencias cruzadas. Genera un índice analítico (back-of-book index) con enlaces a páginas exactas en PDF o fragmentos de texto en EPUB. También construye glosarios a partir de definiciones en bloques de código `glossary` o archivos `glossary.yaml`. El motor de indexación es configurable para ignorar palabras vacías (stopwords) y aplicar stemming en español.

**Parámetros clave:**
- `--index`: boolean, activa generación de índice analítico.
- `--glossary`: ruta a archivo YAML de glosario (formato: `término: definición`).
- `--stopwords-lang`: idioma para filtro de palabras vacías (`es`, `en`, `fr`).
- `--index-stemmer`: boolean, aplica stemming (Snowball) para agrupar variantes morfológicas.

## Cómo Usar

### Paso 1: Configuración del Proyecto y Metadatos
Crea un archivo `metadata.yaml` en la raíz del proyecto con los campos obligatorios y opcionales. Ejemplo mínimo:

```yaml
title: "Guía Técnica de Python"
author: "Juan Pérez"
language: es
identifier:
  type: ISBN
  value: "978-3-16-148410-0"
rights: "Copyright 2025, Juan Pérez. Licencia MIT."
```

Luego, estructura tu contenido Markdown en la carpeta `src/` con un archivo `index.md` como punto de entrada que incluya los capítulos mediante referencias `#include`:

```markdown
#include "capitulos/introduccion.md"
#include "capitulos/instalacion.md"
#include "capitulos/uso-avanzado.md"
```

### Paso 2: Compilación con Parámetros Específicos
Ejecuta el comando de compilación especificando formato, estilos y opciones de indexación. Ejemplo para generar EPUB y PDF con índice analítico en español:

```bash
free_ebook_maker \
  --format epub,pdf \
  --css styles/mi-estilo.css \
  --hyphen-lang es \
  --toc-depth 4 \
  --index \
  --index-stemmer \
  --stopwords-lang es \
  --metadata metadata.yaml
```

El proceso validará la estructura de capítulos, generará el TOC, compilará los formatos y, si `--git-tag` está activo, creará un tag `v1.0.0` en el repositorio.

### Paso 3: Verificación y Post-procesamiento
Revisa los archivos generados en `output/`:
- `output/libro.epub`: validable con `epubcheck` (herramienta oficial de W3C).
- `output/libro.pdf`: verifica que el índice analítico contenga enlaces funcionales usando `pdfinfo` y `pdftotext`.
- `output/libro.mobi`: prueba en Kindle Previewer (solo si se generó).

Para validar metadatos automáticamente, ejecuta:

```bash
free_ebook_maker --validate --metadata metadata.yaml
```

Esto comprueba ISBN, formato de fechas y consistencia de campos Dublin Core.

## Mejores Prácticas

### 1. Integración Continua con Pipelines CI/CD
Configura la compilación de eBooks como un paso en GitHub Actions o GitLab CI. Cada push a `main` debe disparar una compilación completa (EPUB+PDF) y subir los artefactos a un release de GitHub. Utiliza el parámetro `--git-tag` para que el versionado semántico se sincronice automáticamente con los tags del repositorio. Esto asegura que cada versión del libro esté asociada a un commit específico y sea reproducible.

### 2. Estandarización de Estilos con CSS Modular
Define un sistema de estilos CSS base para EPUB y PDF, separando tipografía, colores y layout en módulos reutilizables. Por ejemplo:
- `base.epub.css`: márgenes, tamaños de fuente, interlineado.
- `code.epub.css`: bloques de código, resaltado de sintaxis.
- `tables.epub.css`: tablas responsivas para EPUB.

Mantén estos archivos bajo control de versiones y documenta las variables CSS (como `--font-size-body`) para que cualquier miembro del equipo pueda ajustar la apariencia sin modificar el contenido Markdown.
