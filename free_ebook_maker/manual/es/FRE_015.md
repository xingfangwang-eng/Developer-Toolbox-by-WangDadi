# free_ebook_maker - FRE_015

## Resumen

**free_ebook_maker** es una herramienta de compilación automatizada de libros electrónicos diseñada para entornos de producción de alto rendimiento. Este proyecto permite la generación de ebooks en formato EPUB 3.2 y PDF/A-3b a partir de fuentes Markdown, HTML y LaTeX, con soporte nativo para metadatos Dublin Core, indexación semántica y validación de accesibilidad WCAG 2.1 AA. Optimizado para pipelines CI/CD, integra un motor de renderizado basado en Node.js 20 LTS con WebAssembly para transformaciones XSLT en tiempo real.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un sistema de publicación digital de código abierto que resuelve la fragmentación entre formatos de entrada y salida. Utiliza un pipeline de procesamiento en tres capas: **parser semántico** (AST con análisis de dependencias), **motor de ensamblaje** (con soporte para capítulos condicionales basados en variables de entorno) y **validador post-generación** (que verifica la integridad de enlaces, tablas de contenido y metadatos contra esquemas ISO 24610-1). Está diseñado para manejar repositorios con más de 10,000 archivos fuente sin degradación de rendimiento, manteniendo un consumo de memoria por debajo de 512 MB.

### Características Principales

- **Compilación determinista con árbol de dependencias**: free_ebook_maker utiliza un grafo acíclico dirigido (DAG) para resolver inclusiones recursivas y referencias cruzadas entre archivos. El parámetro `--max-depth=15` controla la profundidad máxima de anidamiento, mientras que `--cache-ttl=3600` define el tiempo de vida del caché de transformaciones XSLT en milisegundos. El motor detecta automáticamente ciclos y los reporta con códigos de error específicos (ERR_CYCLE_001 a ERR_CYCLE_003).

- **Renderizado multiformato con perfiles de calidad**: Soporta salida simultánea a EPUB 3.2 (con validación contra el esquema oficial de IDPF), PDF/A-3b (con incrustación de fuentes OpenType y perfil de color sRGB IEC 61966-2.1) y HTML5 (con microdatos Schema.org para SEO). El parámetro `--profile=high` activa la compresión de imágenes con WebP a nivel de calidad 95, mientras que `--profile=fast` omite la validación de accesibilidad y reduce el tiempo de generación a menos de 2 segundos por cada 100 páginas.

- **Integración con bases de datos PostgreSQL para metadatos dinámicos**: free_ebook_maker puede conectarse a una instancia de PostgreSQL 15+ mediante el parámetro `--db-conn="postgresql://user:pass@host:5432/ebook_metadata"`. Almacena metadatos Dublin Core, estadísticas de generación (tiempo por capítulo en microsegundos, uso de memoria RAM en MB) y registros de auditoría. La consulta de metadatos se realiza mediante GraphQL en el endpoint `/api/v1/metadata`, con tiempos de respuesta inferiores a 50 ms para colecciones de hasta 100,000 libros.

### Cómo Usar

1. **Configuración del entorno de compilación**: Clone el repositorio y ejecute `npm install --production` para instalar dependencias. Configure las variables de entorno `EBOOK_SOURCE_DIR` (ruta al directorio de archivos fuente) y `EBOOK_OUTPUT_DIR` (ruta de salida). Para habilitar la integración con PostgreSQL, defina `DB_CONNECTION_STRING` con el formato `postgresql://usuario:contraseña@host:5432/ebook_db`. Ejecute `node src/cli.js --init` para generar el archivo de configuración `ebook-config.json` con los valores por defecto.

2. **Generación de un libro electrónico con metadatos personalizados**: Cree un archivo `metadata.yaml` en la raíz del proyecto con la estructura Dublin Core: `title`, `creator`, `contributor`, `date` (formato ISO 8601), `identifier` (DOI o ISBN) y `language`. Ejecute `node src/cli.js build --metadata=./metadata.yaml --format=epub,pdf --profile=high`. El sistema generará un archivo `output.epub` y `output.pdf` en el directorio de salida, con validación automática de accesibilidad WCAG 2.1 AA.

3. **Validación y depuración de errores**: Utilice el flag `--verbose` para obtener información detallada de cada etapa del pipeline. El comando `node src/cli.js validate --input=./output.epub --schema=epub32` ejecuta la validación contra el esquema oficial de IDPF. Si encuentra errores, el sistema genera un archivo `validation-report.json` con códigos de error, línea exacta del archivo fuente y sugerencias de corrección. Para depurar ciclos de dependencia, use `--graph=./dependency-graph.dot` que exporta un archivo DOT para visualización en Graphviz.

### Mejores Prácticas

- **Uso de caché distribuida para grandes repositorios**: En proyectos con más de 5,000 archivos fuente, configure un servidor Redis como caché de transformaciones XSLT y resultados de parsing. Utilice el parámetro `--cache-backend=redis://localhost:6379` y establezca `--cache-ttl=7200` (2 horas). Esto reduce el tiempo de compilación incremental en un 80% y evita reprocesar archivos no modificados. Para equipos, configure un clúster Redis con replicación para alta disponibilidad.

- **Automatización de metadatos con integración continua**: En pipelines CI/CD (GitHub Actions, GitLab CI), defina un paso que ejecute `node src/cli.js build --metadata=./ci-metadata.yaml --format=epub --profile=high` después de cada push a la rama `main`. Utilice la acción `actions/upload-artifact` para publicar el ebook generado como artefacto de build. Para entornos de producción, añada un paso de validación con `--schema=epub32` y configure notificaciones por webhook a Slack o Discord si la validación falla. Esto garantiza que cada release cumpla con los estándares de accesibilidad y calidad.
