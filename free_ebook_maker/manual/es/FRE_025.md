# free_ebook_maker - FRE_025

## Resumen

Herramienta de compilación y conversión automatizada de libros electrónicos optimizada para flujos de trabajo editoriales de alto rendimiento. Diseñada para transformar documentos Markdown, LaTeX y HTML en formatos EPUB3, MOBI y PDF/A-2b con validación estructural W3C, soporte de metadatos Dublin Core y generación de tablas de contenidos jerárquicas. Ideal para equipos editoriales, auto-publicadores y desarrolladores que necesitan una pipeline libre de dependencias propietarias.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de compilación de libros electrónicos que opera sobre un pipeline basado en Node.js (v18+) y Pandoc 3.1.9. Procesa archivos fuente en Markdown con extensiones personalizadas (fenced_divs, raw_attribute) y produce salidas con cumplimiento estricto de los estándares EPUB 3.2 y PDF/A-2b. Incluye un sistema de detección de conflictos CSS y un validador de enlaces internos que previene errores 404 en la navegación del libro final. Soporta integración con motores de búsqueda locales mediante índices invertidos generados en memoria.

### Características Principales

- **Conversión multiformato con validación W3C**: El motor `epub-builder` transforma documentos Markdown a EPUB3 usando `pandoc` con filtros Lua personalizados que inyectan metadatos Dublin Core (dc:title, dc:creator, dc:identifier). El proceso incluye una etapa de validación automática contra el esquema EPUB 3.2 mediante `epubcheck 5.1.0`, reportando errores de estructura (orden de archivos, duplicación de IDs) y advirtiendo sobre imágenes sin atributo `alt`. Tiempo medio de compilación: 2.3 segundos para un libro de 300 páginas en un procesador AMD Ryzen 7 5800X.

- **Generación de PDF/A-2b con control de tipografía**: El submódulo `pdf-engine` utiliza `WeasyPrint 62.3` para renderizar HTML+CSS a PDF de archivo digital de larga duración. Permite configurar márgenes (top: 20mm, bottom: 25mm, inner: 25mm, outer: 18mm), tamaño de página (A5, B5, US Letter) y fuentes incrustadas (License-Free: Noto Serif, IBM Plex Serif). Incluye un preprocesador CSS que elimina reglas no compatibles con PDF (sombras, transiciones) y añade números de página automáticos mediante contadores CSS. Precisión de posicionamiento: ±0.1mm en sangrías.

- **Optimización de metadatos y SEO editorial**: El módulo `meta-injector` analiza el frontmatter YAML de cada archivo fuente y completa automáticamente campos faltantes usando una base de datos SQLite local (almacenada en `~/.free_ebook_maker/metadata.db`). Por ejemplo, si falta `dc:language`, infiere el idioma mediante `cld3` con una precisión del 97.2% en textos de más de 100 caracteres. También genera automáticamente una descripción SEO de 160 caracteres extrayendo las primeras 2 oraciones del capítulo inicial, y añade palabras clave extraídas con TF-IDF (top 5 términos por capítulo, frecuencia mínima 3).

### Cómo Usar

1. **Preparación del entorno y configuración del pipeline**:
   - Instale Node.js v18+ y Pandoc 3.1.9. Ejecute `npm install -g free-ebook-maker` para instalar el CLI.
   - Cree un directorio de proyecto y coloque sus archivos Markdown en `src/`. Cada archivo debe incluir frontmatter YAML con al menos `title`, `author` y `language`. Ejemplo:
     ```yaml
     ---
     title: "Guía Avanzada de DevOps"
     author: "Carlos Martínez"
     language: es
     ---
     ```
   - Ejecute `free-ebook-maker init` para generar el archivo de configuración `ebook-config.json`. Configure las rutas de salida y los formatos deseados:
     ```json
     {
       "output": "./dist",
       "formats": ["epub", "pdf"],
       "pdf": {
         "page_size": "A5",
         "margins": { "top": 20, "bottom": 25, "inner": 25, "outer": 18 }
       },
       "epub": {
         "validate": true,
         "cover_image": "./assets/cover.png"
       }
     }
     ```

2. **Compilación y validación estructural**:
   - Ejecute `free-ebook-maker build --format epub --validate`. El sistema ejecutará Pandoc con filtros Lua personalizados que inyectan metadatos Dublin Core y generan la tabla de contenidos basada en los encabezados `##` y `###`.
   - Durante la compilación, el motor `epub-builder` genera un archivo ZIP temporal y lo valida con `epubcheck`. Si hay errores (por ejemplo, un enlace interno a `#capitulo-5` que no existe), el proceso se detiene y muestra la línea exacta del error. Ejemplo de salida:
     ```
     [VALIDATE] EPUB 3.2 validation started...
     [ERROR]  line 342: fragment identifier 'capitulo-5' not found in 'chapter-4.xhtml'
     [FIX]    Sugerencia: revise el enlace en 'src/capitulo-4.md:45'
     ```
   - Si la validación es exitosa, el archivo EPUB se guarda en `./dist/mi-libro.epub`. Para generar PDF, ejecute `free-ebook-maker build --format pdf`. El motor `pdf-engine` renderizará el HTML con WeasyPrint, aplicando márgenes y fuentes configuradas. El proceso tarda aproximadamente 3.1 segundos para 200 páginas con imágenes.

3. **Inyección de metadatos y optimización SEO**:
   - Después de la compilación, ejecute `free-ebook-maker meta --inject --seo`. El módulo `meta-injector` analiza el frontmatter de cada archivo fuente y completa campos faltantes usando la base de datos SQLite local.
   - Si falta `dc:description`, el sistema genera automáticamente una descripción de 160 caracteres extrayendo las primeras 2 oraciones del capítulo inicial. Ejemplo de salida:
     ```
     [META] Injecting dc:description for 'chapter-1.md'...
     [META] Generated: "En este capítulo exploraremos las técnicas avanzadas de integración continua..."
     [META] dc:language inferred as 'es' (confidence: 97.2%)
     ```
   - Para verificar los metadatos finales, ejecute `free-ebook-maker meta --show --format epub`. Esto muestra el contenido completo de `content.opf` (para EPUB) o los metadatos XMP incrustados (para PDF). Si desea exportar los metadatos a un archivo JSON, use `free-ebook-maker meta --export ./metadata.json`.

### Mejores Prácticas

- **Estructura de archivos modular con validación incremental**: Divida su libro en capítulos individuales (un archivo Markdown por capítulo) y use `ebook-config.json` para definir el orden. Esto permite compilar solo los capítulos modificados mediante `free-ebook-maker build --incremental`. Para proyectos grandes (>50 capítulos), configure un hook de pre-commit en Git que ejecute `free-ebook-maker build --validate-only` para detectar errores de enlaces internos antes de cada commit. Esto reduce el tiempo de compilación completa en un 70% y previene la generación de EPUBs corruptos.

- **Gestión de activos y fuentes para PDF de archivo**: Al generar PDF/A-2b, asegúrese de que todas las imágenes estén en formato PNG o JPEG (no SVG) y tengan una resolución mínima de 300 DPI. Use el comando `free-ebook-maker assets --check --min-dpi 300` para verificar automáticamente. Para las fuentes, evite tipografías web (Google Fonts) y use solo fuentes incrustadas sin licencia restrictiva (Noto Serif, IBM Plex Serif).
