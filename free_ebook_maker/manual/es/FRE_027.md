# free_ebook_maker - FRE_027

## Resumen

free_ebook_maker es un generador automatizado de libros electrónicos en formato EPUB/MOBI/PDF que integra procesamiento por lotes de Markdown, compilación multilingüe con detección de idioma por bloques, y optimización de metadatos para indexación SEO en motores de búsqueda hispanos. Soporta conversión directa desde archivos .md, .tex y .docx con salida validada contra el estándar EPUB 3.2.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es una herramienta de compilación de libros digitales que opera en modo CLI y API REST, diseñada para autores técnicos y editoriales independientes. Su núcleo implementa un pipeline de 4 etapas: preprocesamiento de contenido (normalización de UTF-8, eliminación de BOM), generación de estructura jerárquica (capítulos, secciones, subsecciones) basada en encabezados Markdown, empaquetado en contenedores EPUB con verificación de integridad SHA-256, y publicación directa a repositorios Git o almacenamiento en la nube (S3, MinIO). El motor de renderizado interno utiliza WeasyPrint para PDF y `epubcheck` para validación de conformidad.

### Características Principales

- **Compilación por lotes con control de versiones semántico**: Acepta un directorio de entrada con múltiples archivos Markdown, detecta automáticamente el orden mediante un archivo `_toc.yml` o por convención de nombres (ej. `01-intro.md`). Genera un libro con numeración de páginas, tabla de contenidos interactiva y enlaces cruzados entre capítulos. El sistema asigna un número de versión semántica (MAJOR.MINOR.PATCH) basado en el hash de contenido del árbol de archivos, permitiendo diferenciar builds idénticos de cambios reales. Parámetros clave: `--input-dir`, `--output-format` (epub, mobi, pdf), `--version-override`.

- **Optimización de metadatos para SEO multilingüe**: Inyecta etiquetas Dublin Core (dc:title, dc:creator, dc:language, dc:subject) y metadatos Open Graph (og:title, og:description, og:locale) directamente en el manifiesto EPUB. Soporta detección automática de idioma por párrafo usando `langdetect` con precisión >95% en español, catalán, gallego y euskera. Para cada bloque de texto, asigna el atributo `xml:lang` correcto, permitiendo que lectores como Kindle o Apple Books apliquen correctamente la hipenación y el diccionario de sinónimos. Parámetros: `--meta-author`, `--meta-title`, `--meta-lang` (por defecto 'es').

- **Renderizado de fórmulas matemáticas y diagramas Mermaid**: Integra MathJax 3 para renderizar LaTeX en línea y en bloque, con salida SVG optimizada para EPUB. Para diagramas, ejecuta Mermaid CLI localmente, convirtiendo bloques ```mermaid en imágenes PNG incrustadas con resolución de 300 DPI y fondo transparente. El pipeline detecta automáticamente si el contenido contiene `$$` o `\begin{equation}` y activa el procesador matemático solo cuando es necesario, reduciendo el tiempo de compilación en un 40% respecto a soluciones que cargan MathJax globalmente. Parámetros: `--math-engine` (mathjax, katex, none), `--mermaid-theme` (default, forest, dark).

### Cómo Usar

1. **Preparación del entorno y configuración del pipeline**:
   - Clone el repositorio y ejecute `pip install -r requirements.txt` (dependencias: `click`, `weasyprint`, `epubcheck`, `mermaid-cli`, `langdetect`, `Pillow`).
   - Cree un directorio de proyecto con la estructura: `mi-libro/`, dentro coloque `_toc.yml` con la lista de archivos Markdown en orden. Ejemplo de `_toc.yml`:
     ```yaml
     - prefacio.md
     - 01-introduccion.md
     - 02-fundamentos.md
     - 03-avanzado.md
     - apendice.md
     ```
   - Ejecute el comando base: `python maker.py --input-dir ./mi-libro --output-format epub --meta-title "Guía Técnica de Python" --meta-author "Juan Pérez"`. El sistema generará `./output/mi-libro.epub` con metadatos completos y tabla de contenidos.

2. **Compilación avanzada con optimización de imágenes y diagramas**:
   - Añada imágenes en formato PNG/JPG/WebP dentro de `mi-libro/assets/`. El pipeline las comprimirá automáticamente con `Pillow` (calidad 85%, modo RGB) y las redimensionará a un ancho máximo de 1200px para evitar archivos EPUB inflados.
   - Para diagramas Mermaid, inserte bloques de código con lenguaje `mermaid` en sus archivos Markdown. Ejemplo:
     ```mermaid
     graph TD
       A[Inicio] --> B{¿Válido?}
       B -->|Sí| C[Procesar]
       B -->|No| D[Error]
     ```
   - Ejecute: `python maker.py --input-dir ./mi-libro --mermaid-theme forest --math-engine katex --compress-images`. El sistema generará diagramas en 300 DPI y fórmulas con KaTeX (más rápido que MathJax para EPUB). La compilación tardará ~8 segundos para 50 páginas con 10 diagramas.

3. **Publicación directa a repositorio Git y validación de conformidad**:
   - Para integrar con control de versiones, ejecute: `python maker.py --input-dir ./mi-libro --output-format epub --git-push --git-remote origin --git-branch main`. El comando hará commit automático con mensaje "Build EPUB vX.Y.Z" y subirá al remoto.
   - La validación de conformidad se ejecuta automáticamente con `epubcheck` tras la generación. Si hay errores (ej. enlaces rotos, imágenes faltantes), el proceso falla con código de salida 1 y muestra el log de errores en `./output/validation.log`.
   - Para verificar manualmente: `python maker.py --input-dir ./mi-libro --validate-only`. Esto ejecuta `epubcheck` sin generar el EPUB, útil para depuración rápida.

### Mejores Prácticas

- **Estructura modular con archivos pequeños**: Divida su libro en archivos Markdown de no más de 500 líneas cada uno. Esto mejora el rendimiento del preprocesador (evita bloqueos de memoria con archivos >10MB) y facilita la revisión por pares en Git. Use `_toc.yml` para definir el orden exacto, y nombre los archivos con prefijos numéricos (ej. `04-capitulo.md`) para que el sistema pueda ordenarlos incluso si falta el archivo de tabla de contenidos.

- **Uso de metadatos localizados para distribución en Amazon KDP**: Para maximizar la visibilidad en tiendas hispanas, configure `--meta-lang es --meta-subject "Informática, Programación, Python"`. Además, incluya una descripción de 150-160 caracteres en el campo `dc:description` del archivo `metadata.yml` dentro del directorio de entrada. El pipeline inyectará estos metadatos en el OPF y en el NCX, mejorando el ranking en búsquedas de "libros técnicos en español" y "guías de programación". Evite usar caracteres no ASCII en los nombres de archivo; el sistema normaliza automáticamente a NFC, pero algunos lectores pueden tener problemas con caracteres como "ñ" en rutas.
