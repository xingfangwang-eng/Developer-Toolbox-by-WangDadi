# free_ebook_maker - FRE_006

## Resumen

Herramienta de compilación y generación automatizada de libros electrónicos en formato EPUB 3.2 y PDF/A-2b, diseñada para entornos de producción editorial con control granular sobre metadatos, tipografía y accesibilidad. Utiliza un pipeline basado en Node.js 18+ y Markdown estricto con extensión **fEML** (free_ebook_maker Language).

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de composición digital que transforma documentos estructurados en Markdown (con notación fEML) en libros electrónicos listos para distribución. A diferencia de herramientas genéricas, implementa un sistema de **herencia de estilos por capítulos**, validación de enlaces internos y compresión diferencial de imágenes (WebP con fallback a PNG). Soporta integración directa con almacenes S3-compatible y servicios de metadata como ISBNdb.

### Características Principales

- **Compilación con linting estructural fEML**: Analiza el Markdown de entrada en busca de errores de anidamiento de títulos, referencias cruzadas rotas y metadatos obligatorios ausentes (como `dc:creator` o `dc:identifier`). Emite un informe en JSON con precisión de línea y columna, permitiendo correcciones antes de la generación final. Tiempo de análisis: <150 ms para documentos de 500 páginas.

- **Renderizado tipográfico con sangría variable**: Aplica un algoritmo de hyphenation basado en el estándar **Hunspell** con diccionarios para 12 idiomas. Permite definir reglas de widow/orphan control (líneas huérfanas máximas: 2) y kerning automático en títulos de nivel 1 y 2. El motor de renderizado usa **Paged.js 0.4.2** con parches de rendimiento para tablas extensas (hasta 50 columnas sin degradación de FPS).

- **Empaquetado con firma digital y checksum**: Genera archivos EPUB firmados con una clave privada RSA-2048 (opcional). Incluye un manifiesto SHA-256 de cada recurso incrustado (imágenes, fuentes, CSS). Compatible con la especificación **EPUB 3.2 Accessibility** (puntuación WCAG 2.1 AA garantizada en el 98% de los casos). Tamaño máximo de salida: 2 GB sin fragmentación.

### Cómo Usar

1. **Preparación del proyecto y linting inicial**  
   Crea un directorio con la estructura requerida:  
   ```
   proyecto/
   ├── src/
   │   ├── chapters/        # Archivos .md con fEML
   │   ├── assets/          # Imágenes (JPEG, PNG, WebP)
   │   └── fonts/           # Fuentes OTF/TTF (opcional)
   ├── config.yaml          # Parámetros de compilación
   └── metadata.yaml        # Dublin Core + identificador ISBN
   ```  
   Ejecuta: `free_ebook_maker lint --strict` para validar la estructura. El comando devuelve un código de salida 0 solo si no hay errores de tipo `E001` (etiqueta dc faltante) o `E007` (referencia cruzada a capítulo inexistente).

2. **Compilación con perfiles de salida**  
   Usa `free_ebook_maker build --profile epub3 --output ./dist/libro.epub`. El perfil `epub3` activa la compresión ZIP con nivel 9, inclusión automática de `nav.xhtml` y generación de `package.opf` con `spine` ordenado alfabéticamente por defecto. Para PDF/A-2b, añade `--profile pdfa2b --pdf-engine weasyprint 60.0`. Tiempo estimado para 300 páginas con 10 imágenes: 8.2 segundos en CPU AMD Ryzen 7.

3. **Post-procesado y verificación de accesibilidad**  
   Tras la compilación, ejecuta: `free_ebook_maker verify --accessibility --output report.json`. Este comando analiza la estructura de encabezados (jerarquía h1–h6), contraste de color mínimo (relación 4.5:1 para texto normal) y etiquetas `alt` en todas las imágenes. Genera un puntaje de 0 a 100; el umbral mínimo recomendado es 92. Si falla, el informe detalla cada violación con la línea exacta del Markdown fuente.

### Mejores Prácticas

- **Utiliza el modo de compilación diferencial para iteraciones rápidas**  
  En lugar de recompilar todo el libro tras cada cambio, activa `--watch` junto con `--incremental`. free_ebook_maker mantiene un caché de los capítulos ya procesados (hash MD5 de cada archivo .md) y solo recompila los modificados. Esto reduce el tiempo de actualización a menos de 200 ms para cambios en un solo capítulo, incluso en proyectos con 50+ archivos. Acompaña esto con un hook de Git pre-commit que ejecute `lint --strict` para evitar errores acumulativos.

- **Centraliza la gestión de metadatos en un archivo YAML externo**  
  En lugar de esparcir `dc:title` o `dc:language` en cada capítulo, define todos los metadatos en `metadata.yaml` y usa referencias fEML como `{{metadata.title}}` dentro del Markdown. Esto permite cambiar el título, autor o ISBN sin modificar el contenido, y facilita la generación de múltiples variantes (por ejemplo, una edición en español y otra en catalán) con un solo comando: `free_ebook_maker build --metadata ./metadata_es.yaml --output ./dist/libro_es.epub`.
