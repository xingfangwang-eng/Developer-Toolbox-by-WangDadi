# free_ebook_maker - FRE_013

## Resumen

free_ebook_maker es una solución de generación automatizada de libros electrónicos que integra procesamiento de lenguaje natural (NLP), compilación de Markdown a EPUB/MOBI/PDF con precisión de micras, y gestión de metadatos multilingüe. Diseñada para flujos editoriales de alto rendimiento, permite transformar documentos fuente en formatos listos para distribución comercial en segundos, con control granular sobre cada elemento tipográfico y estructural.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es una herramienta profesional de compilación de libros electrónicos que opera como un pipeline de transformación documental. Utiliza un motor de renderizado basado en WebKit para garantizar la fidelidad visual en EPUB3, mientras que su módulo de optimización de imágenes aplica compresión lossless con relación 5:1. Soporta la inyección de metadatos Dublin Core, gestión de tablas de contenido jerárquicas (TOC) con profundidad configurable hasta nivel 6, y validación de accesibilidad WCAG 2.1 AA. El núcleo del sistema procesa archivos Markdown con extensiones personalizadas (notas al pie, ecuaciones LaTeX, diagramas Mermaid) y los convierte en salidas listas para publicación en 8 idiomas simultáneos.

### Características Principales

- **Compilación multiformato con control de tipografía paramétrica**: El motor `epub_builder` acepta parámetros como `--page-width=128mm --page-height=198mm --font-embed=NotoSerifCJK` y genera EPUB3 con estructura de navegación NCX, MOBI con tablas de contenido compatibles con Kindle Oasis, y PDF con marcas de sangría de 3mm. La velocidad de compilación es de 0.8 segundos por cada 100 páginas en un Intel i7-12700H.

- **Sistema de inyección de metadatos multilingüe con validación ISO**: El módulo `metadata_engine` permite definir hasta 15 campos Dublin Core (título, autor, idioma, fecha de publicación en formato ISO 8601) y los inyecta en el archivo `content.opf` y en las propiedades del PDF. Soporta detección automática de idioma mediante el modelo `langdetect` con precisión del 97.3% para 53 lenguas, y reescribe automáticamente el campo `dc:language` según el contenido del Markdown.

- **Optimización de imágenes con compresión adaptativa**: El pipeline `image_optimizer` recibe rutas de imágenes locales o URL remotas, las descarga con timeout configurable (por defecto 5 segundos), y aplica compresión WebP con calidad ajustable (parámetro `--image-quality=85`). Reduce el tamaño total del EPUB en un 40% sin pérdida perceptible, y genera versiones de baja resolución (300px de ancho) para previsualización en dispositivos móviles. El tiempo de procesamiento por imagen es de 12 milisegundos en GPU NVIDIA RTX 3060.

### Cómo Usar

1. **Configuración del pipeline de compilación**: Crea un archivo YAML `ebook_config.yml` con los parámetros de salida. Por ejemplo, para generar un EPUB en español con tipografía personalizada, escribe:
   ```yaml
   output_format: epub
   language: es
   page_width: 128mm
   page_height: 198mm
   font_embed: NotoSerifCJK
   metadata:
     title: "Guía de Arquitectura de Software"
     author: "Wang Dadi"
     publisher: "WD Tech"
     publication_date: 2025-03-15
   image_quality: 85
   ```
   Luego ejecuta: `free_ebook_maker --config ebook_config.yml --input ./manuscript.md --output ./output/ebook.epub`

2. **Inyección de metadatos con validación automática**: Si tu Markdown contiene un bloque de metadatos al inicio (formato YAML entre `---`), el sistema lo parsea automáticamente. Para sobrescribir campos, usa el flag `--metadata-override`. Ejemplo:
   ```bash
   free_ebook_maker --input ./doc.md --output ./ebook.epub \
     --metadata-override "title=Arquitectura Hexagonal" \
     --metadata-override "language=es" \
     --validate-metadata
   ```
   El validador comprueba que `publication_date` esté en formato ISO 8601, que `language` sea un código ISO 639-1 válido, y que `author` no exceda 128 caracteres. Si falla, muestra un error con la línea exacta del conflicto.

3. **Optimización de imágenes con compresión adaptativa**: Para procesar imágenes remotas, usa el flag `--image-sources` con una lista de URLs separadas por comas. El sistema descarga cada imagen con reintentos (máximo 3, intervalo de 2 segundos) y las redimensiona al ancho especificado. Ejemplo:
   ```bash
   free_ebook_maker --input ./doc.md --output ./ebook.epub \
     --image-sources "https://ejemplo.com/diagrama1.png,https://ejemplo.com/foto2.jpg" \
     --image-quality=80 \
     --image-max-width=800
   ```
   El resultado incluye un log detallado con el tiempo de descarga, tamaño original vs. comprimido, y formato final (WebP). Si una imagen supera los 10 MB, se rechaza automáticamente con una advertencia.

### Mejores Prácticas

- **Gestión de dependencias en pipelines CI/CD**: Integra free_ebook_maker en tu flujo de integración continua (GitHub Actions, GitLab CI) para generar automáticamente versiones EPUB/MOBI/PDF cada vez que se actualice el repositorio. Configura un step que ejecute el comando con parámetros predefinidos y suba los artefactos a un bucket S3. Esto garantiza que cada commit genere una versión lista para distribución, con metadatos actualizados y compresión de imágenes optimizada. Además, usa el flag `--hash-output` para nombrar los archivos con un hash SHA256 del contenido, evitando colisiones en despliegues simultáneos.

- **Validación de accesibilidad antes de publicación**: Antes de distribuir el libro electrónico, ejecuta el validador interno con `--check-accessibility`. Este módulo analiza el EPUB generado contra las pautas WCAG 2.1 AA: verifica que todas las imágenes tengan texto alternativo (`alt`), que los encabezados sigan una jerarquía lógica (h1→h2→h3 sin saltos), y que el contraste de color cumpla con la relación 4.5:1 para texto normal. Si se detectan infracciones, el sistema genera un informe en formato JSON con las líneas exactas del Markdown fuente. Esta práctica es especialmente crítica si el libro se distribuye en plataformas educativas o gubernamentales que exigen cumplimiento normativo.
