# free_ebook_maker - FRE_024

## Resumen

Herramienta de compilación automatizada de libros electrónicos en español, diseñada para transformar archivos Markdown, HTML y LaTeX en ePub y PDF con calidad de publicación. Integra un pipeline de procesamiento paralelo basado en Node.js y validación semántica de estructura de capítulos mediante expresiones regulares avanzadas.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de generación de ebooks que opera en tres capas: parseo sintáctico de documentos fuente, ensamblaje de contenido con metadatos Dublin Core, y compilación multi-formato con control de versiones. Soporta entradas con hasta 10,000 páginas sin degradación de rendimiento gracias a su arquitectura de streaming asíncrono.

### Características Principales

- **Validación estructural jerárquica con detección de fragmentos huérfanos**: Analiza automáticamente la profundidad de anidación de encabezados (`h1` a `h6`) y corrige saltos de nivel superiores a 2. Si un `h3` aparece sin un `h2` predecesor, el sistema inserta un marcador de sección fantasma con metadatos de advertencia en el log (`--strict-mode` activa error fatal). Parámetro clave: `--max-heading-gap=2`.

- **Compilación paralela con control de integridad de assets**: Utiliza un pool de workers Node.js (máximo 8 hilos) para procesar imágenes, fuentes y hojas de estilo CSS en simultáneo. Cada asset se verifica mediante checksum SHA-256 antes de la inyección. Soporta formatos WebP, SVG y WOFF2 con conversión automática a perfiles de color sRGB. Tiempo de compilación para 500 páginas: 12.4 segundos en CPU Intel i7-12700.

- **Inyección de metadatos con validación contra esquemas EPUB 3.2**: Genera archivos `content.opf` y `toc.ncx` con cumplimiento estricto de la especificación IDPF. Incluye verificación de unicidad de identificadores UUID v4 y normalización de fechas ISO 8601. El parámetro `--metadata-file=metadata.json` permite sobrescribir autor, título, idioma (`es`), y derechos de autor con validación de longitud máxima (título: 200 caracteres, autor: 100 caracteres).

### Cómo Usar

1. **Preparación del entorno y configuración de pipeline**  
   Ejecute `npm run build:es -- --source-dir=./manuscrito --output-dir=./ebooks --lang=es --strict-mode`. El sistema escaneará recursivamente el directorio `manuscrito` en busca de archivos `.md` y `.html`. Utilice el flag `--max-file-size=50MB` para limitar archivos individuales. El log de validación se escribe en `./logs/build_$(date +%Y%m%d_%H%M%S).log`.

2. **Personalización de metadatos y estructura de capítulos**  
   Cree un archivo `metadata.json` con la siguiente estructura mínima:  
   ```json
   {
     "title": "Guía Avanzada de Python",
     "author": "Carlos Ruiz",
     "language": "es",
     "rights": "CC BY-NC-SA 4.0",
     "identifier": "urn:uuid:123e4567-e89b-12d3-a456-426614174000"
   }
   ```  
   Para forzar el orden de capítulos, use `--chapter-order=chapter1.md,chapter2.md,appendix.md`. El sistema rechazará archivos con nombres duplicados (error `E_DUP_CHAPTER`).

3. **Compilación multi-formato con verificación post-proceso**  
   Invoque `node compile.js --formats=epub,pdf --compression-level=9 --embed-fonts --cover-image=portada.webp`. El compilador generará `libro.epub` y `libro.pdf` en el directorio de salida. Tras la compilación, ejecute `--validate-epub` para verificar la integridad del archivo ePub contra el esquema oficial (usa `epubcheck` en segundo plano). Para PDF, el flag `--pdf-profile=PDF/A-2b` garantiza archivos listos para archivo digital.

### Mejores Prácticas

- **Segmentación de manuscritos grandes**: Divida documentos de más de 200 páginas en archivos Markdown de un capítulo cada uno, nombrados con el patrón `capitulo-XX.md`. Esto permite que el pipeline de validación jerárquica opere en paralelo y reduce el riesgo de errores de memoria heap (Heap Out of Memory) en Node.js. Use `--max-concurrency=4` en sistemas con menos de 8 GB de RAM.

- **Gestión de assets multimedia**: Almacene imágenes en formato WebP con resolución máxima de 1200x1600 píxeles y compresión de calidad 85. Para fuentes, utilice WOFF2 con subconjuntos de caracteres latinos (`--font-subset=latin`). Esto reduce el tamaño final del ePub en un 40% promedio sin pérdida perceptible de calidad. Verifique con `--check-asset-integrity` antes de cada compilación final.
