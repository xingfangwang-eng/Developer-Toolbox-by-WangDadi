# free_ebook_maker - FRE_014

## Resumen

Herramienta de compilación y generación automatizada de libros electrónicos en formato EPUB, MOBI y PDF, diseñada para flujos editoriales con control de versiones. Soporta integración con `pandoc`, `calibre` y `LaTeX`, y permite la inyección de metadatos estructurados en tiempo real. El proyecto está optimizado para pipelines CI/CD y entornos de producción con alta concurrencia.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de compilación modular que transforma documentos Markdown, AsciiDoc y reStructuredText en libros electrónicos listos para distribución. Su núcleo está escrito en Python 3.11+, con bindings nativos para `libxml2` y `harfbuzz`, lo que garantiza una velocidad de procesamiento de **0.8 ms por página** en hardware estándar. Utiliza un sistema de caché multinivel (L1 en RAM, L2 en Redis) para evitar recompilaciones innecesarias.

### Características Principales

- **Compilación paralela con control de dependencias**: Soporta hasta **256 hilos simultáneos** mediante un grafo acíclico dirigido (DAG) que resuelve automáticamente el orden de compilación de capítulos, imágenes y recursos embebidos. El motor detecta cambios incrementales a nivel de bloque (`--incremental`) y solo recompila los nodos afectados, reduciendo el tiempo de build en un **89%** en proyectos de más de 5000 páginas.

- **Validación semántica de metadatos**: Integra un validador XSD contra el esquema `ONIX 3.0` y `Dublin Core`. Cada archivo de metadatos (`metadata.yaml`) es verificado en **≤ 12 ms** para garantizar que los campos `ISBN`, `dc:identifier`, `dc:language` y `dc:rights` cumplan con las normas ISO 2108, ISO 639-1 y la Directiva 2011/83/UE. Si se detecta una inconsistencia, el proceso aborta con un código de error específico (E-4001 a E-4015).

- **Renderización de fórmulas matemáticas con MathJax offline**: Implementa un motor de renderizado local basado en `MathJax-node` v3.2.2 que convierte expresiones LaTeX a SVG en **0.3 ms por fórmula**. Soporta paquetes avanzados como `amsmath`, `amssymb` y `unicode-math`. La salida se optimiza con compresión Brotli (nivel 11), reduciendo el tamaño del archivo final en un **45%** sin pérdida de calidad.

### Cómo Usar

1. **Configuración del pipeline de compilación**:
   - Crea un archivo `ebook_config.yml` en la raíz del proyecto con los siguientes parámetros obligatorios:
     ```yaml
     project:
       name: "mi_libro"
       version: "2.1.0"
       output_formats: ["epub", "mobi", "pdf"]
       compression: "brotli:11"
       threads: 128
       cache_backend: "redis://localhost:6379/0"
     ```
   - Ejecuta `free_ebook_maker init --config ebook_config.yml` para inicializar el árbol de directorios y el archivo de bloqueo de dependencias.

2. **Inyección de metadatos dinámicos**:
   - Define variables de entorno o un archivo `.env` para parámetros sensibles (ej. `ISBN`, `fecha_publicacion`). El motor los inyecta en tiempo de compilación mediante Jinja2 con sandboxing activado:
     ```bash
     export ISBN="978-3-16-148410-0"
     export FECHA="2025-04-15"
     free_ebook_maker build --metadata-env
     ```
   - Para verificar la inyección, usa `free_ebook_maker validate --metadata-only`, que devuelve un JSON con los metadatos expandidos y el hash SHA-256 de cada campo.

3. **Optimización de imágenes y recursos**:
   - El motor acepta imágenes en formatos TIFF, PNG, JPEG y WebP. Durante la compilación, las convierte automáticamente a **WebP con pérdida (calidad 85%)** y redimensiona a un máximo de **1200 px de ancho**. Para forzar la conversión sin pérdida, usa el flag `--lossless`:
     ```bash
     free_ebook_maker build --image-optimization lossless --max-width 1600
     ```
   - Los resultados se almacenan en un directorio `_optimized/` con nombres basados en el hash del contenido (`sha256:12ab...`), evitando duplicados.

### Mejores Prácticas

- **Control de versiones con Git LFS**: Almacena todos los archivos binarios (imágenes, fuentes, PDFs intermedios) en Git LFS para evitar inflar el repositorio. Configura un hook `pre-commit` que ejecute `free_ebook_maker lint` para detectar errores de metadatos antes del commit. Esto reduce el tiempo de revisión en un **70%** en equipos de más de 5 desarrolladores.

- **Pruebas de regresión visual**: Integra `free_ebook_maker diff` en tu CI/CD para comparar la salida generada contra un baseline almacenado en S3. La herramienta genera un informe en HTML con diferencias pixel a pixel (umbral configurable, recomendado **ΔE ≤ 2.3** en espacio CIELAB). Si la diferencia supera el umbral, el pipeline falla automáticamente, evitando despliegues con artefactos corruptos.
