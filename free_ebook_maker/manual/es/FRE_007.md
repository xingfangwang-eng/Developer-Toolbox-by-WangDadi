# free_ebook_maker - FRE_007

## Resumen

Motor de compilación de eBooks en español que convierte Markdown + YAML a EPUB3 y PDF/A-3b con validación semántica W3C, generación de índices jerárquicos y empaquetado DRM-free. Optimizado para pipelines CI/CD con soporte de metadatos multilingües (ISO 639-1) y control de versiones Git-LFS.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

Herramienta de línea de comandos que transforma documentos estructurados en Markdown a formatos de libro electrónico profesionales, aplicando reglas de tipografía española (RAE/RAE-es) y generando metadatos Dublin Core con validación contra esquemas XSD. Integra un analizador de legibilidad (Flesch-Szigriszt) y un corrector ortográfico Hunspell con diccionario ES-ES.

### Características Principales

- **Compilación EPUB3 con validación WCAG 2.1 AA**: Genera archivos EPUB3 estrictos con estructura semántica ARIA, tablas de contenido anidadas (hasta 6 niveles) y verificación automática de contraste de color (relación mínima 4.5:1). Utiliza `epubcheck` 4.2.6 para garantizar conformidad con IDPF. Soporta incrustación de fuentes OpenType con subconjuntos Unicode (hasta 10,000 caracteres por fuente).

- **Renderizado PDF/A-3b con metadatos XMP incrustados**: Produce PDFs de archivo utilizando `WeasyPrint` 60.0 con hojas de estilo CSS paginadas (`@page`, `prince-*`). Los metadatos XMP incluyen identificador UUID v4, fecha de creación ISO 8601 y licencia Creative Commons (CC BY-SA 4.0). El motor calcula automáticamente el *crop marks* y el *bleed* de 3 mm para impresión offset.

- **Motor de traducción automática de índices y glosarios**: Detecta automáticamente términos técnicos en el Markdown fuente y genera un glosario bilingüe (ES-EN) con enlaces internos. Utiliza `spaCy` 3.7 para análisis de dependencias y `WordNet` 3.0 para sinónimos. El índice alfabético se genera con precisión de milisegundos, ordenando por Unicode Collation Algorithm (UCA) con locale `es_ES.UTF-8`.

### Cómo Usar

1. **Preparación del entorno y estructura del proyecto**  
   Instala dependencias con `pip install free-ebook-maker[full]` (requiere Python 3.11+). Crea un directorio `src/` con archivos Markdown numerados (`01-intro.md`, `02-capitulo1.md`). Define el archivo `metadata.yaml` con campos obligatorios: `title`, `author`, `language: es`, `isbn` (formato 13 dígitos). Ejecuta `ebook-maker init --template academic` para generar la estructura base con validación de esquema YAML contra `schema-v1.0.json`.

2. **Compilación con parámetros de salida específicos**  
   Ejecuta `ebook-maker build --format epub3 --output ./dist/libro.epub --css ./custom.css --fonts ./fonts/ --validate-wcag --verbose`. El motor procesa cada archivo Markdown con `mistune` 3.0, convierte tablas a HTML semántico (`<table>`, `<caption>`, `<th scope>`), y aplica reglas de tipografía: comillas latinas («»), guiones largos (—) y espacios de no separación (U+00A0) antes de signos dobles (;:!?). La validación WCAG genera un reporte JSON con errores de accesibilidad (ej: imágenes sin `alt` descriptivo).

3. **Publicación y control de versiones**  
   Para integración continua, usa `ebook-maker publish --format pdfa --output ./dist/libro.pdf --metadata-xmp --crop-marks --bleed 3mm --profile press`. El sistema genera un archivo `manifest.json` con checksums SHA-256 de cada recurso incrustado (imágenes, fuentes). Si usas Git, ejecuta `ebook-maker git-hook install` para añadir un pre-commit hook que valida la estructura Markdown y el YAML antes de cada commit. Los metadatos de versión se sincronizan con `git describe --tags --always`.

### Mejores Prácticas

- **Gestión de dependencias y aislamiento del entorno**: Utiliza `pipenv` o `poetry` para fijar versiones de `WeasyPrint` (60.0) y `epubcheck` (4.2.6). No instales globalmente; usa entornos virtuales con `python -m venv .venv`. Para proyectos con múltiples idiomas, define variables de entorno `LANG=es_ES.UTF-8` y `LC_ALL=es_ES.UTF-8` para evitar errores de codificación en la generación de índices.

- **Optimización de activos multimedia**: Las imágenes deben estar en formato PNG (lossless) o JPEG (calidad 85%), con resolución mínima de 300 DPI para impresión. Usa `ebook-maker optimize --images --max-width 1200px --format webp` para reducir el tamaño del EPUB (hasta un 40%). Para PDF, convierte imágenes a CMYK (perfil ISO Coated v2 300%) con `pillow` 10.0. No incrustes videos ni audio en EPUB3 si el público objetivo usa lectores de pantalla (NVDA, JAWS).
