# free_ebook_maker - FRE_028

## Resumen

free_ebook_maker es una herramienta de generación automatizada de libros electrónicos diseñada para transformar contenido Markdown, LaTeX y HTML en formatos EPUB, MOBI y PDF con precisión quirúrgica. Utiliza un pipeline de compilación multietapa que garantiza la integridad estructural del libro, desde la tabla de contenidos dinámica hasta la indexación de referencias cruzadas. Ideal para autores técnicos, editoriales independientes y desarrolladores que necesitan un control granular sobre el proceso de maquetación digital.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de compilación de libros electrónicos de código abierto que opera sobre un sistema de reglas de transformación basado en AST (Abstract Syntax Tree). A diferencia de herramientas genéricas, no se limita a convertir formatos: aplica **normalización semántica** de encabezados, **reconstrucción de flujo de lectura** para pantallas pequeñas y **optimización de metadatos** para cumplir con los estándares de Amazon KDP, Apple Books y Google Play Books. Su núcleo está escrito en Rust con bindings para Python, lo que permite una velocidad de compilación de **0.3 segundos por cada 100 páginas** en hardware moderno.

### Características Principales

- **Compilación con resolución de dependencias recursivas**: Analiza el árbol de archivos Markdown y resuelve referencias a fragmentos externos (`{{include "capitulo2.md"}}`) mediante un grafo acíclico dirigido (DAG). Si detecta un ciclo de inclusión, aborta la compilación con un mensaje de error que indica la línea exacta y el archivo conflictivo. Soporta **hasta 10,000 nodos** en un solo proyecto sin degradación de rendimiento.

- **Generación de índices temáticos con stemming multilingüe**: Extrae automáticamente términos clave del contenido usando un algoritmo de TF-IDF modificado, y construye un índice alfabético con enlaces internos. El motor de stemming integra lematización para español, inglés, francés y alemán, con un **margen de error inferior al 2%** en la identificación de raíces léxicas. Parámetros configurables: `--min-term-frequency=3` (frecuencia mínima de término), `--max-index-entries=500` (máximo de entradas en el índice).

- **Validación de estructura EPUB3 con perfiles de accesibilidad**: Antes de empaquetar el EPUB, ejecuta una suite de pruebas que verifica el cumplimiento de las especificaciones **EPUB Accessibility 1.1** y **WCAG 2.1 AA**. Reporta problemas como imágenes sin descripción alternativa, tablas sin encabezados de fila o columnas, y enlaces rotos internos. El informe de validación se exporta en formato JSON con la ruta exacta del archivo y el código de error (ejemplo: `{"file":"images/figura3.png","error":"E-IMG-ALT","line":42}`).

### Cómo Usar

1. **Inicialización del proyecto con plantilla de estructura**:
   Ejecuta `free_ebook_maker init --template=technical-manual` para generar un esqueleto de proyecto que incluye carpetas `src/`, `assets/`, `config/` y un archivo `book.toml` con parámetros predefinidos. Luego, modifica `book.toml` para definir el idioma principal (`lang = "es"`), el formato de salida (`output = ["epub", "pdf"]`) y el perfil de compilación (`profile = "production"`). Este perfil activa la compresión de imágenes con **PNGQuant** (pérdida de calidad máxima del 5%) y la eliminación de metadatos redundantes.

2. **Compilación con depuración paso a paso**:
   Usa `free_ebook_maker build --debug --output-dir ./build` para iniciar la compilación. El flag `--debug` activa la salida de logs en tiempo real con marcas de tiempo en **microsegundos** (formato ISO 8601). Si ocurre un error de sintaxis en un archivo Markdown, el compilador se detiene y muestra la línea problemática con un contexto de 3 líneas antes y después. Por ejemplo: `[ERROR] Línea 127 en src/capitulo5.md: tabla mal formada (número de columnas inconsistente: esperaba 4, encontrado 3)`. Corrige el error y repite el comando.

3. **Publicación directa a plataformas mediante API REST**:
   Configura las credenciales en `config/credentials.toml` para Amazon KDP y Google Play Books. Luego ejecuta `free_ebook_maker publish --platform=kdp --draft=true`. La herramienta empaqueta el EPUB, lo firma digitalmente con una clave privada RSA de 4096 bits, y lo envía al endpoint de KDP mediante HTTPS con autenticación OAuth 2.0. El parámetro `--draft=true` sube el libro como borrador sin enviarlo a revisión. El tiempo de respuesta promedio es de **1.2 segundos** para archivos de hasta 50 MB.

### Mejores Prácticas

- **Fragmentación de contenido para compilación incremental**: Divide el libro en archivos Markdown de no más de 200 líneas cada uno. Usa el sistema de inclusión (`{{include ...}}`) para ensamblar capítulos. Esto permite que free_ebook_maker recompile solo los fragmentos modificados (detección mediante hash SHA-256 del contenido), reduciendo el tiempo de compilación de proyectos grandes (más de 500 páginas) de minutos a segundos. Además, facilita la colaboración en equipo mediante control de versiones con Git.

- **Optimización de imágenes para cumplir con los límites de tamaño de plataforma**: Antes de incluir imágenes en el proyecto, ejecuta `free_ebook_maker optimize-images --max-width=1200 --max-height=1800 --format=webp`. Este comando redimensiona todas las imágenes en `assets/` respetando la relación de aspecto original, las convierte a WebP con calidad 85 (compresión sin pérdida perceptible), y actualiza las referencias en los archivos Markdown automáticamente. Esto asegura que el EPUB final no exceda los **50 MB** (límite de Amazon KDP para carga directa) y que las imágenes se rendericen correctamente en lectores de tinta electrónica con pantallas de 300 PPI.
