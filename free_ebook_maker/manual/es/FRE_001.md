# free_ebook_maker - FRE_001

## Resumen

free_ebook_maker es una solución de compilación automatizada para la generación de libros electrónicos en formato EPUB 3.2 y PDF/A-3, optimizada para la conversión masiva de contenido Markdown, LaTeX y HTML5 semántico. Diseñada para pipelines CI/CD, soporta validación W3C, metadatos Dublin Core personalizables y empaquetado con firmas SHA-256. El proyecto se enfoca en la integridad estructural del libro, la accesibilidad WCAG 2.1 y la reducción de latencia en el proceso de renderizado.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de compilación de libros electrónicos de código abierto que opera como una herramienta de línea de comandos y biblioteca Python. Resuelve la fragmentación de formatos al unificar la entrada desde fuentes Markdown extendido, archivos LaTeX y documentos HTML5 con microdatos schema.org. Su núcleo utiliza `lxml` para el análisis sintáctico y `WeasyPrint` para la generación de PDF, logrando un tiempo medio de compilación de 1.2 segundos por documento de 300 páginas en hardware estándar. Está diseñado para entornos serverless y soporta la inyección de variables de entorno para la personalización de estilos CSS.

### Características Principales

- **Validación estructural con esquemas EPUB 3.2**: Implementa un verificador interno que recorre el árbol DOM del archivo OPF y el manifiesto, garantizando que todos los recursos (imágenes, fuentes, hojas de estilo) estén referenciados correctamente. Emite advertencias para enlaces rotos y archivos huérfanos con una precisión del 99.8% en la detección de errores de ruta relativa.
- **Compilación diferencial con caché de fragmentos**: Utiliza un sistema de hash SHA-256 para cada capítulo o sección Markdown. Si el contenido no ha cambiado desde la última compilación, se reutiliza el fragmento HTML pre-renderizado, reduciendo el tiempo de compilación incremental en un 87% para proyectos con más de 50 capítulos.
- **Inyección de metadatos Dublin Core y JSON-LD**: Permite definir un archivo `metadata.yaml` con campos como `dc:creator`, `dc:date`, `dc:language` y un bloque JSON-LD para datos estructurados (ej: `@context: "https://schema.org/Book"`). El compilador valida la coherencia entre ambos formatos y lanza una excepción si hay conflictos de tipo de dato.

### Cómo Usar

1. **Preparación del proyecto y configuración del entorno**: Crea un directorio raíz con la estructura `src/` (contenido Markdown), `assets/` (imágenes, fuentes) y un archivo `book.yaml`. Ejecuta `free_ebook_maker init --template default` para generar la plantilla base. Ajusta el archivo `book.yaml` con los parámetros `title: "Mi Libro"`, `language: "es"`, `css: "style.css"` y `output_format: "epub"`. Verifica la sintaxis YAML con `free_ebook_maker validate --config book.yaml --strict`.
2. **Compilación con optimización de recursos**: Ejecuta `free_ebook_maker build --input src/ --output dist/ --compress-images --quality 85`. El parámetro `--compress-images` activa la optimización automática de PNG/JPEG mediante `Pillow` y `mozjpeg`, reduciendo el tamaño del archivo final en un 40% promedio. Para compilar en PDF, añade `--pdf-engine weasyprint --pdf-variant pdf/a-3u`. Monitorea el progreso con `--verbose`.
3. **Firma y empaquetado final**: Una vez compilado, ejecuta `free_ebook_maker sign --input dist/libro.epub --certificate cert.pem --private-key key.pem` para añadir una firma digital XML-DSig. Luego, `free_ebook_maker archive --input dist/ --output final/ --checksum sha256` genera un archivo ZIP con el checksum incrustado en el nombre (ej: `libro_abc123def456.zip`). Para verificar la integridad, usa `free_ebook_maker verify --archive final/libro_*.zip`.

### Mejores Prácticas

- **Integración continua con pruebas de regresión visual**: Implementa un pipeline en GitHub Actions que ejecute `free_ebook_maker build --output test/` en cada commit. Utiliza `pdftotext` y `diff` para comparar el contenido textual del PDF generado con una versión de referencia, detectando cambios no deseados en el texto o el diseño. Establece un umbral de tolerancia del 0.5% en la diferencia de caracteres para evitar falsos positivos.
- **Gestión de dependencias con entornos virtuales aislados**: Crea un archivo `requirements-freeze.txt` con versiones exactas de las bibliotecas (`lxml==4.9.3`, `weasyprint==60.2`, `Pillow==10.1.0`). Usa `pip install --no-deps -r requirements-freeze.txt` para evitar conflictos con otras herramientas del sistema. Para proyectos grandes, considera usar `docker build` con una imagen base de Python 3.11 slim, reduciendo el tamaño de la imagen final a 180 MB y eliminando paquetes innecesarios.
