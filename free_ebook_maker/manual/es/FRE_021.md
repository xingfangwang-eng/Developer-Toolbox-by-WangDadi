# free_ebook_maker - FRE_021

## Resumen

Plataforma de compilación y generación automatizada de libros electrónicos en formato EPUB3 y PDF/A-3b, diseñada para flujos editoriales con requisitos estrictos de metadatos, tipografía y accesibilidad. Integra procesamiento por lotes, validación semántica de contenido y conversión multiformato con latencia inferior a 200 ms por página.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de publicación digital que transforma documentos Markdown, LaTeX o HTML5 semántico en libros electrónicos profesionales. Opera sobre una arquitectura de microservicios con colas de trabajo asíncronas (Redis Streams) y almacenamiento en caché de fragmentos renderizados. Soporta inyección de metadatos Dublin Core, generación de tabla de contenidos jerárquica y empaquetado conforme a las especificaciones IDPF 3.2 y PDF/A-3b (ISO 19005-3:2012).

### Características Principales

- **Renderizado tipográfico con kerning avanzado y ligaduras contextuales**: Implementa un motor de composición tipográfica basado en HarfBuzz con ajuste de espaciado proporcional (tracking configurable de -0.05 a 0.15 em). Soporta familias tipográficas OpenType con variaciones de eje (wght, wdth, slnt) y genera glifos SVG incrustados para caracteres fuera del plano BMP Unicode. La precisión de posicionamiento alcanza 1/1000 de unidad de em, con renderizado a 300 DPI en salida PDF.

- **Validación estructural y semántica contra esquemas EPUB 3.2**: Utiliza un analizador sintáctico de pila completa que verifica la conformidad con el esquema OPF 2.0.1 y el perfil de accesibilidad WCAG 2.1 AA. Detecta automáticamente etiquetas anidadas incorrectamente, atributos `lang` faltantes en elementos no latinos, y referencias a recursos externos no incluidos en el manifiesto. El tiempo de validación completo para un libro de 500 páginas es inferior a 1.2 segundos en hardware estándar (CPU x86_64 a 2.5 GHz).

- **Compresión diferencial con deduplicación de fragmentos HTML**: Aplica un algoritmo de compresión basado en LZMA2 con ventana de 64 MB y diccionario adaptativo. Detecta bloques repetidos (párrafos, imágenes base64, estilos CSS) a nivel de nodo DOM y los reemplaza por referencias hash SHA-256. La tasa media de compresión alcanza 8:1 en contenido mixto (texto + imágenes JPEG al 85% de calidad), con un overhead de descompresión inferior a 50 µs por fragmento.

### Cómo Usar

1. **Configuración del pipeline de compilación con parámetros de calidad**: Ejecuta el comando `free_ebook_maker build --input-dir ./manuscrito --output-format epub3 --compression-level 9 --metadata-file metadatos.json --font-embed all --image-quality 92 --validation-strict`. El parámetro `--compression-level` acepta valores enteros de 0 a 9 (0 = sin compresión, 9 = máxima compresión con análisis de contexto), mientras que `--image-quality` define el factor de compresión JPEG (0-100). Para PDF/A-3b, añade `--pdf-profile a-3b --pdf-version 1.7 --embed-fonts true`.

2. **Inyección de metadatos Dublin Core con validación de esquema**: Prepara un archivo JSON con los campos obligatorios (`title`, `creator`, `identifier`, `language`, `date`) y opcionales (`publisher`, `rights`, `subject`, `description`). El sistema validará automáticamente contra el esquema XML de Dublin Core Terms (ISO 15836:2009). Usa `free_ebook_maker metadata --import metadatos.json --validate --strict-mode` para verificar la integridad semántica. Los identificadores deben cumplir el patrón `urn:uuid:` seguido de un UUIDv4.

3. **Renderizado por lotes con monitorización de cola de trabajo**: Para procesar múltiples manuscritos simultáneamente, inicia el worker con `free_ebook_maker worker --queue-size 1000 --concurrency 4 --timeout 300`. Los trabajos se distribuyen en 4 hilos paralelos, cada uno manejando un archivo de entrada. Monitorea el progreso con `free_ebook_maker status --job-id <uuid> --format json`. El sistema expone métricas Prometheus en el puerto 9090 (`/metrics`) con indicadores como `render_time_seconds`, `compression_ratio`, `validation_errors_total`.

### Mejores Prácticas

- **Optimización de imágenes para compresión eficiente**: Antes de incluir imágenes en el manuscrito, conviértelas a formato WebP con pérdida (calidad 85-90) o JPEG progresivo (submuestreo 4:2:0). Las imágenes vectoriales deben encapsularse como SVG optimizado con paths simplificados (tolerancia de simplificación 0.001). Evita PNG a menos que sea necesario para transparencia alfa; en ese caso, usa PNG-8 con paleta de 256 colores. El tamaño máximo recomendado por imagen es 800 KB para EPUB y 2 MB para PDF.

- **Estructuración jerárquica del contenido con anclajes semánticos**: Divide el manuscrito en archivos Markdown individuales por capítulo o sección, con nombres de archivo que reflejen el orden numérico (`capitulo-01.md`, `capitulo-02.md`). Utiliza encabezados de nivel 1 (`#`) únicamente para el título del libro, nivel 2 (`##`) para capítulos, y niveles 3-5 para subsecciones. Incluye un archivo `toc.md` con la estructura completa de la tabla de contenidos en formato lista anidada. Esto permite que el motor genere automáticamente enlaces de navegación intra-libro con fragmentos de identificador de párrafo (`#parrafo-123`).
