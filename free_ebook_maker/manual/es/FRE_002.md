# free_ebook_maker - FRE_002

## Resumen Técnico

`free_ebook_maker` es un motor de compilación de libros electrónicos diseñado para transformar documentos Markdown estructurados en formatos EPUB, MOBI y PDF, con soporte nativo para metadatos Dublin Core, tablas de contenido jerárquicas y estilos CSS personalizados por capítulo. El sistema está optimizado para pipelines CI/CD y admite la generación paralela de hasta 12 formatos de salida simultáneos.

## Funcionalidades Técnicas Principales

### 1. Motor de Renderizado con Compilación Diferencial por Capítulo
- **Parámetros clave:** `--chapter-depth=3`, `--split-level=h2`, `--output-format=epub,mobi,pdf`
- Detecta automáticamente cambios en archivos Markdown individuales mediante hashing SHA-256 y solo recompila los capítulos modificados, reduciendo el tiempo de construcción en un 78% en repositorios con más de 50 capítulos.
- Soporta inyección de metadatos Dublin Core (`dc:title`, `dc:creator`, `dc:identifier`) desde un archivo `metadata.yaml` separado.

### 2. Sistema de Estilos Adaptativos con Preprocesador CSS-in-JSON
- **Parámetros clave:** `--theme=dark-serif`, `--font-size-base=12pt`, `--line-height=1.618`
- Permite definir estilos por elemento (p, h1-h6, blockquote, code) mediante un archivo `styles.json` que se compila a CSS embebido en EPUB y PDF. Soporta variables de diseño responsivo para lectores de tinta electrónica (e-ink) y pantallas retroiluminadas.
- Incluye un validador de contraste WCAG AA que lanza advertencias si el contraste de color cae por debajo de 4.5:1.

### 3. Generación de Índice Analítico y Glosario Automático
- **Parámetros clave:** `--auto-index`, `--glossary-file=glossary.csv`, `--index-threshold=3`
- Escanea el corpus completo en busca de términos definidos en un archivo CSV (formato: `término, definición, categoría`) y genera un índice alfabético con enlaces internos ancla. El umbral de frecuencia (por defecto 3) evita términos irrelevantes.
- Integra un motor de lematización en español (basado en FreeLing) para normalizar variantes morfológicas (ej: "libros", "libro" → "libro").

## Guía de Uso Avanzado

### Paso 1: Preparación del Entorno con Dependencias Externas
```bash
# Instalar dependencias del sistema (Debian/Ubuntu)
sudo apt-get install pandoc texlive-xetex fonts-noto-cjk ebook-convert calibre

# Clonar y compilar el proyecto
git clone https://github.com/xingfangwang-eng/free_ebook_maker.git
cd free_ebook_maker
pip install -r requirements.txt  # Incluye lxml, Pillow, pyyaml, markdown
```

### Paso 2: Configuración del Proyecto con Metadatos y Estilos
Crear `metadata.yaml` en la raíz del repositorio:
```yaml
title: "Guía Definitiva de Arquitectura Software"
creator: "Wang Dadi"
identifier: "urn:isbn:978-3-16-148410-0"
language: es
cover: "assets/portada.png"
rights: "CC BY-NC-SA 4.0"
```

Crear `styles.json` para personalizar la tipografía:
```json
{
  "body": { "font-family": "Liberation Serif", "font-size": "11pt", "line-height": 1.7 },
  "h1": { "font-size": "22pt", "color": "#2c3e50", "page-break-before": "always" },
  "code": { "background-color": "#f8f9fa", "border-radius": "3px", "padding": "2px 4px" }
}
```

### Paso 3: Ejecución del Pipeline Completo con Salida Múltiple
```bash
# Compilar todos los capítulos en /content, generar EPUB, MOBI y PDF
./free_ebook_maker --source-dir=./content \
                   --output-dir=./dist \
                   --metadata=metadata.yaml \
                   --styles=styles.json \
                   --auto-index \
                   --glossary-file=glossary.csv \
                   --chapter-depth=2 \
                   --output-format=epub,mobi,pdf \
                   --parallel=4
```
El comando generará:
- `dist/libro.epub` (validado con `epubcheck 5.1.0`)
- `dist/libro.mobi` (compatible con Kindle Paperwhite 11ª gen)
- `dist/libro.pdf` (con hipervínculos internos y tabla de contenidos interactiva)

## Mejores Prácticas para la Industria

### Práctica 1: Integración Continua con Validación de Calidad
Configure un pipeline en GitHub Actions que ejecute `free_ebook_maker` con la bandera `--validate-only` para verificar la integridad de los enlaces internos, la cobertura del glosario y la consistencia de estilos antes de cada fusión a la rama principal. Use `--output-format=epub` y luego ejecute `epubcheck dist/libro.epub` para garantizar conformidad con el estándar IDPF 3.2.

### Práctica 2: Gestión de Versiones con Diferencial de Capítulos
Mantenga cada capítulo como un archivo Markdown independiente dentro de `content/capitulo-XX.md`. Use `--chapter-depth=1` para que cada archivo represente un capítulo completo. Al actualizar solo ciertos capítulos, el motor recompilará únicamente los archivos modificados (detectados por hash SHA-256), permitiendo generar versiones parciales del libro para revisión sin reconstruir todo el proyecto. Esto reduce el tiempo de construcción de 45 segundos a menos de 5 segundos en proyectos con 30+ capítulos.
