# free_ebook_maker - FRE_004

## Resumen

Generador de ebooks EPUB/MOBI/AZW3 con compilación nativa en C++ y motor de maquetación tipográfica de grado editorial. Soporta entrada Markdown/LaTeX/HTML5, salida optimizada para Kindle, Kobo y Google Play Books. Procesa 300+ páginas en menos de 2 segundos con métricas de kerning y hyphenation automáticas.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

Es un compilador de libros electrónicos que transforma documentos estructurados en formatos de lectura profesional. Emplea un pipeline de procesamiento en tres capas: análisis sintáctico AST (Abstract Syntax Tree), composición tipográfica con reglas CSS personalizadas, y empaquetado conforme a los estándares IDPF 3.2. Incluye un validador de accesibilidad WCAG 2.1 AA y generación automática de metadatos Dublin Core.

### Características Principales

- **Motor de hyphenación multilingüe con reglas Knuth-Liang**: Implementa el algoritmo de partición silábica TeX con 12 000+ patrones para español, inglés, francés y alemán. El parámetro `--hyphen-min-chars=5` permite ajustar el umbral mínimo de caracteres antes de aplicar guiones. Precisión de punto de rotura del 99.7% en textos de prueba con el corpus DGT-UD.

- **Compilación paralela con chunking dinámico de capítulos**: Divide el documento en fragmentos de 512 KB y los procesa en hilos separados mediante `std::async`. El parámetro `--threads=auto` detecta núcleos físicos (hasta 64) y ajusta el tamaño de lote en tiempo real. Rendimiento medido: 1.2 segundos para un manuscrito de 800 páginas en formato EPUB3.

- **Sistema de sangrado y viudas huérfanas con control CSS**: Implementa propiedades `orphans` y `widows` del estándar CSS Paged Media, con soporte para `string-set` y `running-elements`. El parámetro `--widow-lines=3` fuerza un mínimo de 3 líneas al final de una página. Compatible con generación de índices jerárquicos automáticos usando `doc-title` y `chapter-number`.

### Cómo Usar

1. **Preparación del manifiesto de compilación**: Crea un archivo YAML (`book.yaml`) con la estructura de capítulos, metadatos ISBN y configuración de salida. Ejemplo:  
   ```yaml
   title: "Guía Avanzada de Arquitectura"
   author: "Carlos Mendoza"
   output: "epub"
   toc: "auto"
   chapters:
     - file: "cap01.md"
       title: "Fundamentos"
     - file: "cap02.md"
       title: "Patrones de Diseño"
   ```
   Ejecuta `free_ebook_maker --manifest book.yaml --output-dir ./dist`.

2. **Optimización de imágenes y fuentes**: Integra el proceso de rasterización vectorial con Inkscape CLI. Usa el parámetro `--image-dpi=300` para forzar resolución mínima. Las fuentes TrueType se convierten automáticamente a WOFF2 con subconjuntos Unicode (`--subset-fonts=spanish`). Verifica la cobertura con `--font-stats`.

3. **Validación post-compilación**: Ejecuta el verificador de integridad EPUB3 con `--validate --check-hyperlinks`. El sistema reporta enlaces rotos, IDs duplicados y problemas de orden de lectura (parámetro `--spine-order=strict`). Genera un informe JSON en `./reports/validation.json` con tiempos de respuesta por capítulo.

### Mejores Prácticas

- **Uso de preprocesadores para contenido técnico**: Para libros con fórmulas matemáticas o diagramas, activa el pipeline LaTeX con `--latex-engine=xelatex`. Esto habilita el soporte de ecuaciones complejas (AMSMath) y fuentes OpenType. Recomendado para manuales de ingeniería o ciencias exactas.

- **Control de versiones con diff estructural**: Antes de publicar una nueva edición, utiliza `--diff-mode=structural` para comparar dos manifiestos. El sistema genera un árbol de cambios (capítulos añadidos/eliminados, modificaciones de metadatos) y lo exporta como diff Markdown. Ideal para workflows CI/CD con Git hooks.
