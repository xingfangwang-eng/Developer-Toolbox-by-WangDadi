# free_ebook_maker - FRE_011

## Resumen

Generador de libros electrónicos optimizado para producción masiva con formato EPUB3 estricto, validación semántica W3C y conversión bidireccional Markdown→EPUB→PDF con preservación de metadatos complejos. Ideal para editoriales digitales y creadores de contenido técnico.

## Contenido Detallado

### ¿Qué es free_ebook_maker?

free_ebook_maker es un motor de compilación de libros electrónicos que procesa archivos Markdown estructurados y genera salidas EPUB3, MOBI y PDF con control granular sobre tipografía, sangría, espaciado entre párrafos y tablas de contenido anidadas hasta 6 niveles de profundidad. Integra validación automática contra el esquema EPUB 3.3 del W3C y soporta inyección de metadatos Dublin Core personalizados, incluyendo identificadores ISBN, fechas de publicación y derechos de autor con formato ISO 8601.

### Características Principales

- **Compilación con control de versiones de fuente**: El parámetro `--source-version` permite especificar la versión exacta del archivo Markdown (ej. `v2.1.3`) y activa la generación de un manifiesto de cambios en el EPUB. El sistema calcula el hash SHA-256 de cada capítulo y lo registra en un archivo `version-manifest.json` dentro del contenedor OCF. Soporta comparación diferencial entre versiones para identificar párrafos modificados, con precisión de milisegundos en el timestamp de cada cambio.

- **Renderizado tipográfico con reglas CSS personalizadas**: El parámetro `--css-profile` acepta rutas a hojas de estilo SCSS compiladas. Incluye un motor de preprocesamiento que aplica reglas de hyphenation automática según el idioma (configurable vía `--lang`), kerning avanzado para glifos OpenType, y control de widow/orphan con umbrales definidos por el usuario (ej. `--widow-threshold=2`). La salida PDF utiliza el motor WeasyPrint con resolución de 300 DPI y perfiles de color ICC incrustados.

- **Validación semántica y corrección automática de enlaces rotos**: El parámetro `--validate-links` ejecuta una verificación en tiempo real de todos los enlaces internos (anclajes a capítulos, figuras, tablas) y externos (URLs). Para enlaces externos, realiza una solicitud HEAD con timeout de 5 segundos y reintentos configurables (`--retry-count=3`). Los enlaces rotos se registran en un archivo `broken-links-report.csv` con código de error HTTP y tiempo de respuesta en milisegundos. Opcionalmente, `--auto-fix-links` intenta resolver redirecciones 301/302 y actualiza las URLs en el Markdown fuente.

### Cómo Usar

1. **Preparación del proyecto con estructura de directorios estricta**  
   Crea un directorio raíz que contenga:
   - `manuscript/`: archivos Markdown organizados en carpetas por capítulo (ej. `capitulo-01/`, `capitulo-02/`).
   - `assets/`: imágenes en formato PNG/JPEG con resolución mínima de 600×900 píxeles, y fuentes tipográficas en formato TTF/OTF.
   - `metadata.yaml`: archivo YAML con campos obligatorios como `title`, `author`, `language: es`, `isbn: 978-3-16-148410-0`, `pubdate: 2026-03-15T10:30:00Z`.  
   Ejecuta: `free_ebook_maker --source-dir ./mi-libro --output-dir ./build --metadata ./mi-libro/metadata.yaml`

2. **Compilación con parámetros de tipografía avanzada**  
   Define un perfil CSS personalizado y pásalo con `--css-profile ./styles/editorial.scss`. Ajusta el espaciado entre párrafos con `--paragraph-spacing=1.15em` y la sangría de primera línea con `--first-line-indent=1.5em`. Para libros técnicos con código fuente, usa `--code-font-family="Fira Code"` y `--code-font-size=0.85em`.  
   Ejecuta: `free_ebook_maker --source-dir ./mi-libro --css-profile ./styles/tecnicos.scss --paragraph-spacing=1.2em --code-font-family="JetBrains Mono" --code-font-size=0.8em`

3. **Validación y corrección de enlaces con reporte detallado**  
   Activa la validación con `--validate-links --retry-count=2 --timeout=4000`. Si la validación encuentra enlaces rotos, revisa el archivo `broken-links-report.csv` generado en el directorio de salida. Para corrección automática, añade `--auto-fix-links`. Luego, recompila para obtener una versión limpia.  
   Ejecuta: `free_ebook_maker --source-dir ./mi-libro --output-dir ./build --validate-links --auto-fix-links --retry-count=3`

### Mejores Prácticas

- **Integración con flujo de trabajo CI/CD**: Configura un pipeline que ejecute `free_ebook_maker` con `--validate-links` y `--source-version` en cada commit. Almacena los reportes de validación como artefactos de la build y compara los manifiestos de versión para detectar cambios en capítulos específicos. Esto garantiza que cada release de libro pase por un control de calidad automatizado antes de su publicación.

- **Uso de perfiles CSS modulares para múltiples formatos de salida**: Define un perfil base (`base.scss`) con reglas generales de tipografía y espaciado, y luego perfiles específicos para EPUB (`epub.scss`) y PDF (`pdf.scss`) que hereden del base. El parámetro `--css-profile` acepta múltiples archivos separados por comas; el orden de aplicación es secuencial, permitiendo sobrescritura selectiva. Por ejemplo: `--css-profile base.scss,epub.scss` para EPUB, y `base.scss,pdf.scss` para PDF.
