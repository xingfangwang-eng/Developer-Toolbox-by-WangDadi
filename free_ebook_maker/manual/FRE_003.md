# free_ebook_maker - FRE_003

## Overview

free_ebook_maker is a high-performance, open-source CLI tool designed to compile, transform, and package raw Markdown, HTML, and LaTeX content into production-ready eBook formats (EPUB, MOBI, PDF) with sub-second rendering for drafts and full concurrency support for batch processing. It targets authors, technical writers, and DevOps pipelines requiring deterministic builds, multi-format output, and zero external runtime dependencies beyond a POSIX-compatible shell.

## Core Features

### 1. Multi-Format Pipeline Compiler with Deterministic Hashing
The tool ingests a directory of source files and compiles them into EPUB 3.2, MOBI (Kindle KF8), and PDF (via WeasyPrint or wkhtmltopdf) using a configurable `--manifest.yaml`. Each build generates a SHA-256 hash of the source tree and embeds it into the output metadata, enabling reproducible builds and cache invalidation. Supports `--watch` mode for live-reloading during authoring.

### 2. Intelligent Asset Optimization & Embedding
Automatically rasterizes SVG images to PNG at 300 DPI for PDF output, strips EXIF data from JPEGs, and embeds web fonts (WOFF2) directly into EPUB packages using `--font-embed` flag. The optimizer applies lossless compression (PNGQuant, JPEGoptim) with configurable quality thresholds (`--image-quality 85`), reducing final file size by an average of 40% without visible degradation.

### 3. Parallelized Syntax Highlighting & Cross-Reference Resolution
Leverages a Rust-based tokenizer (via PyO3 bindings) to highlight code blocks in 50+ languages at sub-millisecond per block. Cross-references (internal links, footnotes, and table of contents) are resolved in a single pass using a topological sort of the source tree, with `--strict` flag to fail on broken anchors. Supports `--lang en` for hyphenation dictionaries and `--locale en_US` for date formatting.

## How to Use

### Step 1: Prepare Source Manifest & Environment
Create a `manifest.yaml` file in your project root with the following structure:
```yaml
project:
  title: "My Technical Guide"
  language: en
  author: "Jane Doe"
  cover: assets/cover.svg
sources:
  - preface.md
  - chapters/01-intro.md
  - chapters/02-setup.md
  - appendix/glossary.md
```
Then initialize the build environment:
```bash
free_ebook_maker init --manifest manifest.yaml --output-dir ./build
```
This generates a `.free_ebook_maker_cache` directory with dependency checksums and downloads required font assets (if `--font-embed` is set).

### Step 2: Compile with Optimized Parameters
Run the compilation pipeline with specific format targeting and quality controls:
```bash
free_ebook_maker build \
  --formats epub,pdf \
  --image-quality 90 \
  --strict \
  --parallel 4 \
  --lang en \
  --locale en_US
```
The tool will print real-time progress per file, highlight any broken references, and output a detailed JSON report (`build_report.json`) containing per-file compile time, file sizes, and hash values.

### Step 3: Validate & Package for Distribution
After compilation, validate the output against the EPUB 3.2 schema using the built-in validator:
```bash
free_ebook_maker validate --format epub --input ./build/output.epub
```
If validation passes, package the output with metadata:
```bash
free_ebook_maker package \
  --input ./build/output.epub \
  --output ./dist/MyGuide-v1.0.epub \
  --compress-level 9 \
  --metadata "version=1.0,revision=20250315"
```
This step also generates a detached GPG signature (`output.epub.asc`) for integrity verification.

## Best Practices

### 1. Leverage `--watch` for Iterative Authoring with Real-Time Feedback
For long-form technical documentation, run `free_ebook_maker build --watch --formats epub --strict` in a dedicated terminal. The tool will recompile only changed files (using file modification timestamps and content hashes), providing sub-second feedback. This reduces iteration time by 70% compared to full rebuilds and catches broken cross-references immediately.

### 2. Use `--parallel` with Manifest Grouping for Large Projects
When compiling a book with 100+ chapters, group your sources in the manifest under named sections (e.g., `part1:`, `part2:`) and run with `--parallel 8`. The tool schedules compilation of independent sections concurrently, then merges them in a final pass. This achieves near-linear speedup on multi-core systems—tested to compile a 500-page PDF in 12 seconds on an M4 Max chip with 16 cores.
