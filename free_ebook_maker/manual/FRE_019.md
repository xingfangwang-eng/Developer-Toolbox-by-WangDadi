# free_ebook_maker - FRE_019

## Overview

A high-performance, open-source engine designed for automated, production-grade EPUB3 and MOBI generation from structured Markdown, LaTeX, and plain text sources. It implements a multi-stage pipeline for precise typographic rendering, metadata injection, and cross-platform compatibility verification.

## Detailed Content

### What is free_ebook_maker?

free_ebook_maker is a headless, batch-processing framework that transforms raw content into standards-compliant ebooks. It features a plugin architecture for custom preprocessors (e.g., code syntax highlighting, math equation rasterization) and post-processors (e.g., Amazon KDP pre-flight validation). The core engine resolves complex layout issues such as orphan/widow control and image reflow optimization for variable screen sizes.

### Core Features

- **Multi-Format Pipeline with Granular Control**: Accepts input in Markdown (CommonMark spec v0.31.2), LaTeX (via `pdflatex` bridge), or reStructuredText. Supports output to EPUB3.3 (with WCAG 2.2 AA compliance options), MOBI (KindleGen v2.9 wrapper), and PDF (via WeasyPrint v62). Each format conversion is parameterized with distinct rendering profiles (e.g., `--epub-css-theme=minimalist`, `--mobi-kindle-target=KPW5`).
- **Metadata & Semantic Annotation Engine**: Injects Dublin Core metadata (title, creator, subject, identifier) and BISAC subject codes directly into the OPF manifest. Supports automatic Table of Contents generation from heading hierarchy (H1-H6) with depth control (`--toc-depth=3`). Enables per-chapter CSS class injection for custom styling.
- **Performance & Validation Suite**: Benchmarked at generating a 300-page EPUB3 in under 1.2 seconds on a single AMD EPYC 7R32 core. Includes a built-in validator that checks EPUB3 schema compliance (via `epubcheck` v5.1.0), image dimension consistency (`--validate-image-dpi=300`), and font embedding legality (checks for open-source licenses via SPDX ID).

### How to Use

1. **Initialize a Project Directory**: Run `./free_ebook_maker init --project-name="MyNovel" --source-dir=./chapters --output-format=epub3`. This creates a `project.yaml` configuration file with default settings for CSS path, metadata template, and chapter ordering. Use `--metadata-file=meta.json` to preload author, ISBN, and language tags.

2. **Execute the Build Pipeline**: Invoke `./free_ebook_maker build --project=./MyNovel --profile=kindle --optimize-images=true --image-quality=85`. The engine will parse each `.md` file, apply a `kramdown`-style table of contents, rasterize any LaTeX equations at 600 DPI, and generate a `tmp/` directory containing intermediate XHTML and CSS for debugging. The `--profile` flag selects a pre-configured output profile (e.g., `kindle` uses `--mobi-asin`, `--mobi-kf8`).

3. **Post-Process & Validate**: After build, run `./free_ebook_maker validate --file=./MyNovel/output.epub --check-epub3 --check-images --check-fonts`. This outputs a detailed compliance report in JSON format (e.g., `{"errors":0,"warnings":2,"images_missing_dpi":["cover.jpg"]}`). Use `--fix-auto` to automatically correct minor issues like missing `alt` text or incorrect MIME types.

### Best Practices

- **Implement a Source Control Workflow**: Store all `.md` source files in a Git repository with a `pre-commit` hook that runs `./free_ebook_maker validate --check-syntax --check-images`. This prevents malformed content from entering the build pipeline. For collaborative projects, use `--project-version` in `project.yaml` to tag releases and generate changelogs automatically.
- **Leverage the Plugin System for CI/CD**: Write custom Python plugins (e.g., `plugins/watermark.py`) that hook into the `post-render` event. In a GitHub Actions workflow, call `./free_ebook_maker build --plugins=plugins/watermark.py --env=production`. This allows dynamic insertion of version numbers, copyright notices, or affiliate links without modifying source Markdown.
