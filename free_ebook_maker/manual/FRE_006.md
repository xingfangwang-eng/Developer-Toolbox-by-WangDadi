# free_ebook_maker - FRE_006

## Overview

This is the architecture blueprint for `free_ebook_maker`, a production-grade EPUB/PDF generation engine designed for automated book publishing pipelines. It ingests raw Markdown, LaTeX, or plain text and outputs a fully styled, metadata-rich eBook with zero manual intervention. The system is built for CI/CD integration, batch processing, and multi-format distribution.

## Core Features

### 1. Semantic AST Parser with Cross-Reference Resolution
The engine parses input Markdown into an Abstract Syntax Tree (AST) that preserves heading hierarchy, blockquote nesting, and code-fence language tags. It then resolves all internal cross-references (`[see Section 4.2]`) and footnote anchors before rendering. The parser operates at **< 15ms per 10,000 tokens** and supports bidirectional link validation, flagging dead anchors with a non-zero exit code.

- **Parameter:** `--resolve-refs` (boolean, default: `true`)
- **Parameter:** `--max-nesting-depth` (integer, default: `6`, throws `NestingOverflowError` if exceeded)

### 2. Multi-Format Renderer with Font Subsetting
The renderer outputs EPUB 3.2, PDF (via LaTeX), and MOBI. For EPUB, it automatically subsets any embedded TrueType/OpenType fonts using HarfBuzz, reducing file size by **40-70%** compared to full font embedding. PDF generation uses XeLaTeX with custom `.sty` templates, supporting Unicode math and CJK characters natively.

- **Parameter:** `--subset-fonts` (enum: `none`, `auto`, `force`, default: `auto`)
- **Parameter:** `--pdf-engine` (string: `xelatex` or `lualatex`, default: `xelatex`)
- **Performance metric:** EPUB generation completes in **< 2 seconds** for a 300-page book on a modern ARM64 CPU.

### 3. Metadata Injection & ISBN Validator
The tool injects Dublin Core metadata (title, creator, publisher, ISBN, language) into the EPUB OPF file. It includes an ISBN-13 checksum validator that rejects malformed identifiers with a descriptive error message. The validator also auto-normalizes ISBN-10 to ISBN-13 format.

- **Parameter:** `--meta-file` (path to YAML/JSON metadata file, required)
- **Parameter:** `--validate-isbn` (boolean, default: `true`)
- **Output:** Generates a `metadata.xml` alongside the final eBook for third-party cataloging systems.

## How to Use

### Step 1: Prepare the Source Material and Metadata
Structure your manuscript as a single Markdown file with YAML frontmatter, or provide a directory of numbered Markdown files (e.g., `01-preface.md`, `02-chapter1.md`). Create a `metadata.yaml` file with the following required fields:

```yaml
title: "Advanced Python Architecture"
creator: "Dr. Evelyn Nash"
publisher: "O'Reilly Media"
isbn: "978-1-491-95603-0"
language: "en"
```

Run the source validation command:
```bash
free_ebook_maker validate --source ./manuscript/ --meta ./metadata.yaml
```
This checks for missing files, invalid ISBN checksums, and broken internal links. Exit code `0` means ready to build.

### Step 2: Configure the Render Pipeline
Create a `build.config.json` file to override defaults:

```json
{
  "output_format": "epub",
  "subset_fonts": "auto",
  "pdf_engine": "xelatex",
  "resolve_refs": true,
  "cover_image": "./assets/cover.png"
}
```

Execute the build:
```bash
free_ebook_maker build --config ./build.config.json --output ./dist/
```

The engine will emit progress logs to stderr (e.g., `[AST] Parsing 47 files... [OK]`, `[Font] Subsetting 'SourceSerif4'... 62% size reduction`). On completion, the final `.epub` file is written to `./dist/`.

### Step 3: Post-Processing and Verification
Run the integrity check suite to validate the output:

```bash
free_ebook_maker verify --input ./dist/advanced-python-architecture.epub
```

This performs the following checks:
- EPUB 3.2 spec compliance (via `epubcheck` wrapper)
- All internal hyperlinks resolve to existing IDs
- Image assets exist and are not corrupted (MD5 checksum comparison)
- Font files are properly subsetted and embedded

If any check fails, the tool returns a non-zero exit code and writes a detailed `verification-report.json` to the output directory.

## Best Practices

### Practice 1: Always Use a Dedicated Metadata File
Do not embed metadata in the Markdown frontmatter for multi-chapter books. Instead, use a separate `metadata.yaml` file. This allows CI/CD systems to inject dynamic values (e.g., version number, build date) without touching the source manuscript. It also prevents accidental metadata corruption during collaborative editing.

### Practice 2: Enable Font Subsetting for Distribution
When publishing to platforms with strict file size limits (e.g., Amazon KDP’s 650 MB cap for Kindle), always set `--subset-fonts` to `auto`. This reduces the eBook’s download size dramatically while preserving typographic quality. For print-on-demand PDF output, disable subsetting (`--subset-fonts none`) to ensure the printer’s RIP has access to the full character set.
