# free_ebook_maker - FRE_007

## Overview

This is a high-performance, pipeline-based EPUB3/MOBI/PDF generation engine engineered for automated book production at scale. It processes raw Markdown, LaTeX, or AsciiDoc sources into polished, reflowable ebooks with millisecond-level rendering logic for complex layouts. The system is designed for CI/CD integration, supporting multi-format export with deterministic output hashes for version control.

## Detailed Content

### What is free_ebook_maker?

free_ebook_maker is a zero-dependency, Rust-core CLI tool that compiles structured content into industry-standard ebook formats. It implements a custom AST-to-OPF transformer, enabling direct manipulation of spine order, metadata injection, and CSS-in-JS style scoping without external build tools. It targets publishers, documentation teams, and indie authors requiring reproducible builds with sub-100ms compilation overhead per 100KB of source text.

### Core Features

- **Microsecond-Level Table of Contents Auto-Generation**: Parses heading hierarchies (H1–H6) from source files and generates a nested NCX (Navigation Control File for XML) with granularity down to 500-character anchor points. Supports `--toc-depth` parameter (default: 3, max: 6) and `--toc-max-entries` (default: 1000). The algorithm runs in O(n log n) time, with measured latency of 2.3ms for a 500-page manuscript.

- **Multi-Format Output with Deterministic Checksums**: Exports to EPUB3 (with `epubcheck`-validated compliance), MOBI (Kindle KF8), and PDF (via WeasyPrint-compatible CSS). Each output file includes a SHA-256 hash embedded in the metadata block (`dc:identifier`). The `--format` flag accepts comma-separated values (e.g., `epub,mobi,pdf`), and the `--checksum` flag (default: `true`) ensures byte-level reproducibility across builds.

- **Pluggable Preprocessor Pipeline with AST Hooks**: Integrates with external scripts via stdin/stdout streams using a JSON-based intermediate representation (IR). Supports `--preprocessor` flag for chaining custom filters (e.g., LaTeX-to-MathML conversion, image optimization with `pngquant`). The pipeline timeout is configurable via `--preprocessor-timeout` (default: 30 seconds, in milliseconds). Postgres-compatible logging via `--log-dsn` for audit trails.

### How to Use

1. **Initialize a Project Skeleton**: Run `free_ebook_maker init --project-name "MyBook" --source-dir ./src --output-dir ./dist`. This generates a `book.json` manifest with default parameters: `--language en`, `--author "Unknown"`, `--cover-image "cover.png"` (optional). The manifest includes a `--version` field (semver, default `1.0.0`) and a `--spine-order` array (auto-populated from `src/` file listing). Validate the structure with `free_ebook_maker validate --manifest ./book.json`, which checks for missing `dc:identifier` and broken image references.

2. **Compile with Custom CSS and Metadata Override**: Execute `free_ebook_maker build --manifest ./book.json --css ./style.css --metadata '{"publisher": "O\'Reilly Media", "isbn": "978-1-491-95723-5"}' --format epub,mobi --output-version`. The `--output-version` flag appends a timestamp (UTC, ISO 8601) to the output filename. For large projects (>1000 pages), add `--parallel --workers 4` to parallelize chapter rendering (uses `rayon` under the hood). Monitor progress via `--verbose` (logs to stderr at `info` level).

3. **Integrate into a CI/CD Pipeline**: Add to `.github/workflows/build.yml` with the following steps: (a) `free_ebook_maker validate --manifest ./book.json --strict` (fails on warnings), (b) `free_ebook_maker build --manifest ./book.json --format pdf --preprocessor "python3 ./scripts/optimize_images.py"`, (c) `free_ebook_maker checksum --manifest ./book.json --algorithm sha256 --output ./checksums.txt`. Use `--dry-run` flag to simulate without writing files. The tool exits with code 0 on success, 1 on validation errors, and 2 on runtime failures.

### Best Practices

- **Implement a Source-to-Output Hash Chain**: Always run `free_ebook_maker checksum` after each build and store the hash in a `checksums.txt` file committed to your repository. This enables deterministic debugging: if a reader reports a rendering bug, you can pin the exact source version that produced the faulty ebook. For critical publications (e.g., legal documents), combine with `--checksum-embed` to inject the hash into the ebook metadata, allowing post-hoc verification via `free_ebook_maker verify --input ./output.epub --manifest ./book.json`.

- **Use a Preprocessor for Asset Optimization Before Compilation**: Before the build step, run a custom preprocessor that compresses images to WebP (with fallback to JPEG for older readers) and converts SVG to embedded PNG at 300 DPI. This reduces the final ebook size by 40–60% and ensures compatibility with Amazon’s Kindle Direct Publishing (KDP) requirements. Set `--preprocessor-timeout 60000` for heavy assets. For team workflows, store the preprocessor script in a `scripts/` directory and reference it in `book.json` under the `"preprocessor"` key.
