# free_ebook_maker - FRE_013

## Overview

`free_ebook_maker` is an open-source, high-performance ebook generation engine designed for automated, batch, and multi-format publishing pipelines. It eliminates manual layout and conversion bottlenecks by integrating real-time Markdown-to-EPUB/MOBI/PDF rendering with granular CSS control, metadata injection, and image optimization at the sub‑second level.

## Core Features

### 1. Multi‑Format Parallel Rendering Engine with Sub‑Millisecond Compilation
- **Parameter**: `--formats epub,mobi,pdf,azw3`  
- **Technical Specification**:  
  - Utilizes a custom `libebook` core based on `pandoc` + `weasyprint`, achieving **≤ 0.8 ms per page** for EPUB generation on a 4‑core AMD EPYC.  
  - Supports **simultaneous output to 4 formats** without sequential locking, reducing full‑book generation time by **73%** compared to Calibre CLI.  
  - Built‑in **CSS‑level hyphenation** and **widow/orphan control** (configurable via `--widow-lines 2 --orphan-lines 3`).

### 2. Automated Metadata & Table of Contents (TOC) Injection with Schema Validation
- **Parameter**: `--meta metadata.yaml --toc-depth 3`  
- **Technical Specification**:  
  - Parses YAML‑based metadata (title, author, ISBN, language, cover image path) and injects it into EPUB `content.opf` and PDF XMP metadata.  
  - Generates a **fully clickable, nested TOC** using `nav.xhtml` for EPUB and `\tableofcontents` for PDF, with **automatic heading‑level detection** (h1–h6).  
  - Validates ISBN‑13 checksum, language codes (ISO 639‑1), and cover image dimensions (minimum **1600×2560 px** for Kindle).

### 3. Intelligent Image Optimization & Lazy‑Loading Pipeline
- **Parameter**: `--image-quality 85 --max-width 1200 --lazy-load`  
- **Technical Specification**:  
  - Converts all embedded images to **WebP** (lossy) with configurable quality (0–100), reducing average file size by **62%** compared to original PNG/JPEG.  
  - Automatically resizes images to a **maximum width of 1200 px** for MOBI and 1600 px for PDF, preserving aspect ratio.  
  - Inserts `loading="lazy"` attributes into HTML for EPUB/MOBI readers, improving initial load time by **40%** on low‑end e‑ink devices.

## How to Use

### Step 1: Prepare Your Markdown Source & Metadata File
Create a clean Markdown file (`manuscript.md`) with standard heading hierarchy (h1 for title, h2 for chapters).  
Define metadata in `metadata.yaml`:
```yaml
title: "The Art of Distributed Systems"
author: "Jane Doe"
isbn: "978-1-234-56789-0"
language: en
cover: "cover.png"
publisher: "O'Reilly Media"
```
Run validation:
```bash
free_ebook_maker validate metadata.yaml
```
Expected output: `✅ ISBN valid | Cover dimensions OK (1600×2560) | Language code 'en' recognized`

### Step 2: Execute Multi‑Format Generation with Custom CSS
Apply a custom stylesheet (`style.css`) with advanced typography rules:
```css
body { font-family: 'Georgia', serif; line-height: 1.8; }
code { font-size: 0.9em; background: #f4f4f4; }
```
Generate all formats in one command:
```bash
free_ebook_maker build manuscript.md \
  --formats epub,mobi,pdf \
  --meta metadata.yaml \
  --css style.css \
  --toc-depth 3 \
  --image-quality 85 \
  --output ./output/
```
Logs will show per‑file timestamps and compression ratios:
```
[14:32:01] EPUB: 2.3 MB → 1.1 MB (52% reduction) | 245 pages | 0.9s
[14:32:02] MOBI: 2.1 MB → 1.0 MB (52% reduction) | 245 pages | 1.2s
[14:32:03] PDF: 4.7 MB → 3.2 MB (32% reduction) | 248 pages | 2.1s
```

### Step 3: Verify Output Integrity & Generate Hash
Check that all internal links, TOC entries, and images resolve correctly:
```bash
free_ebook_maker verify output/manuscript.epub --check-links --check-images
```
Generate SHA‑256 checksums for distribution:
```bash
sha256sum output/*.epub output/*.mobi output/*.pdf > checksums.txt
```

## Best Practices

### Practice 1: Pre‑split Large Manuscripts for Parallel Compilation
For books exceeding **500 pages**, split the Markdown into chapter‑level files (`ch01.md`, `ch02.md`, …). Use the `--input-dir` flag to process them in parallel across multiple CPU cores. This reduces full‑book build time from **linear O(n)** to **near‑constant O(1)** per chapter, with overall speedup proportional to core count.

### Practice 2: Enforce a Strict CSS Grid for PDF Layouts
When generating PDFs for print‑on‑demand (e.g., Amazon KDP), define a **fixed CSS grid** with `@page` rules:
```css
@page { size: 6in 9in; margin: 0.75in 1in; }
```
Use `--pdf-page-numbers` and `--pdf-bleed 0.125in` to ensure proper trim marks. Validate PDF output with `pdfinfo` to confirm page dimensions match the target trim size (e.g., `6.0 x 9.0 in`).
