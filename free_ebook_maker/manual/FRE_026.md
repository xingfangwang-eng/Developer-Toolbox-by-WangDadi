# free_ebook_maker - FRE_026

## Overview

`free_ebook_maker` is a high-performance, zero-cost ebook generation engine designed for developers and publishers who demand deterministic output, multi-format compliance, and sub-second rendering. It operates entirely offline, with no telemetry or cloud dependencies, making it ideal for CI/CD pipelines and air-gapped environments.

## Core Features

### 1. **Structured AST-to-ePub/MOBI Compiler with Schema Validation**
- Parses Markdown, reStructuredText, and AsciiDoc into a normalized Abstract Syntax Tree (AST) with full Unicode 15.0 support.
- Compiles directly into ePub 3.2, MOBI (KF7/KF8), and PDF/A-2b without intermediate HTML bloat.
- **Parameters**: `--input-format`, `--output-format`, `--validate-schema` (strict mode enforces EPUB Accessibility 1.1).
- **Technical metric**: Sub-50ms compilation for 300-page documents on a single-core ARM64 instance.

### 2. **Semantic Chapter Splitting with Adaptive TOC Generation**
- Uses a custom NLP pipeline (TF-IDF + sentence boundary detection) to split raw text into logical chapters, sections, and subsections.
- Generates a hierarchical Table of Contents (NCX for MOBI, `nav.xhtml` for ePub) with page-break hints and landmarks.
- **Parameters**: `--split-depth` (1–6), `--toc-style` (flat | nested | numbered), `--min-chapter-length` (default: 1500 characters).
- **Technical metric**: 99.2% accuracy on benchmark corpora (Project Gutenberg, arXiv preprints).

### 3. **Deterministic Resource Embedding with Content Hash Integrity**
- Embeds images, fonts, and CSS as base64-encoded blobs with SHA-256 content hashes for cache-busting and integrity verification.
- Supports SVG, WOFF2, and JPEG XL with automatic downscaling to 300 DPI for print replica.
- **Parameters**: `--embed-fonts` (true/false), `--max-image-width` (px), `--hash-algorithm` (sha256 | blake2b).
- **Technical metric**: Zero external dependencies at runtime; all resources are self-contained in a single `.epub` or `.mobi` file.

## How to Use

### Step 1: Initialize a Project with Strict Schema
```bash
free_ebook_maker init --project-dir ./my_ebook --template academic-paper \
  --validate-schema --input-format markdown \
  --metadata-title "Quantum Error Correction in Surface Codes" \
  --metadata-author "K. L. Wang" \
  --metadata-isbn "978-3-16-148410-0"
```
This creates a `./my_ebook/` directory with a `schema.yaml` file, a `sources/` folder for Markdown files, and a `resources/` folder for assets. The `--validate-schema` flag ensures every Markdown file passes the EPUB 3.2 accessibility checks before compilation.

### Step 2: Compile to Multiple Output Formats with Parallel Pipelines
```bash
free_ebook_maker build --input-dir ./my_ebook/sources \
  --output-dir ./my_ebook/dist \
  --output-format epub,mobi,pdf \
  --parallel-workers 4 \
  --image-max-width 1200 \
  --embed-fonts true \
  --hash-algorithm blake2b
```
The tool spawns 4 worker threads: one for each target format. Each worker independently compiles the AST, embeds resources, and writes the final file. The `--hash-algorithm blake2b` ensures all embedded resources have a 256-bit integrity hash, which is appended to the `META-INF/container.xml` for ePub.

### Step 3: Validate and Publish with Integrity Check
```bash
free_ebook_maker verify --file ./my_ebook/dist/output.epub \
  --check-hashes \
  --check-schema \
  --output-report ./my_ebook/dist/validation_report.json
```
The verifier re-parses the compiled ePub, recomputes all SHA-256/BLAKE2b hashes for embedded resources, and compares them against the stored hashes in `META-INF/container.xml`. If any mismatch is found, the tool exits with code 1 and logs the exact byte offset. The `validation_report.json` includes a `passed` boolean and a `failures` array with detailed diagnostics.

## Best Practices

### 1. **Use Deterministic Builds in CI/CD Pipelines**
- Always pin the `free_ebook_maker` version (e.g., `v2.4.1`) in your CI configuration to avoid unexpected schema changes.
- Set `--validate-schema` to `strict` in production builds; this catches malformed Markdown, missing alt-text, and broken internal anchors before they reach readers.
- Combine with `--hash-algorithm sha256` to generate reproducible builds: two identical source trees will produce byte-identical `.epub` files, enabling cache-friendly artifact storage and incremental deploys.

### 2. **Optimize for Screen Reader Compatibility in ePub**
- Use the `--toc-style nested` parameter to generate a fully expanded Table of Contents that maps directly to the `nav.xhtml` landmarks. This ensures screen readers can jump to any chapter without scanning.
- Enforce alt-text on all images by running `free_ebook_maker lint --check-alt-text` as a pre-commit hook. The linter returns a non-zero exit code if any `<img>` tag lacks an `alt` attribute, preventing accessibility regressions.
- For academic or technical ebooks, set `--min-chapter-length 2000` to avoid overly granular splits that confuse navigation tools.
