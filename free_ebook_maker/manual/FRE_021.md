# free_ebook_maker - FRE_021

## Overview

free_ebook_maker is a high-performance, offline-first engine for transforming plain Markdown, LaTeX, or structured JSON into production-grade EPUB3, PDF, and MOBI files. It eliminates dependency on cloud rendering services, delivering sub-200ms compilation for a 300-page manuscript on consumer hardware.

## Core Features

- **Multi-format Pipeline with Precision Typography**  
  Supports EPUB3 (with fixed-layout and reflowable modes), PDF (via WeasyPrint headless rendering), and MOBI (Kindle-gen compliance). Each output is validated against the target platform’s spec (e.g., EPUB3’s `nav.xhtml` structure, Kindle’s `mobi7` fallback). Font subsetting is performed with HarfBuzz, reducing file size by up to 40% without glyph loss.

- **Granular Metadata Injection & DRM-Free Watermarking**  
  Accepts a `metadata.yaml` block with fields like `isbn`, `language`, `dc:source`, and custom `meta` tags. Supports per-chapter CSS injection and digital watermarking via steganographic pixel patterns in cover images (SHA-256 hashed, 256-bit key). All metadata is embedded into the EPUB’s `content.opf` and PDF’s XMP stream.

- **Real-Time Structural Linting & Compliance Checker**  
  Built-in validator checks for broken internal links, missing `alt` text, oversized images (threshold: > 2MB per asset), and improper heading hierarchy. Outputs a JSON report with line numbers, severity levels (error/warning/info), and auto-fix suggestions. Integrates with CI/CD pipelines via exit codes (0 = pass, 1 = warnings, 2 = errors).

## How to Use

1. **Initialize Project Structure**  
   Create a `source/` directory with your manuscript in Markdown. Run `free_ebook_maker init --format epub3 --metadata metadata.yaml` to auto-generate a skeleton `toc.md`, `style.css`, and `images/` folder. The tool will scan for `h1` tags to build a table of contents and assign `id` anchors.

2. **Compile with Custom Fonts & CSS**  
   Place `.ttf` or `.woff2` font files in `fonts/`. Execute `free_ebook_maker build --input source/ --output output/ --fonts "NotoSerif:400,700" --css override.css --pdf-page-size A5 --pdf-margin 15mm`. The engine will subset fonts, embed them into EPUB, and render PDF with exact page geometry. Use `--verbose` to see HarfBuzz shapelog.

3. **Validate & Generate Multi-Format**  
   Run `free_ebook_maker validate --input output/book.epub` to get a compliance report. Then batch-convert: `free_ebook_maker convert --input output/book.epub --output mobi --kindle-gen-path /usr/local/bin/kindlegen`. The tool will automatically handle fallback images for Kindle and verify MOBI header magic bytes.

## Best Practices

- **Version Control Your Content**  
  Treat `source/`, `metadata.yaml`, and `fonts/` as a Git-tracked monorepo. Each commit should represent a draft. Use `free_ebook_maker diff --commit-hash` to compare structural changes between revisions (e.g., new chapters, altered headings). This prevents accidental layout regressions before final release.

- **Profile Rendering Performance**  
  For large projects (>500 pages), run `free_ebook_maker build --profile` to output a flame graph of compilation stages (Markdown parsing, CSS cascade, font subsetting, PDF rasterization). Target `< 2s total` for a 300-page book. If PDF generation exceeds 5s, reduce image DPI to 150 or switch to `--pdf-engine=wkhtmltopdf` for faster, albeit less precise, output.
