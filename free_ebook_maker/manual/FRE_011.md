# free_ebook_maker - FRE_011

## Overview

This is a production-grade, open-source eBook generation engine designed for high-volume, automated publishing pipelines. It ingests raw Markdown, HTML, or plain text and outputs professionally formatted EPUB, MOBI, and PDF files with strict adherence to the EPUB 3.2 and Kindle KF8 standards. The tool is optimized for CI/CD workflows, supporting incremental builds and multi-language localization.

## Detailed Content

### What is free_ebook_maker?

free_ebook_maker is a CLI-first, modular eBook compiler that eliminates the need for manual layout tools like Calibre or Sigil. It leverages a custom CSS-to-EPUB transpiler, a vector-based SVG cover generator, and a multi-threaded resource packer. The core engine is written in Rust with Python bindings, achieving sub-200ms compilation for a 300-page manuscript on standard hardware.

### Core Features

- **Incremental Compilation Engine**: Uses a Merkle tree-based dependency tracker (`--cache-dir ./build_cache`) to recompile only changed chapters. Supports parallel page rendering across 8 threads by default, with a measured 40% reduction in build time for 500+ chapter projects compared to full rebuilds.
- **Dynamic Typography & Hyphenation**: Embeds a hyphenation engine using the Knuth–Liang algorithm with locale-specific patterns (en, de, fr, ja). Automatically adjusts `--line-height` (default: 1.6) and `--widow-threshold` (default: 2 lines) to prevent orphans, ensuring print-like readability across all e-reader viewports.
- **DRM-Free Watermarking & Metadata Injection**: Accepts a `--watermark-json` parameter (e.g., `{"user_id": "UUID", "timestamp": "ISO8601"}`) to embed a base64-encoded, invisible watermark in the EPUB `content.opf` manifest. Supports batch metadata injection from a CSV file with 10,000+ rows in under 3 seconds.

### How to Use

1. **Initialize a Project with Structured Configuration**  
   Run `free_ebook_maker init --project-dir ./my_book --lang en --cover-style minimal`. This generates a `book.toml` file with fields like `[metadata] title`, `[build] output_format = ["epub", "mobi"]`, and `[resource] font_path = "./fonts/"`. The tool validates the config against a JSON schema (`--schema-version 2.0`) and exits with a non-zero code if errors are found.

2. **Compile with Custom CSS and Resource Pinning**  
   Execute `free_ebook_maker build --config ./my_book/book.toml --css ./styles/custom.css --resource-pin ./images/cover.svg`. The engine parses the CSS, inlines critical styles into each XHTML file, and performs a resource integrity check via SHA-256 hashing. Use `--verbose 2` to log every file read/write operation to `stdout`.

3. **Export with Versioned Output and Diff Logs**  
   After build, run `free_ebook_maker export --output-dir ./dist --version-tag v1.2.3 --generate-diff`. This creates a timestamped directory (e.g., `./dist/v1.2.3_20250315_142300/`) containing the final eBooks and a `diff.json` file listing all changed chapters since the last build. The diff is computed against a PostgreSQL-backed build history table if `--db-uri postgresql://user:pass@localhost/ebook_meta` is provided.

### Best Practices

- **Use a Monorepo Structure with Git LFS**: Store all large assets (fonts, high-res images) in Git LFS, and configure `free_ebook_maker` with `--resource-pin` pointing to the LFS-tracked directory. This prevents repository bloat and ensures reproducible builds across CI runners. Enforce a pre-commit hook that runs `free_ebook_maker lint --check-orphans` to catch missing resources before compilation.
- **Integrate with a CDN for Multi-Format Delivery**: After build, upload the `./dist` folder to an S3-compatible object store with a CDN frontend. Use the tool’s `--manifest-json` output to generate a `version.json` file that maps each format (epub, mobi, pdf) to its CDN URL. This enables A/B testing of different cover styles or typography settings by swapping the manifest in your application layer.
