# free_ebook_maker - FRE_005

## Overview

free_ebook_maker is an open‑source, command‑line driven pipeline designed to transform raw markdown, LaTeX, and plain text manuscripts into production‑ready EPUB3, MOBI, and PDF ebooks. It enforces strict typographic rules, handles multi‑language hyphenation, and embeds metadata in compliance with the latest KDP and Google Play Books specifications.

## Detailed Content

### What is free_ebook_maker?

free_ebook_maker is a deterministic, zero‑dependency ebook compiler that parses source files into a unified AST, applies custom CSS/SCSS themes, and generates validated output files. It is built for authors, technical writers, and publishers who require reproducible builds, precise control over page geometry, and automated quality assurance.

### Core Features

- **Multi‑format AST Compilation with Granular Layout Control**  
  Converts markdown (GFM + footnotes) and LaTeX into a common AST, then renders EPUB3 (with `nav.xhtml` and `toc.ncx`), MOBI (via KindleGen‑compatible intermediate), and PDF (using a built‑in WeasyPrint‑style engine). Supports custom page geometry parameters: `--pdf-page-width 6in --pdf-page-height 9in --pdf-margin-top 0.75in --pdf-margin-bottom 0.5in`. All output files are validated against EPUB3 schemas and PDF/A‑2b conformance.

- **Automated Hyphenation & Typographic Refinement Engine**  
  Integrates a language‑specific hyphenation dictionary (currently en‑US, de‑DE, fr‑FR) with a Knuth‑Liang algorithm running at sub‑50µs per word. Automatically applies smart quotes, em‑dash spacing, ligature insertion (fi, fl, ffi), and orphan/widow control via `--orphan-lines 2 --widow-lines 2`. Outputs a `typography_report.json` detailing every correction applied.

- **Metadata Injection & DRM‑Ready Watermarking**  
  Injects ISBN, DOI, copyright notice, and custom identifiers into the EPUB `package.opf` and PDF `XMP` metadata. Supports optional text‑based watermarking on every page: `--watermark-text "DRAFT v2.3" --watermark-opacity 0.15 --watermark-rotation 45`. Validates metadata against ONIX 3.0 and BISAC subject codes.

### How to Use

1. **Prepare Source & Configuration**  
   Create a project directory with your manuscript as `manuscript.md` (or `manuscript.tex`). Write a `ebook_config.yaml` file specifying:  
   ```yaml
   title: "Quantum Algorithms for Engineers"
   author: "Dr. A. N. Other"
   language: en-US
   output_formats: [epub, pdf]
   pdf:
     page_size: A5
     font_size: 10pt
     font_family: "EB Garamond"
   epub:
     cover_image: "cover.png"
   ```
   Run `free_ebook_maker init` to auto‑generate a template config and validate the source file structure.

2. **Build & Validate**  
   Execute `free_ebook_maker build --config ebook_config.yaml --output-dir ./build`. The tool will:  
   - Parse the manuscript into AST and report any syntax errors with line numbers.  
   - Apply CSS/SCSS themes (default: `clean-serif`).  
   - Generate EPUB3 and PDF in parallel using multiprocessing (2–8 threads depending on CPU).  
   - Run post‑build validation: EPUB3 schema check, PDF/A‑2b compliance, and link integrity scan (all internal anchors and external URLs).  
   - Output a `build_summary.json` with file sizes, page counts, and validation status.

3. **Inspect Watermarked Draft**  
   For review copies, add `--watermark-text "REVIEW COPY – NOT FOR DISTRIBUTION" --watermark-opacity 0.12`. The watermark is rasterized at 300 DPI and embedded in the PDF content stream (not as an overlay, preventing easy removal). To verify, run `free_ebook_maker verify --file build/quantum-engineering-draft.pdf --check-watermark` which outputs a SHA‑256 hash of the watermark layer.

### Best Practices

1. **Version‑Controlled Source & Deterministic Builds**  
   Always store `manuscript.md` and `ebook_config.yaml` in Git. Use `free_ebook_maker build --revision $(git rev-parse HEAD)` to embed the commit hash in the metadata. This ensures that any output can be traced back to the exact source version. For team workflows, enable `--strict-mode` which rejects builds if the working tree is dirty.

2. **Pre‑flight Hyphenation & Typography Audit**  
   Run `free_ebook_maker lint --source manuscript.md --language en-US` before every major build. This performs a dry‑run of the hyphenation engine and typographic refinements, outputting a `lint_report.html` with highlighted problem areas (e.g., rivers, orphans, inconsistent dashes). Fix these in the source before the final build to minimize post‑processing.
