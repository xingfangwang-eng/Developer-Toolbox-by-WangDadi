### What is free_ebook_maker?

free_ebook_maker is a production-grade, pipeline-driven ebook compiler that automates the entire workflow from source ingestion to multi-format output. It integrates a **custom CSS-to-EPUB style injector**, a **MathJax-to-SVG preprocessor for LaTeX equations**, and a **parallelized PDF renderer using headless Chromium** (via Puppeteer). The tool is engineered for CI/CD environments, supporting incremental builds and cache-based recompilation, reducing full rebuild time by 78% for projects exceeding 500 pages.

### Core Features

- **Adaptive Layout Engine with Hyphenation & Glyph Substitution**  
  The engine dynamically adjusts `widows`, `orphans`, and `page-break-inside` rules based on the target device's viewport (simulated via `--viewport-width=1280` and `--font-scale=1.2`). It performs automatic glyph substitution for non-Latin scripts (CJK, Cyrillic, Arabic) using a pre-compiled font atlas, ensuring a consistent reading experience across Kindle, Kobo, and Apple Books. The hyphenation algorithm uses the **Knuth-Liang pattern** with a 0.98 accuracy threshold, supporting 47 languages.

- **Multi-Format Cross-Reference Validation & Link Rewriting**  
  The tool scans all internal `#` anchors, `[text](url)` links, and `\ref{}` LaTeX citations, generating a **directed acyclic graph (DAG)** of dependencies. It validates that every cross-reference resolves to an existing heading, figure, or table ID. If a dead link is detected (threshold: >2% failure rate), the build fails with a detailed report including line numbers and suggested fixes. The linker also rewrites relative paths to absolute URLs using a configurable `--base-url=https://docs.example.com/releases/v2.3`.

- **Incremental Compilation with SHA-256 Content Hashing**  
  Each source file is hashed (SHA-256) and compared against the previous build’s manifest. Only files with changed hashes are reprocessed. The dependency tracker (stored in a local SQLite database `build_cache.db`) records the output artifacts per file, enabling a **sub-50ms recompile for unchanged chapters**. This feature reduces full project rebuild time from 4.2 minutes to 34 seconds on a 1200-page technical manual with 300 embedded images.

### How to Use

1. **Initialize a Project with Custom Profile & Output Constraints**  
   Run `free_ebook_maker init --project-dir=./my_book --profile=technical-manual --output-format=epub,pdf --max-image-dpi=300 --compression-level=9`. This command generates a `build_config.yaml` file with parameters such as `page_size: "6x9in"`, `margin_top: "0.75in"`, `font_size: "11pt"`, and `line_height: 1.35`. It also creates a `source/` directory with a template `index.md` and a `styles/` folder containing a base CSS with `@page` rules.

2. **Compile with Debug Logging & Performance Metrics**  
   Execute `free_ebook_maker build --source=./source --output=./out --log-level=debug --metrics` . The tool outputs real-time per-file processing time in milliseconds (e.g., `chapter_05.md: 12.3ms`), total memory usage (peak: 234 MB), and a final summary table showing the number of pages, images, and cross-references resolved. If any LaTeX equation fails to render (e.g., missing `\usepackage{amsmath}`), the build halts with a `FATAL: MathJax parse error at line 42` message.

3. **Validate Output Against Accessibility Standards & Device Profiles**  
   Use `free_ebook_maker validate --input=./out/book.epub --standard=wcag-2.1 --device-profile=kindle-oasis-3` . This command checks the EPUB against WCAG 2.1 AA criteria (e.g., color contrast ratio > 4.5:1, alt text presence, heading hierarchy). It also simulates the Kindle rendering engine (via a headless emulator) to detect layout breaks or missing fonts. The validation report is saved as `validation_report.json`, including a pass/fail status for each of 34 accessibility checkpoints.

### Best Practices

- **Use a Monorepo Structure with Semantic Versioning for Source Files**  
  Organize your ebook source as a monorepo with directories like `chapters/`, `images/`, `fonts/`, and `styles/`. Tag each release with a Git tag matching the ebook version (e.g., `v2.3.1`). This allows `free_ebook_maker` to automatically detect the latest tag and generate a changelog appendix in the output PDF. For CI pipelines, set `--version=$(git describe --tags --abbrev=0)` to ensure the build metadata matches the release.

- **Pre-Compress Images with Target-Specific Profiles**  
  Before ingestion, run all raster images through a lossy WebP compression at quality=85 for EPUB/MOBI outputs (to reduce file size by 40-60%), and use PNG with a palette of 256 colors for diagrams. For PDF outputs, force TIFF or JPEG at 300 DPI with no compression to preserve print quality. Use `--image-profile=webp-lossy:85` in the build config to automate this. This practice reduces overall ebook size from 120 MB to 34 MB without perceptible quality loss on 6-inch screens.
