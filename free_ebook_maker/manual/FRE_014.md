# free_ebook_maker - FRE_014

## Overview

This is the production-grade architecture for the `free_ebook_maker` project, designed to convert raw Markdown, LaTeX, and plain text into professionally formatted EPUB, MOBI, and PDF ebooks. The system is optimized for batch processing, semantic chapter splitting, and metadata injection, targeting a latency of under 200 milliseconds per 50-page document on a standard dual-core CPU.

## Detailed Content

### What is free_ebook_maker?

free_ebook_maker is a modular, pipeline-driven ebook compiler that ingests structured or unstructured text, applies a customizable CSS/LaTeX template engine, and outputs a standards-compliant ebook with embedded TOC, cross-references, and DRM-free protection. It supports real-time preview via a WebSocket server and integrates with PostgreSQL for versioned asset storage.

### Core Features

- **Semantic Chapter Smuggling & Auto-TOC Generator**: Leverages a custom NLP tokenizer (WordPiece-based) to detect chapter boundaries even in malformed Markdown (missing `#` or `##`). Generates a hierarchical table of contents with up to 5 levels of nesting. Supports `--toc-depth=5` flag. Outputs a valid NCX (for EPUB 2) or `nav.xhtml` (for EPUB 3) in under 50 ms for 200 chapters.

- **Multi-Format Renderer with LaTeX Equation Rasterization**: Embeds a headless MathJax 3.2.2 instance that renders inline and block equations as SVG with a DPI of 300. Caches rasterized SVGs in a Redis-backed LRU cache (max 10,000 entries) to avoid re-rendering. The `--math-engine=mathjax` parameter also supports fallback to KaTeX (v0.16.9) for faster processing when equations are simple. Outputs vector-quality PDF with `pdfTeX` via a Docker sandbox.

- **Asset Deduplication & Compression Engine**: Automatically detects duplicate images (using perceptual hashing via pHash, Hamming distance < 8) across the entire source tree. Replaces duplicates with symlinks inside the EPUB archive, reducing final file size by up to 40%. Supports `--image-quality=85` (JPEG) and `--compress-level=9` (PNG). Employs MozJPEG for lossy compression and oxipng for lossless, with a configurable timeout of 500 ms per image.

### How to Use

1. **Initialize the Project with a Custom Template and Database Backend**  
   Run `free_ebook_maker init --template=academic --db=postgresql://user:pass@localhost:5432/ebooks --assets-dir=./raw`. This command creates a `config.yaml` file with 32 parameters, including `--toc-depth`, `--math-engine`, and `--output-format=epub,pdf`. It also sets up a PostgreSQL schema with tables for `chapters`, `assets`, and `metadata`, pre-seeded with a UUID primary key for each row.

2. **Compile a Multi-Chapter Document with Cross-References and Equation Support**  
   Execute `free_ebook_maker build --source=./book.md --output=./output --format=epub --toc-depth=5 --math-engine=mathjax --image-quality=85 --compress-level=9 --deduplicate=true`. The pipeline will:  
   - Tokenize the Markdown into 200+ chapters using the NLP tokenizer (accuracy > 99.2% on test corpus).  
   - Render all LaTeX equations as 300 DPI SVGs, caching them in Redis.  
   - Deduplicate images using pHash (Hamming distance threshold = 8).  
   - Assemble the EPUB with a valid `content.opf`, `toc.ncx`, and `nav.xhtml`.  
   - Output a final file at `./output/book.epub` with a SHA-256 checksum.

3. **Validate and Inspect the Output Ebook for Standards Compliance**  
   Run `free_ebook_maker validate --input=./output/book.epub --check-format=epub3 --strict=true`. The validator checks:  
   - EPUB 3 structural validity (e.g., `META-INF/container.xml` exists, `package.opf` has correct `version="3.0"`).  
   - All internal links resolve (no dead cross-references).  
   - Image dimensions are within 1200x1600 pixels (configurable via `--max-image-dimensions`).  
   - No duplicate `manifest` IDs. Reports a pass/fail with detailed logs in `./output/validation.log`.

### Best Practices

- **Use a Git-Based Workflow for Source Files**: Store your Markdown, LaTeX, and image assets in a Git repository with a `.gitignore` that excludes the `output/` and `cache/` directories. Run `free_ebook_maker build` as a pre-commit hook to automatically generate a fresh EPUB on every commit. This ensures traceability and allows rollback to any previous build version. For large projects (>500 chapters), use `--parallel=4` to spawn 4 workers for parallel chapter processing, reducing build time by 60%.

- **Leverage the PostgreSQL Backend for Multi-User Collaboration**: Configure `free_ebook_maker` with a shared PostgreSQL database (e.g., `--db=postgresql://user:pass@cluster.aws.com:5432/ebooks`). Each user can run `free_ebook_maker build` independently; the system uses row-level locking on the `chapters` table to prevent concurrent writes. For team editing, use `--merge-strategy=last-writer-wins` with a timestamp column. Regularly run `VACUUM ANALYZE` on the `assets` table to maintain query performance on the deduplication hash index.
