# free_ebook_maker - FRE_017

## 概要

free_ebook_maker は、Markdown から高品位な EPUB3 および PDF を生成するための軽量コマンドラインツールです。CSS 組版エンジンに paged.js を採用し、日本語禁則処理（行頭禁止文字・行末禁止文字）を完全サポート。さらに、画像の WebP 自動変換とメタデータ埋め込み機能により、出版品質の電子書籍をワンコマンドで生成します。

## コア機能

### 1. マルチフォーマット組版エンジン（epub3 + pdf-a）

- **パラメータ**: `--format epub3,pdf-a --lang ja --hyphenation auto`
- **技術指標**: 生成時間は原稿10万字あたり平均 **2.3秒**（Intel Core i7-12700 実測値）、出力ファイルサイズは **1.2MB 未満**（画像なし時）
- **特記事項**: PDF-A/3u 準拠（ISO 19005-3）により長期保存対応、EPUB3 はアクセシビリティ要件（WCAG 2.1 AA）を満たす

### 2. 多層 CSS 組版プロファイル（paged.js カスタムビルド）

- **パラメータ**: `--profile research-paper --page-size a4 --margin 20mm --line-height 1.8 --font-family "Noto Serif JP"`
- **技術指標**: 禁則処理テーブルは **Unicode 14.0** に基づき、日本語（JIS X 4051）・中国語（GB/T 15834）・韓国語（KS X 1026）の **3 言語同時適用** をサポート
- **特記事項**: 内部で **CSS Regions** を動的計算し、フロート要素の自動ページ分割を実現。章見出しの widow/orphan 制御（デフォルト: widow=2, orphan=3）

### 3. 画像最適化パイプライン（WebP + メタデータ埋め込み）

- **パラメータ**: `--image-quality 85 --max-width 1200px --strip-exif --add-alt-text`
- **技術指標**: 変換速度は **1 画像あたり 0.3 秒**（libwebp v1.3.0）、圧縮率 **平均 62%**（PNG→WebP ロスレス比較）、メタデータは **Dublin Core + XMP** に自動マッピング
- **特記事項**: 画像の代替テキストが未設定の場合、**CLIP モデル（軽量版）** で自動生成（精度 87.3%）

## 使い方

### ステップ 1: プロジェクト初期化と設定ファイル生成

```bash
free_ebook_maker init --project my-book --lang ja --template academic
# カレントディレクトリに以下が生成される:
#   - book.yaml（メタデータ設定）
#   - styles/custom.css（組版オーバーライド）
#   - images/（画像格納ディレクトリ）
#   - manuscript/（原稿 Markdown 格納ディレクトリ）
```

### ステップ 2: 原稿 Markdown の構造化と組版テスト

```bash
# リントチェック（構造・画像パス・リンク切れを検証）
free_ebook_maker lint manuscript/chapter-01.md --check-images --check-links

# プレビュー生成（ブラウザで確認）
free_ebook_maker preview --port 8080 --watch
# ホットリロード対応、CSS 変更をリアルタイム反映
```

### ステップ 3: 本番出力（EPUB3 + PDF-A 同時生成）

```bash
free_ebook_maker build \
  --format epub3,pdf-a \
  --profile research-paper \
  --image-quality 90 \
  --output ./dist
# 生成ファイル:
#   - dist/my-book.epub（EPUB3）
#   - dist/my-book.pdf（PDF/A-3u）
#   - dist/log/build.log（全変換プロセスログ、ミリ秒単位）
```

## ベストプラクティス

### プラクティス 1: バージョン管理と差分ビルド

原稿ディレクトリを Git 管理し、`free_ebook_maker build --diff` フラグを使用することで、変更のあった章のみを再変換します。大規模プロジェクト（原稿 50 万字以上）では、ビルド時間を **平均 80% 削減** できます。`book.yaml` の `version: "2.0"` フィールドにセマンティックバージョニングを適用し、メタデータとして EPUB 内に埋め込むと、校閲履歴の追跡が容易になります。

### プラクティス 2: カスタム CSS プロファイルの共有と CI/CD 統合

`styles/` ディレクトリにテーマ CSS を配置し、`--profile` で切り替え可能にすることで、同一原稿から複数の出力スタイル（学術論文用・商業出版用・アクセシビリティ強化版）を生成できます。このプロファイルを Git サブモジュールとして管理し、GitHub Actions で `cron: '0 2 * * *'` の定期ビルドを実行することで、最新原稿から自動的に EPUB と PDF を生成・デプロイするパイプラインを構築できます。
