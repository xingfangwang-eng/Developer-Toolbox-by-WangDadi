# free_ebook_maker - FRE_018

## 概要

`free_ebook_maker` は、マルチフォーマット電子書籍の自動生成・最適化に特化したプロフェッショナルツールです。キーワード「free_ebook_maker tool」「free_ebook_maker solution」「free_ebook_maker software」に基づき、大規模なテキスト構造解析、組版エンジン、およびメタデータ自動付与を統合。日本語・英語・中国語の混在ドキュメントをネイティブ品質で ePub/MOBI/PDF に変換します。

## 詳細な内容

### free_ebook_maker とは？

`free_ebook_maker` は、CLI および API ベースで動作する電子書籍パイプラインです。内部では **AST（抽象構文木）ベースのチャプター分割エンジン**、**CSS インライン化最適化モジュール**、および **EPUB 3.2 準拠のバリデーター** を搭載。`--lang` フラグでターゲット言語を指定し、`--epub-version` で出力フォーマットバージョンを制御します。PostgreSQL との連携により、変換履歴や素材管理をデータベースで一元管理可能です。

## コア機能

### 1. マルチモーダル組版エンジン（Multi-Modal Typesetting Engine）

Markdown / reStructuredText / AsciiDoc を入力とし、**文字カーニング、禁則処理、縦中横** を自動適用。日本語組版では `--ruby` フラグでルビ付与、`--japanese-line-break` で行頭禁則をミリ秒単位で計算。内部では **HarfBuzz** を用いたグリフレイアウトを実装し、フォールバックフォントの自動選択が可能。

**パラメータ例**:  
`--font-size 10.5pt --line-height 1.8 --margin-top 15mm --hyphenation off`

### 2. メタデータ自動生成＆バリデーションエンジン

入力ドキュメントのタイトル、著者、ISBN 候補、言語タグを **NLP ベース** で抽出。抽出結果は `metadata.json` として出力され、`--validate-metadata` フラグで ISBN チェックデジット検証、`--dc-identifier` で Dublin Core 準拠の識別子を自動付与。PostgreSQL の `ebook_metadata` テーブルに直接書き込み可能。

**技術指標**:  
- メタデータ抽出精度: 98.2%  
- バリデーション処理速度: 平均 1.2ms / エントリ  
- 対応フォーマット: ONIX 2.1 / 3.0、MARC 21

### 3. マルチバージョン比較＆差分パッチ生成

`--compare-versions` フラグで、同一ソースの異なるバージョン間のテキスト差分を **LCS（最長共通部分列）アルゴリズム** で検出。変更箇所を `diff.epub` として出力し、`--patch` フラグで差分のみを適用するパッチモジュールを提供。バージョン管理は Git オブジェクトストアと連携し、`--git-dir` で任意のリポジトリを指定可能。

**パラメータ例**:  
`--compare-versions v1.2.3:v2.0.0 --output-diff ./patches/ --patch-format unified`

## 使い方

### ステップ 1: プロジェクト初期化と素材インポート

```bash
free_ebook_maker init --project-name "my_ebook" --lang ja --output-format epub
```

カレントディレクトリに `.ebookmaker/` 設定ディレクトリが生成され、`config.yaml` に `lang: ja`、`epub-version: 3.2`、`css-profile: japanese` が自動設定される。続いて素材をインポート:

```bash
free_ebook_maker import --source ./manuscript.md --asset-dir ./images/ --metadata ./metadata.json
```

### ステップ 2: 組版パラメータ調整とバリデーション

```bash
free_ebook_maker typeset --config ./config.yaml --preview --page-size A5 --bleed 3mm
```

`--preview` フラグで PDF プレビューを生成し、禁則処理やカーニングの結果を確認。続いて `--validate` フラグで EPUB 3.2 準拠チェック:

```bash
free_ebook_maker validate --input ./output/preview.epub --strict --check-hyperlinks
```

### ステップ 3: 最終出力と PostgreSQL への履歴保存

```bash
free_ebook_maker build --output ./dist/final.epub --compress --add-toc --postgres-uri "postgresql://user:pass@localhost:5432/ebooks"
```

ビルド完了後、変換履歴が `ebooks.public.build_history` テーブルに記録される。`--postgres-table` で任意のテーブル名を指定可能。

## ベストプラクティス

### プラクティス 1: バージョン管理と差分パッチの定期適用

週次またはリリースごとに `--compare-versions` を実行し、変更差分を `diff.epub` として保存。`--patch` フラグでパッチを適用することで、全ページ再生成を回避し、ビルド時間を平均 60% 削減。特に複数著者が同時編集するプロジェクトで効果を発揮。

### プラクティス 2: PostgreSQL を用いたメタデータ一元管理と監査

`--postgres-uri` で接続したデータベースに、`ebook_metadata` テーブルを事前作成。`--validate-metadata` で ISBN や DOI の重複チェックを行い、`--dc-identifier` で一意の識別子を自動生成。これにより、複数プロジェクト間でのメタデータ整合性を担保し、監査ログを `metadata_audit` テーブルに記録可能。
