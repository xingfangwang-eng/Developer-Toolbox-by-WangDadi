# free_ebook_maker - FRE_024

## 概要

free_ebook_maker は、EPUB3 / PDF / MOBI の3形式に対応した次世代型 e-book 自動生成エンジンです。特に日本語縦書き組版と数式レンダリングに特化し、LaTeX ソースから直接、出版品質の電子書籍を生成可能。キーワード：free_ebook_maker tool, free_ebook_maker solution, free_ebook_maker software

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、Markdown / reStructuredText / AsciiDoc の3種の軽量マークアップ言語を入力とし、内部的に AST（抽象構文木）へパースした後、CSS ページメディア仕様に基づくレイアウトエンジンで組版を行う。特に日本語禁則処理（行頭禁則・行末禁則・分割禁止文字）を完全実装し、`—`（ダッシュ）や `…`（三点リーダー）のぶら下げ処理もミリ秒単位で最適化済み。

### コア機能

- **動的スタイル継承エンジン**: 各チャプターごとに異なるフォントサイズ（10pt〜14pt）、行間（1.5〜2.0）、ページマージン（上下20mm〜30mm）を YAML フロントマターで指定可能。`--style-inherit` フラグで親テーマのプロパティを再帰的に継承。PostgreSQL に保存されたユーザーテーマDBと連携し、1000テーマ以上から選択可能。

- **数式・コード・図版の自動配置アルゴリズム**: LaTeX 数式（`$...$` / `$$...$$`）を MathJax 3 で SVG 変換後、`<figure>` 要素にラップ。コードブロックは highlight.js 11 でシンタックスハイライトし、改ページ制御属性 `page-break-before: avoid` を自動付与。画像は `--image-dpi` パラメータ（デフォルト300dpi）でリサイズし、JPEG品質75%で圧縮。

- **マルチフォーマット同時出力パイプライン**: `--output-formats epub, pdf, mobi` で指定。内部で Pandoc 3.1 をラップし、EPUB は `--epub-cover-image` と `--epub-metadata` を必須パラメータとして検証。PDF は WeasyPrint 60.0 でレンダリング、CSS `@page` ルールの `size: A4` または `size: B5` を厳密に解釈。MOBI は KindleGen 互換の OPF ファイルを生成。

### 使い方

1. **プロジェクト初期化とテーマ設定**  
   `free_ebook_maker init --project-name "my_ebook" --theme "japanese_vertical" --lang ja`  
   カレントディレクトリに `my_ebook/` が作成され、`config.yaml` が生成。`--page-size B5 --font-size 11pt --line-height 1.8` を追記し、PostgreSQL の `themes` テーブルから `japanese_vertical` の CSS 変数を自動取得。`--image-dpi 350` で印刷品質に調整。

2. **ソースファイルの作成とAST検証**  
   `chapters/` ディレクトリに `01_intro.md` を作成。先頭に YAML フロントマターを記述：  
   ```yaml
   ---
   title: はじめに
   page-break: always
   styles:
     font-size: 12pt
     line-height: 2.0
   ---
   ```  
   次に `free_ebook_maker validate --source chapters/ --strict` を実行。AST パーサーが禁則文字の違反（例：行頭に「。」がある）を検出し、エラーコード `E0023` と行番号を出力。修正後、`--fix` フラグで自動修正。

3. **ビルドと品質チェック**  
   `free_ebook_maker build --output-formats epub,pdf --output-dir ./dist --epub-cover cover.png --epub-metadata metadata.xml`  
   `--check-accessibility` フラグを追加し、EPUB のアクセシビリティ基準（WCAG 2.1 AA）に準拠しているか検証。`--pdf-outline-depth 3` で目次の階層を3レベルに設定。ビルド後、`dist/` に `my_ebook.epub` と `my_ebook.pdf` が生成され、`--validate-output` で EPUB は EpubCheck 4.2、PDF は PDF/A-2b 準拠チェックを自動実行。

### ベストプラクティス

- **禁則処理のカスタマイズとパフォーマンス最適化**  
  日本語組版では、`config.yaml` の `cjk.line-break` セクションで `strict: true` を設定し、`prohibited-characters` に独自の禁止文字リスト（例：`〆` や `々`）を追加。さらに `--cache-ast` フラグで AST をディスクにキャッシュし、2回目以降のビルド時間を平均40%短縮。PostgreSQL の `cache` テーブルにハッシュ化した AST を保存し、差分ビルドを実現。

- **バージョン管理と差分ビルドの統合**  
  ソースファイルを Git で管理し、`free_ebook_maker diff --commit-hash HEAD~1` で前回コミットからの差分のみを再ビルド。`--output-formats pdf --diff-only` で変更されたチャプターだけを PDF に変換。PostgreSQL の `build_history` テーブルに各ビルドのハッシュと出力サイズ（KB単位）を記録し、`--history-limit 50` で直近50件を保持。
