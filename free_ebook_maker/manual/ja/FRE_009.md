# free_ebook_maker - FRE_009

## 概要

本プロジェクトは、Markdown 原稿から高品質な日本語 EPUB 及び PDF を生成するためのコマンドラインツールです。キーワード「free_ebook_maker tool」「free_ebook_maker solution」「free_ebook_maker software」に基づき、組版エンジンに CSS 印刷仕様（Paged.js）を採用し、出版品質の電子書籍をローカル環境で完結させます。

## 詳細な内容

### free_ebook_maker とは

組版エンジンに Paged.js を内蔵し、CSS によるページマージン制御、ハイフネーション処理、ルビ配置を可能にするプロフェッショナル電子書籍生成ツールです。Markdown → HTML → EPUB/PDF のパイプラインをワンコマンドで実行し、目次自動生成、画像の最適化、メタデータ埋め込みを標準サポートします。

### コア機能

1. **CSS 組版エンジンによるページ精度制御**  
   - `--page-margin` パラメータで上下左右の余白をミリメートル単位（例: `20mm 15mm 25mm 15mm`）で指定可能。  
   - `--hyphenation` フラグで日本語禁則処理（行頭禁則・行末禁則）を自動適用。  
   - `--ruby` オプションにより `<ruby>` タグを解釈し、ルビのベースライン位置を 0.5em 単位で調整。

2. **マルチフォーマット並列出力と差分検証**  
   - `--output epub, pdf, mobi` と指定することで、同一ソースから 3 フォーマットを同時生成。  
   - 生成後の EPUB は `epubcheck` バリデーションを自動実行し、エラー行番号を JSON 形式で出力。  
   - PDF 出力時は `--pdf-version 1.7` で PDF/A-3b 準拠を担保。

3. **リソース埋め込みと外部連携**  
   - `--cover-image` に JPEG/PNG を指定すると、自動で 300dpi にリサンプリングし、ICC プロファイル（sRGB IEC61966-2.1）を埋め込み。  
   - `--metadata` で DC（Dublin Core）要素（title, creator, publisher, identifier）を YAML 形式で注入可能。  
   - 出力先のファイル名には `--timestamp` フラグで UNIX タイムスタンプ（ミリ秒単位）を自動付与し、バージョン管理と競合を防止。

### 使い方

1. **プロジェクトの初期化と設定ファイルの生成**  
   ```bash
   free_ebook_maker init --project my_book --lang ja --output-dir ./output
   ```
   これにより、`my_book/` ディレクトリ以下に `config.yml`（組版パラメータ）、`manuscript/`（Markdown 原稿格納ディレクトリ）、`images/`（画像アセット）が生成されます。`config.yml` 内の `page.width: 128mm` や `page.height: 182mm` を編集することで、文庫本サイズや A5 サイズに即座に変更可能です。

2. **Markdown 原稿の作成とビルド**  
   ```bash
   cd my_book
   echo '# 第一章\n\nこれは本文です。' > manuscript/ch01.md
   free_ebook_maker build --config config.yml --output epub,pdf
   ```
   `build` コマンドはまず `manuscript/` 内の全 `.md` ファイルをファイル名順にマージし、見出しレベル（`#` の数）に応じて自動で目次（nav.xhtml）を生成します。その後、Paged.js エンジンが各ページの改ページ位置を計算し、`--page-margin` に従って余白を適用します。

3. **出力結果の検証とカスタマイズ**  
   ```bash
   free_ebook_maker validate --file output/my_book_20250315_143022.epub
   free_ebook_maker customize --input output/my_book_20250315_143022.pdf --watermark "DRAFT" --opacity 0.3
   ```
   `validate` は `epubcheck` をラップし、エラーが存在する場合は該当行と修正候補を表示します。`customize` コマンドでは既存 PDF に透かしやページ番号（`--page-number bottom-center`）を後挿入可能で、校正用の一時ファイル生成に有用です。

### ベストプラクティス

- **バージョン管理と差分検証の自動化**  
  生成ファイル名に `--timestamp` を常に付与し、`git lfs` で管理することを推奨します。また、`free_ebook_maker diff --left old.epub --right new.epub` コマンドでテキストレイヤーの差分を HTML レポートとして出力できるため、校正時の変更点追跡が容易になります。

- **組版パラメータのプロファイル化**  
  プロジェクトごとに `config.yml` を複数用意し（例: `config_kobo.yml`、`config_kindle.yml`）、デバイスごとに最適化されたマージンとフォントサイズを切り替えてビルドします。特に `--font-size-base 10pt` と `--line-height 1.8` を固定し、`--page-margin` のみ変更することで、リフロー型（EPUB）と固定レイアウト型（PDF）の両方で一貫した可読性を担保できます。
