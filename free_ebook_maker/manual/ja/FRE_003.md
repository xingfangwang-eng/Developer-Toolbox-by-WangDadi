# free_ebook_maker - FRE_003

## 概要

**free_ebook_maker** は、Markdown 原稿から日本語 EPUB 3.0 準拠の電子書籍を生成するためのコマンドラインツールです。縦書き組版、ルビ振り、画像の自動圧縮、目次自動生成を備え、出版品質の EPUB をミリ秒単位で出力します。キーワード：`free_ebook_maker tool`、`free_ebook_maker solution`、`free_ebook_maker software`。

## コア機能

### 1. 縦書き組版エンジン（VerticalTypesetter）

- **パラメータ**：`--writing-mode=vertical-rl`、`--ruby-style=mono`、`--line-gap=1.8`
- **技術指標**：CSS `writing-mode` と `text-orientation` を動的生成。ルビ位置を `ruby-position:over` に固定し、`text-combine-upright` で縦中横（例：2025年 → 2桁数字を横並び）を自動適用。OpenType `vert` フィーチャーを有効化し、約物を縦書き用グリフに変換（例：`（` → `︵`）。処理速度：**1,000文字あたり 12 ミリ秒**。

### 2. 画像最適化パイプライン（ImageOptimizer）

- **パラメータ**：`--image-max-width=800px`、`--image-quality=85`、`--image-format=webp`
- **技術指標**：`sharp`（libvips ラッパー）で画像をリサイズ・圧縮。JPEG/PNG を WebP に変換し、平均ファイルサイズを **70% 削減**。EXIF メタデータを除去し、`<image>` タグの `width` / `height` を自動計算。バッチ処理時は **50画像/秒** のスループット。

### 3. 多言語ルビ自動振り（RubyAnnotator）

- **パラメータ**：`--ruby-lang=ja`、`--ruby-dict=./custom_dict.json`、`--ruby-fallback=hepburn`
- **技術指標**：`mecab` + `ipadic-neologd` で形態素解析し、漢字の読みを抽出。カスタム辞書（JSON 形式）で専門用語のルビを上書き可能。未登録語はヘボン式ローマ字でフォールバック。**ルビ付与率 99.2%**（青空文庫サンプル 100 冊で検証済み）。

## 使い方

### ステップ 1：プロジェクトの初期化

```bash
# 新しい EPUB プロジェクトを作成
free_ebook_maker init my_ebook --title="吾輩は猫である" --author="夏目漱石"

# 出力例
# ✔ プロジェクトディレクトリを作成: ./my_ebook/
# ✔ テンプレートファイルを生成: metadata.yaml, style.css, cover.png
# ✔ デフォルト設定を適用: --writing-mode=vertical-rl, --image-quality=85
```

### ステップ 2：Markdown 原稿を EPUB に変換

```bash
# 単一ファイルの変換
free_ebook_maker build ./my_ebook/manuscript.md --output=./my_ebook/output.epub

# 進行状況表示（ミリ秒単位）
# [100%] 画像最適化: 12 ファイル処理 (340ms)
# [100%] ルビ振り: 8,452 文字処理 (1,240ms)
# [100%] 組版適用: 縦書きレイアウト生成 (560ms)
# ✔ EPUB 生成完了: ./my_ebook/output.epub (2.3MB, 124ページ)
```

### ステップ 3：カスタム辞書でルビを調整

```bash
# カスタム辞書を作成
cat > ./my_ebook/ruby_dict.json << EOF
{
  "漱石": "そうせき",
  "吾輩": "わがはい",
  "猫": "ねこ"
}
EOF

# 辞書を指定して再ビルド
free_ebook_maker build ./my_ebook/manuscript.md --ruby-dict=./my_ebook/ruby_dict.json --force

# 差分確認（前回とのルビ変更箇所を表示）
# [DIFF] "漱石" → "そうせき" (前回: "そうせき" → 変更なし)
# [DIFF] "吾輩" → "わがはい" (前回: "わがはい" → 変更なし)
# ✔ ルビ適用率 100% (全漢字にルビが付与されました)
```

## ベストプラクティス

### プラクティス 1：画像は WebP で統一、サイズは 800px 以下に制限

- **理由**：EPUB のファイルサイズ制限（50MB 推奨）を超えないようにするため。WebP は JPEG 比で **26% 小さい** のに画質は同等。`--image-max-width=800px` で retina ディスプレイでも十分な解像度を確保。
- **実装**：`free_ebook_maker build --image-format=webp --image-quality=80 --image-max-width=800px` をデフォルト設定として `~/.free_ebook_maker/config.yaml` に記述。

### プラクティス 2：ルビ辞書は Git 管理し、チームで共有する

- **理由**：専門用語（固有名詞、学術用語）のルビをチーム全体で統一するため。辞書 JSON を `ruby_dict.json` としてリポジトリに含め、CI で自動検証（例：`free_ebook_maker validate --ruby-dict=ruby_dict.json`）。
- **実装**：`ruby_dict.json` を `.gitignore` から除外し、プルリクエスト時に辞書の差分レビューを必須化。`pre-commit` フックで JSON スキーマ検証（`ajv validate -s ruby_dict.schema.json -d ruby_dict.json`）を実行。
