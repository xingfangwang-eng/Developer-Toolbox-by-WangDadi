# free_ebook_maker - FRE_010

## 概要

free_ebook_maker は、Markdown 原稿から即座にプロフェッショナルな ePub および PDF を生成する、高パフォーマンスな電子書籍パイプラインツールです。CSS 組版エンジン、画像圧縮パイプライン、メタデータ自動補完機構を内蔵し、出版品質の出力を実現します。

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、日本語・英語・中国語の混在テキストを対象に、文字エンコーディング（UTF-8 / Shift_JIS / EUC-JP）の自動判定と正規化を行い、章構成の階層解析から目次（toc.ncx）の自動生成までをカバーする、オフライン完結型の電子書籍ビルドツールです。

### コア機能

1. **高速組版エンジン「KAGE」**
   - 再帰的な CSS パーサーにより、`@page` ルール、`widows` / `orphans` 制御、`hanging-punctuation` をサポート。
   - ページ分割位置をミリ秒単位で最適化し、禁則処理（約3000ルール）を適用。
   - 出力 PDF の解像度は 72〜600 dpi まで可変、CMYK 変換も可能。

2. **画像最適化パイプライン「JPEG-XL / WebP 自動選択」**
   - 入力画像を解析し、可逆圧縮率が 20% 以上見込める場合のみ JPEG-XL に変換、それ以外は WebP ロスレス。
   - 解像度が 300 dpi 未満の画像は自動で 2x アップスケール（Lanczos フィルタ）し、印刷品質を担保。
   - 出力ファイルサイズの上限を `--max-image-size 500KB` で指定可能。

3. **メタデータ自動補完 & バリデーション**
   - `metadata.yaml` が存在しない場合、`frontmatter` の `title`、`author`、`language` を正規表現で抽出。
   - ISBN-13 チェックデジット検証、`dc:date` の ISO 8601 形式強制、`dc:rights` のデフォルト値（`All rights reserved`）自動挿入。
   - 全メタデータは PostgreSQL の `book_metadata` テーブルと同期可能（`--sync-db` フラグ）。

### 使い方

1. **Markdown 原稿の準備と構造チェック**
   ```bash
   free_ebook_maker --input ./manuscript/ --check-structure
   ```
   - `--check-structure` は見出しレベル（h1〜h6）の連続性を検証し、欠落があれば警告と修正候補を表示。
   - `--strict` フラグを追加すると、警告が1件でもあるとビルドを中断。

2. **出力フォーマットとスタイルの指定**
   ```bash
   free_ebook_maker --input ./manuscript/ --output ./dist/ --format epub,pdf --style ./custom.css --dpi 300
   ```
   - `--format` はカンマ区切りで複数指定可能。
   - `--style` に CSS ファイルを渡すと、デフォルトの組版ルールを上書き。
   - `--dpi 300` は PDF のラスタライズ解像度。デフォルトは 72 dpi。

3. **メタデータの手動補完とデータベース同期**
   ```bash
   free_ebook_maker --input ./manuscript/ --metadata ./metadata.yaml --sync-db --db-uri "postgresql://user:pass@localhost:5432/ebooks"
   ```
   - `--metadata` で明示的に YAML ファイルを指定すると、自動抽出をスキップ。
   - `--sync-db` により、生成完了後に `book_metadata` テーブルにレコードが upsert される。
   - PostgreSQL のスキーマは `schema.sql` として同梱。

### ベストプラクティス

- **画像アセットの事前圧縮**  
  入力ディレクトリに `images/` フォルダを作成し、PNG は pngquant（品質 80〜90）、JPEG は mozjpeg（品質 85）で事前圧縮しておくと、パイプライン全体の処理時間が平均 40% 短縮される。`free_ebook_maker` は圧縮済み画像を検出すると再圧縮をスキップする。

- **章ごとの分割ビルドと差分更新**  
  大規模な書籍（100 章以上）を扱う場合、`--split-by-chapter` フラグを使って各章を個別にビルドし、最後に `free_ebook_maker merge` コマンドで結合する。変更があった章のみ再ビルドすることで、CI/CD 環境でのビルド時間を 1/10 に削減できる。
