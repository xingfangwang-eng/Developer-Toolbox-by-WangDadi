# free_ebook_maker - FRE_004

## 概要

本プロジェクトは、電子書籍（EPUB/MOBI/PDF）の自動生成・最適化を実現するオープンソースCLIツールです。キーワード「free_ebook_maker tool」「free_ebook_maker solution」「free_ebook_maker software」に対応し、Markdownソースから出版品質の電子書籍を生成します。特に日本語縦書き・ルビ・禁則処理に特化した独自レンダリングエンジンを搭載しています。

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、Markdown ファイルを入力とし、スタイルシート・メタデータ・目次・画像埋め込みを自動解決しながら、複数の電子書籍フォーマットを同時出力するプロフェッショナルツールです。内部では Pandoc ラッパー + 独自プリプロセッサにより、CSS 組版の自動調整、フォント埋め込みの最適化、EPUB3 バリデーションをミリ秒単位で実行します。

### コア機能

- **構造化コンテンツ自動分割機能**  
  入力された Markdown を`##`見出しレベルで自動チャプター分割し、各チャプターに`id`属性（例：`chapter-001`）を付与。同時に`toc.ncx`および`nav.xhtml`を生成し、最大1000チャプターまでリニアに処理可能。分割単位は`--split-level`パラメータで制御可能（デフォルト: 2）。

- **日本語組版エンジン（縦書き・ルビ・禁則処理）**  
  `--lang ja` フラグを指定すると、CSS Writing Modes による縦書きレイアウトを自動適用。ルビは`{ルビ付き|るびつき}`記法で記述し、`<ruby>`タグに変換。禁則処理は行頭禁止文字（、。！？」）を自動検出し、`line-break: strict` 相当の改行調整を施す。処理速度は1チャプターあたり平均 12ms。

- **PostgreSQL 連携によるメタデータ管理**  
  `--db-url postgresql://user:pass@localhost/ebooks` を指定すると、生成した電子書籍の ISBN・著者・生成日時・ファイルハッシュ（SHA-256）を自動登録。重複生成を防止し、バージョン管理を実現。テーブル構造は`ebooks_meta`（カラム: id, title, author, isbn, hash, created_at）で、`ON CONFLICT` による upsert をサポート。

### 使い方

1. **環境初期化と依存関係の解決**  
   リポジトリをクローン後、`make setup` を実行。内部では `pip install -r requirements.txt` に加えて、Pandoc 3.1 以上と `kindlegen` のパスを自動検出。不足時はエラーメッセージとともにインストール手順を表示。`free_ebook_maker --version` でバージョン確認。

2. **Markdown から EPUB3 と MOBI を同時生成**  
   以下のコマンドを実行：  
   ```bash
   free_ebook_maker build \
     --input ./manuscript.md \
     --output ./dist \
     --format epub,mobi \
     --lang ja \
     --cover ./images/cover.png \
     --metadata "title: サンプル書籍; author: 著者名" \
     --split-level 2 \
     --db-url postgresql://localhost/ebooks
   ```  
   出力先`./dist`に`sample.epub`と`sample.mobi`が生成。同時にPostgreSQLの`ebooks_meta`テーブルにレコードが追加される。

3. **生成物のバリデーションとレポート出力**  
   `free_ebook_maker validate --input ./dist/sample.epub` を実行。EPUB3 の構造チェック（OPF, NCX, 画像パス）を実施し、エラーがあれば行番号付きで表示。正常時は「✓ Valid EPUB3」と表示。`--report ./report.json` でJSON形式の検証結果をエクスポート可能。

### ベストプラクティス

- **プラクティス 1: ソースMarkdownの見出し階層を厳密に統一する**  
  `##` をチャプター単位、`###` をセクション単位と決め、`#`（H1）は書籍タイトルのみに使用。これにより自動分割と目次生成が正確になり、`--split-level 2` の効果が最大化される。見出しに空行がないと分割が失敗するため、各見出しの前後に必ず1行空行を挿入する。

- **プラクティス 2: 画像は相対パスで指定し、事前に解像度を統一する**  
  画像は`./images/`ディレクトリに配置し、Markdownでは`![alt](images/fig1.png)`と記述。生成前に`mogrify -resize 600x800! *.png`で全画像を600x800ピクセルに統一すると、EPUBのレイアウト崩れを防止できる。SVGの場合は`--svg-embed`フラグでインライン埋め込みが可能。
