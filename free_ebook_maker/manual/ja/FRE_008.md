# free_ebook_maker - FRE_008

## 概要

free_ebook_maker は、Markdown ベースの原稿から高品質な EPUB および PDF を生成するためのオフライン対応型ビルドツールです。EPUB3 準拠の固定レイアウト・リフロー両対応、および多言語フォントの自動埋め込みを実現します。

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、電子書籍制作パイプラインを完全自動化するコマンドラインツールです。入力された Markdown ファイル群を解析し、目次自動生成、画像圧縮（JPEG 品質 85% 固定）、CSS インライン化を施した上で、EPUB3.2 および PDF/A-2b を出力します。内部では Node.js v18 のストリーム処理と Sharp ライブラリによる並列画像変換を実装しており、100 ページの書籍を約 2.3 秒でビルド可能です。

### コア機能

- **マルチフォーマット同時出力エンジン**  
  単一のビルドコマンドで EPUB（.epub）、PDF（.pdf）、およびプレーンテキスト（.txt）を同時生成。EPUB 出力時は `--epub-version 3.2` パラメータで固定レイアウトとリフローを切り替え可能。PDF 出力時は `--pdf-page-size A5` や `--pdf-margin 15mm` でページ設定を細かく指定。

- **自動フォントサブセット化と埋め込み**  
  日本語・中国語・韓国語（CJK）フォントに対して、使用文字のみを抽出するサブセット化を実施。Noto Sans CJK JP をデフォルトフォントとして内蔵し、`--font-embed true` フラグで埋め込みを制御。サブセット化後のフォントサイズは平均 1.2MB まで削減（元サイズの約 1/15）。

- **メタデータ自動抽出とバリデーション**  
  原稿先頭の YAML フロントマターからタイトル、著者、ISBN、言語コード（ja など）を自動抽出。さらに `--validate-metadata` オプションで ISBN チェックデジット検証、言語コードの ISO 639-1 準拠チェックを実行。不正なメタデータが検出された場合はビルドを中断し、エラー行番号を出力。

### 使い方

1. **原稿ディレクトリの準備と設定ファイルの作成**  
   プロジェクトルートに `ebook.yaml` を配置し、以下のパラメータを記述します。  
   ```yaml
   title: "サンプル書籍"
   author: "山田太郎"
   language: ja
   output_format: [epub, pdf]
   epub_version: 3.2
   pdf_page_size: A5
   font_embed: true
   ```  
   原稿は `src/` ディレクトリに Markdown ファイルとして格納し、ファイル名の先頭に番号（例: `01_intro.md`）を付与することで結合順序を制御します。

2. **ビルドの実行とパラメータ指定**  
   ターミナルで以下のコマンドを実行します。  
   ```bash
   free_ebook_maker build --config ebook.yaml --output ./dist --compress-images true --image-quality 85
   ```  
   `--compress-images true` で画像を自動圧縮（JPEG 品質 85、PNG は 256 色に減色）、`--image-quality` で品質を微調整可能。ビルド中は各ステップの経過時間（ミリ秒単位）が標準出力に表示されます。

3. **出力ファイルの検証と修正**  
   生成された `dist/` ディレクトリ内の EPUB ファイルに対し、`free_ebook_maker validate --file dist/sample.epub` を実行。EPUB3.2 仕様に違反する箇所（例: 未定義の CSS クラス、壊れた内部リンク）を自動検出し、修正箇所の行番号と推奨修正方法を出力します。エラーが 0 件になるまで修正を繰り返します。

### ベストプラクティス

- **継続的インテグレーション（CI）パイプラインへの組み込み**  
  GitHub Actions や GitLab CI 上で `free_ebook_maker build` と `free_ebook_maker validate` を直列実行し、プルリクエストごとに EPUB の品質を自動チェックします。これにより、誤ったメタデータや壊れたリンクを含む書籍がリリースされるのを防止します。特に `--fail-on-warning` フラグを併用することで、警告レベルの問題でもビルドを中断させることが可能です。

- **画像アセットの事前最適化とキャッシュ戦略**  
  大量の画像を含む書籍（例: 写真集や図解マニュアル）では、`--image-cache-dir ./cache` オプションで画像処理結果をキャッシュします。初回ビルド時に生成されたサムネイルやサブセットフォントをキャッシュディレクトリに保存し、2 回目以降のビルド時間を約 70% 短縮します。また、画像は事前に `--target-width 1200` でリサイズし、縦横比を維持したままファイルサイズを削減することを推奨します。
