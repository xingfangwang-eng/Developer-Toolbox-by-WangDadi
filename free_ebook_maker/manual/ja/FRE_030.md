# free_ebook_maker - FRE_030

## 概要

free_ebook_maker は、Markdown 原稿から EPUB3 / PDF / Mobi 形式の電子書籍を **ミリ秒単位のレンダリング精度** で生成する、オフライン対応の次世代パブリッシングエンジンです。CSS 組版、フォント埋め込み、画像圧縮の各工程を独自のパイプラインで最適化し、**1000 ページ超のドキュメントでも 3 秒以内に出力可能** なパフォーマンスを実現します。

## コア機能

### 1. 動的テンプレートエンジン（DTP モード）
- **パラメータ**: `--template` `--page-size A5` `--margin 15mm`
- ユーザー定義の CSS 変数（`--body-font-size: 10pt`）と条件分岐テンプレート（`{{if japanese}}...{{/if}}`）をサポート。
- 組版エンジン内部で **0.1mm 単位のボックスモデル計算** を実施し、印刷所入稿用の裁ち落とし付き PDF を生成。
- 対応フォーマット: EPUB3（固定レイアウト可）、PDF（PDF/X-1a:2001 準拠）、Mobi（Kindle Previewer 互換）。

### 2. マルチリンガル組版最適化
- **パラメータ**: `--lang ja` `--ruby` `--vertical-rl`
- 日本語縦書き組版では、連続する約物（括弧・句読点）の詰め処理（**約物半角化ルール**）を自動適用。
- ルビ（振り仮名）のベースラインを **0.5px 単位で調整** し、InDesign 書き出し同等の品質を担保。
- 中国語簡体字・繁体字、韓国語ハングル、アラビア語 RTL に個別のハイフネーション辞書を内蔵。

### 3. メタデータ自動抽出 & バリデーション
- **パラメータ**: `--metadata` `--validate`
- Markdown の YAML front matter から著者名、ISBN、発行日を自動抽出し、**ONIX 3.0 フォーマット** に変換。
- 出力前に全リンクの疎通チェック（HTTP ステータスコード 200 確認）を実施。リンク切れがある場合は **エラーコードと行番号を一覧表示**。
- 画像の解像度が 300 DPI 未満の場合、警告を発して自動で補間処理（Lanczos フィルタ）を適用。

## 使い方

### 1. プロジェクト初期化とテンプレート選択
```bash
free_ebook_maker init --project ./mybook --template novel-jp
```
- 実行後、`./mybook/` 配下に `manuscript/`（原稿）、`assets/`（画像・フォント）、`config.yaml`（組版設定）が生成されます。
- `config.yaml` 内で `page_size: A5`、`font_family: "Noto Serif JP"`、`line_gap: 8pt` を指定可能。

### 2. 原稿の組版 & プレビュー
```bash
free_ebook_maker build --input ./mybook/manuscript/chapter1.md --preview
```
- `--preview` フラグにより、**ブラウザベースのリアルタイムプレビュー**（WebSocket 経由）が起動。`localhost:3000` でページ送り、フォント変更、ルビ表示切替を確認できます。
- 組版エラー（禁則違反、画像リンク切れ）は **ターミナルに赤色でハイライト表示**。

### 3. マルチフォーマット一括エクスポート
```bash
free_ebook_maker export --formats epub,pdf,mobi --output ./dist/
```
- EPUB3 出力時は **OPF ファイルの `<spine>` タグを自動生成**。`toc.ncx` と `nav.xhtml` を W3C 標準仕様に準拠。
- PDF 出力時は **PostgreSQL 連携** により、フォント埋め込みライセンス情報をデータベースから参照。ライセンス切れのフォントは代替フォントで自動フォールバック。
- Mobi 出力時は **KindleGen 互換のバイナリビルド** を実行。画像は JPEG 品質 85% に圧縮し、ファイルサイズを **平均 40% 削減**。

## ベストプラクティス

### プラクティス 1: 大規模プロジェクトでのインクリメンタルビルド
- 数千ページの技術書や小説を扱う場合、`--incremental` フラグを推奨。
- 変更があったファイルのみを再組版し、キャッシュ（`~/.free_ebook_maker/cache/`）を活用。初回ビルド時間が 10 秒かかるプロジェクトでも、2 回目以降は **0.8 秒未満** で完了。
- 並列処理数は `--jobs 4` で指定可能。コア数に応じて最適化。

### プラクティス 2: CI/CD パイプラインへの統合
- GitHub Actions や GitLab CI で `free_ebook_maker validate && free_ebook_maker export --formats epub,pdf` を実行。
- バリデーションエラーが 1 件でもある場合は **exit code 1** を返すため、プルリクエストの自動チェックに最適。
- 生成物はアーティファクトとして保存し、リリースタグと同時に自動公開。
