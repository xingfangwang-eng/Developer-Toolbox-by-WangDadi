# free_ebook_maker - FRE_027

## 概要

`free_ebook_maker` は、Markdown / AsciiDoc / ReStructuredText ソースから、**出版品質の EPUB 3.2 / PDF / HTML スライド** を生成するオフライン・コマンドラインツールです。  
キーワード: **free_ebook_maker tool**, **free_ebook_maker solution**, **free_ebook_maker software**。  
全処理をメモリ上で完結させるため、**10,000 ページ超の大規模書籍でもビルド時間 2 秒未満** を実現しています。

## 詳細な内容

### free_ebook_maker とは？

`free_ebook_maker` は、**章単位の差分ビルド**、**画像最適化パイプライン**、**多言語フォント自動埋め込み** を備えた、次世代 eBook ジェネレーターです。  
内部では `pandoc` + `weasyprint` + `libvips` を統合し、**CSS Paged Media 仕様に完全準拠** したページレイアウトを出力します。

## コア機能

### 機能 1: 並列チャプタービルド & キャッシュ機構

- パラメータ: `--workers=4`, `--cache-ttl=3600`
- 各章を独立したプロセスで並列変換。変更のあった章のみ再ビルド（**ミリ秒単位の差分検出**）。
- キャッシュは SQLite に保存。`--cache-dir=/tmp/ebook_cache` でパス指定可能。
- 技術指標: **10 章構成の書籍で初回ビルド 1.2 秒、2 回目以降 0.3 秒**。

### 機能 2: 画像アセットの自動 WebP 変換 & 解像度最適化

- パラメータ: `--image-quality=85`, `--max-width=1200`
- ソース画像（PNG/JPEG/TIFF）を **WebP Lossless または AVIF** に変換。  
  解像度が `max-width` を超える場合はリサイズ、**Exif 回転情報を自動適用**。
- 技術指標: **平均ファイルサイズ 62% 削減、EPUB の読み込み速度 3.2 倍向上**。

### 機能 3: 多言語フォント自動フォールバック & グリフ埋め込み

- パラメータ: `--font-family="Noto Sans JP, Noto Sans SC, serif"`, `--subset-glyphs=true`
- 使用されていないグリフを除去した **サブセットフォント** を EPUB に埋め込み。  
  日本語・中国語・韓国語・キリル文字が混在する文書でも、**フォントサイズ 90% 削減**。
- 技術指標: **フォールバックチェーンは最大 8 段階、PostgreSQL 上のフォントメタデータベースと連携**。

## 使い方

### ステップ 1: プロジェクト初期化 & 設定ファイル生成

```bash
free_ebook_maker init --project-dir=./mybook --lang=ja --format=epub3
```

- カレントディレクトリに `ebook.yml`（YAML 設定）と `chapters/` ディレクトリが生成されます。
- `ebook.yml` には `title`, `author`, `isbn`, `cover-image`, `css-override` を記述可能。
- 内部で `pandoc` の `--metadata-file` を自動生成。**バリデーションは JSON Schema 準拠**。

### ステップ 2: チャプター作成 & アセット配置

```bash
mkdir -p chapters/01_intro chapters/02_methods
echo "# はじめに" > chapters/01_intro/index.md
```

- 各チャプターは `index.md`（または `.adoc` / `.rst`）で記述。
- 画像は `assets/` ディレクトリに配置。`assets/fig1.png` は `![図1](assets/fig1.png)` で参照。
- **自動リンター** が破損画像や未参照ファイルを検出（`--lint` フラグ）。

### ステップ 3: ビルド & 差分検証

```bash
free_ebook_maker build --output=./output/book.epub --diff-mode=chapter
```

- `--diff-mode=chapter` を指定すると、前回ビルドとの差分を **SHA256 ハッシュ** で比較。変更のない章はスキップ。
- 出力ディレクトリには `book.epub`, `book.pdf`, `book.html` が生成。
- ビルドログは `build.log` に記録。**エラー行はソースファイルの行番号付き**。

## ベストプラクティス

### プラクティス 1: CI/CD パイプラインへの統合

- GitHub Actions で `free_ebook_maker build --output=./artifacts/book.epub --cache-ttl=86400` を実行。  
  プルリクエストごとに **プレビュー用 EPUB を自動生成** し、コメントにダウンロードリンクを投稿するワークフローが推奨。
- キャッシュを共有するため、`--cache-dir` には GitHub Actions の `runner.temp` を指定。

### プラクティス 2: 大規模書籍の章分割戦略

- 1 章あたりの Markdown ファイルは **3,000 行以内** に分割。  
  これを超えると `pandoc` の AST メモリ使用量が急増するため、`--split-level=2`（見出しレベル 2 で自動分割）を併用。
- 画像は `assets/` にフラット配置せず、`assets/figures/chapter01/` のように章ごとにサブディレクトリを作成。  
  これにより `--image-quality` のパラメータを章単位で変更可能（例: `--image-quality=80 --chapter=01`）。
