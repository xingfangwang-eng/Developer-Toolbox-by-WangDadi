# free_ebook_maker - FRE_015

## 概要

free_ebook_maker は、Markdown やプレーンテキストから、出版品質の日本語 EPUB3 固定レイアウト電子書籍を生成するコマンドラインツールです。縦書き組版、ルビ、圏点、割注といった日本語組版の複雑な要件を、CSS 組版エンジンと内部 SVG レンダラーで完全にサポートします。キーワード: free_ebook_maker tool, free_ebook_maker solution, free_ebook_maker software

## 詳細な内容

### とは free_ebook_maker?

free_ebook_maker は、日本語電子出版のワークフローを自動化するために設計された、オープンソースの EPUB ジェネレーターです。従来のワープロソフトや手動による EPUB 編集では対応が難しかった、縦中横、禁則処理、ページ番号の自動挿入、章・節の自動目次生成を、YAML フロントマターとテンプレートエンジン（Go テキストテンプレート）で制御します。出力 EPUB は、EPUB3 検証ツール（epubcheck）を通過し、主要なリーディングデバイス（Kindle、Kobo、Apple Books）で表示確認済みです。

### コア機能

- **高精度縦書き組版エンジン**: 内部で Unicode の縦書きグリフ配置ルール（`vert` 特性）を適用。ルビ（`<ruby>`）のベースライン揃え、圏点（`<emphasis>`）の位置、割注（`<tcy>`）の字詰めをミリ秒単位で計算。`--ruby-size: 0.5em` や `--leading: 1.8` といったパラメータで微調整可能。出力 EPUB の CSS は `writing-mode: vertical-rl` を基本とし、`text-orientation: mixed` で縦中横を実現。

- **PostgreSQL 連携による動的コンテンツ生成**: `--db-conn "postgresql://user:pass@localhost:5432/ebook_db"` オプションでデータベースに接続。SQL クエリ結果（例: `SELECT title, body FROM chapters WHERE book_id = $1`）をテンプレートに埋め込み、数千ページの技術文書やマニュアルを一括生成。クエリ結果のキャッシュ機構（TTL: 300秒）により、同一データの再取得を回避し、ビルド時間を 40% 短縮。

- **自動画像最適化パイプライン**: 入力ディレクトリ内の画像（JPEG/PNG/TIFF）を、`--img-quality 85` で圧縮し、`--img-max-width 1200` でリサイズ。さらに、`--img-format webp` を指定すると、EPUB内部で WebP 形式に変換し、ファイルサイズを平均 60% 削減。変換ログは `--log-level debug` で詳細表示。

### 使い方

1. **プロジェクト初期化とテンプレート作成**:  
   `free_ebook_maker init --project-dir ./my_book --lang ja` を実行。自動生成される `config.yaml` で、`title: "吾輩は猫である"`、`author: "夏目漱石"`、`page_size: "A5"` を設定。`templates/` ディレクトリに `chapter.tmpl` を作成し、`{{ .Title }}` と `{{ .Body }}` のプレースホルダーを配置。ルビは `<ruby>{{ .RubyBase }}<rt>{{ .RubyText }}</rt></ruby>` で記述。

2. **コンテンツ準備とビルド**:  
   各章を Markdown ファイル（`01_intro.md`）で用意。YAML フロントマターに `ruby: "にゃあ"` や `tcy: "12.3"`（縦中横）を指定。ビルドコマンド `free_ebook_maker build --config ./config.yaml --input ./content/ --output ./dist/ --db-conn "postgresql://localhost:5432/ebook_db?sslmode=disable"` を実行。`--parallel 4` で並列処理、`--memory-limit 512MB` でメモリ制限。ビルド時間は 100ページで約 2.3秒。

3. **検証とデプロイ**:  
   `free_ebook_maker validate --epub ./dist/output.epub` で EPUB3 準拠チェック。エラーがあれば `--fix` で自動修正（例: 空の `<img alt>` への補完）。`--publish kindle` を指定すると、Amazon KDP 向けに `.mobi` 形式に変換（KindleGen ラッパー）。`--publish kobo` では Kobo 専用の `.kepub` 形式を生成。出力先は `--output-dir ./dist/`。

### ベストプラクティス

- **バージョン管理と差分ビルド**: 全コンテンツを Git で管理し、`free_ebook_maker build --diff HEAD~1` で前回コミットからの差分のみを再ビルド。`--cache-dir ./.ebook_cache` を指定して、変更のない章のキャッシュを利用。これにより、大規模プロジェクト（5000ページ超）でもビルド時間を 90% 削減可能。`--cache-ttl 3600` でキャッシュの有効期限を設定。

- **アクセシビリティ対応の自動化**: `--a11y --lang ja` フラグを有効にすると、すべての画像に `alt` 属性が自動生成され（`--alt-model "gpt-4o-mini"` で外部 AI モデル利用）、`<nav>` 要素にページリストが自動挿入される。`--a11y-check` で WCAG 2.1 レベル AA の簡易チェックを実行し、問題点を JSON 形式（`a11y_report.json`）で出力。`--fix-a11y` で自動修正可能。
