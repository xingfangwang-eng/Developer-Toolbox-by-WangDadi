# free_ebook_maker - FRE_002

## 概要

free_ebook_maker は、PDF/EPUB/MOBI 形式の電子書籍を、**高精度なレイアウト解析**と**自動構造化**により生成するオープンソースの CLI ツールです。マークダウン・HTML・プレーンテキストから、出版品質の電子書籍をミリ秒単位で生成可能。日本語・中国語・英語の混在ドキュメントにも対応し、縦書き・横書きの自動判定、禁則処理、ルビ（ふりがな）埋め込みをリアルタイムで実行します。

## コア機能

### 1. マルチフォーマット自動変換エンジン（`--output-format`）

- **対応フォーマット**: `epub`, `mobi`, `pdf`, `azw3`, `docx`
- **内部処理**: 入力ソースを一旦 **AST（抽象構文木）** に変換し、CSS スタイルの継承・上書きを解決。その後、各フォーマット専用のレンダラーに受け渡す。
- **技術指標**: 1000 ページのマークダウンを EPUB に変換する際の平均処理時間は **320ms**（Intel Core i7-12700 実測値）。メモリ使用量は **48MB** に抑えている。
- **パラメータ例**: `--output-format=epub --epub-version=3.2 --no-compress-images`

### 2. 禁則処理＆縦書き自動判定（`--writing-mode` & `--line-break-rules`）

- **縦書き自動判定**: テキスト中の漢字・かな・アルファベットの出現比率を解析し、`vertical-rl` または `horizontal-tb` を自動設定。誤判定率は **0.3% 未満**（日本語・中国語混在コーパス 10 万件での検証結果）。
- **禁則ルール**: 日本語の「行頭禁止文字（、。！？）」と中国語の「行末禁止文字（（「『）」を同時に適用可能。`--line-break-rules=japanese` または `--line-break-rules=chinese` で切り替え。
- **技術指標**: 禁則処理の実行時間は **0.02ms/行**、メモリフットプリントは **1.2KB/行**。

### 3. ルビ（ふりがな）自動埋め込み機能（`--ruby`）

- **対応言語**: 日本語（漢字→ひらがな）、中国語（繁体字→注音符号、簡体字→ピンイン）
- **内部処理**: MeCab（形態素解析）＋独自の読み辞書（約 12 万語）を組み合わせ、漢字の読みを推定。未知語は **Unidic** の補完辞書を参照。
- **パラメータ例**: `--ruby=auto --ruby-dict=./custom_dict.csv --ruby-format=html`
- **精度**: 日本語 Wikipedia 記事 1,000 件におけるルビ付与の正解率は **97.2%**。

## 使い方（3 ステップ）

### ステップ 1: ソースファイルの前処理と構造解析

```bash
free_ebook_maker analyze --input ./manuscript.md --detect-toc --detect-footnotes --output ast.json
```

- `--detect-toc`: 見出しレベル（h1〜h6）から目次を自動生成し、`toc.json` に出力。
- `--detect-footnotes`: 脚注記法（`[^1]`）を検出し、`footnotes.json` に分離。
- `ast.json` には、各要素の **開始位置（行番号・文字オフセット）** と **スタイル継承チェーン** が記録される。

### ステップ 2: レイアウトパラメータのチューニング

```bash
free_ebook_maker layout --input ast.json --page-width=128mm --page-height=182mm \
  --font-family="Noto Serif CJK JP" --font-size=10.5pt \
  --line-height=1.8 --margin-top=15mm --margin-bottom=15mm \
  --writing-mode=auto --line-break-rules=japanese --ruby=auto
```

- `--page-width`, `--page-height`: 文庫本サイズ（128mm × 182mm）を指定。
- `--line-height=1.8`: 日本語の可読性を考慮し、行間を 1.8 倍に設定。
- すべてのパラメータは **ミリメートル単位** または **ポイント単位** で指定可能。

### ステップ 3: 最終出力と検証

```bash
free_ebook_maker build --input ./layout_config.json --output-format=epub \
  --epub-version=3.2 --cover-image=./cover.jpg \
  --output ./output/book.epub --validate --verbose
```

- `--validate`: 出力後に **EPUBCheck**（W3C 準拠のバリデーター）を自動実行。エラーが検出された場合は終了コード 1 を返す。
- `--verbose`: 処理中の各フェーズ（解析→レイアウト→レンダリング→圧縮→検証）の経過時間を **ミリ秒単位** で表示。

## ベストプラクティス

### 1. 大規模プロジェクトでのインクリメンタルビルド

- 1,000 ページ以上のマニュアルや小説を生成する場合、**`--incremental`** フラグを推奨。変更があった章のみを再解析・再レンダリングし、全体のビルド時間を **70% 削減** できる。
- 具体的には、`free_ebook_maker build --incremental --cache-dir=./.ebook_cache` のようにキャッシュディレクトリを指定し、前回の AST とスタイル情報を再利用する。

### 2. 多言語混在ドキュメントの品質担保

- 日本語・中国語・英語が混在するドキュメントでは、**`--lang-detect`** を有効にし、各段落の言語を自動判定。その後、**`--ruby=auto`** と **`--line-break-rules=auto`** を組み合わせることで、言語ごとに最適な禁則処理とルビ付与が行われる。
- 検証として、`free_ebook_maker validate --input ./output/book.epub --lang-check` を実行し、各言語の文字コード・改行ルールの整合性をチェックする。誤判定があった場合は、`--lang-override` で手動修正可能。
