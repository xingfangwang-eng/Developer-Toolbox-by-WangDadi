# free_ebook_maker - FRE_028

## 概要

free_ebook_maker は、Markdown、ReStructuredText、プレーンテキストなどの軽量マークアップ言語から、出版品質の EPUB 3.0 / Kindle (MOBI) / PDF を生成するオフライン型ビルドツールです。CSS組版エンジンに独自のレイアウトパーサーを採用し、日本語禁則処理（行頭禁則・行末禁則・分割禁止文字制御）をミリ秒単位で解決。単一のYAML設定ファイルで、縦書き・横書き、ノド側余白の自動調整、ルビ（傍注）の自動配置が可能です。Dockerコンテナとしても動作し、CI/CDパイプラインへの組み込みを前提としています。

## コア機能

### 1. マルチフォーマット組版エンジン（組版精度 0.01mm）
- **パラメータ**: `--page-width 148mm --page-height 210mm --margin-top 15mm --margin-bottom 20mm --gutter 8mm`
- 内部でPaged.jsベースのレンダリングパイプラインを実装。CSS `@page` ルールを直接解釈し、`orphans` / `widows` 制御、`break-before` / `break-after` の自動解決を行います。特に `widows: 3` 設定時は、段落最終行が3行未満でページ分割されるのを防止。結果として、プロの印刷所に提出可能なPDF（PDF/X-1a:2001準拠）を生成します。

### 2. メタデータ自動抽出 & DC元号対応
- **パラメータ**: `--metadata metadata.yaml --auto-extract-toc 3`
- 入力Markdownの見出し構造（h1〜h6）から目次（TOC）を自動生成。さらに、`<meta>` タグやYAML front matter から著者名、ISBN、発行日（和暦対応：令和5年 → 2023年）を抽出し、EPUBの `package.opf` に埋め込み。`--dc-date-format ja` フラグで出力日付を「2025年04月08日」形式に変換可能。

### 3. 画像最適化 & バリアフリー対応
- **パラメータ**: `--image-quality 85 --max-image-width 1200 --alt-text-auto`
- 内部でSharpライブラリを使用し、JPEG品質85%、最大幅1200pxで全画像を自動リサイズ。`--alt-text-auto` フラグを有効にすると、画像ファイル名（例：`figure-01-architecture.png`）から代替テキストを推測生成。さらに、`--voice-over-tts` フラグでEPUB内にSSMLタグを埋め込み、スクリーンリーダーでの読み上げ順序を制御可能。

## 使い方

### ステップ1: プロジェクト初期化 & 設定ファイル生成
```bash
free_ebook_maker init --project my-ebook --lang ja --output-dir ./output
```
- カレントディレクトリに `my-ebook/` が作成され、以下のファイルが自動生成されます。
  - `config.yaml`（組版設定）
  - `cover.png`（ダミー表紙）
  - `manuscript/`（原稿格納ディレクトリ）
- 設定ファイル例:
```yaml
book:
  title: "技術書サンプル"
  author: "王大地"
  language: ja
  writing_mode: vertical-rl # 縦書き
  font:
    main: "Noto Serif JP"
    size: 10pt
  page:
    width: 148mm
    height: 210mm
    margin:
      top: 20mm
      bottom: 25mm
      inside: 15mm # ノド側
      outside: 10mm # 小口側
```

### ステップ2: 原稿作成 & メタデータ注入
```bash
# Markdownファイルを作成
echo "# 第1章 はじめに" > manuscript/ch01.md
echo "これはテスト原稿です。" >> manuscript/ch01.md

# メタデータYAMLを作成
cat << 'EOF' > metadata.yaml
isbn: "978-4-XXXX-XXXX-X"
date: 2025-04-08
publisher: "WangDadi Publishing"
rights: "© 2025 WangDadi. All rights reserved."
EOF
```
- 原稿内の `<!-- pagebreak -->` コメントで強制改ページ可能。
- ルビ記法: `{ルビ対象|ルビ}`（例：`{東京|とうきょう}`）

### ステップ3: ビルド & バリデーション
```bash
free_ebook_maker build --config config.yaml --metadata metadata.yaml --format epub,pdf --validate-epub
```
- 内部で `epubcheck` を呼び出し、EPUB 3.0仕様に準拠しているか自動検証。
- `--validate-pdf` フラグでPDF/X-1a準拠チェックも可能。
- 出力例:
```
[INFO] 組版エンジン初期化完了 (0.023s)
[INFO] 画像最適化: 3枚処理 (0.412s)
[INFO] TOC自動生成: 15エントリ検出
[INFO] EPUB生成完了: output/my-ebook.epub (1.2MB)
[INFO] PDF生成完了: output/my-ebook.pdf (4.5MB)
[INFO] バリデーション: エラー0、警告2（推奨: 画像alt属性を手動確認）
```

## ベストプラクティス

### プラクティス1: バージョン管理と差分ビルド
- Gitリポジトリで原稿を管理し、`free_ebook_maker build --diff-only` を使用すると、前回ビルドから変更のあったファイルのみ再処理。大規模書籍（1000ページ以上）でビルド時間を80%削減可能。具体的な運用例:
  ```bash
  git add manuscript/
  git commit -m "第3章を加筆修正"
  free_ebook_maker build --diff-only --cache-dir .build-cache
  ```

### プラクティス2: マルチターゲット出力と自動デプロイ
- CI/CDパイプライン（GitHub Actions等）で、プルリクエスト作成時に自動ビルドし、以下の3種類を生成:
  1. **レビュー用PDF**（ウォーターマーク付き、解像度150dpi）
  2. **配信用EPUB**（DRMフリー、画像圧縮率80%）
  3. **印刷所入稿用PDF**（PDF/X-1a、裁ち落とし3mm、色空間CMYK）
- 設定例（`.github/workflows/build.yml`）:
  ```yaml
  - name: Build for review
    run: free_ebook_maker build --profile review --output-dir ./artifacts/review
  - name: Build for distribution
    run: free_ebook_maker build --profile distribution --output-dir ./artifacts/dist
  ```
