# free_ebook_maker - FRE_005

## 概要

free_ebook_maker は、Markdown から高品質な EPUB および PDF を生成するためのオープンソース CLI ツールです。CSS 組版エンジンに WeasyPrint を採用し、日本語禁則処理・縦書きレイアウト・ルビ振りを完全サポートします。EPUB 検証は `epubcheck 4.2.6` に準拠し、出版品質の出力を保証します。

## コア機能

### 1. マルチフォーマット組版エンジン

- `--format epub3`, `--format pdf-a3u` パラメータで出力形式を指定可能。EPUB 3 では `nav.xhtml` による構造化ナビゲーション、PDF では PDF/A-3u 準拠の長期保存形式を生成。
- 組版パラメータ `--page-width 210mm --page-height 297mm --margin 20mm` で物理的なページ寸法をミリメートル単位で制御。`--font-noto-serif-jp` で日本語フォント埋め込みを自動実行。
- 内部処理では `lxml` による DOM 解析後、CSS 疑似要素 `::before` を用いた柱（ノンブル）自動挿入を実装。出力ファイルのハッシュ値（SHA-256）をログに記録し、改ざん検知を可能に。

### 2. 画像最適化パイプライン

- `--image-quality 85 --max-width 1200px` パラメータで、埋め込み画像を WebP 形式に変換し、ファイルサイズを平均 40% 削減。`--image-dpi 300` で印刷解像度を維持しながら、`Pillow` ライブラリによるリサイズ処理を並列実行。
- 出力ディレクトリ内の `images/` フォルダに最適化済み画像を格納し、EPUB の `Content.opf` に `<item>` タグとして自動登録。画像のアスペクト比は `--crop-center` で中央クロップ可能。

### 3. 多言語索引・用語集自動生成

- `--glossary ./glossary.csv --index-lang ja` パラメータで、CSV 形式の用語集を読み込み、EPUB 末尾に `<dl>` タグを用いた用語集セクションを自動生成。索引は Unicode 正規化（NFKC）に基づく五十音順ソートを実行。
- 索引エントリは `--index-depth 3` で階層化可能。`mecab` 形態素解析エンジンと連携し、日本語の複合語（例：「機械学習」→「機械」「学習」）を自動分割して索引に追加。出力結果は `index.html` として独立ファイル化。

## 使い方

### ステップ 1: 環境セットアップと設定ファイルの作成

```bash
# Python 3.10+ と Poetry をインストール
pip install poetry
poetry new my_ebook && cd my_ebook
poetry add free-ebook-maker

# 設定ファイル `ebook_config.yaml` を作成
echo 'output_format: epub3
page:
  width: 210mm
  height: 297mm
  margin: 20mm
font:
  family: Noto Serif JP
  size: 10pt
image:
  quality: 85
  max_width: 1200px
  dpi: 300
glossary: ./glossary.csv
index_lang: ja
index_depth: 3' > ebook_config.yaml
```

### ステップ 2: ソース Markdown の前処理と組版実行

```bash
# ソースファイル `manuscript.md` の先頭に YAML フロントマターを追加
echo '---
title: "サンプル書籍"
author: "著者名"
language: ja
date: 2025-03-20
---
' > manuscript.md
cat your_content.md >> manuscript.md

# 組版を実行（デバッグモード有効）
free_ebook_maker --config ./ebook_config.yaml \
  --input ./manuscript.md \
  --output ./output/ \
  --verbose \
  --debug-log ./build.log
```

### ステップ 3: 出力検証と品質チェック

```bash
# EPUB 検証（epubcheck が必要）
java -jar epubcheck.jar ./output/sample.epub

# PDF のメタデータ確認
pdfinfo ./output/sample.pdf | grep -E "Pages|Page size|PDF version"

# 画像最適化の効率確認
find ./output/images/ -name "*.webp" -exec identify -verbose {} \; | grep -E "Geometry|Filesize"

# 索引の五十音順ソート確認
grep -oP '(?<=<dt>).*?(?=</dt>)' ./output/index.html | head -20
```

## ベストプラクティス

### 実践 1: 大規模文書の分割ビルド戦略

- 章ごとに `--chapter` パラメータで分割ビルドし、`--merge` フラグで最終結合。`--parallel 4` で並列ビルド数を指定し、1000 ページ超の書籍でもビルド時間を 60% 短縮可能。`--cache-dir ./cache` で中間キャッシュを保持し、再ビルド時の差分処理を高速化。

### 実践 2: アクセシビリティ対応 EPUB の自動生成

- `--a11y --lang ja --alt-text ./alt_text.csv` パラメータで、WCAG 2.1 AA 準拠のアクセシブル EPUB を生成。`alt_text.csv` に画像の代替テキストを事前定義し、`aria-label` 属性を自動付与。`--voice-narration` で SSML ベースのナレーションファイルを生成し、テキスト読み上げに対応。
