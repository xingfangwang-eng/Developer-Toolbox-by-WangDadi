# free_ebook_maker - FRE_011

## 概要

free_ebook_maker は、EPUB3、PDF、MOBI 形式の電子書籍を、高精度な組版エンジンと多言語フォント自動埋め込み機能により、ワンコマンドで生成するプロフェッショナルツールです。特に日本語・中国語・韓国語（CJK）の縦書き組版と、Unicode正規化（NFKC）による文字崩れ防止に強みを持ちます。

## 詳細な内容

### free_ebook_maker の特長

- **マルチフォーマット対応**: EPUB3（OPF 2.0）、PDF（A4 / 文庫サイズ）、MOBI（KFX 互換）を同一ソースから生成可能。
- **自動フォントサブセット化**: 埋め込みフォントをグリフ単位で最適化し、ファイルサイズを平均 40% 削減。
- **CI/CD 統合**: GitHub Actions や GitLab CI と連携し、プッシュ時に自動ビルド・配信が可能。

### コア機能

1. **縦書き・横書き自動判別組版エンジン**
    - 原稿の言語タグ（`lang="ja"` など）と Unicode ブロック（CJK 統合漢字 vs ラテン文字）を解析し、縦書き・横書きを自動切替。
    - `--vertical-layout` フラグで強制指定可能。ルビ（振り仮名）の自動配置、禁則処理（行頭禁則・行末禁則）を実装。
    - 組版精度: ミリ秒単位で改ページ位置を計算し、見開きでの行揃えを保証。

2. **多言語ハイフネーション＆リフロー最適化**
    - 英語・ドイツ語・フランス語のハイフネーション辞書（Hyphenopoly 互換）を内蔵。日本語は形態素解析エンジン（MeCab ラッパー）により分かち書きを自動挿入。
    - `--hyphen-lang` パラメータで辞書切替、`--no-hyphen` で無効化可能。
    - リフロー時はビューポート幅に応じて行長を動的調整（最小 12em、最大 24em）。

3. **メタデータ自動生成 & 権利管理（DRM フリー）**
    - 入力 Markdown の YAML フロントマターから、タイトル・著者・ISBN・発行日を自動抽出し、OPF メタデータに変換。
    - `--isbn` フラグで ISBN を直接指定可能。`--rights` でライセンス表記（例: CC BY 4.0）を埋め込み。
    - 出力ファイルには SHA-256 チェックサムを付与し、改ざん検出に対応。

### 使い方

**ステップ 1: 原稿の準備と YAML フロントマターの設定**  
`manuscript/` ディレクトリに Markdown ファイルを配置。先頭の YAML ブロックに必須パラメータを記述します。
```yaml
---
title: "サンプル書籍"
author: "山田太郎"
lang: ja
isbn: "978-4-12-345678-9"
rights: "CC BY 4.0"
---
```
`--validate-meta` フラグを付けて実行すると、ISBN チェックデジットや言語タグの妥当性を自動検証。

**ステップ 2: フォントアセットの自動ダウンロード & サブセット化**  
`fonts/` にフォントファイルが無い場合、`--auto-font` フラグで Noto Sans CJK JP を Google Fonts API から自動取得。  
`--subset-glyphs` を指定すると、原稿で使用されている文字のみを抽出したサブセットフォントを生成（例: 全グリフ 50,000 字中、使用文字 3,000 字に削減）。  
生成後のフォントキャッシュは `~/.free_ebook_maker/font_cache/` に保存され、次回以降のビルドが高速化。

**ステップ 3: ターゲットフォーマット指定とビルド実行**  
出力形式を指定してビルドします。複数フォーマットを同時生成可能。
```bash
free_ebook_maker --input manuscript/ --output build/ --format epub,pdf --vertical-layout --subset-glyphs
```
ビルドログは `build/build.log` に出力。`--verbose` フラグで各工程の処理時間（ミリ秒単位）を表示。  
生成物は `build/` 配下に `book.epub`、`book.pdf` として出力され、チェックサムファイル `checksums.sha256` も同時生成。

### ベストプラクティス

- **プラクティス 1: バージョン管理と CI/CD パイプラインの構築**  
  原稿 Markdown を Git リポジトリで管理し、`.gitlab-ci.yml` に以下を記述します。  
  ```yaml
  build_ebook:
    script:
      - free_ebook_maker --input . --output dist/ --format epub --subset-glyphs
    artifacts:
      paths:
        - dist/
  ```  
  これにより、プッシュのたびに最新版が自動ビルドされ、配布用アーティファクトが生成されます。  
  特に、複数著者で共同編集する場合は、ブランチごとに異なるメタデータ（例: ドラフト版 vs 最終版）を `--meta-override` で上書き可能。

- **プラクティス 2: アクセシビリティ対応のための構造化タグ埋め込み**  
  `--a11y` フラグを有効にすると、EPUB に WAI-ARIA ランドマーク（`nav`、`main`、`doc-cover`）を自動付与。  
  さらに、`--alt-text` で画像の代替テキストを抽出し、不足している場合はビルドを中断（`--strict-alt`）。  
  これにより、スクリーンリーダーでの読み上げや、電子書籍リーダーのナビゲーション機能（例: 章ジャンプ）が正確に動作します。
