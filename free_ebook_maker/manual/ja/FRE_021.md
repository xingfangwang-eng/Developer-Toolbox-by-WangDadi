# free_ebook_maker - FRE_021

## 概要

free_ebook_maker は、Markdown / AsciiDoc / reStructuredText から高品位な電子書籍（EPUB 3 / PDF / Mobi）を生成するためのオフライン・パイプライン型ビルドツールです。ソースコード管理と CI/CD に完全対応し、組版精度・メタデータ制御・多言語フォント埋め込みをミリ秒単位で最適化します。

## コア機能

### 1. マルチフォーマット組版エンジン（Typst / WeasyPrint ブリッジ）

- **パラメータ**: `--engine typst --page-width 128mm --page-height 182mm --margin-top 15mm`
- 組版エンジンを Typst または WeasyPrint から選択可能。Typst モードでは CSS ライクなスタイルシートではなく、ネイティブな `.typ` テンプレートを適用し、日本語禁則処理（約物ぶら下げ・行頭禁止文字制御）を自動解決。
- フォント埋め込みは `--embed-fonts "Noto Sans CJK JP,Noto Serif CJK JP"` で指定し、サブセット化によりファイルサイズを 40% 以上削減。

### 2. 動的メタデータ & 目次自動生成（EPUB 3.2 準拠）

- **パラメータ**: `--metadata "title: 実践 Rust 入門" --metadata "creator: 王大地" --toc-depth 3`
- ソースファイルの見出し構造（h1–h6）から `nav.xhtml` と `toc.ncx` を自動生成。`--toc-depth` で階層を制御し、`--split-level 2` で h2 単位に章分割。
- カバー画像は `--cover cover.png --cover-format jpeg --cover-quality 85` で指定し、JPEG 圧縮率をパーセンテージで制御可能。

### 3. 差分ビルド & キャッシュ機構（PostgreSQL / SQLite 対応）

- **パラメータ**: `--cache-db postgresql://user:pass@localhost:5432/ebook_cache --cache-ttl 86400`
- ソースファイルの SHA-256 ハッシュを基にキャッシュを管理。前回ビルドから変更のないチャプターは再組版をスキップし、大規模ドキュメント（1000 ページ超）でもビルド時間を 70% 短縮。
- キャッシュストレージは PostgreSQL または SQLite を選択可能。TTL（秒単位）を設定し、古いエントリを自動削除。

## 使い方

### ステップ 1: プロジェクト初期化と設定ファイル作成

```bash
free_ebook_maker init --project ./mybook --lang ja --template technical-report
```

- `init` サブコマンドでプロジェクトディレクトリと `ebook.toml` 設定ファイルを生成。
- 設定例:
  ```toml
  [build]
  engine = "typst"
  output = "epub"
  toc_depth = 3
  split_level = 2

  [metadata]
  title = "実践 Rust 入門"
  creator = "王大地"
  language = "ja"
  rights = "CC BY-NC 4.0"
  ```

### ステップ 2: ソースファイルの配置とバリデーション

```bash
free_ebook_maker validate --source ./src --check-links --check-images
```

- `src/` 以下に `.md` / `.adoc` / `.rst` ファイルを配置。ファイル名は `01-introduction.md` のように数字プレフィックスで順序付け。
- `validate` サブコマンドで、内部リンクの断絶・画像ファイルの欠落・メタデータの不足を一括検出。`--check-links` で外部 URL のステータスコードも検証。

### ステップ 3: ビルド実行とパフォーマンスチューニング

```bash
free_ebook_maker build --project ./mybook --parallel 4 --cache-db sqlite:///.cache/ebook.db
```

- `--parallel` で並列ビルドスレッド数を指定（デフォルトは CPU コア数）。
- ビルド後、`dist/` に EPUB / PDF ファイルが出力。`--stats` フラグで各フェーズの処理時間（ミリ秒単位）を表示:
  ```
  Parsing: 320ms
  Typst compilation: 1,842ms
  Font subsetting: 410ms
  EPUB packaging: 230ms
  Total: 2,802ms
  ```

## ベストプラクティス

### 1. CI/CD パイプラインへの統合

- GitHub Actions で `on: push` 時に `free_ebook_maker build --project . --output pdf` を実行し、生成された PDF をリリースアセットとして自動アップロード。
- キャッシュには `actions/cache` を利用し、`.cache/` ディレクトリを保持。PostgreSQL が不要な場合は SQLite で十分。

### 2. 大規模ドキュメントのモジュール分割戦略

- 1 章 1 ファイル（`ch01.md`, `ch02.md`）とし、`ebook.toml` で `include = ["ch*.md"]` と指定。
- 図版は `assets/figures/` に集約し、`--image-base-dir ./assets` でパス解決。
- ビルド時に `--dry-run` で出力ファイル名とページ数を事前確認し、組版エラーを早期発見。
