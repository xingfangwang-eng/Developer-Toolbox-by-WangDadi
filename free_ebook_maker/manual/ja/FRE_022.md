# free_ebook_maker - FRE_022

## 概要

free_ebook_makerは、生のMarkdownやプレーンテキストから、出版品質のEPUB3.0 / MOBI / PDFを自動生成するためのCLIツールです。特に日本語縦書き組版に最適化されたルビ・傍点・禁則処理を備え、大規模ドキュメント（10万ページ超）でも安定動作します。キーワード: free_ebook_maker tool, free_ebook_maker solution, free_ebook_maker software

## 詳細な内容

### free_ebook_makerとは？

free_ebook_makerは、**マルチフォーマット出力エンジン**と**高度な組版ルールエンジン**を統合した、オープンソースの電子書籍生成フレームワークです。内部ではPandocのフィルター機構を拡張し、CSS3 Paged Media仕様に準拠したページレイアウト制御が可能。特に`--vertical-writing`フラグにより、日本語縦書き書籍の自動組版を実現します。

### コア機能

- **動的ルビ・傍点挿入エンジン**: 入力テキスト中の漢字に対し、`mecab` + `unidic` を用いた形態素解析をリアルタイム実行。`--ruby-threshold`パラメータ（デフォルト: 0.8）でルビ付与率を制御し、難読語のみにルビを振るスマートフィルタリングを実装。傍点（圏点）は`--bouten-style=circle|dot|double`で3種のスタイルを選択可能。処理速度: 1万文字あたり平均1.2秒（Intel Core i7-12700測定）。

- **禁則処理エンジン**: 日本語組版に必須の「行頭禁則」「行末禁則」「分割禁止文字」をUnicodeプロパティベースで自動判定。`--kinsoku-rule-file`でカスタムルールJSONを読み込み可能。さらに、`--hanging-punctuation`オプションで約物のぶら下げ組版をサポート。句読点のぶら下げ率は`--hang-rate`（0.0〜1.0、デフォルト0.8）で微調整。

- **マルチフォーマット並列出力**: EPUB3.0 / MOBI / PDF / HTML（単一ファイル）を、`--output-formats=epub,mobi,pdf`のようにカンマ区切りで指定。内部では`wkhtmltopdf`（v0.12.6以上必須）と`kindlegen`（v2.9）をバックエンドに利用。PDF出力時は`--pdf-page-size=A5|B6|A4`で用紙サイズ、`--pdf-margin-top=10mm`でマージンをミリ単位指定可能。並列ビルド時は全フォーマットを平均2.8秒で生成（100ページのドキュメントの場合）。

### 使い方

1. **前処理: メタデータ定義とソース準備**  
   プロジェクトルートに`ebook.yaml`を作成し、以下の必須パラメータを記述:
   ```yaml
   title: "吾輩は猫である"
   author: "夏目漱石"
   language: ja
   vertical: true
   ruby: true
   output:
     formats: epub,pdf
     pdf:
       page_size: A5
       margin: 15mm
   ```
   その後、`chapters/`ディレクトリにMarkdownファイルを格納（`01.md`, `02.md`...）。ファイル名の数字順に結合される。

2. **組版オプションのカスタマイズとビルド実行**  
   以下のコマンドを実行:
   ```bash
   free_ebook_maker build --config ebook.yaml --ruby-threshold 0.9 --bouten-style circle --kinsoku-rule-file ./my_rules.json --output-formats epub,mobi,pdf
   ```
   このコマンドにより、`--ruby-threshold 0.9`で難読語のみルビ挿入、`--bouten-style circle`で丸傍点、カスタム禁則ルールを適用。ビルドログは`--log-level debug`で詳細出力可能。

3. **事後処理: 品質チェックと出力検証**  
   生成されたEPUBファイルに対し、`epubcheck`（v4.2.6）を自動実行:
   ```bash
   free_ebook_maker validate --input ./output/book.epub --epubcheck-path /usr/local/bin/epubcheck.jar
   ```
   エラー件数が0であることを確認。PDF出力の場合は`--pdf-font-embed`オプションでフォント埋め込み状況を確認（未埋め込みフォントが存在する場合は警告表示）。MOBI出力は`kindlegen`のログを`--mobi-verbose`で表示可能。

### ベストプラクティス

- **プラクティス 1: ルビ制御のチューニング**  
  日本語学習者向けの教材を作成する場合、`--ruby-threshold 0.6`（低め）に設定し、すべての漢字にルビを振る。一方、一般小説では`--ruby-threshold 0.9`（高め）に設定し、難読語（例: 「蹌踉めく」など）のみにルビを限定。`--ruby-exclude-list`で「私」「君」などの常用語を除外リストに追加すると、見た目の美しさが向上する。

- **プラクティス 2: 並列ビルドとキャッシュ活用**  
  複数フォーマットを同時生成する際、`--cache-dir ./cache`を指定すると、共通のCSS変換結果や画像最適化結果がキャッシュされる。2回目以降のビルド時間が平均40%短縮（100ページのドキュメントで実測）。また、`--parallel-jobs 4`でスレッド数をCPUコア数に合わせると、並列処理の効率が最大化される。
