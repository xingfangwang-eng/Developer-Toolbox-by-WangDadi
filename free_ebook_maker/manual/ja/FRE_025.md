# free_ebook_maker - FRE_025

## 概要

`free_ebook_maker` は、Markdown / AsciiDoc / ReStructuredText から高品位な EPUB 3 / PDF / Mobi を生成する、オフライン完結型のパブリッシングパイプラインです。日本語組版における禁則処理・縦中横・ルビ・圏点を完全サポートし、**ミリ秒単位のインクリメンタルビルド** と **PostgreSQL による版管理** を特徴とします。

## コア機能

### 1. マルチフォーマット組版エンジン & 禁則ルール自動適応

- **パラメータ**: `--target-lang ja`, `--line-rules strict`, `--ruby-position inter-character`
- **技術詳細**:
  - 内部で `CJK Line Breaking Algorithm` (JIS X 4051) を実装。約 4,000 種類の禁則文字パターンを事前定義。
  - 縦中横（たてちゅうよこ）を自動検出し、半角数字・アルファベットを 1 文字分に圧縮。
  - ルビ（ふりがな）は `<ruby>` タグからパースし、`writing-mode: vertical-rl` の EPUB 3 固定レイアウトにも対応。
- **パフォーマンス指標**: 1,000 ページ相当の原稿を **320 ミリ秒** で組版完了（Intel Core i7-12700 実測）。

### 2. インクリメンタル差分ビルド & 静的解析キャッシュ

- **パラメータ**: `--incremental --cache-dir .cache --hash-algo blake3`
- **技術詳細**:
  - ソースファイルの BLAKE3 ハッシュを `.cache/manifest.json` に保存。変更のあったファイルのみ再パース。
  - 画像アセットは `ImageMagick` の `identify` 出力をキャッシュし、解像度・色空間の変更のみ再変換。
  - 全ファイルの依存関係グラフを DAG（有向非巡回グラフ）で管理し、並列ビルドを自動スケジューリング。
- **パフォーマンス指標**: 変更ファイル 1 件のみのビルドは **平均 12 ミリ秒**（SSD + 32GB RAM 環境）。

### 3. PostgreSQL 版管理 & メタデータ自動生成

- **パラメータ**: `--db-uri postgresql://user:pass@localhost:5432/ebook_meta`, `--metadata-mode auto`
- **技術詳細**:
  - 各ビルド時に、`books` テーブルに `title`, `author`, `isbn`, `revision_hash`, `build_timestamp` を INSERT。
  - `revisions` テーブルで差分管理。`pg_diff` 関数を用いて、前回ビルドからの変更行数をミリ秒単位で記録。
  - メタデータは `ONNX Runtime` による軽量 NER（固有表現抽出）モデルでタイトル・著者を自動推定。精度 **F1 0.94**。
- **パフォーマンス指標**: 10,000 冊分のメタデータ検索が **2.1 ミリ秒**（インデックス適用済み）。

## 使い方

### ステップ 1: 環境初期化 & データベーススキーマ作成

```bash
# プロジェクトルートで実行
free_ebook_maker init \
  --db-uri postgresql://ebook_user:securepass@localhost:5432/ebook_meta \
  --cache-dir .cache \
  --schema-version 3.2.1
```

- `init` コマンドが自動で `books` / `revisions` / `assets` テーブルを生成。
- `.cache/manifest.json` に空の DAG とハッシュテーブルを作成。

### ステップ 2: ソースファイルのバリデーション & 組版設定

```bash
# 全ファイルの構造チェック（厳格モード）
free_ebook_maker validate \
  --src ./manuscript \
  --ext .md,.adoc,.rst \
  --strict-mode \
  --line-rules jis-x-4051 \
  --ruby-position inter-character \
  --max-image-width 1200px \
  --output-format epub3
```

- 各ファイルの見出し階層、画像リンクの存在、ルビの整合性を検証。
- 違反があればエラーコード（例: `E-0402: 禁則文字列が段落末尾に存在`）と行番号を出力。

### ステップ 3: インクリメンタルビルド & PostgreSQL への版登録

```bash
# 差分のみビルドし、メタデータを自動登録
free_ebook_maker build \
  --incremental \
  --hash-algo blake3 \
  --db-uri postgresql://ebook_user:securepass@localhost:5432/ebook_meta \
  --metadata-mode auto \
  --output ./dist/my_ebook.epub
```

- 変更のないファイルはスキップ。ビルド時間は **変更ファイル数 × 平均 12 ミリ秒**。
- 正常終了後、`books` テーブルにレコードが追加され、`revisions` テーブルに差分行数が記録される。

## ベストプラクティス

### 1. 大規模プロジェクトでは `--incremental` と PostgreSQL 版管理を組み合わせる

- 数百ファイル・数千ページのマニュアルや技術書では、フルビルドに数分かかる場合があります。`--incremental` で変更箇所のみ再組版し、PostgreSQL に各版のハッシュとビルド日時を保存することで、**「どの版でどのファイルが変わったか」** を SQL で瞬時にトレース可能です。
- 例: `SELECT * FROM revisions WHERE build_timestamp > '2025-04-01' AND changed_lines > 50;`

### 2. 画像アセットは `--max-image-width` と `--image-quality` でサイズ最適化

- EPUB のファイルサイズは配信速度やストレージコストに直結します。`--max-image-width 1200px --image-quality 85` を指定することで、**画質を維持しつつファイルサイズを平均 60% 削減** できます（JPEG 圧縮 + リサイズ）。
- 特に写真やスクリーンショットが多いドキュメントでは、ビルド時間も短縮されます（画像変換がスキップされるため）。
