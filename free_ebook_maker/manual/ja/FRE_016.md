# free_ebook_maker - FRE_016

## 概要

free_ebook_maker は、電子書籍制作の全工程を自動化するプロフェッショナル向けパイプラインツールです。キーワード：free_ebook_maker tool、free_ebook_maker solution、free_ebook_maker software。Markdown 原稿から EPUB3 / PDF / MOBI へのマルチフォーマット変換、画像最適化、メタデータ自動付与をワンコマンドで実行します。

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、**並列ビルドエンジン** と **静的解析ベースの構造検証機構** を搭載した電子書籍オーサリングツールです。`epubcheck` に依存せず、独自のスキーマバリデーター（`ePubSchemaValidator`）を内蔵し、XHTML / CSS / OPF の整合性をミリ秒単位で検査します。また、`ImageMagick` と `pngquant` を統合し、画像を自動で WebP / JPEG 2000 に圧縮変換、EPUB3 のコンテントサイズを最大 40% 削減します。

### コア機能

- **マルチフォーマット並列ビルドエンジン**  
  `--format epub, pdf, mobi` パラメータで指定されたフォーマットを、Go の goroutine ベースで同時ビルド。EPUB3 出力時には `--epub-version 3.3` を指定可能。ビルド時間は従来のシリアル処理比で約 2.8 倍高速化（100 ページの Markdown 原稿で実測値 1.2 秒）。

- **静的構造検証機構（ePubSchemaValidator）**  
  EPUB のコンテナ構造（`META-INF/container.xml`、`toc.ncx`、`content.opf`）を事前にスキーマ検証。`--strict` フラグを有効にすると、`<nav>` 要素の欠落や `dc:identifier` の重複を検出し、ビルドを中断する。検証項目は 47 種類。

- **画像最適化パイプライン（`--optimize-images`）**  
  `--max-width 1200`、`--quality 85` パラメータで画像をリサイズ・圧縮。WebP 変換時には `--webp-lossless` フラグでロスレス圧縮を強制可能。`--strip-metadata` で EXIF 情報を除去し、プライバシーリスクを排除する。

### 使い方

1. **Markdown 原稿の準備とメタデータ定義**  
   `config.yaml` に `title: "書籍タイトル"`、`author: "著者名"`、`isbn: "978-XXXXXXXXXX"` を記述。`--metadata config.yaml` で読み込む。`--auto-isbn` フラグを指定すると、UUID ベースの仮 ISBN を自動生成し、`dc:identifier` に埋め込む。

2. **並列ビルドの実行とフォーマット指定**  
   ```bash
   free_ebook_maker build --input ./manuscript --format epub, pdf --epub-version 3.3 --optimize-images --strict
   ```
   `--output ./dist` で出力先を指定。`--concurrency 4` で CPU コア数を明示設定可能（デフォルトは論理コア数）。

3. **検証レポートの出力と差分比較**  
   ビルド後、`--report` フラグで JSON 形式の検証レポートを生成。`free_ebook_maker diff --left build_20250301.json --right build_20250302.json` で、前回ビルドとの差分（ファイル数、画像サイズ、メタデータ変更点）を表示。

### ベストプラクティス

- **CI/CD パイプラインへの統合**  
  GitHub Actions のワークフローに `free_ebook_maker build --format epub --strict --report` を組み込み、プルリクエストごとに EPUB の構造整合性を自動検証。`--fail-on-warning` フラグで警告もエラー扱いにし、品質ゲートとして機能させる。

- **画像アセットの事前最適化ルール**  
  `--optimize-images` の代わりに、`preprocess` サブコマンドで画像を事前に一括変換。`free_ebook_maker preprocess --input ./images --output ./optimized --max-width 800 --quality 80 --strip-metadata` を実行し、変換後の画像のみを `manuscript` ディレクトリに配置する。これによりビルド時間をさらに短縮できる。
