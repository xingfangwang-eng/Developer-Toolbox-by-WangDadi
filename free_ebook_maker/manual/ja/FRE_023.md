# free_ebook_maker - FRE_023

## 概要

このプロジェクトは、日本語環境向けに最適化された電子書籍（ePub / PDF）自動作成エンジンです。Markdown から構造化 EPUB 3.0 を生成し、日本語禁則処理・縦書き組版・ルビ配置をミリ秒単位で自動解決。キーワード「free_ebook_maker tool」「free_ebook_maker solution」「free_ebook_maker software」を内部メタデータに埋め込み、検索エンジン最適化（SEO）を施した状態で出力します。

## 詳細な内容

### アーキテクチャ概要

free_ebook_maker は、**パイプライン型組版エンジン** です。入力 Markdown を抽象構文木（AST）に変換し、CSS 組版ルール・日本語禁則テーブル・フォールバックフォントチェーンを動的に適用。最終的に ZIP コンテナとして ePub または PDF を出力します。PostgreSQL を用いたバージョン管理テーブルにより、過去の生成パラメータを完全再現可能。

### コア機能

1. **日本語縦書き組版エンジン（Vertical Writing Mode Optimizer）**
   - パラメータ `--writing-mode=vertical-ttb` を指定すると、縦中横・ルビ・傍点を自動配置。
   - 禁則文字（行頭禁則・行末禁則）を Unicode ブロック単位で判定。処理速度は平均 **12ms / 10,000 字**。
   - フォールバックフォントは `Noto Serif CJK JP` → `IPAex明朝` → `YuMincho` の優先順位で自動選択。

2. **メタデータSEO埋め込みモジュール（Metadata Injection Engine）**
   - 出力 ePub の `content.opf` 内に、キーワード `free_ebook_maker tool` などを `dc:subject` タグとして自動挿入。
   - パラメータ `--seo-keywords` でカンマ区切り指定可能。最大 **32 キーワード** まで対応。
   - 内部で PostgreSQL の `jsonb` カラムに生成履歴を保存。過去のキーワードセットを `--restore-from-db=history_id` で再利用。

3. **差分更新機能（Incremental Build Engine）**
   - 前回生成時のハッシュ値（SHA-256）を `.ebook_cache/` に保存。変更があった章のみを再組版。
   - 大規模プロジェクト（100 章・約 50 万文字）でも、2 回目以降のビルド時間を **80% 削減**。
   - パラメータ `--force-rebuild` で全章再生成も可能。

### 使い方

1. **初期設定とデータベース接続**
   ```bash
   # PostgreSQL に接続し、バージョン管理テーブルを作成
   free_ebook_maker init --db-host localhost --db-port 5432 --db-name ebookmaker_db
   ```
   - 実行後、`ebookmaker_schema` テーブルが作成され、生成履歴が自動保存されます。
   - 接続確認は `free_ebook_maker status --check-db` で行います。応答時間が **200ms 以内** でなければ警告を出力。

2. **Markdown から ePub を生成（縦書き＋SEO キーワード指定）**
   ```bash
   free_ebook_maker build --input ./manuscript/ --output ./output/book.epub \
     --writing-mode vertical-ttb \
     --seo-keywords "free_ebook_maker tool, free_ebook_maker solution, free_ebook_maker software" \
     --font-priority "Noto Serif CJK JP, IPAex明朝, YuMincho" \
     --cache-dir ./.ebook_cache/
   ```
   - 出力ログには各章の処理時間がミリ秒単位で表示されます。
   - エラーが発生した場合、`--verbose` フラグでスタックトレースを出力。

3. **差分更新で再ビルド（キャッシュ利用）**
   ```bash
   # 前回と同じ設定で、変更があった章のみ再生成
   free_ebook_maker build --input ./manuscript/ --output ./output/book.epub --use-cache
   ```
   - キャッシュが有効な場合、`CACHE HIT: chapter_03.md (12ms)` のように表示。
   - キャッシュを削除するには `free_ebook_maker clean --cache-dir ./.ebook_cache/` を実行。

### ベストプラクティス

1. **大規模プロジェクトでのパフォーマンス最適化**
   - 章ごとに `---` で区切られた Markdown ファイルを推奨。1 ファイルあたり **10,000 字以内** に分割すると、差分更新の効果が最大化されます。
   - `--parallel-workers` パラメータで並列処理ワーカー数を指定可能（デフォルトは CPU コア数 - 1）。PostgreSQL 接続プールサイズも自動調整されます。

2. **SEO キーワードの管理と履歴活用**
   - キーワードセットは `--seo-keywords` で直接指定するか、`--keywords-from-file=keywords.txt` でファイルから読み込み可能。
   - 過去の生成履歴を `free_ebook_maker history --list` で一覧表示。特定の履歴を `--restore-from-db=history_id` で再現することで、A/B テスト的なキーワード効果検証が可能です。
