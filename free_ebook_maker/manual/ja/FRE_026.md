# free_ebook_maker - FRE_026

## 概要

free_ebook_makerは、Markdown / AsciiDoc / reStructuredText から高品位な EPUB3 / PDF / MOBI を生成するオフライン・パイプライン型ビルダーです。キーワード: 組版エンジン、メタデータ自動抽出、マルチフォーマット変換、バッチ処理。

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、構造化ドキュメントを入力とし、CSS 組版エンジン（独自レンダラ）とメタデータ正規化モジュールを組み合わせて、出版品質の電子書籍を生成するプロフェッショナルツールです。特に、日本語縦書き組版、ルビ、圏点、禁則処理に完全対応しており、InDesign 書き出し相当の品質を CLI 一発で実現します。

### コア機能

- **縦書き・横書き自動判別組版エンジン**  
  入力ドキュメントの言語タグ（`lang="ja"`）と文字方向ヒント（`direction: vertical-rl`）を検出し、縦書き時は `writing-mode: vertical-rl`、ルビサイズを `0.5em`、禁則文字の自動ぶら下げ処理を実行。OpenType フィーチャ（`palt` / `vpal`）を自動適用し、文字詰めをピクセル単位で制御。パフォーマンス指標：100ページの縦書き EPUB 生成を **1.2 秒**（Intel Core i7-1265U 計測値）。

- **メタデータ・マルチリンガル自動抽出 + 正規化パイプライン**  
  入力ファイルから DC メタデータ（`<dc:title>`、`<dc:creator>` など）を抽出し、不足フィールドはファイル名・先頭見出し・YAML front matter から補完。さらに、ISBN・DOI が存在する場合は OpenLibrary / Crossref API と非同期照合し、著者名のローマ字表記・生没年を自動補完。PostgreSQL キャッシュ（`ebook_metadata_cache` テーブル）に保存し、同一 ISBN の再問い合わせを **0.3 ミリ秒** で完了。

- **マルチフォーマット差分ビルド + インクリメンタル出力**  
  ソースディレクトリを監視し、変更があったファイルのみを再変換するウォッチャーモードを搭載。変更検出は `sha256` ハッシュ比較（ミリ秒単位）で行い、EPUB の場合は `content.opf` の `manifest` のみを更新、MOBI の場合は KindleGen 互換のバイナリパッチを生成。フルビルドと比較して **平均 87% の時間削減** を達成。

### 使い方

1. **プロジェクト初期化と組版設定**  
   ```bash
   free_ebook_maker init --project ./my_book --lang ja --direction vertical-rl
   ```
   これにより、`my_book/` 直下に `ebook.yaml`（組版パラメータ: `page_width: 128mm`, `page_height: 182mm`, `font_size: 10pt`, `line_gap: 8pt`, `ruby_size: 0.5em`）、`cover/`、`chapters/` が生成される。`--lang` に `ja` を指定すると、禁則文字テーブル（約 120 エントリ）と縦書き用 CSS テンプレートが自動適用される。

2. **ソースファイルの配置とメタデータ補完**  
   `chapters/` に Markdown ファイルを配置。各ファイルの先頭に YAML front matter でメタデータを記述する例:
   ```yaml
   ---
   title: 第1章 組版の基礎
   author: 山田 太郎
   lang: ja
   ruby: 山田|やまだ, 太郎|たろう
   ---
   ```
   `ruby` フィールドに「表記|読み」形式でルビを設定。未設定の場合は、MeCab 形態素解析エンジン（同梱）が自動ルビ振りを実行。精度: 一般名詞で **98.2%**、固有名詞で **91.5%**。

3. **差分ビルドと出力形式指定**  
   ```bash
   free_ebook_maker build --watch --format epub,mobi --output ./dist
   ```
   ウォッチャーが起動し、`chapters/` 内のファイル変更を検出するたびに、`dist/` に `my_book.epub` と `my_book.mobi` が生成される。初回ビルド後は、変更ファイルのみが再変換され、EPUB の場合は `content.opf` の `manifest` の更新と、変更チャプターの XHTML 置き換えのみ実行。2回目以降のビルド時間は **0.8 秒**（1 チャプター変更時）。

### ベストプラクティス

- **プラクティス 1: バージョン管理とメタデータの一元化**  
  プロジェクトルートの `ebook.yaml` に `version: 1.0.0` と `revision_history` を記述し、`git tag` と連携させる。ビルド時に `--inject-revision` フラグを付けると、各ページのフッターに `v1.0.0 (2025-03-21)` が自動挿入される。これにより、複数エディタが同時編集する環境でも、生成物のトレーサビリティが担保される。

- **プラクティス 2: ルビと禁則の事前検証 CI パイプライン**  
  コミットフックまたは GitHub Actions で `free_ebook_maker lint --check-ruby --check-kinsoku` を実行。ルビの親文字数とルビ文字数の不一致（例: `漢字|かんじ` は正しいが `日本|にっぽん` は 2:3 で警告）を検出。禁則文字が行頭に来る可能性がある箇所を CSS `word-break: keep-all` の適用有無で検証し、エラーとして CI を停止する。これにより、出版品質をコードレビュー前に自動担保できる。
