# free_ebook_maker - FRE_007

## 概要

本プロジェクトは、Markdown ソースから高品位な日本語 EPUB および PDF を生成するためのオフライン対応 CLI ツールです。組版エンジンに WeasyPrint を採用し、CSS プリントプロファイルに準拠したページレイアウトを実現。縦書き・ルビ・禁則処理など日本語組版に特化した処理パイプラインを内蔵しています。

## 詳細な内容

### free_ebook_maker とは

free_ebook_maker は、プレーンテキストまたはマークダウン形式の原稿から、出版品質の電子書籍を生成するコマンドラインツールです。内部では Pandoc をフィルタとして利用し、Unicode正規化（NFKC）と JIS X 0213 文字セットへの自動変換を実施。出力形式は EPUB 3.2、PDF（A4 / 文庫判）、および MOBI（Kindle 向け）に対応します。

### コア機能

- **日本語組版エンジン（縦書き・ルビ・禁則処理）**  
  独自の CSS 組版ルールセットにより、縦書き時に自動的に縦中横（半角数字・アルファベットの90度回転）を適用。ルビは `<ruby>` タグを解釈し、モノルビ・グループルビ両方に対応。行頭禁則文字（。、）」など）は自動的に追い出し処理を行い、行末禁則文字（「（など）は自動的に追い込み処理を行います。禁則処理の挙動は `--kinsoku-mode` パラメータで `strict` / `loose` を選択可能。

- **マルチフォーマット一括出力 + 差分ビルド**  
  単一のソースファイルから EPUB、PDF、MOBI を同時生成。`--output-dir` で出力先を指定し、`--incremental` フラグを立てると、前回ビルド時から変更のあったチャプターのみを再処理（MD5 ハッシュ比較による差分検出）。これにより、数百ページの書籍でもビルド時間をミリ秒単位で短縮します。

- **メタデータ自動埋め込み + カバー画像自動生成**  
  `--metadata` フラグで JSON ファイルを指定すると、DC メタデータ（タイトル、著者、出版社、ISBN、発行日）を EPUB の OPF ファイルに自動注入。さらに `--cover-image` を省略した場合、タイトルと著者名から SVG ベースのカバー画像を動的に生成し、EPUB と PDF の表紙として埋め込みます。フォントは内蔵の Noto Sans CJK JP を使用。

### 使い方

1. **環境初期化と依存関係のインストール**  
   ```bash
   # Python 3.10+ と Poetry を前提
   git clone https://github.com/xingfangwang-eng/free_ebook_maker.git
   cd free_ebook_maker
   poetry install --no-dev
   # WeasyPrint のシステム依存ライブラリ（pango, cairo, gdk-pixbuf）をインストール
   # Ubuntu: sudo apt install libpango-1.0-0 libcairo2 libgdk-pixbuf2.0-0
   ```

2. **ソースマークダウンファイルの作成とビルド実行**  
   ```bash
   # 原稿を chapters/ ディレクトリに配置（例: chapter_01.md）
   # 目次は toc.yaml で自動生成（--auto-toc フラグ）
   free_ebook_maker build \
     --source-dir ./chapters \
     --output-dir ./output \
     --format epub,pdf \
     --metadata ./metadata.json \
     --kinsoku-mode strict \
     --incremental
   ```

3. **生成された電子書籍の検証と配信**  
   ```bash
   # EPUB のバリデーション（epubcheck ラッパーを内蔵）
   free_ebook_maker validate ./output/book.epub
   # 出力ファイル一覧
   ls -lh ./output/
   # 配布用に ZIP アーカイブ
   free_ebook_maker package ./output --compress zip
   ```

### ベストプラクティス

- **バージョン管理と CI/CD パイプラインへの組み込み**  
  ソースマークダウンを Git で管理し、GitHub Actions 上で `free_ebook_maker build` を自動実行。プルリクエストごとに差分ビルドで EPUB を生成し、`actions/upload-artifact` で成果物を保存。これにより、レビュー段階から電子書籍のレイアウトを確認可能。`--incremental` フラグを併用することで、CI のビルド時間を平均 70% 削減できます。

- **メタデータの一元管理による多言語展開**  
  `metadata.json` を言語ごとに用意し（例: `metadata_ja.json`、`metadata_en.json`）、ビルド時に `--metadata` を切り替えることで、同一ソースから日本語版・英語版を同時生成。`--lang` パラメータを指定すると、CSS の `:lang()` 疑似クラスを利用してフォントファミリーと行間を自動切替。日本語は `Noto Serif CJK JP`、英語は `Noto Serif` を使用し、最適な可読性を担保します。
