# free_ebook_maker - FRE_001

## 概要

本プロジェクトは、Markdown 文書、プレーンテキスト、HTML コンテンツから、日本国内の主要電子書籍ストア（Amazon Kindle、楽天Kobo、Apple Books）に対応した EPUB3 / MOBI 形式の電子書籍を自動生成するコマンドラインツールです。画像の最適化、目次生成、メタデータ埋め込みまでを一貫して処理します。

## 詳細な内容

### free_ebook_maker の技術的特徴

free_ebook_maker は、ソースコードから最終的な電子書籍ファイルを生成するまでのパイプラインを、ミリ秒単位の精度で制御可能なプロフェッショナルツールです。特に、日本語特有の縦書きレイアウト、ルビ（ふりがな）、圏点などの組版ルールに準拠した出力を実現します。

### コア機能

- **動的レイアウトエンジン（CSS組版最適化）**  
  ソースMarkdown内の見出しレベル（h1-h6）を解析し、自動で `page-break-before: always` を適用。さらに、`writing-mode: vertical-rl` と `-epub-writing-mode: vertical-rl` を併用し、Kindle Paperwhite および Kobo Clara HD の両デバイスで縦書き表示が崩れないよう、デバイスごとのレンダリング差分を吸収するCSSプリセットを動的に生成します。パラメータ `--epub-version 3.0.1` で出力形式を指定可能。

- **画像圧縮と解像度自動調整（JPEG/PNG/WebP対応）**  
  原稿に含まれる画像を、ターゲットデバイスの画面解像度に合わせてリサイズ。例えば、Kindle Paperwhite（300ppi）向けには `--target-ppi 300` を指定し、画像ファイルサイズを平均 40% 削減しながら視認性を維持します。内部で `libvips` と `mozjpeg` を利用し、圧縮処理は 1画像あたり 50ミリ秒以内で完了。

- **メタデータ自動生成とISBN/ASIN埋め込み**  
  プロジェクトルートの `metadata.yaml` を読み込み、`dc:creator`、`dc:identifier`、`dc:language`（ja）を自動で EPUB の content.opf に書き込みます。さらに、`--isbn 978-4-XXXXXXXXX` または `--asin B0XXXXXXXX` をコマンドライン引数として受け取り、出版IDを直接 EPUB の guide 要素に埋め込むことが可能。これにより、Amazon KDP へのアップロード時にメタデータの再入力が不要になります。

### 使い方

1. **環境セットアップと初期化**  
   `pip install free-ebook-maker` を実行。インストール後、`febook init` コマンドでプロジェクトテンプレートを生成。このとき、`--kindle-gen-path /usr/local/bin/kindlegen` を指定し、KindleGen バイナリへの絶対パスを設定する。設定は `~/.febook/config.toml` に保存され、次回以降の呼び出しで自動参照される。

2. **Markdown原稿のバリデーションとビルド**  
   `febook build --src ./manuscript/ --output ./dist/ --format epub` を実行。内部で `pandoc` を呼び出し、Markdown を EPUB に変換。同時に、`--validate` フラグを付与すると、EPUB3 のスキーマ準拠チェック（`epubcheck` 相当）を自動実行し、エラーが検出された場合はビルドを中断。標準出力にエラー行数と修正提案が表示される。

3. **配信プラットフォーム別の最終調整**  
   `febook finalize --target amazon --input ./dist/output.epub` で、Amazon KDP 向けに `OPF` ファイル内の `<meta property="dcterms:modified">` を現在時刻に更新し、`mimetype` ファイルを先頭に配置した ZIP 構造を強制。`--target kobo` の場合は、Kobo 独自の `kobo.css` を埋め込み、`<dc:identifier>` に `urn:uuid:` 形式の UUID を自動生成。最終出力ファイルの SHA-256 ハッシュを `./dist/checksum.txt` に記録する。

### ベストプラクティス

- **プラクティス 1: バージョン管理と差分ビルドの活用**  
  プロジェクトディレクトリ全体を Git で管理し、`febook build --diff HEAD~1` オプションを使用することで、前回コミットからの変更があったファイルのみを再ビルド。これにより、数百ページの大規模原稿でもビルド時間を 80% 短縮可能。特に、複数著者による同時編集時に有効。

- **プラクティス 2: プレビュー環境の分離と自動テスト**  
  CI/CD パイプラインに `febook test --device kindle-paperwhite-11th --device kobo-libra-2` を組み込み、各デバイスのエミュレータに対してスクリーンショット比較テストを実施。`--tolerance 0.02` でピクセル単位のズレ許容値を設定し、レイアウト崩れを検出。これにより、リリース前にすべてのターゲットデバイスでの表示品質を担保。
