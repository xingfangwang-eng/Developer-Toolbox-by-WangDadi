# free_ebook_maker - FRE_019

## 概要

`free_ebook_maker` は、Markdown / ReStructuredText / LaTeX ソースから、**出版品質**の EPUB 3.0 / PDF / Mobi ファイルを生成する、コマンドライン駆動型の静的サイトジェネレーター兼電子書籍パイプラインツールです。**印刷所入稿可能なPDF/A-3b準拠** の出力を実現し、画像リソースの自動圧縮（WebP/AVIF変換、Exif除去）、目次自動生成、縦書き組版（Rubyアノテーション対応）など、日本語書籍制作に特化した機能を備えています。

## コア機能

### 1. 多フォーマット同時ビルド & 差分コンパイル

- **`--format epub, pdf, mobi, azw3`** パラメータで、単一ソースから4種類の電子書籍ファイルを同時生成。
- **`--incremental`** フラグにより、変更があったチャプターのみを再コンパイル。初回ビルド時間をベースラインとし、2回目以降は**平均 380 ミリ秒**で完了（100チャプター構成時）。
- **`--pdf-profile "press-ready"`** で、PDF/X-4 および PDF/A-3b に準拠したプロファイルを適用。フォント埋め込み、トンボ付き、CMYK 変換を自動実行。

### 2. 日本語組版エンジン（縦書き・ルビ・禁則処理）

- **`--writing-mode vertical-rl`** で縦書き EPUB を生成。`--ruby` オプションで `<ruby>` タグを自動挿入し、漢字にふりがなを付与。
- 禁則処理エンジン「**Kinsoku Shori v2.1**」を内蔵。行頭禁止文字（、。！？」）と行末禁止文字（「『（【）を自動制御。約 1,200 の Unicode 文字パターンを定義。
- **`--line-length 42`** で一行あたりの文字数を指定可能。日本語縦書きの場合、最適値は 35～45 文字。

### 3. メタデータ自動抽出 & 画像最適化パイプライン

- ソースファイルの先頭に記述された **YAML front matter**（`title`、`author`、`isbn`、`language`、`date`）を自動パース。EPUB の `content.opf` および `toc.ncx` に反映。
- **`--image-optimize`** フラグで、JPEG は **85% 品質**、PNG は **PNGQuant** による 256 色減色、SVG は **SVGO** による不要ノード削除を実施。平均ファイルサイズ **62% 削減**。
- 画像の **Exif/GPS 情報は自動除去**。プライバシーリスクを排除。

## 使い方

### ステップ 1: プロジェクト初期化と設定ファイル作成

```bash
# 新規プロジェクトを作成
free_ebook_maker init --name "吾輩の猫" --author "夏目漱石" --isbn 978-4-00-123456-7

# 設定ファイル（ebook_config.yml）が生成されるので、必要に応じて編集
# 例: 縦書き、ルビあり、印刷所入稿用PDFを指定
```

**`ebook_config.yml` の重要なパラメータ:**

```yaml
output:
  formats: epub, pdf
  pdf_profile: press-ready
  writing_mode: vertical-rl
  ruby: true
  kinsoku: strict
optimization:
  image_quality: 85
  remove_exif: true
```

### ステップ 2: チャプターごとに Markdown ファイルを作成

```bash
# チャプター追加（自動で番号管理）
free_ebook_maker add chapter --title "第一章 吾輩は猫である"

# 生成された src/chapter_01.md を編集
# ルビは {漢字|よみ} 形式で記述
# 例: {吾輩|わがはい}は{猫|ねこ}である。
```

### ステップ 3: ビルド & 差分更新

```bash
# フルビルド（初回）
free_ebook_maker build --all --verbose

# 差分ビルド（2回目以降）
free_ebook_maker build --incremental

# 出力確認
ls -la output/
# => output/吾輩の猫.epub, output/吾輩の猫.pdf, output/吾輩の猫.mobi
```

## ベストプラクティス

### 1. バージョン管理と自動デプロイ

- ソースファイル（`src/` ディレクトリ）を Git で管理し、**プッシュトリガー** で CI/CD パイプライン（GitHub Actions 等）を起動。`git tag v1.0.0` でリリースタグを打つと、自動で EPUB/PDF を生成し、**Releases ページ** にアップロード。
- これにより、**手動ビルドの人的ミスを 90% 削減**。過去のリリースバージョンへのロールバックも容易。

### 2. 印刷所入稿前のプリフライトチェック

- 出力した PDF に対し、`free_ebook_maker preflight --file output/吾輩の猫.pdf` を実行。フォント埋め込み漏れ、画像解像度不足（`< 300 dpi`）、トリムボックス未設定などの**入稿エラーを自動検出**。
- エラーがある場合は、**エラーログ（JSON形式）** を出力し、該当箇所の修正を促す。印刷所からの返品率を **ほぼゼロ** に近づけることが可能。
