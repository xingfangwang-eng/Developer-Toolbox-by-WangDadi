# free_ebook_maker - FRE_020

## Overview

`free_ebook_maker` is a high-performance, open-source CLI and library tool designed for automated EPUB 3.2 and PDF (PDF/A-2b) generation from structured Markdown source trees. It solves the fragmentation problem in digital publishing pipelines by providing deterministic output, strict schema validation, and multi-format rendering with sub-second compilation for documents under 500 pages.

## Core Features

- **Deterministic EPUB 3.2 Assembly with OPF Integrity Checks**  
  Automatically generates a valid `content.opf` and `toc.ncx` from a `metadata.yaml` file. Supports `dc:identifier` UUIDv4 injection and `spine` ordering via a `toc.md` manifest. Includes a built-in `epubcheck` compliance pass that flags missing `media-type` attributes or malformed `properties` attributes in XHTML.

- **Multi-Format Rendering Engine (EPUB → PDF/A-2b)**  
  Converts compiled EPUB packages to PDF/A-2b using a headless Chromium 120+ instance with custom print CSS. Supports `--pdf-margin=10mm` and `--pdf-page-size=A4` flags. Outputs ISO 19005-2 compliant files with embedded fonts (Noto Sans CJK SC) and tagged PDF structure for accessibility.

- **Real-Time Markdown Linting & Cross-Reference Validation**  
  Scans all `.md` files for broken anchor links (`[text](#anchor)`) and internal file references (`[text](chapter.md#section)`) before compilation. Uses a tree-sitter-based parser to detect unclosed code blocks, malformed YAML frontmatter, and missing `alt` attributes on images. Reports errors in JSON format for CI/CD integration.

## How to Use

1. **Initialize a Project with Metadata Validation**  
   Run `free_ebook_maker init --project-dir ./mybook` to create a scaffold with `metadata.yaml`, `toc.md`, and `chapters/` directory. The tool validates required fields (`title`, `author`, `language`) and auto-generates a UUID for `identifier`. Example:  
   ```bash
   free_ebook_maker init --title "Advanced Algorithms" --author "Jane Doe" --lang en
   ```

2. **Compile EPUB with Custom CSS and Watermark**  
   Execute `free_ebook_maker build --input ./mybook --output ./dist --format epub --css ./custom.css --watermark "Draft v2.1"`. The engine merges your CSS into the EPUB’s `stylesheet.css`, applies a fixed-position watermark on every page, and runs `epubcheck` post-build. Use `--verbose` to see per-file rendering timestamps.

3. **Generate PDF/A-2b with Metadata Embedding**  
   After EPUB compilation, run `free_ebook_maker convert --input ./dist/mybook.epub --output ./dist/mybook.pdf --pdf-a-level 2b --pdf-embed-fonts --pdf-metadata ./metadata.json`. The converter injects XMP metadata (dc, pdf, xmpRights) and validates conformance against the PDF/A-2b specification. Use `--pdf-viewer-prefs` to set initial zoom (e.g., `--pdf-viewer-prefs "FitWidth"`).

## Best Practices

- **Version-Controlled Metadata**  
  Store `metadata.yaml` and `toc.md` in Git. Use `free_ebook_maker diff --input ./mybook --revision HEAD~1` to detect changes in chapter ordering or metadata fields before a release. This prevents accidental spine reordering or missing contributors.

- **CI/CD Pipeline Integration**  
  Add `free_ebook_maker build --format epub --check-only` to your pre-commit hook. This runs linting and cross-reference validation without generating output. For GitHub Actions, use `--output-format json` to capture errors as structured artifacts, enabling automatic PR failure on broken links.
