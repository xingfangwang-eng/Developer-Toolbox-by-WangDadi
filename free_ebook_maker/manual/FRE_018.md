# free_ebook_maker - FRE_018

## Overview

This is a high-performance, cross-format ebook generation engine designed for developers, publishers, and content automation pipelines. The `free_ebook_maker` tool compiles raw Markdown, HTML, or plain text into polished EPUB, PDF, and Mobi files with millisecond-level rendering precision.

## Detailed Content

### What is free_ebook_maker?

`free_ebook_maker` is a command-line and library-grade solution that transforms structured content into standards-compliant ebooks. It validates against the EPUB 3.2 specification, supports embedded CSS and SVG assets, and outputs deterministic byte-level hashes for reproducible builds. The engine leverages a custom `libxml2`-based parser to handle malformed HTML without data loss, and integrates with PostgreSQL for metadata-driven batch processing.

### Core Features

- **Multi-Format Rendering Engine with Deterministic Hashing**  
  Converts input sources to EPUB, PDF, and Mobi simultaneously using a single `--output-formats` parameter (e.g., `--output-formats epub,pdf,mobi`). Each output file is tagged with a SHA-256 hash of the source content and build timestamp, enabling exact reproducibility across CI/CD pipelines. The engine uses `wkhtmltopdf` under the hood for PDF generation, with configurable DPI (`--pdf-dpi 300`) and page size (`--pdf-page-size A4`).

- **Metadata Injection & PostgreSQL Batch Processing**  
  Accepts a `--metadata-file metadata.json` parameter that supports nested JSON structures for title, author, ISBN, language (ISO 639-1), and custom `<meta>` tags. When combined with `--db-connection "postgresql://user:pass@host:5432/ebooks"`, the tool can pull source content and metadata from a PostgreSQL table, process up to 10,000 records per batch, and write output paths back to the database with `status` and `error_log` columns. The batch processor uses `COPY` for high-throughput ingestion and supports `--batch-size 500`.

- **Real-Time Syntax Validation & Asset Embedding**  
  Includes a built-in validator (`--validate-only`) that checks for broken internal anchors, missing image references, and invalid EPUB navigation control files (NCX). The validator reports errors with line numbers and column offsets in microsecond precision. For asset embedding, the tool supports `--embed-fonts` and `--embed-images` flags, which inline base64-encoded fonts and images directly into the EPUB container, eliminating external dependencies. Maximum file size for embedding is configurable via `--max-embed-size 10MB`.

### How to Use

1. **Initialize a Project with Metadata Schema**  
   Create a `metadata.json` file with required fields:
   ```json
   {
     "title": "Technical Guide to Python",
     "author": "Jane Doe",
     "language": "en",
     "isbn": "978-3-16-148410-0",
     "cover_image": "./assets/cover.png"
   }
   ```
   Run `free_ebook_maker init --metadata-file metadata.json` to generate a skeleton directory structure with `content/`, `assets/`, and `output/` folders. The tool validates the JSON schema against a predefined XSD and exits with code 1 if mandatory fields are missing.

2. **Compile Source Content with Custom CSS**  
   Place your Markdown files in `content/` and an optional `style.css` in the project root. Execute:
   ```bash
   free_ebook_maker build --input ./content --style ./style.css --output-formats epub,pdf --pdf-dpi 300 --metadata-file metadata.json
   ```
   The engine parses each `.md` file, applies CSS class-based page breaks (`@page { break-before: always; }`), and generates a temporary HTML file per chapter. For EPUB, it creates a valid `content.opf` and `toc.ncx` with hierarchical headings. The PDF output uses `wkhtmltopdf` with a custom footer template (`--pdf-footer "Page [page] of [topage]"`). On completion, the tool prints the SHA-256 hash of each output file.

3. **Batch Process from PostgreSQL and Validate**  
   Configure a PostgreSQL connection string and run:
   ```bash
   free_ebook_maker batch --db-connection "postgresql://user:pass@localhost:5432/ebooks" \
     --query "SELECT id, title, content FROM drafts WHERE status = 'pending'" \
     --batch-size 500 --output-dir ./batch_output --validate-only
   ```
   The tool streams rows using server-side cursors, validates each record’s content for malformed HTML and broken links, and writes errors to a `validation_errors` table. It uses `COPY` to bulk-insert results back into the database with a `processed_at` timestamp. If `--validate-only` is omitted, the tool also generates the ebook files and updates the `output_path` column.

### Best Practices

- **Use Versioned Metadata with CI/CD Integration**  
  Store `metadata.json` in your Git repository and tag it with semantic versions. In your CI pipeline, run `free_ebook_maker build` with `--commit-hash $(git rev-parse HEAD)` to embed the commit hash into the ebook’s `<dc:source>` metadata. This allows tracing any generated ebook back to its exact source commit. For production releases, always enable `--validate-only` in a separate stage before the build to catch structural errors early.

- **Leverage PostgreSQL Triggers for Automated Publishing**  
  Set up a database trigger that calls `free_ebook_maker batch` whenever a new row is inserted into a `drafts` table with `status = 'ready'`. Use `LISTEN/NOTIFY` in PostgreSQL to avoid polling. For example, after a content editor approves a draft, the trigger fires a notification, and a background worker process runs the batch command with `--output-formats epub,mobi`. This reduces manual intervention and ensures that every approved draft is automatically converted and stored with consistent metadata.
