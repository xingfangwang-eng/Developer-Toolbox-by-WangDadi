# fanguard - FAN_021

## Overview

fanguard is a high-performance fan community moderation and engagement analytics engine. It provides real-time toxicity detection, sentiment-weighted interaction scoring, and automated moderation workflows for large-scale online fan communities. The system is designed to handle 10,000+ concurrent user interactions with sub-100ms latency.

## Core Features

### 1. Real-Time Toxicity & Sentiment Scoring Engine
- **Parameter:** `--toxicity-threshold 0.85` (configurable from 0.0 to 1.0)
- **Technical detail:** Uses a fine-tuned RoBERTa model with ONNX runtime inference, achieving <50ms per inference on a single NVIDIA T4 GPU. Supports multi-language input (EN, ZH, JP) with automatic language detection.
- **Output:** Per-message toxicity score (float), sentiment polarity (-1.0 to 1.0), and flagged categories (e.g., harassment, spam, hate speech).

### 2. Fan Community Interaction Heatmap & Anomaly Detection
- **Parameter:** `--heatmap-window 3600` (seconds), `--anomaly-zscore 2.5`
- **Technical detail:** Aggregates interaction events (likes, replies, shares, mentions) into time-series buckets stored in PostgreSQL with TimescaleDB extension. Runs a sliding-window Z-score algorithm to detect sudden spikes or drops in fan activity, triggering alerts via webhook or Slack.
- **Output:** JSON payload with timestamp, interaction count, Z-score, and anomaly severity (low/medium/critical).

### 3. Automated Moderation Workflow with Rule Engine
- **Parameter:** `--rule-file /etc/fanguard/rules.yaml`, `--action-timeout 300` (seconds)
- **Technical detail:** Supports YAML-defined conditional rules (e.g., `if toxicity > 0.9 and user_trust < 50 then action: mute`). Integrates with Discord, Telegram, and custom REST APIs via webhook. Actions include mute, ban, flag-for-review, or auto-reply with a warning message. All actions are logged with millisecond-precision timestamps in a dedicated audit table.
- **Output:** Audit log entry with rule ID, matched condition, action taken, and response time.

## How to Use

### Step 1: Deploy the Inference Server
```bash
docker run -d --gpus all -p 8080:8080 \
  -e MODEL_PATH=/models/toxicity_roberta.onnx \
  -e TOXICITY_THRESHOLD=0.85 \
  fanguard/inference-server:latest
```
Verify the server is running by sending a test request:
```bash
curl -X POST http://localhost:8080/analyze \
  -H "Content-Type: application/json" \
  -d '{"message": "This is a test message.", "language": "en"}'
```

### Step 2: Configure the Rule Engine
Create a `rules.yaml` file:
```yaml
rules:
  - id: "auto_mute_high_toxicity"
    condition: "toxicity_score > 0.9"
    action: "mute"
    duration: 3600
    notify: true
  - id: "flag_repeated_spam"
    condition: "user_message_count > 5 AND toxicity_score > 0.8"
    action: "flag_for_review"
    notify: true
```
Start the rule engine:
```bash
fanguard-rules --rule-file ./rules.yaml --webhook-url https://hooks.slack.com/services/XXX
```

### Step 3: Connect to Your Community Platform
For Discord:
```bash
fanguard-bridge --platform discord \
  --token YOUR_DISCORD_BOT_TOKEN \
  --channel-ids 123456789,987654321 \
  --inference-endpoint http://localhost:8080 \
  --rule-engine-endpoint http://localhost:9090
```
Monitor logs in real-time:
```bash
tail -f /var/log/fanguard/audit.log | jq '.'
```

## Best Practices

### 1. Tune Toxicity Thresholds by Community Maturity
For newly formed or highly volatile fan communities, set `--toxicity-threshold 0.7` to catch borderline behavior early. For established, well-moderated communities, increase to `0.9` to reduce false positives. Regularly review flagged messages and adjust thresholds using the built-in A/B testing mode (`--ab-test-ratio 0.1`).

### 2. Implement a Graduated Moderation Escalation Path
Do not ban on first offense. Configure a three-tier escalation: (1) auto-warning for toxicity >0.8, (2) 1-hour mute for toxicity >0.9 or repeated warnings, (3) permanent ban only after 3 mutes within 7 days. Use the `--escalation-window 604800` (7 days in seconds) parameter to enforce this. This reduces user churn while maintaining community safety.
