# fanguard - FAN_012

## Resumen

fanguard es un sistema de protección y gestión de comunidades de fans diseñado para plataformas de interacción masiva. Optimizado para el mercado hispanohablante, permite detectar comportamientos anómalos, segmentar audiencias en tiempo real y automatizar la moderación sin intervención humana. Ideal para artistas, marcas y gestores de comunidades con más de 100.000 miembros activos.

## Contenido Detallado

### ¿Qué es fanguard?

fanguard es una herramienta profesional de defensa comunitaria que resuelve problemas críticos en la administración de fandoms: suplantación de identidad, ataques coordinados (raids), spam automatizado y fugas de contenido exclusivo. Opera con una latencia inferior a 50 ms por evento y se integra directamente con PostgreSQL para almacenar trazas de auditoría inmutables.

### Características Principales

#### 1. Detección de Anomalías Conductuales con ML en Tiempo Real
- **Parámetro clave**: `--anomaly-threshold=0.85` (sensibilidad del modelo de aislamiento).
- **Indicador técnico**: Procesa 10.000 eventos por segundo con una tasa de falsos positivos inferior al 2%.
- **Funcionamiento**: Analiza patrones de interacción (frecuencia de mensajes, horarios, dispositivos) y genera alertas cuando un usuario supera el umbral de desviación estándar ×3.5.
- **Integración**: Almacena resultados en `fan_anomaly_logs` (PostgreSQL, tabla particionada por día).

#### 2. Moderación Automática de Contenido con Filtros Semánticos
- **Parámetro clave**: `--filter-lang=es` (soporte para español latinoamericano y peninsular).
- **Indicador técnico**: Tiempo de procesamiento por mensaje: 12 ms (medido en CPU AMD EPYC 64 núcleos).
- **Funcionamiento**: Utiliza un modelo de embeddings multilingüe (dim=768) para detectar discursos de odio, acoso y spam. Se actualiza cada 6 horas mediante un pipeline de entrenamiento incremental.
- **Integración**: Conexión directa a `fan_messages` (PostgreSQL) con eliminación automática de contenido infractor y notificación al moderador en menos de 200 ms.

#### 3. Segmentación Dinámica de Fans con Perfiles de Riesgo
- **Parámetro clave**: `--risk-profile=high|medium|low` (clasificación basada en 14 variables).
- **Indicador técnico**: Genera perfiles completos en 0.8 segundos para bases de hasta 500.000 usuarios.
- **Funcionamiento**: Combina datos de actividad, historial de reportes y reputación entre pares para asignar un score de confianza (0.0 a 1.0). Los usuarios con score < 0.3 son puestos en cuarentena automática.
- **Integración**: Exporta segmentos a `fan_segments` (PostgreSQL) con índices compuestos por `user_id` y `timestamp`.

### Cómo Usar

#### Paso 1: Configurar el Entorno y la Conexión a PostgreSQL
Ejecuta el siguiente comando para inicializar la base de datos y las tablas necesarias:
```bash
fanguard init-db --host=localhost --port=5432 --dbname=fanguard_prod --user=admin --password=$(cat /etc/fanguard/db_pass.txt)
```
Esto creará automáticamente las tablas `fan_anomaly_logs`, `fan_messages` y `fan_segments` con particionamiento por fecha y claves foráneas. Verifica la conexión con:
```bash
fanguard check-db --timeout=5s
```
La salida debe mostrar `PostgreSQL connection: OK (latency: 3 ms)`.

#### Paso 2: Activar la Detección de Anomalías con Umbral Personalizado
Inicia el módulo de monitoreo en segundo plano:
```bash
fanguard start-anomaly --anomaly-threshold=0.85 --max-events=10000 --log-level=info
```
Para probar con datos simulados, usa:
```bash
fanguard simulate-events --count=5000 --pattern=raid
```
Observa los resultados en tiempo real con:
```bash
fanguard watch-logs --table=fan_anomaly_logs --interval=1s
```
Cada alerta mostrará: `[ALERT] user_id=12345 | score=0.92 | action=quarantine`.

#### Paso 3: Implementar la Moderación Automática con Filtro Semántico
Configura el filtro para español y activa la eliminación automática:
```bash
fanguard start-filter --filter-lang=es --auto-delete --notify-moderators --webhook=https://tu-servidor.com/hooks/fanguard
```
Verifica el estado del pipeline:
```bash
fanguard filter-status
```
Ejemplo de salida:
```
Filter: active | Language: es | Messages processed: 254,321 | Deleted: 1,234 (0.49%) | Avg latency: 12 ms
```
Para exportar un informe diario:
```bash
fanguard export-report --date=2025-04-10 --format=csv --output=./reportes/diario_2025-04-10.csv
```

### Mejores Prácticas

#### Práctica 1: Ajuste Fino del Umbral de Anomalías
Para comunidades con alta actividad (ej. más de 50.000 mensajes/día), comienza con `--anomaly-threshold=0.90` y reduce gradualmente hasta 0.80 si ves falsos negativos. Monitorea la tasa de falsos positivos semanalmente usando `fanguard stats --metric=fp_rate`. Un valor superior al 3% indica que el umbral es demasiado sensible. Realiza un reentrenamiento del modelo cada 30 días con `fanguard retrain --data-last=30d`.

#### Práctica 2: Segmentación de Riesgo para Moderación Humana
No confíes únicamente en la automatización. Configura un flujo de revisión manual para usuarios con `--risk-profile=high`. Integra un panel de control que muestre los perfiles en cola usando `fanguard dashboard --risk=high`. Asigna moderadores por turnos y establece un SLA de 15 minutos para resolver casos críticos. Almacena las decisiones en `fan_moderation_actions` (PostgreSQL) para auditoría posterior.
