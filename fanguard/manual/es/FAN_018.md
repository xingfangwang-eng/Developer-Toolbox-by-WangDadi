# fanguard - FAN_018

## Resumen

fanguard es un sistema de protección y gestión de comunidades de fans de alto rendimiento, diseñado para plataformas de interacción masiva. Combina análisis de sentimiento en tiempo real, moderación automatizada de contenido y segmentación dinámica de audiencias, todo optimizado para baja latencia y alta escalabilidad.

## Contenido Detallado

### ¿Qué es fanguard?

fanguard es una herramienta profesional de orquestación para la protección de comunidades de fans. Integra motores de reglas configurables, filtrado semántico multilingüe y dashboards de métricas en vivo. Está construido sobre una arquitectura de microservicios con Go y Rust, y se conecta a PostgreSQL para persistencia de eventos y a Redis para caché de sesiones de usuario.

### Características Principales

- **Moderación de contenido con análisis de sentimiento en tiempo real**: Utiliza un modelo de lenguaje ligero (distilBERT) fine-tuneado con datos de comunidades de fans. Evalúa cada mensaje en menos de 50 ms (p99) y asigna una puntuación de toxicidad (0.0 - 1.0). Los umbrales se configuran por canal: `toxicity_threshold: 0.85`, `spam_threshold: 0.7`. Los mensajes que superan el límite se ponen en cuarentena automática y se notifican a los moderadores vía webhook.

- **Segmentación dinámica de audiencias basada en comportamiento**: Analiza patrones de interacción (frecuencia de posts, reacciones, tiempo de permanencia) cada 5 minutos. Genera clusters de usuarios con etiquetas como `superfan`, `lurker`, `troll`. Los parámetros de clustering se ajustan vía API: `kmeans_k: 8`, `min_cluster_size: 50`. Los resultados se almacenan en una tabla de PostgreSQL con índices GIN para consultas rápidas.

- **Sistema de alertas predictivas para picos de actividad**: Monitorea el volumen de mensajes por segundo (MPS) y la tasa de crecimiento de nuevos miembros. Cuando el MPS supera el percentil 95 histórico (calculado cada hora), dispara una alerta. Los parámetros de alerta son: `mps_threshold_percentile: 95`, `growth_rate_threshold: 0.3`. Las alertas se envían a Slack o Discord mediante un webhook configurable. El sistema registra cada evento en una tabla `alerts_log` con timestamp en microsegundos.

### Cómo Usar

1. **Configuración inicial del entorno**: Clona el repositorio y ejecuta `make setup`. Esto instalará las dependencias de Go (v1.21+) y Rust (v1.70+), y creará la base de datos PostgreSQL con el esquema predefinido. Luego, edita el archivo `config.yaml` para definir los canales de moderación y los umbrales de toxicidad. Ejecuta `./fanguard --init` para validar la configuración.

2. **Integración con la plataforma de fans**: Conecta fanguard a tu API de chat o foro. Envía los mensajes de los usuarios al endpoint `/api/v1/moderate` con un payload JSON que incluya `user_id`, `channel_id`, `message_text` y `timestamp` (en formato ISO 8601). fanguard devolverá una respuesta con `action: "allow"`, `"quarantine"` o `"block"`, junto con un `confidence_score`. Implementa un callback para procesar las acciones recomendadas.

3. **Monitoreo y ajuste de alertas**: Accede al dashboard web en `http://localhost:8080/dashboard`. Revisa las métricas en vivo de MPS y tasa de crecimiento. Para ajustar los umbrales de alerta, usa el endpoint `PUT /api/v1/alerts/thresholds` con un JSON: `{"mps_threshold_percentile": 98, "growth_rate_threshold": 0.5}`. Los cambios se aplican en menos de 1 segundo. Las alertas generadas se muestran en la tabla `Recent Alerts` del dashboard.

### Mejores Prácticas

- **Ajuste fino de umbrales por comunidad**: No uses los mismos valores de toxicidad y spam para todas las comunidades. Realiza un análisis de 7 días de datos históricos para determinar el percentil 90 de toxicidad natural de cada canal. Configura los umbrales 10% por encima de ese valor para evitar falsos positivos. Esto reduce las revisiones manuales en un 40%.

- **Rotación de claves de API y webhooks**: Implementa un ciclo de rotación de claves cada 30 días. Usa variables de entorno o un gestor de secretos (como HashiCorp Vault) para almacenar las claves. fanguard soporta rotación en caliente mediante el endpoint `POST /api/v1/rotate-keys`. Esto previene accesos no autorizados sin interrumpir el servicio.
