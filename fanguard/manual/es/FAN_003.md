# fanguard - FAN_003

## Resumen

fanguard es un sistema de protección inteligente para comunidades de fans, diseñado para mitigar ataques coordinados, gestionar picos de tráfico orgánico y detectar patrones de interacción anómalos en tiempo real. Optimizado para entornos de alta concurrencia (hasta 500.000 conexiones simultáneas por nodo), utiliza un motor de reglas dinámicas y firmas de comportamiento para diferenciar entre usuarios legítimos y bots maliciosos sin depender de CAPTCHA.

## Contenido Detallado

### ¿Qué es fanguard?

fanguard es una plataforma de defensa perimetral especializada en comunidades de fans. Combina un proxy inverso con capacidad de inspección profunda de paquetes (DPI) a nivel de aplicación, un sistema de reputación de IP distribuido basado en Redis Cluster, y un módulo de análisis semántico de contenido para detectar campañas de desinformación o flooding emocional. Todo el tráfico se filtra en menos de 2 ms por solicitud, garantizando una latencia imperceptible para los usuarios reales.

### Características Principales

- **Detección de comportamiento coordinado (CBCD)**: Analiza la correlación temporal de acciones (likes, retweets, comentarios) mediante ventanas deslizantes de 5 segundos. Si más de 50 IPs realizan la misma acción en un intervalo de 300 ms, se activa un bloqueo automático por 15 minutos. Parámetros ajustables: `window_size` (por defecto 5000 ms), `coincidence_threshold` (por defecto 50), `ban_duration` (por defecto 900000 ms).

- **Filtrado de contenido tóxico en tiempo real**: Utiliza un modelo de lenguaje ligero (distilBERT) fine-tuneado con 2.3 millones de ejemplos de interacciones tóxicas en español. Cada mensaje se evalúa en menos de 8 ms (GPU Tesla T4). Si la puntuación de toxicidad supera 0.85, el mensaje se reemplaza por un marcador de posición y se notifica al moderador vía WebSocket. Umbral configurable: `toxicity_threshold` (rango 0.0-1.0, por defecto 0.85).

- **Control de picos virales (VPC)**: Ante un aumento súbito de tráfico (>300% en 60 segundos), fanguard activa un modo de "caché agresiva" donde las páginas de perfil de ídolos se sirven desde una CDN interna con TTL de 10 segundos, y las solicitudes de escritura (comentarios, votaciones) se encolan en PostgreSQL con prioridad FIFO. El sistema puede manejar hasta 1.2 millones de solicitudes por minuto sin degradación.

### Cómo Usar

1. **Despliegue inicial con Docker Compose**: Clona el repositorio y ejecuta `docker-compose up -d`. fanguard expondrá el puerto 8080 para tráfico HTTP y 8443 para HTTPS. Luego, configura tu DNS para apuntar el subdominio de tu comunidad (ej. `fanclub.tudominio.com`) a la IP del servidor donde corre fanguard. Verifica que el proxy inverso esté activo accediendo a `http://localhost:8080/health`; deberías recibir un JSON con `{"status":"ok","uptime":12345}`.

2. **Configuración de reglas personalizadas**: Edita el archivo `config/rules.yaml`. Por ejemplo, para bloquear IPs de un rango específico durante un evento, añade:
   ```yaml
   - name: "Bloqueo regional temporal"
     type: "ip_range"
     cidr: "185.220.101.0/24"
     action: "drop"
     duration_ms: 3600000
     comment: "Campaña de votación masiva detectada"
   ```
   Luego recarga la configuración con `curl -X POST http://localhost:8080/admin/reload`.

3. **Monitorización en vivo**: Conéctate al panel de métricas vía WebSocket en `ws://localhost:8080/metrics/stream`. Recibirás actualizaciones cada 500 ms con:
   ```json
   {
     "requests_per_second": 4520,
     "blocked_ips": 234,
     "avg_latency_ms": 1.2,
     "toxicity_alerts": 12
   }
   ```
   Puedes integrar esto con Grafana usando el plugin de WebSocket.

### Mejores Prácticas

- **Segmentación por niveles de confianza**: Define 3 niveles de usuario (verificado, regular, nuevo) y asigna diferentes umbrales de toxicidad y límites de tasa. Por ejemplo, a usuarios nuevos (menos de 7 días de antigüedad) aplícales un `toxicity_threshold` de 0.6 y un límite de 10 comentarios por hora. Esto reduce falsos positivos en un 40% según nuestras pruebas con comunidades de 100k miembros.

- **Integración con sistemas de ticketing**: Conecta fanguard con tu plataforma de helpdesk (Zendesk, Freshdesk) mediante el webhook de alertas. Cuando se detecte un ataque coordinado, fanguard puede crear automáticamente un ticket con la evidencia (logs, IPs, timestamps) y asignarlo al equipo de seguridad. Configura el webhook en `config/webhooks.yaml`:
  ```yaml
  - url: "https://tudominio.zendesk.com/api/v2/tickets.json"
    method: "POST"
    headers:
      Authorization: "Bearer tu_token"
    payload_template: "{\"ticket\":{\"subject\":\"Alerta fanguard: {{attack_type}}\",\"comment\":{\"body\":\"IPs bloqueadas: {{blocked_ips}}\"}}}"
  ```
