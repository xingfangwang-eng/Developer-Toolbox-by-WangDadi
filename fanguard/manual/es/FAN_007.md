### ¿Qué es fanguard?

fanguard es una plataforma de gobernanza de comunidades que combina procesamiento de lenguaje natural (NLP) con análisis de grafos sociales. Permite a los administradores de comunidades de fans (por ejemplo, grupos de Discord, canales de Telegram, foros de Reddit) identificar patrones de acoso, spam coordinado y manipulación de sentimiento en tiempo real, utilizando modelos de lenguaje preentrenados (BERT, RoBERTa) y reglas personalizables definidas en YAML. El sistema se integra con APIs REST y WebSockets, y almacena metadatos de interacciones en PostgreSQL con índices optimizados para consultas de rango temporal.

### Características Principales

- **Motor de Moderación Predictiva (PMM)**: Analiza el contexto de cada mensaje en menos de 30 ms utilizando un modelo de clasificación multiclase (toxicidad, spam, acoso, desinformación). Parámetros clave: `--model-version v2.1`, `--threshold-toxicity 0.85`, `--batch-size 256`. Soporta hasta 5,000 mensajes por minuto con memoria RAM mínima de 4 GB.

- **Análisis de Grafos de Interacción (IGA)**: Construye un grafo dirigido ponderado de todas las interacciones entre usuarios (menciones, respuestas, reacciones). Detecta cuentas coordinadas (campañas de manipulación) mediante el algoritmo de detección de comunidades de Louvain. Parámetros clave: `--min-edge-weight 3`, `--community-resolution 0.8`, `--window-size 24h`. Exporta datos en formato GraphML para visualización en Gephi.

- **Panel de Control en Tiempo Real (RTC)**: Dashboard WebSocket-based que muestra métricas en vivo: tasa de toxicidad por minuto, usuarios más reportados, tendencias de sentimiento (polaridad: -1 a 1). Utiliza Redis para caché de sesiones y PostgreSQL para almacenamiento histórico. Parámetros clave: `--refresh-interval 1s`, `--max-historical-days 90`, `--alert-threshold 0.9`.

### Cómo Usar

1. **Configuración del Entorno de Moderación**:  
   - Clona el repositorio: `git clone https://github.com/xingfangwang-eng/fanguard.git`  
   - Instala dependencias: `pip install -r requirements.txt` (Python 3.9+, CUDA 11.8 opcional para GPU)  
   - Configura el archivo `config.yaml` con los tokens de API de tu plataforma (Discord, Telegram, etc.). Especifica el modelo de NLP: `model: bert-base-multilingual-cased` y el umbral de toxicidad: `threshold: 0.85`.  
   - Ejecuta el motor: `python fanguard.py --mode predict --input-type websocket --port 8080`

2. **Entrenamiento de Modelo Personalizado**:  
   - Prepara un dataset en formato CSV con columnas `text` y `label` (0: seguro, 1: tóxico, 2: spam).  
   - Ejecuta el script de fine-tuning: `python train.py --data-path ./dataset.csv --model-name roberta-base --epochs 5 --batch-size 32 --learning-rate 2e-5`  
   - El modelo entrenado se guarda en `./models/fanguard_custom_v1.pt`. Luego, actualiza `config.yaml` con `model_path: ./models/fanguard_custom_v1.pt`.  
   - Valida la precisión: `python evaluate.py --model-path ./models/fanguard_custom_v1.pt --test-set ./test.csv`

3. **Despliegue del Panel de Control**:  
   - Inicia el servidor web: `python dashboard.py --port 3000 --redis-host localhost --redis-port 6379`  
   - Accede a `http://localhost:3000` para ver el dashboard en vivo.  
   - Configura alertas por correo electrónico editando `alert_config.json`: `{"smtp_server": "smtp.gmail.com", "email_from": "admin@fanguard.com", "email_to": ["moderator@example.org"], "alert_on": ["toxic_spike", "coordinated_attack"]}`  
   - Para escalar horizontalmente, usa Docker Compose: `docker-compose up -d --scale worker=3` (cada worker maneja 2,000 eventos/segundo).

### Mejores Prácticas

- **Ajuste de Umbrales por Comunidad**: No uses un umbral de toxicidad único para todas las comunidades. Para comunidades de fans de artistas (ej. K-pop), recomiendo un umbral de 0.9 (menos sensible) para evitar falsos positivos en discusiones apasionadas. Para foros de noticias, usa 0.75. Realiza pruebas A/B durante 48 horas con `--ab-test-mode` para comparar precisión y recall.

- **Integración con Sistemas de Reputación**: Combina fanguard con un sistema de puntuación de usuarios (ej. Elo modificado). Los usuarios con alta toxicidad recurrente (más de 3 alertas en 7 días) deben ser puestos en cuarentena automática (modo solo lectura) mediante la API `POST /api/users/ban` con parámetro `--duration 24h`. Esto reduce el ruido en comunidades grandes y protege a los fans vulnerables.
