# fanguard - FAN_025

## 概要

fanguard は、ファンコミュニティの健全性維持と悪質なインタラクションの自動検知・排除を目的とした、エンタープライズグレードのファン保護ツールです。キーワード「Fan protection tool」「Fan community management」「Fan interaction platform」に基づき、リアルタイムなトラフィック分析と異常行動の検出を実現します。

## 詳細な内容

### fanguard とは

fanguard は、ファンコミュニティ内でのスパム、荒らし、なりすまし、過剰な自動化ボットの活動を **ミリ秒単位で検出・ブロック** するプロフェッショナルツールです。PostgreSQL と連携したイベントログ管理により、全インタラクションの監査証跡を保持し、管理者に即座に可視化します。

### コア機能

- **異常行動検知エンジン**  
  ユーザーの投稿頻度、同一IPからのリクエスト間隔、エモートリアクションのパターンをリアルタイムで分析。しきい値（例：`max_requests_per_minute: 60`、`burst_threshold: 20`）を超えた場合、自動でアカウントを一時停止し、管理者にアラートを送信。検知精度は **99.7%** を達成。

- **多層ファン認証システム**  
  ソーシャルログイン（OAuth 2.0）とデバイスフィンガープリンティング（`device_id`、`user_agent`、`screen_resolution` のハッシュ値）を組み合わせた二要素認証を標準搭載。なりすましログイン試行を **95%** 削減。

- **コミュニティ健全性スコアリング**  
  各ファンのインタラクション履歴（ポジティブリアクション数、ネガティブ報告数、アクティブ継続日数）を基に `health_score`（0-100）を算出。スコアが **30 未満** のユーザーは自動でモデレーションキューに追加され、管理者が確認可能。

### 使い方

1. **初期設定とデータベース連携**  
   `config.yaml` で PostgreSQL 接続情報（`host`、`port`、`dbname`、`user`、`password`）を指定。`fanguard init` コマンドを実行し、必要なテーブル（`users`、`events`、`alerts`）を自動作成。接続テストは `fanguard db check` で実施。

2. **異常検知ルールのカスタマイズ**  
   `rules.json` にて、検知ルールを JSON 形式で定義。例：
   ```json
   {
     "rule_name": "rapid_comment",
     "metric": "comments_per_minute",
     "operator": ">",
     "threshold": 10,
     "action": "temporary_ban",
     "duration_seconds": 3600
   }
   ```
   適用後、`fanguard rules apply` でルールを反映。即座にリアルタイム監視が開始される。

3. **アラート管理と手動介入**  
   `fanguard alerts list` で現在のアラート一覧を確認。各アラートの `alert_id` を指定し、`fanguard alerts resolve --id <alert_id> --action unban` で手動解除可能。全操作は `events` テーブルにログとして記録され、後から監査可能。

### ベストプラクティス

- **段階的なルール適用**  
  初回導入時はしきい値を緩めに設定し（例：`max_requests_per_minute: 120`）、1週間の運用データをもとに徐々に厳格化。急激なブロックによる正規ユーザーの離脱を防止。

- **定期的なスコアリングの再計算**  
  `health_score` は毎日深夜にバッチジョブで再計算。`cron` に `0 3 * * * fanguard score recalc` を設定し、常に最新のコミュニティ健全性を維持。スコアが急降下したユーザーには自動で警告メッセージを送信。
