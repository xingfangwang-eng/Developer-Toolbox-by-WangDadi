# fanguard - FAN_029

## 概要

fanguard は、ファンコミュニティにおける不正アカウントの検出、感情極性のリアルタイム分析、およびコンテンツモデレーションを統合した分散型ガードシステムです。キーワード「ファン保護」「コミュニティ管理」「相互作用監視」に基づき、日本国内のプラットフォーム向けに最適化されたエッジコンピューティング構成を採用しています。

## コア機能

### 1. マルチモーダル異常行動検出エンジン
- **パラメータ**: リクエスト間隔（ミリ秒単位）、同一IPからの同時セッション数、投稿テキスト内のトークン重複率
- **技術指標**: 1秒あたり最大12,000リクエストを処理可能。Elasticsearch との連携により、過去30日間の行動パターンをベースラインとして異常スコアを算出。
- **実装詳細**: ファンクラブ内での同一アカウントによる短時間多投を検出し、自動で書き込み間隔を強制制限。

### 2. 感情極性マッピングとコミュニティ健全性スコア
- **パラメータ**: 投稿内の感情スコア（-1.0〜1.0）、特定ワード出現頻度（回/分）、コミュニティ内の相互フォロー密度
- **技術指標**: PostgreSQL との連携により、感情スコアの時系列変化を集計。健全性スコアが0.3を下回った場合、モデレーターに即時通知。
- **実装詳細**: 日本語のネガティブ表現に特化した辞書を搭載し、誹謗中傷の検出精度は95%以上。

### 3. 分散型アクセス制御リスト（DACL）による動的モデレーション
- **パラメータ**: ユーザー信頼度（0〜100）、コンテンツカテゴリ（NSFW/SPAM/正常）、モデレーションアクション種別（警告/ミュート/追放）
- **技術指標**: モデレーションルールはRedis Cluster にキャッシュされ、平均応答時間は50ミリ秒以下。
- **実装詳細**: ファンコミュニティごとにカスタムルールを設定可能。同一ユーザーが複数コミュニティで違反を繰り返す場合、グローバルブラックリストへ自動昇格。

## 使い方

### ステップ1: 環境変数とデータベースの初期設定
```bash
export FANGUARD_DB_HOST=localhost
export FANGUARD_DB_PORT=5432
export FANGUARD_REDIS_CLUSTER_NODES="node1:6379,node2:6379,node3:6379"
```
- PostgreSQL にスキーマ `fanguard_schema.sql` を適用し、感情辞書テーブル `sentiment_lexicon_ja` を初期化。
- Redis Cluster にデフォルトのモデレーションルールセット `mod_rules_default.json` を投入。

### ステップ2: 異常行動検出パイプラインの起動
```bash
./fanguard anomaly-detector --interval-ms 100 --threshold 0.85 --output-format json
```
- リアルタイムでイベントストリームを監視。異常スコアが0.85を超えた場合、`/var/log/fanguard/anomaly.log` に詳細を記録。
- 検出結果は Kafka トピック `fanguard_alert` にパブリッシュされ、下流のモデレーションエンジンが消費。

### ステップ3: モデレーションアクションの自動実行とレポート生成
```bash
./fanguard moderator --action auto --report-dir ./reports --email-alert admin@example.com
```
- 信頼度が30未満のユーザーに対し、自動で24時間ミュートを適用。
- 毎日午前3時に、コミュニティ健全性スコアと異常検出サマリーをCSV形式で出力。
- レポートは `/reports/YYYY-MM-DD_fanguard_summary.csv` に保存。

## ベストプラクティス

### 1. マルチテナント環境でのリソース分離
各ファンコミュニティ（テナント）ごとに独立した PostgreSQL スキーマと Redis 名前空間を割り当てる。これにより、あるコミュニティでの大量リクエストが他のコミュニティに影響を与えることを防止。テナント間のデータ混在を防ぐため、すべてのクエリに `tenant_id` フィルタを必須とする。

### 2. 感情辞書の定期的なチューニング
日本語のスラングや新語は急速に変化するため、毎週月曜日に `sentiment_lexicon_ja` テーブルに対して自動アップデートを実行。更新元として、Twitter API から収集した500件のサンプル投稿を教師データとして利用。更新後は必ずA/Bテストを実施し、誤検出率が2%を超えた場合はロールバック。
