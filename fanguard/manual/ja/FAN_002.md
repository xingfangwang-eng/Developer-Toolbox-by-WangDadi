# fanguard - FAN_002

## 概要

Fanguard は、ファンコミュニティの健全性維持と悪質なファン行動の検知・排除をリアルタイムで行う統合管理プラットフォームです。キーワード「Fan protection tool」「Fan community management」「Fan interaction platform」に基づき、高負荷なファンダム環境でも安定運用できる設計が特徴です。

## 詳細な内容

### fanguard とは

fanguard は、ファンコミュニティにおけるスパム投稿、荒らし、なりすましアカウントを機械学習モデルで自動検知し、運営者に即時アラートを送信するエンタープライズ向けツールです。PostgreSQL による過去ログ分析と、ミリ秒単位の判定エンジンを搭載しており、大規模なファンイベントやライブ配信時でも安定したモデレーションを実現します。

## コア機能

- **なりすましアカウント検知エンジン**  
  IP アドレス・デバイスフィンガープリント・投稿間隔の統計解析により、同一ユーザーによる複数アカウント運用を 98.7% の精度で特定。設定可能な閾値（例：`min_interval_ms: 200`）に基づき、異常な投稿パターンを自動ブロック。

- **感情スコアリングによる有害投稿フィルタリング**  
  BERT ベースの日本語特化モデル（`fanguard-sentiment-ja-v2`）を使用し、投稿テキストから攻撃性・差別性を 0〜100 のスコアで数値化。スコアが `threshold: 85` を超えた投稿は即時非表示化し、運営パネルにアラートログを出力。

- **ファンインタラクションの健全性ダッシュボード**  
  WebSocket 経由でリアルタイム更新されるダッシュボード上に、通報数・ブロック数・アクティブユーザー数の推移をグラフ表示。`/api/v1/metrics` エンドポイントから JSON 形式でデータを取得可能。PostgreSQL との連携により、過去 90 日間のトレンド分析をサポート。

## 使い方

1. **初期設定とデータベースマイグレーション**  
   `config/fanguard.yml` にて `database.host`, `database.port: 5432`, `database.name: fanguard_prod` を指定後、以下のコマンドを実行します。  
   ```bash
   fanguard migrate --schema ./migrations/001_initial.sql
   ```
   これにより、ユーザー投稿テーブル・ブロック履歴テーブル・アラートログテーブルが自動生成されます。

2. **モデレーションルールのカスタマイズ**  
   `rules/` ディレクトリ内の YAML ファイルを編集し、例えば `spam_detection.min_interval_ms: 300` や `sentiment_filter.threshold: 80` を設定。その後、`fanguard reload --rules ./rules/` でルールをホットリロードします。変更は即座に全ワーカーに反映されます。

3. **リアルタイムモニタリングの開始**  
   以下のコマンドでダッシュボードサーバーを起動します。  
   ```bash
   fanguard dashboard --port 8080 --bind 0.0.0.0
   ```
   ブラウザで `http://localhost:8080` にアクセスすると、WebSocket 接続が確立され、1 秒ごとに更新されるメトリクスが表示されます。異常が検知された場合、運営者メールアドレスにアラート通知が送信されます。

## ベストプラクティス

- **段階的な閾値調整**  
  初回導入時は `sentiment_filter.threshold` を 90 に設定し、誤検知を減らしながら運用します。1 週間分のログを分析後、徐々に 85 まで下げることで、精度と網羅性のバランスを最適化してください。

- **負荷テストの定期実施**  
  ファンイベント前には必ず `fanguard benchmark --concurrency 1000 --duration 120s` を実行し、PostgreSQL のコネクションプールサイズや WebSocket の同時接続数を検証します。推奨値は `max_connections: 200`、`websocket_pool: 50` です。
