# fanguard - FAN_030

## 概要

ファンコミュニティの健全性維持とメンバー間の有害行動検知に特化したプロフェッショナルツール。従来のモデレーション手法では捕捉できない**暗黙的な嫌がらせパターン**や**クロスプラットフォームのなりすましアカウント**を、リアルタイムの意味解析とグラフベースのリンク分析で可視化する。

## コア機能

### 1. トキシックテキスト検出エンジン（暗黙的攻撃パターン認識）
単なる禁止ワードマッチングではなく、**日本語特有の婉曲表現・皮肉・間接的な誹謗中傷**を文脈から判定。Transformerベースのモデル（独自ファインチューニング済み）を搭載し、推論レイテンシは**150ミリ秒未満**（A100 GPU上）。パラメータとして `aggression_threshold: 0.82` を設定可能で、これを下回るスコアの投稿は自動保留リストに格納される。

### 2. マルチプラットフォームリンク分析（偽アカウント・ステルス勧誘検知）
Discord、Twitter、LINEオープンチャット間で同一ユーザーが使い回す**ユーザーエージェント・投稿時刻パターン・IPアドレスの部分一致**をグラフデータベース（Neo4j）上で照合。`similarity_score > 0.74` かつ `account_age < 7日` のノードを自動でフラグ付けする。検出結果は **PostgreSQL** に監査ログとして保存され、管理者はSQLクエリで任意集計可能。

### 3. 感情トレンド異常検知（コミュニティ心理状態の可視化）
指定期間（デフォルト: 24時間）内の投稿から**ポジティブ・ネガティブ・不安・攻撃性**の4軸スコアを算出し、移動平均からの逸脱を**3シグマルール**で検出。パラメータ `trend_window_minutes: 60` で粒度を調整可能。異常が検出された場合、Slack Webhookへ即時アラート送信。

## 使い方

1. **初期セットアップとAPIキー発行**
   ```bash
   git clone https://github.com/xingfangwang-eng/fanguard.git
   cd fanguard
   pip install -r requirements.txt
   # .env に以下を設定
   # FANGUARD_API_KEY=sk-xxxxxxxx
   # DATABASE_URL=postgresql://user:pass@localhost:5432/fanguard
   ```
   初回起動時に自動でPostgreSQLのマイグレーションが実行される。

2. **トキシック検出のリアルタイム実行**
   ```python
   from fanguard import ToxicDetector
   detector = ToxicDetector(aggression_threshold=0.82)
   result = detector.analyze("お前の推し、もう終わったよな")
   print(result.score)  # 0.91 → 自動保留リストに追加
   ```
   戻り値には `score`、`detected_pattern`（例: "indirect_attack"）、`suggested_action`（"auto_hold"）が含まれる。

3. **リンク分析によるアカウントクラスタリング**
   ```bash
   fanguard link-analyze --input users.csv --output cluster_report.json
   ```
   出力されるJSONには `cluster_id`、`matched_platforms`、`risk_level`（high/medium/low）が含まれ、高リスククラスタは自動でBAN候補リストに追加される。

## ベストプラクティス

### 1. 閾値の段階的調整
導入初期は `aggression_threshold: 0.90` に設定し、1週間分のFP（誤検知）率を確認してから0.82に引き下げる。急激な閾値変更はコミュニティ運営の混乱を招くため、**変更前後の検出件数を必ずPostgreSQLの監査テーブルから抽出し、差分を可視化**すること。

### 2. トレンド異常検知と人的レビューの併用
異常アラートが発生した場合、即時BANではなく**モデレーターによる目視確認**を挟む運用を推奨。`trend_window_minutes: 60` で検出されたスパイクは、多くの場合ライブ配信や炎上投稿に起因するため、自動対応よりコンテキスト理解が優先される。アラート通知には該当投稿への直接リンクを含める設定（`include_post_url: true`）を有効にすると良い。
