# fanguard - FAN_019

## 概要

fanguard は、ファンコミュニティの健全性維持と不正アカウント対策に特化したプロフェッショナル向けファン保護エンジンです。キーワード「Fan protection tool」「Fan community management」「Fan interaction platform」に基づき、リアルタイムな異常検知と権限管理を実現します。

## 詳細な内容

### fanguard とは

fanguard は、SNSやプラットフォーム上のファン間の相互作用を監視し、スパム・Bot・不正投票・なりすましアカウントを自動で検出・排除するバックエンドツールです。PostgreSQL と Redis を連携し、ミリ秒単位の判定を可能にします。

### コア機能

- **リアルタイム行動スコアリングエンジン**  
  各ユーザーの投稿頻度・いいねパターン・フォロー解除率を動的に分析。閾値（例：1時間あたり200いいね超）を超えた場合、自動でアカウント制限を発動。スコアは Redis にキャッシュされ、応答時間は平均 12ms 以下。

- **マルチプラットフォーム相互検証機能**  
  Twitter・Discord・独自フォーラム間で同一ユーザーの行動を横断的に照合。IPアドレス・User-Agent・投稿時間帯の一致率が 85% 以上のアカウントを「同一運営者疑い」としてフラグ。PostgreSQL の全文検索インデックス（GIN）を活用。

- **カスタムルール自動生成 & テストサンドボックス**  
  管理者が設定した条件（例：「新規アカウントで5分以内に10件のリンク投稿」）に基づき、ルールを自動生成。サンドボックスモードでは、実際のデータに対してルール適用前後の影響度をシミュレーション可能。パラメータ `--simulate` で差分を JSON 出力。

### 使い方

1. **初期設定とデータベース接続**  
   `config.yml` に PostgreSQL の接続情報（ホスト、ポート、データベース名）と Redis のエンドポイントを記述。`fanguard init --db-uri postgresql://user:pass@localhost:5432/fanguard` を実行し、テーブル自動作成とインデックス構築を完了させる。

2. **行動スコアリングポリシーの定義**  
   `policy.json` に重み付けパラメータ（例：`"action_weight": {"like": 2, "comment": 5, "follow": 3}`）と閾値（例：`"threshold": 150`）を指定。`fanguard policy --apply policy.json` でエンジンに反映。

3. **異常検知の実行とレポート出力**  
   `fanguard scan --platform twitter --since 2025-01-01 --until 2025-01-31 --output report.csv` で指定期間のログ分析を実行。結果は CSV に出力され、各アカウントのスコア、フラグ状態、対応履歴が記録される。

### ベストプラクティス

- **段階的制限の導入**  
  即時BANではなく、スコアに応じて「警告→ミュート→一時停止→BAN」の4段階制限を設定する。これにより、誤検知によるファン離れを最小限に抑えつつ、悪質ユーザーの排除精度を向上させる。推奨パラメータは `"escalation_delay": 86400`（秒単位）。

- **定期的なルールメンテナンスとサンドボックス検証**  
  新たなBotパターンが発見された場合、`fanguard sandbox --input new_patterns.json` で既存ルールとの競合を確認してから本番反映する。また、月次で `fanguard audit --anomaly-only` を実行し、未検出の異常パターンを洗い出すサイクルを推奨。
