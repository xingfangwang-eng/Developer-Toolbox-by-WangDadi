# fanguard - FAN_001

## 概要

fanguard は、ファンコミュニティにおける荒らし・スパム・著作権侵害をリアルタイムで検知・防御するためのエンタープライズグレード Fan Protection Engine です。キーワード「Fan protection tool」「Fan community management」「Fan interaction platform」に基づき、大規模ファンクラブ運営者向けに設計されたセキュリティ＆モデレーション基盤を提供します。

## 詳細な内容

### fanguard とは

fanguard は、ファンコミュニティの健全性を維持するために開発された、**AI 駆動型のマルチレイヤーモデレーションツール**です。プラットフォーム（Discord、Slack、独自SNS）に依存せず、Webhook および REST API 経由で統合可能。PostgreSQL と連携し、全モデレーションログをミリ秒単位で記録・分析します。

### コア機能

- **マルチリンガルスパム検知エンジン**  
  日本語・英語・中国語・韓国語を含む 12 言語に対応。Transformer ベースの軽量モデル（パラメータ数 85M）をエッジで実行し、テキスト投稿から 200ms 以内にスパムスコア（0.0〜1.0）を算出。閾値は管理者が 0.05 刻みで設定可能。

- **ファンアート著作権侵害スキャナー**  
  投稿画像を SHA-256 ハッシュと Perceptual Hash（pHash）の両方でインデックス化。既存の登録作品データベース（最大 100 万件）と照合し、類似度 0.85 以上で自動フラグ。照合速度は 1 枚あたり平均 120ms。

- **インタラクション異常検知ダッシュボード**  
  WebSocket を使用したリアルタイムストリーミング。1 秒あたり最大 5,000 イベントを処理。同一 IP からの短時間連続投稿（10 秒以内に 20 件以上）や、新規アカウントによる過剰なメンション（1 時間に 50 件以上）を自動検出し、アクション（警告・ミュート・BAN）をトリガー。

### 使い方

1. **API キーの発行と環境変数の設定**  
   fanguard 管理コンソールにログインし、プロジェクト作成後、自動生成される `FANGUARD_API_KEY` を取得。サーバー環境で `export FANGUARD_API_KEY=sk-xxxx` を設定。PostgreSQL 接続文字列 `DATABASE_URL=postgresql://user:pass@host:5432/fanguard` も同時に設定。

2. **Webhook エンドポイントの登録**  
   モデレーション対象プラットフォームの Webhook 設定画面で、`https://api.fanguard.io/v1/webhook/incoming` を登録。リクエストヘッダーに `Authorization: Bearer ${FANGUARD_API_KEY}` を追加。必要に応じて `X-Fanguard-Community-Id` ヘッダーでコミュニティ識別子（UUID v4）を指定。

3. **モデレーションルールのカスタマイズ**  
   `POST /v1/rules` に JSON ボディを送信。例：  
   ```json
   {
     "community_id": "a1b2c3d4-...",
     "spam_threshold": 0.75,
     "image_similarity_threshold": 0.9,
     "rate_limit": {
       "max_posts_per_10s": 15,
       "max_mentions_per_hour": 30
     },
     "action_on_violation": "mute"
   }
   ```  
   設定後、即座に反映。変更履歴は PostgreSQL に保存され、監査可能。

### ベストプラクティス

- **段階的閾値適用**  
  初期運用時は spam_threshold を 0.9 に設定し、誤検知を最小化。1 週間分のデータを収集後、ログ分析に基づいて 0.75 まで段階的に下げる。これにより、コミュニティ固有のスラングや絵文字多用文化にエンジンを適応させる。

- **画像データベースの定期バックアップとクリーンアップ**  
  ファンアートのハッシュデータベースは、週次で pg_dump によるバックアップを推奨。同時に、90 日以上アクセスのない未確認画像ハッシュを自動削除する cron ジョブ（`DELETE FROM image_hashes WHERE last_seen < NOW() - INTERVAL '90 days'`）を設定し、照合パフォーマンスを維持。
