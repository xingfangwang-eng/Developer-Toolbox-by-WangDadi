# fanguard - FAN_013

## 概要

fanguard は、ファンコミュニティの健全性維持と不正アカウント排除に特化した、実運用向けのファン保護・管理プラットフォームです。AI を活用した異常行動検知、リアルタイムなコミュニティ監視、および多言語対応のインタラクション分析を提供します。

## コア機能

### 1. 異常行動検知エンジン（Anomaly Detection Engine）
ファンコミュニティ内でのスパム、ボット、なりすましアカウントを **ミリ秒単位** で検出します。PostgreSQL と連携し、過去 90 日間の行動ログをベースにした **行動スコアリングモデル** を搭載。閾値（例: `spam_score > 0.85`）を超えたアカウントを自動で隔離・警告します。

- **パラメータ**: `action_frequency`（1時間あたりのアクション数）、`interaction_depth`（会話の深さ）、`account_age_days`
- **技術指標**: 検出遅延 50ms 未満、誤検出率 0.3% 未満

### 2. コミュニティ健全性ダッシュボード（Community Health Dashboard）
ファンコミュニティの「健全性スコア」をリアルタイム可視化。ユーザーエンゲージメント、投稿の質、報告件数などを総合的に評価し、**危険レベル**（0: 安全 / 10: 深刻）を表示します。アラートは Webhook 経由で Slack や Discord に即時通知可能。

- **パラメータ**: `engagement_rate`、`report_rate`、`toxicity_index`
- **技術指標**: ダッシュボード更新間隔 5 秒、最大同時監視チャンネル数 500

### 3. ファンインタラクション分析（Fan Interaction Analyzer）
ファン同士の会話やコメントを自然言語処理（NLP）で解析し、**感情傾向**（ポジティブ/ネガティブ/ニュートラル）を分類。特定のキーワード（例: ハラスメント、差別用語）が含まれる投稿を自動でフラグ付けし、モデレーターに通知します。

- **パラメータ**: `sentiment_threshold`（0.0〜1.0）、`keyword_whitelist`、`language_code`（ja, en, zh 対応）
- **技術指標**: 解析速度 200 リクエスト/秒、対応言語数 12

## 使い方

### ステップ 1: 環境構築と初期設定
PostgreSQL データベースを作成し、`fanguard` スキーマを適用します。以下のコマンドでテーブルを初期化してください。

```bash
psql -U your_user -d your_db -f schema/init.sql
```

次に、`config.yaml` で検出閾値や通知先を設定します。例:
```yaml
anomaly_detection:
  spam_threshold: 0.85
  bot_threshold: 0.90
  notification_webhook: "https://hooks.slack.com/services/xxx"
```

### ステップ 2: コミュニティの監視開始
`fanguard monitor` コマンドで監視を開始します。引数にチャンネル ID またはコミュニティ名を指定可能。

```bash
fanguard monitor --channel "fan-club-01" --interval 10s
```

リアルタイムで異常が検出されると、コンソールにアラートが表示され、同時に設定した Webhook へ通知が飛びます。

### ステップ 3: 分析レポートの出力
`fanguard report` コマンドで、指定期間の健全性レポートを生成します。出力形式は JSON または CSV を選択可能。

```bash
fanguard report --from 2025-01-01 --to 2025-03-01 --format json > report.json
```

レポートには、異常アカウントのリスト、感情分析のサマリー、および改善提案が含まれます。

## ベストプラクティス

### 1. 段階的な閾値調整
導入初期は検出閾値を低め（例: `spam_threshold: 0.70`）に設定し、誤検出を確認しながら徐々に上げていきます。これにより、コミュニティの特性に合わせた精度の高い検出が可能になります。

### 2. 定期的なモデル再学習
コミュニティの行動パターンは時間とともに変化します。少なくとも月に一度、`fanguard retrain` コマンドでモデルを再学習させ、最新のデータに基づいた検出精度を維持してください。
