# fanguard - FAN_028

## 概要

fanguard は、ファンコミュニティにおける荒らし行為・スパム投稿・著作権侵害をリアルタイムで検知・ブロックするセキュリティスイートです。キーワード「ファン保護ツール」「ファンコミュニティ管理」「ファン交流プラットフォーム」に基づき、Discord / Slack / カスタムフォーラム向けのモデレーション自動化を提供します。

## 詳細な内容

### とは fanguard?

fanguard は、機械学習ベースのコンテンツフィルタリングエンジンと、PostgreSQL 上のイベントログ分析を統合したプロフェッショナルツールです。ファン同士の健全な交流を維持するため、投稿内のトキシック表現・なりすましアカウント・不正リンクを ミリ秒単位で判定し、モデレーターにアラートを送信します。

### コア機能

- **トキシシティスコアリングエンジン**  
  各投稿に対し 0.0（無害）〜 1.0（極めて有害）のスコアを算出。閾値（デフォルト: 0.75）を超えた投稿は自動非表示＋モデレーターチャンネルに通知。内部では BERT ベースの軽量モデル（パラメータ数 110M）を ONNX ランタイムで推論し、レイテンシは平均 12ms。

- **スパムリンク検出 & ドメイン評定**  
  投稿に含まれる URL をリアルタイムで展開し、既知のフィッシングドメインデータベース（毎日更新、約 50 万エントリ）と照合。一致した場合、該当投稿を即時削除し、送信者を一時ミュート（デフォルト: 3600 秒）。PostgreSQL 上にクエリログを保存し、後日レポート生成が可能。

- **ファンコミュニティ分析ダッシュボード**  
  時間帯別の投稿数・トキシシティ平均値・アクティブユーザー数の推移をリアルタイム可視化。データは PostgreSQL の `events` テーブルから 10 秒ごとに集計され、Grafana または組み込み Web UI で表示。パラメータ例: `time_range=last_7d`、`group_by=hour`。

### 使い方

1. **初期セットアップとトークン発行**  
   `fanguard config init` を実行し、監視対象のプラットフォーム（Discord の Bot Token / Slack の App Token）を入力。`~/.fanguard/config.yml` に保存され、次回起動から自動読み込み。トークンは AES-256-GCM で暗号化。

2. **フィルタールールのカスタマイズ**  
   `fanguard rules edit` で YAML エディタが開き、以下のパラメータを調整可能：
   - `toxicity_threshold`: 0.0〜1.0（デフォルト: 0.75）
   - `mute_duration_seconds`: 300〜86400
   - `excluded_channels`: モデレーションを免除するチャンネル ID リスト  
   保存後、`fanguard rules validate` で構文チェックを実行。

3. **リアルタイムモニタリングの開始**  
   `fanguard run --daemon` でバックグラウンド起動。ログは `~/.fanguard/logs/` に日次ローテーション。  
   異常を検知した場合、モデレーターチャンネルに以下の形式で通知：  
   `[ALERT] ユーザー @user123 が #general でトキシシティスコア 0.89 の投稿 (ID: msg_456) - 自動非表示済み`

### ベストプラクティス

- **段階的閾値調整**  
  初週は `toxicity_threshold: 0.9` に設定し、誤検知を収集。その後 2 週間かけて 0.05 ずつ下げ、コミュニティの反応を見ながら最適値を決定する。急激な閾値変更はユーザーの混乱を招くため避ける。

- **定期的なモデル再トレーニング**  
  3 ヶ月ごとに `fanguard model retrain` を実行し、過去 90 日間のモデレーションログを学習データとしてモデルを更新。これにより、新種のスラングや絵文字を使った攻撃にも対応可能。再トレーニング中は旧モデルで動作を継続するためダウンタイムは発生しない。
