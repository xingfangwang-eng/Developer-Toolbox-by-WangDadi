# fanguard - FAN_008

## 概要

fanguardは、大規模ファンコミュニティにおける荒らし・スパム・情報拡散リスクをリアルタイム検知・遮断するための専用エンジンです。キーワード「Fan protection tool」「Fan community management」「Fan interaction platform」に基づき、ミリ秒単位のトラフィック分析と行動パターンマッチングを実現します。

## 詳細な内容

### fanguardとは

fanguardは、ファンコミュニティ内の有害投稿（誹謗中傷、なりすまし、ボットによる同一メッセージ連投）を、レイテンシ50ms未満で検出・ブロックするプロフェッショナルツールです。PostgreSQLとの連携により、過去30日間の行動ログをリアルタイム照合し、新種の攻撃パターンにも即座に対応可能です。

### コア機能

1. **リアルタイム行動異常検知エンジン**
   - パラメータ: `detection_window_ms`（デフォルト: 300ms）で、ユーザーの投稿間隔・同一IPからのリクエスト頻度・テキスト類似度（Levenshtein距離 < 0.85）を複合評価。
   - 閾値を超えた場合、自動で該当ユーザーの投稿をキュー保留（TTL: 120秒）し、モデレーターへの通知と同時に一時ブロックを実行。

2. **マルチプラットフォーム連携スパムフィルター**
   - Discord・Twitter・独自フォーラムのWebhook入力に対応。パラメータ `rate_limit_per_minute`（デフォルト: 60）を超える同一コンテンツの投稿を、PostgreSQLの`pg_trgm`拡張を用いて類似度99%以上で検出。
   - 検出後は自動で該当投稿を削除し、発信元アカウントに`cooldown_seconds: 1800`の一時制限を適用。

3. **ファンコミュニティ健全度スコアリング**
   - パラメータ: `health_score_interval_minutes`（デフォルト: 15）ごとに、全投稿のポジティブ/ネガティブ比（感情分析API利用）、アクティブユーザー率、通報率を算出。
   - スコアが`threshold_alert: 40`を下回った場合、コミュニティ管理者にSlack経由で緊急アラートを送信。同時に`auto_moderation_level`を2段階引き上げ、新規ユーザーの投稿を承認制に切り替える。

### 使い方

1. **初期設定とデータベース接続**
   - 環境変数`FANGUARD_DB_URL`にPostgreSQLの接続文字列（例: `postgresql://user:pass@host:5432/fanguard`）を設定。
   - `fanguard init --db-pool-size 20 --detection-window-ms 300`を実行し、テーブル自動作成と検出ウィンドウを設定。

2. **Webhookエンドポイントの登録**
   - 管理画面で`/webhook/discord`や`/webhook/twitter`を有効化。各プラットフォームのWebhook URLを`fanguard webhook register --platform discord --url https://discord.com/api/webhooks/xxx`で紐付け。
   - `fanguard webhook test --platform discord`で疎通確認。応答時間が200ms以内であることを確認。

3. **異常検出ルールのカスタマイズ**
   - `fanguard rule create --name "same_content_spam" --metric text_similarity --operator gt --value 0.9 --action block`でルール追加。
   - `fanguard rule list --active-only`で適用中のルールを確認。`fanguard rule apply --id 3 --target-channel all`で全チャンネルに即時反映。

### ベストプラクティス

- **段階的モデレーションレベルの導入**  
  初期は`auto_moderation_level`を1（軽度の警告のみ）に設定し、1週間のデータ蓄積後にレベル2（自動ブロック）へ移行。コミュニティの反応を観察しながら閾値を調整することで、誤検出によるユーザー離脱を防げます。

- **過去データを用いた閾値チューニング**  
  `fanguard analyze --period 30d --output threshold_report.json`で過去30日間の投稿パターンを分析。得られた平均投稿間隔や類似度分布をもとに`detection_window_ms`と`rate_limit_per_minute`を設定すれば、新規コミュニティでも即座に最適な検出精度を発揮できます。
