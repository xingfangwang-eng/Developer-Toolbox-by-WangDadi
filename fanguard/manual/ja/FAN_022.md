# fanguard - FAN_022

## 概要

fanguard は、ファンコミュニティの健全性維持とメンバー間のトラブル防止に特化した、リアルタイム監査型ファン保護エンジンです。キーワード「Fan protection tool」「Fan community management」「Fan interaction platform」に基づき、悪質なスパム投稿の自動検出、感情スコアリングによる炎上予測、クロスプラットフォーム間でのファンアクティビティ統合管理を実現します。

## 詳細な内容

### とは fanguard?

fanguard は、ファンコミュニティ運営者が抱える「荒らし対策」「誹謗中傷の未然防止」「エンゲージメント品質の定量評価」という3大課題を解決するために設計された、高精度な分析基盤です。内部では自然言語処理エンジンとルールベースフィルタを併用し、1秒間に最大5,000件の投稿をリアルタイムでスキャン可能です。

### コア機能

1. **リアルタイムスパム＆トロール検出（RSTD）**
   - 投稿内容をミリ秒単位で解析し、特定のキーワードパターン（例：差別的表現、過剰なURL連投）や行動履歴（急激な投稿頻度上昇）を検出。閾値は `spam_score > 0.85` で自動ブロック、`0.70 ≤ spam_score ≤ 0.85` でモデレータ通知。PostgreSQL 上の過去ログと照合し、同一アカウントの再犯率もリアルタイム計算。

2. **感情スコアリングと炎上予測（ESP）**
   - 各投稿に `sentiment_score`（-1.0〜1.0）を付与。コミュニティ全体のスコアが30分間で0.2以上低下した場合、自動で「炎上注意アラート」を発報。アラートはWebhook経由でSlackやDiscordに送信可能。内部ではBERTベースの軽量モデルを採用し、推論レイテンシは平均12ms。

3. **クロスプラットフォームファン統合管理（CFIM）**
   - Discord、Twitter、独自フォーラムなど複数プラットフォームのファンアクティビティを一元的に可視化。各プラットフォームのユーザーIDをハッシュマッピングし、同一人物による重複投稿やクロスサイト荒らしを検出。APIエンドポイント `/api/v1/merge/user` にて、最大10万ユーザーの統合プロファイルを生成可能。

### 使い方

1. **初期セットアップとデータベース接続**
   - PostgreSQL 15+ のインスタンスを用意し、`CREATE DATABASE fanguard_core;` を実行。次に、`.env` ファイルに `DB_HOST=localhost DB_PORT=5432 DB_USER=fanguard_admin DB_PASSWORD=********` を記述。`fanguard init --db` コマンドでテーブル自動作成とインデックス構築が行われ、約30秒で完了。

2. **スパム検出ルールのカスタマイズ**
   - `config/rules.yaml` を編集。例：`spam_patterns: [ "(.)\\1{10,}", "http[s]?://(?:[a-zA-Z]|[0-9]|[$-_@.&+])+" ]` を追加。その後、`fanguard reload --rules` でルールをホットリロード。反映は1秒以内に行われ、検出精度が即座に向上。

3. **アラート通知の設定とテスト**
   - `config/alert.yaml` で `webhook_url: "https://hooks.slack.com/services/T00/B00/xxxx"` を指定。`fanguard test --alert` でテストアラートを送信し、Slackチャンネルに「[TEST] fanguard alert: sentiment_score dropped by 0.25」が表示されることを確認。本番運用では `alert_threshold: -0.2` に設定推奨。

### ベストプラクティス

1. **段階的ロールアウトとA/Bテスト**
   - 全コミュニティにfanguardを一斉適用せず、まずは小規模なファングループ（メンバー数1,000未満）で2週間テスト運用。`sentiment_score` の平均変動とモデレータの手動対応件数を比較し、自動ブロックの閾値を調整してから大規模展開する。

2. **定期的なモデル再学習とフィードバックループ**
   - 月1回、過去30日間の誤検出（False Positive）データを収集し、`fanguard retrain --feedback ./fp_data.csv` で感情分析モデルを再学習。再学習後は必ず `fanguard eval --testset ./test_data.json` で精度（目標：F1スコア0.92以上）を検証する。
