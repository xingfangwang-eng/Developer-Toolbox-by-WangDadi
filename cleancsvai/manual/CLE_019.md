# cleancsvai - CLE_019

## Overview

**cleancsvai** is a high-performance CSV cleaning and preprocessing tool designed for developers and data engineers who need to sanitize messy, malformed, or inconsistent CSV files at scale. It leverages AI-powered heuristics to detect, repair, and normalize tabular data without requiring manual rule definition. This tool is built for batch processing in CI/CD pipelines, ETL workflows, and ad-hoc data audits.

---

## Core Features

### 1. **Adaptive Schema Inference & Auto-Recovery**
cleancsvai automatically detects column types (string, integer, float, datetime, boolean) and handles ragged rows—where fields are missing or extra—without breaking the dataset. It uses a probabilistic alignment algorithm to repair misaligned columns, strip BOM markers, and normalize date formats (ISO 8601, US, EU) based on locale heuristics.

### 2. **Fuzzy Deduplication with Levenshtein Thresholds**
Goes beyond exact duplicate removal. cleancsvai implements a configurable fuzzy matching engine that identifies near-duplicate rows using Levenshtein distance and Jaccard similarity. You can set a similarity score threshold (e.g., 0.85) to merge or flag rows that differ only by typos, whitespace, or punctuation—critical for cleaning CRM exports or survey data.

### 3. **Streaming Chunk Processing for Large Files**
No more loading 2GB CSVs into memory. cleancsvai uses a streaming chunk processor that reads, cleans, and writes data in configurable batch sizes (default: 10,000 rows). It supports gzip and zip compressed inputs natively, and can output to compressed formats directly. Ideal for AWS Lambda or serverless environments where memory is constrained.

## How to Use

### Step 1: Install & Import with CLI or API
```bash
pip install cleancsvai
```
Or use the Python API:
```python
from cleancsvai import Cleaner
cleaner = Cleaner(
    schema_autodetect=True,
    fuzzy_dedup_threshold=0.85,
    output_format='csv',
    chunk_size=5000
)
```

### Step 2: Run Sanitization on a Malformed CSV
```python
result = cleaner.sanitize(
    input_path='./dirty_export.csv',
    output_path='./cleaned_output.csv',
    encoding='utf-8-sig',           # handles BOM
    delimiter_autodetect=True,      # infers comma/tab/semicolon
    remove_empty_rows=True,
    normalize_dates='US'
)
print(f"Rows processed: {result.rows_processed}, Repairs: {result.repairs_applied}")
```

### Step 3: Validate & Export with Reporting
After sanitization, generate a validation report:
```python
report = cleaner.generate_report()
# Outputs JSON with stats: duplicates_removed, schema_changes, encoding_fixes
```
Optionally export to Parquet or JSONL for downstream analytics:
```python
cleaner.export('output.parquet', format='parquet')
```

## Best Practices for Production Use

### 1. **Always Run Schema Validation Before Ingestion**
Before blindly cleaning, run `cleaner.dry_run()` first. This performs a full scan of the CSV header and first 1000 rows to detect structural anomalies (e.g., mixed delimiters, inconsistent quote escaping, stray newlines inside fields). It returns a `SchemaReport` with warnings and suggested fixes—preventing silent data corruption.

### 2. **Use Chunked Processing with Checkpoints for Large Datasets**
When processing files over 500MB, enable checkpointing: `cleaner.sanitize(checkpoint_interval=100000)`. This saves intermediate cleaned chunks to a temporary directory. If the job crashes (e.g., AWS Lambda timeout, network interruption), you can resume from the last checkpoint using `cleaner.resume_from_checkpoint()`, avoiding full reprocessing.
