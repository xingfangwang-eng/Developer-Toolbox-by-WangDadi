# cleancsvai - CLE_007

## Overview

This is a high-performance CSV data sanitization and enrichment toolkit built for developers who hate messy spreadsheets.  
Optimized for **English-speaking** SaaS engineers and data ops professionals.

---

### What is cleancsvai?

cleancsvai is a CLI-first and API-ready tool that automatically detects, normalizes, and validates CSV files with minimal configuration.  
It eliminates guesswork from data cleaning by combining pattern recognition, type inference, and rule-based repair — all without uploading your data to a third-party server.

---

### Core Features

1. **Schema-Aware Anomaly Detection**  
   Automatically scans your CSV header and row structure to detect column type mismatches, missing values, and encoding corruption.  
   Supports multi-character delimiters, quoted fields, and mixed-line endings.

2. **Fuzzy Column Matching & Auto-Merge**  
   When your CSV has inconsistent column names (e.g., `email`, `e-mail`, `Email Address`), cleancsvai uses Levenshtein distance and n-gram similarity to align and merge them into a unified schema — no manual mapping required.

3. **Deterministic Data Repair with Rule Engine**  
   Define custom validation rules using a simple JSON or YAML config.  
   The engine applies regex-based transformations, type coercion, and conditional replacement — all executed in-memory with zero data leakage.

### How to Use

1. **Install & Run a Quick Scan**  
   ```bash
   npm install -g cleancsvai
   cleancsvai scan ./messy_data.csv
   ```
   This outputs a JSON report with column stats, anomaly counts, and suggested fixes.  
   No config file needed for basic usage.

2. **Apply Automatic Cleaning with Config**  
   Create a `cleancsvai.config.yaml` in your project root:
   ```yaml
   rules:
     - column: "phone"
       pattern: "^[+]?[0-9]{10,15}$"
       action: drop_invalid
     - column: "date"
       type: datetime
       format: "%Y-%m-%d"
   ```
   Then run:
   ```bash
   cleancsvai clean ./input.csv -o ./clean_output.csv
   ```

3. **Integrate into a CI/CD Pipeline**  
   Use the API mode in your Node.js or Python script:
   ```javascript
   import { cleanCSV } from 'cleancsvai';
   const result = await cleanCSV('./data.csv', { rules: './rules.json' });
   console.log(result.summary);
   ```
   This allows you to plug it into ETL workflows or GitHub Actions for automated data validation.

---

### Best Practices

1. **Run a Dry Scan Before Cleaning**  
   Always execute `cleancsvai scan` first to understand the data quality landscape.  
   This helps you avoid accidental data loss from over-aggressive rules.

2. **Version Your Cleaning Config**  
   Store your rule files in the same repository as your data pipeline.  
   This ensures reproducibility across environments and makes it easy to roll back changes.
