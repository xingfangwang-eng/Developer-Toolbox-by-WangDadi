## What is cleancsvai?

**cleancsvai** is a high-performance, AI-native data cleaning engine designed for developers, data analysts, and SaaS teams who need to sanitize, normalize, and enrich CSV files at scale. Unlike generic spreadsheet tools, cleancsvai leverages pattern recognition and statistical inference to detect and fix structural anomalies, missing values, encoding mismatches, and duplicate records—without writing a single line of boilerplate code.

## Core Features

### 1. Intelligent Schema Inference & Auto-Repair
cleancsvai automatically detects column types, date formats, and encoding inconsistencies across large CSV files. It applies context-aware transformations—like stripping whitespace, normalizing datetime strings, or casting numeric fields—without manual mapping. The engine also flags and repairs mixed-type columns (e.g., a column containing both `"123"` and `"N/A"`) using probabilistic type voting.

### 2. Multi-Pass Deduplication with Fuzzy Matching
Go beyond exact row matching. cleancsvai runs a configurable multi-pass deduplication algorithm that uses Levenshtein distance, Jaccard similarity, and token-set ratio to identify near-duplicate rows. This is critical for CRM exports, lead lists, or survey data where subtle typos or inconsistent casing can cause record duplication.

### 3. Column-Level Statistical Profiling & Anomaly Detection
Every CSV upload triggers an automatic statistical profile: null ratios, value distributions, outlier detection (via IQR and Z-score), and cardinality analysis. cleancsvai surfaces actionable insights like “Column `email` has 12% nulls and 3 invalid formats” or “Column `revenue` contains 4 outliers > 3σ from the mean.” You can apply one-click fixes or export the profiling report as JSON.

## How to Use

### Step 1: Upload & Schema Validation
Upload your CSV via the CLI (`cleancsvai scan ./data/leads.csv`) or the web dashboard. cleancsvai immediately runs a schema validation pass, detecting column count mismatches, header inconsistencies, and encoding issues (e.g., UTF-8 BOM vs. ISO-8859-1). The tool outputs a structured validation report with severity levels (`error`, `warning`, `info`) and suggests automatic fixes.

### Step 2: Apply Cleaning Rules via YAML Config
Define your cleaning pipeline in a `cleancsvai.yaml` file. Example:
```yaml
cleaning:
  - action: normalize_date
    column: created_at
    format: "%Y-%m-%d"
  - action: fill_missing
    column: revenue
    strategy: median
  - action: deduplicate
    method: fuzzy
    threshold: 0.85
```
Run `cleancsvai clean --config cleancsvai.yaml` to execute the pipeline. Each rule is logged with before/after stats for auditability.

### Step 3: Export & Version Control
After cleaning, export the result in multiple formats: clean CSV, Parquet, or JSON Lines. cleancsvai also generates a diff manifest (`.cleancsvai_diff.json`) that records every transformation applied, making it easy to integrate with CI/CD pipelines or data version control systems like DVC.

## Best Practices

### 1. Always Run Profiling Before Cleaning
Don’t blindly apply transformations. Use `cleancsvai profile` to generate a statistical summary of your dataset first. This reveals hidden issues like mixed date formats or high cardinality columns that could break downstream ETL pipelines. Profiling is cheap; fixing corrupted outputs is not.

### 2. Use Fuzzy Deduplication with a Conservative Threshold
When deduplicating CRM or customer data, start with a threshold of 0.9 (on a 0–1 scale) to avoid false positives. After reviewing the flagged pairs, lower the threshold incrementally. This approach minimizes the risk of merging distinct records while still catching real duplicates caused by typos or formatting differences.
