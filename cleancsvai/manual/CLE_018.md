### What is cleancsvai?

cleancsvai is a **rule-driven, AI-augmented CSV sanitizer** that automates schema inference, anomaly detection, and data normalization at scale. It’s not just another CSV cleaner—it’s a **declarative data pipeline** for dirty spreadsheets.

---

### Core Features

1. **Schema-Aware Anomaly Injection Detection**  
   Automatically identifies and flags structural irregularities like mismatched delimiters, inconsistent quoting, or hidden null bytes. Uses entropy-based heuristics to surface corrupted rows before they break your ETL.

2. **Multi-Locale Normalization Engine**  
   Handles mixed date formats (`MM/DD/YYYY`, `DD.MM.YYYY`, `YYYY-MM-DD`), decimal separators (`1,234.56` vs `1.234,56`), and encoding mismatches (UTF-8, ISO-8859-1, Shift-JIS) in a single pass. No more locale-induced data rot.

3. **Idempotent Row Deduplication with Fingerprinting**  
   Generates content-aware hashes (SHA-256 + fuzzy similarity) to deduplicate rows without losing semantically identical but syntactically different records. Perfect for merging CRM exports or scraping outputs.

### How to Use

**Step 1: Declare Your Schema via YAML or Inline Annotation**  
Create a `.cleancsvai.yml` config file or annotate the first row of your CSV with type hints. Example:  
```yaml
columns:
  - name: email
    type: email
    required: true
  - name: date_joined
    type: date
    format: "%Y-%m-%d"
  - name: revenue
    type: float
    locale: de_DE
```
Then run:  
```bash
cleancsvai --config .cleancsvai.yml --input dirty.csv --output clean.csv
```

**Step 2: Execute a Dry Run with Diff Output**  
Before committing, preview changes using the `--dry-run` flag. The tool outputs a side-by-side diff of original vs cleaned rows, highlighting every transformation:  
```bash
cleancsvai --input dirty.csv --dry-run --diff-mode unified
```
This is critical for CI/CD pipelines—fail the build if anomaly count exceeds a threshold.

**Step 3: Apply Post-Processing Hooks**  
Chain custom Python or shell scripts via the `--post-hook` parameter. For example, automatically upload cleaned CSVs to S3 or trigger a dbt model refresh:  
```bash
cleancsvai --input dirty.csv --post-hook "python3 upload_to_s3.py" --post-hook "curl -X POST https://api.dbt.cloud/v1/jobs/123/run"
```

### Best Practices

1. **Treat Cleaning as a CI Step**  
   Integrate `cleancsvai` into your pre-commit hooks or GitHub Actions. Use `--fail-on-error` to reject PRs that introduce data quality regressions. This catches malformed CSVs before they pollute your data lake.

2. **Version Your Cleaning Rules**  
   Store `.cleancsvai.yml` in the same repo as your data pipeline. Tag rules with semantic versions (`v1.0.0`, `v2.1.0`) to ensure reproducibility. When you update a rule, the diff log shows exactly which records changed—audit-friendly and debug-proof.
