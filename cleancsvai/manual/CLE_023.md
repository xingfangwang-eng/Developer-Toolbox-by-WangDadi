# cleancsvai - CLE_023

## Overview

cleancsvai is a production-grade CSV cleaning and preprocessing toolkit designed for data engineers and analysts who need to eliminate noise, standardize schemas, and extract actionable insights from messy datasets. It combines deterministic rule engines with lightweight ML heuristics to handle missing values, type conflicts, and encoding anomalies at scale.

## Core Features

### 1. Schema-Aware Anomaly Detection & Auto-Correction
cleancsvai automatically infers column types (int, float, datetime, categorical) and flags inconsistent entries—e.g., a string in a numeric column or a date in ISO format mixed with MM/DD/YYYY. It applies context-aware corrections (e.g., padding missing digits, normalizing timezones) without breaking row alignment.

### 2. Multi-Strategy Duplicate Deduplication with Fuzzy Matching
Beyond exact row deduplication, cleancsvai supports Jaccard similarity, Levenshtein distance, and TF-IDF vectorization to merge near-duplicate records. You can set a similarity threshold and choose whether to keep the longest, most recent, or most complete row, with full audit logging of merged pairs.

### 3. Streaming Pipeline for Large CSV Files
Process files larger than RAM by using cleancsvai’s chunked reader with configurable buffer size. Each chunk is cleaned independently, then reassembled with consistent header mapping. Out-of-memory errors are eliminated even for 10GB+ CSV dumps.

## How to Use

### Step 1: Load and Profile Your CSV
Use `cleancsvai.profile(path, sample_size=5000)` to generate a data quality report. This returns a `Profile` object containing null counts, unique value ratios, inferred types, and outlier candidates. The profiler also detects delimiter inconsistencies (e.g., mixed commas and semicolons) and suggests a unified separator.

### Step 2: Define Cleaning Rules via YAML Config
Create a `cleaning_rules.yaml` file that specifies per-column actions:

```yaml
columns:
  - name: "transaction_date"
    type: "datetime"
    format: "mixed"           # Auto-detect from multiple formats
    fill_strategy: "forward_fill"
  - name: "amount"
    type: "float"
    outlier_action: "cap"     # Replace values > 3*IQR with boundary
    missing_action: "median"
  - name: "user_id"
    type: "string"
    dedup_strategy: "fuzzy"
    similarity_threshold: 0.85
```

Run `cleancsvai.clean(input.csv, rules.yaml, output.csv)` to apply all transformations.

### Step 3: Validate and Export Cleaned Data
After cleaning, execute `cleancsvai.validate(output.csv)` to run integrity checks: cross-column referential consistency, type enforcement, and null rate after imputation. The validator generates a JSON report with pass/fail status for each constraint. If validation passes, the cleaned CSV is ready for downstream ETL or analysis.

## Best Practices

### Practice 1: Always Profile Before Cleaning
Never apply cleaning rules blindly. Run the profiler first to understand null patterns, skewness, and type mismatches. For example, if a “zip_code” column has 5% entries with 4 digits, you may want to pad with a leading zero rather than drop rows. Profiling also reveals hidden columns (e.g., extra commas) that break parsing.

### Practice 2: Use Streaming for Production Pipelines
In CI/CD or scheduled data ingestion, configure cleancsvai to process files in chunks of 50,000 rows. This keeps memory usage flat and allows partial retries if a chunk fails. Combine with a dead-letter queue for rows that cannot be cleaned (e.g., irrecoverable parse errors) so you never lose raw data.
