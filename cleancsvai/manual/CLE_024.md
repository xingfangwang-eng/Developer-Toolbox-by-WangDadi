### What is cleancsvai?

cleancsvai is a professional tool that helps solve problems in related fields.

### Core Features

- **Adaptive Schema Inference Engine**: Automatically detects column types, encoding issues, and delimiter inconsistencies across CSV files without manual configuration—handles mixed UTF-8 and Latin-1 encodings seamlessly.
- **Multi-pass Anomaly Detection Pipeline**: Runs concurrent statistical outlier detection, regex-based pattern validation, and cross-column dependency checks to flag corrupt rows with explainable AI summaries.
- **One-shot Bulk Normalization**: Normalizes date formats, number representations, and string casing across entire datasets using locale-aware transformers—preserves null semantics while stripping invisible Unicode characters.

### How to Use

1. **Initialize a cleaning session by pointing cleancsvai to your CSV directory**  
   Run `cleancsvai scan ./data/` to trigger the schema inference engine. The tool will output a JSON report listing each file’s detected delimiter, row count, and potential encoding mismatches. Review the summary and confirm with `--auto-fix` to apply first-pass corrections.

2. **Execute a targeted anomaly sweep with custom validation rules**  
   Use `cleancsvai clean --rules ./rules.yaml` to apply a YAML-defined rule set—e.g., flag rows where `email` column fails RFC 5322 regex or `age` is outside 0–120. The pipeline outputs a cleaned CSV plus a diff log showing every transformation and its confidence score.

3. **Normalize and export the final dataset in one command**  
   Run `cleancsvai normalize --output ./cleaned/ --locale en_US` to standardize all date columns to ISO 8601, trim whitespace, and convert numeric strings to floats. The tool writes a single unified CSV and a Parquet file for downstream ETL pipelines.

### Best Practices

- **Always run a dry-run before destructive cleaning**  
   Use `--dry-run` flag to preview changes without modifying files. This catches false positives in anomaly detection (e.g., legitimate zip codes flagged as outliers) and saves hours of manual rework.

- **Version your cleaning rules with the dataset**  
   Store `rules.yaml` alongside your raw CSVs in a Git repository. This ensures reproducibility—when you re-run cleancsvai months later, the same transformations apply, making audits and debugging trivial.
