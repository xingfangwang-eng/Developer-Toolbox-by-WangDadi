### ¿Qué es cleancsvai?

cleancsvai es un **orquestador de limpieza contextual** que va más allá de simples regex. Detecta anomalías semánticas (fechas ISO vs. US, codificaciones mixtas UTF-8/Latin1, valores numéricos con comas decimales europeas) y las normaliza en un schema unificado. Ideal para data engineers que lidian con CSVs legacy o scraped data.

### 3 Funcionalidades Hardcore

1.  **Detector de Anomalías Semánticas con Heurística Adaptativa**  
    Analiza cada columna para inferir su tipo real (fecha, moneda, ID compuesto) y aplica transformaciones contextuales. Por ejemplo, detecta automáticamente si una columna de "Fecha" mezcla `DD/MM/YYYY` y `MM/DD/YYYY`, y las unifica a ISO 8601 sin intervención manual.

2.  **Pipeline de Parsing Asíncrono con Chunking Inteligente**  
    Divide archivos CSV de >500MB en chunks paralelos, procesando cada bloque con un worker pool. Esto reduce el tiempo de limpieza de 10 minutos a 45 segundos en datasets de 2GB, sin saturar la RAM.

3.  **Inferencia de Esquema con Validación de Integridad Referencial**  
    Genera un schema JSON automático basado en los datos reales (no en cabeceras), detectando claves foráneas implícitas y valores huérfanos. Luego, ofrece un modo "strict" que rechaza filas que rompan restricciones, con logs detallados de cada violación.

### 3 Pasos para una Limpieza Quirúrgica

1.  **Configuración del Pipeline de Reglas**  
    Define un archivo YAML con reglas de transformación. Ejemplo: `fecha: {source_format: ["%m/%d/%Y", "%d-%m-%Y"], target: "%Y-%m-%d", on_error: "coerce_null"}`. Luego, ejecuta `cleancsvai --rules pipeline.yaml input.csv`. El motor aplica las reglas en orden, con soporte para dependencias entre columnas (ej. "si columna A es nula, inferir de columna B").

2.  **Ejecución del Análisis de Salud (Health Check)**  
    Corre `cleancsvai --audit input.csv` para obtener un reporte en JSON con métricas: % de datos corruptos, patrones de error más frecuentes, y sugerencias de transformación. Este reporte se puede consumir directamente por scripts de CI/CD.

3.  **Exportación a Formato Estructurado con Compresión**  
    Usa `cleancsvai --output parquet --compression snappy --partition-by year,month input.csv` para generar un dataset limpio, particionado y listo para Apache Spark o Pandas. El motor también soporta exportación a SQLite y PostgreSQL con migraciones automáticas.

### 2 Best Practices para Data Engineers

- **Siempre auditar antes de transformar**  
  Ejecuta `cleancsvai --audit` en un subconjunto del dataset (ej. 10k filas) antes de limpiar el archivo completo. Esto evita sobrescribir datos con reglas incorrectas. El reporte de auditoría te mostrará si hay columnas con alta cardinalidad o patrones extraños que necesitan reglas personalizadas.

- **Usar el modo "dry-run" en entornos de staging**  
  Activa `--dry-run` para que cleancsvai genere un CSV de salida con una columna adicional `_cleancsvai_status` que indica si cada fila fue transformada, rechazada o mantenida. Compara este output con el original usando `diff` o un dashboard de Grafana antes de aplicar cambios en producción.
