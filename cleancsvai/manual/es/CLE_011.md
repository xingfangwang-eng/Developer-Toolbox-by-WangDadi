# cleancsvai - CLE_011

## Resumen

Herramienta de limpieza y análisis de datos CSV potenciada por inteligencia artificial. Elimina duplicados, normaliza formatos, detecta anomalías y estructura conjuntos de datos sucios en segundos. Ideal para data scientists, analistas y desarrolladores que trabajan con datos no estructurados.

## Contenido Detallado

### ¿Qué es cleancsvai?

cleancsvai es un motor de limpieza de datos CSV con capacidades de IA generativa. A diferencia de herramientas tradicionales que requieren reglas manuales, cleancsvai detecta automáticamente patrones, inconsistencias y valores atípicos, aplicando transformaciones inteligentes sin intervención humana. Soporta archivos de hasta 500 MB, con procesamiento en memoria y salida optimizada para pipelines de machine learning.

### Características Principales

- **Deduplicación Semántica con Embeddings**: Detecta filas duplicadas no solo por coincidencia exacta, sino por similitud semántica usando modelos de embeddings. Ideal para limpiar datasets con entradas como "New York, NY" vs "Nueva York, NY".
- **Normalización Automática de Formatos**: Reconoce y estandariza fechas (DD/MM/AAAA → ISO 8601), monedas ($1,234.56 → 1234.56), códigos postales y números de teléfono sin configuración previa.
- **Detección de Anomalías con Isolation Forest**: Incorpora un algoritmo no supervisado que identifica outliers numéricos y categóricos, marcándolos para revisión o eliminación automática.

### Cómo Usar

1. **Integración en Pipeline de Datos**:
   ```bash
   pip install cleancsvai
   python -c "from cleancsvai import Cleaner; c = Cleaner(); c.clean('input.csv', output='clean.csv', dedup=True, normalize_dates=True)"
   ```
   Este comando ejecuta la limpieza completa: elimina duplicados semánticos, normaliza fechas y exporta un CSV limpio. El parámetro `dedup=True` activa el modelo de embeddings; `normalize_dates=True` convierte automáticamente formatos regionales.

2. **Limpieza Interactiva con CLI**:
   ```bash
   cleancsvai --input dirty_data.csv --output clean_data.csv --anomaly-threshold 0.95 --verbose
   ```
   El flag `--anomaly-threshold 0.95` ajusta la sensibilidad del Isolation Forest (0.95 = solo marca el 5% más extremo). `--verbose` muestra en tiempo real cada transformación aplicada.

3. **Uso como Biblioteca en Notebooks**:
   ```python
   from cleancsvai import Cleaner
   import pandas as pd
   
   df = pd.read_csv('raw.csv')
   cleaner = Cleaner(strategy='conservative')  # 'conservative' preserva más datos
   df_clean = cleaner.fit_transform(df)
   print(f"Filas originales: {len(df)} | Filas limpias: {len(df_clean)}")
   ```
   `strategy='conservative'` minimiza la eliminación de datos, ideal para datasets pequeños donde cada fila importa.

### Mejores Prácticas

- **Auditar siempre la salida**: Aunque cleancsvai es preciso, revisa manualmente el archivo `_anomalies.csv` que se genera automáticamente. Contiene las filas marcadas como outliers con su score de anomalía. Úsalo para validar decisiones críticas.
- **Ajustar el threshold según el contexto**: Para datos financieros, usa `--anomaly-threshold 0.99` (solo el 1% más extremo). Para logs de servidores, `0.90` es más apropiado. Experimenta con diferentes valores en un subconjunto antes de procesar el dataset completo.
