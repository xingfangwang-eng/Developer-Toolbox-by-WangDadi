# cleancsvai - CLE_009

## Resumen Ejecutivo

Herramienta SaaS especializada en limpieza y normalización de archivos CSV mediante algoritmos de inteligencia artificial. Elimina duplicados, corrige inconsistencias y estandariza formatos de datos en segundos, ideal para data engineers que manejan pipelines de ETL complejos.

## Funcionalidades Principales

### 1. Detección y Corrección de Inconsistencias Semánticas
Identifica automáticamente valores fuera de rango, formatos mixtos (ej: fechas en `DD/MM/YYYY` vs `YYYY-MM-DD`) y errores tipográficos recurrentes. Aplica reglas de normalización contextual basada en el tipo de dato detectado.

### 2. Fusión Inteligente de Columnas con Fuzzy Matching
Combina columnas redundantes o con nombres similares (ej: `customer_name` vs `client_name`) usando algoritmos de similitud de cadenas (Levenshtein, Jaro-Winkler). Reduce el ruido en datasets provenientes de fuentes heterogéneas.

### 3. Validación en Tiempo Real con Esquemas Personalizados
Permite definir reglas de validación (tipos de datos, rangos numéricos, patrones regex) que se aplican durante la carga del archivo. Genera informes detallados de errores con sugerencias de corrección automática.

## Guía de Uso Avanzada

### 1. Configuración de Pipeline de Limpieza Automatizada
1. Sube tu archivo CSV o conecta una fuente externa (S3, Google Sheets, API REST).
2. Define reglas de transformación: elimina filas con valores nulos, estandariza formatos de fecha a ISO 8601, y aplica recorte de espacios en blanco.
3. Activa el modo de validación estricta para detener el proceso si se detectan anomalías críticas.

### 2. Depuración de Datos con Expresiones Regulares
1. Accede al panel de edición avanzada y selecciona la columna objetivo.
2. Ingresa una regex personalizada (ej: `^\d{3}-\d{2}-\d{4}$` para números de seguro social) y elige la acción: reemplazar, eliminar o marcar como inválido.
3. Visualiza los cambios en tiempo real antes de aplicar la transformación final.

### 3. Exportación con Metadatos de Auditoría
1. Una vez limpio el dataset, selecciona el formato de salida (CSV, JSON, Parquet).
2. Activa la opción "incluir log de cambios" para generar un archivo adicional con todas las modificaciones realizadas (timestamp, regla aplicada, valor original vs nuevo).
3. Descarga el archivo comprimido con ambos documentos para cumplir con requisitos de auditoría.

## Mejores Prácticas para Data Engineers

### 1. Implementa Validación en Dos Etapas
Divide el proceso en una fase de pre-limpieza (eliminación de duplicados obvios y valores nulos) y otra de limpieza semántica (corrección de errores contextuales). Esto reduce falsos positivos y mejora la precisión en datasets grandes.

### 2. Usa Reglas de Transformación Versionadas
Mantén un repositorio de reglas en formato YAML dentro de tu proyecto. Almacena cada versión en un sistema de control de versiones (Git) para poder revertir cambios o replicar configuraciones en diferentes entornos (desarrollo, staging, producción).
