# cleancsvai - CLE_020

## Resumen

Herramienta de limpieza y análisis de archivos CSV basada en inteligencia artificial, diseñada para desarrolladores que necesitan transformar datos desordenados en estructuras listas para producción sin escribir ni una línea de código. Soporta detección automática de anomalías, normalización de formatos y exportación optimizada para pipelines ETL.

## Contenido Detallado

### ¿Qué es cleancsvai?

cleancsvai es un motor de limpieza de datos que combina algoritmos de detección de patrones con modelos de lenguaje natural (LLM) para identificar y corregir errores típicos en archivos CSV: duplicados, valores nulos inconsistentes, formatos de fecha rotos, codificaciones mixtas y caracteres especiales mal escapados. Está pensado para equipos de data engineering que necesitan automatizar la pre-limpieza antes de alimentar bases de datos o dashboards.

### Características Principales (Hardcore)

- **Detección de anomalías semánticas con embeddings**: cleancsvai no solo busca celdas vacías o duplicados exactos. Usa embeddings generados por modelos transformer para identificar registros que, aunque sintácticamente válidos, son semánticamente inconsistentes (ej. un precio en euros mezclado con precios en dólares en una misma columna). El sistema sugiere correcciones contextuales basadas en el conjunto de datos completo.

- **Normalización de esquemas con inferencia de tipos fuzzy**: Analiza cada columna y aplica un algoritmo de inferencia de tipos que soporta formatos mixtos (ej. fechas en "dd/mm/yyyy", "yyyy-mm-dd" y "MMM dd, yyyy" en la misma columna). Convierte todo a un estándar ISO 8601 sin necesidad de especificar el formato de entrada. También detecta y corrige codificaciones UTF-8 vs ISO-8859-1 de forma automática.

- **Exportación con metadatos de linaje**: Al exportar el CSV limpio, cleancsvai genera un archivo JSON con el log completo de transformaciones aplicadas: qué filas fueron eliminadas, qué valores se corrigieron, qué columnas se renombraron y el motivo de cada cambio. Esto es crítico para auditorías y compliance en entornos regulados (GDPR, HIPAA).

### Cómo Usar

1. **Inyección del archivo y análisis previo**:
   Sube tu CSV directamente desde la CLI o mediante un endpoint REST. cleancsvai ejecuta un análisis preliminar que genera un resumen de calidad: número de filas, columnas, valores nulos, tipos inferidos y distribución de anomalías. Recibirás un JSON con métricas como `"calidad_general": 0.87` y recomendaciones de acciones.

2. **Configuración de reglas de limpieza**:
   Define reglas con sintaxis YAML. Ejemplo:
   ```yaml
   limpieza:
     columnas:
       fecha_venta:
         tipo: fecha
         formato_esperado: ISO8601
         accion_si_invalido: corregir_con_ia
       precio:
         tipo: decimal
         separador_decimal: punto
         accion_si_nulo: imputar_mediana
     global:
       eliminar_duplicados: true
       umbral_similitud: 0.95
   ```
   Si no defines reglas, cleancsvai aplica defaults inteligentes basados en el análisis previo.

3. **Ejecución y exportación con tracking**:
   Lanza la limpieza con `cleancsvai clean --input datos_sucios.csv --output datos_limpios.csv --log cambios.json`. El proceso es asíncrono; puedes monitorear el progreso vía WebSocket. Al finalizar, obtienes el CSV limpio y el archivo de linaje. Ejemplo de entrada de log:
   ```json
   {
     "tipo": "corrección",
     "columna": "fecha_venta",
     "fila_original": 234,
     "valor_original": "12/31/2023",
     "valor_corregido": "2023-12-31",
     "confianza": 0.99,
     "algoritmo": "inferencia_fuzzy_fechas"
   }
   ```

### Mejores Prácticas

- **Siempre ejecuta el análisis previo antes de limpiar**: El comando `cleancsvai analyze --input archivo.csv` te da una radiografía completa del estado del dataset. Usa esa información para decidir si necesitas reglas personalizadas o si los defaults son suficientes. Esto evita sobre-escribir datos válidos por error.

- **Integra cleancsvai en tu pipeline CI/CD**: Configura un step en tu GitHub Actions o GitLab CI que ejecute limpieza automática cada vez que se suba un CSV a un repositorio de datos. Combínalo con el log de linaje para tener trazabilidad completa. Ejemplo de step en YAML:
  ```yaml
  - name: Limpiar CSV con cleancsvai
    run: cleancsvai clean --input data/raw/ventas.csv --output data/clean/ventas.csv --log data/logs/ventas.json
  ```
