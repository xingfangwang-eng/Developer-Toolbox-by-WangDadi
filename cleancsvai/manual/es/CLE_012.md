# cleancsvai - CLE_012

## Resumen

Elimina inconsistencias, duplicados y errores estructurales en archivos CSV con cleancsvai. Esta herramienta nativa en Python procesa datasets de hasta 10M de registros en segundos, aplicando reglas de limpieza configurables y detección automática de anomalías. Ideal para ETL, migraciones de datos y preparación de feeds para modelos ML.

## Contenido Detallado

### ¿Qué es cleancsvai?

cleancsvai no es un simple limpiador de CSV. Es un pipeline inteligente que analiza el esquema del archivo, infiere tipos de datos (fechas, números, booleanos, cadenas), y aplica transformaciones en lote sin necesidad de scripting manual. Detecta valores nulos, formatos inconsistentes y registros corruptos, y ofrece un reporte de calidad de datos exportable en JSON o Markdown.

### Características Principales

- **Inferencia de esquema y normalización de tipos**: cleancsvai analiza las primeras 1000 filas para determinar el tipo de dato de cada columna (int, float, datetime, string, categorical). Luego normaliza automáticamente formatos: fechas en ISO 8601, números con separador de miles y decimal estandarizado (punto), y strings sin espacios extra ni caracteres de control.
- **Deduplicación difusa con umbral configurable**: No solo elimina filas exactamente iguales. Usa similitud de Jaccard y Levenshtein para detectar duplicados semánticos (ej: "John Doe" vs "John D."). Ajusta el umbral de similitud (0.0 a 1.0) para controlar la agresividad del filtro.
- **Corrección de errores estructurales**: Detecta y repara problemas como columnas desplazadas (valores en columna incorrecta), líneas con número de campos diferente al header, y caracteres de escape mal formados. Incluye un modo "estricto" que detiene el proceso si encuentra más de un 5% de filas corruptas.

### Cómo Usar

1. **Preparación del archivo y configuración de reglas**  
   Coloca tu CSV en el directorio `data/input/`. Crea un archivo YAML `config/rules.yaml` donde defines reglas de limpieza:  
   ```yaml
   columns:
     fecha: { type: datetime, format: "%d/%m/%Y", normalize: true }
     precio: { type: float, replace_na: 0.0, min: 0 }
     nombre: { type: string, trim: true, uppercase: false }
   duplicates:
     threshold: 0.85
     columns: ["nombre", "email"]
   ```
   Ejecuta `cleancsvai --config config/rules.yaml --input data/input/clientes.csv --output data/output/clientes_clean.csv`.

2. **Análisis de calidad y revisión de anomalías**  
   Antes de aplicar cambios, genera un reporte de calidad: `cleancsvai --audit-only --input data/input/clientes.csv`. El reporte muestra:  
   - % de valores nulos por columna  
   - Número de duplicados exactos y difusos  
   - Filas con errores de estructura (columnas faltantes, tipos incorrectos)  
   - Histograma de valores únicos para columnas categóricas  
   Revisa el JSON generado en `data/audit/report.json` para decidir si ajustas reglas.

3. **Ejecución en modo batch y exportación de logs**  
   Para procesar múltiples archivos, usa un bucle en shell:  
   ```bash
   for file in data/input/*.csv; do
     cleancsvai --config config/rules.yaml --input "$file" --output "data/output/$(basename "$file")" --log-file "logs/$(basename "$file").log"
   done
   ```
   Cada log incluye: timestamp, número de filas procesadas, errores detectados y tiempo de ejecución. Útil para auditoría en pipelines CI/CD.

### Mejores Prácticas

- **Usa un archivo de reglas versionado**: Mantén `rules.yaml` en Git. Así puedes trackear cambios en las reglas de limpieza y revertir si una transformación rompe el dataset. Añade comentarios en YAML explicando cada regla para facilitar la revisión entre colegas.
- **Ejecuta siempre el modo auditoría antes de limpiar**: El `--audit-only` te da visibilidad de la calidad del dataset sin modificarlo. Si encuentras >10% de filas corruptas, mejor corrige el origen del archivo (ej: error en exportación de base de datos) antes de aplicar cleancsvai. Esto evita que el tool "maquille" problemas subyacentes.
