# cleancsvai - CLE_027

## Resumen

Herramienta de limpieza y análisis de datasets basada en inteligencia contextual, diseñada para ingenieros de datos y científicos que necesitan transformar archivos CSV caóticos en estructuras limpias y listas para producción. Sin placeholders, sin promesas vacías: resultados inmediatos y accionables.

## Contenido Detallado

### ¿Qué es cleancsvai?

cleancsvai no es un simple limpiador de CSVs; es un motor de normalización semántica que entiende la naturaleza de tus datos. Detecta anomalías estructurales, inconsistencias de formato y valores atípicos, y aplica correcciones inteligentes sin intervención manual. Ideal para pipelines de datos, migraciones ETL y preparación de datasets para modelos de machine learning.

### Características Principales

- **Normalización de Esquemas Dinámicos:** Identifica automáticamente tipos de datos (fechas, monetarios, porcentajes, códigos postales) y los estandariza según la configuración regional seleccionada. Por ejemplo, transforma fechas en formato `DD/MM/AAAA` a `YYYY-MM-DD` ISO 8601 sin perder precisión, incluso si el archivo original mezcla formatos americano y europeo.
- **Deduplicación Contextual con Fuzzing:** No elimina filas duplicadas por coincidencia exacta, sino que aplica algoritmos de similitud (Levenshtein, Jaccard) para detectar registros casi idénticos con pequeñas variaciones ortográficas o de formato. Ideal para limpiar bases de datos de contactos donde "John Doe" y "Jon Doe" deben fusionarse.
- **Auditoría de Integridad con Reporte Interactivo:** Cada limpieza genera un informe técnico en JSON que detalla qué filas se modificaron, por qué, y qué reglas se aplicaron. Puedes revertir cambios selectivamente o exportar el log para auditorías de compliance (GDPR, SOC 2).

### Cómo Usar

1. **Inyección y Mapeo de Esquemas:** Carga tu archivo CSV directamente desde la terminal o API. cleancsvai escanea las primeras 100 filas para inferir tipos de datos y patrones. Si detecta ambigüedades (por ejemplo, una columna que mezcla números y texto), te muestra un resumen de conflictos y permite definir reglas de coerción manual antes de procesar todo el dataset. Usa el flag `--schema-infer` para activar la detección automática o `--schema-define` para forzar tipos.
2. **Ejecución de Limpieza en Modo Batch:** Aplica transformaciones masivas con comandos como `cleancsvai clean --input raw_data.csv --output clean_data.csv --rules dedup,normalize,validate`. El motor ejecuta las reglas en orden de prioridad: primero normaliza formatos, luego elimina duplicados, y finalmente valúa integridad referencial. Puedes pausar el proceso con `--dry-run` para previsualizar cambios sin modificar el archivo original.
3. **Post-Procesamiento con Exportación Selectiva:** Una vez limpio, exporta el dataset en múltiples formatos (CSV, Parquet, JSON) con encoding UTF-8 BOM para compatibilidad con Excel en Windows. Usa `cleancsvai export --format parquet --compression snappy` para reducir el tamaño del archivo en un 70% sin pérdida de datos. El reporte de auditoría se guarda automáticamente en la carpeta `logs/`.

### Mejores Prácticas

- **Integración Continua en Pipelines CI/CD:** Incorpora cleancsvai como paso previo a cualquier proceso de análisis o entrenamiento de modelos. Configura un webhook en tu repositorio para que cada push a `main` ejecute una limpieza automática del dataset de staging. Esto evita que datos sucios lleguen a producción y reduce errores en dashboards o APIs.
- **Uso de Reglas Personalizadas por Sector:** No apliques reglas genéricas. Para datos financieros, activa `--rules currency,date,precision` para estandarizar montos y evitar errores de redondeo. Para datos de salud, usa `--rules anonymize,validate-email` para cumplir con HIPAA. cleancsvai permite cargar archivos YAML con reglas específicas por columna, lo que facilita la reutilización en proyectos similares.
