# cleancsvai - CLE_014

## Resumen Ejecutivo

**cleancsvai** es un motor de limpieza y normalización de datos CSV impulsado por inteligencia artificial, diseñado para eliminar ruido estructural, corregir inconsistencias semánticas y preparar datasets listos para análisis o migración. Ideal para equipos de datos que trabajan con fuentes heterogéneas y necesitan automatizar tareas repetitivas de calidad de datos.

## Características Técnicas Diferenciales

### 1. **Motor de Inferencia Semántica de Columnas**
cleancsvai analiza automáticamente el contenido de cada columna y sugiere tipos de datos (fecha, moneda, porcentaje, código postal, etc.) basándose en patrones estadísticos y reglas predefinidas. No solo detecta tipos, sino que aplica transformaciones contextuales (ej. normalizar formatos de fecha `DD/MM/YYYY` a `YYYY-MM-DD` sin pérdida de precisión).

### 2. **Pipeline de Corrección Asíncrona con Reglas Condicionales**
Puedes definir reglas de validación y corrección en cadena, como: *"Si columna 'email' contiene '@' y no tiene dominio válido, marcar como inválido; si el campo 'precio' está vacío, rellenar con la mediana del grupo por 'categoría'"*. El motor ejecuta estas reglas en paralelo, con logs detallados de cada transformación.

### 3. **Eliminación Inteligente de Duplicados con Pesos de Similitud**
No es un simple `drop_duplicates()`. cleancsvai permite configurar pesos por columna (ej. 70% nombre, 30% dirección) y usar métricas de similitud (Levenshtein, Jaccard) para fusionar registros casi idénticos, manteniendo el dato más completo de cada grupo.

## Guía de Uso Avanzada

### Paso 1: Carga y Perfilado Automático del Dataset
Ejecuta el comando `cleancsvai profile dataset.csv`. La herramienta generará un informe de calidad: porcentaje de nulos, tipos inferidos, valores atípicos por columna y sugerencias de limpieza. Este perfil es la base para definir reglas personalizadas.

### Paso 2: Definición de Reglas de Limpieza en JSON
Crea un archivo `rules.json` con transformaciones condicionales. Ejemplo para normalizar un campo de teléfono:
```json
{
  "phone": {
    "type": "phone",
    "format": "+34XXXXXXXXX",
    "fallback": "remove_prefix"
  },
  "age": {
    "type": "integer",
    "min": 0,
    "max": 120,
    "outlier_action": "cap"
  }
}
```
Luego ejecuta: `cleancsvai apply dataset.csv --rules rules.json --output cleaned.csv`.

### Paso 3: Revisión y Exportación con Historial de Cambios
Tras la limpieza, el sistema genera un archivo `changes.log` con cada modificación (valor original → valor nuevo, regla aplicada). Puedes exportar el dataset limpio en CSV, Parquet o JSON. Para auditoría, usa `cleancsvai audit cleaned.csv --log changes.log`.

## Mejores Prácticas para Equipos de Datos

- **Integración en Pipelines CI/CD**: Automatiza la limpieza de datos como paso previo a cualquier entrenamiento de modelo o carga en base de datos. Usa `cleancsvai` como una etapa dentro de GitHub Actions o Airflow, garantizando que los datos de entrada siempre cumplan con un estándar mínimo de calidad.

- **Versionado de Reglas con Git**: Mantén tus archivos `rules.json` bajo control de versiones. Cada cambio en la lógica de limpieza debe ir acompañado de un test unitario que valide el comportamiento esperado. Esto evita regresiones y facilita la colaboración entre miembros del equipo.
