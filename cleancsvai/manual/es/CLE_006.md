# cleancsvai - Optimización Semántica para Pipelines de Datos

## Resumen Técnico

cleancsvai es una herramienta de limpieza de datos CSV impulsada por IA, diseñada para entornos de producción donde la integridad del dataset es crítica. A diferencia de soluciones genéricas, cleancsvai aplica heurísticas avanzadas de detección de anomalías y normalización semántica, eliminando la necesidad de scripts ad-hoc. Su motor de análisis contextual permite identificar y corregir inconsistencias en tiempo real, optimizando el flujo de trabajo de científicos de datos e ingenieros de ML.

## Funcionalidades Diferenciales (Hardcore)

### 1. Normalización Semántica con Reglas Híbridas (RegEx + ML)
cleancsvai no solo limpia celdas vacías. Su motor combina expresiones regulares con un modelo ligero de clasificación (Random Forest optimizado) para detectar patrones de datos corruptos que pasan desapercibidos para validadores tradicionales. Ejemplo real: identifica automáticamente `$1,234.56` como un valor numérico válido y lo convierte a `1234.56`, incluso si el formato cambia entre filas.

### 2. Deduplicación Contextual con Hash de Columnas
El algoritmo de deduplicación no se basa en coincidencias exactas. Utiliza un hash compuesto (SHA-256) sobre un subset dinámico de columnas clave, permitiendo detectar duplicados semánticos (e.g., "John Doe" vs. "J. Doe" en la misma columna de nombre). Además, ofrece un modo de fusión de registros que prioriza el dato más reciente o completo según un timestamp de la fila.

### 3. Pipeline de Validación en Streaming con Feedback Loop
cleancsvai puede integrarse como middleware en pipelines de datos en tiempo real (vía API REST o WebSocket). Cada lote de 1000 filas se somete a una validación de consistencia (tipo de dato, rango, unicidad) y, si se detecta una anomalía, el pipeline se pausa y genera un reporte de error estructurado en JSON. Esto permite un feedback loop inmediato sin perder el contexto de la sesión.

## Guía de Implementación (3 Pasos Extremadamente Detallados)

### Paso 1: Configuración del Pipeline de Limpieza con Perfiles de Dataset
Antes de procesar, define un perfil de limpieza. Ejecuta:
```bash
cleancsvai profile --input raw_data.csv --output profile.json --detect-encoding
```
Esto analiza el dataset y genera un archivo JSON con metadatos: tipos de datos inferidos, distribución de nulos, y patrones de formato. Luego, edita el perfil para definir reglas de limpieza (e.g., `"columna_fecha": {"formato_esperado": "%Y-%m-%d", "accion_si_invalido": "reemplazar_con_NA"}`). Esto evita errores de tipo y asegura consistencia en lotes futuros.

### Paso 2: Ejecución de la Limpieza con Modo de Auditoría
Para evitar la pérdida de datos, ejecuta la limpieza en modo auditoría:
```bash
cleancsvai clean --profile profile.json --input raw_data.csv --output clean_data.csv --audit audit_log.csv
```
El archivo `audit_log.csv` registrará cada cambio: fila, columna, valor original, valor corregido y razón (e.g., "outlier_detected", "format_normalized"). Esto es crítico para cumplir con estándares de auditoría (GDPR, SOC2). Si la tasa de corrección supera el 5%, cleancsvai aborta la operación y te pide revisar el perfil.

### Paso 3: Integración en un Flujo de ETL con Python
Usa la API Python para integrar cleancsvai en tu pipeline existente:
```python
from cleancsvai import Cleaner

cleaner = Cleaner(profile="profile.json")
async def process_chunk(chunk_df):
    result = cleaner.clean_dataframe(chunk_df)
    return result.cleaned_df

# Integración con Pandas y Airflow
df = pd.read_csv("raw_chunk.csv")
df_clean = await process_chunk(df)
df_clean.to_csv("clean_chunk.csv", index=False)
```
Esta integración permite manejar datasets de millones de filas sin cargar todo en memoria, usando un iterador de chunks.

## Mejores Prácticas para Entornos de Producción

### 1. Nunca limpies sobre el archivo original
Siempre trabaja con una copia o usa el modo auditoría. cleancsvai permite crear un snapshot automático del dataset original antes de cualquier modificación (`--backup`). Esto es vital en entornos CI/CD donde un error de limpieza puede romper un modelo entrenado.

### 2. Utiliza perfiles de limpieza versionados
Almacena cada perfil (`profile.json`) en un sistema de control de versiones (Git). Cuando cambies la estructura de tu dataset, actualiza el perfil y etiquétalo con la versión del esquema. Esto asegura trazabilidad y permite reproducir limpiezas históricas para auditorías o debugging de modelos.
