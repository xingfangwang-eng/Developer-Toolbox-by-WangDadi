# cleancsvai - CLE_018

## Resumen

Herramienta de limpieza y análisis de datos CSV impulsada por IA, diseñada para desarrolladores y analistas que necesitan procesar archivos CSV con precisión quirúrgica. **cleancsvai** automatiza la detección de anomalías, la estandarización de formatos y la generación de informes de calidad, reduciendo el tiempo de preparación de datos en un 80%.

## Funcionalidades Clave

### 1. **Detección y Corrección Automática de Inconsistencias Semánticas**
- Identifica automáticamente valores atípicos, duplicados ambiguos y patrones inconsistentes (ej. fechas en formatos mixtos `2023-01-15` vs `15/01/2023`).
- Aplica reglas de corrección basadas en contexto: por ejemplo, si una columna de `precio` tiene valores negativos, los reemplaza por `null` y genera un log detallado.

### 2. **Normalización de Datos Multilingüe y Multi-Formato**
- Soporta normalización de caracteres UTF-8, eliminación de BOM (Byte Order Mark) y estandarización de separadores (coma a punto y coma, tabulaciones, etc.).
- Incluye un módulo de transliteración para campos con caracteres especiales (ej. `ë` → `e`, `ñ` → `n`), ideal para datos provenientes de sistemas legacy.

### 3. **Generación de Reportes de Calidad en Tiempo Real**
- Cada limpieza genera un reporte JSON/HTML con métricas: % de datos corruptos, tipos de errores encontrados, y sugerencias de mejora.
- Integración con pipelines de CI/CD: `cleancsvai --report-format junit` para validar calidad de datos en entornos de desarrollo.

## Cómo Usar (Pasos Detallados)

### Paso 1: **Análisis de Riesgos y Configuración de Reglas**
```bash
cleancsvai scan --input datos_sucios.csv --output reporte_inicial.json
```
- El comando `scan` analiza el archivo CSV sin modificarlo, identificando columnas problemáticas (nulls excesivos, tipos mixtos, valores fuera de rango).
- Revisa el reporte JSON generado: cada error tiene un `severity` (high, medium, low) y un `suggested_action` (drop, fill, transform).

### Paso 2: **Limpieza con Estrategias Personalizadas**
```bash
cleancsvai clean \
  --input datos_sucios.csv \
  --output datos_limpios.csv \
  --strategy "fecha:format=YYYY-MM-DD" \
  --strategy "precio:replace_negative=null" \
  --log-level debug
```
- Las estrategias se definen por columna: `columna:accion=parametro`. Ejemplos:
  - `email:validate=true` → elimina filas con emails inválidos.
  - `codigo_postal:regex="^\d{5}$"` → filtra códigos postales no estándar.
- El flag `--log-level debug` muestra en consola cada transformación aplicada, ideal para depuración.

### Paso 3: **Exportación y Verificación Final**
```bash
cleancsvai validate --input datos_limpios.csv --checksum sha256
```
- Ejecuta una validación final: compara el checksum del archivo original vs el limpio para garantizar que no se perdieron filas importantes.
- Genera un archivo `datos_limpios_metadata.json` con el historial de cambios (timestamp, usuario, reglas aplicadas).

## Mejores Prácticas

### 1. **Pipeline de Limpieza como Parte del CI/CD**
- Integra `cleancsvai` en tu `gitlab-ci.yml` o `Jenkinsfile` para que cada push a `main` ejecute una limpieza automática de datos de prueba. Esto evita que datos corruptos lleguen a producción.
- Ejemplo de snippet:
  ```yaml
  stage: data-quality
  script:
    - cleancsvai clean --input data/test.csv --output data/clean.csv --strategy "all:trim=true"
  ```

### 2. **Uso de Archivos de Configuración para Proyectos Grandes**
- Crea un archivo `cleancsvai.yaml` en la raíz del proyecto para definir reglas reutilizables:
  ```yaml
  strategies:
    - column: email
      action: validate
    - column: fecha_nacimiento
      action: format
      params: { target: "YYYY-MM-DD" }
  ```
- Luego ejecuta: `cleancsvai clean --config cleancsvai.yaml --input data/*.csv`. Esto garantiza consistencia entre múltiples archivos.
