# cleancsvai - CLE_024

## Resumen Técnico

**cleancsvai** es una herramienta de limpieza y normalización de datos CSV potenciada por inteligencia artificial, diseñada para desarrolladores, analistas de datos y equipos de data engineering que trabajan con conjuntos de datos inconsistentes, duplicados o mal formateados. Con soporte multilingüe y detección automática de patrones, cleancsvai reduce el tiempo de preparación de datos hasta en un 80 %, eliminando la necesidad de scripts manuales o reglas complejas. Ideal para pipelines de ETL, migraciones de datos y análisis exploratorios, esta herramienta se integra sin fricción en flujos de trabajo existentes.

## Funcionalidades Clave (Hardcore)

1. **Deduplicación Semántica con Embeddings**  
   Utiliza modelos de embeddings (como Sentence-BERT) para detectar y fusionar filas duplicadas que no son idénticas textualmente pero tienen significado equivalente (ej.: "John Doe" vs. "Doe, John"). Ideal para limpiar bases de datos de contactos o registros históricos.

2. **Normalización Automática de Tipos de Datos y Formatos Regionales**  
   Detecta automáticamente formatos de fecha, moneda, números y códigos postales según la configuración regional (español, alemán, japonés, etc.). Convierte de manera precisa cadenas como "12/05/2023" o "1.234,56 €" a tipos estándar (ISO 8601, float, etc.) sin intervención manual.

3. **Corrección de Errores Ortográficos y Coherencia de Categorías**  
   Emplea un modelo de lenguaje ligero (fine-tuned con datos de comercio electrónico y CRMs) para sugerir correcciones ortográficas y unificar categorías inconsistentes (ej.: "Coche", "Auto", "Vehículo" → "Vehículo"). Reduce la ambigüedad en columnas categóricas.

## Cómo Usar cleancsvai (Flujo Profesional)

### Paso 1: Carga y Previsualización del Dataset con Detección de Anomalías  
   Sube tu archivo CSV directamente desde la interfaz web o mediante la API REST (`POST /upload`). cleancsvai analiza automáticamente las primeras 1000 filas para identificar columnas con valores nulos, tipos mixtos o patrones inconsistentes. Obtendrás un resumen visual de anomalías (heatmap de calidad de datos) antes de aplicar cualquier transformación.

### Paso 2: Configuración de Reglas de Limpieza y Transformación  
   Define reglas personalizadas usando una sintaxis YAML simple o selecciona entre plantillas predefinidas (limpieza de contactos, normalización de direcciones, estandarización de monedas). Puedes activar la deduplicación semántica con un slider de umbral de similitud (0.5–0.95) y elegir si conservar la primera o la última ocurrencia. Todo es configurable desde la API o la GUI.

### Paso 3: Ejecución del Pipeline y Exportación del Dataset Limpio  
   Una vez configurado, ejecuta el pipeline de limpieza. cleancsvai procesa el archivo en lotes (chunks de 5000 filas) para evitar timeouts y muestra un log en tiempo real de cada transformación aplicada. Al finalizar, descarga el CSV limpio, un archivo de log detallado (JSON) y un resumen de métricas (filas eliminadas, corregidas, fusionadas). Compatible con exportación a Google Sheets y bases de datos PostgreSQL.

## Mejores Prácticas Recomendadas

1. **Siempre revisa el log de transformaciones antes de aceptar cambios**  
   cleancsvai genera un archivo de log con cada corrección aplicada (valor original → valor nuevo). Revisa manualmente una muestra aleatoria (~5 % de las filas modificadas) para validar que las reglas no estén sobrecorrigiendo datos válidos. Esto es crítico en entornos de producción donde la calidad del dato es auditada.

2. **Usa la función de “dry run” antes de pipelines críticos**  
   Antes de ejecutar una limpieza masiva en tu base de datos de clientes o inventario, activa el modo de simulación (dry run). cleancsvai aplicará todas las reglas pero no sobrescribirá el archivo original, generando solo un reporte de cambios propuestos. Esto te permite ajustar umbrales de deduplicación o reglas de normalización sin riesgo.
