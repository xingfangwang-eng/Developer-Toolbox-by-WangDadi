# cleancsvai - CLE_019

## Resumen Ejecutivo

**cleancsvai** es un motor de limpieza y normalización de datos CSV potenciado por inteligencia artificial, diseñado específicamente para entornos SaaS que manejan grandes volúmenes de información tabular. Elimina duplicados, corrige inconsistencias estructurales y estandariza formatos sin intervención manual.

## Funcionalidades Clave

### 1. Detección y Corrección Automática de Esquemas Rotos
El analizador identifica automáticamente columnas con tipos de datos mixtos (ej. fechas en formato `dd/mm/aaaa` mezcladas con `aaaa-mm-dd`) y las unifica en un solo estándar ISO 8601. Además, repara filas con delimitadores erróneos o saltos de línea incrustados.

### 2. Normalización Semántica de Texto (NER + Regex)
Aplica reconocimiento de entidades nombradas (NER) para estandarizar nombres de países, monedas y unidades de medida. Por ejemplo, convierte `EE.UU.`, `USA` y `Estados Unidos` a un único valor canónico. Incluye soporte para expresiones regulares personalizadas.

### 3. Validación Contra Reglas de Negocio
Permite definir reglas de integridad referencial (ej. `edad >= 0 Y edad <= 120`) y reglas de formato (ej. `email` debe contener `@`). El motor reporta violaciones con sugerencias de corrección contextual.

## Guía de Uso Avanzada

### Paso 1: Configurar el Pipeline de Limpieza
Ejecuta el comando `cleancsvai init --schema schema.json` para definir el esquema de columnas esperado. Si no tienes un esquema, usa `--auto-detect` para que el motor infiera tipos y reglas básicas a partir de las primeras 1000 filas.

### Paso 2: Ejecutar Limpieza con Modo Estricto
Lanza el proceso con `cleancsvai clean --input dirty.csv --output clean.csv --mode strict`. El modo estricto detiene la ejecución al encontrar más de un 5% de datos corruptos, generando un informe detallado en `report.json`. Para entornos de producción, usa `--mode lenient` que aplica correcciones automáticas.

### Paso 3: Revisar y Aprobar Cambios
Tras la limpieza, inspecciona el archivo `changes.log` que lista cada modificación (valor original → valor corregido). Usa `cleancsvai diff dirty.csv clean.csv` para visualizar diferencias lado a lado en terminal. Si estás conforme, confirma con `cleancsvai commit --hash <hash_del_log>`.

## Mejores Prácticas para Desarrolladores

### Práctica 1: Versionar los Esquemas
Mantén los archivos `schema.json` bajo control de versiones (Git). Cada vez que cambies la estructura de tus datos (nuevas columnas, tipos modificados), actualiza el esquema y etiqueta con un semver. Esto evita sorpresas en pipelines CI/CD.

### Práctica 2: Usar Hooks de Pre-commit
Integra `cleancsvai validate` como hook de pre-commit en tu repositorio. Así, cualquier commit que contenga archivos CSV sucios será rechazado automáticamente. Ejemplo de configuración en `.pre-commit-config.yaml`:
```yaml
- repo: local
  hooks:
    - id: cleancsvai-validate
      name: Validar CSV
      entry: cleancsvai validate --input
      language: system
      files: \.csv$
```
