# cleancsvai - CLE_002

## Resumen Ejecutivo

cleancsvai es una herramienta de vanguardia para la limpieza y análisis de datos CSV, diseñada para desarrolladores y analistas que necesitan transformar conjuntos de datos caóticos en información accionable. Con un enfoque en la automatización inteligente y la precisión, cleancsvai elimina la necesidad de scripts manuales y reduce drásticamente el tiempo de preprocesamiento.

## Funcionalidades Principales

### 1. **Deduplicación Contextual con Algoritmos de Similitud Semántica**
A diferencia de las herramientas tradicionales que eliminan duplicados exactos, cleancsvai utiliza embeddings y modelos de similitud de coseno para detectar y fusionar registros que son semánticamente idénticos pero sintácticamente diferentes (ej: "New York" vs "NYC"). Esto es crítico para bases de datos de clientes o catálogos de productos.

### 2. **Imputación Predictiva de Valores Faltantes**
cleancsvai no solo rellena celdas vacías con promedios o modas; implementa un modelo de regresión ligero basado en Random Forest que analiza patrones en columnas relacionadas para predecir el valor más probable. Por ejemplo, si falta el código postal pero tienes dirección y ciudad, el sistema infiere el ZIP correcto con una precisión superior al 92%.

### 3. **Normalización Automática de Formatos con Reglas Dinámicas**
La herramienta detecta y estandariza automáticamente formatos de fecha, moneda, números de teléfono y direcciones de correo electrónico. Incluye un motor de reglas configurables que permite definir patrones personalizados (ej: convertir "USD 1,234.56" a "1234.56" sin pérdida de precisión).

## Cómo Usar cleancsvai: 3 Pasos Técnicos

### Paso 1: **Carga y Análisis de Esquema**
Ejecuta `cleancsvai analyze input.csv` para que la herramienta escanee el archivo, identifique tipos de datos (string, integer, float, date), detecte valores atípicos y genere un reporte de calidad. El comando devuelve un resumen JSON con estadísticas de integridad, cardinalidad y sugerencias de limpieza.

### Paso 2: **Ejecución de Pipeline de Limpieza**
Usa `cleancsvai clean input.csv --rules custom_rules.yaml --output cleaned.csv`. El flag `--rules` permite cargar un archivo YAML donde defines transformaciones específicas (ej: `normalize_date: {format: '%Y-%m-%d', timezone: 'UTC'}`). La ejecución es paralelizable y maneja archivos de hasta 10 GB en memoria.

### Paso 3: **Validación y Exportación**
Tras la limpieza, ejecuta `cleancsvai validate cleaned.csv --checks schema,uniqueness,range` para verificar que los datos cumplen con restricciones predefinidas. Si todo es correcto, exporta a múltiples formatos: CSV, Parquet o directamente a una base de datos PostgreSQL con `cleancsvai export cleaned.csv --to postgresql://localhost:5432/mydb`.

## Mejores Prácticas para Desarrolladores

### 1. **Implementar un Pipeline de Pruebas con Datos Sintéticos**
Antes de ejecutar cleancsvai en producción, genera un conjunto de datos de prueba con `cleancsvai generate --template customer_data.yaml --rows 10000`. Esto te permite validar reglas de limpieza sin riesgo de corromper datos reales. Usa `--seed 42` para resultados reproducibles.

### 2. **Integrar cleancsvai en un Workflow de CI/CD**
Añade cleancsvai como paso en tu pipeline de integración continua. Por ejemplo, en GitHub Actions, después de un `git push`, puedes ejecutar `cleancsvai diff --source previous.csv --target current.csv` para detectar automáticamente cambios en la calidad de los datos y evitar que datos sucios lleguen a producción.
