## Contenido Técnico Detallado

### ¿Qué es cleancsvai?

cleancsvai es un motor de normalización de datos CSV basado en inteligencia artificial, diseñado para arquitectos de datos y desarrolladores backend que necesitan sanitizar datasets masivos sin intervención manual. Detecta automáticamente inconsistencias de formato, valores nulos espurios y anomalías semánticas en campos críticos como fechas, códigos postales o identificadores únicos.

### Funcionalidades Duras (Hardcore Features)

1. **Inferencia de Esquema Dinámico + Corrección Automática**  
   No solo identifica tipos de datos (string, integer, float, datetime), sino que repara automáticamente columnas con formatos mixtos. Ejemplo: si una columna de fecha contiene `2023-01-01`, `01/01/2023` y `1/1/23`, cleancsvai unifica todo a ISO 8601 en milisegundos.

2. **Deduplicación Semántica con Hash Difuso**  
   Va más allá de duplicados exactos: usa similitud de Levenshtein y n-gramas para detectar filas casi idénticas (ej: `John Doe` vs `Jon Doe`), fusionándolas en una única entrada con metadatos de conflicto.

3. **Validación de Integridad Referencial Offline**  
   Comprueba automáticamente que los valores en columnas foráneas (ej: `user_id`) existan en tablas de referencia internas o externas, y genera un reporte de violaciones con sugerencias de reparación (insertar, ignorar o mapear).

### Cómo Usar cleancsvai en tu Pipeline

#### Paso 1: Carga y Previsualización Asíncrona con Streaming  
```bash
cleancsvai load --file ./datos_sucios.csv --chunk-size 5000 --format-detection strict
```
El motor analiza el archivo en chunks de 5.000 filas, mostrando en tiempo real:  
- **Perfil de datos**: % de nulos, cardinalidad de columnas, valores atípicos.  
- **Alertas de estructura**: columnas con tipos mixtos, filas con número variable de campos.

#### Paso 2: Configuración de Reglas de Depuración con DSL Propietario  
Crea un archivo `cleancsvai.rules` con reglas personalizadas:  
```yaml
rules:
  - column: "email"
    action: normalize_email
    options:
      lowercase: true
      remove_whitespace: true
      validate_domain: true
  - column: "phone"
    action: standardize_phone
    options:
      country: "ES"
      format: "international"
```
Ejecuta:  
```bash
cleancsvai apply --rules cleancsvai.rules --output ./datos_limpios.csv
```

#### Paso 3: Validación Final y Exportación con Checksum  
Tras la limpieza, cleancsvai genera un hash SHA-256 del dataset limpio y un log de transformaciones:  
```bash
cleancsvai verify --input ./datos_limpios.csv --checksum-output ./checksum.txt
```
El log incluye:  
- Número de filas corregidas, eliminadas o fusionadas.  
- Detalle de cada transformación aplicada (timestamp, regla, valor original y nuevo).  
- Reporte de integridad referencial con enlaces a documentación.

### Mejores Prácticas para Arquitectos de Datos

1. **Integración en CI/CD con Validación Pre-commit**  
   Añade cleancsvai como paso obligatorio en tu pipeline de datos. Configura un hook pre-commit que rechace cualquier commit si el dataset contiene más del 5% de nulos o duplicados semánticos. Ejemplo de comando:  
   ```bash
   cleancsvai check --file ./data/*.csv --max-null-ratio 0.05 --max-dup-ratio 0.02
   ```

2. **Estrategia de Backup con Versionado de Datos**  
   Antes de limpiar, cleancsvai puede generar automáticamente un snapshot comprimido (gzip) del dataset original con metadatos de fecha y hash. Almacena estos snapshots en un bucket S3 o Azure Blob con retención de 30 días. Esto permite auditorías posteriores sin pérdida de información.
