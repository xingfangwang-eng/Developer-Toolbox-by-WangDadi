# cleancsvai - CLE_029

## Resumen Ejecutivo

Herramienta open-source que transforma archivos CSV caóticos en datasets listos para producción, combinando limpieza inteligente con análisis semántico automatizado. Ideal para equipos de datos que trabajan con fuentes heterogéneas en español y otros idiomas.

## Funcionalidades Clave

### 1. **Detección y Corrección de Inconsistencias Multilingües**
El motor de IA identifica automáticamente patrones de error comunes en datos en español: fechas en formatos dispares (`12/03/2023` vs `2023-03-12`), separadores decimales mixtos (`,` vs `.`), y acentos mal codificados. Aplica reglas contextuales para normalizar sin perder información.

### 2. **Reconciliación de Esquemas Dinámicos**
Cuando columnas tienen nombres similares pero no idénticos (`"Nombre Cliente"` vs `"Cliente_Nombre"`), cleancsvai infiere relaciones semánticas usando embeddings multilingües y propone fusiones inteligentes, reduciendo el trabajo manual de mapeo en un 80%.

### 3. **Pipeline de Validación con Reglas Personalizadas en YAML**
Define constraints complejos (ej: `correo_electronico debe coincidir con regex EMAIL_ES`, `salario > 0 AND salario < 100000`) en un archivo de configuración. El validador ejecuta checks en paralelo y genera un reporte detallado con índices de filas fallidas y sugerencias de corrección.

## Guía de Uso Avanzada

### 1. **Limpieza Automática con Perfilado de Datos**
Ejecuta el comando básico con detección automática:
```bash
cleancsvai clean --input ventas_mes.csv --output ventas_clean.csv
```
El motor analiza el 20% inicial del archivo para inferir tipos de datos, detectar outliers y sugerir transformaciones (ej: convertir `"sí"`/`"no"` a booleanos). Si detecta más de 15% de valores nulos en una columna, la marca como candidata a eliminación.

### 2. **Reestructuración de Columnas con Reglas de Negocio**
Aplica transformaciones específicas usando una receta JSON:
```json
{
  "transformaciones": [
    {"columna": "fecha_pedido", "tipo": "fecha", "formato": "dd/mm/yyyy"},
    {"columna": "precio", "tipo": "moneda", "locale": "es-MX", "decimales": 2}
  ]
}
```
cleancsvai parsea y normaliza en batch, manejando errores de formato con logs detallados (ej: "Fila 45: `31/02/2023` inválido, se asigna null").

### 3. **Validación y Exportación con Auditoría**
Integra la validación en tu CI/CD:
```bash
cleancsvai validate --input ventas_clean.csv --rules reglas_negocio.yaml --output reporte.html
```
El reporte incluye: número de filas válidas/inválidas, distribución de errores por columna, y un archivo `errores.csv` con las filas rechazadas. Ideal para equipos que necesitan cumplir con estándares de calidad de datos (ISO 8000).

## Mejores Prácticas para Equipos de Datos

### 1. **Versiona tus Reglas de Validación**
Mantén el archivo `reglas_negocio.yaml` bajo control de versiones (Git). Esto permite auditar cambios en los constraints a lo largo del tiempo y replicar pipelines de limpieza en diferentes entornos (dev, staging, prod).

### 2. **Usa la Función de Previsualización con Muestras**
Antes de procesar datasets grandes (>100k filas), ejecuta:
```bash
cleancsvai preview --input ventas_mes.csv --sample 1000
```
Esto te mostrará un resumen estadístico (distribución de valores nulos, tipos inferidos, outliers) y te permitirá ajustar las reglas antes de la limpieza completa, ahorrando recursos computacionales.
