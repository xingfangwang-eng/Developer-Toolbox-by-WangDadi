# cleancsvai - CLE_025

**Resumen:**  
cleancsvai es una herramienta SaaS de vanguardia que automatiza la limpieza, normalización y análisis de archivos CSV. Diseñada para desarrolladores y analistas de datos, elimina errores estructurales, duplicados y valores atípicos mediante algoritmos de machine learning, todo desde tu terminal o API. Sin dependencias pesadas, sin configuraciones complejas.

## ¿Qué es cleancsvai?

cleancsvai resuelve el caos de datos sucios: celdas vacías, formatos inconsistentes, columnas mal etiquetadas y ruido semántico. A diferencia de herramientas genéricas, cleancsvai entiende el contexto de tus datos gracias a un motor de inferencia de tipos y reglas de validación inteligentes. Ideal para pipelines de datos, migraciones y preparación de datasets para modelos de IA.

## Características Clave (Hardcore)

### 1. **Limpieza Predictiva con Auto-Detección de Esquemas**
cleancsvai escanea tu CSV y **infiere automáticamente** el tipo de dato de cada columna: fecha, moneda, porcentaje, dirección IP, etc. Luego aplica transformaciones contextuales (ej. unificar formatos de fecha `DD/MM/YYYY` a `YYYY-MM-DD`) sin que escribas una sola regla. Soporta más de 50 patrones de datos comunes.

### 2. **Corrección Semántica Basada en Reglas ML**
No solo elimina duplicados exactos: detecta **casi-duplicados** (ej. "New York" vs "NYC", "USA" vs "United States") y los unifica usando embeddings de palabras y similitud coseno. Además, rellena celdas vacías con valores predichos por un modelo de regresión ligero (XGBoost) entrenado sobre el propio dataset.

### 3. **Modo Pipeline Declarativo con Validación en Tiempo Real**
Define un pipeline de limpieza en un archivo YAML: `clean: { drop_na: true, fill_strategy: median, detect_outliers: zscore }`. cleancsvai ejecuta cada paso en orden, mostrando estadísticas de impacto en vivo (cuántas filas se modificaron, qué columnas se afectaron). Ideal para integración CI/CD o scripts de automatización.

## Cómo Usar (Pasos Detallados)

### 1. **Preparación del Entorno y Carga del Dataset**
Instala cleancsvai vía pip: `pip install cleancsvai`. Luego, carga tu archivo CSV con el comando `cleancsvai load --file datos_sucios.csv --infer-schema`. El motor analizará las primeras 1000 filas para detectar tipos y anomalías estructurales. Recibirás un resumen en terminal: `[INFO] Schema detected: 12 columns, 3 date columns, 1 monetary column (€), 2 columns with 15% nulls`.

### 2. **Ejecución de Limpieza Automática con Personalización**
Ejecuta `cleancsvai clean --auto --output limpio.csv`. Por defecto, aplica: eliminación de filas completamente vacías, corrección de mayúsculas/minúsculas en texto, unificación de formatos numéricos (ej. convertir `1,234.56` a `1234.56`), y relleno de valores nulos con la mediana de la columna (si es numérica) o con el valor más frecuente (si es categórica). Para ajustar, usa `--config mi_pipeline.yaml`.

### 3. **Validación y Exportación con Reporte de Calidad**
Tras la limpieza, ejecuta `cleancsvai validate --report calidad.html`. Esto genera un reporte interactivo con: distribución de tipos de datos, gráficos de valores atípicos, matriz de correlación de columnas, y un score de calidad (0-100). Si el score es < 80, cleancsvai sugiere pasos adicionales (ej. `--normalize-currency`). Exporta el dataset final con `cleancsvai export --format parquet --output datos_limpios.parquet`.

## Mejores Prácticas para Desarrolladores

### 1. **Integración Continua en Pipelines de Datos**
Usa cleancsvai como paso previo a cualquier análisis o entrenamiento de modelos. Por ejemplo, en un script Python:  
```python
from cleancsvai import Cleaner
cleaner = Cleaner()
df_limpio = cleaner.clean('datos_sucios.csv', strategy='auto', fill_na='median')
```
Esto asegura que tus datasets siempre lleguen limpios a tu ETL o base de datos. Además, añade el flag `--fail-if-low-score 80` en tu CI para que el pipeline falle si la calidad no es aceptable.

### 2. **Personalización de Reglas para Datos Específicos del Dominio**
No te limites a la limpieza genérica. Crea reglas personalizadas en un archivo JSON:  
```json
{
  "rules": [
    {"column": "precio", "type": "currency", "format": "EUR", "remove_symbol": true},
    {"column": "email", "type": "email", "validate_domain": true, "drop_invalid": true}
  ]
}
```
Y ejecuta `cleancsvai clean --rules mis_reglas.json`. Esto es crítico para datos financieros, sanitarios o de ecommerce, donde la semántica importa más que la estructura.
