# cleancsvai - CLE_015

## Resumen Ejecutivo

Herramienta de limpieza y análisis de datos CSV con inteligencia artificial integrada, diseñada para desarrolladores que necesitan transformar datasets caóticos en estructuras limpias y listas para producción. Elimina duplicados, corrige formatos inconsistentes y normaliza columnas en segundos.

## Funcionalidades Principales

### 1. Detección y Corrección de Anomalías Estructurales
El motor analiza automáticamente la integridad del archivo CSV: identifica filas con número incorrecto de columnas, valores nulos en campos críticos y patrones de datos atípicos (fechas en formato ISO vs. DD/MM/YYYY). Aplica reglas de saneamiento configurables sin intervención manual.

### 2. Normalización Semántica de Columnas
Reconoce automáticamente tipos de datos (fechas, monedas, códigos postales, direcciones IP) y estandariza formatos según el locale español. Por ejemplo, convierte "1,234.56" a "1.234,56" y "2023-05-15" a "15/05/2023". Soporta mapeo de sinónimos para columnas con nombres inconsistentes (ej: "teléfono", "phone", "contacto").

### 3. Deduplicación Contextual con Reglas Fuzzy
Compara registros usando algoritmos de similitud (Levenshtein, Jaccard) para detectar duplicados no exactos, como "María García López" vs. "Maria Garcia Lopez". Permite definir umbrales de precisión y fusionar campos conflictivos mediante reglas de prioridad personalizadas.

## Guía de Uso Avanzado

### Paso 1: Configurar Pipeline de Limpieza con Reglas Personalizadas
```bash
cleancsvai --input datos_sucios.csv --output datos_limpios.csv \
  --rules normalize_dates:dd/MM/yyyy \
  --rules deduplicate:threshold=0.85,merge_strategy=keep_first \
  --rules fill_missing:column=email,strategy=drop_row
```
Define reglas en cadena: normaliza fechas, elimina duplicados con 85% de similitud y descarta filas sin email. El motor ejecuta las transformaciones en orden secuencial, optimizando memoria para archivos de hasta 500 MB.

### Paso 2: Validar Estructura con Reporte de Calidad
```bash
cleancsvai --validate datos_sucios.csv --report calidad.html
```
Genera un informe HTML interactivo con métricas: porcentaje de datos válidos, distribución de tipos por columna, histograma de valores nulos y gráfico de correlación entre campos. Útil para auditorías rápidas antes de cargar datos a bases de datos.

### Paso 3: Integración con ETL en Tiempo Real
```python
from cleancsvai import StreamCleaner
cleaner = StreamCleaner(rules=["normalize_dates:dd/MM/yyyy", "deduplicate:threshold=0.9"])
for chunk in cleaner.stream("datos_streaming.csv", chunk_size=10000):
    chunk.to_sql("tabla_limpia", con=engine, if_exists="append")
```
Procesa archivos grandes en chunks de 10,000 filas, aplicando limpieza incremental. Ideal para pipelines de datos que requieren baja latencia.

## Mejores Prácticas para el Ecosistema Español

### 1. Manejo de Caracteres Especiales y Codificaciones
Los archivos CSV españoles suelen incluir ñ, tildes y símbolos como €. Configura la codificación explícitamente: `--encoding utf-8-sig` para preservar caracteres. Si trabajas con datos legacy de sistemas Windows, usa `--encoding latin-1` y convierte a UTF-8 con `--normalize_unicode`.

### 2. Estandarización de Direcciones Postales
Para datasets con direcciones españolas, aplica reglas de normalización geográfica: `--rules normalize_addresses:country=ES`. El motor reconoce formatos como "C/ Gran Vía, 12, 28013 Madrid" y los descompone en columnas separadas (calle, número, código postal, ciudad). Combínalo con `--geocode` para añadir coordenadas GPS usando datos abiertos del CNIG.
