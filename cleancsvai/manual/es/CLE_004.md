# cleancsvai - CLE_004

## Resumen Ejecutivo

cleancsvai es un motor de limpieza y normalización de datos CSV impulsado por IA, diseñado para eliminar inconsistencias estructurales, duplicados semánticos y errores de formato en pipelines de datos masivos. Este proyecto está optimizado para SEO con las palabras clave: `CSV file cleaning`, `Data cleaning tool`, `CSV data analysis`.

---

## Contenido Técnico Detallado

### ¿Qué es cleancsvai?

cleancsvai no es un simple limpiador de CSV. Es un **sistema de orquestación de calidad de datos** que combina heurísticas de validación determinista con modelos de lenguaje natural (LLM) para detectar anomalías contextuales (por ejemplo, direcciones mal escritas, fechas en formatos mixtos, valores nulos con significado semántico). Funciona como middleware entre tu fuente de datos (ETL, scraping, APIs legacy) y tu destino analítico (data warehouse, dashboard, modelo ML).

---

### Características Principales (Hardcore)

1. **Deduplicación Fuzzy Basada en Embeddings**  
   A diferencia de los algoritmos de coincidencia exacta (hash), cleancsvai genera *embeddings vectoriales* para cada fila y aplica similitud coseno. Detecta duplicados aunque tengan variaciones tipográficas, campos intercambiados o datos incompletos. Ejemplo: `"John Doe, 123 Main St"` y `"Jon Doe, 123 Main Street"` se fusionan automáticamente.

2. **Normalización de Esquemas Dinámicos**  
   El sistema infiere el tipo de dato real de cada columna (fecha, moneda, porcentaje, dirección IP) y lo normaliza a un formato estándar ISO/UNICODE. Si una columna contiene `"€1.234,56"` y otra `"$1,234.56"`, cleancsvai unifica a `1234.56` con metadatos de moneda. No necesita configuración previa.

3. **Pipeline de Validación con Reglas Regenerativas**  
   Puedes definir reglas de validación en YAML (por ejemplo, `email: regex, required, unique`) y cleancsvai genera automáticamente un *reporte de calidad* con estadísticas de fallos, sugerencias de reparación y un script de corrección. Las reglas se autoajustan tras cada ejecución para reducir falsos positivos.

### Cómo Usar (Pasos Detallados)

**Paso 1: Carga y Previsualización con Análisis de Riesgos**  
Ejecuta `cleancsvai load --input data.csv --preview`. El sistema analiza las primeras 100 filas, identifica columnas con alta tasa de nulos (>30%), formatos inconsistentes y posibles outliers. Devuelve un *heatmap de riesgo* en consola y un archivo `risk_report.json`. Ejemplo de salida:  
```
[WARN] Columna 'phone': 45% de valores no numéricos detectados. Se recomienda normalización.
```

**Paso 2: Ejecución de Pipeline de Limpieza con Políticas**  
Usa `cleancsvai clean --policy strict --output cleaned.csv`. El parámetro `--policy` acepta tres niveles: `lax` (solo corrige errores obvios), `strict` (aplica todas las reglas y rechaza filas irrecuperables), `adaptive` (aprende de errores previos). Durante la ejecución, el motor genera un log en tiempo real con cada transformación aplicada:  
```
[INFO] Fila 234: Normalizando fecha '2024/03/15' -> '2024-03-15'
[INFO] Fila 567: Eliminando duplicado (score: 0.98)
```

**Paso 3: Post-procesamiento y Exportación con Metadatos**  
Finaliza con `cleancsvai export --format parquet --metadata`. cleancsvai exporta el dataset limpio en formato Apache Parquet (compresión columnar) y genera un archivo `metadata.json` con:  
- Número de filas originales vs. limpias  
- Columnas eliminadas o fusionadas  
- Historial de transformaciones aplicadas  
- Distribución de tipos de error (duplicados, nulos, formato)  

Esto permite auditoría completa y trazabilidad.

### Mejores Prácticas (Industry Best Practices)

1. **Integración en Pipelines CI/CD con Validación Preventiva**  
   No esperes a que los datos lleguen al data warehouse. Agrega `cleancsvai check --ci` como paso en tu pipeline de integración continua (GitHub Actions, Jenkins). Esto previene que datos sucios entren a producción. Configura un umbral de calidad (por ejemplo, <5% de errores) para fallar el build.

2. **Uso de Reglas Personalizadas para Datos Específicos del Dominio**  
   Para sectores como fintech o salud, crea un archivo `rules.yaml` con validaciones específicas (IBAN, NIF, código postal). cleancsvai permite cargar reglas desde un repositorio Git, facilitando la colaboración entre equipos de datos y negocio. Ejemplo:
   ```yaml
   - column: "iban"
     type: "iban"
     required: true
     unique: true
     format: "ES\d{22}"
   ```
