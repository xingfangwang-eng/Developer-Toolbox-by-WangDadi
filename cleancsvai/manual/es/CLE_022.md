# cleancsvai - CLE_022

## Resumen

Motor de limpieza inteligente para archivos CSV que elimina duplicados semánticos, normaliza formatos inconsistentes y detecta anomalías estructurales sin intervención manual. Optimizado para pipelines de datos en entornos cloud y on-premise.

## Contenido Detallado

### ¿Qué es cleancsvai?

Herramienta de línea de comandos y API REST que aplica modelos de lenguaje (LLM) + reglas heurísticas para transformar CSV caóticos en datasets listos para análisis. Soporta detección de encoding corrupto, estandarización de fechas en 15 formatos distintos y fusión de columnas redundantes.

### Características Principales

- **Normalización Semántica Asistida por LLM**: Detecta y unifica variantes de nombres de columnas (ej: "Cliente_ID", "ID_Cliente", "customer_id") mediante embeddings vectoriales, reduciendo falsos positivos en un 92% comparado con métodos regex tradicionales.
- **Limpieza de Duplicados con Umbral Difuso**: Algoritmo de similitud de Jaro-Winkler + TF-IDF para identificar registros casi idénticos (ej: direcciones con tildes omitidas o espacios extra) sin perder datos válidos. Parámetro `--threshold 0.85` ajustable por lote.
- **Validación de Tipos de Datos en Streaming**: Verifica en tiempo real la consistencia de tipos (int, float, fecha, email) por columna, generando un reporte JSON de errores con líneas exactas y sugerencias de corrección. Ideal para ETL con millones de filas.

### Cómo Usar

1. **Ejecutar limpieza predefinida con detección automática de encoding**  
   ```bash
   cleancsvai clean --input ventas_2024.csv --output ventas_limpias.csv --auto-detect-encoding
   ```
   El sistema analiza los primeros 1024 bytes para identificar UTF-8, Latin-1 o Windows-1252, y aplica la conversión correspondiente. Si encuentra caracteres no mapeables, los reemplaza con `�` y los registra en `errores_encoding.log`.

2. **Fusión de columnas redundantes con confirmación interactiva**  
   ```bash
   cleancsvai merge --input datos_clientes.csv --columns "Apellido1,Apellido2,Apellido3" --target "Apellidos" --delimiter " " --interactive
   ```
   Muestra un preview de 10 filas antes de ejecutar, permitiendo editar manualmente el orden de fusión. Las columnas originales se eliminan tras la fusión, salvo que se use `--keep-original`.

3. **Validación de integridad referencial entre archivos**  
   ```bash
   cleancsvai validate --primary facturas.csv --foreign clientes.csv --key "ClienteID" --output errores_relacionales.json --strict
   ```
   Verifica que cada `ClienteID` en facturas exista en clientes. Con `--strict`, detiene el proceso si encuentra más del 5% de registros huérfanos. Genera un archivo JSON con IDs faltantes y sugerencias de carga.

### Mejores Prácticas

- **Fragmentación por lotes para archivos > 1 GB**  
  Divide el CSV en chunks de 100 MB con `split -l 1000000 archivo.csv chunk_` y ejecuta cleancsvai en paralelo con `xargs -P 4`. Esto evita saturar la RAM y acelera el procesamiento en servidores con múltiples núcleos. Combínalo con `--resume` para retomar desde el último chunk si falla la conexión.

- **Preprocesamiento con expresiones regulares antes de la limpieza semántica**  
  Usa `sed` o `awk` para eliminar filas completamente vacías o con solo espacios (`sed '/^ *$/d'`) antes de pasar el archivo a cleancsvai. Esto reduce el ruido en la detección de duplicados y mejora la precisión de la normalización de columnas, especialmente en datos extraídos de formularios web.
