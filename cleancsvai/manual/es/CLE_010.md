# cleancsvai - CLE_010

## Resumen Técnico

Herramienta SaaS especializada en limpieza semántica de archivos CSV mediante modelos de lenguaje avanzados. Diseñada para ingenieros de datos que necesitan transformar datasets caóticos en estructuras limpias sin escribir pipelines manuales. Soporta detección de anomalías, normalización de tipado y corrección contextual de valores corruptos.

## Funcionalidades Clave

### 1. Corrección Semántica Basada en Contexto
No solo elimina filas duplicadas o celdas vacías. Analiza el significado de cada campo utilizando embeddings contextuales para detectar y reparar errores lógicos: fechas imposibles (ej: "32/13/2023"), valores fuera de rango en columnas numéricas, o inconsistencias en nombres de ciudades/países. Ideal para migraciones de datos legacy.

### 2. Normalización Automática de Esquemas Dinámicos
Detecta automáticamente el tipo de dato real de cada columna (int, float, datetime, string) y propone un esquema normalizado con reglas de validación. Si una columna contiene "1000,50" y "1.000,50", cleancsvai unifica el formato decimal según la configuración regional (es_ES, en_US, etc.).

### 3. Pipeline de Limpieza Versionable con Logs de Auditoría
Cada transformación queda registrada en un archivo JSON con metadatos: filas afectadas, reglas aplicadas, cambios realizados. Puedes revertir cualquier paso o exportar el pipeline como script reutilizable para integraciones CI/CD.

## Pasos de Uso Avanzados

### 1. Configuración de Reglas de Validación Contextual
```bash
cleancsvai validate --input dirty.csv --rules "fecha_nacimiento:date_format('%Y-%m-%d'), salario:range(0,1000000)" --locale es_ES
```
El motor analiza cada celda y aplica correcciones automáticas: si encuentra "01/02/2023" en un campo configurado como `date_format('%Y-%m-%d')`, lo reformatea a "2023-02-01". Los errores no corregibles se exportan a un archivo `validation_errors.csv` para revisión manual.

### 2. Corrección de Inconsistencias Geográficas
```bash
cleancsvai geo-fix --input clientes.csv --columns "ciudad,pais" --geodb "geonames_es.db"
```
Utiliza una base de datos geoespacial local para estandarizar nombres de lugares: "Barcelona" → "Barcelona (España)", "NYC" → "New York (Estados Unidos)". Incluye detección de errores tipográficos fonéticos: "Madrí" → "Madrid".

### 3. Exportación con Trazabilidad Completa
```bash
cleancsvai export --input cleaned.csv --format postgresql --schema public --audit-log audit_20231027.json
```
Genera un script SQL con todas las transformaciones documentadas. Cada fila insertada incluye un campo `_cleancsvai_audit_id` que referencia el log de auditoría. Ideal para entornos donde se requiere compliance con GDPR o SOX.

## Mejores Prácticas para Equipos de Datos

### 1. Integración en Pipelines ETL con Control de Versiones
No ejecutes limpiezas ad-hoc. Guarda la configuración de reglas en un repositorio Git (ej: `cleancsvai_rules/v1.2.yaml`). Así cada miembro del equipo puede reproducir exactamente la misma transformación, y los cambios quedan auditados. Ejemplo de archivo YAML:
```yaml
rules:
  - column: "email"
    validation: "regex('^[\\w\\.-]+@[\\w\\.-]+\\.\\w{2,}$')"
    action: "drop_row"
  - column: "telefono"
    normalization: "phone_es"
```

### 2. Limpieza Progresiva con Validación Humana
Para datasets críticos (facturación, registros médicos), configura un pipeline de dos fases: primero cleancsvai aplica correcciones automáticas con bajo umbral de confianza (solo cambios con >95% de certeza), luego exporta un archivo de "revisiones pendientes" para que un analista valide manualmente los casos dudosos. Esto reduce el riesgo de errores catastróficos sin sacrificar velocidad.
