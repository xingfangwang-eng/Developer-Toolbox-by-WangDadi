# cleancsvai - CLE_016

## Resumen

Herramienta de limpieza de datos CSV con inteligencia artificial, diseñada para desarrolladores que necesitan procesar datasets complejos sin perder precisión. Elimina duplicados, corrige inconsistencias y normaliza formatos en segundos.

## Contenido Detallado

### ¿Qué es cleancsvai?

cleancsvai es un motor de limpieza de datos que combina heurísticas de NLP con algoritmos de detección de anomalías. No solo elimina ruido en campos CSV, sino que también identifica patrones ocultos y sugiere transformaciones automáticas. Ideal para pipelines de ETL, análisis de logs y preparación de datos para machine learning.

### Características Principales

- **Deduplicación Semántica Avanzada**: Detecta duplicados no exactos basados en similitud de texto, incluso si los valores están escritos de forma diferente (ej: "John Doe" vs "J. Doe"). Utiliza embeddings de palabras y distancia de Levenshtein ponderada.
- **Normalización Automática de Formatos**: Convierte fechas, monedas, números de teléfono y direcciones a un estándar unificado. Soporta más de 50 formatos regionales y detecta automáticamente el esquema del dataset.
- **Detección de Outliers con IA**: Identifica valores atípicos en columnas numéricas mediante clustering espectral y reglas de negocio configurables. Genera un reporte JSON con explicaciones de cada anomalía.

### Cómo Usar

1. **Carga y Previsualización del Dataset**  
   Ejecuta `cleancsvai --input datos_sucios.csv --preview 20` para cargar el archivo y mostrar las primeras 20 filas. La herramienta analizará automáticamente el tipo de datos de cada columna (numérica, categórica, texto libre) y mostrará un resumen de calidad: porcentaje de valores nulos, duplicados y formatos inconsistentes.

2. **Configuración de Reglas de Limpieza**  
   Crea un archivo YAML `reglas_limpieza.yaml` con reglas personalizadas, por ejemplo:  
   ```yaml
   columnas:
     fecha: { tipo: fecha, formato: "%Y-%m-%d" }
     precio: { tipo: moneda, simbolo: "$", decimales: 2 }
     telefono: { tipo: telefono, pais: "ES" }
   reglas:
     - accion: eliminar_duplicados
       criterio: semantico
       umbral: 0.85
   ```  
   Luego ejecuta `cleancsvai --config reglas_limpieza.yaml --input datos_sucios.csv --output datos_limpios.csv`.

3. **Validación y Exportación de Resultados**  
   Usa la flag `--validate` para verificar la integridad del dataset limpio: `cleancsvai --input datos_limpios.csv --validate`. Esto generará un informe HTML con estadísticas de transformación, histogramas de distribución y una tabla de errores residuales. Para exportar en formato Parquet o Feather, añade `--format parquet`.

### Mejores Prácticas

- **Siempre ejecuta `--preview` antes de aplicar transformaciones masivas**. Esto te permite detectar columnas con tipos mixtos (ej: números almacenados como strings) y ajustar las reglas de normalización antes de procesar todo el dataset. Ahorra tiempo y evita corromper datos críticos.
- **Usa la deduplicación semántica solo cuando el dataset tenga nombres, direcciones o descripciones largas**. Para datos puramente numéricos o IDs únicos, la deduplicación exacta es más rápida y suficiente. Configura el umbral de similitud entre 0.7 y 0.9 para evitar falsos positivos.
