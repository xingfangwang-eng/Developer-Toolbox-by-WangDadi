## Contenido Técnico

### ¿Qué es cleancsvai?

cleancsvai es una herramienta profesional de **limpieza y normalización de datos CSV** diseñada para resolver problemas comunes en campos como la ciencia de datos, la automatización de procesos y la integración de sistemas. A diferencia de soluciones genéricas, cleancsvai aplica heurísticas avanzadas y reglas configurables para detectar y corregir inconsistencias, valores nulos, formatos erróneos y duplicados, todo desde tu terminal o pipeline CI/CD.

### Características Principales (Hardcore)

1. **Detección y corrección de tipos de datos con inferencia semántica**  
   cleancsvai analiza cada columna y deduce automáticamente el tipo de dato esperado (fecha, número, booleano, string), incluso si el CSV contiene mezclas como `"2023-01-01"`, `"01/01/2023"` o `"1/1/23"`. Aplica reglas de normalización regional (por ejemplo, formato ISO 8601) y corrige valores fuera de rango sin intervención manual.

2. **Eliminación inteligente de duplicados con umbral de similitud configurable**  
   No solo elimina filas idénticas al 100%. cleancsvai permite definir un **umbral de similitud** (ej. 0.95) usando distancia de Levenshtein o Jaccard, ideal para datasets con errores tipográficos o variaciones menores (ej. `"Madrid"` vs `"Madird"`). Incluye opción de *fuzzy matching* para columnas clave.

3. **Pipeline de transformación en cadena con validación en tiempo real**  
   Puedes encadenar múltiples operaciones (limpieza, normalización, deduplicación, filtrado) en un solo comando. Cada paso se valida contra un esquema JSON opcional, y se genera un log detallado de todas las transformaciones aplicadas (útil para auditoría y reproducibilidad).

### Cómo Usar (Pasos Detallados)

1. **Configuración del pipeline de limpieza**  
   Crea un archivo de configuración YAML (`cleancsvai.yml`) donde definas reglas por columna. Por ejemplo, para un dataset de ventas:
   ```yaml
   columns:
     fecha: { type: date, format: "%Y-%m-%d", locale: es_ES }
     precio: { type: float, min: 0, max: 10000 }
     cliente: { type: string, dedup: { threshold: 0.9 } }
   ```
   Luego ejecuta: `cleancsvai --config cleancsvai.yml input.csv -o output_clean.csv`

2. **Ejecución con validación y logging**  
   Activa el modo verbose para ver cada transformación en tiempo real:
   ```bash
   cleancsvai --verbose --log-level debug input.csv --output cleaned.csv
   ```
   El output incluye estadísticas: `95% de filas válidas`, `12 duplicados eliminados`, `3 valores nulos corregidos`.

3. **Integración en pipeline CI/CD**  
   Añade cleancsvai como paso en tu GitHub Actions o GitLab CI. Ejemplo de snippet:
   ```yaml
   - name: Limpiar datos
     run: cleancsvai --strict --schema schema.json data.csv -o data_clean.csv
   ```
   Si falla la validación, el pipeline se detiene, garantizando que solo datos limpios lleguen a producción.

### Mejores Prácticas para el Uso en Producción

- **Siempre define un esquema de validación**  
  No confíes en la inferencia automática al 100%. Crea un archivo `schema.json` que especifique tipos, rangos y valores permitidos. Esto evita sorpresas cuando los datos de entrada cambian de formato o fuente.

- **Usa el modo `--dry-run` antes de aplicar cambios destructivos**  
  cleancsvai permite ejecutar en modo simulación: `cleancsvai --dry-run input.csv`. Verás qué filas se eliminarían o modificarían sin tocar el archivo original. Ideal para revisar antes de sobrescribir datos críticos.
