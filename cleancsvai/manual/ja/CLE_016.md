# cleancsvai - CLE_016

## 概要

大規模データセットの前処理で発生する「欠損値の埋め合わせ」「型の不一致」「エンコードの破損」といった問題を、機械学習ベースの推論エンジンで自動解決するツール。従来の正規表現ベースのルール定義を排除し、データ構造を動的に解析・修復する。

## コア機能

### 1. スキーマ自動復元と型推論
CSVヘッダーが欠落している、または型が混在するカラムを検出し、統計的特徴量（エントロピー、分布の歪度、欠損率）から最適なデータ型（datetime, float, categorical）を自動割り当て。特にタイムスタンプのパースに強く、ISO 8601、UNIXタイム、和暦表記を自動識別する。

### 2. ファジー重複排除
レーベンシュタイン距離とJaccard係数を組み合わせたハイブリッド類似度計算により、表記ゆれ（全角/半角、大文字/小文字、スペースの有無）を吸収した重複行のマージを実行。閾値はデータセットのエントロピーから動的に決定される。

### 3. 異常値の文脈補完
外れ値を単純削除せず、周辺行の時系列パターン（移動平均、指数平滑化）または同一カテゴリ内の最頻値で補完。数値カラムの分布が歪んでいる場合、Box-Cox変換後に統計モデルを適用する。

## 詳細な操作手順

### 手順1：データプロファイリングの実行
ターミナルで以下を実行：
```bash
cleancsvai profile --input sales_data.csv --output report.json
```
生成された `report.json` には、各カラムのデータ型、欠損率、ユニーク値数、異常値スコア（Zスコア + IQR）がJSON Lines形式で出力される。このレポートを基に、後続のクリーニング戦略を決定する。

### 手順2：ルールベース + MLハイブリッドクリーニング
プロファイリング結果を `--config` フラグで読み込み、自動修復を実行：
```bash
cleancsvai clean --input sales_data.csv --config report.json --output cleaned_data.csv
```
内部的には以下の処理が逐次適用される：
- 欠損値補完（MICEアルゴリズム）
- 型変換（pandasの`infer_objects()`を拡張）
- 重複排除（Jaccard係数 > 0.85）
- 異常値の置換（ローカル外れ因子法）

### 手順3：差分検証とバックアップ生成
クリーニング前後でのデータ分布の変化を可視化：
```bash
cleancsvai diff --original sales_data.csv --cleaned cleaned_data.csv --output diff_report.html
```
生成されたHTMLレポートには、ヒストグラムの重ね書きと、変更された行のサマリー（変更理由、元の値、新しい値）が表形式で表示される。これにより、過剰なデータ改変を防止する。

## 業界ベストプラクティス

### 1. パイプラインの段階的適用
一度に全カラムをクリーニングせず、`--columns` オプションでカラム単位で処理を分割する。例えば、日付カラムのみ先に処理し、その後数値カラムを処理することで、型変換の連鎖エラーを防ぐ。

### 2. バージョン管理との統合
`cleancsvai clean` 実行時に `--git-hash` フラグを付与し、出力CSVのファイル名に現在のGitコミットハッシュを含める。これにより、どのコードバージョンで生成されたデータかが追跡可能になり、再現性が担保される。
