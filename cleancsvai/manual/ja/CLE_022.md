# cleancsvai - CLE_022

## 概要

大規模なCSVデータセットに潜む重複、欠損値、型不一致を機械学習ベースで自動検出・修正するSaaSツール。クリーニングルールの自動提案とワンクリック適用により、データ前処理の工数を80%削減する。

## コア機能

### 1. スキーマ推論と型強制エンジン
CSVの各カラムに対して、値の分布とパターンを分析し、最適なデータ型（日付、数値、ブール、カテゴリ）を自動判定。日付フォーマットの統一（`2024/12/01` → `2024-12-01`）や、数値カラムに混入した文字列の自動除去を実行する。

### 2. ファジー重複除去 with レーベンシュタイン距離
完全一致だけでなく、表記ゆれ（`株式会社` vs `(株)`、全角/半角混在）やスペルミスを含む行を、カスタマイズ可能な類似度閾値（0.8〜1.0）でマージ。大量データでもO(n log n)のインデックス最適化により高速処理を実現。

### 3. 欠損値補完パイプライン
カラムの統計的特性に基づき、平均値/中央値/最頻値での補完、前後行からの線形補間、または機械学習モデル（ランダムフォレスト）による予測補完を自動選択。補完履歴はメタデータとして出力されるため、監査証跡として利用可能。

## 使い方

### ステップ1: CSVをアップロードし、スキーマプロファイルを取得
```bash
curl -X POST https://api.cleancsvai.com/v1/analyze \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@sales_data_2024Q4.csv"
```
レスポンスJSONには、各カラムの検出型、欠損率（%）、重複行数、異常値候補のリストが含まれる。このプロファイルを基に、クリーニング戦略を選択する。

### ステップ2: ルールセットを定義し、プレビューを実行
```python
import cleancsvai

client = cleancsvai.Client(api_key="YOUR_API_KEY")
rules = cleancsvai.RuleSet([
    cleancsvai.Rule("date_column", action="normalize_date", format="%Y-%m-%d"),
    cleancsvai.Rule("price", action="remove_outliers", z_score=3.0),
    cleancsvai.Rule("customer_name", action="deduplicate", threshold=0.85)
])

preview = client.preview("sales_data_2024Q4.csv", rules)
# 変更前後の差分をDataFrameで取得可能
preview.changes.head(10)
```

### ステップ3: クリーニングを実行し、結果をエクスポート
```bash
curl -X POST https://api.cleancsvai.com/v1/clean \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "abc123",
    "rules": [
      {"column": "date", "action": "normalize_date", "format": "%Y-%m-%d"},
      {"column": "price", "action": "remove_outliers", "z_score": 3.0}
    ],
    "output_format": "parquet"
  }'
```
処理後は、クリーンなCSV/Parquet/JSONLのいずれかでダウンロード可能。同時に、クリーニングログ（変更行数、適用ルール、補完値の根拠）を含むJSONレポートが自動生成される。

## ベストプラクティス

### 1. クリーニング前にカラムのドメイン制約を明示する
数値カラムに期待される値域（例：`price >= 0`）や、必須カラムのリストを事前にルールセットに追加することで、異常値をゼロに近づける。特に金融データや在庫管理データでは、この事前制約が後工程でのエラーを防ぐ。

### 2. 大規模データセットでは分割処理＋並列実行を活用する
100万行を超えるCSVは、`page_size`パラメータで10万行単位に分割し、並列APIリクエストを投げることで、メモリ使用量を抑えつつ処理時間を線形に短縮できる。各バッチの結果は自動結合されるため、クライアント側でのマージ処理は不要。
