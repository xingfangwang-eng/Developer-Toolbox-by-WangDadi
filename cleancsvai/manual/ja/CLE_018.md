# cleancsvai - CLE_018

## 概要

CSVファイルの前処理に悩むエンジニアへ。cleancsvaiは、機械学習ベースの異常検知と自動スキーマ推論を統合した、次世代型データクレンジングツールです。重複排除、型変換ミスの修正、欠損値補完をワンコマンドで実行。特に大規模データセット（10万行以上）でのパフォーマンス最適化に強みを持ちます。

## 詳細な内容

### とは cleancsvai?

cleancsvaiは、CSVデータの品質管理を自動化するコマンドラインツールです。従来の正規表現ベースのスクリプトとは異なり、統計的異常検知アルゴリズム（Isolation Forest + Z-score ハイブリッド）を搭載。日付フォーマットの統一、通貨記号の除去、エンコーディング自動判別（UTF-8/SJIS/EUC-JP）をリアルタイムで実行します。特に、ECサイトの商品マスタやログデータのクレンジングに最適化されています。

### コア機能

1. **自動スキーマ推論＆型キャストエンジン**  
   カラムごとに数値・日付・文字列を自動判定し、パースエラーが発生した行のみを隔離。例：`2024/01/15` と `2024-01-15` が混在する日付カラムをISO 8601に統一。型変換失敗行は別ファイルに出力し、手動修正を促す。

2. **ファジー重複検出（Levenshtein距離 + Jaccard類似度）**  
   単純な完全一致ではなく、表記ゆれ（全角/半角、スペース有無）や類似文字列（`株式会社` vs `(株)`）を検出。閾値は0.8以上で自動マージ提案。大規模データでもO(n log n)のインデックス最適化により高速動作。

3. **欠損値のコンテキスト補完**  
   欠損値がある行の前後コンテキストを分析し、線形補間（時系列データ）または最頻値補完（カテゴリデータ）を自動適用。補完結果は信頼度スコア（0-100）付きで表示。例えば、気温データの欠損は前後3行の移動平均で補完。

### 使い方

1. **データプロファイリングと前処理パラメータ設定**  
   `cleancsvai profile --input sales.csv --output report.json` を実行。  
   出力されたJSONレポートで、欠損率・外れ値・型不一致カラムを確認。その後、`--auto-fix` フラグで自動修正を適用するか、`--manual` フラグでインタラクティブモードを起動。  
   *ポイント*: プロファイリング結果は可視化ライブラリ（matplotlib）と連携可能。

2. **カスタムルールによるバッチクレンジング**  
   設定ファイル `cleancsvai.yaml` を作成し、カラムごとのルールを定義：  
   ```yaml
   columns:
     price:
       type: float
       min: 0
       max: 1000000
       replace_nan: 0.0
     date:
       format: "%Y-%m-%d"
       timezone: "Asia/Tokyo"
   ```  
   実行コマンド：`cleancsvai clean --config cleancsvai.yaml --input raw.csv --output clean.csv`  
   ルール違反行は `errors.csv` に自動退避。

3. **差分検出と変更履歴の管理**  
   クレンジング前後の差分をJSON形式で出力：  
   `cleancsvai diff --before raw.csv --after clean.csv --output diff.json`  
   変更内容（補完値・削除行・型変換）をツリー表示。Gitと連携するために、変更ログをコミットメッセージとして生成可能。

### ベストプラクティス

- **パイプライン化による自動化**  
  毎日実行されるバッチジョブでは、`cleancsvai profile` のレポートをSlack通知に連携。異常値が閾値を超えた場合のみ手動介入を促す。GitHub Actions に組み込む場合、`.github/workflows/clean.yml` に以下のステップを追加：  
  ```yaml
  - name: Clean CSV
    run: cleancsvai clean --input data/raw/*.csv --output data/clean/
  ```

- **大規模データの分割処理**  
  メモリ制限のある環境では、`--chunk-size 50000` オプションで50,000行ごとに分割処理。並列実行には `--workers 4` を指定。結果は `cleaned_part_*.csv` として出力され、後で `cleancsvai merge` で統合。これにより、1GB超のCSVもメモリ不足なく処理可能。
