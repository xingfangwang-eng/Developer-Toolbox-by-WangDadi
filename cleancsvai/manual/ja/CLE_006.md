# cleancsvai - CLE_006

## 概要

本プロジェクトは、大規模CSVデータセットの自動クレンジング・正規化に特化したSaaSツール「cleancsvai」の技術ドキュメントです。キーワード：CSVファイルクリーニング、データクレンジングツール、CSVデータ分析。機械学習ベースの異常検知とスキーマ推論により、従来の手作業を90%削減します。

## 詳細な内容

### cleancsvaiとは？

cleancsvaiは、非構造化CSVデータを解析・整形するためのプロフェッショナルツールです。従来のルールベースツールとは異なり、**統計的異常検知**と**パターンマッチング**を組み合わせ、データ型の自動判定、欠損値補完、重複排除を高精度で実行します。特に、異なるエンコーディング（UTF-8, Shift_JIS, EUC-JP）の混在ファイルにも対応し、日本語データセットの前処理に最適化されています。

### コア機能（3つのハードコア機能）

1. **マルチスキーマ自動推論エンジン**  
   - 50万行以上のCSVをストリーム解析し、カラムごとにデータ型（数値、日付、文字列、ブール）を統計的に推定。  
   - 推定結果を元に、型変換ルールを自動生成し、一括適用可能。  
   - 例：`"2023/01/01"` と `"01-01-2023"` が混在するカラムを自動でISO8601に統一。

2. **ファジー重複検出＆マージ機能**  
   - レーベンシュタイン距離とJaccard係数を組み合わせたハイブリッドアルゴリズムで、表記ゆれ（例：「株式会社」vs「(株)」）を検出。  
   - 重複スコア閾値を0.8〜0.95の範囲で設定可能。  
   - マージ時には、最新タイムスタンプのレコードを優先するポリシーを選択可能。

3. **エンコーディング自動判別＆コンバーター**  
   - ファイル先頭512バイトのバイトパターン解析により、エンコーディングを自動識別（UTF-8, Shift_JIS, EUC-JP, ISO-2022-JP）。  
   - 判別後、指定のエンコーディングに一括変換。  
   - 変換ログはJSON形式でエクスポート可能。

### 使い方（3つの詳細な操作手順）

1. **CSVアップロード＆スキーマ初期解析**  
   - CLIコマンド `cleancsvai analyze --input ./data/raw.csv --encoding auto` を実行。  
   - 内部処理：まずファイルの先頭512バイトを読み込み、エンコーディングを判別。その後、全行をストリーム解析し、カラムごとの統計量（平均値、標準偏差、欠損率）を算出。  
   - 出力例：`スキーマ推定完了: カラム数12、日付型3、数値型5、文字列型4、欠損率2.3%`。

2. **クレンジングルールのカスタマイズ**  
   - `cleancsvai config --rules ./rules.json` でルールファイルを指定。  
   - ルール例：  
     ```json
     {
       "column": "price",
       "type": "numeric",
       "outlier_method": "zscore",
       "zscore_threshold": 3.0,
       "action": "replace_with_median"
     }
     ```  
   - 実行後、ルール適用前後のデータ分布をヒストグラムで確認可能（`--visualize` オプション）。

3. **クレンジング実行＆エクスポート**  
   - `cleancsvai clean --input ./data/raw.csv --output ./data/clean.csv --format utf-8` を実行。  
   - 内部処理：スキーマ推定結果とルールファイルに基づき、欠損値補完（平均値、中央値、モード値から選択）、型変換、重複排除を逐次適用。  
   - 終了後、サマリーレポート（`clean_report.json`）が生成され、削除行数、修正行数、変換行数が記録される。

### ベストプラクティス（2つの業界推奨）

1. **パイプライン自動化による継続的データ品質管理**  
   - 毎日生成されるCSVデータに対して、cronジョブで `cleancsvai` を定期実行。  
   - 異常検知アラートをSlackやメールに通知し、データ品質の劣化をリアルタイムに検知。  
   - 推奨設定：閾値は業務要件に応じて調整（例：欠損率5%以上で警告）。

2. **大規模データセットでの並列処理活用**  
   - 100万行を超えるCSVは、`--parallel 4` オプションで4スレッド並列処理を有効化。  
   - メモリ使用量を抑えるため、`--chunk-size 10000`（1万行単位）で分割処理。  
   - これにより、処理時間を60%以上短縮可能。
