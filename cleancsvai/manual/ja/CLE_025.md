# cleancsvai - CLE_025

## 概要

CSVファイルの前処理に悩むエンジニアやデータサイエンティスト向けに設計された、高精度なデータクレンジングツールです。ノイズ除去、型推論、重複排除をワンコマンドで実行し、分析前のデータ準備工数を大幅に削減します。

## コア機能（ハードコア仕様）

### 1. マルチエンコーディング自動検出＋強制統一
ファイルのBOM有無やShift_JIS/UTF-8/ISO-8859-1などの文字コードを自動判別。さらに、全カラムを指定エンコーディングに強制変換し、文字化けやバイト列の混在をゼロにします。内部ではICUライブラリと正規化アルゴリズムを併用し、Unicode正規化（NFC/NFD）にも対応。

### 2. セマンティック型推論＋スキーマバリデーション
数値・日付・メール・郵便番号・電話番号など、20以上のパターンマッチャーを用いて各カラムのデータ型を自動推論。推論結果に基づき、スキーマ違反（例：数値カラムに文字列が混入している場合）を検出し、警告または自動修正（NaN変換・型キャスト）を実行します。

### 3. ファジー重複排除＋近傍レコードマージ
完全一致だけでなく、Levenshtein距離やJaccard類似度を用いたファジーマッチングにより、表記ゆれ（例：「株式会社ABC」「(株)ABC」「ABC Inc.」）を同一レコードとして認識。閾値を調整可能で、マージ時には最新タイムスタンプの値を優先するポリシーも設定できます。

## 使い方（実践手順）

### ステップ1: エンコーディング診断と自動修復
```bash
cleancsvai diagnose input.csv --encoding auto --fix utf-8
```
まず、`diagnose` サブコマンドでファイルの文字コードとBOM有無をスキャン。`--fix utf-8` オプションを付与することで、自動的にUTF-8に統一された出力ファイル `input_cleaned.csv` が生成されます。診断ログには各行のバイト列異常も記録されます。

### ステップ2: 型スキーマの定義とバリデーション
```bash
cleancsvai validate input_cleaned.csv --schema schema.json
```
スキーマファイル（JSON）で各カラムの期待型を定義します。例：
```json
{
  "columns": [
    {"name": "id", "type": "integer", "nullable": false},
    {"name": "email", "type": "email", "pattern": "^[\\w.-]+@[\\w.-]+\\.\\w+$"},
    {"name": "birth_date", "type": "date", "format": "%Y-%m-%d"}
  ]
}
```
バリデーションに失敗したレコードは `invalid_records.csv` に出力され、元ファイルには影響を与えません。

### ステップ3: ファジー重複排除とマージ
```bash
cleancsvai dedup input_cleaned.csv --similarity 0.85 --merge-policy latest --output deduped.csv
```
`--similarity 0.85` で類似度閾値を設定し、`--merge-policy latest` で重複レコード内で最新のタイムスタンプを持つ値を採用。出力先を `--output` で明示的に指定可能。重複検出の詳細レポートは `dedup_report.json` に保存されます。

## 業界ベストプラクティス

### 1. パイプラインの自動化と差分検知
定期的なデータ取り込み処理では、`cleancsvai` をCI/CDパイプラインに組み込み、`--diff-only` フラグで前回処理済みファイルとの差分のみをクレンジング対象にすると効率的です。これにより、フルスキャンによるオーバーヘッドを回避しつつ、品質を維持できます。

### 2. スキーマレジストリとの連携
Apache AvroやJSON Schemaで定義されたスキーマレジストリを参照し、`cleancsvai validate` のスキーマファイルとして動的に生成することを推奨します。これにより、データ基盤全体で一貫したバリデーションルールが適用され、後段の分析やMLモデルへの影響を最小限に抑えられます。
