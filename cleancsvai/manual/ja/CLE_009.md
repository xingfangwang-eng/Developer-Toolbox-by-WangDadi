# cleancsvai - CLE_009

## 概要

cleancsvaiは、機械学習ベースの異常検知とパターン認識を統合した、プロフェッショナル向けCSVデータクレンジングツールです。特に大規模データセットにおけるノイズ除去、型推論、欠損値補完を自動化し、データ分析の前処理時間を最大80%削減します。

## コア機能

### 1. マルチレイヤー異常検知エンジン
単なる重複除去や空白トリムに留まらず、統計的異常値（Zスコア/IQR法）とセマンティックパターン（日付フォーマットの不一致、住所表記ゆれ）を同時に検出。カスタムルールをYAMLで定義可能で、業界固有のデータ品質基準（ISO 8000準拠など）に合わせたチューニングが可能。

### 2. スキーマ推論＆型安全変換
CSVの各カラムに対して、数値・日付・カテゴリ・テキストを自動判別。特に日本語データにおける全角半角混在、和暦西暦混在、電話番号フォーマットの差異を認識し、統一された型に変換。出力時にPostgreSQL/MySQL/BigQuery用のスキーマ定義（DDL）を同時生成する。

### 3. 依存関係グラフによる欠損値補完
単なる平均値補完や前方補完ではなく、カラム間の相関関係（例：郵便番号と都道府県）を自動学習し、最も確からしい値を予測補完。機械学習モデル（LightGBMベース）を内蔵しており、補完精度をリアルタイムで可視化する。

## 使い方

### 1. カスタムルールセットの定義
プロジェクトルートに `cleancsvai_rules.yaml` を作成し、以下のように記述します：

```yaml
rules:
  - column: "phone"
    pattern: "^0\d{1,4}-\d{1,4}-\d{4}$"
    normalize: "strip_hyphen"
  - column: "date"
    pattern: "^(昭和|平成|令和)\d+年\d+月\d+日$"
    convert_to: "iso8601"
```

これを `cleancsvai --rules cleancsvai_rules.yaml input.csv -o output.csv` で実行すると、指定ルールに従って電話番号のハイフン除去と和暦→西暦変換が自動適用されます。

### 2. パイプラインによる段階的クレンジング
複数の処理をチェーンで実行し、中間結果を確認しながら進めるには：

```bash
cleancsvai pipeline \
  --steps "dedup,type_infer,outlier_detect,impute" \
  --dedup-threshold 0.95 \
  --outlier-method iqr \
  --impute-strategy correlation \
  --report report.html \
  input.csv -o clean_output.csv
```

各ステップの統計情報（重複率、異常値率、補完数）が `report.html` に出力され、どのカラムに問題があったかが一目でわかります。

### 3. APIモードでのリアルタイムバリデーション
Webアプリケーションから直接CSVの品質チェックを行う場合、REST APIとして起動します：

```bash
cleancsvai serve --port 8080 --allowed-origins "https://myapp.example.com"
```

エンドポイント `POST /validate` にCSVファイルを送信すると、JSON形式で以下のレスポンスが返ります：

```json
{
  "status": "warning",
  "issues": [
    {"column": "email", "row": 45, "type": "format_error", "value": "user@domain"},
    {"column": "age", "row": 102, "type": "outlier", "value": 245}
  ],
  "suggestions": ["emailカラムに@マーク不足が1件", "ageカラムに異常値（245）が存在"]
}
```

## ベストプラクティス

### 1. バージョン管理と差分監視
クレンジングルールや変換履歴は必ずGit管理し、`cleancsvai diff original.csv cleaned.csv` で変更点を確認。特に顧客データや金融データを扱う場合、監査証跡としての役割を果たします。`.cleancsvai_history/` ディレクトリに全変換ログが保存されるため、1ヶ月前の状態にロールバックすることも可能です。

### 2. バッチ処理の並列化設定
100万行を超える大規模CSVを扱う場合、`--workers 4` オプションで並列処理数を明示的に指定。また、メモリ使用量を抑えるために `--chunk-size 10000` でチャンク分割サイズを調整します。特にAWS LambdaやCloud Runなどのサーバーレス環境では、`--memory-limit 512` でメモリ制限を設定し、OOMエラーを防止します。
