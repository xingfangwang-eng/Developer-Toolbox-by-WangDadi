# cleancsvai - CLE_027

## 概要

大規模データセットの前処理で頭を抱えるエンジニアへ。cleancsvai は、機械学習ベースの異常検知とスキーマ推論を組み合わせ、CSV ファイルのクレンジング・正規化・バリデーションをワンコマンドで完了させる CLI ツールです。特に、不完全なデータやエンコーディングの混在したファイルを扱う際に真価を発揮します。

## 詳細な内容

### cleancsvai とは

cleancsvai は、Rust 製の軽量バイナリとして動作し、Python や Node.js の依存関係を一切必要としません。内部では、統計的異常検知（Z スコア・IQR）とルールベースのパターンマッチングを併用し、欠損値補完、型推論、重複排除を自動実行します。特に、マルチバイト文字（日本語・中国語・韓国語）を含む CSV のエンコーディング自動判別（UTF-8, Shift_JIS, EUC-JP 対応）と、日付フォーマットの統一が強力です。

### コア機能

- **動的スキーマ推論と型強制**  
  全カラムのデータ分布を分析し、int64 / float64 / datetime / string を自動判定。数値カラムに混入した文字列（例："N/A", "---"）を NaN に変換し、日付は ISO 8601 に統一。スキーマを YAML でエクスポートし、次回以降の処理に再利用可能。

- **マルチエンコーディング自動検出と変換**  
  BOM 有無やバイトパターンからエンコーディングを推定（charset-normalizer 相当のアルゴリズム）。Shift_JIS と UTF-8 が混在したファイルでも、カラム単位で適切なエンコーディングを適用し、文字化けを防止。変換後は BOM なし UTF-8 に統一。

- **コンテキスト依存の欠損値補完**  
  単純な平均値補完ではなく、時系列データの場合は前方補完（ffill）と線形補間、カテゴリデータの場合は最頻値補完を自動選択。さらに、欠損率が 50% を超えるカラムは自動削除し、処理ログに警告を出力。

### 使い方

1. **エンコーディング自動判別とスキーマ検証**  
   ```bash
   cleancsvai detect --input messy_data.csv --schema schema.yaml
   ```
   このコマンドは、まずファイルの先頭 1024 バイトをスキャンしてエンコーディングを判定。その後、全行をパースし、カラムごとにデータ型を推定します。推定結果は `schema.yaml` に出力され、手動で修正してから次のステップに再利用できます。例えば、`id` カラムが string と判定された場合、スキーマファイル内で `type: int64` に書き換えることで強制変換が可能です。

2. **異常値除去と欠損値補完を一括実行**  
   ```bash
   cleancsvai clean --input messy_data.csv --output clean_data.csv --strategy auto
   ```
   `--strategy auto` を指定すると、cleancsvai がデータの特性を分析し、最適な補完戦略を自動選択します。具体的には、各カラムの欠損率・データ型・分布を評価し、時系列カラムには `ffill`、数値カラムには `median`、カテゴリカラムには `mode` を適用。異常値は、Z スコアが 3 を超える値を外れ値とみなし、欠損値として扱った上で補完します。

3. **差分監査とログ出力**  
   ```bash
   cleancsvai audit --original messy_data.csv --cleaned clean_data.csv --report audit.log
   ```
   クレンジング前後のデータを比較し、以下の情報を JSON 形式で出力します：
   - 削除された行数とその理由（重複・異常値）
   - 補完された欠損値の数と補完方法
   - 型変換が行われたカラムと変換前後の型
   - エンコーディング変換の有無
   このレポートを CI/CD パイプラインに組み込むことで、データ品質のトレーサビリティを確保できます。

### ベストプラクティス

- **パイプラインの段階的構築**  
  大規模ファイル（100 万行以上）を処理する場合、`cleancsvai detect` でスキーマを事前生成し、手動で確認してから `cleancsvai clean` を実行してください。スキーマの手動調整により、意図しない型変換（例：郵便番号を数値として扱う）を防止できます。また、`--chunk-size 10000` オプションでメモリ使用量を制限し、ストリーミング処理を強制することを推奨します。

- **バージョン管理と再現性の確保**  
  スキーマファイル（`schema.yaml`）とクレンジング設定（`--strategy` オプション）を Git で管理し、データパイプラインの再現性を担保してください。cleancsvai は `--config config.yaml` で全設定を外部ファイルから読み込めるため、環境ごとに異なる戦略を適用する場合でも、設定ファイルを切り替えるだけで対応可能です。さらに、`--seed 42` で乱数シードを固定することで、補完結果の再現性を保証できます。
