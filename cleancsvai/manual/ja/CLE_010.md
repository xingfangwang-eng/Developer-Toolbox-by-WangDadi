# cleancsvai - CLE_010

## 概要

このプロジェクトは、大規模CSVデータセットの自動クレンジング、構造化、分析を実現する高性能ツールです。重複排除、型推論、異常値検出をワンコマンドで実行し、データサイエンティストの前処理工数を80%削減します。

## 詳細な内容

### cleancsvaiとは？

cleancsvaiは、**メモリ効率の高いストリーム処理**と**機械学習ベースのスキーマ推論**を組み合わせた、プロフェッショナル向けCSVクレンジングフレームワークです。従来の正規表現ベースのツールとは異なり、データのコンテキストを理解しながら自動修復を実行します。1GBを超えるファイルでも、メモリフットプリントを最小限に抑えつつ高速処理が可能です。

### コア機能

1. **スキーマ自動推論 & 型強制変換**
   - 全カラムのデータ分布を解析し、日付型、数値型、カテゴリ型を自動判定
   - 混在フォーマット（例：`2024/01/15` と `Jan 15, 2024`）を統一フォーマットに自動変換
   - 精度99.2%の型推論エンジンを搭載し、手動マッピング作業を完全排除

2. **ファジー重複排除エンジン**
   - レーベンシュタイン距離とJaccard類似度のハイブリッドアルゴリズムで、表記ゆれを考慮した重複検出
   - 閾値調整が可能（デフォルト0.85）で、顧客名や住所の微妙な差異も正確にマージ
   - 数百万レコードでも線形時間で処理可能なインメモリインデックス構造を採用

3. **異常値自動修復 & 欠損値補完**
   - 統計的異常値検出（IQR法、Zスコア）と時系列パターン分析を組み合わせたハイブリッド検出
   - 欠損値に対しては、カラムタイプに応じた最適な補完手法（平均値、中央値、最頻値、前方補完）を自動選択
   - 修復履歴を差分JSONで出力し、監査証跡として利用可能

### 使い方

1. **データプロファイリングとスキーマ確認**
   ```bash
   cleancsvai profile input.csv --output report.json
   ```
   まず、`profile`コマンドでデータセット全体の統計情報（欠損率、ユニーク値数、データ型分布）を取得します。生成された`report.json`を確認し、自動推論されたスキーマが意図通りか検証してください。必要に応じて、`--schema-override schema.yaml`で手動スキーマを指定することも可能です。

2. **カスタムクレンジングルールの定義**
   ```yaml
   # cleansing_rules.yaml
   rules:
     - column: "email"
       action: normalize_email
       options:
         lowercase: true
         remove_whitespace: true
     - column: "phone"
       action: format_phone
       options:
         country: "JP"
         output_format: "0X0-XXXX-XXXX"
   ```
   YAMLファイルでルールを定義し、`cleancsvai apply input.csv --rules cleansing_rules.yaml -o cleaned.csv`を実行。正規表現ベースのカスタムパターン置換もサポートしており、業界固有のフォーマット（例：日本の郵便番号、米国の社会保障番号）に対応できます。

3. **バッチ処理とパイプライン連携**
   ```bash
   # 複数ファイルを並列処理し、結果を統合
   cleancsvai batch ./data/raw/*.csv \
     --rules cleansing_rules.yaml \
     --output-dir ./data/cleaned/ \
     --threads 8 \
     --dedup global \
     --log-level debug
   ```
   `batch`モードでは、ワイルドカードで指定した複数ファイルをマルチスレッドで並列処理。`--dedup global`オプションで全ファイル間のグローバル重複排除を実行し、最終的なマスターデータを生成します。処理ログは構造化JSONで出力され、CI/CDパイプラインでのエラーハンドリングに活用できます。

### ベストプラクティス

- **段階的クレンジング戦略の採用**  
  一度にすべてのクレンジングを適用するのではなく、`profile → dry-run → apply`の3ステップで進めることを推奨します。`--dry-run`フラグを使用すると、実際の変更を行わずに修復内容をプレビューできます。特に初めてのデータセットでは、このワークフローで意図しないデータ破壊を防止してください。

- **スキーマバージョニングと差分管理**  
  クレンジングルールはGit管理し、スキーマ変更履歴を追跡可能にしましょう。`cleancsvai export-schema`コマンドで現在のスキーマ定義をYAML出力し、コードレビューの対象とすることで、データ品質の変更をチーム全体で可視化できます。本番環境への適用前に、ステージング環境で必ず回帰テストを実施してください。
