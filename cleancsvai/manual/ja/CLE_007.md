# cleancsvai - CLE_007

## 概要

cleancsvaiは、**機械学習による異常検知**と**パターンベースの自動修復**を統合した、次世代型CSVデータクレンジングツールです。大規模データセットの前処理、重複排除、型不一致の修正をワンコマンドで実行し、分析パイプラインの信頼性を劇的に向上させます。特に**日本語マルチバイト文字**と**数値フォーマットの自動認識**に強みを持ち、データサイエンティストの作業時間を平均70%削減します。

## コア機能（ハードコア3選）

### 1. ファジー型推論エンジン（FTIE）
列ごとに数値・日付・文字列を動的に判定し、**ISO 8601**、**和暦**、**カンマ区切り数値**を自動変換。例えば「2023年4月1日」と「2023/04/01」が混在する列を、統一されたタイムスタンプに正規化します。

### 2. マルチレイヤー重複検出（MLDD）
完全一致に加え、**レーベンシュタイン距離**と**Jaccard係数**を用いたあいまい重複を検出。同一顧客が「株式会社ABC」と「ABC株式会社」で登録されているケースを特定し、統合提案を行います。

### 3. 異常値スコアリング（AVS）
**四分位範囲（IQR）** と**Zスコア**を組み合わせたハイブリッド評価により、外れ値にスコアを付与。例えば「年齢列に-1が含まれている」といった論理エラーを即座にフラグ付けし、削除または補完の判断を支援します。

## 詳細な操作手順

### 1. データセットの前処理と型強制
```bash
cleancsvai infer --input raw_data.csv --output clean_data.csv --force-types int,date,string
```
- `--force-types`で列ごとの型を強制指定（未指定時はFTIEが自動推論）。
- 変換失敗行は`errors.log`に記録され、後続の手動修正が可能。
- 例：`int`列に文字列が混入した場合、`NaN`に変換し、AVSが警告を発行。

### 2. 重複排除とマージ戦略の設定
```python
import cleancsvai as cc
df = cc.load("clean_data.csv")
deduped = cc.deduplicate(
    df,
    threshold=0.85,          # あいまい一致のしきい値（0.0-1.0）
    merge_strategy="latest", # 重複行のマージ方法（"latest", "first", "keep_all"）
    key_columns=["email"]    # 重複判定の基準列
)
deduped.to_csv("deduped_data.csv", index=False)
```
- `threshold=0.85`は、文字列の85%以上が一致する場合を重複と見なす。
- `merge_strategy="latest"`は、タイムスタンプが最新の行を採用。

### 3. 異常値の可視化とフィルタリング
```r
library(cleancsvai)
data <- cc.read("deduped_data.csv")
scores <- cc.anomaly_score(data, method = "hybrid", columns = c("age", "salary"))
plot(scores, type = "histogram", threshold = 3)  # Zスコア閾値3以上を強調表示
filtered <- cc.filter_outliers(data, scores, threshold = 3)
write.csv(filtered, "final_clean.csv", row.names = FALSE)
```
- `method = "hybrid"`はIQR（箱ひげ図）とZスコアを加重平均。
- 閾値`3`は標準偏差3倍を超える値を異常と判定（デフォルト）。

## ベストプラクティス

### 1. パイプラインの段階的適用
大規模データ（10万行以上）では、`infer`→`deduplicate`→`anomaly_score`の順に処理を分割し、各ステップ後にサンプルデータで結果を検証することを推奨。特に日本語データでは、`infer`実行後に文字エンコーディング（Shift_JIS/UTF-8）の自動検出結果を確認してください。

### 2. カスタムルールの優先設定
業界固有のルール（例：医療データで「血圧値が0未満」をエラーとする）は、`--custom-rules`オプションでJSONファイルとして定義します。これにより、AVSのスコアリングに重み付けが反映され、誤検出を抑制できます。
