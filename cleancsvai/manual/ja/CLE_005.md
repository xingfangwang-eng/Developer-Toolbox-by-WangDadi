# cleancsvai - CLE_005

## 概要

大規模データセットの前処理に特化した次世代CSVクレンジングエンジン。機械学習パイプラインの前段処理として設計され、非構造化データを即座に分析可能な状態に変換する。特に日本語・英語混在データのエンコード問題を自動解決し、欠損値補完と異常値検出をリアルタイムで実行する。

## コア機能

### 1. マルチエンコーディング自動検出・統一変換
Shift-JIS, EUC-JP, UTF-8 BOM有無、ISO-8859-1など12種類のエンコーディングを自動判別。混在ファイルを検出した場合、文字化け箇所を特定し、最適なUnicode正規化（NFC/NFD）を適用する。特に日本語CSVで頻発する「半角カナ＋全角英数」の混合を自動統一。

### 2. スキーマ推論型データ型キャスティング
カラムごとに統計的分布を解析し、日付フォーマット（和暦/西暦自動判別）、数値区切り（カンマ/ピリオド）、通貨記号の混在を自動解決。例えば「2024年4月1日」「04/01/2024」「2024-04-01」が同一列に混在していても、ISO8601形式に統一する。

### 3. ファジーマッチング重複除去
レーベンシュタイン距離とJaro-Winkler距離を組み合わせたハイブリッドアルゴリズムで、表記ゆれ（「株式会社」vs「(株)」、「東京都」vs「東京」）を80%以上の類似度で同一レコードと判定。大量データでもO(n log n)の計算量で処理可能。

## 詳細な使い方

### ステップ1: エンコード問題の事前診断と修正
```bash
cleancsvai diagnose --input dirty_data.csv --encoding auto
```
出力される診断レポートで、各行のエンコード状態と文字化けリスクスコア（0-100）を確認。`--fix-encoding` フラグを追加すると、Shift-JISの「①」がUTF-8で「?」になる問題を自動修復。特に大手企業の基幹システムから出力されるCP932形式データで効果を発揮。

### ステップ2: カラム単位の正規化ルール適用
```python
import cleancsvai as cc

config = cc.Normalizer(
    column_rules={
        '電話番号': {
            'format': '国際標準',
            'remove_hyphen': True,
            'country_code': '81'
        },
        '郵便番号': {
            'format': 'XXX-XXXX',
            'convert_halfwidth': True
        }
    }
)
cc.process('cleaned.csv', normalizer=config)
```
この設定で「03-1234-5678」→「+81312345678」、「〒100-0001」→「100-0001」に自動変換。日本語環境特有の全角数字・半角カナ混在も同時に正規化。

### ステップ3: 異常値検出と欠損値補完の自動パイプライン
```bash
cleancsvai pipeline \
  --input cleaned.csv \
  --detect outliers \
  --impute knn \
  --neighbors 5 \
  --output final_dataset.csv
```
外れ値検出にはIQR法＋Zスコアの二重チェックを採用。欠損値補完はK近傍法（k=5）で、数値データは平均値、カテゴリデータは最頻値で補完。出力前に補完箇所をハイライトした差分レポートを自動生成。

## ベストプラクティス

### 1. バッチ処理時のメモリ管理最適化
100万行以上のCSVを処理する場合、`--chunk-size 50000` オプションで分割処理を推奨。内部処理ではApache Arrowのゼロコピー読み込みを利用し、メモリフットプリントを従来のpandas比で60%削減。大規模データセットでは必ず `--temp-dir /mnt/ssd/tmp` で高速ストレージを指定すること。

### 2. 機械学習パイプラインとの統合戦略
scikit-learnのPipelineクラスと直接連携可能なカスタムトランスフォーマーを提供。データクレンジング→特徴量エンジニアリング→モデル学習を一貫したワークフローで実行。特に時系列データの場合、`--timestamp-column` オプションで日付インデックスを自動生成し、欠損値補完に時系列特化のProphetアルゴリズムを選択可能。
