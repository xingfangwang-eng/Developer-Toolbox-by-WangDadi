# cleancsvai - CLE_029

## 概要

大規模データセットの品質問題を解決するため、cleancsvai は機械学習ベースの異常検知とパターン認識を統合した次世代CSVクレンジングツールです。重複レコード、欠損値、フォーマット不整合を自動修復し、データ分析パイプラインの前処理工数を90%削減します。

## ハードコア機能

### 1. マルチエンコーディング・スキーマ自動検出エンジン
UTF-8、Shift_JIS、EUC-JP、ISO-8859-1 など12種類の文字コードを自動判定。さらにカラムごとのデータ型（日付、数値、住所、メールアドレス）を推論し、型不一致をリアルタイムでハイライト表示。例えば「2024/01/15」と「2024-01-15」が混在する日付カラムを自動統一します。

### 2. ファジー重複除去＋コンテキストベース修復
レーベンシュタイン距離とJaccard係数を組み合わせた近似重複検出アルゴリズムを実装。同一顧客が「山田太郎」「ヤマダ タロウ」「Yamada Taro」で登録されている場合、最も完全なレコードを保持し、欠損値を補完。閾値は0.5〜0.95の範囲でスライダー調整可能。

### 3. カスタムルールエンジン＋正規表現バリデーター
YAMLベースのルール定義ファイルにより、業界固有のバリデーションを実装。例えば「電話番号は03-XXXX-XXXX形式のみ許可」「郵便番号は7桁数字」といった条件を設定可能。違反レコードは即座にエクスポート用ログに分離され、修正候補をAIが提案します。

## 詳細な操作手順

### 1. バッチクレンジングパイプラインの構築
```bash
cleancsvai pipeline --input ./raw_data/ --output ./cleaned/ \
  --encoding auto \
  --dedup-threshold 0.85 \
  --rules ./config/validation_rules.yaml
```
- `--encoding auto` で各ファイルの文字コードを個別判定
- `--dedup-threshold 0.85` で85%以上の類似度を重複とみなす
- 出力ディレクトリには `cleaned_*.csv` と `error_log_*.json` が生成

### 2. カラム型強制変換と正規化
```python
import cleancsvai as ccv

df = ccv.read_csv("sales_data.csv")
df = df.force_type("price", "float", on_error="impute_median")
df = df.force_type("date", "datetime", format="%Y/%m/%d")
df = df.normalize("phone", pattern=r"^0\d{1,3}-\d{2,4}-\d{4}$")
df.export("normalized_sales.csv", encoding="utf-8-sig")
```
- `on_error="impute_median"` で変換失敗セルを中央値で補完
- `normalize` は正規表現に合致しない値を `NaN` に置換

### 3. リアルタイム検証ダッシュボード起動
```bash
cleancsvai dashboard --port 8080 --watch ./data/
```
- Webブラウザで `http://localhost:8080` を開く
- 左ペイン：ファイルリストと処理ステータス（進行中/完了/エラー）
- 右ペイン：カラムごとの品質スコア（0〜100）と異常値分布グラフ
- エラーレコードをクリックすると修正提案がポップアップ表示

## 業界ベストプラクティス

### 1. バージョン管理と監査証跡
クレンジング前後のデータをGit LFSで管理し、各変換操作に一意のハッシュIDを付与。`cleancsvai audit --hash <hash_id>` で「いつ、誰が、どのルールで変更したか」を追跡可能に。特に金融・医療データでは監査要件を満たすため必須。

### 2. スケーラブルなパーティショニング戦略
100万行を超えるCSVは、`--chunk-size 50000` オプションで分割処理。各チャンクを並列実行し、最終的にマージ。メモリ使用量を抑制しつつ、処理時間を線形にスケール。AWS LambdaやGoogle Cloud Functionsへのデプロイも公式サポート。
