# cleancsvai - CLE_008

## 概要

大規模データパイプラインにおけるCSV品質問題を解決するために設計された、軽量かつ高精度なデータクレンジング専用ツール。機械学習ベースの異常検知とルールベースの正規化を融合し、ダーティデータをリアルタイムでクリーンな構造化データに変換する。

## コア機能

### 1. スキーマ推論＋型自動修復エンジン
CSVヘッダーと値分布から自動的にカラム型（日付、数値、ブール、列挙型）を推論。型不一致が発生した場合、値のパターン分析に基づいて最適なキャストを提案・自動適用する。例えば「2023-01-32」のような不正日付は自動的に前月末日に補正される。

### 2. ファジー重複排除＋正規化レコメンダー
Levenshtein距離とトークンベース類似度を組み合わせたマルチアルゴリズムで重複行を検出。単純な削除ではなく、どの行をマージすべきか、どのフィールドを優先すべきかをスコアリングし、ユーザーに確認を促すインタラクティブモードを搭載。

### 3. エンコーディング自動検出＋BOM除去パイプライン
Shift_JIS、EUC-JP、UTF-8 BOM付きなど、日本語環境で頻出するエンコーディング問題を自動検出。BOM有無や改行コード（CRLF/LF）の統一も含め、出力前に一貫性を担保。特にレガシーシステムからのCSV取り込み時に威力を発揮する。

## 使い方

### ステップ1： プロジェクト初期化と設定ファイル生成
```bash
cleancsvai init --lang ja --encoding auto --dedup-strength high
```
カレントディレクトリに`cleancsvai.yaml`が生成される。ここで除外カラム、カスタム正規表現ルール、重複判定の閾値（0.0〜1.0）を定義可能。例えば`ignore_columns: ["timestamp", "user_agent"]`のように指定すれば、分析対象から外せる。

### ステップ2： バッチクレンジング実行 with ドライラン
```bash
cleancsvai clean ./input.csv --dry-run --report ./report.html
```
ドライランモードでは実際のファイルは変更せず、検出された問題点（型不一致数、重複候補数、エンコーディング不一致）をHTMLレポートとして出力。レポート内では各行の修正前後の差分がハイライト表示され、どのルールが適用されたかがトレース可能。

### ステップ3： インタラクティブマージと確定出力
```bash
cleancsvai merge --interactive --output ./cleaned.csv
```
直前のドライランレポートを元に、重複行や型修復候補を一つずつ確認しながらマージ。ターミナル上で`y`（採用）、`n`（スキップ）、`e`（編集）の3択で進める。最終的に`cleaned.csv`として出力され、同時に適用済みルールのJSONログが`cleancsvai_log.json`に保存される。

## ベストプラクティス

### 1. パイプラインの前段にcleancsvaiを組み込む
ETL/ELTパイプラインの最初のステージとしてcleancsvaiを配置することで、後続のデータベースロードや分析処理で発生する型エラーや重複キー違反を事前に防止できる。特にAirflowやPrefectとの統合は、CLI呼び出しで完結するため最小限のコード変更で導入可能。

### 2. 定期実行時の差分クレンジング戦略
毎日更新されるCSVに対しては、全件再処理ではなく`--incremental`モードを推奨。前回の処理ログ（`cleancsvai_log.json`）を参照し、新規行と変更行のみを対象にクレンジングを実行。大規模データセット（100万行超）でも処理時間を10分の1に抑えられる。
