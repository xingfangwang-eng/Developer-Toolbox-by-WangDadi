# cleancsvai - CLE_003

## 概要

本プロジェクトは、機械学習と統計的推論を統合した次世代CSVデータクレンジングツールです。従来のルールベース処理では対応困難な**非構造化ノイズ、型推論ミス、重複エントリ**を、コンテキスト認識型AIで排除します。SEO的には「CSVクリーニング」「データ前処理自動化」「AIデータ分析基盤」をキーワードに、日本のデータエンジニアリング市場における競争力を最大化します。

## 詳細な内容

### cleancsvaiとは何か

cleancsvaiは、**バリデーション・正規化・エンリッチメント**を単一パイプラインで実現するSaaS型データ準備基盤です。特に日本語環境（Shift_JIS、EUC-JP混在）における文字化け修復、日付形式の曖昧性解決（令和/西暦自動変換）、数百万行規模のストリーム処理を軽量アーキテクチャで実現。ETL前処理の工数を80%削減します。

### コア機能

1. **コンテキストアウェア型異常値検出**
   - 単なる統計的閾値ではなく、列ごとのセマンティクス（顧客名簿 vs 売上データ）をBERTベースの軽量モデルが学習。例えば「年齢カラムに"不明"が含まれる場合」を異常値としてフラグするだけでなく、業界標準データセットとのコサイン類似度スコアを算出し、置換候補を提示します。

2. **マルチバイト文字自動修復エンジン**
   - 日本語CSVで頻発する「UTF-8→Shift_JIS変換時の文字化け」を、Unicode正規化とN-gram確率モデルで復元。具体的には「�」や「？？？」をコンテキストから推定し、JIS第1・第2水準漢字にマッピング。さらに機密情報（個人番号、クレジットマスク）を検出し、自動マスキングするコンプライアンス機能を内蔵。

3. **スキーマレス動的型推論 + カラム自動マージ**
   - ヘッダー行が欠損している生データでも、最初の100行のパターンから数値/日付/カテゴリを推論。さらに「氏名」「名前」「name」など同義カラムをWord2Vecで検出し、ユーザー承認なしでマージ。重複行の削除だけでなく、ファジーマッチング（Levenshtein距離 < 2）による類似レコード統合も実行。

### 使い方

1. **バルクデータの前処理パイプライン構築**
   - ターミナルで `cleancsvai --input ./raw_data/*.csv --output ./cleaned/ --lang ja --auto-fix encoding` を実行。内部ではまず文字コード検出（ICUライブラリ使用）、次にストリーム分割（1スレッドあたり500KBチャンク）、最後に並列バリデーションが走ります。出力先には元のファイル名に `_cleaned` サフィックスが付与され、処理ログはJSON Lines形式で `./logs/` に保存。

2. **カスタムバリデーションルールの定義**
   - プロジェクトルートに `cleancsvai.rules.json` を作成し、以下を記述：
     ```json
     {
       "columns": [
         {"name": "postal_code", "type": "japan_post", "allow_empty": false, "format": "NNN-NNNN"},
         {"name": "phone", "type": "japan_phone", "regex": "^0[0-9]{9,10}$"}
       ],
       "global_rules": {
         "remove_duplicates": "fuzzy", 
         "fuzzy_threshold": 0.85
       }
     }
     ```
     これを `cleancsvai --rules ./cleancsvai.rules.json --input ./target.csv` で適用。ルール違反行は `--quarantine` フラグで隔離ディレクトリに退避可能。

3. **API経由でのリアルタイムクリーニング**
   - PythonクライアントからRESTエンドポイントにPOST：
     ```python
     import requests
     files = {'file': open('dirty.csv', 'rb')}
     data = {'options': '{"fix_encoding": true, "dedup": "exact"}', 'callback_url': 'https://your-app.com/webhook'}
     resp = requests.post('https://api.cleancsvai.com/v1/clean', files=files, data=data)
     ```
     レスポンスは非同期で、ジョブIDが即時返却。完了後、`callback_url` に結果JSON（クレンジング統計、エラー行サマリー、修正差分）がPOSTされます。

### ベストプラクティス

- **日次バッチ処理でのインクリメンタル活用**  
  毎日生成されるログCSV（例：アクセスログ、購買データ）に対して、`--incremental` フラグを使用。前回処理時のチェックサム（SHA256）を保持し、新規行のみをクリーニング。これによりフルスキャン時間を90%削減。特に数百万行規模のデータセットで効果を発揮します。

- **テスト駆動型データクレンジングの導入**  
  本番投入前に、既知の正解データセット（golden dataset）を `--validate` モードで実行。出力されたCSVと正解データの行単位一致率をF1スコアで評価し、閾値（例：0.98）を下回った場合はパイプラインを停止。CI/CDに組み込むことで、データ品質のリグレッションを防止します。
