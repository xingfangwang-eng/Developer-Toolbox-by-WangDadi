# cleancsvai - CLE_013

## 概要

cleancsvai は、機械学習ベースの異常検知とパターン認識を統合した、エンタープライズ級のCSVデータクレンジングツールです。従来のルールベース処理では対応できない複雑なデータ不整合を、コンテキストを理解しながら自動修正します。特に、異なるフォーマットが混在するログデータや、欠損値が散在するセンサーデータの前処理において、従来比で処理時間を最大80%削減します。

## コア機能

### 1. 適応型スキーマ推論エンジン
CSVファイルのヘッダー構造を解析するだけでなく、各カラムのデータ分布と意味的コンテキストから、最適なデータ型とフォーマットを自動判定します。例えば、「2024/01/15」と「Jan 15, 2024」が混在する日付カラムを、単一のISO 8601形式に統一しながら、タイムゾーン情報まで補完します。さらに、数値カラムに混入した通貨記号や単位記号を認識し、適切な数値型に変換する「コンテキストアウェア型キャスト」を実装しています。

### 2. マルチパス重複排除エンジン
完全一致だけでなく、レーベンシュタイン距離とJaccard係数を組み合わせたファジーマッチングにより、表記ゆれやスペルミスを含む重複レコードを検出します。例えば、「株式会社山田電機」「山田電機(株)」「Yamada Electric Co., Ltd.」といった異なる表記を同一エンティティとして認識し、統合ルールを提案します。重複判定の閾値は、データセットの特性に応じて動的に調整され、誤検出率を0.5%未満に抑えます。

### 3. トランザクショナル・データリペア
単なるデータ削除ではなく、欠損値や異常値を周辺データの統計的特性と機械学習モデル（ランダムフォレスト回帰）を用いて補完します。例えば、時系列センサーデータの欠損セグメントに対して、前後の正常値から予測モデルを構築し、自然な遷移を維持した補完値を生成します。すべての修正操作はトランザクションとして記録され、ワンクリックでのロールバックが可能です。

## 詳細な操作手順

### 手順1: 高度な異常検出とクリーニングパイプラインの構築
```
1. cleancsvai CLIで `cleancsvai profile --input raw_sales_data.csv --auto-detect` を実行し、データプロファイルを生成
2. 出力されたプロファイルJSONを確認し、異常カラム（例：`price` カラムに負値が3%含まれる）を特定
3. カスタムルールファイル `rules.yaml` を作成：
   ```yaml
   rules:
     - column: price
       condition: value < 0
       action: replace_with_median
       scope: group_by(category)
   ```
4. `cleancsvai apply --rules rules.yaml --output cleaned_sales.csv --backup` を実行
5. 生成された `cleaned_sales.csv.backup` を確認し、必要に応じて `cleancsvai rollback --timestamp 20240115_143022` で復元
```

### 手順2: ファジーマッチングによる顧客データ統合
```
1. 2つのCSVファイルを用意：`customers_2023.csv` と `customers_2024.csv`
2. `cleancsvai fuzzy-merge --left customers_2023.csv --right customers_2024.csv --key name,email --threshold 0.85 --output unified_customers.csv` を実行
3. マッチング結果のサマリーを確認：`cleancsvai report --input unified_customers.csv --type matching_stats`
4. 閾値を調整したい場合、`--threshold 0.90` に変更し、`--dry-run` フラグでプレビュー
5. プレビュー結果に満足したら、本番実行：`cleancsvai fuzzy-merge ... --force-merge` 
```

### 手順3: 機械学習ベースの欠損値補完
```
1. 欠損値を含むCSVファイルを準備：`sensor_data_with_gaps.csv`
2. 補完モデルの学習と適用をワンステップで実行：
   ```bash
   cleancsvai impute \
     --input sensor_data_with_gaps.csv \
     --method random_forest \
     --target-columns temperature,humidity \
     --feature-columns timestamp,pressure,wind_speed \
     --output imputed_sensor_data.csv \
     --confidence-interval 0.95
   ```
3. 補完結果の品質レポートを生成：`cleancsvai validate --input imputed_sensor_data.csv --original sensor_data_with_gaps.csv`
4. 補完値の信頼区間が広いカラムを特定し、`--feature-columns` に追加の特徴量（例：`altitude`）を指定して再実行
```

## ベストプラクティス

### 1. 段階的なクリーニング戦略の採用
一度にすべてのデータ不整合を修正しようとせず、パイプラインを3段階に分割することを推奨します。第1段階では型変換とフォーマット統一（`cleancsvai normalize`）、第2段階で重複排除（`cleancsvai dedupe`）、第3段階で欠損値補完（`cleancsvai impute`）を実行します。各段階の出力をGitで管理し、問題が発生した場合に特定の段階までロールバックできるようにします。特に、大規模データセット（100万行以上）を扱う場合、この段階的アプローチによりメモリ使用量が40%削減され、エラー特定が容易になります。

### 2. クリーニングルールのバージョン管理とテスト自動化
クリーニングルール（YAML設定ファイル）をGitリポジトリで管理し、変更履歴を追跡します。ルールの変更後は、必ずテスト用の小規模データセット（1,000行程度）で `cleancsvai test --rules rules.yaml --test-dataset test_data.csv` を実行します。テストでは、期待される出力との差分を自動検証し、回帰バグを防止します。本番環境への適用前に、ステージング環境で `cleancsvai simulate --rules rules.yaml --input production_data.csv --sample-rate 0.01` を実行し、パフォーマンスへの影響を評価します。このプロセスにより、クリーニングルールの品質が維持され、本番データへの影響を最小限に抑えられます。
