# cleancsvai - CLE_024

## 概要

cleancsvaiは、機械学習とルールベース処理を融合した高精度CSVデータクレンジングツールです。ダーティデータの自動検出・修復、スキーマ正規化、異常値補完をワンストップで実現。従来の手作業によるデータ整形工数を90%削減し、分析準備フェーズを劇的に加速します。

## ハードコア機能

### 1. マルチモーダル異常検知エンジン
統計的分布（z-score/IQR）と教師なし学習（Isolation Forest）を組み合わせた2段階パイプラインで、外れ値・欠損・型不一致を同時検出。カラムごとに最適な閾値を自動チューニングし、偽陽性率0.5%未満を達成。

### 2. コンテキストアウェア型値補完
欠損セルに対して、同一レコードの他カラム値・時系列トレンド・カテゴリ相関を考慮したマルチソース補完を実行。数値カラムにはランダムフォレスト回帰、テキストカラムにはBERTベースのセマンティック推論を適用し、補完精度95%以上を担保。

### 3. スキーマフリー正規化パイプライン
入力CSVのカラム名・データ型・フォーマットを動的に解析し、ISO 8601日付形式、UTF-8エンコーディング、統一数値表記（カンマ区切り→ドット区切り）へ自動変換。1000カラム超のワイドテーブルでも5秒以内にスキーマ検出を完了。

## 実践操作手順

### 手順1: 異常検出パイプラインの実行
```bash
cleancsvai detect --input sales_data.csv --output anomalies.json --method hybrid
```
`--method hybrid`フラグで統計的検出＋機械学習検出を同時起動。出力されるanomalies.jsonには、異常行のインデックス、異常タイプ（outlier/missing/type_mismatch）、信頼スコアが含まれる。スコア0.8以上のレコードは自動修復対象となる。

### 手順2: コンテキスト補完による欠損値修復
```python
import cleancsvai as cc
engine = cc.CleaningEngine(strategy='contextual')
engine.load_anomalies('anomalies.json')
engine.repair(input_path='sales_data.csv', output_path='sales_clean.csv', 
              fill_method='random_forest', impute_threshold=0.85)
```
`fill_method='random_forest'`で数値カラムの欠損を周辺データから予測補完。`impute_threshold`を0.85に設定することで、補完確信度が低いセルは強制的にNULLマークを残し、後続の手動レビューを促す。

### 手順3: 正規化後のバリデーションとエクスポート
```bash
cleancsvai validate --input sales_clean.csv --schema schema.json --output validation_report.csv
cleancsvai export --input sales_clean.csv --format parquet --compression snappy
```
`validate`コマンドで、事前定義したschema.json（例：priceカラムはfloat64、dateカラムはdatetime64）との整合性をチェック。エラー行はvalidation_report.csvに抽出される。`export`でParquet形式に変換し、Snappy圧縮でストレージ効率を向上。10GBのCSVが約1.2GBに圧縮される。

## 業界ベストプラクティス

### プラクティス1: 段階的クレンジング戦略の採用
全データを一度にクレンジングするのではなく、最初にサンプリング（全行の5%）でパイプラインの精度を検証。異常検出の偽陽性率が許容範囲（例：2%未満）であることを確認後、フルデータセットに展開する。これにより、大規模データセットでの予期せぬエラーによる処理中断リスクを回避。

### プラクティス2: バージョン管理されたスキーマ定義の活用
cleancsvaiの`schema export`機能で生成したスキーマJSONをGit管理し、データソースの変更に追従。スキーマ変更時は`cleancsvai diff --old schema_v1.json --new schema_v2.json`で差分を可視化し、影響範囲を事前評価してからクレンジングパイプラインを更新する。これにより、プロダクション環境でのデータ品質劣化を防止。
