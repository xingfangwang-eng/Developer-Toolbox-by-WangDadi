# cleancsvai - CLE_021

## 概要

本プロジェクトは、大規模CSVデータの自動クレンジング・正規化・分析を実現するSaaS型ツール「cleancsvai」の技術ドキュメントです。キーワード：CSVファイルクリーニング、データクレンジングツール、CSVデータ分析

## 詳細な内容

### cleancsvaiとは

cleancsvaiは、機械学習ベースの異常検知とルールエンジンを統合した、プロフェッショナル向けCSVデータクレンジングプラットフォームです。従来の正規表現ベースのツールとは異なり、データパターンを自動学習し、欠損値補完、型推論、重複除去をワンストップで実行します。特に、100万行を超える大規模データセットでもメモリ効率の良いストリーミング処理が可能で、データエンジニアやアナリストの前処理工数を80%削減します。

### コア機能

1. **適応型スキーマ推論エンジン**  
   CSVヘッダーとデータ分布から自動でカラム型（日付、数値、カテゴリ、JSON埋め込みなど）を判定。ユーザーがスキーマを定義する必要は一切なく、異種混在データもワンパスで正規化。

2. **非同期パイプラインによる差分クレンジング**  
   変更差分のみを検出・再処理するインクリメンタルモード搭載。毎回全データをスキャンせず、前回のクレンジング結果と比較して新規・更新行だけを処理。CI/CDパイプラインとの統合に最適。

3. **多言語エンコーディング自動修復**  
   Shift_JIS、EUC-JP、UTF-8 BOM有無など、日本環境で頻発する文字化け問題を自動検出・変換。さらに、全角英数字や機種依存文字（①、㈱など）をUnicode正規化し、後続処理の互換性を担保。

### 使い方

1. **CLIによるワンライナー実行**  
   ```bash
   cleancsvai --input ./raw_data.csv --output ./cleaned_data.csv --mode auto
   ```
   上記コマンドで、自動検出モードが起動。カラム型推論、欠損値補完（中央値/最頻値/線形補間から選択）、重複除去が順次実行される。進捗はstderrにJSON形式で出力され、パイプライン処理との連携が容易。

2. **REST APIによる非同期ジョブ管理**  
   ```bash
   curl -X POST https://api.cleancsvai.io/v1/jobs \
     -H "Authorization: Bearer ${API_KEY}" \
     -H "Content-Type: application/json" \
     -d '{"source": "s3://my-bucket/dirty.csv", "rules": ["drop_duplicates", "fill_missing"], "callback": "https://myapp.com/webhook"}'
   ```
   ジョブIDが即座に返却され、バックグラウンドで処理が進行。完了後は指定されたWebhookエンドポイントに結果URLが通知される。大規模データでもタイムアウトを気にする必要はない。

3. **カスタムルール定義（YAMLベース）**  
   ```yaml
   rules:
     - name: "email_validate"
       type: regex
       pattern: "^[\\w\\.-]+@[\\w\\.-]+\\.\\w{2,}$"
       action: reject
     - name: "price_normalize"
       type: transform
       column: "price"
       function: "round(value, 2)"
   ```
   上記ルールファイルを`--rules ./rules.yaml`で指定することで、ドメイン特化のバリデーション・変換を追加可能。正規表現やJavaScript関数もサポートし、柔軟なデータ加工を実現。

### ベストプラクティス

- **パイプラインの段階的適用**  
  初回は`--mode auto`で全体の傾向を把握し、その後カスタムルールで例外処理を追加する二段階アプローチが推奨。特に日本企業のデータでは、全角スペースや機種依存文字が想定外の箇所に混入しているケースが多いため、autoモードのレポートを確認してからルールを調整すると効率的。

- **差分モードの定期実行**  
  cronやGitHub Actionsで1時間ごとに`--mode incremental`を実行し、常にクリーンなデータセットを維持する。これにより、データウェアハウスへの取り込み時にクレンジングを再実行する必要がなくなり、ETLパイプラインのレイテンシを最小化できる。特に、日次バッチ処理で更新される売上データや顧客マスタに有効。
