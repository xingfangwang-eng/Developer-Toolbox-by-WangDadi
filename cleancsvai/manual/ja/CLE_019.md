# cleancsvai - CLE_019

## 概要

大規模データパイプラインにおけるCSVファイルの品質問題を根本から解決する。重複行、欠損値、型不一致、エンコーディング崩れを自動検出・修復し、分析準備時間を90%削減する。

## 詳細な内容

### とは cleancsvai？

cleancsvai は、機械学習ベースの異常検知とルールエンジンを組み合わせた、CSV専用データクレンジングフレームワーク。従来の正規表現や手動ルールに依存せず、カラム間の統計的相関を学習し、異常値をコンテキストに応じて適切に補完または除去する。

### コア機能

- **マルチエンコーディング自動判別・修復**  
  Shift_JIS、EUC-JP、UTF-8 BOM付きなど、混在したファイルを読み込むと同時に、文字化け箇所を特定し、Unicode正規化を適用。さらに、改行コード（CRLF/LF/CR）の統一も自動実行。

- **スキーマ推論＋型安全変換**  
  全カラムのデータ型（int64、float64、datetime64、object）を統計的に推定。日付フォーマット（`2024/01/01`、`2024-01-01T00:00:00Z`、`令和6年1月1日`）を自動認識し、ISO 8601に統一。変換失敗行はログに出力し、元データを保持。

- **重複・部分重複検出＋マージ制御**  
  完全一致だけでなく、Levenshtein距離とJaccard係数を用いたファジーマッチングで類似行を検出。マージポリシー（最新行優先、平均値補完、手動選択）を設定可能。

### 使い方

1. **前処理パイプラインの構築**  
   `cleancsvai pipeline init` を実行し、YAML設定ファイルを生成。`encoding: auto`、`null_threshold: 0.3`（欠損率30%以上のカラムは自動削除）、`type_coercion: strict`（型変換失敗時はエラー停止）を指定。設定後、`cleancsvai pipeline validate` でスキーマとルールの一貫性を検証。

2. **バッチクレンジングの実行**  
   `cleancsvai clean input.csv --output output.parquet --log-level debug` を実行。処理中は進捗バーとリアルタイム統計（削除行数、補完数、型変換数）が表示される。完了後、`output_report.json` に全操作のトレーサビリティが記録される。

3. **結果レビューとロールバック**  
   `cleancsvai review output.parquet --show-changes` で変更箇所をハイライト表示。各行の変更前/後をdiff形式で確認可能。不適切な修正があった場合、`cleancsvai rollback --revision <hash>` で特定の操作のみを巻き戻し、再実行。

### ベストプラクティス

- **パイプラインの段階的適用**  
  初回は`--dry-run`フラグで変更を適用せずにレポートのみ生成。欠損率、重複率、型エラー率を確認し、しきい値をチューニングしてから本実行する。特に日付カラムは手動サンプルチェックを推奨。

- **CI/CDへの統合**  
  GitHub Actionsでデータ取り込み前に`cleancsvai check`を実行。品質スコア（0-100）が設定値（例: 85）未満の場合、パイプラインを自動停止。通知をSlackに飛ばし、ヒューマンレビューをトリガーする。
