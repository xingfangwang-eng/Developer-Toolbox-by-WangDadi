# cleancsvai - CLE_017

## 概要

本ツールは、大規模データセットにおける型推論の曖昧性、エンコーディング衝突、および非構造化カラムの自動正規化を解決するために設計された、プロフェッショナル向けCSVクレンジングプラットフォームです。キーワード：CSVファイルクリーニング、データクレンジングツール、CSVデータ分析。

## 詳細な内容

### cleancsvaiとは？

cleancsvaiは、機械学習ベースの型推論エンジンとメモリマップドファイル処理を組み合わせ、10GBを超える巨大CSVでもストリーム処理で高速クレンジングを実現します。従来のルールベースツールでは対処しきれない、日付フォーマットの混在、特殊文字の埋め込み、欠損値のパターン認識を自動解決します。

### コア機能

- **パターン認識型異常値検出エンジン**：単なるnullチェックではなく、カラムごとの統計的分布（標準偏差、四分位範囲）をリアルタイムで計算し、外れ値を確率的にフラグ付け。教師なし学習による異常パターンの自動分類を実行します。
- **マルチエンコーディング自動解決とバイナリレベルの文字正規化**：UTF-8、Shift_JIS、EUC-JP、ISO-8859-1など16種類の文字コードを自動検出し、混在ファイルでも統一エンコーディングに変換。さらに、全角・半角の統合、制御文字の除去、BOMの自動除去をバイナリレベルで実施します。
- **スキーマ進化追跡と差分マージ機能**：同一データソースから定期的に生成されるCSVのカラム構造変化（追加、削除、型変更）を自動検出し、履歴として保存。異なるバージョンのCSVをスキーマに基づいてインテリジェントにマージ可能です。

### 使い方

1. **型推論を伴った自動クレンジングパイプラインの起動**  
   ターミナルで以下のコマンドを実行し、入力ファイルと出力先を指定します。`--infer-types`フラグを付与することで、全カラムのデータ型を自動判定し、数値カラム内の文字列混在を自動修正します。`--encoding auto`でエンコーディング自動検出が有効になります。
   ```
   cleancsvai --input ./raw_data.csv --output ./cleaned_data.csv --infer-types --encoding auto
   ```

2. **カスタムルールベースの異常値フィルタリング**  
   `--rules`オプションでJSON形式のルールファイルを指定します。例えば、年齢カラムに0から120の範囲制限を設け、メールアドレスカラムに正規表現バリデーションを適用する場合、以下のように設定します。
   ```
   cleancsvai --input ./data.csv --rules ./rules.json --strict-mode
   ```
   ルールファイル例：
   ```json
   {
     "columns": [
       {"name": "age", "type": "integer", "min": 0, "max": 120},
       {"name": "email", "type": "string", "pattern": "^[\\w.-]+@[\\w.-]+\\.\\w{2,}$"}
     ]
   }
   ```

3. **スキーマ差分検出とマージの実行**  
   複数バージョンのCSVをマージする場合、`--merge`フラグと共にバージョン履歴ディレクトリを指定します。cleancsvaiは自動的にスキーマ変更を検出し、欠落カラムにはデフォルト値（`--default-value`で指定）を挿入します。
   ```
   cleancsvai --merge ./history/ --output ./merged.csv --default-value "N/A"
   ```

### ベストプラクティス

- **大規模データセットではメモリマップドモードを明示的に有効化する**  
  デフォルトではファイル全体をメモリに読み込みますが、5GB以上のファイルを扱う場合は`--mmap`フラグを使用し、OSの仮想メモリ管理に委譲することで、物理メモリ消費を最小限に抑えられます。また、`--batch-size 100000`でバッチ処理単位を調整し、GCプレッシャーを軽減します。

- **クレンジング前に必ずプロファイリングレポートを生成する**  
  `--profile`フラグでHTML形式のデータプロファイリングレポートを出力し、欠損率、ユニーク値比率、型不一致の分布を可視化します。このレポートを基にルールファイルを設計することで、過剰なデータ削除を防ぎ、精度の高いクレンジングが可能になります。
