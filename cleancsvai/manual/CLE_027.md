## What is cleancsvai?

**cleancsvai** is a high-performance, AI-powered CSV cleaning engine built for developers who need to sanitize, normalize, and analyze messy tabular data at scale.  
It eliminates the drudgery of manual data wrangling by combining intelligent pattern detection with batch processing — no more regex hell or broken pipelines.

---

## Core Features

### 1. **Adaptive Schema Inference with Auto-Repair**
cleancsvai detects column types (string, numeric, date, boolean) from raw CSV headers and values, then automatically casts mismatched rows.  
It handles mixed encodings (UTF-8, Latin-1, Shift-JIS), trailing delimiters, and quoted field anomalies without breaking the output.

### 2. **Fuzzy Deduplication Engine with Levenshtein & Jaccard Scoring**
Beyond exact matching, the engine identifies near-duplicate rows using configurable similarity thresholds.  
It supports field-level weighting (e.g., “email” > “name”) and outputs a dedup report with confidence scores — critical for CRM or lead list cleanup.

### 3. **Streaming Pipeline with In-Memory Chunking**
Process CSV files up to 500MB without loading the entire dataset into RAM.  
The tool streams rows through a configurable pipeline (validate → clean → transform → export), emitting progress callbacks for real-time monitoring.

## How to Use

### 1. **Initialize the Cleaner with Strict Mode**
```bash
cleancsvai --input messy_data.csv --output clean_data.csv --strict --schema auto
```
This command triggers the adaptive schema inference. If a row fails type validation (e.g., a string in a numeric column), the tool logs the error and skips the row — or you can use `--fail-fast` to halt on first error.

### 2. **Apply Fuzzy Deduplication with Custom Thresholds**
```bash
cleancsvai --dedup --threshold 0.85 --weights email:0.5,name:0.3,phone:0.2
```
The engine compares all rows using a weighted Jaccard + Levenshtein hybrid. Output includes a `_dedup_score` column and a separate `duplicates_report.csv` for manual review.

### 3. **Chain Transformations with a JSON Pipeline Config**
Create a `pipeline.json` file:
```json
{
  "steps": [
    {"type": "trim_whitespace"},
    {"type": "normalize_dates", "format": "ISO8601"},
    {"type": "replace_nulls", "value": "N/A"},
    {"type": "export", "format": "parquet"}
  ]
}
```
Then run:
```bash
cleancsvai --pipeline pipeline.json --input raw.csv --output final.parquet
```
This is ideal for CI/CD data preprocessing workflows where reproducibility matters.

## Best Practices

### 1. **Always Run a Dry Run First**
Use the `--dry-run` flag to preview row-level changes without writing output. This helps catch schema mismatches or aggressive dedup thresholds before they corrupt production data. Combine with `--verbose` to see per-row transformation logs.

### 2. **Version Your Cleaned Outputs**
Append a timestamp or git commit hash to output filenames (e.g., `clean_data_v2.3.1.csv`). This ensures traceability when debugging downstream ML pipelines or dashboards. Pair this with the `--report` flag to generate a JSON audit trail of all transformations applied.
