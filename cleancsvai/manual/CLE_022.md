# cleancsvai - CLE_022

You’re building pipelines, not spreadsheets. **cleancsvai** is a CLI‑first, API‑native CSV cleaning engine designed to eliminate encoding drift, schema drift, and silent row corruption before they hit your ETL or ML stack. No GUI. No fluff. Just deterministic, configurable sanitization.

---

## Three Hardcore Differentiators

1. **Schema‑Aware Anomaly Injection Detection**  
   The engine doesn’t just strip nulls—it compares each row against a user‑defined schema (JSON Schema or Avro) and flags rows that violate type constraints, outlier ranges, or regex patterns. It can even auto‑repair common violations (e.g., coercing `"N/A"` to `NULL` or trimming whitespace) while logging each mutation for audit trails.

2. **Multi‑Locale Encoding Normalizer**  
   Most CSV tools choke on mixed character sets (UTF‑8 BOM, Latin‑1, Shift JIS). cleancsvai scans the first 1,024 bytes to detect encoding, then normalizes everything to UTF‑8 without data loss. It also handles BOM stripping, line‑ending normalization (CRLF → LF), and illegal byte sequence removal—all in a single pass.

3. **Streaming Deduplication with Fuzzy Matching**  
   Unlike naive `drop_duplicates()`, cleancsvai uses Levenshtein distance and token‑set ratio to identify near‑duplicates (e.g., `"Acme Corp"` vs `"Acme Corporation"`). You define a similarity threshold, and the engine consolidates duplicates in‑stream, emitting only the canonical row per cluster. No memory blow‑up on 10M‑row files.

## Three Professional Workflows

### 1. Schema‑Constrained Bulk Cleanup
```bash
cleancsvai clean \
  --input sales_export.csv \
  --output sales_clean.csv \
  --schema schema.json \
  --repair-mode auto \
  --log-level verbose
```
- `schema.json` defines allowed types, ranges, and required fields.  
- `--repair-mode auto` attempts coercion (e.g., string `"1,234.56"` → float `1234.56`).  
- The tool writes a `cleancsvai_repair.log` with every transformed cell and its original value.

### 2. Multi‑File Encoding Audit & Normalization
```bash
cleancsvai normalize \
  --source-dir ./raw_data/ \
  --dest-dir ./normalized/ \
  --target-encoding utf-8 \
  --strip-bom \
  --line-endings lf
```
- Recursively processes all `.csv` files in `raw_data/`.  
- Detects encoding per file (BOM, `charset=` hints, byte pattern).  
- Converts to clean UTF‑8, strips BOM, and normalizes line endings.  
- Outputs a `normalization_report.json` with file‑level encoding metadata.

### 3. Streaming Fuzzy Deduplication with Cluster Report
```bash
cleancsvai dedupe \
  --input leads.csv \
  --output leads_deduped.csv \
  --key "company_name" \
  --threshold 0.85 \
  --cluster-report clusters.json
```
- Compares `company_name` values using token‑set ratio.  
- Groups duplicates into clusters; writes the canonical row (most complete) to output.  
- `clusters.json` maps each canonical row to its group members—critical for CRM dedupe audits.

## Two Industry Best Practices

- **Always run a dry‑run before destructive operations.**  
  Use `--dry-run` flag with `clean` or `dedupe` to preview row counts affected, repair actions, or duplicate clusters—without modifying your source file. This is non‑negotiable for production pipelines.

- **Pin a schema version to your data pipeline configuration.**  
  Store your `schema.json` in version control alongside your ETL code. When your data contract changes, update the schema file and re‑run cleancsvai. This prevents silent schema drift from corrupting downstream models or dashboards.
