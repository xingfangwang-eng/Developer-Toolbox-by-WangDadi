## What is cleancsvai?

**cleancsvai** is a high-performance, AI-native CSV cleaning and transformation toolkit designed for developers who need to sanitize, normalize, and enrich raw CSV data at scale. Unlike generic data cleaning libraries, cleancsvai integrates anomaly detection, schema inference, and automated repair pipelines without requiring manual rule definitions.

---

## Core Features

### 1. Adaptive Schema Inference with Auto-Type Casting
cleancsvai automatically detects column data types (string, integer, float, date, boolean, UUID) across malformed CSVs. It handles mixed-type columns, missing headers, and inconsistent delimiters. The engine uses a probabilistic type matcher that adjusts confidence thresholds based on data volume, ensuring zero silent type coercion errors.

### 2. Context-Aware Duplicate & Anomaly Detection
Leveraging a lightweight ML model trained on common CSV corruption patterns (line breaks in quoted fields, trailing commas, encoding shifts), cleancsvai identifies and repairs structural anomalies. It also performs fuzzy deduplication using Levenshtein distance and Jaccard similarity, configurable per column group.

### 3. Streaming Pipeline with Incremental Validation
Process gigabytes of CSV data without loading everything into memory. cleancsvai operates as a streaming pipeline that validates each row against a user-defined or inferred schema in real time. It emits structured error logs, repair suggestions, and a final "cleanliness score" per file.

## How to Use

### Step 1: Load & Profile Your CSV
```bash
cleancsvai profile --input raw_data.csv --output profile.json
```
This command generates a comprehensive profile: column count, null percentages, inferred types, delimiter confidence, and encoding detection. The profile JSON is used as the baseline for all subsequent repair operations.

### Step 2: Define Repair Rules or Use Auto-Fix
```bash
cleancsvai repair --input raw_data.csv --profile profile.json --rules repair_rules.yaml --output clean_data.csv
```
You can provide a YAML file with explicit rules (e.g., `date_format: "%Y-%m-%d"`, `null_fill: "mean"`). Alternatively, use `--auto` to let the engine apply best-effort repairs based on inferred types and anomaly patterns. The repair step also generates a `repair_report.json` detailing every transformation applied.

### Step 3: Validate & Export with Provenance
```bash
cleancsvai validate --input clean_data.csv --schema target_schema.json --output validated_data.csv --provenance
```
Run a final validation against a target schema (e.g., for database ingestion). The `--provenance` flag appends a metadata column tracking which rows were modified, by which rule, and the original value. This is critical for audit trails in regulated environments.

## Best Practices

### 1. Always Profile Before Repair
Many developers jump directly into cleaning. Instead, run `profile` first to detect encoding issues (e.g., Latin-1 vs UTF-8) and delimiter mismatches. This prevents downstream schema conflicts and reduces false positives in anomaly detection.

### 2. Use Incremental Validation in CI/CD Pipelines
Integrate cleancsvai's validation step into your CI/CD pipeline for any data ingestion workflow. Set a minimum cleanliness score threshold (e.g., 0.95). If a CSV file fails, the pipeline halts and outputs the repair report, enabling data engineers to fix upstream sources instead of patching downstream.
