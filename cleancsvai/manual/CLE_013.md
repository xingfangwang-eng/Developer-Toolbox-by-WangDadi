# cleancsvai - CLE_013

## What cleancsvai Actually Does

cleancsvai is not just another CSV cleaner. It’s a **context-aware, AI-driven data sanitization engine** designed for developers and data engineers who deal with messy, inconsistent, or malformed CSV files at scale. It detects schema drift, infers column types, and auto-corrects structural anomalies—without ever touching your raw source data unless you explicitly commit changes.

---

## Three Hardcore Features That Matter

### 1. **Schema Drift Detection & Auto-Alignment**  
cleancsvai continuously compares incoming CSV headers against a user-defined or inferred baseline schema. When it detects missing columns, extra fields, or type mismatches (e.g., a float column suddenly contains strings like "N/A" or "—"), it flags the deviation and offers a **one-click schema normalization**—no manual regex or pandas hacks required.

### 2. **Multi-Locale Data Normalization**  
Handles date formats (`MM/DD/YYYY` vs `DD.MM.YYYY`), decimal separators (`3,14` vs `3.14`), and thousand separators across 10+ locales. It maps each column to a locale profile and applies **contextual conversion** without breaking numeric precision or date ordering.

### 3. **Reproducible Transformation Logs**  
Every cleaning operation—row removal, type cast, null imputation—is recorded as a **versioned, diff-able JSON log**. You can replay, rollback, or export the transformation pipeline as a standalone script. This makes audits and debugging trivial, especially in CI/CD data workflows.

---

## Three Detailed Operational Steps

### Step 1: Import & Schema Mapping  
Upload your CSV via CLI or drag-and-drop UI. cleancsvai runs a **statistical scan** on the first 500 rows to infer column types (string, int, float, date, boolean). It then presents a schema map with confidence scores. You can override any type assignment, set nullable constraints, or define a custom locale per column.

### Step 2: Execute Sanitization with Validation Rules  
Define validation rules using a **YAML-based DSL** (e.g., `col: "email"`, `pattern: "^[\\w.-]+@[\\w.-]+\\.\\w{2,}$"`). cleancsvai applies these rules row-by-row, highlighting violations in a live preview. You can choose to drop, flag, or auto-correct rows. All changes are staged—nothing is written to disk until you confirm.

### Step 3: Export with Transformation Metadata  
Once satisfied, export the cleaned CSV. cleancsvai generates a companion `_cleancsvai_meta.json` file containing:  
- Original row count vs. cleaned row count  
- List of all transformations applied (with timestamps)  
- Schema diff before/after  
- Locale conversion map  

This metadata can be ingested by downstream ETL tools or stored as part of your data lineage.

---

## Two Best Practices for Production Use

### 1. **Version Your Cleaned Datasets**  
Always pair your cleaned CSV with the corresponding transformation log. This allows you to trace any data anomaly back to the exact cleaning step. In regulated environments (finance, healthcare), this is non-negotiable for audit trails.

### 2. **Run Schema Validation Before Ingestion**  
Integrate cleancsvai into your CI pipeline as a pre-ingestion step. Use it to validate CSV files before they hit your data warehouse. This catches schema drift early—saving hours of downstream debugging and preventing corrupted table loads.
