## What is cleancsvai?

cleancsvai is a deterministic + probabilistic hybrid pipeline designed for developers and data engineers who are tired of writing one-off Python scripts to handle ragged CSV files. It doesn't just strip quotes or fix line breaks—it understands column semantics, detects encoding drift, and auto-patches structural anomalies without losing data integrity. Think of it as a linter + formatter + transformer for your tabular data, but with an AI layer that learns your schema preferences over time.

---

## Core Features (Hardcore)

### 1. Semantic Column Auto-Detection & Type Coercion
cleancsvai doesn't rely on naive header sniffing. It uses a lightweight NLP model to infer column intent (e.g., "DOB" → date, "Amt" → currency) and applies appropriate type coercion—even when headers are missing or garbled. It handles mixed-format dates (`01/02/2023`, `2023-01-02`, `Jan 2, 2023`) by building a unified timestamp representation per column.

### 2. Multi-Encoding Recovery with Byte-Level Heuristics
Many CSV tools fail on files with BOM markers, mixed UTF-8/16, or Latin-1 encoded fields. cleancsvai performs a byte-level scan using a sliding window entropy check, then attempts automatic re-encoding via chardet + fallback heuristics. If a field contains binary garbage, it flags it with a confidence score rather than silently dropping it.

### 3. Structural Fuzz Repair for Ragged Rows
When rows have inconsistent column counts (e.g., quoted commas inside unescaped fields, line breaks inside cells), cleancsvai applies a recursive backtracking parser that re-aligns fields based on expected schema from the first 10 valid rows. It can even split concatenated columns or merge split rows using Levenshtein distance on cell values.

## How to Use (Detailed Steps)

### Step 1: Load & Profile Your CSV
```bash
cleancsvai profile --input dirty_sales.csv --output profile.json
```
This generates a JSON report showing: detected encoding, column count variances, null/sparse column ratios, and potential delimiter conflicts (e.g., semicolons inside quoted fields). Use `--strict` to fail on rows with >10% missing cells.

### Step 2: Apply Structural Fixes with Custom Rules
```python
from cleancsvai import Cleaner

cleaner = Cleaner(
    schema_hints={"date": ["%Y-%m-%d", "%d/%m/%Y"]},
    fix_ragged_rows=True,
    max_field_split_depth=2
)
df = cleaner.transform("dirty_sales.csv", output_format="pandas")
```
The cleaner will attempt to parse date columns with your hints first, then fall back to the AI model. If a row has 12 columns instead of 10, it tries to split or merge based on adjacent cell similarity.

### Step 3: Export with Validation Hooks
```bash
cleancsvai export --input cleaned.parquet --output final.csv --validate-schema schema.yaml --generate-report
```
This runs a final schema validation against your YAML definition (column names, types, constraints). Any violating rows are written to a separate `errors.csv` with an error code and original line number. The `--generate-report` flag produces an HTML dashboard showing before/after stats.

## Best Practices (Industry-Proven)

### Practice 1: Always Profile Before Cleaning
Don't blindly apply transformations. Run `cleancsvai profile` first to understand the shape of your mess. In production pipelines, we recommend caching the profile JSON and using it to parameterize the cleaner—this prevents over-fitting to one file's quirks and keeps your cleaning logic reproducible across data dumps.

### Practice 2: Use Schema Hints for High-Cardinality Columns
If you're cleaning financial or medical CSVs, provide explicit schema hints in a YAML file. For example, forcing `amount` to `Decimal(10,2)` and `zip_code` to `string` prevents the AI from inferring `zip` as an integer (which would drop leading zeros). This is critical when integrating with databases that enforce strict types.
