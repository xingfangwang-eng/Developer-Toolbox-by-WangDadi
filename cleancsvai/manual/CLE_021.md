## What is cleancsvai?

**cleancsvai** is a high-performance, CLI-first data cleaning engine built for developers and data engineers who need to sanitize, validate, and transform messy CSV files at scale—without the overhead of a full ETL pipeline. It leverages intelligent pattern detection and rule-based automation to eliminate manual data scrubbing, making it ideal for CI/CD data pipelines, batch processing, and ad-hoc data audits.

---

## Core Features

### 1. **Adaptive Schema Inference & Type Coercion**  
cleancsvai automatically detects column data types (dates, floats, integers, booleans, mixed types) and applies type coercion with fallback logging. It handles malformed rows gracefully, flagging them without breaking the pipeline—critical for production-grade data ingestion.

### 2. **Fuzzy Duplicate Detection & Deduplication Engine**  
Beyond exact-match deduplication, cleancsvai uses Levenshtein distance and token-based fuzzy matching to identify near-duplicates (e.g., "John Doe" vs "Jon Doe"). You can set custom threshold scores and apply conditional deduplication rules per column group.

### 3. **Regex-Based Column Normalization & Enrichment**  
Define custom regex patterns to normalize inconsistent formats (phone numbers, ZIP codes, email addresses) across thousands of rows in seconds. cleancsvai also supports lookup-based enrichment: map raw values to canonical labels (e.g., "NY" → "New York") using an embedded key-value store.

## How to Use

### Step 1: Profile Your CSV with Schema Detection  
Run the profiling command to generate a schema report:  
```bash
cleancsvai profile --input raw_data.csv --output schema_report.json
```  
This scans the first 10,000 rows (configurable) to infer column types, null ratios, and value distributions. The JSON report is used to validate the entire dataset before cleaning.

### Step 2: Apply Cleaning Rules via Configuration File  
Create a `cleaning_rules.yaml` file to define transformations:  
```yaml
rules:
  - column: "phone"
    normalize: "regex"
    pattern: "^(\+?1)?[-.\s]?\(?(\d{3})\)?[-.\s]?(\d{3})[-.\s]?(\d{4})$"
    replacement: "+1$2$3$4"
  - column: "email"
    validate: "email_format"
    action: "flag"
  - deduplicate:
      columns: ["name", "address"]
      algorithm: "fuzzy"
      threshold: 0.85
```  
Then execute:  
```bash
cleancsvai clean --input raw_data.csv --rules cleaning_rules.yaml --output cleaned_data.csv
```

### Step 3: Audit & Export Cleaned Data with Logging  
After cleaning, generate an audit trail:  
```bash
cleancsvai audit --input cleaned_data.csv --log audit_log.json
```  
This outputs a JSON log containing:  
- Number of rows removed (exact/fuzzy duplicates)  
- Number of cells normalized  
- Rows flagged for manual review  
- Schema drift warnings (if any)  

You can then export the cleaned dataset as CSV, JSON, or Parquet for downstream consumption.

## Best Practices

### 1. **Always Profile Before Cleaning**  
Running `profile` before `clean` prevents silent data corruption. It reveals hidden issues like mixed types in a single column (e.g., "123" and "123.0") that would otherwise break downstream analytics. Set the sample size to at least 10% of your dataset for reliable schema detection.

### 2. **Layer Deduplication After Normalization**  
Perform column-level normalization (e.g., standardizing dates or phone numbers) *before* fuzzy deduplication. This dramatically reduces false positives and improves match accuracy. For high-cardinality datasets, index your deduplication columns first using `--preindex` to accelerate processing.
