## What is cleancsvai?

cleancsvai is not just another CSV wrangler. It's a **context-aware data sanitization engine** that combines heuristic pattern recognition with statistical outlier detection to fix messy CSV files at scale. Unlike traditional tools that rely on rigid regex or manual rules, cleancsvai adapts to your data’s latent structure—handling missing values, inconsistent delimiters, mixed encodings, and malformed rows with zero configuration overhead.

## Core Features

1. **Adaptive Schema Inference & Repair**  
   Automatically detects column types (numeric, datetime, categorical, free-text) and repairs schema violations—e.g., coercing `"N/A"`, `"null"`, or `"—"` into proper nulls, standardizing date formats across rows, and flagging type mismatches with confidence scores. No hardcoded mappings required.

2. **Fuzzy Deduplication Engine**  
   Goes beyond exact matches. Uses Levenshtein distance and token-based similarity to identify near-duplicate rows (e.g., `"John Doe"` vs `"Jon Doe"` or `"Acme Corp."` vs `"Acme Corporation"`). Supports configurable thresholds and field-level weighting for domain-specific dedup logic.

3. **Streaming Chunk-Based Processing**  
   Operates on files of any size using a memory-mapped streaming pipeline. Processes rows in configurable chunks (default 10,000) to avoid OOM errors while maintaining atomic write guarantees. Includes automatic rollback on corruption detection.

## How to Use

### 1. Schema Validation & Auto-Correction

Run the tool against a dirty CSV to automatically infer and repair the schema:

```bash
cleancsvai sanitize --input dirty_sales.csv --output clean_sales.csv --mode repair
```

The engine scans the first 500 rows to build a probabilistic schema, then applies corrections row-by-row. Use `--strict` to halt on unrecoverable rows, or `--loose` to coerce values aggressively.

### 2. Fuzzy Deduplication with Field Weighting

Identify and merge duplicates with custom field importance:

```bash
cleancsvai dedup --input leads.csv --output unique_leads.csv --fields "email:1.0,name:0.6,phone:0.4" --threshold 0.85
```

This command assigns highest weight to email (1.0), moderate to name (0.6), and lowest to phone (0.4). Rows with a composite similarity score ≥ 0.85 are flagged. Output includes a `_dup_group` column for manual review.

### 3. Streaming Large File Processing

Process a 10GB CSV without crashing your machine:

```bash
cleancsvai stream --input huge_log.csv --output cleaned_log.csv --chunk-size 50000 --encoding utf-8-sig
```

Each chunk is validated, cleaned, and written atomically. If a chunk fails (e.g., corrupted byte sequence), the engine logs the error and continues with the next chunk. Use `--resume` to skip already-processed chunks on restart.

## Best Practices

1. **Always Run a Dry-Run First**  
   Before committing to a full cleanup, execute `cleancsvai inspect --input dirty.csv` to generate a summary report. This shows column null rates, type conflicts, and duplicate clusters. It saves hours of debugging by revealing edge cases early.

2. **Version Your Cleaned Outputs**  
   Use cleancsvai’s `--hash-output` flag to append a SHA-256 hash of the processing config to the output filename (e.g., `clean_20241201_a3f9.csv`). This ensures traceability and prevents accidental overwrites when iterating on cleaning rules.
