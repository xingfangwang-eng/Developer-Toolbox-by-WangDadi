# cleancsvai - CLE_017

## Overview

**cleancsvai** is not just another CSV cleaner—it's a **semantic data hygiene agent** designed for developers who hate wrestling with malformed CSVs, inconsistent delimiters, and encoding nightmares. Built for the modern data pipeline, it automates schema inference, outlier detection, and bulk normalization at scale.

## Hardcore Features

### 1. **Adaptive Schema Inference with Type Coercion**
No more manual column mapping. cleancsvai scans your CSV headers and data rows, then infers the most probable data types (int64, float64, datetime, boolean, string) using a lightweight ML heuristic. It automatically coerces mixed-type columns (e.g., `"N/A"`, `"null"`, `"2024-01-01"`) into a unified schema, flagging ambiguous cells for review.

### 2. **Multi-Encoding Auto-Detection & Transcoding**
Stop guessing between UTF-8, ISO-8859-1, Windows-1252, or Shift-JIS. cleancsvai uses a **byte-level statistical detector** (based on character frequency and BOM markers) to detect the encoding of your file. It then transcodes to UTF-8 with optional fallback strategies (e.g., replace, ignore, or escape malformed bytes). This is a lifesaver for legacy system exports.

### 3. **Delimiter-Agnostic Parsing with Escape Heuristics**
Whether your file uses commas, semicolons, tabs, or pipes—even with quoted fields containing embedded delimiters—cleancsvai's parser dynamically detects the delimiter using a **frequency-based voting algorithm**. It also handles escaped quotes (`""` or `\"`) and line breaks inside quoted fields, so you don't have to pre-process your data.

## Step-by-Step Usage

### Step 1: Load & Validate the File
```bash
cleancsvai --input ./messy_data.csv
```
The CLI immediately prints a summary: detected encoding, delimiter, row count, and any initial structural issues (e.g., inconsistent column count). If the file is too large, it streams the first 1000 rows for preview.

### Step 2: Run Schema Inference & Auto-Clean
```bash
cleancsvai --input ./messy_data.csv --output ./cleaned_data.csv --infer-schema --drop-duplicates
```
Behind the scenes, cleancsvai:
- Parses every column, detecting mixed types and applying type coercion.
- Removes exact duplicate rows (configurable via `--drop-duplicates`).
- Strips leading/trailing whitespace from all string cells.
- Normalizes date formats to ISO 8601 (`YYYY-MM-DD`).

### Step 3: Export with Audit Trail
```bash
cleancsvai --input ./messy_data.csv --output ./cleaned_data.csv --audit ./audit_report.json
```
The audit report logs every transformation applied: which rows were dropped, which cells were coerced, and any encoding fallbacks used. You can pipe this into your CI/CD pipeline for data quality monitoring.

## Industry Best Practices

### 1. **Always Run a Dry Run Before Production**
Use `--dry-run` to simulate cleaning without writing output. This prints a diff of changes (rows removed, types changed, encoding modifications) so you can catch unexpected data loss before it hits your database.

### 2. **Integrate cleancsvai as a Pre-Processing Step in Your ETL Pipeline**
Instead of manually cleaning CSVs before ingestion, add `cleancsvai` as a shell command in your Airflow or Prefect DAG. This ensures every file entering your data warehouse is structurally sound, reducing downstream failures in SQL joins or ML model training.
