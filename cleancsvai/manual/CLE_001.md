# cleancsvai - CLE_001

## Overview

**cleancsvai** is a production-grade CLI and web-based tool engineered for developers, data engineers, and analysts who need to strip noise from messy CSV files, infer column types, and standardize datasets without writing glue code. It combines heuristic pattern detection with lightweight ML models to deliver deterministic cleaning pipelines that scale from 100-row test files to multi-million row dumps.

## Core Features

### 1. Schema-Aware Anomaly Rectification
The engine parses each column against an inferred or user-supplied schema (e.g., `ISO8601 datetime`, `E.164 phone`, `ISO 4217 currency`). Rows violating the schema are flagged, auto-corrected where possible (e.g., trimming whitespace, normalizing date formats), or quarantined into a separate error log. This eliminates silent data corruption that plagues regex-only approaches.

### 2. Fuzzy Deduplication with Levenshtein & Phonetic Fingerprinting
Beyond exact duplicate removal, cleancsvai uses configurable Levenshtein distance thresholds and Soundex/Double Metaphone hashing to catch near-duplicates (e.g., “John Doe” vs “Jon Doe”). The dedup engine outputs a merge report so you can audit false positives before committing.

### 3. Multi-Locale Encoding & Delimiter Auto-Detection
The tool scans raw byte sequences to detect character encoding (UTF-8, Latin-1, Shift-JIS, etc.) and delimiter (comma, tab, semicolon, pipe) before any parsing. It then transcodes the entire file to a uniform UTF-8 BOM with consistent newline normalization (CRLF → LF), preventing “mojibake” and broken field boundaries in cross-regional datasets.

## How to Use

### Step 1: Validate & Profile Raw Input
Run `cleancsvai profile --input messy_data.csv --output profile.json` to generate a JSON report containing: column count, row count, null percentage per column, detected data types, and delimiter confidence scores. Review the report to decide which cleaning passes to enable.

### Step 2: Execute Targeted Cleaning Pipeline
Invoke the cleaning engine with explicit flags:
```bash
cleancsvai clean \
  --input messy_data.csv \
  --output clean_data.csv \
  --dedup-threshold 0.92 \
  --normalize-dates "%Y-%m-%d" \
  --strip-whitespace \
  --quarantine errors.csv
```
The engine will stream rows through each filter (whitespace removal → date normalization → dedup → type coercion) and write the cleaned dataset. Non-recoverable rows land in `errors.csv` with a reason code.

### Step 3: Audit & Export Transformed Schema
After cleaning, run `cleancsvai schema --input clean_data.csv --export schema.yaml` to produce a YAML schema definition. This file can be committed to your repo or used as input for ETL pipelines (e.g., dbt, Airbyte). Validate the schema against your downstream system’s expectations by running `cleancsvai validate --input clean_data.csv --schema schema.yaml`.

## Best Practices

### 1. Always Profile Before Cleaning
Never run a cleaning pass blindly. Use the `profile` command first to understand null patterns, encoding issues, and outlier values. This prevents over-correction (e.g., converting a valid “N/A” string to a null) and lets you tune thresholds per column.

### 2. Quarantine Aggressively, Recover Selectively
Set the `--quarantine` flag on every cleaning run. Even if you think your data is clean, edge cases (e.g., mixed-encoding rows, malformed JSON in a CSV field) will surface. After the run, inspect the quarantine file, fix the source of the problem, and re-run only the quarantined rows through the pipeline.
