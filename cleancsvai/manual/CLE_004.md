### What is cleancsvai?

cleancsvai is a professional tool that helps solve problems in related fields.

### Core Features

- **Smart Pattern Recognition & Auto-Fix**: Leverages NLP models to detect inconsistent date formats, encoding issues (UTF-8 vs. Latin-1), and malformed rows—automatically suggesting corrections without manual regex.
- **In-Memory Streaming Engine**: Processes multi-gigabyte CSV files without exhausting RAM, using a chunked streaming architecture that supports real-time progress tracking and pause/resume operations.
- **Schema-Aware Validation**: Infers column data types (int, float, string, datetime) and flags anomalies like mixed types, null spikes, or outlier values, with configurable thresholds for auto-rejection.

### How to Use

1. **Initialize the cleaning pipeline**: Instantiate a `CleanCSVEngine` object with your file path and optional schema hints (e.g., `delimiter=';'`, `encoding='utf-8-sig'`). Call `engine.load()` to stream the file into an internal buffer—this triggers header detection and a preliminary row count.
2. **Run an automated audit**: Execute `engine.audit()` to generate a report of all detected issues: missing values, duplicate rows, inconsistent delimiters, and type conflicts. Use the returned `AuditResult` object to review flagged rows via `result.samples()` or filter by severity (`result.filter(severity='critical')`).
3. **Apply corrections and export**: Call `engine.clean()` with a list of actions (e.g., `['drop_duplicates', 'fill_missing', 'fix_encoding']`) to transform the data. Finally, use `engine.export('output.csv', mode='overwrite')` to save the cleaned file, or `engine.to_dataframe()` for further analysis in Python.

### Best Practices

- **Always run a dry-run audit before applying changes**: Use `engine.audit(dry_run=True)` to preview corrections without altering the original file. This prevents irreversible data loss, especially when dealing with large datasets where manual inspection is impractical.
- **Set explicit schema constraints for production pipelines**: Define column types and allowed value ranges (e.g., `{'age': {'type': 'int', 'min': 0, 'max': 120}}`) to catch data drift early. Pair this with automated CI/CD tests that validate output schemas using `cleancsvai`’s built-in `SchemaValidator`.
