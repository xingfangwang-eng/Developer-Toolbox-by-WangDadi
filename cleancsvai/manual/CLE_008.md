# cleancsvai - CLE_008

## Overview

cleancsvai is a purpose-built data preprocessing engine that automates the detection and repair of malformed CSV structures, encoding mismatches, and inconsistent delimiters—without writing a single regex. Designed for developers who ingest raw datasets from heterogeneous sources (legacy exports, scraped tables, enterprise dumps), this tool reduces manual cleaning time by up to 90% while preserving data integrity.

## Core Features

### 1. Adaptive Schema Inference & Anomaly Repair
cleancsvai dynamically analyzes column types, null patterns, and value distributions across the entire dataset. It identifies and corrects common structural issues—misaligned headers, mixed delimiter usage (e.g., comma vs. semicolon), and truncated rows—by applying a probabilistic repair model that respects the original data lineage.

### 2. Multi-Encoding Auto-Detection with Lossless Normalization
The engine scans byte-level signatures to detect encoding conflicts (UTF‑8, ISO‑8859‑1, Shift‑JIS, etc.) and transparently normalizes all text to UTF‑8 while preserving special characters, diacritics, and multi-byte symbols. No more garbled columns from mismatched charsets.

### 3. Inline Deduplication & Fuzzy Matching
Beyond exact duplicate removal, cleancsvai employs Levenshtein distance and phonetic similarity algorithms to identify near-duplicate records (e.g., “John Doe” vs. “Jon Doe”). Users can set configurable thresholds and choose to merge, flag, or discard ambiguous matches—ideal for cleaning CRM exports or survey data.

## How to Use

1. **Validate & Sanitize a Single CSV**
   Run the CLI command with the input file and let the engine analyze structure, encoding, and anomalies:
   ```bash
   cleancsvai --input raw_export.csv --output cleaned.csv --repair
   ```
   The tool will output a detailed repair log (e.g., “Fixed 12 encoding mismatches, 3 truncated rows, and 1 mixed delimiter line”).

2. **Batch Process Multiple Files with Custom Rules**
   Create a `.cleancsvai` config file to define per-column constraints (e.g., date format, numeric range, allowed values). Then execute:
   ```bash
   cleancsvai --batch ./data/*.csv --config ./rules.yaml --parallel
   ```
   This runs parallel workers per file, applying your domain‑specific rules and generating a unified summary report.

3. **Integrate into a Data Pipeline via API**
   For programmatic use, POST a CSV payload to the REST endpoint:
   ```bash
   curl -X POST https://api.cleancsvai.com/v1/clean \
        -H "Content-Type: multipart/form-data" \
        -F "file=@messy_data.csv" \
        -F "options={\"dedup\":true,\"encoding\":\"auto\",\"repair_headers\":true}"
   ```
   The response returns the cleaned CSV as a stream, along with a JSON metadata block detailing every transformation applied.

## Best Practices

- **Always run a dry‑run first** – Use the `--dry-run` flag to preview the repair plan without modifying the original file. This helps catch unexpected transformations before they corrupt downstream pipelines.
- **Leverage the rule‑based pre‑processor** – For datasets with known domain constraints (e.g., ISO country codes, valid email formats), define a YAML rule file. This ensures that cleancsvai’s heuristic repairs stay within your business logic boundaries, preventing false positives.
