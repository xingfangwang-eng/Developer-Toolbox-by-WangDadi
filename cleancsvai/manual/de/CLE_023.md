# cleancsvai - CLE_023

## Überblick

cleancsvai ist ein spezialisiertes Kommandozeilen-Tool und eine Python-Bibliothek, die inkonsistente, duplizierte oder fehlerhafte CSV-Dateien automatisiert erkennt und korrigiert. Es richtet sich an DevOps-Ingenieure, Datenanalysten und Backend-Entwickler, die täglich mit unstrukturierten Rohdaten arbeiten und keine manuelle Datenbereinigung mehr durchführen möchten.

## Hardcore-Funktionen

### 1. **Schema-Inferenz & Typ-Korrektur (Schema-Aware Repair)**
Das Tool analysiert automatisch den Datentyp jeder Spalte (Integer, Float, Datum, String) und korrigiert inkonsistente Werte – z. B. `"01.02.2023"` vs. `"2023-01-02"` – ohne dass ein vordefiniertes Schema übergeben werden muss. Fehlertoleranz für gemischte Formate, wie sie in realen Datensätzen häufig vorkommen, ist integriert.

### 2. **Fuzzy-Duplikat-Erkennung mit Levenshtein-Distanz**
Statt exakter Duplikate erkennt cleancsvai semantisch ähnliche Zeilen (z. B. `"Max Mustermann"` vs. `"Max Musterman"`) mittels eines konfigurierbaren Ähnlichkeitsschwellwerts. So werden Tippfehler oder OCR-Artefakte zuverlässig eliminiert, ohne dass echte Daten verloren gehen.

### 3. **Anomalie-Detektion auf Basis statistischer Ausreißer**
Das Tool berechnet für numerische Spalten Median, IQR und Standardabweichung und markiert Werte, die außerhalb des 1,5-fachen IQR liegen. Diese können entweder automatisch entfernt oder für eine manuelle Prüfung als separate CSV exportiert werden.

## Detaillierte Anwendungsschritte

### 1. **CSV validieren und Schema analysieren**
```bash
cleancsvai validate --input raw_data.csv --output schema_report.json
```
Dieser Befehl erzeugt einen JSON-Report, der für jede Spalte den erkannten Datentyp, die Anzahl der Nullwerte, die Häufigkeitsverteilung und etwaige Formatinkonsistenzen auflistet. So sehen Sie sofort, welche Spalten eine Bereinigung benötigen.

### 2. **Fuzzy-Duplikate entfernen und mergen**
```bash
cleancsvai deduplicate --input raw_data.csv --output clean_data.csv --threshold 0.85 --keep first
```
Hier wird jede Zeile mit jeder anderen auf Ähnlichkeit geprüft. Der Parameter `--threshold 0.85` bedeutet, dass Zeilen mit einer Levenshtein-Ähnlichkeit von ≥ 85 % als Duplikat gelten. Mit `--keep first` bleibt die erste gefundene Zeile erhalten, die restlichen werden entfernt.

### 3. **Statistische Ausreißer automatisch korrigieren**
```bash
cleancsvai outlier --input clean_data.csv --output final_data.csv --method iqr --action cap
```
Der Befehl identifiziert alle numerischen Ausreißer auf Basis des Interquartilsabstands und setzt diese auf den jeweiligen oberen bzw. unteren Whisker-Wert (`--action cap`). Alternativ können Sie `--action remove` verwenden, um die Zeilen komplett zu entfernen.

## Branchen-Best-Practices

### 1. **Immer eine Sicherungskopie der Rohdaten anlegen**
Bevor Sie cleancsvai auf Ihren Produktionsdaten ausführen, erstellen Sie einen Snapshot der Original-CSV. Das Tool überschreibt standardmäßig keine Dateien, sondern erzeugt eine neue Ausgabe. Dennoch empfehlen wir, die Rohdaten zu archivieren, um bei Bedarf einen vollständigen Audit-Trail zu haben.

### 2. **Bereinigungsschritte schrittweise ausführen und validieren**
Führen Sie die drei Schritte (Validieren → Deduplizieren → Ausreißer entfernen) nicht in einem Durchlauf aus. Validieren Sie zuerst das Schema, prüfen Sie die Duplikat-Erkennung anhand einer kleinen Stichprobe (z. B. `--sample 1000`) und korrigieren Sie dann die Ausreißer erst, wenn die Duplikat-Entfernung abgeschlossen ist. So vermeiden Sie Datenverluste durch falsch positive Erkennungen.
