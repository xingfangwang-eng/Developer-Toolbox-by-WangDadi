# cleancsvai - CLE_027

## Überblick

cleancsvai ist ein spezialisiertes CLI-Tool und eine Python-Bibliothek zur automatisierten Erkennung und Korrektur von Anomalien in CSV-Dateien. Es kombiniert regelbasierte Prüfungen mit maschinellem Lernen, um inkonsistente Datensätze, fehlerhafte Trennzeichen und Encoding-Probleme zu identifizieren – ohne manuelle Nachbearbeitung. Optimiert für Data Engineers, die täglich mit Rohdaten aus unterschiedlichsten Quellen arbeiten.

## Kernfunktionen

### 1. Automatische Schema-Erkennung und Typinferenz mit Fuzzy-Matching
Das Tool analysiert die ersten 1000 Zeilen einer CSV-Datei und erstellt ein dynamisches Schema. Es erkennt nicht nur numerische, datetime- und stringbasierte Spalten, sondern korrigiert auch Tippfehler in Spaltenüberschriften (z. B. `"Geburtsdatum"` → `"Geburtsdatum"`). Bei inkonsistenten Datentypen innerhalb einer Spalte (z. B. `"2023-01-01"` neben `"01/01/2023"`) wird automatisch ein einheitliches Format gewählt und konvertiert.

### 2. Multi-Encoding-Detektion und -Konvertierung
Statt auf UTF-8 zu vertrauen, scannt cleancsvai die Datei auf Bytesequenzen, die auf ISO-8859-1, Windows-1252, Shift-JIS oder andere Encodings hinweisen. Es erstellt einen Confidence-Score pro Encoding und konvertiert die Datei automatisch in UTF-8 mit korrekter BOM-Erkennung. Encoding-Fehler (z. B. `Ã¼` statt `ü`) werden protokolliert und korrigiert.

### 3. Anomalie-Clustering mit Outlier-Erkennung
Mithilfe eines einfachen Isolation-Forest-Modells (trainiert auf numerischen Spalten) werden Ausreißer in Datensätzen markiert. Gleichzeitig erkennt das Tool Duplikate auf Zeilenebene (exakt und fuzzy) sowie leere Zellen, die in kritischen Spalten (z. B. `ID`, `Email`) nicht toleriert werden. Ergebnisse werden als separate `_cleaned.csv` und `_anomalies.csv` ausgegeben.

## So verwenden Sie es

### Schritt 1: Installation und initiale Inspektion
```bash
pip install cleancsvai
cleancsvai inspect ./daten/rohdaten.csv --output-report
```
Der Befehl `inspect` erzeugt einen JSON-Report mit Metadaten: Zeilenanzahl, Spalten, erkannte Datentypen, Encoding-Vorschlag und eine Liste der ersten 10 Anomalien. Nutzen Sie `--output-report` um das Ergebnis als `rohdaten_report.json` zu speichern.

### Schritt 2: Automatisierte Bereinigung mit benutzerdefinierten Regeln
```bash
cleancsvai clean ./daten/rohdaten.csv \
  --schema ./schema.yaml \
  --encoding windows-1252 \
  --fill-missing "mean" \
  --drop-duplicates \
  --outliers-threshold 0.05
```
Hier geben Sie explizit das Encoding vor (`windows-1252`), füllen fehlende numerische Werte mit dem Mittelwert der Spalte (`--fill-missing "mean"`), entfernen exakte Duplikate und markieren Ausreißer mit einem Threshold von 5% (Isolation Forest). Das Schema in `schema.yaml` definiert Spaltentypen und erlaubte Wertebereiche.

### Schritt 3: Validierung und Export
```bash
cleancsvai validate ./daten/rohdaten_cleaned.csv \
  --checks "schema,encoding,duplicates" \
  --output-format csv \
  --export ./export/bereinigt.csv
```
Nach der Bereinigung validieren Sie das Ergebnis: Der Befehl prüft Schema-Konformität, Encoding-Konsistenz und Duplikatfreiheit. Bei Fehlern wird ein detaillierter Log ausgegeben. Mit `--output-format csv` exportieren Sie die bereinigte Datei in das gewünschte Format.

## Best Practices

### 1. Immer ein Backup der Rohdaten erstellen
Führen Sie `cleancsvai inspect` immer zuerst aus, bevor Sie `clean` anwenden. So haben Sie einen vollständigen Überblick über die Datenqualität, bevor Änderungen vorgenommen werden. Speichern Sie den Report als Referenz für nachfolgende Bereinigungsschritte.

### 2. Regelmäßige Schema-Definitionen für wiederkehrende Datenquellen
Wenn Sie regelmäßig CSV-Dateien aus derselben Quelle verarbeiten (z. B. monatliche Exporte aus einem CRM), definieren Sie ein festes Schema in einer YAML-Datei. Das reduziert die Wahrscheinlichkeit von Fehlinterpretationen und beschleunigt die Batch-Verarbeitung. Beispiel:
```yaml
columns:
  - name: "KundenID"
    type: string
    required: true
  - name: "Betrag"
    type: float
    min: 0.0
    max: 100000.0
```
