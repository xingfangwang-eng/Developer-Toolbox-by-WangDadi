# cleancsvai - CLE_022

## Überblick

cleancsvai ist ein spezialisiertes Tool zur automatisierten Bereinigung, Normalisierung und Strukturanalyse von CSV-Dateien. Es richtet sich an Data Engineers, DevOps-Ingenieure und SaaS-Entwickler, die regelmäßig mit inkonsistenten, duplizierten oder fehlerhaften Datensätzen arbeiten – und keine Zeit mit manuellem Regex-Tuning verschwenden wollen.

## Drei harte Kernfunktionen

### 1. **Automatische Schema-Erkennung & Typinferenz mit Konfliktresolution**
Das Tool analysiert jede Spalte auf Datentypen (Integer, Float, ISO-Datum, UUID, Boolesch) und erkennt Inkonsistenzen wie gemischte Formate oder NULL-Werte. Bei Konflikten werden automatisch Korrekturstrategien vorgeschlagen – z. B. `"2024-01-01"` vs. `"01.01.2024"` wird auf ISO-Norm vereinheitlicht, ohne dass der Nutzer manuell eingreifen muss.

### 2. **Fuzzy-Deduplizierung mit Levenshtein & Jaccard-Ähnlichkeit**
Statt exakter Duplikate erkennt cleancsvai auch semantisch identische Einträge (z. B. `"Max Mustermann"` vs. `"Mustermann, Max"`). Über konfigurierbare Ähnlichkeitsschwellen werden Cluster gebildet und zusammengeführt – ideal für CRM-Daten oder Adresslisten aus verschiedenen Quellen.

### 3. **Export mit automatischer Encoding-Erkennung & BOM-Handling**
Das Tool erkennt UTF-8, ISO-8859-1, Windows-1252 und andere Encodings. Beim Export wird automatisch das korrekte Byte Order Mark (BOM) gesetzt, falls das Zielsystem (z. B. Excel unter Windows) dies erfordert. Keine versteckten Sonderzeichen oder kaputten Umlaute mehr.

## Detaillierte Anwendungsschritte

### Schritt 1: Datenquellen validieren & Schema analysieren
```bash
cleancsvai analyze --input data/rohdaten.csv --delimiter auto
```
Das Tool gibt eine strukturierte JSON-Übersicht aus: Spaltennamen, erkannte Datentypen, Anzahl NULL-Werte, minimale/maximale Werte sowie Warnungen zu gemischten Formaten. Nutzen Sie diese Ausgabe als Grundlage für die nächste Konfiguration.

### Schritt 2: Bereinigungsregeln definieren & anwenden
Erstellen Sie eine Konfigurationsdatei `cleancsvai.yaml`:
```yaml
columns:
  email:
    normalize: lowercase
    pattern: '^[\w\.-]+@[\w\.-]+\.\w+$'
  datum:
    format: '%Y-%m-%d'
    fallback: '1970-01-01'
dedup:
  method: levenshtein
  threshold: 0.85
```
Führen Sie dann die Bereinigung aus:
```bash
cleancsvai clean --config cleancsvai.yaml --input data/rohdaten.csv --output data/clean.csv
```
Das Tool protokolliert jede Änderung in einer separaten Logdatei.

### Schritt 3: Qualitätsreport generieren & validieren
```bash
cleancsvai report --input data/clean.csv --metrics duplicates,missing,types
```
Erhalten Sie eine detaillierte Statistik: Anzahl korrigierter Zeilen, verbleibende NULL-Werte, Typkonflikte und Aufwandsabschätzung für manuelle Nacharbeit. Der Report kann als HTML oder Markdown exportiert werden.

## Best Practices

### 1. **Immer mit einer kleinen Stichprobe testen**
Laden Sie nicht gleich 10 GB Rohdaten. Nutzen Sie `--sample 1000`, um die Schema-Erkennung und Deduplizierung zuerst auf einer repräsentativen Teilmenge zu validieren. So vermeiden Sie unerwartete Korrekturen, die später schwer rückgängig zu machen sind.

### 2. **Bereinigungsregeln versionieren**
Legen Sie Ihre `cleancsvai.yaml`-Konfiguration in ein Git-Repository. So können Sie Änderungen nachvollziehen, Regressionen vermeiden und das Setup in CI/CD-Pipelines (z. B. GitHub Actions) automatisieren. Ein `cleancsvai validate`-Befehl prüft die Konfiguration auf syntaktische Korrektheit.
