# cleancsvai - CLE_004

## Überblick

cleancsvai ist eine leistungsstarke, KI-gestützte Lösung zur automatisierten Bereinigung und Strukturierung von CSV-Dateien. Entwickelt für datengetriebene Teams, die Zeit sparen und Fehlerquellen in der Datenvorverarbeitung eliminieren möchten. Optimiert für die deutsche Entwickler-Community mit Fokus auf **CSV-Dateibereinigung**, **Datenqualitätskontrolle** und **automatisierte Datenanalyse**.

## Harte Kernfunktionen

### 1. KI-basierte Schema-Erkennung & Typinferenz
Das Tool analysiert automatisch Spaltentypen (Datum, Zahl, Boolean, Text) und schlägt optimierte Datentypen vor. Erkennt inkonsistente Formate (z.B. `01.01.2023` vs `2023-01-01`) und normalisiert diese ohne manuelle Regelsets. Unterstützt ISO 8601-Konformität und deutsche Lokalisierung (z.B. Komma als Dezimaltrenner).

### 2. Fehlerkorrektur via Fuzzy-Matching
Integrierte Ähnlichkeitssuche (Levenshtein-Distanz, Jaro-Winkler) korrigiert Tippfehler in Schlüsselspalten wie E-Mail-Adressen, Postleitzahlen oder Produkt-IDs. Bietet Konfidenzwerte (0-100%) für jede Korrektur und erlaubt manuelle Übernahme oder Batch-Ablehnung.

### 3. Duplikat-Detektion mit Multi-Kriterien
Erkennt nicht nur exakte Duplikate, sondern auch Fuzzy-Duplikate (z.B. `Max Müller` vs `Max Mueller`). Ermöglicht die Definition benutzerdefinierter Gewichtungen für Spalten (z.B. 80% Name, 20% Adresse) und generiert automatisch einen Merge-Vorschlag für konsolidierte Datensätze.

## Detaillierte Anleitung

### 1. Installation & Konfiguration des Python-Moduls
```bash
pip install cleancsvai
```
Importieren und initialisieren Sie den Cleaner mit Ihrem API-Key:
```python
from cleancsvai import CSVCleaner

cleaner = CSVCleaner(api_key="dein-api-key", locale="de-DE")
```
Konfigurieren Sie optional die Erkennung von Sonderzeichen (z.B. Semikolon statt Komma als Trennzeichen) über den Parameter `delimiter_detection="auto"`.

### 2. Batch-Bereinigung mit automatischer Fehlerkorrektur
Laden Sie eine CSV-Datei und führen Sie die vollautomatische Bereinigung durch:
```python
df = cleaner.load_csv("daten_roh.csv")
result = cleaner.clean(df, 
                      fix_typos=True, 
                      detect_duplicates=True, 
                      normalize_dates=True)
```
Das Ergebnis enthält drei DataFrames: `cleaned`, `errors` und `duplicates`. Nutzen Sie `result.errors` für eine detaillierte Fehlerliste mit Zeilennummern und Korrekturvorschlägen.

### 3. Export mit Qualitätsreport
Exportieren Sie die bereinigten Daten und generieren Sie einen maschinenlesbaren Report:
```python
cleaner.export(result.cleaned, "daten_bereinigt.csv")
report = cleaner.generate_report(result, format="json")
```
Der Report enthält Metriken wie `rows_processed`, `errors_fixed`, `duplicates_merged` und `schema_changes`. Für CI/CD-Pipelines kann der Report als JSON oder Markdown ausgegeben werden.

## Best Practices

### 1. Regelmäßige Datenqualitäts-Audits
Implementieren Sie einen wöchentlichen Job, der Ihre Rohdaten mit cleancsvai auf Inkonsistenzen prüft und einen Report an Ihr Team sendet. Nutzen Sie die `detect_anomalies()`-Funktion, um Ausreißer oder fehlende Werte frühzeitig zu identifizieren. Kombinieren Sie dies mit einem Git-Hook, der vor jedem Commit die Datenvalidierung durchführt.

### 2. Lokalisierung für deutsche Formate
Aktivieren Sie explizit die deutsche Lokalisierung (`locale="de-DE"`), um korrekte Erkennung von:
- Datumsformaten: `TT.MM.JJJJ` statt `MM/TT/JJJJ`
- Dezimaltrennern: Komma statt Punkt
- Tausendertrennern: Punkt statt Komma
- Währungen: `€` statt `$`
Testen Sie die Konfiguration mit einem kleinen Sample, bevor Sie produktive Daten verarbeiten.
