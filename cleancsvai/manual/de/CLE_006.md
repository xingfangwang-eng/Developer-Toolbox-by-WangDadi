# cleancsvai - CLE_006

## Überblick

cleancsvai ist ein spezialisiertes Tool zur automatisierten Erkennung und Korrektur von Dateninkonsistenzen in CSV-Dateien. Es kombiniert regelbasierte Validierung mit maschinellem Lernen, um fehlerhafte Datensätze präzise zu identifizieren und zu bereinigen – ohne manuelle Nacharbeit.

## Kernfunktionen

### 1. Semantische Typinferenz mit Kontextanalyse
Das Tool erkennt automatisch Datentypen (Datum, Währung, Prozent, Telefonnummer etc.) und korrigiert Formatierungsfehler wie inkonsistente Datumsformate (`01.02.2024` vs `2024-02-01`) oder falsche Dezimaltrennzeichen. Dabei berücksichtigt es regionale Konfigurationen und konvertiert intelligent in ein einheitliches Zielformat.

### 2. Fuzzy-Duplikaterkennung mit Levenshtein-Distanz
Statt exakter Übereinstimmungen identifiziert cleancsvai ähnliche Einträge (z.B. `Müller GmbH` vs `Mueller GmbH` oder `Straße` vs `Strasse`) mittels adaptiver Schwellenwerte. Die Ergebnisse werden mit Konfidenzwerten ausgegeben, sodass Sie selbst entscheiden können, welche Dubletten zusammengeführt werden.

### 3. Schema-Validierung mit benutzerdefinierten Constraints
Definieren Sie komplexe Validierungsregeln direkt in der Konfigurationsdatei (JSON/YAML): Spaltenabhängigkeiten, Wertebereiche, Regex-Muster oder Cross-Field-Constraints. Beispiel: `if field_A = "aktiv" then field_B must not be null`. Verstöße werden detailliert protokolliert und optional automatisch korrigiert.

## So verwenden Sie es

### Schritt 1: CSV-Analyse mit automatischer Fehlererkennung
Führen Sie `cleancsvai analyze input.csv` aus. Das Tool scannt Ihre Datei auf häufige Probleme wie:
- Leere Zellen in Pflichtfeldern
- Inkonsistente Zeilenlängen (z.B. 12 statt 13 Spalten)
- Kodierungsfehler (UTF-8 vs ISO-8859-1)
- Unerwartete Sonderzeichen oder Escape-Probleme

Ergebnis: Ein JSON-Report mit detaillierten Fehlerstatistiken, betroffenen Zeilen und Korrekturvorschlägen.

### Schritt 2: Benutzerdefinierte Transformationsregeln anwenden
Erstellen Sie eine `cleancsvai.config.json` mit Ihren Regeln:
```json
{
  "columns": {
    "preis": {
      "type": "currency",
      "format": "EUR",
      "decimal_separator": "."
    },
    "datum": {
      "type": "date",
      "format": "YYYY-MM-DD"
    }
  },
  "cleaning": {
    "remove_duplicates": true,
    "fill_missing": "interpolate",
    "normalize_whitespace": true
  }
}
```
Führen Sie `cleancsvai clean input.csv -c config.json -o output.csv` aus.

### Schritt 3: Validierung und Export mit Audit-Trail
Nutzen Sie `cleancsvai validate output.csv --strict`, um die bereinigte Datei gegen Ihre Constraints zu prüfen. Bei erfolgreicher Validierung erhalten Sie:
- Eine vollständige Änderungshistorie (diff-ähnlich)
- Metadaten zur Datenqualität (Vollständigkeit, Konsistenz, Genauigkeit)
- Einen maschinenlesbaren Report für CI/CD-Pipelines

## Best Practices

### 1. Immer mit einer Sicherungskopie arbeiten
Vor der Bereinigung: `cleancsvai backup input.csv` erstellt automatisch einen Snapshot im `.cleancsvai_backups/`-Ordner. So können Sie bei unerwarteten Transformationen jederzeit zum Originalzustand zurückkehren – ohne manuelle Versionierung.

### 2. Regelmäßige Schema-Updates in CI/CD integrieren
Binden Sie cleancsvai als Pre-Commit-Hook in Ihre Pipeline ein: Bei jedem Push wird automatisch geprüft, ob neue CSV-Dateien den aktuellen Validierungsregeln entsprechen. Das verhindert, dass fehlerhafte Daten ins Repository gelangen, und hält Ihre Datenqualität konsistent.
