# cleancsvai - CLE_030

## Warum cleancsvai?

Jeder Entwickler kennt das Problem: Rohdaten aus CSV-Dateien sind voller Inkonsistenzen, Duplikate und Encoding-Fehler. Manuelle Bereinigung ist fehleranfällig und kostet wertvolle Entwicklungszeit. cleancsvai nutzt maschinelles Lernen, um diese Prozesse zu automatisieren – ohne dass Sie eine einzige Regex schreiben müssen.

## Drei Kernfunktionen, die den Unterschied machen

### 1. Adaptive Schema-Erkennung mit Typinferenz
Im Gegensatz zu statischen Parsern analysiert cleancsvai Ihre CSV-Datei auf semantischer Ebene. Das Tool erkennt automatisch Datumsformate (ISO 8601, US-Format, europäisches Format), Währungssymbole und regionale Trennzeichen. Bei gemischten Datentypen in einer Spalte erfolgt eine intelligente Vereinheitlichung – inklusive Vorschlag für die optimale Datenstruktur.

### 2. Fuzzy-Duplikat-Erkennung mit Levenshtein-Distanz
Standard-Deduplizierung erfasst nur exakte Duplikate. cleancsvai berechnet die Ähnlichkeit von Zeichenketten mittels eines gewichteten Levenshtein-Algorithmus. So erkennen Sie „Müller GmbH“ und „Mueller GmbH“ als Duplikate, ohne False Positives zu riskieren. Der Schwellenwert ist konfigurierbar – ideal für CRM-Importe oder Adressdatenbereinigung.

### 3. Encoding-Autokorrektur mit Fallback-Strategie
UTF-8? ISO-8859-1? Windows-1252? cleancsvai führt einen mehrstufigen Encoding-Detektionsprozess durch. Falls das Encoding nicht eindeutig ist, generiert das Tool eine Konflikttabelle und schlägt die wahrscheinlichste Kodierung vor – mit der Option, manuell einzugreifen.

## Professionelle Anleitung in drei Schritten

### Schritt 1: CSV-Validierung und Vorverarbeitung
Laden Sie Ihre Datei über die CLI oder REST-API:  
```bash
cleancsvai validate ./daten/export.csv --strict
```
Der Befehl `validate` prüft auf strukturelle Integrität: Zeilenanzahl, Spaltenkonsistenz und Zeilenumbrüche. Mit `--strict` werden Warnungen zu Fehlern hochgestuft. Die Ausgabe erfolgt als JSON-Report mit detaillierten Metriken.

### Schritt 2: Konfiguration der Reinigungsregeln
Erstellen Sie eine YAML-Konfigurationsdatei `clean_rules.yaml`:
```yaml
rules:
  - column: "email"
    deduplicate: true
    normalize: lowercase
  - column: "preis"
    parse_currency: true
    target_format: "float"
  - column: "datum"
    source_formats: ["DD.MM.YYYY", "YYYY-MM-DD"]
    target_format: "YYYY-MM-DD"
```
Führen Sie dann die Bereinigung durch:
```bash
cleancsvai clean ./daten/export.csv --config clean_rules.yaml --output ./daten/bereinigt.csv
```
Das Tool wendet die Regeln sequenziell an und protokolliert jede Transformation.

### Schritt 3: Qualitätskontrolle mit Diff-Reporting
Nach der Bereinigung generieren Sie einen Differenzbericht:
```bash
cleancsvai diff ./daten/export.csv ./daten/bereinigt.csv --format html
```
Der HTML-Report zeigt farblich hervorgehoben: gelöschte Zeilen, geänderte Werte und zusammengeführte Duplikate. Perfekt für Audits oder die Übergabe an Stakeholder.

## Zwei Best Practices aus der Praxis

### 1. Progressive Reinigung statt Big Bang
Führen Sie keine vollständige Bereinigung auf dem gesamten Datensatz durch. Starten Sie mit einer Stichprobe von 1000 Zeilen, validieren Sie das Ergebnis und iterieren Sie. cleancsvai unterstützt Sampling über den Parameter `--sample 1000`. So vermeiden Sie unerwartete Seiteneffekte bei Millionen von Zeilen.

### 2. Pipeline-Integration mit CI/CD
Nutzen Sie cleancsvai als Teil Ihrer Daten-Pipeline. Ein Beispiel für GitHub Actions:
```yaml
- name: CSV-Reinigung
  run: |
    cleancsvai clean ./input.csv \
      --config .cleancsvai/rules.yaml \
      --output ./clean/input.csv \
      --fail-on-error
```
Mit `--fail-on-error` bricht der Job bei kritischen Fehlern ab – ideal für automatisierte Datenimporte in Produktionssysteme.
