# cleancsvai - CLE_018

## Überblick

cleancsvai beseitigt manuelle Datenbereinigung und liefert reproduzierbare, regelbasierte CSV-Analyse – speziell optimiert für CI/CD-Pipelines und lokale Datenworkflows. Kein Excel-Gefrickel, keine manuellen Regex-Experimente mehr.

## 3 Hardcore-Features

1. **Adaptive Schema-Erkennung mit Typinferenz**  
   Automatische Erkennung von Datentypen (ISO-Datum, UUID, Geokoordinaten, Währung) inklusive Fehlertoleranz bei gemischten Formaten. Schemata werden als JSON-Schema exportiert und in `cleancsvai schema validate` wiederverwendet.

2. **Deterministische Duplikat-Resolution via Bloom-Filter**  
   Deduplizierung auf Basis von kryptografischen Hashes (SHA-256) in Kombination mit einem Bloom-Filter – minimiert False Positives und skaliert auf Millionen Zeilen mit konstantem RAM-Verbrauch.

3. **Inline-Transformationen mit Lua-Sandbox**  
   Erstelle benutzerdefinierte Transformationsregeln in Lua (isolierte Sandbox ohne Dateisystemzugriff). Beispiel: `row["price_netto"] = tonumber(row["brutto"]) / 1.19` für deutsche MwSt.-Berechnung.

## 3 Detaillierte Anwendungsschritte

### 1. CSV-Schema validieren & Typen korrigieren
```bash
cleancsvai schema detect --input sales_2025.csv --output schema.json
```
Ergebnis: Erkennung von Spalten wie `date` (ISO 8601), `amount` (float), `customer_id` (UUID). Korrigiere manuell im JSON, falls nötig – z. B. `"type": "date"` → `"type": "datetime"` für Millisekunden-Präzision.

### 2. Duplikate mit Bloom-Filter entfernen
```bash
cleancsvai dedup --input input.csv --output clean.csv --bloom-size 1048576 --hash-algo sha256
```
Parameter `--bloom-size` definiert die Filtergröße (1 MB für 1 Mio. Zeilen mit <0,1% False Positive). Nutze `--hash-algo` für Kollisionsresistenz. Überprüfe mit `--dry-run` vor der finalen Ausführung.

### 3. Benutzerdefinierte Lua-Transformation anwenden
Erstelle `transform.lua`:
```lua
function transform(row)
  if row["status"] == "pending" then
    row["priority"] = "hoch"
  else
    row["priority"] = "normal"
  end
  return row
end
```
Ausführung:
```bash
cleancsvai transform --input clean.csv --script transform.lua --output final.csv
```
Lua-Sandbox erlaubt keine I/O-Operationen – sicher für automatisierte Pipelines.

## 2 Best Practices aus der Praxis

1. **Pre-Commit-Hook für CSV-Qualität**  
   Binde `cleancsvai schema validate` in deinen Git-Workflow ein: `pre-commit` prüft jedes neue CSV auf Schema-Konformität und blockiert unvollständige Daten. Beispiel für `.pre-commit-config.yaml`:
   ```yaml
   - repo: local
     hooks:
       - id: csv-schema-check
         name: CSV Schema Validierung
         entry: cleancsvai schema validate --schema schema.json
         language: system
         files: \.csv$
   ```

2. **Batch-Processing für große Datensätze**  
   Nutze `cleancsvai split` und `parallel` (GNU parallel) zur horizontalen Skalierung:
   ```bash
   cleancsvai split --input huge.csv --lines 50000 --prefix chunk_
   ls chunk_*.csv | parallel cleancsvai dedup --input {} --output {}_clean.csv
   ```
   Kombiniere mit `cleancsvai merge` für finale Konsolidierung – reduziert Laufzeit von Stunden auf Minuten.
