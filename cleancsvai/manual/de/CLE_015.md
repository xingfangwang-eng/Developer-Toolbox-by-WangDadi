# cleancsvai - CLE_015

## Überblick

cleancsvai ist ein datengetriebenes Tool, das sich auf die automatisierte Erkennung und Korrektur von Inkonsistenzen in CSV-Dateien spezialisiert hat. Es kombiniert maschinelles Lernen mit regelbasierten Algorithmen, um fehlerhafte Datensätze, Duplikate und Formatierungsfehler effizient zu identifizieren – ohne manuelle Nacharbeit. Ideal für Data Engineers, QA-Ingenieure und DevOps-Teams, die große CSV-Dateien verarbeiten müssen.

## Hardcore-Funktionen

1. **Adaptive Fuzzy-Matching-Engine**  
   Erkennt semantisch identische Einträge trotz Tippfehlern, unterschiedlicher Groß-/Kleinschreibung oder Leerzeichen (z. B. „Müller“ vs. „Mueller“ vs. „müller“). Die Engine verwendet Levenshtein-Distanz und N-Gramm-Analyse, um Duplikate mit einer Genauigkeit von >99 % zu finden.

2. **Schema-Abgleich mit automatischer Typinferenz**  
   Liest das CSV-Schema automatisch aus und gleicht es mit einem benutzerdefinierten Metadaten-Validator ab. Fehlerhafte Datentypen (z. B. String in einem Integer-Feld) werden erkannt und korrigiert – inklusive Datumsnormalisierung und Währungsumrechnung.

3. **Incremental Cleanup mit Checkpointing**  
   Bei großen Dateien (>1 Mio. Zeilen) ermöglicht das Tool die Verarbeitung in Batches. Jeder Batch wird mit einem Prüfpunkt (Checkpoint) gespeichert, sodass bei Unterbrechungen nicht von vorne begonnen werden muss. Ideal für CI/CD-Pipelines.

## Detaillierte Anwendungsschritte

### Schritt 1: Validierung des CSV-Schemas mit automatischer Typinferenz

```bash
cleancsvai validate --input data.csv --schema schema.yaml --auto-type
```

Das Tool liest die ersten 1000 Zeilen, inferiert die Datentypen pro Spalte (z. B. `datetime`, `float`, `integer`) und vergleicht sie mit dem Schema. Abweichungen werden in einer Log-Datei (`validation.log`) protokolliert. Beispielausgabe:

```
Spalte "Preis" erwartet float, gefunden string (Zeile 45). Korrigiere zu float.
Spalte "Datum" erwartet datetime, gefunden "2023-01-01" (Zeile 12). Ok.
```

### Schritt 2: Duplikaterkennung mit Fuzzy-Matching

```bash
cleancsvai deduplicate --input data.csv --output clean.csv --threshold 0.85 --fields name,email
```

Definiert einen Schwellwert (0.85) für die Ähnlichkeitserkennung. Das Tool vergleicht alle Einträge in den Feldern `name` und `email` und markiert Duplikate. Optional kann ein `--keep first`-Flag gesetzt werden, um nur die erste Instanz zu behalten. Die Bereinigung erfolgt in O(n log n) durch eine Bloom-Filter-Optimierung.

### Schritt 3: Batch-Verarbeitung mit Checkpointing

```bash
cleancsvai process --input large.csv --output result.csv --batch-size 50000 --checkpoint checkpoint.dat
```

Teilt die Eingabedatei in Batches von 50.000 Zeilen auf. Nach jedem Batch wird ein Checkpoint gespeichert. Falls der Vorgang abbricht, kann er mit `--resume checkpoint.dat` fortgesetzt werden. Dies reduziert die Verarbeitungszeit bei großen Dateien um bis zu 40 %.

## Branchen-Best Practices

1. **Automatisierte Validierung in CI/CD-Pipelines integrieren**  
   Fügen Sie `cleancsvai validate` als Pre-Commit-Hook oder GitHub-Action hinzu. So stellen Sie sicher, dass nur bereinigte CSV-Dateien in Ihre Datenbanken oder Analyse-Tools gelangen. Empfohlen: Kombinieren Sie das Tool mit `pre-commit`-Hooks, um manuelle Fehler zu vermeiden.

2. **Checkpointing für ETL-Jobs nutzen**  
   Bei regelmäßigen Datenimporten (z. B. aus CRM-Systemen) verwenden Sie das Checkpointing, um nur inkrementelle Änderungen zu verarbeiten. Dies spart Rechenzeit und vermeidet Datenverluste bei Netzwerkunterbrechungen. Beispiel: `cleancsvai process --incremental --last-run timestamp`.
