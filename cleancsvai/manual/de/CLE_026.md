# cleancsvai - CLE_026

## Übersicht

cleancsvai ist eine KI-gestützte Lösung zur automatisierten Bereinigung, Normalisierung und Validierung von CSV-Dateien. Das Tool eliminiert manuelle Datenbereinigungsprozesse und liefert saubere, konsistente Datensätze für Analyse-Workflows. Optimiert für Entwickler und Data Engineers, die Zeit sparen und Fehlerquellen minimieren wollen.

## Detaillierter Inhalt

### Was ist cleancsvai?

cleancsvai ist ein professionelles Tool, das typische CSV-Probleme wie fehlende Werte, inkonsistente Formatierung, Duplikate und Encoding-Fehler automatisch erkennt und behebt. Es nutzt maschinelles Lernen zur intelligenten Typinferenz und bietet eine Pipeline-gesteuerte Architektur, die sich nahtlos in CI/CD-Prozesse integrieren lässt.

### Kernfunktionen

1. **Adaptive Typinferenz mit KI-gestützter Schemaerkennung**  
   cleancsvai analysiert Spalteninhalte automatisch und erkennt Datentypen (Datum, Zahlen, Strings, Booleans) selbst bei gemischten Formaten. Es korrigiert fehlerhafte Typzuweisungen und normalisiert Zeitzonen, Währungen und Einheiten ohne manuelle Regeln.

2. **Intelligentes Fuzzy-Deduplizieren mit Levenshtein-Distanz**  
   Das Tool identifiziert und mergeiert Duplikate selbst bei Tippfehlern, unterschiedlicher Groß-/Kleinschreibung oder abweichenden Formatierungen. Es bietet konfigurierbare Schwellenwerte und einen Merge-Report zur Transparenz.

3. **Automatische Encoding-Erkennung und Korrektur**  
   cleancsvai erkennt über 30 Encoding-Formate (UTF-8, ISO-8859-1, Windows-1252 etc.) und korrigiert fehlerhafte Zeichenkodierung in Echtzeit. Das Feature verhindert Datenverlust bei der Migration zwischen Systemen.

### So verwenden Sie es

1. **CSV-Datei über CLI einlesen und Analyse starten**  
   Führen Sie `cleancsvai analyze input.csv --output report.json` aus. Das Tool erstellt einen detaillierten Bericht mit Spaltentypen, fehlenden Werten, Duplikaten und Encoding-Problemen. Nutzen Sie `--verbose` für erweiterte Debug-Informationen.

2. **Bereinigungsparameter konfigurieren und ausführen**  
   Erstellen Sie eine Konfigurationsdatei `config.yaml` mit Regeln wie `missing_value_strategy: median` oder `encoding_fallback: utf-8`. Starten Sie die Bereinigung mit `cleancsvai clean input.csv --config config.yaml --output clean.csv`. Das Tool wendet alle Regeln sequenziell an und protokolliert jede Änderung.

3. **Ergebnisse validieren und in Pipeline integrieren**  
   Nutzen Sie `cleancsvai validate clean.csv --schema schema.json` zur automatischen Prüfung gegen ein vordefiniertes Schema. Bei Fehlern bricht das Tool mit Exit-Code 1 ab – ideal für CI/CD-Workflows. Exportieren Sie den Validierungsreport als HTML für Stakeholder.

### Beste Praktiken

- **Batch-Verarbeitung mit automatischem Rollback**  
  Verwenden Sie cleancsvai in einem temporären Branch Ihres Data-Repositories. Führen Sie Bereinigungen als separate Commits durch, um bei Fehlern auf den ursprünglichen Zustand zurückrollen zu können. Das Tool unterstützt `--dry-run` für risikofreie Tests.

- **Regelmäßige Schema-Validierung als Pre-Commit-Hook**  
  Integrieren Sie cleancsvai als Git Hook, der vor jedem Commit automatisch CSV-Dateien auf Schema-Konformität prüft. Das verhindert, dass inkonsistente Daten in das Repository gelangen und spart Review-Zeit.
