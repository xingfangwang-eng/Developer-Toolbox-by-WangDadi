## Detaillierter Inhalt

### Was ist cleancsvai?

cleancsvai ist ein professionelles Tool, das speziell für die automatisierte Bereinigung und Normalisierung von CSV-Dateien entwickelt wurde. Es erkennt und korrigiert typische Datenfehler wie inkonsistente Formate, fehlende Werte, Duplikate und Kodierungsprobleme. Die KI-gestützte Analyse ermöglicht eine intelligente Datenaufbereitung ohne manuelle Skripte.

### Kernfunktionen

1. **Intelligente Duplikaterkennung mit Fuzzy-Matching**  
   Erkennt nicht nur exakte Duplikate, sondern auch ähnliche Einträge (z. B. „Müller“ vs. „Mueller“ oder „Max Mustermann“ vs. „M. Mustermann“) mittels Levenshtein-Distanz und Token-basierter Ähnlichkeit. Konfigurierbare Schwellenwerte für Präzision.

2. **Automatische Schema-Inferenz und Typkorrektur**  
   Analysiert Spaltentypen (Datum, Zahl, Boolean, Text) und korrigiert inkonsistente Werte (z. B. „01.01.2024“ vs. „2024-01-01“ oder „ja“ vs. „true“). Unterstützt ISO-8601, US-Formate und europäische Datumsnotationen.

3. **Batch-Validierung mit Regel-Engine**  
   Wendet benutzerdefinierte Validierungsregeln an (z. B. „E-Mail-Format prüfen“, „PLZ auf Ländercode abstimmen“). Erstellt einen detaillierten Fehlerreport mit Zeilen- und Spaltenangaben sowie Vorschlägen zur Korrektur.

### So verwenden Sie es

1. **CSV-Datei laden und Vorschau anzeigen**  
   Öffnen Sie cleancsvai über die CLI oder API und laden Sie Ihre CSV-Datei. Das Tool zeigt automatisch eine interaktive Vorschau der ersten 100 Zeilen an – inklusive erster Validierungswarnungen (z. B. „Spalte ‘Datum’ enthält 12 % inkonsistente Werte“).  
   *Befehl:* `cleancsvai --file input.csv --preview`

2. **Bereinigungsregeln konfigurieren**  
   Wählen Sie aus vordefinierten Profilen (z. B. „E-Commerce-Export“, „CRM-Daten“) oder definieren Sie eigene Regeln über eine JSON-Konfigurationsdatei. Beispiel für eine Regel:  
   ```json
   {
     "column": "email",
     "action": "normalize",
     "rules": ["lowercase", "trim", "validate_email"]
   }
   ```  
   *Tipp:* Nutzen Sie den `--dry-run`-Modus, um Änderungen zu simulieren, bevor Sie die Datei überschreiben.

3. **Bereinigung ausführen und exportieren**  
   Starten Sie die Bereinigung mit `cleancsvai --config rules.json --output cleaned.csv`. Nach Abschluss erhalten Sie eine Zusammenfassung:  
   - Anzahl korrigierter Werte  
   - Anzahl entfernter Duplikate  
   - Fehlerprotokoll als separate CSV (optional)  
   *Exportformate:* CSV, JSON, Parquet (für Big Data).

### Beste Praktiken

- **Immer ein Backup erstellen**  
   Bevor Sie eine CSV-Datei bereinigen, legen Sie eine Kopie des Originals an. cleancsvai überschreibt standardmäßig keine Dateien, aber ein Backup schützt vor unerwarteten Ergebnissen – besonders bei Fuzzy-Matching.

- **Regeln inkrementell testen**  
   Starten Sie mit einer kleinen Teilmenge Ihrer Daten (z. B. 500 Zeilen) und erweitern Sie die Regeln schrittweise. Nutzen Sie den `--dry-run`-Modus, um die Auswirkungen jeder Regel zu prüfen. So vermeiden Sie, dass eine zu aggressive Regel wichtige Daten löscht.
