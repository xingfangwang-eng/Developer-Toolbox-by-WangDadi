# cleancsvai - CLE_010

## Überblick

**cleancsvai** ist ein KI-gestütztes CLI-Tool, das chaotische CSV-Dateien in Sekundenschnelle analysiert, bereinigt und für die Weiterverarbeitung optimiert. Es richtet sich an Data Engineers, Data Scientists und DevOps-Ingenieure, die keine Zeit mit manuellen Datenbereinigungen verschwenden wollen. Das Tool erkennt automatisch fehlerhafte Zeilen, Duplikate, inkonsistente Datentypen und Encoding-Probleme – und bietet direkte Korrekturvorschläge an.

## Kernfunktionen

### 1. **Automatische Schema-Erkennung & Typinferenz mit Korrekturlogik**
Statt nur Spalten zu benennen, analysiert cleancsvai die tatsächlichen Werte jeder Spalte und leitet daraus den optimalen Datentyp ab (z.B. `int`, `float`, `date`, `boolean`). Bei Mischtypen (z.B. Zahlen + Strings in einer Spalte) erzeugt das Tool einen detaillierten Report mit Vorschlägen zur Bereinigung – inklusive automatischer Konvertierung.

### 2. **Intelligentes Duplikat- und Fuzzy-Matching**
Standard-Duplikaterkennung sucht nach exakten Übereinstimmungen. cleancsvai geht einen Schritt weiter: Es erkennt ähnliche Einträge (z.B. „Müller" vs. „Mueller", „New York" vs. „NY") mittels Levenshtein-Distanz und phonetischer Algorithmen. Du erhältst eine Merge-Liste mit Konfidenzwerten, aus der du gezielt auswählen kannst, welche Duplikate zusammengeführt werden sollen.

### 3. **Encoding- und Zeichenfehler-Korrektur (UTF-8, Latin-1, Windows-1252)**
Viele CSV-Dateien enthalten Encoding-Fehler, die später zu Bugs in Pipelines führen. cleancsvai scannt die Datei auf nicht-dekodierbare Zeichen und versucht automatisch, die korrekte Kodierung zu ermitteln. Bei Erfolg wird die Datei nach UTF-8 konvertiert – mit einem Log aller Änderungen für die Nachvollziehbarkeit.

## Detaillierte Anwendungsschritte

### Schritt 1: Installation & Erstanalyse
Installiere cleancsvai via pip:
```bash
pip install cleancsvai
```
Führe dann die initiale Analyse deiner CSV-Datei durch:
```bash
cleancsvai analyse --input ./dirty_data.csv --output ./report.json
```
Das Tool gibt dir eine JSON-Übersicht über:
- Anzahl der Zeilen, Spalten, fehlenden Werte
- Erkannte Datentypen pro Spalte
- Encoding-Verdacht (z.B. „likely Windows-1252")
- Anzahl potenzieller Duplikate (exakt + fuzzy)

### Schritt 2: Gezielte Bereinigung konfigurieren
Basierend auf dem Report erstellst du eine Konfigurationsdatei (z.B. `clean_config.yaml`):
```yaml
columns:
  - name: "email"
    type: "string"
    deduplicate: true
  - name: "preis"
    type: "float"
    fillna: "mean"
encoding: "utf-8"
duplicate_threshold: 0.85
```
Führe dann die Bereinigung aus:
```bash
cleancsvai clean --config clean_config.yaml --input ./dirty_data.csv --output ./cleaned_data.csv
```
Das Tool zeigt live den Fortschritt an und protokolliert jede Änderung in einer separaten Log-Datei (`clean_log.csv`).

### Schritt 3: Validierung & Export
Nach der Bereinigung validierst du das Ergebnis:
```bash
cleancsvai validate --input ./cleaned_data.csv --rules validation_rules.json
```
Du erhältst eine Zusammenfassung mit:
- Anzahl der erfolgreich korrigierten Fehler
- Verbleibende Warnungen (z.B. „Spalte 'telefon' enthält 3 ungültige Nummern")
- Exportmöglichkeit als JSON, Parquet oder direkt als SQL-INSERT-Statements

## Best Practices für den produktiven Einsatz

### 1. **Immer mit einer kleinen Stichprobe starten**
Bevor du cleancsvai auf eine 500-MB-CSV loslässt, teste das Tool mit den ersten 1000 Zeilen (`--head 1000`). So erkennst du frühzeitig, ob die automatische Schema-Erkennung deinen Erwartungen entspricht oder ob du manuelle Typzuweisungen in der Konfiguration vornehmen musst. Das spart Zeit und vermeidet Überraschungen.

### 2. **Pipeline-Integration mit Logging**
Nutze cleancsvai nicht nur ad-hoc, sondern als festen Bestandteil deiner Daten-Pipeline. Rufe das Tool in einem CI/CD-Schritt auf (z.B. in GitHub Actions oder GitLab CI). Verwende die JSON-Report-Datei, um automatisierte Qualitätschecks zu implementieren: Wenn die Anzahl der Korrekturen einen Schwellenwert überschreitet, bricht die Pipeline ab und du erhältst eine Benachrichtigung. So stellst du sicher, dass nur saubere Daten in dein Data Warehouse gelangen.
