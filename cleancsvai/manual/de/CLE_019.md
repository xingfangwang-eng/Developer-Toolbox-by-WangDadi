# cleancsvai - CLE_019

## Überblick

cleancsvai ist ein spezialisiertes CLI- und API-Tool zur automatisierten Bereinigung, Normalisierung und Analyse von CSV-Dateien. Es richtet sich an Data Engineers und DevOps-Ingenieure, die regelmäßig mit inkonsistenten Rohdaten aus verschiedenen Quellen arbeiten. Das Tool erkennt und korrigiert automatisch typische CSV-Fehler wie Trennzeichenkonflikte, Kodierungsprobleme, leere Felder und inkonsistente Datum- oder Zahlenformate – ohne manuelles Preprocessing.

## Hardcore-Funktionen

1. **Adaptive Trennzeichen-Detektion mit Konfliktauflösung**  
   cleancsvai analysiert automatisch das Trennzeichen jeder CSV-Datei (Komma, Semikolon, Tabulator, Pipe) und löst Konflikte auf, wenn in einem Feld selbst Trennzeichen vorkommen (z. B. durch maskierte Anführungszeichen). Es verwendet einen heuristischen Algorithmus, der auf dem Vorkommen von Escape-Sequenzen und Anführungszeichen basiert – kein manuelles `-d`-Flag mehr nötig.

2. **Schema-Inferenz & Typsicherheitsprüfung**  
   Das Tool inferiert automatisch das Schema jeder Spalte (Integer, Float, Datum, String) und validiert die Konsistenz. Bei Abweichungen (z. B. ein String in einer Integer-Spalte) wird der Eintrag entweder korrigiert, gelöscht oder in einen Standard-Nullwert umgewandelt. Das reduziert Runtime-Errors in nachgelagerten ETL-Pipelines drastisch.

3. **Batch-Validierung mit Diff-Reporting**  
   cleancsvai kann mehrere CSV-Dateien parallel validieren und erstellt ein maschinenlesbares Diff-Report (JSON/CSV), das genau zeigt, welche Zeilen und Spalten verändert wurden. Das ist ideal für CI/CD-Pipelines, in denen Datenqualität automatisch geprüft wird.

## Detaillierte Nutzungsschritte

### 1. Installation und Grundkonfiguration

```bash
# Installation via npm (Node.js >= 18)
npm install -g cleancsvai

# Initialisierung eines Konfigurationsdatei (optional)
cleancsvai init --config ./cleancsvai.config.json
```

In der `config.json` können Sie globale Regeln definieren, z. B. Standard-Nullwerte, Datumsformate oder Trennzeichen-Fallback. Ohne Konfiguration arbeitet cleancsvai mit Standardheuristiken.

### 2. CSV-Datei reinigen und normalisieren

```bash
# Einfache Reinigung mit automatischer Erkennung
cleancsvai clean --input ./dirty_data.csv --output ./clean_data.csv

# Mit explizitem Schema und Encoding
cleancsvai clean \
  --input ./dirty_data.csv \
  --output ./clean_data.csv \
  --encoding utf-8 \
  --schema ./schema.json \
  --delimiter auto \
  --null-values "NA,N/A,,null"
```

Das `schema.json` definiert Spaltennamen und erwartete Typen. Bei Typkonflikten wird automatisch korrigiert oder ein Warnhinweis ausgegeben. Das Tool gibt einen Exit-Code ungleich 0 zurück, wenn Fehler auftreten – perfekt für CI/CD.

### 3. Batch-Validierung mit Diff-Reporting

```bash
# Batch-Validierung aller CSV-Dateien in einem Ordner
cleancsvai batch-validate \
  --input-dir ./data/raw/ \
  --output-dir ./data/validated/ \
  --report ./batch_report.json \
  --threads 4
```

Der Report im JSON-Format enthält pro Datei: Anzahl der Zeilen, Anzahl der Korrekturen, Liste der korrigierten Felder und Fehlerkategorien. Sie können den Report automatisch in Ihre Monitoring-Lösung (z. B. Grafana) einspielen.

## Best Practices aus der Praxis

1. **Immer mit einem Schema arbeiten**  
   Auch wenn cleancsvai automatisch inferiert: Definieren Sie explizite Schemas in Ihrer CI/CD-Pipeline. Das verhindert Überraschungen, wenn sich die Struktur der Rohdaten ändert. Nutzen Sie die `--schema`-Option mit einem versionierten JSON-File.

2. **Batch-Validierung vor dem ETL-Lauf**  
   Führen Sie `batch-validate` immer vor dem eigentlichen ETL-Prozess aus. Das spart Zeit, weil Sie fehlerhafte Dateien früh erkennen und nicht erst nach einem gescheiterten Pipeline-Run. Kombinieren Sie das mit einem Slack- oder E-Mail-Webhook für automatisierte Alarme.
