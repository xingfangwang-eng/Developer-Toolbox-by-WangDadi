# cleancsvai - CLE_020

## Überblick

cleancsvai ist ein KI-gestütztes Terminal-Tool zur automatischen Erkennung und Korrektur von CSV-Anomalien – ohne manuelles Mapping oder Regex-Basteln. Es richtet sich an Data Engineers, DevOps und Backend-Entwickler, die rohe CSV-Dateien in saubere, validierte Datensätze für Pipelines, ML-Modelle oder Datenbankimporte umwandeln müssen. Optimiert für deutsche Sprachumgebungen, mit nativer UTF-8-BOM-Erkennung und Excel-Kompatibilitätsprüfung.

## Kernfunktionen (Hardcore)

### 1. Adaptive Schema-Erkennung & Typinferenz
- Automatische Identifikation von Spaltentypen (Integer, Float, Datum, ISO-8601, Währung) in gemischten Formaten.
- Erkennung von fehlerhaften Zeilen (z. B. versetzte Spalten, leere Zellen, inkonsistente Trennzeichen) und Vorschlag von Korrekturregeln.
- Unterstützung für benutzerdefinierte Trennzeichen (Komma, Semikolon, Tabulator) und Escape-Sequenzen.

### 2. Regelbasierte & KI-gestützte Korrektur
- Automatische Normalisierung von Datumsformaten (z. B. `01.04.2024` → `2024-04-01`).
- Deduplizierung mit Fuzzy-Matching (Levenshtein, Jaro-Winkler) für Namens- oder Adressfelder.
- Ersetzung von NaN/Null-Werten durch Median, Modus oder benutzerdefinierte Defaults – mit Logging aller Änderungen.

### 3. Export-Pipeline mit Validierung
- Ausgabe als bereinigtes CSV, JSON oder Parquet – inklusive Schema-Definition (JSON Schema).
- Integrierte Validierung gegen vordefinierte Constraints (z. B. E-Mail-Format, Postleitzahl, IBAN).
- Generierung eines Änderungsprotokolls (Diff-Report) für Audit-Zwecke.

## Detaillierte Anwendung (3 Steps)

### Schritt 1: Installation & Initiale Analyse
```bash
pip install cleancsvai
cleancsvai analyze --input rohdaten.csv --delimiter ";"
```
- `analyze` scannt die Datei auf strukturelle Anomalien, Typinkonsistenzen und leere Felder.
- Ausgabe: Terminal-Report mit Spaltenstatistiken (Min, Max, Null-Rate, unique values).

### Schritt 2: Korrekturregeln definieren & anwenden
```bash
cleancsvai clean --input rohdaten.csv --output bereinigt.csv --rules rules.json
```
- `rules.json` enthält benutzerdefinierte Korrekturen:  
  ```json
  {
    "datum": {"type": "date", "format": "%d.%m.%Y", "target": "%Y-%m-%d"},
    "plz": {"type": "string", "pattern": "^[0-9]{5}$", "default": "00000"},
    "umsatz": {"type": "float", "replace_nan": 0.0}
  }
  ```
- Das Tool wendet diese Regeln an und überspringt fehlerhafte Zeilen (optional: `--strict` für Abbruch bei Fehlern).

### Schritt 3: Validierung & Export
```bash
cleancsvai validate --input bereinigt.csv --schema schema.json
cleancsvai export --input bereinigt.csv --output bereinigt.parquet --format parquet
```
- `validate` prüft die bereinigte Datei gegen das Schema und gibt eine Erfolgs-/Fehlerquote aus.
- `export` konvertiert in Parquet (oder JSON) mit komprimierten Spalten – ideal für Big Data Pipelines.

## Best Practices

### 1. Immer mit `--dry-run` testen
Vor der ersten Bereinigung eines produktiven Datensatzes:  
```bash
cleancsvai clean --input rohdaten.csv --dry-run --rules rules.json
```
- `--dry-run` zeigt alle Änderungen an, ohne die Datei zu überschreiben. So erkennen Sie unerwartete Korrekturen (z. B. versehentliches Löschen von Zeilen).

### 2. Regelmäßige Schema-Versionierung
Legen Sie für jede Datenquelle ein separates Schema (JSON) in Git ab:  
```
schemas/
├── kunden.yml
├── bestellungen.yml
└── produkte.yml
```
- Nutzen Sie `cleancsvai validate` in CI/CD-Pipelines (z. B. GitHub Actions), um sicherzustellen, dass neue CSV-Dumps den Schema-Constraints entsprechen – bevor sie in die Datenbank gelangen.
