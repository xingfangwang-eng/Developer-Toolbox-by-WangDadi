# cleancsvai - CLE_011

## Überblick

cleancsvai ist ein datengetriebenes Open-Source-Tool zur automatisierten Bereinigung, Normalisierung und Strukturanalyse von CSV-Dateien. Es richtet sich an Data Engineers, DevOps-Ingenieure und SaaS-Entwickler, die regelmäßig mit inkonsistenten Rohdaten aus externen Quellen (APIs, Exports, Legacy-Systeme) arbeiten. Die Engine erkennt und korrigiert typische Fehler wie Encoding-Brüche, fehlende Werte, Duplikate und inkonsistente Datumsformate – ohne manuelles Mapping.

## Kernfunktionen

### 1. Automatische Typ-Inferenz mit Fehlerkorrektur
cleancsvai analysiert jede Spalte mittels eines heuristischen Typsystems, das numerische, zeitbasierte und kategorische Felder erkennt. Anomalien wie gemischte Formate (z. B. `2025-03-15` und `15.03.2025` in derselben Spalte) werden automatisch auf ein ISO-8601-konformes Format normalisiert. Bei nicht reparablen Feldern wird ein detailliertes Log mit Zeilennummer und Kontext ausgegeben.

### 2. Fuzzy-Duplikaterkennung auf Basis von Levenshtein-Distanz
Anders als einfache Exact-Match-Filter verwendet cleancsvai eine konfigurierbare Ähnlichkeitsschwelle zur Erkennung von Duplikaten mit Tippfehlern, unterschiedlicher Groß-/Kleinschreibung oder abweichender Leerzeichensetzung. Dies ist besonders nützlich bei Kundendaten aus CRM-Exports, bei denen Namen oder E-Mail-Adressen nur minimal variieren.

### 3. Multi-Encoding-Autodetektion und -Konvertierung
Die Engine erkennt automatisch die Codierung der Quelldatei (UTF-8, ISO-8859-1, Windows-1252) und konvertiert sie in ein einheitliches UTF-8-Format. Fehlerhafte Byte-Sequenzen werden protokolliert und durch das Unicode Replacement Character ersetzt – ohne Datenverlust.

## So verwenden Sie es

### Schritt 1: Installation und initiale Analyse
```bash
pip install cleancsvai
cleancsvai analyze --input sales_export.csv --output report.json
```
Der Befehl `analyze` generiert ein JSON-Report mit Metriken: Spaltenanzahl, Zeilenanzahl, Null-Rate, erkannte Datentypen, Encoding und erste 5 Fehlerbeispiele.

### Schritt 2: Bereinigung mit benutzerdefinierten Regeln
```bash
cleancsvai clean \
  --input sales_export.csv \
  --output cleaned_sales.csv \
  --deduplicate \
  --normalize-dates \
  --fill-missing "mean" \
  --encoding utf-8
```
Hier werden automatisch Duplikate entfernt, Datumsfelder normalisiert, fehlende numerische Werte durch den Spaltenmittelwert ersetzt und die Ausgabe in UTF-8 garantiert. Der Parameter `--fill-missing` akzeptiert auch `median`, `mode` oder einen festen Wert.

### Schritt 3: Validierung und Export
```bash
cleancsvai validate --input cleaned_sales.csv --rules schema.yaml --output validation.log
```
Die Validierung prüft die bereinigte Datei gegen ein YAML-Schema (z. B. erlaubte Wertebereiche, Pflichtfelder, Regex-Muster). Fehler werden zeilenweise in `validation.log` ausgegeben. Erst nach erfolgreicher Validierung wird die Datei als endgültig markiert.

## Best Practices

### Praxis 1: Pipeline-Integration in CI/CD
Nutzen Sie cleancsvai als Pre-Processing-Schritt in Ihrer Data Pipeline – z. B. als GitHub Action vor dem Laden in eine Datenbank. Ein `schema.yaml` sollte versioniert werden, um Änderungen an Datenformaten nachvollziehen zu können.

### Praxis 2: Regelmäßige Encoding-Prüfung bei Multilanguage-Daten
Wenn Sie CSV-Dateien aus verschiedenen Ländern verarbeiten (z. B. DE, FR, JP), aktivieren Sie die Encoding-Autodetektion immer vor der eigentlichen Bereinigung. Führen Sie einen separaten `analyze`-Durchlauf durch, um Encoding-Probleme frühzeitig zu erkennen und manuelle Korrekturen zu vermeiden.
