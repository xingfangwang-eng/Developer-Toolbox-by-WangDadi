# cleancsvai - CLE_028

## Überblick

cleancsvai ist ein KI-gestütztes Datenbereinigungstool, das speziell für Entwickler und Datenanalysten entwickelt wurde, die mit unstrukturierten CSV-Dateien arbeiten. Es eliminiert manuelle Bereinigungsschritte und optimiert den Datenworkflow – von der Erkennung fehlerhafter Zeilen bis zur semantischen Normalisierung. Ideal für ETL-Pipelines, Data-Science-Projekte und Compliance-Audits.

## Hardcore-Funktionen

1. **Automatische Mustererkennung mit Context-Aware Parsing**  
   Erkennt inkonsistente Trennzeichen, Escape-Sequenzen und versteckte Unicode-Fehlercodes (z.B. BOM, NULL-Bytes) und normalisiert sie automatisch in UTF-8 ohne Datenverlust. Unterstützt benutzerdefinierte Delimiter-Profile für Legacy-Systeme.

2. **Semantische Typ-Inferenz mit KI-Korrekturvorschlägen**  
   Analysiert Spalteninhalte auf Basis von Regex-Patterns und ML-Modellen, erkennt falsche Datentypen (z.B. Zahlen als Strings, Datumsformate mit Zeitzonenfehlern) und schlägt automatisch Korrekturen vor – inklusive Validierung durch Referenzdatenbanken.

3. **Inkrementelle Bereinigung mit Delta-Detection**  
   Ermöglicht Batch-Verarbeitung großer Dateien (100k+ Zeilen) durch Streaming-basierte Verarbeitung. Erkennt nur geänderte Zeilen seit dem letzten Lauf und vermeidet redundante Berechnungen. Bietet einen Merge-Mechanismus für versionierte Daten.

## Detaillierte Anwendungsschritte

### Schritt 1: Konfiguration der Datenquellen und Validierungsregeln

```bash
cleancsvai --input dirty_data.csv \
           --delimiter ";" \
           --encoding "utf-8-sig" \
           --rules "rules/custom_rules.yaml" \
           --output clean_data.csv
```

Die YAML-Datei definiert Regeln wie:  
- `type_inference: true` (aktiviert semantische Typ-Erkennung)  
- `null_values: ["N/A", "NULL", "NaN"]` (automatische NULL-Ersetzung)  
- `date_formats: ["%Y-%m-%d", "%d.%m.%Y"]` (Datumsnormalisierung)  

### Schritt 2: Ausführung der Batch-Bereinigung mit Fehlerprotokollierung

```bash
cleancsvai --input large_dataset.csv \
           --batch-size 5000 \
           --error-log errors.jsonl \
           --verbose
```

Das Tool verarbeitet die Datei in Batches, erstellt ein detailliertes JSONL-Protokoll mit Zeilennummern und Fehlertypen (z.B. `row_123: Invalid date format "13.13.2023"`). Nach Abschluss generiert es eine Zusammenfassung mit Metriken wie `rows_fixed`, `rows_removed` und `confidence_scores`.

### Schritt 3: Integration in CI/CD-Pipeline (GitHub Actions Beispiel)

```yaml
- name: CSV Cleanup
  run: |
    cleancsvai --input ${{ github.workspace }}/data/raw.csv \
               --output ${{ github.workspace }}/data/clean.csv \
               --rules .cleancsvai/rules.yaml \
               --fail-on-warning
```

Die Option `--fail-on-warning` bricht den Build ab, wenn kritische Datenfehler erkannt werden (z.B. Duplikate, inkonsistente IDs). Perfekt für automatisierte Data-Quality-Gates.

## Best Practices aus der Industrie

1. **Versionierte Rohdaten vor der Bereinigung sichern**  
   Verwenden Sie Git LFS oder einen S3-Bucket, um die Original-CSV-Dateien zu speichern. cleancsvai kann direkt aus S3 lesen (`--input s3://bucket/raw/`). So können Sie jederzeit auf die unveränderten Daten zurückgreifen und Regressionen vermeiden.

2. **Regelprofile für verschiedene Datenquellen erstellen**  
   Definieren Sie separate YAML-Konfigurationen für CRM-Exports, Log-Dumps und API-Dumps. Verwenden Sie `--profile` zum schnellen Wechsel zwischen Kontexten. Beispiel:  
   `cleancsvai --profile crm_export --input sales.csv`  
   Dies reduziert Fehlkonfigurationen und beschleunigt die Einarbeitung neuer Teammitglieder.
