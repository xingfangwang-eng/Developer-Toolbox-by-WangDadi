# cleancsvai - CLE_025

## Überblick

cleancsvai ist ein spezialisiertes CLI-Tool, das inkonsistente CSV-Dateien automatisch erkennt, bereinigt und in standardisierte Formate überführt – ohne manuelle Regex-Bastelei oder Excel-Makros. Ideal für Data Engineers, die täglich mit schmutzigen Rohdaten aus Legacy-Systemen oder API-Exports arbeiten.

## 3 Hardcore-Funktionen

1. **Automatische Spaltentyp-Inferenz mit Korrekturlogik**  
   Erkennt fehlerhafte Datentypen (z. B. Zahlen als Strings, Datumsformate gemischt) und wandelt sie konsistent um. Bei Konflikten wird eine Konfidenzmatrix ausgegeben.

2. **Fuzzy-Duplikat-Erkennung auf Zeilenebene**  
   Nutzt Levenshtein-Distanz und Token-Überlappung, um semantisch identische Zeilen zu identifizieren – selbst bei abweichender Groß-/Kleinschreibung oder Tippfehlern.

3. **Schema-Export als JSON-Schema**  
   Generiert aus einer bereinigten CSV ein valides JSON-Schema (Draft-07), das direkt in Datenbank-Migrationen oder ETL-Pipelines integriert werden kann.

## 3 detaillierte Anwendungsschritte

### Schritt 1: Dirty CSV analysieren
```bash
cleancsvai scan ./export.csv --output report.json
```
Der Befehl scannt die CSV auf Anomalien: leere Zellen, gemischte Trennzeichen, Zeilen mit abweichender Spaltenanzahl. Der Report enthält eine Heatmap der Problemstellen.

### Schritt 2: Bereinigung mit manueller Bestätigung
```bash
cleancsvai clean ./export.csv --rules rules.yaml --interactive
```
Du definierst in `rules.yaml` Regeln wie `trim_whitespace: true` oder `date_format: "%Y-%m-%d"`. Im interaktiven Modus zeigt das Tool jede Änderung an und wartet auf Bestätigung (Y/n).

### Schritt 3: Schema exportieren und validieren
```bash
cleancsvai schema ./cleaned.csv --output schema.json
```
Das generierte JSON-Schema enthält `type`-Constraints, `pattern` für Datumsformate und `enum` für kategoriale Felder. Nutze es z. B. in Apache Avro oder als Validator für zukünftige Imports.

## 2 Best Practices aus der Praxis

- **Immer zuerst ein Sample bereinigen**  
  Bei CSV-Dateien > 1 Mio Zeilen: Führe `cleancsvai scan` nur auf den ersten 10.000 Zeilen aus, um Ressourcen zu sparen und die Regeldefinition zu iterieren.

- **Regeldateien versionieren**  
  Lege `rules.yaml` unter Git an und referenziere sie in CI/CD-Pipelines. So stellst du sicher, dass alle Teammitglieder dieselben Bereinigungsstandards verwenden – und Änderungen nachvollziehbar bleiben.
