# cleancsvai - CLE_021

## Überblick

cleancsvai ist ein spezialisiertes CLI-Tool und eine Python-Bibliothek, die unstrukturierte CSV-Dateien automatisch erkennt, bereinigt und in analysierbare Formate überführt. Es richtet sich an Datenwissenschaftler und DevOps-Ingenieure, die regelmäßig mit inkonsistenten Rohdaten kämpfen.

## Detaillierter Inhalt

### Was ist cleancsvai?

cleancsvai analysiert CSV-Dateien auf strukturelle Inkonsistenzen, fehlende Werte und Kodierungsfehler. Es nutzt heuristische Algorithmen, um Spaltentypen zu erkennen, Duplikate zu entfernen und Daten zu normalisieren, ohne dass manuelle Regeln definiert werden müssen.

### Kernfunktionen

- **Automatische Typinferenz mit Korrekturvorschlägen**  
  Erkennt numerische, Datums- und Textspalten automatisch. Bei Fehlern (z. B. Buchstaben in einer Zahlenreihe) werden Korrekturvorschläge generiert und die Daten bereinigt, ohne dass der Benutzer eingreifen muss.

- **Erkennung und Auflösung von Kodierungsproblemen**  
  Identifiziert Dateien mit falscher Zeichenkodierung (UTF-8 vs. Latin-1) und konvertiert sie automatisch. Unterstützt auch die Erkennung von BOM (Byte Order Mark) und die Entfernung von unsichtbaren Steuerzeichen.

- **Intelligentes Merging von fragmentierten Tabellen**  
  Erkennt, wenn eine CSV-Datei aus mehreren logischen Tabellen besteht (z. B. durch Zeilenumbrüche oder fehlende Spalten). Extrahiert diese automatisch und gibt sie als separate, bereinigte Dateien aus.

### So verwenden Sie es

1. **Analyse der Rohdaten mit automatischer Fehlererkennung**  
   ```bash
   cleancsvai analyze input.csv --output report.json
   ```  
   Der Befehl erzeugt einen detaillierten Report über fehlende Werte, Datentypen und Kodierungsprobleme. Die Ausgabe erfolgt als JSON, das direkt in CI/CD-Pipelines weiterverarbeitet werden kann.

2. **Bereinigung mit benutzerdefinierten Filtern**  
   ```python
   from cleancsvai import Cleaner
   cleaner = Cleaner()
   cleaner.load("input.csv")
   cleaner.remove_duplicates(columns=["email", "id"])
   cleaner.fill_missing(strategy="median", columns=["age", "salary"])
   cleaner.save("cleaned.csv")
   ```  
   Die Python-API erlaubt es, spezifische Bereinigungsschritte zu definieren, ohne dass die gesamte Datei neu geladen werden muss.

3. **Validierung und Export in mehrere Formate**  
   ```bash
   cleancsvai validate cleaned.csv --schema schema.json --output validation.log
   cleancsvai export cleaned.csv --format parquet --output data.parquet
   ```  
   Die Validierung prüft gegen ein benutzerdefiniertes Schema (z. B. Spaltentypen, Wertebereiche). Der Export unterstützt Parquet, JSON und SQL, sodass die bereinigten Daten direkt in Datenbanken oder Data Lakes übernommen werden können.

### Beste Praktiken

- **Vor der Bereinigung immer ein Backup der Rohdaten erstellen**  
  Speichern Sie die Originaldatei in einem separaten Verzeichnis, bevor Sie cleancsvai ausführen. So können Sie bei Fehlentscheidungen jederzeit auf die unveränderten Daten zurückgreifen.

- **Automatisierung in CI/CD-Pipelines**  
  Integrieren Sie cleancsvai als Schritt in Ihrer Datenpipeline (z. B. mit GitHub Actions oder GitLab CI). Verwenden Sie den `--exit-on-error` Parameter, um den Build fehlschlagen zu lassen, wenn die Datenqualität unter einem definierten Schwellwert liegt.
