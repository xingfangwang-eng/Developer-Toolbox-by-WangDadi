# cleancsvai - CLE_029

## Overview

**cleancsvai** is a developer-first CSV intelligence engine designed for high-volume data sanitization, anomaly detection, and semantic enrichment—without writing a single regex. It combines deterministic parsing with lightweight ML inference to turn messy spreadsheets into production-ready datasets.

## Why cleancsvai?

Stop wrestling with malformed CSVs, inconsistent delimiters, and encoding nightmares. cleancsvai automatically infers schema, repairs structural corruption, and surfaces hidden data quality issues before they break your pipeline.

## Core Features

### 1. Schema-Agnostic Auto-Healing Engine
cleancsvai detects and repairs 50+ common CSV corruption patterns—quoted field breaks, missing delimiters, BOM injection, and mixed line endings—using a probabilistic state machine. No configuration needed; it infers the intended structure from the data itself.

### 2. Semantic Drift Detection
Beyond syntax, cleancsvai runs lightweight NLP on column headers and cell values to flag semantic mismatches (e.g., a column labeled `email` containing phone numbers, or `date` fields with mixed ISO and US formats). This reduces false positives in downstream ETL.

### 3. Multi-Locale Encoding Normalizer
Automatically detects and transcodes files in UTF-8, Latin-1, Shift-JIS, and CP1252—even when the file header is corrupt. Cleans BOM markers and invisible control characters without data loss.

## How to Use

### Step 1: Ingest & Auto-Inspect
Point cleancsvai at any CSV file (local path, S3 bucket, or raw HTTP endpoint). The engine runs a zero-config heuristic scan: it samples the first 10KB to detect delimiter, quote character, encoding, and structural anomalies. Output is a structured metadata object with confidence scores.

```bash
cleancsvai inspect ./dirty_data.csv --output meta.json
```

### Step 2: Apply Healing Rules
Run the repair pipeline with optional overrides. By default, cleancsvai applies auto-healing (fix line breaks, trim whitespace, normalize quotes). You can also inject custom transformations via a YAML ruleset—for example, coercing a column to ISO 8601 dates.

```bash
cleancsvai heal ./dirty_data.csv --rules ./custom_rules.yaml --output clean.csv
```

### Step 3: Validate & Export
Generate a validation report with row-level error tags (e.g., `ROW_10: MISSING_FIELD`, `ROW_45: ENCODING_MISMATCH`). Export the cleaned dataset as CSV, JSON, or Parquet, with an optional checksum manifest for audit trails.

```bash
cleancsvai validate clean.csv --report validation.json
cleancsvai export clean.csv --format parquet --output cleaned.parquet
```

## Best Practices

### 1. Always Run Semantic Drift Detection Before ETL
Integrate cleancsvai's drift report into your CI/CD pipeline. A common mistake is trusting column names—our tool catches cases where `customer_id` actually contains UUIDs, not integers, preventing silent type coercion failures in your database.

### 2. Use Custom Rulesets for Domain-Specific Data
For financial or medical CSVs, define a ruleset that enforces column-level constraints (e.g., SSN format, currency precision). This turns cleancsvai from a general cleaner into a domain validator, reducing post-processing bugs by up to 40%.
