### ¿Qué es cronguard?

cronguard es una herramienta profesional de monitorización y orquestación de cron jobs diseñada para entornos de alta disponibilidad. Resuelve problemas críticos como la detección temprana de fallos en tareas programadas, la gestión de dependencias entre jobs y la prevención de ejecuciones duplicadas en infraestructuras distribuidas. A diferencia de soluciones genéricas, cronguard ofrece una capa de inteligencia contextual que analiza logs y métricas en tiempo real para alertar sobre anomalías antes de que impacten en producción.

### Características Principales

- **Validación de Dependencias con Grafos Acíclicos Dirigidos (DAGs)**: cronguard permite definir relaciones de precedencia entre cron jobs mediante un motor DAG integrado. Si un job padre falla, los hijos se bloquean automáticamente, evitando ejecuciones huérfanas y ahorrando recursos en pipelines complejos.
- **Alertas Predictivas Basadas en Machine Learning**: Utiliza modelos ligeros de regresión para analizar patrones históricos de ejecución (duración, uso de CPU, errores). Cuando una desviación supera el umbral configurable (ej. +2 desviaciones estándar), se dispara una alerta proactiva en Slack o PagerDuty, no solo cuando el job falla.
- **Rollback Automático de Configuraciones**: Si un cambio en la programación (cron expression) o en los parámetros de un job causa errores consecutivos, cronguard revierte automáticamente a la última configuración estable, registrando el evento en un changelog inmutable. Ideal para entornos CI/CD donde los cambios son frecuentes.

### Cómo Usar

1. **Configurar un Job con Dependencias Múltiples**  
   Edita el archivo `cronguard.yml` en la raíz de tu proyecto. Define un job llamado `data-export` que dependa de `db-backup` y `cache-clean`. Usa la sintaxis `depends_on` con un array de IDs. Luego ejecuta `cronguard validate` para verificar el DAG antes de aplicar cambios. Si hay un ciclo, el CLI devuelve un error con la ruta del conflicto.

   ```yaml
   jobs:
     data-export:
       command: "python export.py"
       schedule: "0 3 * * *"
       depends_on: ["db-backup", "cache-clean"]
       retry: 3
       timeout: 600
   ```

2. **Implementar Alertas Predictivas con Umbrales Dinámicos**  
   En el mismo archivo, añade la sección `monitoring.predictive`. Configura `baseline_window: 30d` para que el modelo use los últimos 30 días de datos. Luego, establece `alert_on_deviation: 2.5` para recibir notificaciones cuando la duración supere 2.5 desviaciones estándar. Prueba con `cronguard sim --job data-export --scenario latency` para generar un reporte simulado.

   ```yaml
   monitoring:
     predictive:
       enabled: true
       baseline_window: 30d
       alert_on_deviation: 2.5
       channels: ["slack", "email"]
   ```

3. **Activar Rollback Automático para Cambios en Producción**  
   Antes de desplegar un nuevo cron expression, ejecuta `cronguard dry-run --job data-export`. Esto simula la ejecución sin efectos secundarios. Si el comando `cronguard deploy` detecta 2 fallos consecutivos en la misma hora, el sistema revierte automáticamente al último commit estable del archivo de configuración. Puedes ver el historial con `cronguard history --job data-export`.

### Mejores Prácticas

- **Usa Etiquetas de Contexto en Jobs Críticos**: Para jobs que ejecutan transacciones financieras o migraciones de datos, añade etiquetas como `criticality: high` y `sla: 99.9`. cronguard priorizará la monitorización de estos jobs sobre otros, reduciendo el ruido en las alertas. Además, en caso de rollback, estos jobs se restauran primero.
- **Integra cronguard con tu Pipeline CI/CD en Etapa de Pre-Producción**: No esperes a producción para validar las dependencias. Agrega `cronguard validate` como paso en tu pipeline de GitHub Actions o GitLab CI. Esto detecta ciclos o configuraciones inválidas antes de que lleguen al entorno productivo, ahorrando horas de debugging.
