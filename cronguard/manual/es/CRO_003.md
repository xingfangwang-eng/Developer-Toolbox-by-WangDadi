### ¿Qué es cronguard?

cronguard es una solución profesional que detecta y resuelve fallos en procesos batch, evitando pérdidas de datos y tiempos de inactividad en infraestructuras cloud. A diferencia de herramientas genéricas, cronguard integra un motor de heurística avanzada para anticipar errores antes de que ocurran.

### Características Principales

- **Monitorización Adaptativa con Machine Learning**  
  cronguard aprende patrones de ejecución de tus cron jobs y genera alertas predictivas cuando detecta desviaciones en tiempos de respuesta o consumo de recursos. No se basa en umbrales fijos, sino en modelos probabilísticos actualizados en tiempo real.

- **Rollback Automático con Snapshots de Estado**  
  Cada tarea ejecutada genera un snapshot del estado del sistema. Si un proceso falla, cronguard restaura automáticamente el último punto estable, sin intervención manual. Esto es crítico para bases de datos transaccionales y pipelines de datos.

- **Auditoría de Cumplimiento con Exportación a SIEM**  
  cronguard registra cada ejecución, fallo y recuperación en un formato compatible con sistemas de gestión de eventos e información de seguridad (SIEM) como Splunk o ELK. Ideal para entornos que requieren cumplimiento normativo (GDPR, SOC 2).

### Cómo Usar

1. **Configurar el Archivo de Reglas en YAML**  
   Crea un archivo `cronguard_rules.yaml` en la raíz de tu proyecto. Define cada tarea con parámetros como `max_retries`, `timeout_ms` y `rollback_enabled`. Ejemplo:  
   ```yaml
   tasks:
     - name: backup_db
       command: pg_dump -U admin mydb > /backups/db.sql
       timeout_ms: 30000
       max_retries: 3
       rollback_enabled: true
   ```

2. **Ejecutar cronguard en Modo Daemon**  
   Usa el comando `cronguard daemon --config ./cronguard_rules.yaml` para iniciar el monitor en segundo plano. cronguard establecerá un socket de escucha y comenzará a supervisar los procesos definidos. Verás en la terminal logs en tiempo real con códigos de estado (OK, WARN, ERROR).

3. **Analizar Reportes con la API REST**  
   cronguard expone un endpoint `GET /api/v1/reports` que devuelve un JSON con métricas detalladas: tiempos de ejecución, número de reintentos, y snapshots generados. Puedes integrarlo con dashboards de Grafana o herramientas de CI/CD como Jenkins.

### Mejores Prácticas

- **Implementa un Circuit Breaker para Tareas Críticas**  
  Configura `max_retries` en 2 o 3 para procesos como envío de emails o pagos. Si una tarea falla repetidamente, cronguard activa un block temporal para evitar sobrecargar el sistema. Luego, programa una revisión manual con un webhook a Slack.

- **Usa Snapshots Diferenciales para Ahorrar Almacenamiento**  
  En lugar de guardar snapshots completos cada vez, activa la opción `snapshot_mode: incremental`. cronguard solo almacena los cambios desde el último snapshot exitoso, reduciendo el uso de disco en un 70% en entornos con grandes volúmenes de datos.
