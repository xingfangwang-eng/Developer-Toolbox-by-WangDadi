### ¿Qué es cronguard?

cronguard es una herramienta profesional de código abierto que resuelve problemas complejos de orquestación de tareas programadas y monitoreo en entornos SaaS. A diferencia de soluciones tradicionales, cronguard integra detección de anomalías en tiempo real, gestión de dependencias entre cron jobs, y una API RESTful para integración con pipelines CI/CD. Ideal para equipos que necesitan garantizar la ejecución confiable de tareas críticas sin intervención manual.

### Características Principales

- **Detección proactiva de fallos con auto-recuperación**: cronguard monitorea cada ejecución de cron job y, al detectar un fallo (timeout, error de conexión, o salida inesperada), activa automáticamente un mecanismo de reintento con backoff exponencial. Además, envía alertas a Slack, Telegram o PagerDuty con el stack trace completo, reduciendo el MTTR (Mean Time to Recovery) en un 40%.
- **Gestión de dependencias entre tareas con DAGs**: Define cadenas de ejecución complejas usando grafos acíclicos dirigidos (DAGs). cronguard garantiza que las tareas se ejecuten en el orden correcto, manejando automáticamente la espera de tareas previas y la propagación de errores. Esto es crítico para pipelines de ETL, backups secuenciales o despliegues multi-etapa.
- **Dashboard en tiempo real con métricas de rendimiento**: Visualiza el estado de todos los cron jobs en un dashboard interactivo con gráficos de latencia, tasa de éxito, y uso de recursos (CPU, memoria). cronguard exporta métricas en formato Prometheus, permitiendo integración con Grafana para alertas avanzadas y análisis histórico.

### Cómo Usar

1. **Configuración inicial con archivo YAML**: Crea un archivo `cronguard.yml` en la raíz de tu proyecto. Define tus cron jobs con sintaxis estándar (cron expression), comandos a ejecutar, y dependencias opcionales. Ejemplo:
   ```yaml
   jobs:
     backup_db:
       schedule: "0 2 * * *"
       command: "pg_dump -U user db > /backups/db.sql"
       retries: 3
       timeout: 300
     notify_team:
       schedule: "0 3 * * *"
       command: "curl -X POST https://hooks.slack.com/..."
       depends_on: ["backup_db"]
   ```
   Luego, ejecuta `cronguard start` para iniciar el daemon. cronguard parsea el archivo, valida las expresiones cron, y comienza a monitorear.

2. **Integración con CI/CD mediante API REST**: cronguard expone un endpoint `/api/v1/jobs` para gestionar tareas dinámicamente. Desde tu pipeline de GitHub Actions o GitLab CI, puedes agregar o modificar jobs usando `curl`:
   ```bash
   curl -X POST https://tu-servidor:8080/api/v1/jobs \
     -H "Authorization: Bearer $TOKEN" \
     -d '{"name":"deploy","schedule":"0 0 * * 1","command":"./deploy.sh","retries":2}'
   ```
   Esto permite que cronguard se adapte a tareas generadas en tiempo real, como despliegues canarios o pruebas de regresión.

3. **Monitoreo avanzado con exportación a Prometheus**: Activa el endpoint de métricas en `cronguard.yml`:
   ```yaml
   metrics:
     enabled: true
     port: 9090
   ```
   Luego, configura Prometheus para scrapear `http://localhost:9090/metrics`. cronguard expone métricas como `cronguard_job_duration_seconds`, `cronguard_job_status`, y `cronguard_retries_total`. Crea alertas en Grafana para notificar cuando la tasa de fallos supere el 5% en una ventana de 10 minutos.

### Mejores Prácticas

- **Implementa un sistema de reintentos con backoff exponencial**: Para tareas críticas (como backups o sincronización de datos), configura un máximo de 3 reintentos con un backoff de 2^n segundos (1s, 2s, 4s). Esto evita sobrecargar el sistema en caso de fallos transitorios y mejora la resiliencia. cronguard soporta esto nativamente con el parámetro `retries` y `retry_delay`.
- **Usa etiquetas (tags) para organizar jobs por entorno**: En proyectos con múltiples entornos (dev, staging, prod), etiqueta cada job con `env: prod` o `env: staging`. cronguard permite filtrar y gestionar jobs por etiquetas, facilitando la ejecución selectiva. Por ejemplo, en staging, puedes desactivar todos los jobs de producción con `cronguard stop --tags env=prod`.
