# cronguard - CRO_025

## 概要

cronguard は、高頻度ジョブスケジューリングと Cron 式のリアルタイム監視・障害検知に特化したエンタープライズ向けデーモンツールです。PostgreSQL 上のジョブキューと連携し、ミリ秒単位の実行保証を実現します。

## 詳細な内容

### cronguard の設計思想

cronguard は、従来の cron デーモンが抱える「実行遅延の非可視化」「障害時のロストジョブ」「タイムゾーン非対応」といった課題を解決するために設計されました。内部では独自のスケジューリングエンジン **CronGuard Engine (CGE)** を搭載し、ジョブの実行履歴を `pg_stat_cronguard` テーブルに永続化。障害発生時には自動リトライとアラート通知を実現します。

### コア機能

- **マルチタイムゾーン対応の精密スケジューリング**  
  `cronguard.schedule` に `cron_expression`（例: `0 30 4 * * ? * Asia/Tokyo`）と `timezone_offset`（例: `+09:00`）を指定可能。DST 切り替えにも自動追従し、最大 **±50 ミリ秒** の実行誤差に抑えます。

- **ジョブ依存関係グラフと並列実行制御**  
  `dependencies` パラメータに JSON 形式で先行ジョブのリスト（例: `["job_id_001", "job_id_002"]`）を設定。依存解決には **DAG（有向非巡回グラフ）** を採用し、並列実行可能なジョブは最大 **32 スレッド** で同時処理。PostgreSQL の `LISTEN/NOTIFY` を用いてリアルタイムに状態を同期します。

- **障害検知と自動リカバリ**  
  `failure_policy` に `retry`（最大 5 回、指数バックオフ）または `fallback`（代替ジョブを指定）を設定可能。実行失敗時は `cronguard_alerts` テーブルに **UUID 形式のアラート ID** を即時発行し、Slack Webhook またはカスタム HTTP エンドポイントに通知。

### 使い方

1. **CronGuard デーモンの初期化とジョブ登録**  
   ```bash
   cronguard init --db-host=127.0.0.1 --db-port=5432 --db-name=cronguard_db
   cronguard add-job \
     --name="daily_backup" \
     --cron="0 0 3 * * ? * Asia/Tokyo" \
     --command="/usr/local/bin/backup.sh" \
     --timeout=300000 \
     --dependencies='["pre_check_job"]' \
     --failure_policy=retry
   ```

2. **ジョブ実行状況のリアルタイムモニタリング**  
   ```sql
   -- PostgreSQL 上で直接クエリ
   SELECT job_id, status, started_at, finished_at, error_message
   FROM pg_stat_cronguard
   WHERE job_name = 'daily_backup'
   ORDER BY started_at DESC LIMIT 10;
   ```

3. **障害発生時の自動リトライと手動介入**  
   ```bash
   # リトライ上限に達したジョブを手動で再実行
   cronguard force-run --job-id="job_id_001" --override-timeout=600000
   ```

### ベストプラクティス

- **ジョブの冪等性を保証する**  
  すべてのジョブは冪等（idempotent）に設計してください。cronguard はリトライ時に同一ジョブを再実行するため、副作用が重複しないよう `idempotency_key` パラメータを利用し、PostgreSQL の `ON CONFLICT DO NOTHING` と組み合わせることを推奨します。

- **監視テーブルにインデックスを設定する**  
  `pg_stat_cronguard` テーブルの `job_name` と `started_at` に複合インデックスを作成することで、クエリパフォーマンスが **300% 以上** 向上します。特にジョブ数が 10,000 を超える環境では必須です。
