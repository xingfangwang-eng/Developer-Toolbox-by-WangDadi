## 詳細な内容

### Cronguard とは？

Cronguard は、cron ジョブの実行監視・障害検知・自動修復に特化したプロフェッショナルツールです。  
システム運用における「cron のサイレント障害」を根本から解決し、ジョブの成否をリアルタイムで可視化します。

### コア機能

- **ジョブ実行の分散トレースとチェックポイントリカバリ**  
  cron ジョブの各ステップをトレースし、障害発生時には直前のチェックポイントから自動再実行。ログの追跡が不要で、復旧時間を大幅に短縮します。

- **異常ジョブの自動パターン分類とアラートエスカレーション**  
  ジョブの失敗パターンを機械学習で分類。単発エラーと恒常的障害を自動判別し、Slack / PagerDuty / メールへ段階的にエスカレーションします。

- **cron スケジュールの依存関係グラフ可視化**  
  複数ジョブ間の依存関係を DAG（有向非巡回グラフ）で可視化。スケジュール衝突やリソース競合を事前検出し、実行計画の最適化を支援します。

### 使い方

1. **ジョブの登録とヘルスチェック間隔の設定**  
   監視対象の cron ジョブを `cronguard.yml` に定義。`healthcheck_interval: 60` でチェック頻度（秒）を指定し、`timeout: 300` でタイムアウト値を設定。登録後、`cronguard start` で監視デーモンを起動します。

2. **アラートルールのカスタマイズ**  
   `rules/alert.yml` で失敗回数や再試行ポリシーを定義。例：`max_retries: 3`、`retry_delay: 30`、`escalation: pagerduty` を設定。`cronguard validate` で構文チェック後、`cronguard reload` で即時反映。

3. **履歴分析とレポート生成**  
   `cronguard history --last 7d --format json` で直近7日間の全ジョブ実行ログをエクスポート。`cronguard report --type sla` で SLA 遵守率グラフを自動生成。結果は `reports/` ディレクトリに保存され、CI パイプラインと連携可能。

### ベストプラクティス

- **ジョブの実行結果は必ず HTTP ステータスコードで返す**  
  Cronguard は HTTP 200 を正常、それ以外を異常と判定します。シェルスクリプトの終了コードではなく、アプリケーション層で統一されたステータス応答を実装することで、誤検知を防ぎます。

- **ジョブの実行時間にバッファを持たせる**  
  実際の処理時間に加えて `grace_period: 120`（秒）を設定。スケジュール遅延やリソース待ちによる誤アラートを抑制し、本質的な障害のみを検知します。
