# cronguard - CRO_009

## 概要

cronguard は、分散ジョブスケジューリングの信頼性と可視性を極限まで高める、次世代型監査基盤です。cron の実行漏れや重複、タイムアウトをリアルタイムで検知し、SaaS 運用における SLI/SLO の達成を強力に支援します。本ドキュメントでは、cronguard の中核機能、具体的なセットアップ手順、そして実運用で効果を発揮するベストプラクティスを解説します。

## 詳細な内容

### cronguard とは？

cronguard は、従来の cron 管理が抱える「暗黙の失敗」——ログを見なければ気づかないジョブの未実行や、予期せぬリトライによるデータ不整合——を根本から解決するために設計された、監査・アラート特化型ツールです。Kubernetes CronJob や Linux crontab と統合し、各ジョブの実行結果をチェックサム付きで永続化。分散環境でも一貫した監査証跡を提供します。

### コア機能

- **実行履歴のチェックサム検証と改ざん検知**  
  各 cron ジョブの実行結果に対して SHA-256 ハッシュを自動生成し、改ざんやデータ破損を検知。監査ログとしての完全性を保証します。これにより、コンプライアンス要件の厳しい金融系システムでも安心して導入可能です。

- **依存ジョブのトポロジカルソートとクリティカルパス可視化**  
  複数の cron ジョブ間に依存関係がある場合、実行順序を自動で解決。さらに、全体のスケジュールの中で最も遅延リスクが高い「クリティカルパス」を特定し、ダッシュボード上で強調表示します。

- **タイムアウトとジッターの自動補正（Adaptive Scheduler）**  
  ジョブ実行時間が設定されたタイムアウトを超過した場合、次回のスケジュールを動的に調整。また、分散ノード間のクロックスキュー（時刻ずれ）を検出し、ジッターを吸収することで、実行タイミングの精度をミリ秒単位に向上させます。

### 使い方

1. **監査対象の cron ジョブを cronguard に登録する**  
   `cronguard register --name daily-report --command "python3 /opt/report.py" --schedule "0 6 * * *" --timeout 300s`  
   このコマンドで、`daily-report` というジョブを登録。`--timeout` で最大実行時間を設定し、超過時にアラートを発報する閾値を指定します。登録後、cronguard は自動でジョブの実行履歴を収集するバックエンドエージェントをデプロイします。

2. **依存関係を定義し、実行順序を自動解決する**  
   `cronguard depend --job daily-report --after data-fetch --after validation`  
   上記により、`daily-report` は `data-fetch` と `validation` の両方の完了を待ってから実行されるようになります。依存関係は DAG（有向非巡回グラフ）として管理され、循環参照が検出された場合は即座にエラーを返します。

3. **監査レポートをエクスポートし、外部システムと連携する**  
   `cronguard audit --job daily-report --since 2024-01-01 --until 2024-01-31 --format json > audit_report.json`  
   指定期間の全実行履歴を JSON 形式でエクスポート。出力には各実行のタイムスタンプ、所要時間、終了コード、チェックサムが含まれ、SIEM ツール（例：Splunk, Datadog）への取り込みに最適です。

### ベストプラクティス

- **全ジョブにタイムアウトとリトライ上限を設定する**  
  バッチ処理の肥大化や外部APIの応答遅延により、ジョブが無限に実行され続けるリスクを回避するため、cronguard の `--timeout` オプションと `--max-retries` オプションを必ず指定してください。特にデータ移行ジョブでは、リトライ回数を3回以内に制限し、冪等性を担保することを推奨します。

- **監査ログをデータレイクに保存し、定期的に整合性検証を実行する**  
  cronguard が生成するチェックサム付きの実行履歴は、Amazon S3 や Google Cloud Storage などのオブジェクトストレージに長期保存しましょう。月次で全ログのハッシュを再計算し、cronguard の管理テーブルと突き合わせることで、データの改ざんや消失を早期に発見できます。
