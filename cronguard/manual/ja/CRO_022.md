# cronguard - CRO_022

## 概要

cronguard は、分散環境における cron ジョブの実行監視・障害検知・自動リカバリを実現する、プロダクションレディなツールです。本ドキュメントは、cronguard の導入を検討する日本の DevOps エンジニア向けに、そのアーキテクチャと運用ノウハウを解説します。

## コア機能

### 1. 分散ジョブの分散ロックと冪等性保証

cronguard は、Redis または PostgreSQL をバックエンドとした分散ロック機構を内蔵しています。複数のワーカーインスタンスが同一の cron エントリを重複実行するのを防ぎ、ジョブの冪等性を保証します。ロックの有効期限やリトライ戦略は、YAML ベースの設定ファイルで細かくチューニング可能です。

### 2. ジョブ実行履歴の時系列分析と異常検知

全ジョブの実行結果（終了コード、実行時間、標準出力/エラー）を時系列データベースに保存し、過去の実行パターンからの逸脱を自動検知します。例えば、通常 5 秒で完了するジョブが 30 秒以上かかった場合、または終了コードが非ゼロになった場合に、Slack / PagerDuty へのアラートをトリガーできます。閾値はジョブ単位で動的に設定可能です。

### 3. ジョブチェーンと条件付き実行パイプライン

単一の cron 式だけでなく、ジョブ間の依存関係を定義できます。例えば、「ジョブ A の成功後にジョブ B を実行し、ジョブ B が失敗した場合はジョブ C をスキップして管理者に通知する」といった複雑なワークフローを、シンプルな YAML 記法で構築できます。これにより、バッチ処理のパイプライン全体を cronguard で一元管理できます。

## 使い方

### 1. 初期化と設定ファイルの作成

まず、プロジェクトルートで `cronguard init` を実行し、`.cronguard.yml` を生成します。このファイルに、監視対象の cron ジョブを定義します。以下は、データベースのバックアップジョブと、その結果を検証するジョブの例です。

```yaml
jobs:
  backup_db:
    command: "pg_dump -U postgres mydb > /backups/mydb.sql"
    schedule: "0 2 * * *"
    timeout: 300s
    on_failure:
      - notify: slack
      - retry: 2
  verify_backup:
    command: "ls -l /backups/mydb.sql && wc -c /backups/mydb.sql"
    schedule: "0 3 * * *"
    depends_on:
      - backup_db
```

### 2. エージェントの起動とジョブの登録

`cronguard agent start` を実行すると、エージェントがバックグラウンドで起動し、設定ファイルに基づいてすべてのジョブをスケジューリングします。エージェントは、指定された cron 式に従ってジョブを実行し、結果をローカルの SQLite データベースに記録します。マルチノード環境では、`cronguard agent join` でクラスタに参加させ、分散ロックを有効にします。

### 3. ダッシュボードでのリアルタイム監視

`cronguard dashboard` を起動すると、Web UI が localhost:8080 で利用可能になります。ここでは、全ジョブの実行状態（成功/失敗/実行中/スキップ）をリアルタイムで確認できます。各ジョブの実行履歴をクリックすると、標準出力、エラー、実行時間のグラフが表示され、異常値が赤くハイライトされます。また、フィルター機能で「過去 1 時間以内に失敗したジョブ」だけを表示することも可能です。

## ベストプラクティス

### 1. ジョブの実行時間に基づくタイムアウト設定

cronguard では、各ジョブに `timeout` を設定することを強く推奨します。例えば、データベースのフルバックアップが通常 10 分で完了する場合、`timeout: 600s` と設定します。これにより、何らかの理由でジョブがハングした場合でも、cronguard が強制終了し、次のスケジュールに影響を与えません。タイムアウトが発生した場合のアクション（再実行、通知、チェーン内の後続ジョブのスキップ）も同時に定義しておくと、障害時の自動復旧がスムーズです。

### 2. ジョブの結果をメトリクスとして外部監視システムに連携

cronguard は、Prometheus のメトリクスエンドポイントをデフォルトで公開します。`/metrics` エンドポイントには、ジョブごとの実行回数、失敗回数、実行時間のヒストグラムが含まれています。これを Prometheus でスクレイピングし、Grafana で可視化することで、ジョブのパフォーマンス傾向を長期にわたって分析できます。また、アラートルールを設定すれば、例えば「1 時間以内に 3 回以上失敗したジョブがある」場合に自動でインシデントを発行できます。
