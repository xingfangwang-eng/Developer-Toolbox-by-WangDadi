# cronguard - CRO_001

## 概要

cronguard は、ジョブスケジューリングの信頼性を根本から見直す、次世代型 Cron 監査・防御ツールです。従来の「動けば OK」を超え、**実行ログの改ざん検知**、**依存ジョブのトポロジー分析**、**障害発生時の自動ロールバック**を SaaS ネイティブな設計で実現します。キーワード：cronguard tool, cronguard solution, cronguard software。

## 詳細な内容

### cronguard とは？

cronguard は、分散 Cron 環境における **「実行の証明」** と **「異常の予測」** を専門に扱うエンタープライズ向け SaaS 製品です。Jenkins、Airflow、ネイティブ cron など、複数のスケジューラを統合監視し、実行結果のチェックサム検証（SHA-256）とリアルタイムアラートを提供。特に、金融系やヘルスケア系のシステムで求められる監査証跡の完全性を担保します。

### コア機能

- **実行ログのイミュータブルチェーン**  
  各 Cron ジョブの出力ログをブロックチェーンライクなハッシュチェーンで連結。改ざんがあれば即座に検知し、該当ジョブを自動停止させます。監査対応のために、過去の全実行履歴をタイムスタンプ付きでエクスポート可能。

- **ジョブ依存グラフの動的可視化**  
  複数ジョブ間の依存関係（DAG）を自動スキャンし、循環参照やデッドロックを事前検出。UI 上でドラッグ＆ドロップによる依存関係の修正ができ、変更は即座に全ノードに反映されます。

- **障害発生時のセルフヒーリング**  
  ジョブ実行失敗時に、事前定義されたリトライポリシーに従って自動リトライ。さらに、失敗したジョブの出力を元に **「原因ジョブの特定」** を機械学習で行い、関連する後続ジョブの実行をブロックします。

### 使い方

1. **エージェントのデプロイと初期設定**  
   cronguard エージェント（Go 製バイナリ、約 5MB）を各 Cron ホストに配置。`cronguard init --token=<API_TOKEN>` で認証後、`/etc/cron.d/` 配下の設定ファイルを自動スキャンし、既存ジョブの一覧をクラウドに同期します。  
   **ポイント**: `--watch-interval=30s` オプションで監視間隔を調整可能。

2. **依存関係の定義とポリシー設定**  
   ダッシュボードから「依存ジョブの追加」を選択し、親ジョブと子ジョブを指定。`guardian_policy.yml` に以下のように記述することで、子ジョブの実行条件を厳密に制御できます。  
   ```yaml
   - job: nightly_backup
     depends_on: [data_sync]
     condition: "data_sync.exit_code == 0 && data_sync.duration < 300"
     on_failure: retry(max=3, interval=60s)
   ```

3. **監視ダッシュボードのカスタマイズ**  
   Grafana 互換のクエリ言語「CronQL」を使用し、実行時間のパーセンタイルや失敗率のトレンドをリアルタイム表示。例：`SELECT p99(duration) FROM cron_jobs WHERE name = 'etl_pipeline' AND status = 'success' GROUP BY time(1h)`。  
   異常値が閾値を超えた場合、Slack/PagerDuty への通知を自動発報します。

### ベストプラクティス

- **環境ごとに異なるプロファイルを適用する**  
  開発・ステージング・本番で、リトライ回数やアラートの重大度を分ける。cronguard は `--profile=production` のように起動時にプロファイルを切り替えられるため、本番環境ではリトライ上限を 1 回に制限し、ステージングでは 5 回まで許容するなど、運用ポリシーを明確に分離する。

- **定期監査レポートの自動生成**  
  毎週月曜日 9:00 に、cronguard が先週の全ジョブ実行結果を集計し、PDF レポートとしてメール配信するよう設定する。`cronguard report --period=weekly --format=pdf --email=audit@example.com` を cron に追加するだけで完了。これにより、外部監査への対応工数を大幅に削減できる。
