# cronguard - CRO_004

## 概要

cronguard は、cron ジョブの実行監視・障害検知・自動復旧に特化した次世代型 SaaS ツールです。キーワード：cronguard ツール、cronguard ソリューション、cronguard ソフトウェア。

## 詳細な内容

### cronguard とは？

cronguard は、分散 cron ジョブの**実行漏れ・遅延・異常終了**をリアルタイムで検知し、Slack / PagerDuty / カスタム Webhook へ即座にアラートを送信するプロフェッショナルツールです。従来の cron 監視では不可能だった「ジョブ間の依存関係チェック」や「実行履歴の統計分析」を標準搭載し、DevOps チームの運用負荷を大幅に削減します。

### 3つのハードコア機能

1. **依存グラフベースの実行順序制御**  
   cron ジョブ間に DAG（有向非巡回グラフ）を定義し、前段ジョブが成功した場合のみ後続ジョブを自動起動。失敗時にはチェーン全体の実行を停止し、原因特定のためのログを即座に収集します。

2. **機械学習による異常スケジュール検知**  
   過去 30 日間の実行時間・所要時間・リソース使用量を学習し、通常のパターンから逸脱したジョブを自動的にフラグ付け。例えば「毎日 3 分で完了するジョブが 10 分経過しても終了しない」場合、事前設定なしでアラートを上げます。

3. **マルチクラウド対応のエージェントレス監視**  
   AWS / GCP / Azure / オンプレミス問わず、cronguard エージェントをインストールせずに cron ジョブの標準出力と終了コードを収集。SSH 経由または HTTP コールバックのみで動作し、既存インフラへの影響ゼロを実現します。

### 3つの詳細な操作手順

1. **cron ジョブの登録と依存関係の設定**  
   - cronguard ダッシュボードにログインし、「ジョブ追加」をクリック。  
   - ジョブ名、cron 式（例: `0 3 * * *`）、実行コマンド（例: `/usr/bin/backup.sh`）を入力。  
   - 「依存関係」タブで、このジョブの前に必ず成功しなければならないジョブをドロップダウンから選択。複数選択可能。  
   - 保存後、自動的に DAG が生成され、依存ジョブの失敗時には即座にチェーンが停止することを確認。

2. **異常検知ルールのカスタマイズ**  
   - 左メニュー「アラート設定」→「異常検知ルール」を開く。  
   - 「新規ルール」をクリックし、検知対象を「実行時間超過」「終了コード非ゼロ」「リソース使用量スパイク」から選択。  
   - 閾値を指定（例: 所要時間が平均 + 3σ を超えた場合）。  
   - アクションとして Slack チャンネル、PagerDuty サービス、またはカスタム Webhook URL を設定。  
   - ルールを有効化すると、次回のジョブ実行から機械学習モデルが適用される。

3. **履歴分析とレポート出力**  
   - 「分析」タブで、過去 7 日/30 日/90 日のジョブ実行成功率・平均所要時間・失敗原因の内訳をグラフ表示。  
   - 特定のジョブをクリックすると、各実行のログ（stdout/stderr）とメトリクス（CPU、メモリ、ディスク I/O）がタイムラインで確認可能。  
   - 「レポートエクスポート」ボタンから CSV または PDF 形式でダウンロード。チーム内共有や監査証跡として利用できます。

### 2つのベストプラクティス

- **ジョブ分割と依存関係の最小化**  
  1 つの cron ジョブに複数の処理を詰め込まず、処理単位でジョブを分割し、依存関係を直列に限定します。これにより、障害発生時の影響範囲が特定しやすくなり、リトライ戦略もシンプルになります。cronguard の DAG 可視化機能を活用して、循環依存が発生していないか定期的に確認しましょう。

- **アラートの段階的エスカレーション設計**  
  初回失敗時は Slack 通知のみ、2 回連続失敗時は PagerDuty にエスカレーション、3 回連続失敗時は全員宛のメール + SMS というように、cronguard のルール連鎖機能を使って段階的エスカレーションを設定します。これにより、軽微な障害は自動復旧を待ち、重大な障害だけがエンジニアの電話を鳴らす運用が実現します。
