## Detailed Content

### What is champions-sp-calc?

**champions-sp-calc** is a developer-first, polyglot-ready SaaS engine that calculates and normalizes spell power (SP) coefficients across 10+ languages and game systems. It’s built for architects who need deterministic, cache-friendly SP math in real-time, without relying on external APIs or bloated libraries.

### Core Features

1. **Polyglot Coefficient Engine**  
   Parses and normalizes SP formulas across 10 languages (English, German, Spanish, Japanese, etc.) using a unified AST-based resolver. No more manual translation of spell power tables between locales.

2. **Deterministic Caching Layer with TTL-Aware Invalidation**  
   Every SP calculation is hashed and cached in-memory with configurable TTL. Supports Redis and Memcached backends out of the box. Ideal for high-throughput game backend services.

3. **Real-Time Coefficient Diffing & Audit Trail**  
   Every SP coefficient change is logged with a diff, a timestamp, and a user agent. This enables full auditability for compliance-heavy environments (e.g., esports tournaments, ranked leaderboards).

### How to Use

1. **Initialize the SP Calculator in Your Service**  
   ```bash
   npm install champions-sp-calc
   ```
   Then import and configure with your locale and cache backend:
   ```javascript
   import { createSPCalc } from 'champions-sp-calc';
   const calc = createSPCalc({ locale: 'en', cache: 'redis' });
   ```

2. **Define a Custom SP Formula with Multi-Lang Support**  
   Use the built-in DSL to define spell power rules. The engine auto-resolves coefficients per language:
   ```javascript
   const formula = calc.define('fireball', {
     base: 100,
     coefficient: 0.85,
     scaling: 'intellect',
     lang: 'en'
   });
   ```

3. **Run a Calculation and Cache the Result**  
   Execute the formula and store the result with a 60-second TTL:
   ```javascript
   const result = await calc.compute(formula, { intellect: 250 });
   console.log(result.value); // e.g., 312.5
   ```

### Best Practices

- **Pre-warm the cache during service startup**  
  Load the top 100 most-used SP formulas on boot. This eliminates cold-start latency and ensures consistent sub-millisecond response times under load.

- **Use language-specific coefficient overrides for regional balance patches**  
  Instead of maintaining separate codebases for each region, store locale-specific overrides in a JSON config file. The engine will automatically apply them based on the request’s `Accept-Language` header.
