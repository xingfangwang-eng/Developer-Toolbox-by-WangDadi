## Core Features

### 1. Multi‑League Bayesian SP Engine
Unlike generic rating systems, champions-sp-calc applies a **hierarchical Bayesian model** that dynamically adjusts for league parity, home‑field advantage, and player fatigue indices. The engine outputs a calibrated SP curve instead of a single scalar, enabling probabilistic win/loss thresholds across 40+ sport formats.

### 2. Real‑Time Roster Delta Compiler
Ingests live roster changes (injuries, trades, rest days) and recalculates SP in sub‑second latency. The compiler uses a **weighted adjacency matrix** to map player impact coefficients, automatically flagging statistical outliers (e.g., a star player’s +0.32 SP swing due to a matchup mismatch).

### 3. Deterministic Simulation Sandbox
Run Monte Carlo simulations with **seeded reproducibility**—critical for A/B testing prediction models. The sandbox outputs full distribution histograms, confidence intervals (90% / 95% / 99%), and a JSON‑serializable trace log for audit trails or downstream ML pipelines.

## How to Use

### Step 1: Configure League & Roster Payload
Instantiate the calculator with a league configuration object. Every league requires a `parity_factor` (float 0–1) and a `home_advantage_offset`. Roster data must follow the schema below:

```python
from champions_sp_calc import SPCalculator

calc = SPCalculator(
    league={
        "id": "nba_2024",
        "parity_factor": 0.78,
        "home_advantage_offset": 2.3
    },
    roster={
        "players": [
            {"id": "P001", "rating": 91.4, "fatigue_index": 0.12, "injury_multiplier": 0.85},
            {"id": "P002", "rating": 84.7, "fatigue_index": 0.04, "injury_multiplier": 1.0}
        ]
    }
)
```

### Step 2: Run the SP Projection
Execute the main projection method. The engine automatically fetches any live delta from the Roster Delta Compiler (if the `live=True` flag is set). For deterministic results, pass a `seed` integer.

```bash
calc.project(iterations=10_000, seed=42, live=True)
# Returns: {"sp_curve": [0.23, 0.31, 0.45, ...], "confidence_95": [0.22, 0.48]}
```

### Step 3: Export & Visualize the Distribution
The sandbox outputs a serializable JSON trace. Use the built‑in exporter to generate a histogram or feed the raw array into your own visualization layer (e.g., D3.js, Plotly).

```python
trace = calc.export_trace(format="json")
# Save or stream to a dashboard endpoint
with open("sp_distribution.json", "w") as f:
    f.write(trace)
```

## Best Practices

### Practice 1: Tune `parity_factor` with Historical Data
Never hardcode the league parity factor. Use the tool’s built‑in **calibration mode** (`calc.calibrate(historical_games=500)`) to derive the optimal value. A mis‑calibrated parity factor can inflate SP variance by up to 18%—especially in leagues with high roster turnover (e.g., European football).

### Practice 2: Use Seeded Simulations for CI/CD Pipelines
When integrating champions-sp-calc into a CI/CD pipeline (e.g., nightly model retraining), always pass a fixed `seed`. This guarantees reproducible SP outputs across runs, allowing you to **diff‑test** prediction changes without stochastic noise. Store the seed alongside the trace log for full auditability.
