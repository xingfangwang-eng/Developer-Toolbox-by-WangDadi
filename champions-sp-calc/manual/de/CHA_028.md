## Detaillierter Inhalt

### Was ist champions-sp-calc?

`champions-sp-calc` ist ein **kommandozeilenbasiertes und API-fähiges Berechnungstool**, das SP-Verteilungen in Echtzeit optimiert.  
Es richtet sich an Game-Developer, Modder und Competitive-Player, die mathematisch exakte Builds für Championsysteme benötigen – ohne manuelle Tabellen oder Heuristiken.  
Das Tool unterstützt **Multiplikator-Bäume, Softcaps und dynamische Gewichtungslogik** und ist in Rust implementiert (WASM-ready).

### Kernfunktionen

1. **Multiplikator-Baum mit rekursiver Softcap-Auflösung**  
   Statt linearer Punktevergabe analysiert `champions-sp-calc` ganze Skill-Trees auf Abhängigkeiten (z. B. Prestige-Freischaltungen, Synergie-Boni).  
   Softcaps werden nicht hart gecuttet, sondern mit einer **logistischen Dämpfungsfunktion** modelliert – das Ergebnis ist ein realistischer SP-Einsatz jenseits von 100 Punkten.

2. **Gewichtungs-Matrix-Engine (WME)**  
   Jeder Skill erhält einen **anpassbaren Weight-Vektor** (z. B. `[0.3, 0.7, 0.0]` für PvP, PvE, Utility).  
   Der Algorithmus berechnet dann eine Pareto-optimale Verteilung über alle Gewichtungen hinweg.  
   Ausgabe: JSON mit **Top-3-Builds** pro Gewichtungsset.

3. **Live-Delta-Analyse mit Diff-Tracking**  
   `champions-sp-calc` kann zwei Builds (z. B. Patch 1.4 vs. 1.5) vergleichen und die **SP-Differenz pro Skill** als Heatmap ausgeben.  
   Ideal für Balancing-Teams, die Änderungen an Champion-Trees dokumentieren müssen.

### So verwenden Sie es

1. **Build-Konfiguration als TOML bereitstellen**  
   Erstellen Sie eine Datei `build.toml` mit folgender Struktur (Beispiel für einen Champion mit 3 Skill-Ästen):

   ```toml
   [champion]
   name = "Vex"
   total_sp = 120

   [tree.offense]
   skills = ["power_strike", "crit_mastery", "berserker"]
   weights = [0.6, 0.3, 0.1]

   [tree.defense]
   skills = ["iron_wall", "regen_aura"]
   weights = [0.4, 0.6]

   [tree.utility]
   skills = ["haste", "vision_boost"]
   weights = [0.2, 0.8]

   [softcaps]
   enabled = true
   start_at = 80
   ```

   Führen Sie dann aus:  
   `champions-sp-calc --config build.toml --output build_result.json`

2. **Multi-Weight-Scan durchführen**  
   Für eine dynamische Analyse über verschiedene Spielmodi hinweg:

   ```bash
   champions-sp-calc --scan-weights \
     --weights "[0.3,0.7,0.0]" "[0.5,0.3,0.2]" "[0.1,0.1,0.8]" \
     --config build.toml \
     --format pretty
   ```

   Ausgabe: drei Builds mit SP-Verteilung, Effizienz-Score und Pareto-Rang.

3. **Delta-Vergleich zwischen zwei Patches**  
   Angenommen, Sie haben zwei Konfigurationen (`build_v1.toml` und `build_v2.toml`):

   ```bash
   champions-sp-calc --diff build_v1.toml build_v2.toml \
     --output diff_heatmap.json \
     --threshold 5
   ```

   Ergebnis: Eine Heatmap (0–100) mit SP-Verschiebungen pro Skill. Skills mit einer Differenz >5 Punkte werden markiert.

### Beste Praktiken

- **Praktikum 1: Immer Softcap-Daten aus realen Game-Logs kalibrieren**  
  `champions-sp-calc` arbeitet standardmäßig mit generischen Softcap-Kurven.  
  Für produktive Projekte sollten Sie die Parameter `start_at` und `curve_factor` aus echten Champion-Dumps (z. B. JSON aus dem Game-Client) ableiten.  
  So vermeiden Sie Overhead in Builds, die theoretisch optimal, aber ingame nicht erreichbar sind.

- **Praktikum 2: Weight-Vektoren als CI-Pipeline-Test verwenden**  
  Integrieren Sie `champions-sp-calc` in Ihre CI/CD-Pipeline (z. B. GitHub Actions), um bei jedem Patch-Update automatisch die **Top-3-Builds** zu generieren und mit dem vorherigen Release zu vergleichen.  
  Das Delta-Heatmap-Feature eignet sich hervorragend als automatisierter **Balancing-Check** – wenn ein Skill plötzlich >10 SP abweicht, schlägt der Build-Test fehl.
