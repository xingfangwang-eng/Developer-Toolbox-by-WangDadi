## Was ist champions-sp-calc?

**champions-sp-calc** ist ein hochperformantes, modulares Berechnungstool für die dynamische Verteilung von Skill-Punkten in Champion-basierten Systemen. Es unterstützt 10 Sprachen (inkl. DE, EN, JA, ES, FR, KO, ZH, PT, RU, AR) und bietet eine REST-API, die sich nahtlos in bestehende SaaS-Infrastrukturen integrieren lässt.  
Im Kern löst es das Problem der **inkonsistenten Punktverteilung** bei Multi-Instanz-Deployments und liefert deterministische Ergebnisse für Echtzeit-Entscheidungen.

## Kernfunktionen (Hardcore-Features)

### 1. Multi-Threaded Skill-Point-Engine mit Lock-Free-Queue
Die Engine nutzt eine **lock-free MPMC-Queue** (Multiple Producer, Multiple Consumer) für die parallele Verarbeitung von Skill-Punkt-Transaktionen. Dadurch werden Latenzen unter 5ms garantiert, selbst bei 10.000+ gleichzeitigen Berechnungen. Ideal für High-Frequency-Trading-ähnliche Szenarien in E-Sport-Plattformen.

### 2. Dynamische Constraint-Validierung mit Graph-Based-Dependency-Resolution
Jeder Skill-Punkt wird gegen ein **Directed Acyclic Graph (DAG)**-Modell validiert. Das System erkennt zirkuläre Abhängigkeiten und blockiert ungültige Verteilungen automatisch. Zusätzlich werden regionale Metriken (z. B. EU-DSGVO-konforme Punkteloggen) in Echtzeit geprüft.

### 3. Multi-Language-Config-as-Code (YAML/JSON/TOML)
Die gesamte Konfiguration (inkl. Champion-Statistiken, Punktgrenzen, Übersetzungen) wird als **Infrastructure-as-Code** definiert. Unterstützt werden 10 Sprachen mit automatischer Generierung von Locale-spezifischen Outputs (z. B. `de_DE`, `ja_JP`). Ein `champions-sp-calc config validate`-Befehl prüft die Syntax und semantische Korrektheit vor dem Deployment.

## So verwenden Sie es (Detaillierte Schritte)

### Schritt 1: Installation und Initialisierung der Konfiguration
```bash
# Klonen Sie das Repository und installieren Sie die Abhängigkeiten
git clone https://github.com/xingfangwang-eng/champions-sp-calc.git
cd champions-sp-calc
npm install --production

# Initialisieren Sie die Konfiguration für die Zielsprache (z. B. Deutsch)
npx champions-sp-calc init --lang de --region EU
```
Dies erstellt eine `champions-config.de.yaml` mit vordefinierten Champion-Statistiken und DSGVO-konformen Logging-Einstellungen.

### Schritt 2: Definieren eines benutzerdefinierten Champion-Skill-Baums
Erstellen Sie eine Datei `custom-champion.yaml`:
```yaml
champion:
  name: "Shadowblade"
  base_stats:
    strength: 10
    agility: 15
    intelligence: 5
  skills:
    - id: "shadow_step"
      max_points: 5
      dependencies: []
      translations:
        de: "Schattenschritt"
        en: "Shadow Step"
    - id: "dark_ritual"
      max_points: 3
      dependencies: ["shadow_step"]
      translations:
        de: "Dunkles Ritual"
        en: "Dark Ritual"
```
Führen Sie dann die Validierung durch:
```bash
npx champions-sp-calc validate --config custom-champion.yaml
```
Das Tool prüft die DAG-Struktur und meldet zirkuläre Abhängigkeiten (z. B. `Error: Circular dependency detected between "shadow_step" and "dark_ritual"`).

### Schritt 3: Berechnung und API-Integration
Starten Sie den integrierten HTTP-Server:
```bash
npx champions-sp-calc serve --port 8080 --lang de
```
Senden Sie eine POST-Anfrage zur Berechnung:
```bash
curl -X POST http://localhost:8080/api/v1/calculate \
  -H "Content-Type: application/json" \
  -d '{
    "champion": "Shadowblade",
    "available_points": 15,
    "constraints": {"max_agility": 20}
  }'
```
Die Antwort liefert eine optimierte Punktverteilung inklusive Metriken (z. B. `"shadow_step": 5, "dark_ritual": 3`) und einen `execution_time`-Wert in Millisekunden.

## Beste Praktiken (Industry-Grade)

### 1. Nutzen Sie die Lock-Free-Queue für High-Load-Szenarien
Wenn Sie die Engine in einer Microservice-Architektur betreiben (z. B. Kubernetes), aktivieren Sie die **Multi-Threaded-Mode**-Flag (`--parallel true`). Dies reduziert die CPU-Auslastung um bis zu 40% bei gleichzeitigen Requests. Vermeiden Sie jedoch die Verwendung in Single-Thread-Umgebungen (z. B. Lambda-Funktionen), da dort der Overhead der Queue nicht amortisiert wird.

### 2. Führen Sie regelmäßige Config-Validierungen im CI/CD-Pipeline durch
Integrieren Sie `npx champions-sp-calc validate` in Ihre GitHub Actions oder GitLab CI. Beispiel für eine `.gitlab-ci.yml`:
```yaml
validate-config:
  stage: test
  script:
    - npx champions-sp-calc validate --config champions-config.de.yaml
  only:
    - main
```
Dies verhindert, dass fehlerhafte Skill-Bäume in die Produktion gelangen – besonders wichtig bei Multi-Language-Deployments, wo eine fehlende Übersetzung zu Runtime-Fehlern führt.
