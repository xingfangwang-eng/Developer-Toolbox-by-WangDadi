## Detaillierter Inhalt

### Was ist champions-sp-calc?

**champions-sp-calc** ist ein hochprofessionelles Tool zur präzisen Berechnung und Optimierung von **Spieler-Synergie-Punkten** (SP) in kompetitiven Multiplayer-Spielen.  
Es analysiert Echtzeit-Match-Daten, historische Performance und Team-Kompositionen, um die optimale SP-Verteilung für maximale Win-Raten zu ermitteln – ideal für eSport-Teams und Hardcore-Gamer.

---

### Kernfunktionen

1. **Multi-Variable SP-Optimierung**  
   Berechnet simultan bis zu 12 Einflussfaktoren (z. B. Champion-Kontersynergien, Map-Bonifikationen, Patch-Änderungen) und generiert eine dynamische SP-Rangliste pro Match-up.

2. **Temporal-Decay-Analyse**  
   Integriert einen gewichteten Zeitverfall: Ältere Matches (älter als 14 Tage) werden automatisch heruntergewichtet, sodass nur aktuelle Meta-Daten in die SP-Kalkulation einfließen.

3. **Live-API-Integration**  
   Koppelt sich direkt an offizielle Game-APIs (Riot, Valve, Blizzard) und aktualisiert SP-Werte in Echtzeit – ohne manuelles Nachladen.

---

### So verwenden Sie es

1. **API-Key konfigurieren & Datenpull initialisieren**  
   - Richten Sie einen **OAuth2-Client** in Ihrem Game-Entwicklerportal ein.  
   - Fügen Sie den Key in die `config.yaml` ein:  
     ```yaml
     api_key: "YOUR_API_KEY"
     game: "league_of_legends"
     decay_rate: 0.85
     ```  
   - Führen Sie `champions-sp-calc --pull-history` aus, um die letzten 200 Matches zu laden.

2. **SP-Berechnung mit benutzerdefinierten Gewichten**  
   - Nutzen Sie das CLI-Flag `--weights` für individuelle Gewichtungen:  
     ```bash
     champions-sp-calc --weights "counter:0.4, farm:0.3, kda:0.2, vision:0.1"
     ```  
   - Das Tool gibt eine **tiefensortierte SP-Heatmap** aus – inklusive Konfidenzintervall pro Champion.

3. **Ergebnisse in ein Dashboard exportieren**  
   - Verwenden Sie `--export json` für die Integration in Grafana oder Tableau:  
     ```bash
     champions-sp-calc --export json --output sp_ranking.json
     ```  
   - Das JSON enthält Metadaten wie `patch_version`, `decay_factor` und `timestamp` – ideal für automatisierte Workflows.

---

### Beste Praktiken

1. **Patch-zyklus-gesteuerte Neukalibrierung**  
   Führen Sie nach jedem größeren Game-Patch (z. B. LoL Patch 14.10) eine vollständige SP-Neuberechnung durch.  
   Setzen Sie dazu `--reset-cache` und `--force-recalc` – so vermeiden Sie veraltete Synergiewerte, die auf alten Daten basieren.

2. **Team-spezifische SP-Profile**  
   Erstellen Sie separate Konfigurationsprofile für verschiedene Teams (z. B. `team_alpha.yaml`, `team_beta.yaml`).  
   Jedes Profil speichert eigene `decay_rate` und `weights` – das ermöglicht maßgeschneiderte SP-Optimierung für unterschiedliche Spielstile (Aggressiv vs. Kontrolliert).
