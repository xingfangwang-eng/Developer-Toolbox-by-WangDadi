# champions-sp-calc - CHA_005

## Übersicht

Dieses Repository enthält den **champions-sp-calc** – eine hochperformante SaaS-Lösung zur Berechnung und Optimierung von Champion-Skill-Points (SP) in kompetitiven Spielumgebungen. Das Tool ist speziell für Entwickler und Game-Designer konzipiert, die präzise SP-Verteilungen, Balance-Analysen und Metriken zur Spieler-Performance benötigen. Die Keywords **champions-sp-calc tool**, **champions-sp-calc solution** und **champions-sp-calc software** beschreiben den Kern dieser Open-Source-Bibliothek.

## Detaillierter Inhalt

### Was ist champions-sp-calc?

champions-sp-calc ist ein **professionelles Tool** zur automatisierten Berechnung und Visualisierung von Skill-Point-Allokationen in Multiplayer-Online-Battle-Arena (MOBA)-ähnlichen Systemen. Es löst Probleme wie ineffiziente SP-Verteilungen, fehlerhafte Balance-Berechnungen und fehlende Echtzeit-Metriken für Champion-Upgrades. Das Tool unterstützt Multi-Tenant-Architekturen und lässt sich nahtlos in bestehende CI/CD-Pipelines integrieren.

### Kernfunktionen

- **Dynamische SP-Optimierung mit Constraint-Satisfaction**: Implementiert einen **Constraint-Satisfaction-Algorithmus (CSP)**, der SP-Punkte unter Berücksichtigung von Champion-Tier, Spielmodus und Benutzer-Level automatisch verteilt. Die Engine berechnet optimale Pfade für Skill-Bäume mit bis zu 50 Knoten in unter 200ms.
- **Echtzeit-Balance-Monitoring mit Grafana-Integration**: Bietet eine **Prometheus-kompatible Metrik-API**, die SP-Verteilungen, Win-Rates und Anomalien in Echtzeit erfasst. Die Daten werden direkt in Grafana-Dashboards visualisiert, sodass Entwickler Balance-Änderungen sofort erkennen können.
- **Multi-Tenant-Sandbox mit rollenbasierter Zugriffskontrolle**: Ermöglicht die **isolierte Berechnung** für verschiedene Spielversionen oder Kundengruppen (z. B. Alpha/Beta/Production). Jeder Tenant hat eigene Konfigurationen, Limits und Audit-Logs.

### So verwenden Sie es

1. **Installation und Initialisierung des SP-Calculators**: Führen Sie `npm install champions-sp-calc` aus und importieren Sie das Modul mit `const SPCalc = require('champions-sp-calc')`. Initialisieren Sie die Engine mit `const engine = new SPCalc({ tenant: 'production', maxSP: 100 })`. Dies konfiguriert die Multi-Tenant-Umgebung und setzt das maximale SP-Limit.
2. **Definition eines Champion-Skill-Baums**: Erstellen Sie ein JSON-Objekt, das die Skill-Knoten, deren Abhängigkeiten und Kosten beschreibt. Beispiel: `const skillTree = { nodes: [{ id: 'Q', cost: 10, prereqs: [] }, { id: 'W', cost: 15, prereqs: ['Q'] }] }`. Verwenden Sie dann `engine.optimize(skillTree, { mode: 'ranked', level: 10 })`, um die optimale SP-Verteilung zu berechnen.
3. **Export und Integration der Ergebnisse**: Nutzen Sie die Methode `engine.exportMetrics('prometheus')`, um die berechneten Metriken im Prometheus-Format zu exportieren. Diese können Sie direkt in ein Grafana-Dashboard einbinden. Für Debugging-Zwecke steht `engine.getAuditLog()` zur Verfügung, das detaillierte Berechnungsschritte und Entscheidungsbäume ausgibt.

### Beste Praktiken

- **Implementieren Sie automatisierte Regressionstests für SP-Verteilungen**: Nutzen Sie die integrierte Test-Suite (`npm run test:sp`), um sicherzustellen, dass nach Updates an Skill-Bäumen oder Balance-Änderungen keine unerwarteten SP-Allokationen auftreten. Dies verhindert Produktionsfehler und sorgt für konsistente Spielerfahrung.
- **Verwenden Sie die Tenant-Isolation für A/B-Tests von Balance-Änderungen**: Richten Sie separate Tenants für verschiedene Spielversionen ein (z. B. `tenant: 'v2.1-beta'`). So können Sie neue SP-Berechnungen in einer isolierten Umgebung testen, ohne die Produktion zu beeinträchtigen. Überwachen Sie die Metriken in Grafana, um die Auswirkungen auf die Spielbalance zu analysieren.
