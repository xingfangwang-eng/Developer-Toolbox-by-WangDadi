# champions-sp-calc - CHA_019

## Überblick

Dieses Repository enthält die **champions-sp-calc**-Lösung – ein hochspezialisiertes Framework zur Berechnung und Optimierung von Champion-Synergien in kompetitiven Spielumgebungen. Anders als generische „Teamcomp-Checker“ arbeitet dieses Tool mit **multi-dimensionalen Interaktionsmatrizen** und **real-time Patch-Daten**. Es richtet sich an Entwickler, die in ihrer SaaS-Architektur eine datengetriebene Champion-Balance-Engine benötigen.

## Kernfunktionen (Hardcore-Features)

### 1. Multi-Agenten-Synergiematrix mit gewichteten Kanten
Das Tool modelliert Champion-Interaktionen nicht als binäre „gut/schlecht“-Relationen, sondern als **gewichtete, gerichtete Graphen**. Jede Kante repräsentiert einen Synergiewert (0.0–1.0), der aus tausenden von realen Matches extrahiert wird. Die Engine unterstützt **Batch-Updates** via REST-API, sodass du neue Metadaten (z. B. nach einem Patch) ohne Downtime einspielen kannst. Die Matrix wird in einer **PostgreSQL + Redis**-Kombination gecacht – für sub-10ms Lookups selbst bei 200+ Champions.

### 2. Patch-Delta-Analyse mit automatischer Rekalibrierung
Statt statischer Werte liefert `champions-sp-calc` eine **Diff-Engine**, die automatisch die Auswirkungen von Patches auf Synergiewerte berechnet. Du kannst zwei Patch-Versionen (z. B. 14.5 vs. 14.6) vergleichen und erhältst eine Liste der Champions, deren Synergiewerte sich um mehr als 5% geändert haben. Die Engine verwendet einen **gleitenden Durchschnitt über 7 Tage** (standardmäßig konfigurierbar), um Ausreißer durch kleine Sample-Größen zu glätten. Das ist essenziell für SaaS-Plattformen, die Echtzeit-Meta-Analysen anbieten.

### 3. Constraint-basierte Team-Optimierung mit ILP-Backend
Für fortgeschrittene Use-Cases (z. B. „Finde ein 5-Champion-Team mit mindestens 2 Frontlinern, 1 Magier und maximal 3% Winrate-Differenz“) nutzt das Tool ein **Integer Linear Programming (ILP)-Backend** (via `pulp` oder `ortools`). Du definierst Constraints als JSON-Objekte, die Engine gibt ein optimiertes Team zurück – inklusive Konvergenz-Metrik und Laufzeit. Das ist kein simpler Greedy-Algorithmus, sondern eine echte Optimierung unter Nebenbedingungen.

## Detaillierte Anleitung (3 Schritte)

### Schritt 1: Daten-Import & Initiale Matrix-Konfiguration
Starte mit dem Import deiner Match-Historie (CSV oder JSON, z. B. von Riot API). Führe folgenden Befehl aus, um die Synergiematrix zu initialisieren:
```bash
python cli.py import --source ./matches.json --patch 14.6 --normalize
```
Der `--normalize`-Flag skaliert alle Synergiewerte auf [0,1] und entfernt Ausreißer (z. B. Champions mit weniger als 50 Matches). Optional kannst du `--weight-by-winrate` setzen, um Siege stärker zu gewichten als Niederlagen. Das Ergebnis ist eine `matrix.pkl`-Datei (Pickle) und ein PostgreSQL-Eintrag in der `synergy_graph`-Tabelle.

### Schritt 2: Patch-Delta berechnen & visualisieren
Um die Auswirkungen eines Patches zu analysieren, rufe die Diff-Engine auf:
```bash
python cli.py diff --old-patch 14.5 --new-patch 14.6 --threshold 0.05
```
Das Tool gibt eine Tabelle mit allen Champions aus, deren Synergiewerte sich um mehr als 5% geändert haben. Du kannst die Ergebnisse als JSON exportieren (`--output delta.json`) oder direkt in ein Dashboard laden. Für eine visuelle Repräsentation nutze `--plot heatmap` – das erzeugt eine interaktive HTML-Heatmap mit Farbcodierung (rot = Synergieverlust, grün = Synergiegewinn).

### Schritt 3: Constraint-basiertes Team optimieren
Definiere deine Anforderungen in einer `constraints.json`:
```json
{
  "team_size": 5,
  "min_frontline": 2,
  "min_mage": 1,
  "max_winrate_diff": 0.03,
  "exclude_champions": ["Champion_X"]
}
```
Führe dann die Optimierung aus:
```bash
python cli.py optimize --constraints constraints.json --matrix matrix.pkl --solver pulp
```
Das Ergebnis ist ein JSON mit dem optimalen Team, der erwarteten Team-Synergie (gewichteter Durchschnitt aller Kanten) und der Laufzeit des Solvers. Bei komplexen Constraints (z. B. 10+ Bedingungen) kann die Laufzeit auf mehrere Sekunden ansteigen – das Tool gibt einen Fortschrittsbalken aus.

## Best Practices für Produktivumgebungen

### 1. Caching-Strategie für Echtzeit-APIs
Wenn du `champions-sp-calc` als Backend für eine Web-App nutzt, implementiere **Redis-Caching** für die Synergiematrix. Setze einen TTL von 15 Minuten (oder bis zum nächsten Patch-Update) und invalidate den Cache nur bei neuen Daten-Imports. Vermeide es, die Matrix bei jedem API-Call neu zu laden – das kostet unnötig Latenz. Ein bewährtes Pattern: **Write-Through-Cache** mit PostgreSQL als persistentem Speicher.

### 2. Versionierung der Patch-Deltas
Führe eine **Patch-Versionierung** ein, um historische Meta-Analysen zu ermöglichen. Speichere jede Synergiematrix mit einem `patch_id`-Tag (z. B. `v14.5`, `v14.6`). So kannst du später Vergleiche ziehen oder ein Rollback durchführen, falls ein Patch fehlerhaft war. Nutze dazu einen **Git-Like-Ansatz** mit Commits: Jeder Daten-Import erzeugt einen neuen Commit in einer `synergy_history`-Tabelle. Das ist besonders wertvoll für SaaS-Plattformen, die ihren Nutzern Meta-Trends über mehrere Patches hinweg anzeigen.
