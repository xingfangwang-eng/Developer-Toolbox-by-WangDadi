# champions-sp-calc - CHA_016

## Übersicht

`champions-sp-calc` ist ein spezialisiertes Kommandozeilen-Tool zur deterministischen Berechnung von Skill-Point-Optimierungen in kompetitiven Champion-basierten Spielsystemen. Es eliminiert manuelle Rechenfehler und liefert mathematisch fundierte SP-Verteilungen für maximale Effizienz unter variablen Constraints (Level-Cap, Item-Synergien, Skill-Rotationen). Entwickelt für Spielentwickler, Balanceteams und Hardcore-Min-Maxer, die auf reproduzierbare, scriptbare Analyse angewiesen sind. Kein GUI-Bloat, nur rohe Rechenpower.

## Detaillierter Inhalt

### Was ist champions-sp-calc?

`champions-sp-calc` ist ein deterministischer SP-Optimierer, der auf einem gewichteten Entscheidungsbaum (Weighted Decision Tree, WDT) basiert. Statt heuristischer Näherungen verwendet es eine vollständige Enumeration aller gültigen SP-Verteilungen für einen gegebenen Champion und filtert diese nach benutzerdefinierten Zielfunktionen (z.B. maximale Burst-DPS, minimale Cooldown-Überlappung, optimale Mana-Effizienz pro Rotation). Das Tool liest Champion-Stat-Blobs (JSON/YAML) und generiert eine sortierte Liste optimaler Build-Pfade. Es ist vollständig offline-fähig und für CI/CD-Pipelines konzipiert.

### Kernfunktionen

1. **Deterministischer Branch-and-Prune-Optimierer**  
   Implementiert einen rekursiven Branch-and-Prune-Algorithmus, der den gesamten SP-Baum (max. 30 Ebenen) durchläuft, aber Äste mit garantiert suboptimalen Partialwerten (z.B. Überschreitung des Soft-Caps für eine Primärstat) sofort kappt. Ergebnis: Reduzierung der Rechenzeit von exponentiell auf polynomiell (O(n³) im Worst-Case). Ausgabe als sortiertes JSON-Array mit Metadaten (Rest-SP, Effizienz-Score).

2. **Dynamische Constraint-Engine mit Runtime-Override**  
   Unterstützt Constraints wie „Maximiere Crit-Chance unter Level 12“, „Minimiere Skill-Cooldown bei gleichzeitiger Mana-Regeneration > 5/s“ oder „Erzwinge mindestens 3 Punkte in Skill X vor Level 6“. Constraints werden als JSON-Array übergeben und zur Laufzeit ausgewertet. Overrides können per CLI-Flag (–override) gesetzt werden, ohne die Konfigurationsdatei zu ändern.

3. **Multi-Champion-Batch-Modus mit Differenzanalyse**  
   Ermöglicht den parallelen Vergleich beliebig vieler Champion-Konfigurationen. Gibt für jede Kombination einen Delta-Report aus, der die prozentuale Abweichung der SP-Effizienz zwischen Champions visualisiert. Ideal für Balanceteams, die Patch-Änderungen simulieren oder „Was-wäre-wenn“-Szenarien durchspielen. Ausgabe als CSV oder Markdown-Tabelle.

### So verwenden Sie es

1. **Champion-Konfiguration als JSON bereitstellen**  
   Erstellen Sie eine Datei `champion_config.json` mit folgendem Schema:
   ```json
   {
     "champion": "Aurelion_Sol",
     "base_stats": { "hp": 620, "mana": 350, "ad": 58, "ap": 0 },
     "skills": [
       { "id": "Q", "base_damage": 70, "sp_scaling": 0.65, "max_level": 5 },
       { "id": "W", "base_damage": 50, "sp_scaling": 0.40, "max_level": 5 },
       { "id": "E", "base_damage": 30, "sp_scaling": 0.25, "max_level": 3 },
       { "id": "R", "base_damage": 150, "sp_scaling": 0.90, "max_level": 3 }
     ],
     "constraints": { "maximize": "burst_dps", "min_level": 6, "max_level": 18 }
   }
   ```
   Validieren Sie das Schema mit `champions-sp-calc validate champion_config.json`.

2. **Optimierungslauf starten mit spezifischen Flags**  
   Führen Sie den Optimierer mit expliziter Angabe der Zielfunktion und der gewünschten Anzahl Top-Ergebnisse aus:
   ```bash
   champions-sp-calc optimize --config champion_config.json --objective burst_dps --top 5 --output results.json
   ```
   Das Tool gibt eine Liste der 5 besten SP-Verteilungen zurück, inklusive des berechneten Effizienz-Scores und der Rest-SP. Beispielausgabe (gekürzt):
   ```json
   [
     { "distribution": "Q:5, W:3, E:1, R:2", "score": 0.94, "rest_sp": 2 },
     { "distribution": "Q:5, W:2, E:2, R:2", "score": 0.91, "rest_sp": 3 }
   ]
   ```

3. **Multi-Champion-Vergleich im Batch-Modus**  
   Erstellen Sie ein Verzeichnis mit mehreren Champion-Konfigurationen und starten Sie den Batch-Modus:
   ```bash
   champions-sp-calc batch --input-dir ./champions/ --objective sustain_dps --output-dir ./results/ --format csv
   ```
   Das Tool generiert für jede Champion-Kombination eine Delta-Analyse. Beispiel-Report (Auszug):
   ```
   Champion A vs B: +12.3% sustain_dps bei gleichem SP-Invest
   Champion A vs C: -4.1% sustain_dps, aber +8.7% burst_dps
   ```

### Beste Praktiken

1. **Constraint-basierte Regressionstests in CI/CD integrieren**  
   Nutzen Sie `champions-sp-calc` als Teil Ihrer automatisierten Testsuite, um nach jedem Patch oder Balancing-Update die SP-Effizienz aller Champions zu validieren. Definieren Sie feste Constraints (z.B. „Maximiere Burst-DPS bei Level 6“) und vergleichen Sie die aktuellen Scores mit einer Baseline. Ein Abfall >5% sollte einen Build-Failure auslösen. Dies verhindert unerkannte Nerfs oder Buffs, die das Spielgleichgewicht stören.

2. **SP-Verteilungen als Teil der Champion-Dokumentation versionieren**  
   Generieren Sie für jeden Champion eine optimale SP-Verteilung für die drei häufigsten Spielsituationen (Early Game, Mid Game, Late Game) und checken Sie diese als JSON in Ihr Repository ein. Entwickler können so schnell nachvollziehen, welche Verteilung für ein bestimmtes Szenario empfohlen wird. Kombinieren Sie dies mit einer automatisierte Prüfung, ob die gespeicherten Verteilungen noch mit der aktuellen Champion-Konfiguration übereinstimmen.
