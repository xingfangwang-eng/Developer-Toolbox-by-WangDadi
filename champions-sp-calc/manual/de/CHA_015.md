## Detaillierter Inhalt

### Was ist champions-sp-calc?

**champions-sp-calc** ist ein professionelles, modulares SaaS-Tool, das speziell für die **dynamische Berechnung und Verteilung von Skill-Points** in Champion-basierten Spielsystemen entwickelt wurde. Es löst das Kernproblem der manuellen, fehleranfälligen SP-Verwaltung, indem es eine Echtzeit-API, eine deterministische Berechnungs-Engine und eine integrierte Konfliktanalyse bietet. Das Tool ist als Microservice konzipiert und lässt sich nahtlos in bestehende Game-Backends (Node.js, Go, Python) integrieren.

### Kernfunktionen

1.  **Deterministische SP-Allokations-Engine mit Multi-Constraint-Solver**
    Die Engine berechnet die optimale SP-Verteilung unter Berücksichtigung von Hard-Caps (Level, Rarity), Soft-Caps (Synergie-Boni) und benutzerdefinierten Constraints (z. B. „Mindestens 10 SP in Defensive“). Der Solver arbeitet mit einem **gewichteten, iterativen Algorithmus** (ähnlich einem Simplex-Verfahren), der in unter 50ms selbst bei 500+ Champions ein Ergebnis liefert. Dies eliminiert manuelle Rechenfehler und garantiert eine mathematisch optimale Verteilung.

2.  **Echtzeit-Synergie-Graph-Analyse**
    Statt nur einzelne SP-Werte zu berechnen, analysiert das Tool den gesamten **Synergie-Graphen** eines Teams. Es identifiziert automatisch Breakpoints (z. B. „SP-Level 15 bei Champion A schaltet Team-Buff für Champion B frei“) und visualisiert diese als Directed Acyclic Graph (DAG). Die Analyse wird über eine **WebSocket-API** in Echtzeit gestreamt, sodass UI-Updates ohne Polling erfolgen – kritisch für Live-Strategie-Tools.

3.  **Integrierter Konflikt- und Prioritäts-Manager**
    Bei widersprüchlichen SP-Zielen (z. B. zwei Champions benötigen denselben seltenen Skill-Gem) generiert das System einen **Konfliktbaum** mit Prioritätsstufen. Der Manager bietet drei Modi: *Auto-Resolve* (nach globalen Regeln), *Manual-Override* (mit Live-Vorschau der Konsequenzen) und *Cost-Benefit-Analyse* (zeigt die Auswirkungen auf den Gesamt-DPS/Healing). Dies ist essenziell für Competitive- und Esports-Teams.

### So verwenden Sie es

1.  **Initialisierung der Konfiguration und API-Integration**
    ```bash
    npm install champions-sp-calc
    ```
    Importieren Sie das Modul in Ihren Service und initialisieren Sie den `SPCalcClient` mit Ihrer API-Key und der Spielversion. **Wichtig:** Setzen Sie den `cacheTTL`-Parameter auf `300` (Sekunden), um die Performance zu optimieren, ohne auf frische Daten zu verzichten. Verwenden Sie für Production-Umgebungen zwingend den `clusterMode: true`, um die Last auf mehrere Worker-Threads zu verteilen.

    ```javascript
    const { SPCalcClient } = require('champions-sp-calc');
    const client = new SPCalcClient({
      apiKey: process.env.SP_CALC_KEY,
      gameVersion: '4.2.1',
      cacheTTL: 300,
      clusterMode: true
    });
    ```

2.  **Durchführung einer SP-Optimierung mit benutzerdefinierten Constraints**
    Rufen Sie die `optimize`-Methode auf. Übergeben Sie ein Array von Champion-IDs und ein Constraints-Objekt. Im folgenden Beispiel erzwingen wir, dass der Champion `Aatrox` mindestens 15 SP in `Q` erhält, während der gesamte SP-Pool auf 100 begrenzt ist. Das Ergebnis ist ein JSON-Objekt mit der finalen Verteilung und einem Log der Konfliktlösungen.

    ```javascript
    const result = await client.optimize({
      champions: ['Aatrox', 'Yasuo', 'Jinx'],
      constraints: {
        'Aatrox': { minSP: 15, skill: 'Q' },
        totalSP: 100
      }
    });
    console.log(result.distribution); // Gibt die SP-Verteilung aus
    console.log(result.conflictLog); // Zeigt gelöste Konflikte
    ```

3.  **Aktivierung der Echtzeit-WebSocket-Verbindung für Live-Updates**
    Für eine dynamische UI (z. B. während eines Drafts) öffnen Sie einen WebSocket-Stream. Der Client sendet automatisch Updates bei jeder Änderung der SP-Verteilung oder bei neuen Synergie-Breakpoints. Nutzen Sie die `on('synergyBreakpoint')`-Methode, um spezifische Events zu triggern – z. B. um einen visuellen Buff-Indikator in der UI zu setzen.

    ```javascript
    const stream = client.streamOptimization({
      champions: ['Aatrox', 'Yasuo', 'Jinx'],
      live: true
    });
    stream.on('synergyBreakpoint', (data) => {
      console.log(`Breakpoint erreicht: ${data.champion} - ${data.skill}`);
      // UI-Update auslösen
    });
    ```

### Beste Praktiken

1.  **Caching-Strategie für SP-Berechnungen implementieren**
    Da die Berechnungen deterministisch sind, sollten Sie die Ergebnisse auf Champion-Ebene cachen. Nutzen Sie den integrierten Redis-Adapter (`client.useCache('redis', { host: 'localhost', port: 6379 })`), um die Latenz für wiederholte Abfragen drastisch zu senken. **Achtung:** Invalidieren Sie den Cache bei jedem Game-Patch (z. B. via Webhook), da sich SP-Kosten oder Synergien ändern können. Eine falsche Cache-Validierung führt zu inkorrekten Builds.

2.  **Fehlerbehandlung bei Konflikten mit dem Priority-Manager**
    Konfigurieren Sie den Priority-Manager immer mit einem Fallback-Modus. Setzen Sie `conflictMode: 'cost-benefit'`, um bei unauflösbaren Konflikten automatisch die Variante mit dem geringsten DPS-Verlust zu wählen. Dokumentieren Sie alle manuellen Overrides in einem Log (z. B. `client.getConflictHistory()`), um später eine Regression im Build-Verhalten nachvollziehen zu können. Dies ist besonders in Team-Umgebungen mit mehreren Build-Architekten kritisch.
