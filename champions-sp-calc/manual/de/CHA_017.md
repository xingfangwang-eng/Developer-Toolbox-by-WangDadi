## Detaillierter Inhalt

### Was ist champions-sp-calc?

**champions-sp-calc** ist ein professionelles, in Rust und TypeScript implementiertes Backend-Tool zur Berechnung von **Skill-Points (SP) und Champion-Skalierung** in komplexen, mehrsprachigen Spiel-Ökosystemen. Es löst das Problem der **Latenz-optimierten, konsistenten Zustandsberechnung** über mehrere Datenquellen (APIs, lokale Cache, externe Datenbanken) hinweg – ohne dass eine manuelle Synchronisation erforderlich ist. Die Architektur basiert auf einem **Event-Driven Microservice-Modell** mit integriertem **Circuit Breaker** und **Retry-Logik**, um Ausfälle von Drittanbieter-APIs elegant zu handhaben.

### Kernfunktionen (Hardcore-Features)

1.  **Multi-Language SP Normalization Engine**  
    Das Tool normalisiert Champion-Skill-Points aus 10+ Sprachen (DE, EN, JA, ES, FR, IT, PT, RU, KO, ZH) in ein einheitliches **ISO-639-1-basiertes SP-Mapping**. Es verwendet ein **Custom NLP-Pipeline**-Modul, das auf **spaCy** und **fastText** aufbaut, um regionale Abkürzungen und Slang-Begriffe (z. B. „Ulti“ vs. „Ultimate“) automatisch zu erkennen und in kanonische SP-Werte zu übersetzen.

2.  **Deterministische SP-Skalierung mit Elo-Weighted Multipliers**  
    Statt einfacher linearer Formeln verwendet champions-sp-calc eine **gewichtete Elo-basierte Skalierungsmatrix**. Jeder Champion erhält einen **dynamischen Multiplikator**, der auf den aktuellen Elo-Rankings der Spieler basiert (abrufbar über die **Riot Games API**). Die Berechnung erfolgt in **O(log n)** Zeitkomplexität und unterstützt **Batch-Processing** für tausende Champions gleichzeitig.

3.  **Echtzeit-Cache-Invalidation mit WebSocket-Push**  
    Das Tool integriert einen **Redis-basierten Cache**, der durch **WebSocket-Events** von der externen API invalidated wird. Sobald ein Champion-Update (z. B. Patch 14.10) veröffentlicht wird, sendet champions-sp-calc automatisch ein **Delta-Update** an alle verbundenen Clients – ohne Polling. Die Latenz liegt unter **50 ms** für die erste Aktualisierung.

### So verwenden Sie es (Detaillierte Anleitung)

1.  **Initiale Konfiguration und Datenquellen-Binding**  
    Definieren Sie in der `config.yaml` die **primären und sekundären Datenquellen** (z. B. Riot Games API, lokale JSON-Dumps, externe Redis-Instanz). Nutzen Sie das **CLI-Tool `sp-init`**, um die Sprach-Pipelines zu booten:  
    ```bash
    champions-sp-calc init --langs de,en,ja,es --sources riot,local --cache redis://localhost:6379
    ```  
    Das Tool validiert automatisch die API-Keys und testet die Verbindung zu allen Quellen. Fehlgeschlagene Quellen werden in einer **Fallback-Liste** gespeichert.

2.  **Batch-SP-Berechnung mit Elo-Integration**  
    Führen Sie eine Massenberechnung für alle Champions durch:  
    ```bash
    champions-sp-calc batch --input champions.json --output sp_results.json --elo-multiplier
    ```  
    Das Tool ruft parallel die Elo-Daten von der Riot API ab (Rate-Limiting wird automatisch durch **Token-Bucket-Algorithmus** verwaltet) und wendet die gewichteten Multiplikatoren an. Die Ausgabe enthält für jeden Champion:  
    - `base_sp`: Basis-Skill-Points (normalisiert)  
    - `elo_multiplier`: Aktueller Elo-Faktor  
    - `final_sp`: Endgültige SP (base_sp * elo_multiplier)  
    - `language_confidence`: Konfidenz-Score der Spracherkennung (0.0–1.0)

3.  **Echtzeit-WebSocket-Integration für Live-Updates**  
    Starten Sie den WebSocket-Server mit:  
    ```bash
    champions-sp-calc serve --ws-port 8080 --cache-invalidation true
    ```  
    Verbinden Sie sich mit einem Client (z. B. via `wscat`):  
    ```bash
    wscat -c ws://localhost:8080
    ```  
    Senden Sie ein **Subscribe-Event** für einen Champion:  
    ```json
    {"action": "subscribe", "champion_id": "Aatrox", "lang": "de"}
    ```  
    Bei jedem externen Update (z. B. Patch-Änderung) erhalten Sie automatisch ein **Delta-Objekt** mit den neuen SP-Werten. Der Server sendet nur die geänderten Felder (z. B. `{"champion_id": "Aatrox", "new_sp": 450, "delta": +15}`).

### Beste Praktiken (Industry Best Practices)

1.  **Regionen-spezifische Elo-Caching-Strategie**  
    Vermeiden Sie unnötige API-Aufrufe, indem Sie die Elo-Daten **pro Region** (EUW, NA, KR, etc.) in einem **separaten Redis-Shard** cachen. Setzen Sie einen **TTL von 5 Minuten** für aktive Champions und **30 Minuten** für inaktive. Nutzen Sie den **Cache-Aside-Pattern** mit **Write-Behind**, um die Datenkonsistenz zu gewährleisten. Dies reduziert die API-Last um bis zu 80% bei gleichzeitiger Echtzeit-Genauigkeit.

2.  **Sprach-Pipeline-Monitoring mit Prometheus**  
    Integrieren Sie das **Prometheus-Metrics-Endpoint** (`/metrics`) in Ihre CI/CD-Pipeline. Überwachen Sie spezifisch:  
    - `sp_language_confidence_histogram`: Verteilung der Konfidenz-Scores pro Sprache  
    - `sp_elo_multiplier_latency_ms`: Latenz der Elo-Abfragen  
    - `sp_cache_hit_ratio`: Cache-Trefferquote (Sollwert > 0.95)  
    Setzen Sie **Alerts** auf Basis von **P99-Latenz > 200 ms** oder **Cache-Hit-Ratio < 0.85**, um frühzeitig auf Performance-Engpässe zu reagieren.
