## Detaillierter Inhalt

### Was ist champions-sp-calc?

`champions-sp-calc` ist ein hochperformantes, CLI-basiertes Tool für Entwickler und eSport-Analysten, das die komplexe SP-Verwaltung in Multiplayer-Spielen automatisiert. Es eliminiert manuelle Fehlerkalkulationen und bietet eine deterministische SP-Prognose basierend auf aktuellen Patch-Notes, Champion-Synergien und Gegner-Matchups. Die Architektur ist modular aufgebaut und unterstützt Multi-Threading für Bulk-Operationen.

### Kernfunktionen

1. **Deterministischer SP-Simulator mit Monte-Carlo-Tiefenanalyse**  
   Simuliert 10.000+ mögliche SP-Verteilungen pro Champion unter Berücksichtigung von Item-Builds, Rune-Pfaden und dynamischen Buff/Debuff-Zyklen. Das Ergebnis ist eine Heatmap der optimalen SP-Allokation für jede Spielphase (Early/Mid/Late Game).

2. **Echtzeit-Meta-API-Integration**  
   Direkte Anbindung an offizielle Game-APIs (Riot, Valve, Blizzard) zur automatischen Aktualisierung von Champion-Balances, Item-Änderungen und Patch-Notes. Der Algorithmus gleicht diese Daten mit einer internen Datenbank ab, die über 500.000 gespielte Matches umfasst.

3. **Cross-Platform Export Engine mit JSON/YAML/CSV-Support**  
   Generiert maschinenlesbare SP-Konfigurationen für Build-Tools, CI/CD-Pipelines und Datenanalyse-Frameworks. Inklusive integrierter Validierung gegen Schemas (JSON Schema, OpenAPI 3.0) zur Vermeidung von Inkonsistenzen.

### So verwenden Sie es

1. **Initialisierung und Konfiguration des Projekts**  
   Klonen Sie das Repository und führen Sie `npm install` aus. Erstellen Sie eine `sp-config.yaml` mit Ihren Champion-Daten und Spielparametern. Nutzen Sie `champions-sp-calc init` zur automatischen Generierung einer Basis-Konfiguration inklusive Standard-Item-Sets.

2. **Durchführung einer Batch-SP-Berechnung**  
   Verwenden Sie `champions-sp-calc batch --input ./matches.json --output ./results.csv --threads 8`. Das Tool parst Ihre Match-Historie, berechnet für jede Kombination (Champion vs. Gegner-Champion) die optimale SP-Verteilung und exportiert die Ergebnisse als CSV. Für Debugging-Zwecke aktivieren Sie `--verbose` zur Ausgabe von Logs mit Timestamps und SP-Deltas.

3. **Integration in eine CI/CD-Pipeline**  
   Fügen Sie `champions-sp-calc validate --config ./sp-config.yaml --strict` in Ihren Pre-Commit-Hook oder GitHub Action Workflow ein. Dies stellt sicher, dass jede Änderung an Champion-Daten oder SP-Werten automatisch auf Konsistenz geprüft wird, bevor sie in den Main-Branch gemergt wird.

### Beste Praktiken

- **Versionierung der SP-Konfigurationen**  
  Führen Sie Ihre `sp-config.yaml` und alle generierten SP-Dateien in Git. Nutzen Sie semantische Versionierung (SemVer) für Champion-Updates, um Breaking Changes in Patches nachvollziehen zu können. Empfohlen wird ein separater Branch `sp-data` für manuelle Anpassungen.

- **Automatisierte Regressionstests**  
  Implementieren Sie Nightly-Builds, die `champions-sp-calc test` ausführen. Dieses Feature vergleicht die aktuellen SP-Berechnungen mit einem festgelegten Baseline-Datensatz (z.B. 1000 zufällige Matches) und meldet Abweichungen > 2% als Warnung. So erkennen Sie frühzeitig, ob ein Patch Ihre SP-Strategie beeinträchtigt.
