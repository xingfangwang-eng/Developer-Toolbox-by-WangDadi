## Detaillierter Inhalt

### Was ist champions-sp-calc?

**champions-sp-calc** ist ein hochperformantes, modulares CLI- und API-Tool, das speziell für die Berechnung von **Skill Point (SP) Verteilungen** in Multiplayer-Online-Battle-Arena (MOBA) und RPG-Spielen entwickelt wurde. Es löst das Problem der manuellen, fehleranfälligen Optimierung von Champion-Builds, indem es algorithmisch die effizienteste SP-Allokation unter Berücksichtigung von Cooldown-Reduktion, Mana-Kosten und Synergie-Effekten berechnet.

### Kernfunktionen

1.  **Multi-Varianten-Optimierung mit Constraint-Satisfaction**  
    Das Tool implementiert einen **Constraint-Satisfaction-Algorithmus**, der nicht nur die maximale Schadensausbeute, sondern auch sekundäre Constraints wie Mana-Regeneration, Lebensentzug und Skill-Synergien berücksichtigt. Es berechnet in Echtzeit die optimale SP-Verteilung für bis zu 5 Champions gleichzeitig, basierend auf benutzerdefinierten Gewichtungen.

2.  **Automatisierte Build-Simulation mit Monte-Carlo-Backtesting**  
    champions-sp-calc führt **Monte-Carlo-Simulationen** durch, um die Wahrscheinlichkeit eines erfolgreichen Builds unter variablen Spielbedingungen (z.B. Gegner-Typ, Team-Komposition) zu validieren. Das Ergebnis ist eine Heatmap der SP-Verteilung, die die Erfolgsrate pro Skill-Punkt visualisiert.

3.  **Native Integration in CI/CD-Pipelines via REST-API**  
    Das Tool bietet eine **RESTful API** mit Swagger-Dokumentation, die es ermöglicht, SP-Berechnungen direkt in Continuous-Integration-Workflows (z.B. GitHub Actions, GitLab CI) zu integrieren. Dies erlaubt automatische Build-Validierung vor jedem Release und stellt sicher, dass Champion-Balancing-Updates nicht zu suboptimalen SP-Verteilungen führen.

### So verwenden Sie es

1.  **Konfiguration des Constraint-Modells**  
    Definieren Sie Ihre Champion-spezifischen Constraints in einer `sp_config.yaml`. Beispiel: `max_mana: 500`, `cooldown_reduction: 0.2`, `synergy_bonus: 0.15`. Nutzen Sie das Flag `--validate` um die Konsistenz Ihrer Constraints zu prüfen, bevor die Berechnung startet.

    ```bash
    champions-sp-calc --config ./configs/ahri_sp.yaml --validate
    ```

2.  **Durchführung der Multi-Varianten-Optimierung**  
    Führen Sie die Berechnung mit dem Befehl `--optimize` aus. Das Tool gibt eine JSON-Ausgabe zurück, die die optimale SP-Verteilung sowie die zugehörigen Metriken (DPS, Mana-Effizienz) enthält. Nutzen Sie `--output-format=csv` für die Integration in Excel oder Tableau.

    ```bash
    champions-sp-calc --optimize --champion ahri --config ./configs/ahri_sp.yaml --output-format json
    ```

3.  **Integration in CI/CD-Pipeline**  
    Fügen Sie folgenden Schritt in Ihre `.gitlab-ci.yml` ein, um automatische SP-Validierung vor dem Build durchzuführen:

    ```yaml
    sp-validation:
      script:
        - champions-sp-calc --validate --config ./configs/ahri_sp.yaml
        - champions-sp-calc --optimize --champion ahri --config ./configs/ahri_sp.yaml --output-format json > sp_report.json
      artifacts:
        paths: [sp_report.json]
    ```

### Beste Praktiken

1.  **Versionierung der Constraint-Dateien**  
    Da Champion-Balancing und Skill-Synergien sich ständig ändern, empfiehlt es sich, die `sp_config.yaml` Dateien in einem separaten Git-Repository zu versionieren. Verknüpfen Sie diese mit den Release-Tags Ihrer Spiele-Engine, um reproduzierbare Builds zu gewährleisten.

2.  **Nutzung des Batch-Modus für große Datensätze**  
    Für die Optimierung von mehr als 10 Champions gleichzeitig aktivieren Sie den **Batch-Modus** mit dem Flag `--batch-size=50`. Dies reduziert die API-Latenz um bis zu 60% und ist ideal für automatisierte Nightly-Builds oder Regressionstests.
