## Detaillierter Inhalt

### Was ist champions-sp-calc?

`champions-sp-calc` ist ein professionelles, containerisiertes CLI-Tool zur Berechnung von Skill-Point-Verteilungen und Champion-Synergien. Es nutzt eine hochperformante C++-Engine für die Kernlogik und eine RESTful API-Schnittstelle für die Integration in CI/CD-Pipelines. Das Tool ist speziell für die Verarbeitung von Massendaten in kompetitiven Gaming-Umgebungen optimiert.

### Kernfunktionen

1.  **Dynamische SP-Gewichtung mit Echtzeit-Patch-Analyse:**  
    Das Tool integriert einen Algorithmus, der automatisch die Gewichtung von Skill Points basierend auf den neuesten Champion-Buffs/Nerfs aus offiziellen Patch-Notes anpasst. Es verwendet einen `weighted moving average` über die letzten 5 Patches, um Meta-Verschiebungen sofort abzubilden.

2.  **Multi-Threaded Synergie-Matrix-Berechnung:**  
    Statt einfacher linearer Additionen berechnet `champions-sp-calc` eine nicht-lineare Synergie-Matrix. Diese bewertet Interaktionen zwischen Champions (z. B. „Shield-Break + Execute“) mittels eines proprietären `Interaction-Impact-Score (IIS)`. Die Berechnung ist für 16-Kern-Prozessoren optimiert und reduziert die Rechenzeit um 70% im Vergleich zu Single-Thread-Lösungen.

3.  **Deterministischer Cache mit TTL-basierter Invalidation:**  
    Jede SP-Kalkulation wird in einem Redis-Backend gecached. Der Cache verwendet einen `Time-To-Live (TTL)` Mechanismus, der sich dynamisch an die Stabilität des aktuellen Metas anpasst (z. B. 600s nach einem Major-Patch, 3600s in stabilen Phasen). Dies verhindert redundante Berechnungen und senkt die API-Latenz auf unter 5ms.

### So verwenden Sie es

1.  **Initialisierung des Config-Objekts mit Patch-Flags:**  
    Starten Sie die Berechnung, indem Sie ein JSON-Config-Objekt an die CLI übergeben. Nutzen Sie das `--patch-version` Flag, um das Tool auf eine spezifische Spielversion zu fixieren.  
    **Befehl:**  
    `champions-sp-calc --config ./configs/tournament_meta.json --patch-version 13.10 --output-format json`

2.  **Durchführung einer Batch-Analyse mit CSV-Export:**  
    Für Massenanalysen (z. B. 10.000 Champion-Kombinationen) verwenden Sie den Batch-Modus. Das Tool liest eine Input-CSV mit Champion-IDs und gibt eine angereicherte CSV mit SP-Empfehlungen und Synergie-Scores aus.  
    **Befehl:**  
    `champions-sp-calc --batch-mode --input ./data/champion_pool.csv --output ./results/optimized_sp.csv --threads 8`

3.  **Integration in eine REST-API (Docker-Container):**  
    Starten Sie den integrierten Flask-Server, um die Funktionalität als Microservice bereitzustellen. Nutzen Sie den Endpunkt `/api/v1/calculate` mit einem POST-Request, der ein JSON-Array von Champion-Daten enthält.  
    **Docker-Befehl:**  
    `docker run -p 5000:5000 -e REDIS_URL=redis://cache:6379 champions-sp-calc:latest`  
    **cURL-Beispiel:**  
    `curl -X POST http://localhost:5000/api/v1/calculate -H "Content-Type: application/json" -d '[{"champion_id": 15, "current_sp": 120}]'`

### Beste Praktiken

- **Praxis 1: Nutzung des deterministischen Caches für CI/CD-Pipelines**  
    Integrieren Sie `champions-sp-calc` in Ihre CI/CD-Pipeline (z. B. GitHub Actions). Nutzen Sie den Cache, um bei jedem Build nicht erneut die gesamte Synergie-Matrix berechnen zu müssen. Setzen Sie einen `CACHE_KEY` basierend auf dem Git-Commit-Hash, um sicherzustellen, dass Änderungen am Code sofort neu berechnet werden.

- **Praxis 2: Regionale Meta-Profile für Turniere**  
    Erstellen Sie separate Config-Profile für verschiedene Regionen (EU, NA, KR). Das Tool unterstützt das Laden von regionalen Gewichtungen (z. B. `--region-profile eu`). Dies ist essentiell für professionelle Teams, die sich auf unterschiedliche Meta-Umgebungen vorbereiten müssen.
