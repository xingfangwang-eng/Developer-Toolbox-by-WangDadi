# champions-sp-calc - CHA_012

## Übersicht

Dieses Repository enthält **champions-sp-calc**, ein spezialisiertes CLI-Tool zur Berechnung von **Skill-Punkten (SP)** in kompetitiven Spielumgebungen. Optimiert für Entwickler, die präzise Metriken für Champion-Balancing, Turnier-APIs oder Echtzeit-Analysen benötigen. Keywords: `champions-sp-calc tool`, `champions-sp-calc solution`, `champions-sp-calc software`.

## Detaillierter Inhalt

### Was ist champions-sp-calc?

**champions-sp-calc** ist ein in Rust implementiertes, hochperformantes Tool, das SP-Werte auf Basis von Multi-Faktor-Gewichtungen (KDA, Schadensoutput, Crowd-Control-Dauer, Map-Objective-Beteiligung) berechnet. Es ersetzt rechenintensive manuelle Formeln durch eine deterministische, konfigurierbare Engine – ideal für CI/CD-Pipelines oder lokale Entwickler-Workflows.

### Kernfunktionen (Hardcore-Features)

1. **Multi-Koeffizienten-Matrix mit dynamischer Gewichtung**  
   Statt statischer Formeln verwendet das Tool eine JSON-konfigurierbare Matrix aus bis zu 12 Koeffizienten (z. B. `kill_weight`, `assist_decay`, `cs_per_min_bonus`). Änderungen an der Meta eines Spiels lassen sich ohne Code-Neukompilierung via `--matrix`-Flag einspielen.

2. **Batch-Modus für Replay-Dumps (JSON-Lines)**  
   champions-sp-calc verarbeitet ganze Replay-Dumps im JSON-Lines-Format. Mit `--batch` und `--output csv` generieren Sie aus 10.000+ Spielzügen in unter 2 Sekunden eine SP-Rangliste – inklusive Perzentil-Ranking und Varianzanalyse pro Champion.

3. **Deterministische Seed-basierte Simulation für „What-If“-Szenarien**  
   Über `--simulate` lassen sich hypothetische Spielverläufe (z. B. „Was, wenn Champion X 30 % mehr Schaden macht?“) mit festem Seed reproduzieren. Ergebnis: SP-Delta-Werte, die direkt in Balancing-Patches einfließen können.

### So verwenden Sie es (Detaillierte Schritte)

1. **Installation und Konfiguration der Gewichtungsmatrix**  
   Laden Sie das Binary via `cargo install champions-sp-calc` oder direkt aus den Releases. Erstellen Sie eine `matrix.json` mit Ihren Koeffizienten:  
   ```json
   {
     "kill_weight": 3.0,
     "death_penalty": -1.5,
     "assist_decay": 0.8,
     "objective_bonus": 2.0
   }
   ```  
   Validieren Sie die Matrix mit `champions-sp-calc --validate matrix.json`. Das Tool gibt bei Inkonsistenzen (z. B. negative Gesamtgewichtung) einen detaillierten Fehlerbericht aus.

2. **Batch-Verarbeitung eines Replay-Dumps**  
   Angenommen, Sie haben eine `replays.jsonl`-Datei mit Zeilen wie:  
   ```json
   {"champion": "Aatrox", "kills": 5, "deaths": 2, "assists": 7, "cs": 200}
   ```  
   Führen Sie aus:  
   ```bash
   champions-sp-calc --batch replays.jsonl --matrix matrix.json --output csv --top 10
   ```  
   Das Ergebnis ist eine CSV mit Spalten: `champion, total_sp, percentile, variance`. Perfekt für automatisierte Reporting-Skripte.

3. **Simulation eines Balancing-Szenarios**  
   Um den Effekt einer Buff-Änderung zu testen, nutzen Sie:  
   ```bash
   champions-sp-calc --simulate replays.jsonl --matrix matrix.json --modifier "kill_weight:3.5" --seed 42
   ```  
   Das Tool gibt ein SP-Delta pro Champion aus (z. B. `Aatrox: +12.3 SP`). Mit `--diff-only` sehen Sie nur Champions mit signifikanten Änderungen (>5 SP).

### Beste Praktiken (Industry Standards)

- **Matrix-Versionierung im Repository**  
  Lagern Sie Ihre `matrix.json` unter Versionskontrolle (z. B. `configs/v1.3-meta.json`). So können Sie SP-Berechnungen für historische Patches reproduzieren und in CI/CD automatisiert testen. Empfehlung: Führen Sie bei jedem Matrix-Update einen `--validate`-Check in der Pipeline aus.

- **Kombination mit Echtzeit-Telemetrie**  
  champions-sp-calc lässt sich via `stdin`-Pipe in Live-Datenströme einbinden. Beispiel: Nutzen Sie `tail -f game_events.jsonl | champions-sp-calc --stream --matrix live_matrix.json`, um SP-Werte während eines Turniers sekundengenau zu aktualisieren. Dies ist besonders wertvoll für Overlay-Systeme in Streaming-Setups.
