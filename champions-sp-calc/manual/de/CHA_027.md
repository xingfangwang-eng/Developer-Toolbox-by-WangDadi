## Detaillierter Inhalt

### Was ist champions-sp-calc?

champions-sp-calc ist ein professionelles Tool, das speziell für die Berechnung und Optimierung von Champions-Skill-Punkten in komplexen Spielsystemen entwickelt wurde. Es ermöglicht Entwicklern, präzise Simulationen durchzuführen und Balance-Anpassungen in Echtzeit vorzunehmen.

### Kernfunktionen

1. **Dynamische Skill-Tree-Visualisierung mit Echtzeit-Recalc**  
   champions-sp-calc rendert interaktive Skill-Bäume, die bei jeder Änderung eines Parameters (z. B. Level, Rarity, Synergie-Bonus) automatisch alle abhängigen SP-Werte neu berechnen. Die Engine nutzt einen graphbasierten Abhängigkeitsgraphen, sodass keine manuellen Aktualisierungen erforderlich sind.

2. **Multi-Champion-Synchronisation mit Constraint-Solver**  
   Das Tool unterstützt die gleichzeitige Berechnung von bis zu 10 Champions. Ein integrierter Constraint-Solver stellt sicher, dass globale Ressourcen (z. B. Gesamt-SP-Pool, Limit pro Klasse) eingehalten werden. Bei Überschreitung werden automatisch Optimierungsvorschläge generiert.

3. **Export als JSON-Schema mit Versionierungs-Hash**  
   Jede Berechnung kann als validiertes JSON-Schema exportiert werden, das einen SHA-256-Hash enthält. Dies ermöglicht eine reproduzierbare Integration in CI/CD-Pipelines oder externe Balancing-Tools. Der Hash garantiert Integrität und Nachvollziehbarkeit.

### So verwenden Sie es

1. **Konfiguration des Champion-Pools**  
   Definieren Sie in der Datei `config/champions.json` die Basisparameter jedes Champions: `base_sp`, `growth_rate` und `synergy_matrix`. Nutzen Sie das integrierte CLI-Tool `champions-sp-calc init`, um eine leere Vorlage zu generieren. Beispiel:  
   ```bash
   champions-sp-calc init --template pvp_balanced
   ```

2. **Simulation mit benutzerdefinierten Constraints**  
   Führen Sie die Berechnung mit dem Befehl `champions-sp-calc simulate` aus. Übergeben Sie optional einen `--constraint`-Parameter, um z. B. die maximale SP-Differenz zwischen Champions zu begrenzen:  
   ```bash
   champions-sp-calc simulate --constraint max_sp_diff=50
   ```  
   Das Tool gibt eine detaillierte Log-Ausgabe mit jedem Recaculation-Schritt aus.

3. **Export und Validierung**  
   Nutzen Sie den Befehl `champions-sp-calc export --format json --output result.json`. Anschließend können Sie mit `champions-sp-calc verify result.json` die Integrität des Exports überprüfen. Der Hash wird automatisch gegen die aktuelle Konfiguration geprüft.

### Beste Praktiken

- **Praxis 1: Nutzen Sie die Synergie-Matrix für dynamische Balance**  
  Definieren Sie in der Synergie-Matrix nicht nur positive, sondern auch negative Synergien (z. B. `"type_advantage": -0.15`). Dies verhindert, dass bestimmte Champion-Kombinationen ungewollt übermächtig werden. Die Engine berücksichtigt diese Werte bei jeder Neuberechnung.

- **Praxis 2: Integrieren Sie champions-sp-calc in Ihre CI/CD-Pipeline**  
  Fügen Sie einen Schritt in Ihre GitHub Actions oder GitLab CI ein, der nach jedem Merge die SP-Kalkulation automatisch ausführt und bei Abweichungen von definierten Schwellenwerten (z. B. `max_sp_deviation=10`) einen Build-Failure auslöst. So stellen Sie sicher, dass Balancing-Änderungen nie ungetestet in die Produktion gelangen.
