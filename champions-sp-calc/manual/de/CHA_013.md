## Detaillierter Inhalt

### Was ist champions-sp-calc?

champions-sp-calc ist ein **deterministischer, pipeline-basierter Rechner**, der nicht nur rohe Spell-Power-Werte aggregiert, sondern auch **dynamische Champion-Interaktionen**, **Item-Set-Rotationen** und **Rune-Overlays** in Echtzeit simuliert. Es handelt sich um ein CLI- und API-first Tool, das auf Sub-Millisekunden-Latenz optimiert ist und direkt in CI/CD-Pipelines integriert werden kann.

### Kernfunktionen

1. **Dynamische Synergie-Matrix (DSM)**  
   Eine gewichtete, graphbasierte Datenstruktur, die alle Champion-Kombinationen (z. B. `Aatrox + Soraka` vs. `Aatrox + Yuumi`) auf Basis von Spell-Wechselwirkungen, Cooldown-Reduktion und Mana-Sustain analysiert. Die Matrix wird mittels *Floyd-Warshall*-Algorithmus auf einem 160×160-Champion-Graphen berechnet und liefert einen **Synergie-Score** (0–100) in Echtzeit.

2. **Item-Set-Rotation & Build-Optimizer (ISRO)**  
   Simuliert bis zu 10.000 Item-Kombinationen pro Sekunde unter Berücksichtigung von **Unique-Passives**, **Mythic-Boni** und **Gold-Effizienz**. Der Optimizer verwendet einen *genetischen Algorithmus* (Population: 500, Generationen: 50), um das Pareto-Optimum zwischen Burst-DPS, Sustain und Tankiness zu finden. Ausgabe als JSON-Schema für direkte API-Nutzung.

3. **Rune-Overlay-Engine (ROE)**  
   Ein auf *Constraint Satisfaction Problem (CSP)* basierender Rune-Konfigurator, der nicht nur primäre und sekundäre Pfade optimiert, sondern auch **adaptive Force-Breaks** (z. B. „Wenn Rune X aktiv, dann deaktiviere Rune Y“) berücksichtigt. Die Engine gibt eine **Rune-Hash-ID** aus, die in externen Tools (z. B. Mobalytics, U.GG) importiert werden kann.

### So verwenden Sie es

1. **Installation & Initialisierung der lokalen Datenbank**  
   ```bash
   git clone https://github.com/xingfangwang-eng/champions-sp-calc.git
   cd champions-sp-calc
   npm install --production && npx sp-calc init --patch 14.10
   ```
   Der Befehl `init` lädt die aktuellste Champion-, Item- und Rune-Data-Dragon-Datenbank (validiert via SHA-256) und erzeugt eine lokale SQLite-Instanz. Der Parameter `--patch` erlaubt das Pinnen auf eine spezifische Game-Version für reproduzierbare Ergebnisse.

2. **Synergie-Scan mit benutzerdefinierten Constraints**  
   ```bash
   npx sp-calc scan --champions "Aatrox,Soraka,Yasuo" --role "Top,Support,Mid" --constraints "mana_regen>5,cdr>30"
   ```
   Der Scanner durchläuft die Synergie-Matrix und filtert nach den angegebenen Rollen sowie harten Constraints (z. B. Mana-Regeneration > 5, Cooldown-Reduktion > 30 %). Ausgabe als Tabelle mit den Top-5 Champion-Kombinationen inkl. Synergie-Score und Item-Vorschlag.

3. **API-Export für externe Dashboards**  
   ```bash
   npx sp-calc export --format json --output ./build/synergy_matrix.json
   ```
   Exportiert die vollständige Synergie-Matrix als JSON-Datei (ca. 25 KB) inkl. Metadaten (Patch-Version, Timestamp). Dieses JSON kann direkt in ein Grafana-Dashboard oder eine React-basierte UI eingebunden werden. Optional mit `--compressed` für Gzip-Ausgabe.

### Beste Praktiken

1. **CI/CD-Integration mit Cache-Strategie**  
   Binden Sie `sp-calc scan` in Ihre GitHub Actions oder GitLab CI ein. Nutzen Sie den `--cache` Parameter, um die Synergie-Matrix nach dem ersten Lauf als Artifact zu speichern. Das reduziert die Build-Zeit von 12 s auf unter 0,5 s bei nachfolgenden Runs. Beispiel:  
   ```yaml
   - name: Champion Synergy Check
     run: npx sp-calc scan --champions "${{ inputs.champions }}" --cache ./build/.sp-calc-cache
   ```

2. **Datenvalidierung mittels Hash-Check**  
   Implementieren Sie einen Pre-Commit-Hook, der vor jedem Push die Integrität der lokalen Data-Dragon-Datenbank prüft. Verwenden Sie `sp-calc validate --hash` um sicherzustellen, dass Ihre Berechnungen auf dem offiziellen Riot-Data-Dragon basieren. Das verhindert inkonsistente Ergebnisse durch veraltete oder manipulierte Daten.
