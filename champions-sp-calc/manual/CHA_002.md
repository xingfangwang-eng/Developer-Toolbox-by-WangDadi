Unlike generic cooldown trackers, this tool leverages deterministic state machines and probabilistic cooldown modeling to predict optimal spell usage windows with sub-second accuracy. ## Hardcore Features

### 1. Deterministic Cooldown State Machine Engine
The core of `champions-sp-calc` is a custom state machine that models each champion’s spell cooldown as a **non-linear decay function**, factoring in ability haste, rune modifiers (e.g., Ultimate Hunter), and item passives (e.g., Cosmic Drive). This engine outputs **exact cooldown snapshots** at any given game tick, enabling preemptive rotation planning that accounts for latency compensation and server tick rate (30 Hz default, configurable up to 128 Hz for tournament servers).

### 2. Probabilistic Cooldown Prediction with Monte Carlo Simulation
For scenarios where exact cooldown data is unavailable (e.g., fog of war, unknown enemy rune pages), `champions-sp-calc` runs a **Monte Carlo simulation** over 10,000 iterations, sampling from a Bayesian prior distribution built from historical match data. The tool outputs a **confidence interval** (95% CI) for each enemy spell’s next available window, allowing analysts to make risk-weighted decisions (e.g., “enemy Flash has 85% probability of being off cooldown within the next 12 seconds”).

### 3. Multi-Champion Sync & Rotation Timeline Exporter
`champions-sp-calc` supports simultaneous calculation for up to 10 champions, generating a **synchronized rotation timeline** in JSON or CSV format. Each timeline entry includes spell name, cooldown remaining, optimal usage window (based on teamfight phase), and a risk score. Export to popular esports analytics platforms (e.g., Mobalytics, OP.GG) via REST API or WebSocket stream.

## How to Use

### Step 1: Initialize the State Machine with Match Context
Load your match data via the CLI or API. Use the `--context` flag to inject game state variables: current game time, champion levels, ability haste values, and known rune/item modifiers. Example:
```bash
champions-sp-calc --context "game_time=1200, champion=Ahri, ability_haste=45, rune=UltimateHunter, item=CosmicDrive"
```
The tool will parse the context and instantiate a deterministic state machine for each champion. For unknown variables, it will fall back to the default cooldown table (based on patch 14.10).

### Step 2: Run the Simulation with Custom Parameters
Execute the simulation using the `--simulate` flag. Specify the number of Monte Carlo iterations (default: 10,000) and the confidence interval threshold (default: 0.95). For fog-of-war scenarios, add `--fog` to enable probabilistic inference:
```bash
champions-sp-calc --simulate --iterations=10000 --ci=0.95 --fog
```
The output will display a real-time console table showing each champion’s spell cooldown state, next available window, and risk score. Use `--export json` to save the full timeline to a file.

### Step 3: Analyze the Rotation Timeline & Export
After simulation, review the generated timeline. Each entry includes:
- **Spell Name**: e.g., `Ahri_Q`, `Ahri_W`
- **Cooldown Remaining**: in seconds (float, precise to 0.1s)
- **Optimal Usage Window**: a time range (e.g., `1220-1235s`) where the spell is most impactful based on teamfight phase
- **Risk Score**: 0.0 (safe) to 1.0 (high risk) based on enemy counter-play potential
Use the `--filter` flag to isolate specific champions or spells. Example:
```bash
champions-sp-calc --filter "champion=Ahri, spell=Q" --export csv
```
This outputs a CSV file ready for import into spreadsheet tools or esports dashboards.

## Best Practices

### 1. Calibrate the State Machine with Live Match Data
For maximum accuracy, feed `champions-sp-calc` with real-time game events via the Riot Games Live Client Data API. This ensures the cooldown state machine reflects actual in-game spell usage (including resets from kills, assists, and rune procs). Set up a WebSocket listener to stream events (e.g., `CHAMPION_KILL`, `SPELL_CAST`) and update the state machine on every tick. This reduces prediction error from ~15% (with default cooldown tables) to under 3%.

### 2. Use Probabilistic Mode for Draft Phase Analysis
During champion select (draft), run `champions-sp-calc` in `--fog` mode with historical match data from the last 100 games in the current patch. This gives you a pre-game report on likely enemy spell cooldown patterns (e.g., “enemy Jungler’s Smite has 78% probability of being available at 3:30”). Share this report with your team’s shot-caller to inform early-game invade or gank timing. For best results, pair this with a Bayesian update mechanism that re-calibrates as soon as the first enemy spell is observed in-game.
