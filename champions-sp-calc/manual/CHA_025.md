# champions-sp-calc - CHA_025

## Overview

This is a **purpose-built, locale-agnostic SaaS utility** designed for high-stakes competitive intelligence and SEO-driven market analysis. The `champions-sp-calc` tool is engineered to parse, normalize, and rank sponsorship performance data across multilingual ecosystems—critical for enterprise-level digital asset management and cross-border PPC optimization.

## Core Features

### 1. Multi-Locale Sponsorship Elasticity Score (SES)
This isn’t just a calculator; it’s a **cross-market normalization engine**. The tool ingests raw sponsorship data (impressions, CTR, conversion lag) from 10+ language markets and outputs a **Sponsorship Elasticity Score (SES)** —a proprietary metric that quantifies how a brand’s digital sponsorship responds to local SEO volatility. It auto-adjusts for language-specific SERP features (e.g., Google Shopping in DE vs. Naver Shopping in KR).

### 2. Latency-Aware Attribution Decay Model
Most tools treat attribution as linear. Ours uses a **time-decay, latency-aware model** that factors in the natural lag between a sponsorship impression and a conversion, specifically tuned for B2B SaaS buying cycles (30–90 days). It cross-references this against crawl frequency and indexation delays from Googlebot, Bingbot, and Yandex.

### 3. Competitive Gap Heatmap via NLP
We scrape and vectorize competitor sponsorship copy across 10 languages using a BERT-based multilingual encoder. The tool then generates a **Gap Heatmap**—a visual matrix showing where your competitors are over-investing in low-ROI keywords (e.g., "cheap" or "free") while you can dominate high-intent, low-competition terms (e.g., "enterprise compliance tool in Japan").

## How to Use

### Step 1: Inject Raw Sponsorship Data via API or CSV
Start by pulling your raw sponsorship performance data. The tool accepts a structured JSON payload via REST API (or a flat CSV for manual testing). Each entry must include: `market_locale` (e.g., `en-US`, `de-DE`, `ja-JP`), `impression_count`, `click_through_rate` (as float), `conversion_delay_days` (integer), and `spend_usd`. The engine validates locale codes against a pre-defined ISO 639-1 + region map.

```bash
curl -X POST https://api.champions-sp-calc.io/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{"entries": [{"market_locale": "en-US", "impression_count": 45000, "ctr": 0.032, "conversion_delay_days": 14, "spend_usd": 1200}]}'
```

### Step 2: Run the Elasticity & Decay Engine
Once data is ingested, execute the core calculation pipeline. The tool automatically applies the **Latency-Aware Attribution Decay Model**—it adjusts the weight of each impression based on the median conversion delay for that specific locale. For example, a `ja-JP` campaign with a 45-day delay will have its attribution weight decayed differently than an `en-US` campaign with a 7-day delay. Run the pipeline with:

```python
from champions_sp_calc import Pipeline

pipeline = Pipeline(locale_map="auto", decay_factor=0.85)
results = pipeline.execute(ingested_data)
print(results.ses_scores)
```

### Step 3: Generate & Interpret the Competitive Gap Heatmap
Finally, export the **Gap Heatmap** as an interactive HTML or static PNG. The heatmap highlights underpenetrated keyword clusters per locale. For instance, if your `de-DE` competitor is bidding aggressively on "ERP-Software", but ignoring "Compliance-Tool für KMU", the heatmap will flag this as a **high-opportunity gap** (score > 0.7). Use this to reallocate budget instantly.

```javascript
const heatmap = new GapHeatmap({ locale: 'de-DE', min_opportunity_score: 0.7 });
heatmap.render(document.getElementById('heatmap-container'));
```

## Best Practices

### 1. Normalize Currency & Tax Before Ingestion
When dealing with multi-market data, **never trust raw spend figures**. Always convert all spend to a base currency (USD recommended) using a real-time FX API before ingestion. Additionally, strip out VAT/GST (which varies wildly across EU and APAC markets). The tool does not auto-convert currency—it expects clean, normalized data. This prevents skewed SES scores due to tax inflation.

### 2. Use Locale-Specific Decay Factors for B2B vs B2C
The default decay factor (0.85) is tuned for B2C SaaS. If you’re running B2B campaigns in Germany or Japan (where decision cycles can exceed 90 days), adjust the `decay_factor` to 0.95 or higher. This prevents the model from prematurely discounting high-value, long-tail conversions. Test this by running a simulation with historical data from the last 6 months—compare the predicted vs actual conversion curves.
