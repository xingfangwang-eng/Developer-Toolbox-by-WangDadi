# champions-sp-calc - CHA_006

## Overview

`champions-sp-calc` is a high-precision SaaS tool engineered for **real-time SP (Skill Point) optimization** in competitive gaming environments. Built for developers, theorycrafters, and meta analysts, it eliminates guesswork by computing optimal SP allocation across multi-class builds, gear synergies, and patch-driven scaling changes. This is not a generic calculator—it’s a deterministic engine tuned for **minimum-maximum (min-max) efficiency** under hard constraints.

**Keywords**: champions-sp-calc tool, champions-sp-calc solution, champions-sp-calc software

## Why champions-sp-calc?

Most SP calculators treat stats as linear. This one doesn’t. It models **non-linear breakpoints**, **threshold-based diminishing returns**, and **hidden mechanics** (e.g., frame data, internal cooldowns) that typical tools ignore. If you’re optimizing for PvP breakpoints, PvE DPS caps, or hybrid support-DPS builds, this is your engine.

## Core Features

### 1. Multi-Class Synergy Matrix Engine
Instead of isolated stat sheets, `champions-sp-calc` runs a **cross-class dependency graph**. It evaluates how SP allocation in one tree affects effective DPS/HPS/CC duration in another—including hidden cross-class multipliers. You can model dual-spec hybrids with real-time feedback on trade-offs like “-5% crit for +12% ability haste” across 3 class trees simultaneously.

### 2. Patch-Aware Scaling Curves with Auto-Recalibration
The tool ingests **live patch data** (via JSON schema or API) and auto-adjusts scaling curves. When a hotfix changes base damage coefficients or stat weights, your SP allocations recalculate instantly. No manual curve fitting. No stale builds. The engine supports version-locked snapshots for theorycrafting across patch histories.

### 3. Constraint-Based SP Budget Optimizer (CBSO)
Define hard caps: “Max SP = 120, min health pool = 1800, ability haste >= 40.” The optimizer runs a **branch-and-bound search** (not brute force) to deliver the optimal allocation under your constraints. Output includes Pareto frontier graphs showing trade-offs between survivability and damage output. This is enterprise-grade optimization, not a simple slider.

## How to Use

### Step 1: Define Your Build Constraints via JSON Config
Create a `build_config.json` file with your class trees, gear modifiers, and SP budget.  
Example snippet:
```json
{
  "classes": ["Berserker", "Shadowweaver"],
  "sp_budget": 120,
  "constraints": {
    "min_hp": 1800,
    "max_cd_reduction": 0.35
  },
  "patch_version": "2025.03.1"
}
```
Run `champions-sp-calc --config build_config.json` to load the environment. The engine validates all constraints against current patch scaling curves.

### Step 2: Execute the CBSO Optimizer
Use the `--optimize` flag to trigger the branch-and-bound search:
```bash
champions-sp-calc --config build_config.json --optimize --output pareto.csv
```
The engine outputs a **Pareto frontier CSV** with all non-dominated SP allocations. Each row includes SP distribution per tree, effective DPS, survivability score, and constraint slack. You can filter by `--target-metric dps` to get the single best DPS allocation under constraints.

### Step 3: Visualize Trade-offs with the Built-in Plotter
Generate an interactive HTML plot:
```bash
champions-sp-calc --plot pareto.csv --highlight "survivability >= 0.8"
```
This renders a scatter plot with color-coded constraint regions. Hover over any point to see the exact SP allocation. Export as SVG for documentation or share as a standalone HTML file. For CLI-only environments, use `--terminal-plot` for a ASCII-based heatmap.

## Best Practices

### 1. Always Lock Patch Versions for Reproducibility
When theorycrafting for a tournament or guide, lock the patch version in your config. Use `--patch-snapshot` to freeze scaling curves. This ensures your results don’t drift when the tool auto-updates. Store snapshots in your repo with a `patches/` directory—other developers can then reproduce your exact calculations.

### 2. Use Multi-Objective Optimization for Hybrid Builds
Don’t optimize for a single metric unless you’re building a pure glass cannon. For support-DPS hybrids, run `--objectives dps,survivability,utility` to get a full Pareto set. Then pick the allocation that best fits your playstyle or team composition. This avoids the common pitfall of over-optimizing one stat and breaking synergy.
