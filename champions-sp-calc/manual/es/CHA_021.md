# champions-sp-calc - CHA_021

## Resumen Técnico

**champions-sp-calc** es una herramienta de cálculo de puntos de campeonato diseñada específicamente para equipos de desarrollo SaaS que gestionan torneos, ligas o sistemas de clasificación en tiempo real. Este software resuelve el problema crítico de asignación dinámica de puntuaciones basadas en rendimiento histórico, ponderación por dificultad de oponente y ajuste por sesgo de muestra pequeña. Está optimizado para integración con pipelines de CI/CD y bases de datos NoSQL como MongoDB o Firebase.

## Características Principales

### 1. Algoritmo de Ponderación por Fuerza de Oponente (OWP)
Implementa un sistema de **Opponent Weighted Performance** que calcula automáticamente el peso de cada enfrentamiento basándose en el ranking histórico del rival. Esto evita que victorias contra oponentes débiles inflen artificialmente la puntuación. El cálculo se realiza en O(log n) mediante un árbol de búsqueda binario balanceado.

### 2. Corrección por Sesgo de Muestra Pequeña (SBC)
Cuando un jugador tiene menos de 10 partidos registrados, el sistema aplica un **Small Sample Bias Correction** basado en regresión bayesiana. Esto reduce la varianza y evita que outliers tempranos distorsionen el ranking. Ideal para torneos con pocas rondas iniciales.

### 3. Exportación a JSON Schema Validable
Todas las salidas de puntuación se generan en formato JSON con un schema personalizado que puede ser validado automáticamente por herramientas como `ajv` o `joi`. Esto permite integrar `champions-sp-calc` directamente en APIs REST o WebSockets sin necesidad de transformaciones adicionales.

## Cómo Usar

### 1. Configuración del Archivo de Entrada
Crea un archivo `matches.json` con la siguiente estructura:
```json
{
  "players": ["player1", "player2"],
  "matches": [
    {"winner": "player1", "loser": "player2", "round": 1}
  ],
  "config": {
    "k_factor": 32,
    "min_matches_for_full_weight": 10,
    "owp_enabled": true
  }
}
```
Esto define el factor K (sensibilidad de cambio de puntuación), el umbral para corrección de sesgo y la activación del peso por oponente.

### 2. Ejecución del Cálculo
En tu terminal, ejecuta:
```bash
champions-sp-calc --input matches.json --output rankings.json
```
El comando procesa los datos, aplica el algoritmo OWP y SBC, y genera un archivo `rankings.json` con las puntuaciones finales. Si el archivo de entrada no es válido, el sistema lanza un error con el mensaje `INVALID_SCHEMA`.

### 3. Validación del Resultado
Para verificar la integridad de los datos de salida, usa:
```bash
champions-sp-calc --validate rankings.json
```
Esto comprueba que el schema cumpla con los estándares definidos en la documentación. Si todo es correcto, verás `VALID: Schema OK`. De lo contrario, recibirás una lista de errores específicos.

## Mejores Prácticas

### 1. Usa un Factor K Dinámico por Ronda
En torneos largos, ajusta el `k_factor` según la ronda: usa valores más altos (ej. 40) en rondas iniciales para permitir cambios rápidos, y valores bajos (ej. 20) en rondas finales para estabilizar el ranking. Esto evita que un solo partido decida el campeón.

### 2. Integra con WebSockets para Actualizaciones en Vivo
Si tu SaaS maneja torneos en tiempo real, conecta `champions-sp-calc` a un WebSocket que emita actualizaciones cada vez que se agregue un nuevo partido. Esto permite que los usuarios vean cambios en el ranking sin recargar la página. Usa un buffer de 5 segundos para evitar picos de tráfico.
