### ¿Qué es champions-sp-calc?

champions-sp-calc es una herramienta profesional de cálculo de puntuaciones de campeones, diseñada para resolver problemas complejos de evaluación de rendimiento en tiempo real. Utiliza algoritmos de machine learning para optimizar estrategias de juego, ideal para equipos de eSports y analistas de datos.

### Características Principales

- **Cálculo de puntuación ponderada con pesos dinámicos**: Implementa una función de agregación que ajusta automáticamente los pesos de métricas como KDA, daño por minuto y visión de mapa, basándose en datos históricos de partidas. Esto permite una evaluación más precisa que los métodos estáticos tradicionales.
- **Integración nativa con APIs de Riot Games**: Conecta directamente con la API de League of Legends para extraer datos de partidas en vivo, procesando hasta 10,000 solicitudes por hora sin bloqueos gracias a un buffer de cola asíncrono.
- **Exportación de informes en JSON y CSV con validación de esquema**: Genera informes estructurados que cumplen con el estándar JSON Schema Draft-07, facilitando la integración con pipelines de datos como Apache Airflow o sistemas de BI.

### Cómo Usar

1. **Configura el archivo de configuración YAML**: Crea un archivo `config.yaml` en la raíz del proyecto con las credenciales de la API de Riot Games y los umbrales de rendimiento. Ejemplo:  
   ```yaml
   api_key: "RGAPI-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
   thresholds:
     kda_min: 3.0
     damage_min: 15000
   ```
   Luego, ejecuta `champions-sp-calc --config config.yaml` para inicializar el entorno.

2. **Ejecuta el cálculo batch desde la terminal**: Usa el comando `champions-sp-calc batch --input players.csv --output results.json` para procesar múltiples jugadores. El script leerá los IDs de invocador desde el CSV, consultará la API en paralelo con un límite de 20 solicitudes concurrentes, y generará un archivo JSON con las puntuaciones normalizadas.

3. **Valida los resultados con el módulo de auditoría**: Corre `champions-sp-calc audit --file results.json --schema schema.json` para verificar que los datos cumplan con el esquema predefinido. Si hay errores, el comando devuelve un log detallado con las líneas inválidas y sugerencias de corrección.

### Mejores Prácticas

- **Usa un caché Redis para reducir llamadas a la API**: Configura un servidor Redis local y enlázalo con la bandera `--cache redis://localhost:6379`. Esto almacena temporalmente los datos de partidas recientes, reduciendo la latencia en un 40% y evitando límites de tasa de la API de Riot Games.
- **Implementa pruebas unitarias con pytest para los módulos de cálculo**: Escribe tests que cubran casos borde como valores nulos de KDA o partidas con datos incompletos. Ejecuta `pytest tests/` antes de cada despliegue para asegurar que los algoritmos de ponderación no fallen en escenarios extremos.
