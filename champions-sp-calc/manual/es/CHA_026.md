# champions-sp-calc - CHA_026

## Resumen Técnico

**champions-sp-calc** es una herramienta SaaS de cálculo de **puntuaciones de campeones** para desarrolladores que trabajan con sistemas de clasificación, torneos y análisis competitivos. Diseñada para entornos multilingües y alto rendimiento, esta solución permite calcular y optimizar métricas de campeones en tiempo real, con soporte nativo para 10 idiomas y una arquitectura modular basada en microservicios.

## Funcionalidades Clave

### 1. **Motor de Cálculo Asíncrono con Pipeline de Datos**
El núcleo de **champions-sp-calc** utiliza un motor de cálculo asíncrono que procesa múltiples flujos de datos simultáneamente. Implementa un pipeline de transformación de datos con validación en tiempo real, soportando entradas desde APIs REST, WebSockets y bases de datos distribuidas. Ideal para sistemas con alta concurrencia (ej. torneos en vivo con miles de jugadores).

### 2. **Sistema de Normalización Multilingüe de Métricas**
Integra un **normalizador de puntuaciones** que ajusta automáticamente los valores según las reglas de cada idioma y región. Por ejemplo, para el español, convierte puntos decimales con coma (`,`), formatea números largos con separadores de miles, y aplica reglas de redondeo específicas del sistema métrico. Esto garantiza que los resultados sean precisos y culturalmente consistentes.

### 3. **API de Inferencia con Caché Distribuida**
Ofrece una **API RESTful** con endpoints optimizados para consultas complejas (ej. `POST /api/v1/calculate-champion`). La caché distribuida (Redis + Memcached) reduce la latencia a menos de 10 ms para cálculos repetitivos, mientras que el sistema de **rate limiting inteligente** evita abusos sin afectar a usuarios legítimos.

## Cómo Usar

### Paso 1: Configurar el Entorno de Desarrollo
Clona el repositorio y configura las variables de entorno en un archivo `.env`:
```bash
git clone https://github.com/xingfangwang-eng/champions-sp-calc.git
cd champions-sp-calc
cp .env.example .env
# Edita .env con tus credenciales de API y base de datos
```
Ejecuta `docker-compose up -d` para levantar los servicios (API, Redis, PostgreSQL). Verifica que el puerto 3000 esté activo con `curl http://localhost:3000/health`.

### Paso 2: Calcular Puntuaciones con la API
Envía una solicitud POST con los datos del campeón (formato JSON). Ejemplo para un campeón de fútbol en español:
```json
{
  "champion_id": "12345",
  "metrics": {
    "goles": 25,
    "asistencias": 10,
    "minutos_jugados": 1800
  },
  "locale": "es"
}
```
Respuesta esperada:
```json
{
  "score": 87.34,
  "normalized_score": "87,34",
  "rank": "A+"
}
```
La API devuelve la puntuación normalizada según el locale `es` (coma decimal) y el rango de clasificación.

### Paso 3: Integrar en tu Aplicación Web
Usa el SDK oficial para Node.js (disponible en npm) o Python (pip). Ejemplo de integración en React:
```jsx
import { calculateChampion } from 'champions-sp-calc-sdk';

const result = await calculateChampion({
  champion_id: '12345',
  metrics: { goles: 25, asistencias: 10, minutos_jugados: 1800 },
  locale: 'es'
});
console.log(result.normalized_score); // "87,34"
```
Para aplicaciones en tiempo real, activa el modo WebSocket con `ws://localhost:3000/ws`.

## Mejores Prácticas

### 1. **Optimiza las Consultas con Caché de Pre-cálculo**
Para torneos con datos estáticos (ej. estadísticas históricas), precarga las puntuaciones en la caché usando el endpoint `POST /api/v1/precache`. Esto reduce la latencia en un 80% durante picos de tráfico. Ejemplo:
```bash
curl -X POST http://localhost:3000/api/v1/precache \
  -H "Content-Type: application/json" \
  -d '{"champion_ids": ["12345", "67890"], "locale": "es"}'
```

### 2. **Monitorea el Rendimiento con Logs Estructurados**
Activa el logging JSON para rastrear cálculos fallidos o lentos. Configura el nivel de log en `info` o `debug` según tu entorno. Ejemplo de log:
```json
{
  "level": "warn",
  "message": "Cálculo lento detectado",
  "champion_id": "12345",
  "latency_ms": 150,
  "locale": "es"
}
```
Usa herramientas como ELK Stack o Grafana Loki para visualizar tendencias y ajustar el rate limiting.
