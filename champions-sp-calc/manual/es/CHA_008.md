# champions-sp-calc - CHA_008

## Resumen Técnico

**Champions SP Calc** es una herramienta SaaS de cálculo paramétrico diseñada para ingenieros de producto, arquitectos de software y analistas de rendimiento que necesitan modelar, simular y optimizar puntos de función (SP) en sistemas distribuidos. A diferencia de calculadoras genéricas, esta solución integra lógica de negocio multientorno (cloud, on-premise, edge) y exporta resultados directamente a pipelines CI/CD.

---

## Funcionalidades Clave

### 1. Motor de Cálculo Híbrido con Soporte Multi-Sistema
El núcleo de Champions SP Calc permite definir reglas de cómputo personalizadas mediante un DSL (Domain Specific Language) propio. Soporta herencia de configuraciones, validación semántica en tiempo real y resolución de dependencias circulares. Ideal para arquitecturas con microservicios, donde los SP se distribuyen entre dominios asíncronos.

### 2. Simulación de Escenarios con Historial de Versiones
Cada cálculo se versiona automáticamente con metadatos de entorno, timestamp y hash del modelo de entrada. Puedes comparar dos ejecuciones lado a lado, detectar regresiones paramétricas y revertir a un estado anterior sin perder el contexto de la decisión. Esto convierte a la herramienta en un **sistema de trazabilidad de decisiones técnicas**.

### 3. Exportación Directa a Grafana, Prometheus y APIs REST
Los resultados no se quedan en la interfaz. Champions SP Calc expone un endpoint REST que devuelve JSON estructurado compatible con paneles de monitoreo. Además, permite generar métricas listas para inyectar en Prometheus, facilitando la integración con SLOs y dashboards de rendimiento en producción.

## Guía de Uso Avanzada

### 1. Configuración de un Modelo de Cálculo con Reglas Condicionales
En lugar de usar placeholders, edita el archivo `model.spc` (formato YAML extendido) para definir nodos de cálculo con condiciones lógicas. Ejemplo:

```yaml
nodes:
  - id: "costo_total"
    formula: "sum(horas * tarifa * factor_complejidad)"
    conditions:
      - if: "entorno == 'produccion'"
        then: "apply_overhead(1.15)"
      - if: "region == 'eu-west-1'"
        then: "apply_impuesto(0.21)"
```

Luego ejecuta:
```bash
champions-sp-calc run --model model.spc --env produccion --region eu-west-1
```

### 2. Comparación de Versiones con Dif Estructural
Usa el comando `diff` para comparar dos ejecuciones históricas. Champions SP Calc genera un reporte en Markdown con las diferencias paramétricas, cambios en fórmulas y desviaciones de rendimiento.

```bash
champions-sp-calc diff --from v2.3 --to v2.4 --output diff_report.md
```

Esto es crítico para auditorías técnicas o revisiones de código retrospectivas.

### 3. Integración en Pipeline CI/CD con GitHub Actions
Añade un paso en tu workflow que ejecute el cálculo automático al hacer push a `main`. Ejemplo de acción:

```yaml
- name: Calcular SP con Champions
  run: |
    champions-sp-calc run --model ./models/sp_model.yaml \
      --env staging \
      --output ./results/sp_result.json
  env:
    CHAMPIONS_API_KEY: ${{ secrets.CHAMPIONS_API_KEY }}
```

Los resultados se pueden consumir como artefacto del pipeline o enviarse a un webhook de monitorización.

## Mejores Prácticas Recomendadas

### 1. Versiona los Modelos en tu Propio Repositorio
No dependas únicamente del historial interno de la herramienta. Mantén los archivos `.spc` bajo control de versiones (Git) junto con el código fuente. Esto garantiza que cualquier cambio en la lógica de cálculo quede ligado a un commit y pueda ser revisado en una PR.

### 2. Usa la Exportación a Prometheus para Alertas en Producción
Configura un job de scraping que consuma el endpoint REST de Champions SP Calc cada 5 minutos. Crea alertas en Prometheus que se disparen si la desviación de SP supera un umbral definido (ej: >10% respecto al modelo base). Así evitas sorpresas en facturación o rendimiento.
