# champions-sp-calc - CHA_025

## Resumen

**champions-sp-calc** es una herramienta SaaS de cálculo de puntos de campeonato (SP) diseñada para desarrolladores y equipos de esports que necesitan automatizar la gestión de puntuaciones en tiempo real. Optimizada para SEO con palabras clave como `champions-sp-calc tool`, `champions-sp-calc solution` y `champions-sp-calc software`, esta solución elimina la complejidad de los cálculos manuales y ofrece integraciones nativas con APIs de terceros. Perfecta para torneos, ligas y plataformas de análisis de rendimiento.

## Características Principales (Hardcore)

### 1. **Algoritmo de Ponderación Dinámica (DPA)**
   - Implementa un sistema de ponderación basado en reglas configurables (ELO, K-factor, y ajustes por temporada). El motor evalúa automáticamente el rendimiento histórico y aplica factores de corrección para evitar sesgos en partidas con desventaja de equipo. Ideal para ligas con múltiples divisiones.

### 2. **Cálculo en Tiempo Real con WebSockets**
   - Integración directa con WebSockets para actualizar puntos de campeonato sin latencia. Soporta eventos de juego en vivo (kills, asistencias, objetivos) y sincroniza con plataformas como Twitch o Discord mediante webhooks. No más recargas manuales.

### 3. **Motor de Reglas Personalizables (CRS)**
   - Define tus propias fórmulas de puntuación usando sintaxis JSON o YAML. Desde ponderaciones por rol (ADC, Support, Jungla) hasta bonificaciones por objetivos épicos (Barón, Dragón anciano). El motor valida reglas en tiempo real y ofrece logs de auditoría para depuración.

## Cómo Usar (Pasos Detallados)

### 1. **Configuración Inicial con Variables de Entorno**
   - Clona el repositorio y ejecuta `npm install` para instalar dependencias. Luego, crea un archivo `.env` con los siguientes parámetros:
     ```
     DB_HOST=localhost
     DB_PORT=5432
     API_KEY=tu_clave_secreta
     ELO_DEFAULT=1200
     K_FACTOR=32
     ```
   - Inicia el servidor con `npm run start:prod`. Verifica que el endpoint `/health` responda `200 OK`.

### 2. **Definir Reglas de Puntuación con CRS**
   - Crea un archivo `rules.json` en la carpeta `/config`:
     ```json
     {
       "roles": {
         "ADC": { "weight": 1.2, "bonus_per_kill": 10 },
         "Support": { "weight": 0.8, "bonus_per_assist": 15 }
       },
       "objectives": {
         "baron": { "points": 50, "team_bonus": true },
         "dragon_elder": { "points": 75, "team_bonus": true }
       }
     }
     ```
   - Carga las reglas mediante `POST /api/rules/load` con el archivo. El motor validará automáticamente la estructura y devolverá un hash de integridad.

### 3. **Sincronización en Tiempo Real con WebSockets**
   - Conecta tu cliente WebSocket al endpoint `wss://tuservidor.com/ws`. Escucha eventos como:
     ```javascript
     socket.on('match:update', (data) => {
       console.log(`Nuevo evento: ${data.type} - Puntos actualizados: ${data.sp}`);
     });
     ```
   - Para enviar datos de partida, usa `POST /api/events` con payload JSON:
     ```json
     {
       "match_id": "12345",
       "player_id": "67890",
       "event": "kill",
       "timestamp": 1690000000
     }
     ```

## Mejores Prácticas (Industry Standards)

### 1. **Cacheo de Reglas con Redis para Alta Concurrencia**
   - En torneos con miles de partidas simultáneas, usa Redis para cachear las reglas de puntuación. Configura TTL dinámico basado en la frecuencia de cambios (por ejemplo, 5 minutos si las reglas son estáticas, 30 segundos si se actualizan en vivo). Esto reduce la latencia en un 40% y evita cuellos de botella en la base de datos.

### 2. **Auditoría de Eventos con Logs Estructurados**
   - Implementa un sistema de logging con formato JSON que incluya `event_id`, `timestamp`, `player_id` y `rule_hash`. Almacena estos logs en Elasticsearch para análisis forense. Si un jugador reclama un error de puntuación, puedes reproducir el cálculo exacto usando el hash de las reglas y los eventos originales. Esto es crítico para cumplir con regulaciones de esports profesionales.
