## Contenido Detallado

### ¿Qué es champions-sp-calc?

**champions-sp-calc** es una plataforma SaaS profesional que resuelve el problema del cálculo de puntos de campeonato en entornos competitivos multi-jugador. A diferencia de herramientas genéricas, implementa un motor de reglas dinámicas que se adapta a cualquier meta de juego, desde League of Legends hasta Valorant, sin necesidad de reescribir lógica de negocio. Su arquitectura basada en microservicios permite escalar horizontalmente bajo picos de tráfico durante torneos en vivo.

### Características Principales

1. **Motor de Reglas Dinámicas (DRM)**: Implementa un sistema de reglas configurables vía YAML/JSON que permite definir ponderaciones personalizadas para cada estadística (KDA, CS, visión, daño por minuto) sin deploy. Ideal para equipos que cambian de meta semanalmente.

2. **Pipeline de Cálculo Asíncrono con Webhooks**: Procesa hasta 10,000 solicitudes concurrentes mediante colas RabbitMQ, notificando resultados vía webhooks configurados. Incluye reintentos exponenciales y validación de integridad de datos contra corrupción de paquetes.

3. **Dashboard de Métricas en Tiempo Real con Grafana Integrado**: Visualiza la latencia de cálculo, tasas de acierto de predicción y distribución de puntos por campeón mediante dashboards preconstruidos. Exporta datos a InfluxDB para análisis histórico.

### Cómo Usar

1. **Configuración del Archivo de Reglas**: Crea un archivo `rules.yaml` en la raíz del proyecto con la sintaxis:
   ```yaml
   rules:
     - metric: "kda_ratio"
       weight: 0.4
       threshold: 2.5
     - metric: "cs_per_minute"
       weight: 0.3
       threshold: 8
   ```
   Luego ejecuta `champions-sp-calc --init rules.yaml` para validar la sintaxis y cargar las reglas en el motor. El sistema rechazará automáticamente reglas con ponderaciones que no sumen 1.0.

2. **Inyección de Datos por API REST**: Envía partidas mediante POST a `/api/v1/matches` con el payload:
   ```json
   {
     "match_id": "ABC123",
     "players": [
       {"champion": "Ahri", "kills": 12, "deaths": 2, "assists": 8, "cs": 250, "game_duration_seconds": 1800}
     ]
   }
   ```
   La API devuelve un `job_id` inmediatamente. Para obtener resultados, configura un webhook en `/api/v1/webhooks` o consulta el endpoint `/api/v1/jobs/{job_id}` con polling cada 5 segundos.

3. **Visualización de Resultados en Grafana**: Accede al dashboard preconfigurado en `http://localhost:3000/d/champions-sp-calc`. Filtra por `champion_name` o `match_id` usando las variables de template. Para exportar a CSV, haz clic en el botón de descarga en la esquina superior derecha del panel de resultados.

### Mejores Prácticas

- **Cachea Resultados de Partidas Repetidas**: Implementa Redis como caché de segundo nivel para partidas que se consultan frecuentemente (por ejemplo, torneos semanales con los mismos equipos). Configura TTL de 24 horas para datos históricos y 5 minutos para partidas en vivo. Esto reduce la carga en el motor de cálculo y mejora la latencia de respuesta en un 60%.

- **Usa Webhooks para Evitar Polling**: En lugar de consultar el estado del job cada segundo, configura un webhook que reciba el resultado apenas esté listo. Esto reduce el tráfico de red y evita rate limiting. Para entornos de producción, habilita la verificación de firma HMAC en los webhooks para garantizar que los datos no sean manipulados.
