# champions-sp-calc - CHA_009

## Resumen Técnico

**champions-sp-calc** es una herramienta SaaS de cálculo de probabilidades y rendimiento para equipos de esports y deportes competitivos, diseñada para arquitectos de datos y analistas de rendimiento. Integra modelos estadísticos avanzados (distribución de Poisson, regresión logística y simulación Monte Carlo) con datos en tiempo real de APIs de torneos (Riot Games, Valve, ESL). Su motor de inferencia bayesiana permite ajustar predicciones dinámicamente según el historial de enfrentamientos directos, el estado del jugador y las condiciones del meta actual.

## 3 Características Clave (Hardcore)

1.  **Motor de Simulación Monte Carlo con Kernel Bayesiano Adaptativo**
    No es un simple generador de números aleatorios. El simulador ejecuta 10,000 iteraciones por partida, pero cada iteración pondera los parámetros de entrada (KDA, oro por minuto, tasa de victorias en parches recientes) mediante un kernel bayesiano que se actualiza con cada nuevo dato del ecosistema. El resultado no es una probabilidad plana, sino una distribución de densidades que revela la incertidumbre real del pronóstico.

2.  **Pipeline de Ingesta y Normalización Multitorneo (API-First)**
    champions-sp-calc no depende de hojas de cálculo. Su capa de abstracción `ChampionDataAdapter` se conecta directamente a las APIs de Riot, Steam Web y ESL, normalizando los schemas dispares (JSON, Protobuf, GraphQL) en un modelo de datos unificado (UDM). El pipeline soporta streaming via WebSockets para actualizaciones en vivo durante torneos LAN, con un buffer de eventos que garantiza consistencia eventual sin pérdida de paquetes.

3.  **Analizador de Sinergias y Contrapicks con Embeddings de Campeones**
    Más allá de estadísticas superficiales, la herramienta emplea un modelo de embeddings entrenado sobre 2 millones de partidas rankeadas. Cada campeón se representa como un vector en un espacio latente de 128 dimensiones. El módulo de sinergias calcula la similitud coseno entre vectores de campeones aliados y enemigos, detectando composiciones ganadoras (ej: “Protect the President” o “Pick comp”) y sugiriendo contrapicks con un índice de confianza basado en el meta actual.

## 3 Pasos de Uso (Workflow Profesional)

1.  **Configurar el Contexto de la Partida y el Pipeline de Datos**
    Ejecuta el comando `champions-sp-calc init --tournament LCS --patch 14.10 --streaming true`. Esto inicializa el contexto de análisis, establece la conexión WebSocket con la API de Riot y descarga el último snapshot del meta (tasas de victoria, tasas de selección, índices de prohibición). El sistema valida la integridad de los datos mediante checksums SHA-256 antes de cargar el modelo bayesiano.

2.  **Ejecutar la Simulación con Parámetros Personalizados**
    Para un análisis granular, usa `champions-sp-calc simulate --team "T1" --opponent "Gen.G" --iterations 50000 --confidence 0.95 --output json`. El motor ejecuta las 50,000 iteraciones, aplicando el kernel bayesiano con los datos en vivo del torneo. Durante la ejecución, el log muestra en tiempo real la convergencia de la cadena de Markov y el intervalo de credibilidad del 95% para cada métrica (probabilidad de victoria, primer blood, torre de oro, barón Nashor).

3.  **Interpretar los Resultados y Generar Reportes**
    Al finalizar, el sistema genera un objeto JSON con la distribución completa de probabilidades. Para visualizarlo, ejecuta `champions-sp-calc report --format markdown --output ./reports/analisis_T1_vs_GenG.md`. El reporte incluye gráficos de densidad (KDE) para cada fase del juego (early, mid, late game), una matriz de sinergias entre los 5 campeones seleccionados y un ranking de contrapicks óptimos basado en el embedding de campeones. También exporta un archivo CSV con los datos crudos para análisis posterior en R o Python.

## 2 Mejores Prácticas para Desarrolladores

1.  **Cachea los Embeddings de Campeones Localmente**
    Los embeddings no cambian cada día, pero sí con cada parche mayor (cada 2-3 semanas). Configura un cron job que ejecute `champions-sp-calc update-embeddings --source latest` una vez por semana. Almacena los vectores en una base de datos vectorial local (ej: ChromaDB o FAISS) para reducir la latencia de las consultas de sinergias de 400ms a menos de 10ms. Esto es crítico si integras la herramienta en un dashboard en vivo durante un torneo.

2.  **Ajusta el Prior Bayesiano Según la Fase del Torneo**
    El kernel bayesiano por defecto asume un prior uniforme. Sin embargo, en fases eliminatorias (playoffs, finales), el rendimiento histórico de los equipos en escenarios de alta presión es más relevante. Modifica el archivo de configuración `~/.champions-sp-calc/config.yaml` y establece `bayesian_prior: "playoff_weighted"`. Esto hará que el modelo dé un 30% más de peso a partidas de playoffs anteriores, mejorando la precisión de las predicciones en un 12-15% según nuestras pruebas internas.
