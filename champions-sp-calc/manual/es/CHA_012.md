### ¿Qué es champions-sp-calc?

champions-sp-calc es una herramienta profesional que resuelve problemas complejos de cálculo de puntuaciones, clasificaciones y proyecciones en torneos multideportivos. Ideal para ligas, eSports y competiciones internacionales donde cada punto cuenta y la lógica de desempate es crítica.

### Características Principales

- **Motor de desempate por reglas FIFA/MLB/NBA**: Implementa algoritmos de tie-breaking multi-capa que evalúan automáticamente diferencia de goles, enfrentamientos directos, puntos de set y coeficientes de rendimiento. Soporta personalización de reglas por torneo sin tocar código base.
- **Simulación Monte Carlo para proyecciones en tiempo real**: Ejecuta 10,000+ iteraciones por segundo para predecir probabilidades de clasificación, eliminación y empates. Incluye visualización de percentiles y bandas de confianza para decisiones tácticas.
- **API RESTful con versionado semántico y rate limiting**: Endpoints optimizados para batch processing de hasta 50,000 partidos por solicitud. Cacheo inteligente con Redis y soporte para webhooks en eventos críticos (cambio de líder, empate roto).

### Cómo Usar

1. **Configuración del torneo con schema JSON-LD**: Define la estructura de tu competición usando el schema `champions-sp-calc/tournament/v2`. Incluye parámetros como `tiebreakerRules`, `pointsSystem` y `matchFormat`. Ejemplo:
   ```json
   {
     "@context": "https://champions-sp-calc.example.com/schemas",
     "tournament": {
       "name": "Liga MX 2024",
       "tiebreakerRules": ["goalDifference", "headToHead", "fairPlayPoints"],
       "pointsSystem": {"win": 3, "draw": 1, "loss": 0}
     }
   }
   ```

2. **Inyección de datos de partidos mediante CSV estructurado**: Sube un archivo CSV con columnas obligatorias: `match_id, home_team, away_team, home_score, away_score, date`. El sistema valida automáticamente consistencia (sin partidos duplicados, fechas no futuras). Comando para procesar:
   ```bash
   curl -X POST https://api.champions-sp-calc.com/v1/ingest \
     -H "Authorization: Bearer $TOKEN" \
     -F "file=@matches.csv"
   ```

3. **Ejecución de simulación con parámetros avanzados**: Usa el endpoint `/v1/simulate` con flags como `iterations=10000`, `confidenceLevel=0.95` y `outputFormat=json`. Recibirás un objeto con `probabilities` (por equipo), `scenarios` (mejor/peor caso) y `criticalMatches` (partidos que más impactan). Ejemplo de respuesta parcial:
   ```json
   {
     "team": "FC Barcelona",
     "prob_qualify": 0.87,
     "worst_case_rank": 4,
     "best_case_rank": 1
   }
   ```

### Mejores Prácticas

- **Cachea resultados de simulaciones intensivas**: Si ejecutas simulaciones con más de 50,000 iteraciones, usa el flag `cache=true` para almacenar resultados en Redis con TTL de 5 minutos. Esto reduce latencia de 12s a 0.3s en consultas repetitivas. Ideal para dashboards en vivo durante jornadas de liga.
- **Valida datos de entrada con preprocesamiento offline**: Antes de inyectar datos masivos, ejecuta el comando `validate` con el flag `--strict` para detectar errores comunes: puntuaciones negativas, equipos no registrados, fechas inconsistentes. Esto evita corrupción de rankings y ahorra horas de debugging en producción.
