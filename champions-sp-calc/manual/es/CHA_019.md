# champions-sp-calc - CHA_019

## Resumen

**champions-sp-calc** es una herramienta de cálculo SaaS diseñada para optimizar la asignación de puntos de habilidad en configuraciones de campeones, ideal para desarrolladores que trabajan con sistemas de progresión complejos. Este software permite modelar, simular y validar distribuciones de puntos en tiempo real, asegurando que cada configuración cumpla con restricciones de balance y rendimiento. Perfecto para equipos de game design, simulación de RPGs o sistemas de puntuación dinámica.

## Contenido Detallado

### ¿Qué es champions-sp-calc?

**champions-sp-calc** es un motor de cálculo SaaS que resuelve problemas de optimización combinatoria en la asignación de puntos de habilidad. A diferencia de simples calculadoras, esta herramienta integra algoritmos de búsqueda heurística y validación de restricciones, permitiendo a los desarrolladores modelar árboles de habilidades con dependencias, costos variables y límites de nivel. Está construida sobre una arquitectura serverless con soporte para caching distribuido, ideal para entornos de alta concurrencia.

### Características Principales

- **Optimización por Algoritmo Genético**: Implementa un motor de búsqueda evolutiva que encuentra la distribución óptima de puntos en árboles de habilidades con hasta 200 nodos. Soporta mutación controlada, cruce de configuraciones y selección por fitness, todo configurable vía API REST.
- **Validación de Restricciones en Tiempo Real**: Cada asignación se valida contra un grafo de dependencias (prerrequisitos, cooldowns, sinergias). El sistema rechaza configuraciones inválidas antes de calcular el costo total, reduciendo errores en pipelines de CI/CD.
- **Exportación a JSON Schema**: Genera esquemas de configuración estandarizados (JSON Schema Draft-07) para integrar con herramientas de testing, documentación automática o motores de reglas externos. Ideal para equipos que usan OpenAPI o GraphQL.

### Cómo Usar

1. **Configurar el Árbol de Habilidades**  
   Define la estructura de habilidades en un archivo YAML o JSON. Cada nodo debe incluir `id`, `cost`, `prerequisites` (array de IDs) y `maxLevel`. Ejemplo:  
   ```yaml
   nodes:
     - id: "sword_mastery"
       cost: 3
       prerequisites: []
       maxLevel: 5
     - id: "critical_strike"
       cost: 5
       prerequisites: ["sword_mastery"]
       maxLevel: 3
   ```  
   Luego, ejecuta `champions-sp-calc import --file tree.yaml` para cargar la estructura en la base de datos en memoria.

2. **Ejecutar la Simulación de Puntos**  
   Usa el endpoint `/api/v1/simulate` con un payload que especifique el total de puntos disponibles y el objetivo (maximizar daño, minimizar costo, etc.). Ejemplo con cURL:  
   ```bash
   curl -X POST https://api.champions-sp-calc.dev/v1/simulate \
     -H "Content-Type: application/json" \
     -d '{"totalPoints": 50, "objective": "max_damage", "algorithm": "genetic", "generations": 100}'
   ```  
   La respuesta incluye la distribución óptima, el costo total y un hash de validación.

3. **Validar una Configuración Existente**  
   Si ya tienes una asignación de puntos, pásala al endpoint `/api/v1/validate` para verificar dependencias y límites. Útil en pipelines de CI para evitar regresiones:  
   ```bash
   curl -X POST https://api.champions-sp-calc.dev/v1/validate \
     -H "Content-Type: application/json" \
     -d '{"assignments": {"sword_mastery": 3, "critical_strike": 2}}'
   ```  
   Si la configuración es inválida, el sistema devuelve un array de errores con los IDs de los nodos conflictivos.

### Mejores Prácticas

- **Usa el Modo Batch para Pruebas de Regresión**  
  En entornos de CI/CD, no ejecutes simulaciones individuales una por una. En su lugar, agrupa todas las configuraciones candidatas en un archivo JSON y usa el endpoint `/api/v1/batch-validate`. Esto reduce la latencia de red y permite procesar hasta 100 configuraciones por llamada, con resultados en formato array. Ejemplo:  
  ```json
  {
    "configs": [
      {"assignments": {"sword_mastery": 5, "critical_strike": 3}},
      {"assignments": {"sword_mastery": 2, "critical_strike": 1}}
    ]
  }
  ```

- **Cachea Resultados con ETags**  
  Cada respuesta de simulación incluye un header `ETag` único basado en el hash del árbol de habilidades y la configuración de puntos. Si repites la misma simulación, envía el ETag en el header `If-None-Match`. El servidor responderá con `304 Not Modified` si no hay cambios, ahorrando ancho de banda y cómputo. Ideal para dashboards en tiempo real.
