# champions-sp-calc - CHA_020

## Resumen

**champions-sp-calc** es una herramienta SaaS de alto rendimiento diseñada para calcular y optimizar estrategias de velocidad de punto (SP) en campeones de juegos competitivos con mecánicas de rotación y priorización de acciones. Este software permite a los desarrolladores y analistas de eSports modelar, simular y ajustar en tiempo real las secuencias de habilidades y movimientos, maximizando el DPS y la eficiencia de recursos en entornos de baja latencia. Con soporte multilingüe y una arquitectura orientada a microservicios, champions-sp-calc se convierte en el estándar de facto para equipos de desarrollo de juegos y herramientas de entrenamiento.

## Características Principales

### 1. Motor de Simulación de Rotaciones con Priorización Dinámica
Implementa un algoritmo de planificación de acciones basado en **árboles de decisión con pesos dinámicos**, donde cada habilidad se evalúa según su cooldown, coste de recurso y sinergia con otras habilidades. El motor permite configurar perfiles de campeón personalizados, incluyendo modificadores de velocidad de ataque, reducción de enfriamiento y bonificaciones de equipo. La simulación se ejecuta en tiempo real con una granularidad de 0.1 segundos, generando gráficos de uso de recursos y DPS esperado.

### 2. Optimización de Builds con Machine Learning Integrado
Utiliza un modelo de **aprendizaje por refuerzo (Q-learning)** entrenado con datos de partidas de alto nivel para recomendar secuencias óptimas de objetos y runas. La herramienta analiza el estado actual del campeón (nivel, oro, objetos equipados) y sugiere builds que maximizan el SP en función del matchup y la fase del juego. El modelo se actualiza automáticamente con cada nueva partida procesada, mejorando la precisión de las recomendaciones.

### 3. API RESTful para Integración en Herramientas de Terceros
Expone una API RESTful completa con endpoints para:
- `/api/v1/calculate-sp`: Calcular el SP de una rotación de habilidades dada una configuración.
- `/api/v1/optimize-build`: Obtener la mejor build para un campeón y matchup.
- `/api/v1/simulate-combat`: Simular un combate completo con múltiples campeones y variables ambientales (torretas, dragones, etc.).

La API está documentada con OpenAPI 3.0 y soporta autenticación mediante tokens JWT, permitiendo una integración directa en clientes web, móviles o herramientas de streaming.

## Cómo Usar

### 1. Configurar un Perfil de Campeón Personalizado
Accede al panel de administración y selecciona "Nuevo Campeón". Introduce los parámetros clave:
- **Velocidad de ataque base**: 0.625 ataques/segundo.
- **Coste de habilidad Q**: 50 de maná, cooldown 4 segundos.
- **Multiplicador de daño por nivel**: 0.15 por nivel.

Luego, define las sinergias: por ejemplo, si la habilidad W reduce el cooldown de Q en un 20% durante 3 segundos. La herramienta validará automáticamente la coherencia de los datos y generará un hash único para el perfil.

### 2. Ejecutar una Simulación de Rotación
Desde la interfaz de simulación, selecciona el perfil creado y define la secuencia de habilidades: `[Q, W, E, Q, R]`. Configura la duración de la simulación (por defecto 10 segundos) y activa la opción "Priorizar sinergias". El motor ejecutará la simulación y mostrará:
- **Gráfico de uso de maná y vida** (líneas superpuestas).
- **DPS instantáneo y acumulado** (con intervalos de confianza del 95%).
- **Tiempo hasta la primera rotación completa** (en milisegundos).

### 3. Obtener Recomendaciones de Build con ML
En la sección "Optimización", selecciona el campeón y el matchup (por ejemplo, "Asesino vs Tanque"). La herramienta consultará el modelo de Q-learning y devolverá una lista de objetos ordenados por prioridad, con una puntuación de impacto en el SP. Cada recomendación incluye:
- **Nombre del objeto**: "Filo del Infinito".
- **Costo en oro**: 3400.
- **SP esperado adicional**: +12.7%.
- **Justificación**: "Aumenta el daño crítico en un 25%, sinergizando con la pasiva del campeón."

## Mejores Prácticas

### 1. Validar Perfiles con Datos de Partidas Reales
No confíes únicamente en datos teóricos. Utiliza la función de **importación de logs de partidas** (formato JSON con timestamps de habilidades) para ajustar los parámetros del perfil. Por ejemplo, si la velocidad de ataque real es un 3% menor que la configurada, actualiza el valor base. Esta práctica reduce el error de simulación a menos del 1%.

### 2. Integrar la API en Herramientas de Análisis de Partidas
Si desarrollas una herramienta de replay o un overlay para streamers, conecta la API de champions-sp-calc para mostrar en tiempo real el SP calculado. Por ejemplo, en un overlay de Twitch, puedes mostrar un widget que actualice el SP cada segundo durante una pelea de equipo, ayudando a los espectadores a entender la eficiencia del jugador. La latencia de la API es inferior a 50 ms en condiciones normales.
