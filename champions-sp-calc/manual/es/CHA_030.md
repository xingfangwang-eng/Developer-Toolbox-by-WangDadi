## Contenido Detallado

### ¿Qué es champions-sp-calc?

**champions-sp-calc** es un motor de cálculo SaaS que resuelve problemas de asignación de puntos en competiciones deportivas, torneos de e-sports o cualquier sistema de ranking basado en variables ponderadas. A diferencia de scripts monolíticos, esta herramienta ofrece una API RESTful con validación de reglas de negocio, soporte para múltiples modos de cálculo (Eliminación Directa, Round Robin, Swiss System) y una interfaz CLI para CI/CD pipelines.

### Características Principales

- **Cálculo de puntos con lógica de desempate configurable**: Implementa un algoritmo de desempate basado en head-to-head, diferencia de puntos y coeficiente de rendimiento. Puedes definir reglas personalizadas mediante un archivo YAML (ej. `tiebreaker_rules.yaml`) que se evaluan en tiempo de ejecución sin necesidad de redeploy.

- **Integración nativa con sistemas de eventos en vivo**: Ofrece un endpoint WebSocket (`/ws/calc-stream`) que transmite actualizaciones de puntos en tiempo real cuando se reciben eventos desde un broker de mensajes (Kafka o Redis Pub/Sub). Ideal para dashboards de torneos en vivo.

- **Exportación a formato ELO y compatibilidad con tablas de clasificación**: Genera salidas estructuradas en JSON, CSV o directamente como tablas HTML embebibles. Incluye un formateador ELO que normaliza los puntos según la fuerza del oponente, siguiendo el estándar USCF.

### Cómo Usar

1. **Configurar el archivo de reglas de torneo**: Crea un archivo `tournament_rules.yaml` en la raíz del proyecto. Define parámetros como `points_per_win: 3`, `points_per_draw: 1`, `tiebreaker_mode: head_to_head`. Luego, ejecuta el comando `champions-sp-calc init --config tournament_rules.yaml` para que el motor valide la sintaxis y cargue las reglas en memoria.

2. **Inyectar datos de partidos mediante API REST**: Utiliza el endpoint `POST /api/v1/matches` con un payload JSON que incluya `match_id`, `player_ids` y `result`. Ejemplo: `curl -X POST https://api.champions-sp-calc.com/v1/matches -H "Content-Type: application/json" -d '{"match_id": "M001", "players": ["player_1", "player_2"], "result": "player_1_win"}'`. El sistema devuelve un `match_token` que puedes usar para consultar el estado del cálculo.

3. **Obtener el ranking actualizado**: Llama a `GET /api/v1/ranking?tournament_id=TN001&format=json`. El motor recalcula los puntos en base a las reglas definidas y aplica los desempates automáticamente. Para integraciones en tiempo real, suscríbete al canal WebSocket `/ws/ranking/TN001` y recibirás actualizaciones cada vez que se procese un nuevo partido.

### Mejores Prácticas

- **Usa el modo de cálculo diferido para torneos con alta concurrencia**: Si tu torneo tiene más de 500 partidos simultáneos, activa el flag `--batch-mode` en la inicialización. Esto agrupa los cálculos en lotes de 100 partidos, reduciendo la latencia de la API en un 40% y evitando timeouts en el cliente.

- **Implementa un sistema de caché con Redis para consultas de ranking frecuentes**: Configura el TTL del ranking en 60 segundos usando `--cache-ttl 60`. Para consultas de ranking histórico, utiliza el endpoint `/api/v1/ranking/snapshot?timestamp=ISO8601` que devuelve un snapshot inmutable. Esto evita recalcular el ranking completo en cada petición y mejora la escalabilidad horizontal.
