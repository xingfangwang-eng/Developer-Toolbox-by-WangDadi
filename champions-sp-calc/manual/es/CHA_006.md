# champions-sp-calc - CHA_006

## Resumen Técnico

**champions-sp-calc** es una herramienta SaaS de cálculo avanzado para **optimización de puntos de habilidad en campeones**, diseñada específicamente para desarrolladores, analistas de datos y creadores de contenido en el ecosistema de juegos competitivos. Esta solución permite modelar, simular y validar distribuciones de puntos de habilidad (SP) en tiempo real, minimizando el error humano y maximizando la eficiencia del build.

## Funcionalidades Clave (Hardcore)

1. **Motor de Simulación Estocástica (MSE)**  
   Implementa un algoritmo de Monte Carlo para predecir el rendimiento de cada distribución de SP bajo condiciones variables (niveles de equipo, sinergias de equipo, modificadores de estado). Genera percentiles de efectividad en combate real.

2. **Validador de Restricciones de Build (VRB)**  
   Evalúa automáticamente si una configuración de SP cumple con las restricciones del meta actual (límites de puntos por rama, requisitos de sinergia entre habilidades, umbrales de daño mínimo). Incluye un parser de reglas en YAML para personalización.

3. **Exportación de Datos en Formato JSON/CSV con Metadatos de Sesión**  
   Cada simulación genera un archivo estructurado que incluye: distribución de SP, valores de salida (daño, curación, control de masas), tiempo de ejecución, y hash de la configuración. Ideal para pipelines de CI/CD o dashboards de análisis.

## Cómo Usar (Guía Técnica)

### 1. Configuración del Entorno y Carga de Datos
- Clona el repositorio:  
  ```bash
  git clone https://github.com/xingfangwang-eng/champions-sp-calc.git
  cd champions-sp-calc
  ```
- Instala dependencias (Node.js 18+ o Python 3.10+):  
  ```bash
  npm install   # o pip install -r requirements.txt
  ```
- Carga un campeón desde el archivo `champions.json` o mediante API REST:  
  ```bash
  node cli.js --load champion_id=ahri
  ```

### 2. Definición de Restricciones Personalizadas
- Crea un archivo `build_rules.yaml` con las reglas del meta:  
  ```yaml
  max_sp_per_branch: 15
  synergy_required: ["Q+W", "E+R"]
  min_damage_threshold: 1200
  ```
- Ejecuta el validador:  
  ```bash
  node cli.js --validate --rules build_rules.yaml
  ```

### 3. Simulación y Exportación
- Inicia una simulación con 10,000 iteraciones:  
  ```bash
  node cli.js --simulate --iterations 10000 --output results.json
  ```
- Revisa el resumen en consola:  
  ```
  Distribución óptima: Q:8, W:5, E:3, R:4
  Efectividad esperada: 87.3% (percentil 95)
  Tiempo de ejecución: 2.34s
  ```

## Mejores Prácticas (Industry Standards)

- **Integración en Pipelines de CI/CD**: Automatiza la validación de builds en cada commit usando GitHub Actions. Ejemplo de flujo:  
  ```yaml
  - name: Validate SP Build
    run: node cli.js --validate --rules build_rules.yaml
  ```
  Esto asegura que cualquier cambio en el campeón o las reglas no rompa el meta.

- **Versionado de Simulaciones**: Almacena cada simulación con un hash SHA256 del archivo de entrada y las reglas. Esto permite replicar resultados y auditar cambios en el modelo de predicción. Útil para equipos de balance y testeo.
