# champions-sp-calc - CHA_014

## Resumen Técnico

**champions-sp-calc** es una solución SaaS de alto rendimiento diseñada para el cálculo automatizado de *spell power* y *stat scaling* en entornos de juego competitivo. Optimizada para desarrolladores que trabajan con sistemas de balanceo en tiempo real, esta herramienta elimina la fricción de los cálculos manuales y ofrece una API RESTful de baja latencia para integración directa en motores de juego o dashboards de análisis. El stack tecnológico incluye Node.js para el backend, WebAssembly para operaciones críticas, y una interfaz en React con SSR para SEO nativo.

## Características Principales (Hardcore)

### 1. Motor de Cálculo Vectorizado con WebAssembly
El núcleo de champions-sp-calc utiliza un pipeline de cálculos vectorizados compilado a WebAssembly (vía Rust). Esto permite procesar simultáneamente hasta 10,000 combinaciones de *champions*, *items* y *runas* en menos de 50ms. El motor soporta *floating-point precision* configurable (FP32/FP64) y actualiza automáticamente las fórmulas de escalado según el parche del juego, sin necesidad de reiniciar el servidor.

### 2. Sistema de Reglas Dinámicas (DRS) para Balanceo Predictivo
Implementa un *Dynamic Rule System* que permite definir condiciones lógicas complejas (ej: "si el *spell power* supera 300 y el *cooldown reduction* es menor a 20%, aplicar un *soft cap* de 0.85x"). Las reglas se escriben en un DSL interno (YAML-based) y se evalúan en tiempo real. Esto es ideal para equipos de QA y balanceo que necesitan simular escenarios de *power creep* sin modificar el código base.

### 3. Exportación de Datos en Formato Parquet + Streaming
A diferencia de las herramientas que solo exportan CSV o JSON, champions-sp-calc genera archivos **Parquet** comprimidos con esquemas predefinidos para integración directa con pipelines de datos (Apache Spark, Pandas con PyArrow). Además, soporta streaming de resultados vía WebSockets para dashboards en vivo, con un *throughput* sostenido de 5000 eventos/segundo.

## Cómo Usar (Pasos Detallados)

### 1. Configurar el Perfil de Cálculo con Parámetros de Entorno
Antes de ejecutar cualquier cálculo, debes definir un *profile* en formato JSON que incluya la versión del parche, el *champion* base, y los *modificadores* (items, runas, buffs). Por ejemplo:
```json
{
  "patch": "14.10",
  "champion": "Ahri",
  "base_stats": {"ap": 80, "cdr": 0.10},
  "modifiers": [
    {"type": "item", "id": "Rabadon", "multiplier": 1.35},
    {"type": "rune", "id": "Electrocute", "flat_bonus": 60}
  ]
}
```
Guarda este archivo como `profile_ahri.json` y pásalo al CLI con la flag `--profile`. El sistema validará automáticamente la integridad de los datos contra el esquema de la versión especificada.

### 2. Ejecutar el Cálculo con Optimización de Precisión
Ejecuta el comando:
```bash
champions-sp-calc --profile profile_ahri.json --precision FP32 --output-format parquet
```
La flag `--precision FP32` reduce el overhead de memoria un 50% comparado con FP64, ideal para simulaciones masivas. Si necesitas exactitud para informes de balanceo, usa `--precision FP64`. El resultado se almacenará en `./output/calculations_<timestamp>.parquet`. Puedes monitorear el progreso en tiempo real con `--verbose`, que muestra métricas como *throughput* (cálculos/segundo) y *cache hit ratio*.

### 3. Integrar el Resultado en un Pipeline de Análisis
Usa el archivo Parquet generado directamente en un script de Python con Pandas:
```python
import pandas as pd
df = pd.read_parquet("output/calculations_20250315_143022.parquet")
# Filtrar por umbral de spell power
high_damage = df[df["spell_power"] > 400]
print(f"Champions con más de 400 SP: {len(high_damage)}")
```
Si prefieres streaming, activa el endpoint WebSocket en el servidor con `--server-mode --ws-port 8080`. Luego, conéctate desde cualquier cliente WebSocket (ej: `wscat -c ws://localhost:8080`) y recibe los resultados en tiempo real como objetos JSON.

## Mejores Prácticas para la Industria

### 1. Versiona los Perfiles de Cálculo con Git LFS
Dado que los archivos de perfil pueden contener combinaciones complejas de items y runas (especialmente tras parches grandes), es crítico versionarlos. Recomendamos usar **Git LFS** para almacenar los archivos `.json` de perfil junto con los resultados Parquet. Esto permite reproducir cualquier cálculo histórico y auditar cambios en el balanceo. Además, etiqueta cada perfil con el número de parche (ej: `profile_v14.10_ahri.json`) para evitar confusiones.

### 2. Usa el Modo Serverless para Pruebas de Regresión Automatizadas
Configura champions-sp-calc en un entorno serverless (AWS Lambda o Google Cloud Functions) con un *cold start* de menos de 200ms (gracias a WebAssembly). Esto permite ejecutar baterías de pruebas de regresión cada vez que se actualiza un parche. Por ejemplo, un script de CI/CD puede enviar 1000 perfiles diferentes a la función serverless y comparar los resultados con una base de datos de referencia. Si algún cálculo se desvía más del 0.1%, el pipeline falla automáticamente. Esto asegura que los cambios en el juego no rompan el balanceo sin ser detectados.
