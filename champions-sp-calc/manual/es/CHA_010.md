# champions-sp-calc - CHA_010

## Resumen

**champions-sp-calc** es una herramienta SaaS de cálculo de puntos de campeonato diseñada para optimizar la asignación de recursos en juegos competitivos y sistemas de clasificación. Con un motor de reglas basado en lógica difusa y análisis de datos de rendimiento en tiempo real, permite a desarrolladores y analistas calcular puntuaciones precisas, predecir resultados y ajustar dinámicamente los parámetros de balance. Esta solución es ideal para equipos de esports, plataformas de torneos y aplicaciones de gamificación que requieren cálculos complejos de puntos de campeonato.

## Características Principales

1. **Motor de Cálculo Multivariable con Ponderación Adaptativa**  
   Implementa un sistema de pesos dinámicos que ajusta automáticamente la importancia de cada variable (victorias, derrotas, rendimiento individual, tiempo de juego) según el contexto del campeonato. Soporta configuraciones personalizadas mediante archivos JSON o YAML, permitiendo a los desarrolladores definir reglas de cálculo complejas sin modificar el código.

2. **Integración Directa con APIs de Plataformas de Torneos**  
   Ofrece conectores nativos para APIs de plataformas como Battlefy, Toornament y Challonge, permitiendo importar datos de partidos, jugadores y equipos en tiempo real. Además, expone una API RESTful propia para sincronizar resultados y actualizar clasificaciones de manera automática.

3. **Simulador de Escenarios con Análisis Predictivo**  
   Permite ejecutar simulaciones de múltiples rondas futuras basadas en datos históricos y probabilidades de victoria. Incluye un módulo de regresión lineal y redes neuronales ligeras (TensorFlow.js) para predecir puntuaciones finales y sugerir ajustes estratégicos en la asignación de puntos.

## Cómo Usar

1. **Configuración Inicial del Archivo de Reglas**  
   Crea un archivo `rules.json` en la raíz del proyecto con la estructura de ponderación deseada. Por ejemplo:  
   ```json
   {
     "weight_win": 0.6,
     "weight_loss": 0.2,
     "weight_performance": 0.15,
     "weight_time": 0.05,
     "decay_factor": 0.98
   }
   ```  
   Luego, en tu aplicación, llama a `ChampionsSPCalc.loadRules('rules.json')` para cargar la configuración. El motor valida automáticamente la suma de pesos y aplica un factor de decaimiento exponencial para partidos antiguos.

2. **Cálculo de Puntuaciones con Datos de Partidos**  
   Importa los datos de partidos desde un archivo CSV o desde la API de tu plataforma. Usa el método `calculateScores(playersData)` que acepta un array de objetos con campos como `playerId`, `wins`, `losses`, `kills`, `deaths`, `assists` y `matchDate`. El motor devuelve un objeto con las puntuaciones individuales y un ranking ordenado. Ejemplo:  
   ```javascript
   const scores = calc.calculateScores([
     { playerId: 'P001', wins: 12, losses: 3, kills: 45, deaths: 12, assists: 30, matchDate: '2025-03-15' }
   ]);
   console.log(scores.ranking); // [{ playerId: 'P001', score: 98.7, rank: 1 }]
   ```

3. **Ejecución de Simulaciones Predictivas**  
   Para predecir puntuaciones futuras, utiliza el método `simulateFutureRounds(currentStandings, upcomingMatches)`. Proporciona las clasificaciones actuales y un array de partidos programados con probabilidades de victoria estimadas. El simulador ejecuta 10,000 iteraciones Monte Carlo y devuelve un histograma de probabilidades para cada jugador. Ejemplo:  
   ```javascript
   const predictions = calc.simulateFutureRounds(currentStandings, upcomingMatches);
   predictions.forEach(p => console.log(`${p.playerId}: ${p.medianScore} (Q1: ${p.q1}, Q3: ${p.q3})`));
   ```

## Mejores Prácticas

- **Optimización de Pesos con Validación Cruzada**  
  Para evitar sesgos en los cálculos, realiza una validación cruzada de los pesos usando datos históricos de campeonatos pasados. Divide los datos en conjuntos de entrenamiento (80%) y prueba (20%). Ajusta los pesos hasta que el error cuadrático medio (MSE) entre las puntuaciones calculadas y las reales sea menor a 0.05. Esta práctica garantiza que el motor refleje fielmente la dinámica competitiva.

- **Uso de Caché Distribuida para Cálculos en Tiempo Real**  
  Cuando integres champions-sp-calc en un sistema de torneos en vivo, implementa una capa de caché con Redis para almacenar resultados parciales de cálculos. Por ejemplo, cachea las puntuaciones de cada ronda con una TTL de 5 minutos. Esto reduce la latencia de las consultas de clasificación en un 40% y evita sobrecargar el motor en momentos de alta concurrencia (ej. finales de torneo).
