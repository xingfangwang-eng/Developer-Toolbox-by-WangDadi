## Detailed Content

### What is champions-sp-calc?

champions-sp-calc is a **lightweight, serverless SaaS module** designed to compute composite skill-performance (SP) metrics for competitive gaming champions (e.g., LoL, Dota 2, Valorant). It ingests raw match data (KDA, CS, vision score, objective participation) and outputs a normalized SP rating—enabling developers to build leaderboards, balance patches, or AI-driven coaching dashboards without reinventing the wheel.

### Core Features

1. **Multi-Tier Weighted Aggregation Engine**  
   Unlike simple averages, champions-sp-calc applies a configurable, multi-tier weighting matrix (e.g., early-game vs. late-game performance, role-specific multipliers). This ensures SP scores reflect actual impact, not just raw stats. Built-in support for custom JSON schemas—pipe any game’s telemetry.

2. **Real-Time WebSocket Streaming with Delta Compression**  
   Optimized for live events: the tool exposes a WebSocket endpoint that streams incremental SP updates as match events fire. Delta compression reduces bandwidth by ~70% compared to full payloads. Perfect for embedding in live dashboards or overlay apps for streamers.

3. **Versioned Champion Meta Snapshots**  
   Automatically snapshot SP distributions per game patch. champions-sp-calc stores historical meta states, allowing you to compare champion viability across patches (e.g., “Patch 14.10 vs 14.11”). Exposed as a REST API with time-range filters—ideal for balance analysis and content generation.

### How to Use

1. **Initialize the Module with Your Game Schema**  
   First, define your champion dataset and stat mappings. Use the provided CLI to generate a configuration file:
   ```bash
   champions-sp-calc init --game lol --patch 14.11 --output ./config.yaml
   ```
   This creates a YAML file where you map fields like `kills`, `deaths`, `assists`, `gold_per_minute` to internal weight parameters. For custom games, use `--schema custom` and provide a JSON mapping.

2. **Ingest Match Data via REST or Batch CSV**  
   Push raw match events using the REST endpoint:
   ```bash
   curl -X POST https://api.champions-sp-calc.dev/v1/ingest \
     -H "Content-Type: application/json" \
     -d '{"match_id": "EUW12345", "champion": "Ahri", "stats": {"kills": 8, "deaths": 2, "assists": 10, "cs": 240, "vision_score": 45}}'
   ```
   For bulk analysis, upload a CSV via the `/batch` endpoint. The tool validates fields against your schema and returns a processing receipt with a `job_id`.

3. **Query SP Scores and Export Snapshots**  
   Retrieve computed SP scores with time-series filtering:
   ```bash
   curl "https://api.champions-sp-calc.dev/v1/sp?champion=Ahri&patch=14.11&start=2025-01-01&end=2025-03-01"
   ```
   Returns a JSON array of SP values per match. To export a full meta snapshot for a patch, call:
   ```bash
   curl "https://api.champions-sp-calc.dev/v1/snapshot?patch=14.11&format=csv" -o meta_14.11.csv
   ```
   The CSV includes champion name, mean SP, median SP, volatility, and sample size.

### Best Practices

- **Use Precomputed Weights for Balance Patches**  
  When Riot or Valve drops a balance patch, don’t manually tweak weights. Instead, run `champions-sp-calc calibrate --patch 14.12 --reference 14.11`. The tool auto-adjusts weight coefficients to maintain SP score continuity across patches. This prevents false “champion buff/nerf” signals in your analytics.

- **Implement Client-Side Caching for Live Dashboards**  
  For real-time overlays (e.g., Twitch stream overlays), cache the last 5 minutes of SP deltas in your frontend (localStorage or IndexedDB). Combine with the WebSocket stream’s delta compression to minimize re-renders. Use the `sp:delta` event’s `champion_id` and `score_change` fields to update only affected UI components—avoids full DOM repaints and keeps frame rates smooth.
