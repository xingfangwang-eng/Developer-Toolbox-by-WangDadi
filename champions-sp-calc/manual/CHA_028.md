# champions-sp-calc - CHA_028

## Overview

champions-sp-calc is an enterprise-grade service point (SP) calculator engineered for multi-tenant SaaS architectures. It delivers deterministic SP aggregation and real-time quota simulation across distributed microservices. Optimized for high-throughput environments, this tool eliminates SP drift and ensures SLA compliance in competitive gaming or loyalty ecosystems.

## Core Features

### 1. Deterministic SP Aggregation Engine
Built on a custom DAG (Directed Acyclic Graph) resolver, this engine processes concurrent SP mutations from multiple upstream services with ACID compliance. It uses a lock-free Merkle tree structure to validate SP state transitions, ensuring zero data race conditions even under 10k+ QPS. Ideal for platforms requiring tamper-proof leaderboard recalculation.

### 2. Real-Time Quota Simulation with Predictive Throttling
Leveraging a sliding window algorithm and adaptive rate limiting, the simulator predicts SP exhaustion points before they occur. It integrates with Redis Streams for low-latency event sourcing, allowing developers to test "what-if" scenarios (e.g., spike in user redemptions) without affecting production data. Includes a built-in Prometheus exporter for Grafana dashboards.

### 3. Multi-Tenant Isolation via Namespace Sharding
Implements hierarchical namespace sharding to isolate SP pools per tenant (e.g., game server, region). Each shard operates on its own PostgreSQL partition with autoscaling connection pooling via PgBouncer. The API gateway enforces JWT-scoped access, ensuring that tenant A’s SP mutations never leak into tenant B’s state machine.

## How to Use

### Step 1: Initialize the SP State Machine with Custom Rules
Define your SP rule set in a YAML configuration file. For example, to set a daily cap of 500 SP per user with a 1% decay rate per hour:
```yaml
rules:
  - id: daily_cap
    type: counter
    max: 500
    reset: daily_utc
  - id: decay
    type: exponential
    rate: 0.01
    interval: 3600
```
Run `champions-sp-calc init --config sp_rules.yaml` to bootstrap the state machine. The tool will automatically create the required PostgreSQL partitions and Redis streams.

### Step 2: Simulate Concurrent SP Mutations via CLI
Use the built-in load generator to test SP aggregation under stress:
```bash
champions-sp-calc simulate --tenants 50 --users-per-tenant 1000 --mutations 10000 --mode concurrent
```
This spawns 50 goroutines (one per tenant) that fire SP mutations (earn, redeem, expire) against the API. The tool outputs latency histograms and SP consistency checks (e.g., total SP earned vs. redeemed). Watch for P99 latency spikes—if >50ms, adjust the `--batch-size` flag.

### Step 3: Export SP Audit Logs for Compliance
Generate a cryptographically signed audit trail of all SP state changes:
```bash
champions-sp-calc export --format jsonl --since "2024-01-01T00:00:00Z" --sign-key ./private.pem
```
This produces a JSONL file with each line containing a SHA-256 hash of the previous entry, forming a blockchain-like chain of custody. Use `--verify` to check integrity: `champions-sp-calc export --verify audit_2024.jsonl`. Output is compatible with Splunk and ELK stacks.

## Best Practices

### Practice 1: Use Idempotency Keys for SP Mutations
Always include a unique `idempotency_key` (UUID v4) in every SP mutation request. This prevents double-spending when clients retry failed requests (e.g., due to network timeouts). Configure the key TTL in `sp_rules.yaml` under `idempotency.ttl: 3600`. The tool will reject duplicate keys within the TTL window, returning the original SP state.

### Practice 2: Monitor SP Drift with Baseline Snapshots
Schedule a nightly cron job to take a baseline SP snapshot using `champions-sp-calc snapshot --tenant all`. Compare it with the previous day’s snapshot using the built-in diff tool: `champions-sp-calc diff --snapshot1 baseline_day1.json --snapshot2 baseline_day2.json`. Any divergence >0.1% triggers an alert—this catches silent data corruption or misconfigured rules before they propagate to user-facing leaderboards.
