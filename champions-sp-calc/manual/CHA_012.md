# champions-sp-calc - CHA_012

## Overview

`champions-sp-calc` is a high-performance, precision-engineered service point calculator for competitive esports and sports analytics. Built for developers, data scientists, and tournament operators who need deterministic, auditable scoring logic across multi-format championship brackets—no bloat, no black boxes, just pure, testable math.

## Core Features

### 1. **Multi-Format Weighted Scoring Engine**
Supports Swiss-system, double-elimination, round-robin, and hybrid playoff structures with configurable weight vectors. Assign dynamic coefficients per round, opponent tier, or map score differential—all computed in constant time via a precompiled decision tree. No more duct-taping spreadsheets for tiebreaker logic.

### 2. **Deterministic Tiebreaker Resolution with Provenance**
Implements a cascading tiebreak resolver (head-to-head → map differential → round differential → strength of victory) that outputs a verifiable hash chain for every tiebreak decision. Every tiebreak step is logged with an immutable audit trail, critical for high-stakes tournaments where disputes can cost thousands.

### 3. **Zero-Dependency CLI & REST API**
Ship it as a standalone binary (< 2MB) or spin up the embedded HTTP server for real-time scoring during live events. Accepts JSON, CSV, or protobuf input formats. Integrates natively with CI/CD pipelines for regression testing scoring logic before each patch day.

## How to Use

### 1. Define Your Championship Graph
Create a `config.yaml` with your tournament structure:
```yaml
format: double_elimination
max_teams: 16
scoring:
  win_points: 3
  draw_points: 1
  map_differential_weight: 0.4
tiebreakers:
  - head_to_head
  - map_differential
  - strength_of_victory
```
Then validate the graph with `sp-calc validate config.yaml` to catch structural errors (e.g., bracket depth mismatch, missing byes) before any matches are played.

### 2. Ingest Match Results via Stream or Batch
Pipe live results from your event API:
```bash
curl -X POST http://localhost:8080/api/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{"match_id": "M-042", "team_a": "NRG", "team_b": "G2", "score_a": 3, "score_b": 1}'
```
Or batch-import a CSV of historical matches: `sp-calc import --file results.csv --format csv`. The engine validates each row against the configured bracket state and rejects inconsistent data (e.g., duplicate match IDs, impossible scores).

### 3. Generate Standings & Export Audit Trails
Run `sp-calc standings --output json` to get a ranked list with computed tiebreaker data. For compliance, export a full audit trail:
```bash
sp-calc audit --match-range M-001:M-128 --format html > audit_report.html
```
This produces a human-readable timeline of every scoring event, tiebreaker invocation, and hash proof—ready for review by tournament admins or legal teams.

## Best Practices

### **Version-Control Your Scoring Config**
Treat `config.yaml` as a first-class artifact in your repository. Use `sp-calc diff config_v1.yaml config_v2.yaml` to visualize changes in scoring weights or tiebreaker order before deploying to production. This prevents silent scoring regressions when updating tournament rules mid-season.

### **Run Scoring Regression Tests in CI**
Add a `sp-calc test --fixtures fixtures/` step to your pipeline. Maintain a suite of historical match datasets with known correct standings. The tool will compare its output against your expected results and fail the build if any divergence is detected—essential for leagues that patch scoring logic between splits.
