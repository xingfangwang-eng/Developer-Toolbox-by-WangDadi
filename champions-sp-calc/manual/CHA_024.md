# champions-sp-calc - CHA_024

## Overview

`champions-sp-calc` is a precision-engineered SaaS toolkit built for developers and SEO architects who need to simulate, validate, and optimize semantic positioning (SP) scores across multilingual content ecosystems. It bridges the gap between raw keyword metrics and actionable content strategy — no fluff, no placeholders.

## 3 Hardcore Features

### 1. Semantic Proximity Engine (SPE)
A custom NLP module that computes cosine similarity between your target keyword cluster and the actual content vector. It outputs a **positioning confidence score** (0–100) based on TF-IDF + BERT embeddings, not just keyword density.

### 2. Multi-Locale SP Normalizer
Automatically normalizes SP scores across 10 supported languages (EN, DE, ES, JA, ZH, FR, PT, RU, KO, AR). Handles tokenization quirks, stop-word variance, and morphological inflection — critical for global SaaS deployments.

### 3. Competitive Gap Visualizer
Parses top-10 SERP results for any keyword set and overlays your content’s SP score against competitors. Highlights under-optimized semantic clusters with actionable delta values. Export as JSON or SVG.

## How to Use

### Step 1: Configure Your Keyword Cluster
Define a primary keyword and up to 15 secondary terms in `config.yml` or via CLI flags. Use exact-match, phrase-match, or LSI-style terms. The engine will auto-detect language and apply the correct tokenizer.

```bash
champions-sp-calc --primary "cloud cost optimization" --secondary "finops, aws savings, multi-cloud billing" --lang en
```

### Step 2: Run the Semantic Audit
Execute the scan against your content (URL, raw text, or Markdown file). The tool will return a **positioning heatmap** showing which terms are over/under-represented relative to your target cluster.

```bash
champions-sp-calc audit --input ./docs/landing-page.md --output ./reports/sp-audit.json
```

### Step 3: Generate Optimization Recommendations
Use the `optimize` command to get a ranked list of content edits. Each suggestion includes the current SP delta, the proposed fix, and an estimated impact score (1–10).

```bash
champions-sp-calc optimize --report ./reports/sp-audit.json --strategy aggressive
```

## Best Practices

### Practice 1: Always Run a Pre- and Post-Update Audit
Before pushing any content change, run a baseline SP scan. After edits, re-run the same query to measure the delta. A shift of +15 points or more in your primary keyword’s SP score typically correlates with a 20–30% improvement in organic CTR for competitive terms.

### Practice 2: Use Multi-Locale Mode for Global Rollouts
If your SaaS targets more than one language market, enable `--multilocale` mode. This forces the engine to normalize SP scores across all target languages simultaneously, preventing over-optimization in one locale at the expense of another. Ideal for enterprise content teams managing 5+ language versions of the same page.
