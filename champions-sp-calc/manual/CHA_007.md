# champions-sp-calc - CHA_007

## Overview

**champions-sp-calc** is a high-performance SaaS toolkit engineered for competitive intelligence teams and data-driven SEO strategists who need to reverse-engineer search engine result page (SERP) dynamics at scale. It transforms raw keyword clusters into actionable champion-page scoring matrices, enabling you to pinpoint content gaps, authority thresholds, and SERP volatility patterns without drowning in spreadsheets. Built for Node.js environments and CI/CD pipelines, this tool eliminates the guesswork from topical authority modeling.

## Hardcore Features

### 1. Multi‑Dimensional Champion Score (MDCS) Engine
Instead of relying on a single metric like Domain Rating (DR) or backlink count, champions-sp-calc computes a weighted composite score across **10+ signals**: click-through rate (CTR) distribution, core web vital percentiles, topical TF‑IDF cosine similarity, entity co‑occurrence density, and historical rank stability. The engine outputs a normalized champion score (0–100) that predicts which page is most likely to hold its SERP position under algorithm updates.

### 2. Reverse SERP Volatility Analysis (RSVA)
This feature ingests daily or weekly SERP snapshots (via API or CSV) and automatically detects **rank turbulence events** — sudden shifts in top‑10 composition, new entrants from low‑authority domains, or clustering of thin‑content pages. RSVA highlights “fragile champions” that rank high but show high volatility, allowing you to prioritize content refreshes or link‑building campaigns before they drop out of the top 5.

### 3. Content Gap Heatmap Generation
Using your target keyword set, champions-sp-calc cross‑references the top‑ranking pages and identifies **missing subtopics, question formats, or media types** (video, infographic, interactive). The heatmap overlays keyword difficulty, search volume, and champion score to surface low‑competition opportunities where you can deploy high‑impact content with minimal effort — essentially a built‑in “low‑hanging fruit” detector.

## How to Use (Detailed Operations)

### 1. Configure the Keyword Input Pipeline
Create a `keywords.csv` file with columns: `keyword`, `locale` (e.g., `en-us`), `device` (e.g., `mobile`). Then run the ingestion command:
```bash
champions-sp-calc ingest --input keywords.csv --serp-api-key YOUR_KEY
```
This triggers asynchronous SERP fetching with automatic rate‑limiting (5 requests/second). The tool validates each keyword against Google’s search operators and flags non‑indexable queries (e.g., those returning zero results).

### 2. Execute the Champion Scoring & Volatility Scan
After ingestion, generate the champion matrix:
```bash
champions-sp-calc score --window 30d --metrics dr,ctr,ttfb,entity_density
```
The `--window 30d` flag pulls historical SERP data for the past 30 days. The tool then runs the MDCS engine and RSVA in parallel, outputting two JSON files: `champion_scores.json` (with per‑keyword champion page URL, score, and signal breakdown) and `volatility_report.json` (listing keywords where rank fluctuation exceeds 3 positions in the window).

### 3. Export the Content Gap Heatmap as an Interactive HTML
For immediate strategic action, generate the heatmap:
```bash
champions-sp-calc heatmap --output ./reports/heatmap.html --topics 15
```
This produces a standalone HTML file with a color‑coded matrix (rows = keywords, columns = subtopics/media types). Hover over any cell to see the champion score of the highest‑ranking page for that subtopic. The report also includes a “Priority Actions” table sorted by opportunity score (search volume × gap intensity).

## Industry Best Practices

### Practice 1: Integrate RSVA into Your Weekly SEO Sprint
Don’t wait for monthly rank‑tracking reports. Set up a cron job (or GitHub Action) that runs `champions-sp-calc score --window 7d` every Monday morning. Pipe the `volatility_report.json` into your project management tool (e.g., Linear, Jira) to auto‑create tickets for content refreshes on pages with high volatility. This proactive approach reduces the risk of losing top‑3 positions during Google core updates.

### Practice 2: Use the Heatmap for Cluster‑Based Content Planning
Instead of targeting individual keywords, group keywords into clusters (e.g., “SaaS pricing,” “SaaS features,” “SaaS alternatives”) and run the heatmap on each cluster. Look for subtopics where the champion score is below 40 but search volume is above 500/month — these are prime candidates for “pillar page” creation. Link your new pillar page from existing high‑scoring pages in the cluster to amplify topical authority and boost cluster‑wide rankings.
