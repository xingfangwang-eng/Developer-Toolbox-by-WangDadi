## What is champions-sp-calc?

`champions-sp-calc` is a battle-tested, multi‑tenant SaaS tool purpose‑built for automating spell power calculations in competitive gaming environments. It eliminates manual math, reduces latency in build optimization, and provides real‑time damage output projections based on dynamic stat modifiers, buff stacks, and resist penetration models.

Unlike generic calculators, it integrates directly with game API feeds, supports custom stat weighting, and exposes a composable calculation engine for advanced modding.

## Core Features (Hardcore)

### 1. Real‑Time Stat Chain Solver  
The engine resolves multi‑step stat dependencies (e.g., `Intellect → Spell Power → Crit Rating → Proc Chance`) in under 2ms per update cycle. It handles conditional buff stacking, diminishing returns, and exponential scaling formulas without pre‑computed lookups. This means you can iterate gear sets in real time without frame drops.

### 2. Dynamic Resistance Penetration Matrix  
Instead of a flat penetration slider, `champions-sp-calc` computes a **3D resistance‑penetration matrix** that factors in target level, debuff uptime, and partial resists. It outputs effective damage multipliers per school of magic, with a breakdown of over‑penetration loss and soft‑cap thresholds. This is critical for min‑maxing in high‑tier PvE or PvP scenarios.

### 3. Composable Formula Scripting (Lua/JSON)  
Advanced users can inject custom formula modules via a sandboxed scripting layer. You can override default scaling curves, add multiplicative modifiers, or simulate future patch changes. The engine validates your script against type constraints and runtime safety checks, then hot‑loads it without restarting the service. This is production‑ready for theorycrafters and private server maintainers.

## How to Use (Detailed Steps)

### Step 1: Initialize a Calculation Session via the CLI or API  
```bash
# Using the CLI (requires Node.js 18+)
npx champions-sp-calc init --character "Mage" --level 70 --gear-file ./gear.json

# Or via REST API
curl -X POST https://api.champions-sp-calc.dev/v1/session \
  -H "Content-Type: application/json" \
  -d '{"character_class":"Mage","level":70,"gear":"https://raw.example.com/gear.json"}'
```
This creates a session with a unique ID, loads the character’s base stats, and pre‑fetches the latest game data (spell coefficients, resistance tables). You’ll receive a session token valid for 24 hours.

### Step 2: Apply Buffs and Debuffs as Modifier Layers  
Use the modifier stack to apply temporary effects. Each buff must be defined with its **source**, **duration**, and **stacking rule** (additive, multiplicative, or exclusive).  
```javascript
// Example using the SDK (JavaScript)
const session = await championsSpCalc.loadSession('session_abc123');
session.applyBuff({
  name: 'Arcane Intellect',
  source: 'Self',
  stat: 'intellect',
  value: 30,
  type: 'flat',
  duration: 3600
});
session.applyDebuff({
  name: 'Curse of Elements',
  source: 'Warlock',
  stat: 'fire_resist',
  value: -75,
  type: 'flat',
  duration: 120
});
```
The engine recalculates the stat chain immediately. You can inspect the delta in real time via `session.getDelta()`.

### Step 3: Run a Full Damage Projection and Export the Report  
Execute the solver against a specific spell or rotation:  
```bash
# CLI: simulate a single spell cast
npx champions-sp-calc simulate --session session_abc123 --spell "Fireball" --target-level 73

# Outputs a JSON report with:
# - Base damage range (min/max)
# - Effective damage after resist/penetration
# - Crit probability with current buffs
# - Overkill risk percentage
```
Export the report as JSON, CSV, or an interactive HTML dashboard. The dashboard includes a waterfall chart showing damage contribution per modifier layer.

## Best Practices (Industry‑Proven)

### 1. Always Pre‑Warm the Session Cache for Production Pipelines  
If you’re integrating `champions-sp-calc` into a CI/CD pipeline or a live simulation service, call `session.warmCache()` during deployment. This pre‑loads all game data (spell tables, resistance curves) into memory, reducing first‑call latency from ~800ms to under 50ms. Without this, you may hit cold‑start penalties during automated gear comparisons or regression tests.

### 2. Use the Modifier Stack Logging for Audit Trails  
In competitive or tournament settings, enable `logModifierStack: true` in your session config. This records every buff, debuff, and gear change with a timestamp and source identifier. If a damage output discrepancy arises, you can replay the exact modifier chain and pinpoint the cause (e.g., a double‑stacked buff or a misapplied debuff). This is crucial for maintaining trust in automated scoring systems.
