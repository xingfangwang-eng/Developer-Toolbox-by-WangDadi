# champions-sp-calc - CHA_021

## Overview

`champions-sp-calc` is a high-performance, serverless SaaS tool engineered for real-time **Skill Point (SP) optimization** in competitive gaming and esports analytics. It eliminates manual spreadsheet hell by applying multi-variable regression models to calculate optimal SP allocation across champion builds, factoring in patch changes, win rates, and player-specific playstyle data.

**Keywords**: champions-sp-calc tool, champions-sp-calc solution, champions-sp-calc software

## Detailed Content

### What is champions-sp-calc?

`champions-sp-calc` is not just a calculator—it's a **decision engine** for serious gamers and data-driven coaches. It ingests raw match data, applies weighted SP efficiency metrics, and outputs a ranked list of viable builds. It’s built for developers who want to automate build theorycrafting without touching Python notebooks.

### Core Features

1. **Patch-Aware Dynamic Weighting Engine**  
   Automatically scrapes and parses the latest game patch notes (via API or JSON config) and adjusts SP cost-per-point thresholds in real-time. No more hardcoding stat values that go stale after a Tuesday hotfix.

2. **Multi-Objective Pareto Optimization**  
   Instead of a single “best” build, the tool returns a Pareto frontier of SP allocations across survivability, burst damage, and utility. Each node is annotated with a confidence score based on historical match data.

3. **Playstyle Signature Profiling**  
   Input a `.json` or `.csv` of your last 50 matches—the tool generates a **playstyle vector** (aggressive, defensive, hybrid) and tailors SP recommendations that align with your actual gameplay tendencies, not just theoretical maxima.

### How to Use

1. **Initialize with your champion data**  
   Clone the repo and run `npm install`. Create a `config.yaml` file pointing to your local match history export or a public API endpoint. Example:  
   ```yaml
   champion: "Aurelion Sol"
   data_source: "./data/last_50_matches.csv"
   patch: "14.10"
   optimization_mode: "burst"
   ```

2. **Execute the optimization pipeline**  
   Run `npx champions-sp-calc --config config.yaml --output builds.json`. The tool will:  
   - Parse patch data and apply SP cost adjustments  
   - Run a genetic algorithm (default: 500 generations, 200 population size)  
   - Output a ranked list of builds with SP distributions and expected delta in win rate

3. **Visualize and export the Pareto frontier**  
   Use the built-in `--viz` flag to generate an interactive HTML plot of the Pareto front. Each point is clickable and shows the full SP tree. Export as `.svg` or `.html` for team discussions.

### Best Practices

- **Always pin your patch version in the config**  
   The dynamic patcher is powerful, but for reproducibility in team environments, set `"patch": "14.10"` explicitly. This ensures everyone on your squad gets the same SP calculations, even if the live API updates mid-session.

- **Use playstyle signatures for role-specific builds**  
   Don’t rely on a single “optimal” build for all players. Run the tool with `--profile` and a `.csv` of your team’s match data. This uncovers hidden inefficiencies—e.g., a player with a high aggression score might benefit from SP shifted into early-game burst, even if the global meta favors sustain.
