# champions-sp-calc - CHA_001

## Overview

**champions-sp-calc** is a high-performance, SEO-optimized SaaS tool designed for developers and data analysts who need to compute, validate, and export champion vs. challenger statistical significance in real time. Unlike generic A/B testing calculators, this engine is built for multi-variant, multi-metric scenarios with native support for Bayesian and frequentist models. Keywords: champions-sp-calc tool, champions-sp-calc solution, champions-sp-calc software.

## Detailed Content

### What is champions-sp-calc?

champions-sp-calc is a production-grade statistical engine that eliminates guesswork from experiment evaluation. It ingests raw conversion or revenue data, automatically detects distribution type, and outputs decision-ready metrics—including lift, confidence intervals, and posterior probabilities—without requiring a data science background. Designed for CI/CD pipelines, it exposes both a CLI and a RESTful API, making it ideal for teams that ship experiments weekly.

### Core Features

- **Adaptive Bayesian Sequential Testing** — Automatically switches from fixed-horizon to sequential analysis as data streams in, reducing sample size requirements by up to 40% while controlling false discovery rate. Supports both Beta-Binomial and Gamma-Poisson conjugate priors out of the box.
- **Multi-Metric Composite Scoring** — Weighs up to 12 KPIs (conversion, revenue, retention, etc.) into a single champion score using custom weight matrices. Outputs a normalized composite index with bootstrapped confidence intervals, enabling holistic winner selection without manual trade-off analysis.
- **One-Click Export to Experiment Dashboards** — Generates a JSON payload compatible with Google Optimize, Optimizely, and VWO, plus a markdown report with inline tables and visualizations. Integrates natively with GitHub Actions and GitLab CI for automated post-hoc analysis on every merged experiment branch.

### How to Use

1. **Ingest experiment data via the CLI**  
   Run `champions-sp-calc ingest --file experiments/latest.csv --format wide --metrics conversion,revenue,retention` to parse a wide-format CSV with column headers `variant`, `impressions`, `conversions`, `revenue`, `retention`. The tool auto-detects missing values, imputes zero-conversion rows, and validates that each variant has at least 500 observations. Outputs a validated JSON snapshot to `./.champions/cache/`.

2. **Run a Bayesian analysis with custom priors**  
   Execute `champions-sp-calc analyze --cache ./.champions/cache/ --model bayesian --prior-alpha 2.5 --prior-beta 97.5 --simulations 50000` to compute posterior distributions for each metric. The engine uses Hamiltonian Monte Carlo (via a lightweight C++ backend) to sample 50,000 draws per metric, then outputs the probability that each challenger beats the champion (P(beat)) alongside 95% highest density intervals. Results are printed to stdout as a formatted table and saved to `./.champions/results/analysis.json`.

3. **Export a decision report and push to your dashboard**  
   Call `champions-sp-calc export --results ./results/analysis.json --format optimizely --output ./reports/experiment_42.json` to generate a JSON payload ready for Optimizely’s API. For a human-readable version, use `--format markdown` to produce a report with a lift table, a forest plot (ASCII or SVG), and a decision summary (e.g., “Challenger B wins with 94.3% probability on composite score”). Append `--push` to directly send the payload to your configured experiment dashboard webhook.

### Best Practices

- **Pre-register your metrics and priors before launch** — Use the `champions-sp-calc register --metrics conversion,revenue --priors alpha=2.5,beta=97.5` command to store your analysis plan in a `champions-config.yaml` file. This prevents p-hacking by ensuring all metric definitions and prior parameters are locked before data collection begins. The tool will reject any analysis that deviates from the registered plan unless explicitly overridden with `--force`.

- **Run a sensitivity analysis on your composite weights** — After computing individual metric posteriors, execute `champions-sp-calc sensitivity --results ./results/analysis.json --weight-range 0.2,1.0 --step 0.1` to see how the champion selection changes across 40 different weight configurations. This reveals whether your winner is fragile (e.g., only wins under narrow weight assumptions) or robust. Export the sensitivity matrix as a heatmap (`--format heatmap`) to share with stakeholders during review.
