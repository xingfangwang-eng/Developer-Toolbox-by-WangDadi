## Overview

**champions-sp-calc** is a high-performance, multi-tenant SaaS engine purpose-built for calculating **champion/ challenger statistical significance** in A/B testing and multivariate experimentation pipelines. It eliminates guesswork by delivering **Bayesian and frequentist inference** in real-time, tailored for product growth teams running high-velocity experiments at scale.

## What is champions-sp-calc?

champions-sp-calc is a **serverless-first**, **API-native** statistical calculator that ingests raw experiment data (conversion rates, sample sizes, revenue per user) and outputs **power analysis**, **sequential testing boundaries**, and **lift confidence intervals**—without ever exposing your raw data to third-party servers. Designed for developers who need to embed statistical rigor directly into their CI/CD or experimentation platform.

---

## Core Features

### 1. **Multi-Model Bayesian Posterior Engine**
- Supports **Beta-Binomial**, **Gamma-Poisson**, and **Log-Normal** priors.
- Automatically selects the optimal conjugate prior based on data sparsity and metric type (binary, count, continuous).
- Returns **posterior probability of superiority** and **expected loss** for each variant—no black-box p-values.

### 2. **Sequential GLR (Generalized Likelihood Ratio) Stopping Boundaries**
- Implements **always-valid inference** using the **mixture sequential GLR** framework.
- Dynamically adjusts critical thresholds as sample size grows, preventing **peeking bias** and false positives.
- Exposes a **stopping boundary curve** via REST endpoint for real-time dashboard integration.

### 3. **Delta Method & Bootstrap Confidence Intervals for Ratio Metrics**
- Handles **non-normal metrics** like revenue per user, CTR, or conversion rate per session.
- Applies **delta method** for asymptotic normality or **stratified bootstrap** for small-sample robustness.
- Outputs **95% / 99% confidence intervals** with **bias-corrected accelerated (BCa)** adjustment.

## How to Use

### Step 1: Configure Experiment Parameters via API
```bash
curl -X POST https://api.champions-sp-calc.io/v1/experiment \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "metric_type": "conversion_rate",
    "control_data": {"trials": 10000, "successes": 1200},
    "challenger_data": {"trials": 10000, "successes": 1280},
    "prior": {"alpha": 1, "beta": 1},
    "sequential_boundary": {"alpha_spend": 0.05, "method": "glr"}
  }'
```
**Response:** Returns `posterior_superiority`, `expected_loss`, `lift_ci_95`, and `stop_recommendation`.

### Step 2: Retrieve Sequential Stopping Boundary
```bash
curl -X GET https://api.champions-sp-calc.io/v1/experiment/EXPERIMENT_ID/boundary \
  -H "Authorization: Bearer YOUR_API_KEY"
```
**Response:** Returns a time-series array of critical thresholds (z-scores or Bayes factors) for each interim analysis checkpoint.

### Step 3: Validate & Export Final Decision
```python
# Python SDK example
from champions_sp_calc import ExperimentClient

client = ExperimentClient(api_key="YOUR_API_KEY")
result = client.decide(
    experiment_id="EXPERIMENT_ID",
    decision_threshold=0.95,   # minimum posterior probability
    min_effect_size=0.02       # minimum absolute lift to consider
)
print(result.recommendation)  # "champion", "challenger", or "continue"
```
**Output:** Structured JSON with actionable recommendation, Bayesian power, and a **decision log** for audit trails.

## Best Practices

### 1. **Always Set a Minimum Effect Size (MES)**
- Without an MES, even trivial lifts (e.g., 0.01%) can trigger a "winner" declaration given enough traffic.
- Use `min_effect_size` in your `decide()` call to filter out economically insignificant results. For SaaS metrics, a lift of 2–5% is a common MES baseline.

### 2. **Integrate Sequential Testing into Your CI/CD Pipeline**
- Instead of waiting for a fixed sample size, call `champions-sp-calc` after every 1,000 observations per variant.
- Use the `stop_recommendation` flag to automatically halt experiments and deploy the winner—this reduces time-to-decision by up to 60% without inflating false positive rates.
