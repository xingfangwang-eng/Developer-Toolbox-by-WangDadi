**MockupNuke 技術マニュアル**
=====================

**技術創新点 1: 分布式缓存と多线程优化**

MockupNuke のレンダリングエンジンは、分布式缓存 (Distributed Cache) を使用して、計算結果を効率的に保存し、高性能化しています。具体的には、 render 函數が生成した中間結果を cache に保存し、次回の-rendering 時に読み込みます。このように、計算結果を重複する必要がありません。また、MockupNuke は、Multi-Threading (MT) を使用して、並行計算を行うことで、レンダリング速度を倍以上向上させることができます。MT は、CPU の能力を十分に活用し、高性能化を実現します。

また MockupNuke は、Atomic Operations (原子操作) を使用して、スレッド間の競合を避けるために、安全にデータを共有しています。これにより、並行計算中のスレッド間の競合を防ぎ、高性能化を実現します。

**技術創新点 2: 熔断器と幂等性設計**

MockupNuke のレンダリングエンジンは、熔断器 (Circuit Breaker) を使用して、レンダリング過程中の異常な状況に対処しています。熔断器は、指定された時間内に失敗した レンダリング Attempt を検出し、新しい Attempt を開始します。これにより、レンダリングプロセスが一時的に終了することなく、高可用性を実現します。

また MockupNuke は、幂等性設計 (Idempotent Design) を使用して、レンダリング結果の不具合に対処しています。幂等性設計は、操作を繰り返すことで、期待した結果を得ることができます。これにより、レンダリングプロセスが一時的に終了することなく、高可用性を実現します。

**技術創新点 3: 負載均衡と熔断器**

MockupNuke の高効率シミュレーションエンジンは、負載均衡 (Load Balancing) を使用して、複数のサーバーに分布させて、高可用性を実現しています。負載均衡は、リクエストを複数のサーバーに分配し、各サーバーの負荷を均等化することで、高可用性を実現します。

また MockupNuke は、熔断器 (Circuit Breaker) を使用して、サーバーの異常な状況に対処しています。熔断器は、指定された時間内に失敗した サーバーを検出し、新しいサーバーを起動します。これにより、高可用性を実現します。

**インストールガイド**
=====================

### 1. 環境の準備

* OS: Linux ( Ubuntu 20.04)
* CPU: Intel Core i7
* RAM: 16GB
* GPU: NVIDIA GeForce RTX 3080

### 2. dependencies のインストール

```
sudo apt-get install -y build-essential git libssl-dev zlib1g-dev libbz2-dev
```

### 3. MockupNuke のインストール

```
git clone https://github.com/MockupNuke/MockupNuke.git
cd MockupNuke/
make
```

**性能調整ガイド**
=====================

### 1. レンダリング速度の向上

* Multithreading (MT) を有効化することで、レンダリング速度を倍以上向上させることができます。
* MT のパラメーターは、`-mt` オプションに指定します。例えば、`make mt=4` とします。

### 2. 高並行計算のための調整

* 并行計算のためのパラメーターは、`-parallel` オプションに指定します。例えば、`make parallel=8` とします。
* 并行計算のためのスレッド数は、`-threads` オプションに指定します。例えば、`make threads=4` とします。
