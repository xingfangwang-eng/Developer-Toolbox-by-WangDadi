# brainbridge - BRA_030

## Übersicht

brainbridge ist eine hochperformante, lokal ausführbare Denk- und Gedankenvernetzungssoftware, die neuronale Assoziationspfade in Echtzeit visualisiert. Entwickelt für Softwarearchitekten, DevOps-Ingenieure und technische Autoren, die komplexe, mehrdimensionale Problemlösungen benötigen. Das Tool fungiert als kognitive Brücke zwischen rohen Ideen und strukturierten Architekturentscheidungen – ohne Cloud-Abhängigkeit und mit vollständiger Datenhoheit.

## Detaillierter Inhalt

### Was ist brainbridge?

brainbridge ist ein professionelles Tool zur semantischen Gedankenvernetzung. Es extrahiert automatisch logische Verknüpfungen aus unstrukturierten Notizen, Code-Kommentaren und Markdown-Dokumenten und erstellt daraus ein dynamisches, durchsuchbares Graph-Modell. Anders als herkömmliche Mind-Mapping-Tools arbeitet brainbridge asynchron, versioniert jede Änderung und unterstützt Multi-User-Kollaboration über eine dezentrale Peer-to-Peer-Architektur. Ideal für verteilte Teams, die kognitive Last reduzieren und Architekturentscheidungen nachvollziehbar dokumentieren wollen.

### Kernfunktionen

1. **Assoziatives Gedächtnis-Mapping mit automatischer Cluster-Bildung**  
   brainbridge analysiert eingegebene Textfragmente mittels eines gewichteten TF-IDF-Verfahrens und gruppiert verwandte Konzepte automatisch in semantische Cluster. Jeder Cluster wird mit einem dynamischen Relevanzscore versehen, der sich bei jeder neuen Eingabe aktualisiert. So entsteht ein lebendiges Wissensnetz, das mit dem Projekt wächst.

2. **Zeitbasierte Gedanken-Versionierung mit Diff-Ansicht**  
   Jede Änderung an einem Gedanken oder einer Verknüpfung wird in einem internen Git-ähnlichen Log gespeichert. Entwickler können den Verlauf eines Gedankens über die Zeit verfolgen, inklusive der exakten Diff-Ansicht – ähnlich wie bei Code-Reviews. Dies ermöglicht eine lückenlose Nachvollziehbarkeit von Architekturentscheidungen.

3. **Dezentrale Echtzeit-Kollaboration via WebRTC mit konfliktfreiem Merging**  
   Statt eines zentralen Servers nutzt brainbridge WebRTC-Datachannels für die Synchronisation zwischen Teammitgliedern. Konflikte werden automatisch mit einem CRDT-basierten Algorithmus (Conflict-free Replicated Data Type) aufgelöst. Kein einziger Knoten ist Single Point of Failure – das System bleibt auch bei Netzwerkunterbrechungen voll funktionsfähig.

### So verwenden Sie es

1. **Initiale Wissensbasis aufbauen: Asynchrone Bulk-Importe**  
   Führen Sie `brainbridge import --format=markdown --dir=./docs` aus, um alle Markdown-Dateien im angegebenen Verzeichnis zu parsen. Das Tool extrahiert automatisch Überschriften, Code-Blöcke und Schlüsselbegriffe. Verwenden Sie `--depth=3`, um die Granularität der Cluster-Bildung zu steuern. Nach dem Import öffnen Sie das Web-Interface unter `localhost:8080` und überprüfen die automatisch generierte Graph-Ansicht.

2. **Gedanken manuell verknüpfen und gewichten**  
   Klicken Sie im Graph-Interface auf einen Knoten, wählen Sie „Neue Verknüpfung“ und ziehen Sie eine Linie zu einem anderen Knoten. Geben Sie im erscheinenden Dialog einen Kontext (z. B. „Diese Idee ist eine Abhängigkeit von jener“) und einen Gewichtungsfaktor (1–10) an. Der Faktor beeinflusst die visuelle Dicke der Kante und die Sortierung in Suchergebnissen. Bestätigen Sie mit `Strg+Enter`.

3. **Kollaboration aktivieren: Peer-to-Peer-Session starten**  
   Starten Sie auf einem Knoten `brainbridge session --create --name=team-alpha`. Auf den anderen Knoten führen Sie `brainbridge session --join --id=<session-id>` aus. brainbridge generiert automatisch einen QR-Code, der die Verbindungsdaten enthält. Nach erfolgreicher Verbindung sehen Sie farblich markierte Knoten – jeder Benutzer hat eine eigene Farbe. Änderungen werden in Echtzeit gemerged, Konflikte erscheinen als gelbe Knoten mit Auflösungsvorschlägen.

### Beste Praktiken

- **Architektur-Entscheidungsprotokolle (ADRs) mit brainbridge verknüpfen**  
  Nutzen Sie brainbridge als lebendiges ADR-Repository. Verknüpfen Sie jede Entscheidung mit den zugrunde liegenden Gedanken, Anforderungen und technischen Constraints. Verwenden Sie die Zeitachsen-Ansicht, um zu sehen, wie sich eine Entscheidung im Projektverlauf entwickelt hat. Dies reduziert technische Schulden und erleichtert Onboarding neuer Teammitglieder erheblich.

- **Tägliche 15-minütige „Gedanken-Sync“-Routine**  
  Richten Sie ein tägliches Meeting ein, bei dem jedes Teammitglied 2–3 neue Gedanken oder Verknüpfungen in brainbridge einpflegt. Nutzen Sie die Diff-Ansicht, um die Entwicklung der letzten 24 Stunden zu besprechen. Dies fördert eine Kultur der kontinuierlichen Wissensdokumentation und verhindert, dass implizites Wissen verloren geht. Verwenden Sie den Befehl `brainbridge stats --since=24h`, um eine Zusammenfassung der Aktivitäten zu generieren.
