# brainbridge - BRA_003

## Übersicht

brainbridge ist kein weiteres Mindmapping-Tool. Es ist eine **asynchrone Gedankenverknüpfungs-Engine**, die lose Ideenfragmente in strukturierte, abfragbare Wissensgraphen transformiert. Entwickelt für Developer, die zwischen Chaos und Klarheit navigieren – ohne störende GUI-Overlays.

## Kernfunktionen (Hardcore)

### 1. **Semantische Pfropfung (Semantic Grafting)**
Statt linearer Notizen erlaubt brainbridge das **bidirektionale Verknüpfen** von Gedankenschnipseln über kontextuelle Embeddings. Ein Code-Snippet aus einem Repo kann automatisch mit einer Design-Notiz aus einem anderen Branch fusioniert werden – ohne manuelle Tags.

### 2. **Kognitive Latenzkompensation (Cognitive Latency Compensation)**
Das Tool erkennt **tote Denkpfade** (wiederholte, unproduktive Schleifen) und schlägt automatisch alternative Verknüpfungen vor. Es analysiert die zeitliche Abfolge deiner Eingaben und optimiert den Pfad zur Lösung – wie ein Compiler für Gedanken.

### 3. **Headless API & CLI-First Architektur**
Kein Dashboard-Zwang. brainbridge läuft als **lokaler Daemon** mit REST- und gRPC-Endpunkten. Du kannst Gedanken via `curl` injizieren, via `stdin` pipen oder direkt aus deinem Editor (Neovim, VSCode) triggern. Alles ist logbar, versionierbar und in CI/CD-Pipelines integrierbar.

## Detaillierte Anleitung (Professionell)

### 1. **Initialisierung eines Gedanken-Clusters**
```bash
brainbridge init --workspace "sprint-42" --mode headless
```
Erzeugt ein `.brainbridge/`-Verzeichnis mit einer SQLite-Datenbank und einer `config.yaml`. Alle späteren Einträge werden automatisch in einen **semantischen Vektorraum** eingebettet. Optional: `--embedding-model all-MiniLM-L6-v2` für lokale Inferenz.

### 2. **Injektion asynchroner Gedankenfragmente**
```bash
echo "Kafka Consumer hängt bei Offset 0x3F2 – vermutlich Partition-Rebalance" | brainbridge inject --tag "bug, kafka, debugging"
```
Das Fragment wird nicht nur gespeichert, sondern mit **ähnlichen Einträgen** verknüpft (via Cosine Similarity). Du kannst später via `brainbridge query "Kafka Timeout"` alle verwandten Einträge abrufen – selbst wenn die Keywords nicht exakt übereinstimmen.

### 3. **Export als GraphML oder Mermaid**
```bash
brainbridge export --format mermaid --depth 3 --filter "tag:bug"
```
Generiert ein Mermaid-Diagramm, das die **Top-3-Verknüpfungsebenen** eines Fehlers darstellt. Perfekt für Code-Reviews oder Post-Mortems. Der Export ist deterministisch – gleicher Input erzeugt gleiches Diagramm (ideal für CI).

## Best Practices für Developer (Deutschsprachiger Kontext)

### 1. **Tägliches "Brain Dump"-Ritual**
Nutze brainbridge als **Ersatz für chaotische README-Notizen**. Jeden Abend: `cat ~/notes/today.md | brainbridge inject --workspace "daily-log"`. Am nächsten Morgen: `brainbridge query "gestern"`. So entsteht ein durchsuchbares, verknüpftes Gedächtnis – ohne dass du jemals einen Ordner öffnen musst.

### 2. **Integration in CI/CD für Architektur-Reviews**
Hinterlege Architekturentscheidungen als brainbridge-Einträge mit Tag `adr`. Ein GitHub Action checkt vor jedem Merge, ob der neue Code mit bestehenden ADRs kollidiert. Beispiel: `brainbridge query --tag "adr" --filter "kafka"` warnt, wenn ein neuer Service ohne Event-Bus eingeführt wird.
