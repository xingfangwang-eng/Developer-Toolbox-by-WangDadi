# brainbridge - BRA_004

## Übersicht

brainbridge ist kein weiteres „Thought connection software“-Gimmick. Es ist eine **syntaktische und semantische Brücke** zwischen deinem Gehirn und deiner Codebasis. Statt Notizen in Obsidian oder Markdown zu verlieren, erzeugt brainbridge **ausführbare Gedankengraphen** – direkt aus deiner IDE heraus. Zielgruppe: Entwickler, die mit komplexen Domänenmodellen kämpfen und deren mentale Modelle in Code konservieren wollen.

---

## 3 Hardcore-Funktionen, die den Unterschied machen

### 1. **Semantischer Gedanken-Compiler (SGC)**
brainbridge übersetzt deine Rohtexte (z. B. aus Diktiergerät, Notizen, Slack-Konversationen) in **typsichere ASTs (Abstract Syntax Trees)**. Diese können direkt in TypeScript-Interfaces, Rust-Structs oder Python-Dataclasses exportiert werden. Kein manuelles Mapping mehr zwischen „Idee“ und „Implementierung“.

### 2. **Kontextuelles Gedächtnis-Merging (CMM)**
Statt linearer Changelogs oder loser Markdown-Dateien führt brainbridge **automatische Diffs zwischen deinen Gedankenmustern** durch. Änderst du ein Konzept in deinem Kopf (z. B. „User“ → „Account“), erkennt das Tool semantische Verschiebungen und schlägt Refactorings in deinem Code vor – inklusive Impact-Analyse auf Abhängigkeiten.

### 3. **Interaktive Gedanken-Netzwerk-Visualisierung (IGNV)**
Du siehst deine Ideen als **gerichteten Graphen mit Kantengewichten** (z. B. „starke Kopplung“, „schwache Kohäsion“). Klickst du auf einen Knoten, öffnet sich der zugehörige Codeblock oder die API-Dokumentation. Perfekt für Architektur-Reviews oder Onboarding neuer Teammitglieder.

## 3 Schritte zur produktiven Nutzung

### Schritt 1: **Initialisierung des Gedanken-Index**
```bash
brainbridge init --scope=./src --language=typescript --depth=recursive
```
Das Tool scannt deine gesamte Codebasis, extrahiert **semantische Entitäten** (Klassen, Interfaces, Funktionen) und baut einen lokalen Vektor-Index. Danach startest du den Daemon:
```bash
brainbridge daemon --port=5173 --watch
```
Jetzt läuft ein WebSocket-Server, der auf deine Gedankeneingaben wartet.

### Schritt 2: **Gedanken als Code-Generatoren nutzen**
Öffne dein Terminal oder die integrierte CLI und gib ein:
```
brainbridge think "Wir brauchen eine Factory für Event-Streams mit Retry-Logik und Circuit-Breaker"
```
brainbridge generiert daraus:
- Ein TypeScript-Interface `EventStreamFactory`
- Eine Basisimplementierung mit `retry()` und `circuitBreaker()`
- Automatische Dependency-Injection-Vorlage für NestJS oder Angular

Du kannst das Ergebnis direkt in deine Codebasis mergen:
```bash
brainbridge apply --id=event-stream-factory --target=./src/factories
```

### Schritt 3: **Gedanken-Diffs und Refactoring-Vorschläge**
Änderst du später deine Meinung (z. B. „Event-Streams sollen asynchron sein“), rufst du auf:
```
brainbridge diff --id=event-stream-factory --new-thought="asynchrone Event-Streams mit Backpressure"
```
Das Tool zeigt dir:
- Welche Dateien betroffen sind
- Welche Interfaces geändert werden müssen
- Einen automatischen Merge-Vorschlag inklusive Test-Update

Du bestätigst mit `brainbridge apply --diff`.

## 2 Best Practices für Profis

### 1. **Gedanken als Architektur-Dokumentation (ADR 2.0)**
Statt statischer ADRs (Architecture Decision Records) verwendest du brainbridge als **lebende Dokumentation**. Jeder Gedanke (z. B. „Warum haben wir uns für GraphQL statt REST entschieden?“) wird als semantischer Knoten gespeichert und mit dem relevanten Code verlinkt. Neue Teammitglieder können per `brainbridge explore --domain=graphql` die gesamte Entscheidungskette nachvollziehen – inklusive Code-Referenzen und Timestamps.

### 2. **Pair-Programming mit Gedanken-Streaming**
Nutze brainbridge im **Multi-User-Modus**: Zwei Entwickler können gleichzeitig Gedanken in denselben Graphen einspeisen. Das Tool führt automatisch Konfliktauflösung durch (z. B. „Beide haben das Interface `UserService` definiert – merge mit Präfix `v1_` und `v2_`“). Perfekt für Remote-Teams, die asynchron an derselben Architektur arbeiten.
