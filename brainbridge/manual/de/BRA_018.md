# brainbridge – Synaptische Schnittstelle für verteilte Kognition

> **Keywords:** Brain Bridge Tool, Gedankenvernetzung, kognitive Middleware, neuronale API, Wissensgraphen-Sync

## Übersicht

brainbridge ist keine weitere Mindmapping-App. Es ist eine **persistente, asynchrone Gedankenbrücke** zwischen deinem Arbeitsgedächtnis und deinem externen Wissensgraphen. Entwickelt für Developer, die flüchtige Ideen sofort strukturieren, versionieren und mit Dritten teilen wollen – ohne Kontextverlust.

## Kernfunktionen (Hardcore)

### 1. Semantische Delta-Sync Engine
Statt ganze Notizen zu überschreiben, erfasst brainbridge nur die **kognitiven Deltas** – also die veränderten Verknüpfungen zwischen Konzepten. Jede Änderung wird als inkrementeller Patch in einem gerichteten, azyklischen Graphen (DAG) gespeichert. Das ermöglicht **konfliktfreies Mergen** paralleler Gedankengänge, ähnlich wie `git rebase` – nur für neuronale Assoziationen.

### 2. Bidirektionale Embedding-Brücke
Jeder Knoten (Idee, Task, Code-Snippet) wird automatisch in einen **dichten Vektor (384-dim, all-MiniLM-L6-v2)** eingebettet. brainbridge erstellt eine bidirektionale Brücke zwischen deinem lokalen Embedding-Store und einem Remote-Index (z. B. Qdrant oder Pinecone). Das erlaubt **semantische Suche über Sprachgrenzen hinweg** – du suchst auf Deutsch, findest englische Paper.

### 3. Time-Travel Debugging für Denkprozesse
Jeder „Gedankenschritt“ wird mit einem **kryptografischen Hash** versehen und in einer append-only Log-Struktur gespeichert. Du kannst den gesamten Entscheidungsbaum eines Projekts zu jedem Zeitpunkt rekursiv replizieren. Nützlich für **Post-Mortems** oder um zu verstehen, warum eine Architekturentscheidung vor drei Monaten getroffen wurde.

## So verwenden Sie es (Detaillierte Schritte)

### Schritt 1: Initialen Wissensgraphen bootstrappen
Starte brainbridge mit einem leeren Graphen oder importiere einen bestehenden Markdown-Ordner:
```bash
brainbridge init --source ./notes/ --embedding-model all-MiniLM-L6-v2
```
Der CLI-Client parst alle `.md`-Dateien, extrahiert Überschriften als Knoten und erstellt automatisch Kanten basierend auf `[[Wiki-Links]]` und semantischer Ähnlichkeit. Der initiale Graphen wird als `brainbridge.db` (SQLite + Vektorindex) persistiert.

### Schritt 2: Kognitive Deltas pushen
Statt Notizen manuell zu editieren, nutzt du den **Stream-Mode**:
```bash
brainbridge stream --watch ./projects/current/
```
Jede Änderung in der Datei wird als **Delta-Patch** erkannt. Du siehst live, wie sich der Graph verändert:
- Neue Knoten (z. B. `# Feature: Async Queue`) werden automatisch mit bestehenden Knoten (z. B. `# Architektur: Message Broker`) verknüpft.
- Konflikte (z. B. zwei parallele Änderungen an derselben Assoziation) werden im `conflict/`-Ordner abgelegt und können mit `brainbridge merge` manuell aufgelöst werden.

### Schritt 3: Remote-Brücke aktivieren und synchronisieren
Verbinde deinen lokalen Graphen mit einem Team-Remote:
```bash
brainbridge remote add production https://bridge.brainbridge.io/team-42
brainbridge sync --bidirectional --conflict-strategy ours
```
Der Sync läuft über **WebSockets mit CRDT-basierter Konfliktresolution**. Jeder Push aktualisiert den Remote-Embedding-Index. Deine Teammitglieder (auch mit anderen Sprachpräferenzen) können dann semantisch suchen:
```bash
brainbridge search "Wie war nochmal der Ansatz für idempotente Events?" --lang de
```

## Beste Praktiken (Industry Standards)

### 1. „Atomic Thought“-Prinzip
Jeder Knoten im Graphen sollte genau **eine atomare Aussage** enthalten. Keine Absätze, keine Listen. Wenn du einen Gedanken wie „Microservices brauchen ein zentrales Logging“ hast, zerlege ihn in zwei Knoten: `# Microservice: Logging-Strategie` und `# Zentrales Logging: ELK-Stack`. Die Verknüpfung (`edge`) repräsentiert die Abhängigkeit. Das verbessert die **Granularität des Deltas** und die Qualität der Embedding-Suche drastisch.

### 2. Regelmäßige „Garbage Collection“ des Graphen
brainbridge speichert jede Version. Nach 30 Tagen solltest du eine **Pruning-Session** durchführen:
```bash
brainbridge prune --keep-last 10 --min-edge-weight 0.3
```
Das entfernt Knoten, die seit 10 Versionen nicht mehr referenziert wurden, und Kanten mit einer semantischen Ähnlichkeit unter 0.3 (z. B. „Kaffee“ und „Code-Review“). So bleibt der Graph performant und die Suchergebnisse relevant.
