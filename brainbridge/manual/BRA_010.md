# brainbridge - BRA_010

## Overview

brainbridge is a high-performance cognitive orchestration layer designed for developers who need to map, synchronize, and augment distributed thought workflows across heterogeneous systems. It bridges the gap between raw neural signals, semantic memory graphs, and real-time collaborative reasoning engines. This is not a note-taking app. This is a **thought connectivity fabric** for engineers building next-gen AI-assisted ideation pipelines.

## What is brainbridge?

brainbridge is a modular, API-first framework that enables bidirectional synchronization between internal mental models and external knowledge graphs. It leverages vectorized thought embeddings, temporal reasoning windows, and context-aware routing to turn fragmented ideas into executable system architectures. Built for developers working on semantic search, multi-agent reasoning, or brain-computer interface prototypes.

## Core Features

### 1. Semantic Thought Anchoring Engine
Automatically resolves ambiguous mental fragments into structured, queryable nodes using a hybrid retrieval-augmented generation (RAG) pipeline. Each "thought" is embedded with temporal decay weights and cross-referenced against your private knowledge base. No more context drift.

### 2. Multi-Agent Thought Routing
Deploy parallel reasoning agents that compete or collaborate on a single thought graph. brainbridge uses a priority-based scheduler to route sub-thoughts to the most relevant agent (e.g., one for logic validation, another for creative divergence). Results are merged via a conflict-resolution layer.

### 3. Real-Time Neural Sync Protocol (NSP)
A low-latency WebSocket-based protocol that streams EEG-derived thought markers (alpha/beta band power, P300 spikes) into your pipeline. Integrates with OpenBCI, Muse, or custom hardware. Enables hands-free state transitions and context-aware triggers.

## How to Use

### 1. Initialize a Thought Graph with Custom Embeddings
```bash
brainbridge init --name "project_alpha" --embedder all-MiniLM-L6-v2
```
This creates a local vector store with a configurable embedding model. Use `--dim 768` for higher precision or `--quantize` for edge deployment. The graph is stored in a HNSW index for sub-10ms retrieval.

### 2. Route a Thought to a Reasoning Agent
```python
from brainbridge import ThoughtRouter

router = ThoughtRouter(agent_pool=["logician", "divergent", "critic"])
result = router.route(
    thought="How to optimize latency in a federated learning loop?",
    strategy="parallel_consensus"
)
```
Each agent returns a scored response. The router applies a weighted voting mechanism (configurable via `--consensus_threshold 0.7`). Output includes a confidence score and a trace of agent interactions.

### 3. Stream EEG Markers into the Pipeline
```bash
brainbridge stream --device openbci --channel 4 --trigger_threshold 0.8
```
This launches a real-time listener that maps P300 events to thought anchors. When a spike is detected, brainbridge automatically creates a new node with a timestamp and a snapshot of the current working memory. Use `--output json` for integration with dashboards or CI/CD pipelines.

## Best Practices

### 1. Use Temporal Decay for Stale Thoughts
Set `--decay_rate 0.05` to automatically deprioritize thoughts older than 24 hours. This prevents graph bloat and keeps retrieval relevant. Combine with `--prune_interval 3600` to run background cleanup every hour. For long-running projects, archive decayed nodes to a cold store (S3 or local Parquet).

### 2. Pin Critical Reasoning Agents for High-Stakes Tasks
When running multi-agent routing, pin the "logician" and "critic" agents to the same thread using `--pin_agent logician,critic`. This reduces latency by 40% and ensures consistency in validation-heavy workflows. Use `--fallback_agent baseline` to handle agent failures gracefully.
