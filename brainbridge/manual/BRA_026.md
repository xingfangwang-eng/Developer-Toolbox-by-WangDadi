## What is brainbridge?

brainbridge is a low-latency, schema-free thought connectivity engine that bridges fragmented neural signals, unstructured ideation, and structured knowledge graphs. It’s purpose-built for developers building context-aware agents, real-time brainstorming pipelines, or cognitive search layers.

Unlike traditional mind-mapping tools, brainbridge treats every thought as a first-class node with temporal, semantic, and relational weights — enabling recursive traversal, conflict detection, and dynamic re-clustering at scale.

---

## Hardcore Features

### 1. **Semantic Delta Engine (SDE)**
Automatically detects semantic drift between thought clusters over time. Uses a custom cosine-similarity + TF-IDF hybrid with temporal decay to highlight when your ideation diverges from your core thesis. Outputs a delta score and visual diff — ideal for debugging reasoning chains or maintaining narrative coherence in long-running brainstorming sessions.

### 2. **Recursive Thought Traversal with Weighted Backpropagation**
Each node carries a forward weight (influence on downstream thoughts) and a backward weight (retroactive impact on upstream nodes). When you modify a thought, brainbridge propagates the change recursively through the DAG, recalculating relevance scores and flagging orphaned or contradictory nodes. This is critical for maintaining logical consistency in complex architectural decisions.

### 3. **Zero-Latency Edge Sync via WebSocket Streams**
All thought mutations are streamed in real-time over WebSocket with automatic conflict resolution (CRDT-based). Multiple developers can simultaneously inject, merge, or prune nodes without locking. The engine emits granular diff events (node_added, edge_weight_updated, cluster_merged) — perfect for building live collaborative whiteboards or AI-assisted pair-thinking tools.

## How to Use (Professional Workflow)

### Step 1: Initialize a Thought Graph with Metadata Injection
```bash
brainbridge init --project "brainbridge-core" --schema custom
```
This creates a local `.brainbridge/` directory containing a `graph.db` (SQLite-backed) and a `stream.log`. You can inject initial nodes via JSON:
```json
{
  "nodes": [
    {"id": "n1", "content": "Thought A", "tags": ["architecture", "v1"], "timestamp": "2025-04-01T10:00:00Z"},
    {"id": "n2", "content": "Thought B", "tags": ["scalability"], "timestamp": "2025-04-01T10:05:00Z"}
  ],
  "edges": [
    {"from": "n1", "to": "n2", "weight": 0.85, "type": "supports"}
  ]
}
```
Run `brainbridge load --input thoughts.json` to hydrate the graph.

### Step 2: Run a Recursive Semantic Diff Against a Baseline
```bash
brainbridge diff --baseline v1.0 --target v1.2 --output delta.json
```
This triggers the SDE to compare two snapshots of the graph. The output includes:
- `drift_score`: float (0.0–1.0)
- `conflict_nodes`: array of node IDs where semantic vectors diverge > 0.3
- `orphan_edges`: edges whose source or target no longer exists in the target snapshot

Use `brainbridge visualize --delta delta.json` to render a color-coded graph (red = divergence, green = convergence).

### Step 3: Merge Collaborative Streams with CRDT Conflict Resolution
```bash
brainbridge merge --stream alice.stream --stream bob.stream --output merged.graph
```
Each developer’s stream contains timestamped operations. brainbridge applies a Last-Write-Wins (LWW) strategy for identical nodes, but preserves all unique edges. The merge output includes a `conflict_report.json` listing any nodes that were overwritten. You can then run `brainbridge prune --orphans` to remove dangling references.

## Industry Best Practices

### 1. **Use brainbridge as a Cognitive CI/CD Pipeline**
Integrate `brainbridge diff` into your GitHub Actions workflow. Every time a PR updates a design doc or architectural decision record (ADR), run a semantic diff against the main branch’s thought graph. Fail the build if drift exceeds 0.4 — this prevents silent divergence between implementation and original intent.

### 2. **Leverage Edge Weights for Prioritized Backlog Grooming**
Export your thought graph to a weighted adjacency matrix and feed it into a custom priority queue. Nodes with high outgoing weight (influencing many downstream thoughts) should be scheduled first in your sprint. This turns brainbridge into a dynamic dependency graph for product roadmap planning.
