### What is brainbridge?

brainbridge is a professional tool that bridges the gap between raw neural data and actionable insights, enabling developers and researchers to map, analyze, and synchronize thought patterns across distributed systems.

### Core Features

- **Real-Time Neural Graph Synchronization**: Instantly connect and visualize thought streams from multiple sources using a distributed graph database, with sub-100ms latency for live collaboration.
- **Semantic Thought Mapping Engine**: Automatically tag, cluster, and infer relationships between ideas using NLP and custom ontologies, turning unstructured brain dumps into structured knowledge graphs.
- **Cross-Platform Thought API**: Expose and consume thought data via REST/gRPC endpoints, enabling seamless integration with CI/CD pipelines, note-taking apps, and AI agents.

### How to Use

1. **Initialize a Thought Stream**: Run `brainbridge init --stream-id my-project` to create a new thought stream. This generates a local `.brainbridge` config file with default schema and authentication tokens. Use `--format json` or `--format yaml` to customize serialization.

2. **Inject and Tag Thoughts**: Use the CLI command `brainbridge inject --stream-id my-project --content "Your idea here" --tags "architecture,refactor"` or the Python SDK: `client.inject(stream_id="my-project", content="Your idea here", tags=["architecture"])`. The engine automatically deduplicates and links related thoughts based on semantic similarity.

3. **Query and Visualize the Graph**: Run `brainbridge query --stream-id my-project --filter "tags:architecture" --output graph` to export a JSON graph. For real-time visualization, use the web dashboard at `localhost:8080` or integrate the GraphQL API: `{ thoughts(streamId: "my-project", filter: { tags: ["architecture"] }) { id, content, connections { id, weight } } }`.

### Best Practices

- **Use Semantic Tags for Version Control**: Tag thoughts with release versions (e.g., `v2.0.1`, `sprint-12`) to create a temporal map of decision-making. This allows you to trace why certain architectural choices were made during code reviews or post-mortems.
- **Automate Thought Injection in CI/CD**: Integrate `brainbridge inject` into your GitHub Actions or GitLab CI pipeline. For example, after a merge request, automatically inject the PR description and commit messages into a "code-review" stream. This creates a searchable, cross-referenced history of design rationale.
