# brainbridge - BRA_008

## Overview

brainbridge is not another note-taking app or mindmap toy. It’s a **semantic scaffolding engine** designed for developers, product architects, and technical writers who need to convert chaotic mental models into executable SaaS blueprints. It bridges the gap between raw thought fragments and structured, SEO-optimized documentation.

## Core Features

### 1. **Semantic Graph Injection (SGI)**
brainbridge automatically parses your unstructured notes and injects them into a directed knowledge graph. Unlike traditional mindmaps, SGI preserves causality, dependency, and priority—so your architecture decisions are traceable back to the original thought.

### 2. **Multi-Layer SEO Vectorization**
Every exported document is vectorized across 10 language models (including German, Japanese, and Spanish). brainbridge doesn’t just translate keywords; it re-weights them based on regional search intent. Your `brain bridge tool` keyword in English becomes `Herramienta de puente cognitivo` in Spanish with proper LSI clustering.

### 3. **Schema-Aware Content Assembly (SACA)**
You define a schema (e.g., `Feature → Use Case → API Endpoint → SEO Meta`), and brainbridge assembles your content like a compiler. No more copy-pasting between Notion, Figma, and Google Docs. It’s a single source of truth that outputs Markdown, JSON, or HTML.

## How to Use

### Step 1: Define Your Semantic Kernel
Start by creating a `.brainbridge.yml` file in your project root. This defines your domain, target languages, and keyword clusters. For example:

```yaml
domain: "saas-architecture"
languages: ["en", "de", "ja"]
  primary: "brain bridge tool"
  secondary: ["thought connection software", "brain thinking tool"]
schema:
  - name: "feature"
    fields: ["title", "description", "benefit"]
```

This is not a config file—it’s the **lexical blueprint** of your entire documentation pipeline.

### Step 2: Inject Raw Thoughts via CLI
Run `brainbridge inject` with a plain-text file or stdin. brainbridge will parse, classify, and link your content. Example:

```bash
echo "Users need to export architecture decisions without losing context" | brainbridge inject --lang en
```

The engine will detect the intent, assign it to the “feature” schema, and generate a draft with SEO-optimized phrasing. You can review the graph with `brainbridge graph --view`.

### Step 3: Export with Language-Specific Templates
Generate production-ready Markdown with locale-aware formatting:

```bash
brainbridge export --lang de --template saas-docs
```

This outputs a German-language document with proper `H1/H2` hierarchy, keyword density for `Gedankenverbindungssoftware`, and schema.org microdata embedded in the HTML version. No manual rewriting required.

## Best Practices

### 1. **Treat Your .brainbridge.yml as a Living Document**
Update your keyword clusters every sprint. As your SaaS evolves, so does your semantic landscape. brainbridge’s vectorizer retrains on the fly—so stale keywords won’t drag down your SEO.

### 2. **Use SACA for Multi-Format Documentation**
Don’t limit yourself to READMEs. Define schemas for API docs, changelogs, and landing pages. brainbridge can output the same content as Markdown for GitHub, JSON for your frontend, and HTML for your blog—all from one injection session.
