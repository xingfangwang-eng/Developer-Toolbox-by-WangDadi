### What is brainbridge?

brainbridge is a professional tool that bridges the gap between raw thought and structured execution. It enables developers and architects to map, connect, and synchronize complex mental models into actionable workflows, reducing cognitive load and accelerating decision-making.

### Core Features

1. **Semantic Thought Graph**: Dynamically visualize and link abstract ideas, dependencies, and constraints in a real-time graph database, enabling non-linear exploration of problem spaces.
2. **Context-Aware Snippet Engine**: Automatically capture and index code, documentation, and mental notes with contextual metadata, allowing instant retrieval via natural language queries.
3. **Cross-Platform Sync Bridge**: Seamlessly integrate with IDEs, project management tools, and version control systems to push thought fragments directly into tickets, commits, or design docs.

### How to Use

1. **Initialize a Thought Session**: Run `brainbridge init` in your terminal to create a local workspace. Use the `--mode deep` flag to enable full semantic indexing of your current project directory. This generates a `.brainbridge/` folder with a JSON-LD schema for your thought graph.
2. **Capture and Connect Fragments**: Use `brainbridge capture "Refactor auth middleware to use JWT rotation"` to log a thought. Then, link it to existing nodes via `brainbridge link <node_id> --to <target_id> --relation "depends_on"`. The tool auto-suggests relations based on keyword overlap.
3. **Export to Workflow**: Run `brainbridge export --format jira` to push your graph as structured tickets. Use `--template agile-sprint` to map nodes to epics, stories, and tasks. The tool also supports `--format github-issue` for direct GitHub integration.

### Best Practices

- **Adopt a Daily Thought Dump**: Spend 5 minutes at the end of each day running `brainbridge capture` with a `--tag daily-log` to offload mental clutter. This prevents context-switching overhead and improves sprint planning accuracy.
- **Use Thought Graphs for Retrospectives**: After a release, run `brainbridge export --format mermaid` to generate a dependency diagram of your thought nodes. Compare it against actual commit history to identify gaps in communication or hidden blockers.
