# brainbridge - BRA_011

## Overview

brainbridge is a high-performance neural signal mapping and cross-referencing engine engineered for real-time cognitive state inference. It bridges fragmented neural activity data streams into coherent, queryable thought topologies. Optimized for low-latency EEG/fMRI integration, the tool operates at sub‑10ms processing windows and supports concurrent multi‑subject session merging.

## Core Features

### 1. Real‑Time Neural Graph Synchronization (NGS‑v3)
- **Function:** Maps incoming raw EEG voltage differentials (μV) and fMRI BOLD signal deltas into a unified directed acyclic graph (DAG) where each node represents a discrete cognitive micro‑state.
- **Parameters:** `--sync-interval 5ms`, `--node-capacity 10,000`, `--edge-weight-threshold 0.78`.
- **Technical detail:** Uses a lock‑free concurrent hash map (backed by Redis Streams) to achieve <3ms latency per 1,024‑sample window. Automatically prunes stale nodes older than 2.5 seconds to maintain memory footprint under 512MB.

### 2. Multi‑Modal Thought Vector Alignment (MTVA)
- **Function:** Aligns semantic embeddings from natural language prompts with spatiotemporal neural signatures, producing a normalized “thought vector” (512‑dim float32) that can be compared across sessions or subjects.
- **Parameters:** `--embedding-model all-MiniLM-L6-v2`, `--alignment-threshold 0.92 cosine`, `--output-format parquet`.
- **Technical detail:** Integrates with PostgreSQL via `pgvector` for persistent storage and ANN (Approximate Nearest Neighbor) queries. Supports batch alignment of up to 250,000 vectors per minute on a single NVIDIA A100.

### 3. Causal Inference Pipeline (CIP‑2)
- **Function:** Infers causal relationships between observed neural events and external stimuli (e.g., visual cues, auditory triggers) using a modified Peter‑Clark (PC) algorithm with conditional independence tests.
- **Parameters:** `--alpha 0.05`, `--max-condition-set-size 4`, `--criterion bayesian-information-criterion`.
- **Technical detail:** Each inference run produces a directed acyclic causal graph with confidence intervals (95% CI) per edge. Outputs are serialized as JSON‑LD for downstream graph databases (Neo4j supported). Processing time averages 45ms per 1,000 event pairs.

## How to Use

### Step 1: Initialize the Neural Stream Connector
```bash
brainbridge init --source eeg:openbci:32ch --source fmri:niifti:3T \
  --sync-interval 5ms --redis-host localhost:6379
```
This command launches the data ingestion daemon, binds to the OpenBCI Cyton board (32 channels at 250Hz) and a NIfTI‑formatted fMRI stream. The daemon prints a session UUID (e.g., `sess_a1b2c3d4`) and begins buffering raw data into the shared Redis Stream `brainbridge:raw:stream`.

### Step 2: Execute a Thought Vector Alignment
```bash
brainbridge align --session sess_a1b2c3d4 --prompt "visualize rotating cube" \
  --embedding-model all-MiniLM-L6-v2 --output /data/thought_vectors.parquet
```
The tool extracts the 2.5‑second window following the prompt timestamp, computes the MTVA embedding, and appends the vector to the output Parquet file. Use `--dry-run` to preview alignment quality (cosine similarity score) without writing.

### Step 3: Run Causal Inference on Aligned Data
```bash
brainbridge infer --input /data/thought_vectors.parquet --alpha 0.05 \
  --max-condition-set-size 4 --output /data/causal_graph.jsonld
```
The CIP‑2 pipeline reads the aligned vectors, performs conditional independence tests, and writes the resulting causal graph. Validate the graph with:
```bash
brainbridge validate --graph /data/causal_graph.jsonld --metric shd
```
This returns the Structural Hamming Distance (SHD) against a ground‑truth graph if provided; otherwise, it reports the number of edges with confidence > 0.95.

## Best Practices

### Practice 1: Pre‑Filter Neural Noise with Bandpass Envelopes
Before feeding raw data into the NGS‑v3 pipeline, apply a 0.5–40 Hz bandpass filter to the EEG stream and a 0.01–0.1 Hz high‑pass filter to fMRI BOLD signals. This reduces false positive edges in the causal graph by up to 34% (measured on the HCP‑MMP1.0 dataset). Use `brainbridge preprocess --filter bandpass:0.5-40` to automate this.

### Practice 2: Version‑Lock Embedding Models for Reproducibility
Always pin the exact embedding model version (e.g., `all-MiniLM-L6-v2` with commit hash `a1b2c3d`) in your CI/CD pipeline. A 0.01 shift in cosine similarity thresholds can alter causal inference results by 12–18% in multi‑subject studies. Store model metadata alongside each Parquet file using the `--model-metadata` flag.
