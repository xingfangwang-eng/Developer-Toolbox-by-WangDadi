## Resumen

brainbridge es una herramienta SaaS de arquitectura cognitiva que conecta fragmentos de pensamiento dispersos en flujos de trabajo estructurados. ## Contenido Detallado

### ¿Qué es brainbridge?

brainbridge no es un simple gestor de notas ni un tablero Kanban. Es un **orquestador de pensamiento distribuido** que resuelve el problema de la fragmentación cognitiva en entornos de alta complejidad. Integra APIs, bases de conocimiento y repositorios de código en un solo grafo semántico, permitiendo a los equipos navegar entre decisiones técnicas, requisitos de negocio y artefactos de desarrollo sin perder el contexto.

### Características Principales

- **Grafo de Dependencias Cognitivas**: Cada idea, fragmento de código o decisión de arquitectura se representa como un nodo con relaciones ponderadas. brainbridge analiza automáticamente la fuerza de conexión entre nodos (por ejemplo, `0.85` entre una función en Go y su caso de prueba) y sugiere rutas de navegación óptimas para resolver bugs o refactorizar módulos.

- **Motor de Inferencia Semántica en Tiempo Real**: Utiliza embeddings de lenguaje natural combinados con metadatos de commits de Git para predecir qué artefactos se verán afectados por un cambio propuesto. Esto permite a los desarrolladores validar hipótesis de diseño antes de escribir una sola línea de código.

- **Sincronización Bidireccional con CI/CD**: brainbridge se integra directamente con pipelines de CI/CD (GitHub Actions, GitLab CI) para generar automáticamente un *mapa de impacto* cada vez que se fusiona un PR. Esto evita regresiones silenciosas y acelera las revisiones de código.

### Cómo Usar

1. **Inicialización del Grafo desde un Repositorio Existente**  
   Ejecuta el comando `brainbridge init --repo git@github.com:tu-equipo/proyecto.git --depth full`. Esto escaneará el historial de commits, los comentarios en el código y los tickets de Jira asociados para construir el grafo inicial. brainbridge asignará automáticamente un `ID de nodo` único a cada entidad (por ejemplo, `NODE-42` para una función específica). Luego, usa `brainbridge graph --visualize` para generar un archivo DOT que puedas abrir en Graphviz o en la interfaz web nativa.

2. **Vinculación de Pensamientos Ad Hoc**  
   Durante una sesión de debugging, escribe `brainbridge link --from "NODE-42" --to "IDEA-03" --weight 0.9 --reason "El fallo en la función X se debe a la decisión de usar un pool de conexiones sin timeout"`. Esto crea una arista ponderada que será visible en el grafo global. Puedes verificar la consistencia del grafo con `brainbridge check --consistency`, que detectará ciclos no deseados o relaciones contradictorias.

3. **Exportación de un Mapa de Impacto para un PR**  
   Antes de abrir un pull request, ejecuta `brainbridge impact --branch feature/nueva-api --output pr-report.md`. brainbridge generará un informe Markdown que lista todos los nodos afectados, junto con un nivel de riesgo calculado (bajo, medio, alto) basado en la centralidad del nodo en el grafo. Incluye este reporte directamente en el cuerpo del PR para que los revisores tengan contexto inmediato.

### Mejores Prácticas

- **Establece un peso mínimo por defecto para conexiones triviales**: Configura `min-edge-weight: 0.3` en `brainbridge.yml` para evitar que el grafo se llene de ruido (por ejemplo, relaciones entre variables locales sin impacto arquitectónico). Esto mantiene el grafo manejable y las rutas de navegación relevantes.

- **Programa un barrido semanal de nodos huérfanos**: Usa `brainbridge prune --orphan-days 30` para eliminar nodos sin conexiones activas (por ejemplo, notas de reuniones antiguas o fragmentos de código refactorizados). Esto reduce la latencia en las consultas al grafo y mejora la precisión del motor de inferencia.
