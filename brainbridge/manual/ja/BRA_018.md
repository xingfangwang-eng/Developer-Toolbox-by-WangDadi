# brainbridge - BRA_018

## 概要

brainbridge は、思考の断片を構造化データとして接続・可視化する次世代型マインドマッピングエンジンです。従来のノートツールやマインドマップツールとは異なり、**ニューラルリンクモデル**を採用し、アイデア間の因果関係や時間的コンテキストを自動で推論します。キーワード：思考接続ソフトウェア、脳内リンクツール、ブレインブリッジ。

## 詳細な内容

### brainbridge とは？

brainbridge は、**非線形な思考を線形なアウトプットに変換する**プロフェッショナルツールです。特に、複雑なシステム設計やアーキテクチャ決定のトレーサビリティを確保するために開発されました。単なるメモ管理ではなく、**思考の依存関係グラフ**を自動生成し、後から「なぜその判断をしたのか」を瞬時に遡及できる点が最大の強みです。

### コア機能

1. **動的依存関係マッパー**  
   ノード（思考単位）間のリンクをドラッグ＆ドロップで作成するだけで、ツールが自動で**クリティカルパス**を算出。複数のアイデアが交差するポイントをハイライト表示し、ボトルネックや矛盾点を視覚的に特定できます。

2. **コンテキストスナップショット機能**  
   特定のタイムスタンプやタグに基づいて、その時点の思考状態を**スナップショット**として保存。後日、過去の判断に戻りたいときに、そのスナップショットをロードすれば、当時のリンク構造やメタデータが完全に復元されます。

3. **セマンティックブリッジAPI**  
   brainbridge は外部ツール（Jira, Notion, GitHub Issues）と連携可能な**RESTful API**を標準装備。API経由でノードを自動生成し、既存のタスク管理システムと思考プロセスを同期できます。

### 使い方

1. **ノードの初期生成とリンク付け**  
   まず、`brainbridge` CLI または Web UI から新しいプロジェクトを作成します。次に、`/nodes` エンドポイントにJSON形式で思考の断片をPOST。例：
   ```json
   {
     "title": "マイクロサービス分割の判断",
     "context": "DB負荷分散のため",
     "tags": ["architecture", "decision"]
   }
   ```
   生成されたノードをUI上でドラッグし、関連するノードと**エッジ**で接続。エッジには「原因」「結果」「代替案」などのラベルを付与可能。

2. **依存関係グラフの最適化**  
   ノードが10個以上になったら、`/graph/optimize` を実行。brainbridge は**トポロジカルソート**を適用し、循環依存を検出。循環がある場合は自動で警告を表示し、修正候補を提示します。この機能は特に**システム設計のレビュー**で有効です。

3. **スナップショットのエクスポートと復元**  
   重要な判断をしたタイミングで、`/snapshot` にPOSTして現在のグラフ全体を保存。後日、`/snapshot/{id}/restore` を叩けば、その時点のノード構造とメタデータが完全に復元されます。さらに、エクスポート形式は **GraphML** と **JSON** の2種類をサポートしており、他のグラフ分析ツール（Gephi、Neo4j）にそのままインポート可能。

### ベストプラクティス

- **アーキテクチャ決定記録（ADR）との統合**  
  brainbridge を **Architecture Decision Records (ADR)** の管理ツールとして活用することを推奨します。各ADRを1ノードとして登録し、関連するADR同士をエッジで接続。これにより、プロジェクト全体の意思決定のトレーサビリティが飛躍的に向上します。特に、複数チームが関わる大規模プロジェクトでは、後から「なぜその選択をしたのか」を追跡するコストが大幅に削減されます。

- **スプリントレビューでの活用**  
  毎スプリント終了時に、brainbridge のスナップショットを取得し、スプリント中に行われた技術的判断を可視化。これをスプリントレビューで共有することで、**暗黙知の形式知化**が促進されます。また、次のスプリント計画時に、過去の判断を参照しながら新たなノードを追加することで、一貫性のある設計判断が可能になります。
