# brainbridge - BRA_028

## 概要

brainbridgeは、人間の思考プロセスをデータ構造として捕捉し、AIエージェントとの協業を最適化するためのブリッジツールです。直感的な思考マッピングとセマンティックリンクにより、散逸したアイデアを実行可能なタスクチェーンに変換します。

## brainbridgeとは？

brainbridgeは、思考の断片を**意味的結合（Semantic Bridging）** で連結し、AIエージェントが解釈可能なフォーマットへ自動変換するプロフェッショナルツールです。特に、マルチモーダルな思考データ（テキスト、図解、音声メモ）を統合し、**コンテキスト損失ゼロ** で開発フローに組み込むことを可能にします。

## コア機能（ハードコア3選）

### 1. セマンティック・ブリッジ・エンジン
思考の断片間の意味的距離をベクトル計算し、**自動で関連ノードを結合**。例えば、Slackの議論ログとローカルのコードコメントを横断し、設計意図を再構築します。従来のマインドマップでは不可能な、非線形な思考の追跡が可能です。

### 2. コンテキスト・スナップショット・エクスポート
思考プロセス全体を**AIエージェント（GPT-4, Claude, Gemini）向けのプロンプトテンプレート**として出力。単なるメモではなく、システムプロンプトやFew-shot例として直接利用できるJSON/YAML形式をサポート。これにより、思考→実装のラグを排除します。

### 3. リアルタイム・コラボレーティブ・ブリッジング
複数メンバーの思考マップを**リアルタイムでマージ**し、コンフリクトを自動解決。Gitのマージコンフリクトに似たUIで、思考の分岐を可視化しながら統合できます。リモートチームの非同期ブレインストーミングに最適です。

## 使い方（3ステップで即戦力）

### ステップ 1: 思考ノードのインジェスト
任意のテキスト、音声メモ、またはMarkdownファイルをbrainbridgeにドラッグ&ドロップ。自動的に**ノード分割（Chunking）** され、意味的インデックスが付与されます。例えば、会議の議事録を投入すると、各発言が独立したノードとして認識され、話者の意図タグが自動付与されます。

```bash
# CLI経由でノードを追加
brainbridge ingest --source ./meeting_notes.md --tag "design_review"
```

### ステップ 2: ブリッジの構築と最適化
`brainbridge bridge` コマンドを実行すると、全ノード間の**類似度スコア**が計算され、関連性の高いノード同士が自動リンクされます。閾値（`--threshold 0.75`）を調整することで、リンクの密度を制御可能。結果はインタラクティブなDAG（有向非巡回グラフ）としてブラウザで確認できます。

### ステップ 3: プロンプトとしてのエクスポート
`brainbridge export --format openai` で、現在の思考マップ全体を**システムプロンプト＋ユーザープロンプト**として出力。出力されたテキストをそのままChatGPTやClaudeに貼り付けるだけで、あなたの思考プロセスを完全に理解したAIエージェントがタスクを継承します。

```yaml
# エクスポート例（一部）
system_prompt: |
  あなたはbrainbridgeからエクスポートされた思考マップに基づいて行動します。
  以下のノードは優先度順にソートされています。
context_nodes:
  - id: "node_001"
    content: "マイクロサービス分割の方針"
    priority: 0.92
```

## ベストプラクティス（現場の知見）

### 1. 思考の“デブリ”を定期的に掃除せよ
brainbridgeは思考を貯め込むほどノード数が増加します。週次で `brainbridge prune --orphan` を実行し、**孤立ノード**（どのブリッジにも接続されていないノード）を削除することを推奨します。これにより、検索精度とエクスポート品質が劇的に向上します。

### 2. プロジェクト単位でブリッジを分離せよ
単一のbrainbridgeインスタンスで複数プロジェクトを管理すると、思考ノードが混ざり、エクスポート時にノイズが発生します。`brainbridge init --project "project_name"` でプロジェクトを分割し、`.brainbridge/config` でプロジェクトごとのベクトルDBを分離することを強く推奨します。これにより、AIエージェントに渡すコンテキストの精度が保証されます。
