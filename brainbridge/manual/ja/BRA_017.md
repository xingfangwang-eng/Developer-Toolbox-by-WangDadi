# brainbridge - BRA_017

## 概要

brainbridgeは、散在する思考フラグメントを構造化ナレッジグラフに変換し、直感的な「思考接続」体験を提供するプロフェッショナル向けツールです。メモ、アイデア、プロジェクト間の関連性を動的に抽出し、認知負荷を最小化します。キーワード：思考ブリッジツール、思考接続ソフトウェア、脳内思考ツール。

## 詳細な内容

### brainbridgeとは？

brainbridgeは、人間の思考プロセスを模倣したアソシエーションエンジンを搭載したSaaSツールです。従来のマインドマップやノートツールとは異なり、**暗黙的な関係性**を自動検出し、ユーザーが意識せずとも知識の「橋渡し」を実現します。特に、プロジェクト管理、研究、クリエイティブワークにおける情報断片化問題を解決します。

### コア機能（ハードコア）

1. **セマンティックアソシエーションエンジン**  
   入力されたテキスト、コードスニペット、URLからエンティティ（人物、概念、プロジェクト）を抽出し、TF-IDFとWord2Vecを組み合わせたハイブリッドモデルで関連度スコアを算出。関連性の高いノードを自動リンクし、グラフデータベース（Neo4j）上にリアルタイムで可視化します。

2. **コンテキスト依存の動的フィルタリング**  
   ユーザーの現在のワークスペースやタイムラインに基づき、表示する関連情報を重み付け。例えば、`#architecture` タグが付いたドキュメントを開いている場合、自動的にアーキテクチャ関連のノードのみを強調表示します。これにより、ノイズを排除しつつ、必要な接続を瞬時に発見可能。

3. **双方向ブリッジAPI**  
   RESTful APIとWebSocketを提供し、外部ツール（Slack、Notion、Jira）との連携を標準化。ブリッジを介して、別システムのデータをbrainbridgeのグラフにインポート/エクスポート可能。特に、CI/CDパイプラインと統合することで、コード変更がドキュメントグラフに自動反映される。

### 使い方（詳細ステップ）

1. **初期セットアップとデータインポート**  
   - `brainbridge-cli init` をターミナルで実行し、プロジェクトルートに `.brainbridge.yml` を生成。  
   - `brainbridge import --source ./docs --recursive` で既存のMarkdown/Notionエクスポートファイルをバッチインポート。  
   - インポート後、`brainbridge graph --depth 3` で関連性グラフを確認。ノード間の距離が近いほど、関連度が高いことを示す。

2. **リアルタイムな思考接続の作成**  
   - エディタ（VS Code拡張機能）で新しいメモを作成中、`Ctrl+Shift+B` でbrainbridgeのインライン検索を起動。  
   - 入力中のキーワードに対して、既存ノードとの関連度が高いものがドロップダウン表示される。  
   - 関連ノードを選択すると、自動的に双方向リンクが作成され、グラフ上にエッジが追加される。  
   - さらに、`brainbridge suggest --context` で、現在のノードに基づく「次に読むべきリソース」が提案される。

3. **高度なグラフクエリとエクスポート**  
   - グラフビューで `Ctrl+F` を押し、Cypherクエリを直接入力可能。例：`MATCH (n:Concept)-[:RELATES_TO]->(m) WHERE n.name CONTAINS 'SaaS' RETURN n, m`。  
   - クエリ結果をJSONまたはGraphML形式でエクスポート：`brainbridge export --format graphml --output ./graph.graphml`。  
   - エクスポート後、GephiやCytoscapeでさらに詳細なネットワーク分析が可能。

### ベストプラクティス（業界標準）

1. **プロジェクト開始時に「シードノード」を戦略的に設定する**  
   新規プロジェクトでは、最初に5〜10個の「シードノード」（例：主要な概念、チームメンバー、目標）を手動で作成し、brainbridgeに「文脈の土台」を与える。これにより、その後の自動関連付けの精度が劇的に向上する。特に、アジャイル開発のスプリント開始時に推奨。

2. **定期的なグラフメンテナンスとノードのタグ付け**  
   週次で `brainbridge prune --orphan` を実行し、孤立ノード（リンクがないノード）を削除またはマージする。また、ノードには必ず `#project`, `#research`, `#personal` などのタグを付与し、フィルタリング効率を最大化。これにより、1,000ノードを超える大規模グラフでもパフォーマンスが維持される。
