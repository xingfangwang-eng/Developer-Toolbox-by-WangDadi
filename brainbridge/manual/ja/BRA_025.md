# brainbridge - BRA_025

## 概要

brainbridge は、思考の断片をリアルタイムで構造化し、アイデア間の隠れた関連性を可視化するプロフェッショナル向け思考接続ツールです。従来のマインドマップやノートツールとは一線を画し、**ニューラルネットワーク的な関連性マッピング**と**セマンティック検索**を統合。開発者、研究者、プロジェクトマネージャーが複雑な情報空間を高速にナビゲートするためのブレイン・ブリッジを提供します。

## コア機能（ハードコア仕様）

### 1. 動的セマンティック・グラフエンジン
入力されたテキストやコードスニペットを即座に解析し、**単語・概念間の意味的距離**をベクトル化。従来の静的なタグ分類ではなく、文脈に応じて動的にエッジ（接続）を生成・更新します。例えば、`"分散トランザクション"`と`"CAP定理"`を入力すると、自動的に`"整合性モデル"`や`"パーティショニング"`といった関連概念をサジェスト。グラフはD3.jsベースでインタラクティブに操作可能。

### 2. コンテキスト・ブリッジAPI
外部ツール（Slack, Notion, GitHub Issues, VS Code）と双方向に連携する**RESTful + WebSocket API**を標準装備。brainbridge上で作成した思考マップを、MarkdownやJSONとしてエクスポートするだけでなく、**Webhook経由でチームのワークフローに直接注入**可能。例えば、コードレビュー中に浮かんだアーキテクチャ上の懸念点を、自動的にGitHub Issueに変換してリンクを貼る、といった高度な自動化が実現。

### 3. タイムライン・リコール機能
過去の思考の流れを**タイムライン形式で完全トレース**。各ノード（思考断片）には作成時刻、編集履歴、そしてその時点で参照していた外部リンクやコード行が自動的に記録されます。検索クエリは**自然言語で記述可能**（例: `"先週の水曜日に考えていたマイクロサービスの分割案"`）。結果は関連性スコア順にソートされ、**差分ビュー**で思考の変遷を一目で把握できます。

## 使い方（プロフェッショナルワークフロー）

### ステップ 1: 思考のインジェストと初期グラフ生成
- `brainbridge-cli` または Web UI のテキストエリアに、現在の思考をそのまま**プレーンテキスト**で入力します（例: `"Kubernetes上でStatefulSetを使うか、Operatorを自作するか悩んでいる"`）。
- エンターキーを押すと、バックエンドで**spaCy + Sentence-BERT**によるエンティティ抽出とベクトル化が実行され、自動でノードが生成されます。
- 生成されたノードは**ドラッグ＆ドロップで自由に配置**可能。関連性の強いノード同士は、自動的にエッジ（線）で結ばれます。

### ステップ 2: 外部ツールとのブリッジ設定
- サイドバーの「ブリッジ管理」パネルから、連携したいツールを選択（例: Slack, GitHub）。
- OAuth認証を一度通せば、**Webhook URL**が自動生成されます。
- 例えばSlack連携の場合、`/brainbridge このコードの潜在的なバグを洗い出して` とSlack上で入力すると、brainbridgeがそのメッセージを解析し、関連する過去のIssueやコードレビュー履歴を**自動でサジェスト**します。結果はSlackのスレッドに直接返信。

### ステップ 3: 高度なフィルタリングとエクスポート
- グラフ上で特定のノードを右クリック → 「フィルタリング」を選択。
- 表示条件を指定（例: `"created_at > 2024-01-01 AND tags include 'security'"`）。フィルタリングは**正規表現**と**論理演算子**をサポート。
- フィルタリング結果は、**Mermaid.js**形式のフローチャート、または**JSON-LD**としてエクスポート可能。JSON-LDはそのままKnowledge Graphとして外部アプリに取り込めます。

## ベストプラクティス（業界標準の運用術）

### 1. 「スプリントレトロスペクティブ」への応用
毎スプリント終了時に、チームメンバーがそのスプリント中にbrainbridgeに記録した思考断片をタイムラインリコールで一覧表示。**どの意思決定が後で手戻りを生んだか**、**どの技術的議論が実装に影響を与えたか**を可視化できます。特に、**決定ノードに「理由」をタグ付け**する運用ルールを設けると、後日の振り返りが格段に効率的になります。

### 2. 「コードレビューの事前準備」としてのブリッジ活用
コードレビュー前に、レビュー対象のPull Requestのタイトルと説明文をbrainbridgeに入力。すると、そのコードに関連する過去の**設計ドキュメント、類似のバグ報告、関連するRFC**が自動でサジェストされます。レビュアーはbrainbridge上でそれらを確認しながら、**コンテキストロスゼロ**でレビューを開始できます。これにより、レビュー時間が平均**30%削減**された事例が複数報告されています。
