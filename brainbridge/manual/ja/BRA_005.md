# brainbridge - BRA_005

## 概要

brainbridge は、思考とコードの境界を溶解する次世代型コネクションツールです。従来のブレインストーミングやノートアプリでは実現できなかった、**思考の構造化・可視化・実装化** をシームレスに実行します。特に、散らかったアイデアをリアルタイムで実行可能なロジックに変換する点が、他のツールとの決定的な差別化要因です。

## 詳細な内容

### brainbridge とは？

brainbridge は、**人間の直感的な思考パターン**と**機械的な実行ロジック**を橋渡しする SaaS ツールです。単なるマインドマップやタスク管理ツールではなく、思考の流れをそのままコードスニペット、データフロー図、API エンドポイント設計に変換します。特に、分散チームでのアイデア共有や、プロトタイピングフェーズにおける「思考の非同期同期」を劇的に効率化します。

### コア機能

1. **思考からコードへのリアルタイムコンパイル**  
   ブレインストーミング中に描いたフローチャートやメモを、即座に **TypeScript / Python / Go の疑似コード** に変換。さらに、そのコードをワンクリックで実行可能なスニペットとしてエクスポート可能。思考の「あやふやさ」をそのままロジックに落とし込む、まさに「思考のデバッグ」を実現。

2. **コンテキスト依存の思考グラフ（Context-Aware Thought Graph）**  
   ユーザーが入力するキーワードやタスクの依存関係を動的に解析し、**循環依存や未定義の変数**を自動検出。例えば「ユーザー認証」と「セッション管理」の関係性をグラフ上で可視化し、設計の抜け漏れを防止。さらに、過去のプロジェクトデータを基に「この思考パターン、前回のプロジェクトで問題になったよね？」とアラートを出す。

3. **非同期コラボレーションブリッジ**  
   タイムゾーンが異なるチームメンバー同士で、思考の「差分」を自動マージ。例えば、東京のエンジニアが描いたアーキテクチャ図に対して、サンフランシスコのメンバーがコメントを残すと、そのコメントが **Git のプルリクエストのように** マージ可能な形で提案される。これにより、非同期での設計レビューが本格的に運用可能。

### 使い方

1. **思考フローの初期化と構造化**  
   `brainbridge init` コマンドを実行し、プロジェクトのルートディレクトリに `.brainbridge` ディレクトリを作成。次に、`brainbridge thought "ユーザー管理機能の設計"` と入力すると、自動的に **思考テンプレート**（ユースケース図、データフロー、エラーハンドリング）が生成される。この段階で、既存のコードベース（GitHub リポジトリ）とリンクさせると、自動で変数名や関数名を提案してくれる。

2. **思考グラフのエッジ（関係性）を定義**  
   `brainbridge connect "ログイン処理" -> "セッション管理"` のように、ノード間の依存関係をコマンドラインまたは GUI で定義。この時、**循環依存が発生した場合**、brainbridge は自動で警告を表示し、代替の接続パターンを 3 つ提示する。また、`brainbridge validate` でグラフ全体の整合性チェックを実行。未定義のノードや孤立した思考を検出し、修正を促す。

3. **コードへのエクスポートと実行**  
   `brainbridge export --lang python --output ./src/` で、思考グラフを Python の関数定義に変換。この時、各ノードに割り当てられた「思考ラベル」がそのまま関数名やクラス名になる。さらに、`brainbridge run --test` で自動生成されたテストコード（pytest 形式）を実行。思考の段階でテストケースも同時に生成されるため、**思考の完了 = テストの完了** というワークフローを実現。

### ベストプラクティス

1. **思考の粒度を「1 つの関数」に統一する**  
   brainbridge は、1 つの思考ノードに対して 1 つの関数または 1 つの API エンドポイントを割り当てる設計思想です。例えば「ユーザー登録」という思考ノードには、`register_user()` という関数を割り当てる。これにより、思考グラフがそのままコードのモジュール構造にマッピングされ、リファクタリングやユニットテストが極めて容易になります。逆に、1 つのノードに複数の責務を持たせると、グラフが複雑化し、コードの可読性が低下します。

2. **非同期コラボレーションでは「思考のブランチ」を活用する**  
   チームメンバーが同時に同じ思考グラフを編集する場合、**Git のブランチ戦略**を模倣した「思考ブランチ」を作成することを推奨します。例えば、`brainbridge branch feature/oauth` で新しいブランチを作成し、そこで OAuth 認証の思考を深める。その後、`brainbridge merge` でメインブランチに統合する際に、自動でコンフリクト（思考の矛盾）を検出し、解消方法を提案します。これにより、設計段階でのコンフリクトを事前に防ぎ、コードレビュー時の負荷を軽減します。
