# brainbridge - BRA_024

## 概要

brainbridge は、思考の断片を構造化された知識グラフへと変換する、次世代のコグニティブ・ミドルウェアです。従来のマインドマップツールとは異なり、脳の連想記憶モデルを模倣した非線形なコネクション形成を実現。アイデアの「橋渡し」を自動化し、クリエイティブな問題解決を加速します。

**ターゲットキーワード**: 思考接続ツール、ブレイン・ブリッジ、ニューラル・ナレッジグラフ、アイデア自動構造化

## 詳細な内容

### brainbridge とは？

脳科学の知見に基づく**アソシエーション・エンジン**です。ユーザーが入力した断片的なアイデア（ノード）を、意味的類似性と文脈的依存関係に基づいて自動リンク。従来のツリー構造ではなく、**スパース・コネクテッド・ネットワーク**を生成します。これにより、直感と論理のギャップを埋め、ブレインストーミングからプロトタイピングまでの遷移をシームレスにします。

### コア機能

1.  **セマンティック・アンカリング**  
    入力テキストから**潜在的意味索引（LSI）** を抽出し、ノード間の関連度を0.0〜1.0の浮動小数点でマッピング。閾値（デフォルト0.67）を超えたノードは自動的に「ブリッジ」され、グローバルな知識グラフを形成します。これにより、ユーザーが気づいていない隠れた関連性を可視化します。

2.  **コンテキスト・スナップショット・エクスポート**  
    任意のノードを基点に、そのノードから**3ホップ以内**の全関係性をJSON-LD形式でエクスポート可能。エクスポートデータはRDF準拠であり、外部のセマンティックWebツール（Protégé、Neo4jなど）に直接インポートできます。チーム間での知識継承や、大規模ナレッジベースへの統合に最適です。

3.  **アナロジカル・インファレンス**  
    グラフ内の**構造的類似性**を検出し、異なるドメイン間のアナロジーを自動提案。例えば、「顧客獲得フロー」と「生態系の食物連鎖」のグラフパターンが一致した場合、その類似性をハイライト表示します。これにより、メタファー思考を支援し、革新的なソリューションを導き出します。

### 使い方

1.  **ノードのバルクインポートと初期クラスタリング**  
    `brainbridge import` コマンドを使用し、CSVまたはMarkdownリストからノードを一括インポートします。  
    ```bash
    brainbridge import --format csv --file ideas.csv --anchor-threshold 0.72
    ```
    インポート後、`brainbridge cluster` を実行し、**スペクトラルクラスタリング**アルゴリズムでノードを自動分類。出力されたクラスタIDを確認し、必要に応じて手動でマージまたは分割します。

2.  **手動ブリッジと重み調整**  
    グラフビューアで特定のノードペアを選択し、`/bridge` コマンドで直接リンクを追加。  
    ```brainbridge
    /bridge --source "量子コンピューティング" --target "暗号通貨マイニング" --weight 0.85
    ```
    重みは `--weight` パラメータで0.0〜1.0の範囲で指定。これにより、自動リンクでは捉えきれない専門家の直感を反映できます。

3.  **タイムライン・スナップショットの生成**  
    プロジェクトの進行に合わせて、`brainbridge snapshot` で現在のグラフ状態を保存。  
    ```bash
    brainbridge snapshot --tag "v0.2.4-alpha" --comment "顧客インタビュー後の再編成"
    ```
    スナップショットは `.bbx` 形式で保存され、`brainbridge diff --snapshot1 v0.2.3 --snapshot2 v0.2.4` で差分を可視化。これにより、チームの思考の変遷を追跡し、意思決定のトレーサビリティを確保します。

### ベストプラクティス

1.  **ノード粒度の統一**  
    各ノードは**1つの概念**のみを含むようにしてください。例えば「マーケティング戦略と予算配分」は2つのノードに分割します。これにより、グラフの密度が適正化され、アナロジカル・インファレンスの精度が向上します。また、ノード名は名詞句で統一（例：「顧客獲得コストの削減」ではなく「顧客獲得コスト」）。

2.  **定期的なリバランスとトリミング**  
    ノード数が1000を超えた場合、`brainbridge prune --min-degree 2` を実行し、接続数が1以下の孤立ノードを一時的にアーカイブします。その後、`brainbridge rebalance --target-density 0.15` でグラフ全体の密度を調整。これにより、情報過多による認知負荷を軽減し、重要なコネクションにフォーカスできます。
