# brainbridge - BRA_030

## 概要

brainbridge は、エンジニアの頭の中にある断片的な思考やアイデアを、リアルタイムで構造化・接続するための思考コネクションソフトウェアです。従来のマインドマップやノートツールを超え、セマンティックネットワークとAI推論を組み合わせることで、思考の「ブリッジ」を自動生成します。

## 詳細な内容

### brainbridge とは？

brainbridge は、散らばった思考断片（Thought Fragments）を入力するだけで、それらの関連性を自動解析し、グラフベースの知識構造として可視化・保存するプロフェッショナルツールです。特に、複雑なシステム設計やアーキテクチャの検討時に、頭の中の「暗黙知」を形式知に変換するためのブリッジとして機能します。

### コア機能

1. **セマンティックブリッジエンジン**  
   入力されたテキストやコードスニペットから、意味的類似度と因果関係を自動推定。異なるドメインの思考を「ブリッジ」として結びつける。例えば、バックエンドの設計メモとフロントエンドのパフォーマンス要件を、コンテキストを保ったままリンクする。

2. **リアルタイム思考グラフ（RTG）**  
   思考の追加・編集に応じて、動的にグラフ構造が更新される。ノード間の距離や接続強度を視覚的に表示し、思考の「偏り」や「未接続領域」を発見できる。D3.jsベースのカスタムレンダリングにより、1000ノード以上でもスムーズに動作。

3. **ブリッジ推論API**  
   グラフ上の任意の2ノード間の「最短思考パス」を推論し、関連する思考を自動サジェスト。さらに、欠落している中間思考をAIが補完する「ギャップフィリング」機能を備える。RESTful APIとして外部ツールからの呼び出しも可能。

### 使い方

1. **思考断片のインポートとタグ付け**  
   まず、MarkdownやJSON形式で思考断片をbrainbridgeにインポートする。各断片には、ドメイン（例：`domain:backend`）や重要度（`priority:high`）などのメタタグを付与。以下のようにCLIから一括投入可能：
   ```bash
   brainbridge import --format json --input thoughts.json --tag "architecture, design"
   ```
   これにより、断片は自動でベクトル化され、既存グラフにマージされる。

2. **ブリッジの手動接続と自動推論**  
   グラフビュー上で2つのノードをドラッグ＆ドロップして手動でブリッジを生成。または、ノードを選択して「推論ブリッジ」ボタンをクリックすると、AIが関連性の高いノードをリストアップ。選択したノード間のブリッジ強度（0.0〜1.0）をスライダーで調整可能。

3. **グラフのエクスポートと共有**  
   完成した思考グラフは、標準フォーマット（GraphML、Cytoscape JSON）でエクスポート。さらに、brainbridge独自の`.bbg`形式では、ブリッジ推論のメタデータも保持。チームメンバーへの共有は、埋め込み可能なHTMLウィジェットとして生成可能：
   ```bash
   brainbridge export --format html --output bridge.html --interactive true
   ```

### ベストプラクティス

- **思考の粒度を統一する**  
   brainbridgeは、ノード間の意味的距離を計算するため、各思考断片は「1つのアイデア」に絞ることが重要。例えば、「データベースのシャーディング戦略」と「キャッシュ層の設計」は別ノードとして扱い、ブリッジで結ぶ。粒度が異なると、推論の精度が低下する。

- **定期的なグラフメンテナンス**  
   思考グラフは時間とともに肥大化する。週次で「孤立ノード（接続数0）」や「ブリッジ強度0.1以下の弱接続」をフィルタリングし、不要な断片をアーカイブする。brainbridgeには自動メンテナンススクリプトが付属しており、`brainbridge prune --threshold 0.1` で実行可能。
