# brainbridge - BRA_004

## 概要

brainbridge は、散在する思考を構造化データへと変換し、アイデアの断片と実行可能なソリューションをシームレスに接続するプロフェッショナル向け思考連携ツールです。開発者、プロダクトマネージャー、SaaS アーキテクトが直面する「脳内の非線形なアイデアを、チームが共有可能なロジックツリーに落とし込む」課題を解決します。

## 詳細な内容

### brainbridge とは

brainbridge は、単なるマインドマップツールではありません。**セマンティックグラフエンジン**を内蔵し、ユーザーが入力した断片的なキーワードやメモを、文脈と依存関係に基づいて自動でクラスタリング・構造化します。特に、複雑なシステム設計やアーキテクチャ検討のフェーズにおいて、思考の**過渡的状態**を可視化し、意思決定のボトルネックを特定するために設計されました。

### コア機能

1. **コンテキストアウェア・セマンティックマッピング**  
   入力されたテキストやコードスニペットを、事前学習済みのドメイン固有モデルが解析し、意味的に近いノード同士を自動でリンク。手動でのグルーピング作業を排除し、思考の「暗黙の関連性」を可視化します。

2. **リアルタイム・コグニティブフォーク**  
   特定のノードを起点に、思考の分岐（フォーク）を即座に生成。各フォークは独立したワークスペースとして扱われ、後からマージや差分比較が可能。チームメンバーが同時に異なる仮説を検証する際に、コンフリクトを回避しながら並行思考を促進します。

3. **アーキテクチャ・デシジョンレコード（ADR）自動生成**  
   思考プロセスから得られた最終的な決定事項を、構造化された ADR フォーマット（タイトル、コンテキスト、決定、結果）で自動エクスポート。SaaS の設計レビューやコードレビューにそのまま利用可能な形で出力します。

### 使い方

1. **セマンティックグラフの初期生成**  
   `brainbridge init --type architecture --format json` を実行し、プロジェクトルートに `.brainbridge/config.yml` を作成。その後、任意の Markdown ファイルやコードベースをドラッグ＆ドロップで投入。brainbridge は自動で依存関係を解析し、ノード間のエッジを生成します。必要に応じて `--depth 3` で解析深度を指定可能。

2. **フォークによる仮説検証ワークフロー**  
   グラフ上の重要ノードを右クリックし、「フォークを作成」を選択。フォーク名を `hypothesis-cache-layer` のように命名。このフォーク内での編集は元グラフに影響を与えません。検証後、`brainbridge merge --fork hypothesis-cache-layer --strategy three-way` で統合。コンフリクトが発生した場合は、GUI 上でサイドバイサイドで比較・解決できます。

3. **ADR のエクスポートと CI/CD 統合**  
   最終的な決定ノードを選択し、`brainbridge export --format adr --output ./docs/adr/` を実行。出力された ADR は `adr-001-xxx.md` として保存され、テンプレートエンジンによって自動的にヘッダーとフッターが付与されます。`--ci-mode` フラグを付けると、GitHub Actions や GitLab CI で自動的に PR コメントに ADR を添付するワークフローを生成します。

### ベストプラクティス

1. **設計初期段階での「思考のゴミ箱」を活用せよ**  
   脳内に浮かんだアイデアを、とりあえず brainbridge の「スクラッチノード」に放り込む習慣を持ちましょう。後からセマンティックマッピング機能で自動整理されるため、完璧な構造を最初から求める必要はありません。このプロセスにより、**思考のクリティカルパス**が可視化され、設計判断の遅延を防げます。

2. **フォークの命名規則をチームで統一せよ**  
   フォーク名は `{イニシアチブ}-{仮説番号}` の形式（例：`cache-strategy-hypo-03`）に統一することを推奨します。これにより、後日のマージ時にどのフォークがどの意思決定に対応するかが一目で把握でき、ADR 自動生成時のトレーサビリティが飛躍的に向上します。
