## Overview


## What is brainbridge?

brainbridge is a **neural-link abstraction layer** for developers who need to map, sync, and query distributed thought structures across heterogeneous systems. Think of it as a semantic middleware that bridges human cognition fragments with machine-readable graph schemas — without the overhead of traditional ontology engines.

## Core Features

### 1. Dynamic Semantic Graph Fusion
brainbridge auto-merges fragmented thought data (notes, code comments, API logs, voice snippets) into a **live, queryable semantic graph**. It uses a custom lightweight vector index that updates in real-time, eliminating the need for manual tagging or schema design.

### 2. Context-Aware Thought Routing Engine
Instead of flat search, brainbridge implements a **multi-hop context router** — it traces associative paths across your knowledge base, delivering results that mimic human lateral thinking. This is critical for debugging complex systems or reconstructing design rationale from scattered sources.

### 3. Zero-Latency Bi-Directional Sync Protocol
Built on a **WebRTC-based mesh protocol**, brainbridge synchronizes thought graphs across devices and team members with sub-50ms latency. No central server required. Your local-first architecture stays intact, while collaboration becomes seamless.

## How to Use

### Step 1: Initialize a Thought Node Cluster
```bash
brainbridge init --cluster my-project --mode local-first
```
This creates a `.brainbridge` directory with a distributed graph schema. Run `brainbridge status` to verify peer discovery and node health. The engine auto-detects existing `.md`, `.json`, and `.log` files in the working directory.

### Step 2: Inject Context via CLI or API
```bash
brainbridge inject --source ./notes --tag "architecture:v2" --priority high
```
This parses each file, extracts entities and relationships, and fuses them into the graph. Use `--dry-run` first to preview the semantic nodes. For programmatic access, call the REST endpoint `/v1/graph/inject` with JSON payloads — the engine returns a `fusion_hash` for traceability.

### Step 3: Query with Associative Traversal
```bash
brainbridge query "Why was the auth module refactored?" --depth 3 --format json
```
The router returns a ranked list of thought paths, each with confidence scores and timestamped edges. Use `--visualize` to generate a live D3.js graph in your browser. For CI/CD pipelines, pipe the output to `jq` for automated decision logging.

## Best Practices

### Practice 1: Use Tag Prefixes for Namespace Isolation
When working on multi-repo projects, prefix your tags with the module name (e.g., `auth:decision`, `payment:bugfix`). This prevents cross-contamination in the semantic graph and makes queries more precise. brainbridge respects dot-notation in tag names, so `auth.v2.flow` is valid and searchable.

### Practice 2: Schedule Periodic Graph Compaction
Run `brainbridge compact --threshold 30d` weekly. This merges low-weight nodes (e.g., stale logs, duplicate thoughts) into summary nodes, reducing graph size by up to 40% without losing query accuracy. Automate it via cron or GitHub Actions for zero-maintenance hygiene.
