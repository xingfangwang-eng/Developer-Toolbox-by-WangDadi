# brainbridge - BRA_003

## Overview

brainbridge is not another note-taking app or mind map tool. It’s a **neural-syntax bridge** that translates raw, unstructured thought patterns into structured, queryable data streams. Designed for developers, knowledge architects, and AI pipeline engineers, brainbridge enables real-time semantic linking between fragmented ideas—without forcing you into a rigid hierarchy. Think of it as a **live, bidirectional API for your brain**.

## Core Features

### 1. Semantic Drift Detection & Auto-Link
brainbridge continuously monitors your input for **conceptual drift**—when a thought subtly shifts from one domain to another. It automatically generates weighted, bidirectional links between nodes based on cosine similarity and temporal proximity, creating a **dynamic knowledge graph** that evolves with your thinking. No manual tagging. No hierarchy.

### 2. Contextual Memory Injection
Forget "search." brainbridge uses a **multi-vector retrieval model** to inject relevant past thoughts into your current workspace. When you type a new idea, the engine queries your entire graph—not just keywords, but latent semantic vectors—and surfaces context windows from weeks or months ago. This turns your brainbridge into a **second brain with temporal recall**.

### 3. Exportable GraphQL Schema
Every thought cluster in brainbridge is automatically serialized into a **GraphQL schema**. You can query your own mind like a database: `{ thoughts(where: {emotion: "curiosity", timestamp_gt: "2025-01-01"}) { content, linkedTo { title } } }`. This isn't a gimmick—it’s designed for developers who want to hook brainbridge into CI/CD pipelines, personal dashboards, or AI training loops.

## How to Use

### Step 1: Initialize a Thought Stream
Clone the repo and run `brainbridge init --mode stream`. This spins up a local WebSocket server on `localhost:8443`. Open the CLI or connect via any WebSocket client (e.g., `wscat`). Start typing or pasting raw thoughts—no formatting required. Each line or chunk is hashed, embedded, and stored as a node in a **local vector database** (powered by LanceDB under the hood).

### Step 2: Query with Natural Language + Filters
Use the built-in query DSL: `brainbridge query "ideas about scaling microservices" --limit 5 --since "2025-03-01"`. This fires a **hybrid search**—combining BM25 keyword matching with dense vector retrieval—and returns the top-ranked nodes with their link weights. You can also pipe the output into `jq` for further processing: `brainbridge query "debugging patterns" --format json | jq '.nodes[].content'`.

### Step 3: Export a Subgraph as a GraphQL Endpoint
Run `brainbridge export --subgraph "neural-networks" --output schema.graphql`. This extracts all nodes and edges related to the topic "neural-networks" (based on semantic clustering, not tags) and generates a **standalone GraphQL schema**. Deploy it with any GraphQL server (e.g., Apollo, Yoga) and you now have a live API of your thought process. Ideal for building personal knowledge bots or feeding into RAG systems.

## Best Practices

### Practice 1: Use Temporal Weighting for Decision Logs
When logging architectural decisions, append a timestamp and a confidence score (e.g., `[2025-04-01] | Confidence: 0.85 | Adopt event sourcing for order service`). brainbridge’s drift detection will automatically surface **decision reversals**—if your confidence drops later, the graph will highlight the temporal conflict. This is invaluable for **post-mortems and RFC audits**.

### Practice 2: Hook brainbridge into Your Editor via LSP
Install the `brainbridge-lsp` plugin (available for VS Code and Neovim). As you write code comments or markdown docs, the LSP sends **incremental embeddings** to your local brainbridge server. This means your IDE becomes a live thought collector—no extra context-switching. Later, when you’re debugging, you can query brainbridge directly from the editor: `:BrainbridgeQuery "why did we choose this pattern?"`. It’s like having a **personal Stack Overflow for your own decisions**.
